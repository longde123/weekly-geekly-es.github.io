3CX CFD中的电话调查和CRM搜索，新的WP-Live Chat支持插件，Android应用程序更新

在过去的几周中，我们提供了一些有趣的更新和一项新产品。 所有这些创新和改进均符合3CX政策，即基于UC PBX创建价格合理的多通道呼叫中心。

3CX CFD更新-轮询和搜索CRM组件


新发行的3CX呼叫流程设计器（CFD）Update 3收到了新的组件调查（调查），使没有编程技能的用户可以创建自动电话调查。 要创建调查，请使用可视组件配置向导。



到目前为止，在3CX中创建电话调查表需要程序员的参与，该程序员设置了几个不同的CFD组件并将它们与C＃代码链接在一起。 根据用户的要求，我们创建了现成的组件Survey，它具有以下功能：

  • 讲一般信息，例如在调查开始前打招呼和通知调查已完成。
  • 询问不同类型的问题：“是/否”，“从/到评分”，并且可以简单地记录语音响应。
  • 在CSV文件中收集订户答案，并在必要时添加其他信息。

同样在新的CFD版本中，添加了CRM查找组件。 它允许您从连接到3CX CRM系统的数据中提取数据。 CRM本身像往常一样在3CX管理界面中连接到3CX。 作为请求的结果而获得的数据被发送以用于语音应用CFD的进一步处理。



使用组件的典型示例：

  1. 传送到CRM的来电时，用户的呼叫者ID。
  2. 如果找到具有这样的呼叫者ID的客户端，则请求将从CRM中检索分配给该客户端的经理分机号。
  3. CFD应用程序接收该分机号码，并将呼叫（使用转接组件）转移到经理的分机号码。

因此，客户总是可以找他的服务经理。 以前，CFD没有如此便捷的工具，需要由合格的开发人员再次对接的多个组件进行复杂的交互。

我们重复一遍-要使用CRM查找，您首先需要将其中一个CRM系统连接到3CX，如果您的CRM不在列表中，请使用3CX REST API 。
要使用3CX CFD v16 Update 3，您需要3CX V16 Update 3 。

3CX收购了用于多渠道联络中心的WP-Live Chat插件


我们最近获得了WP-Live聊天支持 ，这是一个受欢迎的聊天插件，具有网站访问者的实时分析功能。 这是WordPress最受欢迎的聊天方式-它拥有超过一百万的下载量和每日一千的下载量。 对WP-Live Chat技术的收购是在3CX v16引入自己的3CX Live Chat插件之后发布的。 所有这些步骤旨在以最实惠的价格实现便捷，标准化的多渠道联络中心。

供参考：WP-Live Chat由南非公司Code Cabin于2014年发布，该公司是电子商务解决方案的开发商。 3CX将积极开发WP-Live聊天支持，同时将其作为独立产品免费提供。 与专有的3CX Live Chat＆Talk不同 ，WP-Live Chat不包括与站点访问者的音频/视频通信，但是具有对在线用户操作的广泛分析。

适用于Android Beta的3CX更新


根据您的反馈，下一个适用于Android的beta 3CX应用程序已获得了多项重要改进。

如果电话移到本地网络之外（并且IP地址的连接已更改为FQDN连接），则在尝试拨打电话时有时会出现“请求失败错误”错误。 现在问题已解决。

现在，与订户名称（在状态和聊天应用程序的界面中）一起，显示远程（通过办公室间中继线连接）PBX 3CX的名称。 如果组织有两名同名员工，但在不同办公室工作（连接到不同的3CX交换机），这将非常方便。 此外，集团电话的名称现在显示在员工的呼叫者ID旁边。 这使您可以快速了解从哪个办公室/哪个PBX呼叫您。

还更新了收听语音邮件的界面。 现在，您将看到带有可用选项的消息的完整列表。 选择适当的选项，该消息将在Google Play音乐的内置播放器中播放。


Android Beta的其他3CX改进：

  • 在提供对手机通讯录的访问权限时，添加了“不再询问”选项。
  • 传输的文件会根据Android 10的开发建议下载到专用文件夹中。
  • 新的下拉式联系人过滤器可让您显示所有联系人，仅3CX联系人，仅显示Android设备上的联系人。
  • 按需最多可容纳3个会议参加者。对于有大量参加者的会议，请使用会议安排程序。

您可以通过连接到Android的3CX beta测试程序来安装该应用程序 。 如果您对应用程序有疑问或有任何建议，请在特殊论坛中进行评论。
