[Peter] Kontainer dan distribusi - JUG.ru bertemu dengan Dmitry Chuiko dan Alexander Belokrylov

Pada hari Kamis, 21 Maret, di kantor Oracle di St. Petersburg, JUG.ru akan bertemu dengan Dmitry Chuiko dan Alexander Belokrylov ( alexbel ) dari BellSoft. Dmitry akan berbicara tentang Jawa dalam lingkungan wadah, dan Alexander akan berbicara tentang membuat dan memelihara distribusi Java-nya.


Partisipasi gratis, diperlukan pendaftaran. Anda dapat mendaftar di tautan ini .


Sedikit tentang BellSoft
BellSoft didirikan pada 2017 oleh orang-orang dari Oracle Development Center di St. Petersburg. BellSoft secara aktif terlibat dalam pengembangan platform Java dan merupakan salah satu dari 5 kontributor OpenJDK bersama dengan Oracle, RedHat, SAP dan Google. Perusahaan ini merilis dan mengelola Liberica JDK, sebuah distribusi Java berbasis OpenJDK untuk serangkaian platform yang lebih luas, termasuk pembangunan kemas dengan Alpine Linux. BellSoft dilisensikan oleh TCK, sehingga semua Liberica JDK dijamin untuk memenuhi standar Java SE.
Aktivitas perusahaan juga terkait dengan pengembangan dan dukungan produk-produk open source kompleks lainnya, seperti OpenJDK, GCC dan LLVM compiler dan platform data besar Hadoop.

1. Aplikasi praktis dari fungsionalitas Java 11 yang baru di lingkungan wadah.


Dmitry Chuyko


Betapa indahnya di Java 11 Anda dapat membuat dan menggunakan gambar runtime Anda - gambar runtime khusus. Tetapi penggunaannya menciptakan persyaratan tambahan untuk perpustakaan dan proses pengembangan. Apakah selalu mungkin dan apakah semua orang membutuhkannya?
Seringkali kami hanya ingin menghemat uang, dan Anda dapat menggunakan gambar universal. Pertanyaannya adalah sumber daya apa yang kita kelola, sejauh mana dan untuk apa. Seringkali ini adalah sumber daya fisik dari satu mesin dan waktu.


Dimulai dengan 10 Jawa, wadah berperilaku tepat, mengerjakannya terus di Jawa 11. Kita akan melihat keefektifan fungsi Java 11 baru untuk lingkungan wadah menggunakan contoh spesifik gambar Docker dari Liberica JDK 11. Tapi ini baru permulaan cerita, jadi seperti dalam wadah dengan Java seluruh sistem operasi dijejalkan.

Tentang pembicara


Dmitry Chuyko adalah seorang insinyur kinerja di BellSoft, salah satu dari 5 kontributor OpenJDK. Sebelum bergabung dengan BellSoft, Dmitry diprogram di Jawa, dan kemudian bekerja pada Hotspot JVM di Oracle. Semua pengalaman sebelumnya dengan Java telah menunjukkan bahwa masalah paling menarik dalam aplikasi mendapatkan solusi mereka di platform yang mendasarinya. Saat ini, Dmitry terutama mengoptimalkan OpenJDK untuk ARM64, perusahaan bahkan telah mengimplementasikan JEP 315 di Java 11 pada topik ini.


2. Membuat distribusi Java Anda sendiri.


Alexander Belokrylov


Beberapa orang berpikir bahwa OpenJDK adalah distribusi Java yang cocok untuk digunakan dalam produksi. Faktanya, OpenJDK adalah basis kode yang perlu dirakit dan diuji untuk bekerja di lingkungan perusahaan.
Dalam laporan ini, Alexander akan membagikan pengetahuannya tentang apa yang diperlukan untuk membuat biner OpenJDK yang sudah jadi, jenis pekerjaan apa di belakangnya, dan juga akan berbicara tentang siapa lagi (dan mengapa) membuat kumpulan biner OpenJDK mereka sendiri.

Tentang pembicara


Alexander Belokrylov telah bekerja di industri TI selama lebih dari 15 tahun, ia telah beralih dari insinyur menjadi manajer. Di Oracle, ia memimpin pengembangan Java ME Embedded, dan kemudian salah satu komponen Oracle IoT Cloud Service. Pada 2017, ia mendirikan BellSoft dengan sekelompok orang yang berpikiran sama, yang memproduksi dan memelihara distribusinya sendiri JDK Java untuk Linux x86, Linux ARM, Windows, Mac dan lainnya, dan juga mengembangkan solusi untuk menganalisis big data dan IoT. Peserta berulang di konferensi JPoint, Joker, JavaOne, JEEConf dan banyak lainnya.


Wawancara


Baru-baru ini, kami melakukan wawancara dengan Alexander Belokrylov dan Alexey Voytylov, yang sangat cocok untuk memahami isu-isu yang bekerja dengan BellSoft dan sudut pandang berdasarkan pada mana logika laporan mereka dibuat. Anda dapat membaca wawancara di sini .
