Instal Veeam Backup & Replication menggunakan skrip PowerShell

Baru-baru ini, kolega saya Dave Kaula memposting skrip publik untuk menginstal Veeam Backup & Replication, yang ia gunakan untuk memperagakan produk di laboratoriumnya. Skrip mungkin bermanfaat bagi mereka yang berencana untuk menyebarkan solusi, misalnya, dalam lingkungan pengujian atau dalam skenario "semua pada satu mesin".

Dave menulis artikel sampul kecil untuk skrip, yang saya bawa perhatian Anda.

Jadi selamat datang di kucing.


Pada artikel ini, saya akan menjelaskan cara menginstal Veeam Backup & Replication 9.5 Update 3 pada mesin virtual berdasarkan platform Hyper-V.

Penting! Kami akan melakukan instalasi sesuai dengan skenario "semua pada satu mesin" , yaitu: server cadangan Veeam (dengan SQL Server 2012 Express Edition termasuk dalam paket instalasi), serta server Veeam Backup Enterprise Manager. Sebelum memulai instalasi, periksa apakah semua persyaratan sistem yang relevan yang dijelaskan dalam dokumentasi dipenuhi. Untuk akun tempat layanan Cadangan Veeam akan berfungsi, hak administrator domain diasumsikan.

Untuk menginstal solusi, Anda harus mengikuti langkah-langkah sederhana ini:

  1. Kami membuat mesin virtual yang menjalankan Windows Server 2016. Misalnya, saya akan membuat VM di Hyper-V Manager dan beri nama Management01 .


  2. Unduh Veeam Backup & Replication dari sini , lalu siapkan file yang ada dengan lisensi komersial, atau minta lisensi untuk versi uji coba atau NFR . Dalam kasus kami, kedua file sudah disimpan di folder F: \ Temp1 .


  3. Di folder ini kami menyimpan skrip ini .
  4. Buka prompt perintah PowerShell (Administratif PowerShell Prompt) dan tentukan path F: \ Temp1 , lalu masukkan perintah: 

    Import-Module .\Install-Veeam-HyperV.PS1 -Force -Verbose

    Kemudian modul Install-Veeam akan dimuat, yang akan kita gunakan untuk menginstal Veeam Backup & Replication.


  5. Kami menjalankan perintah: 

     Install-Veeam -VMName Management01 -GuestOSName -Management01 -VMPath F:\dcbuild_test
  6. Di jendela yang terbuka, pilih file VeeamBackup & Replication_9.5.0.1536.Update3.iso di folder F: \ temp1 .
  7. Kemudian, di jendela pemilihan file lisensi, buka file lisensi dari langkah 2. Saya akan menggunakan lisensi Veeam NFR saya.
  8. Jendela entri akun akan muncul. Seperti yang telah disebutkan, dalam lingkungan pengujian, Veeam Backup sering diberikan akun dengan hak administrator domain, dan contoh ini tidak terkecuali. Kami memasukkan nama pengguna dan kata sandi, dalam contoh ini saya menggunakan akun MVPDays \ Administrator .


  9. Setelah itu, urutan tindakan yang sepenuhnya otomatis dimulai, jadi Anda hanya perlu menunggu hasilnya.



Inilah yang dilakukan skrip:

1) Menambahkan file VHDx baru dari host
2) Melakukan penamaannya (menamai volume Veeam baru)
3) Pasang VHDx baru pada host
4) Salin gambar Veeam ISO
5) Salin file lisensi Veeam
6) Menginstal tipe CLR Sistem 2012
7) Menginstal SQL Server 2012 Express Edition
8) Menginstal Server Veeam Backup & Replikasi 9.5
9) Menginstal Veeam Backup Console
10) Menginstal Veeam Explorer untuk Direktori Aktif
11) Menginstal Veeam Explorer for Exchange
12) Menginstal Veeam Explorer untuk SQL
13) Menginstal Veeam Explorer untuk Oracle
14) Menginstal Veeam Explorer untuk SharePoint
15) Menginstal Veeam Enterprise Manager (web dan Cloud Portal)
16) Menginstal pembaruan 3

Setelah skrip selesai, dialog peluncuran Veeam Backup & Replication yang sudah dikenal akan muncul:



Saya berharap pekerjaan saya yang sederhana akan bermanfaat bagi Anda. Kerja sukses!
