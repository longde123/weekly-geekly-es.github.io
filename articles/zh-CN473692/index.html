中国公司Phicomm在Wi-Fi路由器阵容中拥有一种称为K3C AC1900智能WLAN路由器的设备。

该设备使用了一堆Intel AnyWAN SoC GRX350和Intel Home Wi-Fi芯片组WAV500（ 顺便说一下，华硕Blue Cave使用了相同的硬件：相同的Intel PXB4583EL处理器和Wi-Fi Intel芯片PSB83514M / PSB83524M而不是PSB83513M / PSB83523M ）。

此路由器有多个版本：

  • B1 ， B1G ， B2-中国；
  • A1 ， C1 ， S1 （ VIE1 ）-对于其他国家（ 我得到-C1，固件版本为34.1.7.30 ）。

我对这个IEEE 802.11ac标准路由器有什么兴趣？

因此，有：4个千兆端口（ 1个WAN和3个LAN ），5 GHz频带，对MU-MIMO 3x3：3和USB 3.0的支持。 好，不仅如此。

1.一点背景


可选部分
我以前的路由器是TP-Link TL-WR941ND，硬件版本为3.6（ 4MB闪存和32MB RAM ）。 无论版本如何，标准固件都会无故定期挂起（ 已更新两次，我的硬件的最新更新已于2012年底发布 ）。

我对本机固件感到失望，我请求了石像鬼 （ Emnip，版本1.8；固件是基于OpenWRT的，如果没有人知道的话 ），最后路由器开始正常工作了。

购买WR941时， 它满足我的需求（ 大约10年前 ）具有很好的硬件，但是现在我已经缺乏它的性能了。 端口均为100 Mbit / s，最大Wi-Fi速度为300 Mbit / s。 也许对于Internet来说，这仍然很正常，但是在设备之间在本地网络上传输文件有些慢。 另外，内置的Flash内存不足以实现固件的鲁版化（ 即使通过WinSCP更改文件，我也尝试过 ），也无法安装更大的插件（ 当然，您可以扩展内存，安装固件以增加内存大小，但我的指导不够充分）焊接内存芯片的手 ）。

但是，即使是以上所有情况，也可能不会很快迫使我更换路由器。 我刚刚在今年9月初购买了小米Redmi Note 5来替换过早丢失的Redmi Note 4（ 经过2年的模范服务 ），事实证明RN5和WR941相互不兼容-RN5不想在断开所创建的无线网络连接后重新连接使用WR941（这不是一个孤立的问题，正如我稍后在w3bsit3-dns.com上阅读该主题时发现的那样 ）。


通常，需要更换路由器。 为什么是主题？ 我对它的填充（ 大约一年前在SmallNetBuilder上阅读过 ）及其功能（ 尽管不太可能在不久的将来使用其中的一半 ）感兴趣。 但是即使这样也没有决定性地选择确切的Phicomm K3C（ 我也仔细看了小米Mi WiFi Router 3G ），以及价格合理的价格（ 以32美元的价格购买 ），具有良好的填充和将库存固件更改为完全OpenWRT的能力。 路由器带有制造商提供的OpenWRT版本（ 我读过某处添加了间谍的消息，但未找到任何详细信息 ）。

OpenWRT的修改是在Phicomm K3C上工作的（ 官方OpenWRT不支持Intel WAV500芯片组 ）是由绰号为Paldier的中国人 （ 他的GitHub和此路由器的固件文件页面，这是OpenWRT论坛上路由器的主题 ）进行的。 他还为K3C制作了华硕Merlin固件端口（ 由于安装它需要将RAM从256MB替换为512MB，因此我们不予考虑 ）。

3. OpenWRT固件


  1. 我们通过LAN端口（ 这三个端口中的任何一个 ）将路由器连接到计算机/笔记本电脑，并通过WAN将Internet连接到互联网（ 因为您将需要下载固件（略多于30MB ））。
  2. 我们了解本地网络上路由器的地址（ 稍后我们将需要它，通常是192.168.2.1 ）。
  3. 我们启动了先前下载的RoutAckPro实用程序（其中600kB的重量和一堆中文文本；我不知道将其上传到哪里，但可以在注册后从w3bsit3-dns.com论坛下载 ）。 如果地址与上述地址不同，请以IP格式输入。 单击窗口中的Telnet按钮。 如果一切操作正确，窗口中将显示文本“ 打开Telnet完成 ”。 现在可以关闭该实用程序，即 我们准备好路由器以通过Telnet更改固件。


    RoutAckPro窗口
  4. 使用 PuTTY程序（ Smartty或其他类似程序），我们通过Telnet连接到路由器（ IP与RoutAckPro相同，端口为23 ）。


    具有连接设置的PuTTY窗口。
  5. 在PuTTY控制台中，输入以转到tmp目录： 

    cd /tmp

  6. 我们确定我们需要下载哪种固件（ 铁的版本印在路由器底部的不干胶标签上，在我的情况下是“ H / W C1 ”，即我需要C1固件 ）。
  7. 我们在Paldier网站上选择所需的fullimage.img文件的版本。 对我来说 
      http://k3c.paldier.com/openwrt/C1/fullimage.img

    因此，我们在PuTTY控制台中编写以下内容： 

     wget http://k3c.paldier.com/openwrt/C1/fullimage.img

  8. 然后输入命令 

     /usr/sbin/upgrade /tmp/fullimage.img fullimage 0 1

    并等待有关成功固件的消息。
  9. 之后我们介绍 

     rm -rf /overlay/* sync && sleep 10 && reboot

    并等待路由器重启（ 几分钟 ）。 之后，您可以连接到其Web界面（ 地址192.168.2.1 ，密码admin ）。
  10. 首次启动后，建议进行一次重置（使用路由器上的隐藏按钮，在电源插座右侧，或通过Web界面 ）。


    现在路由器将具有此接口

闪烁的说明是由w3bsit3-dns.com论坛WayOutt的用户发布的， 对此深表感谢 。

如果您不想立即将K3C连接到Internet，并且拥有USB闪存驱动器或带有闪存卡的USB读卡器。
我们跳过了第5步，在第7步中，无需使用wget命令将固件文件下载到路由器，而是将其下载到PC（ 如果以后仍需要 ），然后将文件复制到USB闪存驱动器并将其连接到路由器的USB端口。
在第8段中，我们输入以下命令： 
 /usr/sbin/upgrade /mnt/usb/.run/mountd/sda1/fullimage.img fullimage 0 1

其余项目保持不变。

* 使用OpenWRT固件后，安装闪存驱动器的路径将更改。 对我来说已经变成了这样： 
 /usr/sbin/upgrade /tmp/usb/.run/mountd/sda1/fullimage.img fullimage 0 1
