Zextras储能和分层系统

Zextras Powerstore是Zextras Suite中功能最强大和最先进的Zletras之一。 实际上，它不仅允许您压缩数据并使用重复数据删除算法节省硬盘驱动器上的空间，而且还增加了在Zimbra中分层管理卷的功能。 轻松使用HSM的能力通常成为企业购买和实施Zextras PowerStore的杀手级功能。 这就是为什么在IT界存在一个误解，即PowerStore仅在具有分层存储组织的系统中有效，而在单级系统中将是无用的。 好吧，那根本不是真的。



事实是，尽管PowerStore是在意大利开发的，但它却是一种“瑞士刀”，肯定会在使用Zimbra的任何企业中找到应用。 例如，即使使用单级存储，相同的重复数据删除也可以节省空间。 唯一的区别是，在分层系统中，重复的附件和文本块存储在主存储中，而在辅助存储中，仅存储到它们的透明链接，而在单级存储的情况下，原始对象及其透明链接都被存储将存储在一个地方。

是的，它不会像使用分层存储时那样可靠，但是像以前一样，它将大大节省硬盘驱动器上的空间，并且在使用压缩时，您最多可以节省50％的磁盘空间。 因此，通过使用Zextras PowerStore，企业IT经理可以延迟向多层存储的过渡，并在一个安静的环境中在云存储和本地存储之间做出选择，并招标选择最佳的铁或服务提供商。 因此，您可以节省很多。

另一个优点是搜索速度加快。 由于Zextras PowerStore索引了附件和字母以进行重复数据删除，因此所得到的数据可用于快速搜索文本文档和其他附件。 如今，支持MS Office文档，Libre Office，各种未加密的档案，PDF文件等等。 尽管加快搜索速度并不是一项可以直接降低使用Zimbra拥有服务器的成本的功能，但是它可以显着提高经常使用电子邮件搜索功能的员工的效率。 这些包括，例如，会计师和销售人员。

但是，Zextras PowerStore对于具有分层存储的企业的主要价值当然是易于过渡到分层存储。 方便的图形化外壳和可靠的算法将使向新架构的过渡尽可能简单，轻松。 因此，您可以直接在Zimbra管理控制台中添加辅助存储。 您可以在此处选择新卷是本地卷还是云卷。 支持并从命令行工作。 您可以通过输入zxsuite powerstore doCreateVolume命令并指定必要的参数和属性来创建辅助卷，包括：

  • volume_type-存储类型。 可以是主要，次要或索引
  • volume_name-存储名称
  • volume_path-存储路径
  • volume_compressed-使用压缩
  • compression_threshold_bytes-要压缩的最小文件大小（以字节为单位）



例如，通过输入命令zxsuite powerstore docreatevolume secondary“ volume 2”“ / opt / zimbra / volume_2 /” volume_compressed true compression_threshold_bytes 4096，我们获得了一个新的名为“ volume 2”的辅助卷，它位于/ opt / zimbra / store_2 /中，在其中对其进行了压缩所有文件大小在4 KB以上的文件。

如果需要对现有辅助卷进行更改，则可以使用zxsuite powerstore doUpdateVolume命令和上述属性。 例如，使用控制台命令zxsuite powerstore doupdatevolume 2 secondary“ volume 3”“ / opt / zimbra / volume_3 /” volume_compressed false，我们将卷号2重命名为“ volume 3”，现在将其存储为/ opt / zimbra / store_3 /，并且同时关闭其压缩功能。

您可以使用zxsuite powerstore doDeleteVolume命令删除辅助卷。 例如，zxsuite powerstore doDeleteVolume volume 3命令将删除我们创建的辅助卷。

因此，Zextras PowerStore的冬天不仅对已经使用分层存储的企业有用，对于那些计划迁移到更具成本效益的多层存储的企业也很有用。
