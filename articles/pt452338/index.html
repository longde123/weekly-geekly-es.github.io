Apresentando o Terminal do Windows

O Windows Terminal é um aplicativo de terminal novo, moderno, rápido, eficiente, poderoso e poderoso para usuários de ferramentas e shells de linha de comando, como prompt de comando, PowerShell e WSL.



O Windows Terminal será entregue através da Microsoft Store para o Windows 10 e será atualizado regularmente, garantindo que você esteja sempre atualizado e possa usar os recursos mais recentes e as melhorias mais recentes com o mínimo esforço.



Principais recursos do terminal do Windows


Múltiplas guias


Você perguntou, e nós ouvimos! A função mais frequentemente solicitada para o terminal é o suporte para várias guias, e estamos muito felizes por finalmente termos conseguido fornecer essa função. Agora você pode abrir qualquer número de guias, cada uma conectada a um shell de linha de comando ou aplicativo de sua escolha, por exemplo, Prompt de Comando, PowerShell, Ubuntu no WSL, Raspberry Pi via SSH, etc.



Belo texto


O Windows Terminal usa a aceleração DirectWrite / DirectX GPU com base no mecanismo de renderização de texto. Este novo mecanismo de renderização de texto exibirá caracteres de texto, glifos e caracteres presentes nas fontes no seu PC, incluindo ideogramas CJK, emojis, símbolos de linha de força, ícones, ligaduras de programação etc. Esse mecanismo até renderiza texto muito mais rápido que o mecanismo anterior Console GDI!



Você também terá a oportunidade de usar nossa nova fonte! Queríamos criar uma fonte divertida, nova e com espaço simples, para melhorar a aparência moderna do terminal. Essa fonte não inclui apenas ligaduras de programação, mas também possui seu próprio repositório de código-fonte aberto. Fique ligado para obter mais informações sobre o novo projeto de fonte!



Definições e configurabilidade


Entramos em contato com muitos usuários de linha de comando que gostam de personalizar seus terminais e aplicativos de linha de comando. O Terminal do Windows fornece muitas definições e parâmetros de configuração que oferecem grande controle sobre a aparência do terminal e de cada um dos shells / perfis que podem ser abertos como novas guias. As configurações são armazenadas em um arquivo de texto estruturado, que simplifica a configuração para usuários e / ou ferramentas.

Usando o mecanismo de configuração do terminal, você pode criar vários "perfis" para cada shell / aplicativo / ferramenta que deseja usar, seja PowerShell, Prompt de Comando, Ubuntu ou até conexões SSH com dispositivos Azure ou IoT. Esses perfis podem ter sua própria combinação de estilos e tamanhos de fonte, temas de cores, níveis de desfoque / transparência de fundo, etc. Agora você pode criar seu próprio terminal em seu próprio estilo, personalizado de acordo com o seu gosto!

Mais!


Após o lançamento do Windows Terminal 1.0, planejamos começar a trabalhar com muitos recursos que já estão em nossa lista de pendências, além de muitos recursos que você, como comunidade, provavelmente adicionará!

Quando posso obtê-lo?


Hoje, o Terminal do Windows e o Console do Windows estão disponíveis em código aberto, para que você já possa clonar, compilar, executar e testar o código do repositório no GitHub:

github.com/Microsoft/Terminal

Além disso, neste verão, na Microsoft Store, será lançada uma versão prévia do Windows Terminal para os pioneiros e para fornecer feedback.

E no inverno, planejamos finalmente lançar o Windows Terminal 1.0 e trabalharemos com a comunidade para garantir que esteja totalmente pronto antes do lançamento!

Espere ... você disse em código aberto?


É sim! Temos o prazer de anunciar que estamos abrindo não apenas o Terminal do Windows, mas também o Console do Windows, que contém a infraestrutura de linha de comando no Windows e fornece o tradicional Console UX.

Estamos ansiosos para trabalhar com você para melhorar e expandir os recursos da linha de comando do Windows!

Isso parece incrível, mas por que você não atualiza o console do Windows existente?


O principal objetivo do Windows Console é manter a compatibilidade com as ferramentas, scripts, etc. existentes da linha de comando. Embora tenhamos conseguido adicionar muitos aprimoramentos importantes à funcionalidade do console (por exemplo, adicionar suporte a VT e cores de 24 bits, etc.). postagem no blog ), não podemos fazer mais melhorias significativas no console da interface do usuário sem "quebrar o mundo".

Portanto, chegou a hora de uma nova abordagem.

O Windows Terminal instala e funciona com o aplicativo existente do Windows Console. Se você iniciar o Cmd / PowerShell / etc. diretamente, eles começarão a se conectar à instância tradicional do console da mesma maneira como de costume. Portanto, a compatibilidade com versões anteriores permanece inalterada e, ao mesmo tempo, você pode usar o Terminal do Windows se / quando desejar fazer isso. O Windows Console continuará sendo enviado ao Windows por décadas para dar suporte a aplicativos e sistemas existentes / herdados.

Bem, que tal participar de um projeto ou aplicativo de terminal existente em código aberto?


Estudamos cuidadosamente essa opção durante o planejamento e decidimos que nossa participação em um projeto existente exigiria uma alteração nos requisitos e na arquitetura do projeto, para que fosse muito destrutivo.

Em vez disso, ao criar um novo aplicativo de terminal de código-fonte aberto e o Console do Windows de código-fonte aberto, podemos oferecer à comunidade que trabalhe conosco para melhorar o código e usá-lo em seus respectivos projetos.

Acreditamos que há espaço suficiente no mercado para idéias novas / diferentes sobre o que o terminal pode e deve fazer, e nos esforçamos para ajudar o ecossistema de aplicativos de terminais (e relacionados) a prosperar e se desenvolver através da introdução de novas idéias, abordagens interessantes e inovações interessantes neste espaço.

Convencido! Como participar?


Visite o repositório em github.com/Microsoft/Terminal para clonar, montar, testar e executar o terminal! Além disso, seremos gratos se você reportar bugs e compartilhar feedback conosco e com a comunidade, além de corrigir problemas e fazer melhorias no GitHub.

Neste verão, tente instalar e iniciar o Windows Terminal na Microsoft Store. Se você encontrar algum erro, compartilhe seu feedback através do Hub de Feedback ou da seção Problemas no GitHub, projetada para perguntas e discussões.

Estamos felizes em trabalhar com você! Se você tiver alguma dúvida ou comentário, entre em contato com Kayla @cinnamon_msft e / ou Rich @richturn_ms no Twitter. Estamos ansiosos para ver quais grandes melhorias e recursos você trará para o Terminal Windows e o Console do Windows.
