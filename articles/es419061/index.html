Novedades de GoLand 2018.2

Hola Habr!

GoLand 2018.2 salió la semana pasada. En este artículo hablaremos sobre lo que ha aparecido en la nueva versión.



¿Qué hay de nuevo en breve:

  • soporte para módulos Go (anteriormente conocidos como vgo);
  • Move- refactoring mejorado y nuevas correcciones automáticas;
  • reposición en las familias de acciones de finalización e intención de postfix ;
  • el depurador ahora admite puntos de interrupción que no requieren una interrupción completa del programa, así como la carga diferida de matrices, sectores y diccionarios.

Además, muchas mejoras vinieron de la plataforma IntelliJ: desde innovaciones para sistemas de control de versiones y soporte Touch Bar para MacBook Pro hasta las últimas innovaciones del mundo del desarrollo front-end.

Soporte para módulos Go (vgo)


En febrero, el equipo de Go anunció una versión de prueba de la nueva herramienta de administración de dependencias, e inmediatamente decidimos comenzar a trabajar en su soporte.
GoLand le permite crear proyectos de un nuevo tipo ( Go Module ) y ofrece sincronizar dependencias en un solo clic directamente desde el editor de código.



Lea más sobre el soporte para los módulos Go aquí ( artículo en inglés ).

Mover refactorización


En una versión anterior, GoLand introdujo Move-refactoring, que puede mover rápidamente cualquier anuncio a otro archivo dentro del mismo paquete. En esta versión, le enseñamos cómo mover personajes entre paquetes.

Para ver la imagen completa, agregamos una nueva interfaz que muestra los elementos que dependen del objeto movido. Exporte y transfiera estas dependencias con él.

Edición de código


La nueva acción Agregar alias de importación de puntos le permite hacer referencia a todos los nombres exportados sin usar un nombre de paquete. La acción inversa de Eliminar alias de importación de puntos cancela estos cambios.

Introducir la refactorización de variables locales ahora está disponible a través de Alt + Enter. Una acción rápida escribe el resultado de ejecutar funciones y métodos en una nueva variable local.



Plantillas Postfix


La nueva plantilla .sort agrega un método de clasificación a las matrices y sectores según el tipo de elementos. Si los elementos son de un tipo personalizado, se utilizará el método sort.Slice .

Para facilitar su uso, combinamos las plantillas .rre y .rrv en una nueva: .rr , se puede invocar tanto en expresiones como en nombres de variables.



Estilo de código


Después de estudiar las recomendaciones para los desarrolladores de Go sobre el diseño del código, agregamos resaltado para comentarios que no comienzan con un espacio, y le enseñamos al IDE a formatearlos, agregando un espacio faltante. Después de un tiempo quedó claro que algunas bibliotecas (por ejemplo, easyjson ), por el contrario, ¡requieren su ausencia!

Para las bibliotecas con un requisito similar, hemos agregado una lista de excepciones para la opción Agregar espacio inicial a los comentarios . Ayuda a evitar espacios innecesarios, es suficiente para hacer los prefijos necesarios. Incluido aquí: Configuración / Preferencias | Editor | Código de estilo | Ir | Otros



Depurador


En la nueva versión, el depurador admite puntos de interrupción que no requieren una interrupción completa del programa. Los valores de matrices, sectores y diccionarios se cargan de forma perezosa, y las estructuras ahora se pueden ampliar sin restricciones en el nivel de anidamiento.



Observadores de archivos


Una mejora pequeña pero muy agradable apareció en File Watchers en esta versión. Anteriormente, era necesario configurar File Watchers para cada proyecto por separado, ahora la configuración se puede reutilizar entre proyectos.



Sistema de control de versiones


Brevemente sobre la resolución de conflictos de fusión: es fácil cometer un error, el problema no es fácil de encontrar y difícil de solucionar. Decidimos hacer este proceso menos doloroso, por lo que el sistema de control de versiones ahora agrupa archivos con conflictos de fusión para cada lista de cambios. El cuadro de diálogo mejorado Archivos fusionados con conflictos muestra los nombres de las ramas al ejecutar Pull , Merge o Rebase y agrupa los archivos en directorios.



Hay muchas mejoras para la ventana de registro , ahora puede:

  • Abra todas las ventanas de registro que necesite;
  • Elimina la etiqueta git del commit;
  • Filtrar confirmaciones por ramas seleccionadas;
  • Explore el panel de vista previa de diferencias sin salir de la ventana de registro ;
  • Supervise el estado del repositorio con la nueva acción Examinar repositorio en la revisión .



Además de los cambios para la ventana de registro , VCS ha mejorado en varias formas más.

Interfaz de usuario


Propietarios de MacBook Pro, no aprueben: GoLand 2018.2 es compatible con Touch Bar. Todas las herramientas disponibles para el panel táctil se pueden personalizar para adaptarse a sus tareas: Configuración | Apariencia y Comportamiento | Menús y barras de herramientas .



Otra noticia para los usuarios de macOS: los títulos de las ventanas ahora pueden oscurecerse. Para hacer esto, habilite la opción Usar encabezados de ventanas oscuras en Configuración | Apariencia y Comportamiento | Apariencia



Los iconos cambiaron de color, forma y estilo. Los detalles se pueden encontrar en esta publicación ( en inglés ) de nuestro equipo de diseño de interfaz: comparten cómo tomaron decisiones y qué planean hacer en el futuro.

JavaScript / TypeScript


Por tradición, hablamos un poco sobre los cambios en el soporte de las tecnologías front-end, estos son algunos de los más importantes:

  • Soporte para TypeScript 2.9 y TypeScript 3.0;
  • Muchas nuevas acciones de intención : Implementar interfaz , Crear clase derivada , Implementar miembros de una interfaz o clase abstracta , Generar casos para 'cambiar' e Iterar con 'para..de' .
  • Ahora puede encontrar todo el código no utilizado con la nueva función de cobertura de código.

Puede leer más sobre estas y otras mejoras en la página Novedades del equipo de WebStorm.
