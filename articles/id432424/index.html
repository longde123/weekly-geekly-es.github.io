Infrastruktur Bersertifikat HyperFlex untuk SAP HANA

Kami terus memperkenalkan Anda ke Cisco HyperFlex Hyper-Converged System, dan dalam artikel ini kami akan memperkenalkan infrastruktur berbasis HyperFlex bersertifikat untuk SAP HANA.

Pada Oktober 2018, SAP mensertifikasi HyperFlex untuk peluncuran SAP HANA.

SAP HANA (High-Performance Analytic Appliance) adalah platform penyimpanan dan pemrosesan data berkinerja tinggi berdasarkan komputasi dalam memori. Tujuan utama HANA adalah untuk menyediakan data analitik dengan waktu respons minimum, serta dukungan untuk aplikasi transaksional SAP.

Lansekap SAP terdiri dari server aplikasi dan server DBMS berdasarkan HANA. Server aplikasi telah berjalan dalam bentuk mesin virtual untuk beberapa waktu, dan HANA DBMSs, biasanya diluncurkan pada perangkat keras, yang membuat infrastruktur mahal dan tidak fleksibel, atau dalam lingkungan tervirtualisasi, tetapi dengan sejumlah besar pembatasan dan berdasarkan pada array disk yang sangat mahal. Cisco HyperFlex adalah salah satu sistem hyperconverged pertama yang disertifikasi untuk menjalankan lingkungan produksi HANA dalam lingkungan yang sepenuhnya tervirtualisasi (yaitu, server aplikasi dan DBMS). Pada saat yang sama, HyperFlex adalah platform hemat biaya yang mudah digunakan, digunakan, dan skala.

HANA diluncurkan menggunakan Cisco HyperFlex Certified All-Flash System berdasarkan HX240c M5 All Flash node, dilengkapi dengan dua sentral Cisco UCS 6300 Fabric Interconnects switching devices dengan sistem manajemen node server terintegrasi berdasarkan kebijakan dan template.


HyperFlex adalah solusi terintegrasi yang memungkinkan Anda untuk mengimplementasikan platform virtualisasi berdasarkan blok bangunan standar yang khas - node server dengan hypervisor, disk lokal, dan perangkat lunak khusus yang memungkinkan Anda membangun gudang data yang terdistribusi kesalahan yang didistribusikan. Fitur arsitektur utama HyperFlex adalah distribusi blok dari setiap drive logis dari setiap mesin virtual di semua node di cluster, yang memungkinkan Anda untuk mendistribusikan beban secara merata, sepenuhnya menghilangkan kemacetan dan kemacetan, memberikan kinerja tinggi untuk mesin virtual yang paling menuntut karena penggunaan operasi I / O mesin virtual ini dari semua node, semua perangkat SSD, semua perangkat caching di semua node.

Tes kinerja HyperFlex diberikan dalam artikel sebelumnya tentang Habré , dan juga dalam laporan Lab ESG independen .



Sorotan lain untuk sistem SAP HANA adalah ketersediaan data dan mesin virtual. HyperFlex memungkinkan perlindungan data di berbagai tingkat. Pertama, HyperFlex menyediakan perlindungan data di dalam kompleks dengan menyimpan setiap blok data dalam dua atau tiga salinan pada dua atau tiga node yang berbeda dalam cluster. Kedua, HyperFlex dapat secara asinkron mereplikasi mesin virtual kritis ke kompleks lain di situs lain. Ketiga, ada kemungkinan membangun satu klaster yang berjarak antara dua situs dengan replikasi data sinkron, yang meminimalkan waktu pemulihan dan mengurangi kehilangan data menjadi nol. Tentu saja, pelanggan dapat menggunakan alat ketersediaan di tingkat sistem SAP dan perangkat lunak klaster, serta alat cadangan dan pemulihan yang HyperFlex terintegrasi dengan ketat.

Keuntungan tambahan menjalankan SAP HANA di HyperFlex adalah kemampuan untuk menggabungkan beban kerja HANA produktif dan non-produktif dengan lanskap SAP klasik berdasarkan DBMS tradisional dalam satu cluster HyperFlex, yang sangat menyederhanakan proses migrasi dari SAP klasik ke SAP HANA atau memungkinkan beberapa versi untuk bekerja bersama SAP dalam satu cluster tanpa perlu investasi keuangan tambahan.

Saat ini, HyperFlex tersedia untuk demonstrasi di berbagai wilayah Federasi Rusia di situs mitra kami. Anda dapat mendaftar untuk test drive melalui formulir umpan balik dengan mengklik tautan. Bergabunglah sekarang!
