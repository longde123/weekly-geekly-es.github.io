Cómo aprender API AQUÍ en poco tiempo


 -ŻDónde estás?
-Estoy aqui

A finales de septiembre, el hackatón más grande del mundo tuvo lugar en Kazán. Este evento, según el número de participantes, se incluyó en el Libro Guinness de los Récords.


Durante estas 48 horas, nos encargaron:

Desarrollar un prototipo de una aplicación móvil que permita a un pasajero pedir comida a domicilio en restaurantes ubicados en ciudades a lo largo de la ruta del tren.

Letras de canciones


Por supuesto, durante todo el evento, participamos en la resolución de subtareas heterogéneas sobre nuestro tema. No pasamos las 48 horas aprendiendo la API HERE y escribiendo las tres consultas anteriores.


En este artículo, quiero compartir exactamente la experiencia de usar la API HERE obtenida durante estas 48 horas de insomnio, más precisamente, como desarrollador junior de Python y sistemas distribuidos en la red para implementar la interacción con otros sistemas en la red. El artículo no pretende ser la traducción de toda la documentación de HERE API, solo describe la aplicación práctica para resolver nuestros problemas.


Introduccion


Para optimizar la entrega de las órdenes y su ejecución a tiempo, el servicio de mensajería debe llegar a la plataforma a la hora exacta en que el tren llega a la estación. En este sentido, cada servicio de mensajería debe conocer el tiempo mínimo requerido para la entrega del pedido. Para resolver este problema, fue necesario calcular el tiempo mínimo de viaje entre dos puntos (desde el restaurante hasta la estación de ferrocarril). El camino se calculó en el pueblo, en relación con esto, se decidió considerar varias opciones, a saber, el uso del transporte público, un automóvil personal y sus pies para caminar.


El análisis de las API públicas mostró que las siguientes solicitudes REST son más adecuadas para resolver el problema:


  • Planifique una ruta de A a B usando el transporte público
  • Ruta en coche de a a b
  • Ruta peatonal de a a b

El uso de la API comienza registrándose en la sección de desarrollador del sitio web oficial para generar y recibir las claves de ID de aplicación y CÓDIGO DE APLICACIÓN. La clave API gratuita le permite realizar hasta 250 mil solicitudes por mes. Créeme, esto cubre todas las necesidades de un hackathon.

Practica


Cada solicitud contiene los siguientes campos: 


deplocation = A #   arrlocation = B #   # ,      app_id = os.getenv('HERE_APP_ID') app_code = os.getenv('HERE_APP_CODE')

Encontrar el tiempo de viaje usando el transporte público 


 url = f"https://transit.api.here.com/v3/route.json" query = { 'dep': deplocation, 'arr': arrlocation, 'time': datetime.now().strftime('%Y-%m-%dT%H:%M:%S'), # ,     'app_id': app_id, 'app_code': app_code, 'routing': 'tt' #    } response = requests.get(url, params=query) data = response.json() status = data["Res"] if "Message" in status: print(status["Message"]) exit(-1) if "Connections" in status: route_dut_time = iso8601toSec( status["Connections"]["Connection"][0]["duration"] )

Me gustaría señalar que en esta solicitud, el tiempo se proporciona utilizando ISO 8601. Se implementó la función de convertir la duración del tiempo recibido a segundos iso8601toSec.



Encontrar el tiempo de viaje usando vehículos personales 


 url = f"https://route.api.here.com/routing/7.2/calculateroute.json" query = { 'waypoint0': deplocation, 'waypoint1': arrlocation, 'mode': 'fastest;car;traffic:enabled', #       'app_id': app_id, 'app_code': app_code, 'departure': 'now' #   } response = requests.get(url, params=query) data = response.json() route_dur_time = data['response']['route'][0]['summary']['trafficTime']

No hay problemas con esta solicitud con el tiempo, regresa en segundos.



Encontrar el tiempo de viaje usando sus propias piernas (caminar) 


 url = f"https://route.api.here.com/routing/7.2/calculateroute.json" query = { 'waypoint0': deplocation, 'waypoint1': arrlocation, 'mode': 'fastest;pedestrian', #   'app_id': app_id, 'app_code': app_code } response = requests.get(url, params=query) data = response.json() route_dur_time = data['response']['route'][0]['summary']['travelTime']

En esta solicitud, como en el ejemplo anterior, no hay problemas con el tiempo, regresa en segundos.


Conclusiones


En base a estas solicitudes, obtuvimos tres veces la necesidad de pasar del punto A al punto B. Habiendo calculado el tiempo mínimo y el tipo de movimiento desde ellos, determinamos cuánto tiempo tardó en salir antes de que pudiéramos llegar al punto B en el tiempo indicado.
