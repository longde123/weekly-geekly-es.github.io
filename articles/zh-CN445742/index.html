物联网标准，网络，三张表

物联网的主题以前曾在RIT ++和HighLoad ++会议上滑过，但通常着重于数据仓库或编程的常见问题。 但是去年，我们决定是时候认真讨论IoT了：硬件，数据网络，工业应用，并举行了首届InoThings ++ 2018会议 。

根据参与者的反馈，我们意识到我们不仅需要召开物联网会议-其中有很多会议-社区需要一个完全没有营销和有关彩虹视野的故事的事件，而是旨在解决实际项目的问题。

InoThings ++ 2018表明期望值是合理的，该活动获得5分（满分5分）。照此看，程序委员会负责人兼所有活动参与者Oleg Artamonov的文章均在其中进行了阅读。 今年， 4月4日，我们会做得更好。 我们的演讲者仅是练习，该程序仅包含重要主题。 哪些，请在裁减下阅读。



在InoThings ++上，各个级别的开发人员，架构师和管理人员都在讲话，说的是从业人员-那些动手工作，构建，开发，实施和销售IoT系统的人。 今年将会有更多的从业者：领先公司和电信公司的技术总监，领先的分析师，专家，理学博士，实验室负责人和政府代表。 由于regalia令人头晕，所以我们将不列出所有内容。 只是参加会议-您将自己见到每个人。

物联网实践


整个活动的总体思路是实践中的物联网：在电信运营商，集成商，最终客户和开发人员级别实施IoT项目的实践。 因此，我们将讨论物联网及其周围的一切：设备，软件，测试，加密，协议，业务和法律方面。 间歇泉电信，PVS Studio，非接触式，Sierra Wireless，GoodWAN，三星电子，Lartech和许多其他公司都将演出。

在InoThings ++中，计划10到19个小时安排20个报告和3个圆桌会议 。 该计划与在居民区停车一样严格，但是我们保证会得到喘息的机会-额外的时间和休息时间。 报告将在三个房间中进行：大，中和小。 大型报告-6个，其余报告-7。


时间表的交互式版本在这里 。

演讲的兴趣并不取决于房间的大小，而是取决于主题。 计划委员会从主题上将会议分为三个部分。 三种颜色的轨道是三个主题： 标准和国家法规，项目实施以及设备和系统的开发 。 时间表上的每个部分都有其自己的主题颜色。

每个房间计划召开一个圆桌会议，以讨论物联网的痛点，征求专家意见并与演讲者交谈-准备问题。 稍后关于圆桌会议，但现在让我们看一下报告的主题，这些主题有条件地分为6个块 。

块中的InoThings ++


在“安全和加密”模块中，我们将讨论身份验证和威胁。 为什么传统方法容易受到攻击并且部署起来并不特别方便。 我们将讨论对物联网设备的主要威胁，面临的场景，为什么物联网中的加密不可避免，以及如何通过法律法规，法规和行业专家协会来确保整个物联网的安全性。

技术领先于法规。 尽管物联网中普遍存在相对真空，但监管机构几乎没有规范，限制和关注。 它将很快结束。 在“法律方面”一节中，我们将讨论存在哪些规范，限制和标准，俄罗斯联邦应有哪些，世界上有什么以及如何不使用物联网与国家争执。

在“数据网络，协议，无线电通信”块中，有两个报告，它们与LPWAN相关：分类，问题，机会，解决方案的审查，电源。 有关电源的详细信息，请参见下一节“硬件” 。 它仅包含一份报告 ，但其中一份报告是“适当的营养是电子设备长期成功运行的关键” 。 它是对设计时应注意的外部条件以及一组内部标准（效率，功率，尺寸）的评论。

“软件”块中有5个报告：从选择编程语言到在远程命令和全职命令之间进行选择。

  • 设计另一个控制系统 。
  • 如何用C编写高质量的代码 ，然后切换到Rust 。
  • 如何组织物联网的远程开发而又不着急。
  • 工业经营发展的难点和艰辛。

“ IOT in business”块最大，有6个报告。 让我们从卫星物联网开始，转到航空电信的物联网示例​​和不寻常的LoRa应用案例，了解如何为企业创建有用的东西并教育员工，最后以传统的拯救地球为结尾。 我们所爱的一切。



圆桌会议


每个圆桌会议都与其中一个区块相关，但我们将分别讨论它们。 原则上， 第一张桌子是俄罗斯第一张讨论政府对物联网市场参与的 表格 。 所有相关方都将出席：开发商，公司和国家。 Oleg Artamonov 在另一篇文章中详细介绍了该活动：问题的背景，组织大规模公众讨论的动机以及物联网环境中一长串专家和人物的名单，从技术科学的医生到电信公司的董事。

第二个是关于范式的转变 ：为什么提供成品设备和标准通信服务的经典方法不起作用， 应该归咎于谁，该怎么做 。 Sibintek，Geyser Telecom，MTS和Goodwin Concern参加了会议。

第三张表是关于微电子学的 。 关于国内微电子及其前景。 圆桌会议尚未最终确定，但贝加尔电子公司和NPC Elvis公司将初步确定。

表格彼此不相交，因此您完全有时间向专家提出棘手的问题。
