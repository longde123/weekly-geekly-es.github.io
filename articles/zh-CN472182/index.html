宣布.NET Core 3.1预览版1

我们宣布.NET Core 3.1预览版1 。 .NET Core 3.1将是一个小版本，着重于Blazor和Windows桌面的主要改进，这是.NET Core 3.0中的两个重要改进。 这将是一个长期支持（LTS）版本，预计最终发布日期为2019年12月。

您可以在Windows，macOS和Linux上下载.NET Core 3.1 Preview 1 。

  • .NET Core 3.0 SDK和运行时
  • Docker映像

ASP.NET Core和EF Core今天也发布更新。

Visual Studio 16.4 Preview 2也将在今天发布。 建议您使用.NET Core 3.1预览版进行更新。VisualStudio 16.4包括.NET Core 3.1，因此只需更新Visual Studio即可获得这两个版本。

详细资料：

  • .NET Core 3.1发行说明

已知问题：Visual Studio 16.4安装程序在安装.NET Core 3.1时可能会卸载.NET Core 3.0运行时。 在这种情况下，建议您重新安装或修复.NET Core 3.0 SDK 。



改进之处


.NET Core 3.1计划了一些有针对性的改进。 以下改进尚不可用，但在以后的预览中有望实现。

在.NET Core 3.0中， .NET Core桌面运行时安装程序 （包括WPF和Windows窗体）不会为您安装.NET Core运行时（包括CoreFX和CoreCLR）。 我们将对此进行更改，以使Desktop Runtime安装程序能够自给。 对于服务器方案，运行时和主机捆绑包已经可以通过这种方式工作。

C ++ / CLI是.NET Core 3.0版本的承诺组件。 鉴于它是WPF的必需依赖项，因此已交付了该产品，但是缺少Visual Studio和SDK的开发人员经验。 我们将通过.NET Core 3.1和Visual Studio 16.4提供这种体验。 C ++ / CLI仅在Windows上受支持。

macOS 10.15 Catalina包括一项新的安全要求，即必须对应用程序进行公证 。 我们将满足.NET Core SDK，.NET Core 3.1和所有其他受支持的.NET Core版本的这些要求。 如果您使用.NET Core交付macOS应用程序，我们将与您合作处理公证要求。

闭幕


.NET Core 3.1的主要目标是完善.NET Core 3.0中提供的功能和方案。 .NET Core 3.1将提供长期支持（LTS）版本，至少支持3年。

请安装并测试.NET Core 3.1 Preview 1并提供反馈。 目前尚不支持或建议将其用于生产中。

如果错过了，请查看上个月的.NET Core 3.0公告 。
