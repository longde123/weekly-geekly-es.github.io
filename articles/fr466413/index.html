Échange de sites pour Bitrix et 1C: résoudre les principaux problèmes

Dans cet article, nous avons rassemblé les principaux problèmes d'échange avec des 1 et des listes de contrôle simples pour les résoudre
L'article sera utile pour les programmeurs 1C et les développeurs Bitrix.

Le texte est structuré selon le principe: problème - options de solution. S'il y a une tâche spécifique, cliquez immédiatement sur le lien interne.

Avant de passer aux descriptions, vérifiez d'abord le site avec des tests Bitrix (vérification du système et vérification des droits d'accès). Environ la moitié des problèmes disparaissent ensuite.

Les instructions pour 1C sont données pour UT 11
  • Les prix ne sont pas déchargés
  • Les sections ne sont pas déchargées
  • Photos non téléchargées
  • Les propriétés sont en cours de définition.
  • Erreur 502, mémoire dépassée ou écran blanc
  • Aucun code de caractère n'est créé
  • Nombreuses erreurs 404 après partage avec le site
  • Les sous-sections se déplacent à la racine du site à partir des sections parentes

Les prix ne sont pas déchargés


  • Vérifiez du côté 1 que
  • Accords de contrepartie standard autorisés
  • Un accord standard avec les contreparties est créé, accessible aux utilisateurs externes
  • Le type de prix y est sélectionné.


  • Le produit a un prix avec ce type de prix
  • Dans la "sélection et le tri" des marchandises indiqué cet accord dans le champ "prix"



  • Si vous avez Bitrix Small Business, le type de prix doit être un

Les sections ne sont pas déchargées


Autrement dit, les marchandises arrivent sur le site sans sections

Options pour raisons et solutions

  • Les sections peuvent contenir des champs obligatoires qui ne peuvent pas être reçus de 1c (par exemple, une photo). Ensuite, lors de l'échange, une section ne sera pas créée suite à une erreur
  • Dans Bitrix Small Business, plusieurs types de prix peuvent être configurés dans un téléchargement. Dans ce cas, le déchargement n'aura pas lieu en raison du fait que les prix sont d'abord déchargés, puis de la structure des partitions, et si le déchargement renvoie une erreur de prix dans le déchargement des prix, alors les partitions ne se chargeront pas.

Photos non téléchargées


  • Dans 1C, vérifiez qu'il y a des coches pour télécharger des fichiers et des images
  • Appuyez sur le bouton pour un échange forcé avec des photos.
  • Si cela n’aide pas, vérifiez: dans les paramètres d’échange avec 1C, «Utiliser ZIP» doit être activé sur le côté du site et le module ZIP doit être sur le site.
  • Dans ce cas, la "taille maximale du fichier téléchargé" doit être inférieure à post_max_size et upload_max_filesize dans les paramètres du serveur. Parce que si la photo ne correspond pas à la taille de la limite de chargement du serveur, la division en plusieurs fichiers zip peut résoudre le problème



Les propriétés sont en cours de définition.


La même propriété vient 2 fois ou plus.
La raison en est que dans 1c, les propriétés sont attachées aux types de l'élément et s'il y en a plusieurs, alors plusieurs propriétés sont créées pour la même Et la même.

Comment décider

Dans 1c, vous devez créer une propriété en copiant la précédente et assurez-vous d'indiquer que «attachez cette propriété à un nouveau type d'élément»

Écran 502 ou blanc


Le site se bloque avec une erreur 502 ou avec une erreur dépassant la quantité de mémoire allouée.

Dans les paramètres d'échange avec 1C côté site, le temps pour une étape d'échange est de 1 à 5 secondes
Si cela ne vous aide pas, échangez sans photos.

L'échange peut ne pas avoir lieu car le site ne peut pas couper l'aperçu d'une très grande image.



Aucun code de caractère n'est créé


Très probablement, mbstring n'est pas configuré sur le serveur. Consultez le site avec un test Bitrix interne.



Nombreuses erreurs 404 après partage avec le site


Vous avez probablement une coche «translittérer un code symbolique du nom lors de la mise à jour de la section».

Pour cette raison, lors du changement de nom, l'adresse change également, ce qui est très mauvais pour le référencement




Les sous-sections se déplacent à la racine du site à partir des sections parentes


Cela se produit très probablement lors de l'échange en mode «uniquement les modifications» avec la case à cocher «Ne pas importer le niveau supérieur du groupe de produits s'il est le seul»
