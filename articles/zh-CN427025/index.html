区块链花园中的五块石头


有了这样一张图片，现在就可以描述该行业许多人的心情，当时加密货币的汇率自今年年初开始暴跌。 膨胀的泡沫为区块链技术做了强有力的广告，他们威胁要在每个角落使用。 一年多以前，它飙升至Gartner曲线的最高点，但目前，根据我的感觉，我们处于最低点。

是什么阻止了我们国家（乃至世界）的区块链，根本需要它吗？

1.在干渣中


如果我们抛弃ICO，采矿场和交换器，俄罗斯的公司或大型国有公司仍然鲜有消息：

  • 自2017年初以来，M.Video一直在使用智能合约进行保理业务，其中超过65％的保理交易是在区块链上进行的。 在过去的一年中，使用该技术的交易数量增长了4.7倍，支付数量增长了4.4倍[1]。
  • ANEX“ LEKSAN”创建了一个统一的基础设施，用于开发和实施行业专用的数字系统（通常，其名称为RSCA的怪异而and谐），在其中的功能中可能会进行正式的正式ICO，以保护参与者的权利。 该系统处于与中央银行和财政部的协调阶段[2]。
  • 9月，俄罗斯技术公司（Rostec）与沃斯托克（Vostok）项目签署了一项在数字经济基础设施中合作和共同实施区块链的协议。 他们将建立一个合作项目办公室，并投资约200万美元[3]。
  • 今年7月，VEB区块链能力和数字转换中心与格罗兹尼市政厅一起，在区块链上启动了两个试点项目，涉及住房和公共服务管理以及市政物业租赁[4]。
  • 3月，LANIT公司集团宣布成立数字转型集团，这是数字生态系统的集成商：其任务是创建新的商业模式，包括基于区块链技术的商业模式[5]。
  • 国家技术计划有一个FinNet方向。 [6]它涉及去中心化的金融系统和个性化的网络金融服务：很明显，这里的区块链被视为创建加密货币和交易所的基础，类似于现有的Coinbase，Bitfinex，Binance，Poloniex等。

还有其他这样的例子。 但是，最终的实现仍然很少，我们正在谈论意图。

此外，区块链在实施复杂技术方面继续享有声誉，事实上，没有哪个俄罗斯公司会持续使用它。 即使我们将整个市场作为一个整体，仍然只有两个实际可行且有需求的项目-比特币和以太坊。

然后出现一个问题，是什么阻止了这数百甚至数千个项目的开发？ 因此，我们将继续讨论采用这项技术的障碍。 我认为，在这种情况下，障碍仍然多于优势。 至少现在。

2.休斯顿，我们有问题


2.1。 低稳定性和可扩展性


在区块链项目中，经常提到“每秒处理的事务数”这样的特征。 这个数字差异很大，从比特币的单位到IOTA或EOS等新项目中的数千个（根据声明）。 我们仍然在缓慢但可靠的PoW与快速但集中式（D）PoS以及许多其他共识算法之间寻求平衡。

但是，在迄今为止的两个最大项目中，比特币和以太坊均未解决稳定性和可伸缩性问题。 在公众对它们的兴趣急剧增加的时刻（ICO，密码学，课程推测），交易可以被确认数小时，甚至根本不发送。

2.2。 最终用户登录门槛高


诚然，区块链仍然是众多极客和加密无政府主义者。 对于普通人来说，这仍然是未知的野兽，新闻中写着什么。 我花了几周的时间向我妈妈解释以比特币为例的区块链。

尽管在Internet上似乎有很多信息，但这更有可能产生相反的效果，而且要找到真正有用且可理解的文章或视频并不容易。 知识水平接近零的人如何开始使用以太坊？ 我如何了解我现在可以使用哪些有用的dapp？ 尚无此类信息。 现在，所有这些都类似于万维网的早期阶段，当时有许多不同的网站，但是尚不清楚如何使用所有这些。

2.3。 交易不可逆转


是的，通常被称为技术优势的特性已成为其障碍。 输入地址并进行交易时，没有人能消除人为因素和出错的可能性。 即使这不是由于用户的错误而发生的，也可能是由于恶意黑客的错误而发生的，该黑客将替换其中一个级别的地址。 这样的错误之一可能非常昂贵。

通过银行转帐，您可以对交易提出异议，但是没有这样的事情。 吓到很多 为了在此阶段使用区块链，您需要清楚地了解自己在做什么，可能发生的事情以及如何保护自己免受其影响。

2.4。 算法知识不足


自从开始对最著名的工作量证明共识算法进行“野外”测试以来，已经过去了9年多的时间。 其性能已被实践证实，但是该算法本身具有许多缺点。 因此，社区已经提出了数十种[7]类似算法：

  • 权益证明
  • 委托权益证明；
  • 权限证明；
  • 委派拜占庭式容错；
  • 重要性证明
  • ...
  • 任何东西的证明。

似乎其中许多人甚至还不到两年，更不用说某种理智的测试了。

以太坊中的智能合约是潜在漏洞代码的另一层。 错误在这里可能非常昂贵，[8]非常[9]昂贵。 由于智能合约技术的新颖性，代码审计是非常肤浅的，还是第一次还是根本没有进行。

还值得一提的是全新的TCR技术，在Habré上[10]是原始文章的翻译，其中包含对该算法的第一个或多或少的详细描述。 在对帖子的评论中，您可以听到一个合理的评论，但这通常在哪里应用？为什么？ 是否必要和是否有用将有待观察。

2.5。 缺乏专家


根据IBM商业价值研究院“在政府中建立信任”的一项研究[11]，在公共部门引入区块链的技术问题是与技术合作的技能水平不足 （考虑到该领域缺乏称职的专家）以及缺乏已实施的商业案例 。 我们可以肯定地与最后一个声明争论-这是一个专业知识不足的问题，因为已经有在公共部门实施区块链的实例，但是在俄罗斯却没有。 例如，2016年4月，Bitfury与乔治亚州国家公共登记局签署了一项协议，以启动一个试点项目，以使用Exonum框架将房地产注册中心转移到区块链上。 该系统正在成功运行，现在[12]，飞行员本人在2017年将[13]输入了哈佛商学院的案例收集。 这是一个例外。

但是，我不会否认很多聪明的开发人员来到了区块链，他们厌倦了旧技术并想尝试新的东西。 但是，正如您所知，从来没有很多程序员。

3.可以发挥加什么


投资者的兴趣。 IDC分析师公司预测[14]到2022年，全球在区块链市场上的投资约为117亿美元。 在未来的20年中，这些数字可能会增长10倍以上。

缺乏公认的标准，其公开形成的可能性。 鉴于技术的跨国性，在世界各地的演讲者的参与下，在不同国家举行的大量会议和黑客马拉松[15]，很明显，如果希望进行对话，交流经验并共同做出决定，则可以讨论和更改技术标准。

降低基础设施维护成本。 已经有一些项目可让您与其他用户共享设备上的可用空间或计算资源。 我们每个人都有一部智能手机，处理器上有99％的时间处于空闲状态，所有这些计算能力都可以用光！ （不，不是针对比特币挖矿）。 我认为，未来将取决于口袋和台式机中所有这些巨大的计算资源的经济和合理使用。

信息系统的稳定性和可访问性的新水平 。 如实践所示，当故障点是唯一的服务器时，很容易失去对特定站点的访问。 在新的分散范例中，可以从系统中的任何节点（可能是数千个）获得对系统的访问。

4.结论


现在，我们目睹了许多试图应用区块链的项目。 假设这将毫无痕迹地通过是愚蠢的。 我们仍在等待新的“杀手级应用程序”（您认为它会是什么？），因为它曾经成为Netscape浏览器，或者Facebook如何永久改变了社交互动的机制。 也许这将是多种新技术同时融合的产物-区块链，人工智能和某种虚拟现实。 我们可能不得不重新考虑价值的概念：一些加密无政府主义者已经表达了对个人加密货币出现的幻觉。 当每个人都有自己的货币时，他自己的等价货币取决于他的社会等级（中国好）。

尽管Web 3.0的概念仍处于起步阶段，但越来越多的人认为新的Internet是有价值的Internet。

Web 1.0是关于发布的-任何人都可以编写并具有全球分布。
Web 2.0涉及交互-与他人联系，实时协作，数据。
Web 3.0涉及价值转移-现在可以对资金进行编程并嵌入任何应用程序中。

-Brian Armstrong（@brian_armstrong） 2018年8月24日
