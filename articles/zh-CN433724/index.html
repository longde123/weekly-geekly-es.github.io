介绍和简短的教育计划


有版本，配置文件和编解码器。 因此，您的设备可能具有蓝牙3.0版。 就像多年以来所有版本的蓝牙一样，其中的音乐配置文件是A2DP。 在A2DP配置文件中，您可以使用不同的音乐编解码器（MP3，AAC，SBC，aptX，LDAC等）将音乐文件压缩为可以通过蓝牙传输的比特率。

在蓝牙中，发送设备通常称为SRC（源），而接收设备通常称为SNK（接收器）。

高级音频分发配置文件或A2DP于十年前开发。 在此配置文件之前，存在所谓的“语音层”，该层是为耳机设计的，带宽仅为64 kbit / s。 蓝牙的其余部分为“数据层”。 因此，为了提高声音质量（特别是用于传输立体声信号），开发了A2DP，其本质是捕获旨在“数据传输”的比特率的某些部分并使之适应音乐。

还有一个音频/视频远程控制配置文件-AVRCP，它可以并行工作以提供控制数据的传输：更改音量的命令，切换音轨等等。

A2DP可以从共享池获得什么数据速率？ 实际上， 当前版本的A2DP没有任何限制，但是有一些建议，对于SBC编解码器，它们如下所示：



因此，在48 kHz的采样频率下，最大可用值为345 kbit / s，在44.1 kHz-328 kbit / s的情况下。 由于未压缩CD的质量需要1411 kbps的流，因此上述建议的比特率明显低于未经压缩传输音乐所需的比特率。

有损与无损


进一步论证，区分两种压缩类型（无损压缩和有损压缩）很重要。 两者都减小了文件大小，因此减小了比特率的“负载”。 名称可以说明一切-在无损压缩过程中，信息不会丢失，并且接收器中还原的文件与源文件中的文件相同。 无损压缩通过定义可以比发送原始数据本身更快地描述和传输的模式来减小文件大小。 粗略地说，“在文件中说1000个单词”连续“打招呼”比说“你好” 1000次要快。 信息将是相同的，传输将无损失，但会压缩。

无损压缩率受到限制， 几乎所有现有的无损编解码器（Apple Lossless，FLAC，Ogg Vorbis）都允许实现大约两次压缩，而不再需要压缩 。 该值取决于内容的类型和使用的方法。

有损压缩通过丢弃位来减小文件大小。 有损编解码器会搜索他们认为您听不到的信息，然后将其删除，以希望您不会感到与众不同。 使用这种方法，可以达到什么程度的压缩没有限制-只会抛出更多信息。 MP3 128k压缩CD约13次。 如上所述，部分压缩是无损的。 其他所有内容都是丢弃的数据。 制造商声称的软件或硬件创新无关紧要-您将永远无法恢复扔掉的东西，就像重新粉刷黑白照片一样。

好吧，无论从理论上还是从逻辑上讲，这都是如此。

aptX


您可以在Bluetooth A2DP配置文件中使用各种编解码器。 实际上，您甚至可以使用MP2，MP3，WMA，AAC和ATRAC-但是，必须支持一种编解码器：SBC（子带编解码器）。 该编解码器以高达48 kHz的各种特定采样率支持单声道和立体声流。

所有其他编解码器都是可选的（包括Sony ATRAC系列），公司甚至可以定义自己的编解码器。 但是：为了成功运行，必须在SRC和SNK设备上预先安装这些编解码器的支持。

因此，被称为“最高质量的广泛蓝牙编解码器”的aptX，只有在两个设备都与aptX兼容的情况下才能工作，例如您的智能手机和蓝牙扬声器。

AptX兼容性要求制造商支付许可费，该费用将支付给aptX所有者（CSR / Qualcomm）。 嗯，这已经在上一篇文章中讨论过了，但是我们注意到直到最近，Apple仅在小型MacBook系列中包括aptX，而Android设备仅从第七版操作系统（aptX HD-从第八版）获得了aptX支持。在此版本中，对于某些设备，在软件级别禁用了支持。 显然，许多SNK设备的制造商-无线扬声器，声音面板等-甚至都不会考虑这种支持，尤其是在预算技术方面。

AptX通常被称为“ CD质量”，但事实并非如此，因为A2DP配置文件中将AptX限制为328 kbps，尽管通常指示达到384 kbps。 我们记得，无损压缩可提供最大的两倍增益，因此，以768 kbit / s的速度最多可以传输384 kbit / s，很明显aptX不能达到1411 kbit / s的“ CD质量”所需的数据流，因此，将其称为“接近CD的质量”更为正确-实际上，可以媲美AAC或MP3 320 kbps文件。

特别值得一提的是已经提到的休闲aptX HD编解码器。 实际上，这是相同的aptX，但是具有不同的编码配置文件，并且比特率高达576 kbit / s的可能性。 再一次，简单的数学表明，达到“ CD质量”这一数字还远远不够，尽管它已经非常接近（1152 kbit / s）。

高通公司市场与销售总监Johnny McClintock（JM）董事在接受Sound + Image（S + I）采访时正式证实了我们的结论：

S + I：您能确认aptX HD编解码器是无损还是有损？
JM：从技术上讲，aptX HD是一种有损压缩编解码器。 但是，aptX HD使用4：1的相对较软的压缩率；它本质上不是破坏性的。 由于它基于自适应差分脉冲编码调制，而不是基于心理声学感知原理，因此aptX HD使用预测分析，而不是尝试通过合成耳朵重现人类听力的特征。
S + I：它可以在A2DP配置文件中使用还是使用其他数据？
JM： aptX HD算法将A2DP用于蓝牙。
S + I：可能的最大数据速率是多少？
JM：要实现24位深度并支持48 kHz的采样频率，需要576 kbit / s。

因此，aptX编解码器不允许实现“ CD质量”。

索尼LDAC


第一部分 故事


公平地讲，应该说Sony已经拥有并且已经具有非标准编解码器的蓝牙解决方案-这些是支持MP3的Sony DR-BT50耳机 。 还有一个Sony ATRAC编解码器，对于那些早已被淘汰的微型磁盘来说，每个人都很熟悉。

但是，在这里，澳大利亚索尼公司代表了该公司进入无线音频解决方案LDAC编解码器市场的愿望的新颖性。 该公司只是简单地将其决定命名为“索尼多房间系统” 。



该系统包括很多：三个无线扬声器（包括一个便携式扬声器，一个条形音箱和一个声底座）以及一个SX7微型Hi-Fi系统。 我们还建议添加一种受支持的Sony AV接收机，即STR-DN860或1060。

最终的多房间功能包括DLNA高清晰度的流，从24位/ 96 kHz到24位/ 192 kHz。 此外，还有蓝牙，它使用了索尼自己的LDAC音频编码技术，据该公司称， 当通过蓝牙启动高清音频播放时，其数据传输速率为990 kbps 。

实际上，LDAC具有三个级别，分别以330、660和990 kbps的速率运行-为了获得最高的数据传输速度，您应该选择质量优先的模式。
与aptX一样，仅当源和接收器都装有LDAC时才能使用LDAC。 当前，该列表很小，而且非常昂贵 。

但是回到主要问题：使用蓝牙如何实现？ 正如我们所看到的，990 kbps的数据传输速率比整个立体声A2DP蓝牙配置文件的规范要高得多，更不用说其中的标准编解码器了。

即使我们将其放在一边，也不清楚数字是如何累加的。 未压缩的16位/ 44.1 kHz上的CD质量流的数据速率为1411 kbit / s。 因此，LDAC中指定的990 kbps可以轻松地使用无损压缩，以通过LDAC获得完整的CD质量。 这是可以理解的。

但是，这种高清晰度的声音呢？ 即使我们谈论的是24位/ 96 kHz的较低声明边界-24/96流也应具有4.5 Mbit / s以上的传输速度。 为了以990 kbps的比特率获得这种质量而又不损失，您需要大约5：1的压缩率，这是所有已知的无损编解码器的两倍以上。 也就是说，是否完全一样，编解码器会压缩丢失的数据？ 但是，这种“高清声音”又是什么呢？
