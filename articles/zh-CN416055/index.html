3QX服务器FQDN的分配和支持

我们的用户定期对3CX服务器的FQDN的用途和支持存在疑问（现在可以由3CX提供）：选择FQDN的原理，将此FQDN分配给许可证密钥的原则等。 在本文中，我们将回答许多这些问题。

在安装3CX v15和15.5时，许可证密钥将附加到管理员在服务器安装过程中选择的FQDN。 因此，立即选择正确的FQDN非常重要，这样以后就可以避免不必要的工作。

他们经常问这个问题，为什么我们需要3CX服务器的FQDN？ 使用IP地址难道不是很容易吗？ 我们回答：

  • 用于3CX客户端到Web服务的远程安全（SSL）连接，该服务传达用户状态，呼叫历史记录，聊天，集中式客户端管理和其他功能，这可以称为通用术语：统一通信。
  • 提供对3CX管理界面的安全远程访问
  • 为了实现容错能力，IP电话可以连接到单个FQDN，可以将其解析为主服务器的IP地址和备用服务器的地址。
  • 使用动态IP地址。 如果您的ISP不提供静态IP地址，则3QX FQDN为您提供动态DNS服务。
  • 保护您的许可证。 您购买的许可证已分配给您的域。 如果未经授权使用PBX，将无法使用此键上的PBX，因为 您会立即看到有人使用您的域名和虚假IP地址。

首先，请考虑为3CX服务器选择FQDN，前提是您是首次安装此许可证密钥（尚未绑定FQDN）并使用3CX v15 SP4或更高版本。

3CX域


积极订阅产品更新的3CX v15用户可以从3CX的一个域区域中接收公共域名。 此服务是免费提供的。 除域名外，用户还会从Let's Encrypt中自动收到此FQDN的免费受信任SSL证书。 在安装过程中选择安装程序界面中提供的主机名（用户部分）3CX和域区域（主机部分）后，生成的FQDN（userpart.hostpart.tld）将附加到激活的许可证密钥上。

此外，还根据以下原则为WebMeeting服务创建了一个附加的FQDN：userpart.hostpart.tld-> userpart-tld.3cx.net

常见领域： 

customer.3cx.de -> customer-de.3cx.net Customer.3cx.us -> customer-us.3cx.net

二级域名： 

 customer.3cx.co.uk -> customer-uk.3cx.net customer.3cx.co.nz -> customer-nz.3cx.net customer.3cx.com.tr -> customer-tr.3cx.net customer.3cx.co.za -> customer-za.3cx.net

例外域： 

 customer.elastix.com -> customer-ecom.elastix.net

城市域减少为3个字母： 

 customer.3cx.barcelona -> customer-bar.3cx.net

第二级子域： 

 customer.north.3cx.us -> cusomer-north-us.3cx.net

3CX保留随时撤销分配给您的FQDN的权利，该FQDN具有以下特征：

  • 侵犯其他公司的商标
  • 用于3CX托管以外的目的
  • 包含侮辱和其他此类文本
  • 3CX公司本身需要（在这种情况下，3CX与合作伙伴联系以正确传输FQDN）



我们还注意到3CX用户无权要求另一个用户已经使用的3CX域名。

所有域名均发布为可用。 如果FQDN长时间未使用，则将其释放给其他用户。 在这方面，不能保证您先前链接到许可证密钥的FQDN仍然可用。

自己的网域


如果要在自己的公司FQDN下安装3CX服务器（不是由3CX发行），请记住，这需要3CX Pro或Enterprise版本。 您还必须提前购买该域名的受信任SSL证书（或整个域的通配符证书）。 不幸的是，在这种情况下，不会颁发免费的“让我们加密”证书。



3CX许可证密钥将绑定到您自己的FQDN，并且WebMeeting门户的FQDN的格式如下：userpart.hostpart.tld-> userpart-hostpart-tld.3cx.net

范例： 

 customer.domain.com -> customer-domain-com.3cx.net customer.domain.co.uk -> customer-domain-co-uk.3cx.net pbx.customer.domain-part.co.nz -> pbx-customer-domain-part-co-nz.3cx.net

FQDN 3CX免费许可证


3个月内未使用的免费3CX许可证（NFR附属密钥除外）从其关联的FQDN中取消绑定。 然后，其他用户可以注册这些FQDN。 如果FQDN已被其他用户使用，则您将在密钥激活阶段看到消息“已使用”（请参阅​​下文）。

不同版本的3CX中的DNS TTL


DNS记录（TTL）的缓存时间取决于您拥有的许可证类型。 对于Standard和Pro许可证，它是在6点安装的。 对于企业许可证-300秒。 由于时间很短，因此可以非常快速地（最多5分钟）将用户切换到备份服务器的IP地址，因为TTL越低，FQDN A记录更新到当前IP地址的速度就越快。

更改FQDN


如果您想更改当前的FQDN，请查看我们以前的文章 。

可能的错误


据说，在首次安装期间，3CX安装程序会将您选择的FQDN绑定到您的许可证密钥。 但是，如果所选的FQDN已在使用中，则会出现以下错误： 

 Error creating FQDN: FQDN already in use. Please choose another one

在这种情况下，只需选择其他FQDN。

请注意，使用相同的密钥重复安装3CX将“拉出”与其关联的FQDN。 如果指定其他FQDN，将出现错误： 

 Error creating FQDN: License key already bound to another FQDN

在这种情况下，您需要转到3CX用户门户并检查与该密钥关联的FQDN。 如果所附的FQDN不适合您，请将其从钥匙上解开，然后再继续安装3CX。

以下奇怪的错误也是可能的： 

 Domain zone XYZ is exhausted and can not be used. Please choose another domain.

在这种情况下，只有一种解决方案-选择另一个顶级域。
