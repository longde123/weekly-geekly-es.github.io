Raspberry Pi上的Mozilla WebThings-入门


来自翻译


Mozilla已为智能家居设备建立了通用集线器，以将不同供应商和协议（包括Zigbee和Z-Wave）的设备链接在一起，并在不使用云的情况下从一个地方进行管理。 一年前，有关于第一个版本的新闻 ，今天，我几天前发布了更新文档的翻译，它回答了该项目的大多数基本问题。 我期待在评论中进行讨论和交换意见。


Raspberry Pi的WebThings网关


Mozilla的WebThings网关是用于智能家居系统中网关的软件，它允许通过Internet直接监视和控制智能设备，而无需中介。


你需要什么


  1. Raspberry Pi计算机和电源（Raspberry Pi 3至少需要2A）
  2. microSD卡 （至少8 GB，等级10）
  3. USB适配器 （请参阅兼容适配器列表）

注意： Raspberry Pi 3带有Wi-Fi和蓝牙。 需要USB适配器才能使用Zigbee和Z-Wave等协议连接设备。


1.下载图片


从Mozilla IoT网站下载图像。


2.缝制图像


将图像缝到microSD卡上。 有多种录制方法 。 我们建议使用Etcher 。



  1. 开式蚀刻机
  2. 将存储卡插入计算机的适配器。
  3. 选择图像作为来源
  4. 选择一个存储卡
  5. 点击“ Flash！”

完成后，取出存储卡。


3.下载Raspberry Pi



  1. 将存储卡插入Raspberry PI
  2. 连接USB适配器（如果有）
  3. 打开电源开始下载。

注意： Raspberry Pi的首次启动可能需要2-3分钟。


4. Wi-Fi连接


下载后，网关将创建“ WebThings网关XXXX ”访问点（其中XXXX是Raspberry Pi的MAC地址中的四位数）。 从计算机或智能手机连接到这一点。



连接后，您应该会看到WebThings网关的欢迎屏幕，然后它将开始搜索您的家庭Wi-Fi网络。



从列表中选择您的家庭网络，然后输入密码进行连接。


注意事项：


  • 如果您已连接到“ WebThings Gateway XXXX”访问点，但没有看到欢迎屏幕，请尝试打开位于192.168.2.1的页面。
  • Raspberry Pi也可以使用以太网电缆连接到网络。 在这种情况下，它将尝试自动从路由器获取网络的IP地址。 然后输入浏览器“ http：//gateway.local”以进行网关的初始配置。
  • 如果将网关移动到另一个地方，或者它无法访问原始网络，它将自动切换到接入点模式，以便您可以连接到该网关并配置另一个网络。

5.子域选择


将网关连接到网络后，请确保您要设置的计算机或智能手机在同一网络上。 之后，在浏览器中转到gateway.local地址。


之后，您可以选择注册一个免费的子域，以通过Mozilla的安全隧道访问本地网络外部的网关。



输入所需的子域和电子邮件地址（以在将来重置密码），然后单击“创建”。


注意事项：


  • 您可以跳过此步骤并完全在本地使用网关，也可以自行配置端口和DNS转发。 但是，在这种情况下，如果将来您仍然决定使用Mozilla子域，则必须完全重置网关设置。
  • 如果没有打开gateway.local上的页面，请尝试通过路由器查找网关的IP地址（在已连接设备的列表中查找“ gateway”之类的设备，或者以“ b8：27：eb”开头的MAC地址），然后尝试打开该页面直接通过IP。
  • 如果gateway.local和http：//不起作用，请确保您的计算机和Raspbeery Pi都连接到同一网络。
  • 如果您之前已经注册了子域，请输入其名称和注册时使用的电子邮件地址。 屏幕将显示获得访问权限的说明。

6.创建一个帐户


注册子域后，将打开一个页面，其中包含用于配置网关的以下步骤。 输入您的姓名，电子邮件地址和密码，然后单击“下一步”。



注意：以后可以创建其他帐户。


做完了！


之后，应打开“事物”页面以将智能设备连接到网关。



有关进一步的配置，请参阅《 WebThings网关用户指南》 。
