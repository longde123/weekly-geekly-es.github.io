GitHub sperrt das Repository für das Wort "retard" in der Beschreibung

Vor ein paar Tagen wurde ein Commit für das WebMConverter- Repository vorgenommen , das das Wort Retard (Bremse, Dummkopf) in der Projektbeschreibung korrigiert . Zuvor war die Projektbeschreibung WebM for Retards, die in WebM for Gits geändert wurde. Der Repository-Besitzer zeigte den Text des vom Github gesendeten Briefes an:


Für diejenigen, die nicht gut Englisch sprechen, kurz gesagt -
, ToS ( ) . , «retard» «retarded».
24 . , : ( )

Es ist erwähnenswert, dass das Repository für 3 Tage blockiert wurde, während der Eigentümer versuchte, den Support des Githubs zu kontaktieren.
Eine hitzige Diskussion fand in den Kommentaren zum Commit und im entsprechenden Thema zum Reddit über die Legitimität der Github-Aktionen statt, insbesondere darüber, wie weit dieser Kampf um Minderheitenrechte gehen könnte und ob er die Entwicklung von Open-Source-Projekten mit Github beeinträchtigen würde. Viele bemerkten, dass das Wort git selbst (eine dumme, inkompetente, nervige Person) missbräuchlich ist und beschuldigte den Github der Doppelmoral. Es wurde vorgeschlagen, dass unter dem Deckmantel des Kampfes gegen Diskriminierung anstößige Projekte blockiert werden können.

Es scheint mir, dass dies in RuNet nicht viel diskutiert wurde (ich könnte mich irren). Es war interessant zu wissen, was die Benutzer der Gigtime darüber denken, insbesondere diejenigen, die den Github verwenden.
