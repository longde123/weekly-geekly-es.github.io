Open source tidak menghasilkan uang, karena itu tidak dibuat untuk ini

Cara terbaik untuk melakukan sesuatu adalah dengan setidaknya mencoba

Semua orang tahu bahwa tidak mungkin menghasilkan uang dengan sumber terbuka, bukan?

Saya memikirkan topik ini karena Mozilla ingin mendiversifikasi pendapatannya dalam beberapa tahun ke depan, tetapi kami memiliki satu batasan - semua perkembangan harus dipublikasikan dalam sumber terbuka. Ini adalah puluhan (ratusan?) Dari proyek open source yang sukses. Banyak yang mencoba mendapatkan sesuatu, beberapa berusaha sangat keras. Hasilnya begitu-begitu.

Saya mencoba untuk meluncurkan inisiatif komersial di Mozilla sekarang (jika menulis rencana dan menayangkannya dapat dianggap sebagai "peluncuran"), dan mereka sering bertanya kepada saya: apakah mungkin untuk menjual sesuatu dengan kode sumber terbuka?

Saya tidak punya bukti bahwa kita bisa (atau tidak bisa) menghasilkan. Tetapi saya berani membuat pernyataan berikut: sulit untuk menjual apa yang tidak dimaksudkan untuk dijual.

Kami memperlakukan open source sebagai kebalikan dari produk komersial. Dan ya, lisensi gratis membuat sulit untuk mengumpulkan uang dari pelanggan, meskipun ada banyak bisnis sukses yang menghasilkan uang baik tanpa memaksa siapa pun untuk membayar.

Saya melihat asumsi tersirat yang membuat logika sulit: gagasan bahwa jika sesuatu bermanfaat, itu harus menguntungkan. Ini adalah harapan yang tak terucapkan dan dibenarkan secara moral, semacam Hipotesis Keadilan: jika sesuatu bermanfaat, jika itu membantu orang, dunia membutuhkannya dan memberikan sesuatu kepada orang lain, maka harus ada kemungkinan monetisasi. Seharusnya dimungkinkan untuk melakukan ini setiap hari dan diberi imbalan atas upaya Anda dalam mengembangkan atau mendukung program ini.

Kami percaya bahwa dunia harus seperti itu. Tetapi kita semua tahu bahwa ini tidak benar. Anda tidak dapat mencari nafkah dengan musik. Atau seni. Anda bahkan tidak bisa mencari nafkah untuk anak-anak. Saya pikir ini sebagian besar mendasari kritik terhadap kapitalisme modern: ada terlalu banyak hal penting dan bahkan perlu yang kita butuhkan lebih dari produk komersial apa pun, tetapi hampir tidak mungkin menghasilkan uang dari itu.

Saya tidak akan mencoba memperbaiki dunia di blog saya, saya hanya mencatat: tidak semuanya baik menghasilkan uang.

Tetapi kita tahu bahwa dalam industri perangkat lunak pasti ada uang. Banyak uang! Mungkin uang dalam kode kepemilikan tertutup? Jika OpenSSL ditutup, bisakah ia menghasilkan? Jika dia memiliki sistem lisensi berbayar? Sepertinya tidak mungkin. Lisensi tidak menahannya. Itu tidak terlihat seperti produk yang dirancang untuk keuntungan. Memecahkan masalah penting tidak cukup untuk menghasilkan uang.

Apa yang mereka bayar?

  1. Orang akan membayar sedikit untuk aplikasi; tidak terlalu banyak, sedikit. Penskalaan membutuhkan pemasaran dan investasi, yang hampir tidak pernah dimiliki proyek open source (dan saya ragu bahwa banyak proyek open source akan tahu apa yang harus dilakukan dengan investasi jika mereka mendapatkannya).
  2. Iklan selalu punya uang. Menyedihkan. Jika Anda benar-benar menerapkan iklan, itu dapat menyinggung pengguna sehingga mereka tidak akan terlalu malas untuk mengemas ulang perangkat lunak Anda dengan menghapus iklan dari itu. Saya pikir karena diskriminasi harga seperti itu, opsi iklan tidak akan bertahan.
  3. Hosting lengkap: Saya pikir contoh yang bagus adalah wordpress.com dari Automattic . Bagaimana kabar Hantu ? Ini adalah solusi komprehensif: Anda tidak hanya mendapatkan perangkat lunak, Anda mendapatkan situs web.
  4. Orang akan membayar untuk keputusan individu. Yaitu, konsultasi. Sebagaimana diterapkan pada produk perangkat lunak, perangkat lunak konsultasi diperoleh. Terlepas dari kritik, banyak perusahaan nyata dibangun di atas ini. Saya pikir Drupal termasuk dalam kategori ini.
  5. Orang-orang akan membayar perhatian Anda yang berdedikasi dan konstan. Dengan kata lain: pekerjaan tetap sebagai karyawan. Tampaknya tidak adil untuk memasukkan opsi ini dalam daftar, tetapi merupakan pengembangan alami dari consultingware dan template open source yang dominan sehingga layak disebutkan.
  6. Semuanya terkait dengan perangkat fisik. Orang-orang mengenali nilai kompleks perangkat lunak dan perangkat keras secara keseluruhan.
  7. Saya tidak yakin bagaimana Firefox menghasilkan uang: baik (secara tidak langsung) pada iklan, atau sebagai kompensasi untuk mempertahankan monopoli.

Saya yakin tidak semua ide menarik tercantum di sini.

Tetapi jika Anda memiliki konsep bisnis dan Anda pikir itu bisa berhasil, apa hubungannya sumber terbuka atau tertutup dengan itu? Bukankah kita tahu bahwa kita perlu fokus pada bisnis! Pada klien! Lisensi perangkat lunak tampaknya seperti orang luar. Bahkan perangkat lunak itu sendiri adalah hal yang meragukan untuk fokus, terpisah dari bisnis. Mungkin itu sebabnya Anda tidak dapat menghasilkan uang dengan open source: itu mengalihkan perhatian dari hal utama. Masalahnya bukanlah oposisi antara open source dan open source, tetapi oposisi open source terhadap bisnis itu sendiri.

Anda dapat melihat dari sisi lain: siapa pelanggan Anda? Perangkat lunak open source klasik yang dibuat oleh programmer untuk programmer. Ini sangat sukses, tetapi dirancang untuk orang yang tidak ingin membayar. Mereka ingin mengambil perangkat lunak dan meningkatkan produktivitas pribadi (langkah cerdas, mengingat kenaikan upah untuk programmer). Bisakah kita menjual open source kepada orang lain? Bisakah orang lain melakukan sesuatu dengan kode sumber?

Oleh karena itu, saya tetap pesimis tentang fakta bahwa Anda dapat memperoleh di open source. Dan juga kekecewaan: ada begitu banyak program open source yang bagus, tetapi tidak ada produk komersial. Di sinilah misi open source jatuh meskipun banyak keberhasilan. Perangkat lunak yang benar-benar bekerja dengan orang tidak gratis atau terbuka. Ini memalukan.
