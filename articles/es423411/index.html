iOS CSS de la muerte


Se ha publicado un código en github que conduce a forzar el reinicio de dispositivos iOS (11/12 GM) al visitar una página html. También hace que Mac OS High Sierra / Mojave se congele al usar Safari.


El código que lleva a forzar el reinicio es un código html con una gran cantidad de divs anidados y una función de desenfoque de fondo "loco": 


... div { backdrop-filter: blur(10px); -webkit-backdrop-filter: blur(10px); width:10000px; height:10000px; } </style> </head> <body> <div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div> ...

También publicó una página html de PoC que implementa este error (los usuarios de Safari corren bajo su propio riesgo y riesgo).



Se supone que el error se deslizó a un nivel más bajo que el kit web y puede tener consecuencias más graves. Además, el uso de este error puede ser aplicable en empresas sociotécnicas y bromas estúpidas y prácticas, por lo que le aconsejo que sospeche mucho de todos los enlaces, especialmente porque en iOS todos los navegadores, de hecho, son un complemento para Safari.
