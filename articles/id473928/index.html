Kepunahan file komputer

Layanan teknologi bermodel mengubah kebiasaan Internet kami.




Saya suka file. Saya suka mengubah nama mereka, memindahkan, mengurutkan, mengubah cara mereka ditampilkan di folder, membuat cadangan, mengunggahnya ke Internet, memulihkan, menyalin, dan bahkan mendefrag mereka. Sebagai metafora untuk cara blok informasi disimpan, saya pikir mereka hebat. Saya suka file secara keseluruhan. Jika saya perlu menulis artikel, itu akan muncul di file. Jika saya perlu menerbitkan gambar, itu akan ada dalam file.

Ode ke file .doc


Semua file skeuomorfik. Skeuomorphism adalah kata kunci yang berarti refleksi dari objek fisik dalam bentuk digital. Misalnya, dokumen Word seperti selembar kertas yang terletak di desktop (layar) Anda. File .JPEG terlihat seperti gambar, dan sebagainya. Masing-masing file ini memiliki ikon kecil sendiri, mirip dengan objek fisik yang mereka wakili. Tumpukan kertas, bingkai untuk gambar atau folder Manila. Sangat menawan, bukan?

Apa yang saya benar-benar suka tentang file adalah bahwa ada satu cara untuk berinteraksi dengan mereka, tidak peduli apa yang ada di dalamnya. Hal-hal yang saya sebutkan di atas - menyalin, menyortir, defragmenting - saya dapat melakukan ini dengan file apa pun. Ini bisa berupa gambar, bagian dari permainan atau daftar hidangan favorit saya. Defragmentasi tidak peduli, tidak ada bedanya jenis file itu. Saya telah menyukai file sejak saya mulai membuatnya di Windows 95. Tetapi sekarang, saya semakin menyadari bahwa kita mulai menjauh dari mereka sebagai unit kerja mendasar.


Windows 95. Fakta menarik: kedutan cepat mouse mempercepat OS. Ini tidak terkait dengan artikel; Saya hanya berpikir itu menarik.

Pertumbuhan file .mp3


Sebagai seorang remaja, saya mencoba-coba mengumpulkan dan mendigitalkan vinil, saya adalah seorang kolektor file MP3 yang rajin. Dalam koleksi saya ada banyak file MP3 dengan bit rate 128 Kbps. Anda sangat beruntung jika memiliki penyalin dan Anda dapat menyalin file ke CD, dan kemudian mentransfernya satu sama lain. Volume CD bisa mencapai 700 MB. Ini setara dengan hampir 500 floppy disk.

Saya meninjau koleksi saya dan dengan susah payah meletakkan tag musik: IDv1 dan IDv2. Seiring waktu, orang mulai mengembangkan utilitas yang secara otomatis mengunduh daftar lagu dari cloud sehingga Anda dapat memeriksa dan menentukan kualitas file MP3 Anda. Saya kadang-kadang mendengarkan rekaman-rekaman terkutuk ini, walaupun saya curiga bahwa waktu yang dibutuhkan untuk mengatur dan memvalidasinya jauh lebih besar daripada waktu yang dibutuhkan untuk mendengarkan.


Aplikasi bernama The Godfather. Dia memiliki banyak peluang.

Kemudian, sekitar 10 tahun yang lalu, semua orang secara aktif mulai menggunakan "aplikasi hijau" - Spotify. Menggunakan aplikasi atau situs web mereka, Anda dapat menyiarkan apa pun yang Anda inginkan dan kapan pun Anda mau. Saya pikir itu sangat keren dan nyaman. Tapi apa kualitasnya? Apakah ini lebih baik daripada MP3 128 kbps saya?
Ya, kualitasnya ternyata lebih baik.

Selama semua peristiwa ini, 128 Kbps, yang, kami diberitahu, "tidak bisa dibedakan" dari file WAV besar yang mulai muncul di CD, berubah menjadi sampah. Sekarang bitrate file MP3 mencapai 320 Kbps. Di forum, orang melakukan analisis spektral file, membuat grafik hijau dan biru cerah, untuk "membuktikan" bahwa file terdengar sangat bagus.

Pada saat inilah kabel berlapis emas SCART Monster menjadi terobosan nyata.



Kualitas file pada layanan streaming cukup baik, mereka tersedia di lebih banyak perangkat dan Anda diberi akses ke semua musik yang direkam, bukan hanya MP3, seperti yang ada di komputer Anda. Anda tidak lagi memerlukan koleksi file yang dipikirkan dengan cermat di hard drive Anda. Anda hanya perlu nama pengguna dan kata sandi untuk Spotify.

Bagus, pikir saya, tetapi saya masih memiliki file video besar. Internet terlalu lambat untuk mengirimkan video saya.

Pemakaman file .png


Saya memiliki telepon Sony Ericsson dengan nama menarik k610i. Dia merah dan saya sangat suka. Saya bisa menghubungkannya ke komputer dan menyalin file ke sana. Itu tidak memiliki port headphone, jadi saya harus menggunakan adaptor atau headphone khusus yang dibundel dengannya. Dalam banyak aspek, dia lebih maju dari zamannya.



Kemudian, ketika saya menghasilkan lebih banyak uang dan teknologi bergerak maju, saya membeli iPhone. Tidak diragukan lagi dia cantik. Alumunium hitam, begitu hitam sehingga tampak lebih hitam daripada kegelapan dan kaca medis - detail yang berbatasan dengan ideal, tampaknya diturunkan dari surga oleh para dewa.

Tetapi Apple sangat mempersulit akses kami ke file. Gambar diunggah ke aliran besar, disortir berdasarkan tanggal. Audio di suatu tempat di iTunes. Catatan ... apakah ini daftar? Aplikasi tersebar di desktop. Beberapa file terletak di iCloud. Anda dapat mengirim foto langsung dari iPhone Anda, melalui email, dan menggunakan metode rumit melalui iTunes, Anda dapat mengakses beberapa file di aplikasi tertentu. Tetapi file-file ini bersifat sementara, mereka di-cache dan dapat dihapus tanpa peringatan apa pun. Ini tidak seperti file dari komputer saya yang saya buat dengan hati-hati.
Saya hanya ingin mendapatkan kembali file browser saya.

Di Macbook, iTunes sendiri mengurutkan file musik untuk Anda. Mereka diproses oleh sistem. Musik ditampilkan di antarmuka, dan Anda dapat mengaturnya. Tetapi jika Anda melihat di bawah tenda, melihat file sendiri, Anda dapat melihat lubang kelinci, kekacauan, nama-nama aneh dan folder aneh. "Jangan repot-repot dengan ini," kata komputer, "Saya akan menghadapinya bukan Anda." Tapi saya khawatir!

Saya suka bisa melihat dan mengakses file saya. Tetapi sekarang sistem yang saya gunakan berusaha mencegah hal ini. "Tidak," kata mereka, "Anda hanya dapat mengakses melalui antarmuka yang unik." Saya hanya perlu browser file saya, tetapi sekarang sudah dilarang. Ini adalah peninggalan dari era masa lalu.

Saya tidak bisa menyingkirkan file, folder, dan kontrol yang biasa saya gunakan.


Windows 10. Anda masih dapat bekerja dengan file-file Anda, meskipun kadang-kadang tampak bagi saya bahwa mereka melihat saya dengan putus asa dan harapan.

Caching dan dependensi file .tmp


Saya mulai membuat situs pertama saya saat itu ketika ada gif transparan 1-pixel dalam mode, dan menggunakan tabel dianggap cara yang tepat untuk membuat tata letak dua kolom. Seiring waktu, praktik terbaik telah berubah, dan saya dengan senang hati mengulangi mantra bahwa tabel harus digunakan hanya untuk data tabel, bukan untuk tata letak, secara bertahap dan dengan kesulitan mengubah tata letak sepele saya ke CSS. Setidaknya itu bukan meja, kataku bangga, melihat tata letak tiga kolom saya, yang tidak berfungsi dengan baik di Firefox.



Sekarang, ketika saya membuat situs web, saya memulai instalasi NPM dan mengunduh 65.000 dependensi yang termasuk dalam folder node_modules. Ada begitu banyak file. Tapi aku tidak peduli tentang mereka. Ketika saya membutuhkannya, saya hanya menghapus folder dan menjalankan instalasi NPM lagi. Sekarang mereka tidak ada artinya bagiku.
Beberapa tahun yang lalu situs terdiri dari file, sekarang mereka terdiri dari dependensi.

Suatu hari saya menemukan sebuah situs yang saya tulis dua puluh tahun yang lalu. Saya mengklik dua kali pada file, itu mudah dibuka dan mulai. Kemudian saya mencoba meluncurkan situs web yang saya tulis 18 bulan lalu, dan ternyata saya tidak dapat memulainya tanpa memulai server web, dan ketika saya memulai instalasi NPM, ternyata beberapa file (mungkin satu atau dua) dari 65.000 kesalahan terjadi, akibatnya Node tidak dapat menginstalnya dan situs web tidak dimulai. Ketika semuanya berhasil bagi saya untuk membuatnya berfungsi, ternyata saya membutuhkan database. Dan kemudian dia mengandalkan beberapa API pihak ketiga, tetapi masalah berikut terjadi dengan CORS, karena saya tidak masuk daftar putih di localhost.

Dan situs saya, yang terdiri dari file, terus mengepul. Saya tidak ingin mengatakan bahwa bertahun-tahun yang lalu situs lebih baik, tidak. Saya hanya mengatakan bahwa situs sebelumnya terdiri dari file, sekarang mereka terdiri dari dependensi.

Di mana-mana. Tinta tautan.


Saat menulis artikel ini, tidak ada satu file pun yang rusak. Saya pergi ke Medium dan mulai mengetik. Mengikuti kata-kata saya dikirim ke database.
Unit yang dibuat dipindahkan dari file ke database.

Di satu sisi, itu tidak terlalu penting. Datanya sama, hanya disimpan dalam basis data, bukan dalam dokumen HTML. Bahkan URL mungkin sama, hanya di latar belakang mengambil konten dari jenis penyimpanan lain. Namun, konsekuensinya jauh lebih besar. Konten sepenuhnya bergantung pada infrastruktur, bukan pada kemampuan untuk bekerja sendiri.

Seseorang mendapat perasaan bahwa ini mengurangi nilai keterampilan kreatif individu. Sekarang, alih-alih membuat file Anda sendiri, semuanya hanyalah baris lain di tabel database di suatu tempat di langit. Sebagai contoh, artikel saya, alih-alih berada di dalam file sendiri, Anda dapat mengatakan, "jadilah sendiri," hanyalah perlengkapan kecil di mobil besar.

Namun, ketika saya mengunggah foto ke Google Cloud dan mengunggahnya lagi, file yang dihasilkan berbeda dari yang asli. Ini telah dienkripsi, didekripsi, dikompresi, dan dioptimalkan. Itu manja. Analis spektral pasti akan geram. Ini mirip dengan fotokopi di mana halaman menjadi lebih ringan dan kotor dari waktu ke waktu. Dan hanya kemudian, ketika saya akhirnya mendapatkan file ke tempat di mana saya dapat menggunakannya, saya menemukan bahwa itu "didorong dan ditarik" berkali-kali sehingga hanya cangkang dan bekas kejayaan yang tersisa darinya.
Mengapa konten baru sangat penting?

Tidak ada lagi file .webm


Seperti kebanyakan dari kita, saya memiliki kekacauan dalam layanan Internet saya, semakin banyak kehidupan pribadi saya bercampur dengan pekerjaan. Dropbox, Google Drive, Box, OneDrive, Slack, Google Documents, dan sebagainya. Tentu saja ada banyak lainnya. WeTransfer, Trello, Gmail ... Kadang-kadang mereka mengirimi saya tautan ke Google spreadsheet untuk bekerja, saya membukanya, dan mereka berhasil disimpan di drive Google pribadi saya di sebelah foto seekor ayam cantik yang saya bagikan dengan ibu saya dan dokumen dengan daftar berbagai mouse komputer yang Saya akan membeli pada tahun 2011.

Secara default, di Google Documents, semua file diurutkan sesuai urutan terakhir dilihat. Saya tidak bisa menyortir dan mengaturnya. Semuanya diatur sedemikian rupa sehingga prioritas diberikan kepada file baru, dan bukan untuk apa yang benar-benar penting bagi kami.

Saya pribadi tidak suka transisi ini dari konten abadi ke yang baru. Ketika saya mengunjungi situs, mereka mengiklankan kepada saya hal-hal terbaru yang saya lihat. Kenapa yang baru harus penting? Tidak mungkin bahwa sesuatu yang baru saja dibuat akan menjadi lebih baik dari semua yang telah dibuat untuk sepanjang masa. Apa peluang bahwa setiap kali saya datang ke suatu tempat, puncak pencapaian manusia runtuh pada saat itu? Ternyata, tidak ada penyortiran berdasarkan kualitas. Hanya ada hal baru.


Buku perpustakaan - anehnya, mereka tidak disortir berdasarkan edisi terbaru.

Semua layanan ini, setidaknya bagi saya, sangat membingungkan dan tidak nyaman. Tempat pembuangan di mana peluang kami terakumulasi. Mungkin ini cara semua orang mengelola file mereka? Setiap kali saya menggunakan komputer orang lain, saya selalu kagum dengan jumlah file berantakan yang tersebar di sekitar mereka. Semua file tersebar secara acak, dan tidak ada pertanyaan tentang urutan apa pun. Bagaimana mereka menemukan sesuatu di sana?

Layanan ini telah sepenuhnya menghapus seluruh makna file dari bidang visi kami. File ini di Dropbox: apakah ini versi terbarunya? Atau itu hanya salinan dari apa yang sebenarnya hidup di komputer saya? Atau apakah seseorang mengirim versi baru melalui email? Atau menambahkannya ke Slack? Dengan cara yang aneh, ini membatalkan konten file. Saya tidak percaya mereka lagi. Jika saya melihat file di Dropbox, saya pikir: "Oh, mungkin ada versi yang lebih baru di sana."

Di tempat kerja, saya melihat kolega yang membuat file, mengirimnya melalui email dan bahkan tidak repot-repot menyimpan lampiran ke hard drive. Kotak masuk mereka adalah sistem manajemen file baru mereka. "Apakah kamu mendapatkan meja?" Mereka bertanya. Seseorang melihat pesan yang masuk dan mengirimkannya kembali melalui email. Apakah itu cara kami mengelola data di abad ke-21? Ini adalah langkah mundur yang aneh.



Saya ketinggalan file. Saya masih membuat banyak file saya sendiri, tetapi lebih sering bagi saya kelainan, seperti menggunakan pena daripada pena. Saya kehilangan fleksibilitas file. Dengan fakta bahwa file dapat bekerja di mana saja dan mudah dipindahkan.

File telah digantikan oleh platform perangkat lunak, layanan, ekosistem. Ini tidak berarti bahwa saya mengusulkan pemberontakan terhadap semua layanan. Kami tidak dapat menghentikan kemajuan, menyumbat saluran Internet. Saya menulis tentang ini untuk berduka atas kehilangan kepolosan yang kita miliki sebelum kapitalisme akhirnya menyerbu Internet. Ketika kita menciptakan sesuatu sekarang, kreasi kita hanyalah bagian dari sistem besar. Kontribusi kami sedikit dalam gugus basis data yang tangguh ini. Tapi layak berhenti membayar, karena kita tidak punya apa-apa. Tanpa file "Anda". Layanan dihentikan.
