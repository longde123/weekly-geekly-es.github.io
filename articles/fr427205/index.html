No 12 Unicode
Question
☺ + 1 = ♫
Quatre options:
♦ - ☺ = réponse
♯ - ☺ = réponse
♣ - ☺ = réponse
♥ - ☺ = réponse
Comment décider
Les caractères sont insérés dans HTML à l'aide de leurs codes Unicode à 16 caractères.
La première ligne est un indice: 0x263A + 0x31 = 0x266B , c'est-à-dire l'unité ici n'est pas elle-même, mais le code 0x31 (49).
Ainsi, pour obtenir la réponse sur la calculatrice en mode hexadécimal, soustrayez le second du premier code de caractère. Convertissez la différence résultante en un caractère dans la table Unicode. Sous Windows, charmap.exe est disponible pour cela.
La réponse
♦ - ☺ = ,
♯ - ☺ = 5
♣ - ☺ = )
♥ - ☺ = +
PS si vous avez indiqué un code décimal ou hexadécimal dans la réponse, cela est également compté.

Virus №15
Question
Un virus de réseau hypothétique infecte tout équipement. Chaque machine infectée en infecte une autre toutes les 30 secondes.
Une machine infectée est apparue sur votre réseau 10.0.0.0/[19...25]. Combien de minutes toutes les machines du réseau seront-elles infectées? (On suppose que toutes les adresses sont utilisées)
Comment décider
Les options varient de 10.0.0.0/19 à 10.0.0.0/25. Supposons que nous ayons la version 10.0.0.0/20.
Nous découvrons de quel type d'enregistrement il s'agit - l'adresse réseau et le masque. Il existe des calculatrices en ligne - google sur demande "calculatrice de masque de réseau".

Si vous comptez manuellement, le nombre après la barre oblique signifie combien de bits de 32 seront attribués à l'adresse réseau. Les 32-20 = 12 bits restants peuvent être utilisés pour les adresses de machine, c'est-à-dire il s'avère que 2 12 = 4096 adresses maximum (en fait 4094, car la première adresse 10.0.0.0 est l'adresse réseau et la dernière 10.0.15.255 est diffusée, mais cela n'affecte pas la réponse).

Le nombre de machines infectées augmente de façon exponentielle toutes les 30 secondes: 2, 4, 8 ... Il est facile de voir que le nombre de machines infectées N(t)=22t où t est le temps en minutes. N = 4096 et la fonction inverse nous donnera le temps souhaité d'infection complète:

t= frunclog2N2= frunclog240962=6



PS Il y a un léger défaut dans cette question: il n'est pas indiqué comment arrondir la réponse. Par conséquent, les bons sont comptés comme fractionnaires, ainsi que arrondis vers le haut et vers le bas.

No 18 Addition dans une colonne
Question

Dans l'exemple, certains chiffres sont remplacés par des lettres.
Trouvez ce que ces nombres sont égaux et écrivez la réponse sous la forme ABCDE, par exemple 12345.
Comment décider
Vous pouvez, bien sûr, le résoudre par force brute sur un ordinateur, mais vous pouvez le faire sur une feuille de papier: dans les nombres ajoutés, il y a 8 chiffres, et au total - 9, ce qui signifie qu'il y a eu un report, donc D dans l'ordre le plus élevé de la somme ne peut être qu'un. D = 1 .

Nous examinons la décharge suivante - la décharge des unités: soit B + B = 9, soit B + B = 19, mais 9 est un nombre impair, de sorte que l'unité de transfert provient de la décharge précédente. C'est-à-dire soit 2B = 8, puis B = 4, soit 2B = 18, puis B = 9, mais le neuf se trouve déjà en plein air, ce qui signifie seulement B = 4 .

Nous regardons la décharge de dizaines: on sait maintenant que E + C = 11 .
Nous regardons la dernière décharge (la plus à droite) 9 + 4 = A, donc A = 3 et il y avait une unité de transfert.

Nous regardons le quatrième chiffre après le point décimal: les unités du chiffre précédent n'ont pas été transférées, (4 + 4 + 1 = 9) et 1 + E = C. Nous savons également que E + C = 11, c'est-à-dire C = 11-E, ce qui signifie 1 + E = 11-E, transférez 2E = 11-1, il s'avère E = 5 , C = 6 .
La réponse
34615
PS Certains ont été tellement emportés qu'en réponse, ils ont noté le montant total au lieu des valeurs ABCDE - il a été considéré comme correct comme exception.
