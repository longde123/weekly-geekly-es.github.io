Le registre des packages GitHub prend en charge les packages Swift

Le 10 mai, nous avons annoncé la version bêta limitée de GitHub Package Registry, un service de gestion de packages qui facilite la publication de packages publics ou privés à côté de votre code source. Il prend actuellement en charge les outils de gestion de packages familiers: JavaScript (npm), Java (Maven), Ruby (RubyGems), .NET (NuGet) et Docker, avec d'autres à venir.

Aujourd'hui, nous sommes ravis d'annoncer que nous ajouterons la prise en charge des packages Swift au registre des packages GitHub. Les packages Swift facilitent le partage de vos bibliothèques et de votre code source entre vos projets et avec la communauté Swift.



Disponible sur GitHub, Swift Package Manager est un outil multiplateforme unique pour créer, exécuter, tester et empaqueter votre code Swift. Les configurations des packages sont écrites dans Swift, ce qui facilite la configuration des cibles, la déclaration des produits et la gestion des dépendances des packages. Ensemble, le gestionnaire de packages Swift et le registre de packages GitHub vous faciliteront encore plus la publication et la gestion de vos packages Swift.

Il est essentiel que les développeurs mobiles disposent des meilleurs outils pour être plus productifs. Avec la croissance de l'écosystème Swift, nous sommes ravis de travailler avec l'équipe d'Apple pour aider à créer de nouveaux flux de travail pour les développeurs Swift.

Depuis son lancement, nous avons été surpris de voir votre enthousiasme à l'idée de commencer avec le registre de packages GitHub. Pendant cette période bêta, nous nous engageons à apprendre des communautés et des écosystèmes sur la façon dont il répond à vos besoins et sur ce que nous pouvons faire pour le rendre encore meilleur. Si vous ne l'avez pas déjà fait, vous pouvez vous inscrire à la version bêta limitée maintenant .
