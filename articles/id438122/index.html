Numerologi pada MS SQL - percobaan yang menghibur

Dari zaman kuno orang suka bermain angka. Untuk membuktikan bahwa perbandingan panjang piramida Cheops dengan tinggi adalah ... Saya tidak ingat apa. Fisikawan juga tidak asing dengan ini, misalnya, ada rumus mistis Koid yang menghubungkan massa elektron, muon dan partikel tau. Ada rumus untuk struktur halus konstan - tidak seperti rumus Koide, yang tampaknya sangat buatan. Seberapa validkah formula tersebut? Saya melakukan percobaan.


Ambil N angka: A, B, C ... Dalam percobaan saya, saya membatasi diri menjadi tiga angka. Untuk setiap nomor kita dapat menerapkan fungsi unary: SIN, COS, EXP, LN (Saya membatasi diri saya sendiri sampai empat). Ini memberikan 4 * 3 = 12 angka baru, yang bersama-sama dengan aslinya memberikan 15 angka. Selanjutnya, kami menerapkan operasi biner +, -, *, / untuk kombinasinya. (Anda juga dapat mempertimbangkan orang lain, misalnya, eksponensial, tetapi sekali lagi saya membatasi diri saya menjadi empat). Di sini kombinasi baru adalah 15 * 15 * 4 (pada kenyataannya, lebih sedikit, karena beberapa operasi dilarang, seperti membagi dengan 0, dan untuk + dan * jumlah kombinasi lebih sedikit karena simetri mereka).

Selanjutnya kita dapat mengulangi langkah-langkah ini lebih dan lebih lagi. Sudah pada langkah kedua, 34'513'800 formula (sekarang Anda mengerti mengapa saya membatasi jumlah operasi?) Itu memberi saya untuk A = 1, B = 2, C = 3 bilangan bulat 2'776'355 angka yang berbeda.

Grafik di atas menunjukkan konsentrasi (jumlah angka yang berbeda) untuk sub-rentang panjang 1 dari -60 hingga +60. Skala Y adalah logaritmik. Konsentrasi angka yang terlihat di sekitar 0.



Lakukan zoom untuk rentang -2..2:



Di sini skala Y sudah normal. Puncak sekitar 0 dan 1.

Kami melakukan pembesaran maksimum untuk melihat "struktur halus" dari distribusi angka:



Saya bertanya-tanya seberapa akurat kita dapat mengekspresikan angka arbitrer, katakanlah, 1,23456789? Ini ditentukan oleh (setengah) panjang maksimum segmen antara dua titik yang berdekatan (jika kita tidak beruntung). Di bawah ini perhitungan ditunjukkan dalam bentuk grafik, dan lebih jauh dari nol keakuratan perkiraan menurun:



Jadi, sebagai suatu peraturan, kita dapat mengekspresikan angka apa pun dengan akurasi E-6 hingga E-5. Misalnya, angka 1.23456789 tampaknya berada di antara

cos (ln (3) / cos (3)) + sin (1 / ln (3)) = 1.23456481266341 (0,0002%)
ln (exp (1) * sin (2)) + exp (ln (3) / cos (3)) = 1.23456894186555 (0,000085%)



Akhirnya, menarik apa yang akan terjadi jika alih-alih A = 1, B = 2, C = 3, kita mengambil angka lain, misalnya, A = sqrt (2), B = e, C = pi. Perbandingan kerapatan angka di bagian pertama (123) dan kedua (2epi) yang Anda lihat dalam gambar:



Seperti yang Anda lihat, pada umumnya, tidak ada perbedaan. Sebagai kesimpulan, saya ingin memberi tahu Anda apa yang harus dilakukan dengan MS SQL. Tugasnya sangat lengkap, dan solusi lintas bergabung hanya meminta, yang mengimplementasikan produk Cartesian dari semua angka yang tersedia untuk operasi biner. Anda dapat melihat potongan kecil kode di bagian akhir.

Kode lengkap tidak dipublikasikan karena saya ingin memodifikasinya untuk secara otomatis menghasilkan teks teori konspirasi berdasarkan numerologi. 

-- step 3 insert into Formula (step,path,Value) select 3,path+' '+op, case when op='COS' then COS(Value) when op='SIN' then SIN(Value) when op='EXP' then case when Value<100 then EXP(Value) else NULL end when Value<=0 then NULL when op='LN' then LOG(Value) end from Formula, Unary -- step 4 select L.path+' '+R.path+' '+'+' as path,L.value+R.value as value into p1 from Formula L, Formula R where Ln<=Rn select L.path+' '+R.path+' '+'+' as path,L.value+R.value as value into p2 from Formula L, Formula R where Ln<=Rn select L.path+' '+R.path+' '+'+' as path,L.value+R.value as value into p3 from Formula L, Formula R select L.path+' '+R.path+' '+'+' as path,L.value+R.value as value into p4 from Formula L, Formula R where R.value<>0
