Acer C710: instalando o Windows 10

Para assistir e sentir o ChromeOS há três anos, comprei um Chromebook - Acer C710-2847, a um preço muito bom - US $ 200 (~ 8.000 rublos com entrega). Comprei nos Estados Unidos com entrega por um intermediário.


Diferentemente dos chromebooks lançados na época, o processador Intel no C710 (os primeiros chromebooks ARM apareceram nos quais o Windows da área de trabalho nunca teria sido instalado) e existe a possibilidade de uma atualização - existem dois slots para RAM e um HDD de tamanho normal de notebook (é por isso que Este modelo foi selecionado).

Características
: 11.6", 1366 x 768
CPU: Intel Celeron 847
RAM: 2 GB, 5GB ( 12 GB)
HDD: 300 GB
: 3xUSB 2.0, HDMI, VGA, RJ-45, kensington lock, 3.5 Jack, Card reader.
HD WebCam

Como não havia BIOS, os entusiastas da Internet pareciam tentar instalar um sistema operacional diferente do ChromeOS no laptop.

Depois de jogar com o ChromeOS, além disso, o Ubuntu (ChrUbuntu) foi instalado com a capacidade de alternar entre sistemas; somente o Debian foi instalado com um BIOS preliminar piscando no Coreboot SeaBIOS no qual o laptop foi usado até recentemente.

Chegaram as férias de Ano Novo e apareceu a hora das experiências.

As pesquisas levaram às instruções de instalação do Windows 10 em coolstar.org/chromebook

Configuração do BIOS


No terminal, execute:

cd ~; curl -L -O http://mrchromebox.tech/firmware-util.sh && sudo bash firmware-util.sh

Selecione o terceiro item pressionando o número 3 no teclado.


Somos avisados ​​sobre os possíveis riscos de queimar o dispositivo, confirmamos que fomos avisados ​​pressionando "y" no teclado.


Em seguida, haverá uma pergunta sobre UEFI ou Legacy, selecione UEFI pressionando "U" no teclado.


Estamos aguardando a mensagem verde sobre a instalação bem-sucedida.


Pressione Enter, q, Enter.

Gravação de unidade flash inicializável


Encontramos um computador com o Windows instalado. Faça o download da versão mais recente do Rufus no site oficial . Encontre a imagem do Windows 10 x64. Nós inserimos a unidade flash. Abra o Rufus, selecione a unidade flash USB, selecione a imagem. “Esquema de partição e tipo de interface do sistema”, selecionamos “GPT para computadores com UEFI”! Pressione start e aguarde o final do processo de gravação.


Instale o Windows 10


O touchpad não funcionará; por conveniência, você pode encontrar um mouse USB e conectar-se a um laptop. Nós inserimos a unidade flash USB no laptop, ligamos e pressione Esc. Selecione o item "Boot Manager".


Em seguida, selecione UEFI USB Flash. Instalamos o Windows (no meu caso, apaguei todas as partições e coloquei no disco inteiro).

Instalação do driver


Touchpad


Isenção de responsabilidade:

1. O touchpad funcionará apenas quando o modo de teste estiver ativado.
2. Às vezes, ele não funciona após suspensão / hibernação / reinicialização, é tratado pela reinicialização (às vezes, pode ser necessária uma reinicialização repetida).

No momento, não era possível encontrar uma versão assinada do driver para o touchpad, portanto:

Execute a linha de comando como administrador: Iniciar → Utilitários → Windows → Linha de comando → botão direito → Avançado → Executar como administrador.

Realizamos:

bcdedit -set testsigning on

Reinicie o laptop. Baixe todos os arquivos neste link . Execute e instale o crostouchpad.4.0-2-installer.exe.

Touchpad funciona.

Dispositivos desconhecidos no controle do computador


Abra Iniciar → Ferramentas de administração → Gerenciamento do computador → Gerenciador de dispositivos. Dispositivo desconhecido → Botão direito → Desconectar.

Para cada um dos dois dispositivos restantes com um ponto de exclamação amarelo, Botão direito → Atualizar drivers → Pesquisa automática de drivers.

Um dos dispositivos será enviado e funcionará normalmente com o nome "Controle térmico da família de chipsets Intel® série 7 / C216 - 1E24".

Descompacte 200888.7z. O segundo dispositivo → Botão direito → Atualizar drivers → Procurar drivers neste computador → Especifique o caminho para o arquivo descompactado → Avançar. Estamos aguardando a instalação dos drivers.

O dispositivo será enviado e não será iniciado com o nome "Driver participante do processador da plataforma dinâmica Intel® e da estrutura térmica".

Não foi possível iniciar a operação de dois dispositivos no Gerenciador de dispositivos, apenas desativou o que não estava iniciando.

Driver participante do processador da plataforma dinâmica Intel® e da estrutura térmica → Botão direito → Desativar.

Sumário


Windows 10 está funcionando.

Vídeo pequeno - carregamento e login no Windows


Funciona com saída de áudio, funciona HDMI, VGA, CardReader, câmera web, teclado e touchpad (apenas no modo de teste).

Funciona subjetivamente mais rápido que a combinação “mesmo laptop” / ChromeOS, “mesmo laptop” / Debian ou “mesmo laptop” / Ubuntu.

Trabalhar com um laptop no colo é desagradável, o AIDA64 mostra a temperatura do diodo PCH 79 ° C, a mesma coisa aconteceu no Linux (o problema provavelmente é apenas a minha cópia).

É difícil falar em autonomia, a bateria tem três anos, antes disso o laptop funcionava apenas na rede.

Para executar o Chrome simultaneamente com uma dúzia de guias, sincronização FileZilla, Yandex.Disk (mais de 100 GB), o laptop é suficiente para os olhos.

A saída era um laptop completo (diferente do ChromeOS) com o Windows 10 completo.
