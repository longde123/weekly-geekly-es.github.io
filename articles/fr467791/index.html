Analyse du défilement des pages dans Google Data Studio

Bonjour, je m'appelle Chakkanbaev Ilkhom Je suis l'auteur du blog SeoPulses, je voudrais partager la traduction et la mise en œuvre pratique de l'article michaelhoweely.com . Pour créer un rapport complet et compréhensible pour un blog ou un site d'articles dans Google Data Studio, vous devrez utiliser les outils suivants:

  • Google Tag Manager
  • Google Analytics

Préparation de la création d'un rapport


Pour ce faire, allez dans GTM et sélectionnez dans les paramètres variables:

  • Seuil de profondeur de défilement;
  • Unités de profondeur de défilement;
  • Sens de défilement.



Ensuite, allez dans la section "Déclencheurs" et créez-en un nouveau avec le type "Profondeur de défilement".



Dans les paramètres de déclenchement, sélectionnez "Profondeur de défilement vertical" et les pourcentages (nécessaires pour vous).



Créez maintenant une nouvelle balise et sélectionnez Google Analytics comme type.



Dans la balise, remplissez:

  • Type de suivi: événement;
  • Catégorie: Défilement ou page de défilement. Le nom est requis pour la navigation;
  • Action: {{Chemin de page}};
  • Libellé: {{Seuil de profondeur de défilement}}.
  • En tant que paramètre pour Google Analytics, nous utilisons l'ID Analytics souhaité.



En tant que déclencheur, sélectionnez le déclencheur «Faire défiler la page» créé précédemment.



Une fois la nouvelle version publiée, il sera possible de vérifier que les événements sont transmis à Google Analytics.



Création de rapports dans Google Data Studio


Afin de créer un rapport dans Data Studio, nous utilisons:

Un histogramme dans lequel:

  • Paramètre: étiquette d'événement;
  • Métrique: événements uniques.



Cochez également "Appliquer le filtre" et cliquez sur "Ajouter un filtre". Ici, nous indiquons que la «catégorie d'événement» contient «scroll» (ou un autre nom donné lors de la création du tag dans GTM).





Pour la commodité de la visualisation, il est recommandé de sélectionner «Étiquettes de données» dans les styles d'histogramme.

Tableau:

  • Paramètre: nom de la page;
  • Métrique: pages vues.



Il vaut également la peine d'ajouter un filtre comme plage de dates.

Rapport prêt


Maintenant, en choisissant la période de date souhaitée, nous voyons:

  • Le nom de la page et le nombre de vues pour cette plage de dates;
  • En cliquant sur la page souhaitée, vous pouvez voir dans quelle partie les utilisateurs lisent le plus.

Un exemple de ce rapport peut être consulté dans Data Studio en cliquant sur le lien .
