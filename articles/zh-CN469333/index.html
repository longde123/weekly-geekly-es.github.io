有时，为了确定硬盘快要死了，您需要对应用程序软件检测到的错误做出响应

我规定必须始终安装所有OS更新，可能会稍有延迟（最多一周），我还安装了主板制造商提供的特殊应用程序（APP Center），用于监视驱动程序和某些程序的更新，并且还包括诸如以下的有用程序： ，例如，调整风扇的操作或更新UEFI（在2018年，由于英特尔处理器中发现的问题，我不得不更新3次）。 在此应用程序中，直到最近，一切都按预期工作。 每天我都会检查是否有任何新内容，通常会立即进行设置。 直到最近。 当声卡软件更新到达时（它完全卸载了经典应用程序并安装了一个新应用程序），我稍等片刻，决定安装。 APP Center的一个小问题是，它提供了安装或更新许多我不需要的程序的功能，例如防病毒程序，而购买的程序对我来说非常令人满意，因此，在升级之前，我总是先浏览列表并删除不必要的daws。 就是这个时候。 但是出了点问题。

通常，通常有两个选项卡，在底部有一个“刷新”按钮，另一个是“安装”，每个“取消”按钮。 一次只显示一个标签的内容。 那天出现了三分之一。 现在可以返回事件发生的那一刻，然后在此处发布屏幕快照，然后返回到当前的那一刻（尽管格式是旧的，我还没有删除备份），但是我尝试不使用Acronis True Image，而无需还原-甚至尝试先解决问题。用通常的方法解决。 更重要的是，由于希望拍摄屏幕快照，我不会跳入系统的各种状态（尽管恢复速度很快，并且直接从工作的操作系统开始，但只需要重启即可）。 因此，我依靠记忆。 她可以让您失望：那天我几次浏览了标签页。 看起来，他确实做到了一切都是合乎逻辑的，但是他没有考虑到任何事情。

在更新之前，我没有看第三个选项卡，该应用程序还在驱动程序更新网卡和HDD的相对位置放置了daw。 如果我想更新它们，那么就不要在这一天。 因此，当我看到意外情况时，我立即停止了更新。 好吧，由于我看到已经安装了一些驱动程序，而没有安装一些驱动程序，所以我使用Acronis True Image 2019回滚到系统的早晨副本，有时在这种情况下会这样做以便明天进行更新，但与此同时还要做其他事情。 我整天工作都没有问题，第二天早上，由于硬盘错误，Acronis没有创建备份副本。 显而易见的解决方案是运行chkdsk。 验证失败。 然后chkdsk c：/ scan / v / r。 Chrome缓存中存在错误，为了对其进行修复，必须重新启动。 我同意了 这是最后要做的事情。

在启动时，系统陷入BSOD中，并开始重新启动到下一个BSOD，依此类推。 桌面无法连接。 没问题：我的备份存储在安装了Acronis Survival Kit的外部USB HDD上，在启动时，该USB硬盘使您可以开始在WinRE中恢复备份。 失败 该程序建议从Linux Acronis闪存驱动器启动并继续恢复。 但是我没有事先做这个闪存驱动器。 但是有一个RecoveryDrive闪存驱动器，它从中启动（为了使用其他人的计算机，我必须等待-那是清晨），然后在计算机上放置一个干净的系统。 安装了True Image，并使用该软件的Linux版本创建了可启动的USB闪存驱动器。

我在事件发生前的晚上恢复了系统的副本，并再次检查了硬盘驱动器-相同的错误。 我在《系统》杂志上翻找，发现问题很可能出在一个文件系统过滤器中（加载后立即发生错误，并且日期过长）。 这次，APP Center（此副本中）不再提供升级，因此它恢复到事件发生前的最后一个副本。 成功更新所有驱动程序，再次检查所有内容。 另外安装了smartmontools。 事实证明，在此期间，SMART设法重新分配了超过2200个扇区（几乎100％），通常，HDD已损坏，尽管问题不在于磁盘表面缺陷，而在于软件。 但至少它起作用了。 直到我购买了Acronis True Image 2020的订阅。

更新并再次无法复制。 更准确地说，有时是制作副本，有时不是由于HDD错误。 编写了Acronis技术支持（常规工具无法识别问题），并继续尝试查找这次没有解决的问题。 我还用chkdsk检查了隐藏的驱动器-没问题。 我来到了SeaTools-HDD制造商的软件，专门用于这种情况。 因此它揭示了问题-DST测试未通过。 建议创建一个Linux闪存驱动器并通过从中启动来继续修复。 我做到了 DST也没有结束，进行了全面测试，然后进行了短暂维修。 此后，DST通过了。 并且Acronis True Image 2020开始毫无问题地进行复印。 但是问题仍然存在。

修复后，smartctl显示重新分配的扇区数没有增加，chkdsk（包括隐藏磁盘）的坏扇区数也没有增加，看来问题尚未完全解决，并且可能在将来出现。 这意味着您可能必须更换硬盘。 顺便说一句，Acronis支持在星期一答复（我在星期六发送了请求），但我已经解决了问题（尽管在俄罗斯卢布无法购买Acronis True Image，但仍然有个人支持，自2010年以来我就习惯了我不会更换软件，我现在将以欧元付款）。

也许数据恢复专家将能够澄清我的操作和结论是否正确。 我必须马上说，现在在Windows 10中恢复已知工作的VSS副本并不能为其进一步的正常运行提供信心。
