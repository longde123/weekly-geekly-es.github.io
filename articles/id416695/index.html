Dukungan untuk vSphere 6.7 dan fitur-fitur lain dari Pembaruan & Replikasi Veeam 9.5 9.5 Pembaruan 3a

Seingat Anda, Veeam Backup & Replication 9.5 Update 3 sebelumnya dirilis pada Desember 2017. Sejak saat itu, produsen platform dan aplikasi tempat solusi ini bekerja juga telah merilis sejumlah versi baru - tentu saja, Veeam wajib mendukungnya, yang berhasil dikelola oleh teknisi kami.

Hari ini saya akan memberikan Anda sesuatu yang baru - Pembaruan 3a (nomor ini diberikan karena sebelum akhir tahun ini pembaruan serius lainnya direncanakan, itu akan menjadi Pembaruan 4).
Untuk detailnya, selamat datang di kucing.



Jadi, apa yang bisa menyenangkan Anda di Pembaruan 3a:

  • Dukungan untuk VMware vSphere 6.7 . Perhatikan bahwa NBD / NBDSSL asinkron sekarang didukung - ini dapat meningkatkan kinerja baca untuk mode transportasi jaringan jika Anda bekerja dengan ESXi 6.7. Selain itu, versi baru (HTML5) dari vSphere Web Client plugin untuk Veeam Backup & Replication telah dirilis.
  • Dukungan pra-deklarasi untuk VMware vSphere 6.5 U2 .
    Penting! Meskipun Veeam mencoba untuk mendukung semua kemampuan yang dinyatakan dari versi ini, harus diingat bahwa kesalahan kritis ditemukan dalam kode ESXi 6.5 U2, karena itu vSphere API berhenti bekerja jika CPU dan sumber daya lainnya kelebihan beban cukup tinggi. Saat ini, masalah dikonfirmasi di sisi VMware, pekerjaan sedang dilakukan pada penelitian dan eliminasi. Jika Anda berteman dengan bahasa Inggris, akan lebih bijaksana untuk berlangganan ke topik yang sesuai di forum kami untuk mengikuti perkembangan.
  • Dukungan untuk VMware vCloud Director 9.1.
  • Dukungan untuk VMware Cloud pada AWS versi 1.3 , termasuk kompatibilitas yang telah diumumkan sebelumnya dengan pembaruan di masa mendatang (karena mereka menghapus persyaratan untuk mencocokkan versi API). Selain itu, mesin virtual dengan disk dinamis sekarang didukung.

Juga diterapkan:

  • Dukungan untuk Microsoft Windows Server 1803 sebagai OS tamu mesin virtual. Selain itu, server semacam itu dapat digunakan untuk menginstal komponen Veeam Backup & Replication jarak jauh, serta pengontrol domain yang file database AD-nya ( .dit ) seharusnya dipulihkan menggunakan Veeam Explorer untuk Microsoft Active Directory.
  • Dukungan untuk Microsoft Windows Hyper-V Server 1803 sebagai tuan rumah hypervisor. VM didukung dengan versi 8.3 perangkat keras virtual (perangkat keras virtual versi 8.3).
  • Dukungan untuk Pembaruan Microsoft Windows 10 April 2018 sebagai OS tamu mesin virtual. Selain itu, server seperti itu dapat digunakan untuk menginstal server Veeam Backup & Replication dan komponen solusi apa pun.

... dan peningkatan lainnya


  1. Dengan mengoptimalkan interaksi dengan memori sistem, dimungkinkan untuk meningkatkan kinerja mode Direct Storage Access (DirectSAN) dan Virtual Appliance (Hot Add).
  2. Dukungan yang diterapkan untuk pemulihan Pemulihan Langsung di Microsoft Azure untuk infrastruktur dengan koneksi melalui ExpressRoute dan VPN situs-ke-situs (dalam hal ini, proxy dan alat pembantu hanya dapat memiliki alamat IP pribadi). Untuk mengaktifkan fitur ini, Anda perlu membuat kunci registri AzurePreferPrivateIpAddressesForProxyandLinuxAppliance (DWORD, 1) di cabang Pencadangan dan Replikasi HKLM \ SOFTWARE \ Veeam \ Veeam di server cadangan Veeam.
  3. Saat memulihkan file Linux, sekarang jika Anda memilih Salin Ke (menyalin file yang dipulihkan ke lokasi yang ditentukan), SUID dan SGID disimpan. Juga menambahkan dukungan untuk volume Btrf yang terletak di LVM.
  4. Mereka menyederhanakan pekerjaan dengan Universal Storage API - sekarang Anda tidak perlu menginstal plugins pada setiap mesin dengan Veeam backup console, cukup instal di server.

Anda dapat membaca tentang peningkatan lain dan menyelesaikan masalah dari versi sebelumnya di sini (dalam bahasa Inggris).

Penting! Sebelum menginstal pembaruan, periksa nomor build Veeam Backup & Replication yang Anda kerjakan (di menu utama, pilih Help | About ). Ini harus menjadi salah satu dari angka-angka berikut: 9.5.0.580, 9.5.0.711, 9.5.0.802, 9.5.0.823, 9.5.0.1038, 9.5.0.1335, 9.5.0.1335, 9.5.0.1536.
Setelah menginstal pembaruan, nomor build akan berubah menjadi 9.5.0.1922.

Jika Anda bekerja dengan Veeam Backup & Replication 9.0, maka pertama-tama Anda harus meningkatkan ke 9.5 dengan mengunduh 9.5 ISO dari sini dan mengikuti instruksi .
