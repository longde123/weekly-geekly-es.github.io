Unidade de controle deslizante de vídeo nº 2

Oi, habravchane! Lembra-se do meu controle deslizante de vídeo motorizado de acessórios para móveis, réguas de construção e acrílico?)
Hoje vou mostrar uma coisa nova, um pouco mais simples que o projeto anterior. Desta vez sem trilhos, apenas a parte elétrica.
Leia abaixo!



...
Antecedentes.
Sua motorizado videoslayder eu coloquei no final de julho do ano passado, quando havia apenas um mês antes do estudo, e não havia uma situação de "nem lá nem aqui." Filmei apenas algumas cenas de seis segundos no meu controle deslizante e é isso. Em seguida, o primeiro curso na universidade, uma infusão na vida estudantil, etc. Em geral, eu não estava à altura do controle deslizante. Eu dirigi para casa no fim de semana. Se houver tempo, puxe um controle deslizante. Entrei na natureza algumas vezes para o teste. Mas não é disso que precisamos. Para fotografar beleza - você precisa ficar longe de carro e não em um dia de folga, mas por mais tempo)
outono, inverno, primavera ...
maio. De alguma forma, uma mensagem chega a mim: “Olá! Olhei para o seu artigo sobre um controle deslizante de vídeo motorizadona Gigtime, gostei! Estou envolvido em filmagens de vídeo e, por muito tempo, quero motorizar meu controle deslizante, mas não encontrei uma pessoa em Minsk que fabricasse as unidades para eles. Você vai ajudar?) "
" Claro! " Eu respondi.

Cheguei em casa com o cliente, olhei para o controle deslizante (industrial, sistema IGUS ), conversei sobre isso, discuti o que ele queria da unidade. Sergey, o cliente, queria motorizar apenas um carro deslizante, para iniciantes.
"Bom como você deseja", eu digo. - Pré-pagamento antecipado! 2-3 semanas, e eu trago. "

No começo, comecei a fabricar o alojamento da unidade de controle. Material, como no primeiro rascunho, em plexiglass. Bem provado! Durável, leve, perfeitamente cortado (serra com um dente pequeno) e processado.
Você pode colar!



Então ferro. Baseado no microcontrolador Arduino Nano .



O motor é acionado pelo EasyDriver .



Motor de passo Nema 17 .



Controlamos a velocidade com um potenciômetro .



Energia organizada com base no tamanho das baterias 18650 .



Desta vez, o conector para conectar o mecanismo foi atualizado . Em vez do DIN soviético usado MIC 4 PIN .











Funcional .
O cliente solicitou apenas um modo de operação - o carro se move infinitamente para frente e para trás, a velocidade pode ser alterada com um potenciômetro a qualquer momento. Paramos o carro com a chave seletora quando necessário.

Esboço:
#define xy A0
int Distance = 0;  
int x = 0;
void setup() {                
  pinMode(2, OUTPUT);     
  pinMode(3, OUTPUT);
  digitalWrite(2, LOW);
  digitalWrite(3, LOW);
  pinMode(xy, INPUT);
  
}

void loop() 
{
   x = analogRead(xy); 
x = map(x, 20, 700, 10, 4000); 

  digitalWrite(3, HIGH);
  delayMicroseconds(x);          
  digitalWrite(3, LOW); 
  delayMicroseconds(x);
  Distance = Distance + 1;   
  
  
 if (Distance == 1600*13.8)
  {
  
    if (digitalRead(2) == LOW)
    {
      digitalWrite(2, HIGH);
    }
    else
    {
      digitalWrite(2, LOW);
    }

    Distance = 0;
   
    delay(2500);
  }
}
