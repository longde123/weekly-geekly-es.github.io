GitHub Desktop 2.2：新功能



使用Git的任何人都知道他有很酷的学习路线。 我们从开发人员那里了解到，无论是同事，教授，朋友，还是YouTube视频，大多数人都倾向于向熟人学习。 在GitHub Desktop 2.2中，我们在Git和GitHub上添加了交互式教程的第一个版本，可以成为您的朋友并帮助您入门。 如果您不熟悉 Desktop，可以在desktop.github.com上下载并尝试该教程。 好吧，细节已经删减了。

开始使用


首先，我们将帮助您完成两个主要阶段：创建存储库和连接编辑器。 首次打开程序时，将显示一个欢迎页面，其中包含新选项“创建培训资料库”。 使用此功能，将创建一个培训资料库，指导您了解使用GitHub Desktop使用Git的基本概念。



Git和GitHub入门需要大量工具。 其中最重要的是您的代码编辑器。 在本教程的第一阶段，如果尚未安装编辑器，则将要求您安装该编辑器。



探索GitHub


接下来，我们将告诉您如何使用GitHub Desktop在本地更改代码并将您的工作提交到GitHub。 您将创建一个新分支，更改文件，提交，将其提交到GitHub并打开您的第一个拉取请求。

我们还听说，新用户最初会在Git，GitHub和GitHub Desktop之间感到困惑。 我们将在手册中查看这些差异，并确保它们支持通常的解释。



继续您自己的项目。


在GitHub Desktop 1.6中，我们根据存储库的状态引入了“可能的后续步骤”功能。 现在，您已经完成了本教程，我们还提供了以下步骤：在GitHub上浏览您可能想要贡献的项目，创建一个新项目，或将现有项目添加到Desktop。 我们希望GitHub Desktop是一个可以阐明您下一步工作的工具，无论您是正在工作还是只是新手开发人员。



接下来是什么？


在GitHub Desktop 2.2上工作，我们使产品对初学者更易访问。 我们将根据您的反馈意见改进本教程，并将继续改善GitHub与您的本地计算机之间的连接。 如果您想开始创建某些东西，但不知道该怎么做，可以将GitHub Desktop视为可以帮助您入门的朋友。
