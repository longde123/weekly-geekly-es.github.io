Mencari obat terbaik

Dalam artikel ini, saya akan menunjukkan kepada Anda bagaimana saya bertemu Quest Netvault Backup. Tentang Netvault Backup, ketika perangkat lunak ini masih dimiliki oleh Dell, saya sudah mendengar banyak ulasan positif, tetapi saya tidak harus "menyentuhnya" dengan tangan saya.


Quest Software, juga dikenal sebagai Quest, adalah perusahaan perangkat lunak berbasis di California dengan 53 kantor di 24 negara. Perusahaan ini didirikan pada tahun 1987. Perusahaan ini dikenal dengan perangkat lunaknya yang digunakan oleh para profesional di bidang basis data, dalam manajemen cloud, dalam keamanan informasi, untuk analisis data, untuk pencadangan dan pemulihan. Quest Software diakuisisi oleh Dell pada tahun 2012. Pada 1 November 2016, penjualan selesai dan perusahaan memulai kembali sebagai Quest Software.

Ketat mengenal Quest Netvault belum lama ini. Dalam salah satu proyek, Pelanggan meminta untuk menemukan solusi yang murah dan optimal untuk melindungi infrastrukturnya. Pelanggan sedang mempertimbangkan berbagai perangkat lunak cadangan, salah satu solusinya adalah Quest Netvault Backup. Berdasarkan hasil pengujian, diberikan parameter penting bagi Pelanggan (beberapa di antaranya diberikan pada akhir artikel), Quest Netvault Backup dipilih.

Selain persyaratan dasar, Pelanggan ingin agar perangkat lunak diinstal pada server yang menjalankan Linux. Persyaratan seperti itu tidak diterapkan oleh setiap perangkat lunak cadangan, tetapi Quest Netvault Backup dapat melakukan ini.

Baseline dan persyaratan


Tugas yang ditetapkan oleh Pelanggan adalah merancang sistem yang menyediakan cadangan data dalam jumlah 62 TB. Data ini terkandung dalam sistem aplikasi seperti SAP, Microsoft SQL, PostgreSQL, MariaDB, Microsoft Exchange, Microsoft SharePoint, dll. Sistem aplikasi ini berjalan pada server fisik dan virtual yang menjalankan sistem operasi keluarga Microsoft Windows Server, Linux, dan FreeBSD. Lingkungan virtual dibangun di atas platform virtualisasi VMware vSphere. Infrastruktur terletak di satu situs.

Secara umum, infrastruktur Pelanggan ditunjukkan pada Gambar 1.1.



Gambar 1.1 - Infrastruktur Pelanggan

Analisis memeriksa kemampuan Quest Netvault Backup yang berlaku untuk infrastruktur pelanggan, yaitu: dalam hal melakukan pencadangan, pemulihan, pengelolaan, dan pemantauan data. Fungsi dan prinsip operasi yang khas praktis tidak berbeda dari perangkat lunak vendor lain. Karena itu, saya ingin memikirkan fitur Quest Netvault Backup, yang membedakannya dari alat cadangan lainnya.

Fitur menarik


Instalasi

Ukuran distribusi Quest Netvalt Backup hanya 254 megabita, yang memungkinkan Anda untuk menyebarkannya dengan cepat.

Plugin untuk platform dan tugas yang didukung diunduh secara terpisah, tetapi ini memiliki efek positif pada status target sistem, yang hanya akan memiliki fungsi yang diperlukan untuk melindungi infrastruktur tertentu dan tidak akan kelebihan beban dengan kemampuan yang tidak perlu.

Manajemen

Netvault dikelola melalui shell web WebUI. Login adalah dengan nama dan kata sandi.



Gambar 1.2 - Jendela masuk ke konsol manajemen

Koneksi ke konsol web dilakukan dari komputer mana saja di jaringan menggunakan browser.

WebUI menggunakan antarmuka yang sederhana dan ramah, administrasi tidak menyebabkan masalah, logika manajemen dapat diakses dan dimengerti, jika muncul pertanyaan, maka situs web vendor berisi dokumentasi produk terperinci.



Gambar 1.3 - Antarmuka WebUI

  • Kelola klien, perangkat penyimpanan, dan media

  • melakukan pencadangan dan pemulihan;

Gambar 1.5 - Memantau aktivitas perangkat

  • membuat dan melihat laporan.

Perangkat penyimpanan

Quest Netvault dengan mudah mengimplementasikan aturan penyimpanan 3-2-1, karena ia juga dapat bekerja dengan perangkat untuk penyimpanan online salinan cadangan (sistem penyimpanan disk), serta dengan perangkat untuk penyimpanan jangka panjang (perangkat deduplikasi, perpustakaan pita fisik, autoloaders, virtual tape libraries (VTL) dan shared virtual tape libraries (SVTL)). Cadangan yang dapat di alienasi dapat disimpan di cloud, di situs jarak jauh, atau di media yang bisa diasingkan (misalnya, tape drive).

Ketika bekerja dengan perangkat deduplicating, protokol RDA dan DD Boost khusus didukung. Menggunakan protokol ini:

  • mengurangi beban pada jaringan dan meningkatkan kinerja tugas cadangan, karena data dideduplikasi pada klien dan hanya blok yang diperlukan yang ditransfer. Misalnya, bekerja bersama Quest Qorestor melalui protokol RDA dapat mencapai kinerja hingga 20 terabyte per jam dan kompresi 20 hingga 1;
  • melindungi cadangan dari virus ransomware. Bahkan jika server cadangan itu sendiri terinfeksi dan dienkripsi, cadangannya akan tetap tak tersentuh. tautan .

Pelanggan

Quest Netvault Backup mendukung lebih dari tiga lusin platform dan aplikasi. Anda dapat berkenalan dengan daftar secara lebih rinci di situs web vendor menggunakan tautan (Gambar 1.6). Memeriksa kompatibilitas versi sistem yang dilindungi dengan Quest Netvault Backup dilakukan sesuai dengan dokumen resmi "Panduan Kompatibilitas Pencadangan Netvault Quest" yang terdapat di tautan .

Mendukung begitu banyak sistem memungkinkan Anda merancang solusi untuk infrastruktur tingkat perusahaan yang kompleks. Klien didistribusikan dalam bentuk plugin (mirip dengan vendor - agen lain), yang diinstal pada server. Akibatnya, data dilindungi menggunakan sistem tunggal dengan satu titik manajemen.



Gambar 1.6 - Daftar plugin

Setelah mengunduh plugin untuk platform ini, kami menempatkannya di folder bersama yang kami sambungkan ke Netvault dan kemudian menginstal plugin dari jarak jauh di server yang dilindungi.

Hasil Implementasi


Hasil dari proyek ini adalah sistem cadangan yang digunakan oleh Pelanggan berdasarkan perangkat lunak Quest Netvault Backup dengan arsitektur yang ditunjukkan pada Gambar 1.7.



Gambar 1.7 - Status sistem target

Semua komponen Cadangan Netvault digunakan pada server fisik dengan karakteristik berikut:

  • dua prosesor dengan masing-masing sepuluh inti;
  • 64 GB RAM;
  • dua hard drive SAS 300GB 10K (RAID1)
  • empat hard drive SAS 600GB 15K (RAID10);
  • HBA dengan dua port SAS eksternal;
  • dua port 10 gbps;
  • OS CentOS.

Cadangan online disimpan di Quest Qorestor Standard (back 150TB). Pekerjaan dengan Qorestor dilakukan sesuai dengan protokol RDA. Koefisien deduplikasi pada Qorestor pada akhir operasi uji coba sistem adalah 14,7 banding 1.

Untuk penyimpanan jangka panjang, tape library dengan empat drive standar LTO-7 digunakan. Perpustakaan rekaman terhubung ke server cadangan melalui SAS. Secara berkala, kartrij dibuang dan dipindahkan untuk penyimpanan ke salah satu cabang terpencil.

Semua plugin yang diperlukan diunduh dan ditempatkan di folder jaringan untuk instalasi jarak jauh. Waktu penyebaran dan konfigurasi untuk sistem ini adalah sembilan hari.
