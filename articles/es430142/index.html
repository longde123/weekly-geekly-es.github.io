¿Su Windows es tan viejo?

Como muestra una acalorada discusión de los últimos artículos sobre Habré en los sistemas Windows, una gran cantidad de usuarios avanzados y no muy usuarios dan su preferencia a Windows 7 "obsoleto" y no tienen prisa por cambiar a Windows 10. Los motivos pueden ser muy diferentes: desde la insatisfacción con las políticas de privacidad y mucha telemetría hasta hábitos comunes y falta de voluntad para seguir el ejemplo de los vendedores de Microsoft.

En este artículo quiero considerar el caso en el que el desarrollador requiere Windows 7, e incluso versiones anteriores, para obtener una solución de calidad a sus responsabilidades laborales. No es ningún secreto que la participación de Windows 7 sigue siendo muy grande en el sector corporativo, y dado que Microsoft extendió su soporte extendido hasta 2023, es poco probable que algo cambie drásticamente en el futuro cercano. Y esto sin mencionar el hecho de que existen sistemas bastante antiguos con Server 2003 y 2008, que ejecutan algún tipo de software antiguo, pero extremadamente necesario, que nadie necesita portar sobre la base del principio "funciona - no toque". Sin embargo, actualizar su sistema operativo "desactualizado" es mucho más rápido y fácil de lo que parece.

En el momento en que se requiere probar nuevos desarrollos o complementar la funcionalidad de los existentes, surge la cuestión de instalar sistemas operativos de prueba para probar la operatividad, que no es la mejor idea para llevar a cabo en sistemas de trabajo reales. Y luego resulta que incluso si tiene una imagen funcional o un disco de instalación, Windows 7 SP 1 del modelo 2011 está lejos de ser lo mismo que Windows 7 en las máquinas de los usuarios que han recibido cientos de actualizaciones de seguridad y estabilidad en la última década.

En el caso de que intente instalar actualizaciones utilizando herramientas normales en algunos Windows Server 2008 o Windows 7 SP1, encontrará que el sistema de actualización no funciona y busca actualizaciones sin cesar. Mucha gente sabe que esto está sucediendo, incluso debido a la pila de servicios obsoletos del sistema operativo.

Consideremos el método de máxima sencillez y lo antes posible para actualizar completamente Windows 7 a su estado actual, que incluye, opcionalmente, sin conectar la máquina virtual a Internet, todas las acciones son similares para Windows Server 2008 R2 SP1, Datacenter y Enterprise. Necesitaremos una instalación limpia de Windows 7 SP1 del tamaño de bit apropiado, hecha de una imagen de MSDN o el DVD de instalación en su sistema de virtualización favorito o una máquina real. La entrada y activación de claves en esta etapa se puede omitir y realizar más tarde.
Para aquellos que no tienen una imagen oficial en los zagashniks y usan los servicios de intercambio de archivos para descargarlos, no olviden verificar la suma de verificación del archivo con los disponibles para esta imagen en el sitio web de Microsoft, pueden hacerlo utilizando la versión de prueba de Total Commander, por ejemplo.

Después de instalar con éxito el sistema operativo en sí, necesitaremos descargar de microsoft.com solo tres actualizaciones para el sistema operativo con la profundidad de bits correspondiente. Primero, necesitamos KB3177467 Actualización de la pila de servicio para Windows 7 SP1 y Windows Server 2008 R2 SP1 - Actualización de la pila de servicio de octubre de 2018. Después de instalar la SSU, debe instalar el paquete acumulativo de actualizaciones KB3125574 y reiniciar. Si la instalación falla, es posible que deba preinstalar KB3020369.

El último paso es instalar el paquete acumulativo mensual, cuya versión actual KB4462923 está disponible aquí . En el futuro, la última versión de la actualización acumulativa se encuentra en el motor de búsqueda mediante la consulta de actualización mensual acumulativa de Windows 7 de noviembre, donde noviembre es el mes del paquete de actualización. Un nuevo paquete está disponible en Europa a partir del segundo miércoles de cada mes; la instalación de paquetes anteriores no es, en principio, necesaria.

Por lo tanto, después de instalar estas tres actualizaciones, Windows 7 se actualiza por completo con todas las correcciones de estabilidad y seguridad en este momento. Todas las acciones, excepto los paquetes de servicio de descarga preliminar, se pueden realizar sin conexión a Internet. Además, actualizar la pila de servicios le permitirá recibir nuevas actualizaciones a través de Windows Update rápidamente y sin problemas. Tenga en cuenta que las actualizaciones para otro software de Microsoft, como Office y Dot NET Framework, se realizan por separado.
Como una pequeña fracción de paranoia. Si encuentra el Programa de mejora de la calidad del software de Windows en la barra de búsqueda, prefiero deshabilitar este elemento, ya que Microsoft no planea desarrollar más Windows 7. Esta es probablemente la razón por la que también debería eliminar el servicio diagtrack.
