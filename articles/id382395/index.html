"Hampir pintar" di bawah lantai memanaskan Arduino

Malam saya yang biasa adalah pertemuan komputer. Pada malam yang dingin, sering kali ada keinginan untuk membuat tempat peristirahatan saya lebih nyaman. Lebih tepatnya, dari waktu ke waktu terasa dingin di kaki. Ide-idenya berbeda, sampai membeli sandal USB yang dipanaskan. Namun, mereka semua tampak konyol bagi saya dan tersapu. Dan kemudian suatu hari, menonton saluran YouTube dari salah satu pecinta Arduino, saya menemukan video yang berbicara tentang film inframerah. Melihat film ini, saya segera menyadari: "Itulah yang saya butuhkan!"

Proyek ini dapat dijelaskan secara singkat sebagai berikut: Saya meletakkan sepotong film inframerah di bawah lapisan parket tambahan, menambahkannya sebuah sistem kontrol otomatis menggunakan Arduino, beberapa sensor dan VB.NET. Sekarang, secara berurutan, apa dan bagaimana itu terjadi.



Penolakan


Saya telah terlibat dalam proyek semacam ini selama beberapa tahun, saya melakukannya sendiri. Saya lakukan untuk melakukannya: prosesnya sendiri jauh lebih menarik bagi saya daripada solusi akhir. Itulah sebabnya deskripsi proses dan eksperimen diberikan di bawah ini dengan perincian terperinci. Penggunaan elemen kadang-kadang tidak sepenuhnya dibenarkan dari sudut pandang keuangan - saya mengerti itu. Secara berkala, saya mengubah sesuatu (dalam pendekatan, dalam elemen), tetapi saya pasti tidak akan beralih ke solusi yang sudah jadi, karena ini hanya akan tidak menarik.

Mengapa "hampir pintar"? Saya tidak akan menyebut pengukuran suhu dan pengatur waktu "pintar". Sebagai langkah ke depan, ada ide untuk meningkatkan algoritma kontrol dengan menambahkan fungsi pembelajaran. Maka proyek ini bisa disebut sebaliknya.

Mengapa publikasi ini:
  • menarik untuk mendapat kritik / ide yang membangun
  • memperkenalkan komunitas untuk film inframerah

Jadi ayo pergi!

Setelah memutuskan bahwa saya harus mempersiapkan sebelum tindakan, saya pergi ke mesin pencari untuk mencari informasi dan umpan balik. Komentarnya saling bertentangan. Seseorang menyebut film itu elemen pemanas yang ideal dan mengatakan bahwa film itu berhasil memanaskan seluruh rumah, seseorang mengeluh bahwa itu sama sekali tidak berguna dan meyakinkan bahwa itu semua adalah "perceraian". Penerapan bagian "pintar"


Sambil menunggu bungkusan itu, di kepala saya ada gambaran yang jelas tentang bagaimana lantai hangat saya akan bekerja. Karena ini bukan proyek pertama saya, saya memutuskan untuk memanfaatkan secara maksimal perkembangan yang ada. Bahkan, saya memutuskan untuk menerapkan algoritma dan skema yang sama untuk mengendalikan suhu lantai seperti halnya untuk kontrol cahaya otomatis.

Bandingkan aturan dasar algoritma:
Light
  1. Kami menyalakan lampu jika tingkat iluminasi lebih rendah dari yang ditetapkan
  2. Kami menghidupkan relai untuk jangka waktu tertentu
  3. Kami menghidupkan relai hanya jika ada informasi dari sensor gerak

Pemanas lantai
  1. Kami menyalakan pemanas jika tingkat suhu lebih rendah dari yang ditetapkan
  2. Kami menghidupkan relai untuk jangka waktu tertentu
  3. Kami menghidupkan relai hanya jika ada informasi dari sensor gerak


Semacam diagram blok dari seluruh solusi. Saya meminta Anda untuk tidak menilai skema secara ketat - saya membuatnya khusus untuk publikasi, sehingga metode koneksi jelas dan tidak repot dengan pemilihan ikon yang benar.


Relai daya lantai

Untuk manajemen daya, seikat dua papan digunakan.

Papan pertama menambah Arduino Nano:
  • Poin pemasangan di rak otomatisasi saya (4 salib di setiap sisi)
  • Konektor RJ-45 untuk port input / output (tidak ada pembicaraan tentang jaringan - Saya hanya menggunakan konektor ini untuk berpindah)
  • Input untuk 12V (jika digunakan dalam papan plug-in)
  • Dua resistensi 10 kÎ© untuk menghubungkan sensor analog



Papan kedua:
  • Poin pemasangan di rak otomatisasi saya (4 salib di setiap sisi)
  • Berisi pemicu JK untuk mengingat perintah terakhir
  • L298D power bridge untuk mentransfer peningkatan arus ke koil relay
  • 5V atau 12V relay, tergantung pada versinya
  • Beberapa LED untuk menampilkan status



Saya akan menjawab pertanyaan sebelumnya yang bisa muncul setelah membiasakan diri dengan papan.
  • Mengapa dua papan? Implementasinya disalin dari kontrol lampu yang ada, di mana lebih nyaman bagi saya. Jika saya melakukannya dari awal - kemungkinan besar papan akan menjadi satu.
  • Mengapa memicu? Memang, untuk solusi ini, menurut saya itu mubazir. Hanya di salah satu versi sistem sebelumnya, pengontrol kontrol tidak terhubung ke jembatan L298D secara permanen, tetapi terhubung oleh multiplexer. Karena itu, ada kebutuhan untuk mengingat keadaan yang sudah mapan.

Sensor suhu dan gerak

Saya tidak mulai membuat papan terpisah untuk sensor gerak dan suhu. Sensor gerak datang dengan kontak yang nyaman dan memasangnya di papan tambahan tidak masuk akal. Menghubungkan sensor suhu juga bukan tugas yang sulit - hanya diperlukan satu hambatan tambahan. Sebagai hasilnya, kita dapat mengatakan "pada twist", saya mengumpulkan bagian dengan sensor.

Sensor suhu didorong di dalam kepang kabel CAT5, karena memiliki kontak yang sangat tipis dan tampak sangat rapuh saat disentuh.
