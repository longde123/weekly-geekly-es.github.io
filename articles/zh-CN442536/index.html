将外部邮箱连接到Zimbra

很少有大型企业可以夸耀其最初的规模。 许多大型零售连锁店，集成商，开发商和ISP都是从小型办公室和数十名员工开始他们的业务，这些员工为公司的未来成功而努力。 当然，在这样的小型企业中，实现自己的邮件服务器或协作系统的问题极为罕见，通常在诸如Yandex邮件或gmail之类的公共电子邮件服务中创建员工邮箱。 但是，当企业发展到某个阶段并需要基于Zimbra创建自己的邮件服务器时，就会出现将旧邮箱连接到新帐户的问题。



由于企业的发展几乎总是意味着在Internet上进行积极的广告活动，分发指示电子邮件的名片以及在各种目录中添加邮寄地址，因此即使更改当前邮寄地址数年后，来自潜在客户或活跃客户的信件流也通常不会减弱。 当然，您可以使员工有义务每天检查旧邮箱，但是将两个邮箱合并为一个仍然是一个明智的决定。 除了节省员工的时间和精力之外，该解决方案还可以帮助您快速完全移至新邮箱，因为现在每个字母的答复都将来自新的电子邮件地址。

乍一看，将外部邮箱连接到Zimbra的最简单方法似乎是设置从旧帐户到新帐户的邮件转发。 但是，这种方法有许多问题。 特别是，由于Zimbra内置了垃圾邮件过滤器和防病毒软件，因此字母或某些附件可能根本无法到达新地址。 另外，某些服务可能根本不支持电子邮件的发送，而在如此重要的问题中完全依靠它们也不是一个好主意。

因此，最合适的任务是将接收来自外部邮箱的邮件的任务分配给您控制的服务器。 但是，这是一个小问题，因为Zimbra Collaboration Suite中没有用于处理外部邮箱的内置实用程序。 但是，这并不是什么大问题，因为Zimbra可以与第三方程序和实用程序完美配合。 这就是为什么要从外部地址接收邮件的原因，您可以使用fetchmail程序，该程序可在大多数Linux发行版的存储库中找到。

软件包管理器完成fetchmail的安装后，有必要在/ etc / default / fetchmail文件中生成以下格式，并在必要时取消注释行START_DAEMON = yes并设置守护程序600 。 在第一行，我们允许fetchmail作为系统服务运行，在第二行，我们设置访问外部邮箱的时间间隔（以秒为单位）。 在这种情况下，此参数为600秒，这意味着将每10分钟自动检查一次邮箱。 可以增加此间隔以减少服务器上的负载，或者如果希望字母尽快到达，则可以减小此间隔。

之后，仅保留在fetchmail中指定用于连接到外部邮箱的数据。 例如，假设该实用程序将连接到manager@gmail.com邮箱，然后将邮件转发到manager@company.ru 。 为此，我们需要首先创建fetchmail配置文件，然后对其进行一些更改。 您可以使用cp /usr/share/doc/fetchmail/examples/fetchmailrc.example / etc / fetchmailrc命令执行此操作

之后，在任何文本编辑器中打开文件/ etc / fetchmailrc并在文件末尾设置用于连接到外部邮箱的参数，即我们外部邮箱的授权数据： 

poll imap.gmail.com protocol IMAP user manager@gmail.com is manager@company.ru here password ********* keep ssl
之后，仅需通过依次输入服务fetchmail stop和service fetchmail start命令来重新启动fetchmail，然后通过将测试字母发送到外部邮箱来测试服务。 因此，即使在企业中引入了自己的Zimbra邮件服务器之后，也可以使公司不必将邮件地址的更改通知客户。
