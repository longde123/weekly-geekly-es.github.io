Dari buruh ke pemrogram php. Karir pengembangan yang tidak biasa



Hari ini kami menerbitkan kisah siswa GeekBrains Leonid Khodyrev ( leonidhodirev ), ia berusia 24 tahun. Jalannya di TI berbeda dari cerita yang diterbitkan sebelumnya bahwa Leonid segera mulai belajar PHP segera setelah tentara, yang akhirnya membantunya menemukan pekerjaan yang baik.

Kisah karier saya mungkin berbeda dari yang lain. Saya membaca kisah karir perwakilan TI, dan dalam kebanyakan kasus seseorang dengan percaya diri maju, melakukan semua atau hampir segalanya untuk mencapai tujuan mereka. Tidak demikian halnya dengan saya - saya sama sekali tidak tahu siapa yang saya inginkan dan tidak membuat rencana untuk masa depan. Lebih atau kurang serius, saya mulai memikirkan hal ini ketika saya kembali dari ketentaraan. Tapi mari kita selesaikan.



Pelayan, pemuat, dan asisten pengacara sebagai awal karier


Saya mulai bekerja lebih awal, "spesialisasi" pertama saya adalah distribusi selebaran. Mereka memberi saya banyak kertas, saya membagikan segalanya, tetapi tidak menerima uang. Namun demikian, pengalaman itu bermanfaat - saya mulai mengerti apa yang mungkin saya temui.

Kemudian ia bekerja sebagai loader, pelayan, melakukan berbagai tugas di acara-acara lapangan, menggabungkan ini dengan studi. Dia belajar di perguruan tinggi, pada saat yang sama menguasai topik membuat situs. Saya membuat situs web sederhana pada CMS populer, dan saya menyukainya. Tapi tetap saja, saya mengikuti arus, tidak benar-benar memikirkan apa yang saya butuhkan dalam hidup.

Nah, kemudian mereka membawa saya menjadi tentara, berkat yang saya lihat seluruh negeri. Sudah menjadi tentara, saya memikirkan apa yang ingin saya lakukan di masa depan. Mengingat pengalaman saya dengan situs, saya memutuskan bahwa akan menarik bagi saya untuk bekerja di bidang ini. Dan di ketentaraan, saya mulai mencari kemungkinan belajar jarak jauh. Kursus pengembangan web GeekBrains menarik perhatian saya, yang saya hentikan. Sejauh yang saya ingat, maka saya hanya mengarahkan kunci "pemrograman" atau "pengajaran pemrograman" ke dalam pencarian, melihat situs web kursus, meninggalkan permintaan. Seorang manajer menelepon saya dan saya mulai bertanya secara sistematis tentang segalanya.

Tentu saja, tidak mungkin untuk belajar di tentara, dan tidak ada cara khusus, jadi saya menunda studi saya untuk masa depan.

Hasil dalam IT


Setelah saya keluar, tidak ada uang yang ditambahkan. Untuk memulai pelatihan, saya harus kembali ke pekerjaan pelayan sebelumnya. Ketika saya menerima gaji, saya membeli kursus dan mulai. Sayangnya, menjadi jelas bahwa bekerja sebagai pelayan penuh waktu membutuhkan banyak waktu, yang tidak cukup untuk dipelajari. Sebuah solusi ditemukan dengan cepat - ia mulai membantu seorang pengacara dengan dokumen, dan di "musim tinggi" ia pergi bekerja sebagai pelayan.

Sayangnya, belajar itu sulit, saya berhenti belajar tiga kali. Tetapi kemudian saya menyadari bahwa ini tidak dapat dilanjutkan, pelayan itu baik, tetapi itu jauh lebih penting. Karena itu, saya mengambil waktu istirahat dengan pekerjaan dan sepenuhnya mengabdikan diri untuk belajar. Segera saya menyadari bahwa saya tidak menyukainya, tetapi benar-benar menyukainya. Beberapa saat kemudian, pesanan pertama untuk membuat situs mulai muncul, jadi selain kesenangan, pelajaran ini mulai menghasilkan lebih banyak uang. Entah bagaimana saya mendapati diri saya berpikir bahwa saya melakukan apa yang saya suka, dan mereka juga membayar saya untuk itu! Saya memutuskan masa depan saya saat ini.

Ngomong-ngomong, selama pelatihan, dalam praktik, saya mengembangkan proyek yang sangat serius untuk diri saya sendiri - sistem manajemen konten. Saya tidak hanya menulisnya, tetapi saya dapat menghubungkan beberapa situs. Baca lebih lanjut tentang proyek di sini .



Singkatnya, proyek ini adalah platform yang nyaman bagi pengguna, yang dapat dengan mudah ditingkatkan dengan mengintegrasikan dengan berbagai layanan yang Anda mungkin perlu melakukan bisnis. Target pengguna - pengusaha dan webmaster. Bagi mereka, saya menulis ekstensi "Shop", yang memungkinkan Anda mengelola kategori barang, barang itu sendiri, properti mereka, memproses pesanan.

Ini adalah proyek serius pertama saya, yang dikembangkan menggunakan teknologi yang tidak kalah serius. Tentu saja, ketika Anda mengevaluasinya, jangan lupa bahwa saya mengembangkannya selama pelatihan.

Pekerjaan kantor baru


Saya sudah mengatakan di atas bahwa selama pelatihan saya melakukan pesanan untuk pengembangan situs web. Dan saya sangat menyukainya - bahkan saya tidak ingin bekerja di kantor. Tetapi kemudian saya mulai menyadari bahwa saya membutuhkan pengalaman sebagai tim, karena sebagian besar pengembang pada satu titik atau lain dalam karier mereka mendapatkan pekerjaan formal. Saya memutuskan untuk melakukan ini.

Seperti yang saya ingat sekarang, pada hari Senin pagi saya membuka hh.ru, mengunggah resume, menambahkan sertifikat dan membuat akun publik. Kemudian dia mencari majikan yang paling dekat dengan rumah saya (dan saya tinggal di Moskow) dan mulai mengirim resume.

Secara harfiah satu jam kemudian perusahaan saya tertarik menjawab. Saya ditawari pada hari yang sama untuk wawancara, dan saya melakukannya. Saya perhatikan bahwa tidak ada "tes stres" dan hal-hal aneh lainnya, tetapi saya masih sedikit gugup. Mereka mulai menginterogasi saya dengan ramah tentang tingkat pengetahuan, pengalaman kerja dan umumnya tentang segalanya.

Saya menjawab beberapa pertanyaan tidak seperti yang saya inginkan, tetapi mereka membawa saya. Benar, mereka membuat kami khawatir - pada awalnya mereka mengatakan akan menelepon kembali. Sebenarnya, mereka biasanya menjawab seperti itu ketika mereka tidak ingin mengambil kandidat. Tetapi saya sia-sia khawatir - bel berharga itu berbunyi setelah beberapa jam. Keesokan harinya, setelah mengumpulkan semua dokumen, saya pergi bekerja.

Saya segera dipenjara karena mendukung sistem reservasi online yang memungkinkan agen memesan hotel, transfer, dll. Saya memastikan bahwa semuanya berfungsi dengan baik, memodifikasi fungsionalitas dan menambahkan berbagai fitur (ada juga bug, di mana tanpa ini).

Contoh dari apa yang telah dilakukan:

  • Modul pelaporan pemesanan;
  • Antarmuka platform yang ditingkatkan;
  • Sinkronisasi basis data dengan penyedia layanan;
  • Sistem loyalitas (kode promosi, poin);
  • Integrasi untuk wordpress.


Adapun alat, yang utama adalah:

  • Layout - html / css / js / jquery;
  • Database - pgsql;
  • Aplikasi yang ditulis dalam kerangka php yii2;
  • Perpustakaan pihak ketiga, saya menggunakan banyak yang berbeda.


Jika kita berbicara tentang penghasilan, itu jauh lebih tinggi daripada sebelumnya. Tapi di sini semuanya relatif, karena selama pelatihan saya mendapat sekitar 15.000 rubel sebulan. Terkadang tidak ada sama sekali, karena saya menerima pesanan hanya dari kenalan yang membutuhkan situs.

Tidak ada yang dapat membandingkan kondisi kerja dengan baik - jelas bahwa mereka jauh lebih baik daripada yang saya miliki sebagai buruh atau pelayan. Perjalanan ke tempat kerja hanya membutuhkan waktu 25 menit, yang juga menyenangkan - lagipula, banyak penduduk ibukota menghabiskan lebih banyak waktu. Berbicara tentang Moskow, saya pindah ke ibu kota dari Zelenograd, tempat saya tinggal bersama orang tua saya. Dia pindah ke ibukota selama masa studinya, ketika dia terlibat dalam pembuatan situs sesuai pesanan. Di sini saya suka segalanya, saya tidak akan bergerak, tetapi saya berencana untuk melihat dunia.

Apa selanjutnya


Saya berencana untuk melanjutkan jalur pengembangan saya, karena saya mendapatkan kesenangan dari pekerjaan - inilah yang saya suka. Selain itu, tugas-tugas yang dulu terasa sulit bagi saya sekarang tidak menghadirkan kesulitan sama sekali. Karena itu, saya melakukan proyek yang lebih besar, bersukacita ketika semuanya berubah.

Saya terus belajar, karena beberapa topik yang perlu saya kerjakan bisa sulit dikuasai sendiri. Para guru membantu untuk memecahkan masalah bahkan setelah kursus utama selesai.

Dalam waktu dekat saya ingin belajar bahasa pemrograman baru dan belajar bahasa Inggris.

Kiat untuk memulai


Saya pernah membaca artikel tentang karier orang-orang TI, dan banyak yang mengatakan "tidak perlu takut" dan sejenisnya. Tentu saja, ini benar, tetapi jangan takut - setengah dari pertempuran. Yang terpenting adalah tahu persis apa yang Anda sukai. Cobalah mempelajari dasar-dasar suatu bahasa, misalnya, menggunakan pelajaran dari jaringan, lalu menulis skrip atau aplikasi paling sederhana. Jika Anda menyukainya, maka sudah waktunya untuk memulai.

Dan tip lainnya - jangan menjadi batu sandungan, di mana, seperti yang Anda tahu, air tidak mengalir. Mengapa Baru-baru ini saya belajar bagaimana keadaan dengan beberapa siswa lain. Ternyata, tidak semua orang mendapat pekerjaan. Saya mengundang beberapa orang untuk wawancara di tempat kerja saya, karena perusahaan saya membutuhkan spesialis yang baik. Tetapi pada akhirnya, tidak ada yang datang untuk wawancara, walaupun sebelumnya saya ditanya banyak pertanyaan.

Ini tidak layak dilakukan - jika Anda ingin mencari pekerjaan, maka konsistenlah. Bahkan jika Anda merasa bahwa Anda tidak memiliki pengalaman yang cukup, cobalah untuk melakukan beberapa wawancara - banyak perusahaan mengambil pendatang baru dengan harapan membesarkan spesialis di rumah. Jika wawancara gagal, Anda akan mendapatkan pengalaman berharga dan akan tahu bagaimana proses pekerjaan terlihat dari dalam.
