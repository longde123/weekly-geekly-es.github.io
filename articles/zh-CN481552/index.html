PGConf.Russia 2020即将发布

与往年一样，今年的PGConf.Russia 2020将在2月初举行，即-2月3日-大师班，第4和第5-报告。 这是我们第一次不必推迟接受报告申请的截止日期-人们变得更加自发组织，或者我们想谈论的话题更多。
在本文中，我将讨论会议等待我们的事情。 该站点上的完整程序无需复述，但是我将在此处提供主要（或在我看来是主要）报告。



集群


来自Miro的Victor Eremchenko将与Patroni分享他的经验。

中国HighGo Software的Muhammad Osama将讨论如何使用PgPool-II提供高可用性（HA）以及该产品的最新开发。

一个罕见的话题，就是1C的集群-参见下面有关1C的部分。

Andrey Borodin将讨论在Yandex中创建的Odyssey连接池，以及为什么pgbouncer不足以容纳它们。

绩效：谁拥有更多？


与往常一样，会议上将有有关人和Postgres如何应对沉重负担的报告。

Alexey Lesovsky将讨论PostgreSQL中各种负载缩放方法。

我建议您注意Microsoft的Alice Kuharchik的报告（与此同时，很清楚该公司为何收购Citus的原因）。

Adjust的Chris Travers承诺详细讲述他们如何在postgres中消化PB级的日志 。

来自inCountry的Pavel Konotopov和Leonid Albrecht将讨论创建一个高负载系统来在全球范围内存储个人数据 。 奇怪的是，他们使用nginx平衡了PostgreSQL上的负载。

肖恩·金（Sean Kim）将分享一个使云数据库的生产力提高十倍的秘诀。

InCountry的Alexander Spirin将谈论PostgreSQL和Mongo基准测试的意外结果。

迁移


谢尔盖·布尔米索夫（Sergey Burmisov）将谈论他搬到Clickhouse并回到Postgres的事实，事实证明，这并没有放缓 。

俄罗斯最大的医疗信息系统之一的开发人员Taras Chikin将讨论该系统向PostgreSQL的迁移 。

PL / pgSQL开发人员Pavel Stehule将讨论如何在性能方面从PL / SQL最佳移植到PL / pgSQL 。

Ivan Chuvashov将介绍使用Pentaho进行数据迁移的可能性。

Bars的Alexander Kvaratskhelia将展示在他的公司中创建的自动PL / SQL转换器 。

来自印度的来宾Abkhinav Sagar和Premnat Jangam也将介绍他们从Oracle和DB / 2迁移的方法。

来自Lingua Leo的Oleg Pravdin将展示他有20亿以上用户从MySQL顺利迁移到PostgreSQL项目的经验。

Parma的Andrey Zubkov将展示他创建的pg_profile DBMS历史负载分析工具，该工具可以解决类似于Oracle AWR的问题。

来自开发人员的有关PostgreSQL内部结构和新功能的信息


最古老，最著名的开发人员之一布鲁斯·莫吉安（Bruce Momgian）将讨论设备管理器锁 。

关于Habré上postgres内部结构的一系列文章的作者Egor Rogov （ erogov ）将讨论如何以及为什么在数据库中收集统计信息 。
Heikki Linkangas讨论了如何利用postgres的灵活性并创建自己的数据类型 。

应用开发


Kamil Islamov讨论了PostgreSQL中序列的功能。

Bruce Momgian将分享使用非关系数据的各种方法（不仅仅是JSON！）。

Ivan Frolkov将讨论事务隔离级别，并解释何时以及为何应应用各种级别。

布鲁塞尔大学的Mahmoud Zakr将讨论基于PostgreSQL创建的时空数据存储系统Mobility DB ，以及Mosgortrans的Nina Belyavskaya-讨论该系统如何帮助莫斯科的公共交通 。

Alexey Fadeev将讨论如何使用PL / v8实现GraphQL 。

Postgres Professional的Valery Popov和Nikolai Chadaev将讨论使用强制访问控制创建安全数据库的功能。

1C


PostgreSQL是在1C：Enterprise平台上运行产品的DBMS之一，并且唯一的此类DBMS越来越流行。 一些报告将专门介绍如何在1C模式下正确使用PostgreSQL。

Vyacheslav Gilev团队的Dmitry Shitov将讨论PostgreSQL优化器的设置 。

Vasily Puchkov将讨论大型公司的运营经验和数据库迁移 。

Semyon Troshkin将基于Patroni引入用于1C的高可用性（HA）集群。

Infosoft技术经理Anton Doroshkevich将讨论postgres需要1C昵称 ，以及如何在不存在1C昵称的情况下生活。
