3CX v16 Alpha 2 dan rencana untuk tahun baru

3CX v16 Alpha 2


Pertama-tama, izinkan saya mengucapkan Selamat Natal dan Selamat Tahun Baru kepada Anda dan terima kasih dari lubuk hati kami atas kepercayaan Anda pada produk kami! Kami pasti tidak akan mengecewakan Anda pada tahun 2019!

Tepat sebelum Natal, kami merilis 3CX v16 Alpha 2 dengan beberapa fitur yang sangat menarik. Baca ulasan kami dan tetap disini - masih ada banyak hal menarik di depan!

Indikator BLF klien web




Dalam 3CX v16 Alpha 2, pengguna memiliki kesempatan untuk mengkonfigurasi indikator BLF secara independen di klien tanpa melibatkan administrator. Ini dilakukan di bagian Parameter - Parameter BLF. Anda dapat menetapkan ulang dan memindahkan tombol virtual di klien web, dan setelah mengkonfigurasi ulang telepon IP, mereka akan disinkronkan dengan indikator BLF perangkat keras pada telepon.

Klien web juga menerima Operator Console baru, yang memungkinkan pemantauan semua panggilan dalam sistem, termasuk panggilan dalam Antrian. Konsol memiliki tautan yang mudah digunakan untuk manajer pusat panggilan, memungkinkan Anda untuk terhubung dan memutuskan sambungan operator Antrian. Informasi tentang status operator dan statistik individual Antrian tersedia di Konsol. Ini juga menawarkan ringkasan analitik dan statistik untuk semua panggilan dan Antrian (yang Anda kelola sebagai Manajer Antrian).

Integrasi CRM Tingkat Lanjut


Kami telah berulang kali diminta untuk menyelesaikan masalah perbaikan panggilan dalam log CRM untuk menyimpan catatan panggilan sebagai kontak CRM baru. Dan kami akhirnya mengimplementasikan ini dalam integrasi CRM sisi server. Alpha 2 menambahkan kemampuan untuk merekam data panggilan di semua sistem CRM yang didukung.

PBX Instance Manager 3CX




3CX v16 Alpha 2 memperkenalkan 3CX Instance Manager baru, yang dapat digunakan mitra untuk mengelola instalasi klien PBX secara transparan dan aman (hanya instalasi Linux yang didukung).

Saat mengembangkan Manajer, teknologi internal pemantauan aktif dan kontrol sumber daya yang digunakan oleh pengembang kami digunakan. Perhatikan bahwa agar teknologi ini berfungsi, Anda tidak perlu mengonfigurasi layar jaringan PBX. Secara umum, menggunakan Manajer secara signifikan meningkatkan toleransi kesalahan PBX, karena mereka dapat berhasil diikuti oleh mitra 3CX ketika administrator sistem penuh waktu perusahaan tidak dapat melakukan ini karena beberapa alasan.

Kami memperlakukan privasi data dengan serius, sehingga Anda selalu melihat data mana yang ditransfer ke Instance Manager.



Hanya administrator sistem atau manajer perusahaan yang memutuskan apakah akan mengaktifkan pemantauan, dan apa yang sebenarnya bisa dipantau.

Pembaruan lainnya


Kami memperbarui layanan pengenalan pesan suara , yang sekarang menggunakan Google Speech API yang lebih kuat (bukan Microsoft Bing Speech Services). Harap dicatat bahwa hanya menit pertama dari pesan suara yang dikenali. Teks yang dikenal dikirim ke pengguna melalui email.



Versi baru ini membuat peningkatan kinerja dan keamanan yang signifikan berkat dukungan HTTPS / 2 di server web nginx (yang menggunakan 3CX). Ini memastikan kompatibilitas dengan aplikasi dan perangkat generasi sebelumnya. Pakar teknis tahu berapa banyak HTTPS / 2 meningkatkan kinerja server!

File impor ekstensi (pengguna) sekarang juga dapat menyertakan garis DID yang ditentukan dalam batang SIP. Anda dapat "menarik" tidak hanya pengguna itu sendiri, tetapi juga nomor DID yang ditugaskan kepada mereka - dari satu file CSV. Ini secara signifikan mempercepat transfer konfigurasi ke PBX baru.

Agar rilis baru berfungsi dengan baik, Anda perlu memperbarui set pesan suara di bagian Pengaturan> Pesan Sistem> Pembaruan> Pembaruan.

Masalah yang Diketahui


  • Pratinjau gambar saat ini tidak didukung pada instalasi Linux.
  • Saat beralih dari pengenalan suara Microsoft Bing ke Google Speech, Layanan Transkripsi Voicemail memerlukan kunci API Google Speech yang baru. Hasilkan dan ganti dengan kunci lama.
  • Karena pembaruan Firefox baru-baru ini, indikator BLF di klien web tidak ditampilkan dengan benar. Kami akan memperbaiki masalah ini dalam Beta v16.
  • Log layanan antrian manajer tidak ditimpa secara siklis. Ini akan diperbaiki dalam Beta v16.
  • 3CX Instance Manager tidak berfungsi di kluster failover.

Instal 3CX v16 Alpha 2


Anda dapat mengunduh versi 3CX Alpha 2 untuk Windows dan Linux . Anda juga dapat menginstal 3CX di cloud menggunakan wizard PBX express . Dalam hal ini, Anda akan segera menerima PBX berbasis cloud yang dikonfigurasi dan siap digunakan!

Rencana pengembangan 3CX v16


Berdasarkan posting staf 3CX di forum komunitas , kami telah menyiapkan daftar fitur dan peningkatan baru yang akan muncul tahun depan. Daftar ini tidak dimaksudkan untuk menjadi akurat dan mengikat, dan tidak resmi. Namun, ini akan memungkinkan Anda untuk menavigasi rencana pengembangan 3CX untuk beberapa bulan mendatang. Untuk kenyamanan, daftar ini dibagi menjadi beberapa unit teknologi yang membentuk sistem.

Integrasi, CRM, dan antarmuka skrip


Dalam arah inilah diharapkan jumlah peluang baru terbesar:

  • Peningkatan integrasi CRM, terutama dengan Office 365, Salesforce, dan sistem populer lainnya.
  • Desainer Alur Panggilan akan diganti oleh skrip dalam C #, Powershell dan Python. Mungkin juga dalam antarmuka yang mirip dengan CFD, Anda dapat membuat skrip dalam C # dan kemudian mengeditnya secara manual. Ini akan memungkinkan untuk bekerja dengan kode kesalahan, meminta input DTMF, mentransfer data ke CRM REST API, dll. Dalam waktu dekat kami akan merilis contoh skrip tersebut. Banyak hal kompleks atau yang saat ini tidak mungkin dilakukan dengan 5-10 baris kode. Implementasi penuh antarmuka skrip 3CX baru akan memakan waktu beberapa bulan.
  • Seiring dengan pengembangan skrip, kami akan memperluas antarmuka konfigurasi IVR dan Antrian Panggilan sehingga administrator PBX dapat mengkonfigurasi logika kompleks pemrosesan panggilan tanpa pemrograman.
  • Hal yang sama direncanakan untuk diterapkan ketika memproses ID Penelepon masuk dan keluar, serta dalam aturan masuk dan keluar. Misalnya, Anda bisa menghubungkan skrip pemrosesan tambahan ke aturan keluar.

Klien Web 3CX


  • Semua fitur klien asli 3CX untuk Windows dan Mac akan ditransfer ke klien web. Pada saat yang sama, klien ini masih akan bekerja, tetapi sebagai softphone SIP biasa.
  • Antarmuka klien web akan diperluas dengan fitur-fitur baru. Secara khusus, dimungkinkan untuk menyesuaikan antarmuka dengan menggabungkan informasi dari berbagai sumber pada satu layar.
  • Plugin Chrome dapat dirilis untuk lebih memperluas kemampuan klien web.
  • Untuk kenyamanan pengguna, semua hot key dari klien Windows akan didukung.

3CX Session Border Controller Utility


  • Direncanakan untuk secara signifikan memperluas fungsi SBC. Secara khusus, manajemen telepon jarak jauh dari konsol 3CX (melalui SBC) akan tersedia.
  • Dukungan akan diterapkan hingga 100-150 telepon jarak jauh, tergantung pada kekuatan PC tempat SBC dipasang.

Lisensi 3CX


  • Pelepasan edisi Standar yang bebas dan diperbarui seumur hidup untuk 8 panggilan serentak diharapkan.
  • Pada saat yang sama, beberapa fungsi yang sekarang tersedia dalam Standar akan ditransfer ke edisi Pro. Penting untuk dicatat bahwa ini tidak akan mempengaruhi pengguna lama (kunci lisensi). Perubahan hanya akan muncul dalam lisensi 3CX baru. Perbandingan fitur lengkap akan dipublikasikan setelah rilis 3CX v16.

Plugin Wordpress, firmware ponsel dan laporan


  • Di plugin 3CX untuk Wordpress, obrolan langsung akan muncul di situs, yang dapat dilayani oleh satu operator atau grup, serta ditransfer ke panggilan suara.
  • 3CX akan terus merilis firmware telepon IP sendiri yang telah teruji. Meskipun mereka sedikit tertunda relatif terhadap firmware vendor terbaru, mereka dijamin tidak mengandung kesalahan kritis yang membuatnya sulit untuk bekerja dengan 3CX.
  • Laporan berguna baru akan muncul, khususnya, pada kualitas panggilan VoIP.
