我们使用Google PageSpeed Insights服务分析了俄罗斯最大的网上商店的39个变量-按活动领域和设备划分。 有多少家商店进入绿色区域，内容呈现和整页加载需要花费多长时间，可加快网站访问速度的方法最受欢迎，请在评论中阅读。

为什么所有这些以及对谁有用？


在分析俄罗斯TOP-20在线商店的移动可用性时，我们发现20家商店中只有1家通过了Google PageSpeed Insights服务的测试。 大多数甚至不属于“橙色”区域。


下载速度很重要，原因有两个：


  • 缓慢的网站失去了客户-网站加载的时间越长，返回搜索的愿望就越高；
  • 快速网站更有可能在有机行业中占据更高的位置（请记住，向移动优先索引的过渡）。

我们决定深入研究网站加载速度这一主题，并研究电子商务的实际情况。 我们从DataInsight评级中选择了48家在线商店-来自不同领域的8家在线商店：


  • 汽车产品；
  • 美丽与健康；
  • 衣服，鞋子和配件；
  • 家庭用品；
  • 电子和技术；
  • 百货商店。

使用PageSpeed Insights对每个商店进行了分析，并汇总了汇总表 。




接下来，我们将解释PSI中的指标的含义，并在电子商务的不同领域中对其进行分析。 该信息不仅对开发人员和优化人员有用，而且对了解当前趋势的在线商店所有者也很有用。


有关PageSpeed Insights（PSI）的重要信息


PSI提供有关移动设备和PC上页面加载速度的信息，以及有关如何使其更快加载的提示。


PSI报告分为以下几个块：


  • 页面加载速度得分（性能等级）；



  • 观察数据-显示用户最近30天页面加载速度的数据（摘自Chrome浏览器可用性报告）；



  • 模拟加载-页面加载速度上的数据（使用Lighthouse技术获取它们）；



  • 审核结果-需要提高页面加载速度的任务列表。



我们将更详细地分析并分析每组指标。


性能等级


PageSpeed Insights根据使用Lighthouse工具（单独针对台式机和移动版）对其加载速度的分析，为网站评分。 数据每天更新，涵盖最近30天。


评分等级如下：


  • 90-100点-“绿色”区域。 你可以打开香槟。
  • 50-89点-“橙色”区域。 有缺陷，但您可以活下去。
  • 0-49点-“红色”区域。 一切都不好。

在理想的世界中，该站点应位于绿色区域。 但是现实是严酷的：




从移动设备查看时，在48家在线商店中（在TOP-100评级中为注释），只有一个（！）生活在绿色区域中。 台式机的情况与此类似-只有2个站点获得90分以上。 这仅是所分析站点总数的2％和4％。




从移动设备查看时，将近90％的网站通过了下载速度测试，结果不尽人意。 从台式机观看时-约为52％。


如果您查看按行业划分的平均下载速度，则汽车用品以及服装，鞋子和配件的在线商店的加载速度会更快。 电子商店和百货商店都落后。




观察数据：FCP和FID


首次内容渲染（FCP）


First Contentful Paint（FCP）是从您访问站点到浏览器显示DOM的第一部分内容的时间。 从用户的角度来看，第一个渲染信号表明该站点确实正在加载。


PageSpeed Insights从Chrome浏览器的可用性报告中获取FCP值。 显示第90个FCP百分位数。


FCP值的等级：


  • 0-1秒-第一次呈现内容的速度高；
  • 1-2.5秒-平均速度；
  • 超过2.5秒-低速。

各行业的平均FCP值：

首次输入延迟（FID）


第一次输入延迟（First Input Delay，FID）是指从用户与网站进行交互（单击链接，按钮，使用JavaScript元素）开始到浏览器响应此交互之间的时间。


输入延迟越低，用户等待浏览器的反应就越少。 但是，例如，由于脚本数量巨大，浏览器可以长时间处理它们，这就是为什么输入延迟很高的原因。


PSI通过FID指示第95个百分点-最近30天的平均值。


FID值的等级：


  • 0-50毫秒-高速;
  • 50-250毫秒-平均速度；
  • 超过250毫秒-低速。



与FCP不同，FID值取决于访问站点的设备而有所不同。 从移动设备观看时，第一输入延迟显着高于从PC观看时的延迟。 在8个站点上，用户必须等待超过1.5秒的时间才能反弹，而在三个站点上，用户需要等待3秒以上（这是正常水平的60倍）。


按行业：




在服装，鞋子和配饰的销售领域，无论是在移动设备还是在台式机上，网站对用户操作的反应最快。 美容与健康，电子和技术，百货商店等场所的响应时间更长。


与FCP一样，PSI服务显示FID输入延迟的百分比分布。 一个例子：




从台式机观看时，移动设备中低速下载的百分比是同一指标的2倍以上。 高速下载的份额分别为55％和83％。




模拟页面加载


PSI报告的此块显示使用Lighthouse技术模拟页面加载的结果。 这里介绍了六个指标。 基于它们，形成页面加载速度的点估计。 而且，指标具有不同的权重 。 根据重要性，它们的分布如下：


5（最重要）-交互加载时间（TTI）；


4-下载速度指数（Speed Index）；


3-第一内容的加载时间（FCP）；


2-CPU结束时间（第一个CPU空闲）；


1-内容的足够部分的加载时间（FMP）；


0（未考虑）-最大潜在延迟（FID）。


让我们看一下每个指标。


首次内容下载时间（FCP）


我们已经在上面分析了该指标。 区别在于，在下载模拟块中，数据是使用Lighthouse获取的，在观察块中是从Chrome浏览器的可用性报告中获得的。




在移动设备中，第一个内容的加载时间要长得多。 在48个站点中的9个站点（占19％）上，该时间超过6秒。 从桌面查看时，FCP速度要高得多-在这里只有5个站点（占10％）的第一次渲染持续2秒以上。

足够的内容下载时间（FMP）


第一有意义绘画（FMP）是从页面加载开始到用户看到足够的内容以开始使用页面的时间。


FMP值与FCP进行比较很有趣。 在40％的情况下，从台式机观看时，FMP与FCP相吻合-第一次渲染时，用户会立即看到很大一部分内容。 在其他情况下，用户必须另外等待内容加载（平均-0.3秒）。




从移动设备浏览时，有46％的FCP站点与FMP一致，但是这些指标之间的差距更大-平均为1.4秒（这是从台式机浏览时的4.7倍）。




按行业（移动）：




FMP和FCP之间的差距最大的是在电子和技术领域，而在服装鞋子和配饰的在线商店中，差距最小。 逻辑-为什么如此-很难找到。 可以假定这是由于不同行业中站点的设计特征引起的。


按分支机构（桌面）：




从桌面查看时，情况类似-但FCP和FMP之间的差距较小。 此外，在汽车产品领域，而不是服装，鞋子和配件中，FMP值最低。


下载速度指标


速度索引是显示页面可见部分所花费的时间。 此指标由webpagetest开发和使用。 org 。 该服务获取页面加载的连续屏幕快照，分析内容加载的动态并显示速度索引。




让我们看一下分析站点的速度指数值。




在移动设备中，有54％的站点的下载速度指数高于10秒，而有85％的下载速度指数高于5秒（请记住，速度指数是形成PSI中站点加载速度的点估计的第二个最重要的因素）。


在台式机上，情况要好得多-这里只有23％的网站的速度指数高于5秒。


按行业：




速度指数的最佳值体现在汽车产品以及服装，鞋子和配件的销售中。 移动电子商店站点的加载时间最长-平均超过14秒。


CPU结束时间


以前，此指标称为首次互动的时间（首次互动）。 但是在Lighthouse 3.0中，其名称更改为First CPU Idle。


“第一CPU空闲”显示了从页面加载开始到最小交互的那段时间：


  • 大多数（但不是全部）界面元素可以交互；
  • 该页面平均会在合理的时间内响应大多数手动输入。

将“第一CPU空闲”值与内容的足够部分（FMP）的加载时间进行比较。 底线是查看有多少用户在访问内容后必须等待机会与页面元素进行交互。


在移动设备中，只有8％的站点与First CPU Idle和FMP的值匹配。 在其他情况下，会有间隔（平均-8.7秒）。 也就是说，如果用户要单击页面上的某些内容，则该元素将不会立即可用。




在台式机中，有6％的站点具有相同的“第一CPU空闲”和FMP值。 平均而言，这些指标之间的差距为2.7秒（比移动设备少3.2倍）。




按行业（移动）：




First CPU Idle的最小值在汽车产品销售领域，最大的价值在家庭用品领域。 最大的差距是在服装，鞋子和配饰的在线商店。

总结一下：值得为高速而战吗？


当您查看顶级在线商店的下载速度的分析结果时，您对加快自己的网站速度没有太大的期望。 在48个站点中，只有一个通过了移动和台式机的PSI测试，并保持了较高的头脑。 剩下的一切都不是很糟糕。 这些都是“顶峰”-有团队和金钱。 那小型的区域商店呢？


但是，这样的思路会把你引向错误的地方。


查看“顶部”很有用，但并非总是专注于它们。 除了加载速度外，其他许多因素也导致大型站点滑行。 这里和分类，借出，交付的可用性以及忠诚度计划。 不要忘记品牌的力量。 如果某人正在寻找特定的商店，那么他已经准备好承受长期的负载。


小商店按照其他规则居住。 就种类和服务而言，与大型站点竞争并不容易。 但是由于质量（例如，快速便捷的站点）的存在，您可以保留客户，同时提高您在本地搜索结果中的位置。


当然，在加快站点速度时，您需要比较成本和预期效果。 在某种程度上，您可以免费快速地加速您的网站。 使用插件或简单的代码可以实现相同的延迟加载，缓存设置和GZIP压缩。


但是已经在结构，脚本，服务器响应速度方面进行了优化，可能需要认真地重新设计网站。 而且，如果您的网站在当前状态下能够很好地实现流量转换，那么投资吸引用户-SEO，上下文，在社交网络上投放广告比在PSI中争取优势要好。
