Astra Linux“ Eagle”普通版：Windows之后是否有生命

我们收到了一位操作系统用户的详细评论，并希望与您分享。

Astra Linux是Debian的衍生产品，是俄罗斯在改用开源软件的计划中创建的。 有多种版本的Astra Linux，其中一种旨在用于日常日常使用-Astra Linux“ Eagle”普通版。 从定义上来说，每个人的俄罗斯操作系统都很有趣，我想从一个每天使用三种操作系统（Windows 10，Mac OS High Sierra和Fedora）并且忠于Ubuntu的人的角度谈谈Eagle。 基于这一经验，我将从不同角度考虑系统的安装，接口，软件，开发人员的基本功能以及便利性。 Astra Linux与更常见的系统相比将如何？ 她可以在家中代替Windows吗？



我们把Astra Linux


Astra Linux安装程序与Debian安装程序有很多共同点。 也许第一个甚至更简单，因为默认情况下大多数参数都是固定的。 这一切都始于在不太高的建筑物的背景下的一般许可协议。 甚至在奥勒尔（Orel）。



安装中的重要一点是软件的选择，这是系统默认提供的。 可用的选项满足标准的办公室和工作需求（对于非开发人员）。



另外，最后一个窗口是另外一组设置：阻止解释器，控制台，跟踪，设置执行位等。如果这些字告诉您什么，最好不要打勾。 此外，如果需要，所有这些都可以在以后配置。



该系统安装在资源有限的虚拟环境中（相对较新的系统）。 没有关于速度和性能的抱怨。 下面介绍了已测试的配置。



通常的安装过程是：安装iso映像 ，通过标准的系统安装过程安装它，然后烧毁GRUB引导加载程序。



加载时系统对资源的要求不高-桌面模式启动时大约需要250-300 MB的RAM。



备用启动选项：平板电脑和手机模式


进入系统后，您可以选择以下启动选项之一：安全，桌面，移动或平板电脑。



要在触摸设备上工作，您可以打开屏幕键盘。



让我们看看在不同模式下有趣的事情。 桌面是正常模式，系统类似于Windows。



平板电脑模式适用于大型触摸屏。 除了明显的外部差异（可以在下面的屏幕截图中看到）之外，该界面还有其他功能。 数位板模式下的光标不可见，应用程序关闭按钮位于任务栏上。 全屏应用程序的工作方式略有不同，文件管理器中的文件选择也有所不同。



值得一提的是移动模式-此处的一切与Android中的大致相同。 使用图形环境Fly。 在触摸模式下，需要进行长时间的触摸才能调用上下文菜单。 与台式机和平板电脑相比，移动模式消耗的资源略多。





存在不同的操作模式很方便。 例如，如果您使用带插入式键盘的平板电脑，并因此使用触摸和非触摸用例。

系统更新


在开始使用系统之前，您需要对其进行更新。 基本上，Astra Linux 存储库具有14,000个软件包（ 稳定版 ， 测试版和实验版 ）。 实验分支很快将收到不稳定的更新，因此我们将测试测试分支。 将存储库更改为测试。



我们开始更新存储库并更新系统。 为此，请单击左上方的“更新”按钮，然后单击“标记所有更新”，然后单击“应用”。 重新启动。

用户政策


通过安全策略管理实用程序在系统中创建新用户。



默认情况下，提供远程登录功能（控制面板-系统-登录）。



除了通常的单独会话和远程会话之外，您还可以启动嵌套会话（“开始”-“关闭”-“会话”）。



对于前两个，一切都很清楚。 嵌套会话是从当前会话的窗口开始的会话。



顺便说一句，会话可以在延迟的时间后完成：不要等待冗长的操作结束，而只需配置自动关机即可。



Astra Linux接口和标准软件


Astra Linux Common Edition与几年前的Debian类似。 值得注意的是，向外的Astra Linux Common Edition试图与Windows更加接近。



导航和使用文件系统比Windows更靠近Windows。 通过系统映像，可以连接一组标准软件：办公室，网络，图形，音乐，视频。 系统设置也分组在主菜单中。 默认情况下有四个屏幕可用。


如您所见，LibreOffice作为办公套件安装在系统中

控制面板类似于Windows / Mac / etc，并将基本设置集中在一个位置。



文件管理器具有两个面板的界面，并且能够将档案作为文件夹挂载。





文件管理器可以计算校验和，包括符合GOST R 34.11-2012的校验和。



Mozilla Firefox被安装为标准浏览器。 它看起来很苦行，但同时又足够。 例如，我打开并翻阅了新鲜的Habr。 页面已呈现，系统不会崩溃也不会挂起。



下一个测试是图形编辑。 我们从Habr文章的标题下载了图片，并要求系统在GIMP中将其打开。 这里也没有异常。



轻轻一挥，我们就增加了其中一篇文章的效率测试。 原则上，与标准Linux系统没有区别。



让我们尝试超越简单的脚本，并通过apt-get放置标准软件包。



索引更新后：

对于测试，我们安装了python3-pip，zsh并完成了oh-my-zsh的安装（带有其他git依赖项）。 系统正常运行。

如您所见，该系统在普通用户的标准日常方案框架内运行良好。 如果您希望在这里看到Debian / Ubuntu熟悉的程序，则必须另外使用句柄安装它们（例如，如果您需要ack-grep之类的软件包-它们是通过curl / sh安装的）。 您可以将存储库添加到sources.list，并使用熟悉的apt-get。

本机Astra Linux实用程序


上述工具只是Astra Linux用户可用的一部分。 此外，开发人员还创建了约一百种其他实用程序，这些实用程序可以通过他们用于更新系统的同一存储库提供。



要找到实用程序，只需搜索“ fly”一词-所有必需的实用程序都具有这样的前缀。



很难在一个评论的框架内谈论所有应用程序，因此我们将从简单用户的角度选择一些有用的应用程序。 气象应用程序将显示俄罗斯选定城市的天气预报，它已针对俄罗斯地区进行了优化。



还有一个简单的图形实用程序，其中包含用于搜索文件的多个过滤器和设置。



有其自己的实用程序，用于监视电池电量和各种模式，可通过计时器配置其转换-关闭监视器，睡眠和休眠状态。



用于命令的可执行文件的选择也包装在图形外壳中。 例如，您可以指定运行命令时系统将选择哪个“ vi”。



使用单独的管理实用程序，您可以配置在系统启动时启动哪些应用程序。



还有GPS / GLONASS监视，在电话/平板电脑（通常存在相应的模块）中非常有用。



还有一个自己的简单PDF阅读器，用于在Lawrence Lessig的《免费文化》一书中启动的测试。



您可以在虚拟桌面的“帮助”部分中，在Astra Linux的虚拟导览中阅读有关所有Fly实用程序的信息。

与主要系统对比


从界面和控件逻辑的角度来看，该系统更像是经典的Windows XP，有时甚至是Mac OS的各个元素。

从实用程序，控制台和硬件的角度来看，该系统类似于经典的Debian，对于Ubuntu和Minted的相同用户来说，它相当不错并且熟悉，尽管最先进的系统会错过所有存储库中通常的软件包范围。

如果您将自己的经验强加给潜在用户，那么我对新系统会有积极的期望。 根据他们在Windows / Mac上的经验，普通用户可以轻松地熟悉Astra Linux Common Edition。 使用标准unix-utilities的更多高级Linux用户将配置适合他们的所有内容。

当前版本的Astra Linux基于Debian 9.4，并且还提供了来自Debian 10（4.19）的最新内核。

判决


Astra Linux Eagle普通版的主要优点：

  • 它不会掉落，不会挂起，没有发现严重故障。
  • 成功模仿Windows NT / XP界面。
  • 简单和易于安装。
  • 资源需求低。
  • 预安装了主要软件：LibreOffice办公套件，GIMP图形编辑器等。
  • 大量其他实用程序。
  • 软件包的版本早于Ubuntu的最新版本。
  • 它的存储库比Ubuntu和Debian的存储库小。

结论：针对家庭用户的最新非Ubuntu版本的Ubuntu将比Astra更好。

但是，对于家庭用户而言，坐在LTS发行版上可能并不重要，但是对于组织机构来说，这是一个正常的选择。 因此，针对企业领域的Astra Linux开发人员的选择是明确而合理的。

至于缺点，对于习惯使用Linux的人来说，它们更可能是正确的，因为从外观上看，Astra Linux“ Eagle”更接近Windows，而不是Linux。

作为向政府机构免费软件过渡的一部分，Astra Linux“ Eagle”通用版看起来像是Windows办公版的很好替代品，对于家庭使用而言，它似乎有些保守。
