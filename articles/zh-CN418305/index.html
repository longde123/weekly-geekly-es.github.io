执行脚本时，Python发出多少个对象？

当一些Python程序员在运行一个简单的脚本时发现python解释器分配了多少个临时对象时，他们会感到非常惊讶。

CPython允许您获取分配对象的统计信息，为此，您需要使用其他标志对其进行编译。 

./configure CFLAGS='-DCOUNT_ALLOCS' --with-pydebug make -s -j2

让我们的结论更具可读性： 

 def print_allocations(top_k=None): allocs = sys.getcounts() if top_k: allocs = sorted(allocs, key=lambda tup: tup[1], reverse=True)[0:top_k] for obj in allocs: alive = obj[1]-obj[2] print("Type {}, allocs: {}, deallocs: {}, max: {}, alive: {}".format(*obj,alive))

其中：

  • allocs-自解释器启动以来已分配了多少个对象
  • deallocs-删除了多少个对象（手动或自动）
  • alive-活动（当前）对象的数量（分配-分配）
  • max-自解释程序启动以来最大的活动对象数

如您所见，空的Python REPL设法分配了17,328行和10,550个元组。 这是一些疯狂的物体！ 在这里，您需要记住，为了使REPL正常运行，Python会自动导入其他模块（在空脚本的情况下不会导入）。

框架和代码对象


在上面的示例中，您可以找到许多框架和代码对象。 为什么需要它们？

简而言之，每个代码对象在其自身中存储一个已编译的代码块，然后使用框架对象来执行它们，并按照调用堆栈的原理进行工作。 在Python中，最流行的块是一个函数。 每个新函数都需要自己的代码对象，对此函数的每次调用都需要一个单独的框架对象，Python将在其中存储局部变量。 除了局部变量外，每个框架对象还存储执行该功能所需的大量辅助数据。

所有这些对象从哪里来？


Python是一种非常动态的语言，您必须为此付出代价。 为了支持动态功能，他创建了许多支持对象的临时对象。

例如，声明一个简单函数将创建至少5个字典，5个元组和4个列表。 这些对象将一直存在，直到脚本结束。 反过来，所有这些对象本身都存储了其他对象（它们的元素），这些对象是数十个，有时是数百个用于编译函数内部描述的其他对象。 普通类的描述可以突出显示数百个容器（字典，元组，列表）对象。 不幸的是，这里不可能自动计算分配的对象的确切数量，这些数字是近似的。

为了使Python快速分配大量对象，它使用了大型多层系统来优化内存中对象的分配。

有时，人们想知道解释的语言对我们隐藏了多少细节。 Python使您可以编写优质的代码，而无需考虑很多问题和细节。
