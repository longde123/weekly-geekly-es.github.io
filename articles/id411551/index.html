Lampu Smart Koogeek untuk Apple HomeKit. Kesan pertama

Saya membeli lampu pintar koogeek LB1 dengan Wi-Fi, yang dirancang untuk dikendalikan melalui Apple HomeKit. Lampu terlihat hampir sama dengan lampu LED biasa (di sebelah skala adalah lampu pijar):


Spesifikasi yang diumumkan:

Basis: E27 (lampu konvensional)
Daya: 8 W
Fluks bercahaya: 500 lumens
Temperatur warna: 2700-6000 K
Kedalaman warna: 16000000
Sudut disipasi: 199,9 °
Sumber daya: 25000 jam
Wi-Fi: 802.11 b / g / n, 2,4 GHz

Untuk manajemen saya menggunakan tablet iPad Air dengan iOS 11.3. Saya memasukkan lampu ke kandil, menyalakannya - lampu menyala. Seperti biasa, tanpa membaca manual, saya mulai menghubungkan lampu.
Saya meluncurkan aplikasi "Rumah".



Saya mengklik “Tambah aksesori” (atau tanda tambah di sudut layar, jika sistem sudah memiliki perangkat). Sebuah jendela muncul di mana Anda dapat memindai kode menggunakan kamera atau memasukkannya secara manual, kode ada di stiker di lampu dan di stiker di instruksi.



Saya mengklik ikon lampu.



Dan saya mendapatkan pesan kesalahan.



Tetapi pada tahap ini ternyata manual masih perlu dibaca. Di sana, dalam bahasa Inggris, ditulis dalam warna putih dan putih bahwa iPad harus terhubung ke jaringan Wi-Fi 2,4 GHz, dan saya memiliki titik akses dual-band dan iPad terhubung ke jaringan 5 GHz. Dan, jika Anda berpikir sedikit, semuanya logis - lampu tidak meminta nama jaringan dan kata sandi untuk Wi-Fi. Bagaimana dia bisa tahu? Jelas, Aypad harus mengirimkan data ini kepadanya. Tetapi juga terhubung ke jaringan 5 GHz dan, tentu saja, mentransfer pengaturan untuk jaringannya. Dan lampu di jaringan ini tidak berfungsi. Semuanya jelas - kami beralih iPad ke jaringan 2,4 GHz dan ulangi prosedur. Menghasilkan! Tentu saja, setelah menyelesaikan semua pengaturan, saya kembali mengembalikan iPad ke jaringan 5 GHz, seluruh rentang 2,4 GHz penuh sesak dengan tetangga, kecuali bahwa itu cukup untuk mengontrol bola lampu.

Sekarang semuanya baik-baik saja. Saya mengubah nama menjadi "Lampu" (dengan nama ini akan mungkin untuk mengontrol lampu menggunakan Siri).



Saya klik "Selesai" dan mulai memeriksa. Saya klik pada ikon atau memberikan perintah "Hello Siri, nyalakan lampu" - semuanya berfungsi.



Jika Anda menekan dan menahan ikon lampu, jendela penyesuaian kecerahan terbuka.



Dengan menu pengaturan warna tambahan.







Untuk kenyamanan, ada skenario lampu. Untuk setiap skenario, Anda dapat mengatur status on / off, warna, dan kecerahan. Untuk skenario "Selamat Pagi", saya mengatur kecerahan penuh menjadi putih, untuk "Good Night" - kecerahan minimum menjadi biru (semacam lampu malam), "kuning" - hanya kuning dengan kecerahan penuh, "hijau" - dengan cara yang sama. Sekarang untuk mengontrol lampu, katakan saja "Hello Siri" dan "Selamat malam." Tentu saja, ini semua misalnya, Anda dapat membuat skrip dengan nama apa pun. Tentu saja, komponen sistem HomeKit lainnya dapat ditambahkan ke skenario ini.



Untuk skrip, dimungkinkan untuk mengkonfigurasi otomatisasi berdasarkan timer, berdasarkan geolokasi, dll. Saya menjelaskan pengaturan ini di artikel sebelumnya tentang outlet Elgato yang cerdas.

Ada juga aplikasi Koogeek Home untuk mengendalikan lampu. Tapi saya tidak mengerti, sekilas ada fungsi yang sama dengan HomeKit.

Jika terjadi kesalahan atau untuk menautkan kembali lampu ke sistem lain, dimungkinkan untuk mereset pengaturan lampu sepenuhnya, ini dijelaskan dalam instruksi. Lampu dibeli di Aliexpress, harganya ~ 2000 rubel.
