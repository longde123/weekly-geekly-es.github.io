E-Mail-Migration: So können Sie problemlos von einem Server auf einen anderen wechseln

Das im Titel angegebene Thema mag für angesehene Chabrowiter nicht relevant sein, aber manchmal ist es einfach notwendig, es zur Sprache zu bringen. Tatsache ist, dass ich seit vielen Jahren als Administrator in einem humanitären Forschungsinstitut arbeite, in dem Mitarbeiter über eine solche Qualifikation auf dem Gebiet der modernen Informationstechnologien verfügen, dass die berüchtigte Buchführung aus Anekdoten über IT vor diesem Hintergrund wie eine Sammlung von Philosophen wirkt, die sich allen Geheimnissen des Lebens widmen. Sehr geehrte Wissenschaftler, geben Sie die Namen der Mailserver in russischen Buchstaben ein, anstatt des "@" - Zeichens, schreiben Sie "Hund" in Klammern (und sagen Sie dann, dass es so in der an sie gesendeten Mailadresse geschrieben wurde), und versuchen Sie, mit The Bat! und mache ein paar seltsame Dinge, oft in derselben Botschaft. Es ist sinnlos, sie zu unterrichten, es ist unmöglich, sie zu bekämpfen. Es bleibt ihr Schicksal zu akzeptieren und letztendlich alle Operationen zu automatisieren, die mit der Korrektur ihrer Fehler zusammenhängen.

Eine der unheimlichsten und gefährlichsten Operationen in meiner Praxis war die Migration von Webmail von Server zu Server. Tatsache ist, dass die Mitarbeiter des Instituts bereits drei Service-E-Mails haben: eine enthält einen internen Exchange-Server, die andere wird auf Mail.ru und die dritte auf Google Mail ausgeführt. Nein, ich bin kein Idiot und nicht einmal sie. Dies ist eine Direktivenverordnung, die sich auf einige Abteilungsspiele bezieht. Etwas muss im Institut auf dem "Unternehmensserver" bleiben, etwas im Zusammenhang mit Anträgen und Stipendien muss auf jeden Fall über die russische Post gehen, und die Post von Google Mail von meinen lieben Kollegen ist mit auf jeden Fall notwendigen Dingen wie Dokumenten und Tabellen verbunden Google, Backup auf Festplatte usw. Das einzige Problem ist, dass die sieben Kindermädchen, wie Sie wissen, ein Kind ohne Auge haben - das heißt, in diesem Fall, zwischen den drei Mailservern, die meine Kollegen auf die expliziteste Weise schaffen, die wichtigsten Briefe zu verlieren!

Es gibt ein weiteres Problem, das häufig eine E-Mail-Migration erforderlich macht. Moderne Mail-Dienste ermöglichen häufig das automatische Weiterleiten von Nachrichten von einem Server zu einem anderen, dh das Sammeln von Nachrichten. Und ein Benutzer, der es gewohnt ist, dass seine Nachrichten auf dem Server, z. B. Mail.ru, automatisch in Yandex-Mail kopiert werden, vergisst manchmal, dass er auf diese Weise nicht auf alle Nachrichten, sondern nur auf die Nachrichten zugreifen kann, die später empfangen wurden Einstellungen für die E-Mail-Sammlung. Daher hat er möglicherweise den natürlichen Wunsch, die vollständige Migration von E-Mails vom alten Server auf einen neuen, häufiger genutzten Server abzuschließen, und an wen wird er sich mit diesem Wunsch wenden? Das ist richtig: zum nächsten Systemadministrator!

Ich denke, eine ähnliche Situation ergibt sich für jeden, der auf irgendeine Weise gezwungen ist, mehrere E-Mail-Konten zu haben, geschweige denn diese zu verwalten, oder einfach nur von Server zu Server wechseln möchte, ohne wichtige Informationen zu verlieren. Natürlich kann IT-Bison dieses Problem mit nur zwei Mausklicks lösen. Wenn Sie jedoch in solchen Angelegenheiten unerfahren sind, kann die E-Mail-Migration zu einer schwierigen Aufgabe für Sie werden. Aus diesem Grund habe ich mich entschlossen, kurz über meine Erfahrungen mit dem einfachen Exportieren von E-Mail-Briefen in einen Speicher und dem anschließenden Importieren von E-Mail auf einen anderen Server zu berichten. Vielleicht hilft diese Operation jemandem, kleinere Probleme loszuwerden, oder vereinfacht einfach das Leben!

Briefe exportieren: ein bisschen Theorie, ein bisschen Praxis


Mailserver arbeiten im Prinzip mit Client-Programmen, die eines von zwei Protokollen verwenden: POP3 oder IMAP. Wenn diese Namen plötzlich nichts mehr aussagen (passiert dies?), Versuche ich es mit einfachen Worten zu erklären: Das POP3-Protokoll lädt Nachrichten vom Server auf Ihren Computer herunter und das IMAP-Protokoll verarbeitet sie direkt auf dem Server. Alte E-Mail-Clients haben standardmäßig mit dem POP3-Protokoll gearbeitet (und arbeiten auch weiterhin) und E-Mail-Nachrichten in einem speziell für den Client zugewiesenen Ordner entladen (normalerweise befindet sich dieser Ordner im Benutzerverzeichnis zwischen den Ordnern, in denen die Anwendungsdaten standardmäßig ausgeblendet sind). IMAP ist moderner und mit ihm können Sie auch E-Mails in den lokalen oder Netzwerkspeicher importieren. Die Frage ist also hauptsächlich nicht, wie Sie die benötigten Briefe herunterladen, sondern wie Sie sie an den richtigen Server weiterleiten, um die Mail-Migration durchzuführen. Am einfachsten ist es, das IMAP-Protokoll zu verwenden, alle Briefe in einen Speicher im EML-Format zu kopieren und sie dann in einen anderen Ordner eines anderen Kontos hochzuladen, wobei das Format der Nachrichtendateien im Allgemeinen identisch ist.

Wie kann man das machen?


Die einfachste Methode, die ich zu den niedrigsten Kosten verwende, besteht darin, E-Mail-Briefe mit einem Datenkopierprogramm zu migrieren, das IMAP unterstützt. Dies geschieht in zwei Schritten.

  • Importieren Sie E-Mails aus einem Ordner auf dem Server in einen beliebigen Speicher im EML-Format.
  • Exportieren Sie E-Mail-Briefe über IMAP in einen anderen Ordner auf einem anderen Server.

In diesem Fall verhält sich das Mail-Migrationsprogramm aus Sicht beider Server wie ein regulärer IMAP-Client. (Übrigens müssen Sie auf den meisten Mail-Servern das angegebene Programm als Mail-Client verwenden. Melden Sie sich daher unbedingt bei Ihrem Mail-Konto an und lassen Sie den Server dieses Dienstprogramm in der Liste der verfügbaren Clients über IMAP verwenden, bevor Sie E-Mails mit einem Dienstprogramm migrieren.) Solche Programme erfordern in der Regel ein Minimum an manueller Arbeit, um die E-Mail-Migration vorkonfigurieren zu können. Normalerweise können Sie sogar einen Zeitplan für die regelmäßige automatische Migration von E-Mails von Server zu Server einrichten, wenn Sie dies aus irgendeinem Grund benötigen. Persönlich verwende ich das Programm " Handy Backup " zum Exportieren von E-Mail-Briefen, da es auf fast allen Computern installiert ist und ein Minimum an Einstellungen erfordert, die auch vom Computer des Administrators aus zentralisiert werden. Aber im Großen und Ganzen ist die verwendete Software unwichtig, wenn sie nur E-Mails geschickt direkt auf Webserver exportieren und importieren und ein einziges Format für Briefe auf beiden Servern unterstützen könnte.

Und Microsoft, wie immer ...


Ein weiteres Problem ist die Migration von Exchange- oder Outlook-E-Mail-Nachrichten (ich meine nicht den Outlook.com-Mailserver, sondern den Client), da Microsoft wie üblich einem nicht standardmäßigen Pfad folgt. Wenn Sie in dieser Situation eine spezielle Software zum Exportieren von Outlook-E-Mail-Briefen oder Exchange-Servern zur Hand haben, müssen Sie die Anweisungen zum Migrieren von E-Mail-Briefen mit dem entsprechenden Programm nur lesen. Es ist gut, dass es viele solcher Programme sowie spezielle Plugins für die entsprechende Software gibt, die sich an Microsoft-Produkten orientiert.

POP3-E-Mail-Migration


Manche Leute mögen Perversionen, aber im Allgemeinen ist das nicht so. Daher ist es nicht erforderlich, E-Mails mit dem POP3-Protokoll von Server zu Server zu übertragen. Es ist alt und hässlich. Gehen Sie auf beiden Servern zu IMAP (dazu hat fast jeder Anbieter detaillierte Anweisungen) und führen Sie dann alle Schritte wie oben beschrieben aus (oder verwenden Sie zumindest das in den E-Mail-Dienst integrierte Migrationstool - manchmal gibt es solche Tools, obwohl sie praktisch und praktisch sind Die Logik der Arbeit lässt normalerweise zu wünschen übrig. Sie können auch die altmodische manuelle Methode ausprobieren: Verwenden Sie das Client-Programm, um Briefe von Ordner zu Ordner zu übertragen, oder wählen Sie sie einfach aus und senden Sie sie an einen neuen Server. Als wir noch klein waren, haben wir alle einfach so gehandelt und es schien uns nicht unanständig zu sein. In einer hoffnungslosen Situation können Sie also erneut versuchen, solche manuellen Arbeiten auszuführen ...

Im Allgemeinen erfüllt die Migration von E-Mails von Server zu Server, indem E-Mails nacheinander in das Repository importiert und anschließend über IMAP auf einen neuen Server exportiert werden, alle grundlegenden Kriterien für die Arbeit mit Programmen. Diese Kriterien sind klare Logik, Sicherheit, Automatisierung und die Verfügbarkeit einer großen Anzahl vorgefertigter Tools, die die Aufgabe für Sie erledigen können. Daher hoffe ich, dass meine Notiz für jemanden nützlich ist und mein Leben vereinfacht, wenn die Buchhaltung oder die Planungsabteilung sie plötzlich dringend von Yandex nach Mail.ru, von Google nach Yahoo! oder irgendwo anders, wo der Chef plötzlich mit der Verlagerung von Postaufträgen beschäftigt war. Lassen Sie sich nicht ermüden, Kollegen!
