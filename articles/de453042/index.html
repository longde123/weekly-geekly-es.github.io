Häufig gestellte Fragen zu Windows Terminal Build 2019

Letzte Woche hielt Microsoft seine Build 2019-Konferenz im Washington State Convention Center in Seattle ab . Build ist eine große Veranstaltung, an der mehrere tausend Menschen aus der ganzen Welt teilnehmen, um alles über die aktuelle, neueste und zukünftige entwicklerorientierte Technologie von Microsoft zu erfahren.


Wir hatten das Vergnügen, so viele von Ihnen an unserem Stand zu treffen und alle Ihre Fragen zu beantworten!



Häufig gestellte Fragen


Im Folgenden finden Sie einige der häufigsten Fragen, die wir gehört haben (und noch hören), sowie offizielle Antworten:


Wann und wo kann ich das neue Terminal bekommen?


  1. Sie können den Terminal-Quellcode von GitHub unter https://github.com/microsoft/terminal klonen und auf Ihrem Computer erstellen.
    a. Hinweis: Lesen und befolgen Sie unbedingt die Anweisungen auf der Repo-README-Seite, bevor Sie versuchen, das Projekt zu erstellen. Es sind einige Voraussetzungen und Initialisierungsschritte erforderlich, bevor das Projekt erstellt wird!
  2. Das Terminal als Vorschau wird ab Sommer 2019 im Microsoft Store zum Download verfügbar sein.
  3. Wir beabsichtigen, Windows Terminal v1.0 bis Ende 2019 bereitzustellen, arbeiten jedoch mit der Community zusammen, um v1.0 bereitzustellen, wenn das Terminal von ausreichend hoher Qualität ist.

Ersetzt Windows Terminal die Eingabeaufforderung und / oder PowerShell?


Um diese Frage zu beantworten, klären wir einige Begriffe und Konzepte:


  • Eingabeaufforderung und PowerShell (wie WSL / bash / usw. unter * NIX) sind Shells, keine Terminals und haben keine eigene Benutzeroberfläche
  • Wenn eine Befehlszeilen-Shell / App / Tool ausgeführt wird, startet Windows diese automatisch und verbindet sie mit Windows-Konsoleninstanzen (falls erforderlich).
  • Windows Console ist die standardmäßige „terminalähnliche“ UI-Anwendung, die in Windows ausgeliefert wird und die Benutzer in den letzten 30 Jahren beim Ausführen von Befehlszeilentools unter Windows NT, 2000, XP, Vista, 7, 8 und 10 verwendet haben


Daher wird die Frage höchstwahrscheinlich erneut gestellt: "Ersetzt Windows Terminal die Konsole?"


Die Antwort lautet "Nein":


  • Die Windows-Konsole wird auch in den kommenden Jahrzehnten in Windows ausgeliefert, um die Abwärtskompatibilität mit den vielen Millionen vorhandenen / älteren Befehlszeilenskripten, Apps und Tools sicherzustellen
  • Windows Terminal wird neben der Konsole ausgeführt, wird jedoch wahrscheinlich das bevorzugte Tool für Benutzer sein, die Befehlszeilentools unter Windows ausführen möchten
  • Windows Terminal kann eine Verbindung zu Eingabeaufforderung und PowerShell sowie zu jeder anderen Befehlszeilen-Shell / Tool / App herstellen. Sie können unabhängige Registerkarten öffnen, die mit Eingabeaufforderung, PowerShell, Bash (über WSL oder SSH) und anderen Shells / Tools Ihrer Wahl verbunden sind

Wann kann ich die neue Schriftart bekommen?


Bald! Wir haben keine festgelegte Zeitleiste, arbeiten aber eifrig daran, die Schriftart fertigzustellen. Sobald es veröffentlicht werden kann, wird es in seinem eigenen Repository als Open-Source-Version bereitgestellt.
