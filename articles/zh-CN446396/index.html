GPS周计数器重置



会发生什么？


在2019年4月6日，第二个GPS周翻转计数器将重置为零，第一个是1999年8月21日。这意味着在GPS卫星传输的导航数据帧中，第10周将在第1023周之后显示，在GPS周号下为导航消息分配了10位值（如果考虑到每年大约52周，则1024周还不到20年）。

这将如何影响消费者？


提供此事件的软件中的接收器将继续从导航框中正确解码当前的GPS周编号，对于消费者而言，周编号将继续增加，即零值将自动解释为2048,1会自动解释为2049，依此类推。 但是，如果接收者没有为此做好准备，那么它将回到过去（在1999年甚至在1980年）。

可以/应该做什么？


检查您的导航接收器中是否安装了当前固件版本，如下 大多数制造商都知道这种情况，并进行了软件调整，以使消费者没有任何问题。

请与接收器或软件相关的制造商联系，这些数据取决于消费者。

在消费者端添加拐杖也很有意义-例如，添加支票： 

if ( week < 1023 ) week += 2048;
