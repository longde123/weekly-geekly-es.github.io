Refleksi tentang Hosting Surya untuk Lebah


Semuanya dimulai dengan undian ... undian antara peternak lebah dengan imbalan cerita lucu - mengapa mereka membutuhkannya.


Kemudian kecoak di kepala saya mengambil kendali dan dengan cepat memutar pesan yang menyatakan bahwa saya membutuhkan sarang ini bukan untuk lebah, tetapi untuk menempatkan server pemantauan di sana ;-)


Lebih jauh, fantasi itu menarik bilah Raspberry alih-alih bingkai dengan sarang madu, tetapi ternyata solusi seperti itu sudah ada (dalam foto di atas).


Bahkan, saya mulai berpikir tentang perlunya server web dengan database RRD dari saat publikasi pertama tentang topik pemantauan lebah empat bulan lalu.


Sekarang setelah buah pertama sudah ada, kebutuhan akan server seperti itu menjadi semakin mendesak.


Sebenarnya tentang hal itu juga artikel ke 13 tentang Habr menceritakan.


Distribusi biaya hosting di Ukraina adalah sebagai berikut - untuk $ 30 setahun, Anda bisa mendapatkan pendaftaran nama domain gratis dan server web dengan disk virtual 4GB.


Jadi, untuk mengikat angka-angka ini sehubungan dengan tugas saya - bahkan jika Anda menulis hasil transformasi Fourier empat kali per jam, itu akan keluar sekitar satu kilobyte.


Akibatnya, pangkalan 4GB akan dapat memuat informasi sekitar 400 sarang per tahun.


Untuk memulainya, tampaknya baik-baik saja, tetapi ada satu TETAPI - tidak semua ruang akan diberikan kepada Anda di bawah pangkalan (biasanya hanya seperempat).


Jika Anda sedikit meningkatkan nafsu makan, label harga segera melewati batas seratus dolar - agak berminyak untuk proyek gratis.



Singkatnya, di sini katak sudah membentuk koalisi dengan kecoak dan hal-hal google seperti itu.


Apalagi, untuk seratus meter persegi Anda hanya bisa membeli empat raspberry.


Tapi Tuhan, sungguh mengacaukan mereka, menyelesaikan dan menciptakan sesuatu!


Solusinya harus sesederhana mungkin, mudah dibawa ke hosting normal dan dilindungi dari gangguan daya dan gangguan internet.


Faktanya, sekitar 15 tahun yang lalu saya sudah berurusan dengan organisasi server web hosting di rumah, jadi saya tidak melihat masalah dalam meneruskan domain dan IP.


Jadi, solusi saya untuk masalah memilih platform adalah motherboard berbasis dual-core Celeron J1800 2,4 GHz dengan TDP 10W, tapi setidaknya yang ini:



Setelah mengemas semuanya ke dalam case nettop, saya mendapatkan sistem yang sangat ringkas.


Server dapat diputar pada drive SSD, dan cadangan ke HDD 2,5 "klasik.


Tambahan tambahan - di banyak nettop skema daya asli digunakan - "notebook" PSU dan konverter di dalam unit sistem.


Dan jadi kami sampai pada bagian "cerah" dari cerita.


Tidak, masalahnya bukan menginstal UPS, bahkan yang terkecil akan dapat "menarik" sistem seperti itu selama berjam-jam, tetapi dalam keinginan tersembunyi untuk membuat server mandiri yang tidak terhubung sama sekali dengan kabel (ya, sarang yang sama di bidang yang bersih ;-).



Secara umum, baterai surya 100-110W harus cukup, dipasangkan dengan baterai dari Tavria dan pengontrol muatan, ini akan berfungsi sebagai alternatif yang sangat baik untuk stopkontak di dinding.


Masalah internet? Ada 100Mbit internet di rumah, dan 4G di Kiev, Tuhan melarang semua orang (Anda tidak berpikir bahwa saya benar-benar meletakkan segala sesuatu di lapangan ;-)


Saya tidak menyentuh masalah perangkat lunak karena dua alasan:


  1. Ini adalah topik holivar terpisah
  2. Ya, dan Anda tidak harus memilih sebenarnya - lihat apa yang digunakan hoster, yang akhirnya akan saya datangi, dan masukkan hal yang sama (dari keluarga Linux)

Singkatnya, konfigurasi server adalah Celeron J1800 2-core 2.4GHz, 4GB (2x2) DDR3 SO-DIMM, 32GB SSD-HD, 320GB HDD


Tahukah Anda apa yang paling menyenangkan dalam cerita ini?


Keju gratis! Semua komponen sudah tersedia dan kinerja / stabilitasnya diperiksa!


Saya berharap publikasi ketigabelas secara keseluruhan berhasil!
