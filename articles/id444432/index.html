Penggunaan Linux dan perangkat lunak sumber terbuka di lembaga pendidikan kita: menjadi atau tidak?

Selamat siang, Khabrovchane sayang. Baru-baru ini, saya mulai khawatir tentang pertanyaan: untuk berapa lama monopoli Microsoft akan berlanjut di sektor pasar, yang bertanggung jawab untuk mengirimkan perangkat lunak ke banyak lembaga pendidikan di negara kita (pada kenyataannya, telah ditempati oleh korporasi sejak tahun 90an ).

Biarkan saya memberi Anda contoh konkret: Saya pergi ke lingkaran itu relatif populer untuk siswa sekolah menengah di universitas lokal N (universitas ini cukup bergengsi di kota saya), di sana kami mempelajari sejumlah disiplin ilmu:

  1. "Dasar-dasar Informatika dan Teknik Komputer" (sudah selesai).
  2. "Perangkat ... komputer": OS, "internal" PC (selesai).
  3. "Work on the PC": dasar-dasar Windows Explorer, CMD, MS Office (baru saja selesai).
  4. "Dasar-dasar pemrograman": PascalABC.NET (terima kasih Tuhan, setidaknya bukan Turbo Pascal).

Dan sekarang kritik pada sebagian besar poin:
1. Mungkin satu-satunya disiplin yang kurang lebih cocok untukku. Sistem angka, logika biner, dll. Di sini kami melakukannya tanpa menggunakan OS dan perangkat lunak apa pun, sehingga tidak memiliki hubungan langsung dengan posting saya.

2. Di sinilah kesenangan dimulai. Pertama, hampir semua yang kami lalui adalah sejarah dan kemampuan MS-DOS dan Windows (semua versi, pada Windows XP, karena presentasi dibuat pada awal nol tahun), secara sepintas menunjukkan keberadaan sistem seperti GNU / Linux, * nix dan ... itu saja. Saya tidak melihat sejarah dan kemampuan * nix yang dapat dipahami dalam presentasi ini. Kedua, dalam ceramah, kami berbicara lama tentang perbedaan antara Windows 3.11 dari 95 dan 98; ingat keberadaan mode IDE; bertemu FAT12 / 16/32 dan NTFS (meskipun dosen tidak benar-benar mengatakan apa pun kecuali mendekripsi singkatan NTFS), tetapi saya tidak mendengar sepatah kata pun tentang EXT2 / 3/4, BTRFS, UDF, bahkan tentang tata letak disk - diam! Situasi yang sama adalah tentang BSVV: mereka menjelaskan fungsi Legacy-BIOS, tetapi bahkan lupa tentang UEFI, belum lagi Coreboot dan implementasi BIOS lainnya. Pada saat yang sama, para guru memberi tahu kami tentang "kekuatan terbaru" Pentium 4 dan DDR2, usang floppy disk secara bertahap, dan keunggulan Windows XP dibandingkan ME dan 98 (sekali lagi, laporan 2003-2005).

3. Yah, semuanya cukup jelas: semuanya sudah usang dan / atau berpemilik. Saya tidak menemukan kesalahan pada karakteristik PC itu sendiri: 0,5-2 gigabyte RAM dan Pentium 4 / D sudah cukup untuk komputer klien, sama seperti kita memiliki Windows XP dan server file server pusat, yang sebagian membongkar beban pada HDD sistem. Tetapi saya dapat menemukan pada PC ini hanya beberapa program gratis dari ratusan: Notepad ++ dan versi OpenOffice yang sangat kuno. Yang lainnya: Google Chrome, MS Office 2007/2010 (tidak diaktifkan), Adobe PDF Reader, antivirus, kompiler berpemilik, Turbo Pascal ... Omong-omong, Kubuntu 14.04.x â€‹â€‹diinstal pada beberapa (setidaknya 1 GB RAM), karena instalasi tidak pernah Baik sistem maupun repositori tidak diperbarui. Dan mereka mengatakannya terus terang demi sebuah kutu: Saya belum pernah melihat seorang siswa pun menggunakan Linux. Selain itu, orang yang menginstal distribusi tidak menentukan kata sandi untuk root dan bootloader.

Oleh karena itu, Anda dapat dengan mudah menemukan item Mode Penyelamatan di GRUB2, secara otomatis mendapatkan hak root, remount root di baca / tulis, dan jalankan passwd. Sebenarnya, apa yang telah dilakukan pelayan rendah hati Anda dua kali. Dan semua (semua) OS ada di tangan saya sepenuhnya. Intinya adalah untuk kata sandi Windu, instal antivirus, konfigurasikan proxy, jika semua "perlindungan" ini tidak ada artinya! Tidak hanya itu, ketika saya memberi tahu guru tentang kerentanan ini, dia menjawab saya bahwa dia kurang tertarik dengan masalah ini. Tapi bagaimana caranya? Dilarang menggunakan usb-flash drive tanpa pemindaian anti-virus, kemudian mereka mengabaikan masalah yang ada di permukaan. Ngomong-ngomong, di Kubuntu ini praktis tidak ada perangkat lunak. Baik gcc, atau g ++, atau fpc, tidak lain adalah sistem basis.

Jika kita berbicara tentang kantor, maka penggunaan Libreoffice dan standar dokumen terbuka saya mengarah pada kenyataan bahwa sebagian besar file .odt tidak terbuka dengan benar dalam produk Microsoft, sebaliknya juga benar. Meskipun MS Office (IMHO) adalah produk yang sangat nyaman, Libreoffice, terutama sebagai proyek gratis, juga bagus. Pada prinsipnya, jika bukan karena kepatutan dan kedekatan MS Office, maka makna keberadaan alternatifnya akan diminimalkan (sekali lagi, menurut pendapat saya yang sederhana).

Terlepas dari kenyataan bahwa saya setuju dengan A.V. Stolyarov , saya memiliki pemahaman bahwa beberapa perangkat lunak penting hanya ada untuk Windows dan dalam beberapa tahun ke depan sistem ini pasti tidak akan diganti dalam bidang ini (misalnya, dari pemrograman beberapa mikrokontroler ke meluncurkan perangkat lunak untuk mengelola singkatan dari alignment roda), tetapi setiap tahun perangkat lunak tersebut menjadi semakin berkurang. Selain itu, ini adalah arahan yang sangat jarang, dan * nix tidak sia-sia dianggap sistem yang baik untuk mempelajari dasar-dasar pemrograman.

Moralitas dari dongeng ini Kesimpulan: sayangnya, saya perhatikan bahwa di universitas lokal (kemungkinan besar, tidak hanya dalam hal ini), preferensi diberikan kepada produk-produk berpemilik dan tertutup, bahkan jika mereka harus digunakan secara ilegal. Perangkat lunak lain sering diabaikan, karena guru tidak memiliki insentif untuk meningkatkan pengetahuan mereka yang sebenarnya dan pengetahuan siswa (â€œGuru tidak mau belajarâ€). Ia memunculkan kata-kata bahwa, secara umum, perangkat lunak bebas siap digunakan oleh pengguna, tetapi para pengguna itu sendiri tidak ingin menggunakannya dan / atau mengembangkan diri jika "benar".
