Mettre à jour le panneau de configuration pour CPaaS: réinventer l'interface utilisateur / UX et implémenter l'IDE de personne normale

Peut-être avez-vous déjà réussi à essayer le nouveau panneau alors qu'il était en version bêta ouverte: dans l'ancien panneau, il y avait un bouton spécial pour basculer. Quoi qu'il en soit, le nouveau panneau est désormais accessible à tous.

Et ce n'est pas seulement une version, mais une véritable refonte de la gestion des comptes UI / UX de Voximplant. J'invite tout le monde à couper - vous trouverez un aperçu des innovations les plus importantes.
Juste au cas où: les captures d'écran sont cliquables.


L'application de la toute-puissance


Le concept d '«application» n'est pas nouveau, il a été mis en œuvre lors du lancement de la plateforme. «Application» est un moyen de regrouper des scripts cloud, des numéros de téléphone, des files d'attente d'appels, etc.; il y a une référence explicite au terme "application" dans sa signification logicielle. La création d'applications dans Voximplant a toujours été un must pour une bonne gestion des appels.

Cependant, il était techniquement possible de créer une application à tout moment, par exemple lorsque tout le reste était prêt (un script a été écrit, un numéro a été acheté, etc.). Une telle flexibilité pourrait entraîner des conséquences ambiguës: d'une part, les applications sont requises sans faute, et d'autre part, vous pouvez facilement les oublier après avoir créé d'autres parties du traitement des appels.

Par conséquent, le principal changement concerne cette nuance particulière: la section Applications est désormais le principal point d'entrée dans la gestion des comptes. La section se trouve dans le menu principal, qui s'ouvre en cliquant sur le "hamburger" dans le coin supérieur gauche:


Dans le nouveau panneau, vous créez d'abord l'application, puis vous y tombez et déjà à l'intérieur vous voyez les sections familières: Historique des appels, Scénarios, Utilisateurs, Numéros, etc.

Nouveaux noms et lieux


Certains éléments ont été déplacés, certains sont renommés.

Pour commencer, il convient de noter que les anciennes règles sont désormais routées dans l'application.


Dans les paramètres généraux du compte - Paramètres - maintenant deux nouveaux noms, Sécurité et Webhooks.

La sécurité est une liste blanche IP pour accéder à l'API HTTP et au SIP, voir les onglets correspondants:



Webhooks - c'est ce que les callbacks HTTP sont maintenant appelés (ils étaient sur la page API Access):



De nouvelles fonctionnalités


Et maintenant quelque chose de complètement nouveau.

Les utilisateurs sont désormais créés uniquement dans les applications, tandis qu'un utilisateur ne peut appartenir qu'à une seule application. Petit point: si auparavant vous aviez un utilisateur dans votre compte lié à plusieurs applications, alors afin de ne pas ruiner la logique du travail, nous avons cloné cet utilisateur dans d'autres applications. Dans ce cas, l '"ancien" ID avec statistiques restera dans l'application à laquelle l'utilisateur était attaché pour la dernière fois. Les noms d'utilisateur doivent être uniques dans une seule application. Cela est dû aux modifications apportées à notre API à compter du 1er mars .

Les listes d'appels sont désormais dans le domaine public - une section de l'application leur est dédiée. Grâce à cette fonctionnalité, vous pouvez appeler un grand nombre de clients en mode semi-automatique et en indiquant le nombre de tentatives pour atteindre chaque numéro, l'intervalle entre les tentatives et le nombre maximal d'appels simultanés.



Paramètres de profil mis à jour - Paramètres de profil:

  1. Vous pouvez définir un avatar. Ce n'est pas la fonctionnalité la plus importante, mais il est toujours agréable d'avoir à portée de main une possibilité supplémentaire de personnalisation.


  2. Vous pouvez connecter des comptes OAuth: Github et Google sont disponibles. Maintenant, vous connecter à notre panneau sera encore plus pratique.

Une autre caractéristique est la gestion des sessions actives, des sessions actives; situé dans la section Appels du menu principal. Chaque fois que vous avez au moins une session active, vous pouvez la voir ici. Vous pouvez également en savoir plus sur cette session et la terminer manuellement.


La disposition des scripts a changé. Les scripts créés dans l'ancien panneau sont désormais partagés pour toutes les applications, mais chacune d'entre elles peut être liée à une seule application - cela entraînera la suppression du script du dossier partagé, c'est-à-dire deviendra indisponible pour d'autres applications. Les actions sont disponibles dans le menu contextuel, il vous suffit de cliquer avec le bouton droit sur le script.

Dernier point mais non le moindre, il s'agit de notre nouvel IDE, qui prend en charge la saisie semi-automatique, les touches de raccourci, le diff , le formatage automatique, etc. Toutes les commandes disponibles sont disponibles dans la palette (touche F1) ou dans le menu contextuel (clic droit dans l'éditeur). Nous avons également introduit un nouveau débogueur, dans lequel vous pouvez définir des paramètres de lancement et les utiliser à l'avenir.

Marketplace


Maintenant, le panel a des modèles pour le démarrage le plus rapide possible sur notre plate-forme - vous pouvez les étudier dès maintenant .


Vous pouvez sélectionner la solution la plus appropriée et cliquer sur Installer. Selon le choix, une application, des utilisateurs, une règle, etc. seront créés. Ensuite, vous pouvez essayer cette solution en action en exécutant la démo - le lien apparaîtra après l'installation.

Plus plus


Nous sommes convaincus que l'architecture mise à jour, associée à un puissant IDE, vous aidera à faire plus en moins de temps. Vous pouvez écrire des commentaires dans les commentaires ou directement avec notre support .
