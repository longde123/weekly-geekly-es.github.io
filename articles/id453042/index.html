Kapan dan di mana saya bisa mendapatkan Terminal baru?


  1. Anda dapat mengkloning kode sumber Terminal dari GitHub di https://github.com/microsoft/terminal dan membangunnya di mesin Anda.
    a. Catatan: Pastikan untuk membaca dan mengikuti instruksi pada halaman README repo sebelum mencoba membangun proyek - ada beberapa prasyarat dan langkah inisialisasi yang diperlukan sebelum proyek akan dibangun!
  2. Terminal sebagai pratinjau akan tersedia untuk diunduh dari Microsoft Store pada Musim Panas 2019.
  3. Kami bertujuan untuk memberikan Windows Terminal v1.0 pada akhir 2019, tetapi akan bekerja dengan komunitas untuk mengirimkan v1.0 ketika Terminal berkualitas tinggi.

Apakah Windows Terminal menggantikan Command Prompt dan / atau PowerShell?


Untuk menjawab pertanyaan ini, mari kita perjelas beberapa istilah dan konsep:


  • Command Prompt dan PowerShell (seperti WSL / bash / etc. Pada * NIX) adalah shell, bukan terminal, dan tidak memiliki UI sendiri.
  • Ketika shell / aplikasi / alat baris perintah dijalankan, Windows secara otomatis meluncurkan dan menghubungkannya ke instance Konsol Windows (jika perlu)
  • Windows Console adalah aplikasi UI "terminal-like" standar yang dikirimkan dalam Windows dan yang telah digunakan pengguna selama 30 tahun terakhir ketika menjalankan alat baris perintah pada Windows NT, 2000, XP, Vista, 7, 8 dan 10


Jadi, pertanyaannya kemungkinan besar dinyatakan kembali sebagai "Apakah Windows Terminal mengganti Console?"


Jawabannya adalah "Tidak":


  • Windows Console akan terus dikirimkan dalam Windows selama beberapa dekade mendatang untuk memastikan kompatibilitas dengan jutaan skrip perintah, aplikasi, dan alat yang ada / warisan sebelumnya.
  • Terminal Windows akan berjalan di samping Konsol, tetapi kemungkinan akan menjadi alat pilihan pilihan bagi pengguna yang ingin menjalankan alat baris perintah pada Windows
  • Terminal Windows dapat terhubung ke Command Prompt dan PowerShell, serta shell / tool / app baris perintah lainnya. Anda akan dapat membuka tab independen yang terhubung ke Command Prompt, PowerShell, bash (via WSL atau ssh), dan semua kerang / alat lain pilihan Anda
