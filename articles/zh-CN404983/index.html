如果备份不适合USB闪存驱动器，或者如何不购买外部硬盘

考虑到我收到的评论，我向新读者说明该文章是关于如何保存的，而不备份公共可用材料（文件，网页，印刷媒体），并且仅备份有关它们的文本信息（下载日期，链接，其他书目信息），以及确切的方法。


结果


要备份的文件大小减少了一个数量级。 例如，对于今天（中等强度的工作日），我获得了117 MB的内存，相当于备份了248 KB（大小减少了483倍）。


引言


最近几年，我从事文案写作（广义上来说），我的大部分数据都包括各种格式的文本文件和PDF文档（通常包括很多图形信息）。 我目前正在 Windows 7（320 GB硬盘）下的ASUS Eee PC 1003HAG上工作 。


对于备份数据，我很小心，但是我偏爱本地媒体。 我没有外部硬盘驱动器，但是有两个闪存驱动器：1 GB和2 GB。


问题


我开始遇到闪存驱动器空间不足的情况，但是到目前为止，我还没有购买新媒体的愿望。 基本上，因为真正需要备份的大多数信息仅占我文件的一小部分。


更具体地说，我只需要备份无法从任何地方恢复的信息-我的工作的实际结果：主要是我写的文字和有关我的来源的信息。 同时，通常，我不需要源本身（通常是PDF文档）的备份。 也就是说，事实证明，它们占据了我螺丝上的大部分空间。


解决方案


我的螺丝上有四个部分：


  • 系统（ C ），
  • 工作坊（ W ），
  • 文献（ L ），
  • Redist（ R ）。

我按以下方式分发文件：


  • 在W -基本上是不可恢复的文件（我的工作结果，无论是文本，来源，扫描还是任何其他数据类型）；
  • 关于L对工作有用的从根本上可以恢复的文件（出版物-印刷和电子版本的电子副本），
  • 在R -使用的软件的发行版，
  • 在C ，其他所有内容，包括娱乐文件。

对于W和L坚持严格的数据结构：


  • 在根目录中-仅目录，在目录中-仅文件或目录；
  • 我根据一组规则来命名文件和目录（特别是，我使用当前日期和时间作为唯一标识符，并使用两个字母的代码ISO 3166-1 alpha-2-这是一个单独讨论的主题）；
  • 首先，我将所有下载的文件放在W上的适当目录中（我不会尝试将文件“塞入”某处：如果我没有适当的目录，则创建一个目录）。

我选择PDF作为存储源的单一格式：保存原始文件，将文本文档转换为PDF，将浏览器中的网页和图像打印为PDF。


使用该软件， File Metadata将属性“ Web Page”（ System.Contact.WebPage ）和“导入日期”（Import Date）（ System.DateImported ）添加到Windows资源管理器中以获取PDF文件，我将在下载后立即为任何此类文件填写这些属性。 还添加了一个字段“ Comments”（ System.Comment ），在其中写入有关印刷出版物的书目信息。 还有更多字段：“任务状态”，“标题”，“作者”等。-我积极使用引文管理工具代替工作，但这是单独讨论的主题。 起初，我认为每次输入元数据都是很繁琐且耗时的，但是不，我已经习惯了。


最后，使用上述软件提供的上下文菜单项，将这些属性的值导出到一个特殊的标记XML文件，该文件的名称是程序从相应的PDF文件复制的。


瞧！ 我将PDF文件传输到L上的类似目录，并将XML文件保留在W上W


我将W保留在外部介质上。
