Di bawah kontrol: alat pemantauan disk server VDS

Di Timeweb, kami terus mengembangkan dan berbicara tentang server virtual.

Pada bulan September, kami memberi pelanggan VDS Evo kemampuan untuk memantau pemanfaatan ruang disk. Sekarang klien dapat memperoleh informasi tentang pemanfaatan ruang disk menggunakan widget di halaman server atau dari notifikasi melalui surat dan SMS.



Mengapa


Mungkin, semua orang menghadapi situasi di mana ruang disk akan segera berakhir. Log yang terlalu banyak, cadangan yang usang, atau kesalahan logis pada skrip menyebabkan operasi situs atau seluruh server menjadi salah.

Server virtual - produk di mana penyedia hosting tidak dengan sengaja memasukkan file klien. Anda memilih OS yang diperlukan (Ubuntu, Debian, CentOS, Windows Server, dll.), Dapatkan akses root dan mulai bekerja. Dan hosting mendukung kesehatan seluruh sistem (virtualisasi, jaringan, antarmuka), tanpa masuk ke detail proyek klien.

Namun, berulang kali, menerima aplikasi yang mengklaim bahwa situs tersebut tidak tersedia karena fakta bahwa disk server virtual penuh, muncul ide untuk melengkapi alat panel dengan apa, seperti yang sekarang tampaknya, harus dalam produk default - pemantauan disk.

Anda tidak perlu menginstal panel server tambahan untuk mengevaluasi alat yang diterapkan. Semuanya tersedia segera setelah membuat server pertama Anda.

Bagaimana cara kerjanya




Anda akan selalu tahu kapan ruang disk berakhir. Untuk melakukan ini, kami memungkinkan untuk menerima peringatan melalui email dan SMS.



Seiring proyek web Anda sendiri tumbuh, mau tidak mau mulai mengambil lebih banyak ruang karena mengisinya dengan konten. Log, cadangan juga dihasilkan, basis data bertambah, jadi penting untuk mengetahui secara tepat waktu kapan ruang disk hampir selesai.



Buka bagian "Pemberitahuan" dan tunjukkan apakah Anda ingin menerima pesan sekitar jam saat disk penuh atau pada waktu tertentu. Anda sendiri yang memutuskan apakah akan mengirim informasi semacam itu ke email atau SMS.



Bagaimana perkembangannya


Mereka memutuskan untuk menggunakan perpustakaan libguestf yang sudah jadi untuk mengumpulkan metrik, karena itu baik dalam hal keamanan dan dukungan untuk berbagai sistem file. Secara harfiah semua sistem file klien kami didukung, kecuali ReFS pada beberapa VDS pada OS Windows.

Tidak ada kesulitan dalam mengimplementasikannya, karena kami telah merencanakan fitur di muka dan menggambarkan implementasi dengan sangat rinci. Tapi itu tidak menyelamatkan kita.

Pada pertemuan tim pertama, ketika semua rincian diperhitungkan, banyak koreksi dan ide diterima dari para pengembang. Sebagai hasilnya, ini menghasilkan risalah panjang 13 poin, yang kami pertimbangkan dan kemudian implementasikan.

Pelepasan instrumen diadakan dalam 2 tahap. Pada awalnya, kami meluncurkannya kepada 10% pelanggan, dan seminggu kemudian kami membukanya untuk semua orang. Dalam 1 batch, kami hanya menerima 3 panggilan, di mana ada kecurigaan bahwa alat kami tidak berfungsi, tetapi dengan analisis individual kami menemukan bahwa tidak ada kesalahan, pengumpulan data berfungsi dengan benar.

Untuk meringkas


Sekarang alat diimplementasikan dan tersedia untuk semua pelanggan, bersama dengan alat yang biasa. Kami telah menerima ulasan pertama, tetapi kami berencana untuk terus mengumpulkan umpan balik, yang akan menambahkan fitur baru ke widget atau menambahkan elemen yang sama sekali baru untuk kenyamanan pelanggan yang lebih baik.

Hari ini kami ingin mencari tahu dari pembaca kami momen apa dalam kehidupan perusahaan hosting yang ingin Anda ketahui di publikasi mendatang?
