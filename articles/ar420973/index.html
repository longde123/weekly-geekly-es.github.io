تكوين UFW على خادم Ubuntu 16.04 مع مثال BigBlueButton

UFW (جدار حماية غير معقد) عبارة عن واجهة iptables مصممة لتبسيط عملية تكوين جدار الحماية. أداة iptables موثوقة ومرنة ، ولكن لن يكون من السهل على الوافد الجديد تكوينها بنفسه. أنا جديد على هذا العمل.

بشكل افتراضي ، يتم تمكين نهج ACCEPT في iptables ويبدو كما يلي:



في مثالنا ، لا توجد قيود. السياسة الافتراضية هي قبول.

INPUT - توصيلات الإدخال.
الإخراج - مركبات الإخراج.
إلى الأمام - التوجيه (على سبيل المثال ، من الضروري كتابة مسار إلى الكمبيوتر خلف البوابة).

المنطقة المحددة هي مكان قواعد المستخدم ، ويتم تسجيلها بالأوامر:

للمنفذ:

sudo ufw allow ssh (بواسطة اسم المنفذ)
sudo ufw allow 22 (حدد رقم المنفذ على وجه التحديد)
sudo ufw allow 80/udp (تحديدًا تحديدًا لرقم المنفذ وبروتوكول tcp أو udp)

لنطاق المنفذ: 

 sudo ufw allow 16384:32768/udp

إزالة القواعد: 

 sudo ufw delete allow 80

نحن مهتمون بالخطوط ذات xxx.xxx.xxx.x (xxx.xxx.xxx.x هو عنوان IP لخادمك) ، لأن هذا يشير إلى أن الخدمة تستمع على بطاقة الشبكة للطلبات الواردة من جميع الشبكات (0.0.0.0:*) . لا تهمنا الخطوط ذات 127.0.0.1 نظرًا لأن هذه هي الواجهة الداخلية ، لذلك نحن مهتمون أيضًا بالسطر التي تحتوي على 0.0.0.0 في العمود الرابع (مظلل بخط غامق) ، على سبيل المثال

tcp 0 0 0.0.0.0:80 0.0.0.0:* LISTEN 1359/nginx -g daemo

كما تقول أن المنفذ 80 مفتوح لجميع واجهات الخادم ويقبل الطلبات من جميع الشبكات.

لذلك ، وفقًا لهذا ، نضع القواعد:

sudo ufw allow 22 لـ ssh
sudo ufw allow 80 لخادم الويب nginx أو apache

عدد من قواعد التبديل الحر: 

 sudo ufw allow 5060 sudo ufw allow 5090 sudo ufw allow 2855 sudo ufw allow 2856 sudo ufw allow 5066 sudo ufw allow 8081 sudo ufw allow 8082

لخدمة العقدة:

sudo ufw allow 3000
بعد ذلك ، تحتاج إلى إلقاء نظرة على الوثائق ، حيث سنرى أنه من الضروري فتح المنافذ التالية:

sudo ufw allow 16384:32768/udp - للعمل WebRTC
sudo ufw allow 443
sudo ufw allow 1935

هذا يكمل إعداد ufw الأساسي لخادم BigBlueButton 2.0.

سأضيف فقط أنه من المرغوب فيه إما تعطيل ssh أو تغيير المنفذ ، أو السماح بهذا المنفذ فقط لشبكة معينة ، على سبيل المثال ، لشبكة شركتك.

على سبيل المثال ، تحتوي البوابة على واجهتين للشبكة ، واحدة ظاهرية (للمستخدمين) 192.168.3.2 ، والأخرى الحقيقية 105.xxx.xxx.xxx (إلى الموفر) ، هنا لقاعدة UFW التي نأخذها IP حقيقي ، أو تشير إلى الشبكة التي ينتمي إليها هذا الملكية الفكرية.

أولاً ، قم بتغيير المنفذ في ملف تكوين خادم SSH ، يمكنك استخدام أي محرر نصوص: 

 vim /etc/ssh/sshd_config

ثم نحذف القاعدة التي تسمح بالمنفذ 22:

Sudo ufw delete allow 22 - يسمى أحيانًا OpenSSH.

نقوم بإنشاء قاعدة تسمح بالاتصال بالمنفذ الجديد فقط من شبكة المؤسسة: 

 sudo ufw allow from 105.../26 to any port 4321

ثم نقوم بإعادة تشغيل خدمة ssh: 

 /etc/init.d/ssh restart

وانظر النتيجة 

 netstat -ntlp | grep LISTEN 

 tcp 0 0.0.0.0:4321 0.0.0.0:* listen 5676/sshd

وفي قواعد UFW سنرى من لديه حق الوصول إلى المنفذ: 

 ufw status 

 4321 ALLOW 105.xxx.xxx.xxx/26
