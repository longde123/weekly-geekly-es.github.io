Victor Gamov在jug.msk.ru上介绍Kafka Streams IQ

在2019年5月8日的莫斯科Java开发人员社区jug.msk.ru的会议上， Victor Gamov谈到了Kafka Streams IQ （ 交互式查询 ）。



关于演讲者


Victor在jug.msk.ru上第四次发言 ，他先前的报道是关于Nashorn （ 2014年11月13日 ）， Hazelcast （ 2016年5月4日 ）， Apache Kafka （ 2018年4月23日 ）。 与一年前一样，他目前在Confluent工作 ，但现在是Developer Advocate 。

流行播客Debriefing的共同创始人和主持人， Enterprise Web Development一书的合著者，许多会议（包括JUG.ru集团组织的会议）的演讲者。 Victor的网站 （那里有技术博客 ）， YouTube上的 视频报告 ， Speaker Deck and Notist ， Habr ， Twitter和GitHub上的演示文件。

自上次在jug.msk.ru上发表演讲以来发表的一些报道：

  • “ Kubernetes上的Kafka：创建基于云的流数据处理平台的神秘故事”（ DevOops 2018 ： 演示 ）
  • “ Java Puzzlers NG S03：什么？ 这些是从哪里来的？ （与Baruch Sadogursky一起讨论Oracle Code One 2018 ： 视频 ）
  • “使用Kafka流和Bintray Firehose API打击犯罪”（与Baruch Sadogursky一起讨论了Oracle Code One 2018 ： 视频 ， 演示 ）
  • “跨流：使用KStreams和KSQL重新思考流处理”（ DevNexus 2019 ： 演示 ）
  • “ Kubernetes上的Kafka：真的必须是“艰难之路”吗？” （ 2019年卡夫卡峰会 ： 介绍 ）

关于报告


传统上，在演讲者发言和会议开始之前，Andrei Kogun提醒即将举行的活动-Joker Java会议（10月25日至26日，圣彼得堡）的应用程序是开放的 ，而TechTrain IT节预计将第二次举办（圣彼得堡，24-25日） 8月）。



该报告致力于Kafka Streams API的一部分的Interactive Queries ，它提供了在不使用传统数据库和存储库的情况下访问应用程序的中间状态以及获得的结果的功能。



演示幻灯片显示与代码示例交替显示。 问了很多问题，Victor立刻给出了全面的答案， Twitter上的评论也证实了这一点。



在会议休息期间和会议之后，讨论都继续进行。 JUG.ru会议的一位发言人Ivan Ponomarev参加了会议。他最近在JPoint 2019会议上发表的报告“ Kafka Streams API：超越Hello World”（ 演示文稿 ，到目前为止，视频仅提供给与会人员）可以补充Victor的故事。
