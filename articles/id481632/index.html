Penyedia cloud: siapa yang terbaik di pasar?



Salah satu teman saya datang dengan ide untuk mengoptimalkan infrastruktur TI mereka dengan pergi ke cloud. Dan dia meminta saya untuk mewawancarai semua penyedia cloud untuk menemukan yang terbaik ... Berhenti. Saya bohong Tidak demikian. Dia tidak akrab, tetapi akrab, dan dia benar-benar tertarik pada awan. Tetapi hanya dia membutuhkan rekomendasi, kepada siapa harus berpaling. Dan saya dalam mencari informasi menemukan perbandingan artikel tentang HabrÃ© ( satu , dua , tiga ). Dan dia memutuskan bahwa akan menyenangkan untuk melakukan percobaan yang sama, karena itu adalah kesempatan seperti itu.

Ya, lalu saya pikir itu lucu. Saya tidak memperhatikan fakta bahwa penulis lain menghabiskan beberapa bulan untuk penelitian mereka. Pemuda naif Chukchi. Tapi, karena terlibat dalam pertempuran, dia memutuskan untuk tidak mundur. Bagaimana semuanya berakhir, baca terus.

Jadi, kembali ke intinya. Seorang teman memiliki klinik sendiri dengan banyak peralatan canggih untuk semua jenis ujian. Hasil penelitian - tomograms dan lainnya - gram dihasilkan setiap hari dalam volume besar. Dan mereka harus disimpan di suatu tempat dan ditangani entah bagaimana. Jika perlu, gambar diambil dari basis data operasional dan / atau arsip historis, hasilnya disimpan dalam basis data operasional. File sering memiliki volume yang sangat besar, sehingga beban yang serius ditempatkan di server. Untuk tugas-tugas rutin, server lain digunakan, lebih sederhana. Kenalan TI tidak terlalu berpengalaman, dia lebih dari seorang manajer.

Setelah memperkirakan semuanya secara kasar, saya menyadari bahwa 4 server sebenarnya diperlukan:

  1. Ini melayani input informasi utama, memelihara file karyawan dan jadwal kerja / penerimaan pasien, riwayat pasien, melihat gambar studi pasien dari database operasional dan arsip historis, surat perusahaan, dll. Pada kenyataannya, itu adalah server akses terminal untuk CRM medis.
  2. Ini layanan koneksi peralatan diagnostik dan workstation standar DICOM (transmisi terjadi melalui protokol IP), memperoleh gambar dari itu, pemrosesan awal dan penyimpanan dalam database operasional. Intinya, server diagnostik.
  3. Melayani arsip historis dari akumulasi gambar MRI, CT, EKG, dan studi lainnya. File gambar dapat berbagai ukuran, dengan jumlah atribut layanan yang berbeda (sekarang 32, dengan kemungkinan meningkat menjadi 256). Arsip harus bekerja berdasarkan prinsip perekaman satu kali, banyak tampilan dan ketidakmampuan untuk memodifikasi / menghapus file. Tingkatan penyimpanan minimal 10 tahun. Jumlah awal informasi yang disimpan adalah dari 10 terabyte dengan kemungkinan peningkatan bertahap yang direncanakan berdasarkan permintaan. Anda mungkin perlu penyimpanan objek. Ternyata ini adalah server arsip.
  4. Melayani perangkat lunak untuk analisis grafis gambar MRI, CT, EKG, dll. Digunakan untuk mempelajari penyakit, membuat diagnosis dan merumuskan rencana perawatan awal. Hasilnya disimpan dalam database operasional. Dalam kasus saya, seharusnya server grafis.

Mengirim semua ini ke penyedia, saya hampir pasti akan mendapatkan jawaban yang waras. Tapi kemudian penelitian itu ternyata begitu-begitu. Tidak ada angka pastinya. Setelah berpikir sedikit (setelah membaca banyak, banyak informasi tentang layanan cloud dan menginterogasi semua orang yang setidaknya sekali terkait dengan solusi cloud), saya merumuskan kriteria berikut:

  1. Server akses terminal.
    Win Server, 8 core 2,6 GHz, RAM 64, 300 GB disk. Cadangkan Untuk 30 pengguna. Menghubungkan pengguna - melalui Internet, VPN.
  2. Server Diagnostik
    Win Server, 8 core 2,6 GHz, RAM 64, 3000 GB drive: SSD semua drive, cadangan.
  3. Layanan arsip
    Anda memerlukan penyimpanan untuk arsip gambar yang terakumulasi 10 TB sekarang, dengan kemungkinan meningkat menjadi 300 TB (lebih disukai S3).
  4. Grafis
    Win Server, 8 core 2,6 GHz, RAM 128, disk 4000 GB, grafis 1 vGPU, memori video 8GB, cadangan
    + VPN IPSec 1 alamat ip, lebar saluran 50 Mb.

Model upah per jam diinginkan. Server keempat akan bekerja 4-8 jam sehari, pada malam hari waktu Moskow. Seharusnya ada fungsi menghidupkan / mematikan server dengan pembayaran hanya untuk waktu operasi yang sebenarnya.

Dengan angka-angka ini, saya beralih ke penyedia Rusia. Saya tidak mengambil yang asing, karena masalah ini berkaitan dengan obat-obatan. Dan ini adalah data pribadi yang harus disimpan dan diproses di Rusia. Secara total, saya mengirim 31 permintaan, dan 25 perusahaan menjawabnya.

Bagaimana penyedia dinilai


Teknologi ini mirip dengan yang digunakan sebelumnya. Dibandingkan layanan yang disediakan oleh penyedia berbeda, kemampuan untuk mendapatkan semua yang Anda butuhkan segera, serta total biaya.

Jika perusahaan tidak memiliki layanan, itu menerima minus.

++ Unggul - fungsionalitas atau solusi memenuhi atau melampaui harapan)
+ (Luar biasa - fungsionalitas atau solusi hadir dan memungkinkan Anda untuk menutup kebutuhan)
+ - (Medium - fungsionalitas atau solusi hadir sebagian, ada batasannya)
- (Buruk - fungsionalitas atau solusi hilang atau tidak dapat menutup kebutuhan)
- (Bedovo - fungsi atau solusi tidak dapat menutup kebutuhan dan bahkan menyentuhnya menakutkan)
? (Tidak ada informasi, tidak ada jawaban untuk pertanyaan).

Penafian
Saya harus mengatakan, ada nuansa - saya meminta perusahaan untuk tidak menelepon saya. Bukannya saya seorang yang introvert atau salah sasaran, saya hampir tuli. Saya dapat berbicara di telepon, tetapi dengan kesulitan besar dan dengan bantuan tambahan. peralatan. Dan momen ini ternyata benar-benar menjadi hambatan - mereka sangat ingin berbicara dengan saya. Mereka menelepon selama jam kerja dan di akhir pekan, sore hari dan dini hari. Tersapu oleh longsoran telepon, saya terpaksa menghubungkan teman-teman saya dengan proyek. Ketika mereka bisa, mereka menjawab panggilan. Tapi itu tidak selalu mungkin. Jadi saya minta maaf kepada semua orang yang tidak bisa lewat. Bukan kamu yang curang, akulah yang tidak bisa mendengarkan kamu.

Saya ingin menyoroti poin lain - penegakan aktual percakapan telepon. Para manajer bahkan tidak berpikir bahwa ada orang-orang cacat atau tidak komunikatif. Berkomunikasi dengan klien dengan cara yang nyaman baginya. Ini disebut fokus pelanggan. Di Eropa yang sama, saya pertama kali menulis kepada penjual beberapa kali melalui email sebelum ia menawarkan untuk mengadakan panggilan konferensi. Panggilan kami hampir dijamin jika Anda meninggalkan nomor telepon Anda di suatu tempat. Itu marah.
