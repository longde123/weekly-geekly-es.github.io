Mendistribusikan kembali jendela di antara monitor setelah bangun dari mode tidur

Anda memiliki 2-3 monitor, pada satu browser dan editor kode, pada manual kedua, pada email ketiga, Anda mengurutkan windows sesuai kebutuhan. Lalu kami pergi untuk menuangkan secangkir kopi, monitor dimatikan, Anda datang, menarik mouse dan tiba-tiba semua jendela berada di satu monitor utama, semuanya ada dalam tumpukan, Anda lagi melemparkan jendela dan situasi berulang setiap kali.

Topik ini dibahas pada banyak sumber, termasuk forum Microsoft, tetapi sulit untuk menemukan solusi dan deskripsi masalah yang jelas.

Masalah seperti itu mungkin tidak diamati oleh pemilik monitor tanpa komunikasi dua arah atau dengan DDC / CI dimatikan secara default (pada DELL sebelumnya ini tidak ada, dimatikan dalam menu). Jika Anda hanya memiliki satu monitor, maka windows-nya juga tidak dibangun kembali.

Sekali waktu, banyak pengguna mengeluh tentang masalah seperti itu di Windows, di mana, setelah mematikan monitor, windows tetap tidak terlihat, dan, tampaknya, para pengembang memutuskan untuk memperbaiki titik ini, jika tidak ada monitor, maka windows akan beralih ke salah satu yang dihidupkan.

Ketika monitor terhubung melalui DisplayPort, ketika masuk ke mode tidur, sistem berpikir bahwa itu tidak ada, semua jendela dilemparkan ketika mereka bangun ke monitor lain, dan ketika monitor utama ini terdeteksi setelah sepersekian detik, semua jendela terbang menjauh. Fenomena yang tidak menyenangkan.

Pengguna forum mengeluh tentang hal ini, beberapa ketika mereka mulai melakukan render sepanjang malam hanya mematikan kecerahan, beberapa mengatur layar splash dan mematikan monitor dengan batas waktu.

Beberapa orang bahkan di forum Microsoft meminta untuk membuat pengaturan yang, setelah mematikan monitor, jendela akan tetap di sana atau mereka akan dilemparkan ke tampilan lain.

Ada banyak solusi di sini, jika monitor mendukung penonaktifan DDC, maka Anda dapat melakukannya.
Anda dapat memutuskan kontak, tetapi ini juga semacam keputusan yang kejam.

Saya tidak memiliki kesempatan untuk melihat bagaimana itu pada kartu video Nvidia, tetapi pada AMD itu cukup untuk mematikan layanan AMD External Utility Utility, untuk Nvidia layanan Pembaruan NVIDIA Layanan Deamon dinonaktifkan.

Setelah manipulasi ini, Anda bahkan dapat mematikan monitor dengan tombol dan jendela tidak akan lari darinya, sangat nyaman bagi mereka yang memiliki banyak monitor dan perlu mematikan monitor untuk beberapa saat.

Namun saat seperti itu, saya memiliki DVI-HDMI yang terhubung ke monitor China dengan kabel Cina, dan selalu terlihat seperti ini sebelumnya, pada monitor kedua saya memiliki kode dan sesuatu untuk bekerja, tetapi pada 34-inci utama saya menonton film saat Saya duduk di sofa, saya tidak benar-benar ingin monitor kedua mengganggu saya, saya matikan dan windows tidak membangun kembali, tampaknya DDC / CI yang sama tidak diterapkan pada renda ini.

Tolong jangan menilai dengan ketat dan itu akan berguna jika Anda menambahkan opsi Anda sendiri untuk memecahkan masalah tersebut.

Terima kasih atas perhatian Anda, semoga ada yang berguna.

UPD: Ketika windows tidak dikumpulkan pada satu monitor, tetapi tetap menyala, Anda dapat menekan WIN + P dan pilih "hanya layar komputer" atau "hanya monitor kedua" daripada "memperluas".

Anda juga dapat memilih jendela melalui Alt + TAB, dan kemudian melalui MENANG + panah (kiri / kanan) memindahkannya ke monitor, tergantung pada lokasi monitor dalam pengaturan.
