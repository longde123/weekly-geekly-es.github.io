创建可启动的闪存驱动器


允许我们格式化USB闪存驱动器（从8 GB开始）并在其上写入引导加载程序（将允许我们的系统引导的实用程序）的实用程序称为BootDiskUtility。
这也很不错，因为它允许您下载所谓的“ Recovery HD”，这些微型图像可以让您在发生问题时恢复系统。 它们每个重约500 MB，因此从它们引导时，系统本身会从Apple服务器下载。 只有拥有本地网卡（可立即自行启动）的类似方法才对我们有效。 我没有遇到过这种情况，但是您可以在Internet上搜索哪些网卡不需要单独的工厂。 无论如何，都可以通过单击BDU主菜单中的“ DL Center”来下载所有Recovery HD。

从官方网站上下载它： tyk （小心，很多广告），将我们的闪存驱动器连接到计算机，在列表中选择它，然后单击“格式化磁盘”。 现在，我们的闪存驱动器应格式化并分为两部分。 在此实用程序的设置中，您可以选择Clover下的分区大小，其版本和其他一些参数。 200 MB和最新版本的引导程序将是最好的。
在“三叶草”部分中，我们还将驱动程序置于wifi适配器下。

Keksts（来自拉丁语内核扩展名=内核扩展名）是makoshi的特殊驱动程序，使该驱动程序在理论上不起作用，例如，声卡或网卡。

接下来，将以下一组兴趣点下载并放入三叶草/ kexts /其他文件夹中（我们不考虑FakeSMC，它最初存在于此）：

  • Lilu-其他kext的系统进程的补丁。
  • WhateverGreen是一家图形工厂。
  • USBInjectAll是USB工厂。
  • AppleALC是一家声音工厂。


其余的kekst由您自行决定。

其他kekst的简短列表以及为什么需要它们：
  • FakePCIID是一系列kexts，它们替换驱动程序标识符以确保硬件设备的正确运行。 在我用于hackintosh的那些机器上，不需要这些keksts，但是如果放了它，情况不会更糟。
  • HDMIAudio-用于通过HDMI传输音频。
  • ACPIBatteryManager-修复MacOS上的电池显示。

准备闪存驱动器的最后一步实际上是.hfs文件格式的MacOS映像上的固件。
这样做如下：
  • 从此处或从此处下载系统映像。 MacOS 10.14 Mojave已安装在我的计算机上，如果您的处理器是Sandy Bridge或更高版本，我建议使用它。 如果没有，那么10.13 High Sierra就是您的最佳选择。 从哪里获取图片并不重要；主要是它应该采用.hfs格式。
  • 再次，打开BDU，选择我们闪存驱动器的第二个（最大）部分，单击“还原分区”，在打开的窗口中，单击我们的映像并对其进行闪存。
  • 一切就绪，闪存驱动器已准备就绪。


下载并安装


因此，我们进入了行动的最后部分。
1.我们拿起闪存驱动器，将其插入计算机的USB2.0端口（这很重要，只有USB2.0）并从中启动。
2. Clover`a菜单应打开。 它看起来像这样：



3.通过选择第二行中的第二个图标来打开设置（无法使用鼠标，所有操作均通过键盘执行）。
4.我们编写以下引导参数（引导参数）-神奇的词，为引导加载程序提供一些说明：
kext-dev-mode = 1 -v
该行的第一部分允许您在系统中安装kexts，第二部分-以文本模式开始下载。 原则上，这应该足够了。 另外，存在以下引导参数：
  • rootless = 0-禁用系统完整性保护，无论它是什么-自MacOS 10.11 El Capitan以来，我们决不使用此参数
  • nv_disable = 1-与nVidia的视频卡的首次启动有关。 禁用图形外壳
  • nvda_drv = 1-相反，它包括一个图形外壳。 在已安装驱动程序的情况下，用于nVidia图形的第二次和后续启动。
  • –Disablegfxfirmware-仅激活集成显卡。
  • debug = 0x100-如果在引导过程中发生崩溃，它将停止系统，从而阻止重新引导。 帮助找出错误之前的最后几行。
  • keepsyms = 1-引导时输出更详细的信息
  • cpus = x（而不是x号）-激活指定数量的处理器内核


5.在引导加载程序的“图形注入器”部分中，您可以替换视频卡ID。 第一次没有意义时，它可以那样工作。 如果失败，则可以尝试更改ID。 具体在哪-在您可以在网站上找到applelife.ru之前。 最近，由于某种原因，它不起作用，因此我无法提供链接。 在某些情况下，视频卡的伪造ID可以是任何东西，默认ID除外。

6.单击“从***引导MacOS安装”。 系统开始启动。
如果一切正常，请按照屏幕上的说明进行操作，在“磁盘工具”中将磁盘格式化为“ Macintosh HD”，然后开始安装。
如果一切都不太顺利，请记住在屏幕上加载时出现的最后一行，并将其驱动到搜索引擎中。 我和我所有的黑客朋友都走过了这条路。
7.安装后，系统将自行重启，重复步骤1-5，但是现在单击“从Macintosh HD引导MacOS”。 安装将继续，结果您将进入系统。 但是该系统不如前者，因此您所使用的引导程序位于USB闪存驱动器上。

系统设定


1.运行WiFi适配器的驱动程序以使Internet工作
2.将config.plist，kexts和驱动程序从USB闪存驱动器复制到计算机，然后禁用USB闪存驱动器本身。
3. 从这里下载并安装Clover。 在安装过程中，为UEFI BIOS配置Clover。
4.现在，在Finder中，您可以看到带有Clover的EFI部分。 我们将kexts，驱动程序和config.plist放在此处，就像它们在闪存驱动器上一样。
5.下载Clover Configurator程序以编辑config.plist文件。 确切需要编辑的内容-我不会说，按照前言中的链接阅读。 我只是不想重复几次相同的信息。
6.重新启动以验证性能。

细微差别/错误解决方案的集合


  1. 安装系统10.13、10.14、10.15时，需要Drivers64UEFI文件夹中的apfs.efi驱动程序。 如果缺少该文件，则在安装的第一阶段完成后，您将无法继续进行安装-Clover将看不到带有系统的磁盘。
  2. 确保在文本模式（-v参数）下使用引导，直到确定系统运行正常。
  3. 最好不要在Clover`e中打勾未知参数。
  4. 我遇到的错误如下：将hack安装在硬盘驱动器本身上时，系统会在完成前几分钟保持静止状态。 我为此忍受了很长时间，直到尝试使用旧版本的三叶草。
  5. 无需破解SSD，该系统运行速度非常快，并且可以在7200 rpm的普通硬盘上运行。
