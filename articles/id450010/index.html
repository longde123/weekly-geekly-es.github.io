Penyimpanan, cadangan, dan katalogisasi foto

Di sini secara berkala menulis posting tentang cara menyimpan dan membuat cadangan foto mereka - well, hanya file. Dalam posting terakhir seperti itu saya menulis komentar yang agak panjang, berpikir sedikit dan memutuskan untuk menyebarkannya di posting tersebut. Apalagi saya sedikit mengubah metode backup ke cloud, semoga bermanfaat bagi seseorang.

Server rumah, di mana banyak hal berikut terjadi:



Apa yang harus disimpan?


Yang paling penting dan banyak adalah foto. Kadang-kadang video, tetapi sangat kadang-kadang - ini memakan terlalu banyak ruang dan terlalu banyak waktu, karena saya tidak terlalu menyukainya, saya hanya merekam video pendek yang berada di tumpukan yang sama dengan foto. Saat ini, arsip foto menempati sekitar 1,6 terabyte dan tumbuh di suatu tempat sekitar 200 gigabytes per tahun. Hal-hal penting lainnya jauh lebih sedikit dan ada lebih sedikit masalah dengan mereka dalam hal penyimpanan dan cadangan, selusin atau dua gigabyte dapat dijejalkan ke banyak tempat gratis atau sangat murah, mulai dari DVD dan diakhiri dengan flash drive dan awan.

Bagaimana cara disimpan dan dicadangkan?


Seluruh arsip foto saya sekitar 1,6 terabyte saat ini. Salinan master disimpan pada SSD dua-terabyte di komputer rumah. Pada kartu memori, saya mencoba untuk tidak menyimpan foto lebih lama dari yang diperlukan, pada kesempatan pertama saya menggabungkannya ke desktop atau laptop (saat di jalan). Meskipun saya tidak menghapus flash drive, jika masih ada ruang. Salinan tambahan tidak pernah sakit. Dari laptop setibanya di rumah, semuanya juga turun di desktop.



Salinan harian folder dengan foto dibuat di server rumah (dengan jenis cermin berdasarkan Drivepool, di mana duplikasi folder penting dikonfigurasi). Omong-omong, saya masih merekomendasikan Drivepool - selama bertahun-tahun menggunakan satu kesalahan. Itu hanya bekerja. Anda tidak hanya harus melihat antarmuka Rusia-nya, saya mengirim terjemahan yang lebih layak kepada pengembang, tetapi saya tidak tahu kapan itu akan diterapkan ( pembaruan : mereka mengatakan mereka akan mencoba dalam rilis berikutnya). Sementara itu, dalam bahasa Rusia, ini adalah program untuk mengelola kolam (kelola kolam).



Anda dapat, tentu saja, membuat salinan lebih sering, jika banyak pekerjaan telah dilakukan dalam sehari, maka saya dapat dengan paksa menjalankan tugas. Meskipun sekarang saya masih berpikir untuk mulai menyalin pada mengubah file, saya ingin berhenti menjaga desktop dihidupkan sepanjang waktu, biarkan server bekerja lebih banyak. Program ini adalah GoodSync.



Sampai baru-baru ini, file dari desktop GoodSync yang sama diunggah ke cloud Onedrive. Sebagian besar file yang saya miliki bukan pribadi, jadi saya mengunggahnya apa adanya, tanpa enkripsi. Apa yang bersifat pribadi diunggah oleh tugas terpisah, dengan enkripsi.

Onedrive dipilih karena langganan Office 365 Home Premium untuk 2000 per tahun menghasilkan lima (dan sekarang enam) terabyte di cloud. Bahkan dengan potongan per terabyte. Sekarang, bagaimanapun, freebie telah naik harga sedikit, tetapi beberapa minggu yang lalu masih ada opsi untuk 2600-2700 setahun (Anda perlu mencari pengecer). Saya melihat ini, ketika tahun lalu MS menaikkan harga, dan bahkan berhenti menjual langganan di situs, karena saya mengaktifkan langganan selama lima tahun ke depan, masih ada 1800-2000 kotak yang dijual (tentu saja, ada juga beberapa kotak sebagai cadangan) untuk mengambil, tapi saya tidak berani berpikir sejauh ini).



Kecepatan unduhan adalah maksimum untuk tarif saya, 4-5 megabyte / detik., Pada malam hari hingga 10. Saya melihat crashplan pada satu waktu - itu bagus di sana jika megabyte per detik dimuat.

5TB seumur hidup seharga $ 2-3 dengan ebay - hal yang sangat acak. Untuk rentang hidup bisa sangat singkat, sejauh ini tiga bulan adalah rekor. Ini bukan masalah mencadangkan ke tempat yang dapat dicakup kapan saja. Bahkan untuk sepeser pun.



Tetapi sekarang, karena dia memutuskan untuk menyeret sebagian tugas dari desktop ke server, menyalin ke Onedrive ditransfer ke Duplicati. Biarlah beta, tapi saya sudah menggunakannya selama beberapa bulan dan sejauh ini sudah berfungsi cukup stabil. Karena Duplicati masih menyimpan cadangannya dalam arsip, dan tidak dalam jumlah besar, saya memutuskan untuk mengenkripsi semua yang diunduh dengan alat bawaan. Semuanya sama untuk memulihkan, jika itu, melalui Duplicati, itu perlu. Jadi biarkan semuanya mengenkripsi.

Mengingat bahwa saya memiliki terabyte berkeping-keping, cadangan ke cloud terdiri dari beberapa tugas. Di sinilah cadangan diunggah ke cloud. 2019 banjir dengan cepat - ada lima puluh foto dalam beberapa hari, saya telah melakukan perjalanan sejauh ini, dan 2018 perlahan-lahan mengalir. Kecepatan unduhan saat ini bukan hari maksimum, saluran dimuat dan semua itu.



Di cloud, folder cadangan terlihat seperti ini - ada banyak arsip zip, ukuran arsip disesuaikan saat membuat tugas:



Sekitar sebulan sekali saya membuat salinan di drive eksternal, yang disimpan di lemari. Saya terhubung dan menjalankan tugas dengan tangan saya di GoodSync yang sama. Meskipun, tentu saja, dimungkinkan untuk mengatur peluncuran saat menghubungkan disk - tetapi saya tidak selalu perlu membuat salinan ketika saya menghubungkan disk.

Dengan cara yang baik, Anda akan memerlukan lokasi penyimpanan jauh yang lain - milik Anda dan tidak terlalu berawan. Di server saya, yang terletak di platform penyedia, saya sudah menyiapkan disk untuk bisnis ini sejak lama, tetapi semua tangan tidak dapat menjangkau. Tapi karena saya mulai menyeret semuanya di bawah duplikat, saya pikir saya akan melakukannya sekarang, setelah saya memuat ulang semuanya di onedrive.



Bagaimana cara katalog?


Di sini pertanyaan dibagi menjadi dua - tingkat sistem file, di mana katalogisasi berada pada tingkat folder dan katalogisasi logis untuk lebih banyak parameter, karena struktur folder masih terbatas dalam kemampuan.

Ya, saya mengambil foto di rabies. Karena Anda dapat membuat jpg dari mentah kapan saja, tetapi tidak sebaliknya. Setelah saya memotret dengan jpg + mentah, sehingga saya dapat dengan cepat mengunggah foto ke ponsel saya dan mengirimnya ke Internet (sulit untuk mentransfer mentah ke ponsel saya). jpg kemudian terhapus saat menyalin ke desktop. Tapi sekarang telepon mulai cocok untuk saya dalam hal kualitas foto (untuk mengunggah ke Internet), oleh karena itu saya sepenuhnya menolak jpg di kamera. Tetap dari saat-saat ketika saya tidak punya mirrorless, atau datang dari telepon.



Pada tingkat sistem file, tampilannya kira-kira seperti ini: di bagian atas folder, sumbernya. Nama-nama fotografer biasanya.



Level yang lebih rendah adalah topik. Semua kurang lebih sama, mungkin ada tema pribadi (misalnya, "Anjing," beberapa topik mungkin tidak.



Berikutnya adalah tahun. Di dalam folder tahun demi hari. Mungkin ada sesi foto terpisah di folder jika foto mengalahkan topik per hari.

Akibatnya, jalur ke file mungkin terlihat seperti ini: My \ Trips \ 2018 \ 2018-04-11 Berlin \ French Station \ P4110029.ORF

Saya mengambil foto dengan dua kamera, biasanya secara bergantian, tetapi kadang-kadang saya mengambil keduanya - kemudian saya membuang foto dari mereka ke dalam satu folder. Yang utama adalah bahwa waktu harus disinkronkan, jika tidak maka Anda harus menghitung perbedaan dan menyesuaikan tanggal pemotretan untuk semua file (di lightroom itu sederhana, tetapi membosankan untuk mempertimbangkan perbedaan waktu).

Untuk foto dari ponsel ada folder terpisah di tingkat kedua, tetapi jika perlu, foto tersebut dapat dikirim ke folder tematik.

Katalog logis di atas folder - Adobe Lightroom . Tentu saja, ada banyak program untuk katalogisasi dan pemrosesan, tetapi lightroom cocok untuk saya, itu cukup menggembirakan (dan mereka bahkan memberikan photoshop dalam kit), dan selama beberapa tahun terakhir ini juga menjadi kurang lambat. Meskipun, tentu saja, ada juga transisi lengkap ke SSD yang membantu.

Semua foto hidup dalam satu direktori. Struktur folder dari paragraf sebelumnya digunakan terutama, di atasnya adalah informasi EXIF, geo-tag, tag dan tanda warna. Anda juga dapat mengaktifkan pengenalan wajah, tetapi saya tidak menggunakannya.

Berdasarkan semua hal di atas, Anda dapat membuat "koleksi pintar" - pilihan dinamis untuk properti file tertentu - dari parameter pemotretan hingga teks dalam komentar.



Semua tag disimpan dalam file, riwayat pengeditan dalam file XMP di sebelah parit. Katalog lightroom didukung oleh lightroom itu sendiri seminggu sekali ke folder tertentu, dari mana ia kemudian jatuh ke onedrive. Nah, ditambah, melalui agen veeam, disk desktop desktop dituangkan ke server setiap hari - dan direktori disimpan pada disk sistem.

Dan bagaimana dengan semua foto? Apa, tidak ada jenis file lain?


Ada, kenapa tidak. Metode pencadangan tidak berbeda (jika Anda perlu mencadangkan sama sekali), dan metode pembuatan katalog tergantung pada jenis konten.

Pada dasarnya, pengurutan pada tingkat folder sudah cukup, tag tidak diperlukan. Hanya untuk film dan acara TV, digunakan kataloger terpisah. - Server Media Plex. Dia juga server media, seperti namanya. Tapi di sana kuda itu tidak tergeletak, biasanya disortir dengan baik jika seperempat perpustakaan film, dan sisanya berada di folder "! Untuk mengurutkan".
