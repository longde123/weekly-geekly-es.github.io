Cómo un administrador de Linux manejó un coro de niños usando un sistema de reconocimiento de música en Ubuntu 16.04. Micrófono y Acordes

¿Qué necesita el verano para divertirse? Necesito un poco de música! Pero si la música se graba con notas en papel, y usted es un administrador ordinario del sistema y se le asignó la tarea de crear un coro infantil, entonces, como una de las soluciones a este problema, presento varios programas útiles. Están escritos en Java y están disponibles tanto en el repositorio Ubuntu 16.04 como en el Github para resolver el problema del reconocimiento automático de notas, crear partes musicales, descargar y editar el producto musical resultante en formatos populares y distribuir archivos (así como golpes de cabeza) a artistas individuales. El tema, espero, será interesante para los lectores de nuestro blog.





Imagine la situación real de un campamento infantil. Era necesario reunir rápidamente al coro y enseñar a los niños una nueva canción, pero no había nadie que quisiera el puesto de director musical estipulado por la mesa de personal. Preguntó al administrador del sistema local. Y se fue!

Al tener una computadora portátil con Ubuntu 16.04 instalado, el administrador resolvió el problema de la siguiente manera.

1. Instale el potente editor de música gratuito MuseScore .

Vista general con una canción infantil abierta "My Russia".



Puedes abrir un archivo MIDI:



Funciona con archivos de karaoke.




Si alguien tiene un teclado MIDI, también se puede conectar a un programa para editar e ingresar notas.

2. Se tomaron varias puntuaciones populares. Los ejemplos están en las capturas de pantalla. Las canciones, por supuesto, son populares, ¡por eso ya se han roto, incluso en coro, incluso en solitario!

3. Pero vivimos en el siglo XXI. Se decidió intentar cantar algo nuevo. Pero resultó que divirtiéndose cantando para el programa, nadie conoce las notas. Entonces, ¿qué hace el administrador de Linux?

A. Ponga el sistema de reconocimiento OCR TESSERACT. Por ejemplo, desde aquí: https://github.com/tesseract-ocr/tesseract/wiki . Tenga cuidado, no necesita instalar 4.0, sino la versión anterior: 3.5.
Asegúrese de verificar la instalación de idiomas que se utilizan con precisión en obras musicales y notas (inglés, italiano, alemán, francés, ruso).

B. ¡Instala un increíble sistema de reconocimiento de notas! La única limitación es el escaneo de 300 DPI, no más de 3.5 MB, una página en PDF.
Necesitas ponerlo desde aquí: 

$> git clone https://github.com/Audiveris/audiveris.git cd audiveris gradlew.bat clean build ./gradlew clean build ./gradlew run ./gradlew run -PcmdLineArgs="your,arguments,here"

Hay suficientes puntajes predeterminados para trabajar con una o dos hojas de puntaje, pero para cosas de orquesta recomiendo construir y ejecutar al menos -Xmx = 11G.

Si todo está instalado correctamente, debe comenzar desde el directorio donde funcionaba el comando git clone.

La interfaz se ve así:



Necesitas usarlo así:

1. Abra el archivo PDF.

2. Libro -> libro Trancribe

3. Espere el reconocimiento de las notas.

Comenzando:



4. Exporte la música resultante XML 3.0 a través de Libro-> Exportar.

5. Abra el archivo en MuseScore y expórtelo a formato MP3.

Nosotros escuchamos y disfrutamos.

6. Si no puede esperar para grabar los pasajes de micrófono de los ruiseñores locales con falsete, entonces también está el programa de aplicación de micrófono Nootka para este caso.
Este es un buen entrenador vocal. Puede hacerte tocar escalas por notas, puede probar tu audición, puede ayudarte a afinar el instrumento. Muestra gráficamente cuán exactamente ingresas la nota y, si la obtienes, la dibuja en un instrumento musical. Por ejemplo, la misma melodía en mi actuación.
