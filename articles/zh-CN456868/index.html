V16 Update 2 Beta中的新3CX SBC远程连接管理器

上周，我们发布了下一个3CX v16 Update 2 BETA更新。 它包括从头开始重写的3CX SBC（会话边界控制器）远程连接管理器。

回想一下，如果两个或更多IP电话位于远程位置，建议使用3CX SBC将远程办公室连接到中央3CX PBX。 同时，如同到PBX的本地连接一样，SIP信号通过3CX服务器传递，媒体流（语音）直接在远程设备之间传递，而无需离开办公室。 这不仅提高了通信质量，而且减轻了外部通信信道的负担，将本地呼叫“留在”了本地网络中。 此外，3CX SBC通过单个TCP端口建立所有VoIP流量的通道，从而大大简化了边界路由器的配置。 SBC允许您远程管理IP电话-重新配置，更新固件等，还可以对远程办公室和3CX服务器之间的VoIP流量进行加密，以提供更多的隐私。

与V16 Update 2 Beta一起，我们还发布了俄语版的3CX快速入门指南 。 在交换中创建号码时，它会通过扩展名参数中指定的电子邮件发送给用户。

特色3CX SBC v16


新的3CX SBC v16具有以下功能：

  • 简化服务安装。 要安装SBC，只需指定3QX服务器FQDN和SBC隧道身份验证密码（3CX隧道技术）（请参阅下文）。
  • 服务管理。 现在可以在3CX界面中远程更新和重新配置SBC（请参见下文）。 SBC自动更新仅在Linux和Raspberry Pi上可用。
  • 集中监控所有连接的SBC。 对于每个SBC干线，管理员都会看到连接状态更改。 还可以使用该服务的统计信息，事件日志记录和电子邮件通知。

请注意，SBC v16仅适用于3CX V16 Update2。必须在3CX服务器上安装受信任的SSL证书（不支持自签名证书）。

安装3CX更新


除了3CX界面中的标准更新（仅适用于Linux）之外，您当然可以下载完整的系统发行版：

  • Windows： 3CX v16 Update 2 Beta
  • Linux： Debian 9 ISO上的3CX v16-在“ 3CX更新通道”屏幕上，选择Beta。
  • 在Debian和Raspberry Pi上手动安装3CX。

完成变更日志 。

配置3CX SBC连接


3CX SBC可以安装在最新版本的Raspberry Pi （Raspbian Linux）上，并且最多可以处理30个IP电话。 建议每个电话配置不超过10个BLF指示器，因为 状态消息下载服务。 但是，如果在推荐的硬件上的Windows或Debian Linux上安装SBC，则允许使用更多手机。

计划在其上安装SBC的计算机必须具有可用端口5060（TCP和UDP），并且不能是DNS，DHCP，SQL Server或类似的网络资源。 通常，不建议将此计算机用于其他任务。

来自服务器端3CX的SBC连接


在“ SIP中继”部分的“添加3CX SBC”按钮中创建一个新的SBC连接。



指示SBC连接的名称（不要使用特殊字符），例如HeadquartersToMoscowBranch。 然后，在连接设置页面上，复制身份验证ID和Web链接。 这些参数是在远程办公室中安装SBC时指定的。



考虑SBC连接的参数。

基本标签：

  • 连接信息-连接名称和隧道密码。
  • 网络信息-用于连接远程SBC的URL和ID。

统计标签：

  • 隧道状态，正常运行时间，软件版本，此连接的呼叫数和用户。

选项标签：

  • 隧道安全级别（TLS或TCP）
  • 第一RTP音频端口号是通过SBC的音频流使用的第一端口。
  • 启用容错-输入3CX备份服务器的外部IP地址。 如果主服务器发生故障，SBC将切换到备用服务器。
  • 日志记录和故障排除-选择日志记录级别。 不要设置“长时间的详细日志记录级别”-这会导致过载，尤其是在使用Raspberry Pi的情况下。

这些按钮在3CX SIP中继接口部分中也可用：

  • “推送配置”-将更新的配置发送到选定的SBC服务。
  • “更新”-将SBC服务更新为最新的稳定版本。

在远程办公室中安装SBC服务


在3CX中配置SBC连接后，可以在远程办公室中安装SBC。

在Debian和Raspbian Linux上安装SBC




从命令行安装：

稳定版 

wget http://downloads.3cx.com/downloads/sbc/3cxsbc.zip -O- | sudo bash

测试版 

 wget http://downloads.3cx.com/downloads/sbc/3cxsbcV16.zip  -O- | sudo bash



从ISO映像安装：

完成Linux安装后，在3CX Update Channel屏幕上，选择3CX SBC Stable或3CX SBC Beta。



然后从SBC连接设置中指定3CX服务器URL和身份验证ID。



现在，您可以连接远程IP电话 （连接模式-通过SBC连接）。

在Windows上安装SBC


运行3CX SBC Windows 版分发工具包 ，并在安装过程中指定服务器URL和身份验证ID。



在Linux上手动更新SBC（从先前版本升级）


更新APT存储库： 

 sudo apt update

更新到稳定版本： 

 sudo apt upgrade 3cxsbc

Beta更新： 
 sudo apt -t testing upgrade 3cxsbc

手动更新以前版本的SBC（仅Linux）时，相应的SBC连接会自动出现在3CX管理界面中。 选择一个连接，然后单击“发送配置”按钮。 更新配置后，您可以连接IP电话。
