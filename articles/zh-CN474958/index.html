你问我如何过这样的生活 ？ 好吧，你必须从远处开始。
事实是，根据情况的意愿，在过去的几年中，我被迫分析客户填写的纸质表格的电子副本，并将其结果包含在适当的数据库中。 最初的工作是艰苦而又忘恩负义的，如果不是在某种情况下，这可能会使任何人疯狂……

在某个时候，研究各种样本调查表中重复的标准部分的过程鼓励我尝试创建一个能够从各种字符的混搭中独立提取用户数据的系统。 嗯，不是完全在全自动机器上，但是无论如何，总比手动排序和键入数据要好。
我撰写的第一个此类程序仍然完全是自行编写和手工制作的-坦率地说，我仍然不敢解释为什么在大多数情况下仍能完成其任务。 然后，我无意间在广大的Internet上找到了适合我的框架 -就是说，该例程库用于执行所有必要的操作。 该框架的使用基于所有计算机科学家长期以来所熟悉的正则表达式，这使得可以在此基础上创建模式，并最终确定了文本要求的提取物的现成定义，即需要从文本区域中提取已经描述的模式的值的那些区域，将其替换为特定的文本。

怎么会更容易制定呢？ 好吧，你们中许多人可能已经看到了所谓的报告日志文件，看起来像这样：
127.0.0.1--[2009年6月10日：10：00：00 +0000]“ GET /example.html HTTP / 1.1” 200-“ example.com ”“ Mozilla / 4.0（兼容； MSIE 7.0； Windows NT 5.1 ）“

即使对于没有经验的用户，也可以很明显地看到，在某个时间点的某个网络地址（在这种情况下为本地），访问外部资源以便使用某种类型的浏览器和操作系统检索某些数据。 对不对 可以根据需要在报告文件中重复多次类似的行，以便根据这些行来区分可以分解的清晰结构（分为不同的语义结构）。 一名技术专家对其进行研究，提取有用的信息，并在此基础上得出有关特定程序，系统，用户操作等的必要结论。

实际上，实际上，存在用于这种结构的句法分解的大量工具。 根据我的需要调整了这些框架之一，我为自己扩展了功能，以允许从任意文本中无限制地选择这样的有序结构。 从代表简单垃圾的程序的角度来看，它们之间的空白可以用您喜欢的任何字符填充。

顺便说一句，我仍然想知道为什么以前没有在很多地方使用过这种简单的想法-至少，我找不到关于这种活动的任何提法，而且几乎没有关于我的发展的评论。 没错，我遇到了一个陌生的家伙，他向我发送了一封感谢信，并向我保证，从现在开始，每个人现在都可以借助这种废话来整理书籍的目录。 对我自己来说，我只是惊讶于我的方法如此不寻常的范围，并开始进一步思考此方法还可以做些什么。

这个新想法似乎已经变得更加疯狂-我开始开发一种工具，该工具已经能够独立确定任意文本中的所有有序和重复结构。 当然，这里还没有解决该问题的现成标准解决方案-必须连接神经网络，该网络通常是我们自己大脑的近似模型。 好吧，对人类大脑的各种品质而言，我们仍然遥遥无期-充其量，目前，我们只能以这种方式模仿蟑螂有意义的活动，仅此而已。 但是，正如他们所说，对此表示感谢。

然后在工作过程中，我突然想到了一个新的猜测：如果我们的生活发展经历了大致相同的情景，该怎么办？ 好吧，也就是说，从一开始，就有大约一个程序能够从最初的混乱字符集中提取未来原始生物的有意义的DNA序列，然后将它们组织成必要的结构？ 然后，在使其自身功能复杂化的过程中，该程序将源代码排序为越来越有组织的结构，直到它出现为止……我能够清楚地解释所有这些工作原理。

然后，在这里，最终，真正的含义出现在我身上，体现在所谓的“常规”的概念中（我们用行话称呼这些非常常规的表达式）。

如果您以前从未处理过此类设计，那么也许您无需开始。 对于没有经验的人来说，它们看起来太沉重且难以阅读，以至于无法从中分离出任何有用的信息。

虽然，也许我会告诉您最重要的功能之一。 这就是所谓的Kleene星号 （*），位于我们需要的任何字符序列之后，意味着该序列可以在我们文本中的任意位置出现此任意次数，包括一次。 实际上，一个天才的事物为重复结构的自我生成开辟了道路。 它以纪念美国著名数学家和逻辑学家斯蒂芬·克莱因（Stephen Kleene）的名字命名，而史蒂芬·克莱因（Stephen Kleene）实际上是自己发明了这些规则的。
