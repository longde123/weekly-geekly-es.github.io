GitLab membuat perubahan untuk pengguna produk cloud dan komersial



Pagi ini saya menerima surat dari GitLab tentang perubahan perjanjian layanan. Terjemahan dari surat ini akan di potong.

Terjemahan:

Pembaruan penting untuk Perjanjian Layanan dan Layanan Telemetri kami

Pengguna GitLab yang terhormat!

Kami telah memperbarui Perjanjian Layanan kami mengenai penggunaan layanan telemetri kami.

Pelanggan yang sudah ada menggunakan produk eksklusif kami (Gitlab.com dan layanan Edisi Perusahaan pada peralatan mereka) dapat, mulai dengan versi 12.4, lihat sisipan skrip js tambahan yang berinteraksi dengan GitLab dan / atau layanan telemetri pihak ketiga (misalnya, Pendo).

Untuk pengguna Gitlab.com: setelah peningkatan, Anda harus menerima Perjanjian Layanan baru kami. Akses ke antarmuka web dan API akan diblokir sampai penerimaan kondisi baru.
Ini dapat menyebabkan jeda dalam layanan melalui API kami untuk pelanggan yang menggunakan integrasi dengan API kami hingga persyaratan diterima setelah masuk melalui antarmuka web.

Untuk pengguna dengan perangkat keras mereka: GitLab Core tetap menjadi perangkat lunak gratis. GitLab Community Edition (CE) tetap menjadi pilihan bagus jika Anda ingin menginstal GitLab tanpa menggunakan perangkat lunak berpemilik. Ini dirilis di bawah lisensi MIT , dan tidak akan berisi perangkat lunak berpemilik. Banyak proyek open source menggunakan GitLab CE untuk kebutuhan SCM dan CI mereka. Sekali lagi, tidak akan ada perubahan pada GitLab CE.

Perubahan utama:

Gitlab.com (versi SaaS dari GitLab) dan paket eksklusif untuk instalasi sendiri (Starter, Premium dan Ultimate) sekarang akan menyertakan sisipan tambahan dalam skrip JavaScript (baik open source dan proprietary) untuk berinteraksi dengan GitLab dan, mungkin , dengan layanan telemetri pihak ketiga (kami akan menggunakan SaaS Pendo ).

Kami akan mengungkapkan semua opsi untuk penggunaan tersebut dalam kebijakan privasi kami, termasuk tujuan menerapkan data yang dikumpulkan. Kami juga akan memastikan bahwa setiap layanan telemetri pihak ketiga yang kami gunakan memiliki standar perlindungan data, setidaknya tidak lebih buruk daripada yang sudah digunakan di GitLab, dan kami akan berusaha keras untuk mematuhi SOC2. Pendo mematuhi SOC2.

Jika Anda memiliki pertanyaan, silakan hubungi kami di support@gitlab.com

Terima kasih

Tim Gitlab
