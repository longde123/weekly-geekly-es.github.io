Memperkenalkan Azure DevOps

Azure DevOps dirilis! Hari ini kita akan berbicara secara singkat tentang platform baru ini, solusinya, proses transisi dari Visual Studio Team Services dan tentang penggunaan dalam proyek nyata. Bergabunglah sekarang!



Hari ini kami mengumumkan perilisan Azure DevOps. Berkomunikasi dengan pelanggan dan pengembang kami di seluruh dunia, kami menyadari bahwa DevOps sangat penting untuk kerja tim yang sukses. Azure DevOps mewujudkan lebih dari 15 tahun pengalaman menciptakan alat pengembangan perangkat lunak. Selama sebulan terakhir, lebih dari 80.000 pengguna di Microsoft Corporation dan ribuan pelanggan kami, baik besar maupun tidak, telah menggunakan layanan DevOps untuk membuat produk.

Saat ini, layanan ini menghadirkan alat yang digunakan pada setiap tahap pengembangan, yang memungkinkan Anda membuat perangkat lunak lebih cepat dan lebih baik. Ini adalah penawaran terlengkap yang disajikan di cloud publik. Azure DevOps meliputi:

Semua layanan Azure DevOps terbuka dan dapat dikembangkan. Mereka sangat bagus untuk semua jenis aplikasi, terlepas dari lingkungan, platform atau cloud. Mereka dapat digunakan bersama sebagai solusi DevOps lengkap atau secara terpisah, dengan layanan lain. Pengembangan Layanan Visual Studio Team (VSTS)


Azure DevOps adalah hasil pengembangan Layanan Tim Visual Studio (VSTS). Pengguna VSTS akan secara otomatis dimigrasikan ke Azure DevOps. Pengguna yang ada tidak akan kehilangan fungsionalitasnya, tetapi akan mendapatkan lebih banyak pilihan dan kontrol. Transparansi dan integrasi penuh, fitur khas VSTS, dipertahankan. Layanan Azure DevOps bekerja bersama dengan mulus. Kami akan mulai mentransfer pengguna hari ini, dan selama beberapa bulan ke depan mereka akan melihat semua perubahan. Apa artinya ini?

  • URL akan berubah dari abc.visualstudio.com ke dev.azure.com/abc. Pengalihan dari alamat visualstudio.com akan diaktifkan, sehingga Anda tidak akan menemukan tautan yang rusak.
  • Karena layanan telah berubah, antarmuka juga telah diperbarui . Kami akan terus meningkatkannya berdasarkan umpan balik pengguna dari versi uji coba. Untuk pengguna baru, ini akan diaktifkan secara default. Dalam beberapa bulan mendatang, itu akan menjadi yang utama bagi pengguna yang ada.
  • Layanan Team Foundation Server (TFS) lokal akan terus diperbarui, pembaruan akan serupa dengan yang dirilis untuk Azure DevOps. Setelah merilis versi TFS berikutnya, produk akan menerima nama baru - Server Azure DevOps dan akan diperbarui dalam mode sebelumnya.

Pelajari cara menerapkan perubahan ini sekarang untuk pengguna VSTS saat ini.
