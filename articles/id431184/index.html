WIFI jaringan multi-hop mesh menggunakan teknologi Mesh Connex

Saat membangun jaringan nirkabel, seringkali tidak mungkin untuk menyediakan koneksi kabel untuk titik akses tertentu, dan perlu untuk mengkonfigurasi jembatan nirkabel point-to-point (WDS), teknologi ini dipelajari dan diimplementasikan dengan baik oleh sebagian besar penawaran yang ada di pasar. Lebih sulit untuk memecahkan masalah membangun jaringan wifi mesh multi-hop. Untuk memfasilitasi tugas ini, seluruh lini produk ExtremeWireless Wing mendukung teknologi Mesh Connex.
Dasar Mesh Connex adalah standar IEEE802.11s, RFC 3561 , serta desain Extreme Networks sendiri.



Menggunakan Mesh Connex, Anda dapat berhasil mengkonfigurasi mesh multi-hop di konfigurasi statis dan dinamis.

- Multi-hop statis Connex Mesh - mode infrastruktur, dapat digunakan saat membangun objek seperti laut atau bandara, taman rekreasi, karier;
- Multi-hop Mobile Mesh Connex - jaring dinamis ketika titik akses dipasang pada objek bergerak (bus, kereta, truk sampah ...)
Juga, kedua mode dapat digunakan secara bersamaan dalam jaringan mesh yang sama.



Poin Akses dapat berfungsi sebagai elemen mesh berikut:

  • MPR (Mesh Point Root) - AP dengan akses jaringan kabel
  • MP (Mesh Point) - Memperbaiki AP tanpa akses jaringan kabel
  • VMM (Vehicle Mobile Mounted) - AP dipasang pada objek bergerak

Pada saat yang sama, tidak ada batasan jumlah titik akses root pada jaringan, tingkat prioritas yang berbeda dapat ditetapkan untuk MPR yang berbeda. Menggunakan fungsi Backhaul Detection, MPR memeriksa apakah ada akses ke jaringan kabel dan mengalihkan lalu lintas ke root lain jika ada masalah dengan port kabel. Untuk memantau sumber daya kritis, Anda dapat mengonfigurasinya agar diperiksa menggunakan permintaan ARP atau ICMP, dalam hal ini, bahkan jika backhaul kabel pada MPR dinaikkan, lalu lintas akan dialihkan melalui MPR lain.



Perbedaan utama antara Mesh Connex dan teknologi lain yang digunakan untuk membangun jaringan multi-hop mesh adalah protokol routing yang menghitung rute optimal antara node mesh. Protokol perutean dalam jaringan mesh nirkabel dapat proaktif atau reaktif.

  • Proaktif - pembaruan terus-menerus dari tabel perutean penuh, yang bahkan dengan jaringan berukuran sedang meningkatkan lalu lintas â€œoverheadâ€ layanan, sambil mengambil waktu dan sumber daya untuk konvergensi lengkap
  • Reaktif - protokol secara perlahan merespons perubahan dalam jaringan, menyimpan rute hanya ke tetangga, tetapi pada saat yang sama memiliki penundaan yang lebih tinggi ketika mencari rute

Mesh Connex menggunakan kedua pendekatan ini secara bersamaan, dalam mode hybrid. Awalnya, informasi routing dikumpulkan secara reaktif dalam mode on-demand, ketika setiap MP mempelajari rute hanya ke MPR, dan kemudian, saat jaringan beroperasi, pengiriman lalu lintas dioptimalkan menggunakan mode proaktif.

Dalam jaringan mesh multi-hop nirkabel, sering terjadi bahwa bahkan jika tetangga memiliki SNR yang lebih baik secara keseluruhan, rute ke titik root melalui itu akan memiliki karakteristik yang lebih buruk, sehingga Mesh Connex menghitung rute berdasarkan data berikut:

- Kualitas Tautan - seberapa baik suar tetangga terdengar (yaitu, nilai awalnya "diprediksi" dan kemudian diukur selama pertukaran paket)
- Tautan Metrik - angka dari 0 -65535, diukur berdasarkan LQ + RSSI + Data Rate rata-rata
- Path Metric - Metrik Tautan menjumlahkan hingga MPR

Manfaat penting lain dari MeshConnex adalah:

  • Skalabilitas tinggi (jumlah poin bisa dari 2 hingga 1000)
  • Latensi rendah per-hop - 2,2 ms
  • Jaringan mungkin atau mungkin tidak memiliki "backhaul" kabel
  • Node mesh dapat mobile (semua atau beberapa)
  • Mobile node dapat bergerak dengan kecepatan hingga 140 km / jam (dengan "handoff" ~ 200 ms)
  • Konvergensi cepat
  • Kemampuan penyembuhan diri jaringan jika terjadi kecelakaan
  • Dukungan multicast melalui mesh

Jika Anda perlu membangun jaringan nirkabel dengan elemen jaring untuk tempat parkir, zona industri, zona festival ruang terbuka, tambang, tambang, pelabuhan udara dan laut, sebaiknya gunakan Mesh Connex untuk transportasi kereta api. Di Rusia dan negara-negara tetangga, puluhan proyek semacam itu telah berhasil dilaksanakan hingga saat ini.
