我们超频了JTAG路由器


如果您突然需要在克里米亚调试几个微控制器/微处理器，请在希姆基一个闷热的办公室喝冰沙。 如果微处理器板位于移动的物体上，并且无法到达JTAG调试器（该板位于气球/直升机上）。 如果您突然间只需要在主机和要调试的电路板（例如，高压设备）之间进行电气隔离。 而且它对制造商（STM，Broadcom，Xilinx等）或架构（ARM，MIPS，FPGA等）仍然便宜，开朗且通用。 然后，您需要一个路由器，是的，例如这样的路由器。

因此，这是Rostelecom在全国范围内发行的Sagem F @ ST2704 V2。 我们拥有SoC BCM6328 MIPS架构内核，320 MHz，一对焊接的USB端口[1]。 有wifi和以太网。 最好的部分是该模型的openwrt发行。 设备为实现上述目标所需的一切。

立刻想到了选择st-link并尝试通过网络转发USB的想法。 看起来像拐杖，很可能承诺工作不会很快并且不是很稳定，开销很大。 我们进一步研究可以做什么。

您可以将openocd移植到openwrt，拿起st-link或ftdi芯片并启动gdb服务器。 幸运的是，openocr已经移植到了openwrt。 似乎对此有足够的了解。 但是我想看看openocd给我们的其他选择。 此处是sysfsgpio接口的文档。 您需要的是，可以通过芯片焊接引脚上的/ sys / class / gpio控制tck，tdi，tdo信号以及OS linux的常规方式。

我们尝试。 首先，将openwrt（使用chaos_calmer分支）与openocd一起收集。 默认情况下，在焊接的GPIO上，光指示功能以及用于执行某些命令（rfkill，reset和wpsc）的轮询按钮是固定的。 为了避免它们干扰，我通过从程序集中删除相应的内核模块将其关闭。

当然，这是由于/ sys / class / gpio中的文件处理缓慢。 在openocd中戳戳。 查找RaspberryPi的接口驱动程序（src / jtag / drivers / bcm2835gpio.c）。 从测试[5]来看，它的速度应与st-link相似。 这在很大程度上是由于直接访问GPIO寄存器而实现的。 我们将对SoC进行同样的操作，对于整个bcm63xx芯片系列也是如此。

与sysfsgpio相比，它增加了两个选项：

  • bcm63xx_gpio_jtag_delay
  • bcm63xx_gpio_adresses

第一个设置设置开关引脚之间的延迟，它是RaspberryPi驱动程序的bcm2835gpio_speed_coeffs的间接模拟，它设置了jtag频率。 例如，在零延迟的情况下，开关频率约为1兆赫，一切工作都非常稳定，但是为了可靠性，最好能够设置此参数。

第二种选择是bcm2835gpio_peripheral_base的模拟，仅因为它需要为该寄存器注册两个地址，该地址设置引脚的输入/输出功能，以及负责gpio上输入/输出逻辑值的寄存器。 首先，它从内核头文件中获取寄存器值。 但是，这些值都无效。 事实证明，不能直接从用户空间访问外设寄存器，即 必须在内核中重新映射。 很好，gpio驱动程序已经为我实现了这一点，可以从/ proc / iomem中获取必要的值。
