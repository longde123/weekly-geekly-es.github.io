File yang dihapus mulai kembali ke folder Dropbox. Bug diperbaiki



Pengembang Dropbox memperbaiki bug karena file yang dihapus bertahun-tahun lalu mulai dipulihkan di akun pengguna. Sekarang pengguna diizinkan untuk menghapus file "bangkit dari kematian" lagi. Perusahaan berjanji bahwa mereka tidak akan pernah muncul lagi.

Beberapa pakar keamanan telah lama menduga bahwa ketika menghapus file di Dropbox, mereka tidak dihapus, tetapi disimpan di hosting - seperti yang terjadi dengan informasi yang "dihapus" di Facebook . Seperti, ketika "menghapus" file hanya ditandai sebagai dihapus (flag yang sesuai diatur). Itu menghilang dari akun, tetapi terus disimpan di server untuk berjaga-jaga.

Dropbox mengklaim ini bukan masalahnya.

Menurut kebijakan Dropbox, perusahaan menghapus file secara permanen 60 hari setelah pengguna mengklik tombol "Hapus". Mukjizat-mukjizat yang telah terjadi baru-baru ini seharusnya hanyalah "bug".

Dalam beberapa minggu terakhir, beberapa pengguna mulai melaporkan bahwa file dan folder yang sebelumnya dihapus mulai muncul di akun mereka. Pada orang-orang, ini menyebabkan kebingungan. File biasanya sangat tua, seolah-olah mereka datang entah dari mana.

Dalam forum dukungan teknis, beberapa diskusi dibuka. Mereka kemudian digabungkan menjadi satu utas . Dalam beberapa kasus, folder muncul di akun yang sudah dihapus pada 2009, dalam kasus lain - yang lebih baru.

"Saya memiliki masalah yang sama - beberapa folder berbeda dengan file lama dari 2009-2011, dihapus beberapa tahun yang lalu, tiba-tiba muncul dalam satu hari," salah satu korban menulis. "Dan aku pasti tidak terhubung ke komputer lama." Pengguna menjelaskan bahwa file tidak dapat disinkronkan dari PC lama di mana salinan tetap.

"Saya menghapus file-file ini lebih dari enam tahun yang lalu," tulis yang lain. "Perangkat tempat mereka disimpan sudah lama dibuang."

Pengguna lain mengatakan bahwa ketika dia melihat folder baru dengan file di akun, dia mengubah kata sandi untuk akun dan mengganggu semua koneksi dengan server - dia pikir akunnya diretas.

Setelah banyak keluhan pengguna, pada 19 Januari 2017, perwakilan Dropbox mengakui bahwa ada masalah.dan mengumumkan rilis tambalan. Perusahaan menjelaskan bahwa alasannya adalah bug yang tidak memungkinkan beberapa file dan folder dihapus sepenuhnya dari server bahkan ketika pengguna menghapusnya dari akun Dropbox: “Dengan memperbaiki bug ini, kami secara tidak sengaja mengembalikan file dan folder ini ke akun pengguna. Itu kesalahan kami; alasannya bukan intervensi pihak ketiga, dan Anda tidak diretas, ”salah satu pengembang menulis.

Artinya, perusahaan benar-benar menyimpan file "dihapus" secara formal di server-nya selama bertahun-tahun - tampaknya, ia tidak melihat beberapa terabyte data tambahan.

Benar, ini tidak berlaku untuk semua file. Dihapus dan kemudian dipulihkan file dan folder dalam kasus ini memiliki "metadata inconsistency." Menurut Dropbox, karena ketidakkonsistenan metadata, file dan folder khusus ini dikarantina, dikeluarkan dari prosedur penghapusan file hingga metadata diperbaiki.

“Selama waktu ini, tidak ada yang memiliki akses ke file atau folder Anda dan tidak ada file lain yang mengalami bug,” kata Dropbox. Bug diperbaiki, dan file dan folder "baru lahir" tidak akan memengaruhi kuota disk pengguna.

Penghapusan informasi palsu adalah perilaku normal beberapa perusahaan cloud dan jejaring sosial. Misalnya, Facebook juga tidak menghapus pesan pribadi, foto, dan informasi lain yang datang ke server jika pengguna kemudian memutuskan untuk menghapusnya dari akunnya (karena ia percaya, "hapus").

Informasi Anda di cloud bukan informasi Anda.


Informasi pribadi tentang pengguna adalah aset berharga yang dimiliki perusahaan, jadi tidak ada yang mau menghapusnya secara gratis. Ingat kisah mahasiswa hukum Max Shrems? Dia menuntut agar Facebook mengirim CD dengan semua data pribadi yang dikumpulkan selama tiga tahun aktivitas di situs.

Pria itu menerima disk dengan dokumen PDF lebih dari 1.200 halaman, semua informasi dalam file itu dibagi menjadi 57 kategori (pekerjaan, pendidikan, teman, pandangan politik, hobi, foto, dll.). Seperti katanya, bahkan KGB tidak memiliki dokumen lengkap tentang warga.

Tetapi yang paling mengejutkan adalah, antara lain, file yang dikirim berisi pesan yang dihapus, sesi obrolan, nama-nama mantan teman dan data lain yang ditandai dihapus: benar.

Dropbox sulit dicurigai licik. Sangat sulit baginya untuk menyimpan semua informasi yang dihapus selama bertahun-tahun. Di sini kita berbicara tentang sejumlah besar data. Diketahui bahwa perusahaan file hosting pergi ke segala macam trik untuk setidaknya sedikit mengoptimalkan ruang disk dan menghilangkan informasi berlebih. Ini bagi mereka kerusakan finansial langsung.

Tapi “menghapus” file dari cloud hosting, Anda tidak akan pernah bisa 100% yakin bahwa itu benar-benar dihapus. Saya bertanya-tanya berapa banyak perusahaan yang masih memiliki "bug" seperti itu.
