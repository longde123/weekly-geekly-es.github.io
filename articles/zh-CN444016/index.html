利用Microsoft Game Stack取得更多成就


Microsoft的信念是赋予人和组织更大的权力-这是我们公司的基因。 我们宣布了一项新计划，即Microsoft Game Stack，其中我们致力于将Microsoft工具和服务整合到一起，这些工具和服务将使像您这样的游戏开发人员（无论您是刚起步的独立开发人员还是AAA工作室）都可以实现更多目标。


这是新旅程的开始，今天我们只是迈出了第一步。 我们认为Microsoft非常适合兑现这一承诺。 我们公司在游戏以及建立面向开发人员的平台方面有着悠久的历史。


如今，全球有20亿玩家在各种设备上玩各种游戏。 社区中对视频流，观看和共享的关注与对比赛或竞争的关注一样多。 作为游戏创作者，无论他们身在何处或使用何种设备，您都在每天努力使玩家不断参与，激发他们的想象力并激发他们的灵感。 我们正在引入Microsoft Game Stack，以帮助您做到这一点。


本文在我们的博客中。

微软游戏堆栈到底是什么？


Game Stack将我们所有的游戏开发平台，工具和服务（例如Azure，PlayFab，DirectX，Visual Studio，Xbox Live，App Center和Havok）整合到一个强大的生态系统中，任何游戏开发人员都可以使用。 Game Stack的目标是帮助您轻松发现创建和操作游戏所需的工具和服务。


云在游戏堆栈中扮演着至关重要的角色，而Azure满足了这一至关重要的需求。 Azure提供了计算和存储等构建块以及来自机器学习和AI的云原生服务，以推送通知和混合现实空间锚。 Azure已在包括中国在内的全球54个地区提供服务，并将继续投资建设高度安全和可持续的云基础架构以及为游戏开发人员提供的其他服务。 Azure的全球规模将使Project xCloud流技术具有规模，从而为全球玩家提供出色的游戏体验，而不论他们的设备和位置如何。


像Rare，Ubisoft和Wizards of the Coast这样的公司已经与Azure合作，托管多人游戏服务器，安全可靠地存储玩家数据，分析游戏遥测，保护其游戏免受DDOS攻击以及训练AI创造更身临其境的游戏玩法。


尽管Azure是Game Stack的一部分，但重要的是要指出Game Stack是与云，网络和设备无关的。 而且我们不止于此。


有什么新消息？


Game Stack的下一部分是PlayFab，这是用于构建和操作实时游戏的完整后端服务。 一年前，我们通过收购欢迎PlayFab加入Microsoft。 今天，我们很高兴地宣布，我们即将把PlayFab引入Azure系列。 Azure和PlayFab共同构成了一个强大的组合：Azure带来了可靠性，全球规模和企业级安全性； PlayFab为Game Stack提供托管游戏开发服务，实时分析和LiveOps功能。 去年秋天，我们看到了这两个平台可以与PlayFab Multiplayer Servers一起使用，它允许您通过使用Azure云计算动态托管服务器来安全地启动和扩展多人游戏。


用PlayFab的联合创始人James Gwertzman的话来说，“现代游戏创作者不像电影导演那样，而更像巡游导演。 要取得长期成功，就需要让玩家参与不断的创造，试验和运营周期。 不再只是发布游戏并继续前进。” 这就是一年前为什么我们通过收购欢迎PlayFab加入Microsoft的原因。 PlayFab支持从iOS和Android到PC和Web到Xbox，Sony PlayStation和Nintendo Switch的所有主要设备； 以及所有主要游戏引擎，包括Unity和Unreal。 PlayFab还将继续支持所有未来的主要云。


今天，我们也很高兴宣布预览中的五项新PlayFab服务。


在今天的公开预览中：


  • PlayFab对接会 ：适用于多人游戏的强大对接会，改编自Xbox Live对接会，但现在可用于所有游戏和所有设备。

在今天的私人预览中（ 与我们联系以加入预览 ）：


  • PlayFab Party：语音和聊天服务，改编自Xbox Party Chat，但现在可用于所有游戏和所有设备。 Party利用Azure认知服务进行实时翻译和转录，使更多玩家可以访问游戏。
  • PlayFab见解：将强大的实时游戏遥测与来自其他多个来源的游戏数据相结合，以衡量您的游戏性能并创建可行的见解。 在Azure数据资源管理器的支持下，Game Insights将提供与现有的第一方和第三方数据源（包括Xbox Live）的连接器。
  • PlayFab PubSub：通过Azure SignalR支持的持久连接，将您的游戏客户端订阅从PlayFab服务器推送的消息。 这可以实现诸如实时内容更新，配对通知和简单的多人游戏玩法等场景。
  • PlayFab用户生成的内容：通过允许玩家创建并与其他玩家安全共享用户生成的内容来与社区互动。 该技术最初是为支持Minecraft市场而构建的。

不断发展的Xbox Live社区


游戏堆栈的另一个主要组件是Xbox Live。 在过去的16年中，Xbox Live已成为世界上最活跃，最具参与度的游戏社区之一。 它也是一个安全且包容的网络，打破了游戏玩家跨设备连接的界限。


今天，我们很高兴Xbox Live成为Microsoft Game Stack的一部分，提供身份和社区服务。 在Game Stack下，Xbox Live将扩展其跨平台功能，因为我们引入了一个新的SDK，它将这个社区带入iOS和Android设备。



现在，移动开发人员将能够通过Xbox Live接触到地球上一些高度参与和热情的游戏玩家。 这些只是移动开发人员的部分好处：


  • 受信任的游戏身份：使用新的Xbox Live SDK，开发人员可以专注于创建出色的游戏，并利用Microsoft的受信任的身份网络来支持登录，隐私，在线安全和子帐户。
  • 无摩擦集成：全新的点菜服务产品，并且没有Xbox Live认证通行证，使移动开发人员可以灵活地构建和更新游戏。 开发人员只是使用最适合他们需求的服务。
  • 充满活力的游戏社区：接触Xbox Live不断发展的社区，并在众多平台上联系游戏玩家。 寻找创造性的方法来实现成就，Gamerscore和“英雄”统计数据（具有自己的游戏外体验），以保持游戏玩家的参与度。

其他游戏堆栈组件


游戏堆栈的其他组件包括Visual Studio，混合器，DirectX，Azure App Center，Visual Studio，Visual Studio Code和Havok。 在接下来的几个月中，随着我们努力改善和发展Game Stack，您将看到这些服务之间更深的联系，因为我们将它们统一起来可以无缝地协同工作。


作为如何进行这种集成的一个示例，今天，我们将PlayFab和以下游戏堆栈组件组合在一起：


  • App Center：来自App Center的崩溃日志数据现在已连接到PlayFab，通过将崩溃日志重新绑定到各个玩家配置文件，您可以实时更好地理解和响应游戏中的问题。
  • Visual Studio代码：使用PlayFab的Visual Studio Code的新插件，编辑和更新Cloud Script变得更加容易。

创造您的世界，成就更多


当我们将重点扩展到云时，平台的性质可能会发生变化，但是我们对像您这样的游戏开发人员进行授权的承诺坚定不移，并且我们期待着Microsoft Game Stack的发展。 当我们开始将所有这些出色的服务和技术融合在一起时，我们的团队就为这些可能性而感到鼓舞和兴奋。 在进行过程中，请务必与我们分享您的反馈，以便我们可以帮助您实现更多目标。 如果您在GDC，请在Moscone中心南大厅的Microsoft展位前停下来，试用许多新服务，并详细了解未来的令人兴奋的机遇。
