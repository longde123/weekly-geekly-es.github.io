我们渴望告诉谁今年将演出。 许多发言人正在就主题进行谈判，因此有人拥有话题，很快就会有人出现。

今年， Yuri Selivanov （加拿大多伦多）将首次来找我们-Python 核心开发人员 ，专家软件开发人员，EdgeDB的创始人，uvloop，asyncpg和asyncio的作者。 Yuri已为Cisco，Pinterest，ABB，Nintendo等公司提供建议。 它的软件被Facebook和Instagram等公司用来提高生产力。

Andrey Svetlov （乌克兰基辅）-Python 核心开发人员 ，许多Python库的作者和积极参与者，包括asyncio，aiohttp，aiopg，aiozmq，著名的Python编程博客的作者 ，PyCon俄罗斯会议的程序主管和发言人。 两年前，我们就Python语言以及更多内容采访了 Andrew。 如果您还没有看过，请阅读:)

Christian Heimes （德国汉堡）-自2007年以来担任Python核心开发人员 ， Red Hat的高级软件工程师，PSF成员。 克里斯蒂安（Christian）将做一个演讲“ SSLError，现在怎么办？”，他将简要讨论主要的加密原语，握手协议，证书的内部结构和公钥基础结构。 您将了解TLS / SSL的最佳做法，调试工具和诊断方法，以及如何管理证书。

大会的另一个主题是来自Swagger / OpenAPI开发人员Stephan Jaensch （德国汉堡）之一的Yelp的 Techid 。 Stefan将做一个演讲“使用更大的代码库键入注释”，它将告诉您如何开始使用注释，如何充分利用注释以及如何避免在开始使用它们时出现的问题。

Melanie Warrick （美国旧金山）-Google Cloud高级开发倡导者。 在此之前，Melanie是DeepLearning4J（开源Java平台）的创始人和开发人员，还曾在Change.org从事机器学习。 在过去的两年中，她在许多人工智能会议上发表了演讲，她最喜欢谈论关于解决机器学习问题的工作。

Eigen Technologies的开发经理，指数的CTO，人工智能和机器学习专家Alejandro Saucedo （英国伦敦）将在PyConRu上发表有关使用Python和Airflow的工业数据管道的演讲。 这个动手演讲将帮助您弄清楚如何使用Airflow设置机器学习。

Kate Heddleston （美国旧金山）是Shift Technologies创业公司的软件工程师，该公司是买卖二手车的交易平台，她从事python项目。 此外，凯特（Kate）是Opsolutely的创始人兼首席执行官，该平台可帮助工程团队在云中部署基础架构。

Mail.Ru的开发人员，Mail.ru教育项目的Perl和Python老师， Python频道 Vadim Pushtaev的作者将告诉他们如何在Search上编写单元测试。 从小事情：如何命名，每个测试的结构等，到大问题：TDD的问题，如何弄湿，如何处理数据库等外部系统，它们如何与夹具一起使用，为什么需要测试私有方法等？ 。

Python具有与C代码的出色集成功能，这使您可以在保持灵活性的同时以低成本优化对性能至关重要的功能。 CNS开发技术总监Sergey Borisov将举办研讨会“ Cython-C Programming for People”，该研讨会将展示可用于解决此类问题的工具，并与研讨会参与者一起编写具有简单协议的异步客户端。

Yandex内部服务开发人员Marina Kamalova会告诉您可以从哪个python组件创建文本聊天机器人，从收到用户消息到收到机器人的响应（NLU，NLG，ML分类器），如何使该机器人适应不同的Messenger和不仅是Messenger，而且还以Telegram API为例说明了如何提高漫游器的弹性。

过去，EasyTen和伦敦大学学院的数据科学家和机器学习工程师，现在是Poteha Developers的首席执行官Vitaliy Davydov，在他的报告中将考虑基于Python的无服务器微服务的示例。 经过一小部分理论工作之后，Vitaly将在实时模式下在AWS Lambda上部署一个简单服务并进行测试。

中枢神经系统开发总监，MIPT工业编程和编程语言理论讲师Alexey Kuzmin将讨论异步编程的优缺点，并说明这种机制如何在Python中起作用。 考虑一些有用的库和工具。 最后，我们将讨论如何正确地测量和调试异步代码。

Ingram Micro Cloud开发部门的负责人Alexander Khayorov将作题为“ Python 2k18开发人员环境”的演讲。 Sasha将讨论所有工具，辅助实用程序和整个生态系统，这些都将伴随着使用Python进行的现代商业和工业开源开发。 将诸如pyenv，pipenv和pipsi之类的内容放在一起。 为了充实视野，它将涉及程序包管理器，语言的各种版本以及其他流行语言中的库依赖项。 通过描述和比较，我们将了解真相，并可能会得出最佳的开发人员环境。

Avito Dmitry Khodakov的技术主管将讨论构建加载的微服务框架时的典型问题和陷阱。 关于分析异步应用程序； 龙卷风和aiohttp异步之间的根本区别； 并在接近战斗的情况下诚实地比较龙卷风和aiohttp。 该报告对于处理异步和微服务的经验丰富的Python开发人员很有用，这些异步和微服务面临着异步应用程序的扩展和调试问题。
