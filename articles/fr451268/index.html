Victor Gamov à propos de Kafka Streams IQ sur jug.msk.ru

Le 8 mai 2019, lors d'une réunion de la communauté moscovite des développeurs Java jug.msk.ru, Victor Gamov a parlé de Kafka Streams IQ ( Interactive Queries ).



À propos de Speaker


Viktor s'exprime sur jug.msk.ru pour la quatrième fois, ses précédents reportages concernaient Nashorn ( 13 novembre 2014 ), Hazelcast ( 4 mai 2016 ), Apache Kafka ( 23 avril 2018 ). Comme il y a un an, il travaille actuellement chez Confluent , mais maintenant en tant que développeur avocat .

Co-fondateur et animateur du podcast populaire Debriefing , co-auteur du livre Enterprise Web Development , conférencier de nombreuses conférences (y compris celles organisées par JUG.ru Group ). Site Web de Victor (avec un blog technique là-bas), rapports vidéo sur YouTube , fichiers de présentation sur Speaker Deck and Notist , Habr , Twitter et GitHub .

Quelques reportages faits depuis le dernier discours sur jug.msk.ru :

  • «Kafka sur Kubernetes: l'histoire mystérieuse de la création d'une plate-forme de traitement de données en continu basée sur le cloud» ( DevOops 2018 : Présentation )
  • "Java Puzzlers NG S03: Quoi? D'où viennent-ils?! " (avec Baruch Sadogursky sur Oracle Code One 2018 : vidéo )
  • «Combattez le crime avec Kafka Streams et l'API Bintray Firehose» (avec Baruch Sadogursky sur Oracle Code One 2018 : vidéo , présentation )
  • «Traverser les flux: repenser le traitement des flux avec KStreams et KSQL» ( DevNexus 2019 : présentation )
  • "Kafka sur Kubernetes: faut-il vraiment que ce soit" The Hard Way "?" ( Sommet Kafka 2019 : présentation )

À propos du rapport


Traditionnellement, avant la présentation de l'orateur et le début de la réunion, Andrei Kogun rappelle les événements à venir - les candidatures pour la conférence Joker Java (Saint-Pétersbourg, 25-26 octobre) sont ouvertes et le festival TechTrain IT devrait se tenir pour la deuxième fois (Saint-Pétersbourg, 24-25 Août).



Le rapport était consacré aux requêtes interactives , qui font partie de l'API Kafka Streams , qui permet d'accéder à l'état intermédiaire de l'application et aux résultats obtenus sans utiliser de bases de données et référentiels traditionnels.



Le diaporama de présentation alternait avec des exemples de code. Un grand nombre de questions ont été posées, auxquelles Victor a immédiatement donné des réponses complètes, ce qui est également confirmé par des critiques sur Twitter .



La discussion s'est poursuivie pendant la pause de la réunion et après celle-ci. La réunion a été suivie par l'un des orateurs des conférences JUG.ru, Ivan Ponomarev, dont le récent rapport à la conférence JPoint 2019 «Kafka Streams API: un pas au-delà de Hello World» ( présentation , la vidéo n'est disponible pour le moment que pour les participants à la conférence) peut compléter l'histoire de Victor.
