使用PowerShell脚本安装Veeam Backup＆Replication

最近，我的同事Dave Kaula发布了一个公共脚本，用于安装Veeam Backup＆Replication，他用它在实验室中演示了该产品。 该脚本对于计划部署解决方案的人员可能有用，例如，在测试环境中或在“一台机器上全部”的场景中。

Dave为该脚本写了一篇简短的封面文章，我引起了您的注意。

所以欢迎猫。


在本文中，我将介绍如何在基于Hyper-V平台的虚拟机上安装Veeam Backup＆Replication 9.5 Update 3。

重要！ 我们将根据“一台机器上所有”场景执行安装，即：Veeam备份服务器（安装包中包含SQL Server 2012 Express Edition）以及Veeam Backup Enterprise Manager服务器。 在开始安装之前，请检查是否满足文档中描述的所有相关系统要求。 对于将使用Veeam Backup服务的帐户，假定使用域管理员权限。

要安装解决方案，您需要遵循以下简单步骤：

  1. 我们创建一个运行Windows Server 2016的虚拟机。例如，我将在Hyper-V Manager中创建一个VM，并将其命名为Management01 。


  2. 从此处下载Veeam备份和复制，然后准备具有商业许可证的现有文件，或请求试用版或NFR的许可证。 在我们的例子中，两个文件都已经存储在F：\ Temp1文件夹中 。


  3. 在此文件夹中，我们保存此脚本 。
  4. 打开PowerShell命令提示符（Administrative PowerShell Prompt），并指定路径F：\ Temp1 ，然后输入以下命令： 

    Import-Module .\Install-Veeam-HyperV.PS1 -Force -Verbose

    然后，将加载Install-Veeam模块，该模块将用于安装Veeam Backup＆Replication。


  5. 我们执行命令： 

     Install-Veeam -VMName Management01 -GuestOSName -Management01 -VMPath F:\dcbuild_test
  6. 在打开的窗口中，选择文件夹F：\ temp1中的文件VeeamBackup＆Replication_9.5.0.1536.Update3.iso 。
  7. 然后，在许可证文件选择窗口中，从步骤2中打开许可证文件。 我将使用我的Veeam NFR许可证。
  8. 帐户输入窗口将会出现。 如前所述，在测试环境中，经常为Veeam Backup分配一个具有域管理员权限的帐户，该示例也不例外。 我们输入用户名和密码，在此示例中，我使用MVPDays \ Administrator帐户。


  9. 此后，将开始一个完全自动化的动作序列，因此您只需要等待结果即可。



这是脚本的作用：

1）从主机添加新的VHDx文件
2）执行其命名（命名新的Veeam卷）
3）在主机上安装新的VHDx
4）复制Veeam ISO映像
5）复制Veeam许可证文件
6）安装2012 System CLR类型
7）安装SQL Server 2012 Express Edition
8）安装Veeam Backup＆Replication 9.5服务器
9）安装Veeam备份控制台
10）安装Veeam Explorer for Active Directory
11）安装Veeam Explorer for Exchange
12）安装Veeam Explorer for SQL
13）安装Veeam Explorer for Oracle
14）安装Veeam Explorer for SharePoint
15）安装Veeam企业管理器（Web和云门户）
16）安装更新3
