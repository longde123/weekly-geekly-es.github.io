使用Veeam Availability Console：配置托管备份服务托管备份



我们最近启动了一项新的客户端备份策略管理服务，即Veeam托管备份。 它基于Veeam可用性控制台门户。 Veeam已经讨论了门户的功能。 简要回顾一下主要内容：

  1. 对于客户 -监控哈萨克斯坦共和国的任务和基础设施，接收有关计费的信息；
  2. 对于提供商 -管理备份客户端：配置，监视和维护。

在我的文章中，我想谈谈如何在门户中工作以及如何为虚拟环境配置托管备份。 但是首先，简要概述Veeam托管备份的性能。 即使资源位于客户端站点上，它也可以将整个备份过程外包。 该服务对启用了Veeam Cloud Connect服务的VM使用许可模型。

这对客户意味着什么？ 首先，可以在服务提供商的云中获取虚拟机的额外备份副本，而无需支付Veeam Cloud Connect许可证的费用。 仅为备份（Cloud Connect备份）或复制（Cloud Connect复制）支付了存储空间。 其次-支付使用许可的费用，即与永久许可之类的许可相比，成本上的灵活性。

但是回到Veeam Cloud Connect门户：设置托管备份的第一步是创建一个客户帐户。

步骤1：建立组织


当您收到用于连接到Managed Backup Veeam服务的应用程序时，必须创建一个“组织”及其“帐户”。

  1. 为此，请转到Veeam可用性控制台并打开“公司”部分。

  2. 转到“新公司”部分，然后选择“公司信息”标签。 在客户端输入公司名称和员工的联系方式：

  3. 在“用户信息”选项卡中，设置用于后续连接的登录名和密码：


需要说明的重要一点是，如果客户端仅订购了托管备份服务，则不会将云中的资源分配给他。 如有必要，可以将Cloud Connect资源快速添加到客户端的订阅中（备份存储，硬件计划）。

步骤2：订阅计划


创建客户帐户后，您需要创建订阅计划并为分配的资源设置价格：VM的许可证成本，Cloud Connect备份的存储量，技术支持服务等。

  1. 转到“结算”部分，然后选择“新建”以创建新的订阅。 或使用先前创建的订阅计划（如果适用）。

  2. “编辑计划”部分将打开。 在其中，选择“托管备份”字段，然后输入每月资源的价格：
    • 托管服务-备份管理；
    • 托管VM-每个VM的许可证成本（Cloud Connect备份\复制，托管服务）；
    • 工作站代理-工作站的Veeam代理费用；
    • 服务器代理-Veeam服务器代理成本；
    • 云代理-在虚拟环境VMware vSphere，Microsoft Hyper-V，Microsoft Azure或Amazon Cloud中的VM上部署的Veeam备份代理。


    资源成本可以在以后分配或更改。
  3. 选择“云备份”选项卡，然后输入一个月的备份成本：
    • VM备份-Veeam备份和复制许可证；
    • 服务器备份-服务器Veeam代理许可证的费用；
    • 工作站备份-工作站的Veeam代理许可证的费用；
    • 云存储库-云存储量；

  4. 选择创建的订阅计划：

  5. 现在，根据所选计划，客户将为使用的资源生成每月账单。 看起来像这样：



步骤3：客户端访问


公司成立后，我们会向客户发送一封欢迎电子邮件，其中包含用于进入VAC（Veeam可用性控制台）的数据。 这封信是直接从门户发送的。

  1. 为此，请转到“公司”部分，选择所需的公司，然后单击“发送欢迎电子邮件”。



  2. 这就是从门户发送的信件的样子。

将帐户信息发送给客户端后，您可以继续配置Veeam托管备份服务。

步骤4：备份设定


首先，我们将与客户端商定其基础架构的资源和服务器数量：备份服务器，代理服务器等等。

商定资源后，即可在客户端的服务器上部署Veeam基础架构和许可证。

  1. 安装Veeam备份和复制。 根据客户的要求，我们部署了Veeam的所有必要元素：
    • 备份服务器
    • 代理服务器\存储库（网关）服务器；
    • 企业经理
  2. 成功部署系统后，请转到Veeam备份和复制控制台：

  3. 我们选择客户资源。
  4. 安装必要的许可证。
  5. 我们将许可证连接到客户端服务器。
  6. 我们根据客户的愿望和要求创建备份任务。

步骤5：将客户端连接到Veeam Cloud Connect


现在，将客户端基础架构连接到Veeam Cloud Connect DataLine服务。

  1. 为此，请打开Veeam Backup＆Replication控制台，转到“服务提供商”选项卡，单击“添加服务提供商”，然后输入客户端的DNS或IP地址。

  2. 确保选中此框以允许访问提供程序以进行远程安装和服务器管理。

  3. 输入密码并登录以连接：

  4. 连接到Veeam Cloud Connect之后，客户端服务器将可用于远程管理。 要连接到Veeam可用性控制台或Veeam备份服务器，请在提供商的控制台中使用Cloud Gateways。

配置Veeam托管备份服务后，服务提供商工程师将负责管理服务器。 客户端可以使用Veeam Availability Console门户从其任务中独立恢复VM /文件。

如有必要，可以授予客户端访问Veeam Backup Enterprise Manager的权限。 它将是先前为客户部署的Veeam基础架构的一部分。
