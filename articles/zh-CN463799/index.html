我对Windows 10不满意的地方

我在这里遇到了“导致我从Windows 10切换到Linux的10个原因”的另一个清单，并决定列出我今天使用的Windows 10中我不喜欢的东西。 在可预见的将来，我不会切换到Linux，但这并不意味着我对操作系统中的所有更改都感到满意。

我会回答“如果您不喜欢Windows 10，为什么不继续使用Windows 7？”的问题。

我的工作与技术支持有关，包括数十台计算机。 因此，使用当前版本的操作系统，而不是从“我不使用这十个您的任务”下的任务中获得otmazyvatsya，会更有收益。 我住七岁的时候，我记得她，我知道，从那时起，那里什么都没有改变。 但是有十二个在不断变化，您更新得太晚了-有些设置会爬到另一个地方，行为的逻辑也会改变，等等。 因此，为了跟上生活，我在日常使用中都安装了Windows 10。



现在，我告诉您我对她的不满意。 由于我不仅是用户，而且还是管理员，所以从两个角度来看，我会感到不满意。 谁不亲自使用它，而只有管理员使用它-他将不会遇到一半的事情，而简单的用户将不会遇到第二个问题。

更新内容


如果关闭，打开，运行期间以及计算机空闲时，没有要求的安装更新是有害的。 Windows家庭版的用户通常没有对更新的官方控制。 企业版本的用户具有一些相似的控制方式-“工作时间”，“推迟一个月”，“仅针对业务进行更新”-但迟早它们会被更新所取代。 如果您推迟了很长时间，那么在最不适当的时刻。



充满关于“他来到演示文稿，打开笔记本电脑，然后将整个更新放置一个小时”或“将计算保留了一整夜，然后计算机安装了更新并重新启动”的故事。 根据最近的个人经验-上周五，我们的员工关闭了计算机（从10 Home），他写道：“我放了更新，请勿关闭。” 好的，我没有将其关闭，我离开了。 电脑完成后，关闭。 星期一早上，一名员工进来，打开了电源，并继续安装更新。 有一个旧的Atom，因为安装肯定会持续两个小时左右，甚至可能会更长。 并且，如果安装中断，则Windows将回滚更新，好像更新的时间不超过设置的时间。 因此，我从来不建议您中断安装，除非一个小时已经显示30％并且不会移动到任何地方。 因此，即使在Atom上也无法提供更新。

理想的版本是Windows Update的先前版本，在该版本中我看到了安装的版本，可以完全禁用该更新，禁用不必要的更新，仅配置手动安装等。

当然，今天有多种方法可以关闭更新。 最简单的方法是阻止访问路由器上的更新服务器。 但这将是断头台头痛的一种治疗方法，如果没有关键的更新，可能迟早会复发。

通过在启动时按F8禁用安全模式


谁打扰了？ 现在，要进入安全模式，您需要启动操作系统，从那里按一个特殊的按钮，重新启动后，您将到达所需的位置。

而且，如果系统无法启动，则您需要等到Windows本身了解到它无法启动-才可以选择安全模式。 直到现在，她并不总是了解这一点。

返回F8的魔术命令： bcdedit / set {default} bootmenupolicy legacy

不幸的是，这只能提前在计算机上完成，如果您带来了一个未加载的陌生人，那么您就必须以不同的方式进入安全模式。

遥测




收集有关系统的信息并发送给Microsoft。 实际上，我不是隐私的忠实拥护者，而且我主要是基于“难以捉摸的乔”的原则生活-谁需要我？ 虽然，这当然并不意味着我将护照扫描件发布到了互联网上。

MS的遥测是非个人的（据推测），它的存在并不会给我带来太多麻烦。 但是它消耗的资源可能非常明显。 不久前，我从i5-7500（4核，3.4GHz）迁移到AMD A6-9500E（2核，3GHz，但是旧的慢速架构），这对工作产生了非常明显的影响。 后台进程不仅占据了大约30％的处理器时间（在i5上它们是不可见的，挂在远端内核的某个地方并且没有干扰），而且通过遥测组装和发送数据的过程开始占用了100％的处理器。

介面变更


当界面在版本之间变化时，就可以了。 但是，当在同一OS版本中，按钮和设置从一个部分迁移到另一部分时，进行设置的位置很少，甚至有些重叠，这会令人不快。 尤其是当新的“选项”与旧的“控制面板”完全不同时。

通知事项


再次-有人使用它们吗？ 单击角上的数字时，会显示一些无用的信息。 有时，在角落里，一些消息会弹出几秒钟，单击后会执行操作，并且不会发出其他信息。 例如，当您单击邮件本身时，一条消息，指出防火墙已禁用，它将重新打开防火墙。 是的，上面写着有关它的内容-但消息在屏幕上停留了很短的时间，您可能没有时间阅读最后一句话。

但是真正的嘲笑是您进入全屏模式的消息，Windows不会打扰您。 仅在全屏模式下，这些消息是透明的，但仍然挂在角落。 当您单击此角时-假设您正在玩，并且游戏中有一些按钮-您将被扔到桌面上。 不再显示消息的位置-您在桌面上。 当您返回游戏时，按钮顶部的角落再次出现透明消息。

从一开始的想法就不错-在一个地方收集所有程序的通知，但是实现是la脚的。 另外，“所有程序”都不会急于将其通知放在那里，而是以老式的方式显示。
