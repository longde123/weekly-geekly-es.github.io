Mengapa MS-DOS menyimpan buffer baca floppy selama tepat dua detik?

MS-DOS 2.0 menyimpan buffer disk baca, tetapi bukan buffer tulis. Buffer baca penting: ia menghindari membaca ulang. Dan itu dapat dibatalkan ketika volume dilepas.

Tapi, hei, kita bukan melompati disket. Kami hanya membawa mereka keluar.

Drive era itu dapat dibuka secara manual dan floppy disk dihapus kapan saja. Spesifikasi menyediakan status untuk membuka pintu drive, tetapi IBM tidak menerapkan bagian spesifikasi ini untuk menghemat satu gerbang logis. Pemasok peralatan akan melakukan apa saja untuk menghemat setidaknya satu sen .

Tapi buffer baca ini sangat penting untuk kinerja. Tanpa itu, Anda harus mulai dari awal setiap operasi I / O, membaca kembali daftar isi volume, menemukan entri direktori, melihat tabel alokasi blok untuk mencari cluster gratis berikutnya ... dan floppy disk jelas bukan drive tercepat, sehingga semua operasi ini tertunda selama beberapa detik.

Untuk meninggalkan setidaknya beberapa cache, pengembang MS-DOS meluncurkan benchmark: seberapa cepat seseorang dapat mengubah disket dalam drive IBM PC?

Manajer proyek MS-DOS 2.0 Mark Zbikowski sedang duduk dengan stopwatch sementara Aaron Reynolds dan Chris Peters mencoba mengubah floppy disk pada komputer IBM secepat mungkin.

Mereka tidak bisa melakukannya lebih cepat dari dua detik.

Dengan demikian, cache MS-DOS kedaluwarsa dalam dua detik. Jika dua akses floppy disk terjadi dalam interval dua detik, maka upaya kedua menggunakan cache dari floppy disk lama.

Saya tidak tahu apakah kebijakan modern dua detik untuk membersihkan cache adalah konsekuensi langsung dari persaingan di kantor, tetapi saya suka berpikir bahwa ada semacam koneksi.
