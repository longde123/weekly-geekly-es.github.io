Perusahaan baru akan mendukung OpenJDK 8 dan 11 - kami memahami situasinya

Oracle menghentikan dukungan untuk OpenJDK lama untuk pelanggan perusahaan. Tetapi dalam posting ini, Red Hat akan menggantikan perusahaan. Kami membahas alasan keputusan dan opini publik.


/ Pixabay / Tasos_Lekkas / PL

Perubahan penjaga


Pada Januari 2019, Oracle menghentikan dukungan OpenJDK 8 dan 11 gratis untuk pelanggan perusahaan. Sekarang, pembaruan keamanan untuk versi platform yang lebih lama hanya dapat diperoleh dengan berlangganan berbayar untuk Oracle Java SE Advanced dan Java SE Suite. Untuk pengguna individu, pembaruan akan tersedia hingga 2020.

Situasi itu mengkhawatirkan komunitas TI. Kekhawatiran terbesar adalah dengan Java 8, yang masih merupakan versi paling populer dari platform perangkat lunak. Misalnya, ini digunakan oleh pengembang Minecraft dan banyak digunakan di lingkungan cloud . Solusi Oracle, menurut para pakar keamanan informasi, dapat secara serius merusak keamanan ekosistem Java.

Jalan keluar dari situasi itu ditawarkan di Red Hat. Raksasa IT ini mengaku bertanggung jawab atas pembaruan OpenJDK 8 dan 11. Perusahaan akan mendukung mereka masing-masing hingga 2023 dan 2024.

Mengapa proyek ini menarik bagi Red Hat


Sejarah RedHat dan OpenJDK telah ada sejak 2007. Kemudian platform tidak sepenuhnya terbuka - sekitar 5% dari kode dipinjam dari aplikasi pihak ketiga. Red Hat memutuskan untuk memperbaiki kesalahpahaman dan bersama dengan Sun Microsystems (sekarang Oracle) meluncurkan proyek IcedTea . Tujuannya adalah untuk menghapus semua kode hak milik dari basis kode OpenJDK.

Setahun kemudian, OpenJDK menjadi open source, dan Red Hat mulai menggunakannya dalam produk mereka. Sejak itu, raksasa IT terus berpartisipasi dalam pengembangan dan pengembangan platform. Dari pembaruan terbaru, perusahaan telah mengusulkan untuk memasukkan algoritma pengumpulan sampah Shenandoah ke Jawa untuk meningkatkan produktivitas.

Red Hat terkait dengan OpenJDK oleh fakta bahwa perusahaan memiliki salah satu manajer teknis dari proyek open source ini, Andrew Haley. Di Red Hat, ia mengelola tim pengembang Java. Haley sudah melakukannya
Proyek Oracle "Intercepted" - dia bertanggung jawab atas OpenJDK 6 dan 7. "Seumur hidup" dari versi keenam sudah berakhir, dan dukungan untuk yang ketujuh akan berhenti tahun depan ( tabel 1 ). Oleh karena itu, Red Hat memiliki pengalaman dan sumber daya untuk mengawasi OpenJDK 8 dan 11.

Pendapat


Menurut analis, solusi Red Hat penting dalam hal keamanan informasi. Seperti yang telah kami katakan, banyak perusahaan dapat dibiarkan tanpa tambalan untuk melindungi aplikasi dan layanan mereka. Tepat sebelum pengumuman Oracle tentang penghentian dukungan perusahaan, platform menerima pembaruan keamanan di mana 254 bug diperbaiki.

Siapa lagi yang melakukan Java


Sebelumnya, Oracle menolak untuk mendukung set spesifikasi Java EE (Enterprise Edition), yang menggambarkan arsitektur platform server untuk tugas-tugas perusahaan menengah dan besar. Hak atas proyek pada tahun 2017 diteruskan ke organisasi nirlaba Eclipse Foundation dan sekarang platform ini disebut Jakarta EE.

Eclipse Foundation memperbarui produk-produk Edisi Perusahaan yang lebih lama dan memperluas fungsinya. Misalnya, pada awal 2019, pembaruan server GlassFish dirilis , yang menambahkan kompatibilitas dengan Java 8. Di masa depan, organisasi berencana untuk mengintegrasikan Jakarta EE dengan teknologi open source populer lainnya - Docker, Kubernetes, NoSQL.

Contoh lain adalah platform JavaFX. Ini dirancang untuk membuat aplikasi mobile dan desktop dengan antarmuka grafis yang kaya. Pada 2018, JavaFX diisolasi sebagai modul terpisah dan dihapus dari OpenJDK. Gluon memutuskan untuk mendukung platform. Organisasi baru-baru ini merilis Java FX versi 12, yang menambahkan fitur baru untuk aplikasi Android yang terkait dengan WebView. Gluon berencana untuk terus memperbarui produk.
