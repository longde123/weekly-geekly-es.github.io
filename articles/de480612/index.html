Nehmen Sie diese Änderungen vor, um die Barrierefreiheitsstandards für das Webdesign zu erfüllen.

Hallo habr Ich mache Sie auf eine Übersetzung des Artikels „Nehmen Sie diese Änderungen vor, um die Barrierefreiheitsstandards für das Webdesign zu erfüllen“ von Richard Pilton aufmerksam

Was ist Zugänglichkeit?


Barrierefreiheit (in zunehmendem Maße auch als Barrierefreiheit bezeichnet) ist das Design von Produkten, Geräten oder Diensten, die von möglichst vielen Menschen genutzt werden können, einschließlich Menschen mit Behinderungen wie Sehbehinderungen, körperlichen Schwierigkeiten, Lernschwierigkeiten oder Taubheit.



Verfügbarkeitsvorteile


Der Hauptvorteil des erfolgreichen Entwurfs einer erschwinglichen Plattform besteht darin, dass Ihre Anwendung oder Website für alle zugänglich ist. Es gibt jedoch auch viele andere Vorteile.

Vermeiden Sie Diskriminierung: Wenn Ihre Website oder Anwendung für jedermann gedacht ist, bedeutet dies, dass Sie eine Plattform geschaffen haben, die keine Diskriminierung zulässt. Die meisten Unternehmen diskriminieren Personen nicht absichtlich, aber einige Unternehmen konzentrieren sich nur auf visuelle und nicht auf grundlegende Prinzipien. Dies ist äußerst wichtig für Organisationen des öffentlichen Sektors, die von einem breiten Publikum genutzt werden. Durch die Fokussierung auf das Design können Website-Entwickler bestimmte Funktionen oder Kommunikationsmethoden eliminieren, die für die Bereitstellung des Zugriffs für bestimmte Personengruppen von entscheidender Bedeutung sind. Die Schaffung einer Plattform, die nicht nur die gesetzlichen Anforderungen erfüllt, sondern sich auch an die Bedürfnisse aller anpasst, wird eine solche Diskriminierung vermeiden.

Markenreputation : Eine zugängliche Website oder Anwendung kann die Reputation Ihrer Marke verbessern, indem sie ein breiteres Publikum anzieht, was letztendlich zu einem erhöhten Datenverkehr führt. Ebenso kann die Barrierefreiheit Sie von der Konkurrenz abheben, da sie zeigt, dass Sie für absolut alle Benutzer geeignet sind. Dies kann ein nützliches Marketinginstrument für Ihr Unternehmen sein.



Zu berücksichtigende Merkmale


Lassen Sie uns darüber sprechen, was Sie tun müssen, um die Empfehlungen zu befolgen und wie Sie eine erschwingliche Designstrategie erstellen können.

Hier ist eine Liste von Elementen, die besondere Aufmerksamkeit erfordern.


1. Layout: Stellen Sie sicher, dass Sie eine klare und einfache Gliederung haben. Überladenes Design kann Verwirrung stiften. Benutzer sollten in der Lage sein, auf die Größe von Funktionen zu reagieren und diese bei Bedarf zu vergrößern (z. B. mit Sehschwäche). Daher ist ein adaptives Layout von entscheidender Bedeutung.

2. Farbe: Für diejenigen, die farbenblind sein können, ist eine gut durchdachte Farbpalette erforderlich. Verlassen Sie sich nicht nur auf die Farbe, um wichtige Elemente Ihrer Benutzeroberfläche zu identifizieren. Fügen Sie in diesem Fall Text hinzu, um diese Bereiche anzugeben, damit alle Benutzer die angezeigten Informationen vollständig interpretieren und reibungslos auf der Site navigieren können.

3. Navigation: Klare Indikatoren und kurze Task-Streams sorgen für eine reibungslose Navigation. Die Schaltflächen müssen konsistent, insbesondere der Aufruf zum Handeln, und klar gekennzeichnet sein. Menüleisten und Suchfelder sollten ebenfalls in einem ähnlichen Format vorliegen, damit Benutzer problemlos auf Ihre Plattform geleitet werden können.

4. Zugänglichkeit der Tastatur: Betrachten Sie diejenigen, die die Maus nicht verwenden können. Dies bedeutet, dass Sie eine Site entwickeln, damit Benutzer (falls erforderlich) Ihre Ressource nur über die Tastatur navigieren können. Zum Beispiel mit verschiedenen Schaltflächen, einschließlich Pfeilen, Leertaste und Eingabe. Dies ist besonders wichtig für Menschen mit Behinderungen.

5. Messaging: Messaging sollte sorgfältig durchdacht werden. Einige Benutzer haben möglicherweise Probleme beim Lesen. Sie sollten daher in Betracht ziehen, den Text zu formatieren. Sprechen Sie einfach Englisch, verwenden Sie keine langen Sätze oder Absätze und vermeiden Sie Kursivschrift und ungleiche Leerzeichen. Denken Sie auch über Farbe und Schrift nach, ist sie lesbar und lesbar?

6. Audio und Video: Interaktivität sollte gegebenenfalls in jede Entwurfsstrategie einbezogen werden. Es ist wichtig zu bedenken, dass die Aufnahme von Audio- oder Videodaten die Verwendung verfügbarer Funktionen erfordert, insbesondere für Plattformen im öffentlichen Sektor. Benutzer sollten in der Lage sein, anzuhalten, die Lautstärke anzupassen und die Titel zu steuern. Untertitel sind auch eine gute Idee für das Posten von Inhalten an gehörlose Benutzer sowie für das Einschließen von Textversionen von Audiodateien.



Tests, Tests und Tests noch einmal


Sobald Sie klare Ziele festgelegt und eine barrierefreie Plattform entwickelt haben, ist der nächste Schritt das Testen. Hierzu gibt es verschiedene Ansätze, einschließlich automatisierter Eingabehilfen, mit denen Sie Fehler finden können. Sie können jedoch auch einfache Schritte ausführen, bevor Sie in diese Tools investieren. Wenn Sie beispielsweise die Maus deaktivieren und mit nur einer Tastatur auf der Website navigieren, können Sie die Verfügbarkeit der Tastatur überprüfen. Wenn Sie die Textgröße auf 300% erhöhen, wird angezeigt, ob Ihre Plattform für Personen mit Sehbehinderungen zugänglich ist.



Eine Person, die durch Ausprobieren an Ihrer Schnittstelle herumstochert, ist ein weiteres Testwerkzeug. Testen Sie die Plattform mit Ihrer Zielgruppe, Menschen mit und ohne Behinderungen, und sehen Sie, wie sie auf den Inhalt reagieren. Sie können Ihnen zeigen, dass die verfügbare Funktion nicht funktioniert. Dann können Sie zu Funktionen zurückkehren, die nicht funktioniert haben, und über alternative Lösungen nachdenken.
