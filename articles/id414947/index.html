Tanya jawab


Karena ReactOS saat ini ditujukan untuk kompatibilitas dengan Windows Server 2003, Git versi 2.10.0 dipilih sebagai kelinci percobaan, yang terakhir yang mengklaim dukungan untuk Windows XP dan 2003.


Pengujian dilakukan pada baris perintah ReactOS, gejala eksternal dari masalah agak campuran. Misalnya, ketika memulai git tanpa menentukan argumen tambahan, ia tanpa masalah akan menampilkan informasi bantuan di konsol. Tapi itu layak untuk dicoba git clone atau setidaknya git --version , dalam banyak kasus konsol tidak menampilkan apa - apa, atau kadang - kadang menunjukkan pesan rusak dengan tegas:

Kata penutup


Dengan demikian, bug lain yang secara tidak langsung menghambat perakitan ReactOS di dalam ReactOS diperbaiki melalui upaya kolektif. Suatu kebetulan lucu adalah kenyataan bahwa sesaat sebelum itu bug lain diperbaiki di msvcrt yang sama (yaitu, dalam fungsi qsort ), yang tidak memungkinkan merakit driver USB di ReactOS.


Saya berpartisipasi dalam pengembangan banyak proyek yang ditulis dalam berbagai bahasa pemrograman, baik sumber tertutup maupun sumber terbuka. Saya telah bekerja dengan proyek ReactOS sejak 2014, tetapi saya mulai aktif membantu dan menulis kode hanya pada 2017. Sangat menarik untuk bekerja di bidang ini, karena ini adalah sistem operasi keseluruhan! Orang merasakan skala besar dari hasilnya, di mana upaya diinvestasikan, serta perasaan menyenangkan bahwa satu bug menjadi kurang! :)


Seseorang mungkin akan bertanya-tanya mengapa saya membantu ReactOS, dan bukan Linux misalnya. Kebetulan secara historis bahwa dalam kebanyakan kasus saya menulis program untuk Windows, dan bahasa pemrograman favorit saya adalah Delphi. Mungkin itu sebabnya arsitektur Windows NT, bersama dengan Win32 API, sangat menarik bagi saya, dan proyek sistem operasi gratis ReactOS menghidupkan mimpi lama - dalam praktiknya, ini memungkinkan Anda untuk mengetahui bagaimana semuanya bekerja dari dalam.
