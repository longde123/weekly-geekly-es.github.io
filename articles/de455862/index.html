DATA API für Amazon Aurora Serverless


Wie funktioniert die Verbindung zu einer herkömmlichen Datenbank? Sie öffnen eine Verbindung, verwenden sie, um eine oder mehrere SQL-Abfragen oder andere Anweisungen zu verarbeiten, und schließen dann die Verbindung. Sie haben wahrscheinlich eine Clientbibliothek verwendet, die für Ihr Betriebssystem, Ihre Programmiersprache und Ihre Datenbank spezifisch ist. Irgendwann wurde Ihnen klar, dass das Erstellen von Verbindungen viel Zeit in Anspruch nahm und Speicherplatz im Datenbankmodul beanspruchte.

Die DATA-API kümmert sich um die Verwaltung und Skalierung langfristiger Datenbankverbindungen und gibt Daten in JSON-Form zur einfachen Analyse zurück. Der gesamte Datenverkehr ist kryptografisch und erfolgt über HTTPS.

Was ist die DATA-API?

ExecuteStatement - führt eine einzelne SQL-Abfrage aus.
BatchExecuteStatement - Führen Sie eine SQL-Abfrage für ein Datenarray aus.
BeginTransaction - Starten Sie eine Transaktion und geben Sie eine Transaktionskennung zurück.
CommitTransaction - Schließen Sie die Transaktion ab und schreiben Sie die darin ausgeführten SQL-Vorgänge fest.
RollbackTransaction - Rollback einer Transaktion.

Die einzige Einschränkung besteht darin, dass jede Funktion innerhalb einer Minute abgeschlossen sein muss und bis zu 1 Megabyte Daten zurückgeben kann.

Mit Amazon Kinesis Data Analytics können Sie Ihren Echtzeitanwendungen jetzt AWS-Ressourcen-Tags zuweisen


Mit zunehmendem Datenfluss steigt die Notwendigkeit, diese Daten auf besondere Weise zu verarbeiten und zu analysieren. Amazon Kinesis Data Analytics ist einer dieser Dienste, mit denen Sie durch Schreiben von SQL- oder Java-Code (mit Apache Flink) Daten aus verschiedenen Quellen erfassen sowie in Echtzeit verarbeiten und analysieren können. Nachdem Sie ein vollständigeres und mehrdimensionales Bild Ihrer Daten erhalten haben, können Sie auf einfache Weise neue nützliche Informationen extrahieren und schnell auf die Bedürfnisse von Unternehmen und Kunden reagieren.

Mit der wachsenden Anzahl solcher Anwendungen wächst die Notwendigkeit, sie kompetent zu verwalten. Amazon hat kürzlich angekündigt, dass es jetzt möglich ist, Ihren Anwendungen in Amazon Kinesis Data Analytics wie anderen AWS-Assets Asset-Tags hinzuzufügen. Das Tag ist ein Schlüssel-Wert-Paar, wobei der Wert ein optionaler Teil ist. Die Verwendung von Tags ist eine einfache, aber effektive Möglichkeit, AWS-Ressourcen zu verwalten und Daten zu organisieren. Beispiele für mögliche Tags:

Anwendung: Anwendungsname

Amazon S3 Path Deprecation Plan - Der Rest der Geschichte


In einem der beliebtesten S3-Dienste von Amazon Web Services (AWS) wird die Methode zum Abrufen von Dateien durch Angabe eines Pfadstils bald veraltet sein, z. B. s3.amazonaws.com/usmanovbf/docs/Bulat_Usmanov_CV.pdf oder mit einer Region - s3-us-east-2.amazonaws.com/usmanovbf/docs/Bulat_Usmanov_CV.pdf , wobei usmanovbf der Bucket und /docs/Bulat_Usmanov_CV.pdf der Schlüssel für die Datei in diesem Bucket ist.

Bevorzugt wird eine andere Methode - unter Verwendung eines virtuellen Hosts (virtuell gehosteter Stil), z. B. usmanovbf.s3.amazonaws.com/docs/Bulat_Usmanov_CV.pdf oder auch mit der Region - usmanovbf.s3-us-east-2.amazonaws. com / docs / Bulat_Usmanov_CV.pdf

Der Übergang wird voraussichtlich am 30. September 2020 erfolgen. Aufgrund der Offenheit der AWS-Ingenieure gegenüber der Community werden alle bis zu diesem Datum erstellten Links im Pfadstil nicht gelöscht. Das heißt, sie bleiben voll funktionsfähig. Kurz nach diesem Datum können Sie nur noch die zweite Methode verwenden - den virtuell gehosteten Stil.

Diese Änderung wird aus zwei Hauptgründen diktiert:
1. Ein zentraleres Modell, das über einen Pfad auf Dateien zugreift, schränkt die effektive Skalierung ein, da der gesamte Datenverkehr auf eine kleine Anzahl von Eingabepunkten gelangt: Unterdomänen der dritten Ebene s3, s3-us-east-2, s3-us-east -1 und so weiter. Aus dem gleichen Grund wird es unpraktisch, mit DNS-Namensauflösung, Sicherheit und Reflexion von DDOS-Angriffen zu arbeiten.
Mit dem virtuell gehosteten Stil können Sie den Datenfluss flexibler steuern, da Sie an einen separaten Subdomain-Bucket und an eine Region gebunden sind, z. B. usmanovbf und s3-us-east-2, wie im obigen Beispiel.

2. AWS-Ingenieure stehen nicht still. Um neue Funktionen einzuführen und die alten aufzugeben, müssen sie auf den virtuell gehosteten Stil umsteigen. Beispielsweise ist geplant, die alten Verschlüsselungsmethoden aufzugeben.

Weitere Informationen zur Migration auf die neue Methode finden Sie im offiziellen AWS-Blog.
