JDBC pools dan penanganan file yang efisien: Java mitap 3 Desember di St. Petersburg



Kami mengundang Anda ke Jawa mitap tentang pertunjukan , yang akan diselenggarakan pada 3 Desember di kantor St Petersburg Wrike. Mari kita bicara tentang level rendah, tetapi penting untuk pengembang dari segala level yang terkadang tidak memiliki cukup ruang di konferensi-java.

Dalam program


Alexander Kolesnikov, Wrike - JDBC Pools Battle

Alexander bekerja sebagai backend techlide / SRE di Wrike. Sebagian besar layanan backend di perusahaan ditulis dalam Java, penyimpanan utama adalah database relasional (PostgreSQL). Toleransi kesalahan dan kinerja database sangat penting Wrike memberi pengguna layanan uptime 99,99%.

Dalam laporan itu, Alexander akan menjawab pertanyaan-pertanyaan berikut:

  1. Untuk apa kolam JDBC?
  2. Mengapa ada begitu banyak implementasi, dan apa perbedaannya;
  3. Kumpulan mana yang akan dipilih untuk aplikasi Anda; 4) Pengaturan kolam apa yang perlu diperhatikan.

Dmitry Govorukhin, GridGain - Manajemen file yang efektif untuk pengembang Java

Dmitry adalah pemimpin tim / pengembang utama di GridGain. Spesialisasi utama adalah sistem terdistribusi, masalah kinerjanya, toleransi kesalahan dan penskalaan.

Dmitry akan berbicara tentang pengalaman pribadi dalam mengembangkan aplikasi intensif disk dan rake yang dikumpulkan selama pengembangan Apache Ignite Native Store:

  • apa lagi selain beban kerja Anda yang memengaruhi kinerja;
  • apa yang perlu Anda lakukan untuk memeras maksimal dari disk;
  • kesalahan umum dan kesalahpahaman saat bekerja dengan Java IO API.

Kami menunggu Anda di kantor Wrike pada 3 hingga 19 Desember. Acara ini gratis, diperlukan pendaftaran. Setelah program resmi - komunikasi gratis, jaringan, pizza dan bir.

Lebih lanjut tentang cara menuju ke sana.
