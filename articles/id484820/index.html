FAQ.Net - program pencatatan gratis untuk Windows dengan desain yang diperbarui

Pendahuluan


Selama 3 bulan, aplikasi FAQ.Net telah diunduh lebih dari 800 kali. Setiap orang yang menggunakan buku catatan dan mencari solusi turnkey untuk menyimpan informasi, catatan, atau dokumen penting (dalam bentuk teks, gambar, tabel) dalam satu basis data, unduh FAQ.Net. Aplikasi ini portabel, gratis, tanpa iklan dan tidak memerlukan akses Internet. Kode sumber diposting di github .

Komunitas VK menyelenggarakan instruksi video dan berita aplikasi. Setiap orang dapat bergabung dengan komunitas, berpartisipasi dalam survei, dan mengajukan pertanyaan tentang pengoperasian aplikasi FAQ.Net.





FAQ.Fitur aplikasi Net


  • Semua catatan (dokumen RTF) disimpan dalam database SQLite tunggal
  • Ada "Kamus kiat" (lihat detail ) - ini adalah tooltips yang dibuat oleh pengguna dan dapat digunakan:
    • untuk menavigasi catatan lain di dalam aplikasi
    • sebagai penerjemah, yang sangat membantu ketika mempelajari kata-kata asing baru
    • untuk menyimpan tautan Internet di satu tempat
    • untuk menyimpan deskripsi subjek, acara, kata, dll.
  • Pelengkapan kata-kata secara otomatis sesuai dengan "Kamus permintaan" (IntelliSense) dengan kombinasi tombol "Ctrl + Spasi"
  • Pengaturan antarmuka yang fleksibel (font, warna, ikon)
  • Menambahkan gambar. Jika gambar disisipkan dari editor Paint melalui buffer (mis., Menggunakan kombinasi tombol Ctrl + C - Ctrl + V), maka gambar tersebut dapat diedit dengan mengklik dua kali
  • Buat tabel
  • Membuat salinan cadangan dari basis data
  • Pratinjau dan cetak dokumen
  • Pencarian cepat pada semua catatan
  • Tab Favorites berisi daftar catatan yang ditandai oleh pengguna.
  • Tab Journal menampilkan 50 catatan yang paling baru dimodifikasi

Daftar perubahan di FAQ.Net versi 2.9


  1. Memperbaiki tampilan aplikasi. Menambahkan gradient fill untuk banyak komponen yang warnanya dapat disesuaikan oleh pengguna
  2. Tema yang diterapkan untuk kustomisasi tampilan yang cepat
  3. Kemampuan untuk memilih satu set ikon, serta menambahkan ikon Anda sendiri atau mengganti yang sudah ada
  4. Kemampuan untuk menyembunyikan / menampilkan ikon untuk menghemat ruang layar
  5. Transfer kotak kombo ke bilah menu dengan perataan kanan (ini adalah kotak kombo untuk memilih gaya font, ukuran font dan penskalaan dokumen)
  6. Tombol Putaran Kembali / Maju untuk menavigasi dokumen yang sebelumnya dibuka
  7. Menambahkan kemampuan untuk memasukkan gambar dari file. Ketika Anda memilih beberapa file dengan gambar, mereka digabungkan menjadi satu, dengan kemampuan untuk menyesuaikan resolusi dengan cepat dengan mengubah ukuran jendela.
  8. Tampilan tambahan jumlah karakter dalam catatan
  9. memperbaiki Memperbaiki tampilan tooltips agar tidak melampaui batas area dokumen yang ditampilkan

Sesuaikan Penampilan


Dalam salah satu artikel saya, saya jelaskan secara rinci, dengan contoh, cara universal untuk menyesuaikan tampilan aplikasi oleh pengguna di bawah WinForms , di mana saya menerima penilaian ketat bahwa gaya tidak boleh diubah oleh pengguna, bahwa gaya harus ditentukan oleh pengaturan OS, dll.

Tetapi setelah membuat cara sederhana untuk menyesuaikan tampilan (di mana semua warna, font, tampilan objek disimpan dalam satu file XML), sangat mudah untuk menerapkan tema dan sekarang tema ini dapat dibuat oleh pengguna sendiri, yang dapat dengan mudah mendistribusikannya ke komputer lain dengan menyalin file XML.



Apa yang akan ditingkatkan lebih lanjut dalam aplikasi FAQ.Net?


Ada banyak ide untuk mengembangkan aplikasi Net FAQ, berikut adalah beberapa di antaranya:

  • kemampuan untuk melampirkan file ke catatan
  • menyinkronkan catatan dengan internet
  • tambahkan keranjang
  • tag untuk catatan
  • penyorotan sintaks dari fragmen teks yang dipilih sesuai dengan kata-kata dari "kamus permintaan"
  • perluas daftar format (tambahkan Excel, HTML)

PS: Setelah melakukan survei terhadap pembaca Habr yang disegani dengan topik " Apa yang dibutuhkan dalam aplikasi pencatatan? " Saya menemukan bahwa saya perlu mengembangkan ke arah lintas platform. Saya harap ini bisa diimplementasikan.
