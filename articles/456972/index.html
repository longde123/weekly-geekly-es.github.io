5 excelentes formas de animar aplicaciones React en 2019



La animación en las aplicaciones React es un tema popular y discutido. El hecho es que hay muchas formas de crearlo. Algunos desarrolladores usan CSS agregando etiquetas a las clases HTML. Gran manera, vale la pena usar. Pero, si desea trabajar con tipos complejos de animaciones, debe tomarse el tiempo para estudiar GreenSock, es una plataforma popular y poderosa. También hay muchas bibliotecas y componentes para crear animaciones. Hablemos de ellos.

Este artículo analiza cinco formas de animar aplicaciones React:

  • CSS
  • ReactTransitionGroup;
  • Reaccionar animaciones
  • Reaccionar-revelar;
  • TweenOne y Ant Design.

CSS


Este método fue mencionado al principio, y es realmente bueno. Si en lugar de importar bibliotecas de JavaScript y usarlo, el ensamblaje será pequeño, el navegador no necesitará muchos recursos. Y esto, por supuesto, afecta el rendimiento de la aplicación. Si su animación debe ser relativamente simple, preste atención a este método.

Un ejemplo es un menú animado:



Es relativamente simple, con una propiedad CSS y un disparador como className = "is-nav-open" para la etiqueta HTML.

Hay varias formas de usar este método. Por ejemplo, cree un contenedor sobre la navegación y luego invoque los cambios de campo. Como la navegación tiene un ancho constante de 250 px, el ancho del contenedor con la propiedad margin-left o translateX debe ser el mismo ancho. Si necesita mostrar la navegación, agregue className = "is-nav-open" para el contenedor y mueva el contenedor al margen izquierdo / translateX: 0;.

Finalmente, la fuente de animación se verá así: 

export default class ExampleCss extends Component { handleClick() { const wrapper = document.getElementById('wrapper'); wrapper.classList.toggle('is-nav-open') } render() { return ( <div id="wrapper" className="wrapper"> <div className="nav"> <icon className="nav__icon" type="menu-fold" onClick={() => this.handleClick()}/> <div className="nav__body"> Lorem ipsum dolor sit amet, consectetur adipisicing elit. Repito, si la animación es relativamente simple, entonces este método es el principal. Los usuarios apreciarán el rendimiento del navegador.

ReactTransitionGroup


El componente ReactTransitionGroup fue desarrollado por el equipo de la comunidad ReactJS. Con él, puede implementar fácilmente animaciones y transiciones CSS básicas.

ReactTransitionGroup está diseñado para cambiar las clases al cambiar el ciclo de vida del componente. Tiene un tamaño pequeño, debe instalarse en el paquete para la aplicación React, lo que aumentará ligeramente el tamaño general del ensamblaje. Además, puede usar CDN.

ReactTransitionGroup tiene tres elementos, estos son Transition, CSSTransition y TransitionGroup. Para comenzar la animación, debe envolver el componente en ellos. El estilo, a su vez, debe escribirse en clases CSS.

Aquí está la animación, y luego la forma de implementarla.



El primer paso es importar el CSSTransitionGroup desde react-transition-group. Después de eso, debe ajustar la lista y establecer la propiedad transitionName. Cada vez que agrega o elimina un elemento secundario en un CSSTransitionGroup, obtiene estilos animados. 

 <CSSTransitionGroup transitionName="example"> {items} </CSSTransitionGroup>

Al establecer la propiedad transitionName = "example", las clases en las hojas de estilo deben comenzar con el nombre del ejemplo. 

 .example-enter { opacity: 0.01; } .example-enter.example-enter-active { opacity: 1; transition: opacity 300ms ease-in; } .example-leave { opacity: 1; } .example-leave.example-leave-active { opacity: 0.01; transition: opacity 300ms ease-in;

Arriba hay un ejemplo usando ReactTransitionGroup.

También necesita lógica y dos métodos para implementar el ejemplo de agregar una lista de contactos.

El primer método handleAdd: agrega nuevos contactos, obtiene un nombre aleatorio, que luego coloca en la matriz state.items.

Para eliminar un contacto por índice en la matriz state.items, use handleRemove. 

 import React, { Component, Fragment } from 'react'; import { CSSTransitionGroup } from 'react-transition-group' import random from 'random-name' import Button from './button' import Item from './item' import './style.css'; export default class ReactTransitionGroup extends Component { constructor(props) { super(props); this.state = { items: ['Natividad Steen']}; this.handleAdd = this.handleAdd.bind(this); } handleAdd() { let newItems = this.state.items; newItems.push(random()); this.setState({ items: newItems }); } render () { const items = this.state.items.map((item, i) => ( <Item item={item} key={i} keyDelete={i} handleRemove={(i) => this.handleRemove(i)} /> )); return ( <Fragment> <Button onClick={this.handleAdd}/> <div className="project"> <CSSTransitionGroup transitionName="example" transitionEnterTimeout={500} transitionLeaveTimeout={300} > {items} </CSSTransitionGroup> </div> </Fragment> ); } };

Reaccionar animaciones


React-animations es una biblioteca que se basa en animate.css. Es fácil trabajar con ella, tiene muchas colecciones de animación diferentes. La biblioteca es compatible con cualquier biblioteca de estilo en línea que admita el uso de objetos para definir los cuadros principales de una animación, incluidos Radium, Afrodita o componentes con estilo.



Sé lo que piensas:



Ahora veamos cómo funciona esto con el ejemplo de una animación de rebote.



Primero, importe la animación desde las animaciones de reacción.

const Bounce = styled.div`animation: 2s $ {keyframes` $ {bounce} `} infinite`;

Luego, después de crear el componente, envolvemos cualquier código HTML o componente para la animación. 

 <bounce><h1>Hello Animation Bounce</h1></bounce>

Un ejemplo: 

 import React, { Component } from 'react'; import styled, { keyframes } from 'styled-components'; import { bounce } from 'react-animations'; import './style.css'; const Bounce = styled.div`animation: 2s ${keyframes`${bounce}`} infinite`; export default class ReactAnimations extends Component { render() { return ( <Bounce><h1>Hello Animation Bounce</h1></bounce> ); } }

Todo funciona, la animación es muy simple. Además, hay una gran solución para usar la animación de rebote al desplazarse: reaccionar-animar-en-desplazamiento .

Reaccionar-revelar


El marco React Reveal tiene animaciones básicas, que incluyen desvanecimiento, reflexión, escalado, rotación y más. Permite trabajar con todas las animaciones usando accesorios. Por lo tanto, puede especificar configuraciones adicionales, incluyendo posición, retraso, distancia, cascada y otros. Se pueden utilizar otros efectos CSS, incluidos la representación del lado del servidor y los componentes de alto orden. En general, si necesita animación de desplazamiento, debe usar este marco. 

 import Fade from 'react-reveal/Fade'; <Fade top> <h1>Title</h1> </Fade>



Hay cinco bloques en total, cada uno de ellos tiene una página de pantalla completa y un título. 

 import React, { Component, Fragment } from 'react'; import Fade from 'react-reveal/Fade'; const animateList = [1, 2, 3, 4, 5]; export default class ReactReveal extends Component { render() { return ( <Fragment> {animateList.map((item, key) => ( <div style={styles.block} key={key}> <Fade top> <h1 style={styles.title}>{`block ${item}`}</h1> </Fade> </div> ))} </Fragment> ); } } const styles = { block: { display: 'flex', alignItems: 'center', justifyContent: 'center', width: '100%', height: '100%', background: '#000', borderBottom: '1px solid rgba(255,255,255,.2)', }, title: { textAlign: 'center', fontSize: 100, color: '#fff', fontFamily: 'Lato, sans-serif', fontWeight: 100, }, };

Ahora presentamos la constante animateList. La matriz incluye cinco elementos. Después de usar el método de matriz de mapa, es posible representar cualquier elemento en los componentes de Fade insertando elementos en el encabezado. Los estilos que se definen en la constante de estilos obtienen estilos CSS cortos tanto para el bloque como para el encabezado. Arriba hay cinco bloques con animación Fade.

TweenOne y animación en Ant Design


Ant Design es una biblioteca React UI que contiene una gran cantidad de componentes útiles y fáciles de usar. Es adecuado si necesita crear interfaces de usuario elegantes. Desarrollado por Alibaba, que utiliza la biblioteca en muchos de sus proyectos.



El ejemplo tiene bastantes componentes animados. La mayoría de ellos tienen animaciones similares, por lo que una implementación de ejemplo será más simple que la anterior. Esto incluirá solo una bola, una bola verde y un elemento adicional, por ejemplo, un cuadrado rojo.



La animación utiliza el componente TweenOne, que necesita PathPlugin para establecer correctamente la trayectoria. Todo esto funcionará solo si se coloca
PathPlugin en TweenOne.plugins. 

 TweenOne.plugins.push(PathPlugin);

Las principales opciones de animación son las siguientes:

  • duración - tiempo de animación en ms;
  • facilidad - suavidad de la animación;
  • yoyo - cambio de movimiento hacia adelante y hacia atrás con cada repetición;
  • repetir: repetir la animación. Necesitas usar -1 para una animación sin fin;
  • p - las coordenadas de la ruta para la animación;
  • easePath: coordenadas de la ruta suave para la animación.

Los dos últimos parámetros son muy específicos, pero no se preocupe por ellos, todo funciona como debería. 

 const duration = 7000; const ease = 'easeInOutSine'; const p = 'M123.5,89.5 C148,82.5 239.5,48.5 230,17.5 C220.5,-13.5 127,6 99.5,13.5 C72,21 -9.5,56.5 1.5,84.5 C12.5,112.5 99,96.5 123.5,89.5 Z'; const easePath = 'M0,100 C7.33333333,89 14.3333333,81.6666667 21,78 C25.3601456,75.6019199 29.8706084,72.9026327 33,70 C37.0478723,66.2454406 39.3980801,62.0758689 42.5,57 C48,46.5 61.5,32.5 70,28 C77.5,23.5 81.5,20 86.5,16 C89.8333333,13.3333333 94.3333333,8 100,0'; const loop = { yoyo: true, repeat: -1, duration, ease, };

Ahora puede comenzar a crear el objeto de animación.

  • redSquare contiene los parámetros del bucle más la coordenada Y, la duración y el retraso.
  • greenBall contiene la ruta con los parámetros del objeto x, y - el valor de p. Además, la duración, la repetición y la suavidad son una función de TweenOne.easing.path, que tiene dos parámetros.
  • camino - easyPath.
  • lengthPixel es una curva que se divide en solo 400 secciones.
  • track: un óvalo con ejes, tiene estilos de ciclo y un parámetro de rotación. 

 const animate = { redSquare: { ...loop, y: 15, duration: 3000, delay: 200, }, greenBall: { path: { x: p, y: p }, duration: 5000, repeat: -1, ease: TweenOne.easing.path(easePath, { lengthPixel: 400 }), }, track: { ...loop, rotate: 15, }, };
