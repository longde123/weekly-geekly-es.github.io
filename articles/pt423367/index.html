Apresentando o Azure DevOps

Lançamento do DevOps do Azure! Hoje falaremos brevemente sobre essa nova plataforma, suas soluções, o processo de transição do Visual Studio Team Services e sobre o uso em projetos reais. Inscreva-se agora!



Hoje anunciamos o lançamento do Azure DevOps. Ao nos comunicarmos com nossos clientes e desenvolvedores em todo o mundo, percebemos que o DevOps é essencial para o sucesso do trabalho em equipe. O Azure DevOps incorpora mais de 15 anos de experiência na criação de ferramentas de desenvolvimento de software. Para projetos de código aberto, você pode usar o desenvolvimento e implantação contínuos com os Pipelines do Azure gratuitamente.


Como confirmação de que pretendemos fornecer ferramentas abertas e flexíveis para todos os desenvolvedores, nos Pipelines do Azure, você pode executar o desenvolvimento gratuito gratuito e a implantação de até dez tarefas simultaneamente para cada projeto de código aberto gratuitamente. Os pipelines do Azure são ideais para qualquer tipo de projeto, pois possuem um cluster de nuvem Linux, MacOS e Windows.

Muitos dos maiores projetos de código aberto já usam os Pipelines do Azure para desenvolvimento e implantação contínuos, como Atom , CPython , Pipenv , Tox , Visual Studio Code e TypeScript , e essa lista está crescendo a cada dia.

Queremos fornecer um serviço de primeira classe para todos. Portanto, os desenvolvedores de projetos de código aberto usam a mesma infraestrutura que os usuários de serviços pagos.

Desenvolvimento do Visual Studio Team Services (VSTS)


O Azure DevOps é o resultado do desenvolvimento do Visual Studio Team Services (VSTS). Os usuários do VSTS serão migrados automaticamente para o Azure DevOps. Os usuários existentes não perderão a funcionalidade, mas terão mais opções e controle. A total transparência e integração, os recursos distintos do VSTS, são preservados. Os serviços do Azure DevOps funcionam perfeitamente juntos. Hoje começaremos a transferir usuários e, nos próximos meses, eles verão todas as alterações. O que isso significa?

  • Os URLs mudarão de abc.visualstudio.com para dev.azure.com/abc. Os redirecionamentos do endereço de visualstudio.com serão ativados, para que você não encontre links quebrados.
  • Como o serviço mudou, a interface também foi atualizada . Continuaremos a aprimorá-lo com base no feedback do usuário da versão de avaliação. Para novos usuários, ele será ativado por padrão. Nos próximos meses, ele se tornará o principal para os usuários existentes.
  • O serviço local do Team Foundation Server (TFS) continuará sendo atualizado, as atualizações serão semelhantes às lançadas para o Azure DevOps. Após o lançamento da próxima versão do TFS, o produto receberá um novo nome - Azure DevOps Server e será atualizado no modo anterior.
