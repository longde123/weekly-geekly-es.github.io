Kami menghidupkan kembali pengereman Samsung Galaxy TAB 2 WiFi

Saya mengingatkan resep saya untuk merevitalisasi tablet pengereman yang lama dan memperbarui peranti lunaknya.

Tahun ini saya melakukan audit dan menjual perangkat dan komponen saya yang lama dan tidak digunakan. Saya harus mengatakan bahwa saya benci menonton video dan membaca dari telepon, karena tidak ada perendaman total - bekerja dan tidak banyak panggilan, surat masuk, pesan dari kurir cukup mengganggu saya. Waktu berlalu, dan saya mempelajari penawaran toko dan memilih rasio harga / besi, tetapi tidak berhenti pada apa pun.

Itu adalah giliran membersihkan garasi, di mana saya menemukan Samsung Galaxy Tab 2 7.0 "P3110 (tanpa slot kartu SIM). Untuk bersenang-senang, saya menyalakannya, berkedip di layar dan menyalakan Android 4.1.

Bagus sekali, saya pikir, perangkat keras yang andal, jika selama beberapa tahun di ruang yang tidak dipanaskan itu dimulai dan dinyalakan. Mungkin saya akan mengambilnya di bawah buku saya. Sebenarnya, ini adalah bagaimana saya mendapat kesempatan untuk membaca kembali isi halaman beberapa kali, sementara teks bergulir ke yang berikutnya, dan selangkah demi selangkah untuk menonton acara TV favorit saya.

Pertama, dengan firmware terbaru, banyak layanan masuk, yaitu "pembaruan", "debugging" dan "dari pabrikan", yang tidak dapat diputuskan, tetapi jika Anda menghentikannya, kinerja akan meningkat secara signifikan.

Kedua, versi lama ketika saya mencoba menempatkan 50% perangkat lunak dari Play Market memberi tahu saya bahwa "mungkin perangkat lunak tidak dioptimalkan untuk versi OS Anda."

Setelah mempelajari hasil pencarian, dan tersandung pada penyebutan firmware khusus Lineage-OS, diputuskan untuk mengejek perangkat, karena masih belum ada manfaat praktis.

Proses firmware dijelaskan dalam berbagai manual, tetapi masih mencari jalan memakan waktu, saya menerbitkan resep saya.
UPD: situs resmi lineage-os
Dari sini kami mengambil firmware khusus , bootloader TWRP alternatif, kabel USB, driver USB Samsung (untuk mengunggah firmware TWRP), perangkat lunak ODIN versi terbaru, Open Gapps - Play Market untuk versi Android yang dipilih, kartu microSD 8GB, komputer desktop Windows 10.

Persiapan:

1. menyebarkan Odin dari arsip (tidak memerlukan instalasi) ke disk, instal driver USB Samsung.

2.1. Unduh TWRP. Ternyata, ODIN menjahit dari format tar atau tar.md5 khusus. Dan di situs web TWRP ada file img dengan loader baru. Android dari versi 7 mengubah format gambar boot, loader 2.8 tidak menerima format ini. Oleh karena itu, paragraf 2.2

2.2. Unduh utilitas format file tar , yang akan menerima ODIN. Ubah nama file TWRP yang diunduh menjadi "recovery.img" dan letakkan di direktori "bagian" utilitas. Kami meluncurkan file BAT dan dalam beberapa detik kami mengambil file twrp-3.1-p3110.tar dari direktori "paket", setelah menjawab pertanyaan apa yang kami lakukan (titik 1 - file tar) dan tombol "Y".

3. Unduh Gapps, dalam kasus saya, jenis prosesor adalah ARM, OS 7.1, piko Gapps (set minimum untuk Play Market berfungsi)

Proses:

4. Isi OS MicroSD, Gapps. Kami memasukkan ke dalam tablet.

5. Pada tablet mati, tekan dan tahan tombol daya dan tombol volume turun. Kami masuk ke menu firmware bootloader dan klik "Konfirmasi" - tombol kenaikan suara - driver layar tidak memuat dalam mode firmware bootloader.

6. Kami menghubungkan tablet dan komputer dengan kabel, port Com tertentu (dalam kasus saya COM6) akan muncul di ODIN. Pilih item â€œAPâ€ dan tentukan file firmware TWRP (twrp-3.1-p3110.tar) di dalamnya. Kami melakukan reboot tablet karena ODIN menyelesaikan firmware.

7. Tahan tombol daya dan tambahkan suara sampai layar kosong dan model tablet muncul lagi. Selanjutnya, jendela bootloader TWRP terbuka.

Ada tiga hal yang harus dilakukan: membuat cadangan firmware saat ini untuk rollback, menghapus bagian boot, cashe dan data, mengisi OS dan Gapps.

8. Kami membuat cadangan partisi, memeriksa Sistem, Cache, Data, Boot (Penyimpanan Internal juga dapat ditentukan jika Anda sebelumnya menyimpan data penting dari sana - ini adalah memori internal tablet). Geser untuk melakukan pencadangan, tunggu.



9. Hapus dengan lembut (Wipe) menggunakan bagian Advanced Wipe - kotak centang yang berlawanan dengan Sistem, Cache, Data, dan geser.

10. Kami melanjutkan untuk menginstal OS: Instal, pilih arsip lineage-os, tombol add archive gapps. untuk memenuhi.

11. Setelah selesai, pilih Reboot. Pada tahap ini, saya tidak merekomendasikan untuk bereksperimen dengan tawaran bootloader untuk menginstal Super SU - OS saya tidak lepas landas, klik Jangan Instal.

Dari kesalahan:
Tidak ada cukup RAM untuk menginstal OS dan Gapps - kami memilih arsip Gapps yang lebih kecil.
OS boot (logo terlihat seperti senyum yang sangat sedih) dan tidak ada yang terjadi - lepaskan microSD dan coba boot lagi
Tidak ada Play Market di OS - versi Gapps yang salah
Saya masih belum memiliki mode lansekap untuk desktop, meskipun aplikasi berjalan dalam mode lansekap. Saya mencari solusi, mungkin ada di komentar.



Terima kasih atas perhatiannya !!!
