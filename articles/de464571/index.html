Gehirn + VPS für 30 Rubel =?

Es ist schön, wenn alle notwendigen Kleinigkeiten zur Hand sind: ein gut schreibender Stift und ein Notizbuch, ein angespitzter Bleistift, eine bequeme Maus, ein paar zusätzliche Drähte usw. Diese unauffälligen Dinge ziehen keine Aufmerksamkeit auf sich, sondern sorgen für ein komfortables Leben. Die gleiche Geschichte mit verschiedenen mobilen und Desktop-Anwendungen: für lange Screenshots, um die Bildgröße zu reduzieren, um persönliche Finanzen, Wörterbücher, Übersetzer, Konverter usw. zu berechnen. Haben Sie einen solchen VPS - der kostengünstig ist, immer zur Hand ist und viele Vorteile bringt? Nein, nicht die, die Sie in Ihrer Firma haben, sondern Ihre eigene "Tasche". Wir dachten, dass es ohne einen kleinen VPS im Jahr 2019 irgendwie traurig war, genau wie ohne den üblichen Füllfederhalter bei einem Vortrag. Warum traurig sein? Sommer ist. Nun, wie im Sommer. IT-Sommer: Sitzen Sie zu Hause, schneiden Sie Lieblingsprojekte ohne Bedauern. Im Allgemeinen gedacht und getan.

Wir lesen Artikel von Wettbewerbern und Anwendern, die vor 3-4 Jahren darüber geschrieben haben, warum kostengünstiges VPS nicht benötigt wird. Nun, richtig, dann war VPS "für einen Cent" reines Marketing und konnte keine normalen Arbeitsmöglichkeiten bieten. Aber die Zeiten ändern sich, die Kosten für virtuelle Ressourcen werden immer niedriger und für 30 Rubel pro Monat sind wir bereit, dies anzubieten:

  • Prozessor: Intel Xeon 2 GHz (1 Kern)
  • Linux-System (Debian, Ubuntu, CentOS zur Auswahl)
  • 1 dedizierte IPv4-Adresse
  • 10 GB für die Datenspeicherung auf schnellen SSDs der Enterprise-Klasse
  • RAM: 512 MB
  • Abrechnung pro Sekunde
  • Unbegrenzter Verkehr

Der Tarif unterliegt zusätzlichen technischen Einschränkungen, Details auf der Seite unseres coolen Angebots - VPS für 30 Rubel.

Wer sollte einen solchen virtuellen Server verwenden? Ja, fast jeder: Anfänger, Enthusiasten, erfahrene Entwickler, DIY-Fans und sogar einige Unternehmen.

Wofür ist dieser VPS geeignet?


Wir glauben, dass Habrs Leser definitiv ihren eigenen Weg finden werden, eine solche Konfiguration anzuwenden, aber sie haben beschlossen, ihre eigene Auswahl an Ideen zusammenzustellen - sonst braucht es plötzlich jemand, aber die Männer wissen es nicht?

  • Veröffentlichen Sie Ihre einfache Website, Ihr Portfolio, Ihren Lebenslauf mit Code und so weiter. Natürlich macht eine selbst gestaltete Website einen positiven Eindruck auf den Arbeitgeber. Platzieren Sie es auf Ihrem VPS und seien Sie selbst für die Sicherheit und Stabilität der Website verantwortlich und nicht für die Mitarbeiter von Shared Hosting.
  • Verwenden Sie VPS für Bildungszwecke: Veröffentlichen Sie Ihr Projekt, studieren Sie die Funktionen des Servers und des Server-Betriebssystems, experimentieren Sie mit DNS, graben Sie eine kleine Schulungssite.
  • Für die Telefonie. Manchmal braucht ein einzelner Unternehmer, Freiberufler oder ein sehr kleines Unternehmen dringend IP-Telefonie, und die Betreiber dieser Telefonie sind sehr gierig. Es ist ganz einfach: Wir nehmen unseren Server, kaufen eine Nummer beim IP-Telefonisten, richten eine virtuelle Telefonanlage ein und erstellen (falls erforderlich) interne Nummern. Die Einsparungen sind enorm.
  • Verwenden Sie einen Server, um Ihre Anwendungen zu testen.
  • Verwenden Sie den Server für DIY-Experimente, einschließlich zum Verwalten und Sammeln von Daten von den Sensoren eines Smart-Home-Systems.
  • Eine ungewöhnliche Anwendungsmethode besteht darin, einen virtuellen Assistenten des Börsenhandels, einen Handelsroboter, auf dem Server zu platzieren. Sie sind voll verantwortlich für die Stabilität und Sicherheit des Servers, was bedeutet, dass Sie ein kontrolliertes Instrument für den Handel an den Aktienmärkten erhalten. Nun, plötzlich mag oder plant jemand :-)

Es gibt eine solche VPS-Anwendung im Unternehmensbereich. Neben dem bereits erwähnten Telefondienst können Sie einige interessante Teile implementieren. Zum Beispiel:

  • Platzieren kleiner Datenbanken und Informationen, die Mitarbeitern aus der Ferne zur Verfügung stehen, z. B. mithilfe von FTP. Auf diese Weise können Sie schnell neue Analysen, aktualisierte Konfigurationen für Verkäufer, Präsentationen usw. austauschen.
  • Gewähren Sie Benutzern oder Clients temporären Zugriff, um Software oder Multimedia zu demonstrieren.

VPS-Probefahrt für 30 Rubel - für Sie erledigt


30 Rubel sind so klein, dass selbst die Zurückhaltung, eine Karte zum Bezahlen und Testen zu bekommen. Wir sind auch manchmal so faul, aber diesmal haben wir alles für Sie getan. Bevor wir die Server im Kampf starten, haben wir einen Test durchgeführt, um alle Details zu überprüfen und zu zeigen, wozu die Server zu diesem Tarif in der Lage sind. Um es interessanter zu machen, haben wir Extremsportarten hinzugefügt und überprüft, wie sich diese Konfiguration verhält, wenn Dichte und Last die von uns festgelegten Werte überschreiten.

Der Host wurde von einer Reihe virtueller Maschinen belastet, die verschiedene Aufgaben auf dem Prozessor ausführten und das Festplattensubsystem aktiv verwendeten. Ziel ist es, eine hohe Platzierungsdichte und eine Last zu simulieren, die vergleichbar oder größer als die des Kampfes ist.

Zusätzlich zur konstanten Last haben wir 3 virtuelle Maschinen installiert, die mithilfe von sysbench synthetische Metriken erfassen, deren durchschnittliche Ergebnisse unten angegeben sind, sowie 50 virtuelle Maschinen, die zusätzliche Last erzeugt haben. Alle virtuellen Testmaschinen hatten dieselbe Konfiguration (1 Kern, 512 GB RAM, 10 GB SSD). Als Betriebssystem wurde das Standard-Debian 9.6-Image ausgewählt, das Benutzern auf RUVDS angeboten wird.

Die Ladung wurde in Art und Größe simuliert, vergleichbar mit dem Kampf:

  • Einige der virtuellen Maschinen wurden mit geringer Last gestartet.
  • Einige Maschinen "verdrehten" ein Testszenario, das die Belastung des Prozessors simuliert (mithilfe des Stress- Dienstprogramms).
  • Auf den restlichen virtuellen Maschinen haben wir ein Skript ausgeführt, das dd verwendet, um Daten von vorbereiteten Daten mit einer mit pv festgelegten Einschränkung auf die Festplatte zu kopieren (Beispiele finden Sie hier und hier ).

Wie Sie sich erinnern, hatten wir drei Autos, die synthetische Metriken sammelten.

Auf jedem Computer wurde zyklisch alle 15 Minuten ein Skript ausgeführt, das Standard-Sysbench-Tests für Prozessor, Speicher und Festplatte ausführt.

#!/usr/bin/python
import mysql.connector as mariadb
from flask import Flask, json, request, abort
from mysql.connector.constants import ClientFlag

app = Flask(__name__)

def getFields(cursor):
    results = {}
    column = 0
    for d in cursor.description:
        results[d[0]] = column
        column = column + 1
    return results

PAGE_SIZE = 30

@app.route("/")
def main():
    return "Hello!"

@app.route("/employees/<page>", methods=['GET'])
def getEmployees(page):
    offset = (int(page) - 1) * PAGE_SIZE
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees')
    cursor = connection.cursor()
    cursor.execute("SELECT * FROM employees LIMIT {} OFFSET {}".format(PAGE_SIZE, offset))
    return {'employees': [i[0] for i in cursor.fetchall()]}

@app.route("/employee/<id>", methods=['GET'])
def getEmployee(id):
    id = int(id)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees')
    cursor = connection.cursor()
    cursor.execute("SELECT * FROM employees WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    employee = {}
    found = False
    for row in cursor.fetchall():
        found = True
        employee = {
            "birth_date": row[fields["birth_date"]],
            "first_name": row[fields["first_name"]],
            "last_name": row[fields["last_name"]],
            "gender": row[fields["gender"]],
            "hire_date": row[fields["hire_date"]]
        }
    if not found:
        abort(404)
    cursor.execute("SELECT * FROM salaries WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    salaries = []
    for row in cursor.fetchall():
        salary = {
            "salary": row[fields["salary"]],
            "from_date": row[fields["from_date"]],
            "to_date": row[fields["to_date"]]
        }
        salaries.append(salary)
    employee["salaries"] = salaries
    cursor.execute("SELECT * FROM titles WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    titles = []
    for row in cursor.fetchall():
        title = {
            "title": row[fields["title"]],
            "from_date": row[fields["from_date"]],
            "to_date": row[fields["to_date"]]
        }
        titles.append(title)
    employee["titles"] = titles
    return json.dumps({
        "status": "success",
        "employee": employee
    })

def isFieldValid(t, v):
    if t == "employee":
        return v in ["birdth_date", "first_name", "last_name", "hire_date"]
    else:
        return false

@app.route("/employee/<id>", methods=['PATCH'])
def setEmployee(id):
    id = int(id)
    content = request.json
    print(content)
    setList = ""
    data = []
    for k, v in content.iteritems():
        if not isFieldValid("employee", k):
            continue
        if setList != "":
            setList = setList + ", "
        setList = setList + k + "=%s"
        data.append(v)
    data.append(id)
    print(setList)
    print(data)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees', client_flags=[ClientFlag.FOUND_ROWS])
    cursor = connection.cursor()
    cursor.execute("UPDATE employees SET {} WHERE emp_no = %s".format(setList), data)
    connection.commit()
    if cursor.rowcount < 1:
        abort(404)
    return json.dumps({
        "status": "success"
    })

@app.route("/salary", methods=['PUT'])
def putSalary():
    content = request.json
    print(content)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees', client_flags=[ClientFlag.FOUND_ROWS])
    cursor = connection.cursor()
    data = [content["emp_no"], content["salary"], content["from_date"], content["to_date"]]
    cursor.execute("INSERT INTO salaries (emp_no, salary, from_date, to_date) VALUES (%s, %s, %s, %s)", data)
    connection.commit()
    return json.dumps({
        "status": "success"
    })
