GitHub mengunci repositori untuk kata "retard" dalam deskripsi

Perlu dicatat bahwa repositori diblokir selama 3 hari sementara pemilik mencoba menghubungi dukungan github.
Diskusi yang memanas membuka komentar tentang komitmen dan topik terkait reddit mengenai legitimasi tindakan github, khususnya, seberapa jauh perjuangan untuk hak-hak minoritas ini dapat berjalan dan apakah itu akan membahayakan pengembangan proyek sumber terbuka menggunakan github. Banyak yang mencatat bahwa kata git itu sendiri (orang yang bodoh, tidak kompeten, dan menyebalkan) kasar dan menuduh github memiliki standar ganda. Diusulkan bahwa di balik perlindungan melawan diskriminasi, proyek-proyek yang tidak disetujui dapat diblokir.

Sepertinya saya bahwa ini tidak banyak dibahas di RuNet (saya bisa saja salah). Sangat menarik untuk mengetahui apa yang dipikirkan oleh para pengguna gigtime, terutama mereka yang menggunakan github.
