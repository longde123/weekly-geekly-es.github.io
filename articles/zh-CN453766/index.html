软件无线电-它如何工作？ 第6部分

哈Ha

在上一部分中，我们研究了使用GNU Radio发送简单信号的可能性。 现在，我们将进一步探讨如何传达更复杂的内容。 让我们从业余无线电信号WSPR开始，然后创建一个可运行的软件QAM调制解调器。



与以前的情况一样，我们将无需编写任何代码即可完成此程序，该程序也将是跨平台的，并且将能够在OSX / Linux和Windows下运行。 我还将展示如何在不使用任何硬件的情况下使用GNU Radio调试调制解调器。

继续下割。

对于尚未使用GNU Radio的用户，建议阅读第4部分和第5部分，其中描述了使用该程序的原理。

WSPR


让我们从一个简单的WSPR开始 -这种通信是专门为弱信号的传播测试而设计的，即 我们需要的是LimeSDR等设备的功率电平不超过100mW。 WSPR信号在非常窄的频带内以非常低的速度（每条消息2分钟，大约30个字节）传输，这使得它甚至可以在噪声水平以下被接收。 使用GNU Radio发送此类信号。

首先，您需要记录信号。 为此，请使用WSJT程序，设置所有必需的参数（电源，呼号，位置等）。 在设置中将虚拟音频电缆指定为输出音频设备，并以WAV记录信号。 需要在任何编辑器中修剪边缘的暂停（例如，酷编辑），因此，我们应该获得一个持续时间约为2分钟的文件。

现在，在GNU Radio Companion中创建一个图形。



该方法并没有声称效率最高，但是非常简单明了。 WSPR信号最初为1500 Hz，记录的wav文件的采样频率为22050 s / s。 首先，我们对信号进行57/5次重新采样，以使采样频率达到所需的250.000 s / s。 然后我们将频率上移10KHz（有用信号将在11.5KHz的频率处），将信号转换为接收器所需的复数形式，并切除多余的滤波器，剩下11-12KHz的频率。

WSPR信号是有时间限制的，每隔一分钟（0：00、0：02等）传输一次。 我通过肉眼手动启动了到GNU Radio的传输，确定了以小时为单位的间隔，任何想要的人都可以在Python中添加脚本来自动开始传输。

我们正在等待合适的时间，打开发射器，接收器并检查结果。



如果您愿意，还可以根据输入数据（呼叫符号，位置，发送功率）将WSPR文件自动生成添加到程序中，可以在github上获取用于Python的WSPR生成示例。

有趣的是，尽管信号仍在解码，但在432 MHz处的频率漂移已经非常明显。 但是在1.3 GHz的频率下，漂移变得如此之大，以至于WSPR不再被接受-SDR需要一个具有更稳定信号的外部参考发生器（或至少在传输过程中进行软件频率校正，尽管这样做不太方便）。

如果SDR允许您以低频传输，则可以尝试在HF频带上传输。 因此，使用HackRF，可以从室内天线以14 MHz的频率在1000 km处发射1000 km的信号，这被认为是一个很好的结果。 尽管高频（433 MHz和1.3 GHz）对于实验来说可能更加有趣，但是信号仅以直接的视线传输，因此对于此类实验，您需要在接收方再有一个参与者。 这样的测试的第二个优点是，只有延迟不传输到wspr中的HF，但是对高频的掌握却少得多。 因此，使用LimeSDR或USRP之类的板卡，您可以进行非常有趣的超高频接收和传输实验。

QAM调制解调器


让我们走得更远。 WSPR是一种相当简单的格式，让我们做一些更有趣的事情-一个成熟的（差不多）调制解调器。 使用正交幅度调制时，信号的幅度和相位都会同时变化，从而可以以更高的速度传输数据（但占用的带宽也更大）。

考虑第一部分-发射器。



如您所见，我们从data.txt文件中读取数据，然后以25KHz的采样频率将数据发送到数据包编码器，该数据包编码器将流转换为4位代码。 该流进入正交调制器，然后将采样频率增加到250KHz的发射机频率，信号向上移动80KHz（许多接收机在零频率处有一个峰值，这会产生干扰）。 星座图矩形组件设置调制参数-字符数以及相位和幅度偏移。

第一部分准备就绪。 我们开始“传输”并看到我们的信号。



我们甚至不需要任何设备就可以测试发射器-为此，有一个特殊的Channel Model模块-通信通道模型。 您可以在此处设置噪声，频移等。

这就是我们的信号在传输前后的样子。 顺便说一下，正如您所看到的，低于120 dB的发射机的所有“谐波”都远远低于噪声水平。



现在接待处。 实际上，同一件事，只是相反的顺序。



另外，您可以在最后一个UDP Sink块处停止。 目前尚不清楚原因，但是在GNU Radio中没有用于查看文本数据的组件。 因此，我们仅通过UDP将数据发送到任何本地端口（我选择999）。

为了接收，我们用Python编写了一个简单的程序。 

import socket UDP_IP = "127.0.0.1" UDP_PORT = 999 sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) # UDP sock.bind((UDP_IP, UDP_PORT)) sock.settimeout(1.0) while True: try: data, addr = sock.recvfrom(64) # buffer size is 64 bytes print("Msg:", data) except socket.timeout: pass

结果：运行脚本，运行GNU Radio，并在控制台中查看收到的消息。



如您所见，一切正常，并且您既没有接收器也没有天线:)
