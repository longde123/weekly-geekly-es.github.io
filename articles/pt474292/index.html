Instalação Java


O JavaFX, é claro, requer que você tenha o JDK instalado. A obtenção das dependências necessárias varia para diferentes versões do Java.

O JavaFX foi introduzido como parte do release do Java 8. No entanto, mais tarde foi removido do JDK e
portado para um módulo separado no Java 11.

Isso significa que, se você estiver direcionando o Java 8-10, já terá todas as dependências necessárias como parte do seu JDK. Viva! Se você estiver usando uma versão mais recente do Java - ou seja, 11+, precisará obter as dependências separadamente.

Maven


Obter e gerenciar dependências manualmente é bastante inconveniente e, em um aplicativo real, você raramente faria isso. É muito melhor usar um sistema de gerenciamento de dependências como Maven ou Gradle. Dessa forma, você pode simplesmente declarar quais dependências você está usando e quais versões, e o sistema cuidará do resto.

Arquétipo Maven


Obviamente, você pode configurar seu projeto Maven manualmente, do zero. No entanto, você pode preferir uma maneira mais conveniente de criar a estrutura e o conteúdo do projeto base para você usar o Maven.

O Maven tem o conceito de arquétipos, o que significa essencialmente que você pode criar diferentes tipos de projetos a partir de um modelo. Existem inúmeros arquétipos para vários tipos de projetos e, felizmente, existem alguns para o JavaFX. O arquétipo que você pode escolher depende de qual versão do Java você está usando.

Você pode ler mais sobre os arquétipos do Maven na seguinte postagem: Tutorial dos arquétipos do Maven .

Arquétipo Java 8


Você pode usar com.zenjava: javafx-basic-archetype ou pode encontrar outros arquétipos por conta própria, se esse não for o seu caso.

Você pode gerar facilmente um projeto a partir da linha de comando usando o Maven usando o arquétipo acima: 

mvn archetype:generate -DarchetypeGroupId=com.zenjava -DarchetypeArtifactId=javafx-basic-archetype

Como alternativa, você pode criar um novo projeto Maven a partir de um arquétipo diretamente no seu IDE.

Arquétipo Java 11


Para Java 11, você pode usar org.openjfx: javafx-archetype-simple .

Para criar um projeto, basta executar: 

 mvn archetype:generate -DarchetypeGroupId=org.openjfx -DarchetypeArtifactId=javafx-archetype-simple

Ajuste manual do Maven


Se você quiser mais controle, poderá configurar manualmente seu projeto Maven sem gerá-lo a partir do arquétipo.

Primeiro você precisa de dois componentes. O primeiro é o Maven Plugin para Java FX .

Basta adicionar o seguinte ao seu pom.xml: 

 <build> <plugins> <plugin> <groupId>org.openjfx</groupId> <artifactId>javafx-maven-plugin</artifactId> <version>0.0.3</version> <configuration> <mainClass>com.example.App</mainClass> </configuration> </plugin> </plugins> </build>

Por favor note que 
 <mainclass>
deve apontar para sua classe principal, que possui um método principal e estende javafx.application.Application. Falaremos sobre isso no próximo artigo da série.

A segunda parte adiciona uma dependência para os controles JavaFX : 

 <dependency> <groupId>org.openjfx</groupId> <artifactId>javafx-controls</artifactId> <version>11.0.2</version> </dependency>

Gradle


Atualmente, Gradle não suporta a criação de projetos diretamente de arquétipos .

Você pode usar o plug-in não oficial do Gradle Archetype e os arquétipos do Maven mencionados acima.
Como alternativa, você pode criar seu projeto usando o Maven a partir de um arquétipo e convertê-lo em um projeto Gradle usando o seguinte comando no diretório que contém o seu pom.xml: 

 gradle init


Gradle manual


Como no manual do Maven, você precisa adicionar o plug-in JavaFX: 

 plugins { id 'application' id 'org.openjfx.javafxplugin' version '0.0.8' }

E a dependência para os controles: 

 javafx { version = "11.0.2" modules = [ 'javafx.controls' ] }

JavaFX SDK


Há outra possibilidade de usar o JavaFX localmente. Você pode baixar o JavaFX SDK. Ele contém todas as bibliotecas necessárias que você pode vincular ao projeto no seu IDE ou adicionar ao caminho de classe.

Isso pode ser útil quando você não estiver familiarizado com Gradle ou Maven e apenas para o desenvolvimento local.

Ao distribuir seu aplicativo, isso se torna inconveniente, pois você deve certificar-se de incluir todas as dependências necessárias.

Com essa opção, você pode gerar um projeto sem o Maven / Gradle no seu IDE, que contém todos os arquivos necessários. No IntelliJ IDEA, você pode simplesmente acessar: 

 File → New → Project → JavaFX
