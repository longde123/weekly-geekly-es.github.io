翻译：为什么权力下放很重要

以下文章中的Chris Dixon（冒险老人，换句话说）报告了这张照片：


  1. 开放协议构成了Internet的基础。 但是它们的主要品质是开放性，促使有兴趣的人在不回头的情况下使用它们（“他们不会淹没普通的船”）。 因此网络已经发展壮大。
  2. 通过开放协议，开始建立满足用户需求的业务。 这催生了集中式解决方案（Google / Facebook / Twitter等），该解决方案对用户有所帮助，但开始减缓技术增长并引起了公开辩论。 规模负担使市场领导者放慢了脚步。
  3. 由于Internet存储了“以人类思想的代码铸造”（作者的用语）的空间，而且空间是无限的，因此有可能以软件的形式无休止地重建满足人们需求的选项。 该软件层易于适应情况并几乎立即传播到全世界。
  4. 加密网络保留了开放性的吸引力（1）和“软件雾化器”的功能（3），因此，集中化的IT公司（2）将发生巨大变化。

互联网的第一个和第二个时代


在Internet的第一个时代-从1980年代到2000年代初期-服务以整个Internet社区拥有的开放协议运行。 人们和组织可以完全信任游戏规则不会改变，从而从容地巩固自己在网络上的地位。 那时，主要的网络巨头诞生了，例如Yahoo，Google，Amazon，Facebook，LindedIn和YouTube。 同时，诸如AOL之类的集中式平台开始失地。


在2000年代中期，第二个互联网时代已经开始，一直持续到今天。 开放协议的技术能力跟不上商业高科技公司（尤其是Google，Apple，Facebook和Amazon（GAFA））的软件和服务的快速发展。 随着大量网络流量开始流向移动应用程序，智能手机市场的革命只为火上浇油。 结果，用户从开放服务切换到更复杂和集中的服务。 甚至用户也可以通过GAFA软件和服务过滤器访问开放协议（网络）。


好消息是，数十亿人现在可以免费使用许多出色的数字解决方案。 坏消息是，对于初创企业，内容制造商和其他市场参与者而言，扩大其互联网影响力变得更加困难，因为集中式平台规则的突然改变充满了利润和受众的损失。 反过来，这抑制了发展和创新，剥夺了互联网的活力和多样性。 集中化还产生了新的社会问题，围绕着这些问题目前正在进行激烈的辩论：假新闻，国家机器人工厂，社交网络用户的平台化（剥夺大学言论自由），违反欧盟关于隐私的法律以及数据处理系统和评论的偏见。 多年来，围绕这些问题的紧张局势只会加剧。


“ Web 3”-互联网的第三时代


集中化问题的可能解决方案之一是大型互联网公司的状态控制。 类似的提议完全基于互联网类似于旧的通信形式（电话，广播，电视）的假设。 但是，它们之间的连接仅取决于硬件，而Internet具有最前沿的软件。 创建后完全依赖硬件的网络实际上是无法重建的。 但是，如果网络的运行取决于软件，那么市场力量和企业创新将影响其发展。


互联网是理想的软件网络。 它具有一个相当简单的核心 ，围绕该核心可以组装数十亿台计算机，并且始终可以对计算机进行重新编程。 实际上，软件是数字包装中纯人类的思想，它为设计开辟了无限的范围。 连接到网络的计算机几乎可以运行任何用户软件。 如果可以想像，可以使用正确的方法在Internet上快速传播。 基于这些杠杆和技术创新，互联网的整个体系结构都基于此。


互联网的发展仍处于起步阶段：在接下来的几十年中，中央互联网服务将被重建多次。 这些变化背后的推动力将是加密经济网络，它是比特币发起并由以太坊接管的想法的更普遍应用。 加密网络结合了前两个时代的优势：它们位于Internet社区的手中，属于去中心化，未来的功能将取代最复杂的集中式服务。


为什么要分散？


权力下放常常被误解。 例如，一些人认为，加密货币的拥护者之所以提倡权力下放，完全是因为他们的自由主义者的观点，或者是因为他们在政府的审查制度中挣扎。 但是，权力下放并不重要。


集中式平台有什么问题？ 他们有可预测的生命周期。 在旅程的开始，他们正在努力争取用户基础并争取第三方（开发人员，其他公司，媒体）的支持。 这就增加了服务的价值，并且从定义上讲，该平台是具有多方面网络效应的系统。



平台生命周期的Sigmoid及其用户


随着平台适应性的提高，它对用户和第三方的影响力越来越大。 在曲线的顶部，平台和用户之间的动态变成了对立游戏。 之后，持续增长的最简单方法是收集用户数据并与这些第三方竞争，试图夺回他们的利润和用户。 故事中的示例：Microsoft和Netscape，Google和Yelp，Facebook和Zynga，Twitter及其非官方客户。 借助iOS和Android之类的操作系统，情况会更好，但即使并非没有罪过：它们会对应用程序收取30％的不菲税，出于牵强的原因拒绝访问商店，并根据需要替换第三方应用程序的功能。


对于独立开发商和市场参与者而言，从协作到竞争的这种过渡似乎是一种设置。 随着时间的流逝，即使是最好的企业家，开发商和投资者也开始意识到，在集中式平台之上进行构建并不是一个好主意。 十几年来的历史直接证明了这一点。 此外，它还损害了最终用户：他们牺牲了隐私，允许其他人管理其数据，并且容易受到安全系统故障的影响。 而且在将来，集中化的问题只会加深。


加密网络简介


加密网络是通过Internet形成的网络：


1）使用诸如区块链之类的机制来达成共识以保证运行并更新网络状态


2）使用加密货币（硬币/代币）来激励共识方（矿工/验证者）和其他网络参与者。


一些加密网络（例如以太坊）是用于通用编程的平台，几乎适用于任何目的。 其他加密网络的目标狭窄，相同的比特币用作存储手段的存储，Golem用于执行计算，Filecoin用于分散的文件存储。


早期的Internet协议是由工作组和NGO根据技术标准创建的，其受欢迎程度取决于满足所有Internet用户的要求。 在Web开发的早期阶段，该系统运行良好，但是自90年代初以来，实际上还没有新的流行协议。 加密网络借助令牌解决了这个问题，令牌为开发人员，网络用户和支持其工作的所有人提供了足够的经济激励。 在技​​术设计方面，这样的网络也更加灵活和多样化。 例如，他们可以维持自己的状态，并同时对其进行操作，这在以前是不可能的。


加密货币具有多种防御机制，可确保它们在增长时保持中立，并防止它们下滑到欺骗性的集中式平台。 首先，加密网络与其用户之间的所有通信只能在开源中进行。 其次，加密网络基于“ 语音 ”和“ 出口 ”机制。 社区根据链（即通过协议本身）和链的原理（通过围绕协议建立的社会机制）向参与者发出声音（语音）。 参与者可以通过出售硬币或在特殊情况下通过创建协议的分支来退出网络。


简而言之，加密网络的参与者受到共同的激励，即这种网络的发展和令牌的价值。 由于这种结构，尽管所有怀疑论者都做出了预测，但比特币仍然蓬勃发展，尽管以太坊等成功竞争对手的出现也是如此。


迄今为止，密码网络中仍然存在足够的缺陷，可以防止它们违反集中式网络给出的现状。 现在的主要问题是性能和可伸缩性。 在接下来的几年中，将解决这些问题，并设计加密空间的基础设施。 然后，基于完成的基础架构，将创建应用程序。


为什么分权制胜


列出加密货币的优势是一回事，宣称加密货币将获胜则是另一回事。 让我们来看一下这些观点，这些观点描绘了对未来的乐观看法。


Web服务和软件由开发人员制作。 世界各地有数百万经验丰富的开发人员。 在这些公司中，只有一小部分在大型IT公司工作，而在那些从事新产品开发的公司则更少。 历史上大多数最重要的软件开发项目都是通过初创公司或独立开发人员社区启动的。


“无论您是谁，大多数聪明的人都为别人工作”
比尔·乔伊

分散式网络将引领互联网的第三个时代，其原因与第一个时代相同：它们将赢得企业家和开发商的心。


一个生动的例子是在零黎明之初，Wikipedia与像Encarta这样的集中竞争者的竞争。 如果我们在上个十年的初期比较这两种服务，那么Encarta无疑是赢家：主题种类繁多，信息更准确，并且总体上来说，产品更好。 但是，维基百科的发展要快得多，这要归功于志愿者的活跃社区，他们被权力下放和网站内容的开放访问所吸引。 到2005年，维基百科已经成为网络上最受欢迎的信息资源。 2009年，Encarta项目关闭。


教训是：比较集中式系统和分散式系统时，必须将它们视为动态过程，而不是静态成品。 在启动时，集中式系统通常会更具吸引力，但它们的进一步开发完全取决于赞助公司的责任。 分散式系统通常会以原始方式进入市场，但如果采用正确的策略，则过程中每个新参与者的影响力都会成倍增长。


加密网络具有几个不断增长的周期：主协议的开发人员，其他加密网络的开发人员，第三方应用程序的开发人员以及监视网络的服务提供商。 所有这些过程都受到令牌的刺激，这正是由于加密货币以反应性的速度发展的原因，正如我们在比特币和以太坊中已经看到的那样（然而，由于挖掘比特币，这并不总是会带来良好的后果，同样会浪费能源）。


哪个过程将引领新时代，集中化还是分散化？ 最后，一切都取决于产品的质量，而产品的质量又只有一个问题：谁会拥有更多的专业开发人员和企业家？ GAFA的优势在于：巨额资金，庞大的用户群和功能完善的基础架构。 反过来，对于开发商和企业家而言，加密网络的潜力可能更具吸引力。 如果他们能够取胜，那么在资源上超越GAFA并不困难，这最终意味着闪电般的发展和更好的产品。


如果您问'89年代的人们他们的生活缺乏什么，他们几乎不会说：“我们需要一个分散的信息节点网络，这些节点通过超文本相互连接”。
农夫与农夫

在启动时，集中式平台通常拥有出色的应用程序：Facebook有足够的社交“芯片”，而iPhone有自己的关键应用程序。 但是，去中心化平台通常是原始启动的，没有任何明显的方向。 因此，在找到市场位置之前，他们需要经历两个阶段：


1）建立平台与开发者/企业家之间的和谐关系，在此基础上，整个生态系统和


2）在平台/生态系统与最终用户之间建立和谐。


这个两步过程通常使得即使是经验丰富的技术人员也很难分辨去中心化平台的全部潜力。


互联网的下一个时代


分散式网络不是万灵药，也无法挽救Internet的所有麻烦。 但是与其他系统相比，它们绝对是赢家。


比较Twitter上的垃圾邮件问题和通过电子邮件比较垃圾邮件的问题。 Twitter所有者已阻止独立开发人员访问其网络。 因此，他本人必须处理所有垃圾邮件。 同时，数百家公司通过电子邮件解决垃圾邮件问题，注入了数十亿美元的风险投资和私人资金。 是的，垃圾邮件尚未完全消除，但是现在情况显然要好得多了，这都是因为电子邮件协议是去中心化的，这使得独立的利益集团可以基于此创建业务，而不必担心游戏规则会突然改变。


或处理网络管理问题。 信息如何分类和过滤？ 谁将被晋升，谁将被禁止？ 对于大型平台，此问题和其他重要问题由单独的一组员工解决，这些员工对任何人都不承担任何责任。 在加密网络中，这些问题由整个社区解决，所有决策机制都是透明和开放的。 我们从现实世界中知道，民主不是完美的，但是没有更好的选择。


集中式平台一直统治着市场，以至于人们只是忘了还有其他选择。 如此强大的替代方案恰好是加密网络：它们将使用户控制网络的开发，并为独立开发人员，公司和内容制作者创造透明的条件。 在互联网的第一个时代，分散已经证明了其价值。 我们希望她下次再来。
