Smart IdReader SDK: reconocimiento integrado en proyectos en Python y PHP

Nosotros, Smart Engines , continuamos con la serie de artículos sobre cómo integrar nuestras tecnologías de reconocimiento ( pasaportes , tarjetas bancarias y otras) en sus aplicaciones. Anteriormente escribimos sobre incrustación en iOS y Android , mostramos cómo incrustar el reconocimiento en el bot de Telegram , y hoy hablaremos sobre cómo trabajar con las interfaces Python y PHP de la biblioteca de reconocimiento Smart IDReader para usar en un servidor interno.


Por cierto, la lista de lenguajes de programación que admitimos, además de los discutidos aquí, incluye C #, Objective-C, Swift y Java. Como antes, admitimos todos los sistemas operativos y arquitecturas populares y muchos impopulares, y nuestras aplicaciones de demostración gratuitas están disponibles para descargar desde la App Store y Google Play .


Por tradición, la versión demo del Smart IDReader SDK para Python y PHP junto con ejemplos está disponible en Github y está disponible aquí.


Poner una envoltura


Dado que la versión del complemento debe coincidir con la versión del intérprete, no suministramos un módulo listo para usar, sino un ensamblador automático que le permite recopilar e implementar el contenedor de biblioteca de reconocimiento Smart IDReader necesario en su máquina (se admiten Python 2/3 y PHP 5/7). Para hacer esto, debe ejecutar el script de ensamblaje apropiado, indicando la ruta a nuestra biblioteca libsmartidEngine.so y la versión del intérprete para la que necesita construir el módulo. Por ejemplo, para Python, se ve así: 


bash build_python.sh ../../bin 3

PHP funciona de manera similar, solo el segundo argumento es la ruta a la configuración de php que usa: 


 bash build_php.sh ../../bin /usr/bin/php56-config

Importante: después del ensamblaje, el módulo resultante se referirá a libSmartidEngine.so en la ruta absoluta, ¡así que primero instale la biblioteca en un directorio conveniente para usted!


El ensamblador automático desempaquetará el SWIG suministrado en el ensamblaje, generará un módulo y lo verificará ejecutando la prueba.


El contenedor de la biblioteca incluye dos archivos: un módulo escrito en Python / PHP y una extensión para el intérprete (_pySmartIdEngine.so para Python y phpSmartidEngine.so). Para integrar el reconocimiento en el proyecto, debe importar el módulo y conectar la extensión.
En el caso de Python, para esto es suficiente escribir explícitamente la ruta al módulo y la extensión: 


 sys.path.append(os.path.join(sys.path[0], '../../bin/')) sys.path.append(os.path.join(sys.path[0], '../../bindings/')) import pySmartIdEngine

En PHP, un módulo se importa de la misma manera: 


 require(“phpSmartIdEngine.php");

pero con la extensión es un poco más complicado: el enlace dinámico no funciona en algunas versiones, por lo que para trabajar constantemente con la extensión debe colocarlo en la carpeta donde se encuentran todas las extensiones para su versión (por ejemplo / usr / lib / php56 / modules) y conectarse, escritura en php.ini predeterminado 


 extension=phpSmartisEngine.so

En el ejemplo con GitHub, solo le decimos al intérprete que use nuestro php.ini con la extensión ya registrada 


 php56 -c php.ini smartid_sample.php

Entonces, descubrimos la conexión, ¡no dude en estudiar un ejemplo!


Aprendiendo la interfaz


El ejemplo comienza con tres argumentos: la ruta a la imagen a reconocer, la ruta al archivo con la configuración del motor y el tipo de documento a reconocer (las pruebas ejecutan el ejemplo para rus.passport.national (pasaporte de un ciudadano de la Federación de Rusia)) 


 python smartid_sample.py ../../testdata/passport_rf_2.jpg ../../data-zip/bundle_mock_smart_idreader.zip rus.passport.national

Nos detendremos en los métodos para cargar la imagen con más detalle: puede ser la ruta al archivo, una línea en Base64 y un búfer en la memoria (ambos datos en los formatos RGB \ YUV NV21 y el archivo jpeg). Los buffers pueden ser externos, o pueden extraerse de un objeto de la clase Image (después de crearlo especificando la ruta al archivo o usando el mismo buffer)


Importante: solo se puede acceder a la imagen en formato RGB o YUV en el módulo Python utilizando un objeto bytearray (cree un objeto del tamaño requerido, páselo al método CopyToBuffer, vea un ejemplo), ¡en PHP solo puede trabajar con una cadena de formato Base64!

Conclusión


Consideramos trabajar con SmartIdEngine SDK en Python y PHP, en los ejemplos reflejamos todas las dificultades que puede encontrar en el proceso de implementación del módulo en una máquina en funcionamiento.
