在圣彼得堡测试NB-IoT网络的可用性（2019年8月）



不久前，移动运营商开始尝试新的无线接入技术-NB-IoT。 让我提醒您，以防万一该技术是为需要不时将少量数据传输到远程服务器的低功耗设备而设计的。 我决定查看2019年8月圣彼得堡NB-IoT的发展情况。



该计划很简单：检查三个运营商的NB-IoT网络的可用性，以及在圣彼得堡的几个点通过NB-IoT将数据传输到远程服务器的能力。


为了进行实验，我使用了Neoway N21 NB-IoT模块的调试板和三个运营商的特殊SIM卡：MTS，Megafon和Beeline。 测试在室内进行，使用增益为3 dBi的天线。 N21可以包含各个频率范围（AT + NVSETBAND命令），因此我同时决定找出当前正在使用的频段。

1.圣彼得堡的NB-IoT MTS网络：


  • 以商业运作模式运作；
  • 使用800 MHz（B20，主要）和1800 MHz（B3）频段；
  • 有一个NB-IoT覆盖图（通过测试结果确认）， https://spb.mts.ru/personal/podderzhka/zoni-obsluzhivaniya/nasha-set ;
  • TCP / UDP数据包的传输相对稳定；
  • PSM模式有效。

2.圣彼得堡的NB-IoT扩音器网络：


  • 在测试模式下工作；
  • 使用900 MHz（B8，主要）和800 MHz（B20）的频率范围；
  • 没有覆盖图，但是NB-IoT网络在圣彼得堡的许多地区都可以使用；
  • 有时在注册过程中很奇怪，而在传输TCP / UDP数据包时却运行不稳定（但这也许是正常的，因为到目前为止还没有人做出任何承诺）；
  • PSM模式不起作用（至少我没有成功）。

3.圣彼得堡的NB-IoT Beeline网络：


  • 在测试模式下工作；
  • 使用1800 MHz（B3）的频率范围；
  • 没有覆盖图；在圣彼得堡很少有地图；
  • TCP / UDP数据包的传输相对稳定；
  • PSM模式有效。

我没有NB-IoT Tele2（Rostelecom）SIM卡，我在哪里可以得到它，是否有覆盖范围，我不知道。


据我所知，目前只有MTS宣布NB-IoT网络的商业运营并提供有关覆盖区域的信息。 同时，在MTS覆盖图下，有以下注释：“为支持NB-IoT技术的设备在频率范围800、900、1800中指示了NB-IoT网络服务区域。”
有趣的是，在圣彼得堡发行的商用NB-IoT MTS SIM卡还可以在莫斯科，彼尔姆，叶卡捷琳堡，下诺夫哥罗德，罗斯托夫使用。 在莫斯科发行的测试SIM卡NB-IoT Beeline在圣彼得堡工作。 圣彼得堡发行的测试SIM卡NB-IoT扩音器拒绝在叶卡捷琳堡测试网络NB-IoT的覆盖范围内工作（本地SIM卡有效）...


我也很感兴趣的一个问题是，NB-IoT SIM卡将如何在可同时在NB-IoT网络和GPRS网络中工作的双模设备中工作。 这样的设备已经在市场上出现了（例如， N25模块或MTR-MNG2路由器 ），因此问题并非闲置。 无论如何，当我尝试在GPRS设备中使用它们时，我的SIM卡NB-IoT MTS和Beeline已经“崩溃”了。 更准确地说，操作员的设置丢失了，此后，SIM卡拒绝在NB-IoT网络上注册。 没有在扩音器SIM卡上进行过这样的实验。


PS此处提供的数据是2019年8月在圣彼得堡进行测试得出的结果，不能声称其无可挑剔的准确性和绝对可靠性。 此外，情况可能会很快改变。 一切顺利！
