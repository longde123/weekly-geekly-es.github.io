Sites dynamiques sans serveur sur les pages Github (pour ceux qui ne savent pas, sans serveur utilisent des serveurs API tiers)



Beaucoup ont sûrement entendu parler des pages github et que vous pouvez y héberger des sites statiques (générés et téléchargés là-bas). Mais je voudrais qu'il soit dynamique dans ce cas, c'est-à-dire qu'il serait possible de remplir le code une fois et de le remplir comme sur un vrai TsMSke, de plus, il y avait une autorisation, des données chargées dynamiquement et tout ça. Non? Je voulais donc ça. Sous la coupe, je vais montrer ce que j'ai fait.

Pour commencer, comme promis, je vais vous montrer ce qui s'est passé, puis je vais vous dire les détails (garder un œil sur la modification des onglets et des adresses de page).


Alors qu'est-ce qui se passe ici? J'ai récemment écrit sur l'écriture de cms sans tête, nommés Prisma CMS .

Mais comme je l'ai écrit, l'idée initiale était que, bien que le serveur et le front soient amicaux, ils doivent encore pouvoir vivre leur propre vie et sont des unités indépendantes. Et si le serveur suit la tendance actuelle des headless-cms / api-first-cms, alors l'avant suit un autre concept - sans serveur. Autrement dit, si nous lançons un front séparé, nous spécifions simplement le point de terminaison du serveur API (ce n'est pas nécessaire sur le même domaine), et les données seront extraites par l'API (et l'autorisation fonctionnera).

Mais ce n'est pas tout. Sur le site, j'ai écrit un article sur le fait que sur prisma-cms.com tout le monde peut créer un site Web sur un sous-domaine qui utilisera une seule base de données, mais vous pouvez concevoir individuellement votre propre site Web pour vous-même. Donc, si vous copiez le référentiel prisma-cms.github.io pour vous - même , vous obtiendrez votre propre site, comme indiqué dans la vidéo.

Mais vous pouvez même augmenter votre propre serveur et site. Voici le code source du site: https://github.com/prisma-cms/prisma-cms.com . Déployez sur votre serveur et exécutez. Si tout fonctionne, vous spécifiez ici votre API de point de terminaison et la collectez avec la commande PUBLIC_URL =. / Yarn build
Tout, le site est prêt. Maintenant, en suivant les instructions officielles, remplissez le contenu du dossier de construction sur le github et vous avez votre propre site Web avec votre API (vous pouvez utiliser une API pour travailler avec un tas de sites statiques distincts, chacun pouvant être avec sa propre conception).

Et ce n'est pas pour rien que j'ai utilisé une image intitulée Social coding. Comme vous pouvez le voir dans la vidéo, tous les modèles créés sur l'un des sites à l'aide d'une seule base de données peuvent être utilisés dans la conception d'autres sites. C'est-à-dire qu'à l'avenir, s'il y a suffisamment d'intérêt, il sera possible de créer un marché unique de modèles et de parties de modèles pour créer rapidement des sites à partir de composants prêts à l'emploi.

Bon codage!
