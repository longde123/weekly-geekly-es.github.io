Menggunakan Windows Server tanpa explorer dari sudut pandang pengguna Windows biasa

Salam untuk semua "kelangsungan hidup" mereka di bawah Windows Server tanpa explorer


Hari ini saya akan menguji program biasa untuk Windows yang tidak biasa.

Saya akan mulai dari awal


Ketika Anda menyalakan komputer, boot Windows standar muncul, tetapi setelah boot tidak membuka desktop, tetapi baris perintah dan tidak lebih.



Unduh file melalui Internet dari baris perintah


Karena saya tidak tahu metode lain untuk mengunduh file setelah instalasi bersih ke pengguna normal (kecuali untuk menghubungkan hard drive kedua dengan sistem yang berbeda), saya menggunakan utilitas bitsadmin.exe bawaan

Pertama, buat folder tempat kami akan mengunggah file 

md c:\download

Kemudian saya mengunggah file yang saya butuhkan ke situs saya.

Misalnya, instal analog dari explorer standar - Explorer ++ 

 bitsadmin.exe /transfer "Download" https://_/files/Explorer++.exe C:\download\Explorer++.exe

Proses pengunduhan:



Selesai mengunduh:



Mengingat waktu MS-DOS, kami menulis Explorer++.exe di baris perintah.
Explorer ++ yang diunduh oleh saya terbuka



Saya juga mencoba untuk meluncurkan konduktor yang biasa dengan menariknya keluar dari sistem saya, tetapi ia menolak untuk bekerja, meskipun saya sangat memintanya. Melalui Powershell, Anda dapat menginstal tidak hanya penjelajah standar, tetapi juga MMC, Eventvwr, PerfMon, Resmon dan Powershell ISE, tetapi ini akan menghabiskan sumber daya dan tidak akan sesuai dengan subjek publikasi.



Tes perangkat lunak


Pertama-tama, berhentilah menggunakan baris perintah untuk mengunduh file, ganti dengan Internet Download Manager.

Sebagai pengelola file saya menggunakan explorer ++, Anda dapat mengunduh yang lain, hanya sampai di tangan saya.

Saya menginstal Archiver 7zip, itu juga bekerja tanpa masalah.
Saya tidak menginstal peramban Firefox, jadi saya memilih Chromium Portable. Anda tidak akan dapat mengunduh file melalui browser, jadi kami akan terus menggunakan Internet Download Manager.
Pemutar media VLC mulai normal
qBittorrent juga tidak menyebabkan masalah saat startup
Thunderbird memberikan kesalahan yang sama dengan Firefox
Saya akan berbicara tentang perangkat lunak perkantoran sedikit kemudian.

Permainan


Sayangnya, game modern melalui Steam tidak akan berfungsi. Saat startup, klien lumpuh karena kesalahan.



Perangkat lunak perkantoran


Microsoft Office tidak cocok dengan konsumsi sumber daya. Dan Libre Office tidak dimulai, yang sangat menyedihkan.

Versi web Microsoft Office berfungsi melalui peramban, sehingga Anda dapat dengan mudah mengedit dokumen, tetapi Anda tidak dapat menyimpannya di mesin lokal. Saya belum menguji Google Documents, tetapi saya pikir itu akan berhasil juga.

Terakhir


Menggunakan Windows Server tanpa explorer sebagai "desktop" tidak masuk akal. Jika sumber dayanya hampir habis, yang terbaik adalah memilih salah satu distribusi Linux yang ringan.

Dalam publikasi ini, saya menunjukkan contoh apa yang terpikir oleh saya untuk diuji, ini bukan manual pelatihan, tetapi konten hiburan. Terima kasih sudah membaca.
