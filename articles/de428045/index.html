PHP-Interna


  • [RFC] Preloading - Das Konzept von Dmitry Stogov ist in einen vollwertigen RFC eingebettet . Es wird vorgeschlagen, das Hochladen von Dateien vor der Verarbeitung der Anforderungen zu ermöglichen. Alle auf diese Weise geladenen Klassen und Funktionen sind im Rahmen nachfolgender Anforderungen dauerhaft verfügbar, ohne dass eine Einbeziehung erforderlich ist. Neben Produktivitätssteigerungen eröffnet eine solche Gelegenheit den Weg für weitere Optimierungen.
  • [RFC] Improven openssl_random_pseudo_bytes () - Die Funktion openssl_random_pseudo_bytes() möglicherweise nicht openssl_random_pseudo_bytes() eine unsichere Zufallsfolge. In diesem Fall wird vorgeschlagen, eine Ausnahme auszulösen.

      Versteckter Text
      Im Rahmen des Concurrent-PHP / Ext-Async- Projekts zur Implementierung der Unterstützung für asynchrone E / A- Vorgänge in PHP wird ein HTTP-Client implementiert, der den PSR-Standards entspricht. Dies zeigt die Möglichkeit, die Unterstützung von Concurrent-PHP / Ext-Async in vorhandene PHP-Bibliotheken zu integrieren und gleichzeitig die Abwärtskompatibilität aufrechtzuerhalten. Es ist wichtig zu beachten, dass bei der Implementierung von Coroutine mithilfe von Generatoren die asynchrone Funktion / Methode ein Generator-Objekt zurückgibt, wodurch verhindert wird, dass die Schnittstelle abwärtskompatibel ist
