Cálculo de quadrados mágicos usando uma GPU

Oi habr.

O tema dos "quadrados mágicos" é bastante interessante, porque por um lado, são conhecidas desde a antiguidade; por outro, o cálculo do "quadrado mágico" até hoje é uma tarefa computacional muito difícil. Lembre-se de que, para construir um NxN de “quadrado mágico”, é necessário inserir os números 1..N * N para que a soma de suas horizontais, verticais e diagonais seja igual ao mesmo número. Se você simplesmente escolher o número de todas as opções para organizar os números de um quadrado 4x4, obteremos 16! = 20.922.789.888.000 opções.

Considere como isso pode ser feito com mais eficiência.



Para começar, repetimos a condição do problema. Você precisa organizar os números em um quadrado para que eles não se repitam, e a soma das horizontais, verticais e diagonais era igual ao mesmo número.

É fácil provar que essa soma é sempre a mesma e é calculada pela fórmula para qualquer n:



Vamos considerar quadrados 4x4, então a soma = 34.

Denote todas as variáveis ​​por X, nosso quadrado ficará assim:



A primeira e óbvia propriedade: Se a soma do quadrado for conhecida, as rajadas extremas podem ser expressas pelos 3 restantes:

X14 = S - X11 - X12 - X13
X24 = S - X21 - X22 - X23
...
X41 = S - X11 - X21 - X31

Assim, um quadrado 4x4 na verdade se transforma em um quadrado 3x3, o que reduz o número de opções de pesquisa de 16! até 9! 57 milhões de vezes. Sabendo disso, começamos a escrever código, ver o quão complicada é a busca exaustiva por computadores modernos.

C ++ - versão multithread


Reescrever um programa usando fluxos é basicamente simples, embora um pouco complicado. Felizmente, hoje existe uma opção quase esquecida - o uso do suporte ao OpenMP (Open Multi-Processing). Essa tecnologia existe desde 1998 e permite que as diretivas do processador digam ao compilador quais partes do programa serão executadas em paralelo. O OpenMP também é suportado no Visual Studio, portanto, para transformar um programa em multiencadeado, basta adicionar uma linha ao código: 

 int squares = 0; #pragma omp parallel for reduction(+: squares) for (int x11 = 1; x11 <= MAX; x11++) { ... } printf("CNT: %d\n", squares);

A diretiva #pragma omp parallel for indica que o próximo loop for pode ser executado em paralelo, e o parâmetro adicional squares define o nome da variável, que será comum para threads paralelos (sem isso, o incremento não funcionará corretamente).

O resultado é óbvio: o tempo de execução foi reduzido de 102 para 18 .

Computando com NVIDIA CUDA


Se você não entrar em detalhes, o processo de cálculo realizado na placa de vídeo pode ser representado como vários blocos de hardware paralelos (blocos), cada um dos quais executa vários processos (threads).



Por exemplo, podemos dar um exemplo da função de adicionar 2 vetores da documentação da CUDA: 

 __global__ void add(int n, float *x, float *y) { int index = threadIdx.x; int stride = blockDim.x; for (int i = index; i < n; i += stride) y[i] = x[i] + y[i]; }

As matrizes x e y são comuns para todos os blocos, e a própria função é executada simultaneamente em vários processadores ao mesmo tempo. A chave aqui é o paralelismo - os processadores de placa de vídeo são muito mais simples que uma CPU comum, mas existem muitos deles e eles estão focados especificamente no processamento de dados numéricos.

É disso que precisamos. Temos uma matriz de números X11, X12, .., X44. Vamos começar o processo de 16 blocos, cada um deles executando 16 processos. O número do bloco corresponderá ao número X11, o número do processo ao número X12 e o próprio código calculará todos os quadrados possíveis com os X11 e X12 selecionados. É simples, mas existe uma sutileza - os dados precisam não apenas ser calculados, mas também transferidos da placa de vídeo, para isso, armazenaremos o número de quadrados encontrados no elemento zero da matriz.

O código principal é muito simples: 

 #define N 4 #define SQ_MAX 8*1024 #define BLOCK_SIZE (SQ_MAX*N*N + 1) int main(int argc,char *argv[]) { const clock_t begin_time = clock(); int *results = (int*)malloc(BLOCK_SIZE*sizeof(int)); results[0] = 0; int *gpu_out = NULL; cudaMalloc(&gpu_out, BLOCK_SIZE*sizeof(int)); cudaMemcpy(gpu_out, results, BLOCK_SIZE*sizeof(int), cudaMemcpyHostToDevice); squares<<<MAX, MAX>>>(gpu_out); cudaMemcpy(results, gpu_out, BLOCK_SIZE*sizeof(int), cudaMemcpyDeviceToHost); // Print results int squares = results[0]; for(int p=0; p<SQ_MAX && p<squares; p++) { int i = MAX*p + 1; printf("[%d %d %d %d %d %d %d %d %d %d %d %d %d %d %d %d]\n", results[i], results[i+1], results[i+2], results[i+3], results[i+4], results[i+5], results[i+6], results[i+7], results[i+8], results[i+9], results[i+10], results[i+11], results[i+12], results[i+13], results[i+14], results[i+15]); } printf ("CNT: %d\n", squares); float diff_t = float(clock() - begin_time)/CLOCKS_PER_SEC; printf("T = %.2fs\n", diff_t); cudaFree(gpu_out); free(results); return 0; }

Selecionamos o bloco de memória na placa de vídeo usando o cudaMalloc, executamos a função dos quadrados, indicando 2 parâmetros 16.16 (o número de blocos e o número de threads) correspondentes aos números 1..16 repetidos, depois copiamos os dados através do cudaMemcpy.

A própria função dos quadrados repete essencialmente o código da parte anterior, com a diferença de que o aumento do número de quadrados encontrados é feito usando atomicAdd - isso garante que a variável seja alterada corretamente durante chamadas simultâneas.
