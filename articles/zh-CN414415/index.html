6月21日召开的Tarantool会议-不仅涉及Tarantool，而且涉及内存中的计算

您是否听说过Tarantool会议或即将召开的 T + Conf会议 ？ 当然，Tarantool的每个人都知道，我们现在将向您介绍会议的所有内容。



T + Conf-会议不仅涉及Tarantool本身，而且通常涉及内存计算 ，Tarantool / Redis / Memcached，协作式多任务处理和Lua的使用 ，以开发高负载的容错服务 。

何时何地？


很快，2018年6月21日， 在Mail.ru Group办公室 （Leningradsky Prospekt 39，第79页）。

这将是一个全职工作 -报告将从10:00开始，至19:00结束。 我们确定了两个流，其中一个完全用于企业架构 ，第二个结合了与Tarantool周围的生态系统相关的性能，包括教程的格式。 因此，它们不仅会吸引有经验的开发人员 ，而且一方面会吸引初学者 ，另一方面也会吸引公司部门的建筑师 。

可以通过参考来研究时间表，以下是所有报告的简要说明-没有那么多报告 ，我们可以负担得起，不要错过任何人。

社区和教程


VShard-Tarantool中的水平缩放


Vladislav Shpileva 将讨论内部设备VShard （这是Tarantool DBMS中分片的替代实现），以及其子系统和实现，并附有使用示例。

在VShard中，重新平衡是分阶段执行的，您可以指定任意分片函数以确保关联数据的局部性，分片函数的计算结果存储在每个记录中，而无需重新计算。

Tarantool迁移到Kubernetes


与大多数数据库一样，Tarantool非常容易在单个实例中部署到Kubernetes，但是随着负载的增加，无法通过单个命令扩展服务。 作为报告的一部分， Alexander Golovko （Valarm）计划从Tarantool服务的角度讨论在Kubernetes内部转移所有微服务的过程，存在的困难以及用于解决这些困难的工具。

我们从头开始为Tarantool创建一个高负载的应用程序


教程格式的Vladimir Perepelitsa 将证明 Tarantool是一个具有巨大潜力的数据库，可用作应用服务器 ，并告诉您如何释放这种潜力。 我们还将通过创建队列服务器的示例逐步考虑Lua语言的优缺点。



在.Net项目中使用Tarantool


Evote.com的Anatoly Popov回忆说.Net不仅是Windows，而且还是Linux和OS X，并计划分享 在.Net Core 中使用Tarantool 的经验，并讨论其优缺点，问题和解决方法。 并回答一个重要的问题：是否可以将2M RPS从一台服务器压缩到.Net Core？

Tarantool复制：配置和使用


Tarantool的最新版本具有几个附加功能，可轻松在集群中配置和使用复制。 DBMS开发人员之一Georgy Kirichenko在他的报告中将考虑设备的基本原理，详细讨论状态向量的内部结构-vclock。 而且直接来自第一手-这不适合您阅读文档。

Tarantool 2.0的新功能


Cyril Yukhin （Tarantool）计划审查应该添加到主要版本中的新功能 ：支持SQL语言的方式和程度，同步复制的审查，分片，交互事务的新方法以及事务中的DDL。

在Tar​​antool上创建可用于生产的应用程序


Tarantool是一个应用程序服务器，这意味着您必须能够在其上正确编写应用程序。 来自KTS Studio的Igor Latkin 将展示如何针对不同情况正确构建布局应用程序 ，这些现有的库和模块不仅使开发过程变得容易，而且在生产中启动时也使生活变得更加轻松。

Andrey Knyazev 将讨论计费系统向MegaFon的转换，先决条件（垂直扩展的僵局，负载增长等），技术挑战是什么以及最终发生了什么。 特别是，诸如解决复杂问题之类的显而易见的课程需要时间 ，而由于某种原因，这总是被人们遗忘。

非易失性内存字节和DBMS


IBS的Andrei Nikolaenko 计划解决与响应式字节可寻址非易失性存储器有关的许多问题，最近，这已在多家供应商中出现。 还将提供其他研究人员手写和发布的测试结果 。 不仅在今天而且在不久的将来考虑已处理的TB的价格和速度的每个人都将很感兴趣。



我们如何测试基于GridGain的新Sberbank平台


帕维尔·利普斯基（Pavel Lipsky）承诺会解释为何Sberbank会转向内存数据网格（IMDG）技术，以及他们如何测试基于GridGain 的新平台 。

Tarantool作为物联网应用服务器


诺基亚公司的Vladislav Zaitsev将在物联网方面着手探讨这个话题-并告诉您 物联网为何是控制系统，Tarantool来自何处以及为什么如此，为什么不使用（几乎）数据库以及如何将其全部组合在一起。



Tarantool /乙烯基内饰，调音和监控


当然，Tarantool DBMS架构师Konstantin Osipov不禁在这次会议上发表讲话。 他将发表一份报告，在其中他将以最一致和最详细的方式，功能以及最重要的是针对该引擎的调优和性能监视机制 ，检查Vinyl体系结构（Tarantool中磁盘存储引擎的实现）。

我们将在圆桌会议结束时召开一个圆桌会议，圆桌会议的程序委员会名为“ Tarantool Roadmap ”-我们将讨论对社区重要的问题，概述前景，表达关注和愿望，并花费时间进行公开交流。

如您所见，Tarantool开发人员有话要说，这并不奇怪。 但是，还有许多报告是基于真实经验的，而不是出于玩具目的，而是在大型公司中。 他们的故事，从选择技术开始，将对那些刚开始的人以及已经学习到微妙之处并正在寻找巨大机会的人很有用。
