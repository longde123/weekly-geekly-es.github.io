Bagaimana generator frasa situs jahat diizinkan mencuri $ 4 juta

Ars Techica baru-baru ini menerbitkan materi yang menggambarkan bagaimana generator frase frase iotase.io seed, iotaseed.io, yang berbahaya (dan sekarang tidak lagi berfungsi), memungkinkan pembuatnya mencuri cryptocurrency IOTA senilai hampir $ 4 juta dari dompet pengguna.



Menurut deskripsi publikasi, situs web "menyimpan data pada setiap frase awal yang dihasilkan bersama dengan informasi tentang dompet yang terkait, memungkinkan pemilik situs web (atau mereka yang meretasnya) untuk hanya menunggu token tiba di dompet, dan kemudian tarik mereka ke akun Anda. " Dalam artikel ini, saya memutuskan untuk mempelajari sisi teknis dari penipuan ini.

Kami sedang mencari kode


Versi asli situs web iotaseed.io telah digantikan oleh pesan yang menyatakan bahwa itu dinonaktifkan. Untungnya, salinan situs disimpan oleh Wayback Machine.

Situs web mengarah ke repositori github, memperingatkan pengunjung bahwa kode terbuka untuk ditinjau, tetapi merekomendasikan pengguna untuk membuat seed di situs web itu sendiri, karena repositori mungkin berisi kode yang lebih baru dan belum diuji.

Berdasarkan informasi ini, saya berasumsi bahwa kode berbahaya hanya bertindak di situs, dan itu tidak ada dalam repositori. Pendekatan semacam itu akan memungkinkan untuk menyembunyikan fragmen kode yang digunakan untuk mencuri benih. Repositori hanya berisi kode bersih yang tidak menimbulkan pertanyaan atau kecurigaan. Ini akan menjelaskan rekomendasi untuk hanya menggunakan situs web dan bukan repositori. Jika ini benar, maka setiap upaya untuk membandingkan JavaScript di situs dengan JavaScript di repositori Github akan mengarah ke deteksi jelas dari pintu belakang di situs.

Sayangnya, repositori iotaseed.io github merujuk pada norbertvdberg / iotaseed yang sudah dihapus (akun juga telah dihapus). Wayback Machine hanya menyimpan halaman repositori utama . Menanggapi upaya untuk melihat kode atau mengunduh arsip zip-nya, WM mengatakan bahwa halaman-halaman ini tidak diarsipkan padanya. Namun demikian, melihat penghitung garpu di sudut kanan atas, kita dapat menemukan bahwa kode ini dipalsukan oleh 8 akun yang berbeda. Setelah memeriksa artikel terkait di basis pengetahuan GitHub, kita belajar bahwa menghapus repositori tidak memengaruhi pekerjaan forknya. Ini berarti bahwa salinan dari repositori ini mungkin masih ada di suatu tempat dalam luasnya layanan.



Pencarian cepat dari komit yang terlihat dalam salinan halaman yang disimpan oleh Wayback Machine memungkinkan Anda untuk mengetahui hal-hal berikut:



Sepertinya eggdroid / eggseed3 adalah salah satu fork dari kode asli iotaseed.io. Semua 26 komit di dalamnya dibuat oleh norbertvdberg, yaitu, pemilik-pengguna yang sama dari repositori asli. Sekarang kami memiliki file situs web dan JavaScript dari github yang kami miliki, saatnya untuk membandingkannya dan menemukan perbedaannya.

Menganalisis kode


Generator seed terdiri dari banyak file JavaScript berbeda yang digabungkan menjadi satu file all.js diperkecil menjadi all.mini.js. Dialah yang sebenarnya digunakan pada halaman. Oleh karena itu, saya membandingkannya dengan file js yang sama dari salinan WM yang disimpan.

Berdasarkan ini, kita dapat menyimpulkan bahwa file ./images/logo_small_bottom.png dihasilkan dengan cepat oleh server iotaseed.io.

Selama pembuatan file ini, nomor yang digunakan oleh generator nomor acak yang diperbaiki diubah dan mungkin disimpan di suatu tempat sehingga penyerang nantinya dapat menggunakannya untuk mencuri IOTA. Akibatnya, situs web tersebut menghasilkan benih yang berbeda untuk pengguna yang berbeda. Namun, situasi dengan generasi acak di sisi server juga bukan yang terbaik, karena diketahui bahwa setidaknya satu orang menerima benih di situs yang telah digunakan seseorang. Peragaan perbedaan kode tersedia di tautan ini.

Menggunakan resmi Javascript-perpustakaan dari IOTA , kita dapat menetapkan bahwa disebutkan sebelumnya Sid XZHKIPJIFZFYJJMKBVBJLQUGLLE9VUREWK9QYTITMQYPHBWWPUDSATLLUADKSEEYWXKCDHWSMBTBURCQD sesuai dengan alamat PUEBLAHRQGOTIAMJHCCXXGQPXDQJS9BDFSCDSMINAYJNSILCCISDVY99GMKAEIAICYQUXMIYTNQCJYVDX . Menurut situs web ini , dompet kosong, tetapi situs penjelajah riwayat transaksi lainnya memberikan kesalahan 404 . Ini berarti bahwa saya melakukan kesalahan ketika mendekodekan alamat, atau saya tidak mengerti cara kerja jaringan IOTA.

Kesimpulan


Pintu belakang disembunyikan dengan licik. Itu pasti ditempatkan di sana dengan niat jahat, dan bukan karena kesalahan dengan penggunaan kriptografi. Tidak jelas sampai akhir apakah itu ditambahkan oleh pemilik repositori github dan situs norbertvdberg, atau apakah akun hostingnya diretas. Betapapun, dilihat dari reaksi pemilik, yang kemudian menghapus akunnya di GitHub , Reddit dan Quora , ternyata situs itu awalnya dirancang untuk mencuri dana pengguna.

Penyerang mengambil banyak langkah untuk menyembunyikan pintu belakang. Sekilas ke dasbor pengembang di peramban tidak akan mengungkapkan apa pun yang mencurigakan. Sebagai contoh, data: tautan pada tahap pertama dari pintu belakang dimulai dengan iVBORw0KGgo yang sesuai dengan awal header PNG yang valid yang dikodekan dalam basis 64. Ini berarti bahwa url seperti itu dapat dengan mudah keliru untuk penyisipan gambar - tindakannya cukup normal untuk perpustakaan pemberitahuan js. Bagian dari kode javascript dimuat dari gambar, dan permintaannya adalah satu-satunya permintaan jaringan. Sayangnya, semua ini ternyata cukup untuk membuat banyak orang percaya bahwa semuanya sesuai dengan situs.

Penelitian yang cermat terhadap panel pengembang di browser memungkinkan Anda melihat permintaan ini.



Secara umum, kejadian ini harus diambil sebagai pengingat bahwa ketika datang ke cryptocurrency (dan terutama dalam jumlah besar!), Paranoia bisa menjadi hal yang baik. Anda seharusnya tidak pernah mengandalkan layanan online seperti generator frase benih atau dompet web, memercayai mereka jumlah yang signifikan untuk Anda. Anda hanya perlu menggunakan perangkat lunak yang telah menjalani peninjauan dan audit menyeluruh oleh komunitas.

Dalam kasus kami, iotaseed.io benar-benar memposisikan dirinya sebagai solusi open source, menunjukkan keterbukaan dan kemampuan untuk memverifikasi kode oleh siapa pun. Sepertinya ini sudah cukup untuk meyakinkan beberapa orang, tetapi tidak satupun dari mereka yang berpikir bahwa kode asli yang berjalan di situs web dapat diubah. Pemeriksaan yang cermat akan mengungkap fakta ini, memberi kita contoh lain tentang konsekuensi serius apa yang bisa ditimbulkan oleh kepercayaan buta pada label open-source, terutama di bidang cryptocurrency.
