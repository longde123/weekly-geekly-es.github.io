Git é legal! Seu boné

Este talvez seja o meu artigo mais curto. Além disso, a ideia em si é óbvia.

Muitas vezes, em projetos WEB, há tarefas de fornecer estática para produzir lançamentos externos . Por exemplo, você precisa fazer upload de arquivos para o SPA com informações de marketing: comissões atuais, novas perguntas frequentes etc.

Geramos esses arquivos no painel de administração. Em seguida, eles espalharam rsync-com nos nós. As desvantagens de tantas soluções. A organização do registro e controle dos direitos de acesso são alguns dos mais graves.

Uma vez surgiu uma idéia para usar o git para fornecer estática de marketing aos produtos. I.e. os arquivos também são gerados no painel do administrador e, em seguida, são confirmados automaticamente para o usuário que os gerou e enviados para um nabo de marketing separado.

E então, a mágica familiar do CI / CD git funciona.

Além disso, agora temos acesso a lucros nos quais nem pensávamos - controle automático da execução real de tarefas de marketing através da integração do Jira e gitlab.
