Neu im Frühjahr 5. Projektmigration zum Frühjahr 5

Hallo! Fortsetzung einer Reihe von Migrationen zu neuen Versionen von Frameworks.
Das letzte Mal war es JUnit 5 in 10 Minuten. Diesmal ist Frühling 5.

Wenn sie im Frühjahr 5 über neue Dinge sprechen, fangen sie meistens an, über den Jet-Stack zu sprechen, wobei viele der neuen Dinge, die darin auftauchten, weggelassen wurden. Es gibt nur sehr wenige reaktive Treiber für Datenbanken, daher hoffe ich, dass mein Videoüberblick über andere wichtige Updates für die meisten Spring-Benutzer relevant sein wird. Darüber hinaus werden alle Abhängigkeiten zusammen mit Spring 5.x aktualisiert: Spring-Data 2.x, Spring-Security 5.x. Im Januar dieses Jahres wurde unser Schulungsprojekt für Spring 5 / JPA Enterprise (Topjava) von Spring 4.3.x auf Spring 5 migriert. Der Vorgang war recht einfach und am Ende eines kurzen Videos für 10 Minuten werde ich auch alle Änderungen im Projektcode auflisten.


Neu im Frühjahr 5


  • Verwendetes JDK 8+ (Optional, CompletableFuture, Zeit-API, java.util.function, Standardmethoden)
  • Java 9-Unterstützung (Automatic-Module-Name in 5.0, Modul-Info in 6.0+ , ASM 6)
  • Unterstützt HTTP / 2 ( TLS , Push ), NIO / NIO.2, Kotlin

  • Andere Änderungen

    • Null-Sicherheitsanmerkungen ( @Nullable ), neue Dokumentation
    • Komponentenindex zur Kompilierungszeit (Alternative zum Scannen von Klassenpfaden)
    • Kompatibel mit Java EE 8 (Servlet 4.0, Bean Validation 2.0, JPA 2.2 , JSON Binding API 1.0)
    • Neue gemeinsame Holzbrücke - spring-jcl
    • Unterstützung für JUnit 5 + Testverbesserungen (bedingt und gleichzeitig)
    • Unterstützung entfernt: Portlet, Velocity, JasperReports, XMLBeans, JDO, Guava
  • Spring-Data-JPA 2.x, Spring-Security 5.x.
  • Reaktivität ( Web on Reactive Stack )
