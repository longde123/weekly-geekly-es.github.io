Commandes Linux de base pour les testeurs et plus

Préface


Bonjour à tous! Je m'appelle Sasha et je teste le backend (services Linux et API) depuis plus de six ans. La pensée de l'article m'est venue après la prochaine demande d'un testeur familier pour lui dire ce qui peut être lu sur les commandes Linux avant l'interview. Habituellement, un candidat au poste d'ingénieur QA doit connaître les commandes de base (si, bien sûr, cela signifie travailler avec Linux), mais comment comprenez-vous les commandes à lire lors de la préparation d'un entretien si vous avez peu ou pas d'expérience Linux?

Par conséquent, bien qu'il ait déjà été écrit à ce sujet à plusieurs reprises, j'ai quand même décidé d'écrire un autre article «Linux pour les débutants» et de lister ici les commandes de base que vous devez connaître avant toute interview dans le département (ou l'entreprise) où ils utilisent Linux. J'ai pensé aux commandes et aux utilitaires et aux paramètres que j'utilise le plus souvent, j'ai recueilli les commentaires de mes collègues et je les ai regroupés dans un seul article. L'article est conditionnellement divisé en 3 parties: d'abord, de brèves informations sur les bases des E / S dans le terminal Linux, puis une vue d'ensemble des commandes les plus élémentaires, et la troisième partie décrit la solution des problèmes typiques sous Linux.

Chaque équipe a de nombreuses options, elles ne seront pas toutes répertoriées ici. Vous pouvez toujours entrer « man <command> » ou « <command> --help» pour en savoir plus sur la commande.

Un exemple: 

[user@testhost ~]$ mkdir --help Usage: mkdir [OPTION]... DIRECTORY... Create the DIRECTORY(ies), if they do not already exist. Mandatory arguments to long options are mandatory for short options too.   -m, --mode=MODE  set file mode (as in chmod), not a=rwx - umask   -p, --parents   no error if existing, make parent directories as needed   -v, --verbose   print a message for each created directory   -Z          set SELinux security context of each created directory                          to the default type       --context[=CTX] like -Z, or if CTX is specified then set the SELinux                          or SMACK security context to CTX       --help   display this help and exit       --version output version information and exit GNU coreutils online help: <http://www.gnu.org/software/coreutils/> For complete documentation, run: info coreutils 'mkdir invocation'


Si une commande met trop de temps à se terminer, elle peut être terminée en appuyant sur Ctrl + C dans la console (un signal SIGINT est envoyé au processus).

Un peu sur la sortie de commande


Lorsqu'un processus démarre sur Linux, 3 flux de données standard pour ce processus sont créés: stdin , stdout et stderr . Ils sont numérotés respectivement 0, 1 et 2. Mais nous nous intéressons maintenant à stdout et, dans une moindre mesure, à stderr . D'après les noms, il est facile de deviner que stdout est utilisé pour générer des données et stderr est utilisé pour afficher des messages d'erreur. Par défaut, lors de l'exécution d'une commande sous Linux, stdout et stderr impriment toutes les informations sur la console, cependant, si la sortie de la commande est volumineuse, il peut être pratique de la rediriger vers un fichier. Cela peut être fait, par exemple, comme ceci: 

 [user@testhost ~]$ man signal > man_signal

Si nous imprimons le contenu du fichier man_signal , nous verrons qu'il est identique à ce qu'il serait si nous exécutions simplement la commande ` man signal` .

L'opération de redirection ` > ` utilise stdout par défaut. Vous pouvez spécifier explicitement la redirection stdout : ` 1> `. De même, vous pouvez spécifier la redirection stderr : ` 2> `. Vous pouvez combiner ces opérations et ainsi séparer la sortie normale de la commande et la sortie des messages d'erreur: 

 [user@testhost ~]$ man signal 1> man_signal 2> man_signal_error_log

Vous pouvez rediriger stdout et stderr vers un seul fichier comme suit: 

 [user@testhost ~]$ man signal > man_signal 2>&1

L'opération de redirection « 2> & 1» signifie rediriger stderr vers où stdout est dirigé.

Un autre outil pratique pour travailler avec les E / S (ou plutôt, c'est un moyen pratique de communication interprocessus) est le tuyau (ou un pipeline ). Les pipelines sont souvent utilisés pour lier plusieurs commandes: la commande stdout est redirigée vers stdin comme suit, et ainsi de suite dans la chaîne:

Commandes Linux de base


pwd


Imprime le répertoire courant (de travail). 

 [user@testhost ~]$ pwd /home/user

date


Imprimez la date et l'heure actuelles du système. 

 [user@testhost ~]$ date Mon Dec 16 13:37:07 UTC 2019 [user@testhost ~]$ date +%s 1576503430

w


Cette commande indique qui est connecté au système. De plus, le temps de disponibilité et LA (moyenne de charge) sont également affichés. 

 [user@testhost ~]$ w 05:47:17 up 377 days, 17:57, 1 user, load average: 0,00, 0,01, 0,05 USER   TTY FROM       LOGIN@ IDLE JCPU PCPU WHAT user   pts/0 32.175.94.241  05:47 2.00s 0.01s 0.00sw

ls


Imprimez le contenu du répertoire. Si vous ne passez pas le chemin, le contenu du répertoire actuel sera affiché. 

 [user@testhost ~]$ pwd /home/user [user@testhost ~]$ ls qqq [user@testhost ~]$ ls /home/user qqq [user@testhost ~]$ ls / bin boot cgroup dev etc home lib lib64 local lost+found media mnt opt proc root run sbin selinux srv swap sys tmp usr var

Personnellement, j'utilise souvent les options -l (format de liste longue - sortie vers une colonne avec des informations supplémentaires sur les fichiers), -t (tri par heure de modification de fichier / répertoire) et -r (tri inversé - en combinaison avec -t les fichiers les plus "frais") sera ci-dessous):

Il existe 2 noms de répertoire spéciaux: " . " Et " .. ". Le premier est le répertoire courant, le second est le répertoire parent. Ils peuvent être facilement utilisés dans diverses commandes, en particulier ls : 

 [user@testhost home]$ pwd /home [user@testhost home]$ ls .. bin boot cgroup dev etc home lib lib64 local lost+found media mnt opt proc root run sbin selinux srv swap sys tmp usr var [user@testhost home]$ ls ../home/user/ qqq

Il existe également une option utile pour afficher les fichiers cachés (commencez par " . ") - -a : 

 [user@testhost ~]$ ls -a . .. 1 .bash_history .bash_logout .bash_profile .bashrc .lesshst man_signal man_signal_error_log .mongorc.js .ssh temp test .viminfo

Et vous pouvez également utiliser l'option -h - sortie au format lisible par l'homme (attention aux tailles de fichiers): 

 [user@testhost ~]$ ls -ltrh total 16K -rwxrwx--x 1 user user 31 Nov 26 11:09 temp -rw-rw-r-- 1 user user 6.0K Dec 3 16:02 1 drwxrwxr-x 2 user user 4.0K Dec 4 10:39 test

cd


Changer le répertoire actuel. 

 [user@testhost ~]$ pwd /home/user [user@testhost ~]$ cd /home/ [user@testhost home]$ pwd /home

Si vous ne passez pas le nom du répertoire comme argument, la variable d'environnement $ HOME , c'est-à-dire le répertoire personnel, sera utilisée. Il peut également être pratique d'utiliser ` ~ `, un caractère spécial signifiant $ HOME : 

 [user@testhost etc]$ pwd /etc [user@testhost etc]$ cd ~/test/ [user@testhost test]$ pwd /home/user/test

mkdir


Créez un répertoire. 

 [user@testhost ~]$ mkdir test [user@testhost ~]$ ls -ltr total 38184 -rw-rw-r-- 1 user user 39091284 Nov 22 14:14 qqq drwxrwxr-x 2 user user   4096 Nov 26 10:29 test

Parfois, vous devez créer une structure de répertoires spécifique: par exemple, un répertoire dans un répertoire qui n'existe pas. Afin de ne pas entrer mkdir plusieurs fois de suite, vous pouvez utiliser l'option -p - elle vous permet de créer tous les répertoires manquants dans la hiérarchie. De plus, avec cette option, mkdir ne retournera pas d'erreur si le répertoire existe. 

 [user@testhost ~]$ ls qqq test [user@testhost ~]$ mkdir test2/subtest mkdir: cannot create directory 'test2/subtest': No such file or directory [user@testhost ~]$ mkdir -p test2/subtest [user@testhost ~]$ ls qqq test test2 [user@testhost ~]$ ls test2/ subtest [user@testhost ~]$ mkdir test2/subtest mkdir: cannot create directory 'test2/subtest': File exists [user@testhost ~]$ mkdir -p test2/subtest [user@testhost ~]$ ls test2/ subtest

rm


Supprimer le fichier. 

 [user@testhost ~]$ ls qqq test test2 [user@testhost ~]$ rm qqq [user@testhost ~]$ ls test test2

L'option -r vous permet de supprimer récursivement les répertoires avec tout leur contenu, l'option -f vous permet d'ignorer les erreurs lors de la suppression (par exemple, un fichier qui n'existe pas). Ces options permettent, en gros, de garantir la suppression de toute la hiérarchie des fichiers et répertoires (si l'utilisateur en a les droits), par conséquent, elles doivent être utilisées avec prudence (l'exemple de plaisanterie classique est " rm -rf / ", dans certaines circonstances, il supprimera si pas l'ensemble du système, il y a beaucoup de fichiers importants pour ses performances). 

 [user@testhost ~]$ ls test test2 [user@testhost ~]$ ls -ltr test2/ total 4 -rw-rw-r-- 1 user user  0 Nov 26 10:40 temp drwxrwxr-x 2 user user 4096 Nov 26 10:40 temp_dir [user@testhost ~]$ rm -rf test2 [user@testhost ~]$ ls test

cp


Copiez un fichier ou un répertoire. 

 [user@testhost ~]$ ls temp test [user@testhost ~]$ cp temp temp_clone [user@testhost ~]$ ls temp temp_clone test

Cette commande possède également les options -r et -f ; elles peuvent être utilisées pour garantir que la hiérarchie des répertoires et des dossiers sera copiée vers un autre emplacement.

mv


Déplacez ou renommez un fichier ou un répertoire. 

 [user@testhost ~]$ ls -ltr total 4 drwxrwxr-x 2 user user 4096 Nov 26 10:29 test -rw-rw-r-- 1 user user  0 Nov 26 10:45 temp -rw-rw-r-- 1 user user  0 Nov 26 10:46 temp_clone [user@testhost ~]$ ls test [user@testhost ~]$ mv test test_renamed [user@testhost ~]$ mv temp_clone test_renamed/ [user@testhost ~]$ ls temp test_renamed [user@testhost ~]$ ls test_renamed/ temp_clone

chat


Imprimez le contenu du ou des fichiers. 

 [user@testhost ~]$ cat temp Content of a file. Lalalala...

Il convient également de prêter attention aux commandes head (imprimer les n premières lignes ou octets du fichier) et tail (plus à ce sujet plus tard).

queue


Affiche n dernières lignes ou octets du fichier. 

 [user@testhost ~]$ tail -1 temp Lalalala...

L'option -f est très utile - elle vous permet de sortir de nouvelles données dans un fichier en temps réel.

moins


Parfois, le fichier texte est trop volumineux et il n'est pas pratique de l'afficher avec la commande cat . Ensuite, vous pouvez l'ouvrir avec la commande less : le fichier sera affiché dans les parties, la navigation est disponible dans ces parties, la recherche et d'autres fonctionnalités simples. 

 [user@testhost ~]$ less temp

Il peut également être pratique d'utiliser moins avec un tuyau : 

 [user@testhost ~]$ grep "ERROR" /tmp/some.log | less

tuer


Envoyez un signal au processus. Par défaut, un signal SIGTERM est envoyé , ce qui met fin au processus. 

 [user@testhost ~]$ ps ux USER     PID %CPU %MEM  VSZ RSS TTY  STAT START TIME COMMAND 531   1027147 0.0 0.0 119956 4260 ? Puisqu'un processus peut avoir des gestionnaires de signaux, kill ne mène pas toujours au résultat attendu - une interruption instantanée du processus. Pour «tuer» un processus à coup sûr, vous devez envoyer un signal SIGKILL au processus. Cependant, cela peut entraîner une perte de données (par exemple, si le processus doit enregistrer certaines informations sur le disque avant de se terminer), vous devez donc utiliser cette commande avec précaution. Le numéro de signal de SIGKILL est 9, donc la version courte de la commande ressemble à ceci: 

 [user@testhost ~]$ ps ux | grep sleep 531   1034930 0.0 0.0 107960 636 pts/1  S+ 15:21 0:00 sleep 300 531   1034953 0.0 0.0 110516 2104 pts/2  S+ 15:21 0:00 grep --color=auto sleep [user@testhost ~]$ kill -9 1034930 [user@testhost ~]$ ps ux | grep sleep 531   1035004 0.0 0.0 110516 2092 pts/2  S+ 15:22 0:00 grep --color=auto sleep

En plus des SIGTERM et SIGKILL mentionnés , il existe de nombreux signaux différents, leur liste peut être facilement trouvée sur Internet. Et n'oubliez pas que les signaux SIGKILL et SIGSTOP ne peuvent pas être interceptés ou ignorés.

ssh


Le client OpenSSH SSH vous permet de vous connecter à un hôte distant. 

 MacBook-Pro-User:~ user$ ssh user@11.11.22.22 Last login: Tue Nov 26 11:27:39 2019 from another_host [user@testhost ~]$ hostname testhost

Il y a beaucoup de nuances dans l'utilisation de SSH, ce client a également beaucoup de fonctionnalités, donc si vous le souhaitez (ou en avez besoin), vous pouvez en lire plus en détail .

Copiez les fichiers entre les hôtes ( ssh est utilisé pour cela).

écho


Affichez une ligne de texte. 

 [user@testhost ~]$ echo "Hello" Hello

Les options -n pour ne pas compléter la ligne avec des sauts de ligne à la fin, et -e pour activer l'interprétation de l'échappement avec "\" sont à noter ici. 

 [user@testhost ~]$ echo "\tHello\n" \tHello\n [user@testhost ~]$ echo -n "\tHello\n" \tHello\n[user@testhost ~]$ [user@testhost ~]$ echo -ne "\tHello\n" Hello

De plus, en utilisant cette commande, vous pouvez afficher les valeurs des variables. Par exemple, sous Linux, le code de sortie de la dernière commande terminée est stocké dans la variable spéciale $? , et ainsi vous pouvez savoir exactement quelle erreur s'est produite dans la dernière application en cours d'exécution: 

 [user@testhost ~]$ ls  #    1 man_signal man_signal_error_log temp test [user@testhost ~]$ echo $?  #  0 —    0 [user@testhost ~]$ ls qwerty  #   ls: cannot access qwerty: No such file or directory [user@testhost ~]$ echo $?  #  2 — Misuse of shell builtins (according to Bash documentation) 2 [user@testhost ~]$ echo $?  #  echo   ,  0 0

Résolution de problèmes Linux typiques


Changer le propriétaire du fichier


Vous pouvez modifier la propriété d'un fichier ou d'un répertoire à l'aide de la commande chown : 

 [user@testhost ~]$ chown user:user temp [user@testhost ~]$ ls -l temp -rw-rw-r-- 1 user user 31 Nov 26 11:09 temp

Dans le paramètre, cette commande doit recevoir le nouveau propriétaire et le nouveau groupe (facultatif), séparés par deux points. De plus, lorsque vous changez le propriétaire du répertoire, l'option -R peut être utile - les propriétaires de tout le contenu du répertoire changeront également.

Modifier les autorisations de fichier


Ce problème est résolu à l'aide de la commande chmod . À titre d'exemple, je donnerai l'installation des droits "le propriétaire est autorisé à lire, écrire et exécuter, le groupe est autorisé à lire et à écrire, tout le reste - rien": 

 [user@testhost ~]$ ls -l temp -rw-rw-r-- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod 760 temp [user@testhost ~]$ ls -l temp -rwxrw---- 1 user user 31 Nov 26 11:09 temp

Le premier 7 (c'est 0b111 dans la représentation binaire) dans le paramètre signifie «tous les droits pour le propriétaire», le second 6 (c'est 0b110 dans la représentation binaire) - «lire et écrire», eh bien, 0 n'est rien pour les autres. Un masque de bits se compose de trois bits: le bit le moins significatif («droit») est responsable de l'exécution, le bit suivant («milieu») est pour l'écriture et le plus ancien («gauche») est pour la lecture.
Vous pouvez également définir des droits à l'aide de caractères spéciaux ( syntaxe mnémonique ). Par exemple, dans l'exemple suivant, les droits d'exécution de l'utilisateur actuel sont d'abord supprimés, puis renvoyés:

 [user@testhost ~]$ ls -l temp -rwxrw---- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod -x temp [user@testhost ~]$ ls -l temp -rw-rw---- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod +x temp [user@testhost ~]$ ls -l temp -rwxrwx--x 1 user user 31 Nov 26 11:09 temp

Cette commande a de nombreuses utilisations, je vous conseille donc d'en lire plus (en particulier sur la syntaxe mnémonique, par exemple, ici ).

Imprimer le contenu d'un fichier binaire


Cela peut être fait en utilisant l'utilitaire hexdump . Voici des exemples de son utilisation.

 [user@testhost ~]$ cat temp Content of a file. Lalalala... [user@testhost ~]$ hexdump -c temp 0000000  C on  tent    ofaf  il 0000010  e . \n  L ala  lala .  . . \n 000001f [user@testhost ~]$ hexdump -x temp 0000000  6f43 746e  6e65 2074 666f  6120 6620 6c69 0000010  2e65 4c0a  6c61 6c61 6c61  2e61 2e2e 000a 000001f [user@testhost ~]$ hexdump -C temp 00000000 43 6f 6e 74 65 6e 74 20 6f 66 20 61 20 66 69 6c |Content of a fil| 00000010 65 2e 0a 4c 61 6c 61 6c 61 6c 61 2e 2e 2e 0a |e..Lalalala....| 0000001f

En utilisant cet utilitaire, vous pouvez afficher des données dans d'autres formats, cependant, ces options pour leur utilisation sont le plus souvent utiles.

Rechercher des fichiers


Vous pouvez rechercher un fichier par son nom dans l'arborescence des répertoires à l'aide de la commande find :

 [user@testhost ~]$ find test_dir/ -name "*le*" test_dir/file_1 test_dir/file_2 test_dir/subdir/file_3

D'autres options et filtres de recherche sont également disponibles. Par exemple, vous pouvez trouver des fichiers dans le dossier de test qui ont été créés il y a plus de 5 jours:

 [user@testhost ~]$ ls -ltr test total 0 -rw-rw-r-- 1 user user 0 Nov 26 10:46 temp_clone -rw-rw-r-- 1 user user 0 Dec 4 10:39 created_today [user@testhost ~]$ find test/ -type f -ctime +5 test/temp_clone

Rechercher du texte dans des fichiers


La commande grep vous aidera à faire face à cette tâche . Elle a de nombreuses utilisations, voici un exemple des plus simples.

 [user@testhost ~]$ grep -nr "content" test_dir/ test_dir/file_1:1:test content for file_1 test_dir/file_2:1:test content for file_2 test_dir/subdir/file_3:1:test content for file_3

Une façon populaire d'utiliser la commande grep est de l'utiliser dans un canal :

 [user@testhost ~]$ sudo tail -f /var/log/test.log | grep "ERROR"

L'option -v vous permet de faire l'effet de grep 'et l'inverse - seules les lignes qui ne contiennent pas le modèle passé à grep seront affichées .

Afficher les packages installés


Il n'y a pas de commande universelle, car tout dépend de la distribution Linux et du gestionnaire de paquets utilisé. L'une des commandes suivantes vous aidera probablement:

 yum list installed apt list --installed zypper se —installed-only pacman -Qqe dpkg -l rpm -qa

Voir combien d'espace prend une arborescence de répertoires


Un cas d'utilisation pour la commande du :

 [user@testhost ~]$ du -h -d 1 test_dir/ 8,0K test_dir/subdir 20K test_dir/

Vous pouvez modifier la valeur du paramètre -d pour obtenir des informations plus détaillées sur l'arborescence de répertoires. Vous pouvez également utiliser la commande en combinaison avec le tri :

 [user@testhost ~]$ du -h -d 1 test_dir/ | sort -h 8,0K test_dir/subdir 16K test_dir/subdir_2 36K test_dir/ [user@testhost ~]$ du -h -d 1 test_dir/ | sort -h -r 36K test_dir/ 16K test_dir/subdir_2 8,0K test_dir/subdir

L'option -h de la commande sort vous permet de trier les tailles écrites au format lisible par l'homme (par exemple, 1K, 2G), l'option -r vous permet de trier les données dans l'ordre inverse.

"Rechercher et remplacer" dans un fichier, dans des fichiers dans un répertoire


Cette opération est effectuée à l'aide de l'utilitaire sed (sans l'indicateur g à la fin, seule la première occurrence de «old-text» dans la chaîne sera remplacée):

 sed -i 's/old-text/new-text/g' input.txt

Vous pouvez l'utiliser pour plusieurs fichiers à la fois: 
 [user@testhost ~]$ cat test_dir/file_* test content for file_1 test content for file_2 [user@testhost ~]$ sed -i 's/test/edited/g' test_dir/file_* [user@testhost ~]$ cat test_dir/file_* edited content for file_1 edited content for file_2

Dériver la colonne de la sortie


Pour faire face à cette tâche aidera la awk . Dans cet exemple, la deuxième colonne de la sortie de la commande ps ps s'affiche :

 [user@testhost ~]$ ps ux | awk '{print $2}' PID 11023 25870 25871 25908 25909

Il convient de garder à l'esprit que awk a des fonctionnalités beaucoup plus riches, donc si vous avez besoin de travailler avec du texte sur la ligne de commande, vous devriez en savoir plus sur cette commande.

Vidage du trafic réseau


Pour intercepter le trafic réseau sous Linux, l'utilitaire tcpdump est utilisé . Pour vider le trafic sur le port 12345, vous pouvez utiliser la commande suivante:

 [user@testhost ~]$ sudo tcpdump -i any -A port 12345

L'option -A indique que nous voulons voir la sortie en ASCII (par conséquent, c'est bon pour les protocoles de texte), -i any indique que nous ne sommes pas intéressés par l'interface réseau, port - quel port le trafic à vider. Au lieu du port, vous pouvez utiliser l' hôte ou une combinaison d' hôte et de port ( hôte A et port X ). Et l'option -n peut également être utile - ne convertissez pas les adresses en noms d'hôtes en sortie.
Et si le trafic binaire? Ensuite, l'option -X nous aidera à produire des données en hexadécimal et ASCII:

 [user@testhost ~]$ sudo tcpdump -i any -X port 12345
