解决“短左移”问题

长时间使用键盘时的舒适度是影响生产率和手部健康的重要因素。 通常，我们必须保持连续几个小时不间断地操作，因此，除了实际选择合适的键盘（按键行程高度，击键硬度，从操作点到停止位置的击键深度，按键表面的触感，键盘外壳的刚度，键盘布局选项等）外，建议这样做想一些简单的技巧就可以毫不费力地提高键盘的可用性。

在我看来，在“中等或低键行程，中等硬度，仅用于繁重工作（非游戏）的无线键盘”类别中，领导者是Logitech键盘。

不幸的是，该公司的大多数本地化版本的全尺寸键盘型号（带有拉丁语和第二语言字符，请参见文章末尾的注释）不属于“标准布局，两个长Shift，大Enter，长退格的全尺寸键盘”类别。

之所以要“不包含”，是因为Logitech设计师和工程师一直希望通过在其右方放置一个带有斜杠的重复键（向前，向后，竖线）来“缩短”左Shift。

通常，要按下长的Shift键，您可以使用水平放置的小手指，或者使用小指和无名指或其他一些选项-直观地假设左Shift键足够长，并且您无需尝试更靠近键盘的左边缘。 因此，经常会错误地按下向左复制“斜杠”的键。

我曾经遭受这种不便，特别是当我不得不从长Shift左键的键盘切换到短键盘-Logitech的键时。
谷歌搜索-事实证明这很困难，因为很难提出查询来选择解决此特定问题的方法，因为诸如“通用键盘驱动程序”之类的结果有些不同。

事实证明，该解决方案非常简单-KeyTweak实用程序。 据我了解，该站点已不再存在，但是您可以从许多软件门户网站下载它。 不需要安装-这通常是正确的，因为很可能您只需要在特定的计算机上运行一次实用程序（否则，某人会因低级别的键重映射而迷失了方向）。

屏幕截图清楚地显示了接待的实质：



它只是用左Shift的低级代码替换了左侧重复的“斜杠”键的低级代码。 因此，现在按下重复键类似于按下左Shift键。

注册表中的设置如下所示（您可以不使用该实用程序）：


（注册表分支\ HKEY_LOCAL_MACHINE \ SYSTEM \ CURRENTCONTROLSET \ CONTROL \键盘布局）

由于这是注册表中的设置，因此不再需要KeyTweak实用程序-它不是驱动程序，而只是一个界面，用于方便此类设置。

在这种情况下：

  • 左移本身并没有改变；
  • 要按左Shift，可以按它，可以按“斜杠”交替键，可以同时按Shift和交替-就像您按了很长的左Shift键一样； 代理人完全参与类似于快捷键Shift的键盘快捷键（使用Ctrl，Alt和其他快捷键）；
  • 正确的转变没有改变；
  • 带有斜杠的右右键（位于Enter旁边）没有任何更改-它既有效又有效；
  • 不仅在Logitech键盘上进行了测试，还在其他多家制造商的键盘上进行了测试；
  • 该设置在特定的PC上完成一次，更改键盘时您无需执行任何操作-如果新键盘具有左“斜杠”键，则使用该设置；如果没有这样的键（即，左Shift键很长），则该设置不会产生任何影响。

关于“本地化”键盘的注释是Logitech-据我了解，缩短的Shift键仅对于本地化键盘是一个问题。 对于同一型号，仅拉丁字母键盘具有正常的长Shift键（请参阅下面的官方照片链接），而本地化的键盘则为短键（我提供链接的所有键盘在俄罗斯零售中均具有短的Shift键）。 设计思想的深度是什么-我不明白。 当然，也有例外-键盘型号的左Shift键长，无论国际化如何。
