FAQ sur Windows Terminal Build 2019

La semaine dernière, Microsoft a tenu sa conférence Build 2019 au Washington State Convention Center à Seattle . Build est un grand événement auquel participent plusieurs milliers de personnes du monde entier pour tout savoir sur la technologie orientée développeur actuelle, la plus récente et future de Microsoft.


Nous avons eu le plaisir de vous rencontrer si nombreux sur notre stand et de répondre à toutes vos questions!



Foire aux questions


Voici quelques-unes des questions les plus fréquentes que nous avons entendues (et entendons toujours), ainsi que les réponses officielles:


Quand et où puis-je me procurer le nouveau terminal?


  1. Vous pouvez cloner le code source du terminal à partir de GitHub sur https://github.com/microsoft/terminal et le créer sur votre machine.
    a. Remarque: assurez-vous de lire et de suivre les instructions sur la page README du repo avant d'essayer de construire le projet - il y a quelques prérequis et étapes d'initialisation requis avant que le projet ne se construise!
  2. Le terminal en tant qu'aperçu sera disponible en téléchargement sur le Microsoft Store d'ici l'été 2019.
  3. Nous visons à livrer Windows Terminal v1.0 d'ici la fin de 2019, mais nous travaillerons avec la communauté pour livrer la version 1.0 lorsque Terminal est de qualité suffisamment élevée.

Le terminal Windows remplace-t-il l'invite de commandes et / ou PowerShell?


Pour répondre à cette question, clarifions quelques termes et concepts:


  • L'invite de commandes et PowerShell (comme WSL / bash / etc. sur * NIX) sont des shells, pas des terminaux, et n'ont pas d'interface utilisateur propre
  • Lorsqu'un shell / application / outil en ligne de commande est exécuté, Windows les lance automatiquement et les connecte aux instances de la console Windows (si nécessaire)
  • La console Windows est l'application d'interface utilisateur standard de type terminal livrée avec Windows et utilisée par les utilisateurs au cours des 30 dernières années lors de l'exécution d'outils de ligne de commande sous Windows NT, 2000, XP, Vista, 7, 8 et 10.


Ainsi, la question est très probablement reformulée comme "Le terminal Windows remplace-t-il la console?"


La réponse est «non»:


  • La console Windows continuera à être livrée dans Windows pendant des décennies à venir afin d'assurer une compatibilité descendante avec les millions de scripts, applications et outils de ligne de commande existants / hérités
  • Le terminal Windows fonctionnera aux côtés de la console, mais deviendra probablement l'outil préféré des utilisateurs souhaitant exécuter des outils de ligne de commande sous Windows
  • Windows Terminal peut se connecter à l'invite de commandes et à PowerShell, ainsi qu'à tout autre shell / outil / application en ligne de commande. Vous pourrez ouvrir des onglets indépendants connectés à l'invite de commandes, PowerShell, bash (via WSL ou ssh) et tout autre shell / outil de votre choix

Quand puis-je obtenir la nouvelle police?


Bientôt! Nous n'avons pas de calendrier défini, mais nous travaillons avec impatience pour finaliser la police. Une fois prêt à être publié, il sera open source dans son propre référentiel.

Appel à l'action!


Assurez-vous de suivre @cinnamon_msft et @richturn_ms sur Twitter et revenez souvent sur notre blog en ligne de commande au cours des semaines et des mois à venir pour en savoir plus sur Terminal et nos progrès vers la version 1.0.


Si vous êtes un développeur et que vous souhaitez vous impliquer, veuillez visiter le référentiel Terminal GitHub , et passer en revue et discuter des problèmes avec l'équipe et la communauté, et si vous avez le temps, contribuez à des RP contenant des correctifs et des améliorations pour nous aider à créer le terminal génial!


Si vous n'êtes pas développeur mais que vous souhaitez toujours essayer le Terminal, téléchargez le Terminal depuis la boutique Microsoft après l'avoir publié cet été et assurez-vous de nous envoyer des commentaires sur ce que vous aimez, ce que vous n'aimez pas, etc. .


Nous nous réjouissons de vous entendre!
