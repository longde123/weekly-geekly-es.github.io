OpenSceneGraph：从源代码和Hello World构建



引言


OpenSceneGraph （以下称为OSG）是一个用C ++编写的开放式跨平台框架，它代表图形引擎，该图形引擎为程序员提供了OpenGL的对象接口。 在我们的国家，这种引擎并不是特别流行，即使在哈布雷（Habré）上，我也只看到或多或少像样的出版物 。 OSG在国外有很多使用，例如，它是免费FlightGear飞行模拟器的基础， Morrowind游戏的一种开放实现，称为OpenMW，其开发也从Ogre引擎转移到OSG。 俄语文档几乎很少，在英语文档中，只有一小部分来自开发人员的书籍可以被提及： OpenSceneGraph 3.0。 入门指南和OpenSceneGraph 3.食谱 。

尽管如此，由于以下原因，该引擎还是很有趣的：

  1. C ++中的开放式跨平台实现。
  2. 模块化架构。
  3. 通过内置插件系统可扩展。
  4. 图形数据的多线程处理的可能性以及用于其实现的内置工具
  5. 通过智能指针机制进行动态内存管理

我认为对于Habr的读者来说，更详细地了解这个项目将很有趣。 俄语OSG知识库的补充不会多余。 我将就此主题发布的所有材料均基于OpenSceneGraph 3.0。 初学者指南 ，但这不是其翻译，而是对此处介绍材料的创造性修改。 如果您对此主题感兴趣，请要求

在计算机上获取最新版本的OSG的唯一肯定方法是从源代码构建库。 Windows的现有二进制安装程序着重于MS Visual C ++编译器。 对我来说，对于我的项目，有必要使用GCC编译器，或者更确切地说，使用它的MinGW32版本，该版本包含在Qt框架开发工具中。 因此，我们需要：

  1. 使用MinGW32 5.3版编译器和QtCreator IDE安装和配置Qt框架
  2. Windows版Git客户端
  3. Windows的Cmake实用程序

期望读者熟悉Qt项目中使用的QtCreator IDE和qmake构建系统。 另外，假定读者了解使用Git版本控制系统的基础知识，并且原则上具有非零编程技能。

3.组装和安装引擎的调试和发行版本


现在运行cmd shell，该快捷方式位于桌面的开始->程序-> Qt-> Qt 5.11.2-> Qt 5.11.2（MinGW 5.3.0 32bit）上



运行中的命令行会话设置了mingw32构建工具正常工作所需的整个环境。 转到OSG源目录 

 C:\Qt\Qt5.11.2\5.11.2\mingw53_32>D: D:\> cd OSG\build-win32-debug

发出命令 

 D:\OSG\build-win32-debug>cmake -G "MinGW Makefiles" -DCMAKE_INSTALL_PREFIX=E:\Apps\OSG -DCMAKE_BUILD_TYPE=DEBUG ../OpenSceneGraph

让我们更详细地分析参数的含义：

  • -G“ MinGW Makefiles”-表示有必要为mingw32-make实用程序生成Makefile
  • -DCMAKE_INSTALL_PREFIX = E：\ Apps \ OSG-设置OSG的安装路径
  • -DCMAKE_BUILD_TYPE = DEBUG-指示应构建引擎的调试版本。

让我们创建一些系统环境变量，这些变量指示库，头文件和OSG插件的路径。 在我的示例中，它看起来像这样



必须创建其名称在屏幕快照中用红色圈出的变量。 创建变量后，为了使它们能够被开发工具看到，尤其是QtCreator需要至少登录系统（注销并以当前用户身份登录），或者可能地，重新引导系统（这是Windows！）。

5.将Hello World写入QtCreator


我们将以一个简单的示例开始熟悉OpenSceneGraph图形引擎，这是在使用某些“ Hello world！”进行编程时惯常使用的示例。

在编写任何代码之前，我们将为qmake构建系统配置项目，我们将在整个系列文章中使用该系统。 在我们感兴趣的文件系统中，创建以下目录结构 

 OSG-lessons/ |-data/ |-OSG-lessons/ | |-hello/ |-include/ |-src/

在hello目录中，创建具有以下内容的hello.pro文件

全文hello.pro 
 TEMPLATE = app TARGET = hello DESTDIR = ../../bin win32 { OSG_LIB_DIRECTORY = $$(OSG_BIN_PATH) OSG_INCLUDE_DIRECTORY = $$(OSG_INCLUDE_PATH) CONFIG(debug, debug|release) { TARGET = $$join(TARGET,,,_d) LIBS += -L$$OSG_LIB_DIRECTORY -losgd LIBS += -L$$OSG_LIB_DIRECTORY -losgViewerd LIBS += -L$$OSG_LIB_DIRECTORY -losgDBd LIBS += -L$$OSG_LIB_DIRECTORY -lOpenThreadsd } else { LIBS += -L$$OSG_LIB_DIRECTORY -losg LIBS += -L$$OSG_LIB_DIRECTORY -losgViewer LIBS += -L$$OSG_LIB_DIRECTORY -losgDB LIBS += -L$$OSG_LIB_DIRECTORY -lOpenThreads } INCLUDEPATH += $$OSG_INCLUDE_DIRECTORY } unix { CONFIG(debug, debug|release) { TARGET = $$join(TARGET,,,_d) LIBS += -losgd LIBS += -losgViewerd LIBS += -losgDBd LIBS += -lOpenThreadsd } else { LIBS += -losg LIBS += -losgViewer LIBS += -losgDB LIBS += -lOpenThreads } } INCLUDEPATH += ./include HEADERS += $$files(./include/*.h) SOURCES += $$files(./src/*.cpp)

我们将更详细地分析这些字母。 

 TEMPLATE = app TARGET = hello DESTDIR = ../../bin


变量指定项目模板（应用程序-应用程序），可执行文件的名称（hello）以及汇编后可执行文件的放置目录。

根据组装项目的OS，我们定义变量，这些变量指示库目录和OSG头文件的路径。 这是OSG_BIN_PATH和OSG_INCLUDE_PATH环境变量派上用场的地方-现在，在哪里安装OSG库都无关紧要。 任何想要在其计算机上使用此项目的人都将在其系统中编写适当的环境变量，而无需编辑构建脚本。
