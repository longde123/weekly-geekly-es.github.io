Recopilamos en rpm WireMock, una utilidad para crear stubs sobre servicios web en Fedora COPR, análogo de PPA en Ubuntu

WireMock es una utilidad de biblioteca java para crear stubs sobre servicios web. Crea un servidor HTTP, al que podríamos conectarnos como un servicio web real.


Fedora COPR es un servicio de alojamiento gratuito para alojar repositorios personalizados (similar a AUR en Arch Linux o PPA en Ubuntu). De las características, la capacidad incorporada de recopilar paquetes rpm especificando el nombre de PIP y RubyGems.


En esta publicación escribiré cómo recopilar rpm de su repositorio comprometiéndome con Fedora COPR .


Por ejemplo, tome el repositorio wiremock-rpm en el que creé las especificaciones y los archivos de la unidad systemd. Al crear su repositorio, puede tomar wiremock-rpm como base y cambiar lo que necesita. Omito la escritura de especificaciones y archivos de unidad systemd. Creo que lo sabes


Creando un proyecto en Fedora COPR


Ir a Fedora COPR .



Nombre del proyecto : especifique el nombre del paquete. Por lo general, es el mismo que el nombre de su repositorio git.


Descripción : una breve descripción del proyecto.


Instrucciones : Cómo instalar tu paquete. Indique lo siguiente: 


yum -y install yum-plugin-copr yum copr enable ---fedora-copr/- yum -y install - systemctl start -

Página de inicio : especifique la página de inicio del programa que desea construir o su repositorio.



Opciones de compilación : en Chroots indicamos en qué sistema operativo desea recopilar el paquete.



Resulta como en la captura de pantalla.



Otras opciones : si necesita Internet, marque la casilla Habilitar el acceso a Internet durante las compilaciones en Booleanos



Después de crear el proyecto, vaya a Paquetes .



Nombre del paquete: especifique el nombre del paquete.


URL de clonación: especifique el repositorio de git.


Subdirectorio: es mejor no utilizar este elemento y mantener la fuente y el archivo de especificaciones en la raíz del proyecto. Si tiene las fuentes en un directorio y el archivo de especificaciones en otro directorio, puede haber problemas con la compilación.


Archivo de especificaciones : ruta al archivo de especificaciones.



Configuración del paquete genérico : asegúrese de verificar la reconstrucción automática



Después de crear el paquete, vaya a Configuración , luego a Integraciones. A continuación, en la página, copiamos el webhook del sistema donde se encuentra su repositorio git.



Vaya a Configuración donde se encuentra su repositorio git. Luego para github, vaya a webhook. Añadir webhook. Inserte URL de carga útil, seleccione Tipo de contenido application / json


Ahora sobre WireMock.


Instale la maqueta de alambre de acuerdo con las instrucciones. El directorio de trabajo de wiremock en mi proyecto es /usr/lib/wiremock /. Este directorio contiene wiremock.jar y el directorio de asignaciones. En el directorio de asignaciones hay archivos json con solicitudes que envía a wiremock y respuestas preparadas.


Ejemplo de http://wiremock.org/docs/running-standalone/ : 


 { "request": { "method": "GET", "url": "/api/mytest" }, "response": { "status": 200, "body": "More content\n" } }

Enviamos una solicitud a / api / mytest y obtenemos: 


 curl http://localhost:8080/api/mytest More content

Un ejemplo de mi json preparado: 


 { "request": { "method": "GET", "url": "/503" }, "response": { "status": 503, "body": "503 Service Unavailable\n" } }
