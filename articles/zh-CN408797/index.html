蓝屏死机简史



尽管微软不希望这样做，但这是一种邪教。 在90年代，这与使用Windows作为Paint and Solitaire的基本体验是相同的部分，但是今天并不常见。

当然，我说的是“ 死亡蓝屏”，BSoD。 年轻的PC用户不知道这种恐慌传播屏幕曾经有多广泛，或意味着什么。 您所做的一切都丢失了，您的计算机需要重新启动-然后可能需要十分钟。

我们中那些记得他的人正试图忘记他-但这并不容易。

时至今日，BSoD是无法正常工作的可识别符号，但是为什么它甚至出现了呢？ 这是一段短暂的旅程，沿着记忆之路的一个难以区分的部分，您的父母警告您有危险。

Windows 3.1：屏幕Ctrl + Alt +删除




Windows 3.1没有BSoD：崩溃时，您会看到黑屏。 如果幸运的话，此屏幕上会显示DOS命令输入字段，您可以从中再次启动Windows。 如果没有，则重启时间到了。

但是，通过按Ctrl + Alt + Delete可以调出蓝屏。 将来，他将启发BSoD的设计。

有趣的是，正如雷蒙德·陈（Raymond Chen）在博客中所记录的那样，蓝屏的文字由公司的未来总监史蒂夫·鲍尔默（Steve Ballmer）管理微软系统部门时所写。

Windows 95和Windows 98：死亡的第一个蓝屏


很难高估Windows 95的重要性：想像一下早期iPhone机型的炒作，而只是桌面操作系统的炒作。 人们从字面上排在商店里。 是的，90年代很奇怪：人们对台式计算机的新功能感到满意。 但是没有人对此屏幕充满热情。



每当程序或驱动程序崩溃时，BSoD就会显示出来。 他提供了有关问题原因的某种加密信息，然后就下一步该如何做提供了更清晰的建议。

从理论上讲，按任意键都应关闭该程序并使您返回Windows桌面，但实际上它很少起作用。 根据Raymond Chen的说法，此消息的早期版本表示“有可能继续以正常模式工作”，但是由于过于乐观，因此删除了此行。

Windows 2000：也许检查过病毒？




通过Windows 2000的发行，Microsoft扩展了其蓝屏提示集。 关于可能返回桌面的所有参考均已消失，并立即建议用户关闭计算机。 从扫描病毒到检查硬盘驱动器的完整性，如果再次出现此问题，还有一套跟踪错误的想法。

Windows XP，Vista和7：更多提示




Windows XP延续了在蓝屏上增加提示数量的趋势。 关于哪个程序导致问题的信息仍然不清楚，但是至少它给出了一些可以在Google上搜索的代码。 屏幕的其余部分充满了各种技巧。 仍然建议用户关闭计算机，但也建议他们检查所有程序是否安装正确，并且他们还提供了一些想法来查找原因。

在Vista中，蓝屏变化不大，但开始出现更多。 Windows 7减少了其外观数量，但未更改其外观。

Windows 8：没有细节的悲伤图释




Windows 8完全改变了BSoD。 类似于终端的文本屏幕消失了，它被现代系统字体和ASCII中巨大的悲伤笑脸所取代。 最有趣的是，几乎所有有关问题原因的信息都已消失，寻找解决方案的技巧也已消失。

这不是像以前那样严重的问题，因为蓝屏在今天已经不那么普遍了。 您可以通过检查日志或使用收集此类信息的第三方程序来找出导致PC崩溃的原因。

在Windows 10上，屏幕看起来相同。

蓝屏遗产


微软希望情况有所不同，但是直到今天，死亡的蓝屏仍然是Windows的象征。 这一事实启发了有史以来最伟大的办公游戏之一-BSoD屏幕保护程序。 它是由Sysinternals（后来被Microsoft收购）制造的，它使任何计算机看上去都像被冻结了一样，直到您按下键或抽动鼠标为止。 很好笑
