Keajaiban kemasan dari Microsoft: kernel Linux di Windows 10 dan mesin IE di dalam Chromium Edge

Pada konferensi tahunan untuk pengembang , Microsoft membuat beberapa presentasi yang cukup penting sekaligus. Kami memilih dua dari mereka.

Pertama: pada musim panas 19H2 Windows 10, kernel Linux lengkap berdasarkan versi 4.19 tanggal 22 Oktober 2018 untuk subsistem Linux untuk Windows sendiri (WSL - Windows Subsystem Linux) akan dikirimkan.

Kedua: di perusahaan di masa depan membangun reinkarnasi Chromium dari Microsoft Edge euthanized, inti dari IE akan diintegrasikan.

Berita pertama penting dan berguna bagi pengembang, yang kedua seperti lelucon buruk.


Kami membuat mesin IE di browser Chromium Anda sehingga Anda tidak lupa bahwa ini adalah browser Edge

Kernel Linux pada Windows 10


Menambahkan kernel Linux lengkap ke Windows 10 adalah langkah logis yang melanjutkan kerja metodis Microsoft dengan komunitas. Sebelumnya, hanya emulasi kernel Linux yang tersedia di Windows. Bersamaan dengan kernel Linux di dalam sistemnya, perusahaan mengumumkan rilis Windows Terminal - aplikasi Windows baru untuk akses terpusat ke PowerShell, Cmd dan, pada kenyataannya, kernel Linux itu sendiri dalam paket WSL.

Perwakilan perusahaan mengklaim bahwa 4.19 bukan versi terbaru dari kernel untuk subsistem mereka. Di masa depan, WSL akan diperbarui dan mengikuti versi stabil dari kernel Linux, yang juga penting. Selain itu, pengenalan kernel penuh dalam sistem menghilangkan kebutuhan untuk persaingan, yang, secara langsung, akan meningkatkan produktivitas dan kenyamanan pengembang yang bekerja dari bawah Windows. Selain itu, WSL didasarkan pada kode sumber terbuka, yaitu, setiap pengembang akan dapat membuat perubahan yang diperlukan untuk subsistem dan membuat garpu kernel.

Ini bukan langkah besar pertama menuju komunitas pembangunan. Sebelumnya, perusahaan mengintegrasikan Bash ke dalam sistemnya , yang dapat disebut langkah yang sangat serius bagi Microsoft yang sangat konservatif.

Tindakan Microsoft mengejutkan, tetapi tidak lagi mengejutkan: di bawah kendali Satya Nadella, perusahaan itu benar-benar masuk ke komunitas open source dan semakin berfokus pada bekerja secara langsung dengan pengembang dan segmen perusahaan, sebagaimana dibuktikan oleh tindakan Microsoft selama 3-4 tahun terakhir.

Monster Microsoft'a atau mengapa di Internet Explorer inti Chromium Edge


Semua orang yang bergidik mengingat Internet Explorer, dan perancang tata letak yang lama juga menangis pada saat yang sama. Ketika Microsoft mengumumkan Spartan dan ditinggalkannya IE, di seluruh dunia mulai membuka sampanye, tetapi kenyataannya ternyata lebih buruk dari yang kami harapkan. Masih ada solusi, portal, dan situs yang dibuat untuk bekerja secara eksklusif di bawah IE versi lama. Sebagian besar, solusi ini hidup dalam solusi perusahaan tertutup. Microsoft pernah memiliki pilihan: untuk memaksa perusahaan di seluruh dunia untuk memperbarui produk dan solusi internal mereka karena pengabaian lengkap perusahaan terhadap IE, atau untuk mulai berjuang "ke arah yang salah". Karena penolakan total terhadap IE terancam dengan miliaran kerugian untuk Microsoft dan kerusakan reputasi, raksasa itu harus menyeret kompatibilitas browsernya dari tahun ke tahun dengan halaman-halaman yang diadaptasi untuk bekerja di Internet Explorer.

Bahkan setelah dengan keras meninggalkan mesin EdgeHTML dan mentransfer pengembangan "browser default" untuk Windows 10 ke mesin Chromium, Microsoft tidak dapat melepaskan diri dari warisan mengerikannya di hadapan Internet Explorer.

Oleh karena itu, terutama untuk Enterprise-builds dari Windows 10, perusahaan akan mengirimkan versi Chromium Edge dengan "kejutan" - mesin IE bawaan di Edge itu sendiri. Solusi aneh semacam itu akan memastikan kompatibilitas browser baru dengan solusi korporat lama yang dulunya dipakukan dengan ganas ke Internet Explorer.

Saat Anda membuka halaman apa pun yang diadaptasi untuk Internet Explorer di Chromium Edge yang baru, browser akan secara otomatis mengaktifkan "mode kompatibilitas IE" dan menghubungkan tampilan yang sesuai dengan Internet Explorer 11. Anda dapat mengetahui apakah halaman tersebut diluncurkan dalam mode kompatibilitas dengan ikon IE khusus di sebelahnya dengan bilah alamat di halaman browser.



Microsoft sendiri menegaskan bahwa perusahaan yang menggunakan portal web usang harus beralih ke Chromium Edge baru dan sepenuhnya meninggalkan praktik menggunakan IE + browser lain. Tentu saja, setiap spesialis teknis akan mengatakan bahwa langkah seperti itu untuk menanamkan mesin yang sudah ketinggalan zaman dalam produk baru itu berlebihan, tetapi, pada kenyataannya, sekarang Microsoft berusaha menyelamatkan muka dan melakukan sedikit kegiatan amal. Pertanyaannya adalah apakah raksasa itu akan pernah bisa mengatakan tidak kepada bisnis dan akhirnya menembak keledai. Itu belum berhasil.

Konferensi dimulai kemarin, 6 Mei, dan akan berlangsung hingga tanggal 8, jadi Microsoft memiliki setidaknya satu hari untuk lebih mengejutkan kita.
