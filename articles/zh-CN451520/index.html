VDI：便宜又开朗



下午好，亲爱的哈布罗维派，朋友和熟人。 作为序言，我想谈谈一个有趣的项目的实施，或者正像现在所说的那样，谈论一个有关VDI基础结构部署的有趣案例。 似乎有很多关于VDI的文章，它们是循序渐进的，是直接竞争对手的比较，也是循序渐进的，还有竞争解决方案的比较。 似乎可以提供一些新的东西？

而许多文章所没有的新内容则是对实施的经济效果的描述，所选解决方案的拥有成本的计算，甚至更有趣的是与类似解决方案的拥有成本的比较。 在这种情况下，根据文章标题，关键字很便宜 ：这意味着什么？ 在今年年初，我的一位同事，一位朋友和一位朋友，要以最少的“窗口”来实施VDI，即免费的虚拟机管理程序，Linux桌面，免费的数据库，以及通过我们“心爱的”微软最大限度地降低成本的其他方法。

为何使用“最小窗口”？ 在这里，我脱离了进一步的叙述，并描述了为什么我对这个特定主题的公开感兴趣。 我的一个朋友（我在项目部署过程中曾提供过帮助）在一家拥有500多名员工的普通公司中工作，并非所有软件都是合法的，但是正在努力对其进行优化，大多数前端信息系统都适用于WEB，直到整天晴朗，我一直心情愉快分配给公司的Microsoft“个人经理” 收藏家没有来，也开始不，不报价，不问，而是要求紧急强行使一切合法化，从而得出基于开放源和媒体使用的解决方案的许多结论。 发布。 公司似乎不介意，但是这种紧要关头和紧迫感，逼近威胁，刺激了长期的进口替代计划， 以最大程度地减少MS产品的使用和对OpenSource的最大关注。 外部人员可能不会特别相信软件巨人的代表所描述的情况，但是一次，在Microsoft员工亲自陪同下，在指定的压力下，类似的情况会一一重复出现。

另一方面，这是审查IT部门的开发策略以使付费软件产品的使用多样化的另一个触发因素。 再次，OpenSource解决方案在企业中的渗透趋势正在增强，IT-OS 0219会议上对此主题进行了讨论，并且下面的幻灯片完全证实了这一点。


因此，在上述组织中，设定了目标：在最大程度地引入和使用OpenSource解决方案的同时，强制完成MS产品的许可。 为了用户访问，决定从“ terminalok”和Windows VDI完全切换到Linux VDI。 之所以选择Citrix VDI，是因为管理人员少，分支机构众多，扩展部署和已购买产品的部署简单。

在本文的第一部分中，我想着重于计算Linux VDI基础架构的TCO所有权，并在XenDesktop和老式XenServer的老百姓中选择基于Citrix Virtual Apps and Desktops解决方案的解决方案，尽管现在它被称为Citrix Hypervisor（哦，这次更名，更改了几乎所有产品的名称）产品系列），以及相应的Linux台式机。 每个人似乎都清楚地知道，VDI / APP的协同作用是将Vmware用作虚拟机管理程序，将Citrix用作应用程序交付控制器，以及将Microsoft用作来宾OS。 但是，如果需要相同的技术却又以最低的成本怎么办？ 好吧，让我们计算一下：

在开始时，我将讨论DO的配置，然后再转向新平台是“值得的”。
为了使图片简单和完整，让我们仅考虑软件部分，因为设备已经可用并且可以执行其任务。

因此，一开始它就是……拥有出色的EMC存储，HP c7000的Blade篮筐和7个G8服务器在虚拟化VDI中的作用。 服务器上安装了具有Hyper-V角色的Windows Server 2012R2，并使用了SCVMM。 部署了基于XenDesktop 7.18的收购的VDI平台，并部署了多个终端场。 了解部署情况和许可大量软件的必要性之后，让我们比较一下部署Linux VDI和完整的Microsoft统包解决方案的成本。 决定逐步实施翻译，在最初阶段，公司分支机构受到影响，第二阶段是GO中剩余工作的转移。



在终端场中，1C主要在标准办公室套件，邮件，文件和Internet的VDI桌面上工作（它们的主要功能是专门读取和打印）。

了解所需软件的列表后，让我们计算Microsoft解决方案的总拥有成本。

Windows Server：

根据Microsoft许可要求，必须满足以下条件：

  1. 您需要许可服务器中的所有物理核心。
  2. 2核服务器许可证的最小软件包为8个。 （或一份16核许可证）。
  3. 2核处理器许可证的最小软件包为4个。 （如果处理器数量大于两个，则启用此规则）。
  4. 标准许可证包授予在一台服务器上使用Windows Server的一个物理实例和两个虚拟实例的权利。
  5. 数据中心许可证包提供了在单个服务器上使用Windows Server的一个物理实例和任意数量的虚拟实例的权利。

事实证明，如果您需要在服务器上安装Windows Server和Windows工作站的13个以上虚拟副本，则购买我们将考虑的Datacenter版本在经济上是可行的。

Windows 10 VDI：

根据Microsoft的许可政策，必须从具有有效Microsoft VDA（虚拟桌面访问）订阅的设备上执行从客户端操作系统对虚拟桌面的访问，但“软件保证”所涵盖的PC除外。 就我们而言，我们实际上需要每年获取和续订300个DVA许可证。

“我正在从VMware / Citrix /其他供应商那里购买VDI软件。

我仍然需要Windows VDA吗？ 是的 如果要从任何非SA设备（包括瘦客户端，iPad等）访问Windows客户端OS作为数据中心中的来宾操作系统，则无论选择哪种VDI软件供应商，Windows VDA都是合适的许可工具。 唯一不需要Windows VDA的情况是，如果使用软件保障范围内的PC作为访问设备，因为虚拟桌面访问权限是SA的好处，因此得到了好处。”

SCVMM：

虚拟基础架构管理系统系统中心虚拟机管理器包含在Microsoft System Center软件包中，没有作为单独的产品提供。 关于这种方法，没有必要先讨论我们拥有的东西，然后再拥有我们的东西。

考虑到许可要求：

  1. “您需要许可服务器中的所有物理核心。
  2. 2核服务器许可证的最小软件包为8个。 （或一份16核许可证）。
  3. 2核处理器许可证的最小软件包为4个。 （如果处理器数量大于两个，则启用此规则）。
  4. 标准许可证包提供了在一台服务器上管理一个物理和两个虚拟操作系统的权利。
  5. Datacenter许可包提供了在一台服务器上管理一个物理和任意数量的虚拟OS的权利。”



标明的价格是价格，当然，如果有这种数量，可以有折扣，但是与Cisco或Lenovo的GLP价格不同，忘记50％或70％的折扣。 从MS的经验来看，很难看到超过5％。 事实证明，仅第一年，拥有成本将超过500万卢布，三年之内拥有成本将约为900万卢布。 这个数字不小，但对于一家中型公司，我会说这是巨大的。 事实证明，从经济角度来看，这一决定似乎并不那么简单。

展望未来，我要说的是，在计算了该项目的决策后，管理层在批准该计划时做出了肯定的决定。
