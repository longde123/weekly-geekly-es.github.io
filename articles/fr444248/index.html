Test sans tête dans un navigateur. Avantages et inconvénients

Bonjour encore. Cette publication est dédiée au lancement du cours Automation of Web Testing , qui commence cette semaine. Bonne lecture!

Qu'est-ce qu'un test sans tête?

Le test sans tête est un lancement dans le navigateur d'un test pour les performances de tous les composants de l'interface utilisateur sans afficher cette interface elle-même. Un tel test lance des scripts dans le navigateur, mais ne se lance pas et n'affiche pas l'interface utilisateur par elle-même.



Pourquoi pourriez-vous avoir besoin d'utiliser des navigateurs sans tête? Cette approche présente de nombreux avantages et inconvénients. L'utilisation d'un navigateur sans tête peut ne pas être très utile pour surfer sur le net, mais elle est indispensable pour automatiser les tâches et les tests.

Pourquoi utiliser un navigateur sans tête?

Suivre l'argent est un cliché, cependant, je pense toujours que c'est un indicateur clé de l'émergence des tendances, et donc à quoi vous devez absolument faire attention. Par exemple, Sauce Labs vient de lancer un nouveau service appelé Sauce Headless, une solution basée sur le cloud pour tester sans tête vos solutions.

Je sais que les gens de Sause sont des gens intelligents. Ils ne développeront rien pour rien. Ils doivent obtenir des commentaires des utilisateurs afin de comprendre que toute fonctionnalité est nécessaire.

Je suis sûr qu'ils ne seront pas seuls dans leur quête pour développer des tests sans tête.
Alors que nous avançons dans le cycle de vie du développement logiciel, les testeurs doivent fournir de plus en plus rapidement des commentaires aux développeurs. Une façon d'y parvenir consiste à effectuer des vérifications rapides avec un navigateur sans tête.

Automatisation du développement logiciel

Je suis pour l'automatisation de toutes les mains, mais l'automatisation ne concerne pas seulement les tests.
Il s'agit d'une histoire sur tous les aspects que vous pouvez automatiser pour gagner du temps ou des efforts dans le cycle de vie du développement logiciel, du contrôle qualité ou des tests, du DevOps ou de l'installation. Les navigateurs sans tête sont l'un des puissants outils d'automatisation.

Les navigateurs sans tête sont plus rapides que les navigateurs classiques

L'une des différences les plus importantes entre les navigateurs sans tête et les navigateurs classiques est leur vitesse. Le fait est que lorsque vous chargez une page dans un navigateur sans interface graphique, elle n'a pas besoin de passer du temps à charger CSS, JavaScript et à rendre HTML.

Je ne dirais pas que la différence sera énorme, mais dans certains cas, vous pouvez suivre l'augmentation de la vitesse de moitié ou même de 15. Ainsi, si les performances sont importantes pour vous, un navigateur sans tête sera un bon choix.

Gratter avec un navigateur sans tête

Un autre avantage des navigateurs sans tête est leur capacité à collecter des données à partir de sites Web . Pour ce faire, vous n'avez pas besoin de lancer manuellement le navigateur. Vous pouvez utiliser la technologie sans tête pour simplement supprimer le code HTML. Et le rendu de l'ensemble du navigateur est complètement facultatif.

Supposons que votre cas nécessite des statistiques sportives ou des comparaisons de prix sur différents sites.

Étant donné que vous n'avez besoin que de données, il n'est pas logique de lancer une essence de navigateur à part entière et de charger inutilement la machine, car moins il y a de charge, plus vous obtenez le résultat rapidement. Il ne s'agit pas nécessairement de tests, il vous suffit d'utiliser le bon outil pour travailler. En général, je pense que les testeurs sous-estiment en vain les navigateurs sans tête dans leur domaine.

Ainsi, si vous avez besoin d'obtenir des données d'une page Web, vous n'avez pas besoin de l'ouvrir et de donner une charge supplémentaire en ouvrant un navigateur, toutes les fonctionnalités nécessaires sont capables de fournir un navigateur sans tête.

Gagnez du temps pour les développeurs

Je sais que de nombreux développeurs utilisent des navigateurs sans tête pour tester leurs sites et leurs applications. La possibilité de l'utiliser à partir de la ligne de commande sans mettre à jour ou lancer manuellement le navigateur leur fait gagner beaucoup de temps et d'efforts. Par exemple, Rob Frizzel , l'auteur de PhantomJS CookBook dans une interview avec TestTalks a expliqué pourquoi ses développeurs utilisent le navigateur sans tête PhantomJS:
«Bien que PhantomJS lui-même ne soit pas un cadre de test, c'est un« rayon de lumière utile dans une mine de charbon »qui donne confiance; si tous les tests sont réussis, vous deviendrez beaucoup plus confiant que votre code est vraiment bon. »
Surveillance des performances avec des scripts de navigateur sans tête

Une autre option pour utiliser un navigateur sans tête est de surveiller les performances des applications réseau.

Certains les utilisent même pour automatiser le rendu et prendre des captures d'écran de l'écran d'un site Web afin de tester des mises en page dans un style automatisé.

Je pense que c'est l'une des raisons pour lesquelles Google a développé une nouvelle API Chrome sans tête appelée Puppeteer . Il vise principalement à aider les développeurs.

Idées clés Test du navigateur sans tête

En plus de l'idée précédente, il existe également d'autres options d'utilisation des navigateurs sans tête, dont je vais vous parler maintenant:

  1. Exécutez des tests sur une machine sans moniteur;
  2. Réglage des données;
  3. Test SSL;
  4. Simulation de plusieurs navigateurs sur une seule machine;
  5. Exécution de tests sur un système sans tête, tel que Linux sans interface graphique;
  6. Réception et visualisation de fichiers PDF;
  7. Test de balisage, car les navigateurs sans tête peuvent afficher le HTML et le CSS en tant que navigateurs à part entière, vous pouvez donc les tester avec des styles.

Cas où vous ne souhaitez pas utiliser un navigateur sans tête

Bien sûr, il peut arriver que vous souhaitiez utiliser un navigateur complet au lieu d'un navigateur sans tête. Par exemple:

  1. Vous devez simuler de vrais utilisateurs;
  2. Vous devez voir comment fonctionne le test;
  3. Si vous devez déboguer, cela peut être délicat avec un navigateur sans tête.

Navigateurs sans tête populaires

  1. Google Puppeteer - Headless Browser Puppeteer est une bibliothèque de nœuds. Il fournit une API de haut niveau pour contrôler le chrome ou le chrome sans tête à l'aide du protocole DevTools. Il peut également être configuré pour utiliser du chrome ou du chrome à part entière (sans tête).
  2. Google Chrome à partir de la version 59;
  3. Firefox versions 55 et 56;
  4. PhantomJS - kit web sans tête sur l'API JavaScript. Il prend en charge rapidement et nativement diverses normes Web: traitement d'arborescence DOM, sélecteurs CSS, format JSON, Canvas et SVG. * Maintenant, il n'est pas pris en charge. Pour cette raison, vous ne voudrez peut-être pas l'utiliser.
  5. HtmlUnit est un navigateur sans interface graphique pour le code Java. Il modélise des documents HTML et fournit une API qui vous permet d'ouvrir des pages, de remplir des formulaires, de cliquer sur des liens, etc. Ainsi que dans votre navigateur "normal".
  6. Splinter - Splinter est un navigateur sans tête orienté Python. Il est open source et est utilisé pour tester des applications Web à l'aide de Python. Par exemple, vous pouvez l'utiliser pour automatiser les actions du navigateur, telles que la visite de pages Web et l'interaction avec leurs éléments.
  7. jBrowserDriver - Un pilote de navigateur Web intégré et programmable compatible avec la spécification Selenium WebDriver - sans tête, basé sur WebKit, en Java pur.

Quand devez-vous utiliser un navigateur sans tête pour les tests?

La réponse à cette question dépend de l'objectif de votre test.

D'une part, les gens disent souvent: «N'utilisez jamais de navigateurs sans tête. Un vrai utilisateur ne l'utilisera jamais, alors pourquoi devriez-vous? " Bien que, d'un autre côté, nous entendons: "Vous devez toujours utiliser un navigateur sans tête, car il est plus rapide et plus rapide en matière de tests, cela signifie mieux"

Comme nous le savons, chaque position aura des partisans et de l'opposition, mais ici il ne s'agit que de choisir les bons outils pour résoudre certains problèmes.

N'oubliez pas que vous devez utiliser l'outil optimal pour le travail et toujours vous demander comment cela affectera l'utilisateur final, vous devez vous demander quel est le but du test chaque fois que vous faites un choix entre ces deux points de vue.

Selon la tradition établie, nous attendons vos commentaires, et rappelons également que dans quelques heures il y aura une journée portes ouvertes , où vous pourrez en savoir plus sur notre cours.
