6. Periksa Titik Memulai R80.20. Mulai di SmartConsole



Selamat datang di pelajaran ke-6. Hari ini kita akhirnya akan bekerja dengan GUI Titik Periksa yang terkenal. Apa yang kebanyakan orang suka dengan Check Point, dan sebagian membenci. Jika Anda ingat pelajaran terakhir, maka saya katakan bahwa pengaturan keamanan dapat dikontrol baik melalui SmartConsole atau melalui API khusus yang hanya muncul di versi R80. Dalam tutorial ini, kita akan mulai dengan SmartConsole . Maaf, tetapi topik API meninggalkan kursus kami.

Apakah


Saya ingin mencatat berapa banyak antarmuka R80 telah berubah dibandingkan dengan rilis sebelumnya. Untuk melakukan ini cukup sederhana, karena perbedaannya hanya kolosal. Di sini Anda melihat antarmuka R77.30:



Omong-omong, SmartDashboard disebut, bukan SmartConsole. Dan sangat mirip bahkan dengan rilis tertua, seperti R65 dan bahkan lebih muda. Yaitu antarmuka manajemen kebijakan keamanan tidak banyak berubah, baik secara eksternal maupun logis, selama bertahun-tahun. Namun semuanya telah berubah secara dramatis dengan munculnya keluarga R80.

Telah menjadi




Perbedaan visual pada wajah. Tampilannya menjadi lebih modern, indah, tetapi ini bukan yang terpenting. R80 secara mendasar telah mengubah logika konsol. Sebagai contoh, sangat sulit bagi saya untuk beralih ke konsol baru. Namun, setelah bekerja dengannya, saya menyadari bahwa dia bahkan bukan apa-apa) Menurut pendapat subjektif saya, bekerja di konsol R80 jauh lebih nyaman daripada di R77.30. Tapi, ini lebih merupakan kebiasaan. Banyak yang masih meludah di antarmuka baru.
Saya tidak melihat alasan untuk berbicara tentang konsol dalam gambar, lebih baik mari kita lihat itu "hidup". Di bawah ini Anda akan menemukan tutorial video dengan peragaan antarmuka. Untuk satu, kami akan menghubungkan gateway ke server manajemen kami.
