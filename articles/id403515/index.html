Beberapa file sistem Windows tersumbat dengan sampah metadata Adobe


Berbagi sampah Adobe XMP dalam file sistem Explorer.exe dan ApplicationFrame.dll

Volume file sistem Windows telah berkembang dari 250-350 MB di Windows 98 menjadi 700-800 MB di Windows XP, dan kemudian menjadi 10-15 GB di Windows 7 dan terus bertambah. Jika Anda bertanya-tanya mengapa distribusi Windows 10 memakan banyak ruang disk dan sistem mengkonsumsi begitu banyak RAM (dibandingkan dengan Linux), maka ketahuilah bahwa sebagian kecil, kesalahan yang sangat kecil karena ini terletak pada Adobe. Adobe yang mengembangkan dan mempromosikan teknologi Extensible Metadata Platform (XMP) , yang mengembang ukuran file (seperti PNG), dan kemudian file PNG yang membengkak ini dimasukkan ke dalam binari sistem Windows.

Bagian XMP kecil dalam volume total distribusi Windows 10 (hanya 5341271 byte tambahan ), meskipun sebagian besar file individual ditempati oleh metadata tambahan. Dan ini bukan beberapa tanda tangan digital dari program Sound Forge yang retak, dengan bantuan sistem file WAV untuk Windows 95 dibuat , tetapi metadata PNG yang sama sekali tidak perlu dan tidak informatif.

Apa itu Adobe XMP? Sederhananya, ini adalah platform metadata yang memungkinkan Anda menjejalkan berbagai informasi ke dalam PNG, GIF, JPEG, PSD, TIFF dan banyak lainnya. Platform itu sendiri memungkinkan berbagai program untuk bertukar data melalui XMP. Artinya, satu program dapat menyimpan di sana, misalnya, profil warna, dan program lain akan mengekstraknya. Berbagai tajuk, pengidentifikasi, koordinat GPS, informasi warna, dan lainnya tertanam dalam wadah ini.

Biasanya, XMP diwakili oleh konstruksi dari subset model RDF, yang pada gilirannya biasanya diserialisasi sebagai XML. Sayangnya, banyak pengguna sendiri tanpa mengetahui ukuran file gambar mengembang dan mengisinya dengan sampah Adobe XMP saat menyimpan file. Hanya saja dalam beberapa program ini adalah pengaturan penyimpanan default. Misalnya, ini adalah pengaturan default di Adobe Photoshop.

Dalam format XML, data ini membutuhkan banyak ruang. Selain itu, mereka menggunakan sumber daya CPU saat mem-parsing XML selama pemrosesan file.



Pemrogram dan pengguna Windows 10 Rafael Rivera menjadi tertarik: berapa banyak Adobe XMP sebenarnya dalam file yang berbeda? Untuk menjawab pertanyaan ini, ia menulis utilitas eoraptor , yang kode sumbernya diterbitkan di Github.

Rafael mengatakan dia sangat terkejut ketika program tersebut menemukan sejumlah besar sampah XMP bahkan dalam file sistem Windows 10! Dia tidak menulis program untuk menganalisis gagasan Microsoft ini, tetapi dengan cepat mempelajari file grafik. Tetapi ternyata pengembang Microsoft tidak berdosa - dan menanamkan informasi yang tidak perlu bahkan dalam file sistem!

Misalnya, System Explorer (Windows Explorer), yang dimuat ke dalam memori pada awal sistem operasi, sebanyak 20% terdiri dari sampah. Atau ambil file sistem lain ApplicationFrame.dll, yang juga dimuat ke dalam memori di awal sistem - ini bertanggung jawab atas header dan bingkai jendela aplikasi. Di dalamnya, konten XML yang tidak perlu menempati 41% dari total ukuran perpustakaan!

Rafael Rivera menulis bahwa konten XMP mau tidak mau menembus ke berbagai komponen sistem yang berkaitan dengan grafis, pemirsa gambar dan komponen terkait.

Masalah ini tampaknya sepele, tetapi sebenarnya menunjukkan beberapa non-optimalisasi sistem operasi. Komponen tambahan dalam file sistem memiliki sedikit efek pada semua aspek kinerja OS: pada waktu boot, pada ukuran gambar dan penggunaan ruang disk setelah menginstal sistem, pada jumlah pembaruan, pada validasi ketika menjalankan file dengan tanda tangan digital Authenticode, dll. Yang khas, Microsoft memiliki departemen yang mengoptimalkan kinerja Windows. Secara teori, ini harus di bidang kompetensinya. Ini adalah hal-hal yang harus mereka lacak.

Microsoft dapat dengan cepat dan mudah memperbaiki situasi dengan hanya menekan semua gambar PNG yang merupakan bagian dari file sistem. Untungnya, ada banyak utilitas yang mengoptimalkan PNG tanpa kehilangan kualitas, secara efektif menghapus metadata yang tidak perlu dari sana. Mungkin perusahaan akan melakukan optimasi dan memperbarui file sistem dengan build terdekat.

Dan sekarang kita harus mengakui bahwa upaya mengoptimalkan sistem operasi di Microsoft tidak pada level tertinggi atau bahwa pengembang memiliki beberapa alasan untuk menyimpan metadata XMP dalam file sistem. Mungkin mereka harus mengambil contoh dari kolega mereka dari tim pengembangan browser Microsoft Edge. Mereka sudah mengoptimalkan semua file: mereka membuang kelebihan sampah di PNG menggunakan PNGDistill , dan kemudian mengompresi biner menggunakan algoritma Google ZopFli .

Manajer program untuk Microsoft Internet Explorer Eric Lawrence baru-baru ini berbicara tentang pengoptimalan.

Setiap pemilik situs dapat menghemat lalu lintas hanya dengan memproses file PNG-nya dengan utilitas PNGDistill ( modul Python ).
