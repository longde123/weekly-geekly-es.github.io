Python Creator: Saya lelah, saya akan pergi

Sekarang PEP 572 1 sudah siap, saya ingin itu menjadi PEP terakhir yang harus saya lawan dengan susah payah ketika berhadapan dengan begitu banyak orang yang membenci pendapat saya.


1 Proposal Peningkatan PEP atau Python - proposal untuk ekspansi Python.

PEP 572 akan tetap dalam sejarah tidak hanya sebagai PEP terakhir dari era Guido van Rossum, tetapi juga berkat operator penugasan baru, yang menyebabkan reaksi emosional ambigu pengguna Python.

Saya ingin sepenuhnya mengecualikan diri dari proses pengambilan keputusan. Saya masih akan tersedia untuk beberapa waktu sebagai pengembang senior 2 reguler, dan saya akan terus menjadi mentor 3 , saya bahkan akan memiliki lebih banyak waktu untuk ini. Tetapi pada intinya, saya mengirim diri saya dengan cuti permanen dari posisi diktator seumur hidup murah hati 4 , dan mulai sekarang Anda akan sendirian.


2 Pengembang Python Senior - Python Core Dev, seseorang yang memiliki hak untuk melakukan tanpa review langsung ke master. Menandatangani perjanjian khusus. Diterima oleh proses yang kompleks.

Github menunjukkan 590 kontributor ke master, di mana 67 berkomitmen dan bergabung dengan master untuk bulan lalu.

3 Mentor - untuk mengajar, meminta, membantu pemula untuk memahami segalanya.

4 Diktator seumur hidup yang murah hati - sebuah istilah yang diciptakan secara kebetulan pada hari ketika organisasi Python resmi pertama dibentuk dalam grup surat dan Guido van Rossum tidak mendapatkan jabatan "normal".

Suatu hari, ini semua harus terjadi, pada akhirnya bus 5 yang terkenal ini selalu bersembunyi di tikungan, dan saya tidak menjadi lebih muda. (Saya tidak akan menanggung daftar semua masalah kesehatan saya).


5 Ini merujuk pada istilah "faktor bus" - faktor bus, atau faktor risiko yang terkait dengan fakta bahwa figur kunci suatu perusahaan atau proyek dapat tiba-tiba mati atau kehilangan kapasitas hukumnya. Biasanya menunjukkan kurangnya distribusi dalam pengambilan keputusan dan adanya konsentrasi kekuasaan atau tanggung jawab yang berlebihan di tangan satu orang.

Saya tidak akan menunjuk diri saya sebagai penerus.


Apa yang akan kamu lakukan sekarang? Ciptakan demokrasi? Anarki? Kediktatoran? Federasi?


Saya tidak khawatir tentang pengambilan keputusan harian di bug tracker 6 di Github. Mereka meminta pendapat saya di sana sangat jarang, dan biasanya tidak ada kebutuhan khusus untuk itu. Jadi dengan ini, saya pikir, Anda bisa bekerja seperti biasa.


6 masalah GitHub

Pertanyaan-pertanyaan yang sekarang mungkin yang paling penting:


  • Bagaimana keputusan PEP akan dibuat?
  • Bagaimana orang baru akan diterima sebagai pengembang senior

Mungkin kita bisa mengatur proses ini menjadi PEP, mungkin ini bahkan akan membuat sesuatu seperti konstitusi. Tapi ini masalahnya, saya meninggalkan Anda semua (komite saat ini) untuk menangani hal ini sendiri, tanpa saya.
