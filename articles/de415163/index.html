Warum Netzwerk? Office - einfach und übersichtlich: mobile Geräte, Post. Lager - Dies ist die Abrechnung von Waren für die Zulassung, Datenerfassungsterminals: Registerbewegung, Ankunft und Abreise. Sobald sie den Stoff aus dem Lager in die Produktion bringen, ist eine sofortige Kontrolle erforderlich. Das Lager verlassen - sofort aufnehmen. All dies über ein Netzwerk erfordert Wi-Fi. In einer regulatorischen Produktion müssen alle pharmazeutischen Geräte an ein Netzwerk angeschlossen sein, um Informationen darüber zu übertragen, wie viele Tabletten und mit welcher Zusammensetzung. Dies ist eine Regulatoranforderung.

Ab 2018 tritt in Russland ein Dekret in Kraft, nach dem die Apotheker den Verbrauchern alle Informationen über Arzneimittel zur Verfügung stellen müssen - Authentizität, Haltbarkeit, Produktionsort, Unternehmensdaten, Zusammensetzung, spezifische Charge des Arzneimittels, einen eindeutigen Code für eine separate Packung. Sie können diese Daten herausfinden, indem Sie einfach den Barcode lesen, der sich direkt auf der Verpackung befindet. Durch Lesen des Barcodes kann der Apotheker leicht überprüfen, wie viele Medikamente in der Apotheke verfügbar sind, wie viele verkauft werden usw.

Access Points werden von einem Cluster aus zwei Cisco Wireless LAN Controller 5520-Controllern verwaltet. Access Points werden über das offene CAPWAP-Protokoll (RFC5415) verwaltet. Zugriffspunkten in dieser Architektur werden nur die Funktionen zum Verbinden von drahtlosen Clientgeräten und zum Verschlüsseln der übertragenen Daten auf der Datenverbindungsschicht zugewiesen.



Der Nachteil ist, dass ein Controller-Fehler zum Ausfall des gesamten drahtlosen Netzwerks führt. Aus diesem Grund wird eine Redundanz der Steuerungen durchgeführt. Controller werden in einem Failovercluster zusammengefasst. Bei einem Ausfall des Hauptcontrollers werden die Zugriffspunkte von der Sicherungssteuerung gesteuert, ohne die Benutzersitzung zu unterbrechen.

Der Hauptcontroller verfügt über eine Lizenz zum Verwalten von 300 Zugriffspunkten. Bei einem Ausfall des primären Controllers wird diese Lizenz auf dem Backup-Controller aktiviert, für den keine separaten Lizenzen für den Backup-Controller erworben werden müssen. Mit dem Wachstum des drahtlosen Netzwerks kann die Anzahl der vom Controller-Cluster unterstützten Zugriffspunkte mit Lizenzen auf bis zu 1500 erweitert werden.

Die zentrale Verwaltung und Überwachung von WLAN-Geräten erfolgt mithilfe des Prime Infrastructure-Systems, das auf einem virtuellen Server auf Basis von VMware ESXi installiert ist. Dies dient dazu, Gerätefehlfunktionen im Netzwerk zu erkennen und zu lokalisieren, um statistische Informationen über die Funktionsweise des gesamten Netzwerks oder seiner einzelnen Komponenten zu erhalten.
