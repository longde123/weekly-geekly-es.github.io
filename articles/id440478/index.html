3CX v16 Beta 1 dengan Dukungan Raspberry Pi Dirilis

Kami dengan senang hati mempersembahkan versi berikutnya dari 3CX v16 Beta 1. Rilis terakhir kami sudah dekat! lebih dekat ke rilis final. Sekarang izinkan saya memperkenalkan fitur-fitur baru, peningkatan, dan dukungan untuk platform baru yang telah muncul di Beta 1.

Bekerja pada platform Raspberry Pi



Tidak sering, produk perangkat lunak yang kompleks diadaptasi untuk mendukung arsitektur perangkat keras baru. Ya, itu tidak mudah diimplementasikan untuk sistem waktu nyata, yaitu 3CX.

Tapi kami berhasil! 3CX v16 sekarang berjalan pada perangkat seukuran sabun seperti Raspberry Pi 3B +. Kami telah menyiapkan panduan instalasi untuk v16 pada Raspberry Pi, yang diberikan di bawah ini. Instal dan gunakan secara gratis !

Merekam percakapan di penyimpanan eksternal


3CX v16 Beta 1 mencakup sistem manajemen perekaman percakapan baru yang mendukung pemindahan file rekaman ke penyimpanan eksternal, mencari, memilih, dan secara otomatis menghapus catatan lama. Sekarang Anda dapat secara otomatis, sesuai jadwal, mentransfer catatan percakapan ke Google Drive atau server FTP dengan mengaktifkan opsi Pengarsipan Jarak Jauh. Dengan demikian, Anda membebaskan ruang berharga pada disk (atau Flash-card) dari server PBX. Selain itu, solusi ini mempercepat pemutakhiran atau instalasi ulang 3CX.



Bagian Rekaman Percakapan juga memiliki fitur Bersihkan . Ini menghapus catatan lokal atau eksternal. Ini juga memungkinkan Anda untuk mentransfer catatan lokal ke penyimpanan eksternal (arsip). Sekarang Anda dapat mengurangi biaya PBX perangkat keras dengan memindahkan arsip percakapan ke penyimpanan cloud eksternal yang murah. Fungsi Filter memungkinkan Anda memilih rekaman percakapan untuk periode yang diinginkan atau ke nomor telepon tertentu. Baca lebih lanjut tentang fitur ini di panduan kami.

Panel Operator ATS di jendela terpisah




Klien web dalam 3CX v16 Beta 1 menerima pembaruan penting - tab panel operator PBX sekarang dapat dibuka di jendela terpisah. Manajer dan operator pusat panggilan dapat menggunakan jendela terpisah untuk memantau panggilan dengan cara yang sama seperti yang mereka lakukan pada klien 3CX untuk Windows. Secara umum, semua fungsi klien untuk Windows yang terkait dengan pemantauan panggilan telah ditransfer ke klien web - semuanya berfungsi di browser!

Screencast dalam 3CX WebMeeting tanpa plugin


Saat memperkenalkan sistem baru, muncul pertanyaan penting - kerumitan peluncuran awal. Mencoba menyederhanakan permulaan bekerja dengan konferensi video, kami menggunakan teknologi terbaru dari browser Chrome 72 dan menyelesaikan WebMeeting untuk menolak memasang plug-in tambahan.



Sekarang pengguna dapat menampilkan layar mereka dengan atau tanpa plug-in ClickCall 3CX (versi Chrome yang lebih lama).

Cukup pilih mode tampilan - layar penuh atau jendela aplikasi - dan desktop Anda akan ditampilkan di konferensi! Untuk menggunakan fitur ini, instal versi terbaru Chrome .

Menginstal PBX 3CX pada Raspberry Pi


Dimulai dengan 3CX v16, Anda dapat menginstal PBX pada perangkat Raspberry Pi terbaru, model 3B +. Sistem ini diinstal pada Raspbian Stretch OS, distribusi Linux Debian untuk arsitektur AR Raspberry Pi. Anda dapat mengontrol OS Debian baik melalui antarmuka web 3CX (perintah administrasi dasar) atau melalui antarmuka baris perintah jika Anda adalah pengguna Linux yang berpengalaman.

Harap dicatat bahwa menginstal 3CX pada Raspberry Pi direkomendasikan untuk sistem dengan 8 panggilan simultan dengan 16-24 pengguna. Setiap pengguna dapat mengkonfigurasi hingga 8 indikator BLF rata-rata (mentransmisikan status BLF menempatkan beban pada server PBX).

Persyaratan tambahan:

  • Kartu Memori: 32 + GB Kelas 10 Micro SDHC
  • Sumber Daya: Kompatibel dengan Raspberry Pi, 2.5 A, Micro USB

Mempersiapkan Kartu SD untuk Raspbian OS


Untuk menginstal Raspbian pada kartu micro-SD, unduh dan unzip gambar Raspbian Stretch Lite dari sini .



Bakar gambar ke kartu SD menggunakan utilitas Etcher atau Win32 Disk Imager . Kemudian masukkan kartu dan mulai perangkat.

Setelah memuat OS, masuk dengan kredensial administrator default ( pi pengguna, kata sandi raspberry ). Pi kata sandi default dapat diubah dengan perintah passwd .

Perhatikan bahwa pada Raspbian Linux, gunakan perintah sudo untuk melakukan tugas administratif. Misalnya, untuk memeriksa pembaruan paket di repositori, gunakan perintah sudo apt update .

Tetapkan nama host dan alamat IP perangkat


Untuk mengatur nama host pada sistem Pi, jalankan utilitas konfigurasi Raspbian dengan perintah sudo raspi-config .



Pilih 2. Opsi Jaringan> N1 Nama Host dan OK di kotak dialog peringatan. Masukkan nama host dan reboot perangkat. Anda dapat memeriksa nama host baru dengan perintah hostname.

Untuk mengatur alamat IP statis dari antarmuka eth0, modifikasi file /etc/dhcpcd.conf menggunakan editor nano: sudo nano /etc/dhcpcd.conf



Pergi ke akhir file dan ubah pengaturan: ip_address, router (gateway default) dan domain_name_servers. Gunakan pengaturan yang sesuai untuk jaringan Anda, misalnya: 

 #eth0 static IP configuration interface eth0 static ip_address=192.168.1.111/24 static routers=192.168.1.1 static domain_name_servers=192.168.1.1 8.8.8.8

Simpan pengaturan baru dan sudo reboot perangkat sudo reboot . Anda dapat memeriksa alamat IP baru dengan perintah ifconfig eth0 .

Instal 3CX di Raspbian




Untuk menginstal 3CX pada Raspbian, masuk melalui SSH sebagai pi pengguna, dan kemudian:

Setel kunci 3CX APT untuk otentikasi paket: 

 wget -O- http://downloads-global.3cx.com/downloads/3cxpbx/public.key | sudo apt-key add -

Buat file sumber 3CX APT (rilis dan uji): 

 echo "deb http://downloads-global.3cx.com/downloads/debian stretch main" | sudo tee /etc/apt/sources.list.d/3cxpbx.list echo "deb http://downloads-global.3cx.com/downloads/debian stretch-testing main" | sudo tee /etc/apt/sources.list.d/3cxpbx-testing.list

Perbarui sumber dan paket APT 

 sudo apt update

Periksa ketersediaan paket PBX 3CX 

 sudo apt-cache policy 3cxpbx

Instal rilis stabil PBX 3CX 

 sudo apt -t stable install 3cxpbx

Anda juga dapat menginstal rilis uji. 

 sudo apt -t testing install 3cxpbx



Menginstal paket memerlukan waktu sekitar 5-10 menit, tergantung pada kinerja perangkat.



Setelah instalasi selesai, pilih metode pengaturan 3CX awal: (1) melalui browser web dan (2) melalui baris perintah. Jika opsi pertama dipilih, masukkan URL konfigurasi awal di browser: http: // <device_ip>: 5015 (lihat manual terperinci ).

sudo /usr/sbin/3CXWizard --cleanup Wizard Konfigurasi Awal 3CX dengan perintah sudo /usr/sbin/3CXWizard --cleanup menginisialisasi konfigurasi 3CX. Ini menghapus semua data, termasuk cadangan dan rekaman percakapan! Jika perlu, simpan data ini di perangkat lain.

Menghapus 3CX di Raspbian


Untuk menghapus instalasi 3CX di Raspbian gunakan salah satu dari perintah berikut:

  • sudo apt remove 3cxpbx - File sistem 3CX dihapus, menyimpan data yang berfungsi (catatan percakapan, basis data PBX, file pengguna, dll.). Gunakan jika di masa depan Anda berencana untuk menginstal ulang 3CX.
  • sudo apt remove --purge 3cxpbx - sepenuhnya menghapus sistem 3CX, termasuk data operasional.
