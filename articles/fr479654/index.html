Générateur de vue Django

Comment se faire des amis un projet sur cadre repos avec vue? Vous pouvez créer un projet vue dans le répertoire avec le projet django en utilisant vue-cli. En y accédant et en tapant build yarn, à la sortie, nous obtenons des fichiers index.html dans le répertoire dist et les répertoires css et js. Oui, css et js peuvent être mis en statique et créer un modèle à partir de index.html. En ajoutant les sous-répertoires static / frontend et templates / frontend et les fichiers __init__.py et urls.py au répertoire avec le front (disons que nous l'avons appelé frontend), nous pouvons ajouter frontend à INSTALLED_APPS. Dans urls.py, quelque chose comme path('', TemplateView.as_view(template_name='frontend/index.html'), name='frontend-index'), suffit path('', TemplateView.as_view(template_name='frontend/index.html'), name='frontend-index'),

Pour que les fichiers arrivent là où ils devaient être assemblés, nous corrigeons la construction dans package.json en "vue-cli-service build && (rm -rf static/frontend/ 2>/dev/null || true) && sed 's/=\\//=\\/static\\/frontend\\//g' dist/index.html > templates/frontend/index.html && mv dist static/frontend"

En même temps, nous fixons les chemins avec sed.

Pour tout automatiser, j'ai fait django_vue_generator . Il ajoute les commandes de gestion start_frontend et build_frontend. Mais ce n’est pas tout! Il génère automatiquement des formulaires avec validation sur le client à partir de vos sérialiseurs. Et il crée des composants avec des formulaires pour le projet drf de vue. Et les composants de la liste sont les mêmes pour eux. Avec pagination et filtration.




La vue de liste peut être redéfinie à l'aide des emplacements.
