第三方SSD上的Mac OS X 10.10.4和TRIM更新

在最近的开发者大会上，WWDC 2015宣布了Mac OS X 10.11 El Capitan的下一版本。通过对新产品的更深入了解，苹果公司提供了为第三方SSD强制包含TRIM的功能。万岁！的确，新操作系统的发布将不得不等到秋天。但是让大家高兴的是，trimforce命令已经在当前的10.10.4更新中。

最近，我的灵感来自于此geektimes.ru/post/200362构建了Fusion Drive。该驱动器选择了128 GB的Samsung 850 Pro。添加SSD的效果令人难以置信（70 MB / s-> 260 MB / s）。在Mac OS X 10.10上我唯一没有做的就是我没有使用TRIM启动器强制包含TRIM，因为为此您必须干预系统安全策略。我希望从某种程度上来说，SSD预留空间的功能就足够了。过度配置也是一个有趣的故事。在家用驱动器中，仅为这些目的保留了7％的空间，与服务器模型不同，服务器模型保留了大约三分之一的空间。另外，您可以在家用驱动器上创建容量较小的分区，从而为过度配置腾出了必要的空间。问题是我在命令行上不够强壮，而且我不知道在何处配置什么内容，从而无法通过创建Fusion Drive的算法仅选择磁盘的一部分。

2-3个月后，计算机的响应能力明显下降。我运行测试，看到预期的图片：



SSD已满，未启用TRIM命令，一切都不正常。
后须藤trimforce enable命令和计算机重新启动后，速度也很快恢复了正常：



现在它仍然是了解工作的稳定性。到目前为止，已经工作了几个小时-飞行正常。

在安装更新10.10.4之前，我遇到了一个问题-App Store没有看到更新的存在。我从Apple网站下载了单独的更新文件，但仍然无法安装。
在苹果论坛上找到的问题的解决方案：
-删除Library / update文件夹的内容
-删除/Library/Caches/com.apple.SoftwareUpdate/-
删除/Library/Preferences/com.apple.SoftwareUpdate.plist-
重新启动
-通过App Store定期更新
