Chrome 50 dejó de ser compatible con Windows XP, Vista y versiones anteriores de OS X



Google hizo lo que prometió hace mucho tiempo: dejó de admitir los viejos sistemas operativos Windows y OS X en la nueva versión de su navegador Chrome. La interrupción de la asistencia significa que las actualizaciones para los usuarios de Windows XP, Windows Vista, OS X Snow Leopard, OS X Lion y OS X Mountain Lion no están disponibles. Para ellos, la última versión de Chrome es la 49. Las versiones anteriores del navegador funcionarán en estas versiones del sistema operativo, pero los usuarios ya no recibirán actualizaciones de seguridad. No hay garantía de que en versiones anteriores de la autenticación de Chrome continúe funcionando para los servicios de Google.

El hecho de que en 2016 cese el soporte para estos sistemas operativos, Google anunció en abril de 2015, es decir, hace un año. Al mismo tiempo, la interrupción del soporte para versiones anteriores de Windows y OS X se planeó a fines del año pasado, pero Google decidió extender el soporte para sistemas operativos hasta abril de 2016.

La razón de lo que sucedió es que los fabricantes de estos sistemas operativos ya no los admiten (con algunas excepciones, ya que las actualizaciones de seguridad aún se están lanzando para Windows Vista).

Si desea actualizar, hay dos soluciones al problema. El primero es actualizar el sistema operativo, que es lógico. El segundo es instalar Firefox 45, que recibe actualizaciones para Windows XP y OS X 10.6, u Opera, que también se ejecuta en XP y OS X 10.7.



En cuanto a las actualizaciones en la nueva versión de Chrome, hay bastantes . La principal innovación se refiere a las notificaciones push. Entonces, ahora con la ayuda de estas notificaciones, no solo se realizan notificaciones a los usuarios, también es posible interactuar con ellos a través de cuadros de diálogo. Además, los sitios web ahora podrán acompañar sus mensajes con vibración o sonido.

Ha aparecido otra característica interesante: la precarga declarativa, que le permite descargar contenido masivo para que el usuario no espere una carga completa del contenido de la página durante mucho tiempo. El desarrollador puede proporcionar este tipo de carga de recursos agregando el atributo link rel = "preload" en el código HTML.
