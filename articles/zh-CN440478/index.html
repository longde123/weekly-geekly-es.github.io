发布了具有Raspberry Pi支持的3CX v16 Beta 1

我们很高兴介绍3CX v16 Beta 1的下一个版本。最终版本指日可待！ 更接近最终版本。 现在，让我介绍Beta 1中出现的对新平台的新功能，改进和支持。

在Raspberry Pi平台上工作



复杂的软件产品通常不适合支持新的硬件体系结构。 是的，要实现实时系统3CX并不容易。

但是我们做到了！ 3CX v16现在可以在Raspberry Pi 3B +等肥皂大小的设备上运行。 我们已经为Raspberry Pi上的v16准备了安装指南，如下所示。 免费安装和使用 ！

在外部存储上记录对话


3CX v16 Beta 1包括一个新的对话记录管理系统，该系统支持将记录文件传输到外部存储，搜索，选择和自动删除旧记录。 现在，您可以通过启用“远程存档”选项，根据计划自动将对话记录传输到Google云端硬盘或FTP服务器。 因此，您可以释放PBX服务器磁盘（或闪存卡）上的宝贵空间。 此外，该解决方案还可以加快3CX的升级或重新安装速度。



对话记录部分还具有新的清除功能。 它删除本地或外部记录。 它还允许您将本地记录传输到外部存储（存档）。 现在，您可以通过将会话存档移动到廉价的外部云存储中来降低硬件PBX的成本。 筛选功能使您可以选择所需时间段内或特定电话号码的对话记录。 在我们的指南中阅读有关这些功能的更多信息。

ATS操作员面板在单独的窗口中




3CX v16 Beta 1中的Web客户端进行了重要更新-PBX操作面板的选项卡现在可以在单独的窗口中打开。 呼叫中心管理员和话务员可以使用单独的窗口来监视呼叫，就像在Windows 3CX客户端中一样。 通常，与呼叫监视相关的所有Windows客户端功能都已转移到Web客户端-所有功能都可以在浏览器中使用！

没有插件的3CX WebMeeting中的截屏视频


引入任何新系统时，都会出现一个重要问题-初始启动的复杂性。 为了简化开始视频会议的工作，我们使用了Chrome 72浏览器的最新技术并完成了WebMeeting的安装，以拒绝安装其他插件。



现在，用户可以显示带有或不带有3CX Click2Call插件（Chrome的旧版本）的屏幕。

只需选择显示模式-全屏或应用程序窗口-您的桌面将显示在会议中！ 要使用此功能，请安装最新版本的Chrome 。

在Raspberry Pi上安装3CX PBX


从3CX v16开始，您可以在最新的Raspberry Pi设备（型号3B +）上安装PBX。 该系统安装在Raspbian Stretch OS（Raspberry Pi ARM架构的Debian Linux发行版）上。 您可以通过3CX Web界面（基本管理命令）或命令行界面（如果您是经验丰富的Linux用户）来控制Debian OS。

请注意，建议在Raspberry Pi上安装3CX，以用于同时进行16-24个用户的8次通话的系统。 每个用户平均最多可以配置8个BLF指示器（传输BLF状态会给PBX服务器带来负担）。

附加要求：

  • 记忆卡：32 + GB Class 10 Micro SDHC
  • 电源：兼容Raspberry Pi，2.5 A，Micro USB

为Raspbian OS准备SD卡


要将Raspbian安装在micro-SD卡上，请从此处下载并解压缩Raspbian Stretch Lite映像。



使用Etcher或Win32 Disk Imager实用程序将映像刻录到SD卡。 然后插入卡并启动设备。

加载操作系统后，使用默认的管理员凭据（用户pi ， raspberry密码）登录。 可以使用passwd命令更改默认密码pi。

请注意，在Raspbian Linux上，使用sudo命令执行管理任务。 例如，要检查存储库中的软件包更新，请使用sudo apt update命令。

设置主机名和设备IP地址


要在Pi系统上设置主机名，请使用sudo raspi-config命令运行Raspbian配置实用程序。



选择2.网络选项> N1主机名，然后在警告对话框中选择确定。 输入主机名，然后重新启动设备。 您可以使用hostname命令检查新的主机名。

要设置eth0接口的静态IP地址，请使用nano编辑器修改/etc/dhcpcd.conf文件： sudo nano /etc/dhcpcd.conf



转到文件末尾并更改设置：ip_address，路由器（默认网关）和domain_name_servers。 使用适合您的网络的设置，例如： 

 #eth0 static IP configuration interface eth0 static ip_address=192.168.1.111/24 static routers=192.168.1.1 static domain_name_servers=192.168.1.1 8.8.8.8

保存新设置，然后重新启动sudo reboot设备。 您可以使用ifconfig eth0命令检查新的IP地址。

在Raspbian上安装3CX




要在Raspbian上安装3CX，请以pi用户身份通过​​SSH登录，然后：

设置用于数据包认证的3CX APT密钥： 

 wget -O- http://downloads-global.3cx.com/downloads/3cxpbx/public.key | sudo apt-key add -

创建一个3CX APT源文件（发行和测试）： 

 echo "deb http://downloads-global.3cx.com/downloads/debian stretch main" | sudo tee /etc/apt/sources.list.d/3cxpbx.list echo "deb http://downloads-global.3cx.com/downloads/debian stretch-testing main" | sudo tee /etc/apt/sources.list.d/3cxpbx-testing.list

更新APT源和软件包 

 sudo apt update

检查PBX 3CX软件包的可用性 

 sudo apt-cache policy 3cxpbx

安装PBX 3CX的稳定版本 

 sudo apt -t stable install 3cxpbx

您也可以安装测试版本。 

 sudo apt -t testing install 3cxpbx



根据设备的性能，安装软件包大约需要5-10分钟。



安装完成后，选择初始的3CX设置方法：（1）通过Web浏览器和（2）通过命令行。 如果选择了第一个选项，请在浏览器中输入初始配置URL：http：// <device_ip>：5015（请参阅详细手册 ）。

使用sudo /usr/sbin/3CXWizard --cleanup命令sudo /usr/sbin/3CXWizard --cleanup 3CX初始配置向导sudo /usr/sbin/3CXWizard --cleanup初始化3CX配置。 这会删除所有数据，包括对话的备份和记录！ 如有必要，将此数据保存在另一台设备上。

在Raspbian中删除3CX


要在Raspbian中卸载3CX，请使用以下命令之一：

  • sudo apt remove 3cxpbx删除3CX系统文件，保存工作数据（对话记录，PBX数据库，用户文件等）。 如果将来打算重新安装3CX，请使用。
  • sudo apt remove --purge 3cxpbx完全删除3CX系统，包括操作数据。
