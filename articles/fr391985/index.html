Non seulement Apple est sous le coup du pistolet. Le gouvernement oblige d'autres développeurs à afficher le code source



Il s'avère que les autorités américaines ont tenté à plusieurs reprises d'obtenir le code source de nombreux programmes. Maintenant, ils essaient d'obtenir le code source iOS d'Apple, mais les mêmes exigences ont été avancées par rapport à d'autres sociétés, écrit ZDNet en référence à une source anonyme .

En ayant le code source à la disposition exclusive, il est beaucoup plus facile pour les services spéciaux de rechercher les vulnérabilités dans les programmes. Ces vulnérabilités sont ensuite exploitées à des fins de renseignement et pour protéger la sécurité nationale.

Selon le même schéma, basé sur le précédent Lavabit, Apple a désormais besoin de la source.

ZDNet a demandé des commentaires à une douzaine de grandes sociétés informatiques du Fortune 500, mais aucune d'entre elles n'a accepté de répondre directement à la question de savoir si elles avaient reçu des demandes du tribunal secret du FISC avec une demande de publication du code source ou si elles n'avaient pas reçu de telles demandes. Du silence des sociétés informatiques, certaines conclusions peuvent être tirées .

Cisco a déclaré qu'il "n'avait jamais et ne partagerait jamais de code source avec ses clients, en particulier les gouvernements". IBM a cité une déclaration similaire de 2014 .

Microsoft, Juniper Networks et Seagate ont officiellement refusé de répondre. Aucune réponse n'est venue de Dell, EMC, Lenovo, Micron, Oracle, Texas Instruments ou Western Digital.

Notez que certaines de ces sociétés sont mentionnées dans les signets NSA de 50 pages et le catalogue de portes dérobées , qu'Edward Snowden a déclassifié en 2013.

Dans le cas de San Bernardino, Apple a déclaré qu'il ne fournissait à aucun gouvernement un code source iOS exclusif, bien que les agences gouvernementales de divers pays, y compris les États-Unis, auditent régulièrement les nouvelles versions d'iPhone - Apple leur envoie des échantillons pour audit.

Cependant, si le code source iOS avait été fourni à des services spéciaux à la demande du FISC, même la haute direction d'Apple ne le savait peut-être pas, car le tribunal fonctionne dans une atmosphère de strict secret. Il gère 99% de toutes les demandes de renseignement et de surveillance. Par exemple, ces dernières années, il a approuvé la collecte de métadonnées pour tous les abonnés mobiles américains, l' écoute électronique des canaux de communication dans d'autres pays , ainsi que des opérations de piratage ciblées de la NSA .
