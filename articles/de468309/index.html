Einführung in die Cascadia-Code-Schriftart

Cascadia Code ist endlich da! Sie können es direkt von der Release-Seite in unserem Repository auf GitHub installieren oder beim nächsten Windows Terminal-Update automatisch erhalten.



Warten Sie, wie lautet der Cascadia-Code?


Cascadia Code wurde im Mai dieses Jahres auf der Microsoft Build-Veranstaltung vorgestellt. Dies ist die neueste Monospace-Schriftart von Microsoft, die eine neue Befehlszeilenerfahrung und Code-Editoren bietet. Cascadia Code wurde Hand in Hand mit der neuen Windows Terminal App entwickelt. Diese Schriftart wird für die Verwendung mit Terminalanwendungen und Texteditoren wie Visual Studio und Visual Studio Code empfohlen.

Software-Ligaturen


Cascadia Code unterstützt Software-Ligaturen! Software-Ligaturen sind beim Schreiben von Code am nützlichsten, da sie durch Kombinieren von Zeichen neue Glyphen erstellen. Dies hilft, den Code für manche Menschen lesbar und „freundlich“ zu machen.



Hinweis: Wenn Sie Visual Studio Code verwenden, müssen Sie zur Verwendung von Ligaturen die Schriftligaturen in den Einstellungen aktivieren.



Haben wir erwähnt, dass es Open Source ist?


Ja, du hast richtig gehört! Cascadia Code ist auf GitHub unter der SIL Open Font License verfügbar, sodass Sie auf Wunsch einen Beitrag leisten können!

Hinweis: Das Repository verfügt derzeit über keine ordnungsgemäße Testinfrastruktur. Daher müssen alle Pull-Anforderungen manuell getestet und von Microsoft genehmigt werden. Dies verlangsamt die Koordination von Pull-Anfragen. Bitte haben Sie Verständnis.

Warum heißt es Cascadia Code?


Der Name Cascadia Code stammt aus dem Windows Terminal-Projekt. Vor der Veröffentlichung lautete der Codename für Windows Terminal Cascadia. Tatsächlich verwenden einige der Quelldateien in Windows Terminal immer noch diesen Namen! Uns hat die Idee gefallen, die Schriftart nach dem Codenamen des Projekts zu benennen. Dies ist eine solche Hommage an Windows Terminal.

Wir haben am Ende des Schriftnamens „Code“ hinzugefügt, um anzuzeigen, dass diese Schriftart hauptsächlich für die Programmierung bestimmt ist. Insbesondere hilft es festzustellen, ob es Software-Ligaturen enthält.

Um sicherzustellen, dass Cascadia Code die richtige Wahl für den Schriftnamen ist, haben wir zusammen mit anderen Optionen für die von uns berücksichtigten Namen eine Umfrage auf Twitter durchgeführt. Wir sind allen sehr dankbar für die Teilnahme und haben uns sehr darüber gefreut, dass Cascadia Code der Gewinner war.



Wer ist der Designer von Cascadia Code?


Vielen Dank an Aaron Bell, Designer von Cascadia Code. Wir hatten das Glück, im vergangenen Jahr mit ihm zusammenzuarbeiten, und er hat wirklich gute Arbeit geleistet. Aaron Bell, ein Schriftdesigner und Inhaber von Saja Typeworks , ist seit vielen Jahren bei Microsoft. Zum Beispiel war er 2015 der Designer von Selawik ! Sie können seine Aktivitäten auf Twitter ( @aaronbell ) und in unserem Repository auf GitHub verfolgen . Wir planen, weiter daran zu arbeiten, um den Cascadia-Code zu verbessern und ihn zu einer großartigen Schriftart für alle zu machen!

Was wird als nächstes mit Cascadia Code passieren?


Die Cascadia Code-Schriftversion 1909.16 ist derzeit auf GitHub verfügbar. Es folgt der Syntax von Windows-Versionen und erhält Updates zu neuen und verbesserten Glyphen.
