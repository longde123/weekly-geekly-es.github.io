Administrar certificados SSL / TLS en las nubes y contenedores, no trabajo humano


De la presentación de Venafi: cómo la instalación manual de certificados dificulta la integración y el despliegue continuo de aplicaciones

Los servicios y contenedores en la nube se han convertido en el estándar de facto para implementar aplicaciones web. Sin embargo, la integración de los certificados SSL / TLS en el entorno DevOps sigue siendo demasiado complicada y lenta. Muchas tareas aún se realizan manualmente, y esta es una carga muy grande en los devops. En un entorno virtual con contenedores, la cantidad de máquinas en la red aumenta drásticamente, y aún se necesita la protección de las conexiones máquina-máquina y las comunicaciones entre ellas. Si en dicho entorno la emisión de certificados y prácticas de gestión está mal establecida, la falta de autenticación confiable de cada máquina aumenta la superficie de ataque.

Si todo se hace manualmente, los desarrolladores suelen dar prioridad a la velocidad y la simplicidad, en lugar de la seguridad . A veces, en aras de la velocidad, eligen opciones más simples: crear su propia autoridad de certificación (CA) con certificados autofirmados, algoritmos de cifrado débiles, importar certificados raíz no confiables, protección inadecuada de claves secretas para CA raíz e intermedias. Y a veces los desarrolladores no usan SSL / TLS para cifrar las comunicaciones entre máquinas y contenedores.

Para resolver este problema, aparecieron varios servicios nuevos en el mercado que se integran directamente en el ciclo continuo de integración / entrega (CI / CD) y automatizan el proceso.

Estos servicios ofrecen una seguridad mejorada y una mayor productividad de desarrollo, así como también el cumplimiento de los estándares regulatorios de seguridad como PCI-DSS, NIST e HIPAA. Y el soporte requiere solo unas pocas líneas de código. Venafi, uno de estos servicios desde abril de 2017, es especialista en soluciones de seguridad de la información.


Venafi Cloud Place en CI / CD Conveyor

Venafi Cloud for DevOps es un servicio en la nube integrado que integra convenientemente la infraestructura de claves criptográficas y certificados digitales en las populares plataformas corporativas DevOps. La compañía anunció recientemente la integración de Venafi Cloud con la infraestructura de clave pública GlobalSign PKI.

Venafi Cloud ayuda a administrar los certificados SSL / TLS. Puede probar la plataforma como parte de una versión beta gratuita .

Características clave:


  • Seguimiento de todos los certificados externos.
  • Monitoreo y visualización continuos donde se instala cada certificado interno (se utiliza un escáner liviano).
  • Identificación de vulnerabilidades potenciales.
  • Solicitud automática y renovación de certificados, integración con una autoridad de certificación. Los certificados se entregan en segundos. Emitir certificados directamente a las canalizaciones de CI / CD y aplicar políticas apropiadas para cada entorno.


  • Instalación automática de certificados a través de la API REST, integración con herramientas DevOps y el servidor ACME (Entorno de gestión de certificados automatizado).
  • Generación de informes.

Inicialmente, Venafi Cloud ofrece integración con las herramientas DevOps, que incluyen Hashicorp Terraform, Hashicorp Vault, SaltStack, Ansible, Docker y Jetstack Cert-Manager. Venafi Cloud y GlobalSign PKI DevOps proporcionan interfaces estándar bien documentadas, incluida la API REST, el SDK de código abierto VCert (disponible en Go y Python) y ACME. Las empresas de todos los tamaños ahora pueden tener un servicio de identificación de máquina en su infraestructura híbrida y múltiples nubes, lo que ayuda a aumentar la velocidad de DevOps.



Las funciones principales de Venafi Cloud se enumeran en la tabla.

Función Descripción
Contenedorización
  • Automatice la gestión del ciclo de vida del certificado con Kubernetes y Jetstack Cert-Manager
  • Generación de claves y solicitudes de certificados de Docker y el contenedor Venafi Key Management. Los certificados se proporcionan de forma segura a otros contenedores en el mismo host Docker que los contenedores Venafi.
Orquestación
  • Uso de Terraform con la generación de claves, que se puede hacer referencia en los planes para la adquisición y despliegue sin problemas de certificados.
Gestión de la configuración
  • Usando SaltStack para simplificar el proceso de obtención e implementación de certificados, usando la integración de Venafi para transferir certificados a través del sistema de pilares Salt.
Manejo de secretos
  • Hacer cumplir políticas con HashiCorp Vault para certificados emitidos a través de la API de HashiCorp Vault.
Servicios de soporte
  • REST API para solicitar certificados, ver políticas para emitir certificados, ver certificados emitidos, transferir certificados directamente a aplicaciones web de Microsoft Azure, etc.
  • Generación de claves para simplificar la obtención de certificados utilizando VCert, sin tener que escribir código que interactúe con la API REST de Venafi.
  • Los desarrolladores de aplicaciones pueden integrar la generación de claves y las tareas de administración de certificados en aplicaciones personalizadas utilizando VCert SDK, un kit de desarrollo de software multiplataforma escrito en Go.
  • Automatice la administración de certificados para infraestructura externa, como subsistemas de equilibrio de carga, utilizando el servidor Venafi ACME con certificados GlobalSign .
