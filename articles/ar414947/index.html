نظرًا لأن ReactOS يهدف حاليًا إلى التوافق مع Windows Server 2003 ، فقد تم اختيار الإصدار 2.10.0 من Git كأرنب تجريبي ، وهو الأخير الذي يدعي دعم نظامي التشغيل Windows XP و 2003.


تم إجراء الاختبار على سطر الأوامر ReactOS ، وكانت الأعراض الخارجية للمشكلة مختلطة إلى حد ما. على سبيل المثال ، عند بدء git دون تحديد وسيطات إضافية ، فإنه بدون أي مشاكل معروضة معلومات المساعدة في وحدة التحكم. لكن الأمر كان يستحق تجربة git clone أو على الأقل git clone في معظم الحالات لم تعرض وحدة التحكم أي شيء على الإطلاق ، أو عرضت أحيانًا رسالة معطلة مع التأكيد:

كان من المفيد جدًا التحقيق في المشكلة المتمثلة في أن عميل git كان مفتوح المصدر ، ولم يكن العثور على الخط الذي حدث فيه الاستثناء صعبًا: https://github.com/git-for-windows/git/blob/4cde6287b84b8f4c5ccb4062617851a2f3d7fc78/exec_cm. ج # L23 


 char *system_path(const char *path) { /*  */ #ifdef RUNTIME_PREFIX assert(argv0_path); //    assert(is_absolute_path(argv0_path)); /*  */ #endif strbuf_addf(&d, "%s/%s", prefix, path); return strbuf_detach(&d, NULL); }

وتم تحديد قيمة المتغير argv0_path بواسطة هذا القسم من الكود: 


 const char *git_extract_argv0_path(const char *argv0) { const char *slash; if (!argv0 || !*argv0) return NULL; slash = find_last_dir_sep(argv0); if (slash) { argv0_path = xstrndup(argv0, slash - argv0); return slash + 1; } return argv0; }

خاتمة


وهكذا ، تم إصلاح خلل آخر أعاق بشكل غير مباشر التجميع الذاتي لـ ReactOS داخل ReactOS من خلال الجهود الجماعية. صدفة مضحكة هي حقيقة أنه قبل فترة وجيزة تم إصلاح خطأ آخر في نفس msvcrt (أي في وظيفة qsort ) ، والتي لم تسمح بتجميع برامج تشغيل USB في ReactOS.


أشارك في تطوير العديد من المشاريع المكتوبة بلغات برمجة مختلفة ، مغلقة ومفتوحة المصدر. لقد كنت أعمل مع مشروع ReactOS منذ عام 2014 ، لكنني بدأت في المساعدة وكتابة التعليمات البرمجية فقط في عام 2017. من المثير للاهتمام بشكل خاص العمل في هذا المجال ، لأنه نظام تشغيل كامل! يشعر المرء بالحجم الهائل للنتيجة ، حيث تم استثمار الجهود ، بالإضافة إلى الشعور اللطيف بأن حشرة واحدة أصبحت أقل! :)


ربما يتساءل شخص ما لماذا أساعد ReactOS ، وليس Linux على سبيل المثال. حدث ذلك تاريخيًا أنه في معظم الحالات أكتب برامج لـ Windows ، ولغة البرمجة المفضلة لدي هي دلفي. ربما هذا هو السبب في أن بنية Windows NT ، إلى جانب Win32 API ، مثيرة للاهتمام للغاية بالنسبة لي ، ويجلب مشروع نظام التشغيل المجاني ReactOS إلى الحياة حلمًا قديمًا - عمليًا ، يتيح لك معرفة كيفية عمل كل شيء من الداخل.
