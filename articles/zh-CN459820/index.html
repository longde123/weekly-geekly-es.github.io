只需刷卡：纽约地铁如何使用OS / 2

老式技术已经在纽约市的地铁结构中使用了数十年，有时甚至以意想不到的方式突然出现。 OS / 2粉丝的文章


纽约人和42 Ulitsa地铁站（又称时代广场）的游客致电。 听起来像是在开玩笑。 实际上不是：其中一个人很高兴他到达了那里。 另外，这种情况非常令人讨厌。 人们知道如何以最快的速度离开那里。 另一个不是-他不会说英语。 纽约客和游客是不同的人，但目前他们是一个人。 两者都受到大都会运输局（MTA）运输公司的怪癖和1990年代初期中等成功的平均操作系统前所未有的可靠性的影响。

在2016年的平均工作日中，纽约地铁运送了570万人[作比较：莫斯科地铁的这一数字为670万人/约。 翻译]。 这是自1948年以来的最高平均水平。 如果问一般的纽约客，他很可能会说：“就这些吗？” 难以置信是可以理解的，因为有800万永久居民居住在城市中，并且在高峰时段或节假日有时人数会激增至2000万，显然，很多人喜欢乘出租车。


纽约地铁的旋转门

很难押注未来，但实际上MTA做到了。


3月，Tedium 发表了一篇关于IBM在操作系统微内核上的大赌注，其中包括其著名的OS / 2 OS的变体。 它详细描述了该比率导致公司遭受的损失。 但是，IBM对操作系统成功的信心使其他公司也做出了类似的假设。

但是，最大的赌注是MTA，即大都会运输局（Metropolitan Transit Authority），它需要想出一种摆脱代币的方法，并进入一个一切都应该数字化的时代。 结果，狂热的MetroCard卡出现了 。 自从1993年发布以来，带有黄色条纹的黄色塑料薄片已成为纽约人钱包的主要主题。

当前的纽约地铁通行方式的历史在公共基础设施的安排及其如何为公众服务的细节方面很有趣。 但是在此之前，了解当前系统的外观将很有用。 因为当您建造与纽约地铁一样重要的东西时，它最终应能按预期工作。

实际上，您只有一次尝试-任何错误都很可能会导致数十亿的维修费用和数百万民众的烦恼。 在众多选择中，最可靠的选择之一变成了IBM最大的错误之一。


五张专用于David Bowie的特殊MetroCard，由Spotify支付。 在2018年秋天的几周内，该公司将百老汇-拉斐特街/布莱克街站变成了西村，成为波普艺术的纪念碑，以纪念附近的艺术家。 除了使用MetroCards的背面进行广告（以及无广告的地方）之外，MTA还不断提供由大型品牌赞助的特殊版本的卡。 至尊卡选项需要花费大量金钱，但有时MTA会在品牌上得分很高，并且确实很酷。



作为IBM的操作系统，它夸大了宣传，但并没有成为特别的东西，找到了家，为数以百万计的人提供了服务


在有关OS / 2的文章中 ，提到了有关微内核和其他事物的许多有趣的细节，但是在本文中，该OS仍具有其支持者这一事实与该主题最相关。 好吧，没有它的地方。

MTA最终决定使用OS / 2来数字化地铁某些方面的原因，反映了围绕OS于1990年代初发布的炒作。 但是，对话和发展是在几年前开始的。 微软和IBM并没有特别推广这一点，而是致力于下一代OS。 尽管人们认为盖茨和微软在现代故事中使IBM与其MS-DOS相结合，但当时IBM显然有不同的想法。

显然，IBM并没有哀悼所损失的利润，而是意识到自己缺乏知识，因此从基础知识出发，首先与Microsoft一起开发了下一代OS。 就像您可能已经猜到的那样，这项冒险以与MS-DOS故事几乎相同的方式结束了IBM。 但是，在1980年代后期的很短时间内，MTA主管只是在寻找方法来拒绝在地铁中使用令牌并用预付卡代替令牌。 优势是显而易见的-这使得加价和引入代币支付更加容易。 旅客有机会在单程或往返之间进行选择，并且在一定时间内没有任何选择。

为了介绍这一革命性的更新，MTA求助于著名的公司IBM。 当时这很有意义。


OS / 2版本2.1

OS / 2和MTA顾问Neil Waldhauer在一封电子邮件中说：“几年来，您可以押注OS / 2。”

要了解原因，您需要了解该时间。 Waldhauer继续说道：“自从Linux和Windows尚未出现以来，它就一直在发展。 OS / 2似乎是未来的安全选择。”

由于缺乏选择，MTA选择了最好的。 他作为相当复杂的系统中的关键软件组件之一工作了几十年。

正如Waldhauer所说，他可以进一步生存：“让我说，尽管系统支持MetroCard，但OS / 2将继续工作。”

非常有趣的一点是，由于MTA只是在放弃MetroCard的过程中，转而使用各种形式的非接触式支付。 过渡将提高工作效率并帮助MTA收集更多收入。

听起来很有趣，但是很容易发现问题-如果您研究当前MetroCard系统的奇怪功能。


我的MetroCard，“同志骄傲月”的六月版本。 有趣的是，它将比只能使用一年的标准MetroCard延长四个月。

神秘的磁条及其如何影响人们的生活


简而言之，从代币到MetroCard的过渡花费了很多年，而且一点也不顺利。 令牌已于2003年正式停止使用。那时，该城市的所有车站都接受了MetroCard，但没人喜欢它。

进入地铁通常很容易，但是到处都有关于插卡接线的抱怨。 显然，许多问题与系统不同部分之间的通信中断有关。 尽管OS / 2用于连接大型主机的地铁系统的不同部分，但传入组件的标准并不是最高的。 纽约市任何一个车站的旋转门都因反复无常而臭名昭著-但他们能够与IBM系统配合使用。


ATM也依赖OS / 2

尽管OS / 2在消费市场上失败了，但它的可靠性令人难以置信，这使其在工业和工业系统中具有很长的使用寿命-自动柜员机是其中的一个应用示例。 Waldhauer说：“记住MTA中使用的所有OS，我可以说OS / 2可能是系统中最可靠的部分，除了大型机。” 它仍然在2019年的纽约地铁中使用。 IBM很久以前就放弃了它，甚至在2001年甚至允许另一家公司为其提供软件支持。（今天，一家名为Arca Noae的公司出售了官方支持的OS / 2版本ArcaOS ，尽管其大多数用户的情况与MTA相似）。

NYC地铁中的OS / 2充当指挥者的角色。 它有助于将人们使用的各个部分与人们不使用的部分结合起来。 Waldhauer指出：“系统上没有可供用户使用的OS / 2应用程序。 OS / 2主要用作复杂的大型机数据库和地铁和公共汽车中常规使用的简单计算机之间的接口。 但总的来说，OS / 2计算机分布在整个系统中。”

我们谈论的是80年代末设计的操作系统，是90年代初发布的，这是两家技术巨头之间复杂关系的一部分。 MTA必须忽略这个故事的大部分内容，因为它已经做出了决定，而且更改路线会花费很多钱。

后端与纽约人和游客遇到的那些设备之间的协调非常荒谬。 如果您想像一下，让我们回到Waldhauer：“我觉得开发人员计划将MetroCard与大型机数据库一起使用，而一些随机的电子设备会将它们捆绑在一起。”


纽约地铁的代币，根据使用日期从左到右：1953-1970； 1970-1980； 1979-1980年； 1980-1986； 1986-1995； 1995-2003年。

现在让我们谈谈磁条。 无论品牌如何，任何MetroCard底部的黑条都应该起作用。 出于显而易见的原因，它的实际工作方式是一个秘密。

“人们入侵了MetroCard，” Waldhauer说。 “如果您考虑使用磁性编码，那么那里的比特是如此之大，以至于它们可以在放大镜中看到。 磁条的编码是如此秘密，以至于我从未见过。 人们为免费旅行可以做什么，真是太神奇了。”

今天重要吗？ 是的，原则上不是。 MTA明确表示，它计划转换为非接触式付款，就像使用伦敦的Oyster卡一样。 但是，此过程有其自身的问题。 他们甚至雇用了伦敦系统的前负责人，并设定了彻底摆脱MetroCard的最终目标。


新推出的OMNY系统，将在未来几年内推出

将来，人们将可以像今天的阵容一样乘坐迪斯尼乐园的过山车进入纽约的地铁。 此过程将需要一个人将连接到Internet的设备（无论是电话还是智能手表）引导到旋转门。 运气好的话，我们将拥有一个带有MetroCard的新系统。 但是没有保证。

创造纽约地铁的实际技术需求影响了该市几乎所有居民。 纽约人正在使用新的付款方式，那些有能力为此付款的人会这样做。 其余的将坐在家里。
