提供者的分布式数据中心：什么以及为什么

在其存在的几十年中，数据中心已经从小型计算机室发展到计算机中心，数据中心，云和分布式数据中心。 分布式数据中心有哪些用途？ 著名的思科系统公司确定其创建目标：

  • 防灾能力
  • 数据处理连续性
  • 应用移动性
  • 系统迁移
  • 能力/能力建设
  • 分布式服务
  • 地理本地化服务

当所有数据，服务和应用程序都集中在一个数据中心中时，中型组织通常会拥有足够的集中式模型，从而可以保证一定水平的数据可用性和安全性。 但是，通常需要一个备用数据中心， 该中心与使用高速通信通道的主数据中心组合到单个计算机集中。



对于大型地理分布的组织来说，执行收集，存储，处理和交付数据功能以支持公司业务流程的分布式数据中心系统（主数据中心和几个区域中心）可能是最佳的解决方案。 区域数据中心也可以用作备份站点，在高峰负载下使用它们或实施防灾解决方案（DR）。 即使在很大一部分IT资源或通信渠道出现故障的情况下，它也将确保业务流程的连续性。

但是，创建自己的数据中心是一项昂贵的项目。 在计算构建备份数据中心的成本时，有必要考虑IT基础架构和软件许可/支持的成本。 根据一些估计 ，这占创建数据中心总成本的60％以上。


通常，与数据中心服务提供商联系而不是在建设和维护自己的站点上投入大量资金才有意义。 并非偶然的是，在俄罗斯最近的艰难岁月中，大型零售，社交网络和一些IT公司增加了将IT基础架构转移到外包的项目数量 。 租用IT基础架构更具成本效益，并为业务提供了更多的灵活性，使您可以启动新服务并开发现有产品而无资金成本。

用例


分布式数据中心将各种风险降到最低，并且Roskomnadzor最近对站点和子网的阻止（尤其是导致许多云服务的工作中断）表明，任何意外都是可能的。 一些公司在国外购买服务器时发现了摆脱这种情况的出路，在那里他们可以组织VPN，并通过它启动所有必要的云服务。

对于冗余和某些服务，访问站点至关重要，提供商应考虑到这一点。 例如，RUVDS RUCLOUD自己的位于Korolev的数据中心具有三个5 Gb / s的Internet通道，从而保证了高速传输。 通道连接到最大的通讯中心M-9（MMTS-9）和M-10。 客户可以在任何地点选择设备。 因此，用户进入俄罗斯的速度至少为250 Mbit / s，这使他可以与自己的虚拟服务器（VPS / VDS）舒适地工作。

例如，RUCLOUD数据中心通过带宽为10 Gb / s的直接通信通道连接到M-9数据中心中的通信主干。 这使客户能够保证位于不同数据中心的虚拟服务器之间的高速流量交换。

此外，提供商可以通过最大限度地利用IT基础架构资源（大多数企业数据中心中不具备规模经济）来提高备份数据中心的效率。

在典型的分布式数据中心布局中，站点的局域网（LAN）和存储区域网（SAN）相互连接。 服务器所连接的局域网的网段被组合成L2域，这使应用程序可以透明地在站点之间移动服务器IP地址。 得益于SAN服务器的组合，可以使用不同站点的存储资源。

从管理和运营的角度来看，地理位置分散的数据中心（并且有四种类型的此类数据中心）对于用户而言可能看起来像是一个系统。 当分布式数据中心向外部用户提供服务时，可以使用一个服务和支持界面。

它有什么作用？


分布式数据中心还能给提供商及其客户带来什么？ 对于供应商公司而言，这意味着扩大活动范围，进入新市场并允许客户提供新的服务点和更大的选择可能性的可能性。 地理分布的国际数据中心能够为各种参数提供最佳条件。 此外，此类提供商通常具有灵活的定价策略，允许他们考虑到各个要求来选择放置IT设备和资费的最佳选择。

特别是对于客户而言：

  1. 能够使用指定的灾难恢复选项创建各种灾难恢复配置 （灾难恢复）。 备份数据中心还可用于托管IT基础架构，以确保公司应用程序的高可用性。 例如，Roskomnadzor阻止“额外” IP地址对许多与电报无关的Internet服务和站点产生了负面影响。 如果一个国家/地区的IP地址受到类似的阻止，则另一个国家/地区的部分基础设施将可以解救。
  2. 能够将工作组织得尽可能接近所需的目标市场，并使由于“对用户的服务逼近”而导致的网络延迟最小化。 例如，瑞士和英国经常由在欧洲站点上进行交易的贸易商 （在线商店的所有者）选择。
  3. 能够在多个辖区工作 ，遵守当地法律，例如要求处理俄罗斯联邦，欧盟和英国境内的客户数据。
  4. 通过保存数据甚至帐户中的资金轻松地迁移到新站点的能力。 例如，如果客户使用俄罗斯联邦的服务器并决定将数据传输到伦敦，则他只需要请求支持服务，不久之后，他的数据就会“移动”到英国首都。
  5. 分布式云数据中心也被认为是提供IaaS服务的最佳解决方案。

容灾和高可用性


如上所述，通过将IT系统放置在多个数据中心中，可以确保其容灾能力和高可用性。 每个数据中心都在其中发挥作用，但是通常，独立实施和支持这样的体系结构通常是一项艰巨的任务。 此外，高可用性的概念取决于公司业务的具体情况。 为了利用具有多个数据中心的这种拓扑，必须对数据中心的体系结构，在其中运行的应用程序和过程进行适当的实施。


通过将其放置在多个数据中心中，可以实现IT系统的容灾能力和高可用性。

抗灾数据中心（DR）用于托管关键业务应用程序的IT基础结构，以在主数据中心发生灾难或灾难时支持IT服务。 在这种情况下，需要确保在紧急情况下从主数据中心迁移过程中的损失达到可接受的水平。

可能有几个保护区，具体取决于：

  1. 系统对于公司核心业务的重要性。
  2. IT系统停机带来的风险和潜在损失。
  3. IT系统的性能和可靠性要求。
  4. 一组需要冗余和高可用性的系统。
  5. 组织备份数据中心的成本。
  6. 目标数据中心拓扑。

公共云可以承担备份数据中心的某些功能。 在公共云中托管应用程序以提高灵活性并优化IT基础架构成本通常是最明智的解决方案。 因此，从长远来看，可以通过使用公共云服务来减少数据中心的数量。

容灾假设符合RPO / RTO恢复点（Recovery Point Objective，RPO）和恢复时间（Recovery Time Objective，RTO）的必需参数。 RPO / RTO越低，解决方案就越昂贵。 在这种情况下，通常使用数据复制和“热备用”系统。 使用较大的RPO / RTO，您可以备份到磁带或云，并进行冷备份。

地理分布的故障转移群集可用于确保业务连续性。 集群系统通常需要在数据中心之间“拉伸” L2 VLAN，尽管某些应用程序支持通过L3网络进行集群。

虚拟服务的移动性意味着可以在数据中心之间移动虚拟机。 这需要扩展VLAN并提供对LUN卷的连续访问。 这样的解决方案可以成为云负载管理，其平衡以及站点之间负载迁移的基础。

同时，站点之间的距离是关键因素。 距离越短，等待时间越短，性能越高，通信越容易。 您可以使用同步复制。
