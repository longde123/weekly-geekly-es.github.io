Mac OS X 10.10.4 dan Pembaruan TRIM pada SSD Pihak Ketiga

Pada konferensi pengembang baru-baru ini, WWDC 2015 mengumumkan versi Mac OS X 10.11 El Capitan berikutnya. Dengan berkenalan lebih dalam dengan produk baru tersebut, ternyata Apple memberikan kemampuan untuk memaksa dimasukkannya TRIM untuk SSD pihak ketiga. Hore! Benar, rilis OS baru harus menunggu sampai musim gugur. Tetapi untuk semua orang, perintah trimforce sudah ada dalam pembaruan 10.10.4 saat ini.

Baru-baru ini, saya terinspirasi oleh posting ini di sini geektimes.ru/post/200362 membangun Fusion Drive. Drive memilih Samsung 850 Pro pada 128 GB. Efek penambahan SSD luar biasa (70 MB / s -> 260 MB / s). Satu-satunya hal yang tidak saya lakukan pada Mac OS X 10.10 adalah saya tidak menggunakan TRIM enableruntuk memaksa dimasukkannya TRIM, karena untuk ini Anda harus campur tangan dalam kebijakan keamanan sistem. Saya berharap bahwa sampai taraf tertentu kemampuan SSD over-provisioning akan cukup . Kelebihan ketentuan juga merupakan cerita yang menarik. Dalam drive rumah tangga, hanya 7% dari ruang yang dicadangkan untuk tujuan ini, tidak seperti model server, di mana sekitar sepertiga dari volume dicadangkan. Atau, Anda dapat membuat partisi dengan kapasitas lebih kecil pada drive domestik, membebaskan ruang yang diperlukan untuk penyediaan berlebih. Masalahnya adalah saya tidak begitu kuat di baris perintah dan saya tidak tahu di mana dan apa yang harus dikonfigurasikan sedemikian rupa untuk memilih hanya bagian dari disk dengan algoritma untuk membuat Fusion Drive.

Setelah 2-3 bulan, daya tanggap komputer menurun secara nyata. Saya menjalankan tes dan melihat gambar yang diharapkan:



SSD penuh, perintah TRIM tidak diaktifkan, semuanya buruk.
Setelah sudo trimforce mengaktifkan perintah dan komputer dinyalakan kembali, kecepatan dengan cepat kembali ke normal:



Sekarang tetap untuk memahami stabilitas. Sejauh ini, beberapa jam kerja - penerbangan normal.

Sebelum menginstal pembaruan 10.10.4, saya mengalami masalah - App Store tidak melihat adanya pembaruan. Saya mengunduh pembaruan sebagai file terpisah dari situs web Apple, tetapi saya masih tidak dapat menginstalnya.
Solusi untuk masalah yang ditemukan di forum Apple:
- Hapus isi Perpustakaan / folder pembaruan
- Hapus /Library/Caches/com.apple.SoftwareUpdate/
- Hapus /Library/Preferences/com.apple.SoftwareUpdate.plist
- Reboot
- Pembaruan rutin melalui App Store
