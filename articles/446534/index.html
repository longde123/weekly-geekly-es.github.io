Visual Studio 2019 lanzado



Hoy, una nueva versión del IDE de Microsoft está disponible: Visual Studio 2019 y su "primo" Visual Studio 2019 para Mac han sido lanzados.

Visual Studio se encuentra en una posición un tanto extraña, y los desarrolladores también pueden preguntar por qué existe tal lanzamiento. Visual Studio 2017 ha recibido nueve actualizaciones menores e innumerables parches desde su lanzamiento hace dos años . Cada una de estas versiones trajo nuevas características y correcciones de errores , y el uso de Visual Studio resultó ser similar a, por ejemplo, Google Chrome, donde cada nueva versión trae un flujo constante de mejoras incrementales.


Modo Live Share, a la izquierda, el código está abierto en Visual Studio 2019, a la derecha, en Visual Studio Code

Y después de todo, Microsoft está promocionando (y usando) este modelo incremental interactivo en servicios como Azure DevOps , y se puede comparar con el desarrollo continuo de Office 365 y Visual Studio Code actualizados mensualmente . Cuando se utiliza este enfoque para el desarrollo, uno puede preguntarse por qué se estaba molestando con "Visual Studio 2019": simplemente tengamos "Visual Studio" y siempre se actualizará.

¿Por qué apegarse al antiguo enfoque de lanzamiento? Hay consumidores que compran licencias perpetuas, y una nueva versión principal facilita la realización de ciertos cambios, por ejemplo, dejar de admitir plataformas más antiguas o escalar la biblioteca C ++. Visual Studio 2019 (finalmente) deja de ser compatible con Windows XP para proyectos de C ++, por lo que debe usar el antiguo compilador de Visual Studio 2017 si aún desea apuntar a un sistema operativo obsoleto. Además, la nueva versión principal es el momento adecuado para realizar grandes cambios en la interfaz, y lo primero al instalar Visual Studio 2019 será una nueva pantalla de bienvenida , una nueva interfaz para crear proyectos y una nueva barra de título que incluye de inmediato tanto el menú de la aplicación como una búsqueda de funciones rediseñada en el IDE .



Además, la nueva versión trae cosas que no estaban en 2017. Lo que más me atrae es que el sistema Live Share ha alcanzado la etapa de disponibilidad general. Este es un sistema de edición colaborativo que funciona tanto en Visual Studio como en Visual Studio Code, permitiendo a las parejas de desarrolladores codificar y depurar juntos, mientras ven una interfaz frente a ellos que coincide con sus preferencias personales. La versión de vista previa inicial de Live Share, que apareció en noviembre de 2017, solo admitía JavaScript (junto con su exitosa versión de Microsoft de TypeScript) y C #.

En respuesta a la demanda del usuario, C ++ y Python se agregaron a Live Share. Hasta ahora, Python para Visual Studio todavía es nuevo; El soporte para este lenguaje de secuencias de comandos se agregó en Visual Studio 2017 con una de las actualizaciones. Visual Studio 2019 amplía esto con soporte para varios tiempos de ejecución de Python (facilitando el cambio entre intérpretes y versiones), un depurador más funcional y un complemento IntelliSense más inteligente.

Los desarrolladores de C ++ obtienen una optimización mejorada en el compilador, soporte mejorado para proyectos creados por CMake y soporte parcial para un perfil de por vida , un conjunto de reglas que permiten al compilador advertir sobre el uso inseguro de punteros e iteradores.

Ahora que GitHub es parte de Microsoft, la integración con GitHub está ganando impulso en Visual Studio; en 2019, aparece el soporte para el modelo de solicitud de extracción de Github para gestionar la integración de parches en la base de código directamente en el IDE. También hay soporte para la función "stash" de git, que le permite guardar un conjunto de cambios para cambiar a otra rama sin tener que confirmar estos cambios y sin correr el riesgo de perderlos.

Y al igual que con cualquier nueva versión de Visual Studio, existe la serie habitual de actualizaciones de compilador y versiones de idioma: vista previa de las características de C # 8.0, nuevas refactorizaciones y similares.

Visual Studio para Mac (basado en el IDE de Xamarin cuando Microsoft compró Xamarin ) también se actualizó hoy. Su primera versión, de hecho, fue un cambio de marca de la aplicación Xamarin Studio (con la incorporación del compilador C # y las bibliotecas .NET de Microsoft) y tuvo poco que ver con el Visual Studio "real".

Sin embargo, parece que Microsoft está tratando seriamente de acercar estos productos en aquellos aspectos en los que tiene sentido. En Visual Studio para Mac 2019, apareció una vista previa de un nuevo editor de texto, basado en el mismo motor que en Visual Studio para Windows, con una interfaz nativa y funciones para macOS. Esto significa que ambos Visual Studio ahora tienen capacidades similares en cosas como IntelliSense, adiciones de código y soluciones rápidas. El nuevo editor no está habilitado de forma predeterminada, pero se puede habilitar para C # y XAML, y después de llevarlos a un estado estable, se planea agregar más idiomas. La pantalla de bienvenida ahora también se ve muy similar a la de Windows:



Microsoft reúne dos Visual Studio en otras áreas: el depurador de Unity en Mac y Windows ahora es el mismo, y en el futuro, la actualización tiene la intención de llevar parcialmente Windows Xamarin Forms XAML a Mac.

Además de todo esto, hay mejoras en el rendimiento y la estabilidad, así como muchas mejoras en la accesibilidad.

En el caso de ambas versiones de Visual Studio, Microsoft enfatiza la importancia de los comentarios de los usuarios durante el proceso de desarrollo. Tanto las actualizaciones menores como las principales se basaron en comentarios, por ejemplo, en los casos de Python y Live Share, aparecieron nuevas características como respuesta directa a las solicitudes de los usuarios. Un flujo constante de lanzamientos menores le permite a Microsoft proporcionar nueva funcionalidad a los usuarios mucho más rápido de lo que sería con algunas actualizaciones importantes, y esta funcionalidad se puede modificar y ampliar en respuesta a los comentarios. En comparación con los viejos tiempos, cuando comenzaste con errores en el sitio web de Microsoft Connect solo para hacerlos más profundos, el estado actual de las cosas es una mejora refrescante.
