So erstellen Sie ein schönes Datenbankschema in yEd Graph Editor

Ich möchte Ihnen zeigen, wie Sie mit Hilfe der mittlerweile beliebten yEd-Anwendung, mit der verschiedene Diagramme und Schemata erstellt werden, ein vollständiges Datenbankschema (mit der gesamten Notation) erstellen können.

Für den Anfang haben wir also einen Standardcontainer mit Elementen zum Erstellen eines ERD (Entity Relationship Diagram). Ihre Bezeichnungen sind in der Abbildung dargestellt.



Ein kleiner Exkurs, wenn Sie nur Text in das Diagramm einfügen müssen, d. H. Ohne Block lösen die folgenden einfachen Schritte aus einem Kommentar von Jaiprakash dieses Problem schnell. "Wählen Sie einen beliebigen Link -> Etikett hinzufügen -> Text eingeben -> in Knoten konvertieren."

Wir stellen also das bedingte Problem der Konstruktion des folgenden Schemas.



Erstellen Sie einen Rahmen aus Grundelementen.



Um die Notation von Primär- und Fremdschlüsseln hinzuzufügen, müssen Sie ein wenig erfinden. Wählen Sie das gewünschte Element aus und fügen Sie ihm eine neue Bezeichnung hinzu, die wir KEY nennen. Weil Es befindet sich unter dem Hauptgerät und wählt es dann mit der geklemmten Ctr aus.





Stellen Sie die Platzierung (Platzierung) ein, indem Sie Intern auswählen: oben links (innen, oben links), Farbe wie gewünscht, Abstand (Abstand von den Kanten) auf Null, Einfügungen (Einfügung in Pixel) auf 20 2 0 0, sodass sie dem Attributfeld gegenüberliegen Mit einem kleinen Abstand stellte die Schriftfamilie (Schriftart) das Segoe-UI-Symbol ein, weil Es unterstützt genau die erforderlichen Zeichen.



Symbole zum Kopieren finden Sie beispielsweise in der Windows-Symboltabelle.





Machen Sie dasselbe mit den Pfeilen.



Um mehrere Tabellen zu einem Block zu kombinieren, können Sie die Gruppierung verwenden.
