3CX SBC Remote Connection Manager baru di V16 Perbarui 2 Beta

Minggu lalu kami merilis Pembaruan 2 BETA 3CX v16 berikutnya. Ini termasuk Remote Connection Manager yang ditulis ulang dari awal 3CX SBC (Session Border Controller).

Ingatlah bahwa 3CX SBC direkomendasikan untuk digunakan untuk menghubungkan kantor jarak jauh ke pusat PBX 3CX jika dua atau lebih telepon IP berada di lokasi yang jauh. Pada saat yang sama, pensinyalan SIP melewati, seperti halnya koneksi lokal ke PBX, melalui server 3CX, dan aliran media (suara) melewati antara perangkat jarak jauh secara langsung, tanpa meninggalkan kantor. Ini tidak hanya meningkatkan kualitas komunikasi, tetapi juga membongkar saluran komunikasi eksternal, "meninggalkan" panggilan lokal dalam jaringan lokal. Selain itu, 3CX SBC terowongan semua lalu lintas VoIP melalui port TCP tunggal, sangat menyederhanakan konfigurasi router perbatasan. SBC memungkinkan Anda mengelola telepon IP dari jarak jauh - mengkonfigurasi ulang, memperbarui firmware, dll., Dan juga menyediakan enkripsi lalu lintas VoIP antara kantor jarak jauh dan server 3CX untuk memberikan privasi tambahan.

Bersama dengan V16 Update 2 Beta, kami juga merilis Panduan Cepat 3CX dalam Bahasa Rusia. Itu dikirim ke pengguna (melalui email yang ditentukan dalam parameter ekstensi) saat membuat nomor dalam pertukaran.

Fitur 3CX SBC v16


3CX SBC v16 baru memiliki fitur berikut:

  • Instalasi layanan yang disederhanakan. Untuk menginstal SBC, Anda hanya perlu menentukan 3QX server FQDN dan kata sandi otentikasi terowongan SBC (teknologi 3CX Tunnel) (lihat di bawah).
  • Manajemen layanan. SBC sekarang diperbarui dari jauh dan dikonfigurasi ulang dalam antarmuka 3CX (lihat di bawah). Pembaruan otomatis SBC hanya tersedia di Linux dan Raspberry Pi.
  • Pemantauan terpusat untuk semua KUK yang terhubung. Untuk setiap trunk SBC, administrator melihat perubahan status koneksi. Juga tersedia statistik layanan, log acara, dan pemberitahuan email.

Perhatikan bahwa SBC v16 hanya berfungsi dengan Pembaruan 3CX V16 2. Sertifikat SSL tepercaya harus diinstal pada server 3CX (sertifikat yang ditandatangani sendiri tidak didukung).

Instal Pembaruan 3CX


Selain pembaruan standar di antarmuka 3CX (hanya untuk Linux), Anda tentu saja dapat mengunduh distribusi sistem lengkap:

  • Windows: 3CX v16 Pembaruan 2 Beta
  • Linux: 3CX v16 pada Debian 9 ISO - Dari layar Saluran Pembaruan 3CX, pilih Beta.
  • Menginstal 3CX secara manual pada Debian dan Raspberry Pi .

Konfigurasikan Koneksi 3CX SBC


3CX SBC dapat diinstal pada versi terbaru dari Raspberry Pi (Raspbian Linux), dan dapat menangani hingga 30 telepon IP. Setiap telepon disarankan untuk mengkonfigurasi tidak lebih dari 10 indikator BLF, karena layanan pengunduhan pesan status. Namun, jika Anda menginstal SBC pada Windows atau Debian Linux pada perangkat keras yang disarankan , lebih banyak ponsel diperbolehkan.

Komputer tempat Anda berencana menginstal SBC harus memiliki port gratis 5060 (TCP dan UDP) dan tidak boleh berupa DNS, DHCP, server SQL, atau sumber daya jaringan serupa. Secara umum, tidak disarankan komputer ini digunakan untuk tugas-tugas lain.

Koneksi SBC dari sisi server 3CX


Buat koneksi SBC baru di bagian "SIP Trunks", tombol "Add 3CX SBC".



Tunjukkan nama koneksi SBC (jangan gunakan karakter khusus), misalnya, HeadquartersToMoscowBranch. Kemudian, pada halaman pengaturan koneksi, salin ID otentikasi dan tautan web. Parameter ini ditentukan saat menginstal SBC di kantor jarak jauh.



Pertimbangkan parameter koneksi SBC.

Tab dasar:

  • Informasi koneksi - nama koneksi dan kata sandi terowongan.
  • Informasi Jaringan - URL dan ID untuk menghubungkan SBC jarak jauh.

Tab Statistik:

  • Status terowongan, waktu aktif, versi perangkat lunak, jumlah panggilan dan pengguna untuk koneksi ini.

Tab opsi:

  • Tingkat Keamanan Terowongan (TLS atau TCP)
  • Nomor port audio RTP pertama adalah port pertama yang digunakan oleh aliran audio yang melewati SBC.
  • Aktifkan toleransi kesalahan - masukkan alamat IP eksternal dari server cadangan 3CX. Jika terjadi kegagalan pada server utama, SBC akan beralih ke server cadangan.
  • Logging dan troubleshooting - pilih level logging. Jangan atur “Level detail logging untuk waktu yang lama” - ini menyebabkan kelebihan, terutama jika Anda menggunakan Raspberry Pi.

Tombol-tombol ini juga tersedia di bagian antarmuka SIP Trunk 3CX:

  • "Push config" - Mengirim konfigurasi yang diperbarui ke layanan SBC yang dipilih.
  • "Perbarui" - Memperbarui layanan SBC ke versi stabil terbaru.

Instal layanan SBC di kantor jarak jauh


Setelah Anda mengkonfigurasi koneksi SBC di 3CX, Anda dapat menginstal SBC di kantor jarak jauh.

Menginstal SBC pada Linux Debian dan Raspbian




Instalasi dari baris perintah:

Versi stabil 

wget http://downloads.3cx.com/downloads/sbc/3cxsbc.zip -O- | sudo bash

Versi beta 

 wget http://downloads.3cx.com/downloads/sbc/3cxsbcV16.zip  -O- | sudo bash



Menginstal dari gambar ISO:

Setelah menyelesaikan instalasi Linux, pada layar 3CX Update Channel, pilih 3CX SBC Stable atau 3CX SBC Beta.



Kemudian tentukan URL server 3CX dan ID otentikasi dari pengaturan koneksi SBC.



Sekarang Anda dapat menghubungkan telepon IP jarak jauh (mode koneksi - koneksi melalui SBC).

Instal SBC pada Windows


Jalankan 3CX SBC untuk kit distribusi Windows dan tentukan URL server dan ID otentikasi selama proses instalasi.



Memperbarui SBC di Linux secara manual (upgrade dari versi sebelumnya)


Perbarui repositori APT: 

 sudo apt update

Perbarui ke versi stabil: 

 sudo apt upgrade 3cxsbc

Pembaruan beta: 
 sudo apt -t testing upgrade 3cxsbc

Ketika memperbarui secara manual versi SBC sebelumnya (hanya Linux), koneksi SBC yang sesuai secara otomatis muncul di antarmuka manajemen 3CX. Pilih koneksi dan klik tombol "Kirim Konfigurasi". Setelah memperbarui konfigurasi, Anda dapat menghubungkan telepon IP.
