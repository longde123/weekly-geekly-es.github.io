â–Tentang panel, merek, dan perusahaan


Plesk adalah perangkat lunak berpemilik yang dikembangkan di Novosibirsk dan dirilis pertama kali di AS pada tahun 2001. Selama hampir 20 tahun, hak atas platform secara berturut-turut diperoleh oleh berbagai perusahaan yang mengubah merek dan nama. Sejak 2015, Plesk adalah perusahaan Swiss independen dengan beberapa cabang (termasuk Novosibirsk) dan staf sekitar 500 orang (termasuk spesialis Rusia baik di kantor pusat maupun di cabang).

Tiga versi terbaru:

  • Plesk 12.5 (2015)
  • Plesk Onix (2016-2019)
  • Plesk Obsidian (2020)

Panel ini multibahasa. Ditulis dalam PHP, C, C ++. Dukungan untuk beberapa versi PHP, serta Ruby, Python dan NodeJS; dukungan penuh untuk Git; Integrasi Docker Toolkit SEO. Setiap instance Plesk secara otomatis diamankan menggunakan SSL / TLS.

OS yang didukung: Windows dan berbagai versi Linux. Di bawah ini Anda dapat melihat persyaratan untuk OS ini.

Program ini dirilis dalam berbagai majelis, yang masing-masing dirancang untuk audiens penggunanya sendiri. Sebagai contoh, panel memungkinkan administrator untuk mengelola semua layanan sistem secara terpusat menggunakan antarmuka web tunggal dan mengurangi biaya perawatan, memberikan tingkat fleksibilitas dan kontrol yang diperlukan. Dan untuk perusahaan yang menjual hosting virtual dan berdedikasi, panel ini memungkinkan Anda untuk mengatur sumber daya server ke dalam paket dan menawarkan paket ini kepada pelanggan - perusahaan atau individu yang ingin meng-host situs mereka di Internet, tetapi tidak memiliki infrastruktur TI yang diperlukan untuk ini.

Antarmuka


Di Plesk, setiap grup pengguna memiliki antarmuka sendiri, disesuaikan dengan kebutuhan mereka. Dalam antarmuka penyedia hosting - alat untuk menyediakan hosting, termasuk sistem penagihan terintegrasi untuk bisnis otomatis. Perusahaan yang menggunakan platform untuk mengelola infrastruktur web mereka sendiri dapat menggunakan berbagai operasi manajemen server: pemulihan sistem, pengaturan server web, dan sejenisnya. Mari kita lihat dua versi terakhir dari platform - Plesk Onyx dan Plesk Obsidian - melalui mata seorang administrator web.

â– Fungsi untuk administrator web


Akun pengguna. Buat akun pengguna terpisah dengan kredensial mereka sendiri. Tetapkan peran dan langganan pengguna untuk setiap pengguna atau grup pengguna.

Berlangganan Buat langganan dengan serangkaian sumber daya dan layanan spesifik yang terkait dengan paket layanan, dan beri pengguna akses sesuai dengan peran pengguna. Batasi jumlah sumber daya sistem (CPU, RAM, disk I / O) yang dapat digunakan oleh langganan tertentu.

Peran Pengguna Mengaktifkan dan menonaktifkan fungsionalitas dan ikon untuk masing-masing pengguna. Berikan berbagai tingkat akses ke pengguna yang berbeda di tingkat berlangganan yang sama.

Paket layanan. Buat paket layanan yang menentukan distribusi sumber daya Anda: misalnya, ruang disk, bandwidth, dan fitur lain yang ditawarkan kepada pelanggan Anda.

Dukungan server surat. Secara default, server surat Postfix dan Courier IMAP diinstal pada Plesk untuk Linux, dan MailEnable on Plesk untuk Windows.

Perlindungan DKIM, SPF dan DMARC. Plesk mendukung DKIM, SPF, SRS, DMARC untuk otentikasi email.

OS yang didukung Versi terbaru Plesk untuk Linux / Unix mendukung beberapa platform, termasuk Debian, Ubuntu, CentOS, Red Hat Linux, dan CloudLinux.

Manajemen basis data. Memindai, memulihkan, melaporkan, memperbaiki basis data yang didukung.

Kepatuhan dengan standar PCI DSS di luar kotak. Amankan server Anda dan dapatkan kepatuhan PCI DSS di server Linux.

Perencanaan tugas. Atur waktu dan tanggal untuk menjalankan perintah atau tugas tertentu.

Pembaruan sistem. Perbarui semua paket sistem yang tersedia di server secara manual atau otomatis tanpa membuka konsol.

Plesk Migrator. Migrasi tanpa perlu menggunakan baris perintah. Sumber yang didukung: cPanel, Confixx, DirectAdmin dan lainnya.

Administrasi server Linux


Administrator dapat menggunakan beberapa alat tambahan yang disediakan dengan distribusi Plesk standar untuk menambahkan tugas otomatisasi khusus, membuat cadangan dan memulihkan data, dan memulihkan komponen Plesk dan pengaturan sistem. Alat tersebut mencakup beberapa aplikasi mandiri, utilitas baris perintah, dan kemampuan untuk mengintegrasikan skrip khusus dengan Plesk. Untuk melakukan tugas manajemen server dengan mudah, ada instruksi langkah-demi-langkah yang berisi bagian-bagian berikut:

  • Pengantar Plesk. Menjelaskan komponen dan layanan utama yang dikelola Plesk, persyaratan lisensi, dan cara menginstal dan memperbarui komponen Plesk.
  • Konfigurasi host virtual. Menjelaskan konsep host virtual dan implementasinya di Plesk. Memberikan instruksi tentang mengapa dan bagaimana mengubah konfigurasi mereka.
  • Manajemen layanan. Berisi deskripsi sejumlah layanan eksternal yang digunakan pada server Plesk, dan instruksi untuk mengatur dan menggunakannya.
  • Pemeliharaan sistem. Menjelaskan cara mengubah nama host server, alamat IP, dan lokasi direktori untuk menyimpan file host virtual, cadangan, dan konten email. Bab ini juga mencakup alat baris perintah Plesk, mesin scripting untuk acara Plesk, dan monitor layanan, yang memungkinkan Anda untuk memantau dan memulai kembali layanan tanpa masuk ke Plesk.
  • Cadangkan, pulihkan, dan migrasi data. Menjelaskan cara membuat cadangan dan memulihkan data Plesk menggunakan utilitas baris-perintah pleskbackup dan pleskrestore, dan menyediakan alat untuk memigrasi data yang dihosting antar server.
  • Statistik dan Log. Menjelaskan cara melakukan perhitungan statistik berdasarkan permintaan untuk ruang disk dan penggunaan lalu lintas, dan untuk mengakses log server web.
  • Tingkatkan produktivitas. Memberikan informasi tentang cara meningkatkan Plesk dengan perangkat lunak.
  • Tingkatkan keamanan. Ini berisi instruksi tentang cara melindungi server Plesk dan situs yang dihosting di dalamnya dari akses tidak sah.
  • Sesuaikan tampilan dan nuansa GUI Plesk. Memperkenalkan tema Plesk yang dapat digunakan untuk menyesuaikan tampilan dan identitas korporat Plesk, dan menjelaskan cara menghapus elemen GUI Plesk tertentu atau mengubah perilaku mereka.
  • Lokalisasi Memperkenalkan metode pelokalan PUI GUI ke dalam bahasa yang Plesk tidak menyediakan pelokalan.
  • Pemecahan masalah. Menjelaskan metode pemecahan masalah untuk layanan Plesk.
