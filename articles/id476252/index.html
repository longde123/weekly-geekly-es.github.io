Migrasi email: cara mudah berpindah dari satu server dan pergi ke yang lain

Topik yang disebutkan dalam judul mungkin tidak relevan untuk Khabrovit yang disegani, tetapi kadang-kadang hanya perlu untuk mengangkatnya. Faktanya adalah bahwa saya telah bekerja sebagai administrator selama bertahun-tahun di sebuah lembaga penelitian kemanusiaan, di mana karyawan memiliki kualifikasi sedemikian di bidang teknologi informasi modern sehingga pembukuan terkenal dari anekdot tentang TI pada latar belakang ini tampak seperti kumpulan filsuf yang didedikasikan untuk semua rahasia kehidupan. Para ilmuwan yang terhormat, berhasil memasukkan nama server surat dalam surat Rusia, alih-alih tanda "@", tulis "anjing" dalam tanda kurung (dan kemudian katakan bahwa itu ditulis di alamat surat yang dikirimkan kepada mereka), cobalah mengirim email ke WhatsApp menggunakan The Bat! dan melakukan banyak hal aneh, sering kali dalam pesan yang sama. Tidak ada gunanya mengajar mereka, tidak mungkin untuk melawan mereka; tetap menerima nasib mereka dan pada akhirnya mengotomatiskan semua operasi yang terkait dengan koreksi kesalahan mereka.

Salah satu operasi yang paling menyeramkan dan berbahaya dalam praktik saya adalah migrasi surat Web dari server ke server. Faktanya adalah bahwa karyawan lembaga sudah memiliki tiga surat layanan: satu termasuk server Exchange internal, yang lain bekerja di Mail.ru, dan yang ketiga - di Gmail. Tidak, ini bukan aku idiot, dan bahkan bukan mereka. Ini adalah perintah langsung yang terkait dengan beberapa game departemen. Sesuatu harus tetap berada di dalam institut di server "perusahaan", sesuatu yang berkaitan dengan aplikasi dan hibah tentu harus melalui surat Rusia, dan surat Gmail dari rekan terkasih saya terhubung dengan hal-hal yang tentu diperlukan seperti dokumen dan tabel Google, cadangan ke disk, dll. Satu-satunya masalah adalah bahwa ketujuh pengasuh, seperti yang Anda tahu, memiliki anak tanpa mata - yaitu, dalam hal ini, antara tiga server surat rekan-rekan saya dengan cara yang paling eksplisit berhasil kehilangan surat yang paling penting!

Ada masalah lain yang sering mengharuskan migrasi email. Layanan surat modern sering memungkinkan penyampaian pesan secara otomatis dari satu server ke server lain, yaitu, koleksi email. Dan pengguna yang terbiasa dengan fakta bahwa pesannya di server, katakanlah Mail.ru, secara otomatis disalin ke surat Yandex, kadang-kadang lupa karena dengan cara ini ia mendapatkan akses tidak ke semua pesan, tetapi hanya untuk mereka yang diterima setelah pengaturan koleksi surat. Oleh karena itu, ia mungkin memiliki keinginan alami untuk menyelesaikan migrasi lengkap email dari server lama ke yang baru, yang lebih sering digunakan, dan kepada siapa ia akan pergi dengan keinginan ini? Itu benar: kepada administrator sistem terdekat!

Saya pikir situasi serupa muncul untuk siapa pun yang entah bagaimana terpaksa memiliki beberapa akun email, apalagi untuk mengelolanya, atau hanya ingin pindah dari server ke server tanpa kehilangan informasi penting. Tentu saja, bison TI dapat dengan mudah menyelesaikan masalah ini dengan dua klik mouse, tetapi jika Anda tidak berpengalaman dalam hal-hal seperti itu, maka migrasi email dapat berubah menjadi tugas yang sulit bagi Anda. Oleh karena itu, saya memutuskan untuk membagikan pengalaman saya secara singkat tentang cara mudah mengekspor surat ke beberapa penyimpanan dan kemudian mengimpor surat ke server lain. Mungkin operasi ini akan membantu seseorang menyingkirkan masalah kecil atau hanya membuat hidup lebih mudah!

Surat ekspor: sedikit teori, sedikit praktik


Pada prinsipnya, server surat bekerja dengan program klien menggunakan salah satu dari dua protokol: POP3 atau IMAP. Jika nama-nama ini tiba-tiba tidak memberi tahu Anda apa-apa (apakah ini terjadi?), Saya akan mencoba menjelaskan dengan kata-kata sederhana: protokol POP3 mengunduh pesan dari server ke komputer Anda, dan protokol IMAP memprosesnya langsung di server. Klien email lama secara default bekerja (dan terus bekerja) dengan protokol POP3, membongkar pesan email dalam folder yang khusus dialokasikan untuk klien (biasanya terletak di suatu tempat di direktori pengguna, di antara folder dengan data aplikasi disembunyikan secara default). IMAP lebih modern, dan dengan itu Anda juga dapat mengimpor email ke penyimpanan lokal atau jaringan. Jadi pertanyaannya terutama bukan bagaimana mengunduh surat yang Anda butuhkan, tetapi bagaimana meneruskannya ke server yang tepat untuk melakukan migrasi email. Opsi termudah adalah dengan menggunakan protokol IMAP, menyalin semua huruf ke beberapa penyimpanan dalam format EML di atasnya, dan kemudian mengunggahnya ke folder lain di akun lain, mengambil keuntungan dari kenyataan bahwa format file pesan umumnya sama.

Bagaimana cara melakukannya?


Metode paling sederhana yang saya gunakan dengan biaya terendah adalah memigrasi surat surat menggunakan semacam program penyalinan data yang mendukung IMAP. Ini dilakukan dalam dua langkah.

  • Impor email dari folder di server ke penyimpanan apa pun dalam format EML.
  • Ekspor surat ke beberapa folder lain di server lain melalui IMAP.

Dalam hal ini, program migrasi surat dari sudut pandang kedua server berperilaku seperti klien IMAP biasa. (Omong-omong, sebagian besar server email akan mengharuskan Anda untuk menggunakan program yang ditentukan sebagai klien email, jadi pastikan untuk masuk ke akun email Anda dan memungkinkan server untuk menggunakan utilitas ini dalam daftar klien yang tersedia melalui IMAP sebelum memigrasi email dengan utilitas apa pun). Program semacam itu biasanya membutuhkan pekerjaan manual minimum untuk melakukan pra-konfigurasi migrasi email. Biasanya Anda bahkan dapat mengatur jadwal migrasi otomatis reguler dari server ke server, jika Anda memerlukannya karena alasan tertentu. Secara pribadi, saya menggunakan program Handy Backup untuk mengekspor surat, karena diinstal pada hampir semua mesin dan memerlukan pengaturan minimum, yang juga terpusat dari mesin administrator - tidak perlu pergi ke mana pun. Tetapi, pada umumnya, perangkat lunak yang digunakan tidak penting, jika saja ia dapat dengan terampil mengekspor dan mengimpor surat langsung ke server Web, dan mendukung format tunggal untuk huruf pada kedua server.

Dan Microsoft, seperti biasa ...


Sakit kepala terpisah adalah migrasi Exchange atau Outlook e-mail (maksud saya bukan server surat Outlook.com, tetapi klien), karena Microsoft, seperti biasa, mengikuti jalur non-standar. Nah, jika dalam situasi ini Anda memiliki perangkat lunak khusus untuk mengekspor surat surat Outlook atau server Exchange, maka tugas itu disederhanakan sebelum membaca instruksi untuk memigrasi surat surat yang menjalankan program yang sesuai. Baik bahwa ada banyak program seperti itu, serta plugin khusus untuk perangkat lunak terkait yang berorientasi pada produk Microsoft.

Migrasi Email POP3


Beberapa orang menyukai penyimpangan, tetapi secara umum tidak demikian. Oleh karena itu, tidak perlu mentransfer email dari server ke server menggunakan protokol POP3, itu sudah tua dan jelek. Pergi ke IMAP di kedua server (bagaimana melakukan ini, hampir setiap penyedia memiliki instruksi terperinci), dan kemudian melakukan segala sesuatu seperti yang dijelaskan di atas (atau setidaknya menggunakan alat migrasi yang dibangun ke dalam layanan surat - kadang-kadang ada alat seperti itu, meskipun kenyamanan mereka dan logika kerja biasanya menyisakan banyak yang diinginkan). Anda juga dapat mencoba metode manual kuno: menggunakan program klien, mentransfer surat dari folder ke folder, atau cukup memilihnya dan mengirim ke server baru. Suatu ketika, ketika kita masih kecil, kita semua bertindak seperti itu, dan itu tampaknya tidak senonoh bagi kita, jadi dalam situasi tanpa harapan, Anda dapat mencoba lagi untuk melakukan pekerjaan manual seperti itu ...

Secara umum, migrasi email dari server ke server dengan mengimpor surat secara berurutan ke repositori, dan kemudian mengekspor email ke server baru melalui IMAP, memenuhi semua kriteria dasar untuk bekerja dengan program. Kriteria ini adalah logika yang jelas, keamanan, otomasi dan ketersediaan sejumlah besar alat siap pakai yang tersedia yang dapat melakukan pekerjaan untuk Anda. Jadi, saya berharap ini catatan saya akan berguna bagi seseorang dan menyederhanakan hidup saya dalam kasus-kasus ketika departemen akuntansi atau departemen perencanaan tiba-tiba perlu untuk mentransfernya dari Yandex ke Mail.ru, dari Google ke Yahoo! atau di tempat lain, di mana kepala tiba-tiba sibuk dengan dislokasi pesanan surat. Jangan biarkan diri Anda lelah, kolega!
