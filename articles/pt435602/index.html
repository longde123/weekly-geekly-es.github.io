Microsoft Learning


O primeiro recurso que você precisa prestar atenção é o Microsoft Learning . Este é o principal recurso recomendado pela Microsoft para aprender o Azure. Esse recurso, que pode ser usado para aprender o Azure a partir do zero, enquanto você escolhe sua função (desenvolvedor, arquiteto, administrador, especialista em dados ou AI (no entendimento da Microsoft, é claro) ).
Existem muitas peças relativamente pequenas que podem ser estudadas por 30 a 60 minutos por dia durante 1 a 3 meses, enquanto cobrem uma ampla camada de material.



Um dos pontos mais importantes é que existem muitos pequenos trabalhos de laboratório que ajudam a construir a memória muscular. Por exemplo, vamos ver todas as seções do Azure que podem ser de interesse dos desenvolvedores .

Claro, cerca de 1 a 3 meses - isso é do zero. Se você trabalha com o Azure nos últimos dois anos, esse recurso deve ser analisado rapidamente, encontre as lacunas no seu conhecimento e continue com a preparação para o exame.

Função PluralsightIQ


Para começar, você pode fazer o teste de conhecimento do Azure gratuitamente em 3 (de 8) áreas: arquitetura , desenvolvimento , infraestrutura , engenheiro de IA , engenheiro de dados , engenheiro de DevOps , engenheiro de segurança , cientista de dados e obter de 0 a 300 do seu conhecimento na saída. Cada função consiste em vários testes de 6 a 9, o que permite analisar de maneira abrangente seu conhecimento. (Você precisa criar uma conta, não precisa vincular um cartão, também não é necessário iniciar uma avaliação gratuita. Os testes estão disponíveis sem ele). Exemplo de arquitetura



Na minha opinião, esses são bons testes qualitativos ( existe um qualitativo, mas existe um método quantitativo que dará valor ) para avaliar seu conhecimento.

Para cada pergunta, é dado um limite de tempo , dependendo do tamanho do texto da pergunta + todas as opções de resposta. Mas se você deseja invadir o sistema, será difícil, porque não há muito tempo.

Por tudo isso, a estimativa é não linear , ou seja, Passei 17 das 20 perguntas - você receberá 255 das 300. Depende da complexidade das tarefas e em que momento você estava errado. Pessoalmente, esse sistema é um pouco incompreensível para mim (acontece que, se você cometeu um erro no início, é ruim, mas se mais próximo do final é menos assustador), mas fornece um resultado qualitativo.



A propósito, você pode divertir seu orgulho, porque ao final de cada teste, eles dão não apenas uma avaliação, mas também a porcentagem de pessoas que passaram melhor que você. É bom ver que 1-3% são melhores que você.

Vejo esse teste como uma maneira de dividir a massa de pessoas que desejam estudar em vários grupos, de acordo com suas habilidades iniciais.
Depois de passar no teste, você recomendará cursos que colmatam lacunas de conhecimento, mas esses cursos já custam muito.

Openedx


Existem muitos recursos semelhantes com o mesmo conteúdo. Porque
A Microsoft distribui esse conteúdo por meio de seus parceiros de Educação, e esses, por sua vez, tornam o produto fora do conteúdo. Portanto, o conteúdo pode ser encontrado em uma dúzia de portais diferentes. Mas eles não me deram no EPAM, porque Não somos um parceiro educacional.

Existem cursos para cada seção listados na descrição do exame. Isso é um aperto, despejo cerebral, que você deve saber para passar nos exames.

Você se inscreve em um curso (inscreve-se), lê o conteúdo, realiza pequenas tarefas de teste e passa um pequeno teste no final. Não estou pronto para garantir que, depois de aprender o texto de todos os módulos, você será aprovado no exame, mas se você já conhece o Azure, esse recurso ajuda muito a reduzir o tempo de preparação. Além disso, um certificado é gerado no OpenEdx. O certificado parece sólido, com a assinatura do máximo, embora não esteja claro quanto esses certificados têm valor.

Na minha opinião subjetiva, os cursos com o OpenEDX são o mais próximo possível do que eles pedem nos exames da série AZ e, ao mesmo tempo, são gratuitos.

Documentação oficial


Este é um ótimo lugar para estudar como só isso é a fonte oficial da verdade.

Você pode ler, assistir e até tentar repetir o que é mostrado na documentação por horas e até dias inteiros. Eu vejo 3 problemas:

  1. Falta de trabalho de laboratório. Este é um enorme sinal de menos, porque em muitas pessoas, a memória “muscular” é melhor que a memória visual.
  2. Documentação desatualizada localmente (ou pelo menos capturas de tela). A documentação é enorme. Por exemplo, na última vez que verifiquei, havia 9000 fotos, que não foram mencionadas na documentação. É verdade que havia 50 mil desses referenciados, mas muitos precisavam ser atualizados. Com texto, a mesma coisa.
  3. Às vezes, a documentação simplesmente não tem respostas para as perguntas necessárias e a única maneira de encontrar uma solução é fazê-lo, tentando todas as opções.

Mas todos esses três problemas ainda não superam o fato de que a documentação é a principal fonte de conhecimento teórico .

Minha maneira recomendada de estudar e passar nos exames e certificação


Se você é gerente de projeto usando o Azure, recomendo o seguinte conjunto de recursos e etapas de aprendizado:
  1. OpenEdx - AZ-900 Materiais preparatórios
  2. Provavelmente, você tem muito mais para o google.
  3. Exame AZ-900 (um exame que confirma que você conhece todos os principais conceitos do Azure)


Se você é ITPro
  1. OpenEdx - Uma visão geral dos serviços básicos. O trabalho de laboratório pode não ser aprovado, mas a leitura do conteúdo estará correta
  2. OpenEdx- AZ-103 Materiais preparatórios
  3. No RoleIQ Pluralsight -> RoleIQ -> Administrador do Microsoft Azure (faça testes para verificar suas habilidades)
  4. Exame AZ-900 (um exame que confirma que você conhece todos os principais conceitos do Azure)
  5. Exame AZ-103


Se você é um desenvolvedor (e conhece .net / C #)
  1. Cursos sobre OpenEdx - Uma visão geral dos serviços básicos é necessária aos desenvolvedores, como nem todos os projetos podem ser realizados no PAAS puro, mas você ainda precisa entender os serviços básicos de infraestrutura, porque os mesmos AKS se baseiam, afinal, em VMs conectadas a Vnets com Load Balanced, IP público, grupos de segurança de rede etc. .
  2. OpenEdx - AZ-203 Materiais preparatórios
  3. No RoleIQ Pluralsight -> RoleIQ -> RoleIQ Pluralsight -> RoleIQ -> Desenvolvedor do Microsoft Azure (faça testes para verificar suas habilidades)
  4. AZ-900 Opcional, mas útil (um exame que confirma que você conhece todos os principais conceitos do Azure. )
  5. Exame AZ-203

Se você não conhece C # /. Net, será mais difícil passar, mas é possível, porque todas as linguagens do tipo C são muito semelhantes e você pode olhar para a API sem conhecer o C #.

Se você é um arquiteto (solução ou qualquer outro tipo). Esse caminho será o mais longo, mas "Não há caminhos reais para a geometria".
  1. OpenEdx - Uma visão geral dos serviços básicos. O trabalho de laboratório pode não ser aprovado, mas a leitura do conteúdo estará correta
  2. OpenEdx - AZ-300 / AZ-301 Materiais preparatórios
    • OpenEdx - AZ-103 materiais preparatórios (Não é necessário, mas muito útil, porque você pode olhar para os mesmos serviços de um ângulo diferente)
    • OpenEdx - AZ-203 materiais preparatórios (Não é necessário, mas muito útil, porque você pode olhar para os mesmos serviços de um ângulo diferente)
  3. Eu recomendo fazer o exame AZ-900. (Um exame confirmando que você conhece todos os principais conceitos do Azure. Para arquitetos, este é um ponto de verificação onde você pode ter certeza de que está no caminho certo)
  4. No RoleIQ Pluralsight -> RoleIQ -> RoleIQ Pluralsight -> RoleIQ -> RoleIQ Pluralsight -> RoleIQ -> Microsoft Azure Solution Architect (faça testes para verificar suas habilidades)

    • RoleIQ Pluralsight -> RoleIQ -> RoleIQ Pluralsight -> RoleIQ -> Desenvolvedor do Microsoft Azure (opcional, mas a prática adicional não prejudicará)
    • RoleIQ Pluralsight -> RoleIQ -> Administrador do Microsoft Azure (opcional, mas a prática adicional não será prejudicial)
  5. Explore o Centro de Arquitetura do Azure
  6. O exame AZ-300/301 é obrigatório, pois não há um significado particular para entregar um sem o outro.
