倾倒垃圾时，最主要的是不要开始看着它。 第一部分？

不久前，我的朋友高兴地拍打着绑在尾巴上的空罐零食，我的朋友们通过了一个垃圾桶，看到旁边站着一袋可疑的整洁外表。 看看里面，他们决定也让我看看。 包内装有几乎完整的iRU Stilo-3014COMBO笔记本电脑。

谁也对在引擎盖下寻找并在现代环境中尝试感兴趣？


简短的外部检查和随后的搜索显示，在少女时代，这是一台MiTAC 8080笔记本电脑，并提供了服务手册（请查看elektrotanya或亲自撰写）。

对于初学者-安装数据：

  • CPU：采用Micro-FCPGA封装的Intel Pentium M（Banias）处理器。 在这种情况下为1.4 GHz。
  • 频率合成器：ICS950810。
  • 北：Montara-GM（2个DDR内存插槽，在266 MHz时已安装256 MB，集成视频）。
  • 南：ICH4-M（PCI，IDE，LPC，USB总线）。
  • 到S-Video接口的电视信号整形器：CH7011A。
  • 超级I / O控制器：PC87393（ISA总线，串行/ IR和并行端口）。
  • ISA总线上的可选微控制器：Hitachi H8 / F3437，用于维修键盘，PS / 2端口，温度传感器ADM1021A和风扇。
  • LAN 10/100：VT6105。
  • 传真/调制解调器：Billonton MDC56S-I 56Kbps传真调制解调器。
  • PCMCIA和读卡器控制器：CB710。
  • IEEE1394（FireWire）控制器：VT6307L。
  • AC'97代码：STAC9750。
  • 1个miniPCI插槽，准备安装无线模块。
  • 硬盘：从20到40 GB，以及两组硬件均缺失。
  • DVD / CD-RW驱动器。
  • 显示：14.1英寸TFT 1024x768，雾面.QD141X1LH03-MP01或N141X6-L0
  • 电池：锂离子电池，14.8 Vol。 （4S），4400 mAh（2P？），已耗尽。
  • BIOS：4MB，Insyde软件。
  • 预装Windows XP Home（应该在丢失的HDD上）。

据我了解，填充在当时很典型。 例如，直到2012年，我一直使用98和XP以下操作系统的计算机在处理器（P III）上性能较弱，而SDRAM却是后者的三倍。 我不能说他比目前使用3Gb DDR3的i3笨拙得多。
其中的功能-几乎是方形的设计，用Fn键代替Ctrl'a，在显示器的两个表面上都有电源指示灯，并且在前边缘有一个小键盘，可在不打开笔记本电脑本身的情况下播放音频CD。

在查看了复位信号电路（所有复位均已发生）和电源（如果没有启动，则没有像MAX1907这样的PU505转换器向处理器内核供电）之后，我决定尝试“延迟”该稳定器的启动。 将一个0.1 F电容器焊接到PR61电阻电路（先前稳定器的PGOOD就绪输出和SYSPOK许可输入）可以大大提高成功启动的频率。 使用1微法拉的电容器，启动变得稳定。 这种不稳定可能是由于该稳定器的每个臂中的三个关键晶体管之一缺少“火花”引起的：





现在-轮到研究BIOS了，但是并没有显示特别丰富的设置或服务信息。 在BIOS级别，上网本是五到八年前。

可能的下载源是HDD，CD / DVD，LAN，USB-FDD。 正如我所读到的那样，由于将USB闪存转换为USB-FDD需要额外的动作，并且HP提供的最简单的实用程序没有产生预期的效果，因此我从带有多个实时系统的便携式HDD安装了磁盘。 由于Windows PE不存在-我将自己限制为使用CheckIt和antiX-17_386-full的Free DOS，这是我所知道的最不需要的现代发行版。
CheckIt成功地测试了我当时想到的所有内容。 除键盘外，几个键不起作用，几个键产生幻像点击。

跟随触摸板信号（已指定PS / 2接口）后，我开始组装RP36电阻器，该电阻提供PS / 2线的“上拉”功能。 但是，尝试使用相邻的KM_DATA和KM_CLK线来连接外部键盘失败。 也许-它已保留，没有在H8 / F3437服务微控制器的当前固件中激活。 将外部键盘切换到触摸板线会导致它们发生冲突（尽管在工作中，带键盘的条形码扫描仪可以并行工作）。 如果您不使用外部USB键盘，则可以选择“触摸板+越野车内置键盘”或“外部键盘”。



另外，在查看电路和检查电路板的过程中，发现了一个非焊接的驱动器，其位置标识为U515（在PCMCIA插槽框架下），并且RS-232信号输出到控制点。



当然，您可以深入/ sys /类来搜索和测试合适的键盘或PS / 2和ttyS。 但这需要更多的实质性兴趣和对这个问题的关注。
