Présentation de 3CX V16 Update 4 et du WebMeeting FQDN 3CX unifié

Juste avant les vacances, nous avons publié la prochaine mise à jour 4 de 3CX V16! Nous avons également un nouveau nom universel MCU 3CX WebMeeting et de nouveaux types de stockage pour les sauvegardes et les enregistrements de conversation 3CX. Considérons tout dans l'ordre.

3CX V16 Update 4


La prochaine mise à jour 3CX propose un choix de périphériques audio dans le client Web, la version finale de l'extension 3CX pour Chrome et de nouveaux types de stockage de sauvegarde. De plus, la mise à jour 4 a reçu un certain nombre d'améliorations de stabilité et de sécurité apportées par les développeurs pendant la phase de test.

Comme vous le savez , la mise à jour est apparue comme une extension 3CX pour Google Chrome, qui implémente un softphone VoIP basé sur un navigateur. La version finale de l'extension a déjà été publiée dans le Chrome Web Store . Le softphone Web vous permet de recevoir des notifications lors des appels, même si le navigateur est réduit ou même fermé, et intercepte également les clics sur les numéros des pages Web - et les compose directement ou via un téléphone IP connecté.

Dans le Chrome Store, recherchez le mot «3CX» et installez l'extension dans le navigateur. Connectez-vous ensuite à votre client Web 3CX sous votre compte et cliquez sur «Activer l'extension 3CX pour Chrome». L'extension nécessite l'installation de 3CX V16 Update 4 et de Google Chrome V78 et supérieur. Si vous avez installé l'ancienne extension Click to Call 3CX, vous devez la désactiver. Les utilisateurs de V16 Update 3 et versions antérieures doivent d'abord installer Update 4, puis recharger la page avec le client Web ouvert, afin que l'option d'activation de l'extension apparaisse.



La mise à jour 4 a également introduit une sélection de périphériques audio PC dans le client Web 3CX (et, par conséquent, une extension pour Chrome). Vous pouvez sélectionner des périphériques audio pour «Dynamics» (d'où vous entendez la voix) et «Speakerphone» (d'où vous entendez l'appel). Ceci est très pratique si vous utilisez un casque - vous pouvez maintenant émettre des appels vers des haut-parleurs externes et entendre qu'ils vous appellent. Le choix des appareils audio se fait dans la section du client Web "Paramètres"> "Personnalisation"> "Audio / Vidéo".



L'installation de la mise à jour se fait comme d'habitude - dans l'interface 3CX, allez dans la section «Mises à jour», sélectionnez «v16 Update 4» et cliquez sur «Télécharger la sélection».

Vous pouvez également installer une nouvelle distribution v16 Update 4:

  • Windows: 3CX v16 Update 4 .
  • Linux: 3CX v16 sur Debian 9 ISO - Dans l'écran " 3CX Update Channel ", sélectionnez "Stable".

Journal des modifications complet dans cette version.

FQDN unique pour 3CX WebMeeting


Cette semaine, nous avons également effectué un ajustement utile que les administrateurs système apprécieront - le nom de réseau universel pour le service WebMeeting 3CX est «mcu.3cx.net». Si vous disposez d'un réseau sécurisé, vous ne pouvez ouvrir ce nom de domaine complet que dans les paramètres du pare-feu. Désormais, vous n'avez plus besoin de connaître et d'ouvrir chaque adresse IP séparément. Le nouveau nom de domaine complet est également pratique pour hiérarchiser le trafic entre vous et WebMeeting.

Vous pouvez savoir quelles adresses correspondent à «mcu.3cx.net» à l'aide de la commande standard nslookup mcu.3cx.net .



Si un serveur est temporairement indisponible, son adresse IP sera automatiquement supprimée de la liste.

Nouveaux types de magasins d'enregistrement d'appels pris en charge


Nous voulons également attirer votre attention sur les nouveaux types de référentiels pris en charge pour les enregistrements d'appels introduits dans la mise à jour v16 4 bêta. Il s'agit de SFTP, de partages Windows et de FTP sécurisé (FTPS et FTPES). Désormais, le serveur 3CX peut s'intégrer dans un environnement réseau prenant en charge une variété de technologies. Par exemple, SSH (Secure SHell) est l'un des protocoles de transfert de fichiers les plus populaires sur Internet, pris en charge par diverses plates-formes et offre une protection des données cryptographiques.


Pour utiliser le serveur SSH, accédez à Sauvegarde> Emplacement. et spécifiez le chemin d'accès et les informations d'identification (ou la clé de serveur OpenSSH). Si vous devez créer ou convertir une clé OpenSSH, consultez ce guide . La configuration de votre propre serveur OpenSSH est décrite ici .

Le protocole SMB est familier à tous les administrateurs Windows. Soit dit en passant, il est également pris en charge avec succès sur les périphériques NAS, Raspberry Pi, Linux et Mac (Samba).



Son utilisation est tout aussi simple - spécifiez le chemin des boules SMB et les identifiants d'accès.
Soit dit en passant, si vous êtes confronté à la tâche de synchroniser des copies de sauvegarde ou des enregistrements de conversations entre plusieurs serveurs, nous vous recommandons d'utiliser l'utilitaire Linux rsync. En savoir plus sur son utilisation dans cet article.
