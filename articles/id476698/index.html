Bagaimana Apple Membunuh Teknologi Web


Perusahaan telah membuatnya sangat sulit untuk menggunakan teknologi web pada platform-nya, dan berharap para pengembang akan bertahan.

Bahasa pemrograman untuk membuat situs web sering digunakan untuk menulis aplikasi. Ini sebagian besar disebabkan oleh perangkat lunak yang memungkinkan pengembang untuk "menggunakan kembali" kode yang mereka tulis untuk web dalam produk yang mereka buat untuk sistem operasi seperti Linux, Android, Windows dan macOS.

Tetapi Apple memiliki alasan untuk tidak menyukai penggunaan kembali teknologi web. Dia ingin App Store diisi dengan aplikasi eksklusif, tidak tersedia di setiap platform. Dengan perubahan kebijakan baru-baru ini , perusahaan telah membuatnya sedikit lebih sulit bagi pengembang untuk menerbitkan aplikasi berbasis web.


Artikel ini disiapkan dengan dukungan Perangkat Lunak EDISON, yang mengembangkan sistem pemeriksaan medis elektronik dan juga mendukung perangkat lunak .


Perlahan, App Store mulai menolak aplikasi yang dibuat menggunakan alat populer bernama Electron, yang memungkinkan pengembang untuk membuat semua aplikasi mereka pada kode web. Beberapa aplikasi paling populer di App Store, seperti Slack, Spotify, Discord, dan WhatsApp, termasuk dalam kategori ini.

Dalam sebuah diskusi tentang Github , beberapa pengembang mengatakan bahwa untuk aplikasi mereka yang dibangun menggunakan Electron - yang telah disetujui di masa lalu - kegagalan telah muncul dengan penjelasan bahwa aplikasi ini "mencoba menyembunyikan penggunaan API pribadi", yang merupakan API yang dibuat untuk internal penggunaan Apple, bukan untuk pengembang pihak ketiga. Menggunakan API pribadi untuk membuat aplikasi publik biasanya tidak disetujui karena mereka dapat berubah atau pecah seiring waktu, dan Apple melarang aplikasi yang menggunakannya.

Elektron telah menggunakan API pribadi selama bertahun-tahun dan telah melakukannya tanpa masalah. API pribadi ini memungkinkan pengembang, misalnya, untuk secara signifikan meningkatkan konsumsi daya , sementara alat resmi Apple menurunkan pengalaman pengguna. Dalam sebagian besar kasus ini, Apple tidak memberikan alternatif nyata bagi pengembang yang ingin mengakses fungsi API pribadi ini.

Sekarang tidak mungkin bahwa ribuan pengembang yang telah membuat aplikasi mereka menggunakan Electron akan dapat merilis pembaruan untuk mereka kecuali Electron merilis perubahan signifikan dalam implementasinya.

Pengembang dapat mendistribusikan aplikasi mereka dari situs web mereka, mengundang pengguna untuk mengunduhnya secara langsung. Tetapi ini berarti mengabaikan fitur seperti mekanisme pembaruan otomatis di App Store dan sinkronisasi iCloud. Dan metode akses langsung ke pembeli ini dapat segera diblokir, karena persyaratan notaris kontroversial Apple mungkin memerlukan revisi.

Apple memiliki sejarah kemajuan teknologi web yang tertunda pada platformnya. Di iOS, Apple tidak mengizinkan browser pihak ketiga yang sepenuhnya independen , mengharuskan semua aplikasi untuk menggunakan browser Safari mereka saat merender konten web. Meskipun browser seperti Chrome dan Opera tersedia di App Store, mereka harus menggunakan mesin browser Safari untuk merender halaman web, bukan milik mereka. Ini berarti bahwa Apple memiliki monopoli tentang bagaimana pengguna iPhone dan iPad mendapatkan akses ke web. Untuk mendorong pengembang membuat aplikasi iOS asli, alih-alih menggunakan teknologi web, Apple mengabaikan bagian populer dari spesifikasi web terbuka yang diterapkan oleh browser lain.

Praktik anti-persaingan halus Apple tidak terlihat sangat terisolasi, tetapi bersama-sama mereka membentuk strategi yang jelas.

Misalnya, teknologi WebRTC memungkinkan Anda melakukan panggilan video di browser web tanpa perangkat lunak tambahan. Dia bekerja di alat seperti Google Meet. Tetapi Apple sangat lambat mengimplementasikan spesifikasi , mengabaikan fungsi-fungsi utama, dan teknologi ini tidak berfungsi ketika tertanam dalam aplikasi .

Apple juga menggagalkan standar yang muncul yang disebut Progressive Web Apps (PWA), yang, seperti Electron, memungkinkan pengembang untuk membuat aplikasi untuk desktop dan perangkat seluler - memperkenalkannya sedemikian rupa sehingga terlalu tidak konsisten untuk memungkinkannya mengandalkan. PWA tidak memiliki masalah yang sama jika pengguna membuka aplikasi di Chrome atau Firefox, tetapi pengguna iPhone dan iPad tidak dapat menginstal browser pihak ketiga, yang membuat teknologi berbasis PWA mengalami kegagalan.

Pengembang menggunakan teknologi seperti Electron dan PWA karena mereka menyediakan pembaruan lebih cepat pada platform yang berbeda tanpa banyak basis kode yang berbeda. Beberapa berpendapat bahwa ini mengarah pada aplikasi berkualitas rendah, tetapi saya akan mengatakan bahwa alternatifnya adalah kurangnya aplikasi secara umum atau aplikasi yang jarang diperbarui, karena melayani produk Windows, Mac, dan web yang unik sulit dan mahal. Apple baru-baru ini merilis kerangka kerja bersaing yang disebut Catalyst , yang memungkinkan pengembang aplikasi iPad dengan cepat meluncurkannya di macOS, alat yang hebat untuk pengembang yang hanya berfokus pada pengguna Apple, tetapi bukan mereka yang membuat aplikasi lintas platform.

Metode anti-persaingan terselubung Apple tidak acak, tetapi mereka membentuk strategi yang jelas: membuat pembuatan aplikasi berbasis web di platform Apple begitu menyakitkan sehingga pengembang tidak akan memikirkannya. Sekarang karena App Store tidak menerima aplikasi yang dibangun dengan Electron, pengembang cenderung menemukan cara-cara kreatif untuk hal ini, tetapi Apple diatur untuk terus-menerus memainkan permainan kucing dan mouse, karena berencana untuk memiliki kontrol lebih besar atas aplikasi mana yang dapat berjalan pada platform. di masa depan

Jenis-jenis perubahan ini dapat dibuat atas nama privasi atau keamanan, tetapi kenyataannya adalah bahwa argumen terlihat lemah ketika pengguna dan pengembang tidak punya pilihan karena Apple mengendalikan platform, mesin browser, dan metode distribusi. Terlepas dari pendapat Anda tentang kualitas aplikasi pada Electron, pilihan itu penting.

Kontrol Apple atas ekosistem aplikasinya adalah jenis monopoli baru yang sulit dipahami dan sulit diatasi oleh anggota parlemen karena tidak ada jalan keluar dari pembatasan ini ketika perusahaan mengendalikan metode distribusi dan platform itu sendiri.
