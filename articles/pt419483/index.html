PowerShell e Shift + Ins, ou como obter a velocidade Hermes ao trabalhar com GPP


A área de transferência e o PowerShell irão acelerar, mas não serão cegos.


Usar o GPP para adicionar arquivos de registro é assustador e inconveniente - todos esses ramos do registro, tipo de chave, valores ... Especialmente se os ramos e valores forem muito bons. Mas existem alguns hackers que podem acelerar significativamente o trabalho com políticas de grupo.


Obviamente, você pode travar um script de logon com o comando import da ramificação do registro. Mas este não é o nosso método.


O primeiro caminho. Um pouco complicado


A primeira opção é criar seu próprio modelo de GPO. Este método é especialmente conveniente se você precisar alterar os parâmetros de valor, dependendo do usuário. Se você não quiser aprender os princípios da formação de modelos, basta exportar o arquivo reg "correto" e convertê-lo em um arquivo de modelo usando o script REG_2_ADMXL.vbs publicado na galeria de scripts Technet.


Suponha que desejamos facilitar a vida dos usuários, parafusando vários mecanismos de pesquisa no Internet Explorer da seguinte maneira:


  • se você inserir o texto da consulta na barra de endereço " g ", a consulta seria pesquisada no Google;
  • " Y request text" - em Yandex;
  • e " w query text" na Wikipedia russa.

Para fazer isso, basta preparar o arquivo de registro: 


Windows Registry Editor Version 5.00 [HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\SearchUrl\W] @="http://ru.wikipedia.org/wiki/:Search?search=%s" [HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\SearchUrl\Y] @="http://www.yandex.ru/yandsearch?rpt=rad&text=%s" [HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\SearchUrl\G] @="http://www.google.com/search?q=%s"

Converta-o em um modelo com o seguinte comando: 


 cscript REG_2_ADMXL.vbs IE-search.reg Ru-ru IE-search.admx

O modelo resultante e a pasta com o arquivo de idioma serão lançados no diretório de modelos. Agora, nossa política aparecerá no snap-in de gerenciamento de GPO.



Modelo instalado.


O mecanismo não é muito conveniente em termos de manutenção, mas em princípio funciona. E não conduza nada manualmente.


Outra opção seria usar os cmdlets do PowerShell para trabalhar com o GPO. Por exemplo, para restaurar o item de menu de contexto "Enviar" para os usuários, o conjunto de cmdlets será o seguinte: 


 Import-module -Name GroupPolicy New-GPO -Name SendTo Set-GPRegistryValue -Name "SendTo" -key "Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders" -ValueName SendTo -Type ExpandString -value "C:\Users\Default\AppData\Roaming\Microsoft\Windows\SendTo" Set-GPLink -Name SendTo -Target "ou=Users,dc=domain,dc=com" -LinkEnabled Yes

Parece que a opção para apenas uma chave. Se houver muitas chaves, por conveniência, você precisará reinventar a roda e converter novamente o arquivo de registro em um conjunto de cmdlets do PowerShell.


Você pode ler mais sobre como trabalhar com políticas de grupo e como automatizar tudo isso em um dos artigos anteriores, “ Mergulhando em modelos e domando GPOs do Windows ” . Bem, eu vou falar sobre o meu método favorito.


O segundo caminho. Copiar - Colar (quase)


Como você deve ter notado (eu não percebi isso imediatamente), ações em objetos, incluindo operações padrão da área de transferência, estão disponíveis no snap-in Gerenciamento de Diretiva de Grupo.



O menu de contexto nas configurações do registro GPP.


Se você copiar um objeto para a área de transferência e colá-lo, esse objeto é um arquivo xml. Isso significa que o arquivo pode ser pré-gerado e adicionado ao snap-in sem preencher manualmente os campos.


Para isso, Malcolm McCaffery escreveu um roteiro especial. Ele gera um arquivo xml com base no arquivo de registro exportado. O script pode ser encontrado no blog do autor.


O script do autor contém chamadas extras para a função Convert-Reg2Xml e um bloco de parâmetros extra. Esses erros são facilmente corrigidos durante uma execução de teste através do PowerShell ISE. No entanto, por precaução, joguei a versão corrigida no pastebin . Há também uma bifurcação desse script no github e até um serviço online que faz o mesmo.

Usar um script é extremamente simples. 


 Convert-Reg2Xml -regPath input.reg -xmlPath output.xml

O xml resultante pode até ser simplesmente arrastado para o campo para as configurações do registro.


Vamos analisar um exemplo específico. Primeiro, exporte o ramo de registro “correto”. Neste exemplo, faremos uma política para permitir a exibição de arquivos, pastas, extensões ocultas e, ao mesmo tempo, faremos o auto start do ctfmon.exe para alterar perfeitamente o layout.

Com destreza adequada, o processo leva menos tempo do que a leitura deste texto.


Para maior comodidade, você pode adicionar um item para arquivos de registro no menu de contexto para converter arquivos em geral em dois cliques.


Será suficiente criar um novo tipo de ação para objetos do tipo regfile e gravar o caminho no script desejado. Para sua conveniência, compilei isso, é claro, em um arquivo de registro: 


 Windows Registry Editor Version 5.00 [HKEY_CURRENT_USER\Software\Classes\regfile\shell\convert2xml] @="convert2xml" [HKEY_CURRENT_USER\Software\Classes\regfile\shell\convert2xml\command] @="powershell.exe -File C:\\temp\\script.ps1 %1"

O caminho para o script é especificado na última linha.


Para que o script resolva isso, você deve adicionar as seguintes linhas: 


 $output=(Split-Path -Path $args[0]) + "\output.xml" Convert-Reg2Xml -regPath $args[0] -xmlPath $output

Agora, quando você chama o menu de contexto, outro item será exibido; quando você clica nele, o arquivo xml necessário é exibido ao lado do arquivo de registro.



Novo item de menu de contexto.


Obviamente, para fazer alterações simples no registro de usuários e computadores, esses mecanismos são supérfluos. Mas, para configurar um grande número de parâmetros, esses hackers são muito convenientes.
