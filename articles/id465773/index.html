Memperbarui laptop dengan Windows 10 1903 - dari batu bata hingga hilangnya semua data. Mengapa pembaruan bisa lebih dari sekadar pengguna?

Microsoft Corporation pada versi terbaru dari sistem operasi Win10 menunjukkan kepada kita keajaiban dari kemungkinan pembaruan. Setiap orang yang tidak ingin kehilangan data dari pembaruan 1903 diundang ke cat.

Beberapa poin yang jarang diperhatikan dalam mendukung Microsoft adalah asumsi penulis artikel, diterbitkan sebagai hasil percobaan, dan tidak mengklaim dapat diandalkan.

  1. Ada daftar aplikasi yang jelas akan bertahan setiap pembaruan. Beberapa aplikasi lawas mungkin mematahkan pembaruan karena fitur yang tidak terdokumentasi.
  2. Windows 10 secara aktif mempromosikan konsep bahwa penguji perangkat lunak terbaik adalah penggunanya.
  3. Jika Anda secara tidak sengaja bekerja dengan banyak koleksi multimedia dan perangkat seluler dari Microsoft, sistem mungkin runtuh karena algoritma pengindeksan tidak berdokumen

Jarang disebutkan informasi Wikipedia tentang UWP

Baca hanya untuk pengembang hardcore
Universal Windows Platform (Universal Windows Platform, disingkat UWP) adalah platform yang dibuat oleh Microsoft dan pertama kali diperkenalkan di Windows 10. Tujuan platform ini adalah untuk membantu membuat aplikasi universal yang berjalan pada Windows 10 dan Windows 10 Mobile tanpa mengubah dalam kode. Ada dukungan untuk membuat aplikasi seperti itu di C ++, C #, VB.NET dan XAML. API diimplementasikan dalam C ++ dan didukung dalam C ++, VB.NET, C #, F # dan JavaScript. Dirancang sebagai ekstensi untuk Windows Runtime (platform yang diperkenalkan pada Windows Server 2012 dan Windows 8), ini memungkinkan Anda untuk menjalankan aplikasi pada berbagai platform perangkat keras.


Jadi, informasi teoretis dibangun. Mari kita lanjutkan berlatih.

Saya membeli laptop baru di bawah 10-ku.

Saya terkejut bahwa setelah menghubungkan hard drive kedua, file media diindeks untuk waktu yang sangat lama. Untuk bekerja dengan multimedia di perangkat eksternal, lama saya menginstal Zune player. Sistem mulai diperbarui tidak pada tempatnya. Sebagai hasilnya, dengan pembaruan 1903, saya diizinkan memilih waktu untuk pembaruan.

Saya memilih ...

Biasanya, Windows 10 memperbarui ketika melihat pembaruan. Tapi! Pembaruan 1903 berskala besar, dan setelah tiga jam beroperasi, komputer mulai menunjukkan hal-hal yang tidak pantas.

Saya mulai menginstal pembaruan - Saya kehilangan pengguna. % Nama Pengguna% .0001 ...
Ada nama pengguna, dan setelah reboot itu berubah. Ternyata ini adalah reaksi terhadap media player.

Ada dua disc. Satu sistem, yang lain untuk data.

Disk kedua ternyata menjadi batu bata.



Ini berarti bahwa untuk alasan yang tidak diketahui, dari awal dan akhir disk dari snap-in Windows, itu terputus oleh megabita di bawah sistem file yang tidak dikenal.

Saya melihat apa yang terjadi.

Menjadi perlu untuk menjalankan snap-in untuk menghapus perubahan ini.
Tetapi yang terburuk adalah karena media player, pembaruan tidak dapat merekam
pengaturan sistem pengguna. Mungkin tidak ada yang memikirkannya.



Akibatnya, pembaruan menyalin file pengguna ke pengguna baru, dan sekarang komputer tidak akan dapat masuk ke jaringan karena kenyataan bahwa tidak ada pengguna di domain, registri telah terbang, karena banyak program, sumber daya, dan ikon dipertajam di bawah nama pengguna.

Anda harus mengganti nama pengguna di registri secara manual, instal ulang
bagian dari program dan bersihkan ribuan file yang direferensikan
sumber daya.

Inilah pemainnya untuk Anda - ia dapat membuang pembaruan!
Begitu banyak untuk pembaruan - itu membuang sistem.

Tetapi registri masih rusak!

Dan Microsoft tidak memiliki editor yang baik (atau, lebih baik, mekanisme kembalikan aplikasi).

Dan tombol mulai - aplikasi UWP - menghilang selamanya setelah mencoba mengembalikan nama pengguna ke registri.

Beberapa kata di akhir artikel.

  1. Jika bukan karena struktur drive C, maka kemungkinan besar akan ada batu bata. Jika hanya ada satu disk, kemungkinan kehilangan data akan meningkat berkali-kali lipat.
  2. Pembaruan menghancurkan catatan domain, program harus dikonfigurasi ulang, bahkan Visual Studio dari Microsoft tidak mengalami serangan seperti itu.
  3. Secara eksperimental telah ditetapkan bahwa aplikasi UWP masih menyimpan informasi pengguna di suatu tempat, tetapi tidak ada cara yang efektif untuk bekerja dengan informasi pengguna UWP, apalagi, ada kecurigaan bahwa karena fakta bahwa pengembang untuk Android dan iOS tidak terburu-buru untuk porting mereka. aplikasi untuk Windows Mobile, standar tidak akan didukung dan dikembangkan di masa depan.
