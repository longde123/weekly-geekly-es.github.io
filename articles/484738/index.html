Tutorial de integración continua para Laravel 6 en Google Cloud Run

Hola a todos! En previsión del inicio de los cursos "Framework Laravel" y "Backend-developer in PHP" , continuamos compartiendo material interesante sobre el tema. Vamos



En los últimos años, la popularidad de Laravel ha aumentado dramáticamente . La comunidad de Laravel es incluso de la opinión de que Laravel hizo que el proceso de trabajar en PHP sea más agradable en comparación con el dolor que era antes. Laravel 6 tiene algunas características nuevas y geniales . Obtener una URL de trabajo súper escalable para su aplicación lleva horas, si no días. Y crear algo como Kubernetes es una tarea desalentadora. Es en tales casos que Google Cloud Run está en la mejor luz, porque con él puede obtener una URL HTTP de trabajo para cualquiera de sus aplicaciones de contenedor en minutos.

Google Cloud Run no tiene servidor y está completamente administrado por Google. Obtiene súper escala, pago por segundo, URL de HTTP y su propia asignación de dominio.

Si desea ejecutar contenedores sin estado, Cloud Run es la forma más sencilla de hacerlo. En este artículo, explicaré en detalle cómo hacer que su aplicación Laravel 6 se ejecute en la nube de Google con Integración Continua (CI).

Requerimientos Requeridos


  • Está familiarizado con PHP / Composer y sabe acerca de Laravel (creo que de lo contrario no estaría aquí);
  • Sabes cómo usar Git desde la consola;
  • Su código CI / CD está alojado en GitHub y usted sabe cómo usarlo;
  • Está familiarizado con Docker, tal vez incluso con el ensamblaje de etapas múltiples;
  • Tiene una cuenta de Google Cloud en funcionamiento (le otorgan un préstamo sin intereses de $ 300 por 1 año, no hay razón para no iniciarlo).

Por qué Cloud Run es ideal para principiantes


Por dos razones, que describiré a continuación:

  • Aprenderá sobre las mejores prácticas y el software, como Docker y CI / CD.
  • El marco principal funcionará después de hacer clic en un botón, elegir dos cosas y cinco minutos de espera, y puede obtener una URL HTTP que funcione. Cuanto mas facil

7. Agregue docker y ejecute docker-compose para ejecutar la aplicación localmente

Ahora agreguemos Docker y hagamos Docker-compose para ejecutar la aplicación localmente sin PHP o servicio artesanal .

Esta parte del tutorial está inspirada en el artículo de Nicolas Laravel en Google Cloud Run .

Si quieres saber más sobre Docker y Laravel, mira este artículo .

Para comenzar, ejecute los siguientes comandos para actualizar la rama maestra, ya que agregamos un nuevo archivo de flujo de trabajo utilizando la interfaz GitHub: 

 git checkout master git fetch git pull --rebase origin master # as we added the workflow file from github interface git checkout -b docker

Agregue la clave al archivo .env.example , .env archivo .env , como se muestra a continuación: 

 APP_NAME=Laravel APP_ENV=local APP_KEY=base64:DJkdj8L5Di3rUkUOwmBFCrr5dsIYU/s7s+W52ClI4AA= APP_DEBUG=true APP_URL=http://localhost

Dado que esto es solo una demostración, puede hacerlo, pero en aplicaciones reales, tenga cuidado con los secretos. Para aplicaciones listas para producción, deshabilite la depuración y otras cosas relacionadas con el desarrollo.

Agregue dicho Dockerfile a la raíz del proyecto: 

 FROM composer:1.9.0 as build WORKDIR /app COPY . /app RUN composer global require hirak/prestissimo && composer install FROM php:7.3-apache-stretch RUN docker-php-ext-install pdo pdo_mysql EXPOSE 8080 COPY --from=build /app /var/www/ COPY docker/000-default.conf /etc/apache2/sites-available/000-default.conf COPY .env.example /var/www/.env RUN chmod 777 -R /var/www/storage/ && \ echo "Listen 8080" >> /etc/apache2/ports.conf && \ chown -R www-data:www-data /var/www/ && \ a2enmod rewrite

Agregue el siguiente archivo a docker/000-default.conf : 

 <VirtualHost *:8080> ServerAdmin webmaster@localhost DocumentRoot /var/www/public/ <Directory /var/www/> AllowOverride All Require all granted </Directory> ErrorLog ${APACHE_LOG_DIR}/error.log CustomLog ${APACHE_LOG_DIR}/access.log combined </VirtualHost>

Después de eso agrega docker-compose.yml : 

 version: '3' services: app: build: context: ./ volumes: - .:/var/www ports: - "8080:8080" environment: - APP_ENV=local

Reduzcamos todo a lo principal.

Si está tratando de entender todo lo escrito aquí, puede ser exhaustivo, así que permítame contarle sobre las partes principales:

  • Utilizamos la imagen oficial de Docker Apache de PHP para ejecutar Laravel, tiene PHP versión 7.3.
  • Usamos una compilación de varias etapas para obtener dependencias con Composer, y luego las copiamos a la imagen principal del acoplador, que tiene PHP 7.3 y Apache.
  • Dado que Google Cloud Run requiere que el servidor web escuche en el puerto 8080, usamos 000-default.conf para configurar esto.
  • Para hacer que Docker-compose sea más fácil de ejecutar, ingrese solo un comando docker-compose up.
  • Ahora que ha leído hasta este punto, ejecute docker-compose desde la raíz y todo debería funcionar en http://localhost:8080 , en este caso comprenderá que Laravel 6 se ejecuta localmente en Docker.

8. Agregue una implementación al botón Google Cloud .

Ahora implementemos esta instalación de Laravel en Google Cloud Run de una manera simple. En esta etapa, se cree que organizó su solicitud de extracción desde la rama acoplable.

Ejecutemos los siguientes comandos: 

 git checkout master git fetch git pull --rebase origin master git checkout -b cloud-run-button

Agregue lo siguiente a su archivo readme.md :

### Run on Google cloud run

[![Run on Google Cloud](https://storage.googleapis.com/cloudrun/button.svg)](https://console.cloud.google.com/cloudshell/editor?shellonly=true&cloudshell;_image=gcr.io/cloudrun/button&cloudshell;_git_repo=https://github.com/geshan/laravel6-on-google-cloud-run.git)

Reemplace cuidadosamente la última parte de la URL HTTP de su repositorio, por ejemplo, si su repositorio está ubicado en la dirección a la que cambiará, luego confirme y presione .

Su solicitud de extracción debería verse así.

9. Implementar en Google Cloud Run

Después de combinar su solicitud de extracción, vaya a la página del repositorio y haga clic en el botón Ejecutar en Google Cloud .



Después de eso, si ha iniciado sesión en su cuenta de Google y tiene la configuración de la nube de Google con un proyecto, haga clic en el botón "Continuar". Puede que tenga que esperar un poco, después de esto:

  1. Elija un proyecto: elija un proyecto para implementar esta aplicación
    Elija una región: elija una región para implementar esta aplicación, generalmente elijo us-central-1
    Luego espere a que el contenedor se recopile y expanda, el proceso en mi caso se ve así:

Si todo va bien, en Google Cloud Shell verá las URL de HTTP en las que puede hacer clic para ver que su aplicación Laravel se está ejecutando, como se muestra a continuación:



Lo que sucedió arriba es esto:

  1. Después de elegir una región, el script creó una imagen del contenedor de Docker desde el Dockerfile en el repositorio;
  2. Luego introdujo la imagen ensamblada en el Registro de contenedores de Google;
  3. Después de eso, utilizando la CLI de gcloud, implementa la imagen ensamblada en Cloud Run, que devuelve la URL.
