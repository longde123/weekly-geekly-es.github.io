物联网的引入正在改变世界，使我们每个人都变得更好。

物联网解决方案实施的越多，就需要更多的数据进行敏锐的分析，甚至需要人工智能来迅速接收相关信息。 此类信息可让公司实时查明偏差和技术故障。 例如，汽车行业使用防撞，交通路线和非道路检测系统等智能技术改善了交通安全。

综上所述，物联网系统中的实时决策还受到成本，外形尺寸限制，数据传输延迟和其他标准的限制。 Google Cloud IoT尝试通过在终端设备上进行机器学习并引入Cloud IoT Core，Cloud IoT Edge，Edge TPU等服务来改变现有状况。



云物联网核心


Cloud IoT Core是完全可管理的服务，允许快速安全地连接，配置和接收来自许多设备的数据。 使用Cloud Pub / Sub，Core可以将分散设备中的数据合并到单个全局系统中。 如果与其他Google Cloud IoT服务结合使用，Core建议使用复杂的解决方案来实时收集，分析和可视化IoT数据。 反过来，这又允许创建多功能模型，以优化或预测对您的业务有价值的数据。

Cloud IoT Core支持常规的MQTT和HTTP协议，这使客户可以利用许多现有设备来构建其IoT系统。 这项服务可在Google基础架构中运行，该基础架构会实时自动扩展，并在需要时可以连接一百万个设备。

核心包含：

设备管理员
通过控制台或软件设置和管理单独的设备。 在连接上使用设备标识和身份验证机制。 您还可以配置每个设备的逻辑，并使用云远程控制它。

协议桥
它提供的最好的功能是用于组件负载平衡的连接端点，对MQTT和HTTP的本机支持，将遥测信息存储到Pub / Sub中。

Clout IoT Core收集发布在Cloud Pub / Sub中的数据以进行进一步分析。 使用Google BigQuery执行特定的分析，或使用Cloud Machine Learning Engine来应用机器学习。 之后，您可以使用Google Data Studio中的各种报告和信息仪表板来可视化结果。

此外，为了简化和加快各种设备与Core的连接，您可以利用Google IoT的附加配置服务。 它使客户和制造商都可以入职。 此服务使用硬件级别的可靠安全性（aka加密芯片或安全元件（SE））来安全地配置设备。 这样就可以在无人干预的情况下将数百万个设备提供给正确的内核。

云物联网边缘


扩展Google Cloud数据处理并在自动传输线，涡轮机械等边缘设备上使用机器学习的有用工具。 这样，设备可以使用其传感器数据实时运行并在本地预测结果。 Edge可以在基于Linux的操作系统上工作。 它包含两个运行时组件（Connect和ML）和ASIC芯片定制硬件加速器（TPU）。

  • 边缘连接-用于将边缘设备安全连接到云，软件和微程序更新，核心数据传输管理。
  • Edge ML-用于在机器上推断机器学习TensorFlow Lite模型。 好处：处理能力提高，等待时间短，多功能性等。
  • Edge TPU-用于在边缘运行TensorFlow Lite模型的AI芯片 高效，优化并具有多种用途。 如果与Cloud TPU一起使用，可能会是一个巨大的优势。

好处：

  1. 图像，视频，手势，声音和运动的边缘局部处理。 比将原始数据发送到云更有效。
  2. 无需发送大量（可能是机密的）数据。
  3. 使用JSON Web令牌验证边缘设备。 比TLS堆栈相互认证更有效。

Edge TPU的其他优势


对于TensorFlow Lite机器学习模型，专用集成电路Edge TPU芯片是新的必须。 它专门用于：

  • 在云中更快地进行机器学习准备。
  • 在边缘设备上更快地进行ML分析。
  • 使传感器能够做出本地实时智能决策。
  • 在边缘高效地启动精确的AI。

随着对要求机密性，低停留时间和高容量的互连设备的需求迅速增加，在边缘设备上启动AI模型变得越来越普遍。 Edge TPU是完美的选择，因为它以较低的物理资源和能源消耗提供了高效率。

Edge TPU开发板


提供所有所需边缘连接的基板。 您可以在模块（SOM）电路上安装或卸下系统，并将Edge TPU模块集成到设备中。





Edge TPU加速器


将Edge TPU协处理器添加到系统的USB设备。 配备Type-C连接器的Accelerator可以连接到任何基于Linux的系统，以实现快速ML输出。 该外壳包含Rapsberry Pi板连接器孔。



这样，您可以创建一个智能平台，该平台将快速使用传感器实时监控各种组件和整体制造效率。
