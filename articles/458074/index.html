Ir idioma: selección ORM

Go ganó fama como uno de los lenguajes de programación más simples, incluidas sus ventajas: facilidad para escribir y leer código, en la mayoría de los casos, soporte más simple para la base de código. Hablamos de varias ventajas de Go, gracias a las cuales en SimbirSoft lo usamos en varios proyectos altamente cargados con varias arquitecturas, tanto de servicio web como de microservicio (SOA).

Solicitud


Este lenguaje de sistema es bueno para la mayoría de las tareas de red, microservicios y macro servicios, servicios web, así como para herramientas del sistema; no sin razón, a muchos expertos de DevOps les encanta. Go encuentra su uso en Machine Learning, y algunos desarrolladores incluso escriben sitios web con él. En nuestra práctica, recurrimos a Go cuando desarrollamos CRM y productos de software separados para uso interno, por ejemplo, para compañías de seguros. El repositorio estándar de idiomas Go también tiene un paquete para el desarrollo móvil, este paquete aún es compatible, aunque hay herramientas más adecuadas para esto.

Go: desarrollo y reconocimiento


En Google, el desarrollo del lenguaje Go comenzó en 2007. Antes de eso, la compañía usaba Java, Python y C ++, sin embargo, tenían ciertos inconvenientes (en particular, alto consumo de memoria) o uso limitado. Entonces Google comenzó a desarrollar un nuevo lenguaje para sus necesidades.

¿Cuáles son los requisitos para el idioma?

  • Un archivo ejecutable (sin dependencias)
  • Compilación rápida
  • Tiempo de ejecución rápido (al menos a la par con Java y C ++)
  • Recolector de basura minimizando la fuga de recursos
  • Optimización para sistemas multiprocesador.
  • Mecanografía estática
  • Minimalismo
  • Ligereza sintáctica
  • Multiparadigness (imperativo, funcional, orientado a objetos)

Para aquellos que no trabajan con Go, recuerden brevemente la historia de su creación. Google comenzó a pensar en la especificación del idioma en 2007. La versión 1.0 tuvo lugar en 2012, y en 2015 se lanzó la versión 1.5, ya sin una sola línea de código C ++. A partir de ese momento, el lenguaje se compila, lo que nos permitió acelerar el desarrollo de tareas individuales en Go.

La edad promedio de reconocimiento de un nuevo idioma en la comunidad es de aproximadamente 10 años, pero Go ya es uno de los idiomas más populares. En junio de 2019, Go subió 15º en términos de popularidad en el mundo, según el índice TIOBE , agregando tres posiciones en un año. Entre las razones de su éxito están las siguientes:

  • Biblioteca estándar fuerte
  • Buena sintonización estándar
  • Las funciones más necesarias se presentan en el "cuadro", y las bibliotecas se dan a código abierto
  • Comunidad muy amigable

Ir herramientas


Uno de los beneficios de Go es su amplio conjunto de herramientas estándar. En nuestra opinión, hemos seleccionado algunas de las utilidades más interesantes (de hecho, hay muchas más):

  • benchcmp: comparación de referencia, rendimiento antes y después de los cambios
  • go cover - análisis de la cobertura del código por pruebas
  • godoc - generación de documentación basada en comentarios
  • Goimports: análisis de importación (eliminación de elementos no utilizados, adición de elementos faltantes)
  • gorename - refactorización
  • presente - ajuste de presentación
  • ir al veterinario: error frecuente
  • Go Test: ejecuta pruebas y puntos de referencia
  • vaya a arreglar: corrija los cambios al cambiar la API (antes de la versión 1.0)
  • go tool pprof - perfilado
  • Ir fmt - formateo de código por estilo de código
  • Detector de carrera: encontrar una carrera de datos en tiempo de ejecución

Competitividad


Otra fortaleza de Go es que es fácil escribir código competitivo en este idioma. Para hacer esto, simplemente use palabras clave antes de llamar a la función.

ORM en Go


Se cree que una de las debilidades de Go es ORM. Quizás esto se deba en parte al hecho de que el lenguaje es joven; en él, muchas herramientas necesarias apenas comienzan a desarrollarse. En particular, puede usar el paquete database / sql para trabajar con bases de datos. ¿Cuáles son los requisitos para el paquete?

  • Paquete general para todas las bases de datos sql
  • Portátil: no contiene ninguna característica en el campo de trabajar con bases de datos. Idealmente, todo debería funcionar correctamente si cambia el controlador.
  • Conversión de tipos Go a tipos de bases de datos estándar (debería ser posible procesar tipos específicos)
  • Contiene grupo de conexión
  • Seguridad de roscas para gorutinas

Consulta SELECT de muestra utilizando la base de datos / sql 


defer rows.Close() for rows.Next() { var message string err := rows.Scan(&message) if err != nil { log.Panic(err) } fmt.Println(message) } if err := rows.Err(); err != nil { log.Panic(err) }

Como puede ver en el ejemplo, se deberán realizar muchas acciones de plantilla cada vez. Por ejemplo:

  • cerrar objeto de fila: diferir filas. Cerrar ()
  • verificar errores err: = rows.Err (); err! = nulo desde no está claro por qué filas.Next () nos devolvió falso

De todo esto, quiero usar algo más universal y simple, por ejemplo, ORM.

Tipos de implementaciones de ORM


Cuando los desarrolladores se enfrentan a la elección de ORM para trabajar con Go, a menudo se guían por la calificación en el github. Cuando elegimos ORM para nosotros, Beego obtuvo la mejor calificación (18 mil "estrellas"). Sin embargo, en la práctica, ORM es parte del marco, por lo que la evaluación no es bastante objetiva (si selecciona solo ORM). También entre los líderes, Gorm, que se ha desarrollado desde 2013, admite transacciones y tiene muchas otras ventajas.



Sin embargo, como ha demostrado nuestra experiencia con Gorm, es posible que surjan ciertos problemas al usarlo, especialmente al probar: por ejemplo, la carrera de datos o la eliminación de una clave foránea. Podemos suponer las siguientes causas de tales errores en Gorm:

  • Se compromete inmediatamente en master
  • Etiquetas faltantes
  • Constructor sql impredecible

Por el momento, en nuestro trabajo, preferimos varios ORM que no utilizan la interfaz {}: estos son Kallax, Reform y Sqlboiler. Usualmente usamos el paquete Golang para migraciones y el repositorio de orm-bench con los puntos de referencia ORM en Go. Al mismo tiempo, no debe perseguir solo puntos de referencia. Más a menudo, la velocidad no es tan importante como la calidad del código y su funcionalidad.

Para resumir


Las ventajas del lenguaje Go para el desarrollador y para las empresas incluyen una mayor productividad del desarrollo, la capacidad de usar menos servidores, la facilidad sintáctica y un tiempo de capacitación más rápido para los nuevos desarrolladores, un gran conjunto de herramientas y una comunidad amigable. A pesar de que es un idioma joven, está en constante crecimiento y desarrollo. Las características de Go lo convierten en el lenguaje óptimo para una amplia variedad de proyectos, desde tareas de red hasta microservicios y blockchain.
