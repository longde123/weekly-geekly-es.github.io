报告书


它以Nicolai Josuttis的主题演讲“ C ++ 17-最大的陷阱”开始 。 尼古拉（Nikolai）是《 C ++标准库和C ++模板》一书的作者，并且是标准化委员会的成员已有近20年的时间。 他的报告可能是“未来与现代C ++”主题中最热门的报告之一。 该报告将介绍该语言的问题，其背后的体系结构，应归咎于谁以及该怎么做。


第二份报告来自英特尔的Ruslan Harutyunyan。 Ruslan-正在创建一个标准的C ++库，用于自动驾驶技术领域。 在“现代C ++分配器”报告中，我们了解了分配器的历史，分析了其设计中的错误，查看了标准库提供的现代分配器，并了解了它们解决了哪些问题。


在第三份报告中， “编译时优化。 我们将为编译器提供更多的信息，” DINS的Kirill Tikhonov将展示如何向编译器传递有关所使用的数据及其作用的信息：从单个编译单元，链接时间优化和静态初始化到节制和模板元编程。


响应式C ++是响应式编程教程。 我们将与Artec 3D的Alexander Schmidt一起，弄清楚这种方法（使用RxCpp）如何帮助我们从处理复杂的异步代码中抽象出来，同时简化其测试。


并完成了“仅移动C ++设计”的广播-著名演讲者Ivan Chukich的报告。 Ivan在贝尔格莱德大学数学系计算机科学博士学位系工作，专门从事编程语言的设计，并为KDE做出了贡献。 在此报告中，我们将介绍不可复制的仅移动类型（“线性类型”）和允许此操作的C ++功能的体系结构。 例如，查看来自C ++ 20或&&-限定函数的概念。


总计：新C ++标准的功能，优化和反应性。 拥有一个愉快的星期五所需的一切。
