如何在Windows的React Native上部署应用程序开发环境

美好的一天！


决定开始在React Native上开发应用程序时，我遇到了部署环境的问题。 今天，我想分享设置的经验。

当然， 官方网站上有详细的说明，但是仅遵循这些建议，进行所有设置非常困难。



因此，让我们开始：

节点，Python2，JDK


  • 安装NodeJS 。 在撰写本文10.11.0时，我具有最新版本
  • 安装Python2和JavaSE 。 二手jdk-10.0.2


反应本机CLI


  • 安装React Native CLI

npm install -g react-native-cli

Android开发环境


  • 下载并安装Android Studio

    开始后，选择一个自定义安装



    接下来，选中“ Android虚拟设备”框，并指定是否到Android文件夹-或将其保留为C：\ Users \％USERNAME％\ AppData \ Local \ Android \ Sdk
    最主要的是，途中不应有西里尔字母！
    尤其是这样，将来可能会出现问题。 例如，我的系统用户名位于西里尔字母“ C：\ Users \ Alexander”，并且在启动应用程序后Grandle找不到路径，因为该路径看起来像“ C：\ Users \ ???????????” \ ...“



    点击“下一步”。 我们保留建议的RAM数量，单击“下一步”并安装。
  • Android SDK-打开工作室并转到设置
    外观和行为→系统设置→Android SDK 。

    在“ SDK平台”选项卡上，打开“显示软件包详细信息”复选框，然后设置“ Android 8.0（Oreo）” ：

    • Android SDK平台26
    • 英特尔x86 Atom_64系统映像
    • Google API Intel x86 Atom_64系统映像



    现在选择“ SDK工具”选项卡，启用“显示软件包详细信息”复选框
    并安装软件包

    • “ 28.0.3”

    单击"Apply" 。


环境变量


  • ANDROID_HOME
    为ANDROID_HOME创建一个变量：



    • 变量名ANDROID_HOME
    • 可变值（可以在Android Studio中查看）


  • JAVA_HOME
    以此类推，创建JAVA_HOME变量：

    • 变量名称： JAVA_HOME
    • 变量值：C：\ Program Files \ Java \ jdk-（版本）

  • 更改系统变量Path：



    添加4个值：

    1. ％JAVA_HOME％\ bin
    2. ％ANDROID_HOME％
    3. ％ANDROID_HOME％\平台工具
    4. ％ANDROID_HOME％\值班员


我们创建应用程序并运行模拟器


  • 在控制台中转到对我们方便的文件夹，然后键入：
    react-native init MyTestProject
  • 我们在Android Studio中打开我们的项目，然后打开AVD管理器



    如果未下载AVD，请下载并运行
  • 接下来，转到应用程序文件夹中的控制台，然后键入：
    react-native run-android

    之后，我发现应用程序启动了。

本文使用了官方的React Native手册。
