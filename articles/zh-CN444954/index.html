开放视频管理系统回顾



视频监控长期以来一直用于解决各种问题。 在1950年代和60年代，它们是模拟闭路视频监控系统，CCTV（闭路电视）这个名字揭示了这一概念的含义。

即使在90年代，发展也没有停滞不前，也就是说，在1996年，Axis Communication公司开发了世界上第一台数字IP摄像机。 摄像机的技术特性似乎并不那么雄心勃勃，与模拟摄像机相比，它们看上去很苍白。 尽管如此，还是售出了大约10,000台第一台IP摄像机。

IP摄像机的进入是视频监控系统发展的起点，摄像机的出现要求出现与之配合使用的软件。

随着时间的流逝，出现了新的IP视频监控摄像机以及软件。 开发过程有两种方式：一些公司开发了可与任何IP摄像机配合使用的软件，另一些公司则生产了自己的摄像机以及专门用于其摄像机的软件。

这种开发方法将视频监视系统分为“开放”和“封闭”系统。 开放平台系统朝着与照相机无关的多特征支持方向发展，而封闭制造商则是封闭的-单厂商系统仅支持使用自己的生产设备进行工作。

但是，现在有这样一种趋势，即封闭系统开始寻求支持第三方设备，但是距离开放平台还很远。

此外，值得一提的是视频分析。 如今，它已成为VMS不可或缺的一部分，VMS正在迅速发展，可以促进并帮助运营商避免在庞大的视频信息流中迷失方向。 并且还可以解决业务问题，并且通常将视频转换为易于操作的数据。 因此，现代化的集成视频监控系统应支持分析工作，并支持与访问控制和建筑物管理系统的集成。

市场上有很多开放的视频监控平台。 为了对它们有所了解，我们将简要回顾一些VMS，并在VMS比较表中提供更多详细信息。



MileStone系统。 XProtect


Milestone公司自90年代末以来一直在开发用于IP视频监视系统的软件。 由于其活动的起源，它与视频监控设备生产的领导者Axis Communication紧密合作。

十年来，Milestone一直是基于IHS Markit的VMS市场的领导者，也是视频监控开放平台概念的创始人。

该产品线由XProtect软件组，七个版本和网络记录器的Husky系列代表。

该产品线涵盖了视频监控系统的各种需求：从简单的小型系统到复杂的，地理位置分散的系统，而对每台服务器或整个系统的摄像机数量没有限制。

要熟悉该软件产品，可以在8个设备上完全免费下载任何版本的试用版或使用Essential +产品的较新版本，而不受相机分辨率，连接用户，存储期限等的限制。 。



Milestone在驱动程序级别支持最多数量的IP设备，现在，它已来自各种制造商的7,000多种经过验证的模型以及大量现成的集成产品-视频分析（边界保护，面部识别，车牌识别，停车解决方案，商业分析，声音分析），访问控制系统，建筑物管理系统等。

要自定义功能，可以使用所有版本的免费SDK和/或更高版本的配置API。

Milestone具有非常简单的许可系统。 许可包括一个基本软件许可，与服务器硬件和设备许可无关。 设备的许可不限制其功能，并允许您使用“压缩”设备中的最大值：流（通道）数，I / O端口，声音支持，与存储卡一起使用，内置分析。 基本软件许可证不限于一种安装，可以不受限制地安装在各个位置和站点上。 在相同的基本许可证范围内重新连接设备不需要额外的许可证，例如，这使得执行两次输入成为可能。 在大多数情况下，将录像机连接到Milestone XProtect需要一个许可证，因此您可以将NVR连接到64个通道。 使用记录器的唯一缺点是缺乏对记录器上存档的直接访问。



Milestone XProtect对用户和同时连接的数量没有限制（一个版本限制为五个客户端连接）。 它支持三种类型的用户：专有用户，Windows和Active Directory。 为了使用VMS的方便和灵活，共有三个完全免费的客户端：胖客户端，移动客户端和Web客户端，使您可以处理实时的存档视频并响应系统中的警报。 使用移动客户端，您可以将带有摄像头的设备转换为IP摄像头，并广播视频，并可以将其记录到存档中。

对于系统的配置，使用了附加软件，它是客户端服务器软件，可以在服务器网络可用范围内的任何位置使用。

Milestone不开发自己的分析。 公司的理念是创建一个方便，开放的工具来集成和处理视频：查看，录制并确保其安全性和真实性，并且分析公司应寻求分析的发展，因此，大量的技术合作伙伴和集成解决方案在Milestone产品组合中。



为了扩展功能，Milestone XProtect在其产品组合中提供了多个模块，其中包括Transact-用于处理交易（文本信息）并将其与视频进行比较； LPR-车牌识别系统； Retail-用于与ERP系统集成的模块；以及ACS-与访问控制系统集成。

Milestone XProtect VMS设计为在Windows环境中使用，服务器和客户端（厚客户端）均如此。 它使用MS SQL保存配置和各种服务信息，默认版本为Express。 里程碑支持在虚拟环境中的工作。

基因技术公司。 全播


Genetec自90年代末开始运营，是视频监控系统的市场领导者之一。

Genetec是统一安全平台的开发商。 主要产品是安全中心和Omnicast。 后一个产品是视频监视系统本身，第一个是最高点，它是其他系统组件的集合体，并且在使用Security Center时会显示一些Onmicast系统功能。

因此，Omnicast分为三个版本，对系统中的摄像机数量，用户和其他功能有不同的限制。

Genetec能够满足视频监控系统的各种需求：从简单的小型系统到复杂的，地理位置分散的系统，对每台服务器或整个系统的摄像机数量没有限制，但对记录服务器的流量有限制因此，在安装大量摄像机时，可以为200 Mb / s的速度准备好使用许多记录服务器。



没有免费的Genetec版本，因此要熟悉该软件产品的唯一选择是索取演示版本。

Genetec Onmicast支持大约6000个IP设备和大量集成的视频分析解决方案（边界保护，面部识别，车牌识别，业务分析）访问控制系统，建筑物管理系统等。 可在Genetec网站上找到有关设备支持和集成解决方案搜索的信息，因为例如，只能在Security Center中提供支持，而不能在Omnicast中获得支持。



为了扩展功能，Genetec提供了SDK。

Genetec根据设备数量获得许可，并带有基本许可的链接。 支持的硬件功能取决于集成级别，可以支持多个通道，声音，I / O端口，与存储卡一起使用，板上分析。 该许可证与某些系统信息相关联，因此PC配置更改可能会导致其取消并需要重新激活。

根据版本，Genetec Omnicast对连接的用户数量有限制，仅在较高版本中没有限制。 Onmicast支持的用户帐户类型是一种-专有，使用安全中心时可以与Active Directory集成。

为操作员提供了胖客户端软件，没有移动客户端。 为了支持移动设备和Web访问，您将需要安装移动服务器作为Security Center的组件。 移动服务器是许可组件。 反过来，移动访问不仅将提供查看和处理来自世界任何地方的视频的能力，而且还将使用移动设备的摄像机作为IP摄像机。

Genetec没有开发自己的分析模块，并且有很多合作伙伴可以集成各种视频分析。

Genetec功能可以通过自己的模块扩展，这就是AutoVu车牌识别系统和Synergis门禁系统。 使用这些组件意味着存在安全中心。

Genetec还生产软件和硬件系统Streamvault。

Genetec Omnicast专为Windows（服务器和客户端）而设计。 要配置系统，请使用单独的软件。 它使用MS SQL存储各种服务信息，支持在虚拟环境中工作。

Luxriot。 埃沃


Luxriot是一家美国视频监控系统软件开发公司，除软件外，它还为服务器提供预安装的软件和用于模拟视频监控系统的采集卡。



Luxriot带有三种软件版本，可满足小型和大型视频监视系统（包括地理分布的监视系统）的需求。 在获得的许可范围内，系统对每个服务器或系统的摄像机数量没有限制。

您可以使用演示许可证申请来熟悉Luxriot产品，也可以使用该产品的免费版本。 免费版有很多限制，主要限制是：9个支持的通道，最大摄像机分辨率高达2Mp和每个系统一个用户连接。 每个需求可将免费许可证扩展为最多16个通道。



Luxriot VMS系列支持约6000个IP设备，并支持少量第三方视频分析解决方案和解决方案（如访问控制系统）的集成。 至于车载分析相机，Luxriot支持的制造商数量有限。

Luxriot许可基于渠道，并作为捆绑销售，用于较新版本和较旧版本中的每个渠道。 由于现在市场上的大多数IP摄像机都是多通道（配置文件）摄像机，因此在某些情况下，按通道许可会带来不便，并且在使用多通道设备时需要额外的许可。 Luxriot中的许可证与服务器硬件绑定在一起，因此，如果您更改配置，则可能需要重新激活。

尽管用于集成第三方解决方案的解决方案的选择很少，但是VMS功能可以通过其自己的分析模块进行扩展。 Luxriot开发了以下视频分析模块：外围保护，业务分析以及车牌和面部识别。



提供API / SDK以解决有关功能扩展和集成的问题。

用户连接仅在免费版本中受到限制。 支持两种类型的用户帐户：专有帐户和Active Directory。

运营商可以使用三种类型的客户端：胖客户端，移动客户端和Web客户端。 客户端应用程序是完全免费的，没有额外的许可。 在移动客户端设备上使用摄像头可以将视频广播和录制到系统中。

Luxriot设计用于Windows，其服务器部分以及操作员的主要软件-Windows和MacOS。 Luxriot使用专有数据库来保存配置和系统信息，并支持在虚拟环境中的工作。

网络Optix。 NxWitness /韩华。 Wisenet波


该VMS有两种名称，来自各个供应商。 进行OEM的原因是，Network Optix公司为Hanwha为其VMS NxWitness贴上了品牌，在市场上被称为Wisenet Wave。 Network Optix在这方面不仅限于与韩华合作，因此，您可以从其他供应商那里找到熟悉的界面。



Network Optix的起源于2010-2011年，主要软件NxWitness代表了市场。

NxWitness存在一个版本，没有分成多个版本。 在所有考虑的产品中，VMS具有最紧凑的安装程序，并且被开发人员定位为一种非常轻巧的产品，不需要硬件资源，而且不会牺牲功能。

该VMS适用于各种大小的对象，易于缩放，甚至适用于地理分布的对象。 在NxWitness中，每个服务器限制为128个设备，并且单个系统中建议的服务器数量不超过50。从理论上和实践上，服务器数量可能会更多，但是开发人员应参与计划系统，并且存在此类安装。



要了解NxWitness，可以在制造商的网站上获得演示许可证，并且可以完全免费地使用NxWitness观看实时视频或用作视频代理服务器。

NxWitness支持大约5000个IP设备。 它根据设备获得许可，并且在连接NVR时，每四个通道均获得许可。 NxWitness本身可以与NVR（特别是Hanwha的NVR）配合使用，并支持访问设备本身上的存档。 许可证与硬件绑定，更改服务器配置后，您可能需要重新激活许可证。 许可证使使用IP设备的完全受支持的功能（I / O端口，声音，板载分析）成为可能，因为对于通道，NxWitness仅支持双通道操作-主通道和辅助通道。 在NxWitness架构中，观看视频时，辅助流起到减少网络或运营商硬件负载的作用。 默认情况下记录第二个流，但是可以禁用记录。

该系统具有少量现成的视频分析集成解决方案，但是免费提供了多功能API和SDK。 通过使用API​​，您可以完全自动化系统的日常工作，直至完全配置或非标准监视。

系统中有两种类型的帐户：专有帐户和LDAP。 有三种用于VMS的客户端软件：胖客户端，移动客户端和Web客户端。 所有客户都是绝对免费的。 系统配置程序与视频程序结合在一起，单个界面和分离发生在用户权限级别。



NxWitness的服务器部分支持Windows和Linux环境中的工作，客户端部分-Windows，Linux和MacOS。 ARM架构对软件服务器部分的支持是其独特之处。 供应商测试了单板计算机Raspberry Pi和Banana Pi的工作，尽管从理论上讲，在其他类似设备上的使用没有限制。

NxWitness可以部署在虚拟环境以及Docker中。

卫星创新。 宏观镜（Eocortex）


卫星创新公司是一家俄罗斯公司，以Macroscop的名义为专业市场的视频监控系统开发软件解决方案，其欧洲市场称为Eocortex。
Macroscop的历史可以追溯到2008年，并开发软件，视频分析功能和网络记录器。



Macroscop包括三个软件版本，具有各种支持的相机，服务器，客户端连接和分析支持。 Macroscop适用于小型物体和大型物体。 在旧版本中，系统中的摄像机数量没有限制，但是每台服务器的摄像机数量限制为479。

该产品的演示版本可供查看，没有Macroscop的免费版本。

Macroscop产品线支持约5,000个IP摄像机和某些型号的录像机。 许可是基于通道的，现在市场上的大多数IP摄像机都是多通道（配置文件），在某些情况下，这种类型的许可带来了不便，并且在使用多通道设备时需要额外的许可。 该许可证与服务器硬件绑定，因此，如果您更改配置，则可能需要重新激活。 激活密钥后，无法将其转移到另一台计算机。 或者，可以使用没有绑定的USB密钥。

没有观察到Macroscop摄像机对车载分析的支持。 为此，Macroscop提供了软件内置的智能模块的使用。 该系统大约有15个模块，它们根据渠道数量分别获得许可，而更高版本的模块则免费包含了一些模块。 有趣的事实是，变形是Fisheye相机的一大特色，属于智能模块，并另外获得了许可。



Macroscop拥有自己的SDK，可满足集成和扩展系统功能的需求。 几乎没有免费的第三方视频分析解决方案可用，但与某些访问控制系统集成在一起。



该系统支持一种专有帐户类型。 Active Directory用户支持已停止。 有三种与VMS配合使用的客户端软件：主，移动和Web。 所有客户都是绝对免费的。 有一个用于系统配置的单独配置器。
服务器和客户端部分支持Windows环境中的工作。

视频监视系统的选择必须全面，透彻并面向未来。 主要选择可以是封闭式和开放式视频监控系统之间，也可以是具有嵌入式分析功能的系统之间，也可以不具有集成功能的系统之间进行选择。 并且您始终必须考虑是否需要集成第三方系统。

选择什么？ 对于我来说，选择是显而易见的-开放平台，最好是在这个词的整个范围内。

开放平台将为您提供灵活性和选择权-无需绑定IP设备，硬件系统和分析。 视频监控平台的开放性将为您提供一个行动领域-测试和选择能够真正解决您的问题的解决方案。

因此，在选择系统时，重要的是要注意其创新性和业绩记录。 您需要调整自己，以充分利用产品。 而且，您不应该只关注每条通道的成本，而应该更合理地估算通道成本与其价值之比。
