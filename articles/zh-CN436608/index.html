昨天， 我写了博客，介绍我为新开发人员创建的内容，这些内容已到达我们在卢卡的Techedge办事处。


教学是我开始喜欢的东西，是我爱学习和分享知识的自然结果。 而且我认为，重要的是，新生或年轻的开发人员必须具备一些精心设计的内容，也许还应包括年长者在旅途中学到的一些技巧。


该练习背后的想法是涵盖SAPUI5开发人员应了解和理解的每个主题。


该练习将在我的GitHub项目openui5-exercise中提供 。


练习



练习将分为轻而易的步骤，以逐步构建一个完整的CRUD SAPUI5主从应用程序。


该项目将从Netweaver Gateway Demo ES5中使用/创建SAP OData服务的数据。

从哪里开始


正如我所说，本练习将基于通过SAP Netweaver Gateway演示提供的OData服务。 您必须在演示系统上创建一个用户才能访问这些信息。


配置使用的SCP驾驶舱目的地


登录到您的SAP SCP帐户。 进入“ 连接性>目标”，并使用网关演示系统博客文章中提供的信息创建新目标 。
我已经为（懒惰的！）准备了配置，因此您只需单击“ 导入目标”按钮。 只需记住在点击保存之前配置您的用户名/密码并对其进行测试。 


Description=SAP Gateway Demo System Type=HTTP TrustAll=true Authentication=BasicAuthentication WebIDEUsage=odata_abap, bsp_execute_abap, odata_gen Name=ES5 WebIDEEnabled=true ProxyType=Internet URL=https\://sapes5.sapdevcenter.com sap-client=002 User=YOU_ES5_USER_HERE WebIDESystem=ES5

提示：我建议您以与我过去相同的方式在SAP SCP驾驶舱中呼叫目的地。 这样，当您从存储库分支中克隆“练习步骤”时，就不会有命名冲突
