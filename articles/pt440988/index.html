O Google abriu a zona de domínio .dev



Em 2015, o Google despertou o descontentamento de alguns desenvolvedores, tendo adquirido a zona de domínio .dev para uso exclusivo. Agora a situação está mudando e qualquer pessoa tem a oportunidade de comprar um domínio.

Bem, agora não é ninguém: de 19 a 28 de fevereiro, o Programa de acesso antecipado com uma taxa de registro de domínio adicional de até US $ 11.000 está em vigor. Mas todos os dias o valor diminui e, a partir de 28 de fevereiro, haverá apenas um pagamento padrão de US $ 12 / ano, portanto, será adequado não apenas para grandes empresas, mas também para pequenos blogs de desenvolvimento pessoal.

Várias empresas já estão ativas na nova zona (e isso não significa apenas "colocar um redirecionamento"). Aqui estão alguns exemplos:

  • web.dev e opensource.dev - sites diretamente do Google
  • github.dev - O GitHub não redireciona para sua página principal, mas oferece aos desenvolvedores o arquivo de um site pessoal usando as páginas do GitHub
  • ycombinator.dev - o famoso acelerador de inicialização YCombinator reuniu links para seus materiais para desenvolvedores
  • slack.dev - parece ter criado um site separado da página principal do site api.slack.com/tools
  • jetbrains.dev - em essência, uma página de destino dos produtos JetBrains pelos quais você já é conhecido
  • codecademy.dev - parece o Codecademy principal usual, mas sem o botão "Atualizar para o Pro". Uma versão simplificada com apenas cursos gratuitos?
  • women.dev - código de mulheres que
  • mdn.dev - Documentos da Web MDN da Mozilla

Em endereços como microsoft.dev ou apple.dev, nada abre, mas você não poderá comprá-los. Mas o habr.dev ainda está livre!

Você pode verificar a disponibilidade do domínio e prosseguir com a compra usando um site especial do Google get.dev .



É engraçado que na ilustração deste site haja uma colocação discreta do produto na coluna Página inicial do Google. Assim como a zona .app aberta anteriormente, o .dev foi originalmente projetado para HTTPS: navegadores que suportam HSTS (incluindo todos os populares) acessarão todos os sites .dev apenas via HTTPS.

Anteriormente, os endereços com o final .dev às vezes eram usados ​​pelos desenvolvedores para testar seus produtos, mas em 2017, havia problemas com a aparência de uma zona de domínio.

Bem, vamos agora recitar os comentários: é "uma zona útil que estamos perdendo há muito tempo" ou "outra zona sem sentido, foi ótimo conviver com .com para tudo em sequência"? Você quer um domínio lá? Deseja que as empresas iniciem sites orientados a desenvolvedores lá ou a seção "para desenvolvedores" no site principal é suficiente? Estamos realizando conferências para desenvolvedores, você acha que a zona .dev para sites lógicos dessas conferências é mais que suficiente?
