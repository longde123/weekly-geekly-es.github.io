Subrayar en Python

Recientemente tuve la oportunidad de escuchar una charla sobre buenas y malas prácticas de programación en C. En él, en particular, se tocó el tema de decodificar código de programa de aspecto divertido ( emoticones en C). Luego hubo un debate sobre la conveniencia de usar un código tan confuso para probar las habilidades de un candidato para un puesto de programador durante las entrevistas. La disputa no condujo a un consenso.

Considere una posible pregunta sobre emoticones durante una entrevista para un puesto que implique el conocimiento del lenguaje de programación Python.

Desafío


Hay dos expresiones: 

  1. _+_ 
  2.  _|_

¿Cuál de estas dos expresiones probablemente conducirá a errores y por qué?

Notas de expresión
Cabe señalar que estas expresiones no se leen bien, por lo que no es aconsejable usarlas en el código de trabajo, pero solo se pueden usar como un rompecabezas.
También debe recordar que los emoticones en la tarea no están escritos pitónicamente (no cumplen con el estándar de estilo PEP 8 ).

Solución


En Python, el guión bajo no es una palabra clave ; por lo tanto, en general, se puede usar para asignar ciertos valores a las variables.

La tarea se reduce a la consideración de casos de errores durante la adición y con "bit a bit" o (por brevedad, simplemente "o") una variable. Es decir para buscar los tipos integrados de Python (clases) para los que se implementa una operación y para buscar los tipos integrados donde se implementa otra operación.

En la consola de Python, es fácil verificar que haya valores de subrayado tales como:

  1. No hay error al evaluar cada expresión 
     >>> _ = 10 >>> _+_ 20 >>> _ = 10 >>> _|_ 10
  2. No hay error al evaluar la primera expresión 
     >>> _ = '10' >>> _+_ '1010' >>> _ = '10' >>> _|_ Traceback (most recent call last): File "<stdin>", line 1, in <module> TypeError: unsupported operand type(s) for |: 'str' and 'str'
  3. No hay error al calcular la segunda expresión 
     >>> _ = {1, 0} >>> _+_ Traceback (most recent call last): File "<stdin>", line 1, in <module> TypeError: unsupported operand type(s) for +: 'set' and 'set' >>> _|_ {0, 1}
  4. Hay un error en el cálculo de cada expresión. 
     >>> _ = {1: 0} >>> _+_ Traceback (most recent call last): File "<stdin>", line 1, in <module> TypeError: unsupported operand type(s) for +: 'dict' and 'dict' >>> _|_ Traceback (most recent call last): File "<stdin>", line 1, in <module> TypeError: unsupported operand type(s) for |: 'dict' and 'dict'

La magia de Python es que la implementación de adiciones, y "o", y otros operadores está en los llamados "métodos mágicos" descritos para las clases requeridas.
Además, este es el método 

 __add__

Para "o", este es un método 

 __or__

El nombre de cada método mágico contiene guiones bajos, dos al principio y al final de la palabra correspondiente.

Encontrar información sobre las clases integradas de Python.
Puede intentar encontrar la información requerida en la documentación de los tipos integrados en el sitio web oficial de Python: https://docs.python.org/3.7/library/stdtypes.html .
Puedes ejecutar en una terminal Python 

 >>> import builtins >>> help(builtins)

Prefiero usar la documentación de la versión exacta de Python que estoy usando actualmente, refiriéndome a la interfaz web del módulo pydoc.

Para hacer esto, en el símbolo del sistema, ingrese, por ejemplo: 

 python3 -m pydoc -p 3344

y abra el módulo de documentación del navegador http: // localhost: 3344 / builtins.html .

¿Cuál es el papel del guión bajo en nombrar objetos?


Respuesta 2.
El guión bajo se puede usar para separar palabras en diferentes estilos de nombres de entidades en Python :

  1. Al nombrar en minúsculas 

     lower_case_with_underscores
  2. Al nombrar en mayúsculas 

     UPPER_CASE_WITH_UNDERSCORES

Algunos nombres limitados a guiones bajos están reservados :

  1. Con un guión bajo al comienzo de un nombre, no se importa durante la importación general 

     from module import *
  2. Con dos guiones bajos en ambos lados del nombre se considera sistémico, no se recomienda utilizar dichos nombres para otros fines.
  3. Con dos guiones bajos al comienzo de un nombre, se considera privado (no público) para la clase


¿Cuál es el papel de un solo guión bajo?


Respuesta 3.
En el intérprete, si el guión bajo no se usó explícitamente como una variable, almacena el resultado de la última acción sin errores que se muestra en la pantalla. 

 $ python3 Python 3.6.8 (default, Feb 14 2019, 22:09:48) [GCC 7.4.0] on cygwin Type "help", "copyright", "credits" or "license" for more information. >>> _ Traceback (most recent call last): File "<stdin>", line 1, in <module> NameError: name '_' is not defined >>> 2 * 5 10 >>> _ 10 >>> _ = 1 >>> _ 1 >>> 2 * 5 10 >>> _ 1

Esto se debe a que sys.stdout se escapa con displayhook . Tome un ejemplo de la documentación oficial : 

 def displayhook(value): if value is None: return # Set '_' to None to avoid recursion builtins._ = None text = repr(value) try: sys.stdout.write(text) except UnicodeEncodeError: bytes = text.encode(sys.stdout.encoding, 'backslashreplace') if hasattr(sys.stdout, 'buffer'): sys.stdout.buffer.write(bytes) else: text = bytes.decode(sys.stdout.encoding, 'strict') sys.stdout.write(text) sys.stdout.write("\n") builtins._ = value
