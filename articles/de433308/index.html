Übertragen der PBX-Konfiguration an den 3CX PBX Express-Dienst

In diesem Artikel wird beschrieben, wie Sicherungskopien vorhandener PBX-Installationen in 3CX PBX Express wiederhergestellt werden. Die Möglichkeit, die Konfiguration wiederherzustellen, ermöglicht es beispielsweise, den lokalen Server in die Cloud zu verschieben, das Hosting zu ändern oder die TK-Anlage in der Cloud nach einem schwerwiegenden lokalen Fehler wiederherzustellen. Die einzige Voraussetzung beim Verschieben der Konfiguration ist, dass die Option "Lizenzdaten und FQDN des Namens" in der Sicherung aktiviert sein muss.

Beachten Sie Folgendes, bevor Sie Ihre PBX auf PBX Express umstellen:

  • Beenden Sie alle Dienste Ihrer aktuellen (alten) Telefonanlage, bevor Sie die Konfiguration in der Cloud wiederherstellen.
  • Die IP-Adresse des PBX-Servers im öffentlichen DNS wird automatisch geändert, wenn der vollqualifizierte Domänenname von 3CX verwendet wird und die Adresse in den 3CX-Parametern (Einstellungen → Netzwerk → Öffentliche IP) auf dynamisch eingestellt ist.
  • Die IP-Adresse muss manuell geändert werden, wenn der vollqualifizierte Domänenname von 3CX verwendet wird und die Adresse in den 3CX-Parametern auf statisch eingestellt ist (Einstellungen → Netzwerk → Öffentliche IP).
  • Wenn Sie einen benutzerdefinierten vollqualifizierten Domänennamen verwenden, ändern Sie die externe Adresse in 3CX und aktualisieren Sie den A-Eintrag im öffentlichen DNS, der jetzt auf die neue Server-IP-Adresse verweisen sollte.

3CX-Clients stellen automatisch eine Verbindung her, sobald der Server unter ihrem vollqualifizierten Domänennamen wieder erreichbar ist, und aktualisieren dann die Konfiguration automatisch. IP-Telefone müssen möglicherweise neu konfiguriert werden. In der Regel müssen die Einstellungen für die automatische Abstimmung von Telefonen in der 3CX-Schnittstelle von „Local Area Network“ in „STUN“ oder „SBC“ geändert werden. Wenn die Telefone den STUN-Server verwenden, müssen sie erneut eine Verbindung herstellen, sobald der PBX-Server wieder verfügbar ist. Stellen Sie bei Verwendung des SBC-Dienstprogramms sicher, dass es gemäß der Konfiguration der Cloud-TK-Anlage konfiguriert ist.

Sie können die Auflösung der neuen PBX-IP-Adresse (nach einer Weile, wenn die Replikation von Änderungen am DNS abgeschlossen ist) mit dem Befehl 'nslookup [3CX-FQDN]' überprüfen. Wenn Sie 3CX FQDN verwenden, verwenden Sie den Befehl 'nslookup [3CX-FQDN] 8.8.8.8'. Denken Sie daran, dass DNS-Änderungen von 8 Minuten bis 6 Stunden repliziert werden können.

Beim Übertragen von PBX-Anlagen auf PBX Express HTTPS-Cloud-Hosting wird der 3CX-Server-Port auf 443, der SIP- und der 3CX-Tunnel-Port auf 5060 bzw. 5090 festgelegt.

Backup




Stellen Sie wie oben erwähnt die Option "Lizenzdaten und FQDN des Namens" ein. Es wird außerdem empfohlen, die Option "System-Sprachnachrichten und Musikdateien in der Warteschleife" zu aktivieren. Verwenden Sie keine Großbuchstaben, Sonderzeichen oder Leerzeichen im Namen. Bei Bedarf können Sie das Sicherungskennwort verschlüsseln.
Es wird nach Möglichkeit nicht empfohlen, den Anrufverlauf, die Benutzervorlagen, die Voicemail und die Konversationsaufzeichnungen zu übertragen, da dies die Größe der Sicherungsdatei erheblich erhöht. Dementsprechend erhöht sich die Datenwiederherstellungszeit auf dem neuen Server - Sie können nicht vorhersagen, wann die Übertragung endet.

Backup-Wiederherstellung


Für die Wiederherstellung muss auf die Archivdatei über HTTP oder HTTPs zugegriffen werden können. Sie können jeden Webserver sowie Dropbox, OneDrive oder Google Drive verwenden.
Laden Sie die Sicherungsdatei von der 3CX-Verwaltungsoberfläche herunter.



Und übertragen Sie es in ein geeignetes Repository.

Dropbox


Laden Sie die Datei in Dropbox hoch, klicken Sie darauf und wählen Sie "Teilen"> "Link erstellen"> "Link kopieren".



Google Drive


In Google Drive sollte die Option zum Erstellen eines freigegebenen Links wie folgt ausgewählt werden:



Dann kopieren Sie den Link.

Backup auf Microsoft OneDrive


Klicken Sie in Microsoft OneDrive mit der rechten Maustaste auf die Datei und wählen Sie "Freigeben". Stellen Sie die Parameter wie unten gezeigt ein und klicken Sie auf "Link kopieren".



Wiederherstellung in PBX Express


Gehen Sie zu https://pbxexpress.3cx.com und wählen Sie Backup wiederherstellen. Fügen Sie den Archivlink ein und geben Sie das Kennwort ein, wenn Sie die Verschlüsselung verwendet haben. Geben Sie auch die Zeitzone der PBX-Instanz und die E-Mail-Adresse des Administrators an (sie erhält eine Benachrichtigung über die Wiederherstellung).



Die Sicherungsdatei wird direkt von ihrem Speicherort auf die Cloud-basierte PBX-Instanz heruntergeladen und das System ist in wenigen Minuten betriebsbereit.
