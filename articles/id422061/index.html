Server kompak dengan pemuatan kode yang disesuaikan dan log sql oleh satu tim pakar untuk pengembangan produktif

Saya ingin berbagi pengalaman menggunakan payara-mikro dalam pengembangan aplikasi perusahaan. Saya berharap seseorang akan menghemat waktu, karena keputusan seperti itu tidak segera datang. Jika Anda sudah menggunakan Payara atau Glassfish sebagai server industri, atau hanya akan terjun ke dunia javaee, maka payara-mikro dan artikel ini untuk Anda.


Anda akan memerlukan aplikasi web, yang dirakit melalui pakar ke dalam arsip perang, dan Java 8 (tidak memeriksa yang lebih lama).


Pertama, saya akan memberikan teks lengkap dari profil, setelah itu kita akan menguraikannya menjadi beberapa bagian dan menambahkan detail yang hilang yang tidak terlihat dalam kode sumber.

Hal pertama yang harus dilakukan adalah mengunduh arsip payara-micro itu sendiri. Inilah yang dilakukan oleh plugin pertama ( maven-dependency-plugin ). Arsip adalah server aplikasi lengkap yang dapat diluncurkan dari baris perintah sebagai aplikasi java sederhana.


Plugin terbaru ( payara-micro-maven-plugin ) tidak memiliki fitur dan dikonfigurasi untuk versi payra-micro 4.1.2.173 yang berfungsi dengan baik . Dari kemampuan plugin, fungsi pengemasan aplikasi web ke dalam bundel khusus, siap dijalankan, digunakan di sini. Ada cara lain untuk meluncurkan payara-mikro, tetapi di sini mereka tidak akan berguna bagi kami.


Tetap menghasilkan baris perintah yang diinginkan dan menyatukan semuanya. Exec-maven-plugin digunakan untuk memulai dari baris perintah. Mari kita pertimbangkan opsi baris perintah lebih detail: 


 <configuration> <executable>java</executable> <arguments> <!--http://blog.payara.fish/using-hotswapagent-to-speed-up-development--> <argument>-XXaltjvm=dcevm</argument> <argument>-javaagent:hotswap/hotswap-agent.jar</argument> <argument>-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=9009</argument> <argument>-Duser.language=en</argument> <argument>-Duser.region=US</argument> <argument>-Ddb.ora.url=jdbc:p6spy:oracle:thin:@localhost:1521:XE</argument> <argument>-Ddb.pg.url=jdbc:p6spy:postgresql://localhost:5432/postgres</argument> <argument>-jar</argument> <argument>${project.build.directory}/${project.build.finalName}-microbundle.jar</argument> <argument>--domainConfig</argument> <argument>src/main/setup/domain.xml</argument> <argument>--deploy</argument> <argument>${project.build.directory}/${project.build.finalName}.war</argument> <argument>--rootDir</argument> <argument>${project.build.directory}/payaramicro</argument> </arguments> </configuration>

Dua argumen pertama ( -XXaltjvm = dcevm -javaagent: hotswap / hotswap-agent.jar ) diperlukan untuk mengubah kode sumber tanpa me-restart aplikasi. Berkat argumen kedua ( -agentlib: jdwp = transport = dt_socket, server = y, suspend = n, address = 9009 ), kami segera meluncurkan aplikasi dalam mode debug, yang memungkinkan kami untuk mengubah tidak hanya tubuh metode itu sendiri melalui cara menerapkan perubahan kode dalam debugging di IDE , tetapi juga menambahkan bidang baru dan banyak lagi. Anda hanya perlu terhubung dengan debugger dari IDE, kompilasi file sumber yang dimodifikasi dan terapkan perubahannya.


Dua argumen berikut ( -Duser.language = en -Duser.region = US ) diperlukan agar klien jdbc bekerja dengan benar dengan Oracle. Pengaturan untuk menghubungkan ke DBMS tes dilewatkan melalui argumen -D . Dalam aplikasi saya, saya menggunakan tautan seperti $ {db.ora.url} ke parameter ini di file sumber daya, yang secara otomatis membuatnya pada saat startup. Substitusi file yang menciptakan sumber daya dengan pengaturan tes atau industri juga diterapkan melalui profil dalam pakar dan kemampuannya untuk memfilter file teks.


Argumen kunci saat startup menunjukkan path ( --domainConfig src / main / setup / domain.xml ) ke file pengaturan domain, serta path ke file perang ( --deploy $ {project.build.directory} / $ {project. build.finalName} .war ) untuk digunakan ke server. Pengaturan domain saya peroleh berdasarkan file standar di dalam arsip. Saya akan memberikan konten sebagai contoh. Untuk digunakan dalam pengembangan pengaturan ini lebih dari cukup.
