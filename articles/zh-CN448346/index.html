从GNU到毁灭战士：TechTrain 2019宣布


这些人是谁？除了头发长短，还有什么使他们团结在一起的？ 许多游戏玩家可能从照片中认出了John Romero ，自由软件的支持者不需要介绍Richard Stallman ，而PostgreSQL的用户可能对Oleg Bartunov非常了解。 共同点已经很明显了：这三个人都已经从事IT多年了，并且参与了流行的项目。 但是，三个人在做这么多不同的事情时是如何在同一张图片中出现的呢？

原因是他们将参加同一活动-您也可以在那里。 8月24日至25日，我们将在圣彼得堡举办TechTrain IT节 ，这三人将在其中演讲。 还有什么呢？

对于初学者：TechTrain到底是什么？ 很难用一个词来形容节日，而与此同时发生了很多事情。 让我们尝试从一种更易于理解的会议形式开始，并解释差异。

除了TechTrain，我们还定期组织针对特定主题（“ JS开发”，“测试”，“ DevOps”）的硬性会议，这些会议是为苛刻的专业人员而设计的。 毫无疑问，它们的好处。 但是与此同时，我们意识到不同格式的事件对IT社区将是有用的。

首先，它不是高度专业化的，而是立即用于与IT有某种联系的每个人。 在这里，您可以松开当前任务的提示，然后回头看看周围正在发生的事情。 测试人员可以在这里与devopa爱好者交谈，高级开发人员可以与学生交谈。

其次，不是认真的专业人士，而是同时提供丰富的信息和娱乐性。 如果您的目标是在IT中尽可能高效地工作，那么TechTrain可以为您提供帮助。 但是，如果您只想闲逛两天，也没有问题，选择日期作为日期也不是偶然。

第三，不是封闭和精英，而是大众化和负担得起的。 专业会议通常是由雇主支付的，观众的参与通常很有价值。 在这里，我们试图在活动允许的范围内尽可能降低门票价格。

第四，规模大而变化多端：如果会议的所有内容都围绕报告而定，并且在会议举行期间大厅空了，那么各种口味的活动会同时发生。

那么，一旦8月24日至25日在世博会上可以看到什么呢？

报告书




在TechTrain中，报告不像会议那样重要（您无法参加任何会议，仍然感到满意），但这并不意味着我们对报告质量的重视程度不高。 从某种意义上说，相反，您必须考虑更多。 当您为经验丰富的开发人员准备.NET会议时，通常会很清楚观众在等待什么，他们已经知道了什么。 但是，如果各种各样的人从初学者到“乳齿症”走到一起，那么应该有哪些报道才能使每个人都感兴趣呢？ 在这种情况下，是否有可能不会陷入过度的复杂性或“ hello world”？

我们刚刚在YouTube上打开了YouTube上去年TechTrain的视频-您可以看到他们的期望。 针对首次举办TechTrain的事实进行了调整，因此我们还没有能够改善程序的观众反馈。 并针对当时所有的讲者都是俄语的事实进行了调整：甚至没有人梦到约翰·罗梅罗。 也就是说，这次应该更准确，更大。



此外，我们活动的传统元素是讨论区，报告后，演讲者将在其中详细回答问题。 当像斯托曼这样的人聚集在一起参加活动时，它变得尤为重要：我们认为许多人都想问他，而不仅仅是在报告主题上。 因此，TechTrain不仅是一个从下至上查看传奇的机会，而且还是一个可以从中亲自学习一些东西的机会。

谁将在八月演出？ 除了已列出的三个人之外，我们还将命名Java开发商EvgenyBorisov 。 其余名称将稍后显示-有关此名称的最相关信息将在网站上 。

社区




每天，在办公室外面，您都可以完全抛弃所有与编程有关的内容。 但是对于许多人来说，编程语言不仅是一种赚钱的工具，而且使用这种语言的其他开发人员也不只是随机的“同伴”。 人们希望团结起来，安排一些有趣的事情，并分享有益的经验。 因此，发烧友创建了进行主题会议的本地IT社区。

在这些社区将人们聚集在一起的同时，我们的TechTrain通过为他们提供摊位将这些社区聚集在一起。 他们花了令人兴奋的两天时间，才能看到其他人并通过进入新的观众展示自己-并能够为该观众安排一些有趣的活动。

如果通常社区会为在场很长一段时间的人举行认真的演讲会，那么在这种活动的展台上（那里有许多不同背景的人）会提出不同的形式：对各种观众来说更生动。

究竟是什么-取决于每个社区，他们自己应该对自己的展台负责。 例如，去年，PiterJS举办了“黑暗中的代码”竞赛（盲目强加），DotNetRu主持了圆桌会议，各种各样的观众都愿意参加。 我们向PiterJS和DotNetRu询问了他们的印象；稍后我们将发布单独的帖子。

社区还将有机会进行报告，为此，有一个特殊的开放“演示阶段”。 如果您总是有目的地浏览“常规”报告，那么在演示场景中，只需路过即可看到有趣的表演。

今年，我们将吸引比去年更多的社区。 如果您自己只是这样一个社区的组织者，请关注此页面 （如果您知道这样的组织者，请向他发送此链接）。

公司介绍




传统的公司摊位当然也会。 而且，如果在狭窄的会议上选择合适的公司（在Java大会上将有与Java非常相关的公司），那么您可以在这里期待多样性。 去年，可以看到JetBrains（著名的IDE的创建者），Leroy Merlin（IT数量远远超过乍看之下的人）和金融初创公司Revolut。

公司还将有自己的演示平台，这不仅是“我们公司正在做什么”。 例如，在2018年，有诸如“ Git flow”之类的主题表演，一位演讲者介绍了他的个人项目，该作品以巴赫风格自动合成音乐。 然后，许多人路过听过的音乐实例，停下来观看“这里发生了什么”，然后非常感兴趣地听了这份报告。 此外，两家公司还宣布了抽奖结果-当抽奖很多时，比在展位上更方便。

去年，在TechTrain上有50个展位（包括公司和社区在内），今年我们打算将其展位数量增加到100个，并且我们还在努力实现多样性：我们对初创公司和巨头都感兴趣； 和针对开发人员的核心解决方案，以及一些怪胎。 我们认为，如此丰富的每个人都可以为自己找到一些有趣的东西。

也许您现在正在考虑您的公司是否也应该参加一个展台（并非每天都能见到成千上万的IT人员现场观众）。 对于这种情况，该站点有一个特殊的页面，其中包含有关参与的信息。

其他




列表不限于这些-将会有各种各样有趣的东西。 以前的TechTrain的访问者可以记住，例如，复古计算机上的游戏，Mosigra桌子和Brain Slaughter测验。 目前尚无新活动的娱乐清单，但总体思路是相同的。

将有我们自己的展台-如果您对与我们的会议有关的任何事情感兴趣，那么在这里查找任何东西将很方便。 将为每个人提供舒适的基础设施（TechTrain是我们的第一个针对儿童区域的活动）。

除此之外，另一件事很重要。 无论我们安排哪种活动，对于观众来说不仅重要的是他们在活动中看到的内容，而且还包括其他观众之间的观看者。 与老朋友相识总是很有价值的，与他们相处比在灿烂的孤独中有趣得多。 问题就来了：您的熟人上TechTrain的可能性有多大？

我们的全球目标是：使音乐节成为俄罗斯的主要IT活动，而恰恰是您在两天之内可以结识所有有趣的人的地方。 我们正在朝这个方向逐步发展：去年，在TechTrain上大约有2,000人，而今年我们预计将有近4,000人，因此，如果您有熟悉的Petersburg IT专家，那么很可能其中就有数千人。

总结


距离八月底还有很多时间-有人计划四个月内的一个周末吗？ 提前开售票有意义吗？ 我们是这样认为的。 看到“ John Romero”这个名字的人，会立即在日历上追溯日期，无论日期是什么。 知道何时值得在圣彼得堡，有人可以更好地计划一个假期。 对于某人来说，简单地保存更为重要：临近活动日期，门票价格逐渐上涨。 因此，现在可以在网站上以最低价格购买它们。

而且，如果您还没准备好做出决定，那么同一站点也将为您提供帮助：随着时间的流逝，有关演讲者，社区，公司和活动的信息将越来越多。

我们期待与您在八月见面！
