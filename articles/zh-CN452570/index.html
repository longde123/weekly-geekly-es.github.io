Linux Install Fest-侧视图

几天前，在下诺夫哥罗德（Nizhny Novgorod）发生了“限制互联网”时代的经典事件-Linux Install Fest 05.19 。



NNLUG（一个Linux用户区域组织）长期以来一直支持这种格式（〜2005年）。

如今，不再习惯于“从螺丝到螺丝”进行复制并以新的方式分发毛坯。 每个人都可以访问Internet，并且每个茶壶都照耀着互联网。
同时，教育内容仍然重要。 这次节日已经证实了它的重要性。

组织者邀请演讲者讨论Linux和自由软件领域中的任何有趣话题。 结果，最终列表涵盖了严重的“管理”任务，图形，游戏领域以及音频和音乐应用程序。

当演讲者在NNLUG网站上注册主题时，组织者发布了公告，包括在Habré上 。 立即以GD编制了一份任务清单，向任何想参加准备工作的人开放。

组织者计划了什么？


8个报告，最新Linux发行版的使用演示，各种口径的游戏机支架以及最终的音乐演奏会（后来证明）。

所有这些都在NTRC宽敞的礼堂中进行，声音优美，角落里有茶几。

现在解散的项目有一个独特的代表-PocketChip。 当地黑客说服所有者给了一周的审核时间。

同时，Sergei和Alex恢复了演示机，并立即启动了以下Linux发行版的安装：

  • Ubuntu 18.04.2
  • 鲁本图19.04
  • Solus 4.0 Budgie
  • Astra Linux CE（2.12）
  • Alt-Linux。 该版本不是新的，因此我们未指定。

RPi 3上的Ubuntu MATE 04/18/2位于一旁。

一位参与者正确地问：
为什么会有那么多不同的 Linux？
问题是正确的，我找不到答案。 我让Debian Lenny在家用计算机上运行KDE3，它足以完成典型的办公室和多媒体任务。

似乎除了杂色的桌面之外，不同的发行版可能仍然具有自己独特的理念，方法，配置和安全性差异。 NNLUG社区推荐的许多发行版都必须推迟以进行进一步的研究。

扰流板下的一些照片屏幕快照和分布的表面印象：


Ubuntu 04/18/2。 不成功的拍摄基本上反映了我对Gnome的最初印象：平板电脑。 原则上，如果您习惯于“分散眼睛”到一堆图标上，那还不错。



鲁本图04/19。 Kuteshno简洁。 也许我的选择是第一名候选人。



Solus 4.0 Budgie。 当然很漂亮：半透明的窗口，可以通过运行的应用程序进行分组，但是有点错误。



Astra Linux CE（2.12）。 个人排名第二。 由于预期会安装很多东西（由于站点的大小和公告），它的安装时间最长。 安装后，他要求输入复杂的密码，不久后，他弹出了一个复选框列表，这些复选框确定了更强大的安全级别。 给定创建方法-将来进行更详细研究的候选人。

Alt-linux召回了遥远的KDE3。 使用专门的培训软件非常简单。
在他中间是基本的！




Ubuntu MATE 04/18/2。

稍有延迟，活动正式开始。 接下来是报告的主观观点。 您可以在6小时流的录制中完全了解它们。



Denis谈论了一个有趣的Meshroom软件包。 简而言之，该程序基于从不同角度拍摄的50-100张对象的照片，构建具有已形成纹理的3D模型。 演示了调整的细微差别和结果。



弗拉基米尔（Vladimir）添加了虚拟引力，提到Proxmox VE：主要是用户案例和一般印象。 这个基于debian的工具的发行并不频繁，但是需要更多特殊条件来支持和更新。



Innocent讲述了一个针对儿童GCompris的好的游戏开发包。 该软件适合3岁以上的小人使用，并且包含不同类型和类型的小型游戏。 游戏形式还可以刺激儿童的成长：开阔视野，发展逻辑，自闭症儿童的方法。

根据Denis（他的第二份报告），Blender 2.8的新版本比某些类似物更稳定。 功能调整。 界面更改。



Artyom Kashkanov（ radiolok ）赞扬Nextcloud。 他说他已经使用了很长时间并且非常成功。 拥有本地“ DropBox模拟”是一个不错的选择。

Artyom Poptsov（ avvvp ）提到了伟大的Audacity，通过展示正确的正弦曲线来介绍了哲学种子。 好的包装，好的压缩器，事实上是Linux下初始声音处理的标准（我个人认为）。

Ilya在下一次报告中描述并演示了有关音频处理的更多详细信息。 作为职业音乐家和作曲家，他成功地使用了Ubunty Studio。 这个故事涉及Linux音频系统的基础知识以及更具体的Supercollider和Pure Data软件包。



最终，Fedor在N个维度上展开了自由软件的范式，并在不费劲的情况下完全“将其压缩”。 历史背景，事实，比较-该报告证明是对FOSS的阴险批评。 自由软件的拥护者受到了伤害，逐渐地，叙述变成了一个“保持火花”并且对情况真的无动于衷的人的圆桌会议。



在安装区域（最后几排有几个座位），迷你项目“在PentiumMMX上安装FreeDOS”开始沸腾。 同时，熨烫机只有20GB的IDE HDD，没有USB。 DVD ROM不在手边。

FreeDOS的官方映像格式由Ivan帮助。



然后跳线有些大惊小怪。 可拆卸的DIN键盘无法正常工作-未按下两个Enter键...当地的CADR黑客获救，事实证明它在架子上完全一样。 没错，时间浪费了，“测试人员”只能从系统安装程序的HDD启动。 快速找出如何在同一HDD上安装系统失败，该项目显然将保留到下一个节日。

总结


约有一百人参加了这次活动。 尽管背离了法规和技术覆盖面，但人们还是喜欢它。 为“大师班”和“研讨会”格式的一系列更具体的活动预留了空间-现在说细节还为时过早，让我们看看该倡议是否仍然存在。

大约有10个人参加了该组织，通过聊天进行了口头协调。
分配了7天的准备时间。 预算为零。 晋升-4种专门资源上的职位。 对帖子的评论可以分为两类：“安装节节不相关”和“仍然举办此类活动真是太酷了”。
