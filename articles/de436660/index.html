Die VS Code Roadmap 2019 - ENTWURF

Da 2018 zu Ende ist, ist es jetzt an der Zeit, in die Zukunft zu schauen. Wir schauen normalerweise 6 bis 12 Monate und legen Themen fest, an denen wir arbeiten möchten.


Während wir gehen, lernen wir und unsere Einschätzung einiger der aufgelisteten Themen ändert sich. Daher können wir Themen hinzufügen oder löschen, wenn wir fortfahren.


Wir beschreiben einige Initiativen als „Untersuchungen“, was bedeutet, dass unser Ziel in den nächsten Monaten darin besteht, das Problem und mögliche Lösungen besser zu verstehen, bevor die eigentliche Funktionsarbeit geplant wird. Sobald eine Untersuchung abgeschlossen ist, werden wir unseren Plan aktualisieren, indem wir entweder die Initiative verschieben oder uns dazu verpflichten.


Wie immer werden wir auf Ihr Feedback hören und unsere Pläne bei Bedarf anpassen.

Themen


Unsere Roadmap behandelt folgende Themen:


  • Werden Sie der beste Editor für alle, die sich auf Eingabehilfen verlassen.
  • Leistung, Skalierbarkeit, Wartungsfreundlichkeit, Sicherheit
  • einige der meistgesuchten Benutzerfunktionen in Angriff nehmen
  • Polieren und ein konstant langsames Rinnsal von Design-Erfrischungen
  • bereits vorhandene Funktionen schrittweise verbessern
  • Aktivieren Sie verantwortungsbewusst Erweiterungen mit breiteren Anforderungen an die Erweiterbarkeit

Grundlagen


  • Machen Sie VS Code zu einem hervorragend zugänglichen Entwicklertool. Wir werden mit unserer Community zusammenarbeiten, um Input und Anleitung zu erhalten, und Sie müssen uns ehrlich halten.
  • Halten Sie die Startzeiten für Benutzer auf allen Plattformen in einem vorhersehbaren und geeigneten Bereich und verbessern Sie die Gesamtleistung für große Arbeitsbereiche:
    • Laden Sie beim Start weniger Code und untersuchen Sie, wie Sie die Wiederherstellungszeit der Workbench verbessern können, indem Sie den Rapid-Rendering- Ansatz erweitern.
    • Implementieren Sie ein neues Baum-Widget, das besser skaliert und in die Workbench übernommen werden kann (Explorer, Suche, Einstellungen, Gliederung, Debugger).
  • Verbessern Sie die Wartungsfreundlichkeit
    • Machen Sie es einfach, Erweiterungen zu identifizieren, die sich negativ auf die Gesamtleistung von VS Code auswirken.

Werkbank


  • Workbench-Layout
    • Die Unterstützung für abnehmbare Werkbankteile ist unsere am häufigsten gestellte Funktionsanforderung, deren Implementierung aufgrund von Architekturproblemen schwierig ist. Wir werden untersuchen, wie wir diese Einschränkung umgehen können. Diese Untersuchung konzentriert sich auf das Trennen von Terminals (zweithäufigste Feature-Anfrage) und Editoren.
    • Aktivieren Sie ein flexibleres Bedienfeld- / Seitenleistenlayout.
  • Bieten Sie Filterung und schnelle Tastaturnavigation in Bäumen auf der Workbench.
  • Untersuchen Sie die Anzeige von Suchergebnissen nicht nur in der Seitenleiste oder einem Bedienfeld, sondern auch in einem Editor. Auf diese Weise können wir zusätzliche Kontextinformationen für jedes Spiel anzeigen.
  • Verbessern Sie die Arbeit mit dem Datei-Explorer in großen Arbeitsbereichen
    • Untersuchen Sie "Arbeitssätze" von Dateien und Ordnern
    • Durchsuchen Sie flache Ordnerhierarchien im Explorer
  • Untersuchen Sie, wie Sie sicher eine umfassendere Anpassbarkeit in der Workbench gewährleisten können
    • Untersuchen Sie benutzerdefinierte Ansichtsfenster und Bedienfelder.
    • Untersuchen Sie benutzerdefinierte Editoren, die mit der Begrüßungsseite vergleichbar sind.

Ux


  • Verbessern Sie die Präsentation und das Verhalten auf der ganzen Linie schrittweise weiter. Beispiele sind:
    • Harmonisieren Sie Schwebeflüge, Abschlusselemente und Abschlusselementdetails
    • Ikonographie
    • Willkommensseite
    • Verwenden Sie Registerkarten anstelle der Terminal-Dropdown-Liste
  • Erfahren Sie, wie Sie fließendes Design unter Windows integrieren können

Herausgeber


  • Untersuchen Sie, wie Sie den Editor von Grammatiken isolieren können.
  • Untersuchen Sie die Unterstützung für semantische Färbung
  • Untersuchen Sie, wie Sie die Pflege von Textmate-Grammatiken vereinfachen können
  • Stellen Sie die Lokalisierungsunterstützung im eigenständigen Monaco-Editor wieder her. Diese Unterstützung wurde ausgesetzt, als wir die Unterstützung für Sprachpakete für VS Code hinzugefügt haben.

WSL-Unterstützung


  • Verbessern Sie die WSL-Unterstützung.
    • Untersuchen Sie, wie wir Erweiterungen aktivieren können, um die in WSL verfügbaren Tools zu nutzen.
    • Untersuchen Sie, welche anderen Szenarien durch die für die WSL erforderlichen Änderungen aktiviert werden

Sprachen


  • Verbessern Sie die Auswahl erweitern, um die Semantik der Programmiersprachen besser einzuhalten.
  • Verbessern Sie die Unterstützung für das Navigieren und Präsentieren komplexer Fehlerbeschreibungen, wie sie beispielsweise von TypeScript for React oder Vue generiert wurden.
  • Aktivieren Sie Programmiersprachenerweiterungen, um Anrufhierarchien und Typhierarchien zu unterstützen.

TypeScript


Wir werden weiterhin eng mit dem TypeScript-Team zusammenarbeiten, um die umfassendsten Erfahrungen in Bezug auf Codebearbeitung, Navigation und Verständnis für TypeScript und JavaScript zu liefern. Siehe auch die TypeScript-Roadmap .


  • Verbessern Sie die Integration von tslint, indem Sie es als TypeScript Server- Plugin ausführen .

Debuggen


  • Unterstützt Daten-Haltepunkte
  • Verbessern Sie Schwebe- und Inline-Werte, indem Sie das Wissen über die Programmiersprache nutzen
  • Investieren Sie weiterhin in die Dokumentation von Debugging-Rezepten für gängige Konfigurationen

Erweiterungen


Benutzer von Erweiterungen


  • Unterstützt die Installation einer Erweiterung, ohne die Workbench neu laden zu müssen. Dies ist unsere dritthäufigste Feature-Anfrage .
  • Integrieren Sie die vom Developer: Show Running Extensions angezeigten Laufzeitinformationen Developer: Show Running Extensions Erweiterungen anzeigen in die vorhandene Erweiterungs-Benutzeroberfläche, z. B. das Erweiterungs-Viewlet und den Erweiterungs-Editor.
  • Verbessern Sie das Erweiterungsempfehlungssystem.
  • Erhöhen Sie den Verbrauch von Erweiterungen und verbessern Sie den Prozess für den Umgang mit böswilligen Erweiterungen.
    • Der vorhandene Prozess funktionierte, wie das Problem mit dem Ereignisstrom gezeigt hat, aber wir haben auch festgestellt, dass Verbesserungspotenzial besteht: Unterstützung für die Deinstallation einer bestimmten Version einer Erweiterung, Zugriff auf Informationen darüber, warum eine Erweiterung deinstalliert wird, automatische Neuinstallation einer Erweiterung, wenn das Problem behoben ist.
  • Unterstützung hinzufügen, um nur signierte Erweiterungen zu aktivieren (siehe nächster Abschnitt).

Erweiterungsautoren


  • Arbeiten Sie mit Autoren von Erweiterungen zusammen, um deren Erweiterungen zu verbessern. Beispiele sind: Verwenden Sie Webpack , um die Installation und Aktivierung zu verbessern, Abhängigkeiten einer Erweiterung zu minimieren und sicherzustellen, dass vscode nur eine Entwicklungsabhängigkeit ist.
  • Aktivieren Sie Erweiterungen, um zusätzliche plattformspezifische Komponenten zur Installationszeit der Erweiterung zu installieren.
  • Unterstützung der Veröffentlichung signierter Erweiterungen.
  • Unterstützung für verifizierte Publisher hinzufügen.
