[Ekaterinbourg, annonce] UralJS # 9 - trois rapports sur les microservices, les tests et la journalisation des erreurs à l'avant


Clôture de la saison estivale! UralJS # 9 se tiendra le samedi 25 août au bureau de Kontur sur la Broad River. Nous expliquerons pourquoi l'architecture de microservice est nécessaire pour les fournisseurs frontaux, enseignerons les tests pour vraiment appuyer sur les boutons et découvrirons comment détecter une erreur sur l'ordinateur de l'utilisateur avant d'appeler le support technique.


Voici une description détaillée des rapports:


Vova Sannikov, Point - Frontend de microservice


Dans le backend, les microservices sont courants. Il existe des dizaines de méthodologies bien développées pour la construction, l'assemblage, la mise à l'échelle et l'architecture. Chez notre client, une approche de service est rarement utilisée et il n'y a presque pas de matériaux sur la conception.


Je vais vous dire comment l'architecture de microservice est utile dans le front-end, pourquoi nous avons voulu la faire glisser, sous quelle forme nous l'avons mise en œuvre et ce dont nous avons bénéficié.


Dima Lazarev, Contour - Cliquez sur le bouton. À la dure


Lorsque nous écrivons des tests, nous voulons tout à la fois: que le code soit simple et concis, le support est facile, l'exécution est rapide et la couverture est à cent pour cent. Malheureusement, il faut souvent chercher des compromis.


Je vais vous dire combien il est difficile de réaliser des scripts utilisateur honnêtes, même aussi simple que de cliquer sur un bouton. Je considérerai plusieurs outils de test et partagerai la solution que nous utilisons dans le projet.


Anton Konev, Yandex - Erreurs de journalisation sur le frontend


Vous → Service ← Utilisateurs. Cela fonctionne pour vous, mais ce n'est pas le cas. Je vais vous dire comment comprendre avant l'utilisateur sans vélo que quelque chose s'est mal passé et corriger la situation.


Inscrivez- vous pour le thé, le café et les collations pour tous les invités.
