Lanjutkan Programmer Jujur


Bagian 1. Keterampilan Lembut


  1. Saya diam di pertemuan. Saya mencoba membuat wajah yang penuh perhatian dan cerdas, bahkan jika saya tidak peduli.
  2. Orang menganggap saya positif dan bisa dinegosiasikan. Saya selalu melaporkan dengan sopan dan goyah bahwa itu ditulis untuk membuat kaku. Dan hanya sekali. Maka saya tidak berdebat. Dan ketika saya menyelesaikan tugas dan ternyata menjadi apa, saya tidak tertawa, dan tidak mengatakan "Saya berbicara!"
  3. Saya tidak peduli yang kaku untuk dilewati. Jika pelanggan tertarik pada pendapat saya, dia tidak akan menyewa seorang manajer proyek, seorang pengamat produk, seorang ahli scrum, seorang pria yang gesit dan seorang desainer UI. Hipsters ini bahkan membentuk segala macam opini, visi dan chip pemasaran.
  4. Saya disiplin. Saya datang kerja jam 9 dan berangkat jam 6. Saya merasa sangat nyaman. Saya bisa terlambat membayar dua kali lipat atau jika tugasnya menarik.
  5. Saya memiliki selera humor yang bagus dan pengalaman hidup yang kaya. Saya dapat dengan mudah mengganggu pekerjaan tim selama setengah hari dengan cerita-cerita tentang bagaimana hari Sabtu saya pergi. Tapi saya jarang melakukannya, karena saya pikir saya dibayar bukan untuk ini, tetapi bagi saya untuk melewatkan thread kaku.
  6. Saya membalikkan timlidisme Anda, Anda sendiri tahu di mana. Saya sendiri dapat melewati beberapa jenis kaku, tetapi dengan tampilan yang cerdas, jelaskan kepada bawahan saya bahwa mereka harus melakukan semacam kaku di atas kekuatan saya.
  7. Saya benar-benar luar biasa dalam presentasi. Apalagi jika Anda perlu menghadirkan bottom yang belum selesai. Saya mahir memintas bug pada presentasi program. Sekali, selama dua jam, saya mempresentasikan jendela login, karena program tidak bekerja lebih jauh. Dan login tidak selalu berhasil.
  8. Ketika semuanya mengganggu saya, saya diam-diam berhenti, tetapi jangan pergi ke departemen dan jangan pergi "Semuanya buruk, kita ada di bawah, semua orang bodoh."

Bagian 2. Keterampilan Keras


  1. Warisan adalah hal yang saleh jika hanya 1 anak yang diwarisi dari paus.
  2. Saya menggunakan enkapsulasi hanya ketika Ide digarisbawahi dalam warna kuning dan menulis, metode ini dapat dibuat pribadi. Hal yang sama dengan final.
  3. Saya tidak pernah menggunakan volatile, final, dan banyak lainnya.
  4. Saya tidak peduli apa yang harus digunakan: ArrayList atau LinkedList. Saya selalu menggunakan ArrayList.
  5. Saya tidak bisa menggunakan getter dan setter di Jawa jika saya tahu tidak ada yang akan membaca kode saya. person.name = "john". Jika saya tahu seseorang akan membacanya, saya malu.
  6. Saya masih tidak mengerti mengapa antarmuka diperlukan di java, kecuali untuk callback dan lambd. Semua contoh dengan penggunaannya dibuat-buat dan saya bisa membuatnya lebih mudah tanpa mereka.
  7. Saya tidak tahu cara kerja gc, saya tidak pernah menggunakannya. Secara umum, selama 6 tahun dalam ingatan saya dia hanya disebutkan sekali. Selain wawancara, tentu saja.
  8. Saya memiliki lobak di github, tetapi saya tidak akan menunjukkannya kepada Anda. Dia adalah pribadi saya, dan saya menguliti di sana seperti yang saya inginkan. Anda tidak pulang dengan jas berekor, ayolah?
  9. Saya bisa dan saya suka melewati bagian depan jika saya bosan dengan bagian belakang. Saya sudah lupa reaksinya dan tertinggal. Tapi sepertinya aku ingat Sencha.

Bagian 3. Prestasi


  1. Saya membuat 3 situs yang dikunjungi lebih sedikit orang daripada sebelumnya. Ketika saya membuat 2 situs saya tahu bahwa tidak ada yang akan mengunjungi mereka. (Diharapkan bahwa mereka akan mengambil alih dunia)
  2. Saya membuat tiga aplikasi web (ExtJs-Java-Docker) di mana dua tidak pernah digunakan dalam prod, tapi saya menggunakan dua kali. (Diharapkan mereka akan mengambil alih dunia).

    Ketika saya melakukannya, saya tahu itu akan terjadi, karena saya tidak percaya pada pengguna yang hafal manual 20 halaman, saya sendiri menyajikan karya saya dengan manual tercetak di tangan saya.
  3. Saya membuat aplikasi Android asli 8 layar, di mana tidak ada yang melangkah lebih jauh dari yang kedua, itu diunduh 107 kali di pasar Google (diharapkan untuk mengambil alih dunia).
  4. Suatu kali saya memperbaiki bug tertinggi selama dua hari, dan kemudian saya menyadari bahwa tidak ada yang mengunjungi bagian situs ini selama sekitar tiga tahun. Dan itu adalah bagian yang sangat sehat dari situs ini, yang menghabiskan banyak jam kerja.
  5. Saya menghabiskan sekitar satu minggu untuk kotak kombo untuk pergi bukan dari atas, tetapi ke kanan.
  6. Saya memimpin 4 orang dan kami melakukan satu proyek selama setengah tahun, yang saya sendiri bisa lakukan dalam seminggu. Dan ya, ini adalah proyek dari paragraf 2.
  7. Saya mengatur caching permintaan dalam bahasa Mongu pada aplikasi yang memiliki satu orang sehari.
  8. Saya melakukan klien email perusahaan, meskipun ada ratusan yang gratis dan semuanya lebih baik.
  9. Saya melakukan idealisasi piksel (atau apa namanya?) Di bagian depan.
  10. Saya mendesain ulang pustaka Material UI untuk React karena desainer lepas Kurgan UI kami memutuskan ia lebih baik dalam desain daripada Matthias Duarte, Wakil Presiden Desain Google, Sarjana Ilmu Komputer dengan perbedaan dari University of Maryland, dengan tambahan . pendidikan seni dan sejarah seni, kepala Maryland Art Art Gallery.

    Saya tidak pernah mengerti mengapa membuat kembali hal-hal baik yang dilakukan orang-orang pintar untuk Anda dan memberikan secara gratis, terutama jika Anda jelas bodoh.
  11. Saya membuat fitur selama sebulan, yang, dengan perhitungan paling optimis, akan melawan balik 437 tahun. (memesan pel untuk wanita pembersih) di ERP.
  12. Saya ulang satu kaku dari awal 7 kali, karena TK itu berubah. Akibatnya, dia menjadi lebih buruk dari dia.
  13. Selama 4 jam saya mencari tahu mengapa sen dalam tagihan dibulatkan secara tidak benar, terlebih lagi, saya tahu sebelumnya bahwa saya tidak dapat memperbaikinya, jika tidak saldo tidak akan bertemu.
  14. Saya melakukan microservice untuk meningkatkan keandalan logika bisnis utama, dan ya, microservice ini jatuh 20 kali lebih sering daripada logika bisnis.

    Tetapi kemudian mereka membuat seluruh departemen yang terdiri dari 12 orang untuk meningkatkan keandalan dari layanan microser keandalan ini, dan sekarang layanan microser crash 20 kali lebih sering, melakukan transaksi paruh waktu dan kehilangan data tanpa jejak. Ketika saya berhenti, mereka memutuskan untuk membuat microservice keandalan untuk microservice keandalan.
