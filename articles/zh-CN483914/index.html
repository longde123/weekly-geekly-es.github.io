我们邀请您参加DINS JavaScript EVENING：我们讨论API设计并使用代数数据类型解决问题

我们很高兴在今年1月29日于我们19岁的Staro-Petergofsky的办公室举行的第一个公开DINS JavaScript EVENING上看到前端开发人员。

今天晚上，DINS的Dmitry Zlygin将讨论“良好” API的标准，并分享如何使您的界面更方便。 Devexperts的Anton Sutarmin向我们介绍了代数数据类型，并通过示例展示了它们如何帮助解决问题。

删减部分-有关报告和演讲者的更多信息，一个注册参加会议，广播信息和上次会议资料的链接。



报告书


前端的API设计（Dmitry Zlygin，DINS）

每天我们使用不同的编程接口-浏览器，Lodash或Axios等各种库，框架-React，Angular，Jest等。 我们自己为组件，功能，类创建新的API。 有时结果很好，有时不是很好。 让我们讨论一个好的API与一个“不好的” API的区别，当然，我们将确定“好的”的标准。 如何使您的界面方便且足以完成任务。 如何处理我们无法影响的第三方API。

该报告将与入门级及更高级别的Frontend开发人员和QA自动化工程师相关。 没有铁杆。

超过5年的时间，Dmitry一直在专门开发前端。 与Netscape Navigator和IE 4.0一起使用的JavaScript在第三个十年中取得了不同的成功。 随着Vue.js的出现，他放弃了后端，转而进入前端阵营，但是随着时间的流逝，他意识到了React的所有魅力。 他目前在DINS从事大型React项目。

“ TypeScript中的代数数据类型”（Anton Sutarmin，Devexperts）

他们谈到每个铁杆的功能编程，关于免疫力和清洁度的每个人都已经厌倦了聆听。 让我们尝试更进一步，看看FP世界中使用最广泛的另一个概念-代数数据类型。 让我们从实践的角度出发，我们将在他们的帮助下解决3个问题，并一路摸索到底是什么以及如何使用它。 示例是关于TypeScript的，但不必知道它：在此过程中，我将告诉您一些必要的事情。

该报告适用于熟悉功能范例的所有级别的前端开发人员，或希望在程序员的知识中添加其他工具的前端开发人员。

四年来，Anton一直是前端开发人员，最近有点不满。 他喜欢函数式编程和表现型系统。 当他看到漂亮的代码和漂亮的界面时，他感到很高兴。
