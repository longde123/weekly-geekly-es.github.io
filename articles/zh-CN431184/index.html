使用Mesh Connex技术的WIFI多跳网状网络

建立无线网络时，通常无法为特定的接入点提供有线连接，并且必须配置点对点无线网桥（WDS），市场上现有的大量现有产品都对这种技术进行了充分的研究和实施。 解决建立多跳wifi网状网络的问题更加困难。 为了完成此任务，整个ExtremeWireless Wing产品线都支持Mesh Connex技术。
Mesh Connex的基础是IEEE802.11s标准， RFC 3561以及Extreme Networks自己的开发。



使用网格连接器，您可以在静态和动态配置中成功配置多跳网格。

-多跳静态网格物体连接-基础结构模式，可以在构建诸如海上或机场，休闲公园，职业等对象时使用；
-多跳移动网状连接体-在移动物体（公共汽车，货车，自卸车...）上安装了接入点时的动态网格
同样，两种模式可以在同一网状网络中同时使用。



接入点可以用作以下网格元素：

  • MPR（网状点根）-具有有线网络访问权限的AP
  • MP（网状点）-固定AP，无有线网络访问
  • VMM（车载移动安装）-AP安装在移动物体上

同时，对网络上根访问点的数量没有限制；可以将不同的优先级分配给不同的MPR。 使用回程检测功能，MPR可以检查是否可以访问有线网络，如果有线端口有问题，则将流量重定向到另一个根。 为了监视关键资源，您可以将其配置为使用ARP或ICMP请求进行检查，在这种情况下，即使提高了MPR上的有线回程，流量也将通过另一个MPR重定向。



Mesh Connex与其他用于构建多跳网状网络的技术之间的主要区别在于路由协议，该协议可计算网状节点之间的最佳路由。 无线网状网络中的路由协议可以是主动的或被动的。

  • 主动-不断更新完整的路由表，即使在中型网络中也是如此，这会增加服务“开销”流量，同时占用时间和资源以实现完全融合
  • 反应式-协议缓慢响应网络中的变化，仅存储到邻居的路由，但同时在搜索路由时具有更高的延迟

Mesh Connex以混合模式同时使用这两种方法。 最初，当每个MP仅研究到MPR的路由时，以按需模式被动地收集路由信息，然后，随着网络的运行，使用主动模式优化流量传递。

在无线多跳网状网络中，经常会发生以下情况：即使邻居整体上具有更好的SNR，通过它到达根点的路由也会具有较差的特性，因此Mesh Connex根据以下数据来计算路由：

-链路质量-邻居的信标被听到的程度（即，最初是“预测”值，然后在交换数据包时对其进行测量）
-链接指标-0到65535之间的数字，基于LQ + RSSI +平均数据速率进行测量
-路径指标-链接指标总计为MPR

MeshConnex的其他重要优点是：

  • 高扩展性（点数可以从2到1000）
  • 低每跳延迟-2.2毫秒
  • 网络可能有也可能没有有线“回程”
  • 网格节点可以是移动的（全部或部分）
  • 移动节点可以以高达140 km / h的速度移动（“切换”为200 ms）
  • 快速收敛
  • 发生事故时的网络自我修复能力
  • 网格上的组播支持

如果您需要为停车场，工业区，露天节庆区，矿山，采石场，空中和海上港口构建带有网格元素的无线网络，建议您使用Mesh Connex进行铁路运输。 迄今为止，在俄罗斯和周边国家已经成功实施了数十个此类项目。
