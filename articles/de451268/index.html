Victor Gamov über Kafka Streams IQ auf jug.msk.ru

Am 8. Mai 2019 sprach Victor Gamov bei einem Treffen der Moskauer Community der Java-Entwickler jug.msk.ru über Kafka Streams IQ ( Interactive Queries ).



Über den Sprecher


Viktor spricht zum vierten Mal auf jug.msk.ru. Seine früheren Berichte betrafen Nashorn ( 13. November 2014 ), Hazelcast ( 4. Mai 2016 ) und Apache Kafka ( 23. April 2018 ). Wie vor einem Jahr arbeitet er derzeit bei Confluent , jetzt jedoch als Developer Advocate .

Mitbegründer und Moderator des beliebten Podcasts Debriefing , Mitautor des Buches Enterprise Web Development , Sprecher zahlreicher Konferenzen (einschließlich der von der JUG.ru Group organisierten ). Victors Website (mit einem technischen Blog am selben Ort), Videoberichte auf YouTube , Präsentationsdateien auf Speaker Deck und Notist , Habr , Twitter und GitHub .

Einige Berichte seit der letzten Rede auf jug.msk.ru :

  • „Kafka on Kubernetes: Die mysteriöse Geschichte der Schaffung einer Cloud- basierten Plattform für die Verarbeitung von Streaming- Daten“ ( DevOops 2018 : Präsentation )
  • "Java Puzzlers NG S03: Was? Woher kommen die ?! " (mit Baruch Sadogursky auf Oracle Code One 2018 : Video )
  • "Verbrechensbekämpfung mit Kafka Streams und der Bintray Firehose API" (mit Baruch Sadogursky auf Oracle Code One 2018 : Video , Präsentation )
  • „Crossing the Streams: Stream-Verarbeitung mit KStreams und KSQL überdenken“ ( DevNexus 2019 : Präsentation )
  • "Kafka auf Kubernetes: Muss es wirklich" The Hard Way "sein?" ( Kafka Summit 2019 : Präsentation )

Über den Bericht


Traditionell erinnert Andrei Kogun vor der Präsentation des Redners und dem Beginn des Treffens an die bevorstehenden Veranstaltungen - Bewerbungen für die Joker Java-Konferenz (St. Petersburg, 25.-26. Oktober) sind offen und das TechTrain IT-Festival wird voraussichtlich zum zweiten Mal stattfinden (St. Petersburg, 24-25.) August).



Der Bericht war interaktiven Abfragen gewidmet , die Teil der Kafka Streams- API sind und die Möglichkeit bieten, auf den Zwischenstatus der Anwendung und die erzielten Ergebnisse zuzugreifen, ohne herkömmliche Datenbanken und Repositorys zu verwenden.



Die Präsentations-Diashow wechselte sich mit Codebeispielen ab. Es wurde eine große Anzahl von Fragen gestellt, auf die Victor sofort umfassende Antworten gab, was auch durch Bewertungen auf Twitter bestätigt wird.



Die Diskussion wurde sowohl in der Pause als auch danach fortgesetzt. An dem Treffen nahm einer der Redner der JUG.ru- Konferenzen , Ivan Ponomarev, teil, dessen jüngster Bericht auf der JPoint 2019- Konferenz „Kafka Streams API: Ein Schritt über Hello World hinaus“ ( Präsentation , Video steht bisher nur Konferenzteilnehmern zur Verfügung) Victors Geschichte ergänzen kann.
