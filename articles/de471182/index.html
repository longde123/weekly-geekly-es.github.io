GitHub Desktop 2.2: Was ist neu?



Jeder, der Git benutzt, weiß, dass er eine coole Lernlinie hat. Wir haben von Entwicklern gelernt, dass die meisten Menschen normalerweise von Bekannten lernen, ob sie ein Kollege, Professor, Freund oder sogar ein YouTube-Video sind. In GitHub Desktop 2.2 fügen wir die erste Version eines interaktiven Tutorials zu Git und GitHub hinzu, das Ihr Freund werden und Ihnen den Einstieg erleichtern kann. Wenn Sie Desktop noch nicht kennen, können Sie das Tutorial unter desktop.github.com herunterladen und ausprobieren. Nun, die Details sind unter dem Schnitt.

Erste Schritte


Zunächst helfen wir Ihnen, zwei Hauptphasen zu durchlaufen: Erstellen eines Repositorys und Verbinden eines Editors. Wenn Sie das Programm zum ersten Mal öffnen, wird eine Begrüßungsseite mit der neuen Option "Trainings-Repository erstellen" angezeigt. Mit dieser Funktion wird ein Schulungsrepository erstellt, das Sie durch die grundlegenden Konzepte der Arbeit mit Git mit GitHub Desktop führt.



Viele Tools werden benötigt, um mit Git und GitHub zu beginnen. Das wichtigste davon ist Ihr Code-Editor. In der ersten Phase des Tutorials werden Sie aufgefordert, einen Editor zu installieren, falls Sie noch keinen haben.



Entdecken Sie GitHub


Als Nächstes erfahren Sie, wie Sie mit GitHub Desktop Code lokal ändern und Ihre Arbeit an GitHub senden können. Sie erstellen einen neuen Zweig, nehmen Änderungen an der Datei vor, schreiben sie fest, senden sie an GitHub und öffnen Ihre erste Pull-Anfrage.

Wir haben auch gehört, dass neue Benutzer anfänglich Verwirrung zwischen Git, GitHub und GitHub Desktop erfahren. Wir werden uns diese Unterschiede im Handbuch ansehen und sicherstellen, dass sie die üblichen Erklärungen unterstützen.



Fahren Sie mit Ihrem eigenen Projekt fort.


In GitHub Desktop 1.6 haben wir die Funktion "Mögliche nächste Schritte" eingeführt, die vom Status Ihres Repositorys abhängt. Nachdem Sie das Lernprogramm abgeschlossen haben, bieten wir außerdem die folgenden Schritte an: Erkunden von Projekten auf GitHub, zu denen Sie möglicherweise beitragen möchten, Erstellen eines neuen Projekts oder Hinzufügen eines vorhandenen Projekts zu Desktop. Wir möchten, dass GitHub Desktop ein Tool ist, das Ihre nächsten Schritte klärt, unabhängig davon, ob Sie gerade arbeiten oder nur ein unerfahrener Entwickler sind.



Was weiter?


Mit GitHub Desktop 2.2 machen wir das Produkt für Anfänger zugänglicher. Wir werden das Tutorial basierend auf Ihrem Feedback verbessern und die Verbindung zwischen GitHub und Ihrem lokalen Computer weiter verbessern. Wenn Sie etwas erstellen möchten, aber nicht wissen, wie, stellen Sie sich GitHub Desktop als einen Freund vor, der Ihnen beim Einstieg hilft.
