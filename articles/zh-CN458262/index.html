会议室的在线设计师-选择视频会议的最佳解决方案

三年前，我们发表了有关VKS会议室设备的文章。 这是劳动和不只是一个人的专业知识的结果，而是一些专门从事复杂视频会议解决方案集成多年的公司的结果。 乍一看，视频会议和电话有很多共同点：共同的任务，相似的设备集和相似的解决方案体系结构。 但是，实际上，从集成商的角度来看，视频会议项目的计算和实施与电话明显不同。

视频会议仍然需要专业知识，并且通常还需要一种设计方法，而电话越来越采用“盒装”解决方案的形式。

而主要的困难恰恰在于会议室和会议室的解决方案，而不是个人使用的设备。 选择了摄像头和头戴式耳机后，现在很难犯错了，另一件事是具有声学和光学功能的大房间。 会议室视频会议设备具有如此众多的特性，以至于它们对于许多用户的价值和作用仍然是个谜。

即使您更详细地了解它们，客户也经常会遇到以下问题：

  • 为什么我不能在此会议室放置常规网络摄像头？
  • 该相机可以增加多少椅子？
  • 如果我们将电视移到桌子上，最近的参与者会进入画面吗？
  • 这个扬声器的灵敏度足以满足我们的房间需求吗？

即使了解房间和设备的参数，在不诉诸于房间中设备的实际安装的情况下，也不容易将它们组合成视觉图像。

在已经提到的文章中，我解决了如下可视化设备覆盖图的问题。

收集了设备的所有精确数值特性，例如视角，麦克风捕获半径，光学变焦比，屏幕对角线等。 根据现场数据或在实验室中，凭经验计算出不正确的参数。

例如，根据对角线的数十个真实会话和主观意见的平均测试分数，计算出取决于对角线的距电视屏幕的最大距离。 简而言之，我将此称为我们的专业知识。

在计划中添加协商设备之前，有必要确定VKS的类型。 有两个选项可供选择： 软件和硬件 。
当终端SIP / H.323客户端称为“硬件编解码器”，并且PC上安装的VKS软件将其转变为“软件客户端”时，这些表述是过去的遗留。 现在，这些概念之间的界线已经非常模糊，但是由于设备兼容性问题仍然存在，因此仍需要进行分离。

因此，我们重复：

VKS硬件系统基于SIP / H.323终端构建，其控制元素是遥控器或触摸屏。

VKS软件系统是基于PC和专业音频/视频设备构建的。 绝大多数操作系统软件终端是Windows。 这使您可以将VKS终端变成通用解决方案-专用软件使该终端成为SIP / H.323客户端，WebRTC控制台或任何云或本地VKS服务的客户端，包括 软件应用程序（Yealink，Skype，Teams，Polycom等）

问题“该选择什么？”。 首次与客户联系时听到的第一个合理问题。 有一个答案。 对于它的形成，我们需要稍微远离设计者的主题，并深入研究视频会议的体系结构。
首先，我们将确定术语以同步对特定概念的理解。

电视会议是一个对称或不对称的事件，涉及两个以上的参与者，他们可以彼此见面或聆听，也可以充当旁观者（取决于会议的场景）。
视频通话 -两个订户之间的视频通信会话。
客户端 -视频会议系统的软件或硬件终端
VKS或MCU服务器 -接收/发送/处理从客户端接收的视频流以及视频会议管理的设备。

视频会议建立在客户端-服务器架构上。 这意味着要将客户召集到一个多点活动中，您需要VKS服务器或VKS 云服务 （Skype，Google Hangouts Meet，ZOOM等）。
因此， 客户端的选择取决于计划使用哪种服务器或VKS服务。 全部与传输协议有关。

如果MCU支持SIP和H.323协议，则适合使用终端硬件解决方案或已安装软件的软件（Polycom RD，Yealink VCD等）。

如果视频会议是基于具有专有音频/视频传输协议的服务（例如Skype）构建的，则硬件SIP终端将无用，因为它不支持安装专用软件。 在这种情况下，将使用基于PC的客户端。
有支持许多连接选项的服务器解决方案（例如Yealink Meeting Server ）：

  • SIP / H.323-使用硬件终端
  • WebRTC-通过浏览器连接
  • 移动和桌面应用程序 -具有软件SIP / H.323客户端的智能手机，平板电脑或PC

通过如此广泛的连接选项，选择终端解决方案的选择更多地取决于设备的特性。 但是重要的是要记住硬件和软件终端的优点：

  • 硬件终端无需服务器参与即可进行点对点视频通话。 即 SIP / H.323客户端不仅可以从MCU接收呼叫，还可以直接从SIP / H.323客户端接收呼叫。 对于专有系统，在大多数情况下这是不可能的：例如，在没有Skype服务器或云的情况下，我们无法从Skype客户端调用到Skype客户端。
  • 该软件平台又使您可以赋予终端多功能性。 在一台PC上，我们可以安装专用软件产品和标准SIP / H.323客户端。 除此之外，PC还可以执行与视频会议无关的其他方案
