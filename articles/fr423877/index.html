29-31 octobre: ​​création d'un cluster Kubernetes prêt pour la production


Southbridge mène des cours intensifs en direct et en ligne via Kubernetes.


Le matériel est destiné à ceux qui connaissent Linux, Docker, Kubernetes, Ansible, Helm et Git.


Intensif est avant tout une pratique. Chaque membre créera son propre cluster dans le cloud Selectel .
La partie théorique n'est pas une relecture des manuels, mais l'expérience et les recommandations des intervenants.


Thèmes d'emploi:


Thème №1: Le processus de création d'un cluster de basculement de l'intérieur


• Travailler avec Kubeadm
• Pratique


Sujet n ° 2: Comprendre les contrôleurs Kubernetes


• Informateur, ListWatcher, gestionnaire d'événements de ressources, SharedInformer, Workqueue


Thème №3: Autorisation dans un cluster en utilisant un fournisseur externe


• Intégration avec LDAP: Nginx et Python / Coreos dex


Sujet n ° 4: Réseautage


• Introduction à CNI
• Politique de sécurité du réseau
• Pratique


Sujet n ° 5: applications de cluster sécurisées et hautement disponibles


• PodSecurityPolicy, PodDisruptionBudget


Sujet n ° 6: applications avec état dans un cluster


• Démarrage d'un cluster PostgreSQL
• Démarrage d'un cluster MongoDB


Sujet n ° 7: mise en œuvre de stratégies de déploiement autres que RollingUpdate


• Canari, bleu / vert


Thème # 8: Sauvegarde et récupération après sinistre


• Sauvegarde et récupération de cluster à l'aide de Heptio Ark et etcd


Sujet n ° 9: Garder les secrets dans le coffre-fort. Avantages et inconvénients de l'utilisation d'un cas réel


• L'évolution de la gestion des secrets
• Caveau pour pratiquer, histoire
• Avantages et inconvénients de l'utilisation de Vault


Sujet n ° 10: Dépannage à Kubernetes


• Dépanner dans un cluster
• Dépannage etcd (problèmes, débogage et solution)


Thème №11: Travaux pratiques, dockerisation de l'application et lancement dans le cluster (déploiement canari, revue dynamique, sortie moniteur)
Le travail pratique, la dockérisation des applications et le réglage CI / CD incluent des fonctionnalités avancées:


• Déploiement des Canaries
• Examen de l'environnement dynamique
• Suivi des versions après déploiement, restauration automatique de la version précédente en fonction des résultats de PromQL (Prometheus)
