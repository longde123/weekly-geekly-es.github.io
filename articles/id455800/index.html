Matriks 1.0 - Rilis Protokol Pesan Desentralisasi

Pada tanggal 11 Juni 2019, para pengembang dari Matrix.org Foundation mengumumkan rilis Matrix 1.0, sebuah protokol untuk mengimplementasikan jaringan gabungan berdasarkan sejarah linear dari peristiwa-peristiwa di dalam grafik asiklik (DAG). Cara paling umum untuk menggunakan protokol adalah dengan mengimplementasikan server pesan (misalnya, server Synapse, klien Riot) dan "menghubungkan" protokol lain satu sama lain melalui jembatan (misalnya, implementasi pada libpurple dengan dukungan untuk XMPP, Telegram, Discord, dan IRC).



Inovasi utama (dan prasyarat untuk penggunaan) dari server Synapse 1.0 - implementasi protokol Matrix 1.0 - adalah penggunaan sertifikat TLS (Let's Encrypt gratis juga cocok) untuk domain server, yang menyediakan transfer data yang aman antara server yang berpartisipasi dalam jaringan gabungan. Karena itu, jika Anda menggunakan sertifikat yang ditandatangani sendiri untuk server rumah Anda - Anda harus membuat sertifikat yang valid - jika tidak, server Anda akan berhenti berinteraksi dengan server lain di jaringan.

Rencana rilis protokol Matrix 1.0 diumumkan di Brussels pada bulan Februari 2019 di konferensi Open Source FOSDAM 2019 terbesar sebagai bagian dari implementasi teknologi Yayasan Matrix.org untuk menyediakan infrastruktur komunikasi pemerintah Perancis.

Menariknya, dua bulan yang lalu, server matrix.org menjadi sasaran serangan hacker, oleh karenanya diperlukan untuk membuat ulang database server matrix.org (dengan hilangnya riwayat obrolan terenkripsi yang disimpan di server) - serta meluncurkan kembali aplikasi Riot untuk Android - karena kebocoran kunci dan kata sandi. Peretas meninggalkan rekomendasi untuk meningkatkan proses bisnis dan keamanan server (terkait dengan kerentanan di Jenkins - platform untuk otomatisasi pengembangan dan pengujian perangkat lunak). Server "rumah" Matrix tidak terpengaruh, dengan pengecualian tidak tersedianya "stiker" sementara untuk pesan pengguna dan layanan sekunder lainnya.

Klien Riot.im yang paling populer (versi saat ini 1.2.1) - tersedia baik dalam implementasi desktop dan pada kebanyakan platform seluler - telah mendekati klien serupa untuk Slack dan Telegram dalam hal kenyamanan dan keandalan.



Seperti yang sudah saya tulis , server Synapse agak tidak menuntut perangkat keras - Anda dapat menggunakan mikrokomputer ARM ODROID-XU4 seharga $ 49 untuk server "rumah", dan karena penampilan mesin virtual berdasarkan prosesor ARM Graviton di Amazon Cloud pada akhir tahun lalu, murah membuat cadangan "pusat data rumah mini" di cloud Amazon.
