GitHub Package Registry将支持Swift软件包

5月10日，我们对GitHub Package Registry进行了有限的beta测试，这是一个软件包管理服务，可轻松在源代码旁边发布公共或私有软件包。 该服务当前支持熟悉的软件包管理工具：JavaScript（npm），Java（Maven），Ruby（RubyGems）、. NET（NuGet），Docker映像等等。

我们很高兴地宣布，我们会将Swift软件包支持添加到GitHub软件包注册表中。 使用Swift软件包，可以轻松地在自己的项目中以及与Swift社区共享您的库和源代码。 我们将与Apple的人员合作进行此工作。



这篇GitHub博客文章

Swift Package Manager在GitHub上可用，是一个用于构建，运行，测试和打包Swift代码的跨平台工具。 这些配置是用Swift编写的，可以轻松设置目标，宣传产品和管理软件包依赖项。 结合使用Swift Package Manager和GitHub Package Registry，您可以轻松发布和管理Swift软件包。

对于移动开发者来说，拥有更好的工具来提高生产力非常重要。 随着Swift生态系统的发展，我们很高兴与App​​le团队合作，为Swift开发人员创建新的工作流程。

自从GitHub Package Registry启动以来，我们已经看到社区大力参与使用该工具。 在测试期间，我们寻求向社区学习Package Registry如何满足各种需求以及我们可以做些什么来使其变得更好。 如果您尚未尝试过GitHub Package Registry，则可以在此处申请Beta版 。
