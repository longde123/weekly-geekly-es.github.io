Accordion, faq, spoiler, dan widget drop-down lainnya



Anda dapat membuat akordeon, faq, spoiler, dan sejenisnya menggunakan Div dan JavaScript.
Tetapi lebih baik: Detail dan Ringkasan

Ini didukung oleh semua browser modern dan ini adalah kode yang diformat semantik dengan benar, yang penggunaannya akan memiliki keuntungan:

  1. Orang-orang penyandang cacat akan lebih mudah menggunakan situs Anda! Perangkat lunak mereka (pembaca layar dan sejenisnya) sangat memahami tag html5 dan akan memprosesnya dengan benar dan memberi tahu orang-orang tentang konten dengan benar.
  2. Koherensi teks akan meningkat, dan mesin pencari akan dapat mengindeks situs dengan lebih baik, karena mereka akan lebih memahami bagaimana teks yang terlihat dan tersembunyi terkait.
  3. Ini akan tersedia untuk mengontrol elemen-elemen dari keyboard dan perangkat lain.
  4. Jumlah kode javascript yang akan dimuat berkurang, yang meningkatkan kecepatan pemuatan halaman, kecepatan pemrosesan, dan kebenaran.
  5. Meningkatkan kinerja di Lighthouse, Google PageSpeed dan alat serupa lainnya.
  6. Ini berfungsi saat javascript dimatikan.

Minus:

  1. Browser yang lebih lama tidak tahu tag seperti itu dan tidak akan menyembunyikan informasi.

HTML: 

<details> <summary>- </summary> <p>  —   </p> </details> <details open> <summary>-  2</summary> <p>        </p> </details> <details> <summary>-  3</summary> <p>  —   </p> </details>

Ngomong-ngomong, tampilan default dari tag Details sangat mirip dengan spoiler dari Habr, hanya saja Anda perlu mengecat ulang sedikit, menggarisbawahi dan mendapatkan semantik yang benar, tanpa javascript dan div, spoiler Habrovsk.



Sayangnya, penanda default memiliki dua kelemahan:

  1. Browser lama tidak melihatnya.
  2. Browser Webkit tidak mengizinkan perubahan simbol marker.

Untuk alasan ini, penanda default harus disembunyikan dan buat milik Anda.


Pertimbangkan contoh Detail / Ringkasan pertama dengan penanda teks yang dimodifikasi:



CSS: 

 summary::-webkit-details-marker{display:none;} summary::-moz-list-bullet{list-style-type:none;} summary::marker{display:none;} summary { display:inline-block; padding: .3em .5em .3em .4em; font-size:1.4em; cursor: pointer; } summary:before { content: "+"; margin-right: .3em; } details[open] > summary:before { content: "–"; } summary ~ * { padding:0 1em 0 1em; } summary:focus { outline:0; box-shadow: inset 0 0 1px rgba(0,0,0,0.3), inset 0 0 2px rgba(0,0,0,0.3); } details{ display:block; margin-bottom: .5rem; }

Untuk marker default, kami menampilkan: tidak ada dan kami menampilkan alternatif dengan ringkasan: sebelum {content: "+";}

ringkasan: fokus - sapukan dengan bayangan kotak, ini diperlukan untuk keyboard agar elemen aktif terlihat dan Anda dapat bergerak dengan tombol tab dan membuka dan menutup dengan spasi.

Untuk tag ringkasan, saya atur tampilan: inline-block - ini agar tidak meregang ke lebar penuh dan hanya kata-kata yang dapat diklik, dan bukan seluruh baris.

Sebelum Anda menghapus garis besar, pikirkan 100 kali bagaimana Anda dapat menggantinya, sehingga seseorang dapat melihat fokus dan dapat bergerak dari keyboard atau perangkat lain.

Jika Anda perlu sehingga ketika Anda membuka satu spoiler, yang lain ditutup, Anda harus menggunakan javascript, di bawah ini saya memberikan contoh kode JS sederhana yang akan menyelesaikan masalah ini.

JavaScript: 

 var details = document.querySelectorAll("details"); for(i=0;i<details.length;i++) { details[i].addEventListener("toggle", accordion); } function accordion(event) { if (!event.target.open) return; var details = event.target.parentNode.children; for(i=0;i<details.length;i++) { if (details[i].tagName != "DETAILS" || !details[i].hasAttribute('open') || event.target == details[i]) { continue; } details[i].removeAttribute("open"); } }
