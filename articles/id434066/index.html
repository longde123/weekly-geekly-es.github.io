Sberbank FFD 1.05

Ketika saya melihat artikel ini di sini, saya menangis. Sebuah pukulan tepat di hati yang membuat iri Cupid sendiri. Tetapi dalam artikel ini tidak ada informasi apa yang harus dilakukan.

Masalah dengan situs web Sberbank baru adalah bahwa itu tidak diindeks oleh mesin pencari. Tentu saja Tentu saja Jika kami mengarahkan URL baru mereka ke mesin pencari, maka kami tidak akan melihat apa pun. Tidak ada pencarian di situs juga. Pencarian Google melalui situs tidak akan memberi kami apa pun. Yah, tanahnya adalah Winnie-the-Pooh.

Jika Anda menggunakan REST untuk menerima data dan Anda tidak dapat mengulang kode untuk persyaratan baru, maka Anda

1. Buka panel admin Sberbank dan periksa apakah kasir daring Anda dapat berfungsi mulai 1,05. Ada dalam versi sistem CRF harus 1,05.

2. Lengkapi permintaan Anda dengan tombol-tombol berikut 

"itemAttributes": { "attributes" : [ { "name": "paymentMethod", "value": "1" }, { "name": "paymentObject", "value": "4" } ] }

itemAttributes harus di setiap item item.

3. Dan permintaan penuh 

 { "orderCreationDate": 1544553466000, "customerDetails": { "email": "test@222.ru", "phone": "42", "contact": "Max" }, "cartItems": { "items": [ { "positionId": 1, "name": "FFD 1.05", "quantity": { "value": 1, "measure": "ed" }, "itemCode": "item_1", "tax": { "taxType": 0, "taxSum": 0 }, "itemPrice": 100000, "itemAttributes": { "attributes" : [ { "name": "paymentMethod", "value": "1" }, { "name": "paymentObject", "value": "4" } ] } } ] } }

Ini adalah minimum yang diperlukan untuk memperbarui kode Anda di bawah FFD 1.05. Jika tiba-tiba Anda sangat terkejut dengan format itemAttributes, maka Anda tidak sendirian. Bagaimana menerjemahkan apa yang ditunjukkan di sini menjadi apa yang tertulis di atas, saya tidak terlalu membayangkan. Tapi saya akan senang jika Anda memberi tahu saya di komentar di mana Anda dapat menemukan contoh pemformatan itemAttributes di situs web mereka.
