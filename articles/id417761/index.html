GitLab bergerak dari Azure ke Google Cloud Platform. Berita Relokasi dan Tanggal Pemeliharaan


Terjemahan sebuah posting dari blog GitLab.com.


GitLab.com berencana 28 Juli untuk pindah ke Google Cloud Platform. Kami memberi tahu Anda apa yang akan terjadi pada Anda sekarang dan di masa depan.


Bagi kami, prioritas selalu untuk meningkatkan kinerja dan keandalan GitLab.com . Baik itu maupun yang lainnya secara bertahap tumbuh, dan sementara itu kami merencanakan perubahan serius yang mampu memberikan terobosan nyata: menjadikan GitLab aplikasi cloud ( cloud native ) untuk Kubernetes.


Kami siap mengambil langkah besar menuju cloud asli: pindah dari Azure ke Google Cloud Platform (GCP). Azure adalah penyedia hebat bagi kami, tetapi GCP lebih baik mendukung Kubernetes, yang penting bagi rencana kami yang berjangkauan luas. Dan akan ada efek langsung, termasuk penyimpanan data terenkripsi yang diaktifkan secara default dan caching lebih cepat karena integrasi ketat GCP dengan CDN kami.


Waktu Perawatan untuk Migrasi ke GCP


Ada posting blog tentang migrasi , tetapi di sini kami secara singkat menginformasikan kepada komunitas bahwa kami berencana untuk bermigrasi akhir pekan depan (28 Juli). Pemeliharaan akan berlangsung dalam 2 tahap, yang kita semua beri tahu.


Apa yang harus Anda ketahui:


Layanan berikut tidak akan tersedia selama pemeliharaan:


• Situs SaaS ( GitLab.com tidak akan tersedia, tetapi about.gitlab.com dan docs.gitlab.com akan terus berfungsi)
• Git ssh
• Git https
• pendaftaran
• CI / CD


21 Juli, Sabtu, pukul 13:00 UTC (16:00 waktu Moskow)


Sebagai bagian dari pengujian, kami melakukan perawatan singkat pada hari Sabtu, 21 Juli ini, pukul 16:00 waktu Moskow . Ini adalah ujian terakhir kesiapan. Pemeliharaan membutuhkan waktu 1 jam.


28 Juli, Sabtu, pukul 10:00 UTC (13:00 waktu Moskow)


Pada hari pemindahan, kami akan memulai perawatan pukul 13:00 waktu Moskow. Menurut rencana, GitLab.com akan dihentikan selama 2 jam. Jika ada perubahan, kami akan melaporkan melalui saluran yang ditunjukkan di bawah ini. Pada akhir periode ini, GitLab.com akan berjalan pada GCP.


• Halaman Status GitLab
• GitLab Status Tweeter


Domain Kustom di Halaman GitLab


Jika Anda memiliki domain individual di Halaman GitLab :


• Kami akan meninggalkan proxy, Anda akan punya waktu untuk mengubah DNS.
• Mulai 28 Juli, GitLab Pages akan pindah ke 35.185.44.232.
• Jangan mengubah DNS ke alamat baru sampai kami menyelesaikan migrasi.
• Kami akan melaporkan secara terpisah di blog ketika alamat Halaman GitLab di Azure dinonaktifkan.


Jika Anda memerlukan dukungan selama migrasi, tulis ke Dukungan GitLab .


Semoga beruntung!
