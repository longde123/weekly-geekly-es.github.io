Windows终端介绍

Windows终端程序是一种新的，现代的，快速的，高效的，功能强大的终端应用程序，适用于命令行工具和Shell（例如命令提示符，PowerShell和WSL）的用户。



Windows终端程序将通过Microsoft Store交付到Windows 10，并将定期更新，以确保您始终保持最新状态，并能够以最小的努力使用最新的功能和最新的改进。



Windows终端的主要功能


多个标签


您问了，我们听到了！ 终端最常请求的功能是支持多个选项卡，我们很高兴终于能够提供此功能。 现在，您可以打开任意数量的选项卡，每个选项卡都连接到您选择的命令行外壳或应用程序，例如，命令提示符，PowerShell，WSL上的Ubuntu，通过SSH的Raspberry Pi等。



美丽的文字


Windows Terminal使用基于文本呈现引擎的DirectWrite / DirectX GPU加速。 这个新的文本渲染引擎将显示文本字符，字形和PC上以字体显示的字符，包括CJK表意文字，表情符号，电力线符号，图标，编程连字等。该引擎甚至比以前的引擎更快地渲染文本。 GDI控制台！



您还将有机会使用我们的新字体！ 我们想要创建一种有趣的，新的等距字体，以改善终端的现代外观。 该字体不仅包含编程连字，而且还具有自己的开放源代码存储库。 请继续关注有关新字体项目的更多信息！



设置和可配置性


我们已经联系了许多喜欢自定义终端和命令行应用程序的命令行用户。 Windows终端提供了许多设置和配置参数，可以很好地控制终端的外观以及可以作为新选项卡打开的每个外壳程序/配置文件。 设置存储在结构化的文本文件中，从而简化了用户和/或工具的配置。

使用终端配置机制，您可以为要使用的每个shell /应用程序/工具创建多个“配置文件”，包括PowerShell，命令提示符，Ubuntu，甚至是与Azure或IoT设备的SSH连接。 这些配置文件可以具有自己的字体样式和大小，颜色主题，背景模糊/透明度级别等组合。现在，您可以按照自己的样式创建自己的终端，以个性化您的独特品味！

更多！


Windows Terminal 1.0发布后，我们计划开始使用积压的许多功能，此外，作为社区的您可能还会添加许多功能！

什么时候可以买到？


今天，Windows终端和Windows控制台可在开源中使用，因此您已经可以从GitHub上的存储库中克隆，编译，运行和测试代码：

github.com/Microsoft/Terminal

另外，今年夏天，Microsoft商店将为先驱发布Windows Terminal的预览版，并提供反馈。

在冬季，我们计划最终发布Windows Terminal 1.0，我们将与社区合作，以确保在发布前它已完全准备就绪！

等等...您是在开源上说的吗？


是的，是的！ 我们很高兴地宣布，我们不仅要开放Windows终端，而且还要开放Windows控制台，该控制台包含Windows中的命令行基础结构并提供传统的控制台UX。

我们渴望与您一起改善和扩展Windows命令行的功能！

这听起来很棒，但是为什么不升级现有的Windows控制台呢？


Windows控制台的主要目标是保持与现有命令行工具，脚本等的向后兼容性。尽管我们设法对控制台功能进行了许多关键的改进（例如，添加了VT支持和24位颜色等。 博客文章 ），如果不“打破常规”，就无法对UI控制台进行进一步的重大改进。

因此，现在是一种新的，新鲜的方法的时候了。

Windows Terminal安装并与现有的Windows Console应用程序一起使用。 如果直接启动Cmd / PowerShell / etc。，它们将以与通常相同的方式开始连接到传统控制台实例。 因此，向后兼容性保持不变，同时，如果需要，您可以使用Windows终端。 Windows控制台将继续交付Windows数十年，以支持现有/旧版应用程序和系统。

那么，如何参与开源中现有的终端项目或应用程序呢？


我们在计划过程中仔细研究了此选项，并决定我们参与现有项目将需要更改项目的要求和体系结构，以免造成破坏性。

相反，通过创建新的开放源代码终端应用程序和开放源代码Windows控制台，我们可以为社区提供帮助，以改进代码并将其用于各自的项目中。

我们相信市场上有足够的空间来容纳关于终端可以和应该做什么的新的/不同的想法，并且我们通过引入新的想法，有趣的方法和激动人心的创新，努力帮助终端（及相关）应用程序生态系统蓬勃发展和发展。在这个空间。
