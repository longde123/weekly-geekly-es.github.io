Metode atribusi di pelacak populer: Pencocokan ID Perangkat, Instal pengarah dan Sidik Jari

Dalam artikel ini, atas nama agensi teknologi pemasaran seluler Mobio , kami akan menjelaskan dan membandingkan metode atribusi dasar Pencocokan ID Perangkat, Instal perujuk dan Sidik Jari yang menggunakan pelacak aplikasi seluler.

Bayangkan situasinya: aplikasi sedang dipromosikan oleh tim in-house, serta oleh beberapa agensi. Pemilik aplikasi perlu memahami siapa yang membawa pengguna: tim mereka sendiri, agensi, atau apakah itu lalu lintas organik. Tugas ini membantu menyelesaikan pelacak seluler. Tugas utama pelacak, misalnya, seperti Sesuaikan, AppsFlyer, Appmetrica, adalah untuk membandingkan pemasangan klik pada iklan yang mengarah ke pemasangan ini. Pada artikel ini kita akan berbicara tentang metode dasar atribusi.


Pencocokan ID Perangkat


Pencocokan ID Perangkat adalah metode atribusi menggunakan id perangkat / id iklan - GAID (Google Advertising ID) atau iOS IFA (Apple Identifier for Advertising). Ini adalah metode atribusi utama dan paling akurat.

Cara kerjanya:

- Ketika Anda mengklik spanduk, jaringan iklan mengganti berbagai parameter ke dalam tautan pelacakan, termasuk GAID atau iOS IFA, tergantung pada OS;
- Pengguna melalui pengalihan melalui pelacak aplikasi seluler; parameter klik yang diganti oleh jaringan iklan disimpan di server;
- Setelah menginstal dan membuka aplikasi, SDK pelacak seluler mengirim informasi tentang perangkat ke server, termasuk pengidentifikasi iklan;
- Di server, parameter perangkat dibandingkan ketika aplikasi pertama kali dibuka dan ketika diklik.

Fitur Pencocokan ID Perangkat:

- Metode atribusi 100% paling akurat;
- Ini hanya berfungsi jika jaringan iklan mengirimkan id perangkat / id iklan;
- Jendela atribusi di pelacak yang berbeda mencapai 30 hari.

Pasang pengarah


Pemasang pengarah adalah metode pengaitan yang didasarkan pada parameter unik INSTALL_REFERRER, yang diteruskan dengan referensi.

Cara kerjanya:

- Pengguna mengklik spanduk iklan dan meneruskan pengalihan melalui pelacak aplikasi seluler, pelacak menambahkan parameter & pengarah ke tautan, yang nilainya diganti dengan pengidentifikasi unik;
- Pengguna menginstal dan membuka aplikasi, setelah itu pelacak aplikasi seluler menerima nilai parameter pengarah dari toko aplikasi;
- Pelacak membandingkan pengarah, diganti di tautan, dengan yang dikeluarkan toko aplikasi, dan menghasilkan atribusi.

Fitur Instal pengarah:

- Akurasi yang digunakan tidak kalah dengan Pencocokan ID Perangkat;
- Atribusi dimungkinkan tanpa mentransfer id perangkat / id iklan dari jaringan iklan;
- Hanya berfungsi untuk Google Play store;
- Jendela atribusi sama dengan Pencocokan ID Perangkat.

Sidik jari


Sidik jari - metode yang didasarkan pada informasi tentang perangkat yang dikumpulkan pelacak pada saat pengalihan.

Cara kerjanya:

- Ketika pengguna mengklik spanduk, pengguna melewati pengalihan melalui pelacak aplikasi seluler, pada saat ini pelacak mengumpulkan informasi yang tersedia tentang perangkat pengguna: browser-agen pengguna, alamat IP perangkat, waktu, dll. Kemudian pengguna diarahkan ke toko aplikasi;
- Ketika pengguna menginstal dan membuka aplikasi, SDK pelacak mengumpulkan informasi tentang perangkat dan mengirimkannya ke server pelacak;
- Pelacak membandingkan parameter pada klik dan selama instalasi. Mengaitkan penginstalan dengan klik;

Fitur sidik jari:

- Jendela atribusi (interval waktu antara klik dan pembukaan aplikasi pertama) di sebagian besar pelacak tidak lebih dari 24 jam;
- Mengaktifkan atribusi tanpa mengirimkan pengidentifikasi iklan dari jaringan iklan. Misalnya, ketika menggunakan milis;
- Bukan metode atribusi 100% akurat.

Kami memeriksa metode atribusi dasar yang digunakan oleh pelacak aplikasi seluler. Pertama-tama, metode Pencocokan ID Perangkat digunakan. Meskipun akurasi tinggi dan kemampuan untuk menetapkan jendela atribusi yang agak besar, metode ini membutuhkan id perangkat / id iklan yang diterima dari jaringan iklan, dan ini tidak selalu memungkinkan. Untuk Android, ada alternatif yang sama akuratnya - Pasang pengarah dengan jendela atribusi yang sama, yang tidak memerlukan transfer id perangkat / id iklan. Jika kedua metode ini tidak memungkinkan, sidik jari digunakan, itu tidak seakurat metode yang disebutkan di atas, ia memiliki jendela atribusi yang agak kecil, tetapi masih memiliki tempat untuk menjadi.
