Ketika cadangan tidak cocok pada USB flash drive, atau bagaimana tidak membeli HDD eksternal

Mempertimbangkan komentar yang saya terima, saya mengklarifikasi untuk pembaca baru bahwa artikel ini adalah tentang bagaimana saya menyimpan, tanpa membuat cadangan bahan yang tersedia untuk umum (file, halaman web, media cetak), dan mencadangkan hanya informasi tekstual tentang mereka (tanggal pengunduhan, tautan, lainnya informasi bibliografi), dan bagaimana tepatnya saya melakukannya.


Hasil


Ukuran file yang akan didukung telah menurun dengan urutan besarnya. Misalnya, untuk hari ini (hari kerja dengan intensitas sedang) saya memperoleh 117 MB, yang setara dengan 248 KB sebagai cadangan (penurunan ukurannya 483 kali lipat).


Pendahuluan


Beberapa tahun terakhir saya telah terlibat dalam copywriting (dalam arti kata yang luas), dan sebagian besar data saya terdiri dari file teks dalam berbagai format dan dokumen PDF (termasuk sering dengan banyak informasi grafis). Saat ini saya sedang mengerjakan ASUS Eee PC 1003HAG pada Windows 7 (320 GB HDD).


Saya berhati-hati dalam membuat cadangan data, tetapi saya lebih suka media lokal. Saya tidak memiliki HDD eksternal, tetapi ada dua flash drive: 1 GB dan 2 GB.


Masalah


Saya mulai menemui kekurangan ruang pada flash drive, tetapi sejauh ini tidak ada keinginan untuk membeli media baru. Pada dasarnya, karena sebagian besar informasi yang benar-benar perlu didukung adalah sebagian kecil dari file saya.


Lebih khusus lagi, saya hanya perlu mencadangkan informasi yang tidak dapat saya pulihkan dari mana pun - hasil aktual dari pekerjaan saya: terutama teks yang saya tulis dan informasi tentang sumber saya. Pada saat yang sama, sebagai suatu peraturan, saya tidak memerlukan cadangan sumbernya sendiri (biasanya dokumen PDF). Yaitu, mereka, ternyata, menempati sebagian besar ruang di sekrup saya.


Solusi


Ada empat bagian pada sekrup saya:


  • Sistem ( C ),
  • Workshop ( W ),
  • Sastra ( L ),
  • Redists ( R ).

Pada saat yang sama, saya mendistribusikan file sebagai berikut:


  • pada W - file yang pada dasarnya tidak dapat dipulihkan (hasil pekerjaan saya, apakah itu berupa teks, sumber, pindaian, atau tipe data lainnya);
  • pada L - file yang dapat dipulihkan secara fundamental berguna untuk pekerjaan (publikasi - salinan elektronik dari cetakan dan elektronik),
  • pada R - distribusi perangkat lunak yang digunakan,
  • di C , yang lainnya, termasuk file hiburan.

Untuk W dan L mematuhi struktur data yang ketat:


  • di root - hanya direktori, di direktori - baik hanya file, atau hanya direktori;
  • Saya memberi nama file dan direktori sesuai dengan seperangkat aturan (khususnya, saya menggunakan tanggal dan waktu saat ini sebagai pengidentifikasi unik dan kode dua huruf ISO 3166-1 alpha-2 - ini adalah topik untuk diskusi terpisah);
  • pertama, saya menempatkan semua file yang diunduh di direktori yang sesuai di W (pada saat yang sama, saya tidak mencoba untuk â€œmenjejalkanâ€ file di suatu tempat: jika saya tidak memiliki direktori yang sesuai, saya membuat satu).

Saya memilih PDF sebagai format tunggal untuk menyimpan sumber: menyimpan yang asli, saya mengonversi dokumen teks ke PDF, mencetak halaman web dan gambar dari browser saya ke dalam PDF.


Menggunakan perangkat lunak, File Metadata menambahkan properti "Halaman Web" ( System.Contact.WebPage ) dan "Tanggal Impor" ( System.DateImported ) ke Windows Explorer untuk file PDF, yang saya isi untuk file tersebut segera setelah mengunduh. Juga menambahkan bidang "Komentar" ( System.Comment ), di mana saya menulis informasi bibliografi tentang publikasi cetak. Dan beberapa bidang lainnya: "Status Tugas", "Judul", "Penulis", dll. - Saya secara aktif menggunakan alat manajemen kutipan alih-alih bekerja, tetapi ini adalah topik untuk diskusi terpisah. Awalnya saya pikir akan membosankan dan menghabiskan waktu untuk memasukkan metadata setiap waktu, tapi tidak, saya sudah terbiasa.


Akhirnya, menggunakan item menu konteks yang disediakan oleh perangkat lunak tersebut, saya mengekspor nilai properti ini ke file XML markup khusus, nama yang disalin oleh program dari file PDF yang sesuai.


Dan voila! Saya mentransfer file PDF ke direktori serupa di L , dan meninggalkan file XML di W


Saya memesan file dengan W pada media eksternal.
