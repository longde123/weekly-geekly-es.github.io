Presentamos Tartiflette: una implementación de código abierto de GraphQL para Python 3.6+

Amigos, en vísperas de las vacaciones de mayo, decidimos no bombardearlos con artículos técnicos complicados, por lo que encontramos material bastante interesante y, lo más importante, fácil de leer, cuya traducción nos complace compartir con ustedes. Este material queremos coincidir con el lanzamiento del curso "Desarrollador web en Python" .

El original se puede encontrar aquí .



La adquisición de Vivendi dailymotion hace tres años fue un punto de inflexión para nuestra organización. Esto nos permitió repensar el vector de nuestro trabajo, repensar nuestro trabajo de principio a fin. Aprovechamos la oportunidad para evaluar el movimiento diario en general, para repensar nuestra infraestructura y, lo que es más importante, la arquitectura de nuestros productos.

Al final, el autoanálisis confirmó lo que ya sabíamos: queríamos distribuir geográficamente nuestra plataforma y desarrollar API , aplicaciones móviles y de TV. Esto marcó el abandono de la estructura monolítica actual y la adopción de un enfoque orientado a la API. Este artículo describe el camino que seguimos.

Criterios y conceptos de validación

Comenzamos el proyecto definiendo criterios API, que finalmente se redujeron a cuatro puntos:

  • Proporcionando una buena DX (experiencia del desarrollador), fácil uso e implementación;
  • El cultivo de una comunidad sólida y en crecimiento , que permita la agregación y el uso de tecnología generalizada;
  • Comportamiento similar a una puerta de enlace en términos de arquitectura de software , destinado a simplificar la transformación de la integración de nuestro sistema de un monolito a SOA (arquitectura orientada a servicios);
  • Capacidad para implementar herramientas de administración de API, es decir, administración de API, documentación y acceso a datos.

A continuación, seleccionamos varios modelos de API y probamos varios conceptos para comprender su idoneidad:

  • Rest API con Swagger
  • API GraphQL con grafeno
  • API con Falcor



Después de pruebas rigurosas, descubrimos que GraphQL y sus implementaciones de Graphene cumplían nuestros criterios mejor en comparación con otros modelos. Esto permitió a nuestros desarrolladores front-end usar más fácilmente nuestra API y, al mismo tiempo, simplificar su uso en aplicaciones cliente (por ejemplo, React JS y Apollo Client). En nuestra arquitectura, GraphQL también es más simple y más eficiente como pasarela de patrones. Al final, finalmente decidimos seguir adelante con GraphQL y Graphene.

Camino a la producción

En abril de 2017, después de un intenso desarrollo de seis meses, comenzamos la producción con nuestra API. En el verano, cambiamos todos los productos de Dailymotion (web, móvil y TV) a nuestra API GraphQL.



Cuando elegimos GraphQL hace tres años, todavía estaba en beta y no alcanzó la popularidad que tiene hoy. Fuimos los primeros jugadores importantes en este campo y esto hizo que nuestra reconstrucción interna fuera aún más agradable.

Nacimiento de Tartiflette

En los primeros meses de 2018, después de más de seis meses de usar Graphene, decidimos dar un paso más y escribir nuestro propio motor GraphQL. Esto nos permitió implementar algunos requisitos que Graphene no cumplió. Hemos desarrollado criterios para nuestro propio motor. El debe:

  • Proporcione el mejor DX para los desarrolladores de Python;
  • Use SDL (lenguaje de definición de esquema);
  • Use asyncio como un motor de ejecución independiente;

Después de casi un año de desarrollo y muchas semanas de probar nuestra infraestructura (en promedio se procesaron más de 100 millones de llamadas por día), estamos orgullosos de ofrecer a la comunidad GraphQL nuestro propio motor Tartiflette de código abierto.

ADN Tartiflette

Tartiflette es una implementación del servidor GraphQL construida en Python 3.6+
El esquema GraphQL se describe utilizando el nuevo lenguaje de definición de esquema (SDL) ;
El rendimiento es un elemento clave de nuestro trabajo y esto se refleja en Tartiflette ;
Construido con Zen of Python en mente. No demasiado complicado.

Hola mundo en tartiflette 



import asyncio from tartiflette import Engine, Resolver @Resolver("Query.hello") async def resolver_hello(parent, args, ctx, info): return "hello " + args["name"] async def run(): ttftt = Engine(""" type Query { hello(name: String): String } """) result = await ttftt.execute( query='query { hello(name: "Chuck") }' ) print(result) # {'data': {'hello': 'hello Chuck'}} if __name__ == "__main__": loop = asyncio.get_event_loop() loop.run_until_complete(run())

Puede encontrar información sobre la nueva funcionalidad en el manual en tartiflette.io .

¿Qué pasará después con Tartiflette?

El Tartiflette de código abierto es solo el primer paso. Aquí hay algunas ideas y planes para el futuro para el desarrollo de Tartiflette:

  • Documentación de "pulido" para usuarios y la comunidad;
  • Proporcionando más ejemplos y ampliando casos de uso para inspirarte a experimentar con las capacidades de Tartiflette ;
  • Mayor productividad , principalmente a nivel del motor de ejecución.
