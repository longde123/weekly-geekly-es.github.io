ADAM-6200 I / O Modul


Modul I / O jarak jauh dirancang untuk berkomunikasi dengan berbagai jenis periferal. Ini adalah elemen penting dalam membangun sistem industri. Keduanya dapat menerima sinyal dari perangkat lain, dan mengirim sinyal kontrol ke perangkat, berintegrasi dengan sistem kontrol pusat SCADA melalui MODBUS TCP, MQTT, HTTP , dll.

Perangkat I / O generasi sebelumnya yang digunakan dalam industri biasanya terhubung menggunakan antarmuka serial RS-232/485, yang membuat skalabilitas sistem semacam itu menjadi sulit. Seri modul ADAM-6200 dibedakan oleh kemampuan untuk terhubung melalui Ethernet, serta sakelar bawaan untuk 2 port Ethernet, yang memungkinkan menghubungkan perangkat secara seri dalam suatu rangkaian.

Selain transmisi sinyal, modul ADAM-6200 dapat memainkan peran pengontrol logika yang dapat diprogram untuk menyelesaikan tugas otomatisasi sederhana. Berkat dukungan bahasa logika grafis bersyarat, mereka dapat diprogram bahkan tanpa pengetahuan bahasa pemrograman.

Artikel ini membahas karakteristik perangkat, ruang lingkup dan fungsi tambahan perlindungan terhadap kegagalan.

Spesifikasi teknis


Seri ADAM-6200 disajikan oleh berbagai model untuk kebutuhan industri apa pun. Perangkat dapat dihubungkan untuk pekerjaan otonom bersama (peer-to-peer), yang memungkinkan Anda untuk mendapatkan kombinasi port yang diperlukan.

  • ADAM-6217 - 8 saluran input analog
  • ADAM-6224 - 4 saluran output analog
  • ADAM-6250 - 15 saluran I / O digital
  • ADAM-6251 - 16 saluran input digital
  • ADAM-6256 - 16 saluran output digital
  • ADAM-6260 - 6 saluran relai
  • ADAM-6266 - 4 saluran relai


Protokol yang didukung


  • Modbus TCP adalah standar industri untuk menghubungkan perangkat melalui IP. Semua perangkat dalam seri mendukung protokol ini.
  • MQTT adalah protokol Internet of Things modern, sering digunakan untuk terhubung ke sistem cloud seperti Microsoft IoT hub dan sejenisnya. Pesan di dalam MQTT dikirim dalam format JSON .
  • HTTP REST API - antarmuka web dan REST API memudahkan untuk mengintegrasikan perangkat ke dalam solusi yang ada berdasarkan platform apa pun: NodeJS, PHP, Python, dll.
  • SNMP - digunakan untuk mengonfigurasi pengaturan jaringan dan mengumpulkan statistik.

Fleksibilitas API HTTP memudahkan untuk mengintegrasikan perangkat ADAM-6200 ke platform apa pun yang ada, tanpa perlu protokol industri tertentu.

Pemrograman Menggunakan Logika Bersyarat


Tugas otomatisasi sederhana yang tidak memerlukan daya pemrosesan besar dapat diprogram secara langsung pada pengontrol ADAM menggunakan Logika Kondisi Grafis. Misalnya, aktifkan sirene jika terjadi kecelakaan, tutup katup jika sensor, dll. Lingkungan GCL hampir tidak memerlukan keterampilan pemrograman dan memungkinkan Anda untuk membuat skrip otomatisasi lengkap menggunakan kursor mouse.



Kembangkan skrip GCL di Adam / Apax .NET Utility . Sebagai kondisi, keadaan input digital atau analog, penghitung sinyal diskrit, timer internal, dll. Dapat digunakan. Aturan dapat merujuk satu sama lain dan membentuk kondisi yang lebih kompleks. Baca lebih lanjut tentang lingkungan pengembangan GCL dalam dokumentasi .


Lingkungan Pengembangan Logika Bersyarat

Koneksi Seri (Rantai Daisy)


Perangkat seri ADAM-6200 memiliki dua port ethernet on board, yang memungkinkan mereka untuk dihubungkan secara seri, dengan rantai daisy. Ini sangat menyederhanakan topologi jaringan dan menghilangkan kebutuhan untuk switch tambahan ketika membangun bagian-bagian terpencil dari jaringan.



Berbeda dengan koneksi yang menggunakan antarmuka RS-232/485, koneksi ethernet membuatnya mudah untuk mengintegrasikan pengontrol ke jaringan IP apa pun, serta menghubungkan objek jarak jauh melalui Internet menggunakan terowongan VPN, tanpa perlu menggunakan pengonversi antarmuka tambahan.

Perlindungan Power Break (Auto-Bypass)


Fungsi Auto-Bypass secara otomatis diaktifkan ketika daya terputus pada perangkat perantara dalam rantai. Dalam mode ini, perangkat bertindak sebagai koneksi pasif, seolah-olah kabel terhubung langsung. Dalam hal ini, ada batasan untuk koneksi UTP, jadi penting untuk mempertimbangkan panjang total kabel antara dua titik tetangga dan tidak melebihi 50 meter di setiap sisi, karena panjang maksimum koneksi pasif adalah 100 meter. Waktu fungsi kerja otonom hingga 4 hari.


Berkat Auto-Bypass, komunikasi tidak terputus bahkan ketika perangkat perantara mati energi

Kesimpulan


Modul Universal ADAM-6200 I / O memiliki dukungan protokol yang luas: Modbus TCP, MQTT, HTTP REST, dan dapat dengan mudah diintegrasikan ke dalam sistem SCADA klasik, serta ke dalam setiap produk perangkat lunak modern.

Dukungan untuk program dalam bahasa GCL memungkinkan Anda untuk menyelesaikan tugas otomatisasi sederhana tanpa menggunakan perangkat tambahan. Interoperabilitas peer-to-peer memungkinkan pertukaran data secara langsung antar perangkat, tanpa menggunakan server pemrosesan data perantara.

Kemampuan untuk menghubungkan perangkat secara berurutan dalam rantai membuatnya mudah untuk membangun segmen besar jaringan tanpa peralatan tambahan, dan fungsi Auto-Bypass melindungi terhadap jeda baris karena kegagalan perangkat di rantai.
