DataGrip 2019.1：支持新数据库，初始化脚本，新检查等

你好 我们来看一下DataGrip 2019.1中的新功能。 回想一下，DataGrip的功能包含在我们的其他付费IDE中，WebStorm除外。



支持新数据库


在此版本中，四个数据库在我们的工具中获得了官方支持：

Apache Hive是基于Hadoop平台的数据库管理系统。
Greenplum是用于基于PostgreSQL的数据仓库的分析型DBMS。
Vertica-大数据分析的列基础。
雪花 -云数据存储。 如果我们谈论关系数据库，那么Snowflake 询问最多 。 在此版本中，我们仅支持SQL；稍后将发布instrotecation。



连接方式


我们对数据库连接对话框进行了更改：我们试图使其更加清晰和方便。

一般


在此选项卡中，主要发生了重构。

连接类型字段以前称为URL类型 ，位于最底部。 但是，由于此字段中的值确定了进一步的过程，因此它现在位于顶部。

在输入登录名和密码后找到“ 数据库”字段，因为需要身份验证才能通过Ctrl / Cmd + Space显示数据库列表。



在上一篇文章的评论中，有很多有关保存密码的讨论 。 添加了新选项，并创建了一个下拉列表。 该列表的值是：

-不要保存密码。
-保存直到DataGrip重新启动（以前“不保存”选项以这种方式起作用）。
-保存到会话：直到与数据源断开连接。
-永远保存。



为避免混淆，请通过上下文菜单输入空密码。



现在，“ 测试连接”结果显示在窗口本身中，没有额外的单击和对话框。



而且，如果未下载驱动程序，DataGrip将提供此功能。 以前，“ 测试连接”按钮随后被阻止，这使用户感到困惑。



选件


只读 ， 自动同步 ， 事务控制设置已从“常规”选项卡移至此处。

新品 ：

- 每N秒运行一次保持活动查询：每N秒粘贴一次数据源。 对于我们不支持的数据库，您可以自己编写一个保持活动请求。 这是在驱动程序设置中完成的。

-N秒后 自动 断开连接 ：在此处输入的值（以秒为单位）将指示DataGrip，之后它将自动与数据源断开连接。

- 启动脚本 ：您可以在此处输入一个请求，该请求将在每次创建连接时执行。 回想一下，如果单连接
模式未启用，将为每个新控制台创建一个新连接。



模式


树中显示的对象的过滤器已移至此处。



导航和搜索


最近位置清单

新的“最近的位置”窗口显示了您最近的位置。 列表项是您最近编辑或观看的一小段代码。 如果您记住上下文但不记得文件名，这将很有用。 在DataGrip中，通常会发生这种情况，因为所有控制台的命名方式都相似：)默认情况下，键盘快捷键是：
Ctrl / Cmd + Shift +E。



如果您之前使用此键盘快捷键来显示最近修改的文件列表，那么现在请使用Ctrl / Cmd + E双击。

搜寻方式

我们删除了从平台上获得的不必要选项： Module和Project 。 现在，默认情况下，DataGrip中的“ 在路径中查找”将在所有位置搜索。 我们还为“ 附件目录”添加了一个新的搜索区域-它仅包含“文件”面板中的文件和文件夹。



导航结果中的动作

现在，从导航结果中，适用于代码或树中对象的操作。 例如，您正在寻找一张桌子。 您可以在结果窗口中执行以下操作。

-查看DDL： Ctrl / Cmd +B。
-打开数据： F4 。
-打开“修改表”窗口： Ctrl / Cmd + F6 。
-在其他上下文中显示： Alt + F1 （例如，在树中显示）。
-参见一般信息： Ctrl + Q / F1 。
-生成SQL： Ctrl / Cmd + Alt + G。



使用代码


自动完成组合元素
对于CREATE和DROP自动完成功能提供了组合选项。



不要忘记缩写。



新检查

如果使用未打开的游标，DataGrip将发出警告。



默认情况下，以下两项检查是关闭的，但有些检查可能需要。

如果使用未命名的参数，它将突出显示。



GOTO运营商宣誓就职。



处理文件


添加了默认项目文件夹的设置。 新项目将在此文件夹中创建。



立即另存为控制台的操作：
-提供默认的项目文件夹。
-请记住最后的选择。



他们在文件树中添加了“ 分离目录”操作：取消固定文件夹。 以前，要取消固定文件夹（即不在此树中显示），您必须单击Delete ，然后DataGrip询问：您要删除还是取消固定？ 这是不便和不可理解的:)



数据库树


我们已经为DB2进行了自省。 这意味着我们像以前一样使用查询而不是通过JDBC驱动程序来获取有关数据库对象的信息。 因此，以前不存在的对象出现在树中：触发器，类型，方法，模块，计数器，角色等。



树包含上下文：数据源的名称位于顶部。



对于不受支持的数据库，绘制了图标：那些为其数据源创建数据源的图标将不再引起混淆。



我们还绘制了抽象图标，可以在驱动程序设置中使用它们。
