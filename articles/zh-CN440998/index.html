OpenStack扩展产品组合并参与CI / CD

OpenStack基金会宣布，该组织不仅将支持 IaaS的开源项目， 还将支持从事容器化，CI / CD和外围计算解决方案开发的公司。

我们告诉方向变化与什么有关，以及谁在与该基金合作。


/照片eflon CC BY

为什么要改变方向


根据2018年OpenStack基金会的一项调查，大约81％的全球公司认为在未来几年中，开源解决方案将成为云的关键部分。 同时，OpenStack的代表指出 ，企业很难一起“使用”各种开源解决方案。 因此，该组织决定扩大该基金的项目范围，以形成一个单一的生态系统并简化开放工具的集成。

除了云系统（仍在组织的产品组合中）之外，OpenStack还将开发CI / CD项目和容器。 为此，该基金开辟了一个新的方向-OpenDev 。 在此基础上，将为各种工具（包括与OpenStack生态系统无关的工具）形成托管。

为了进一步强调改变方向的重要性，OpenStack将其常规的OpenStack Summit活动重命名为Open Infrastructure Summit。 现在，会议将讨论的不是OpenStack，而是一般的开源项目：从云技术到IT基础架构的安全性。

谁与基金合作


计划参与者由OpenStack Foundation董事会成员亲自选拔。 将会选择全新的和已经开发的开源项目。 该基金将为公司的发展提供所有可能的帮助-提供计算资源并任命导师。 现已成为OpenStack Foundation一部分的四个项目：

飞艇 一堆用于自动化云设置的工具。 该系统提供了使用YAML文档管理容器的功能，从而有助于简化Kubernetes集群的工作。 为了创建容器，飞艇使用了 Helm，同时使用了Ceph的功能来存储数据，并使用Calico项目来组织虚拟网络环境。 飞艇由AT＆T和SKT移动运营商开发。 因此，该解决方案有望帮助电信运营商为5G网络部署IT基础架构。

卡塔集装箱。 该项目的目标是创建安全的虚拟机，其性能可与容器媲美-它们的启动时间不到100毫秒。 在这种情况下，该项目将采用英特尔的最佳实践-透明容器的概念，该概念最初侧重于容器环境的安全性。 Kata Containers的源代码可以在GitHub上找到 。

StarlingX。 用于基于外围计算来组织基础结构的平台。 它将帮助将物联网设备连接到单个云系统。 StarlingX为开发人员提供了自动配置和支持应用程序的工具。 该平台于2018年10月发布-有关该平台的文档可在官方网站上找到 。

Zuul。 持续集成和代码交付的系统（CI / CD）。 Zuul专注于具有大量开发人员承诺的大型项目。 该解决方案会自动测试代码中的每个建议更改，并分析它是否会导致服务违规。 该项目自2012年以来一直存在。 在此期间，Zuul设法引进了宝马，GoDaddy和Wikimedia等公司。

IT社区意见


OpenStack基金会的代表期望新的工具和服务能够帮助吸引用户加入OpenStack生态系统。 据专家称，进入中国市场的机会对于OSF尤其重要，因为在该市场中，该组织的项目并不受欢迎。 OpenStack对于中国公司来说很有趣，但他们通常对该功能不满意。 扩展项目池将有助于“弥补” OpenStack服务中的空白并简化与它们的工作（因为它们被认为很难实施）。

同时，IT行业的一些代表对OpenStack Foundation计划表示了担忧，例如Ubuntu和Canonical的创建者Mark Shuttleworth 。 据他介绍 ，OpenStack基金会感兴趣的领域的扩大可能导致该组织不会对其主要产品给予足够的重视。

在不久的将来OSF 将开始两倍地发布OpenStack更新的事实可能会加剧这种情况。 Shuttleworth怀疑，以这种发布速度和对第三方项目的支持速度，该组织将能够维持其领先服务的质量。

还有谁在开发开放云项目


除了OpenStack基金会之外，Linux基金会还在开发开源云解决方案（ 我们在博客中讨论了该基金会的新计划）。 为此，该组织成立了两个子基金-Cloud Native Computing Foundation（CNCF）和Cloud Foundry Foundation（CFF）。

Cloud Native Computing Foundation开发的最著名的项目之一是Kubernetes，但该基金还支持其他服务和工具。 其中，最成熟的是Prometheus监视系统，Envoy负载平衡器和CoreDNS DNS服务。 大型IT公司（例如：阿里巴巴，思科，谷歌，IBM，微软） 参与了该组织项目的开发。

Cloud Foundry Foundation正在开发一个统一的系统，用于管理数据中心的IT基础架构。 该组织的主要服务是应用程序运行时和容器运行时，它们分别帮助开发人员在云或容器中运行应用程序。 Cloud Foundry Foundation与Google，IBM，Dell和Huawei合作。

OpenStack基金会计划与Linux基金会紧密合作。 根据OpenStack代表的说法，该公司并非试图与其他基金竞争，而是希望联合起来并加强开源社区。 目前尚不清楚这种愿望是否会扩散到两个基金的任何联合项目中，但是，他们今年OpenStack基金会的一些计划可能会在今年春天在丹佛举行的主题会议上公布。
