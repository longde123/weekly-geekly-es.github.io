Artículos de la Conferencia de Minsk C ++ CoreHard Spring 2018


Del 7 al 8 de abril, la comunidad CoreHard celebró la octava conferencia internacional CoreHard Spring 2018 en Minsk sobre lenguaje C ++ y tecnologías relacionadas "hardcore" en Minsk. A la conferencia asistieron oradores de las principales empresas de TI en Bielorrusia, Rusia, Ucrania, Alemania y los Países Bajos. Hablaron sobre su experiencia en desarrollo y pruebas.

Ahora los informes de la conferencia están disponibles gratuitamente en YouTube y todos pueden familiarizarse con ellos. Gracias a los organizadores de la conferencia.


Evgeny Okhotnikov. 25 años de historia de C ++ volando ante mis ojos



El autor del informe conoció a C ++ en 1991, y desde 1992, C ++ ha sido el principal lenguaje de desarrollo para el hablante. ¿Qué le pasó a la lengua durante este tiempo? ¿Cómo y por qué se hizo popular? ¿Cómo comenzó el estancamiento en el desarrollo de C ++? ¿Cómo ha perdido C ++ su popularidad? ¿Hay espacio para C ++ en el mundo moderno? Tratemos de hablar sobre esto, basado en 25 años de experiencia en programación en C ++.

Sergey Solozhentsev. Enfoque basado en características para el desarrollo de juegos



Hablemos de cómo han cambiado los enfoques para trabajar con objetos de juego en juegos de match 3, y cómo llegamos al enfoque actual de componente de característica para crear objetos de juego mutables. ¿Cuáles son sus ventajas sobre otros enfoques probados para crear tutoriales, numerosos modos y revisión de código?

Andrey Yakimov. Capturar funciones en Windows en aplicaciones que usan C / C ++



Hablemos de enlaces de funciones en aplicaciones escritas en varios lenguajes y tecnologías. El informe considerará: conceptos básicos, aplicaciones, variedades de métodos de intercepción y sus características técnicas, bibliotecas listas para usar.

Andrey Karpushin. C ++ para web usando Emscripten



En el informe, el autor habla sobre su experiencia práctica en la compilación y uso de C ++ en páginas web, sobre lo que se debe hacer para portar una aplicación en C ++ a la web, sobre las limitaciones asociadas con esto, sobre la integración del código C ++ con JavaScript y otros elementos web páginas

Vasily Nemkov. El núcleo de la billetera multidivisa Multy



3.5 blockchain de 2 megabytes: cómo, por qué y por qué. Características del desarrollo de soluciones blockchain multiplataforma.

Alexander Markevich. De C ++ a Objective-C



En este informe, el autor habla sobre el hecho de que Obj-C / C ++ es una extensión de C / C ++ existente. También considerará las características del lenguaje, hablará sobre la administración de memoria (retener / liberar vs. ARC vs. shared_ptr / unique_ptr), sobre Swift y por qué es mejor o peor que Obj-C / C ++.

Mikhail Matrosov. C ++ multifacético en la práctica



En C ++, hay muchas formas de resolver el mismo problema. Tomaremos el problema real de la práctica del autor y exploraremos una serie de herramientas de C ++ para resolverlo: contenedores STL, boost.range, rangos de C ++ 20, rutinas. Compararemos las soluciones en términos de sus interfaces y rendimiento, y también veremos cómo una solución puede obtenerse fácilmente de otra si el código está organizado correctamente. En el proceso, observamos las características de C ++ 17: constexpr if, sentencias de selección con inicializador, std :: not_fn, etc. Se prestará especial atención a los algoritmos estándar (mi tema favorito).

Pavel Belikov. Cómo funciona el análisis de flujo de datos en un analizador de código estático



Analysis Data Flow (flujos de datos) es una tecnología para analizar el código fuente de los programas que se utiliza ampliamente en diversas herramientas de desarrollo: compiladores, linters, IDE. Hablaremos de ello utilizando el desarrollo de un analizador estático como ejemplo. Considere la clasificación y los diversos tipos de análisis de flujo de datos, las tecnologías relacionadas que se complementan entre sí y los problemas que surgen durante su desarrollo, y las sorpresas que C ++ nos presenta cuando tratamos de analizarlo. Durante el informe, analizaremos varios errores encontrados en proyectos reales que utilizan esta tecnología.

Alexander Zaitsev. Herramientas de perfil de código C ++



Sucede que a veces su aplicación comienza a realizar tareas aparentemente mundanas durante mucho tiempo y consume una gran cantidad de RAM. Y usted, como desarrollador, no tiene idea de por qué sucede esto (pero le interesa). En el transcurso de la charla, hablaremos sobre herramientas que pueden comprender las razones del comportamiento extraño de nuestros programas. Si no le temes a palabras como Valgrind, gprof, gperftools y muchas otras, ¡bienvenido!

Alexander Chuprina. Configuración del entorno para la compilación cruzada basada en Docker



Cómo configurar / actualizar entornos de manera rápida y fácil para proyectos de compilación cruzada para varias plataformas (basadas en Docker), cómo cambiar rápidamente entre ellas, cómo organizar CI y pruebas (basadas en GitLab y Docker) utilizando estos ladrillos.

Alexey Tkachenko. Plataforma multiplataforma de generación de código C ++



El informe considerará la generación de código al compilar varias construcciones de lenguaje, tanto simples como complejas, en varias plataformas, tanto las comunes x86 / x64 como aquellas que reciben menos atención: ARM, AVR. También habrá ejemplos de procesadores completamente exóticos como PowerPC e incluso MicroBlaze. El énfasis principal se colocará no en el procesamiento de datos, sino en la comparación de varias construcciones de código con instrucciones de las plataformas de destino.

Vadim Vinnik. Procesamiento de colecciones: una esencia y muchas manifestaciones



La programación de hoy está dividida en muchas áreas que están bastante lejos unas de otras. ¿Existe una programación conceptualmente unificada y sobre qué descansa esta unidad? Las respuestas a estas y otras preguntas son proporcionadas por el autor del informe.

Nicolai Josuttis. Cuidado con C ++ 17



El diablo está en los detalles. Esto también se aplica a C ++ 17. Obtenemos nuevas características interesantes, pero también obtenemos cosas nuevas para cuidar y recordar. Esta charla discute algunas de las características interesantes cuando pueden dar lugar a sorpresas.
