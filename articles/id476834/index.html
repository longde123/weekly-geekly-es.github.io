Red Hat OpenShift 4.2: Alat Baru

Pada Oktober 2019, OpenShift 4.2 dirilis , yang intinya melanjutkan kursus menuju otomatisasi dan optimalisasi pekerjaan dengan cloud.



Ingatlah bahwa pada bulan Mei 2019 kami memperkenalkan Red Hat OpenShift 4, generasi berikutnya dari platform Kubernetes kami, yang kami desain ulang untuk menyederhanakan pengelolaan aplikasi kemas dalam lingkungan produksi.

Solusi ini dibuat sebagai platform pemerintahan sendiri dengan pembaruan otomatis perangkat lunak dan fungsi manajemen siklus hidup di cloud hybrid dan didasarkan pada Red Hat Enterprise Linux dan Red Hat Enterprise Linux CoreOS yang terbukti. Dalam versi 4.2, fokusnya adalah membuat platform lebih nyaman bagi pengembang. Selain itu, kami menyederhanakan pekerjaan mengelola platform dan aplikasi untuk administrator cluster dengan menawarkan cara migrasi dari OpenShift versi 3 ke 4, serta mengimplementasikan dukungan untuk konfigurasi tanpa akses ke Internet.

Di mana kecepatan di sini?


Versi 4.2 sangat menyederhanakan bekerja dengan Kubernetes, menawarkan mode konsol manajemen OpenShift baru yang dioptimalkan untuk tugas-tugas pengembangan, serta alat dan plugin baru untuk membangun wadah, mengatur pipa CI / CD dan menerapkan sistem tanpa server. Semua ini membantu programmer untuk lebih fokus pada tugas utama mereka - membuat kode aplikasi, tanpa terganggu oleh fitur Kubernetes.


Lihat topologi aplikasi di konsol pengembang.


Mode pengembang baru dari konsol OpenShift

Alat pengembang baru di OpenShift 4.2:

  • Mode pengembangan konsol web membantu pengembang fokus pada hal utama, hanya menampilkan informasi dan konfigurasi yang diperlukan. UI canggih untuk melihat topologi dan membangun aplikasi membuatnya mudah untuk membuat, menyebarkan, dan memvisualisasikan aplikasi dan sumber daya kluster kemas.
  • Toolkit odo adalah antarmuka baris perintah khusus untuk pengembang yang menyederhanakan pengembangan aplikasi pada platform OpenShift. Mengatur interaksi seperti git push, CLI ini membantu pengembang untuk membuat aplikasi pada platform OpenShift tanpa usaha ekstra, tanpa mempelajari seluk-beluk Kubernetes.
  • Konektor OpenShift Red Hat untuk Microsoft Visual Studio Code, JetBrains IDE (termasuk IntelliJ) dan Eclipse Desktop IDE menyediakan integrasi yang mudah dengan alat-alat yang digunakan dan memungkinkan Anda untuk mengembangkan, membangun, men-debug, dan menyebarkan aplikasi OpenShift di lingkungan pengembang yang sudah dikenal.
  • Redhat OpenShift Deployment Extension untuk Microsoft Azure DevOps . Memberikan pengguna toolkit DevOps ini dengan kemampuan untuk menyebarkan aplikasi mereka pada Azure Red Hat OpenShift atau dalam cluster OpenShift lainnya pada platform Microsoft Azure DevOps.

OpenShift penuh pada laptop


Red Hat CodeReady Containers , yang merupakan cluster OpenShift yang sudah jadi yang dioptimalkan untuk ditempatkan di workstation atau laptop, memberikan peluang untuk mengembangkan aplikasi berbasis cloud dalam mode lokal.

Service mesh Service Mesh


Solusi OpenShift Service Mesh kami , berdasarkan proyek sumber terbuka Istio, Kiali dan Jaeger dan operator Kubernetes khusus, menyederhanakan pengembangan, penyebaran, dan pemeliharaan aplikasi pada platform OpenShift, menyediakan alat yang diperlukan dan mengadopsi otomatisasi aplikasi cloud berdasarkan arsitektur modern seperti seperti layanan microser. Solusi ini memungkinkan para programmer untuk dibebaskan dari kebutuhan untuk secara mandiri menyebarkan dan memelihara layanan jaringan khusus yang diperlukan untuk aplikasi yang dibuat dan logika bisnis.

Red Hat OpenShift Service Mesh , tersedia untuk OpenShift 4 , secara harfiah dirancang untuk pengembang yang secara harfiah â€œdari dan keâ€ dan menawarkan fitur-fitur seperti pelacakan, metrik, visualisasi dan pemantauan komunikasi jaringan, serta menginstal dan mengonfigurasi mesh layanan dalam satu klik. Selain itu, solusi ini menawarkan keuntungan dari sudut pandang manajemen operasional dan keamanan, misalnya, enkripsi lalu lintas antara server dalam pusat data dan integrasi dengan gateway API Red Hat 3scale .

Komputasi tanpa server


Solusi kami yang lain, OpenShift Serverless , membantu Anda menyebarkan dan menjalankan aplikasi yang dengan mudah menyesuaikan permintaan atas dan ke bawah ke level nol. Solusi ini, dibangun berdasarkan proyek Knative dan tersedia dalam versi Technology Preview, dapat diaktifkan pada cluster OpenShift 4 menggunakan operator Kubernetes yang sesuai, yang memudahkan untuk memulai dan menginstal komponen yang diperlukan untuk menyebarkan aplikasi tanpa server atau fungsi di OpenShift. Mode pengembang konsol OpenShift, yang muncul dalam versi 4.2, memungkinkan Anda untuk menggunakan opsi tanpa server dalam proses pengembangan standar, seperti Impor dari Git atau Deployan Image, dengan kata lain, Anda dapat membuat aplikasi tanpa server langsung dari konsol.


Mengkonfigurasi penyebaran tanpa server di konsol OpenShift

Selain integrasi dengan konsol pengembang, ada peningkatan tanpa server lain dalam versi baru OpenShift. Secara khusus, ini adalah kn - antarmuka baris perintah Knative yang menyediakan operasi yang mudah dan intuitif, memungkinkan Anda untuk mengelompokkan objek yang diperlukan untuk aplikasi; mengambil snapshot dari kode dan konfigurasi, dan juga memungkinkan untuk memetakan titik akhir jaringan ke versi atau layanan tertentu. Semua fitur ini, tersedia dalam versi Technology Preview melalui operator OpenShift Serverless, membantu pengembang memulai dengan arsitektur tanpa server dan menyebarkan aplikasi mereka secara fleksibel dalam cloud hybrid tanpa terikat dengan infrastruktur tertentu.

CI / Konveyor Awan CD


Integrasi dan Pengiriman Berkelanjutan (CI / CD) adalah praktik pengembangan utama saat ini yang meningkatkan kecepatan dan keandalan penyebaran perangkat lunak. Toolkit CI / CD yang baik memungkinkan tim pengembangan untuk mengoptimalkan dan mengotomatiskan proses umpan balik, yang sangat penting untuk keberhasilan pengembangan tangkas. Di OpenShift, Anda dapat menggunakan Jenkins klasik atau solusi OpenShift Pipelines baru kami sebagai alat tersebut.

Jenkins adalah standar de facto saat ini, tetapi kami menghubungkan masa depan container CI / CD dengan proyek open source Tekton. Oleh karena itu, OpenShift Pipelines dibangun berdasarkan proyek ini dan lebih baik mendukung pendekatan berbasis cloud seperti pipeline-as-code ("pipeline as code") dan GitOps. Dalam OpenShift Pipelines, setiap langkah dilakukan dalam wadahnya sendiri, sehingga sumber daya hanya digunakan selama pelaksanaan langkah ini, yang memungkinkan pengembang untuk sepenuhnya mengontrol saluran pengiriman, plug-in dan kontrol akses mereka tanpa menggunakan server CI / CD pusat.

OpenShift Pipelines masih dalam tahap Pratinjau Pengembang dan tersedia sebagai operator yang sesuai, yang dapat digunakan di setiap cluster OpenShift 4. Jenkins dapat digunakan baik dalam OpenShift versi 3 dan 4.

Manajemen Kontainer Cloud Hibrid


Instalasi otomatis dan pembaruan OpenShift menjadikan cloud hybrid sedekat mungkin dengan cloud kanon dari sudut pandang pengalaman. OpenShift keempat sebelumnya tersedia untuk platform cloud publik utama, lingkungan cloud pribadi, platform virtualisasi, dan server bare-metal, tetapi versi 4.2 menambahkan dua platform cloud publik baru ke daftar ini - Microsoft Azure dan Google Cloud Platform, serta cloud OpenStack pribadi .

Pemasang OpenShift 4.2 telah diselesaikan untuk berbagai lingkungan target, serta dilatih untuk pertama kalinya untuk bekerja dengan konfigurasi terisolasi (tidak terhubung ke Internet). Instalasi mandiri dan mode proxy wajib dengan kemampuan untuk menyediakan rantai sertifikat khusus (bundel CA) membantu memastikan kepatuhan dengan standar peraturan dan protokol keamanan internal. Mode pemasangan senyap memungkinkan Anda untuk selalu memiliki versi terbaru dari OpenShift Container Platform di mana tidak ada akses ke Internet, atau di lingkungan dengan kebijakan pengujian gambar yang ketat.

Selain itu, menggunakan tumpukan OpenShift penuh menggunakan Red Hat Enterprise Linux CoreOS, versi ringan dari Red Hat Enterprise Linux, memungkinkan Anda untuk menyiapkan cloud Anda dalam waktu kurang dari satu jam setelah instalasi dimulai.

Red Hat OpenShift memungkinkan Anda menyatukan proses membuat, menyebarkan, dan mengelola aplikasi kontainer di cloud dan infrastruktur di lokasi. Dengan instalasi yang lebih sederhana, otomatis, dan lebih cepat, OpenShift 4.2 sekarang tersedia di platform AWS, Azure, OpenStack, dan GCP, memungkinkan organisasi untuk mengelola platform Kubernetes mereka secara efisien dalam cloud hybrid.

Migrasi mudah dari OpenShift 3 ke OpenShift 4


Alat migrasi beban kerja baru membuatnya mudah untuk ditingkatkan ke OpenShift 4.2 dari versi platform sebelumnya. Mentransfer beban dari cluster lama ke yang baru sekarang jauh lebih cepat, lebih mudah dan dengan minimum operasi manual. Cukup bagi administrator klaster untuk memilih sumber OpenShift 3.x klaster, tandai di atasnya proyek yang diperlukan (atau namespace) dan kemudian tentukan apa yang harus dilakukan dengan volume permanen yang sesuai - salin ke kluster atau transfer OpenShift 4.x target. Setelah ini, aplikasi terus berjalan di cluster sumber sampai administrator menyelesaikan pekerjaan mereka.

OpenShift 4.2 mendukung berbagai skenario migrasi:

  • Data disalin menggunakan repositori antara yang didasarkan pada proyek Velero. Opsi ini memungkinkan Anda untuk bermigrasi dengan perubahan dalam sistem penyimpanan ketika, misalnya, Gluster digunakan di kluster asli dan Ceph digunakan di kluster baru.
  • Data tetap dalam repositori saat ini, tetapi terhubung ke cluster baru (saklar volume permanen).
  • Menyalin sistem file melalui Restic.

Malam pertama benar


Seringkali, pengguna kami ingin dapat mencoba inovasi OpenShift yang direncanakan jauh sebelum rilis rilis baru. Oleh karena itu, dimulai dengan OpenShift 4.2, kami memberi pelanggan dan mitra akses ke bangunan awal malam . Harap dicatat bahwa majelis ini tidak dimaksudkan untuk digunakan dalam produksi, tidak dilengkapi dengan dukungan, tidak didokumentasikan dengan baik dan mungkin memiliki fungsi yang tidak lengkap. Kualitas majelis ini tumbuh ketika Anda mendekati versi final.

Rakitan bangunan malam memungkinkan pelanggan dan mitra untuk membiasakan diri dengan fitur-fitur baru pada tahap awal pengembangan, yang dapat berguna dalam hal perencanaan untuk penyebaran atau mengintegrasikan OpenShift dengan solusi pengembang ISV sendiri.
