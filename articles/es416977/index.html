ESCENARIOS


Lazurite le permite crear disparadores y scripts complejos con condiciones y bucles , es decir. el usuario puede "programar" el programa utilizando el constructor incorporado para cualquiera de sus necesidades.


Cualquier secuencia de comandos tiene su propio interruptor en la interfaz de usuario: "mosaico". Dependiendo del tipo de datos de script, el interruptor se verá diferente.

Además, hay tres tipos de scripts en Lazurite: acción simple, script complejo y script remoto.


Ventana para seleccionar el tipo de script que se creará.


Acción individual


Las acciones individuales son escenarios que no requieren una lógica compleja, como encender la luz o cambiar el nivel de sonido utilizando el "mosaico" en el menú del interruptor. En tal escenario, no puede crear un bucle o condición; en él solo puede vincular una acción separada del complemento (sobre ellos a continuación) al conmutador.

Script remoto


Lazurite tiene un servidor incorporado en el que se ejecuta el servicio WCF. Esto es lo que permite que las aplicaciones cliente reciban información sobre los scripts y realicen algunas acciones. Pero, además, otro servidor Lazurite puede conectarse al servidor Lazurite y usar scripts de servidor remoto. Para usar esta funcionalidad, debe crear un script remoto .


La compatibilidad con scripts remotos le permite crear una red de dispositivos con Lazurite instalado. Es decir en los scripts de un servidor, puede llamar scripts u obtener valores de script de otro servidor.


Por ejemplo, en una casa grande, puede crear un sistema distribuido en el que varios nodos con Lazurite instalado se ubicarán en diferentes pisos (habitaciones, edificios), con los cuales la unidad central de Lazurite se comunicará. El administrador configurará el sistema y decidirá qué tan independientes serán los dispositivos esclavos, si se utilizarán como intérpretes simples o si se les incorporará cierta autonomía.



DISPARADORES


Los disparadores son ciertas acciones que se activan cuando se producen condiciones especiales. Se diferencian del escenario en que no tienen un tipo de datos y no tienen un "mosaico". El activador para activar un activador es cambiar el escenario seleccionado, es decir Si queremos realizar alguna acción cuando la temperatura cambia, la humedad cambia, la apariencia del usuario en el hogar, etc., entonces necesitaremos crear un disparador para el escenario correspondiente. El constructor del disparador es una copia del constructor del script, donde puede crear bucles, condiciones, acciones.


Disparador para activar un sensor de movimiento.


ACCION


Cualquier línea agregada en el constructor del script es una "acción". Cada complemento instalado en Lazurite agrega algunas características nuevas al programa: acciones. Cualquier trabajo con Z-Wave, Modbus, geolocalización, etc. producido a través de esta funcionalidad. En consecuencia, cualquier acción, como un script, tiene su propio tipo de datos .

SEGURIDAD


En Lazurite, puede configurar el acceso para ver y cambiar el valor del script. Esto puede ser necesario cuando creamos un usuario en el sistema para nuestro invitado o cuando solo queremos ocultar un script que realiza funciones auxiliares y no debe mostrarse a los usuarios.


Ventana de configuración de seguridad de script.


Los siguientes elementos están disponibles para configurar la seguridad:


  • "Negar para cualquier uso";
  • "Denegar para todos los grupos excepto ...": prohibición del uso del script para usuarios que no son miembros de los grupos seleccionados;
  • "Denegar para todos los usuarios excepto ...": prohibición de utilizar el script para todos los usuarios excepto los seleccionados;
  • "Denegar para grupos": prohibición de ejecución para grupos de usuarios seleccionados;
  • "Denegar la ejecución desde la red": los scripts con esta restricción no se pueden mostrar en las aplicaciones cliente y no se pueden usar como un "script remoto";
  • "Denegar para el menú de inicio rápido": el script no se mostrará en el menú de inicio rápido;
  • "Denegar para usuarios": prohibición de ejecución para usuarios seleccionados;
  • "Denegar el uso de otro script / disparador": el script no se puede ejecutar en otro script o disparador.

Por defecto, cualquier script está prohibido para comenzar desde la red y para el menú de inicio rápido.


USUARIOS Y GRUPOS


Los usuarios de Lazurite son una parte integral del sistema. Sin una cuenta, los usuarios no podrán acceder a los scripts necesarios y administrar dispositivos en la casa. Si no hubiera autenticación, cualquier persona se conectaría al sistema y ejecutaría cualquier script, lo cual es extremadamente indeseable.


Para mayor comodidad, las configuraciones de seguridad de los scripts de usuario se pueden agrupar, por ejemplo, puede crear un grupo "Invitados" y evitar que este grupo ejecute ciertos scripts.

Onda Z


Z-Wave es un protocolo de comunicación inalámbrica utilizado en domótica. Para implementar el protocolo, se utilizan módulos de RF en miniatura de baja potencia. Ahora en el mercado de ZWave hay una lista bastante extensa de dispositivos, como bombillas, enchufes, módulos de relés, interruptores, cerraduras, sensores de temperatura (humedad, movimiento, fugas), controladores de temperatura, dispositivos de control de aire acondicionado, etc. Los multisensores también son comunes: dispositivos que combinan varios sensores (por ejemplo, un sensor de luz, presencia, temperatura, sensor de puerta, etc.), lo cual es muy atractivo desde el punto de vista de los costos financieros y la ubicación en la habitación.


En Lazurite, el módulo ZWave le permite realizar todas las operaciones estándar del dispositivo, como eliminar, agregar dispositivos y restablecer el controlador. También puede usar varios controladores ZWave, el usuario final no verá la diferencia. Por ejemplo, puede tomar lecturas de temperatura de un sensor que está conectado al controlador "A" y encender el aire acondicionado usando el controlador "B". No hay restricciones, cualquier acción y verificación se llevan a cabo de manera estándar y se agregan en el diseñador de scripts .


El nodo se selecciona en la imagen, el multisensor es ZW100, luego se selecciona su parámetro de usuario, con el que puede medir la temperatura en la habitación .


Después de crear una secuencia de comandos con este nodo, aparecerá un mosaico con información de temperatura.

GEOLOCACIÓN DEL USUARIO


Lazurite tiene la funcionalidad de un rastreador GPS. Cualquier aplicación cliente envía información de geodatos al servidor Lazurite, donde estos datos se pueden procesar, guardar y mostrar. Las acciones presentadas en el complemento UserGeolocationPlugin están diseñadas para trabajar con la geolocalización de los usuarios y le permiten calcular valores como la distancia del usuario a un lugar, la ubicación del usuario en un lugar específico, etc.


Algunas acciones requieren la presencia no solo del usuario, sino también de un lugar o una "ubicación", que debe configurarse y configurarse en el programa. Tal lugar puede ser una casa, trabajo, ciudad, estación de metro. Se necesitan acciones con este parámetro para realizar cualquier acción sobre la apariencia o la distancia del usuario desde esta "ubicación". Un lugar tiene tres parámetros: coordenadas centrales, radio y nombre.
