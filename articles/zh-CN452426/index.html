使用GitHub凭据登录到Azure DevOps

在Microsoft，我们专注于这一想法，它为开发人员提供了快速创建高质量应用程序的机会。 实现此目的的一种方法是提供涵盖软件开发生命周期各个阶段的一系列产品和服务。 这包括IDE和DevOps工具，基于云的应用程序和数据平台，以及操作系统，人工智能，物联网解决方案等等。 所有这些人都集中在开发人员，团队和组织中的个人以及开发人员社区的成员中。

GitHub是最大的开发人员社区之一，对于全球数百万开发人员而言，在GitHub上的身份识别已成为其数字生活的重要方面。 认识到这一点，我们很高兴地宣布改进，这些改进将帮助GitHub用户开始使用我们的开发人员服务，包括Azure DevOps和Azure。


您的GitHub凭据现在可以用于访问Microsoft服务。


现在，我们为开发人员提供了在任何Microsoft登录页面上使用其现有GitHub帐户登录Microsoft在线服务的机会。 现在，使用GitHub凭据，您可以通过OAuth登录到任何Microsoft服务，包括Azure DevOps和Azure。

您将看到通过单击“使用GitHub登录”登录帐户的机会。

通过GitHub登录并授权Microsoft应用程序后，您将收到一个与GitHub凭据关联的新Microsoft帐户。 在此过程中，您也可以选择将其链接到现有的Microsoft帐户（如果已有的话）。

登录到Azure DevOps


Azure DevOps为开发人员提供了一组服务，以帮助他们计划，创建和提交任何应用程序。 借助GitHub身份验证支持，我们能够简化与Azure DevOps服务的工作，例如持续集成和持续交付（Azure管道）； 敏捷计划（Azure委员会）； 和私有包的存储，例如用于NuGet，npm，PyPi等的模块（Azure Artifacts）。 Azure DevOps免费供个人和最多五人的小组使用。

要使用您的GitHub帐户开始使用Azure DevOps，请在Azure DevOps页面上单击“使用GitHub免费开始”。



登录过程完成后，将直接带您到Azure DevOps中访问的最后一个组织。 如果您不熟悉Azure DevOps，将被带到为您创建的新组织中。

访问所有Microsoft在线服务


除了访问开发人员服务（例如Azure DevOps和Azure）之外，您的GitHub帐户还可用于访问从Excel Online到Xbox的所有Microsoft在线服务。

使用这些服务进行身份验证时，您可以在单击“登录选项”后选择您的GitHub帐户。



我们对您的隐私的承诺。


首次使用GitHub帐户访问Microsoft服务时，GitHub会要求您使用个人资料信息的权限。

如果您同意，GitHub将提供您的GitHub帐户的电子邮件地址（公共和私有），以及个人资料信息，例如您的姓名。 我们将使用此数据来确定您在我们的系统中是否有一个帐户，如果没有，则需要创建一个新帐户。 将您的GitHub ID连接到Microsoft不会使Microsoft可以访问您在GitHub上的存储库。 如果诸如Azure DevOps或Visual Studio之类的应用程序需要使用您的代码，则它们将分别请求访问您的存储库，而您将需要分别征得他们的同意。

尽管您的GitHub帐户用于登录到您的Microsoft帐户，但它们仍然保持独立-一个简单地使用另一个作为登录方法。 您对GitHub帐户所做的更改（例如，更改密码或启用两因素身份验证）不会更改您的Microsoft帐户，反之亦然。 您可以在“安全性”选项卡上的帐户管理页面上管理 GitHub和Microsoft身份之间的连接。

立即开始学习Azure DevOps


转到“ Azure DevOps”页面，然后单击“使用GitHub免费启动”以启动。
