Elasticsearch yang sangat mudah diakses dan dapat diukur di Kubernetes


Dalam posting sebelumnya, kami meningkatkan set replika MongoDB dan memperkenalkan StatefulSet. Kami sekarang akan mengambil orkestrasi dari cluster ketersediaan tinggi Elasticsearch (dengan node master lainnya, node data, dan node klien) dan menggunakan ES-HQ dan Kibana.


Anda akan membutuhkan:


  1. Pemahaman dasar tentang Elasticsearch, jenis-jenis simpulnya, dan perannya.
  2. Cluster Kubernetes yang berfungsi dengan setidaknya tiga node (setidaknya empat core, 4 GB).
  3. Kemampuan untuk bekerja dengan Kibana.

Arsitektur Penerapan



  • Simpul data Elasticsearch digunakan sebagai StatefulSet dengan layanan tanpa kepala sehingga kami memiliki pengidentifikasi jaringan yang stabil .
  • Masternode Elasticsearch dikerahkan sebagai ReplicaSet dengan layanan tanpa kepala. Ini untuk penemuan otomatis .
  • Pod pada node klien Elasticsearch dikerahkan sebagai ReplicaSet dengan layanan internal sehingga Anda dapat mengirim permintaan baca / tulis ke node data.
  • Pod Kibana dan ElasticHQ digunakan sebagai ReplicaSet dengan layanan yang tersedia di luar cluster Kubernetes , tetapi terletak di dalam subnet (mereka tidak terbuka ke luar kecuali diperlukan).
  • HPA (Horizonal Pod Autoscaler) digunakan untuk node klien dan bertanggung jawab untuk penskalaan horizontal pada beban tinggi.
    "Ingat untuk mengonfigurasi lingkungan:
    1. Variabel ES_JAVA_OPTS .
    2. Variabel CLUSTER_NAME .
    3. Variabel NUMBER_OF_MASTERS untuk penyebaran master untuk menghindari situasi otak terpecah. Jika kita memiliki 3 tuan, sebutkan 2.
    4. Aturan anti-afinitas untuk perapian serupa untuk memastikan keandalan yang tinggi jika simpul kerja jatuh.
      "

Scaling


Kami dapat menggunakan layanan autoscaling untuk node klien tergantung pada batas CPU. Contoh HPA untuk simpul klien: 


 apiVersion: autoscaling/v1 kind: HorizontalPodAutoscaler metadata: name: es-client namespace: elasticsearch spec: maxReplicas: 5 minReplicas: 2 scaleTargetRef: apiVersion: extensions/v1beta1 kind: Deployment name: es-client targetCPUUtilizationPercentage: 80

Penskalaan otomatis menambahkan pod pada node klien ke cluster, dan ini dapat dilihat pada log dari pod apa saja pada node master.


Untuk pod pada node data , Anda hanya perlu menambah jumlah replika di panel kontrol Kubernetes atau di konsol GKE. Node data yang dibuat itu sendiri akan ditambahkan ke cluster dan akan mulai mereplikasi data dari node lain.


Saya tidak perlu melakukan penskalaan otomatis pada node master - mereka hanya menyimpan data tentang status cluster. Tetapi jika Anda akan menambahkan data node, pastikan bahwa jumlah master node dalam cluster adalah ganjil , dan jangan lupa untuk mengubah variabel NUMBER_OF_MASTERS untuk lingkungan.
