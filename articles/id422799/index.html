Bagaimana Microsoft menyembunyikan seluruh server dan bagaimana menemukannya

- Lihat gopher SSAS?
- Tidak ...
- Dan saya tidak melihat. Tapi dia!


(Ditembak dari film "DMB")

Dukungan untuk model data tabular (tabular) yang relatif baru, yang bertentangan dengan OLAP multidimensi yang sudah mapan, dibangun ke dalam sejumlah produk Microsoft. Dimulai dengan SQL Server (SS) dan berakhir dengan Excel. Dan jika semuanya jelas dengan SS, itu berisi produk SQL Server Analysis Services (SSAS) yang terpisah. Tetapi bagaimana dengan dukungan untuk bahasa permintaan DAX dan fungsi tabular lainnya di Power BI, SharePoint, atau Excel? Mari kita mencari seorang gopher menggunakan contoh Power BI Desktop .

Saya pikir banyak orang memperhatikan bahwa ketika menginstal Power BI Desktop di komputer, struktur folder Microsoft SQL Analysis Services (SSAS) meluas secara paralel. Ini adalah komponen server opsional yang merupakan bagian dari edisi yang lebih tinggi dari SQL Server dari Microsoft (khususnya, edisi Enterprise dan Business Intelligence). Dia bertanggung jawab (seperti yang Anda tebak dari namanya) untuk fungsi analitik dan, khususnya, pembuatan dan akses ke kubus OLAP, dan, mulai dari versi 2012, dan untuk dukungan penuh dari ideologi baru gudang data multidimensi - tabular (atau tabular) model data. "Binatang" ini diciptakan untuk mendukung penyimpanan data massal (tidak seperti penyimpanan string standar untuk OLTP) dan bekerja dengan seluruh data yang diatur dalam memori akses acak (dalam memori). Lebih detail di sini .


Struktur folder kerja Power BI Desktop (Instalasi berdiri sendiri)

Dimasukkannya komponen server dalam perangkat lunak desktop terlihat cukup logis dari sudut pandang pengembangan, dan yang paling penting, dukungan lebih lanjut untuk keseragaman model data tabular, umum untuk produk server unggulan, dan untuk lingkungan pengembangan desktop dan bahkan untuk Excel. Secara khusus, Power BI Desktop hanya mengintegrasikan inti server SSAS, yang bertanggung jawab untuk mendukung model yang sangat tabular ini dan permintaan data DAX. Ketika Anda memulai aplikasi Power BI Desktop, mesin xVelocity (vertiPaq), yang dibangun ke dalam server SSAS, dimulai, yang melakukan semua pekerjaan kasar untuk melayani permintaan DAX, penyeimbangan beban, manajemen memori dan mentransfer hasilnya ke renderer yang sudah menampilkan hasil dalam grafik, Histogram, PivotTables (matriks), dan elemen visual lainnya yang tersedia di Power BI.



Apa yang diberikan pengetahuan tentang dapur bagian dalam ini kepada kita? Dan fakta bahwa struktur modular seperti itu memungkinkan Anda untuk menggunakan Power BI Desktop sebagai server analitik uji tanpa perlu menggunakan lingkungan server SQL yang lengkap dengan Layanan Analisis Microsoft. Dan tanpa beban lisensi yang sesuai (mis. Gratis). Tentu saja, pendekatan ini tidak dapat digunakan dalam solusi industri. Namun, untuk penyebaran cepat dari infrastruktur pengujian dan verifikasi model data yang dibangun, tidak ada yang lebih baik dari yang Anda bayangkan.

Home Master Tips


Untuk menentukan nomor port aktual tempat Layanan Analisis "pocket" lokal diluncurkan, cara termudah adalah dengan menggunakan alat gratis yang luar biasa, "pisau Swiss" untuk setiap peretas DAX Studio DAX .
Saat menghubungkan dari DAX Studio ke model data Anda (file PBIX yang diperlukan harus dibuka di Power BI Desktop), pengaturan koneksi (seperti yang ditunjukkan pada gambar di bawah) yang dapat Anda gunakan untuk mengakses model data ini dari data lain akan ditampilkan di bilah status aplikasi aplikasi, seperti dari Excel atau instance lain dari Power Bi Desktop, secara lokal atau melalui jaringan.



Nomor port akan berubah setiap kali Anda memulai PBI Desktop. Dalam contoh saya, ini adalah 52308. Dalam kasus Anda, kemungkinan besar akan ada beberapa nomor lain dari kumpulan port Power BI Desktop.

Untuk menyambung kembali ke model data ini, Anda akan membutuhkan tidak hanya nomor port yang ditugaskan secara acak, tetapi juga nama database yang dihasilkan secara acak dalam bentuk GUID. Nomor port diperoleh sesuai dengan metode yang dijelaskan di atas, dan pengidentifikasi dari database yang diperlukan (GUID model data) dapat diperoleh dengan membuat kueri SQL berikut di DAX Studio: 

select * from $SYSTEM.DBSCHEMA_CATALOGS

Dalam kolom 
  CATALOG_NAME dan DATABASE_ID

akan menjadi nama GUID yang diperlukan.

Solusi lokal


File PBIX yang dikembangkan oleh saya tersedia untuk diunduh melalui tautan (123kB), yang membentuk daftar semua model data yang dimuat ke dalam memori (termasuk yang ada di dalam file ini). Artinya, daftar akan memiliki banyak baris karena ada file PBIX berjalan.
Pembaruan: Versi ini telah ditingkatkan untuk kompatibilitas dengan rilis Agustus Desktop PBI.

Penafian : File ini hanya dapat digunakan untuk tujuan demonstrasi dan pengujian, disediakan "sebagaimana adanya", dengan semua bug, kesalahan, dan lainnya, dibentuk 90% melalui antarmuka Power Query / PowerPivot dan tidak dapat menjadi contoh yang baik untuk mempelajari pemrograman Power BI .


Antarmuka program


Untuk mengonfigurasi, setelah mengklik tombol [Ubah Permintaan] , ubah parameternya 
  USER_NAME
dan mungkin 
  WORK_FOLDER
.


Home Master Tips


Power BI Desktop dapat digunakan pada komputer lokal dalam dua mode berbeda:

  1. Aplikasi yang berdiri sendiri
  2. Aplikasi Windows

Yang pertama diunduh dari situs https://powerbi.com dan diinstal dalam beberapa klik sebagai aplikasi yang paling umum. Yang kedua diinstal melalui Microsoft Store untuk mencari " Power BI "

Kode biner aplikasi dalam kedua versi adalah sama hanya dalam kasus pertama, Anda harus secara independen memantau rilis versi baru aplikasi, rilis yang Microsoft bakes seperti pancake. Frekuensi reguler pembaruan Power BI Desktop adalah sebulan sekali, Layanan Pelaporan Power BI rata-rata sekali dalam satu kuartal.

Instalasi melalui Microsoft Store akan menyelamatkan Anda dari memantau pembaruan versi perangkat lunak, tetapi ada kemungkinan kecil (tetapi tidak nol) bahwa dalam versi yang diperbarui Anda secara tak terduga akan menghadapi masalah yang tidak diamati pada rilis sebelumnya.

Secara pribadi, untuk berjaga-jaga, saya menyimpan kedua edisi, secara teratur mengisi kembali koleksi pribadi saya dengan rilis Desktop PBI bulanan untuk dapat memutar kembali ke yang sebelumnya.

Fitur lain dari kedua mode ini adalah masing-masing dari mereka memiliki struktur sendiri folder kerja. Untuk mengkonfigurasi file di atas, ini penting (lihat gambar).
