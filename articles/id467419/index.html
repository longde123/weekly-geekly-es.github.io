Rilis Final EE 8 Jakarta Dirilis

Eclipse Foundation telah merilis spesifikasi baru untuk platform & profil web Jakarta EE 8.
Mereka juga memperkenalkan server aplikasi EE Jakarta dan test suite (TCK) untuk menguji implementasi yang kompatibel lainnya.


Pada 10 September 2019, Jakarta EE 8 dirilis.

Spesifikasi EE Jakarta tersedia di situs web mereka .

Mulai dari Anotasi Jakarta ke Halaman Server Jakarta.

Bersamaan dengan spesifikasi Eclipse, Eclipse Glassfish Server 5.1, sebuah server aplikasi open source yang mengimplementasikan platform Jakarta EE 8, telah dirilis.

Pengembangan server Jakarta EE 8 dari berbagai penyedia server Java diharapkan. Jadi, server IBM Open Liberty telah disertifikasi sebagai Jakarta EE 8-compatible.

Eclipse setuju untuk mengambil alih pengembangan pada 2017 . Alih-alih memperkenalkan Jakarta EE 9 yang lebih baru, Eclipse bertujuan untuk migrasi yang lebih lancar, secara bertahap menggantikan Java EE dengan Jakarta EE.

Pengembangan EE Jakarta


EE Jakarta akan digunakan (menurut Eclipse) untuk membangun solusi modern seperti layanan cloud dan layanan Microsoft.

Fitur masa depan untuk memperbarui EE Jakarta termasuk pembaruan Java SE yang mendasari EE Jakarta.

Versi saat ini adalah Java SE 8, kandidat berikutnya adalah Java SE 11 (ini adalah build berikutnya setelah SE 8 dengan siklus dukungan yang panjang).

Juga, bekerja dengan teknologi cloud seperti Kubernetes, Docker akan ditingkatkan.

Spesifikasi terpisah juga diharapkan ditingkatkan, seperti Layanan Web Jakarta RESTfull.

Selain itu, Eclipse Foundation harus mendefinisikan proses untuk mengubah ruang nama javax ke jakarta. Ini mungkin memerlukan kompilasi ulang program, meskipun Eclipse bekerja pada kompatibilitas ke belakang untuk mengurangi tingkat keparahan migrasi.

Namespace javax telah kompatibel mundur selama 20 tahun, tetapi dimiliki oleh Oracle .
