Cadangan, Bagian 5: Menguji Bacula dan Cadangan Veeam untuk Linux


Artikel ini akan membahas berbagai alat perangkat lunak cadangan "besar", termasuk yang komersial. Daftar kandidat: Veeam Agent untuk Linux, Bacula.


Ini akan memeriksa pekerjaan dengan sistem file, sehingga lebih mudah untuk dibandingkan dengan kandidat sebelumnya.


Hasil yang Diharapkan


Karena kedua kandidat adalah solusi siap pakai yang universal, hasil yang paling penting adalah prediktabilitas kerja, yaitu, waktu kerja yang sama ketika memproses jumlah data yang sama, serta beban yang sama.


Gambaran Umum Veeam Agent untuk Linux


Program cadangan ini bekerja dengan perangkat blok, yang memiliki modul untuk kernel Linux, yang memastikan integritas cadangan dengan melacak blok data yang diubah. Penjelasan lebih rinci ada di sini .


Proses membuat cadangan file bekerja berdasarkan modul kernel yang sama: snapshot perangkat blok dibuat, yang dipasang di direktori sementara, setelah data disinkronkan file dengan file dari nugget ke direktori lokal lain, atau dihapus melalui protokol smb atau nfs, di mana beberapa file dibuat di format berpemilik.


Proses membuat cadangan file tidak berhasil. Sekitar 15-16% dari kecepatan eksekusi turun menjadi 600 kb / s dan lebih rendah, dengan penggunaan cpu 50%, yang berpotensi menyebabkan 6-7 jam proses pencadangan, sehingga prosesnya terhenti.


Aplikasi untuk dukungan teknis Veeam dibuat, staf yang diusulkan menggunakan mode blok sebagai solusi.


Hasil dari mode cadangan blok-demi-blok adalah sebagai berikut:


Runtime program dalam mode ini adalah 6 menit per 20 GB data.


Secara keseluruhan, tayangan yang cukup bagus dari program bekerja, tetapi dalam tinjauan umum itu tidak akan diperhitungkan karena lambatnya mode operasi file.


Tinjau Bacula


Bacula - perangkat lunak client-server untuk membuat cadangan, secara logis terdiri dari beberapa bagian, yang masing-masing melakukan bagian kerjanya sendiri. Ada Direktur, yang digunakan untuk manajemen, FileDaemon adalah layanan yang bertanggung jawab untuk cadangan, StorageDaemon adalah layanan penyimpanan cadangan, Konsol adalah antarmuka ke Direktur (ada TUI, GUI, opsi Web). Kompleks ini juga termasuk dalam ulasan karena, meskipun memiliki ambang masuk yang sangat besar, ini adalah cara yang cukup populer untuk mengatur cadangan.


Dalam mode cadangan penuh

Dalam mode ini, Bacula menunjukkan dirinya cukup dapat diprediksi, melakukan cadangan dalam rata-rata 10 menit,
Profil beban adalah sebagai berikut:



Ukuran cadangan sekitar 30 GB, seperti yang diharapkan saat bekerja dalam mode operasi ini.


Saat membuat cadangan tambahan, hasilnya tidak jauh berbeda, kecuali untuk ukuran repositori, tentu saja (sekitar 14 GB).


Secara umum, kami melihat beban yang seragam pada satu inti prosesor, serta fakta bahwa indikatornya mirip dengan tar biasa dengan kompresi yang diaktifkan. Karena kenyataan bahwa pengaturan bacula untuk cadangan sangat, sangat luas, tidak berfungsi untuk menunjukkan keuntungan yang jelas.


Hasil


Secara umum, situasi ini tidak menguntungkan bagi kedua kandidat, kemungkinan besar karena fakta bahwa mode file untuk membuat salinan cadangan digunakan. Pada bagian selanjutnya, kami juga akan mempertimbangkan proses pemulihan dari cadangan, kesimpulan umum dapat dibuat dengan total waktu.
