Panduan untuk menyiapkan server BIND DNS di lingkungan chroot untuk Red Hat (RHEL / CentOS) 7

Terjemahan artikel disiapkan untuk siswa kursus Keamanan Linux . Apakah menarik untuk dikembangkan ke arah ini? Tonton rekaman siaran kelas master oleh Ivan Piskunov "Keamanan di Linux dibandingkan dengan Windows dan MacOS"



Pada artikel ini, saya akan menjelaskan langkah-langkah untuk menyiapkan server DNS pada RHEL 7 atau CentOS 7. Untuk demonstrasi, saya menggunakan Red Hat Enterprise Linux 7.4. Tujuan kami adalah untuk membuat satu catatan A dan satu catatan PTR untuk zona pencarian maju dan mundur.


Pertama instal paket rpm yang diperlukan untuk server DNS.


CATATAN: Untuk RHEL, Anda harus memiliki langganan RHN aktif , atau Anda dapat mengonfigurasi repositori mandiri lokal di mana pengelola paket yum dapat menginstal paket dan dependensi rpm yang diperlukan. 


# yum install bind bind-chroot caching-nameserver

Pengaturan saya: 


 # hostname golinuxhub-client.example  IP- 192.168.1.7 # ip address | egrep 'inet.*enp0s3' inet 192.168.1.7/24 brd 192.168.1.255 scope global dynamic enp0s3

Karena kita akan menggunakan chroot, kita perlu menonaktifkan layanan. 


 # systemctl stop named # systemctl disable named

Kemudian salin file yang diperlukan ke direktori chroot.
CATATAN Gunakan argumen -p ke perintah cp untuk mempertahankan kepemilikan dan kepemilikan.

Kosongkan isi named.conf dan rekatkan yang berikut ini. 


 [root@golinuxhub-client etc]# vim named.conf options { listen-on port 53 { 127.0.0.1; any; }; # listen-on-v6 port 53 { ::1; }; directory "/var/named"; dump-file "/var/named/data/cache_dump.db"; statistics-file "/var/named/data/named_stats.txt"; memstatistics-file "/var/named/data/named_mem_stats.txt"; allow-query { localhost; any; }; allow-query-cache { localhost; any; }; }; logging { channel default_debug { file "data/named.run"; severity dynamic; }; }; view my_resolver { match-clients { localhost; any; }; recursion yes; include "/etc/named.rfc1912.zones"; };

Informasi yang berkaitan dengan zona tersebut harus ditambahkan ke /var/named/chroot/etc/named.rfc1912.zones . Tambahkan entri di bawah ini. File example.zone adalah file lookup maju, dan example.rzone adalah file reverse zone.


CATATAN PENTING: Zona pencarian terbalik berisi 1.168.192, karena alamat IP saya adalah 192.168.1.7 


 zone "example" IN { type master; file "example.zone"; allow-update { none; }; }; zone "1.168.192.in-addr.arpa" IN { type master; file "example.rzone"; allow-update { none; }; };

File yang terkait dengan zona ada di sini: 


 # cd /var/named/chroot/var/named/

Selanjutnya, buat file untuk zona maju dan mundur. Nama file akan sama seperti di atas dalam file named.rfc1912.zones . Kami sudah memiliki beberapa templat default yang dapat kami gunakan. 


 # cp -p named.localhost example.zone # cp -p named.loopback example.rzone

Tambahkan konten berikut ke file zona maju. Di sini kita membuat catatan A untuk localhost (golinuxhub-client) dan satu lagi untuk server (golinuxhub-server). 


 # vim example.zone $TTL 1D @ IN SOA example. root ( 1 ; serial 3H ; refresh 15M ; retry 1W ; expire 1D ) ; minimum IN NS example. IN A 192.168.1.7 golinuxhub-server IN A 192.168.1.5 golinuxhub-client IN A 192.169.1.7

Selanjutnya, tambahkan konten ke file zona terbalik. Di sini kita membuat catatan PTR untuk golinuxhub-client dan untuk golinuxhub-server. 


 # vim example.rzone $TTL 1D @ IN SOA example. root.example. ( 1997022700 ; serial 28800 ; refresh 14400 ; retry 3600000 ; expire 86400 ) ; minimum IN NS example. 5 IN PTR golinuxhub-server.example. 7 IN PTR golinuxhub-client.example.

Sebelum kita memulai layanan bernama-chroot , periksa konfigurasi file zona. 


 [root@golinuxhub-client named]# named-checkzone golinuxhub-client.example example.zone zone golinuxhub-client.example/IN: loaded serial 1 OK [root@golinuxhub-client named]# named-checkzone golinuxhub-client.example example.rzone zone golinuxhub-client.example/IN: loaded serial 1997022700 OK

Semuanya terlihat bagus. Sekarang periksa file konfigurasi menggunakan perintah berikut. 


 [root@golinuxhub-client named]# named-checkconf -t /var/named/chroot/ /etc/named.conf

Jadi, semuanya berhasil. 


 [root@golinuxhub-client named]# echo $? 0
