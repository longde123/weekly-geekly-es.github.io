Amazon Echo Dot vs. Google Home Mini - asisten mana yang lebih baik?

Antarmuka suara untuk berinteraksi dengan perangkat Internet of Things (IoT) dengan cepat menyerang kehidupan sehari-hari. Jika sebelumnya mekanik, sakelar sentuh dan tombol cukup cocok sebagai alat komunikasi dengan benda-benda teknis, sekarang pengguna yang cerewet akan senang mengatakan ungkapan sederhana: "Alexa, nyalakan lampu ruang tamu." untuk menyalakan lampu, dll. Namun, sekali untuk ini hanya perlu "bertepuk tangan." Sekarang, sebaliknya, kecerdasan buatan asisten suara membuat upaya yang cukup percaya diri untuk memahami permintaan suara yang dibuat secara sewenang-wenang. Di posting terakhir, Apa yang Kita Ketahui Tentang Amazon Alexa? Atau kesan pertama Amazon Echo Dot ", infrastruktur asisten suara Amazon Alexa ditinjau menggunakan Echo Dot sebagai contoh. Jadi sekarang saatnya untuk mempertimbangkan pesaing di ceruk harga yang sama - Google Home Mini.


Apakah Amazon Echo Dot dan Google Home Mini mirip satu sama lain, atau mereka perangkat yang sama sekali berbeda?

Google adalah salah satu yang pertama menggunakan kemampuan antarmuka bicara dalam desainnya, misalnya, pada perangkat Android. Dan bukan kebetulan bahwa solusi perangkat keras dari raksasa pencarian segera melampaui Alexa. Semuanya diketahui dengan perbandingan, jadi dalam publikasi ini Anda tidak dapat melakukannya tanpa analogi. Perlu dicatat bahwa saat ini pasar untuk asisten suara cukup besar, misalnya, Google Now, Microsoft Cortana, Siri, Alice, dll. Tetapi ini adalah implementasi kecerdasan di awan, dan asisten suara pribadi adalah perangkat fisik yang tentunya harus menjadi bagian dari rumah pintar, kantor, mobil, dll. Namun, para pengembang sistem ini berpikir demikian. Pengguna awam sekarang, sebaliknya, menghargai potensi perangkat semacam itu dan prospek keamanan dan privasi mereka sendiri. Meskipun hampir semua orang menggunakan smartphone, dan kebanyakan dari kita tidak menyegel webcam laptop, yang baru selalu terlihat tidak dijelajahi dan sedikit mengintimidasi.

Jika kita berbicara tentang pengeras suara cerdas dengan antarmuka suara, sekarang harganya sekitar $ 100 USD dan lebih dari $ 300 USD, bukan karena pilihan asisten, tetapi karena pemilihan akustik yang memenuhi permintaan pengguna yang serba guna. Selain itu, jika Anda tahu apa yang diharapkan dari solusi perangkat keras asisten suara itu sendiri, maka akan mungkin untuk menjawab pertanyaan mendasar tentang kelayakan penambahan tambahan untuk sistem speaker di rumah atau kantor yang cerdas, dll. Karena pilihan solusi awal cukup sempit, sekarang semuanya tergantung pada membandingkan Amazon Echo Dot dengan Google Home Mini, setidaknya dua solusi ini berharga hingga $ 50 USD dan memungkinkan Anda untuk mengembangkan kemampuan Anda dengan akustik yang lebih solid, bekerja bersama dengan perangkat lain berdasarkan berbagai solusi untuk asisten suara perusahaan dan, tidak diragukan lagi, hampir seluruh jajaran sistem modern untuk rumah pintar.

Pada saat yang sama, menjadi jelas bahwa masalah memastikan keamanan siber bukan lagi ungkapan kosong, terutama dalam hal banjir di tingkat kehidupan sehari-hari dengan perangkat IoT. Tidak diragukan lagi, produsen dalam segala hal prihatin dengan privasi pengguna perangkat mereka dan keamanan informasi, tetapi proyek-proyek seperti IoT Inspektur Universitas Princeton selalu menarik.

Secara tradisional, kita belajar tentang alat perekam: $ arecord -l dan playback: $ aplay -l. Sebagai tanggapan, kami tertarik pada nomor kartu dan perangkat yang sesuai.


Perangkat Audio di Raspberry Pi 3, Menjalankan Raspbian

Setelah menerima informasi tentang perangkat audio, hanya tinggal mengisi file ".asoundrc", yang harus disimpan dalam direktori "/ home / pi". Dalam kasus kami, dengan kartu audio-USB eksternal, pengaturan untuk subsistem suara ALSA adalah sebagai berikut:

pcm.!default { type asym capture.pcm "mic" playback.pcm "speaker" } pcm.mic { type plug slave { pcm "hw:1,0" } } pcm.speaker { type plug slave { pcm "hw:1,0" } }

Di mana nomor kartu dan nomor perangkat yang sesuai: "1.0", sesuai dengan "Audio USB" kami. Namun, ketika instalasi mulai berhasil, semuanya berjalan dengan sangat baik.


Untuk menyesuaikan volume, Anda dapat menggunakan konsol AlsaMixer

Langkah selanjutnya adalah mendaftarkan proyek baru di Actions Console . Untuk melakukan ini, pilih nama proyek dan aktifkan penggunaan Google Assistant API untuknya. Kemudian, kami mendaftarkan model perangkat kami ( Register the Device Model ), sebagai hasilnya kami mendapatkan file "credentials.json", dalam format JSON, dengan deskripsi perangkat kami. File ini harus dipindahkan ke direktori kerja “/ home / pi /”, misalnya, jika kita mendaftarkan perangkat dari komputer lain, kita dapat menggunakan protokol SSH: “$ scp ~ / Downloads / credentials.json pi @ raspberry-pi-ip-address : / home / pi / ". Ngomong-ngomong, di masa depan kita akan membutuhkan pengetahuan tentang pengidentifikasi: "proyek-my-dev" dan "model-saya", namun, yang akan selalu tersedia dari Actions Console. Anda dapat melewati langkah berikutnya tanpa menentukan properti Menentukan sifat. Properti ini akan dibutuhkan selama pengembangan Tindakan Anda (Tindakan), misalnya, untuk reaksi dengan frasa: "Oke Google, nyalakan." untuk LED.

Akhirnya, mari beralih ke menginstal Google Assistant SDK . Mengapa menginstal dependensi terlebih dahulu, di mana disarankan untuk menginstal versi ketiga Python: 

 $ sudo apt-get update $ sudo apt-get install python3-dev python3-venv $ python3 -m venv env $ env/bin/python -m pip install --upgrade pip setuptools wheel $ source env/bin/activate

Seperti yang dapat Anda lihat dari tim, lingkungan virtual sedang berlangsung untuk proyek. Kemudian kami memasang komponen yang diperlukan di dalamnya:
