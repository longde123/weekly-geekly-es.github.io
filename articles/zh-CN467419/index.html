Jakarta EE 8最终版本发布

Eclipse Foundation已发布了有关Jakarta EE 8平台和Web配置文件的新规范。
他们还介绍了Jakarta EE应用服务器和测试套件（TCK），以测试其他兼容的实现。


2019年9月10日，雅加达EE 8发布。

Jakarta EE规范可在其网站上找到 。

它们的范围从“雅加达注释”到“雅加达服务器页面”。

与Eclipse规范一起，已经发布了实现Glass的开源应用程序服务器Eclipse Glassfish Server 5.1，该服务器实现了Jakarta EE 8平台。

有望从各种Java服务器提供商开发Jakarta EE 8服务器。 因此， IBM Open Liberty服务器已经通过了Jakarta EE 8兼容认证。

Eclipse 同意在2017年接管开发工作 。 Eclipse并没有引入较新的Jakarta EE 9，而是希望实现更平滑的迁移，逐渐用Jakarta EE替代Java EE。

雅加达EE开发


（根据Eclipse）将使用Jakarta EE构建现代解决方案，例如云服务和微服务。

更新Jakarta EE的未来功能包括作为Jakarta EE基础的Java SE更新。

当前版本是Java SE 8，下一个候选版本是Java SE 11 （这是SE 8之后的下一个具有长期支持周期的版本）。

此外，与Kubernetes，Docker等云技术一起使用将得到改善。

单独的规范也有望得到改进，例如Jakarta RESTfull Web服务。

另外，Eclipse Foundation必须定义将javax名称空间更改为jakarta的过程。 尽管Eclipse正在努力向后兼容以降低迁移的严重性，但这可能需要重新编译程序。

javax命名空间向后兼容20年了，但是由Oracle拥有 。
