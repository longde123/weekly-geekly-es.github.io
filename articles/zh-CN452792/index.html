企业用户固态固态硬盘评论金士顿DC500R

金士顿最近发布了专为高恒定负载而设计的金士顿DC500R企业级固态硬盘。 现在，许多记者正在积极测试新产品并发布有趣的材料。 我们想与哈勃分享金斯敦DC500R的详细评论之一，其测试将吸引读者。 原始文档位于Storagereview网站上，并以英文发布。 为方便起见，我们已将材料翻译成俄文并置于切口下。 祝您阅读愉快！



金士顿DC500R驱动器基于3D TLC NAND闪存技术。 选件的容量分别为480 GB，960 GB，1.92 TB和3.84 TB，这为希望省钱的公司或根本不需要大容量驱动器的公司提供了更多选择。 这篇评论讨论了3.48 TB版本，其声明的顺序读取和写入速度分别为555 MB / s和520 MB / s，以及在98,000和28,000输入操作的恒定负载下的4 KB块读取和写入速度。 -每秒输出（IOPS）。 作为该产品系列的一部分，金士顿还提供针对混合用途应用进行了优化的DC500M。

测试信息

企业级测试实验室StorageReview提供了充足的机会在接近实际条件的环境中测试存储设备。 实验室包括各种服务器，网络设备，电源系统和其他网络基础结构。 这使我们的员工能够创造接近真实的条件，以准确评估设备性能。
评审中包含有关环境和协议的信息，以便IT专业人员和负责购买存储系统的人员可以评估达到相应结果的条件。 测试设备的制造商不支付检查费用，也不控制检查费用。

应用程序工作量分析

为了正确评估企业级存储设备的性能参数，对适合于实际环境的基础结构和应用程序工作负载进行建模非常重要。 因此，为了评估Samsung 883 DCT SSD，我们使用SysBench实用程序测量了MySQL OLTP数据库的性能，并通过模拟TCP-C工作负载来测量了Microsoft SQL Server OLTP数据库的性能 。 在这种情况下，对于应用程序，每个驱动器将处理2到4个配置相同的虚拟机。

SQL Server性能

为每个SQL Server虚拟机配置两个虚拟磁盘：容量为100 GB的启动磁盘，以及用于承载数据库和日志文件的容量为500 GB的磁盘。 在系统资源方面，每个虚拟机都配备有16个虚拟处理器，64 GB DRAM和LSI Logic SAS SCSI控制器。 以前，使用Sysbench工作负载，我们测试了I / O速度和驱动器容量利用率。 反过来，SQL测试有助于评估延迟。

作为测试的一部分，SQL Server 2014已部署到运行Windows Server 2012 R2的来宾虚拟机。 使用Quest Benchmark Factory for Databases软件创建负载。 StorageReview的Microsoft SQL Server OLTP数据库测试协议使用事务处理性能委员会基准C性能评估程序（TPC-C）的当前版本。 该实时事务处理性能评估程序可模拟复杂应用程序环境的过程。 通过TPC-C测试，您可以比人工性能测试更准确地确定数据库环境中存储基础架构的优缺点。 作为测试的一部分，SQL Server虚拟机的每个实例都使用333 GB（1500规模）的SQL Server数据库。 在15,000个虚拟用户的负载下进行了事务处理期间的性能和延迟测量。

SQL Server测试配置（针对每个VM）：
•Windows Server 2012 R2
•磁盘空间：已分配600 GB，已使用500 GB
•SQL Server 2014
-数据库大小：1500规模
-虚拟客户端数量：15,000
-RAM内存缓冲区：48 GB
•测试时间：3小时
-2.5小时-初级阶段
-30分钟-直接测试

根据SQL Server事务处理性能估计，金士顿DC500R仅略微落后于三星883 DCT，显示总性能为每秒6,290.6个事务（TPS）。



甚至比TPS指标更好，评估SQL Server的性能可以衡量延迟。 在这里，两个驱动器-三星860 DCT和金士顿DC500R-都显示相同的时间：26.5 ms。



使用Sysbench的性能

下一个测试使用Percona MySQL数据库 。 使用SysBench实用程序评估了OLTP性能。 在这种情况下，将测量平均TPS和延迟，以及最不利情况下的平均延迟时间。

每个Sysbench虚拟机都使用三个虚拟磁盘：大约92 GB的启动磁盘，具有大约447 GB的预安装数据库的磁盘和具有270 GB的测试数据库的磁盘。 在系统资源方面，每个虚拟机都配备有16个虚拟处理器，60 GB的DRAM和LSI Logic SAS SCSI控制器。

Sysbench测试配置（针对每个VM）：

•CentOS 6.3 64位
•Percona XtraDB 5.5.30-rel30.1
-数据库表数：100
-数据库大小：10,000,000
-数据库线程数：32
-RAM内存缓冲区：24 GB
•测试时间：3小时
-2小时-初级阶段，有32个线程
-1小时-直接测试，32个线程

根据Sysbench事务处理性能等级，DC500R落后于竞争对手，每秒提供1,680.47笔事务。



在平均延迟方面，DC500R以76.2 ms的数字位居榜首。



最后，在最坏情况下（第99个百分位数）测试延迟时间之后，DC500R再次以134.9 ms的结果排在列表的末尾。



VDBench工作负载分析

在测试存储设备时，应用程序测试比综合测试更可取。 但是，尽管它们的结果不符合实际条件，但是由于任务的可重复性，综合测试可方便地确定基本指标和比较竞争解决方案。 这些测试提供了广泛的配置文件，从四个角测试和示例数据库迁移测试到跟踪来自各种VDI环境的捕获。 在所有这些情况下，都使用带有脚本处理器的单个vdBench工作负载生成器来自动化和收集大型计算测试集群中的结果。 这样就可以将相同的工作负载用于各种驱动器，包括闪存阵列和单个驱动器。 作为测试的一部分，我们将数据完全填充到驱动器中，然后将它们划分为容量为原始容量25％的分区，以模拟应用程序负载并评估驱动器的性能。 这种方法与完全熵测试不同，完全熵测试在恒定负载下立即使用整个磁盘。 因此，以下结果反映了更稳定的记录速度。

个人资料：
•4k随机读取：只读，128个线程，输入/输出速度为0到120％
•4K随机写入：仅写入，64个流，输入/输出速度为0到120％
•顺序读取64 KB：只读，128个线程，输入/输出速度从0到120％
•顺序写入64 KB：仅写入，64个流，输入/输出速度从0到120％
•综合数据库：SQL和Oracle
•VDI复制（完整副本并创建链接副本）

在第一个VDBench工作负载测试（4K随机读取）中，金士顿DC500R显示了令人印象深刻的结果：延迟1毫秒直到达到80,000 IOPS的速度水平，最大速度为80,209 IOPS，延迟为1.59毫秒。



在第二次测试（随机写入4 KB）中，所有测试的驱动器均显示几乎相同的结果：速度略高于63,000 IOPS，延迟为2毫秒。



转到顺序加载，我们首先查看了64K读取。 在这种情况下，金士顿驱动器在毫秒内保持延迟，直到达到5200 IOPS（325 MB / s）。 该磁盘的最高数字为7183 IOPS（449 MB / s），延迟为2.22 ms，使该磁盘在总体排名中排名第二。



在测试顺序记录操作时，金士顿设备将延迟保持在1 ms以下，最高速度达到5700 IOPS（356 MB / s），从而胜过所有竞争对手。 最大速度为6291 IOPS（395 MB / s），延迟为2.51 ms。



之后，我们继续执行SQL任务，在这三个测试中，金士顿DC500R是唯一延迟时间超过毫秒的设备。 在第一种情况下，磁盘的最大速度为26411 IOPS，延迟为1.2 ms。



在SQL 90-10测试中，金士顿以27339 IOPS的最大速度以1.17毫秒的延迟位居最后。



SQL 80-20测试中发生了同样的事情。 在这种情况下，金士顿设备的最大速度为29576 IOPS，延迟为1.08 ms。



Oracle工作负载测试结果将DC500R放回了最后一位，但是在两次测试中，该设备仍然显示了毫秒级的延迟。 在第一种情况下，金士顿的最大磁盘速度为29098 IOPS，延迟为1.18毫秒。



在第二项测试（Oracle 90-10）中，DC500R达到24555 IOPS，延迟为894.3μs。



在第三个测试（Oracle 80-20）中，金士顿设备的最大速度为26401 IOPS，延迟级别为831.9μs。



然后，我们继续复制VDI-创建完整和链接的副本。 在测试完整版VDI的下载时，金士顿驱动器再次未能超越竞争对手。 该设备支持低于1 ms的延迟，最高速度约为12000 IOPS，最大速度为16203 IOPS，延迟为2.14 ms。



在测试VDI的初始登录副本时，金士顿设备的性能更好，最终（略有优势）获得了第二名。 该驱动器支持一毫秒内的延迟，直到达到11000 IOPS的速度为止，最大速度为13652 IOPS，延迟为2.18毫秒。



同样，金士顿硬盘在星期一登录测试结果中获得了第二名，获得了完整的VDI副本。 希捷Nytro 1351硬盘的最高速度略高一些，但金斯敦设备在整个测试过程中通常显示出较低的延迟水平。 DC500R的最高速度为11897 IOPS，延迟为1.31毫秒。



在测试VDI链接副本的下载时，金士顿排在最后。 延迟已经低于1毫秒，速度低于6000 IOPS。 DC500R的最高速度为7861 IOPS，延迟为2.03毫秒。



但是，根据“初始登录”测试的结果，该驱动器再次获得第二名：仅在达到峰值性能后，延迟才超过毫秒的限制，最终达到7950 IOPS，延迟为1.001 ms。



在最后一次测试VDI链接副本的过程中-星期一登录-该磁盘还显示了第二个结果：最大速度为9205 IOPS，延迟为1.72毫秒。 在毫秒级之外，达到6400 IOPS的速度时，延迟超过了。



结论


DC500R是金士顿面向企业用户的最新固态硬盘。 DC500R的尺寸为2.5英寸。 可用容量从480 GB到3.84 TB不等。 该驱动器基于3D TLC NAND闪存技术，并结合了大量资源和高性能。 对于3.48 TB磁盘，连续读取和写入速度分别为555 MB / s和520 MB / s，具有恒定负载分别为98,000和28,000 IOPS且资源容量为3504 TBW的读取和写入速度。

为了评估金士顿DC500R的性能，我们将其与其他流行的SATA SSD进行了比较，包括三星860 DCT和883 DCT驱动器以及Seagate Nytro 3530驱动器。 Kington DC500R能够保持竞争对手的水平，在某些情况下甚至超过了竞争对手。 测试应用程序工作负载时，金士顿DC500R在处理SQL任务时表现出色，在每秒事务数（6291.8 TPS）和延迟时间（26.5 ms）方面排名第二。 在具有更强写入性能的Sysbench测试中，DC500R以1,680.5 TPS性能最终排在列表的末尾，平均延迟为76.2 ms，最不利情况的延迟为134.9 ms。

作为测试4 KB随机读取和写入块的一部分，金士顿DC500R的速度为80209 IOPS，读取时延迟为1.59 ms，以及速度为63000 IOPS，写入时延迟为2 ms。 作为64 KB读写测试的一部分，DC500R的速度分别为7183 IOPS（449 MB / s），2.22毫秒的延迟和6291 IOPS（395 MB / s），2.51毫秒的延迟。 在使用SQL和Oracle数据库进行综合测试以及提高写入速度要求的情况下，DC500R的性能很差。 在SQL工作负载方面，金士顿DC500R在所有三个测试中均处于最后位置，并且是延迟时间超过毫秒的唯一驱动器。 但是，在测试Oracle时，情况要好得多。 在三分之二的测试中，驱动器支持的延迟小于1毫秒，位居第二。 使用完整和相关的VDI副本进行测试时，金士顿DC500R表现出不错的性能。

通常， 金士顿DC500R SSD是同类中的高质量设备，值得更多关注。 无论我们多么喜欢高性能技术（NVMe和类似技术），SATA驱动器仍然是处理任务的首选解决方案，在这些任务中，可靠性起着至关重要的作用，例如加载服务器或存储控制器。 在物有所值的情况下，这种驱动器也是用于存储服务器数据的经济解决方案。 此外，就固态硬盘的整体拥有成本而言，它们还提供了所有优势，固态硬盘从硬盘驱动器的背景中脱颖而出。 DC500R的性能使该设备能够根据我们与其他同类驱动器进行的多次测试得出的结果来占据头把交椅。 DC500R是一款出色的SATA驱动器，适用于需要可靠且生产效率高的驱动器，大容量资源和多种容量选择的场景。
