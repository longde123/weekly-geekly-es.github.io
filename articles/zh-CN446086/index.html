Linux历史。 第三部分：新市场和旧“敌人”

我们记得 90年代发布的第一批​​商业Linux发行版。 现在到了2000年代-这个操作系统开始征服消费者市场的时期。


/ Flickr / 利亚姆·奎因 / CC BY-SA

迈向消费者


十到二十年前，Linux在PC操作系统市场中的份额还很低 ，不到2％。 原因之一是缺少熟悉的桌面环境。 随着KDE和GNOME的出现，情况开始发生变化。 它们仍然是最受欢迎的Linux桌面环境。 这两种产品成为消费市场分销诞生的最初信号之一。 他没有让自己等待。

2004年，发布了基于Debian架构的Ubuntu 4.10。 该项目的出现归功于企业家和开发商Mark Shuttleworth。

马克虽然从事Debian的工作，但其中发现了许多缺点，这些缺点与他理想的操作系统的想法不符。 沙特尔沃思（Shuttleworth）认为，有必要改变更新周期的方法，并专注于产品的可用性。 最后，他和几个同事决定创建Canonical，并开始开发他的发行版。

开源社区热烈欢迎具有稳定更新发布计划的新系统-每年两次。 操作系统问世三年后，它成为最受欢迎的桌面Linux发行版。 用户选择 Ubuntu来简化安装和安全性。 多年来，我们可以说Mark Shuttleworth创建一个方便且易于理解的Linux产品的想法成真。

正如开源软件历史上经常发生的那样，在Ubuntu的第一个版本发布后不久便提出了一种改进替代方案的想法。 2006年，Linux Mint发行版的创建者Clement Lefebvre受到了这个想法的指导。 它的操作系统的第一个版本基于Kubuntu，但是已经有了v2.0，开发人员转而使用Ubuntu。 从那一刻起，Mint从其前身汲取了越来越多的想法，两年后，更新周期终于“绑在了” Ubuntu版本上。

Linux Mint的一个功能是支持专有软件，包括用于播放多媒体文件的插件和编解码器。 对于Linux发行版来说，这样的举动并不常见-开发社区坚持使用开放模型。 但是，新方法确保了Mint的成功-Linux用户终于能够使用熟悉的Windows应用程序。 在过去的八年中，Mint一直是内核历史上最受欢迎的三种Linux发行版之一。

个人电脑市场之战


戴尔注意到了Ubuntu在2000年代中期的流行和便利。 早在1998年，该公司就成为第一家在整个产品线中提供Linux的主要制造商。 但是，戴尔的开源OS支持计划仅持续了几年。 封闭的间接原因被称为 “需求不足”。

2007年，该项目随着基于Ubuntu的PC和笔记本电脑的发布而恢复。 迈克尔·戴尔本人使用了该分发工具包并赞赏其便利性，这一点发挥了重要作用。 他联系了Canonical，公司开始为Dell计算机准备软件。 预装了Ubuntu的计算机在美国，英国，法国和德国销售。

华硕对此信号作出了回应。 该公司发布了2007年Asus Eee上网本。 它的主要功能是预装的Linux系统。 支持开源OS的选择使制造商可以大大降低设备的价格并吸引社区的注意。

但是，由于华硕（Asus）-Xandros选择的分布，社区的反应参差不齐。 OS是同名公司的商业产品，未完全遵守自由软件的原则。 很快，第一个基于Windows XP的Asus Eee发布了。 微软已大大降低了制造商的许可费用，并在新的细分市场中占有一席之地。 之后，Linux 开始失去在新市场中的份额。

但是崩溃没有发生。 部分原因是Google于2009年发布了Chrome操作系统。 新的操作系统已成为另一款Linux设备-Chromebook的基础。 莱纳斯·托瓦尔兹（Linus Torvalds）曾经说过，这款笔记本电脑的发布可能是PC上Linux未来的转折点。 四年后，这些设备的销量超过了美国的Apple Mac。 但是我们将在下一篇文章中讨论。

移动时代


在Linux的基础上构建了许多移动操作系统。 例如，诺基亚于2005年发布的Maemo OS基于Debian，并借鉴了GNOME的大多数GUI元素和库。 2004年，移动操作系统历史上最早的开发者之一-Palm OS的开发者将其操作系统转移到Linux内核。 该计划产生了两个独立的项目-Access Linux Platform（ALP）和Palm webOS。 第一个没有得到分配，Palm和其他制造商的几款移动设备都是基于webOS推出的。

那几年对于Linux社区而言，更重要的事件发生在完全不同的领域。 Android Inc.于2003年在加利福尼亚成立。 推出一年后，团队开始寻找投资者。 该公司最初制定战略的基础产品是数码相机的操作系统。 在这个小众市场上没有成功，团队改变了重点。 因此出现了Android系统的想法-当时在移动OS领域的领导者Symbian和Microsoft Windows Mobile的竞争对手。 随着2005年朝这个方向的最初发展，该公司被Google 收购 。

这家IT巨头早在2006年就向电信公司推出了运行新操作系统的设备的第一款原型。 这是一部带有QWERTY键盘且没有触摸屏的手机。 2007年，媒体获悉 Google的移动操作系统将在Linux上运行，并且该公司已经在与移动运营商以及智能手机制造商进行谈判。 该版本应在不久的将来发布。

但是在同年，第一部iPhone进入了市场，这就是Google必须紧急在其手机上添加触摸屏并推迟发布日期的原因。 最终，在2008年，Google与T-Mobile一起推出了第一款Android智能手机T-Mobile G1或HTC Dream。 对G1的需求一开始就超出了预测 ，并且其发布标志着世界上最受欢迎的移动OS的历史的开始。 数十个Android版本和数千个智能手机使Linux成为这一利基市场的领导者 。

生态系统发展


除了在Linux社区中发布新发行版之外，其他重要事件还发生在2000年代。

长期以来，内核开发人员一直没有使用代码管理系统，这使得难以支持像Linux这样的大型项目。 因此，在2002年，Linus Torvalds决定改用BitKeeper。 尽管软件是一种商业产品，但是Linux开发人员有机会免费使用它。 尽管如此，社区还是发现选择SCM系统的决定引起争议。

2003年，BitKeeper提出了一个替代方案。 莱纳斯·托瓦尔兹（Linus Torvalds）介绍了他自己的用于管理核心开发的项目Git。 如今，Git已成为开发环境中最受欢迎的版本控制工具。

开放项目的生态系统也在发展。 活动，非商业产品的生产，许可活动-自90年代初以来，围绕这些活动创建了越来越多的组织。 其中一个后来变成了Linux Foundation财团。

该组织的原型是在非营利组织LI的基础上形成的。 在2000年，LI管理出现问题，公司更名为开放源代码开发实验室（OSDL）。 2003年，Linus Torvalds正式成为 OSDL 的成员。 该财团包括惠普，英特尔，IBM等公司。 四年后，OSDL与另一个非营利组织Free Standards Group合并。 通过该联盟，成立了Linux Foundation，该基金会仍参与开源项目的融资及其标准化。

旧冲突与新视野


新时代带来了Linux新挑战。 在2003年，拥有Unix的SCO Group公司声称 Linux和其他发行版滥用了原始OS的代码，从而侵犯了版权。 上合组织发起的船只均未获得申请人的青睐。

同时，Microsoft 发起了 Get the Facts广告活动，该活动公开比较了Linux和Windows的服务器版本（当然支持后者）。 微软的活动可能与Linux在企业领域的份额不断增长有关。 2004年，Unix和Linux 占据了该市场的近三分之一 ，这严重威胁了IT巨头的地位。

作为新广告活动的一部分，该公司声称Windows Server 2003的总拥有成本低于Linux。 开源社区以各种方式驳斥了这些事实。 在过去的几年中，虚拟纠纷的规模不断扩大，直到Microsoft在2007年关闭该活动为止。

但是，所有这些并没有阻止Microsoft与开源公司建立合作关系。 2006年，SUSE Linux Enterprise Server发行权的所有者Novell 宣布与Microsoft建立合作关系。 同样重要的是，这一已有13年历史的步骤可能会间接影响Microsoft的云战略。 随后，SUSE Linux Enterprise Server 成为了公司云平台上最早可用的发行版之一。

如果从更广泛的角度看待这种情况，事实证明，当今世界上大多数数据中心都使用开源OS。 在下一篇文章中，我们将更多地讨论Linux如何引领云革命。
