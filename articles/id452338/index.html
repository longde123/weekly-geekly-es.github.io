Memperkenalkan Terminal Windows

Windows Terminal adalah aplikasi terminal baru, modern, cepat, efisien, kuat, dan kuat untuk pengguna alat dan cangkang perintah-perintah seperti Command Prompt, PowerShell, dan WSL.



Terminal Windows akan dikirimkan melalui Microsoft Store ke Windows 10 dan akan diperbarui secara berkala, memastikan bahwa Anda akan selalu mengikuti perkembangan dan akan dapat menggunakan fitur-fitur terbaru dan peningkatan terbaru dengan upaya minimal.



Fitur Utama Terminal Windows


Banyak tab


Anda bertanya, dan kami mendengar! Fungsi yang paling sering diminta untuk terminal adalah dukungan untuk beberapa tab, dan kami sangat senang bahwa kami akhirnya dapat menyediakan fungsi ini. Sekarang Anda dapat membuka sejumlah tab, yang masing-masing terhubung ke shell command-line atau aplikasi pilihan Anda, misalnya, Command Prompt, PowerShell, Ubuntu di WSL, Raspberry Pi melalui SSH, dll.



Teks yang indah


Windows Terminal menggunakan akselerasi GPU DirectWrite / DirectX berdasarkan mesin rendering teks. Mesin rendering teks baru ini akan menampilkan karakter teks, mesin terbang dan karakter yang ada dalam font pada PC Anda, termasuk CJK ideogram, emoji, simbol powerline, ikon, ligatur pemrograman, dll. Mesin ini bahkan membuat teks lebih cepat daripada mesin sebelumnya Konsol GDI!



Anda juga akan memiliki kesempatan untuk menggunakan font baru kami! Kami ingin membuat font monospace yang menyenangkan, baru, untuk meningkatkan tampilan terminal yang modern. Font ini tidak hanya mencakup ligatur pemrograman, tetapi juga akan memiliki repositori Open-Source sendiri. Tetap disini untuk informasi lebih lanjut tentang proyek font baru!



Pengaturan dan Konfigurasi


Kami telah menghubungi banyak pengguna baris perintah yang suka menyesuaikan terminal dan aplikasi baris perintah mereka. Windows Terminal menyediakan banyak pengaturan dan parameter konfigurasi yang memberikan kontrol besar terhadap penampilan terminal dan masing-masing shell / profil yang dapat dibuka sebagai tab baru. Pengaturan disimpan dalam file teks terstruktur, yang menyederhanakan konfigurasi untuk pengguna dan / atau alat.

Menggunakan mekanisme konfigurasi terminal, Anda dapat membuat beberapa "profil" untuk setiap shell / aplikasi / alat yang ingin Anda gunakan, baik itu PowerShell, Command Prompt, Ubuntu, atau bahkan koneksi SSH ke perangkat Azure atau IoT. Profil-profil ini dapat memiliki kombinasi gaya dan ukuran font, tema warna, keburaman latar belakang / tingkat transparansi, dll. Sekarang Anda dapat membuat terminal sendiri dengan gaya Anda sendiri, yang disesuaikan dengan selera unik Anda!

Lagi!


Setelah merilis Windows Terminal 1.0, kami berencana untuk mulai bekerja dengan banyak fitur yang sudah ada dalam jaminan kami, di samping banyak fitur yang mungkin akan ditambahkan oleh Anda, sebagai komunitas,!

Kapan saya bisa mendapatkannya?


Hari ini, Terminal Windows dan Konsol Windows tersedia di Open-Source, sehingga Anda sudah dapat mengkloning, mengkompilasi, menjalankan, dan menguji kode dari repositori di GitHub:

github.com/Microsoft/Terminal

Juga, musim panas ini di Microsoft Store akan dirilis versi pratinjau Terminal Windows untuk perintis dan memberikan umpan balik.

Dan di musim dingin, kami berencana untuk akhirnya merilis Windows Terminal 1.0, dan kami akan bekerja dengan komunitas untuk memastikan bahwa itu sepenuhnya siap sebelum kami rilis!

Tunggu ... apa Anda bilang di Open-Source?


Ya itu! Kami dengan senang hati mengumumkan bahwa kami membuka tidak hanya Terminal Windows, tetapi juga Windows Console, yang berisi infrastruktur command-line di Windows dan menyediakan UX Console tradisional.

Kami ingin bekerja sama dengan Anda untuk meningkatkan dan memperluas kemampuan baris perintah Windows!

Kedengarannya luar biasa, tetapi mengapa Anda tidak hanya memutakhirkan Windows Console yang ada?


Tujuan utama Konsol Windows adalah untuk menjaga kompatibilitas ke belakang dengan alat baris perintah yang ada, skrip, dll. Meskipun kami berhasil menambahkan banyak perbaikan kunci ke fungsi konsol (misalnya, tambahkan dukungan VT dan warna 24-bit, dll. Lihat ini posting blog ), kami tidak dapat membuat peningkatan signifikan lebih lanjut pada konsol UI tanpa "menghancurkan dunia."

Oleh karena itu, saatnya telah tiba untuk pendekatan baru yang segar.

Terminal Windows menginstal dan bekerja dengan aplikasi Windows Console yang ada. Jika Anda memulai Cmd / PowerShell / etc. secara langsung, mereka akan mulai menghubungkan ke instance konsol tradisional dengan cara yang sama seperti biasa. Dengan demikian, kompatibilitas mundur tetap tidak berubah, dan pada saat yang sama, Anda dapat menggunakan Terminal Windows jika / ketika Anda ingin melakukan ini. Konsol Windows akan terus dikirim ke Windows selama beberapa dekade untuk mendukung aplikasi / sistem lama yang ada.

Nah, bagaimana dengan berpartisipasi dalam proyek terminal atau aplikasi yang ada di Open-Source?


Kami dengan cermat mempelajari opsi ini selama perencanaan dan memutuskan bahwa partisipasi kami dalam proyek yang ada akan membutuhkan perubahan persyaratan dan arsitektur proyek sehingga itu akan terlalu merusak.

Sebagai gantinya, dengan membuat aplikasi terminal Open-Source baru dan Open-Source Windows Console, kami dapat menawarkan komunitas untuk bekerja bersama kami untuk meningkatkan kode dan menggunakannya dalam proyek masing-masing.

Kami percaya bahwa ada cukup ruang di pasar untuk ide-ide baru / berbeda tentang apa yang dapat dan harus dilakukan terminal, dan kami berusaha untuk membantu ekosistem aplikasi terminal (dan yang terkait) berkembang dan berkembang melalui pengenalan ide-ide baru, pendekatan menarik dan inovasi menarik di ruang ini.

Yakin! Bagaimana cara berpartisipasi?


Kunjungi repositori di github.com/Microsoft/Terminal untuk mengkloning, merakit, menguji, dan menjalankan terminal! Selain itu, kami akan berterima kasih jika Anda akan melaporkan bug dan berbagi umpan balik dengan kami dan komunitas, serta memperbaiki masalah dan melakukan perbaikan pada GitHub.

Musim panas ini, cobalah menginstal dan memulai Terminal Windows dari Microsoft Store. Jika Anda menemukan kesalahan, bagikan umpan balik melalui Hub Umpan Balik atau bagian Masalah di GitHub, yang dirancang untuk pertanyaan dan diskusi.

Kami senang bekerja sama dengan Anda! Jika Anda memiliki pertanyaan atau tanggapan, jangan ragu untuk menghubungi Kayla @cinnamon_msft dan / atau Rich @richturn_ms di Twitter. Kami ingin melihat peningkatan dan fitur hebat apa yang akan Anda bawa ke Terminal Windows dan Windows Console.
