Zabbix 4.4的新增功能

Zabbix团队很高兴宣布Zabbix 4.4的发布。 最新版本随附以Go语言编写的新Zabbix代理，为Zabbix模板设置标准，并提供高级可视化功能。



让我们看一下Zabbix 4.4中包含的最重要的功能。

Zabbix新一代代理商



Zabbix 4.4引入了一种新型的代理zabbix_agent2，它提供了广泛的新功能和高级监视功能：

  • 它是用Go语言编写的。
  • 一个用于监视各种服务和应用程序的插件框架。
  • 维护检查之间的状态的能力（例如，维护持久数据库连接）。
  • 内置调度程序以支持灵活的时隙。
  • 通过传输大量数据来有效利用网络。
  • 目前，该代理在Linux上运行，但是在不久的将来，我们将使其在其他平台上可用。

→请参阅文档以获取新功能的完整列表。

注意！ 现有的Zabbix代理将仍然受支持。

→ 下载

Web挂钩和可编程动作/通知逻辑

与外部通知和票证发行系统的集成已得到显着改善，这使我们能够使用内置的JavaScript引擎定义所有处理逻辑。 此功能简化了与外部系统的双向集成，从而允许从Zabbix用户界面一键访问以在您的票务系统中进行记录，生成聊天消息等等。

设置Zabbix模板的标准

我们为创建模板引入了许多标准和定义明确的准则 。

XML / JSON文件的结构已大大简化，这使您可以仅使用文本编辑器手动编辑模板。 大多数现有模板已得到增强，以使其与新标准保持一致。

对TimescaleDB的官方支持

除了MySQL，PostgreSQL，Oracle和DB2，我们现在正式支持TimescaleDB。 TimescaleDB提供几乎线性的性能水平，以及自动自动删除旧历史数据的功能。

在本文中，我们将性能与PostgreSQL进行了比较。

基于数据元素和触发器的知识库



Zabbix 4.4提供了对数据元素和触发器的更加直观的描述。 这些信息对工程师有很大帮助，为他们提供了有关所收集物品的含义和目的的所有可能详细信息，问题的详细信息以及解决方法。

高级可视化



工具栏和相关的窗口小部件已通过各种方式进行了改进，从而简化了它们的创建和管理，并且还增加了通过单击更改窗口小部件设置的功能。 仪表板格栅的大小现在适合于支持宽屏和大屏幕。

问题显示小部件已得到增强，以支持聚合表示，并且引入了新的小部件来显示原型图。

另外，现在所有小部件都可以以无头模式显示。

条形图和数据聚合



Zabbix 4.4支持直方图，并且图形小部件现在可以使用各种聚合函数来聚合数据。 这两个功能的结合极大地促进了长期数据分析和带宽规划。

对新平台的官方支持


Zabbix 4.4现在可在以下平台上运行：

  • SUSE Linux Enterprise Server 15
  • Debian 10
  • 树莓10
  • RHEL 8
  • Mac OS / X的代理
  • Windows版MSI代理

所有可用平台均可在下载部分中找到。

一键式云安装

Zabbix可以作为容器或现成的磁盘映像轻松安装在各种云服务上：

  • ws
  • 蔚蓝
  • 谷歌云平台
  • 数字海洋
  • 码头工人

可靠的自动注册



Zabbix的新版本允许将PSK加密用于添加的主机的自动加密设置的自动注册。 现在，您可以将Zabbix配置为仅使用PSK，仅未加密或两者都允许自动注册网络设备。

用于预处理的高级JSONPath



Zabbix现在支持高级JSONPath语法，该语法允许对JSON数据进行复杂的预处理，包括聚合和搜索。 预处理还可以用于低级别检测，使其成为用于自动化和检测的极其强大的工具。

用户宏描述



自定义宏提供了很好的功能，可简化Zabbix的配置并大大简化配置更改。 支持自定义宏描述将有助于记录每个宏的用途，这将大大简化其管理。

更高效的高级数据收集



与WMI，JMX和ODBC相关联的对象的数据收集和发现已通过新的检查以JSON格式返回对象数组的方式得到了改进。 我们还增加了对用于监视Linux平台的VMWare和系统服务的VMWare数据仓库的支持，以及用于将CSV转换为JSON的新型预处理。

Zabbix 4.4的其他新功能和改进功能

  • 从LLD预处理XML数据
  • 依赖项的最大数量已增加到1万。
  • 在JSONPath预处理中添加了自动类型转换
  • 主机名实时包含在导出文件中。
  • Windows代理现在支持英语性能计数器
  • 能够在发生错误时忽略预处理中的值
  • 最近的数据已得到扩展，不仅可以访问历史数据，还可以访问操作数据。
  • 编辑触发器描述的功能已被删除，对它们的访问已大大简化
  • 取消了对Jabber和Eztexting内置媒体文件类型的支持，改为使用Web挂钩或外部脚本
  • 更新了默认仪表板
  • 自动注册的主机有机会指定选项“连接到dns”或“连接到IP”
  • 为触发URL添加了宏支持{EVENT.ID}
  • 不再支持屏幕元素“屏幕”
  • 上一次创建的仪表板小部件类型将被记住并在以后重用
  • 小部件标题的可见性可针对每个小部件进行自定义。

Zabbix 4.4的新功能的完整列表可以在新版本的注释中找到。
