Englischer Code

Englisch ist vielleicht das einzige, das zum Schreiben von Code geeignet ist. Die darauf geschriebenen Wörter sind relativ kurz und ändern je nach Kontext fast nicht ihre Form. Hauptsache aber, dass sich die imperative Stimmung der Äußerungen im Englischen kaum vom Indikativ unterscheidet. Dies ist wichtig, da sich im Paradigma der objektorientierten Programmierung der Name der Methode auf das Verhalten der Entität bezieht und je nach Gerät der dynamischen Sprachen ein Methodenaufruf eine Nachricht an die Zielentität sendet, die entscheidet, was damit zu tun ist. Betrachten Sie dies anhand von Beispielen:

unbedingt bestellen 

 ->

oder Verhalten erklären 

 .()

Sogar die Infinitivoption sieht aus wie ein wahrnehmbarer Akzent der amerikanischen Ureinwohner 

 .()

während auf Englisch sowohl das eine als auch das andere als geschrieben werden 

 mailClient.receiveMessages()

Daher sieht der Code, der in den Landessprachen geschrieben ist, in den Augen des Betrachters fast immer wie „Scheiße“ aus, und auf Englisch besteht die Möglichkeit, dass er mit Selbstgefälligkeit akzeptiert wird, unabhängig davon, welches Paradigma jeder Programmierer denkt.
