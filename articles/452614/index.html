Cómo comenzar a programar en Adobe Illustrator. Parte dos

Esta publicación es una continuación de la primera parte , donde se presentó el script Expandir máscara de recorte y se describió en detalle qué y cómo lo hace, y en el camino se consideraron los principios básicos de la creación de tales programas en general. En esta parte, continuaré la historia de cómo agregar nuevas funcionalidades al programa para obtener un "producto terminado" de la "pieza de trabajo". Aquí no puede prescindir de una inmersión más profunda en el área temática, que es una de las condiciones necesarias para crear un producto completo. Entonces, comienza la inmersión!


Las siguientes primitivas gráficas se pueden usar como un contorno de máscara en Adobe Illustrator: un esquema simple (Ruta), un esquema compuesto (Ruta Compuesta), una forma compuesta (Forma Compuesta) y objetos de texto (Texto de Punto y Texto en la Ruta). Por el momento, el script solo funciona con contornos simples, como se puede ver en el siguiente código, donde PathItem es una llamada al elemento Path.

Antes de eso, clipPath la variable clipPath , pero no le asignamos un valor. 


  var clipPath;

Esto significa que su valor aún no se ha determinado, es decir Es undefined . Si ahora seleccionamos una máscara cuyo contorno será, digamos, la ruta compuesta y ejecutamos el script, el programa producirá un error en la última línea de la parte funcional del script, 


  clipPath.remove();

clipPath la condición en el bucle no se cumplirá, la variable clipPath permanecerá undefined y es imposible aplicar el método remove() a algo indefinido. Para evitar esta situación, haremos lo siguiente: establecer clipPath en null , que, a diferencia de undefined , ya es algo más específico que al menos puede verificar.


Pensemos cómo determinar si algún camino compuesto es el contorno de nuestra máscara. Cuando digo "pensemos", significa que sugiero buscar en la documentación y encontrar la propiedad que necesitamos. Por analogía con PathItem estamos buscando la propiedad de clipping . Resulta que el objeto CompoundPathItem no tiene dicha propiedad, pero hay una propiedad pathItems través de la cual puede obtener contornos PathItem simples que tienen la propiedad de clipping .

Ahora podemos convertir nuestros pensamientos / búsquedas en código. En primer lugar, clipPath que clipPath no se definió en la iteración anterior, y luego copiamos el bloque de código ya escrito y le hacemos pequeños cambios.

El siguiente "paciente" es la forma compuesta. ¡Aquí se vuelve bastante interesante! En la documentación no encontramos tal objeto en absoluto. Que hacer Primero, definamos a qué clase de objetos pertenece. Para averiguarlo, escribiremos un pequeño código auxiliar, que luego tiraremos. Como se mencionó en la primera parte , no abordamos el problema de las herramientas utilizadas para escribir / depurar código. Por lo tanto, suponga que este es un archivo separado, que luego simplemente va a la papelera. El código será el siguiente:

En la primera línea, creamos un enlace al objeto seleccionado, en la segunda, mostramos un mensaje acerca de qué tipo es. Seleccione el contorno de la máscara en Adobe Illustrator, es decir. el mismo objeto de Forma compuesta y ejecute el script. En el cuadro de mensaje, vemos que la Forma compuesta es un elemento de complemento. Nos deshacemos del código auxiliar, volvemos a la documentación nuevamente, pero no encontramos la propiedad de recorte o pathItems en PluginItem. En general, nada que nos ayude a indicar inequívocamente que este objeto es el contorno de la máscara. A partir del script, ni siquiera puede determinar qué tipo de complemento es. Algún módulo externo y eso es todo!


Eso es una emboscada! - exclamas en tus corazones. Y el cerebro está trabajando febrilmente, con la esperanza de resolver un problema sin solución. Y luego, después de pasar por todas las opciones posibles e imposibles, presiona desesperadamente Del y elimina la odiada Forma compuesta. Y aquí, por el rabillo del ojo en la paleta Layers , notas que después de esta acción, el contenedor de máscara, que era Clip Group , se convirtió en solo Group . ¿Qué podría significar? Y el hecho de que la propiedad clipped del objeto clipped de true volvió false . ¡Aquí está, una solución que podría funcionar! Por supuesto, esto es, en general, un truco, pero qué diferencia hace si ayuda a determinar el circuito deseado.

El algoritmo para determinar el contorno de la máscara representado por el objeto de Forma compuesta será el siguiente: iterar sobre todos los objetos de máscara en el bucle y cuando encontremos PluginItem, elimínelo y verifique si la propiedad clipped del contenedor de máscara ha cambiado. Si se volvió false , entonces este es nuestro circuito. Lo único que funciona para este truco es actualizar DOM Illustrator después de eliminar el objeto, lo que se puede hacer usando el método app.redraw() . Entonces aún debe recordar devolver el objeto remoto, que se realiza mediante el método app.undo() .


A continuación se muestra el código para la ruta de Forma compuesta: 


  if (clipPath == null) { for (var i = 0; i < clipGroup; i++) { if (sel[0].pageItems[i].typename == 'PluginItem') { sel[0].pageItems[i].remove(); app.redraw(); if (sel[0].clipped == false) { app.undo(); clipPath = sel[0].pageItems[i]; break; } else { app.undo(); } }; }; };

Ahora, de todas las opciones posibles para el tipo de objetos que pueden ser el contorno de la máscara, solo queda el texto (o TextFrameItem , en la terminología de las referencias de script de ilustrador ). Pasamos a la documentación una y otra vez, no encontramos las propiedades de clipping allí. Pero esta vez, ya no nos preocupamos tanto por esto y tranquilamente descubrimos que TextFrameItem tiene una propiedad kind que determina el tipo de objeto de texto ( TextType ). Descubrimos que puede haber tres tipos: AREATEXT, POINTTEXT y PATHTEXT. El primer tipo no es interesante para nosotros, ya que no se puede usar como un esquema de máscara, y los otros dos siguen siendo tan interesantes. Solo queda encontrar un truco que nos ayudará a determinar ahora no el contorno, sino el objeto de texto, que es el contorno de la máscara. Y ese truco será el equipo, Convertir a tipo de área, que convertirá POINTTEXT a AREATEXT. Al igual que con la Forma compuesta, se clipped un cambio implícito en la propiedad clipped .


En consecuencia, el código para un TextFrameItem de tipo POINTTEXT será el siguiente: 


  if (clipPath == null) { for (var i = 0; i < clipGroup; i++) { if (sel[0].pageItems[i].typename == 'TextFrame' && sel[0].pageItems[i].kind == 'TextType.POINTTEXT') { sel[0].pageItems[i].convertPointObjectToAreaObject(); app.redraw(); if (sel[0].clipped == false) { app.undo(); clipPath = sel[0].pageItems[i]; break; } else { app.undo(); } }; }; };

Solo queda un TextFrameItem tipo PATHTEXT. Desafortunadamente, al convertir PATHTEXT a AREATEXT, la propiedad clipped no cambia. Pero como este es el último candidato posible para el título "esquema de máscara", es posible utilizar tal comportamiento. Es decir, verificamos que después de ejecutar el clipped Convert To Area Type , la propiedad clipped sigue siendo true . A continuación se muestra el código para un TextFrameItem de tipo PATHTEXT. 


  if (clipPath == null) { for (var i = 0; i < clipGroup; i++) { if (sel[0].pageItems[i].typename == 'TextFrame' && sel[0].pageItems[i].kind == 'TextType.PATHTEXT') { sel[0].pageItems[i].convertPointObjectToAreaObject(); app.redraw(); if (sel[0].clipped == true) { clipPath = sel[0].pageItems[i]; break; } else { app.undo(); } }; }; };
