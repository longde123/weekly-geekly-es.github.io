Secara singkat tentang WSL


Dengan pembaruan Windows, menjadi mungkin untuk menggunakan hal seperti WSL (Windows Subsystem untuk Linux). Belum lama ini, WSL2 muncul dengan modifikasi SANGAT keren. WSL2 menggunakan teknologi virtualisasi terbaru dan terbaru untuk menjalankan kernel Linux di dalam mesin virtual utilitas yang disederhanakan. Ini berarti bahwa atribut seperti isolasi dan perlambatan hilang.

Pengembangan


Sekarang kami melanjutkan langsung ke pengembangan. Saat memilih editor, saya akan tetap menggunakan Visual Studio Code . Ini mencakup banyak alat, termasuk debugger, yang menjadikannya IDE yang lengkap. Menciptakan lingkungan virtual pada sistem yang berbeda berbeda. Misalnya, setelah menerapkan perintah 

 python -m venv venv

a .bat aktivator dan file python.exe dibuat di Windows, yang akan digunakan saat Linux menyalin sistem biner python. VS Code cukup fleksibel dengan ekstensi.

Ctrl + Shift + X
akan memungkinkan Anda untuk dengan cepat beralih ke add-on. Dalam pencarian, masukkan WSL dan unduh ekstensi pertama dari daftar.



Untuk pengembangan yang nyaman, perlu menambahkan Python, Git History, Django ... Dan kemudian apa yang diinginkan jiwa.







Setelah itu, nyalakan ulang editor dengan berani, setelah itu Anda dapat membuka proyek Anda
File -> Buka Folder
Setelah memuat, perhatikan sudut kiri bawah layar.



Ini akan membuka kembali folder menggunakan WSL



Perbedaannya adalah bahwa VS Code dalam hal ini beralih ke kernel Linux, yang merupakan lapisan bawah dari kernel Windows. Ini akan memungkinkan Anda untuk menggunakan juru bahasa yang diinginkan, serta terminal langsung dari editor.





Anda juga dapat menggunakan Terminal Windows. Ini akan secara signifikan meningkatkan efisiensi kerja, karena WSL juga terintegrasi ke dalamnya.





Secara kompleks, semua ini memastikan pekerjaan yang nyaman di Windows dengan kemampuan Linux. Dan juga faktor utama adalah bahwa lingkungan kerja memiliki akses ke sistem utama, tetapi diisolasi darinya oleh kernel dan sistem file dasar.
