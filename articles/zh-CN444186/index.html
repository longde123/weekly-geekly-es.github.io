颤振1.2。 最新消息

2月26日，Flutter团队宣布了第一个稳定的产品版本 Flutter 1.2。 对新消息感兴趣的每个人都带来了更新-我要猫。



该发行版中的主要更改可以简要描述为三点：

  • 提高稳定性，提高框架和插件的速度和质量；
  • 改善现有小部件的显示和功能；
  • 一种新的基于Web的工具，供开发人员在Flutter上创建应用程序。

此处可以查看所有以发布请求形式发布的更改。

提高稳定性，提高框架核心的速度和质量


  1. 将Dart SDK更新到2.2版。 根据开发人员的说法，新版本将性能提高了11-16％，但代价是将应用程序的大小增加了1％。 更多细节在这里 。
  2. 视频播放器插件中的错误修复。 此外，在Android的幕后，他们从支持库迁移到AndroidX。
  3. Webview插件中的错误修复。 修复了内存泄漏和小错误。
  4. 谷歌地图插件中的错误修复。 修复了iOS中图标的显示。 Android已将play-services-maps更新为16.1.0。
  5. 添加了对Android版AppBundle的支持。
  6. 正在开发用于应用内购买的插件 （但您已经可以感觉到了）。

改善现有小部件的显示和功能


  1. 添加了对使用浮动光标编辑文本的支持。
  2. 增加了对更广泛的动画减速功能集的支持。
  3. 添加了新的键盘事件（未特别提及）和鼠标悬停处理，以为对桌面类操作系统的更深入支持做准备。 显然，无所不在的Fuchsia OS已经关闭，并将支持台式机。
  4. 改进了Material和Cupertino小部件的工作和渲染。

开发人员在Flutter上创建应用程序的新Web工具


引入了一个新的分析和调试工具： DevTools 。 它可用于Android Studio ， VS Code和命令行 。 以下是其主要功能列表：

  • 小部件检查器 。 将代码中的小部件层次结构呈现为小部件树。
  • 时间轴视图 。 一个可帮助您分析动画期间的CPU，GPU和延迟的时间线。
  • 源代码级调试器 。 代码分析工具。 允许您在断点上调试应用程序并分析调用堆栈。
  • 记录视图 。 我认为这是显而易见的。 系统和用户都可以在其中写入日志事件的窗口。



接下来是什么？


Flutter团队制定了2019年计划。 你可以在这里看到它 。 全球目标有两个：

  • 项目代号Humminbird 。 目的是增加对Flutter中Web开发的支持。
  • Flutter桌面嵌入项目 。 目的是增加对Flutter的支持，以开发Windows，Mac和Linux等台式机类操作系统。

Flutter Create比赛：在Dart上5KB可以做什么？


Flutter团队还宣布了比赛 。 其本质是您需要在Flutter上编写最酷，最漂亮的应用程序。 但是同时您有一个限制：该代码占用的空间不得超过5KB。 配备齐全的iMac Pro带来了无数奖项。 比赛持续到4月7日，优胜者将在Google I / O上宣布。


结论


Flutter变得越来越流行，并且已经进入Github上排名前20位的软件存储库。 他的团队甚至启动了他们的YouTube频道 。 社区每个月都在扩大，工具也在不断完善。 是否使用它是您的选择。 但是它的发展方式确实令人鼓舞。 阅读您对评论中的创新的看法将很有趣。 这就是我的全部。 希望Google不会破坏您的Play商店，也不能破坏您的商店！
