Memperkenalkan .NET Core 3.1 Pratinjau 2

Memperkenalkan .NET Core 3.1 Pratinjau 2 . .NET Core 3.1 akan menjadi rilis kecil yang berfokus pada peningkatan utama untuk Blazor dan desktop Windows, dua rilis besar baru dari .NET Core 3.0. Ini akan menjadi rilis dukungan jangka panjang (LTS) dengan tanggal peluncuran final yang diharapkan pada bulan Desember 2019.

Anda dapat mengunduh .NET Core 3.1 Preview 2 untuk Windows, macOS, dan Linux.

  • .NET Core 3.1 Pratinjau 2 SDK dan Runtime
  • Gambar Docker

ASP.NET Core dan EF Core sekarang juga tersedia.

Visual Studio 16.4 Pratinjau 3 dan Visual Studio untuk Mac 8.4 Pratinjau 3 sekarang juga tersedia. Ini adalah pembaruan yang diperlukan untuk menggunakan .NET Core 3.1 Pratinjau 2. Visual Studio 16.4 termasuk .NET Core 3.1, sehingga pembaruan Visual Studio sederhana akan menginstal Anda berdua.

Detail:

  • .NET Core 3.1 Catatan Rilis
  • Catatan Rilis GitHub



Perangkat tambahan


Peningkatan terbesar dalam rilis ini adalah dukungan untuk C ++ / CLI (AKA "managed C ++"). C ++ / CLI hanya tersedia di Windows. Perubahan untuk C ++ / CLI dibuat terutama di Visual Studio. Untuk menggunakan C ++ / CLI, Anda perlu mengatur beban kerja "Pengembangan Desktop di C ++" dan komponen "Dukungan C ++ / CLI". Anda dapat melihat komponen yang dipilih ini (terakhir ditampilkan) pada gambar di bawah ini.



Komponen ini menambahkan beberapa pola yang dapat Anda gunakan:

  • Perpustakaan Kelas CLR (.NET Core)
  • Proyek Kosong CLR (.NET Core)

Jika Anda tidak dapat menemukannya, cukup temukan di kotak dialog Proyek Baru.

Kesimpulan


Tujuan utama .NET Core 3.1 adalah untuk meningkatkan fitur dan skrip yang kami perkenalkan di .NET Core 3.0. .NET Core 3.1 akan menjadi rilis dukungan jangka panjang (LTS) , didukung setidaknya selama 3 tahun.

Silakan instal dan uji .NET Core 3.1 Pratinjau 2 dan tinggalkan ulasan Anda. Harap dicatat bahwa versi ini tidak direkomendasikan untuk digunakan dalam produksi.

Jika Anda tiba-tiba melewatkannya, baca pengumuman .NET Core 3.0 .
