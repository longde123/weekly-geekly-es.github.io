DataGrip 2019.2: Gestion des connexions, recherche de données, filtrage dans la navigation

Salut Nous parlons de ce que nous avons fait dans DataGrip en quatre mois. Si vous utilisez le support de base de données dans nos autres IDE, ce message est aussi pour vous.



Boîte à outils des services


La plateforme IntelliJ IDE a une nouvelle fenêtre d'outils: Services. Dans DataGrip, les connexions sont affichées ici.



Chaque nœud d'arborescence, qu'il s'agisse d'un service, d'une source de données, d'une connexion, peut être ouvert dans un onglet distinct: cliquez sur Afficher dans le nouvel onglet de la barre d'outils. Ou utilisez le glisser-déposer. Si un cercle vert est allumé sur la connexion, cela signifie qu'elle est active. Vous pouvez fermer la connexion à partir du menu contextuel.

Les résultats de la requête sont attachés aux connexions correspondantes. Si vous souhaitez retourner la vue «comme avant», désactivez l'arborescence des services dans les paramètres d'affichage en cliquant sur l'icône d'engrenage.



Minuteur de demande


Dans la fenêtre Services , ils ont ajouté la possibilité de voir combien de temps la demande prend: l'horloge tourne à droite de la console à partir de laquelle elle s'exécute.

Docker


Un autre service qui peut apparaître dans cette fenêtre est Docker, si vous utilisez le plugin approprié.



Le raccourci clavier pour afficher / masquer la fenêtre Services est Alt / Cmd + 8 .

Recherche de données


Recherchez les données, même si vous ne vous souvenez pas où elles sont stockées. Sélectionnez une source de données, un groupe de sources ou, inversement, uniquement des tables individuelles, puis appuyez sur Ctrl / Cmd + Alt + Maj + F. Ou sélectionnez Recherche plein texte dans le menu contextuel.



Dans la boîte de dialogue, entrez les éléments à rechercher. À gauche, une liste de sources de données. Le bas de la fenêtre affiche les requêtes que DataGrip exécutera pour la recherche.



Les résultats ressemblent à ceci:



En cliquant sur le résultat, l'éditeur de données s'ouvrira, dans lequel un filtre sera appliqué pour n'afficher que les lignes trouvées. S'il y a trop de colonnes et qu'il n'est pas clair où se trouve ce que vous avez trouvé, utilisez la recherche de texte: Ctrl / Cmd + F.



Détails:
- Dans PostgreSQL , MySQL et MariaDB, vous ne pouvez rechercher que par colonnes indexées. Pour ce faire, sélectionnez « Uniquement les colonnes avec des index de recherche en texte intégral » dans la liste «Rechercher dans» .
PostgreSQL aura cette requête: where col @@ plainto_tsquery('query')
Dans MySQL et MariaDB, c'est: where match(col) against ('query' in natural language mode)
- Dans Oracle , DataGrip utilisera le contexte, les index ctxrule, ctxcat, le cas échéant.
- Dans SQL Server, si des index de texte intégral sont créés, DataGrip génère une requête avec la clause ` WHERE CONTAINS (col, N'text ') `.
- La recherche recherche dans les colonnes pour lesquelles LIKE n'est pas pris en charge. Par exemple, les colonnes JSON seront pré-converties en chaîne.
- Pour Cassandra, DataGrip exécute plusieurs requêtes pour une seule table car l'instruction OR n'est pas prise en charge.

Editeur de données


Redimensionnement rapide des pages


Modifiez la taille de la page dans l'éditeur de données sans accéder aux paramètres.



Noms des onglets


Vous pouvez maintenant nommer vous-même les onglets des résultats: écrivez un nom dans les commentaires avant la demande.



Si vous n'aimez pas que tout commentaire précédent devienne un nom, indiquez le mot suivi d'une ligne pour le titre. Cela se fait dans les paramètres appropriés.



Explorateur de bases de données


Tables de sauvegarde rapides


Vous pouvez copier des tableaux en utilisant le glisser-déposer pendant une longue période, mais avant cela ne fonctionnait pas si vous le faites glisser dans le même schéma. Maintenant ça marche! C'est pratique si vous sauvegardez des données avant une opération importante.



Création rapide de groupe


Créez des groupes de sources de données en faisant glisser une source vers une autre. Si vous devez déplacer la source de données vers un groupe existant, cela se fait également avec la souris. Nous rappelons aux amateurs de clavier qu'il existe une touche F6 pour cela.



Mise à jour forcée


La nouvelle action Forcer l'actualisation chargera des informations sur tous les objets de la base de données, après avoir préalablement vidé le cache. Cela peut être utile si quelque chose est «coincé» dans votre mémoire et que vous ne voyez pas l'image réelle.



Composés actifs


À partir de cette version, un petit cercle vert sur l'icône de la source de données signifie que la connexion à celle-ci est active.



Commentaires sur les tableaux


Après avoir activé la vue | Apparence | Les descriptions dans les commentaires du tableau des vues arborescentes sont affichées dans l'Explorateur.



Filtrage des sources de données


Si vous avez de nombreuses bases de données similaires, il n'était pas pratique de rechercher des objets: par exemple, si vous recherchez une table et que vous en avez dix, elles apparaissent toutes dans la sortie.
Vous pouvez maintenant spécifier où chercher: sélectionnez une source de données ou un groupe de sources spécifique.



La même chose fonctionne lors de la recherche de code dans Find In Path .



Aide à l'écriture de code


Objets système


Les bases de données ont des catalogues système - les métadonnées y sont stockées: informations sur les tables, les procédures, les fonctions intégrées.

Les objets de ces répertoires sont souvent utilisés dans les requêtes, ils doivent donc être affichés en auto-complétion et correctement mis en évidence: l'IDE ne doit pas penser que ce sont des objets inconnus.

Auparavant, pour que cela fonctionne, des circuits système devaient être ajoutés à l'explorateur, car DataGrip ne connaissait que les objets qui étaient inspectés. Mais les circuits du système ne changent pas; obtenir des informations à leur sujet à chaque fois n'a pas de sens. De plus, ils ne sont pas toujours nécessaires dans le conducteur et interfèrent avec les plans de travail là-bas.

L'éclair devant le diagramme signifie que DataGrip a toujours des informations sur ses objets, même si vous ne les avez pas ajoutées à l'explorateur.



Voici quelques exemples de schémas système:
PostgreSQL : pg_catalog, information_schema
SQL Server : INFORMATION_SCHEMA
Oracle : SYS, SYSTEM
MySQL : information_schema
DB2 : SYSCAT, SYSFUN, SYSIBM, SYSIBMADM, SYSPROC, SYSPUBLIC, SYSSTAT, SYSTOOLS

Avertissements et corrections


À partir de la fenêtre d'avertissement, vous pouvez maintenant résoudre immédiatement le problème dont vous avez été informé. Par exemple, dans une fenêtre de faute de frappe, DataGrip propose de le corriger et si l'EDI avertit qu'il n'y a pas d'alias, vous êtes invité à le saisir. Pour ce faire, cliquez sur l'action en bas à gauche ou appuyez sur Alt + Maj + Entrée .


Alt + Entrée fonctionne toujours et affiche une liste de toutes les actions possibles dans ce contexte.

Nouveaux paramètres pour le mouvement du chariot


Nous avons modifié le comportement de transport par défaut. DataGrip déplace maintenant le chariot à la fin du mot actuel si vous sautez par-dessus les mots. Ce comportement est configuré dans les paramètres : Editeur | Général



En règle générale, cette action est effectuée par des flèches lorsque vous appuyez sur Ctrl pour Windows / Linux et sur Opt pour MacOS. Le comportement par défaut dans différents systèmes d'exploitation est différent: maintenant dans notre IDE, cela fonctionne «comme sur un Mac».

En savoir plus


  • DataGrip 2019.2 utilise JetBrains Runtime 11, fork non certifié d'OpenJDK 11.
  • [Cassandra] Vous pouvez maintenant éditer des colonnes des types suivants: set, list, map, tuple, udt, inet, uuid , timeuuid.
  • Nouveaux éléments dans la liste de saisie semi-automatique IS NULL et IS NOT NULL. Auparavant, les comparaisons avec NULL devaient être saisies séparément.
  • Sauter en dehors du crochet de fermeture / devis avec Tab est activé par défaut. En appuyant sur la touche Tab, le curseur se déplace vers l'extérieur du bloc de guillemets et de parenthèses.
  • Entourer une sélection d'un guillemet ou d'une accolade est activé par défaut. Si vous tapez un guillemet ou un crochet avec un fragment de code sélectionné, le fragment apparaîtra entre guillemets ou crochets.
  • L'action Introduire l'alias fait désormais partie du menu Refactoring.
  • Configurer le travail avec PostgreSQL 12. Plus d'informations ici: DBE-8384 .
  • Logique ajoutée au mode lecture seule . Auparavant, ce paramètre comprenait deux modes à la fois: au niveau de l'EDI (analyses DataGrip qui demande de modifier les données) et au niveau du pilote (s'il est implémenté). Dans l'EDI, nous avons ajouté la possibilité d'exécuter une demande après un avertissement, si nécessaire. Mais auparavant, le mode lecture seule bloquait toujours de telles demandes au niveau du pilote: DBE-8145 . Maintenant ne bloque pas.
