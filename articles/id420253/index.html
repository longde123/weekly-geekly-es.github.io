Bagaimana API perbankan terbuka mengubah dunia keuangan

Industri keuangan sedang mengalami revolusi digital. Bank konservatif mengikuti tren modern dan mulai memberikan informasi kepada pihak ketiga yang sebelumnya dianggap sebagai kerahasiaan perbankan. Mengapa ini terjadi, kepada siapa dan mengapa itu diperlukan - kita akan mengerti dalam artikel ini.

Pengungkapan Bank


Kecenderungan berasal di Eropa, misalnya, di Jerman, Proyek Open Bank telah berkembang sejak 2010, sebuah proyek yang mendukung pengungkapan data perbankan dan menikmati dukungan bank-bank terbesar di negara ini.

Di Inggris, pada bulan September 2015, dengan dukungan otoritas publik, inisiatif Open Banking Standard diluncurkan, yang bertujuan untuk meningkatkan persaingan dan akses ke layanan di pasar keuangan. Menurut inisiatif tersebut, bank harus memberi pihak ketiga (yang disebut perusahaan keuangan dan teknis) informasi tentang saldo pelanggan dan akses ke rekening mereka saat ini. Penerapan prinsip data perbankan terbuka telah menjadi keharusan bagi 9 bank besar di Inggris, yang melayani lebih dari 80% warga negara.

Menggunakan Teknologi API


Informasi ditransmisikan dan disediakan menggunakan API atau Antarmuka Pemrograman Aplikasi, yang dalam bahasa Rusia berarti "antarmuka pemrograman aplikasi". Dengan kata sederhana - ini adalah daftar perintah, pertanyaan, jawaban, yang dengannya program komputer saling bertukar informasi dan berinteraksi, "memaksa" satu sama lain untuk melakukan tindakan apa pun.

Contoh penggunaan API adalah aplikasi mobile banking yang memungkinkan klien untuk memeriksa saldo mereka, melakukan pembayaran dan melakukan operasi perbankan lainnya langsung dari perangkat mobile mereka.

Peluang dan manfaat API perbankan terbuka


Tren yang mulai populer di dunia adalah menyediakan API perbankan untuk semua organisasi yang berencana bekerja di sektor keuangan. Pendekatan ini membuka sejumlah keuntungan bagi bank dan konsumen.

Misalnya, menjadi mungkin untuk membuat aplikasi seluler universal yang dengannya pengguna dapat melihat informasi untuk setiap bank yang menjadi klien mereka. Sebelumnya, untuk setiap bank, pengguna membutuhkan aplikasi terpisah, yang mempersulit pilihan penawaran dan layanan yang relevan yang disediakan oleh bank.

Aplikasi multi-perbankan memungkinkan Anda untuk:
  • Lihat saldo Anda secara online dan kelola akun Anda dengan mudah di berbagai bank.
  • Sederhanakan operasi perdagangan (memilih bank untuk pembayaran, pemeriksaan kredit cepat, melakukan pembayaran dari aplikasi).
  • Menerapkan aplikasi hipotek / kredit yang dapat mengkonfirmasi solvabilitas warga dari perangkat seluler, menyederhanakan proses memperoleh hipotek / pinjaman.

Akibatnya, pelanggan tidak perlu menggunakan berbagai saluran komunikasi dan aplikasi untuk operasi perbankan rutin, perbankan menjadi nyaman, visual dan nyaman.

Akses terbuka ke API perbankan bermanfaat bagi bank:

  • Bank akan memperluas saluran distribusi produk dan layanan keuangan mereka dengan bantuan organisasi pihak ketiga dan sistem informasi.
  • Perluasan saluran distribusi - potensi untuk meningkatkan basis klien bank.
  • Bank akan memiliki akses ke informasi dan layanan dari organisasi pihak ketiga, yang akan memberikan kesempatan untuk lebih menavigasi tren yang muncul di bidang jasa keuangan.
  • Dan akhirnya, API terbuka akan memberikan kesempatan untuk memantau pasar baru dari dalam dan mungkin mengalahkan organisasi di balik perkembangan paling sukses di bidang ini untuk memperkuat posisi pasarnya sendiri.

Seperti yang Anda lihat, keputusan tentang API perbankan terbuka membawa manfaat bagi semua pihak dalam proses ini. Sekarang kita akan mempertimbangkan bagaimana konsep ini diterapkan di berbagai negara di dunia.

Pengalaman menggunakan API perbankan terbuka


Di negara-negara Uni Eropa, sejak Januari 2018, standar PSD 2 (Petunjuk Pembayaran Eropa) telah berlaku, yang mewajibkan bank untuk menyediakan basis data klien dan antarmuka program (API) untuk 3 orang yang berencana bekerja di sektor keuangan. Dengan demikian, direncanakan untuk meningkatkan persaingan di pasar pembayaran online seluler dengan memasukkan perusahaan teknologi di pasar.

Di Amerika Serikat, tidak ada peraturan pemerintah tentang bank API terbuka. Namun demikian, situasi keuangan yang menguntungkan menyebabkan pembentukan agregator bank Mint, situs yang sudah pada tahun 2016 digunakan oleh sekitar 20 juta penduduk Amerika Serikat dan Kanada.

Di Singapura, Otoritas Moneter (MAS) mendukung prinsip-prinsip API perbankan terbuka. Asosiasi Bank Singapura (ABS) dan MAS telah menyiapkan dokumen Keuangan-sebagai-Layanan: API Playbook yang berisi informasi dan rekomendasi untuk sektor keuangan, yang dirancang untuk memastikan pertukaran informasi keuangan yang efektif, serta menciptakan landasan untuk meluncurkan proyek-proyek perbankan yang inovatif.

Platform informasi API terbuka India Stack dibuat di India, yang mencakup sistem dan layanan seperti: Aadhaar - sistem identifikasi biometrik digital terbesar di dunia; Perusahaan Pembayaran Nasional India; Digital locker - platform untuk pembentukan dan verifikasi dokumen dalam bentuk digital, dll.

IndiaStack memungkinkan pengembang dan satraps teknologi untuk memasuki pasar aplikasi seluler untuk mengidentifikasi dan mengotentikasi pengguna layanan keuangan.

Pengalaman menggunakan API perbankan terbuka di Rusia


Di Rusia, pada tahun 2016, atas prakarsa Sberbank, Alfa Bank dan perusahaan keuangan lainnya, organisasi FinTech diciptakan dengan tujuan mengembangkan teknologi keuangan di negara tersebut. Juga di Rusia, FintechSmart sedang mengembangkan - serikat asosiasi yang mengembangkan lingkungan untuk startup finansial dan komunitas RusFinTech, yang menyelenggarakan seminar dan konferensi tentang topik teknologi keuangan.

Berbicara tentang prospek pengembangan API perbankan terbuka di Rusia, perlu disebutkan inisiatif yang diluncurkan oleh Bank Rusia bersama dengan organisasi fintech.

Arah ini melibatkan pengembangan skenario aplikasi API terbuka, proyek percontohan, serta pengembangan standar dan dokumen tentang penggunaan API terbuka di Rusia. Sudah, bank-bank terbesar di negara itu, seperti Sberbank, Alfa-Bank, VTB 24, Gazprombank dan Otkritie Bank, siap untuk menyediakan API mereka ke organisasi pihak ketiga.

Tren pengembangan API perbankan terbuka menunjukkan bahwa arah pengembangan aplikasi mobile untuk bisnis organisasi fintech, yaitu, aplikasi multi-perbankan di Rusia, akan menjadi relevan dalam waktu dekat. Menurut agen riset Markswebb, 18 juta orang menggunakan bank seluler di Federasi Rusia, yang merupakan sekitar 33% dari total pemirsa internet di negara itu. Setidaknya satu bank Internet digunakan oleh 35,3 juta orang, atau 64,5% dari semua pengguna Internet Rusia. Usia pengguna internet dan mobile banking berkisar antara 18 hingga 64 tahun.

Pelanggan Wellsoft dari sektor keuangan sudah menunjukkan minat di bidang ini dan sedang menjajaki kemungkinan mengembangkan aplikasi multi-bank menggunakan API terbuka.

Pengalaman Wellsoft dalam mengembangkan aplikasi seluler untuk bank memungkinkan penerapan solusi tersebut saat ini, jika klien menginginkannya.
