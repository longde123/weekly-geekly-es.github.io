电报封锁触发了国内初创企业的成本上升


图片来源：Kommersant

对于一些国内软件开发商而言，云服务的成本已经增加了一倍甚至更多。 原因是Roskomnadzor进行了“地毯式轰炸”，是针对俄罗斯整个IP地址池进行的。 至于费用的增加，该问题主要影响到仅被迫改用俄罗斯类似Google Cloud和Amazon的小型初创公司。 但是大中型开发公司并未感到重大变化。

电报阻塞导致云服务成本增加了100％，因此，家用软件的成本也增加了。 这是由Equio Alexey Vagin公司总经理向 Kommersant 报告的 。 他说，在小型公司中，那些在国外但为俄罗斯开发云产品的公司遭受的损失最大。 在封锁Messenger之前，这些公司仅使用俄罗斯云服务来存储客户的个人数据，一般信息已发布在诸如Google Cloud和Amazon Web Services之类的国外服务的服务器上。

4月16日之后，由于外国云服务的许多IP地址在俄罗斯联邦境内被封禁，一些IT公司发现自己处境不利。 开发人员别无选择，只能转向使用更昂贵的国内“云”来存储其客户的数据。 “例如，对于在Google Cloud Platform上拥有3万至4万活跃用户的云而言，一种非常简单的服务器配置要花费330美元，即约21000卢布。 每月，而在俄罗斯供应商上进行类似配置将花费6.4万卢布。 每月”，Vagin说。

据瓦金说，与“信使”封锁有关的“平等”费用至少增加了300万卢布。 每年。 Teachbase首席执行官弗拉基米尔·谢尔巴科夫（Vladimir Shcherbakov）表示，他的业务已经遭受了至少100万卢布的财务损失。 同时，不基于云服务的创业公司已经绕过了“价格上涨”的问题。

该公司Hot-WiFi的代表表示，鉴于目前的情况，他们还不得不从国外的数字海洋公司转用俄罗斯Selectel公司，一次性支付了30万卢布。 购买云服务器，额外的员工时间和搬迁本身。 一个半月后，服务器成本没有增加。 顺便说一下，尽管国外云服务的许多IP地址已经被解除封锁，但由于担心重复锁定，其用户并不急于返回。

大中型企业生存的秘密在于，这类公司最初更喜欢只使用其服务器。 仅在极端情况下，他们才决定开始使用国外或国内的第三方服务。 Trueconf开发总监Dmitry Odintsov讲述了这一点。 据他介绍，这样的开发人员使用内部系统进行通信，因此信息泄漏的风险要低得多，并且阻止Telegram的副作用实际上不会影响它们。

大型软件开发者协会Russoft也表示，它不受锁的影响。 “我们从一些公司那里收到了有关封锁后果的信息，但他们拒绝讨论细节。 此外，封锁更是一个政治问题。 我们是一个为会员利益着想的国家行业协会，而不是一个政治组织。

Flexbby（斯科尔科沃居民）甚至早些时候估计俄罗斯公司阻止电报的潜在损失为10亿美元，亚马逊和谷歌约为9.4亿美元。据该公司称，截至2018年4月，俄罗斯有400多家公司遭受了中断。数据仓库，业务应用程序，人工智能系统，支付，客户服务等的运作。 所有这些都可以称为锁的“副作用”。 到4月底，超过1800万个与Telegram无关的IP地址被阻止。 到6月中旬，他们的人数下降到379万。
