Raspberry Pi y SIM7600E 4G HAT Modem

Buen dia a todos. En este artículo, me gustaría hablar sobre mi experiencia con el módem 3G SIM7600X, que se puede conectar a la Raspberry Pi. Este artículo puede ser útil para aquellos de ustedes que necesitan usar Raspberry de forma autónoma, en ausencia de una red WIFI.

Parámetros del sistema


Raspberry Pi 3B +
Sistema operativo: Raspbian GNU / Linux 10 (buster)
Módem SIM7600E 4G HAT, versión de firmware LE11B08SIM7600M22



Conectar un módem a Raspberry


El módem SIM7600E 4G HAT actualmente cuesta alrededor de $ 80. Puedes pedirlo aquí .

El módem tiene una ranura de 40 pines a través de la cual se conecta a la Raspberry. Hay una ranura para tarjeta SIM en la parte posterior de la placa de módem. Usé la tarjeta SIM del megáfono.



Tenga en cuenta que cuando enciende el módem, deben encenderse dos indicadores LED: PWR y NET.



La documentación dice que los comandos AT para controlar el módem deben transmitirse a través de su puerto USB. Por lo tanto, necesitamos un cable separado para conectar este puerto al puerto USB de Raspberry.



Ahora intentemos conectarnos al módem usando minicom . Puede usar los archivos del dispositivo / dev / ttyUSB2 o / dev / ttyS3 . 

sudo apt-get install minicom sudo minicom -D /dev/ttyUSB2

Asegúrese de que el módem responda adecuadamente a los comandos AT. Por ejemplo, ¿usando el AT + CPIN? Puede verificar si el módem ve la tarjeta SIM. 

 AT OK AT+CPIN? +CPIN: READY

Obteniendo coordenadas GPS.


El módem SIM7600 está equipado con un receptor GPS. Para usarlo, primero conecte la antena GPS externa a la toma GNSS. Después de eso, llevaremos la antena a la zona de recepción de señal GPS confiable (es decir, al menos la arrojaremos por la ventana, ya que el GPS no está atrapado en edificios de concreto).



Para encender el receptor GPS, ejecute el comando: 

 AT+CGPS=1 OK

Verifique que el receptor haya comenzado: 

 AT+CGPS? +CGPS: 1,1

Intentemos obtener las coordenadas GPS: 

 AT+CGPSINFO +CGPSINFO: ,,,,,,,,

Puede ser necesario repetir el último comando varias veces hasta que el receptor "atrape satélites": 

 AT+CGPSINFO +CGPSINFO: ,,,,,,,, AT+CGPSINFO +CGPSINFO: 2232.43534,N,11404.325345,E,300618, 0855320.0,96.0,0.0,0.0

Conexión de un módem a una red GSM y una red 3G.


Para acceder a la red, es conveniente utilizar la utilidad qmcli lanzada recientemente. Muchas gracias a la persona con el apodo mkrzysztofowicz, por una descripción detallada de este método . (Antes de eso, tenía que usar las utilidades sakis3g y umtskeeper , que no eran del todo confiables ni demasiado convenientes). Para comenzar, instale los paquetes necesarios: 

 sudo apt-get update sudo apt-get install libqmi-utils udhcpc

El módulo de radio del módem SIM7600 está apagado de forma predeterminada. Enciéndelo: 

 sudo qmicli -d /dev/cdc-wdm0 --dms-set-operating-mode='online'

El indicador NET del módem debería comenzar a parpadear. Esto significa que el módem ha cambiado para buscar y conectarse a una red GSM. Verifique que se encuentre la red y que el nivel de señal no sea demasiado bajo: 

 sudo qmicli -d /dev/cdc-wdm0 --dms-get-operating-mode sudo qmicli -d /dev/cdc-wdm0 --nas-get-signal-strength sudo qmicli -d /dev/cdc-wdm0 --nas-get-home-network

El controlador qmi-wwan, al ver el módem SIM7600 conectado a la Raspberry, crea la interfaz de red wwan0. Por defecto, 802-3 está instalado como el protocolo en esta interfaz, sin embargo, el protocolo que necesitamos es aparentemente raw-ip . Hagamos este cambio: 

 sudo qmicli -d /dev/cdc-wdm0 -w #    ,   wwan0 sudo ip link set wwan0 down echo 'Y' | sudo tee /sys/class/net/wwan0/qmi/raw_ip sudo ip link set wwan0 up

Ahora conéctese a la red 3G. Puede encontrar los parámetros para APN en la página de su proveedor de GSM. 

 sudo qmicli -p -d /dev/cdc-wdm0 --device-open-net='net-raw-ip|net-no-qos-header' --wds-start-network="apn='<YOUR_APN>',username='<YOUR_USERNAME>',password='<YOUR_PASSWORD>',ip-type=4" --client-no-release-cid

Por ejemplo, en el caso de Megaphone: 

 apn='internet',username='gdata',password='gdata'

Finalmente, obtenga la dirección IP: 

 sudo udhcpc -i wwan0

Con el comando ifconfig, puede ver que la interfaz wwan0 ahora tiene asignada una dirección IP específica. Si es así, felicidades, ¡todo salió bien! 

 ifconfig wwan0: flags=4305<UP,POINTOPOINT,RUNNING,NOARP,MULTICAST> mtu 1500 inet 10.245.223.144 netmask 255.255.255.224 destination 10.245.223.144

Eso es todo! Les deseo todo el éxito en nuestro difícil negocio.
