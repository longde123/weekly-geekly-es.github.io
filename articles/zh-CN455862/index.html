AWS的新增功能：DATA API，Kinesis Data Analytics，S3 Path

适用于Amazon Aurora Serverless的DATA API


连接到传统数据库如何工作？ 您打开一个连接，使用它来处理一个或多个SQL查询或其他语句，然后关闭该连接。 您可能使用了特定于您的操作系统，编程语言和数据库的客户端库。 在某个时候，您意识到创建连接要花费大量时间并占用数据库引擎上的内存。

DATA API负责管理和扩展长期数据库连接，并以JSON形式返回数据以便于分析。 所有流量都是加密的，并且通过HTTPS发生。

那么什么是DATA API？

ExecuteStatement-执行一个SQL查询。
BatchExecuteStatement-对数据数组执行一个SQL查询。
BeginTransaction-开始交易并返回交易标识符。
CommitTransaction-完成事务并提交在其中执行的SQL操作。
RollbackTransaction-回滚事务。

唯一的限制是每个功能必须在一分钟内完成，并且最多可以返回1兆字节的数据。

Amazon Kinesis Data Analytics现在允许您将AWS资源标签分配给实时应用程序


随着数据流的增加，以特殊方式处理和分析此数据的需求增加了。 Amazon Kinesis Data Analytics是其中一项服务，使您能够通过编写SQL或Java代码（使用Apache Flink）来开始从不同来源收集数据，并进行实时处理和分析。 现在，您已经收到了更为完整和多维的数据图，您可以轻松提取新的有用信息，并快速响应业务和客户的需求。

随着此类应用程序数量的增加，有效管理它们的需求也在增长。 亚马逊最近宣布，现在可以像其他AWS资产一样，将资产标签添加到Amazon Kinesis Data Analytics中的应用程序中。 标签是键值对，其中值是可选部分。 使用标签是管理AWS资源和组织数据的简单但有效的方法。 可能的标签示例：

环境：暂存
应用程序：应用程序名称
项目：项目名称

使用标签，您可以向应用程序添加更多上下文。 对于Amazon Kinesis Data Analytics应用程序，可以使用标签：
确定特定Amazon Kinesis Data Analytics应用程序的账单。
控制对应用程序资源的访问。
出于用户自己定义的目的-您可以根据用户标签的存在来确定应用程序的功能。
您可以在此处找到有关Amazon Kinesis Data Analytics应用程序标签的更多信息。

Amazon S3路径弃用计划-故事的其余部分


在Amazon Web Services（AWS）最受欢迎的S3服务之一中，通过指定路径样式来获取文件的方法很快就会过时，例如s3.amazonaws.com/usmanovbf/docs/Bulat_Usmanov_CV.pdf或带有区域- s3-us-east-2.amazonaws.com/usmanovbf/docs/Bulat_Usmanov_CV.pdf ，其中usmanovbf是存储桶，/ docs / Bulat_Usmanov_CV.pdf是此存储桶中文件的密钥。

将优先考虑另一种方法-使用虚拟主机（虚拟托管样式），例如usmanovbf.s3.amazonaws.com/docs/Bulat_Usmanov_CV.pdf或区域-usmanovbf.s3-us-east-2.amazonaws。 com / docs / Bulat_Usmanov_CV.pdf

过渡预计将于2020年9月30日完成。 由于AWS工程师对社区开放，因此直到该日期（包括该日期）之前创建的所有路径样式链接都不会被删除。 也就是说，它们将保持完全运行，在此日期之后，您只能使用第二种方法-虚拟主机样式。

此更改是由两个主要原因决定的：
1.更加集中的模型（使用路径样式访问文件）限制了有效缩放，因为所有流量都流向一小部分输入点：第三级s3，s3-us-east-2，s3-us-east的子域-1等。 出于同样的原因，使用DNS名称解析，安全性和DDOS攻击反射功能变得很不方便。
虚拟主机样式由于绑定到单独的子域存储桶和区域（例如上例中的usmanovbf和s3-us-east-2）而使您可以更灵活地控制数据流。

2. AWS工程师不会停滞不前，为了引入新功能并放弃旧功能，他们需要切换到虚拟托管样式。 例如，计划放弃旧的加密方法。

您可以在官方的AWS博客上找到有关迁移到新方法的更多信息。
