Einführung in 3CX V16 Update 4 und Unified FQDN 3CX WebMeeting

Kurz vor den Feiertagen haben wir das kommende 3CX V16 Update 4 veröffentlicht! Wir haben auch einen neuen universellen Namen MCU 3CX WebMeeting und neue Speichertypen für Backups und 3CX-Unterhaltungsaufzeichnungen. Betrachten wir alles in der richtigen Reihenfolge.

3CX V16 Update 4


Das nächste 3CX-Update bietet eine Auswahl an Audiogeräten im Webclient, die endgültige Veröffentlichung der 3CX-Erweiterung für Chrome und neue Arten von Backup-Speichern. Darüber hinaus erhielt Update 4 eine Reihe von Stabilitäts- und Sicherheitsverbesserungen, die von Entwicklern während der Testphase vorgenommen wurden.

Wie Sie wissen , erschien das Update als 3CX-Erweiterung für Google Chrome, die ein browserbasiertes VoIP-Softphone implementiert. Die endgültige Version der Erweiterung wurde bereits im Chrome Web Store veröffentlicht . Mit dem Web-Softphone können Sie Benachrichtigungen in Anrufen empfangen, auch wenn der Browser minimiert oder sogar geschlossen ist. Außerdem können Sie auf Webseiten Klicks auf Nummern abfangen und diese direkt oder über ein verbundenes IP-Telefon wählen.

Suchen Sie im Chrome Store nach dem Wort "3CX" und installieren Sie die Erweiterung im Browser. Melden Sie sich dann in Ihrem 3CX-Webclient unter Ihrem Konto an und klicken Sie auf "3CX-Erweiterung für Chrome aktivieren". Die Erweiterung erfordert 3CX V16 Update 4 und Google Chrome V78 und höher. Wenn Sie die alte Click-to-Call-Erweiterung von 3CX installiert haben, müssen Sie sie deaktivieren. Benutzer von V16 Update 3 und früheren Versionen müssen zuerst Update 4 installieren und dann die Seite mit dem geöffneten Webclient neu laden, damit die Option zum Aktivieren der Erweiterung angezeigt wird.



Mit Update 4 wurde auch eine Auswahl von PC-Audiogeräten im 3CX-Webclient eingeführt (und dementsprechend eine Erweiterung für Chrome). Sie können Audiogeräte für „Dynamics“ (von wo aus Sie die Stimme hören) und „Speakerphone“ (von wo aus Sie den Anruf hören) auswählen. Dies ist sehr praktisch, wenn Sie ein Headset verwenden. Jetzt können Sie Anrufe an externe Lautsprecher ausgeben und hören, dass diese Sie anrufen. Die Auswahl der Audiogeräte erfolgt im Bereich des Webclients „Einstellungen“> „Personalisierung“> „Audio / Video“.



Die Installation des Updates erfolgt wie gewohnt. Gehen Sie in der 3CX-Benutzeroberfläche zum Abschnitt „Updates“, wählen Sie „v16 Update 4“ und klicken Sie auf „Download Selected“.

Sie können auch eine saubere v16 Update 4-Distribution installieren:

  • Windows: 3CX v16 Update 4 .
  • Linux: 3CX v16 unter Debian 9 ISO - Wählen Sie im Bildschirm " 3CX Update Channel " die Option "Stable".

Vollständiges Changelog in dieser Version.

Einzelner FQDN für 3CX WebMeeting


Diese Woche haben wir auch eine nützliche Änderung vorgenommen, die Systemadministratoren zu schätzen wissen werden - der universelle Netzwerkname für den 3CX-WebMeeting-Dienst lautet „mcu.3cx.net“. Wenn Sie ein sicheres Netzwerk haben, können Sie diesen FQDN nur in den Firewall-Einstellungen öffnen. Jetzt müssen Sie nicht mehr jede IP-Adresse einzeln kennen und öffnen. Mit dem neuen FQDN können Sie auch den Datenverkehr zwischen Ihnen und WebMeeting priorisieren.

Mit dem Standardbefehl nslookup mcu.3cx.net können Sie herausfinden, welche Adressen „mcu.3cx.net“ entsprechen.



Wenn ein Server vorübergehend nicht verfügbar ist, wird seine IP-Adresse automatisch aus der Liste entfernt.

Neue Arten von unterstützten Anrufaufzeichnungsspeichern


Wir möchten Sie auch auf die neuen Arten von unterstützten Repositories für Anrufaufzeichnungen aufmerksam machen, die in v16 Update 4 Beta eingeführt wurden. Dies sind SFTP, Windows Shares und Secure FTP (FTPS & FTPES). Jetzt kann der 3CX-Server in eine Netzwerkumgebung integriert werden, die eine Vielzahl von Technologien unterstützt. Beispielsweise ist SSH (Secure SHell) eines der beliebtesten Dateiübertragungsprotokolle im Internet, das von einer Vielzahl von Plattformen unterstützt wird und kryptografischen Datenschutz bietet.


Um den SSH-Server zu verwenden, gehen Sie zu Backup> Location. und geben Sie den Pfad und die Anmeldeinformationen (oder den OpenSSH-Serverschlüssel) an. Wenn Sie einen OpenSSH-Schlüssel erstellen oder konvertieren müssen, lesen Sie dieses Handbuch . Hier wird beschrieben, wie Sie Ihren eigenen OpenSSH-Server einrichten .

Das SMB-Protokoll ist allen Windows-Administratoren vertraut. Übrigens wird es auch auf NAS-Geräten, Raspberry Pi, Linux und Mac (Samba) erfolgreich unterstützt.



Die Verwendung ist ebenso einfach: Geben Sie den Pfad der SMB-Bälle und die Zugangsdaten an.
Übrigens, wenn Sie vor der Aufgabe stehen, Sicherungskopien oder Aufzeichnungen von Gesprächen zwischen mehreren Servern zu synchronisieren, empfehlen wir die Verwendung des Linux-Dienstprogramms rsync. Lesen Sie mehr über seine Verwendung in diesem Artikel.
