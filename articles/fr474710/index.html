Sous un chat des thèses de performances et le programme.

React: Lever l'état, c'est tuer votre application - Andrey Goncharov, Hazelcast




Avez-vous entendu parler de «lever l'état»? L'un des douze concepts clés de la documentation officielle de React peut-il entraîner des performances médiocres? Dans le cadre du rapport, nous créerons la grille la plus simple sur React. Étape par étape, nous analyserons les problèmes de productivité qui se posent. Nous verrons que parfois O (1) n'est pas assez rapide. Nous allons profiler et refactoriser jusqu'à ce que l'application s'exécute plus rapidement que vous n'avez le temps de dire «Réagir»


Le Saint Graal: lancement de React à Java - Oleg Korovin, camarades de classe




Il y a beaucoup de rumeurs sur un site mystérieux et dur à Odnoklassniki. Beaucoup ont entendu dire que tout se faisait là-bas en Java. Même le frontend. Mais que se passe-t-il vraiment là-bas? Ce rapport vous expliquera comment cela s'est produit, quels étaient les problèmes et comment la solution a été trouvée à l'aide des nouvelles machines virtuelles GraalVM et React. Je vais parler de:
  • Pourquoi nous n'avons pas pris NodeJS, Kotlin, Nashorn et d'autres solutions
  • Pourquoi a été choisi GraalVM
  • Comment fonctionne JS dans GraalVM et quels sont ses avantages
  • Comment construire et exécuter React dans cette configuration
  • Quelles tâches apparaissent et comment les résoudre
  • Comment démarrer une migration progressive, un énorme projet


Des bugs que vous ne rencontrerez jamais - Tim Chaptykov, VK




Nous avons beaucoup d'utilisateurs sur VK, donc souvent à première vue un bug mineur peut être désagréable.
Je vais vous raconter quelques histoires sur les bogues que j'ai rencontrés, sur certaines solutions techniques qui sont nécessaires compte tenu de nos spécificités, et essayer de vous faire part de ce que ça fait de travailler en tant que développeur front-end sur VKontakte.
