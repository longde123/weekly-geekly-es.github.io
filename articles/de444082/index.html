Der Auszubildende Vasya und seine Geschichten über die Idempotenz-API

Eine API-Methode heißt idempotent, deren wiederholter Aufruf den Status nicht ändert. Hier gibt es einen subtilen Punkt: Das Ergebnis eines idempotenten Anrufs kann sich ändern. Wenn Sie beispielsweise die idempotente API aufrufen, um eine Bestellung erneut zu erstellen, wird die Bestellung nicht erneut erstellt, aber die API kann sowohl mit 200 als auch mit 400 antworten.

Kotlin als Zukunft der Android App-Entwicklung

Seit dem Aufkommen von Version 1.0 sind nur wenige Jahre vergangen, und die Sprache wurde von vielen großen Unternehmen und Entwicklern übernommen. Alles dank der Tatsache, dass es den Entwicklern der Sprache gelungen ist, die zu Beginn des Pfades festgelegten Ziele zu erreichen - ein Tool zu entwickeln, das bequemer und sicherer als Java, aber weniger kompliziert als Scala ist.
