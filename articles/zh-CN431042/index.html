新闻与发布


  • PHP 7.3.0RC6-分支7.3的第六个，也是最后一个候选版本。 屏住呼吸，我们正在等待12月6日的最终发布。 同时，您可以在此处和此处的最新评论中再次看到7.3中的新增功能。 与7.2相比， 几乎没有可用的基准显示约10％的性能提升。
  • 基于QUIC的HTTP / 3-决定了下一个HTTP标准将使用QUIC，而QUIC则通过UDP而不是TCP进行工作。 可以使用CloudFlare的QUIC审查，包括NAT问题和其他细微差别。
  • 可用的PhpStorm 2018.3
    •DQL支持
    •PHP CS修复程序
    •新的重构
    •同时部署到多个主机
    •GitHub拉取请求
    和其他许多改进！
  • Symfony离开了PHP-FIG -Symfony的负责人Fabien从PHP-FIG组的成员列表中 删除了自己和框架。 并评论了他在Twitter上的决定：

    PHP-FIG **是为PHP项目创建通用基础的一种好方法，**互操作性优先** #php

    -Fabien Potencier（@fabpot） 2018年11月20日

    有鉴于此， 安东尼·费拉拉（Anthony Ferrara）给PHP-FIG的公开信中说：“请停止解决常见问题，这似乎仍然有意义。 解决50％的问题，而不是99％。”
    两年前，Laravel，Propel，Doctrine，Guzzle，Stash和Aura的代表已经离开PHP-FIG小组，这主要是因为他们没有特别参加讨论。

PHP内部


  • RFC：密码哈希注册表 -下一版本的PHP接受此报价，并允许扩展注册自己的哈希算法。
  • RFC：数组中的扩展运算符 -建议为数组添加一个运算符...
    隐藏文字 
     $parts = ['apple', 'pear']; $fruits = ['banana', 'orange', ...$parts, 'watermelon']; // ['banana', 'orange', 'apple', 'pear', 'watermelon'];
  • [RFC]预加载-对文件预加载的RFC投票已经结束。 所有核心团队成员都一致支持该提议。 因此，明年7.4，我们将有机会将所有文件预加载到opkash中。 这些文件中声明的所有函数和类都可用于所有后续请求，就像它们是内置元素（例如strlen（）或Exception）一样。
    在Composer和Symfony （由@dmitrybalabka发送）中，有关于支持此功能的详细讨论，并且还提供了Ayesh / Composer-Preload插件概念。
