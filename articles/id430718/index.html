Untuk objek apa layak menggunakan pengawasan video cloud?



Baru-baru ini, orang semakin memilih untuk menyewa. Sewa apartemen agar lebih mobile. Pilih berbagi mobil alih-alih perawatan mobil Anda sendiri yang mahal. Sewa cloud alih-alih menyimpan file di smartphone mereka.

Apakah prinsip yang sama berfungsi dengan pengawasan video? Kami melakukan analisis dalam hal ini disarankan untuk meninggalkan server video kami sendiri demi pengawasan video cloud.

Tiga jenis objek diambil untuk penelitian: kecil (16 kamera), sedang (64 kamera) dan besar (128 kamera) - dan tiga tahun penggunaan aktif. Kami mengevaluasi server rakitan, serta server video Line yang stasioner dan berbasis cloud, tidak termasuk biaya kamera CCTV.

Harga peralatan yang ditunjukkan di bawah ini relevan pada akhir 2018 dan tergantung pada daya komputasi yang diperlukan dari server untuk menjamin operasi sejumlah kamera tertentu. Biaya pengawasan video cloud diperhitungkan dari tarif sewa saat ini dari server cloud Liniya , yang mulai dari 90 rubel per bulan (termasuk 7 hari penyimpanan arsip) dan tergantung pada jumlah kamera dan bit rate mereka.

Analisis dilakukan berdasarkan lima parameter utama:

1. Biaya server , yang dihitung, sebagaimana disebutkan di atas, berdasarkan jumlah kamera pada objek tertentu dan karakteristiknya.

2. Biaya hard drive untuk merekam arsip dengan kedalaman 7 hari rekaman konstan dari semua kamera yang terhubung ke sistem.

Adalah penting bahwa ketika menyewa server cloud Line, arsip tidak terbatas pada tujuh hari, tetapi untuk ruang disk yang dialokasikan untuk bitrate kamera yang disarankan sebesar 2 Mbps untuk perekaman arsip berkelanjutan terus menerus. Jika Anda mengatur rekaman untuk deteksi di zona dengan aktivitas rendah, maka kedalaman arsip yang dipilih dapat bertahan 3-4 minggu atau lebih.

3. Biaya listrik untuk memelihara peralatan dan memastikan pengoperasian yang berkelanjutan di fasilitas.

4. Internet - 10, 30, 50 Mbps, masing-masing untuk objek kecil, sedang, dan besar. Jika perekaman akan dilakukan pada server terpusat, total bitrate yang disarankan dari kamera adalah 36, 144, 288 Mbps. Biaya kecepatan Internet di berbagai daerah dapat sangat bervariasi. Karena itu, tarif minimum digunakan untuk perhitungan.

5. Penggantian layanan - titik paling kontroversial, yang tidak mungkin untuk menemukan formula universal untuk perencanaan biaya. Mari kita bahas lebih detail.

Server dapat bekerja secara stabil untuk seluruh periode operasi, dan dapat gagal setelah enam bulan. Bahkan jika peralatan masih dalam garansi, penggantian akan memakan waktu dan saraf. Proses dengan pemasok mungkin memakan waktu lebih dari satu bulan, dan downtime sangat penting dalam beberapa kasus. Pengguna harus membeli server baru untuk memulihkan sistem sesegera mungkin.

Mengingat hal ini, diputuskan untuk menggunakan skema perhitungan biaya berikut untuk kemungkinan perbaikan: untuk tahun pertama, 10% dari biaya peralatan (server + disk untuk arsip) digunakan, untuk tahun kedua 20% dan untuk tahun ketiga 30%.

Dalam kasus server tetap Line, biaya penggantian layanan pada tahun-tahun pertama dan kedua tidak diperhitungkan. Ini karena fakta bahwa kerusakan diminimalkan, karena semua peralatan diuji secara menyeluruh sebelum dikirim ke klien. Ada pengecualian untuk aturan, tetapi masalah dengan cepat diselesaikan dengan memilih solusi optimal untuk klien. Entah penggantian cepat bagian server yang salah dilakukan, atau jika klien tidak dapat mendiagnosis secara independen, kami mengambil seluruh server dan memperbaikinya. Ketika waktu henti peralatan sangat penting, klien dapat mengganti bagian yang diperlukan dengan penggantian biaya lebih lanjut pada bagian kami.

Biaya seperti pajak properti, biaya pemeliharaan peralatan, pekerjaan instalasi server server, dan konsumsi listrik untuk pengendalian iklim tidak diperhitungkan.

Pengawasan video untuk 16 kamera


Mari kita mulai dengan objek kecil dengan 16 kamera FullHD (1920 x 1080) resolusi, 12 fps, kedalaman arsip adalah 7 hari. Selanjutnya, karakteristik seperti yang paling populer di antara klien cloud akan digunakan.



Pada biaya awal, server cloud adalah pemimpin yang tidak perlu dipersoalkan. Pada tahun kedua penggunaan, server video stasioner Liniya ternyata menjadi pilihan paling ekonomis, karena biaya hanya dibatasi oleh biaya pemeliharaannya. Pada tahun ketiga, pengawasan video cloud kembali memimpin.

Untuk penggunaan selama tiga tahun, solusi paling hemat dengan perbedaan hampir sepertiga dari biaya adalah penyewaan server cloud video Line.

Jika seiring waktu menjadi perlu untuk memperluas sistem pengawasan video dan menambahkan lebih banyak kamera, maka kinerja server stasioner mungkin tidak cukup. Ini akan memerlukan biaya baik untuk modernisasi atau untuk pembelian server video kedua. Menggunakan server cloud, cukup kirimkan aplikasi untuk menambahkan kamera. Harga sewa akan meningkat, tetapi masih akan jauh lebih rendah daripada biaya server video Anda sendiri.

Pengawasan video untuk 64 kamera


Kami beralih ke objek medium dengan 64 kamera. Di sini, untuk analisis, 5 jenis server digunakan: solusi cloud, dua PC desktop klasik dan dua server dalam enclosure rak Supermicro, dengan kemampuan "panas" untuk mengganti hard drive dan menggunakan serangan perangkat keras.

Sistem server rak sangat dapat diandalkan, tetapi jauh lebih mahal dan juga memerlukan kondisi penggunaan tertentu: Anda memerlukan ruang terpisah dengan dukungan untuk kondisi suhu. Biaya-biaya ini tidak diperhitungkan di sini, karena pembeli solusi semacam itu kemungkinan besar sudah memiliki ruang server.

Jika biaya server tidak dianggarkan, hasilnya bisa menjadi server biasa dalam versi klasik, yang telah menjadi pemimpin di segmen ini. Berikut adalah persyaratan yang lebih ringan untuk pemeliharaan dan pemeliharaan peralatan.

Menurut hasil analisis, server video cloud untuk 64 kamera hanya bermanfaat digunakan satu tahun, karena tidak ada biaya awal untuk pembelian peralatan. Untuk pengoperasian sistem yang lebih lama, paling menguntungkan menggunakan server klasik. Dialah yang, lebih dari tiga tahun digunakan, menjadi pilihan paling murah. Menyewa server cloud untuk objek jenis ini ternyata menjadi yang paling mahal dan mengambil tempat terakhir.

Pengawasan video untuk 128 kamera


Tahap akhir dari penelitian ini adalah analisis manfaat menggunakan satu atau beberapa jenis server video pada fasilitas besar, hingga 128 kamera IP. Untuk server stasioner, kami mempertimbangkan solusi rack-mount.



Pada tahun pertama penggunaan, pertama-tama dalam hal ekonomi, menyewa server cloud adalah dengan margin besar karena kurangnya kebutuhan untuk membeli peralatan. Pada tahun kedua penggunaan di tempat pertama adalah server video "NVR 128 SuperStorage Line". Untuk tahun ketiga, biaya pemeliharaan kedua server stasioner adalah setara.

Akibatnya, selama tiga tahun penggunaan, server perakitan sendiri adalah solusi optimal untuk objek besar. Dengan margin kecil 94.000 rubel, server di bawah merek Line tertinggal, perbedaan dengan pemenang hanya 6,6%.

Untuk objek apa layak menggunakan pengawasan video cloud?


Menurut hasil penelitian, menyewa server cloud bermanfaat untuk digunakan pada objek kecil hingga 16 kamera. Tetapi, sebelum memutuskan mendukung opsi tertentu untuk mengatur pengawasan video, ada baiknya memperhatikan fitur-fitur berikut:

1. Jika Anda perlu menggunakan sistem pengawasan video selama kurang dari dua tahun, pembelian server stasioner akan menimbulkan biaya awal yang tinggi, dan pada akhir umur peralatan itu akan sulit dijual, karena peralatan dengan cepat menjadi usang.

2. Setelah tiga tahun menggunakan peralatan, kemungkinan kegagalan server itu sendiri dan hard drive untuk menyimpan arsip meningkat. Seringkali, memperbarui satu komponen memerlukan penggantian beberapa komponen lagi.

Dalam praktiknya, kami dihadapkan dengan mereka yang selama 3 tahun dipaksa untuk terus-menerus terlibat dalam memperbarui / mengganti komponen, dan ada juga kasus di mana server telah bekerja selama lima tahun tanpa keluhan. Banyak tergantung pada kondisi operasi dan kualitas setrika.

3. Kebutuhan untuk melengkapi ruang server untuk beberapa klien menjadi argumen utama yang mendukung server cloud.

4. Jika terjadi pencurian atau kegagalan peralatan, penyerang tidak akan dapat menghancurkan arsip yang direkam di server video cloud. Jadi, beberapa pengguna menggabungkan metode pengarsipan: semua kamera direkam pada server stasioner, dan kamera utama diduplikasi pada penyimpanan cloud.

5. Saat menggunakan pengawasan video cloud, ada risiko kehilangan sebagian arsip jika situs tersebut memiliki Internet yang tidak stabil. Penting untuk menyediakan saluran Internet cadangan dan catu daya otonom.

6. Jika Anda melakukan prabayar cloud server rental selama lebih dari satu tahun, dimungkinkan untuk menghitung biaya individual.

Apa yang harus dipilih sebagai dasar sistem pengawasan video, semua orang memutuskan sendiri. Kami berharap penelitian kecil kami akan membantu menentukan dan menyediakan beberapa jebakan.

Ini akan menarik untuk mendengar di komentar pendapat pengguna pengawasan video cloud. Sistem apa yang Anda gunakan untuk berapa banyak kamera, apa pro dan kontra cloud dalam sistem keamanan?
