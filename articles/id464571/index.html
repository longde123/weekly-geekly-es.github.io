Brain + VPS selama 30 rubel =?

Sangat menyenangkan ketika semua hal-hal kecil yang diperlukan ada di tangan: pena dan notebook yang menulis dengan baik, pensil yang tajam, mouse yang nyaman, beberapa kabel tambahan, dll. Hal-hal yang tidak mencolok ini tidak menarik perhatian, tetapi menambah kehidupan yang nyaman. Kisah yang sama dengan berbagai aplikasi mobile dan desktop: untuk screenshot panjang, untuk mengurangi ukuran gambar, untuk menghitung keuangan pribadi, kamus, penerjemah, konverter, dll. Apakah Anda memiliki VPS - yang tidak mahal, selalu ada dan membawa banyak manfaat? Tidak, bukan yang Anda miliki di perusahaan Anda, tetapi Anda sendiri, "saku". Kami berpikir bahwa tanpa VPS kecil pada 2019 itu entah bagaimana sedih, sama seperti tanpa pulpen biasa di sebuah kuliah. Kenapa harus sedih? Musim panas. Seperti musim panas. Dia adalah miliknya sendiri - VPS kami untuk tiga puluh


Kami membaca artikel dari pesaing dan pengguna yang menulis 3-4 tahun lalu tentang mengapa VPS murah tidak diperlukan. Ya, benar, maka VPS "untuk satu sen" adalah pemasaran murni dan tidak dapat menawarkan peluang kerja normal. Tetapi waktu berubah, biaya sumber daya virtual semakin rendah dan selama 30 rubel sebulan kami siap menawarkan ini:

  • Prosesor: Intel Xeon 2 GHz (1 inti)
  • Sistem Linux (Debian, Ubuntu, CentOS untuk dipilih)
  • 1 alamat IPv4 khusus
  • 10 GB untuk penyimpanan data pada SSD cepat kelas perusahaan
  • RAM: 512 MB
  • Tagihan Per Detik
  • Lalu lintas tidak terbatas

Tarif ini tunduk pada batasan teknis tambahan, detail pada halaman penawaran keren kami - VPS untuk 30 rubel.

Siapa yang harus menggunakan server virtual seperti itu? Ya, hampir semua orang: pemula, penggemar, pengembang berpengalaman, penggemar DIY, dan bahkan beberapa perusahaan.

Untuk apa VPS ini cocok?


Kami berpikir bahwa para pembaca Habr pasti akan menemukan cara mereka sendiri untuk menerapkan konfigurasi seperti itu, tetapi mereka memutuskan untuk mengumpulkan ide-ide pilihan mereka sendiri - jika tidak, tiba-tiba seseorang membutuhkannya, tetapi orang-orang itu tidak tahu?

  • Posting situs sederhana Anda, portofolio, resume dengan kode dan sebagainya. Tentu saja, situs web yang dirancang sendiri memberikan kesan positif pada majikan. Letakkan di VPS Anda dan bertanggung jawab atas keamanan dan stabilitas situs Anda sendiri, dan bukan oleh staf hosting bersama.
  • Gunakan VPS untuk tujuan pendidikan: memposting proyek Anda, mempelajari fitur-fitur server dan sistem operasi server, bereksperimen dengan DNS, menggali situs pelatihan kecil.
  • Untuk teleponi. Terkadang seorang wirausahawan perorangan, pekerja lepas, atau perusahaan yang sangat kecil benar-benar membutuhkan IP-telephony, dan operator telepon ini sangat rakus. Sangat sederhana: kami mengambil server kami, membeli nomor dari operator IP-telephony, membuat PBX virtual dan membuat nomor internal (jika perlu). Penghematannya sangat besar.
  • Gunakan server untuk menguji aplikasi Anda.
  • Gunakan server untuk eksperimen DIY, termasuk untuk mengelola dan mengumpulkan data dari sensor sistem rumah pintar.
  • Metode aplikasi yang tidak biasa adalah menempatkan di server sebagai asisten virtual perdagangan pertukaran, robot perdagangan. Anda akan bertanggung jawab penuh untuk stabilitas dan keamanan server, yang berarti Anda akan mendapatkan instrumen yang terkendali untuk berdagang di pasar saham. Nah, tiba-tiba seseorang menyukai atau merencanakan :-)

Ada aplikasi VPS di bidang korporat. Selain layanan telepon yang telah disebutkan, Anda dapat menerapkan beberapa bagian yang menarik. Sebagai contoh:

  • Untuk menempatkan basis data dan informasi kecil yang akan tersedia untuk perjalanan karyawan jarak jauh, misalnya, menggunakan ftp. Ini akan memungkinkan Anda untuk dengan cepat bertukar analitik baru, konfigurasi yang diperbarui untuk tenaga penjualan, presentasi, dll.
  • Berikan akses sementara kepada pengguna atau klien untuk menunjukkan perangkat lunak atau multimedia.

Uji coba VPS selama 30 rubel - dilakukan untuk Anda


30 rubel sangat kecil sehingga bahkan keengganan untuk mendapatkan kartu untuk membayar dan menguji. Kami juga terkadang sangat malas, tetapi kali ini kami melakukan segalanya untuk Anda. Sebelum memulai server dalam pertempuran, kami melakukan tes untuk memeriksa semua detail dan menunjukkan kemampuan server pada tarif ini. Untuk membuatnya lebih menarik, kami menambahkan olahraga ekstrim dan memeriksa bagaimana konfigurasi ini berlaku jika kepadatan dan beban melebihi nilai yang kami tentukan.

Tuan rumah berada di bawah beban sejumlah mesin virtual yang melakukan berbagai tugas pada prosesor dan secara aktif menggunakan subsistem disk. Tujuannya adalah untuk mensimulasikan kepadatan penempatan tinggi dan beban yang sebanding atau lebih besar dari yang tempur.

Selain beban konstan, kami memasang 3 mesin virtual yang mengumpulkan metrik sintetis menggunakan sysbench, hasil rata-rata diberikan di bawah ini, dan 50 mesin virtual yang menciptakan beban tambahan. Semua mesin virtual uji memiliki konfigurasi yang sama (1 core, RAM 512 GB, SSD 10 GB), gambar standar debian 9.6, yang ditawarkan kepada pengguna di RUVDS, dipilih sebagai sistem operasi.

Beban disimulasikan di alam dan ukuran yang sebanding dengan pertempuran:

  • Beberapa mesin virtual dimulai dengan beban rendah.
  • Beberapa mesin "memutar" skenario pengujian yang mensimulasikan beban prosesor (menggunakan utilitas tegangan )
  • Di sisa mesin virtual, kami menjalankan skrip yang menggunakan dd untuk menyalin data dari data yang telah disiapkan ke disk dengan set pembatasan menggunakan pv (contoh dapat ditemukan di sini dan di sini ).

Juga, seingat Anda, kami memiliki tiga mobil yang mengumpulkan metrik sintetis.

Sebuah skrip dijalankan pada setiap mesin secara siklis setiap 15 menit, yang menjalankan tes sysbench standar untuk prosesor, memori, dan disk.

Script sysbench.sh 
#!/bin/bash
date +"%Y-%m-%d %H:%M:%S" >> /root/sysbench/results.txt
sysbench --test=cpu run >> /root/sysbench/results.txt
sysbench --test=memory run >> /root/sysbench/results.txt
sysbench --test=fileio --file-test-mode=seqwr run >> /root/sysbench/results.txt
sysbench --test=fileio --file-test-mode=seqrd run >> /root/sysbench/results.txt
sysbench --test=fileio --file-test-mode=rndrw run >> /root/sysbench/results.txt

#!/usr/bin/python
import mysql.connector as mariadb
from flask import Flask, json, request, abort
from mysql.connector.constants import ClientFlag

app = Flask(__name__)

def getFields(cursor):
    results = {}
    column = 0
    for d in cursor.description:
        results[d[0]] = column
        column = column + 1
    return results

PAGE_SIZE = 30

@app.route("/")
def main():
    return "Hello!"

@app.route("/employees/<page>", methods=['GET'])
def getEmployees(page):
    offset = (int(page) - 1) * PAGE_SIZE
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees')
    cursor = connection.cursor()
    cursor.execute("SELECT * FROM employees LIMIT {} OFFSET {}".format(PAGE_SIZE, offset))
    return {'employees': [i[0] for i in cursor.fetchall()]}

@app.route("/employee/<id>", methods=['GET'])
def getEmployee(id):
    id = int(id)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees')
    cursor = connection.cursor()
    cursor.execute("SELECT * FROM employees WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    employee = {}
    found = False
    for row in cursor.fetchall():
        found = True
        employee = {
            "birth_date": row[fields["birth_date"]],
            "first_name": row[fields["first_name"]],
            "last_name": row[fields["last_name"]],
            "gender": row[fields["gender"]],
            "hire_date": row[fields["hire_date"]]
        }
    if not found:
        abort(404)
    cursor.execute("SELECT * FROM salaries WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    salaries = []
    for row in cursor.fetchall():
        salary = {
            "salary": row[fields["salary"]],
            "from_date": row[fields["from_date"]],
            "to_date": row[fields["to_date"]]
        }
        salaries.append(salary)
    employee["salaries"] = salaries
    cursor.execute("SELECT * FROM titles WHERE emp_no = {}".format(id))
    fields = getFields(cursor)
    titles = []
    for row in cursor.fetchall():
        title = {
            "title": row[fields["title"]],
            "from_date": row[fields["from_date"]],
            "to_date": row[fields["to_date"]]
        }
        titles.append(title)
    employee["titles"] = titles
    return json.dumps({
        "status": "success",
        "employee": employee
    })

def isFieldValid(t, v):
    if t == "employee":
        return v in ["birdth_date", "first_name", "last_name", "hire_date"]
    else:
        return false

@app.route("/employee/<id>", methods=['PATCH'])
def setEmployee(id):
    id = int(id)
    content = request.json
    print(content)
    setList = ""
    data = []
    for k, v in content.iteritems():
        if not isFieldValid("employee", k):
            continue
        if setList != "":
            setList = setList + ", "
        setList = setList + k + "=%s"
        data.append(v)
    data.append(id)
    print(setList)
    print(data)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees', client_flags=[ClientFlag.FOUND_ROWS])
    cursor = connection.cursor()
    cursor.execute("UPDATE employees SET {} WHERE emp_no = %s".format(setList), data)
    connection.commit()
    if cursor.rowcount < 1:
        abort(404)
    return json.dumps({
        "status": "success"
    })

@app.route("/salary", methods=['PUT'])
def putSalary():
    content = request.json
    print(content)
    connection = mariadb.connect(user='admin', password='q5XpRomdSr', database='employees', client_flags=[ClientFlag.FOUND_ROWS])
    cursor = connection.cursor()
    data = [content["emp_no"], content["salary"], content["from_date"], content["to_date"]]
    cursor.execute("INSERT INTO salaries (emp_no, salary, from_date, to_date) VALUES (%s, %s, %s, %s)", data)
    connection.commit()
    return json.dumps({
        "status": "success"
    })
