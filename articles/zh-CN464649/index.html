Zimbra开源版的文档协作系统

在现代业务中共同编辑文档的重要性很难被高估。 在法务部门员工的参与下起草合同和协议，在线上级主管的监督下撰写商业报价等能力，使该公司节省了以前在大量批准上花费的数千工时。 因此， Zextras Suite 3.0的主要创新之一就是 Zextras Docs的出现-该解决方案使您可以直接在Zimbra Collaboration Suite开源版Web客户端中组织与文档的全面协作。

当前，Zextras Docs支持与文本文档，表格和演示文稿的协作，并且还知道如何处理大量文件格式。 解决方案界面与任何文本编辑器的界面都没有太大不同，因此企业员工无需花费大量时间进行培训即可切换到使用Z​​extras Docs。 但是，与以往一样，最有趣的是“幕后”。 让我们一起看看Zextras Docs的工作原理以及该文档协作解决方案可以提供的优势。



最重要的是，Zextras Docs将吸引那些已经在其企业中使用Zimbra OSE和Zextras Suite的用户。 使用此解决方案，可以在不增加信息系统数量的情况下将新服务引入生产环境，因此可以在不增加IT基础结构拥有成本的情况下进行。 请注意，Zextras Docs仅与Zimbra OSE 8.8.12版及更高版本兼容。 因此，如果您仍在使用Zimbra的过时版本，我们强烈建议您升级到8.8.15 LTS版本。 由于长期的支持，该版本将保持最新状态并安全数年，并且还将与所有当前的新增版本兼容。

Zextras Docs的优势还可以包括在企业基础架构上完全部署的可能性。 这避免了将信息传递给第三方的情况，这在使用混合服务或云服务时通常是这样。 这就是为什么Zextras Docs非常适合具有严格信息安全策略的企业以及那些希望完全控制企业中发生的数据流和流程的系统管理员的原因。 此外，本地部署的文档协作服务使您可以避免在服务提供商出现问题或高速Internet访问出现问题时与云服务不可访问性相关的风险。

Zextras Docs由三部分组成：独立服务器，扩展和冬季。 这三个部分各自发挥作用：

  • Zextras Docs服务器是LibreOffice引擎，经过增强后可与Zimbra OSE进行协作和集成。 在Zextras Docs服务器上，用户访问的所有文档都将被打开，处理和存储。 必须将其安装在运行Ubuntu 16.04，Ubuntu 18.04或CentOS 7的专用计算节点上。如果Zextras Docs服务的负载足够大，则可以一次为其选择多个服务器。
  • Zextras Docs扩展程序是内置于Zextras Suite中的，因此无需安装。 由于有了此扩展，用户可以连接到Zextras Docs服务器，并在使用多台服务器时实现负载平衡。 另外，通过Zextras Docs扩展，多个用户同时连接到一个文档，文件也从本地存储下载到服务器。
  • 反过来，需要Zextras文档Zimlet才能将该服务集成到Web客户端中。 多亏了他，Zimbra Web客户端才有机会创建以及预览Zextras Docs文档的功能。



为了在企业中部署Zextras Docs，必须首先为其选择一个或多个物理或虚拟服务器。 之后，您需要从Zextras网站下载适用于Ubuntu 16.04 ， Ubuntu 18.04或CentOS 7的服务器应用程序发行版 ，然后将其解压缩并安装。 在安装的最后阶段，服务器将要求您指定LDAP服务器的IP地址，以及用于将有关新服务器的数据输入LDAP的用户名/密码对。 请注意，安装完成后，所有其他基础结构节点都将看到每个Zextras Docs服务器。

由于Zextras套件中已经包含Zextras Docs扩展，因此仅足以使那些有权访问文档协作工具的用户和组使用它。 可以从Zimbra管理控制台激活Zextras Docs Zymelet。 请注意，将Zextras Docs服务器添加到Zimbra OSE基础结构后，您需要更新Zimbra代理服务器配置。 为此，只需以Zimbra用户身份运行/ opt / zimbra / libexec / zmproxyconfgen文件 ，然后运行zmproxyctl restart命令以重新启动代理服务。
