在实践中采用进口替代。 第3部分。操作系统



我们继续我们有关进口替代的系列文章。 在以前的出版物中， 考虑了用“国内”系统（特别是“国内生产”系统管理程序） 替换已部署系统的选项 。

现在该谈谈目前通信部注册表中包含的“本地”操作系统。

0. 起点


我发现自己不知道如何比较LINUX发行版。 我爬进了Wikipedia ，但它变得更加清晰。 要考虑什么标准？ 以什么为参考点？ 对于我来说-服务器操作系统最有说服力的标准是稳定性。 但是在测试的框架中，“稳定性”这个词听起来至少很奇怪。 好吧，我花了一周时间研究了扩展的系统……但是，在几年的正常运行时间甚至不是平均值的情况下，一周的时间并不是一个指标。 压力测试？ 如何在支架上装载系统？ 此外，需要加载的是操作系统，而不是应用程序，并且必须加载它以使其崩溃。如果没有崩溃，该如何进行比较？

但是，我得出的结论是，可以有条件地从“本地”操作系统的发布者那里获得稳定性。 以Astra为例，例如Debian，ROSA-Mandriva，Calculate-Gentoo等。 而且仅对于Alt而言，自Mandrake分离以来已经存在了很长时间，因此可以将其视为独立发行版（相对于所有其他“本地” OS）。 但是请记住，这都是极其任意的，因为不知道开发人员填充了什么源代码，以及随着操作系统安全性的提高而进行了更改。

更具可追溯性的标准是OS交付软件包及其存储库中软件包的组成。 但是在这件事上，我们必须从必要性的要求出发。 我有自己需要解决的任务，也有自己的任务，选择软件的方法应该是这样的：“任务就是选择软件”，而不是相反，这在非营利组织中经常发生……

因此，在“移动”期间需要部署的服务：

  • 邮件服务器
  • 扎比克斯
  • 数据库管理系统
  • Web服务器
  • Jabber服务器
  • 后备
  • 办公套餐
  • SUFD和银行客户
  • 邮件客户端
  • 浏览器

AD，DNS，DHCP，CertService保留在Windows服务器上（对此的解释在上一篇文章中给出）。 但是公平地说，我注意到目录服务可以在相同的SAMBA或FreeIPA上引发，并且某些发行版本声明“其”目录服务（Astra Linux目录，ALT，ROSA目录，Lotos目录）。 DNS和DHCP也可以在任何Linux发行版上使用，并且并不是每个人都需要认证服务器。

邮件服务器 。 我喜欢Zimbra 。 我和她一起工作，她很舒服，知道如何从Exchange收集数据，了解很多东西。 但是它只能部署在ROSA Linux上。 您可以将其放在操作系统的其余部分上，但不会被认为是合法的。 另一方面，我遇到了Zimbra，每个“家用”操作系统都有自己的一套邮件服务器。

扎比克斯 他没有竞争对手。 在进口替代的框架内-甚至更多。 Zabbix是Alt Linux，RED OS，Astra和ROSA的一部分。 在计算中，它被标记为“不稳定”。

数据库管理系统 PostgreSQL支持所有“本地”操作系统。

Web服务器 。 Apache在所有服务器操作系统中。

Jabber服务器 。 通常，计划引入Bitrix24 ，但是我已经习惯了一切都已经进行了很长时间的事实，因此我正在考虑选择基于jabber的公司聊天。 我习惯于OpenFire 。 它是Calculate ，RED OS和ROSA的一部分。 作为ROSA，Alt，RED OS和Astra的一部分，也有ejabberd。

备份 。 Bacula是Astra，Rosa，Alt，Calculate，AlterOS的一部分。

办公套件 。 免费的办公套件Libre Office安装在所有客户端（通常是服务器）的“本地”操作系统中。

电子邮件客户端 。 在所有客户端（通常是服务器）“本地”操作系统中都使用Thunderbird 。

浏览器 在所有操作系统上至少都可以使用Mozilla Firefox 。 Yandex.Browser也可以安装在所有操作系统上。

通过SUFD和银行，客户变得更加复杂。 正式地，所有这些都可以在几乎所有“家用”操作系统上运行。 实际上，要进行验证非常困难，因为您需要带用户，带到经过测试的机器上并说“ try”。 很烦。 因此，我将第一次离开旧的方案-一个带有Windows的客户端，并为每个银行的虚拟机向其投掷一个令牌。 幸运的是，Linux肯定会知道如何转发令牌。 在那里将会看到。

接下来，我们继续选择满足我们需求的操作系统。 但是为了客观起见，我尝试从商业和工业部的注册表中涵盖尽可能多的操作系统。

因此，最后，我决定只写我对测试过程中每个发行版的个人看法和想法。 好吧，还有一些有用的信息，而不是非常有用的信息。

1.1。 阿斯特拉Linux


官方网站

当前版本：

父发行版是Debian。

软件包的组成可以在这里找到。 （在“操作系统的组成”部分的品牌软件图片下，不起眼的按钮“详细信息”。）

安装需要很长时间。 将OS部署到虚拟机将近一个半小时……也就是说，如果需要将其部署到1500台域PC，则将花费大量时间。

这是Debian。 这是一个过时的Debian。 无论是在程序集还是在存储库中，Astra软件包都比其父软件包更老。 如果有紧急需求，可以连接Debian存储库，但是，这会自动取消任何导入替换（在这种情况下，您可以通过Debian存储库apt update && apt upgrade更新系统，并且它将继续工作...尽管我不确定野兽到底是什么，我以防万一。

桌面“飞”。 对于服务器，尽管它简化了某些操作，但从根本上来说，根本不需要GUI。 但是对于没有它的用​​户操作系统，无处不在。 通常，它会留下良好的印象，同时尽可能靠近Windows，这将为用户简化向此OS的过渡。 系统中有很多“飞”的东西，所有这些都是由JSC“ NPO RusBITech”开发的。 大多数情况下，热键与Windows上的功能相同。 Win + E打开“资源管理器”，Win打开任务栏菜单，依此类推。 通常，最有可能的是，开发人员试图使外观尽可能接近Windows。

操作系统加入AD，允许您配置授权等。 在测试过程中，它被证明是稳定的（据测试操作期间的判断），而不是任性的，简单易用的Debian OS。

如果需要，可以不从存储库安装软件包。 我尝试以OpenFire为例。 您下载了Debian的软件包，并且所有内容都已安装完毕。

为了解决我的问题，它可以用作部署Zabbix，jabber-server，PosgreSQL，Apache的平台。 作为用户操作系统，它可以满足所有要求（Nice界面，LibreOffice，Thunderbird，Firefox）。 SUFD和客户未测试银行。

特别版与通用版的不同之处在于，特别版适用于处理国家机密和其他机密文件，并且经过认证。 通用-“普通”操作系统，可以在不需要认证的地方使用，并且您不需要使用秘密。

3. 总结


要部署Zimbra Collaboration Suite OSE，我至少需要1个ROSA Enterprise Linux Server副本，最好是2个，以配置代理。

为了提高所有其他服务，使用Astra Common Edition或EDIT OS是有意义的，因为由于价格便宜，将来这些系统的价格将是最优惠的。但是我个人对Astra印象深刻。由于它是免费的，因此可以

在Calculate Linux的基础上部署一些非关键服务。但是由于缺乏支持，这些应该是此类服务，对于企业而言，停机时间对于企业而言并不重要，因为系统管理员直接对其性能负责。

自定义操作系统-我更喜欢相同的Astra CE。她拥有最新的办公套件，用户友好的GUI，该系统可以执行所需的任何操作。是的，它比竞争对手便宜。

如果需要部署目录服务器和其他基础结构服务，则至少从兼容性的角度出发，考虑将要部署到用户的同一家族的OS是有意义的。就我而言，如果我仍然需要这样做，则很可能是Astra CE。
