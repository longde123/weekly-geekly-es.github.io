Mengapa Windows XP masih mencuri pada 2019, atau CTOD?

Tentang Habré dalam komentar untuk artikel tentang rilis versi baru sistem operasi, rilis model baru laptop, perangkat penyimpanan data, modul memori, dll. pendapat tersebut secara teratur dinyatakan bahwa hanya versi terbaru dari sistem operasi dari vendor terkenal yang memungkinkan geek modern untuk tidak meluncur ke kota yang suram ... dia merasa seperti orang, dan hanya orang yang memiliki Windows 8, 10, 11, 9000 yang diinstal (garis bawahi apa yang diperlukan ), akan populer di kalangan perempuan pengusaha dan pelanggan. Untuk alasan yang disebutkan di bawah ini, saya menganggap pendapat ini sangat keliru dan bahkan cacat, menunjukkan ketidakmampuan pemilik komputer untuk secara optimal menggunakan sumber daya perangkat keras dan perangkat lunak yang dimilikinya.

Berikut ini adalah sistem yang, pada 2019, memungkinkan pemiliknya, yang bekerja di bidang TI, bukan seorang programmer, untuk memuluskan semua jenis urusan komputer tanpa masalah dan digunakan untuk menjelajahi Internet, mengunduh torrent, melakukan pembukuan rumah, mengirim email, berkomunikasi dengan teman melalui konferensi audio dan video, interaksi dengan bank dan lembaga pemerintah, membuat presentasi dan ceramah, pemrosesan foto amatir dan membuat dan mengedit video rumah amatir (untuk menontonnya di e di komputer), mengisi dan memelihara koleksi rekaman musik (untuk mendengarkannya bukan di komputer), merekam arsip pada disk CD dan DVD, dll., dll.

Besi


Saya membeli laptop saya pada tahun 2010 seharga 300 dolar Kanada (~ 250 USD). Saya selalu membeli komputer termurah, dan bahkan pada saat itu laptop tidak dapat membanggakan fitur-fitur unggulan:

  • Prosesor : AMD Turion II Dual-Core M600
  • Frekuensi CPU : 2,4 GHz
  • Kartu Video : Radeon 4200
  • Kapasitas RAM : 2 GB
  • Hard Drive : 140 GB, SATA
  • Drive : DVD-RW
  • Slot dan port ekspansi : 3 x USB 2.0, pembaca kartu SD, VGA, HDMI, ethernet, wi-fi.

Saya perhatikan bahwa murahnya komputer membuatnya terasa dan kemudian saya harus mengubah banyak komponen - peralatan Hewlett Packard tidak terkenal karena keandalan dan daya tahannya. Tetapi artikel ini masih bukan tentang isian besi, tetapi tentang sistem operasi dan serangkaian program dan utilitas yang dengan senang hati hidup di bawah kendali OS "usang" pada perangkat keras sederhana ini, terutama dengan standar saat ini.

Perangkat lunak


Windows XP



Laptop datang terinstal dengan Windows 7. Setelah disulap dengan nLite, untuk mengintegrasikan driver SATA untuk hard disk ke dalam file instalasi (ini diperlukan karena BIOS laptop tidak memiliki opsi untuk beralih antarmuka IDE / SATA), saya dapat menginstal Windows XP dengan Paket Layanan 2, yang setelah optimalisasi mulai bekerja dengan cerdas dalam konfigurasi berikut:



  • Kapasitas RAM: ~ 300 MB,
  • volume file OS pada disk: 211 MB (tidak termasuk driver printer),
  • swap file: 0 MB,
  • tidak adanya Internet Explorer dan semua pustaka dan jejaknya (kecuali folder Content.IE5 dan History.IE5 yang berukuran 32768 byte),
  • menjalankan layanan:



    Sebagian besar layanan lain dinonaktifkan atau dihapus sama sekali (melalui Jauh), termasuk layanan pembaruan Windows,
  • hanya satu pengguna yang dipasang di sistem: Administrator.

Saya juga ingin memikirkan menonaktifkan file swap.


Kebutuhan untuk swap file dalam sistem operasi Microsoft adalah mitos yang sangat umum - diyakini bahwa ketika dimatikan, sistem menjadi sepenuhnya tidak beroperasi, hang, crash, biru "layar kematian" muncul, hard drive beralih ke frekuensi resonansi, itu crash dan pecah unit sistem menjadi rusak ... Dari dari semua cerita horor, masalah sebenarnya adalah satu - ketika program yang berjalan mengkonsumsi semua RAM yang tersedia dan crash, karena memori ini tidak cukup untuk itu. Yah, pada dasarnya itu. Pada komputer yang menggunakan program dan paket intensif sumber daya (Google Chrome, IDE , MS Office, dll.), Cukup untuk menginstal RAM lebih sedikit, tetapi pada sistem yang tidak bekerja dengan program besar, 640 KB cukup untuk semua 2- RAM 4 GB akan cukup untuk perangkat lunak bekerja dengan nyaman - dan file swap dapat sepenuhnya dinonaktifkan!

Namun, menonaktifkan file swap memiliki lebih banyak keuntungan daripada menggunakannya: program dan utilitas hanya mengkonsumsi RAM cepat, jangan menulis atau membaca data ke / dari swap secara terus-menerus, bekerja dengan file pengguna jauh lebih cepat karena sistem operasi tidak perlu menunggu salah satu program selesai bertukar. Kita semua menemukan situasi di mana Anda perlu dengan cepat membuka / mengedit / mencetak file dari disk, dan komputer "swap", indikator operasi hard disk tetap menyala, antarmuka sistem kehilangan respons, kepala berteriak, pelanggan menginjak-injak dan merobek rambut di kepala pengembang. - Jadi, percayalah pengalaman saya selama bertahun-tahun bekerja dengan berbagai macam workstation yang menonaktifkan swap meningkatkan kecepatan mesin Windows tanpa berlebihan pada waktu (!) Dan secara praktis tidak mempengaruhi stabilitas pekerjaannya. Saya menulis "secara praktis" hanya karena saya jujur ​​mengatakan bahwa crash program terjadi, tetapi mereka sporadis, mereka dapat diprediksi dan berhasil dicegah.

Ketika saya mendapatkan komputer pribadi di tempat kerja yang baru, setelah saya mendapatkan kontrol administratif atasnya, hal pertama yang saya lakukan adalah pergi ke pengaturan sistem dan mematikan file swap. Selalu! Saya melakukan ini di Windows 98, XP, 7, 8 - dan tidak pernah mengalami masalah dengan kurangnya RAM dan sering crash. Pada saat yang sama, sangat jelas bahwa saya tidak akan membahayakan diriku sendiri, sebaliknya - dengan menonaktifkan file halaman saya membuat pekerjaan saya lebih efisien dan menjaga rambut di kepala saya untuk saya sendiri, pengembang, dan pelanggan.
