Pembuat Wikipedia menjawab pertanyaan: pemrograman, tidur, buku, tips seumur hidup

"Setiap hari saya bangun dan melakukan hal paling menarik yang bisa saya pikirkan."


Ketika saya pertama kali melihat tanggapan 961 Jimmy Wales untuk Quora untuk pertama kalinya , saya menulis beberapa yang paling menarik untuk "melihat ke kepala" orang yang mungkin membuat proyek yang paling berguna bagi umat manusia dengan komponen TI yang diucapkan (Jimmy sendiri tidak mempertimbangkan Wikipedia TI sebuah proyek, dan menganggapnya sebagai proyek komunitas sosial).

Beberapa jawaban tumpang tindih, beberapa bertentangan satu sama lain, karena ini adalah gado-gado 2011-2019, tetapi mereka semua berguna, menurut pendapat saya, untuk mengasah pemikiran dan kemampuan Anda untuk memberikan jawaban yang sopan dan mengajukan pertanyaan. Memahami apa yang dipandu oleh Jimmy Wales ketika membuat keputusan, bagaimana dia membuat pilihan, bagaimana dia memprioritaskan, bagaimana dia menghancurkan kehidupan.

Dalam pembacaan jawaban kedua dan ketiga (961 ... jangan tanya berapa banyak musim panas yang cerah dan malam berbintang yang dibutuhkan), saya sudah mengidentifikasi beberapa blok tematik, yang akan saya terbitkan nanti. Tulis pertanyaan Anda di PM, dalam komentar, melalui pos (alexey.stacenko@gmail.com), yang dapat ditanyakan kepada Jimmy Wales atas nama habrasociety (seperti yang pernah kami tanyakan pada Alan Kay ).

Pemrograman


- Apakah Jimmy Wales memiliki akun di GitHub?

- Ya, saya punya akun di GitHub. Saya menggunakannya untuk proyek-proyek Ruby on Rails.

"Bagaimana Jimmy Wales belajar memprogram dan apa urutan di mana dia belajar bahasa?"

"Aku tidak bisa mengingat semua detailnya, tetapi aku akan memberitahumu bahwa aku bisa." Namun, penting untuk diingat: Saya bukan programmer yang hebat. Saya pikir saya bahkan bukan programmer yang sangat baik. Tapi saya seorang programmer.

Ketika saya masih muda, paman saya membuka salah satu toko komputer pertama di kota saya. Mungkin yang pertama. Penjualan komputer Commodore.

Di Commodore, di bawah bimbingannya dan menggunakan berbagai buku, saya belajar Basic. Seperti yang mungkin Anda ketahui, bahasa yang mengerikan.

Kemudian, di perguruan tinggi, saya belajar sedikit Fortran dan S. Cukup banyak.

Kemudian, begitu saya lulus dari sekolah pascasarjana (di bidang keuangan), saya menyadari betapa bermanfaatnya pemrograman bagi saya. Jadi saya terjun ke dalamnya sekali di musim panas dan belajar sedikit C dan C ++. Saya menggunakan tutorial Coronado C dan C ++, yang pada masa itu didistribusikan sebagai shareware.

Kemudian, ketika saya bekerja di bidang keuangan di Chicago, saya belajar Perl untuk mulai membangun situs. Tentu saja, antara lain, saya harus belajar HTML dan, pada akhirnya, sedikit Javascript, tetapi ini adalah hari-hari pertama di Internet, sebelum Javascript menjadi sangat penting.

Saya mulai memahami Perl dengan sangat baik, dan memang, ini adalah bahasa yang paling saya kenal.

Baru-baru ini, saya mempelajari buku tentang Ruby dan mengambil kursus Ruby on Rails (kursus Michael Hartl sangat bagus). Saya sama sekali bukan ahli, tetapi selama proyek saya bekerja di jejaring sosial tempat saya berperan dalam The People's Operator, saya sangat senang bahwa saya meluangkan waktu untuk mempelajari Rails, karena saya dapat membaca kode, saya tahu bahwa itu mungkin dan itu tidak mungkin, dan saya suka bekerja dengan orang-orang yang tahu lebih banyak dari saya, cukup kompeten sehingga saya bisa belajar dari mereka.

- Bagaimana Anda belajar memprogram?

- Saya menggunakan Tutorial Coronado C untuk belajar C dan kemudian Tutorial Coronado CPP. Secara teknis, ini bukan bahasa pemrograman pertama saya sejak saya belajar Basic dan sedikit Fortran sebelumnya. Tetapi ini adalah pertama kalinya saya benar-benar belajar bagaimana memprogram dengan mempelajari pointer dan pemrograman berorientasi objek.

Buku-buku teks ini masih tersedia di Internet saat ini, mungkin dengan sedikit atau tanpa perubahan, karena dasar-dasar bahasa-bahasa ini abadi, saya kira.

- Apakah Jimmy Wales tahu cara memprogram?

- Ya saya tahu.

Jika Anda melihat melalui Quora, Anda akan menemukan jawaban lain yang menggambarkan kisah saya, tetapi saya akan memberi tahu Anda apa yang dapat saya lakukan sekarang.

Tahun lalu, saya mengambil cuti sebulan untuk belajar Ruby on Rails. Saya mencapai tingkat profesionalisme awal, dengan mengandalkan keterampilan pemrograman lama yang belum pernah saya gunakan selama bertahun-tahun.

Selama 3 hari terakhir, saya terjun langsung ke basis kode dan meluncurkan di laptop saya kode prototipe untuk jaringan sosial untuk Operator Rakyat, di mana saya memimpin pengembangan. Saya pikir penting bahwa pemimpin proyek mengetahui teknologi dan dapat berfungsi, tetapi ini tidak terjadi ketika saya akan menjadi pengembang utama, karena keterampilan saya terlalu dilupakan untuk cepat dan akurat.

- Tidak. Saya seorang programmer, meskipun dibandingkan dengan programmer profesional nyata, saya bukan programmer yang sangat baik.

Tidur


- Jimmy Wales mengklaim bahwa tidur sangat penting untuk kesuksesan. Apa yang dia pikirkan tentang begitu banyak orang yang sangat sukses, sehingga hanya tidur sedikit?

Bagi beberapa dari mereka, saya pikir itu wajar. Jika Anda sendiri tidak bangun setelah tidur singkat, maka Anda mungkin bukan milik orang-orang seperti itu.

Beberapa dari mereka, saya pikir, membuat kesalahan. Mereka akan lebih sukses jika mereka beristirahat dengan benar.

Dan bagi banyak orang, atau kebanyakan rumor, saya pikir ini tidak benar. Orang menyukai kisah orang yang tidak perlu tidur lebih dari beberapa jam sehari, mungkin dengan cara tertentu, karena itu membuat mereka merasa lebih baik tentang kekurangan mereka sendiri - yaitu, jika saya berbeda secara genetik, atau jika Saya ingin membuat diri saya menderita, saya bisa lebih sukses, tetapi saya tidak bisa atau lebih suka tidak.

Saya pikir banyak orang berpikir salah dalam bidang ini.

"Berapa jam dalam sehari Jimmy Wales tidur?" Kurang dari rata-rata orang dewasa? Saya selalu berpikir bahwa orang-orang yang benar-benar sukses hanya tidak membutuhkan banyak tidur untuk tetap waspada dan sangat fungsional.

- Saya tidur selama 8 jam kira-kira setiap malam. Ketika saya bisa, saya tidur lebih banyak. Saya percaya pada mimpi. Saya pikir kesalahan besar yang dilakukan oleh anak muda ketika mereka ingin sukses adalah membuat mereka tidak bisa tidur. Ini sangat bodoh.

Bagaimana kurang tidur mempengaruhi kemampuan mental? Apakah orang-orang hebat seperti Einstein, Feynman dan Ramanujan memiliki cara untuk tidur lebih sedikit? Jika tidak, bagaimana mereka melakukan pekerjaan mereka?

Saya pikir itu mungkin untuk menipu mimpi, tetapi saya pikir sebagian besar bukti menunjukkan kebalikan dari apa yang Anda pikirkan dan lawan dari apa yang dipikirkan kebanyakan orang. Tetapi seharusnya demikian, dan ini seharusnya merupakan penipuan tidur yang keren. :-)

Cara terbaik untuk menipu mimpi adalah dengan tidak mengejar semacam rezim aneh yang, kami harap, menipu tubuh kita untuk kurang tidur. Anda dapat melatih kurang tidur - tetapi Anda akan secara kognitif melemah dan kurang produktif, jadi apa gunanya?

Jadi menipu mimpimu sesuai dengan sistem salah satu penipu tidur terbesar sepanjang masa, Benjamin Franklin. "Pensiun dini dan kebangkitan awal membuat seseorang sehat, kaya, dan bijaksana."

Tidurlah dan pastikan tidur Anda efektif. Tidak ada kafein di sore hari. Tidak ada layar dan lampu bercahaya di malam hari. Tidurlah lebih awal. Bangunlah secara alami, tetapi jauh lebih awal daripada jika Anda disosialisasikan dan menonton TV sampai tengah malam.

Jika seseorang mengajukan pertanyaan, saya tidak melakukannya sendiri. Tetapi saya memiliki periode yang sangat produktif dalam hidup saya, dan sekarang saya harus menjadi lebih baik.

- Bagaimana saya bisa mengatasi kurangnya fokus pada GMAT?

- Sebagai pedoman, bertahun-tahun yang lalu saya mencetak 750 poin pada skala GMAT total pada skala 200-800.

Saran saya untuk Anda: tidur. Tidur 9 jam sehari. 10 jika Anda membutuhkannya. Satu atau dua jam ekstra penelitian sehari tidak akan ada gunanya bagimu jika kamu begitu lelah sehingga kamu hanya melakukannya karena rasa tugas atau daya saing atau sesuatu yang orang katakan kepada kamu.

Tidur banyak. Bangun. Minumlah kopi. Burrow ke dalam buku-buku dan jadilah superman seperti yang Anda impikan. Jadilah orang yang fokus dan fokus selama beberapa jam sehari, dan kemudian Anda akan tahu bagaimana melakukannya selama pengujian.

Kemudian hentikan penggunaan kafein lebih cepat sehingga Anda bisa ... tidur. Tidur 9 jam sehari. 10 jika Anda membutuhkannya.

- Bagaimana saya bisa mengatasi penurunan aktivitas? Saya seorang siswa kelas 12. Meskipun saya bersemangat tentang matematika, saya tidak ingin belajar apa pun. Saya merasa mengantuk sepanjang waktu. Bagaimana saya bisa belajar? Saya bermeditasi, tetapi itu tidak berguna.

- Meskipun saya setuju sampai batas tertentu dengan jawaban lain, saya akan menyarankan bahwa 9 jam tidur di malam hari adalah normal (Anda seorang remaja, Anda harus tidur sedikit lebih jika mungkin), tetapi 5-6 jam lagi tidur pada siang hari adalah berlebihan sejauh Anda dapat berkonsultasi dengan dokter Anda.

Perlu membaca tentang kantuk berlebihan di siang hari.

Adapun keengganan untuk terlibat dalam studi "resmi", saya hanya bisa bersimpati. Di sini saya setuju dengan jawaban yang lain, tetapi saya dapat mengatakan bahwa saya tidak pernah menggunakan saran saya sendiri dan, oleh karena itu, memiliki kinerja yang buruk di sekolah selama sebagian besar masa remaja saya. Terkadang Anda hanya perlu mencetak dan menerobos beberapa misi yang membosankan.

Tetapi sekali lagi, saya pikir bahwa tidur yang berlebihan adalah tanda peringatan dari tubuh Anda. Pergi memeriksanya!

- Apakah Jimmy Wales terbang kelas satu?

Saya melakukan perjalanan di udara lebih dari siapa pun.

Terkadang saya terbang kelas satu. Orang sering membayar untuk perjalanan saya ketika saya datang untuk memberikan pidato untuk mereka, dan ini biasanya kelas bisnis, tetapi kadang-kadang kelas satu. Dan karena saya terbang cukup banyak, saya terkadang mendapatkan pembaruan dari kelas bisnis ke kelas satu.

Saya hampir tidak akan pernah terbang kelas satu jika saya membayar sendiri penerbangannya. Kelas bisnis sering sepadan karena saya bisa tidur dan menjadi lebih produktif, serta menghabiskan lebih sedikit waktu di luar rumah (tidak perlu merencanakan waktu ekstra untuk pulih dari penerbangan tanpa banyak tidur). Tetapi meningkatkan dari kelas bisnis ke kelas pertama biasanya tidak sepadan.

Saya pernah membayar untuk perjalanan kelas satu dari London ke Meksiko, karena untuk alasan yang tidak diketahui itu sebenarnya lebih murah daripada kelas bisnis.

Akhirnya, setiap kali saya berbicara tentang penerbangan, saya ingin menekankan bahwa Wikimedia Foundation hanya membayar 1 penerbangan saya ... Saya pikir itu hampir sepuluh tahun yang lalu. Ketika saya bekerja dengan IMF (misalnya, saya menghadiri rapat dewan), saya bersikeras bahwa saya membayar sendiri penerbangan saya.

Tentang #philtech Global Initiative
#philtech (teknologi + filantropi) adalah teknologi terbuka dan dideskripsikan secara publik yang menyelaraskan standar hidup sebanyak mungkin orang dengan menciptakan platform transparan untuk interaksi dan akses ke data dan pengetahuan. Dan memenuhi prinsip-prinsip filtech:

1. Buka dan direplikasi, bukan kepemilikan secara kompetitif.
2. Dibangun di atas prinsip pengorganisasian diri dan interaksi horizontal.
3. Berkelanjutan dan berorientasi pada perspektif, daripada mengejar manfaat lokal.
4. Dibangun di atas data [terbuka], bukan tradisi dan kepercayaan
5. Non-kekerasan dan non-manipulatif.
6. Termasuk, dan tidak bekerja untuk satu kelompok orang dengan mengorbankan orang lain.

PhilTech Accelerator dari Startup Teknologi Sosial adalah sebuah program untuk pengembangan intensif proyek-proyek tahap awal yang bertujuan untuk menyamakan akses ke informasi, sumber daya dan peluang.
