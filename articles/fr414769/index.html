Lancez le reciblage mobile avec Appsflyer: paramètres, rapports et liens

Si vous achetez du trafic dans une application mobile, un tracker mobile doit y être pour surveiller la qualité du trafic impliqué. Dans cet article, au nom de Getloyal, un système de reciblage automatisé pour les applications mobiles , nous vous montrerons comment lancer le reciblage mobile avec Appsflyer , comment le configurer et où afficher les statistiques.



Rapport de reciblage Appsflyer


Appsflyer fournit un tableau de bord spécial, qui se trouve dans l'onglet «Re-Targeting» dans le menu latéral. Des statistiques sont collectées si:

1. S'il s'agit vraiment d'un retour d'utilisateurs existants
2. Et si dans le lien vous écrivez le paramètre: "is_retargeting = true"



Appsflyer propose deux versions de conversion de reciblage:

  • Redistribution - l'utilisateur a désinstallé l'application précédemment installée et, après avoir cliqué sur l'annonce, l'a réinstallée.
  • Réengagement - l'utilisateur a ouvert l'application actuellement installée

Cas: Imaginez que vous lancez une campagne UA sur une plate-forme publicitaire où vous ne pouvez pas exclure la base d'utilisateurs actuelle. Cela signifie que vous en attirerez de nouveaux ou en reviendrez d'anciens.
Dans le cas d'Appsflyer, les nouveaux paramètres seront affichés dans le tableau de bord Présentation, où ils appartiennent. Si vous n'avez pas ajouté la valeur "is_retargeting = true" au lien (ce que vous n'auriez peut-être pas fait pour les nouveaux utilisateurs de la campagne), vous ne verrez pas les conversions dans le rapport de reciblage, même s'ils ont une place.

Paramètres de reciblage Appsflyer


Vous devez savoir et faire ce qui suit:

1. Tout d'abord, avant de lancer une campagne dans les paramètres de l'application dans Appsflyer, vous devez activer la "mesure des campagnes de reciblage".



Sans activer ce paramètre, vous ne pouvez pas mesurer les ouvertures d'application (réengagements), et seules les réattributions seront prises en compte dans le tableau de bord.

2. La fenêtre d'attribution de réengagement est le nombre de jours pendant lesquels tous les événements après que l'utilisateur a été de nouveau attiré sont enregistrés dans la source de la redistribution. La période commence par le fait de la réattribution et se termine par le jour qui est défini dans les paramètres. La valeur par défaut est de 30 jours. Vous pouvez le changer si vous ajoutez «af_reengagement_window =» dans le lien et définissez le nombre de jours souhaité: de 30 à 90 jours.

3. Fenêtre d'analyse d'attribution - il s'agit d'une fenêtre bien connue après un clic / une vue pendant laquelle la réattribution peut être comptée. Au clic, la valeur standard est de 7 jours. Configurable de 1 à 30 jours. Modifie en ajoutant le paramètre "af_click_lookback =" au lien.

4. Fenêtre de re-attrubution - la période de temps après la première installation, pendant laquelle, si l'utilisateur désinstalle l'application et la réinstalle, la réinstallation ne sera pas comptée dans une nouvelle source. Ce paramètre n'est pas lié au reciblage, car si un utilisateur clique sur une campagne de reciblage pendant cette période, la réinstallation ne sera comptée que sur la source du reciblage. La fenêtre de redistribution standard est de 90 jours. Configurable de 1 à 24 mois. En savoir plus sur cette période dans la documentation Appsflyer.

Cas: Avant-hier, vous avez acquis un nouvel utilisateur sur les publicités Facebook et il a effectué le paiement le même jour. Aujourd'hui, vous l'avez retourné par le biais d'une campagne de reciblage, et il a effectué un autre paiement après cela. Comment Appsflyer traitera-t-il l'activité? (Supposons que les paramètres sont standard).

La réattribution sera comptabilisée dans le tableau de bord de reciblage, et le deuxième paiement sera comptabilisé pour cette campagne (et toutes les suivantes dans les 30 jours (sauf si un autre paramètre (30-90) a été spécifié dans le lien - fenêtre de réengagement).

Lien de reciblage Appsflyer


Comme décrit ci-dessus, pour suivre l'activité de reciblage dans Appsflyer, vous devez insérer le paramètre "is_retargeting = true" dans le lien. Lors de la configuration des liens de suivi, ce paramètre peut être la valeur par défaut de chaque lien. Veillez à utiliser le paramètre qui transmet l'ID utilisateur - IDFA ou GAID. Exemple de lien: app.appsflyer.com/myapp?pid=partner&c={mycampaignasket&is_retargeting=true&advertising_id={GAID_macroasket&idfa={IDFA_macro}

En ce qui concerne les ouvertures de liens directs ouverts, ne vous inquiétez pas que les réattributions se produisent sans enregistrer un clic. Cela se produit car les ouvertures de lien profond se produisent sans redirection dans le navigateur. Pour suivre les transitions de lien profond, les paramètres du lien de suivi après le point d'interrogation doivent être remplacés dans le lien profond.

Mesure de reciblage Appsflyer


La partie la plus importante est de savoir comment le tracker mesure l'efficacité du reciblage.
Prenons l'exemple précédent (la veille de l'achat d'un nouvel utilisateur sur les publicités Facebook, et il a effectué le paiement le même jour. Aujourd'hui, vous l'avez renvoyé via une campagne de reciblage, et il a effectué un autre paiement après cela.) Appsflyer comptera la réallocation (régénération) et toutes les activités pendant la fenêtre de redistribution seront enregistrées dans la source de reciblage.

Mais! Les mêmes événements seront comptabilisés dans la source d'installation, dans les publicités Facebook dans ce cas. Qu'est-ce que cela signifie? Appsflyer ne réécrit pas la source d'installation. Chaque utilisateur peut se voir attribuer à la fois une source d'installation et une source de reciblage. Et les événements dans le cadre de la fenêtre de redistribution seront comptabilisés dans les deux tableaux pré-établis: UA et reciblage.

Changez le cas. Supposons que vous ayez attiré un utilisateur de Facebook, il a payé 24 $ jusqu'au moment de la redistribution. Après la redistribution, l'utilisateur a payé 14 $ supplémentaires. En conséquence, dans Appsflyer, vous verrez:

  • Revenus des utilisateurs de Facebook 38 $
  • Reciblage des revenus des utilisateurs - 14 $


Il peut sembler que vous avez gagné un total de 52 dollars. Mais les revenus par utilisateur ne seront que de 38. L'image du bas montre comment Appsflyer comptabilise les revenus par source.



Conclusion


Ne résumez pas les revenus de l'achat de trafic et du reciblage, car le second n'est pas unique: chaque événement de reciblage a une source originale, sur laquelle il est également pris en compte. Vous pouvez en savoir plus sur le calcul de la somme des événements et des revenus dans la documentation Appsflyer .

Pensées principales


  • Le tracker est adapté pour lancer une campagne de reciblage mobile;
  • Appsflyer partage l'activité de reciblage et de campagnes UA - il existe un tableau de bord spécial pour le reciblage;
  • Appsflyer a une fenêtre de redistribution - une période de temps pendant laquelle les événements sont enregistrés dans une source de reciblage;
  • Pour démarrer une campagne de reciblage avec Appsflyer, vous devez insérer «is_retargeting = true» et le paramètre de collecte d'ID de périphérique dans le lien;
  • Appsflyer ne réécrit pas la source d'origine. Chaque utilisateur peut avoir la source d'installation et la source de redistribution en même temps, et un événement unique sera enregistré sur les deux sources.
