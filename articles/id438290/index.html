Post-mortem dengan GGJ-2019: bagaimana cara mendapatkan gundukan, tetapi tetap membuat game

GGJ-2019 adalah selai game global yang diadakan setiap tahun pada akhir pekan terakhir Januari. Ini berbeda dari yang lain dalam dukungan luas (sponsor, semua urusan), sejumlah besar peserta, serta fakta bahwa itu terjadi secara eksklusif offline. Selanjutnya, semua pengalaman yang saya peroleh di acara tersebut akan dijelaskan dalam kaitannya dengan situs SUSU (Chelyabinsk). Kesimpulan yang disajikan dalam artikel ini tidak mengklaim universalitas, tetapi saya harap Anda menemukan mereka berguna.
Spoiler alert: artikel ini memiliki daftar game finalis dari acara tersebut, beberapa di antaranya cukup dapat dimainkan.


Membangun tim


Karena saya terlambat (semua karena kemacetan lalu lintas di jalan raya antar-kota), kebetulan saat kedatangan saya tim sudah terbentuk. Kesimpulan pertama: jangan terlambat untuk pembentukan tim. Pertanyaan yang diajukan dalam acara obrolan mengungkapkan bahwa tidak ada yang membutuhkan programmer JS / PHP / Python dalam tim. Saya hampir tidak menemukan audiens kuliah yang besar ditunjukkan dalam program acara, hanya untuk menemukan bahwa audiensi tertutup rapat. Saya menemukan jumlah penonton melalui obrolan, di mana para peserta pergi, dan menyadari bahwa ada masalah yang signifikan: bahkan wali yang “baik” tidak dapat menjelaskan bagaimana menemukan mereka. Tersesat di suatu tempat di tengah jalan, berkeliaran tanpa tujuan di suatu tempat di salah satu gedung SUSU, memikirkan segala hal yang tidak tahan lama ketika salah satu guru melihat saya, dan bertanya apakah dia bisa membantu saya dengan sesuatu. Saya bersyukur kepada pria ini bahwa dia bisa membimbing saya ke tempat yang tepat. Ngomong-ngomong, dia sangat terkejut bahwa bahkan orang yang bukan penduduk datang ke acara di SUSU. Kemudian, berjalan di sekitar auditorium (dua yang berada di lantai pertama, pergi ke lantai delapan, mungkin ke gedung lain, saya tidak berani takut hilang lagi), saya menemukan bahwa sebagian besar tim akan menggunakan Unity, dua tim Unreal Engine , tim lain memilih Construct. Dari sini saya membuat kesimpulan kedua - layak sebagai programmer ke game tertentu jika Anda menghabiskan waktu di muka untuk mengenal Unity.

Saya punya rencana cadangan: jika saya tidak menemukan tim, saya akan bekerja secara mandiri. Selain itu, saya memiliki satu permainan yang telah lama saya impikan untuk berkembang secara signifikan ke genre tetangga, dan di sini saya memiliki peluang besar, untungnya, tema acara tersebut, “apa arti rumah bagi Anda”, sangat memungkinkannya. Kesimpulan ketiga: memiliki rencana cadangan terkadang berguna.

Diversifikasi


Setelah memperkirakan peluang saya untuk membuat game yang bagus dan / atau menarik dalam pikiran saya, diputuskan untuk menggunakan penyelam untuk mengimbangi kurangnya desain grafis dan desain game, yang, sayangnya, game saya tidak pernah bisa benar-benar membanggakan.
Apa itu penyelam?

Diversifiers * adalah cara yang bagus untuk menambah kesenangan atau menantang diri sendiri untuk menghabiskan akhir pekan. Terkadang mereka bisa sangat berguna dalam mempersempit atau menyempurnakan ide Anda. Ini adalah batasan mutlak yang bisa Anda tambahkan ke game untuk GGJ19 bersama dengan tema utama tema. Ini dapat membantu permainan Anda menonjol dari kerumunan, dan Anda dapat memilih hingga 4 penyelam saat mengirim permainan!

Dari uraian ini jelas bahwa ini seperti pencapaian, bukan hanya untuk pemain, tetapi untuk pengembang game. Ini daftar penyelam yang saya pilih:

Ephemeral - Buat game yang setiap pemain hanya bisa main sekali.
Super Secret Stash - Tambahkan ruang tersembunyi ke gim Anda.
Di halaman web - Gameplay Anda disembunyikan di halaman web yang tampaknya normal.
Tingkatkan dengan lagu - Gim Anda harus bertahan persis selama trek musik.

Tentang permainan


atau bagaimana melakukannya - atau bagaimana tidak melakukannya, mengingat bahwa saya tidak mengambil tempat)

Tidak lama sebelum selesai, dan sekarang saya sudah mulai mengerjakan proyek saya di tumpukan tradisional untuk saya: HTML5.Canvas, Javascript vanilla. Mula-mula bentang alam ditarik, lalu benda-benda, lalu kabut perang. Jika dua yang pertama adalah array dua dimensi yang sudah diisi sebelumnya (sayangnya, ini tidak berubah menjadi bagel), maka kabut perang, meskipun merupakan array, dihitung setiap kali (logis), berdasarkan posisi terbaru dari protagonis. Semua ini dilakukan pada hari pertama, kecuali kabut perang - ia muncul pada malam kerja pertama.

Tentang manajemen waktu, phishing


Pada pagi hari kedua saya mengambil kertas dan menulis jadwal:

9-11 bekerja dengan grafik dan dialog
11-12 bekerja dengan musik
12-13 bekerja dengan endgame.

Ini memungkinkan pekerjaan diselesaikan hampir tepat waktu. Kesimpulan lain : buat jadwal pada hari kedua dan ikuti, catat waktu yang dihabiskan dan tolak detail yang tidak perlu jika Anda tidak punya waktu. Kesalahan utama saya adalah, bagi saya, bahwa saya menempatkan Ephemerality pada penyelam, yang membawa lebih banyak sakit kepala - bagi saya dan para pemain - daripada menjadi baik. Akan lebih baik jika saya membuat generator kartu yang buruk di jam yang sama yang saya habiskan untuk itu. Saya juga menerapkan "Super Secret Stash" dan "Meningkatkan dengan Lagu" pada hari kedua.

Tasogare doki


Anda bermain sebagai gadis kecil yang tersesat. Anda harus menemukan jalan pulang, di mana ibu dan ayah sedang menunggu Anda. Hindari monster atau mereka akan memakanmu. Mereka melihat 1 kotak baik secara diagonal dan lurus. Temukan rumah sebelum musik berakhir atau monster memakanmu!

Namun, itu adalah pengalaman lain yang sangat merugikan saya. Terlepas dari kenyataan bahwa veteran acara ini Nikita Popov (ia berpartisipasi dalam acara itu untuk keempat kalinya!) Memberitahu saya tentang hal ini di tengah hari pertama, saya tidak berhasil membangun kembali diri saya dan tidak punya waktu untuk membangun kembali strategi saya. Halaman game macet

Pengamatan yang menarik: selama penghargaan, para hakim tidak ragu untuk menyebutkan kelemahan dan preferensi mereka dengan keras. Di antaranya adalah: game bertema ruang angkasa (a la Asteroid), game excel (game tentang bunga yang dibuat oleh tim karyawan universitas di mana perlu untuk menyiraminya, berlindung dari hawa dingin, dan sebagainya) dinominasikan, game dengan grafik yang lebih baik, dan seterusnya ... Tahun berikutnya mereka menjanjikan hadiah untuk kode terbaik atau semacamnya. Setelah upacara, untuk berjaga-jaga, saya bertanya apakah orang-orang yang mengacaukan Machine Learning ke permainan tentang tembakan raket di GGJ-2019 dapat mengandalkan mengambil hadiah dalam nominasi ini. Jawabannya mengecewakan: terlepas dari tema hype, Machine Learning bukan asisten mereka di sini. Jadi mereka yang, selama pengumuman hasil wasit, bekerja di notebook, dapat mengandalkan poin bonus nyata dari para juri untuk tahun berikutnya.

Oleh karena itu kesimpulan akhir: di sini tidak ada penyelam akan membantu permainan yang lemah dalam wasit, tetapi pengetahuan tentang tempat "lemah" juri dapat memainkan peran positif.

Saya harap pengalaman saya dicatat karena suatu alasan. Yah, saya harap Anda juga akan masuk ke Tycoon Entry, seperti yang terjadi pada saya dan lusinan orang lain di acara itu.
