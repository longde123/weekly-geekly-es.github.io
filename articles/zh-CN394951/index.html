最简单的方法

许多试图摆脱对新OS版本的强加依赖的人试图做的第一件事是删除Get Windows 10（GWX）应用程序，该应用程序负责更新系统以及侵入式窗口和消息的外观。在某些时候，在下一次系统更新之后，毫无例外，在装有Windows 7和8 / 8.1的系统的所有计算机上都安装了Get Windows 10（GWX）。不幸的是，所有摆脱它的尝试都不会成功-使用多态过滤器绕过技术的Get Windows 10（GWX）在下一次更新期间不可避免地会再次出现在系统中。

由于与Windows 10过渡直接或间接相关的所有内容都通过无关紧要的更新加载到系统中，因此最简单，最有效的方法是通过禁用所有更新来阻止此访问通道。

之后，如果仍然有更新系统的建议，那么使用其中一种与公用程序无关的公用程序清理计算机就足够了，该公用程序删除了与恼人的升级有关的所有内容（例如，Never 10，我不要Windows 10等）。 ）。应该注意的是。为了不损害系统，选择自己的实用程序版本“清洁程序”值得查看已经设法进行测试并确保其正常工作的有经验的用户的建议。GWX控制面板

手动删除更新

当然，如果要减少自动更新功能，则很有可能会丢失将来的关键软件包，从而纠正系统问题并提高抵御大量新威胁的能力。因此，花费一些时间并有选择地仅删除那些引起GWX实用程序安装的更新是合乎逻辑的。

对于Windows 7，在“更新中心”中，您将需要在代码KB3035583，KB2952664，KB3021917，KB2977759和KB3083710下找到更新。在Windows 8，KB3035583，KB2976978和KB3083711上。它们需要从系统中删除。



也可以在命令行（CMD）上进行强制清洁。要实现该目标，请在命令行上键入：wusa / uninstall / KB：3035583，然后使用上面列表中的所有数字（系统上的数字）执行此简单操作。重新引导系统后，您将需要转到“更新中心”并标记“有害”软件包，以防止随后重新安装它们。

微软用于安装无处不在的GWX的另一个“空白”是公司最初发布的用于其他目的的软件包：KB30383710，KB2990214和KB3014460。删除适当的软件包后，应禁止系统在注册表级别对其进行更新，以使软件包在重新引导后不会返回系统。

下一步是在注册表级别禁止OS更新。为此，请在以下地址创建一个名称为DisableOSUpgrade的32位DWORD密钥（如果您没有所需的分区，则还需要创建它）：

HKLM \ SOFTWARE \ Policies \ Microsoft \ Windows \ WindowsUpdate，DWORD值：DisableOSUpgrade = 1
HKLM \ Software \ Microsoft \的Windows \ CurrentVersion \ WindowsUpdate的\ OSUpgrade ， 价值DWORD：ReservationsAllowed = 0



此外，您可以输入一个字符串：HKLM \ SOFTWARE \政策\微软 \的Windows \ GWX， 价值DWORD：DisableGwx = 1
