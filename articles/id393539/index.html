Luncurkan Nextion Editor 0.34 di bawah Wine

Belum lama ini, produk ITEAD yang luar biasa dirilis - layar HMI Nextion. Banyak orang tahu tentang bagaimana Anda dapat menghubungkan layar ke mikrokontroler, misalnya, selain Arduino dan menampilkan informasi grafis. Jika Anda mencoba sedikit, Anda dapat menambahkan panel sentuh resistif dan Anda mendapatkan perangkat sentuhan nyata. Masalahnya hanya kecepatan rendah mikrokontroler dan jumlah RAM yang kecil. Ada banyak pustaka arduino untuk solusi seperti itu, beberapa tidak sepenuhnya kompatibel satu sama lain, dan beberapa sepenuhnya disetel untuk pengontrol tertentu. Ini sangat menyulitkan pengembangan dan dalam beberapa kasus membuat penskalaan menjadi tidak mungkin. Sepertinya pada satu titik, orang-orang dari ITEAD berkata "Cukup untuk menanggung ini" dan meluncurkan perusahaan penggalangan danake generasi layar baru untuk perangkat tertanam. Proyek itu bernama Nextion. Layar ini adalah perangkat untuk menampilkan gambar dan berinteraksi dengan antarmuka grafis, tetapi gambar itu sendiri dan semua pola dijahit langsung ke perangkat dalam bentuk firmware. Layar seperti itu sudah berkomunikasi dengan mikrokontroler melalui UART dan sama sekali tidak memuatnya, yang memungkinkan Anda mengalokasikan sumber daya untuk tugas sensor pemungutan suara, memantau parameter secara real time, pada kenyataannya, apa yang harus dilakukan mikrokontroler.
Untuk pemrograman, atau lebih tepatnya pengembangan firmware, Editor Nextion tersedia.dan hanya di bawah Windows. Ini menyedihkan banyak, karena bahkan pada tahap penggalangan dana, pengembang berjanji untuk merilis editor lintas-platform atau versi untuk Mac dan Linux pada akhir 2015. Rupanya, ada yang tidak beres, karena sampai sekarang komunitas Linux belum melihat alat yang sangat diinginkan, dan para pengembang memberi mereka sarapan. Ternyata, menjalankan editor di Linux sangat mungkin dan penulis dengan senang hati akan memberi tahu Anda cara untuk

segera melakukan reservasi. Hanya mereka yang tahu apa itu anggur dan setidaknya dapat membayangkan bagaimana cara menggunakannya, yang dapat menggunakan instruksi ini. Turun.


Anda akan membutuhkan Wine yang baru diinstal. Pengguna Ubuntu dapat memanfaatkan versi terbaru dari repositori PPA Wine-staging

sudo add-apt-repository ppa:pipelight/stable

sudo apt-get update

sudo apt-get install --install-recommends wine-staging


Maka Anda perlu meletakkan paket kompatibilitas

sudo apt-get install wine-staging-amd64


Saya masih tidak bisa mendapatkan awalan untuk berjalan dengan benar melalui winetricks, tetapi saya berhasil melakukannya melalui PlayonLinux .
Karena itu, letakkan dan jalankan.

Klik pada tombol "Instalasi".



Pilih "Instal program yang tidak ada dalam daftar" dan klik "Next".





Pilih "Instal program pada disk virtual baru" dan klik "Next."



Pilih nama yang berubah-ubah, misalnya, "Nextion" dan klik "Next."



Pilih "Install libraries tambahan" dan klik "Next"



Pastikan untuk memilih "instalasi windows 32 bit" dan klik "Next"



Untuk beberapa waktu, skrip pembuatan awalan akan berfungsi dan kemudian jendela untuk memilih komponen yang diperlukan akan terbuka. Kita perlu menandai dotnet45 dan klik "Next" dan setelah instalasi selesai, menolak untuk menginstal aplikasi dengan menutup jendela.



Setelah semua, Anda harus kembali ke tahap awal dan klik tombol "Instal" lagi, melalui langkah-langkah lagi, tetapi kali ini pilih "edit atau perbarui aplikasi yang ada" dan klik "Next".



Centang kotak "show virtual disks" dan pilih sendiri (I have Nextion).



Kali ini kami memilih "Konfigurasi Anggur" dan "Instal Perpustakaan Tambahan", klik "Berikutnya".



Pastikan untuk memilih "Windows XP", klik "OK" dan lanjutkan.



Kami menandai "ie8" di antara komponen dan kembali ke titik terakhir, menolak untuk menginstal aplikasi.



Itu saja, perfix sudah dirakit, dapat ditemukan di
/home/<username>/.PlayOnLinux/wineprefix/


Anda dapat menggunakannya langsung dari sini, atau Anda dapat menyalinnya ke folder dengan awalan Anda.

Anda masih tidak dapat menginstal lingkungan pengembangan, tetapi Anda dapat menggunakan opsi yang belum dibongkar.
Ngomong-ngomong, penulis telah membuka tiket dengan permintaan untuk mengunggah arsip zip sebagai tambahan pada pemasang, tetapi Anda dapat melakukan hal yang sama , saya pikir ini akan mempercepat penampilan kit distribusi =)

Untuk memulai secara pribadi, penulis menggunakan q4wine yang tidak ramah karena kebiasaan. Anda dapat menggunakan apa yang Anda suka dengan opsi ini.

Itu saja, Anda dapat menggunakan lingkungan pengembangan di Linux. Tampaknya semuanya dikompilasi, tetapi penulis tidak bertanggung jawab atas bug dan gangguan.
Seharusnya entah bagaimana hasilnya .

NB Jika Anda sangat malas dan Anda pecinta solusi cepat, maka Anda dapat menggunakan awalan atau skrip yang sudah disiapkan dan arsip ZIP dengan penginstal yang belum dibongkar . Saya berharap bahwa paket ZIP segera akan muncul di situs dan akan dimungkinkan untuk menambahkan solusi TurnKey lengkap berdasarkan PlayOnLinux.

Nikmati menggunakan =)
