GitLab 11.6 dirilis dengan fitur serverless dan cluster Kubernetes untuk grup


Kami dengan bangga mempersembahkan rilis GitLab 11.6, di mana kami memperluas kemampuan arsitektur tanpa server di GitLab dan menambahkan cluster grup Kubernetes untuk menyederhanakan pekerjaan dengan infrastruktur cloud asli.


Menyebarkan perkembangan tanpa server ke cloud apa pun melalui GitLab


Komputasi tanpa server diproses secara dinamis, dan ketika kode dieksekusi, sistem mengakses sumber daya cloud, sehingga mengoptimalkan alokasi dan distribusi sumber daya yang diperlukan untuk menjalankannya. Popularitas pendekatan ini di kalangan pengembang semakin meningkat, karena memungkinkan mereka untuk fokus pada apa yang benar-benar penting - misalnya, pada penulisan kode - dan tidak memerlukan pemahaman tentang infrastruktur yang mendasari di mana perangkat lunak Anda akan bekerja.


Komputasi tanpa server di GitLab menggunakan Knative, platform Kubernetes yang dirancang untuk membuat, menyebarkan, dan mengelola proyek tanpa server. Fitur ini memberikan pengembang cara mudah untuk membuat pengembangan tanpa server mereka dan mengelolanya bersama dengan kode lain dalam antarmuka yang sudah akrab. Untuk bisnis, arsitektur tanpa server memungkinkan Anda untuk menggunakan strategi multi-cloud dan menghindari ikatan dengan penyedia cloud tertentu.


GitLab Terus Menyederhanakan Pengembangan Aplikasi Cloud Asli


Dengan registry kontainer bawaan dan integrasi Kubernetes, GitLab memungkinkan Anda memulai dengan kontainer dan pengembangan cloud lebih mudah dari sebelumnya. Dimulai dengan rilis 11.6, pengguna dapat membuat kelompok grup Kubernetes yang dapat digunakan di semua proyek dalam suatu grup atau subkelompoknya. Ini semakin menyederhanakan konfigurasi proyek cloud asli dan memungkinkan pengembang untuk fokus pada pengembangan aplikasi berkualitas.


Untuk liburan, kami merilis banyak fitur keren.


Favorit kami: perubahan yang diajukan , terminal web di IDE Web dan grafik kerentanan di panel keamanan grup . Kontribusi tim sekarang lebih mudah untuk diterima, karena perubahan yang diusulkan dalam komentar untuk perbedaan permintaan penggabungan (di lokalisasi Rusia "permintaan penggabungan" GitLab) dapat diterima dengan satu klik. Juga dalam rilis ini, fitur pertama dari IDE Web yang berjalan di server, terminal web, menjadi tersedia. Bagan kerentanan baru muncul di panel keamanan grup yang ditambahkan dalam rilis terakhir, yang menunjukkan perubahan dalam jumlah kerentanan setiap hari, memungkinkan Anda untuk melacak metrik perubahan.


Baca terus dan Anda akan mengetahui semua tentang inovasi liburan dalam rilis GitLab 11.6!

Fungsi tanpa server (versi alpha)


(INTI, STARTER, PREMIUM, ULTIMATE, GRATIS, BRONZE, SILVER, EMAS)


Berdasarkan integrasi dengan Knative ( artikel asli , terjemahan ) yang ditambahkan dalam rilis GitLab 11.5, kami menyajikan fitur baru - fungsi tanpa server di bawah kendali Knative, yang dapat dibuat pengguna dalam repositori mereka.


Anda perlu mendefinisikan fungsi-fungsi Anda di file serverless.yml dari repositori dan menggunakan .gitlab-ci.yml . Setiap fungsi akan ditempatkan pada cluster Anda, dan Knative akan mengurus semuanya dan ekspansi berdasarkan sumber daya yang diperlukan. Fitur ini akan memungkinkan pengembang aplikasi untuk merilis versi baru lebih cepat tanpa khawatir tentang pengaturan dan pemeliharaan infrastruktur.

(INTI, STARTER, PREMIUM, ULTIMATE, GRATIS, BRONZE, SILVER, EMAS)


Memulai pekerjaan tertentu dalam hal permintaan penggabungan menjadi lebih mudah. Ini memberikan lebih banyak kontrol atas perilaku pipa dan juga membuka akses ke variabel lingkungan baru yang menunjuk ke cabang target dan menggabungkan ID permintaan, yang memungkinkan Anda untuk menerapkan strategi perilaku lainnya.

Terminal Web untuk IDE Web (Beta)


(ULTIMATE)


Web IDE membuatnya lebih mudah dan lebih cepat untuk melakukan perubahan dan memproses umpan balik permintaan menggabungkan, menghilangkan kebutuhan untuk menyimpan perubahan dan mengubah cabang secara lokal. Tetapi membuat perubahan besar tanpa terminal untuk menjalankan tes, bereksperimen dengan REPL (Read-Eval-Print Loop - loop "baca-hitung-output") dan mengkompilasi kode tidaklah mudah.


Sekarang melalui IDE Web Anda dapat meluncurkan terminal web, bekerja di editor dan memiliki terminal di tangan seperti Anda akan bekerja secara lokal - melacak apa yang datang melalui API atau memeriksa sintaks Anda di REPL. Terminal web adalah fitur pertama dengan pemrosesan sisi server di IDE Web, dikonfigurasi menggunakan file .gitlab/.gitlab-webide.yml .


Terminal web interaktif belum tersedia di GitLab.com dan saat ini perubahan tidak disinkronkan antara editor dan terminal web. Anda dapat mengikuti kemajuan pengembangan di sini , dalam rilis berikutnya kami akan menambahkan dukungan untuk perubahan mirroring dan pratinjau waktu-nyata .
