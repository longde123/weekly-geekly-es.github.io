知识产权管理问题


在开始讨论推动新范式形成的问题之前，我要说的是，在我工作过的所有公司中，只有17％的公司可以提供知识产权的整个生命周期，但事实并非如此。

还应理解，“管理”始终是具有以下已定义定义的过程：
管理是一个过程，在此过程中，某人领导一群人，他们在其中执行某人的某些特定要求。
对于大多数人而言，不幸的是，管理始终是恐惧与痛苦 ， 控制与力量的代名词，但在生活中并非总是如此。 此安装扩展到IT公司的知识产权管理。 结果，许多人没有注意到为什么正是由于俄罗斯联邦无法管理知识产权而没有真正的IT公司。

∞生命管理


在深入研究问题并回答“为什么？”之前，我想举几个可能进行管理的生活领域的例子。

我数了八个不断变化的领域。

  • 业务-每个人都可以管理它，了解增加销售需要什么。
  • 第二个区域是状态。 在这里也很清楚，谁来控制它以及有效管理需要什么。
  • 第三个方面是一个通常易于管理的人，会有钱，但是正如一个人所说， “有钱，有钱，没有钱！” ，另一人继续说： “没有钱，但您坚持！” 。
  • 第四个领域是社会。 这很难管理，金钱对社会没有关键作用。
  • 第五领域是文明。 管理起来更加困难。
  • 最后-这些是自然和机器人。 理论上的那个和另一个都是可控的，但实际上并非总是可控的。

IT公司的知识产权


俄罗斯联邦民法典第1225条对知识产权进行了以下定义：
1.受到法律保护（知识产权）的法人，商品，工程，服务和企业的智力活动和等同的个性化手段，其结果是：

1）科学，文学和艺术作品；
2）电子计算机程序（计算机程序）；
3）数据库；
4）性能；
5）留声机；
6）广播或有线广播或电视广播（广播或有线广播组织的广播）；
7）发明；
8）实用新型；
9）工业设计；
10）选拔成绩；
11）集成电路拓扑；
12）生产秘密（专有技术）；
13）公司名称；
14）商标和服务标志；
15）原产地名称；
16）商业名称。

2.知识产权受法律保护。
在俄罗斯，知识产权主要在原始社区级别进行管理，包括“收集”：

  • 各种品牌，商品名称，商标和服务标志；
  • 撰写了各种关于不公开知识产权信息的协议和限制，只忘记了专门与生产有关的知识，而与计算机程序的供应无关，甚至与许可有关；
  • 出版物，彩色新闻资料袋，带有小册子的商品目录和网站设计布局以及商品和服务目录。

创建计算机程序的过程的解释


在考虑创建计算机程序的传统过程之前，应该理解，这里有一个IT业务，只是有一个业务充当IT业务的客户。

创建计算机程序的传统过程

如果我们考虑按照这种方式进行编程，则创建计算机程序的过程如下：

  1. 市场研究和业务需求的识别；
  2. 以开发计算机程序的技术规范的形式体现业务需求；
  3. 创建计算机程序；
  4. 测试计算机程序；
  5. 计算机程序的实现和复制。

有些公司甚至可能在计算机程序中注册了精美的证书。

然后，由于对信息交换方式的新解释的出现，业务需求发生了变化，业务所有者决定重写计算机程序以满足新的需求，并且该循环一次又一次地重复。 许多IT公司为此过程和品牌投入了数十亿美元的资金，甚至没有试图满足业务需求，只是保存他们的品牌，有时是以品牌本身为代价。

即使新技术为企业所熟知，但由于惯性，它将不断忽略它并通过不断升级计算机程序来保存品牌，从而重复创建计算机程序的传统周期。

结果，在第n次变更周期（有些人错误地认为这是敏捷）之后，由于公司或州一级的价值变更，计算机程序可能无法通过测试。


有一个原因：
学会和国家都不知道您以计算机程序形式表达的需求。 他们看到的只是您的品牌。

因此，为了节省业务，经理建议为每个州，每个社区编写有关技术交互的协议，并收到成百上千的需要关注和金钱的协议。 缔结所有这些协议的唯一目的是尽早获取有关“一项新技术已经出现”事件的信息。

在门户网站“ Rosneft.ru”，“ Kremlin.ru”和“ zakupki.gov.ru”上搜索“协议”标签的结果。

如果从这种角度考虑知识产权的生命周期，那么由于缺乏知识产权交换，俄罗斯联邦的IT公司无法管理任何事情。

知识产权的交换总是意味着从一个管理领域转移到另一个管理领域。 没有全额转让，就没有交换。 部分传输-不起作用。 因此，当一个人收到临时使用的计算机程序时，据他所知，他不是完全所有者，因此，使用知识产权的所有责任均由企业承担。

这个商家知道吗？ 我认为不会，否则他将不会花费数十亿美元来重复传统程序。

回到艺术的可能性


许多人坚信这些是橘子，但也许只是秋天。

人主要通过沉思与艺术进行互动，这种沉思旨在鼓励创造新的形式，但是对于某些人来说，艺术是始终需要交换金钱的产品。 与社会分离的个人或企业始终需要不断提醒自己： 可以感动，无法谈论的内容，可以谈论的内容以及保密协议在哪里生效 。

实际上，计算机程序可能与Art没有什么不同。

为此，您必须遵守两个主要原则。

第一个原则是作者原则
承包商始终是客户的作者或合著者。

第二原则
客户可以在任何时候将知识产权作为公共领域的对象，承包商只有在N年后才有权使用此权利。

根据这两个原则，IT公司始终计划编写新的计算机程序 ，从而使企业能够获得专有知识产权。

第二个原则允许客户为企业客户启动知识产权复制过程。

结果，创建计算机程序的过程变成了以下过程：


  1. 选择要为其创建计算机程序的管理区域
  2. 在选定的管理领域进行市场调查和确定业务需求
  3. 以开发计算机程序的技术规范的形式实现业务需求
  4. 创建计算机程序
  5. 考虑选定管理领域的计算机程序测试
  6. 实施和转让计算机程序
  7. 关于将知识产权转移到公共领域的决策
  8. 在公共资源库中发布计算机程序。

由于IT公司总是创建新的计算机程序，因此其成本将比传统的计算机程序创建过程中的成本略高。 但是，由于不受客户对源代码的可复制性和修改的限制以及源代码本身的排他性，因此以这种方式开发的计算机程序将不仅在业务领域而且在其他选定领域也具有更高的价值。

PI许可


PI许可证和其他免费许可证之间的主要区别是排他性和归属。

许多人可能会注意到与免版税许可的类型有一些相似之处，但事实并非如此，这是与该许可的主要区别，我称之为PI-License。
PI-许可证 免版税 CC0 CC归属
成本低 没有啦 是的 是的 是的
排他性 是的 没有啦 没有啦 没有啦
局限性：流通，使用方法 没有啦 是的 没有啦 没有啦
归因 是的 没有啦 没有啦 是的
代码修改 是的 是的 是的 是的

因此，创建的程序将转移给客户，而承包商保留作者的所有权，并在N年后自动将其转移到公共领域（CC0）。
知识产权向公共领域的转移使我们既不像现在这样在狭here的志同道合的人中谈论我们的需求，但这种情况提供了将知识从一个政府领域转移到另一个政府领域的机会。

PI许可证壁垒


  1. 缺乏能够编写新计算机程序的合格IT人员
  2. 对IT业务缺乏重建流程的渴望
  3. 数量有限的适合免费使用的编译器，软件组件和库
  4. 计算机程序的庞大性以及对第三方组件和库的高度依赖
