Apa itu Peringatan Email?

"Peringatan Email" adalah fungsi dari router Vigor , yang digunakan untuk memberi tahu pengguna bahwa router sedang mengalami serangan DoS atau menunjukkan internal yang diblokir oleh aturan filter URL yang coba dikunjungi oleh situs web pengguna internal.



Seri Vigor 2920T ditampilkan di bawah ini untuk demonstrasi untuk menjelaskan cara memasang DoS Defense.

Jika Anda ingin menerima informasi tentang serangan DoS, silakan periksa Aktifkan Pertahanan DoS sebelum menyiapkan Pemberitahuan Email.
1. Masuk ke WUI Vigor 2920.
2. Buka Firewall >> DoS Defense Setup (Firewall >> DoS Security Settings).
3. Tandai item yang diperlukan.



Jika Anda ingin menerima informasi dari Log Akses URL, harap aktifkan fungsi Aktifkan Log Akses URL sebelum mengatur Pemberitahuan Email.
1. Masuk ke WUI Vigor 2920.
2. Buka CSM >> Profil Filter Konten URL (CSM >> Filter Profil Konten URL).



3. Klik Indeks 1 dan buka halaman berikutnya.



4. Buka Pemeliharaan Sistem >> Syslog / Peringatan Email dan konfigurasikan Peringatan Email sebagai berikut.



Server SMTP Masukkan alamat IP dari server SMTP yang Anda gunakan untuk mengirim peringatan email.
Kirim Masukkan alamat email yang akan menerima pemberitahuan kesalahan email.
Jalur balik Masukkan alamat email kotak surat lain untuk menerima semua pesan yang dikembalikan jika terjadi masalah serius dengan kotak surat penerima.

Alamat email yang diketik di area ini juga bertindak sebagai alamat Pengirim sementara router Vigor mengirim alamat email alternatif.

Otentikasi Jika server SMTP yang Anda konfigurasi perlu mengautentikasi pengguna, harap aktifkan dan masukkan nama pengguna dan kata sandi yang sesuai dalam paragraf berikut.

Nama Pengguna Silakan masukkan otentikasi nama pengguna server SMTP.
Kata Sandi Silakan masukkan otentikasi kata sandi server SMTP.

Catatan: Hanya alamat IP server SMTP yang dapat dimasukkan di bidang server SMTP. Anda tidak dapat memasukkan nama domain dari server SMTP secara langsung. Untuk mendapatkan alamat IP dari server SMTP, gunakan perintah nslookup untuk mendapatkan alamat IP.



Catatan: Ada beberapa server SMTP gratis tanpa otentikasi di Internet. Jika Anda memasukkannya sebagai server SMTP, maka Anda tidak perlu mengaktifkan elemen Otentikasi dan mengkonfigurasi Nama Pengguna dan Kata Sandi.

Setelah menyelesaikan pengaturan di atas, Anda akan secara berkala menerima pemberitahuan melalui email (termasuk tentang serangan DoS dan URL Access Log). Jika Anda membiarkan Jalur Pengembalian kosong, Anda akan menerima pemberitahuan berikut melalui email. Pengirim akan ditampilkan dengan "MAILER-DAEMON".



Catatan: Jika Anda memilih Keduanya di bidang Prioritas di CSM >> Profil URL Content Filter (CSM >> Profil URL Content Filter) dan beberapa pengguna internal mencoba mengunjungi situs web, Anda akan menemukan entri yang relevan, misalnya , "CF Keyword Block 172.17.1.21.38096 -> www.google-analytics.com , 80 PR tcp len 20 861 -PA -1618639177 -479936808."

Jika Anda mengkonfigurasi fungsi Jalur Kembali, Anda akan menerima pemberitahuan berikut melalui email. Item “Pengirim” dari notifikasi surat akan menampilkan kotak surat yang Anda konfigurasikan dalam item “Tujuan kembali”.
