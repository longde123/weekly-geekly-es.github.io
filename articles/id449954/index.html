Instal Zimbra Open-Source Edition pada CentOS 7

Selama desain implementasi Zimbra di perusahaan, manajer TI harus memilih sistem operasi di mana node infrastruktur Zimbra akan beroperasi. Saat ini, hampir semua distribusi Linux kompatibel dengan Zimbra, hingga OS merah domestik dan ROSA. Biasanya, pilihan untuk menginstal Zimbra di perusahaan adalah Ubuntu atau RHEL, karena perusahaan komersial mengembangkan distribusi ini. Namun, sering kali, pilihan manajer TI jatuh pada Cent OS, yang merupakan fork-driven dari distribusi RHEL komersial dari Red Hat.



Persyaratan sistem minimum Zimbra meliputi 8 GB RAM di server, setidaknya 5 gigabytes ruang kosong di folder / opt, serta nama domain yang lengkap dan data MX. Sebagai aturan, masalah terbesar pemula justru muncul dengan dua poin terakhir. Keuntungan besar dari CentOS 7 dalam hal ini adalah memungkinkan Anda untuk menetapkan nama domain server pada tahap pemasangan sistem operasi. Ini memungkinkan Anda untuk menginstal Zimbra Collaboration Suite tanpa masalah, bahkan untuk pengguna yang sebelumnya tidak memiliki pengalaman dengan Linux.

Dalam kasus kami, nama domain server tempat Zimbra akan diinstal adalah mail.company.ru. Setelah penginstalan selesai, tetap hanya menambahkan baris dari formulir 192.168.0.61 mail.company.ru mail , di mana alih-alih 192.168.0.61, perlu mendaftarkan alamat IP statis server Anda. Setelah itu, Anda perlu menginstal semua pembaruan paket, serta menambahkan catatan A dan MX ke server menggunakan perintah dig -t A mail.company.ru dan menggali -t MX company.ru . Dengan demikian, server kami akan memiliki nama domain lengkap dan sekarang Anda dapat dengan mudah menginstal Zimbra di atasnya.

Anda dapat mengunduh arsip dengan versi distribusi Zimbra saat ini dari situs resmi zimbra.com . Setelah arsip dibongkar, tetap hanya menjalankan skrip instalasi bernama install.sh. Serangkaian perintah konsol yang Anda butuhkan untuk ini adalah sebagai berikut:

mkdir zimbra && cd zimbra
wget files.zimbra.com/downloads/8.8.12_GA/zcs-8.8.12_GA_3794.RHEL7_64.20190329045002.tgz --tidak periksa sertifikat
tar zxpvf zcs-8.8.12_GA_3794.RHEL7_64.20190329045002.tgz
cd zcs-8.8.12_GA_3794.RHEL7_64.20190329045002
./install.sh



Segera setelah ini, program pengaturan Zimbra Collaboration Suite dimulai. Pertama-tama, Anda harus menerima perjanjian lisensi untuk melanjutkan instalasi ZCS. Langkah selanjutnya adalah pemilihan modul untuk instalasi. Jika Anda ingin membuat satu server surat, masuk akal untuk menginstal semua paket sekaligus. Jika Anda bermaksud membuat infrastruktur multi-server dengan kemampuan untuk menskala, Anda harus memilih hanya beberapa paket yang diusulkan untuk instalasi, seperti yang dijelaskan dalam salah satu artikel kami sebelumnya.

Setelah instalasi selesai, menu pengaturan Zimbra akan terbuka langsung di terminal. Jika Anda memilih instalasi server tunggal, Anda hanya perlu mengatur kata sandi administrator. Untuk melakukan ini, pertama pilih item nomor 7, dan kemudian item 4 untuk mengatur kata sandi administrator, yang harus minimal 6 karakter. Setelah kata sandi ditetapkan, tekan tombol R untuk kembali ke menu sebelumnya, lalu tombol A untuk menerima perubahan.

Setelah instalasi Zimbra selesai, buka port yang diperlukan untuk operasinya di firewall dengan perintah firewall-cmd --permanent --add-port = {25,80.110,143,443,465,587,993,995,5222,5223,9071,7071} / tcp , dan kemudian mulai ulang firewall dengan perintah tersebut firewall-cmd --muat ulang

Sekarang kita hanya perlu memulai Zimbra dengan perintah zimbra layanan mulai untuk memulai. Anda dapat mengakses konsol administrasi di browser dengan membuka company.ru : 7071 / zimbraAdmin / . Akses untuk pengguna email akan diberikan di mail.company.ru . Perhatikan bahwa jika ada masalah atau kesalahan selama operasi Zimbra, jawabannya harus dicari dalam log, yang dapat ditemukan di folder / opt / zimbra / log .

Setelah instalasi Zimbra selesai, Anda juga dapat menginstal ekstensi Zextras Suite, yang dapat meningkatkan keandalan dan efektivitas biaya menggunakan Zimbra dengan menambahkan fitur yang relevan dengan bisnis ke dalamnya. Untuk melakukan ini, Anda perlu mengunduh arsip dengan versi terbaru Zextras Suite dari Zextras.com dan membongkar. Setelah itu, buka folder yang belum dibongkar dan jalankan skrip instalasi. Seluruh proses dalam bentuk konsol adalah sebagai berikut:

wget download.zextras.com/zextras_suite-latest.tgz
tar xfz zextras_suite-latest.tgz
cd zextras_suite /
./install.sh semua



Setelah itu, Zimbra Anda akan memiliki kesempatan untuk mengarsipkan dan mendeduplikasi data dalam penyimpanan surat, me-mount volume sekunder, mendelegasikan hak administrator untuk pengguna lain, menggunakan obrolan online langsung di klien web Zimbra, dan banyak lagi.

Untuk semua pertanyaan yang terkait dengan Zextras Suite, Anda dapat menghubungi perwakilan perusahaan "Zextras" Katerina Triandafilidi melalui email katerina@zextras.com
