IPv6的情况如何，或阻碍过渡到该协议的新版本的原因-讨论情况

IPv6 诞生于二十多年前。 一直以来，它尚未得到广泛的分发，至少其实施进展缓慢。 例如，据Google称，只有24％的用户通过IPv6连接到公司的搜索资源。 今天，我们将向您介绍该协议在世界范围内的发展情况，并讨论了为什么公司仍应考虑实施该协议。


/ Flickr / 克里斯蒂安·科伦 / cc

世界形势-IPv6实施情况


自2009年以来，Google一直在收集与其服务的IPv6连接的统计信息。 根据该公司网站上显示的时间表，使用新协议的用户略多于20％。 其中大多数在美国（将近39％），德国（38％）和印度（34％）。

对于俄罗斯，此处只有1.3％的Google服务用户使用新协议。 但是，随着越来越多的运营商和Internet提供商向其客户提供IPv6，这一数字正在逐渐增加。 例如，由于许多客户要求我们这样做，因此我们在1cloud中也支持该协议的新版本。

除了Google之外，互联网注册商还监视IPv6的实施进度。 APNIC实验室的研究负责人Geoff Huston表示，通过IPv6访问Internet的大多数用户位于印度（44％）和美国（21％）。 与Google统计数据的差异可能是由于该IT巨头最近才开始在印度市场积极推广其产品 。

是什么阻碍了向IPv6的过渡？


技术难点

人们认为，构建单独的IPv6网络可能会花费大量时间。 在某些情况下，这是真的。 例如，由视频游戏开发公司组成的SIE Worldwide Studios已实施“ IPv6项目”已有7年了。 该团队不得不重新考虑网络体系结构，摆脱NAT和过时的防火墙规则。 但是，到目前为止，它们还不能完全消除IPv4的“遗留”：它们都还使用专用IP地址（ RFC 1918 ）和/ 26和/ 30的CIDR范围。

长期过渡的原因之一是公司网络的规模 ：它涵盖了约14个开发工作室，配备了1.5万台设备。 同时，当团队开始其有关IPv6网络实施的项目时，许多硬件和软件与新协议不兼容。

从那时起，情况有所改善，硬件默认情况下支持IPv6，并且网络上有大量的指南来实现新版本的协议。 它们没有考虑所有可能的困难，但是可以让您在更短的时间内完成转换。 许多公司已经迁移。

同时，互联网提供商和云运营商简化了使用新协议的工作。 许多供应商已经支持IPv6，可以帮助实现。

“当然，如果该公司没有简化迁移的工具和指南，那么协议的适应速度就会放慢，” 1cloud开发部门负责人谢尔盖·贝尔金（Sergey Belkin）说道。 -要迁移到IPv6，您需要从业务的问题，需求和需求开始，仔细制定迁移计划。

为了简化针对客户的新版本协议的工作，我们决定针对用户的常见问题准备一系列分步说明，并在其中提供一些设置建议。 我们希望这将有助于减轻实施IPv6的困难。”

昂贵的

英国互联网服务提供商Spitfire Network Services的工程师Graham Lewis 强调 ，在一切都可以在IPv4上运行之前，不会有大规模的协议转换为新版本。 一些公司担心迁移将不得不花费大量资金购买新设备，并且只有大型IT公司才能负担得起IPv6的大规模试验。

但是，社区已经提出了某些可以降低成本的方法。 例如，购买新设备可以“定时”到服务器机队的计划升级，这使您可以用一块石头杀死两只鸟。 而且，如果公司使用云提供商的服务，那么他可以自己承担一部分费用。 在这种情况下，您只需付费即可连接新的IP地址。

不同的提供商有不同的价格，但是，总的来说，该价格应低于网络开发所能使用的价格。 例如，在1cloud中，连接新的IPv6的费用为50卢布。 每月的维护费用也一样（对于第一个地址，您无需支付“订阅费”）。 另外，如有必要，您可以关闭IPv6服务以节省金钱。

怀疑地址没有结束

互联网注册商长期以来一直在警告IPv4地址即将用尽。 RIPE在四月提供了最后一个免费块，而其他注册服务商在很久以前就用完了块-2011年有人从中释放出块。 最后一个仍具有IPv4地址的注册商是AFRINIC ，但是其“资源”即将终止（大概在明年4月）。

IT社区的一些成员认为，无需IPv6即可解决IPv4耗尽问题。 您只需要返回无人认领的地址，然后开始将它们按实际需要的数量而不是成块地发行给公司。 但是有一个问题-由于没有保留其注册表，因此很难找到这些地址。

解决该问题的另一种方法可能是在特殊拍卖中买卖地址。 例如，一年前，麻省理工学院（MIT） 发现了 1400万个未使用的IP地址。 其中八百万决定出售。 但是，有一个困难。 地址的不变（和不受控制）转售会导致路由表增加，这将导致路由器的内存问题。

为什么仍值得实施IPv6


实际上，IPv4从来都不是正式版本。 Vint Cerf是Internet的“父亲”之一，也是TCP / IP协议栈的开发人员，他解释说 ：“创建IPv4协议时，开发人员不知道将有多少设备连接到网络。 因此，地址的数量是根据地球上的居民数量计算的。 实验的结果表明世界上有多少个需求可以解决之后，它计划推出该协议的“成熟”版本。”

但是IPv4摆脱了实验的限制，并开始在各地使用。 因此，许多公司打算切换（或已经切换）到该协议的新版本。 IPv6不仅被设计为替代结尾的IPv4，而且从根本上来说是具有其他功能的新解决方案。

IPv6的好处之一是可以选择退出NAT。 该协议的新版本使您可以为网络上的每个设备分配一个特定的地址，从而消除了识别用户的困难并简化了路由 。

像许多其他技术一样，NAT完全是一种既可以做得更好又可以做坏的工具。 是的，它允许您通过将多个内部IP地址转换为一个外部公共IP地址来节省地址，” Sergey Belkin说。 -但是，此功能会产生许多问题。

例如，如果在使用NAT的公司中，所有员工都将使用同一资源，则服务器可能会“认为” DoS已经开始。 它将阻止访问具有公司IP的所有设备。 向IPv6的过渡-每个设备都有其自己的地址-完全杜绝了“错误” DoS攻击的情况。“

切换到IPv6的另一个原因是它提高了性能。 在欧洲，非洲和大洋洲，新协议比IPv4更快。 在亚洲和美洲，新旧协议的速度大致相同，但是这种情况在可预见的将来将不得不改变。 根据APNIC的杰夫·休斯顿（Jeff Houston）的说法，当每个人都实现IPv6时，协议性能将显着提高，并且将在连接建立速度和可靠性方面绕过IPv4。

结果如何


Google上新协议的适应曲线正在上升。 但是，何时向全球过渡到IPv6仍是一个悬而未决的问题。

有人推测，到2021年，数千个最受欢迎的站点将支持 IPv6。 据英国IT记者兼PC Pro专业人士Derien Graham-Smith（Darien Graham-Smith）的编辑所说，为了向IPv6进行大规模过渡，一切准备就绪，公司只需要决定这一步骤即可。 区域互联网注册商的代表预测 ，由于IPv4地址的耗尽，在未来10年内将向IPv6迁移。

格雷厄姆·史密斯（ Graham Smith）说，就像是开玩笑的一半，认真的一半一样，在这种情况下，您至少要再考虑一百万年就不必考虑IP地址用完了。

您可以立即开始尝试IPv6网络。 例如，在1cloud.ru上，您可以在几分钟内部署虚拟测试环境 。 IPv6可以在我们所有站点上连接，但圣彼得堡的SDN-1除外（该服务将在稍后出现）。
