异步通信是远程工作效率更高的真正原因。

现在异步通信不仅在远程站点上实现



插图： 尹为鸿

一项又一项的研究 证明 ，远程工作者比办公室的同事生产力更高。

唯一不清楚的是为什么。

是的，人们可以避免高峰时段的交通拥堵，从而节省时间（和原因）。 他们不受办公室分心的影响。 他们对工作时间表有控制感。 还有更多的时间留给家人，朋友和爱好。

但是，除了上班外，所有这些优势并不是异步通信的副产品-自主位置的结果-员工决定何时与同事进行通信。

许多经理考虑组织远程工作，但很少有人考虑更多异步通信的想法。 尽管我相信未来取决于远程工作 ，但异步通信是团队绩效中更为重要的因素，无论它是否可以远程工作。 异步不仅可以提供最佳结果，还可以使人们做更多重要的工作，并过着更加自由和充实的生活。

根据我自己的远程和很大程度上异步公司Doist的经验，我将尝试解释什么是异步通信，它如何提高生产率以及您可以采取哪些具体步骤来创建更异步的环境。

什么是异步通信？


本质上， 异步通信是指您在不等待立即响应的情况下发送消息 。 例如，一封电子邮件。 我在几个小时内打开并回答信件。

相反， 同步通信是指您发送消息时，收件人立即读取消息并立即答复 。 像会议一样的个人通讯是纯同步通讯的示例。 您说些什么，我会立即收到信息并立即做出回应。

但是，数字通信形式（例如实时消息传递）也可以是同步的。 您发送一条消息，我收到通知，然后打开Slack实时阅读消息并回复。 甚至电子邮件也主要被视为一种同步的通信形式。 Yahoo Labs 在 2015年进行的一项研究表明，对电子邮件的平均响应时间仅为两分钟。

在探讨更异步的协作方法的好处之前，让我们看一下为什么我们应该质疑我们目前（主要是同步的）工作和交流方式。

实时不断沟通的问题


如果员工在办公室外工作效率更高，则办公室出了点问题。

根据《哈佛商业评论》上的“ 协作超载”文章，在过去的20年中，员工开始将更多的时间花在与同事沟通上50％。 最多有80％的时间用于电子邮件通信（平均每天六小时 ），会议（平均15％ ）以及最近的即时通讯应用程序（平均来说，Slack发送200条消息到天 ，尽管最活跃的消息一千条也不例外）。

这种趋于持续不断的交流的趋势意味着，一般的脑力劳动者应该围绕几次会议计划一个工作日，而在此期间，他会心不在works地工作，被电子邮件和Slack分散注意力。

更糟糕的是，移动技术的发展意味着工作通讯不再局限于实际的工作场所或工作时间。 我们可以在一天中的任何时间检查电子邮件并回复邮件。 结果，我们永远不会完全关闭电源。 正如一位上班族在《纽约杂志》的评论中所说：“我曾经醒来，关闭闹钟，然后检查Tinder。 现在，我醒来并查看Slack。”


Slack声称用户在工作日要花费9个小时或更多时间。 90分钟的主动使用时间长达9个小时，这是一大堆干扰

如果能产生结果，那么这种异常同步的工作方式将是合理的。 但是，越来越多的证据表明，实时通信的开销使得难以专注于工作，耗尽了智力资源，并且通常使得难以取得有意义的进步。

我已经反对实时消息传递应用程序了 ，但是值得总结一下适用于大多数形式的同步通信的主要问题：

  • 不断分心 。 它们分散了注意力并阻碍了工作进度。 诸如编程，编写，设计，制定策略和解决问题之类的高效，认知要求很高的活动需要长期深入的，专注的工作 。 同步通信不允许为此分配长时间。

    “深层工作”一词是由乔治敦大学计算机科学教授和Cal Newport著作的作者介绍的

  • 连接优先而不是生产力 。 实时地，鼓励您保持联系并随时保持联系。 如果断开连接，则讨论将在没有您的情况下继续进行，因此您将无法回答甚至看不到它们。 为了不错过重要的决定和讨论，人们尝试始终保持在线状态，并参加尽可能多的会议，这会损害他们的健康和生产力。
  • 不必要的压力 。 永久可用性意味着员工无法控制自己的时间。 白天，他会响应请求，而不会主动设置自己的时间表。 一项研究表明，人们补偿因分心而浪费的时间 ，试图更快地工作，这会导致“更多的压力，沮丧，浪费更多的精力和压力”。 同步文化直接导致倦怠。
  • 讨论质量下降和解决方案欠佳 。 当您需要立即回答时，人们没有时间仔细思考关键问题并给出深思熟虑的答案。 您对这种情况的第一反应通常不是最好的。

异步环境的好处


大多数人将分散注意力视为例行公事，但是Doist， Gitlab ， Zapier ， Automattic和Buffer等一些公司正在尝试实现异步方法。 以下是更好地控制个人通讯的一些主要好处：

  • 工作时间控制=更快乐和更高效率的员工 。 在异步环境中，没有设置的工作时间。 员工几乎可以完全根据自己的生活方式，生物节律和职责（例如，照顾孩子！）来安排工作日的方式。 晚上有些工作，因为对他们来说更方便。 每天早上，我都会与儿子待一个小时，异步组织中没有人会注意到这一点。
  • 高质量的沟通，而不是肤浅的回答 。 诚然，异步通信速度较慢，但​​通常质量较高。 人们学会更清晰，彻底地交流，避免不必要的chat不休。 他们有时间思考特定的问题或想法，并给出更周到的答案。 人们可以反省自己的反应，而不必为自己的反应做好准备（作为一个额外的优势：当我们有时间去思考它时，我们不太可能看到沉思的情绪爆发，在过去的八年中，我们没有遇到一个严重的问题，沟通不正确）。
  • 更好的计划=减少压力 。 如果无法在最后一分钟发送ASAP请求，则需要进行高级计划。 人们学会了如何计划工作量并更仔细地协调活动，以便同事有足够的时间来回答他们的要求。 这样可以减轻压力，最终可以更有效地完成工作。
  • 默认情况下进行深度工作 。 由于员工在收货时不必阅读消息，因此他们可以将大量时间投入到为组织创造最大价值的工作上。 他们每天返回1-3次以批量答复消息，并且不会在工作与消息/会议之间来回跳转。

但是！ 您仍然需要同步通讯

我们的通讯栈

通常，在以下情况下使用同步通信：

  • 您希望建立人与人之间的相互理解（例如，一对一对话或团队会议是合适的）。
  • 必须携带重要信息或讨论一些敏感话题。
  • 有许多未知因素，您想集体讨论各种想法和解决方案。
  • 有许多快速变化的变量，并且您想快速地将相同的信息带给每个人，例如，在有关项目启动的会议上。
  • 发生了危机，需要立即引起注意，例如服务器崩溃。 我们仅将Telegram与随附的通知一起用于紧急通信。

同步通信应该是例外，而不是常规。


每个团队在星期一（而不是（有时是另外））每周一次会议，而不是（每周一次）召开会议，并发布上周当前计划的结果。 因此，每个人都可以看到其他人在不同团队中正在做什么。

如何在团队中创建异步文化


向异步的过渡不会在一夜之间发生。 它要求工具，流程，习惯和文化发生深刻变化。 您可以单独和作为团队领导者采取以下一些具体步骤。

员工可以做什么：


  • 过度沟通 。 发送消息时，请提供尽可能多的信息。 可视化屏幕截图或截屏视频。 清楚地说出您在另一个人的需求以及在什么时间范围内。 在细节和编辑文本上花费几分钟（以便更好地理解）可以节省异步环境中来回交谈的时间。

    缺乏沟通 过度沟通
    您是否可以在有时间的情况下就内容发送报告？ 您能否在下周二（美国东部标准时间）14-00之前通过Google Analytics（分析）发送带有博客指标的简短报告（约1页）？ 请注意以下几点：热门帖子，唯一身份观看的次数，跳出率，点击率和未来帖子的计划。 这是此类报告的一个很好的示例： 内容报告模板 。 我将不胜感激！

    尽可能清晰，仔细地撰写您的信息。
  • 提前计划，以便人们有时间查看您的消息 。 例如，“我希望在两天内完成此操作，最好在您的参与下完成”，而不是“我在一小时内需要您的答复”。
  • 始终检查您的文档共享设置 。 这似乎有些琐碎，但是如果有人必须请求访问权限，则可能会将整个过程延迟几个小时甚至几天。
  • 在开会之前，先开始直播或创建文档 。 在会议之前共享所有信息并讨论关键问题，以便每个人都可以充分理解该主题。
  • 会议结束后，写下讨论和结果 。 开始流或继续文档，以便所有错过的人都可以找到此信息。 我们甚至开始尝试用视频录制会议，以便每个人都可以异步参加。
  • 关闭通知 。 相反，请在一天中留出一定的时间间隔 ，以检查并回复电子邮件和消息。


  • 有效地使用等待时间 。 如果有事情要做，等待答案不会是一个大问题。

团队领导可以做什么：


  • 鼓励每个人掌握书面沟通作为一项核心技能 。 这将减少等待时间并快速传达事物的本质。 在异步环境中，每个人都必须成为优秀的作家。
  • 根据结果​​而不是响应速度或工作小时数来评估人员 。 我们已经详细介绍了性能评估 。
