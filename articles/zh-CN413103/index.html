3CX v15.5 Update 5 Beta和REST与AmoCRM集成发布

本周，我们介绍了3CX v15.5 Update 5 BETA更新。 此更新引入了新的报告类型，改进了对话记录，增加了新的CRM集成模块（包括俄语AmoCRM）和其他改进。 考虑一下此版本中出现的内容。

新型报告


新的中继线呼叫报告可让您查看此中继线（或DID线路）的来电数量。


该报告对于跟踪特定号码上收到的呼叫数量非常方便。 信息可用于例如分析广告活动的有效性或打给组织各个部门的电话数量。

也可以排除持续时间少于一定时间的呼叫。 这样可以消除因通话时间太短而导致的统计数据失真。 您可以在任何报告中按持续时间过滤呼叫。

现在，在3CX v15.5 Update 5 Beta中，您可以为系统中的多个呼叫队列创建摘要（合并）报告。 观看视频 。


呼叫中心经理经常要求提供此机会。 现在，主管可以在一份报告中分析几组操作员的工作。 如果呼叫中心服务于多个项目，则绝对需要这样的报告！

对话记录改进


Update 5扩展了呼叫记录选项。 现在，管理员可以启用或禁用用户以停止记录对话。 此外，管理员可以指定要记录的呼叫：全部或仅外部呼叫。 通常，内部调用不会引起您的兴趣，因此无法编写。 这大大减少了服务器上的负载并节省了磁盘空间。 对话记录现在可以选择以ADPCM格式保存。 这种格式将录制文件的大小减少了约4倍，但无法在标准音乐播放器和浏览器上播放（但大多数第三方播放器都可以播放）。 观看视频 。

我们计划在将来的3CX更新中添加更多的呼叫记录功能。

新的服务器CRM集成模块


我们正在努力将REST与CRM系统集成。 在Update 5中，已扩展了对CRM系统的支持，此外还重新设计了集成指南 。 而且，可以直接从3CX界面加载XML CRM集成模板。



添加了对以下CRM的服务器支持：

  • AmoCRM（有关更多信息，请参见下文）
  • Exactonline
  • 佐霍夫2
  • Opencnam
  • 维蒂格
  • 更新了Salesforce，Freshdesk和Connectwise的模板。

紧急电话和公司聊天


现在，在3CX v.15.5 Update 5 Beta中，对于用户组/分机范围，您可以指定将紧急呼叫路由到的线路。


同样，许可证中的同时呼叫数不再影响紧急呼叫。 如果有电信运营商的空闲线路，您可以使用任何许可证进行紧急呼叫。 请参阅安装视频 。

另一个创新-管理员现在可以为所有用户禁用公司聊天。

恢复管理员密码并替换许可证密钥


有时会发生3CX管理界面的密码丢失或被遗忘的情况。 以前，要恢复密码，必须联系3CX技术支持，但是现在您只需将密码发送到指定为系统中管理员地址的电子邮件地址即可。

现在，您可以即时更换许可证密钥，这对于希望从免费版本切换到商业版本的用户来说可以节省大量时间。 以前，如果您的密钥不能直接更新（无替换），则需要重新安装3CX服务器。 现在，您只需指定一个新密钥而不是旧密钥，系统就可以继续工作。 观看视频 。

详细的管理员通知


为了确保安全，3CX会阻止尝试输入错误的登录名或密码的IP地址。 但是，以前很难确定问题的确切根源。 现在，详细的通知将帮助管理员找出IP阻塞的原因或解决发送错误授权数据的VoIP设备中的问题。

其他改进v15.5 Update 5 Beta


  • 备份到Google云端硬盘支持Google API v1.32，从而提高了性能和稳定性。
  • 支持.NET Core 2.0.7并修复了不支持AVX扩展的处理器（Intel NUC，Celeron和此列表中的其他处理器）...
  • 系统的停车位数量已增加到250。
  • 现在，默认配置最多允许1024个同时呼叫。
  • 修复了报告中与后续呼叫转移持续时间有关的错误。
  • 改进了语音邮件的传递。
  • 改进了对P-Asserted-Identity SIP扩展的支持。
  • 支持TLS 1.2（3CX现在完全兼容PCI）。 但是，不可能自动配置旧式设备和客户端。 在这种情况下，建议您切换到TLS 1.0。
  • FTP备份现在支持EPSV（扩展被动模式）。
  • Yealink T4X的模板现在支持多个EXP40扩展模块。
  • PUSH通知的速度已经提高（计划在更新6中进一步改进）。

请参阅3CX v15.5 Update 5功能视频和详细的 变更 日志。

3CX服务器与AmoCRM的集成


在此更新中，我们“返回”了与流行的俄罗斯AmoCRM系统的服务器集成。 集成显示来电中的联系人或公司数据，允许您创建新联系人，以及将通话记录保留在联系人卡中。
使用我们的3CX Click2Call浏览器插件可以拨打AmoCRM的电话。

要启用集成，请在AmoCRM门户中，转到“设置”>“ API”并复制API密钥。



在3CX界面中，转到“设置”>“ CRM集成”>“服务器端”（“选项”>“ CRM集成”>“服务器端”选项卡），然后选择amocrm。



指定三个参数：

  • 您的AmoCRM门户地址
  • 已注册AmoCRM门户的用户的电子邮件地址
  • 上面复制的API密钥

这样就完成了3CX和AmoCRM CRM的集成！
