Instal 3CX di hosting untuk 2,99 Euro / bulan. dalam 10 menit

Anda mungkin tahu bahwa saat ini perusahaan 3CX menyediakan hosting PBX gratis untuk semua pengguna baru di infrastruktur cloud Google selama 1 tahun. Namun, untuk pengguna 3CX yang ada atau mereka yang segera ingin menggunakan hosting komersial agar tidak "bergerak" di masa mendatang, masuk akal untuk menginstal PBX pada mesin virtual Linux yang murah di pusat data yang dapat diandalkan. Keuntungan menginstal 3CX pada mesin virtual Anda sendiri dibandingkan dengan yang disebut "PBX virtual" adalah Anda sendiri yang sepenuhnya mengontrol layanan PBX dan data rahasia - riwayat dan rekaman percakapan.

3CX telah bekerja keras untuk menyederhanakan instalasi pengguna tanpa pelatihan khusus. Anda hanya perlu mengikuti langkah-langkah untuk membuat akun cloud dengan hati-hati, di mana instance server virtual dengan 3CX PBX yang terinstal kemudian akan dibuat secara otomatis.

3CX telah mensertifikasi beberapa penyedia cloud, termasuk Google Cloud , Azure, dan Amazon, untuk menginstal PBX. Agak terpisah adalah pusat data besar Eropa OVH, yang menawarkan harga yang sangat rendah untuk server cloud dasar, dari 2,99 Euro per bulan. Pada server seperti itu, dimungkinkan untuk menempatkan PBX untuk 16 panggilan simultan, mis. untuk sekitar 64 pengguna. Penting untuk dicatat bahwa PBX untuk 16 panggilan simultan 3CX menawarkan tahun pertama secara gratis .

OVH menggunakan infrastruktur OpenStack, sehingga para spesialis yang akrab dengannya akan sangat mudah untuk mengetahui. Tetapi jika Anda belum pernah membuat layanan cloud, gunakan panduan langkah demi langkah kami - dan hemat dukungan teknis!

Aktivasi Awan OVH


Untuk memulai, buat akun di OVH . Anda akan menerima email dengan kredensial Anda, termasuk ID Pelanggan (tipe bn112346-ovh).

Pergi ke OVH Manager dan aktifkan proyek cloud (Cloud) di atas (1).



Tunjukkan nama proyek (2), yang kemudian akan terlihat di panel kontrol OVH. Anda juga harus menentukan kartu pembayaran atau akun Paypal untuk mengaktifkan cloud (jumlah - 10 Euro).



Pada akhirnya, setelah pembayaran dan beberapa menunggu, Anda akan menerima pemberitahuan email bahwa cloud Anda diaktifkan.



Membuat Pengguna OpenStack


Ketika cloud sudah siap, buka Cloud Manager , lalu ke bagian OpenStack (2) dan buat pengguna (Anda dapat menentukan nama dan nama keluarganya) (3). Kredensial pengguna ini (4) kemudian digunakan untuk membuat 3CX PBX dalam panduan PBX Express.



Pastikan untuk memperbaiki nama akun dan kata sandi, seperti mereka ditampilkan hanya sekali. Setelah itu, buka konsol manajemen OpenStack (5) dengan kredensial pengguna yang dibuat.

Membuat Kunci SSH


Buat kunci enkripsi SSH untuk terhubung ke mesin virtual. Di bagian atas, pilih pusat data tempat Anda ingin membuat mesin virtual (1). Disarankan untuk meninggalkan pusat data WAW1 (Warsawa), karena secara geografis paling dekat dengan Eropa Timur.



Lalu pergi ke Hitung (2)> Pasangan Kunci (3) dan tentukan nama kunci (4). Selanjutnya akan ditampilkan di wizard PBX Express. Jika Anda berencana untuk terhubung ke mesin virtual PBX 3CX melalui SSH , salin kunci ke tempat yang aman (5).

Kunci keamanan harus dihasilkan untuk setiap pusat data OVH tempat Anda berencana membuat mesin virtual. Oleh karena itu, jika Anda ingin membuat instance 3CX di pusat data lain, misalnya, "GRA1" (Eropa Barat), ulangi prosedur ini.

Ini melengkapi persiapan hosting OHV.

Wisaya PBX Express


Di panduan PBX Express, pada langkah 5, pilih penyedia OVH dan masukkan nama pengguna dan kata sandi pengguna yang kami buat di atas.



Pilih proyek di mana mesin virtual akan dibuat, dan wilayah yang diinginkan. Nama kunci enkripsi yang kami buat sebelumnya akan secara otomatis ditampilkan. Pilih jenis mesin virtual yang sesuai dengan kapasitas PBX Anda . Anda bisa mulai dengan vps-ssd-1.



Harap perhatikan bahwa faktor-faktor berikut ini mempengaruhi pemuatan server PBX:

  • merekam percakapan (dalam pembaruan terbaru, Anda hanya dapat menulis panggilan eksternal, dan tidak semua berturut-turut)
  • konferensi suara
  • jumlah batang SIP
  • integrasi dengan sistem dan basis data CRM
  • jumlah pengguna aktif di klien web 3CX
