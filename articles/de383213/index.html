Folgen eines Forum-Datenschutzstreits

Vor nicht allzu langer Zeit wurde in einem Profilforum über Datenschutz in Windows 10 gesprochen. Es gibt bereits eine allgemeine Standarddebatte: Sendet es private Daten, was und warum? Wie so oft wurden Westen zerrissen, gegenseitige Anschuldigungen niedergeschlagen, Studien von mabritischen Wissenschaftlern veröffentlicht und so weiter. Irgendwann ärgerte ich mich über das nächste Mal, dass beim Aktivieren von Windows 10 30 Megabyte Daten vom Camcorder des Laptops gesendet werden. Ich entschied mich, die Vorgänge in Windows 8.1 zu überprüfen, dann Windows 10 in der Nähe (erneut) zu installieren und das Verhalten zu vergleichen .

Ich muss sofort sagen, dass der Plan nicht die Installation von Dutzenden erreichte. Fragen stellten sich bereits in den acht.

Wie können Daten von Windows gesendet werden?

Sofort fällt Ihnen ein Taskplaner ein, in dem Sie (plötzlich) jede Aktion nach Ereignis oder Zeit planen können. Die zweite Option ist ein Standarddienst, der neben regelmäßigen Aktivitäten auch Eventualverbindlichkeiten ausführen kann. Lesezeichen, einfach ausgedrückt. Leider erlaubt mir meine Qualifikation nicht, das Vorhandensein solcher Dienste sicher zu identifizieren.

Daher wurde die Auswahl eingegrenzt, um den Planer zu überprüfen. Ich denke, wenn etwas von dort gesendet wird, wird es dort sterben . Ich habe meine Acht im Januar neu angeordnet, nicht mit Standardeinstellungen, sondern mit Deaktivierung aller Funktionen, die für mich unnötig waren (die Standardeinstellungen waren übrigens eines der Argumente im Streit). So wurde es als Referenzsystem geplant.

Ich entferne die Liste der Aufgaben in Powershell (Get-ScheduledTask) und dann klammert sich das Auge an diesen Block:
\ Microsoft \ Windows \ Customer Experience Impr ... BthSQM Ready
\ Microsoft \ Windows \ Customer Experience Impr ... Consolidator Ready
\ Microsoft \ Windows \ Customer Experience Impr ... KernelCeipTask Ready
\ Microsoft \ Windows \ Customer Experience Impr ... UsbCeip Ready

Wie das? Ich erinnere mich, dass ich CEIP definitiv ausgeschaltet habe. Ich überprüfe - "C: \ windows \ system32 \ rundll32.exe" werconcpl.dll, ShowCEIPDialog (um ehrlich zu sein, ich bin eigentlich nicht so pervers und suche einfach nach dem Wort "CEIP" in Start / Start):



Alles ist in Ordnung, aber Senden ist dennoch enthalten.

Ich öffne taskschd.msc und schaue mir die Protokolle dieser Aufgaben an (in Klammern stehen Beschreibungen der Aufgaben selbst):

  • Consolidator( Windows, ) — 12/08/2015

* Wichtiger Hinweis - gestern in der Zeitschrift war ein anderes Datum: der 10. August. Heute habe ich diese Aufgaben aufgenommen, um einen Screenshot zu machen. Ich kann davon ausgehen, dass frühe Ereignisse nach Abschluss der Aufgabe gelöscht werden. Wenn ja, dann kann ich nicht sagen, wann sie wirklich aufgenommen wurden.

Es ist klar, dass die Aufgaben nach dem Einschalten weiter arbeiteten. Bis sie manuell getrennt werden (PShell vom Administrator):
Get-ScheduledTask -TaskPath "\ Microsoft \ Windows \ Programm zur Verbesserung der Kundenerfahrung \" | Disable-ScheduledTask
