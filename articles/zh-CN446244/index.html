Chrome扩展程序用于Web开发并与GitHub一起使用



大家好 我想与大家分享一些我在开发，分析网站，使用GitHub以及在日常生活中使用的Chrome扩展程序。 我相信对于大多数读者来说，这些扩展列表并不是一个开篇，但是那些第一次了解它们并安装它们的人无疑会喜欢使用新工具。

我认为创建带有详细描述工具的长鞋履是不值得的，它们很简单并且可以理解。 因此，我将尝试简要地谈谈使用它们的优点。

使用GitHub


在我看来， Octotree是最有用，最方便的扩展。 它允许您以树的形式查看源代码，该树显示在页面的左侧。 新导航比标准导航方便得多。 如果您正在处理带有大量嵌套目录的大型项目，则Octotree将特别有用。

GitHub Repository Size是一个方便，简单的插件，可显示单个存储库的大小以及其中每个文件的大小。

OctoLinker是一个扩展，可通过将简单的代码元素转换为可单击的链接来提供快速的代码导航。 通过此链接，您可以快速找到源。 根据编程语言的不同，扩展名将显示源文件或将用户带到外部站点。 转换为链接的元素会用粉红色的点标记，以便您可以立即看到哪里有交互式链接。

GitHub Hovercard可从GitHub网站上的任何位置即时访问用户信息，存储库，问题和提交。 只需将鼠标悬停在信息上即可。

npmhub当在github.com上查看具有package.json文件的存储库时，此扩展名将分析软件包中的依赖项，并在README文件下方显示链接及其说明。

github npm stats显示NPM网站上的软件包下载统计信息（如果有）。

精致的Github显示最近发布的早午餐，还允许您重新处理请求并显示对评论做出响应的用户的化身。

等距贡献 -此扩展名没有以前的有用，但非常有趣）。 它允许您在GitHub上的常规贡献图和等距像素艺术版本之间切换。

使用网站


滴管是查看网站上使用的颜色并创建自己的调色板的绝佳工具。 我几乎每天都使用它））。

全页截屏 。 想象一下，您已经在3-5个屏幕上开发了一个着陆平台，却不知道如何使其全屏显示。 按下打印屏幕5次，然后在编辑器中一个接一个地制作作品？ 哦，好！ FPSC是个很棒的工具，可以在几秒钟内获取当前页面的屏幕截图。 如果您需要在不下载特殊服务的情况下检查站点优化，以及完全捕获您创建的站点，这将非常有用。

WhatFont-在此扩展程序的帮助下，您只需单击扩展名并将光标移动到所需的位置，即可轻松找到站点上使用的字体。 简单而优雅。 通过按下“ Esc”按钮禁用。 还显示文本颜色，高度，字体大小和其他参数。

PerfectPixel-此扩展将帮助您以逐像素精度开发网站。

CSS Peeper是专门为设计人员设计的CSS分析工具。 扩展的目的是允许设计人员专注于设计，并花费尽可能少的时间来研究代码。 只需尝试一下操作，您就会了解所有内容。

日常工作


Google翻译是另一个工具，我几乎每天都无法做到。 这不是页面，这是扩展。 有了它，您可以直接在页面上快速翻译所选文本，而无需进入翻译器的页面。 附言 不会翻译长文本，但这没关系。
