12 de diciembre mañana resumen de Java


  • El final de la vida de Spring Boot 2.1.x está programado para el 1 de noviembre de 2020. Algunas actualizaciones seguirán allí antes de esta fecha, y después de eso desaparecerán. Necesita ser actualizado a 2.2. El procedimiento de actualización en sí no parece complicado, las instrucciones están en la wiki . También hay una lista de versiones compatibles .
  • Se lanzó la primera versión del kit de herramientas Kilt , diseñada para manejar paquetes de internacionalización. En realidad, en el nombre de Kilt, la letra i es i18n, l es l10n, t es t9n. La biblioteca puede convertir paquetes de / a XLS (X) (útil para un pequeño equipo de traductores), proporciona una fachada de tipo seguro para trabajar con ellos en Java (no, la letra K en el nombre no es Kotlin) y refactorizaciones simples dentro del paquete, como reformatear texto Puede funcionar como una aplicación de línea de comandos independiente, como tarea para Ant, como complemento para Maven. Casi el 100% del código está escrito en Java. 198 se compromete en GitHub, solo diez estrellas (no es sorprendente, el autor lo anunció solo hoy).
  • Payara Server ahora es oficialmente compatible con Jakarta EE 8. Le recuerdo que este es un middleware Java EE de código abierto con un enfoque en nubes híbridas. Presente en GitHub . Jakarta EE (si ha estado en animación suspendida durante los últimos años) es el sucesor de Java EE, a través de Party, el futuro de Cloud Native Java. A partir de la versión 5.193.1 , este producto se une a la feliz familia de Glassfish (Eclipse Foundation), OpenLiberty (IBM) y Wildfly (RedHat). Si estaba preocupado por el futuro de Wildfly allí, ahora tenemos dos implementaciones alternativas.
  • La última versión de Spring Cloud Gateway en GitHub es 2.2.0. En la próxima versión 2.2.1, podemos implementar un nuevo interruptor automático en la parte superior de la conocida biblioteca resilience4j (más de mil commits y cinco mil estrellas en GitHub). Detrás de esto hay una pequeña historia trágica. Este reemplazo no ocurre a instancias del talón izquierdo de los hipsters, sino porque Netflix congela su maravillosa Hystrix a medida que la compañía cambia su stack tecnológico. Gibb Spencer (cofundador de Netflix) nos prometió que Hystrix y Ribbon continuarán apoyando en el marco de Spring Cloud, pero todos los que lo necesitan ya se han retirado del barco con horror. (Este tweet incluso recibió una mención en This Week in Spring de Josh Long) . Les recuerdo que la segunda alternativa para resilience4j es el Sentinel del Grupo Alibaba. Robert Winkler, de Deutsche Telekom, Mahmoud Romeh, de IBM, Dan Maas, de Target Corporation (esta es una red de tiendas minoristas estadounidenses con una facturación de 72 mil millones de dólares) y Bogdan Storozhuk . Gente seria, tienes que tomarlo. El proceso de adaptación se puede observar en elementos que forman el sistema como Spring Cloud Gateway, y puede leer más sobre esto en el blog de Pyotr Minkovsky (este es el autor del libro Mastering Spring Cloud ).
