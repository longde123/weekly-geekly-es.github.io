El mejor lenguaje de programación del mundo.

Cada programador tiene un sueño: crear su propio lenguaje de programación. El mejor y más conveniente, por supuesto, no como los existentes. Para mí personalmente, la idea de la solución era crear un lenguaje en el que no hubiera absolutamente nada repetitivo, que fuera lo más breve posible, pero extremadamente elocuente. Durante dos años he estado tratando de lograr el resultado deseado, pero donde sea que empiece, al final, después de cortar todo lo superfluo, siempre obtuve Lisp. Y entonces se me ocurrió una idea ingeniosa: tomar Lisp y mejorarlo de acuerdo con mis ideas. En honor a lo primero que me llamó la atención, el proyecto se llamó Sova.


Para que el lenguaje fuera de la caja funcione en el servidor, en la web y en las plataformas móviles, decidí compilarlo en Javascript, que, en mi opinión, no es una broma uno de los idiomas mejor diseñados de nuestro tiempo y, lo que es más importante, tiene un poderoso ecosistema npm, Node.js, React and React Native.


Y antes de comenzar nuestro viaje, para despertar inmediatamente el interés en seguir leyendo el artículo, aquí hay un ejemplo de un archivo de aplicación Express en Sova: 


=-> './database' database =-> 'express' express = application express () application.get '/' -> (request response) (response.sendStatus 200) application.get '/user/:id' -> (request response) (response.send (database.getUserById request.params.id)) application.listen 8080 -> () (console.log 'Application is listening on port 8080')

Bueno, empecemos ...


Paso 1: Extraer paréntesis de Lisp usando la sangría de Python



Aunque este cómic me hace sonreír cada vez que lo miro, estoy completamente en desacuerdo con su declaración principal. Los soportes no son elegantes; los soportes son repetitivos.


Por lo tanto, lo primero en mi maravilloso idioma decidí deshacerme de los corchetes. La mejor solución fue la sangría relevante, al igual que en Python. Tomemos, por ejemplo, una pieza de código lisp: 


 (* 2 (+ 1 2) (- 4 (/ 2 1)))

Por supuesto, esta expresión puede extenderse en diferentes líneas, pero esto no nos alivia del payaso de los ojos obstruidos. 


 (* 2 (+ 1 2) (- 4 (/ 2 1)))

Ahora veamos cuán elegante y aireado es posible escribir la misma expresión en Sova con la ayuda de una sangría significativa: 


 * 2 + 1 2 - 4 (/ 2 1)

Es decir, en Sova, la expresión a (bc) (d (ef)) equivalente a la expresión: 


 a bc d ef

Paso 2: haz que el lenguaje sea conciso


Lo que no me gustó en la mayoría de los lenguajes de programación fue la obstrucción de la sintaxis sin nada significativo, pero ocupando un lugar en la pantalla con un texto repetitivo: palabras clave adicionales, signos de puntuación sin sentido y muchos otros. Incluso en ceceo común, en lugar de caracteres simples y comprensibles, las palabras se usan a menudo para denotar operaciones simples, como la misma defn .


Declaración de constantes


Tomemos, por ejemplo, la declaración de una constante en Javascript: 


 const a = 1

Sova es un lenguaje extremadamente funcional y todas las variables son inmutables, por lo que no necesita especificar una palabra clave adicional const, pero todo está escrito simplemente: 


 = a 1

Las funciones


El elemento central de cualquier lenguaje es la función. Tan minimalistas se ven en Sova: 


 = addOne -> number + number 1 = doubleAndAddOne -> number = doubled (* number 2) addOne doubled console.log (doubleAndAddOne 2)

Como en cualquier lenguaje funcional, la última expresión en el cuerpo de la función es retornable. Es decir, el código anterior en JavaScript compilado se verá así: 


 const addOne = number => { return number + 1 } const doubleAndAddOne = number => { const doubled = number * 2 return addOne(doubled) } console.log(doubleAndAddOne(2))

Comparaciones y condiciones


Una expresión condicional en Sova puede tener dos o un argumento.
Aquí hay ejemplos de condiciones que tienen dos argumentos: 


 console.log ? true 1 0 console.log ? (> 2 1) 'Greater' 'Less' console.log ? (> 2 1) ? (> 1 2) 'x' 'y' 'z'

Y aquí, por ejemplo, en la función checkNumber, devolvemos valores por condición: 


 = checkNumber -> number ? (=== number 1) (<- 'One') ? (=== number 2) (<- 'Two') ? (<= number 9) (<- 'From three to nine') 'Ten or more' console.log (checkNumber 1) console.log (checkNumber 4) console.log (checkNumber 11)

En JavaScipt compilado, se ve así: 


 const checkNumber = number => { if (number === 1) return 'One' if (number === 2) return 'Two' if (number <= 9) return 'From three to nine' return 'Ten or more' } console.log(checkNumber(1)) console.log(checkNumber(4)) console.log(checkNumber(11))

Colecciones


Matriz


La colección principal de cualquier idioma es una matriz. Así es como se ve la declaración y la deconstrucción de la matriz en Sova: 


 = list | 1 2 3 4 console.log list console.log list.map (-> x (+ x 1)) = (| first second) list console.log first console.log second

En JavaScript compilado, se verá así: 


 const list = [1, 2, 3, 4] console.log(list) console.log(list.map(x => x + 1)) const [first, second] = list console.log(first) console.log(second)

Objeto


La segunda colección más importante es el hashmap. En Sova, el anuncio y la deconstrucción del mapa se ven así: 


 = map : a 1 b 2 c : d 3 e 4 f 'Hello' console.log map = (: a (c (: de))) map console.log a console.log d console.log e

En JavaScript compilado, se ve así: 


 const map = { a: 1, b: 2, c: { d: 3, e: 4 }, f: 'Hello' } console.log(map) const { a, c: { d, e }} = map console.log(a) console.log(d) console.log(e)

Si queremos llamar a un método en un objeto, entonces hay dos formas de hacerlo. Podemos llamarlo como object.method parameter1 parameter2 o como .method object parameter1 parameter2 . El segundo método nos permite crear una cadena de llamadas a métodos.


Importar y exportar módulos


Importar


Puede importar módulos al código Sova desde otros archivos .sv , así como desde archivos .js . Por ejemplo, en este ejemplo, se importan dos módulos: data/index.js y handler/index.sv : 


 =-> './data' (: greeting name) =-> './handler' handle handle greeting name

En JavaScript compilado, se ve así: 


 const { greeting, name } = require('./data') const handle = require('./handler') handle(greeting, name)

La importación de los módulos JavaScript y Sova hace posible integrar ligeramente Sova en un proyecto Javascript existente.


Exportar


En este ejemplo, la función se exporta desde el módulo: 


 <-= -> (greeting name) console.log greeting console.log name

En JavaScript compilado, se ve así: 


 module.exports = (greeting, name) => { console.log(greeting) console.log(name) }

Ejemplos de uso


Para ver toda la belleza, la ligereza y la concisión de Sova, debe mirar un programa más o menos grande. Por ejemplo, aquí hay un programa que calcula la edad promedio de las personas y encuentra el nombre de la persona cuya edad es más cercana a la edad promedio. 


 =-> 'lodash' _ = people | : (name 'Alice') (age 24) : (name 'Bob') (age 15) : (name 'Chris') (age 46) : (name 'Daniel') (age 35) : (name 'Elisabeth') (age 29) : (name 'Fred') (age 52) = averageAge / .reduce (.map people (-> man man.age)) -> (xy) (+ xy) 0 .length people = manWithClosestToAverageAge _.minBy .map people (-> man (: (name man.name) (distance (Math.abs (- averageAge man.age))))) 'distance' console.log averageAge console.log manWithClosestToAverageAge.name

Debido al hecho de que el lenguaje está compilado en JavaScript, el desarrollo para cualquier plataforma es posible. Por ejemplo, aquí hay un pequeño ejemplo de una aplicación React para navegadores web: 


 =-> 'react' React =-> 'react-dom' ReactDOM =-> './styles' styles = (: createElement:e) React = App -> ((: name)) e 'div' (: (style styles.container)) e 'div' (: (style styles.hello)) 'Hello' e 'div' (: (style styles.name)) name ReactDOM.render (e App (: (name 'John'))) (document.getElementById 'root')

También en el repositorio hay ejemplos de aplicaciones Express server y React Native para plataformas móviles.


Conclusión


Por lo tanto, Sova incorpora lo mejor de varios otros idiomas:


  • la simplicidad y el poder de Lisp
  • Sangrado de Python limpio
  • tiempo de ejecución y ecosistema de JavaScript

El código del compilador con ejemplos de uso del lenguaje se encuentra aquí https://github.com/sergeyshpadyrev/sova . Me alegrará ver las estrellas en el repositorio de todos aquellos a quienes les gustó el concepto del lenguaje y quisieran continuar trabajando en él. Pero inmediatamente te advertiré que por el momento esto es solo una prueba de concepto, e incluso jugar con el idioma debido a la falta de documentación y algunas características es extremadamente difícil. Por ejemplo, el lenguaje aún no tiene manejo de excepciones, clases y otras cosas necesarias. Y ejecutar Windows es poco probable que tenga éxito.


En los siguientes pasos, planeo complementar y estabilizar la sintaxis, reescribir el analizador, escribir pruebas para el analizador y el traductor, escribir documentación. Mientras tanto, gracias por su atención y hasta pronto.
