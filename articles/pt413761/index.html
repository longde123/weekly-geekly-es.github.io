Como impedir que o Windows 10 seja reiniciado após atualizações

Olá pessoal. Surpreendentemente, a resposta a esta pergunta (mais precisamente, "como fazer tudo para que fosse como antes") está mal iluminada em Runet. E, em geral, ainda é preciso procurar uma solução pronta.
No entanto, o problema precisa ser resolvido porque o sistema não recupera todos os aplicativos após uma reinicialização.

Então:

  1. Vá para as configurações de diretiva. Isso funcionará mesmo em uma máquina local sem domínio.
    Para iniciar o editor de políticas, faça: Iniciar → Executar (ou Win + R) → gpedit.msc
  2. À esquerda na árvore, selecione: Configuração do computador, Modelos administrativos, Componentes do Windows, Windows Update.
  3. Aqui estamos interessados ​​em dois parâmetros:

    1. Não execute a reinicialização automática ao instalar atualizações automaticamente se Não houver reinicialização automática com usuários conectados para instalações agendadas de atualizações automáticas (Ativado). Viva, não haverá mais uma reinicialização (se houver sessões ativas).
    2. Preste atenção também no parâmetro Configurar atualizações automáticas. Se você colocá-lo no estado Desativado, o sistema deixará de baixar e instalar secretamente as atualizações.

      Se você deseja que o sistema nunca reinicie espontaneamente nunca (mesmo na ausência de sessões ativas), é isso que você precisa fazer.

Após desativar a reinicialização, o sistema fornecerá periodicamente uma boa janela solicitando que você reinicie o computador (como era antes):


A propósito, essa janela é ainda melhor: permite adiar a reinicialização "até a noite", o que significa um lembrete apenas uma vez por dia, e não a cada 4 horas, como era antes.

PS: Conforme observado corretamente nos comentários, o editor de políticas está disponível apenas em versões mais íngremes que o Professional. Nas versões mais novas, você precisará fazer alterações diretamente no registro.
