Minimale CentOS / Fedora / RedHat-Installation

Ich habe keinen Zweifel daran, dass die edlen Spender - Linux-Administratoren - bestrebt sind, die auf dem Server installierten Pakete zu minimieren. Es ist wirtschaftlicher, sicherer und vermittelt dem Administrator ein Gefühl der vollständigen Kontrolle und des Verständnisses der Prozesse.

Ein typisches Szenario für die Erstinstallation des Betriebssystems sieht daher so aus, als würde man die Mindestoption auswählen und sie dann mit den erforderlichen Paketen füllen.



Die vom CentOS-Installationsprogramm angebotene Mindestoption ist jedoch nicht ganz minimal. Es gibt eine Möglichkeit, die Größe der Erstinstallation des Systems auf standardisierte Weise zu reduzieren.

Wenn Sie das CentOS-Betriebssystem in Arbeit verwenden, entdecken Sie früher oder später die Automatisierung der Installation mithilfe des Kickstart-Mechanismus. Ich habe CentOS schon lange nicht mehr als Standardinstallationsprogramm installiert. Während der Arbeit hat sich ein ausreichendes Arsenal an Konfigurations-Kickstart-Dateien angesammelt, sodass Sie Systeme automatisch bereitstellen können, einschließlich auf LVM, Krypto-Partitionen, mit einer minimalen GUI usw.

Und jetzt, in einer der Versionen der 7. Version, hat RedHat Kickstart eine erstaunliche Option hinzugefügt, mit der das Image des installierten Systems weiter minimiert werden kann:

--nocore

Deaktiviert die Installation der Core- Paketgruppe, die ansonsten standardmäßig immer installiert ist. Das Deaktivieren der Core- Paketgruppe sollte nur zum Erstellen von Lightweight-Containern verwendet werden. Die Installation eines Desktop- oder Serversystems mit --nocore führt zu einem unbrauchbaren System.
RedHat warnt ehrlich vor den möglichen Konsequenzen der Verwendung dieser Option. Die jahrelange Verwendung durch mich in einer realen Umgebung bestätigt jedoch ihre Stabilität und Anwendbarkeit.

Das Folgende ist ein Beispiel für eine minimale Kickstart-Installationsdatei. Der Mutige kann yum davon ausschließen. Machen Sie sich bereit für Überraschungen: 

install text url --url="http://server/centos/7/os/x86_64/" eula --agreed firstboot --disable keyboard --vckeymap=us --xlayouts='us' lang en_US.UTF-8 timezone Africa/Abidjan auth --enableshadow --passalgo=sha512 rootpw --plaintext *** ignoredisk --only-use=sda zerombr bootloader --location=mbr clearpart --all --initlabel part /boot/efi --fstype="efi" --size=100 --fsoptions="umask=0077,shortname=winnt" part / --fstype="ext4" --size=1 --grow network --bootproto=dhcp --hostname=localhost --onboot=on --activate #reboot poweroff %packages --nocore --nobase --excludedocs yum %end %addon com_redhat_kdump --disable %end

Ich möchte darauf hinweisen, dass CentOS / RedHat Fedora bei der Interpretation der Option gegenüber loyaler ist. Letzteres wird das System so stark abdecken, dass eine Neuinstallation mit wichtigen Dienstprogrammen erforderlich ist.

Als Bonus gebe ich Ihnen einen "Zauber" für die Installation einer minimalen grafischen Umgebung in CentOS / RedHat (Version 7): 

 yum -y groupinstall x11 yum -y install gnome-classic-session systemctl set-default graphical.target

Sowohl das minimale Image des Betriebssystems als auch die minimale grafische Umgebung wurden von mir getestet und funktionieren auf realen Systemen.
