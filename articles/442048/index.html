Cómo Google Programmer resuelve problemas comunes



De un traductor : Estamos publicando una traducción de un artículo de Steve Merrit , un empleado de Google que habla sobre cómo resuelve los problemas de programación típicos. La publicación será útil principalmente para programadores novatos.

En este artículo hablaré sobre mi estrategia para resolver los problemas que surgen durante el trabajo en un proyecto, de principio a fin. Lo uso en el flujo de trabajo diario en Google, así como cuando trabajo con codificadores de todos los niveles (colegas, graduados de bootcamps, estudiantes universitarios). Una técnica estructurada minimiza el tiempo dedicado a la depuración y al mismo tiempo conduce a la creación de un mejor código.

Por cierto, la misma estrategia a menudo funciona durante las entrevistas en grandes corporaciones tecnológicas. Hace tres años conseguí un trabajo en Google gracias a ella.

Paso a paso


Mostraré ejemplos en forma de problemas típicos para revelar el tema.

Problema: "Dadas dos líneas, sourceString y searchString, debe devolver el primer índice cuando sourceString aparece en searchString. Si searchString no está en sourceString, devuelve -1 ".

1. Dibújalo


Comenzar a escribir código de inmediato no es una buena idea. Primero debes delinear una forma de resolver el problema. Comience formando una hipótesis y evidencia de su punto de vista. Y trabaje solo cuando ya tenga un plan claro. Si esto no se hace, entonces cuando el trabajo ya ha comenzado, puede encontrar el hecho de que las piezas individuales de código no se corresponderán entre sí.

La solución a menudo puede ser no trivial, incluso si la tarea parece simple. La planificación en papel lo ayuda a encontrar el enfoque correcto y a asegurarse de que funcione en otras situaciones. Y aprenderá todo esto incluso antes de que se escriba la primera línea de código.

Así que no empieces a escribir código, ni siquiera lo pienses. Tendrás mucho tiempo para trabajar. Eres una computadora humana y resuelves el problema.

Ponga el algoritmo de solución en papel. Si algo te ayuda a visualizar tu plan, hazlo. La tarea es resolver el problema con lápiz y papel, sin teclado.

Ven con una entrada simple. Si la función "pasa la cadena", entonces "abc" es el primer excelente ejemplo. Intenta comprender cuál debería ser el resultado correcto. Luego piense cómo entendió el problema, qué pasos se tomaron.

Imagine que las cadenas tienen los siguientes valores:

sourceString: "abcdyesefgh"
searchString: "yes"

Entonces podemos ver que searchString está dentro de sourceString. ¿Pero cómo llegamos a esto? Comenzamos desde el principio de sourceString y lo leímos hasta el final, observando cada fragmento de tres caracteres para ver si coincide con la palabra "sí". Por ejemplo, "abc", "bcd", "cde", etc. Cuando llegamos al índice 4, encontramos "sí" y, por lo tanto, decidimos que había una coincidencia, y comienza en el índice 4.

Tuve un maestro en el instituto que se propuso elaborar las instrucciones para crear un sándwich de mantequilla de maní. Para una instrucción detallada y comprensible, nos prometieron la calificación más alta.

Escribí lo siguiente:

“Abre la mantequilla de maní, extiende sobre el pan. Pon otro pedazo de pan encima y listo.

Pensé que lo había logrado hasta que la maestra tomó la mantequilla y untó el pan, que todavía estaba en una bolsa de plástico.

Los programas, como mi maestro, requieren instrucciones muy detalladas para hacer posible la tarea. Por lo tanto, cuando creamos el algoritmo, nos aseguramos de proporcionar todo, todos los escenarios posibles. Devolver la respuesta correcta cuando el partido se ENCUENTRA es excelente, pero es necesario devolver la respuesta incluso si el partido NO SE ENCUENTRA.

Intentemos nuevamente con otro par de líneas:

sourceString: "abcdyefg"
searchString: "yes"

Aquí, comenzamos desde el principio de sourceString y lo leímos hasta el final, observando cada fragmento de tres caracteres para ver si coincide con la palabra "sí". Cuando llegamos al índice 4, encontramos yef, que fue casi una coincidencia, pero incompleta, ya que el tercer personaje era diferente. Por lo tanto, continuamos leyendo hasta llegar al final de la línea, y luego decidimos que no había coincidencia, por lo que devolvimos -1.

Creamos una serie de pasos (en la programación que se llama algoritmo) que realizamos para resolver el problema, e intentamos ejecutar varios escenarios, cada vez obteniendo el resultado correcto. Por el momento, podemos estar seguros de que nuestro algoritmo funciona, y ahora es el momento de formalizarlo, lo que nos llevará al siguiente paso.

2. Escribimos el algoritmo en palabras.


Esto hace que los pasos sean reales, lo que significa que podemos consultarlos más adelante al escribir el código.

  • Comience al principio de la línea.
  • Examinamos todas las combinaciones de tres caracteres (o cuántos caracteres hay indicados en searchString).
  • Si alguno de ellos es igual a searchString, devolvemos el índice actual.
  • Si llegamos al final de la línea sin encontrar una coincidencia, devuelve -1.

3. Escribimos un pseudocódigo


El pseudocódigo no es realmente un código, pero pretende ser un código. Un ejemplo de lo que estoy hablando, dado nuestro algoritmo:

for each index in sourceString,
there are N characters in searchString
let N chars from index onward be called POSSIBLE_MATCH
if POSSIBLE_MATCH is equal to searchString, return index
at the end, if we haven't found a match yet, return -1.

Puedo hacerlo aún más como un código real como este:

for each index in sourceString,
N = searchString.length
POSSIBLE_MATCH = sourceString[index to index+N]
if POSSIBLE_MATCH === searchString:
return index
return -1

4. Traducimos todo lo que podemos al código


Ahora tenemos que ocuparnos de la sintaxis, los parámetros de la función y las reglas del lenguaje. Tal vez no puedas escribir todo, y eso es normal. ¡Escriba en el código lo que sabe con seguridad! 

 function findFirstMatch (searchString, sourceString) { let length = searchString.length; for (let index = 0; index < sourceString.length; index++) { let possibleMatch = <the LENGTH chars starting at index i> if (possibleMatch === searchString) { return index; } } return -1; }

Tenga en cuenta que dejé parte de este código en blanco. ¡Esto es intencional! No estaba seguro de la sintaxis para procesar cadenas en JavaScript, pero más adelante en eso.

5. No confíes en la suerte


Un error bastante común, especialmente para los programadores novatos, es usar algo que se encuentra en la red con la esperanza de que simplemente funcione. El fragmento encontrado simplemente se inserta en su propio proyecto sin probarlo. Cuantas más secciones de su programa no comprenda, más irreal será la finalización exitosa del trabajo.

La probabilidad de un error se duplica cuando agrega cualquier elemento del que no está seguro. Como resultado, el proceso simplemente se sale de control.

Comentario: la probabilidad de un error se puede calcular utilizando la secuencia de Mersenne: a (n) = (2 ^ n) - 1

Prueba tu código. Encontrar algo en línea es genial, pero antes de agregar un fragmento a su programa, intente esta sección por separado de todo.

En el paso anterior, dije que no sabía cómo seleccionar una determinada parte de la cadena usando JavaScript. Vamos a buscarlo en google.

https://www.google.com/search?q=how+to+select+part+of+a+string+in+javascript

El primer resultado es de w3schools. Un poco anticuado, pero funcionará:

http://www.w3schools.com/jsref/jsref_substr.asp

Supongo que debería usar substr (index, searchString.length) para resaltar la parte sourceString cada vez. Pero hasta ahora esto es una suposición y nada más. Así que lo comprobaré primero.

let testStr = "abcdefghi"
let subStr = testStr.substr(3, 4); // simple, easy usage
console.log(subStr);
"defg"
subStr = testStr.substr(8, 5); // ask for more chars than exist
"i"

Ahora sé exactamente cómo funciona esta función. Por lo tanto, cuando agregue este fragmento a mi programa, ya sabré que si no funciona, el problema no está en la sección agregada.

Y finalmente, agrego la última parte del código. 

 function findFirstMatch(searchString, sourceString) { let length = searchString.length; for (let index = 0; index < sourceString.length; index++) { let possibleMatch = ( sourceString.substr(index, searchString.length)); if (possibleMatch === searchString) { return index; } } return -1; }
