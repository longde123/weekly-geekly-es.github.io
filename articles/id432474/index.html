kue mRemoteNG lagi

Jika Anda adalah administrator dari beberapa lusin server Windows dan Linux, banyak switch dan router, maka tanpa manajer koneksi jarak jauh Anda dapat dengan cepat dan andal menjadi gila.

TL; DR . mRemote, yang perkembangannya sudah lama ditinggalkan, telah menemukan kehidupan baru. Jika Anda menggunakan RDCMan atau Remote Desktop gratis dari Devolusi - coba mRemoteNG!

mRemote adalah manajer koneksi jarak jauh yang sangat populer di masa lalu. Sayangnya, pengembangannya ditinggalkan sekitar tahun 2009. Ketidaksempurnaan dalam antarmuka, gangguan dengan versi baru RDP memaksa saya untuk mengabaikannya.

Untuk beberapa waktu, Remote Desktop Connection Manager (alias RDCman ) dari Microsoft tidak ada .



Switch dan server Linux hidup di konsol SuperPutty terpisah. Saya tidak antusias tentang utilitas pertama atau kedua. Saya tidur sangat buruk karena kata sandi yang disimpan, yang dienkripsi sesuai dengan prinsip Keamanan melalui ketidakjelasan. Gagal !

Kemudian dia beralih ke VisionApp vRD .



Komersial, ternyata mendapatkan lisensi NFR. Manajer yang bagus, tapi buggy. Dan tidak terlalu fungsional. Poin kuatnya adalah penyimpanan koneksi di SQL Server, ini diperlukan agar seluruh tim Anda memiliki basis data koneksi yang sama. Sekarang produk tersebut disebut ASG Remote Desktop .

Pengalaman dual vRD membuat saya memikirkan alternatif. Tidak perlu mencari waktu yang lama, pada tahun 2014 Remote Desktop Manager dari orang-orang Kanada dari Devolutions sudah mendominasi pasar.



Produk hebat yang sedang dikembangkan sangat intensif. Tampaknya memiliki semua yang Anda inginkan. Isi otomatis di konsol web. Modul PowerShell sendiri. Integrasi elegan (Anda tidak bisa mengatakan sebaliknya) dengan KeePass - silakan.
Dan yang paling penting, pencarian server cepat. Berdasarkan nama, deskripsi, dengan tag. Ini meningkatkan kemudahan administrasi dengan urutan besarnya.
Kami membeli beberapa lisensi untuk bekerja. Masalahnya, seperti biasa, adalah satu - harga. $ 200 per lisensi adalah hal yang manusiawi untuk Enterprise dan tidak terlalu untuk penggunaan pribadi.

Ada versi gratis dari produk ini. Di dalam RDM flagship yang sama, tetapi fungsinya agak sederhana. Kurangnya warisan kata sandi dan cinta akan memori zhora terpaksa mencari-cari dan kembali mencari alternatif untuk penggunaan pribadi.

Dan dia ditemukan. Ternyata mRemote tua yang baik dibawa ke keadaan operasional lebih dari.



Proyek open source berlisensi di bawah GPLv2. Dilihat oleh halaman di GitHub, ia memiliki setidaknya 2 pengembang aktif dan beberapa lusin komuter. Dan mereka benar-benar memperbaiki bug dan menambahkan fitur baru.

Program ini berfungsi dengan baik di Windows 10. Mod dempul bertanggung jawab untuk SSH, dan bukan komponen pihak ketiga seperti di RDM (yang pihak ketiga biasanya "bug / hang / slow down" dalam praktiknya).

Secara singkat membahas fungsi.

Koneksi : RDP, SSH, ICA, VNC, Telnet, HTTP / HTTPS.



Warisan : properti koneksi apa pun, termasuk kata sandi. Warisan hierarkis.



Penyimpanan koneksi : dalam file, dalam SQL Server.



Enkripsi : enkripsi penuh dari file koneksi, pilihan tiga algoritma, enkripsi kata sandi dalam SQL Server.



Impor koneksi : dari pemindai port bawaan, Active Directory, RDCMan.



Ada yang mencari koneksi, tetapi sayangnya dengan RDM tidak pergi ke perbandingan apa pun.
Ya, proyek ini jauh dari desain komersial terbaik, tetapi sepenuhnya berfungsi, berkembang dan memiliki forum yang relatif ramai tentang Reddit dan pelacak bug.
Saya meringkas . Sepertinya saya bahwa hari ini mRemoteNG adalah alternatif terbaik untuk manajer koneksi komersial. 7zip menggantikan WinZIP dan WinRAR. qBittorent menggantikan mTorrent.

Hal-hal baik perlu didukung. Terutama Open Source.
