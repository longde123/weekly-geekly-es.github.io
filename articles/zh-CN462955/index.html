现场人员培训和认证的数字化转型

今天，我们将讨论一种解决方案，该解决方案可以使员工的持续培训过程自动化，并从该过程中排除书面工作。

我们的客户是一家联邦制造公司。 她的采购员负责商店中的货物可用性，货架代表，POSM材料（广告），卖方对产品的了解等。主管负责培训，评估和控制现场员工。 为此有公司标准和明确规定的流程。

以前，有关培训或认证结果的宝贵信息仅记录在纸上。 白天，主管和采购员一起经过了20个销售点，并在每个销售点中都填写了摘要表。 然后将其处理以给人反馈。



True Engineering的我们创建了一个技术工具，可以帮助主管计划团队的发展，记录结果并进行盘点。 本文将提供详细信息。

开发过程是如何组织的


开发过程是多阶段的。 在零售店的售货员操作期间提供培训。 伴奏与培训和认证交替进行。

持续发展的过程解决了两个问题：

1.由于该领域的频繁护送，使初学者的职业快速成长

主管选择负责人并设定2个支持目标，例如，发展演讲技巧和反对异议。 然后，他打印出调查表，并与采购员一起沿着路线到达20个销售点。 他整天监视采购员的工作，检查员工是否考虑了上次提出的建议和意见，提出建议并将所有结果记录在问卷中。 最后，他总结并起草了一份发展计划，在下一个伴奏中将予以考虑。

2.通过定期的认证和网点审核来保持经验丰富的员工的工作质量

不仅要进行认证，还要进行培训：主管陪同员工在零售商店中工作，并监督其工作质量。 审核是在没有采购员的情况下进行的。 主管本人会去购物，事先已为每个网点打印了关键指标。 在访问期间，他填写了摘要表格，并在电话上做了照片报告。 主管根据结果调整开发计划并提出建议。

因此，到了一天结束时，主管已经完成了纸质表格。 然后，他需要额外的时间来准备最终结果以反馈给员工，并将整个团队的结果转移到Excel电子表格中。 所有这些都有助于跟踪团队发展的动态并计划下一次活动。 一个月之内，主管必须与每位下属会面，平均每人有7位下属。

在此过程中存在明显的困难：


  1. 很多文书工作。 人们填写了纸质问卷-不方便且累人。
  2. 数据是零散的：销售商的评级是纸质的，产品的布局图片在电话中。 为了提供反馈，需要将它们放在一起进行处理。
  3. 为了评估级别并了解特定员工或团队的发展动力，有必要手动计算每个问卷的分数并将这些结果写入Excel电子表格。
  4. 很难实现跨领域学习的目标-以便每次下一次会议都能产生一个单一结果并考虑到过去的会议。 押运的历史记录保留在纸质形式中，与Excel表中的结果无关，因此也没有进行任何分析。

解决方案组成


我们的解决方案包括一个Web门户和一个iPad的移动应用程序。



主管在门户上计划为期一周的培训，认证和网点审核。 在陪同下，主管在移动应用程序中填写调查表：根据公司标准评估工作质量，计划目标，指出员工的优缺点并制定自我发展的任务。 然后，它将反馈提供给售货员，并将结果存储到门户。

返回门户后，主管将分析团队或特定员工在此期间的结果和发展动态。 根据这些信息，他计划进行以下活动。



我们开发了门户网站Angular 5（当时的最新版本）、. Net Core 2.2上的后端以及Swift上的移动应用程序。

后端是在微服务架构的基础上实现的，并且根据单一职责将其分为9个独立的服务。 一种服务只处理计划，另一种服务-保存并显示结果，第三种服务-与移动应用程序进行交互，第四种服务-处理照片等。 因此，一项服务的更改绝不会影响另一项服务的工作。



微服务通过RabbitMQ +大众运输总线相互通信。 每个服务都有其自己的数据库，禁止访问“外部”数据库。

还有一个单独的客户数据库，用于存储用户及其角色，从属关系，销售点列表，产品信息和其他数据的列表。 此数据由Service.Integration集成服务同步。 （ 我们之前在Habr上写了更多有关体系结构的文章 ）。

培训和认证的数字化转型


• 活动策划

事实证明，在计划过程中，主管依赖太多因素：

  • 员工能力水平，
  • 活动顺序
  • 最近的培训，认证，审核的结果，
  • 优点和缺点
  • 设定发展目标
  • 每个员工花了一个月的时间。

因此，我们不仅提出了UI / UX，还提出了一种经过深思熟虑和直观的计划方法，以便主管可以看到所有用于决策的摘要信息，而不会在学习过程中遗漏任何东西。

主管在门户网站上看到了他的整个团队，并计划了下个月的一系列活动，同时考虑了有关所花费的时间和之前进行的活动的所有信息。 这种方法为整个团队提供了更加统一的培训：所有关注的人员和指标都得到了关注。 如果本周初学者经过培训，则需要计划下周的认证，以了解自那时以来的进展，然后可以审核出口。 所有这些都很容易在每月演示文稿中进行组织，而计划则可以通过拖放轻松更改。



每周表演已用于计划活动的更详细tin，以免忽略流程。



• 活动设置

主管打开计划的活动卡，该卡将自动替换过去护送员工的有关优缺点的信息，以便在为即将到来的培训选择目标时更容易记住他的水平并做出明智的决定。 在这种情况下，用户可以添加评论并查看先前护送的结果。

计划活动的卡片与该事件的问卷模板直接相关。 我们为所有类型的事件创建了模板：用于培训，认证和审计的支持。 模板是一组不同的问题和答案，在此基础上构成了问卷。

在模板中，您可以创建问题列表，问题序列，类型，权重和形式：

  • 有答案是/否
  • 从列表中选择一个选项，
  • 文本框等。

可以将不同的问题分组。 例如，您可以生成带有产品列表的块，并且在每个字段的对面是一个用于输入数量的字段。 或将问题列表按活动领域分组，例如，所有问题以测试处理异议的技能。

Web门户与计划服务一起使用，该服务通过数据总线将计划活动中的所有更改通知其他服务。

• 移动应用程序与门户的同步

在门户上进行计划后，主管将打开一个移动应用程序，该应用程序将自动调用Service.Mobile，并请求一份计划的支持和审核列表以及相应的问卷模板。

• 支持并填写问卷

在移动应用程序的主屏幕上，显示有关员工及其能力水平的信息，以及在维护或审核过程中必须完成的一系列调查表：

  • 现场培训的准备工作-将在一天之初完成，
  • 问卷模板-为每个网点填写，
  • 工作标准-在工作日结束时完成。



主管陪同下属在商店里，观察他的工作并回答问卷。 在每个配置文件中，我们都会显示员工优缺点的摘要，以便主管将重点放在开发特定技能上。 我们还将显示带有当前支持结果的标尺，以便主管可以在此处和现在对其下属进行工作。

在支持过程中，主管对一组问题进行评论，并提供照片报告，以便在工作日结束时提供全面的反馈。 每份调查表都有一整天的优点和缺点，是进一步规划员工发展的指南。



•将配置文件发送到服务器

在一天结束时，主管将向售货员提供更多反馈，并将维护结果发送到门户。 首先，已完成问卷的基本数据消失了，然后是照片。



• 处理结果

概要文件发送到服务器并显示在Web门户上，其结构类似于模板。 数据也以通用格式存储，并且与其他组件分开存储。 因此，我们从问卷结构的变化甚至报告中都获得了独立性。

结果使领导者可以建立绩效各方面的报告：整个团队和每个员工。

  • 其中包含有关一个月内创建了多少次会议和创建哪种类型，发生了多少次会议的信息，
  • 您可以获得有关下属能力的报告（所有内容均以点为单位进行评估，以便可以直观地看到动态情况），
  • 您可以分析整个团队的发展动力。 如有必要，请查看特定支持或审核的详细结果，以及问题的答案，评论，照片报告和其他说明，以了解更改的原因。
  • 整个俄罗斯都有每个地区和城市的统计数据。

该报告可以保存为PDF或Excel。

使用SQL Server Reporting Services生成报告。 实际上，使用报告并不是一件容易的事，因为我们希望即使在表格中也要保持设计风格的连续性，细微差别足以满足另一篇文章的要求。 我们保证在不久的将来，我们将写一篇详细的文章介绍所使用的解决方案。

结果


该解决方案使在不同质量级别上对员工进行培训和认证成为可能，以分析员工的能力和技能。 它已经运行了仅10个月，在这段时间内，已经进行了3,700多次维护和审核。

iPad应用程序已替换摘要纸张配置文件。 大大提高了灌装速度和监督人员的效率。 现在还有更多时间用于观察，培训和提供详细的反馈。

门户部分使您可以单击查看报告，按员工，团队和每种活动类型上传动态。 从报告中，您可以看到已完成的问卷调查，还可以查看与每个员工进行的每次会议的详细信息。 了解更改的原因。 查看照片报告和评论。 并基于许多因素和集成分析计划团队的进一步发展。

进一步发展


该系统的体系结构具有很大的扩展和开发潜力。
