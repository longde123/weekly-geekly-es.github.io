智能手机在哪里发展：成为成熟计算机的路线图

在销售下降的情况下，有一些原因可以推测智能手机市场的疲软：似乎是由iPhone推出并建立在其内在潜在创意发展的基础上的现代智能手机时代即将结束。 但这并不是完全意义上的时代的终结，而只是链式电话→手机→智能手机→智能（？）即将发生的演进过渡。 很有可能，在下一阶段，这将不再是“背景”，今天可以将大量特定且不可剥夺的背景移到背景之外（例如内存）或进行很大的更改（例如输入和输出方法） ），则无法处理当今熟悉的类别。 在扣除所有可能的“变量”之后，为了只留下最低限度的最低要求，我得到了4个条件：便携式个人通信计算设备。 但是，这里更重要的不是确保代代相干性的因素，而是使该设备具有新品质的因素，这将使我们可以谈论进化的下一阶段。

实际上，最能满足这四个参数的“新质量”的最有前途和设计思想之一就是将功能强大的PC的全部功能推向智能手机外形的悠久而众所周知的想法。 与沿着AR或VR突变的路径移动相比，此选项的优势是非常明显的，它增加了必要的其他未知数-创建某些VR或AR功能，这将是绝对必须具备的，这证明了将开发重点放在实现上是合理的它的能力。

重新发明智能格式的PC将具有已经可以理解的优势，足以为它积极扎根。

  2. 解决软件碎片化的问题。 我记得在讨论Windows Phone挑战苹果和Google的移动操作系统的机会时，我首先读到了这样一个想法，即智能手机最重要的参数是可用于其的应用程序数量。 除此之外，我们还可以补充说，减少平台数量将释放其开发的潜在潜力，由于减少了用户细分，因此使其开发既便宜，更快又更具吸引力。
  3. 解决多平台站点的问题。 我们正在谈论站点尝试掌握移动平台的尝试，其复杂性可以通过Habr的例子直接说明，除了桌面版本的启动和重新启动移动应用程序，同时开发站点的移动版本，试图保持平衡，兼顾这三个方面之外，最后，做出选择支持开发该网站的移动版本。 我认为，这种选择现在是最好的选择，但是如果花在应用程序上的精力立即用于开发Web版本，那么它可以从用户那里获得多少功能。 单个操作系统将使您摆脱昂贵的不必要实体的烦恼，并且输出方法的开发可以将过去强制性的移动和全尺寸屏幕开发发送出去。
  4. 智能手机和平板电脑与笔记本电脑之间的竞争即将结束。 笔记本电脑市场似乎正在显示出增长的迹象-但是，很有可能是由于台式机市场份额的更大蚕食。 带有便携式设备的笔记本电脑的竞争一直是单方面的：小工具攻击了笔记本电脑-并且没有理由相信这种情况会有所改变，因为a）带有强大智能手机的笔记本电脑属于同一价格类别，显然让很多用户无法选择， b）由于紧凑性，移动性和移动网络比Wi-Fi更好的覆盖范围，几乎不可能放弃智能手机而使用笔记本电脑-仅作为补充，而拒绝PC /笔记本电脑支持智能手机/平板电脑 - 真正的实惠选择，尽管有妥协活用了许多严重程度不一。 市场的这种蚕食没有任何好处-主要是针对笔记本电脑用户。 我对笔记本电脑从内存和处理器中的“卸载”感到非常满意，如果它们使笔记本电脑更容易使用，范围更广，价格更实惠，则可以将它们转变为专注于键盘显示三合一的瘦客户机。

柔软的 扎根将计算机与智能手机以一种外形形式结合的主要原因之一-通用操作系统-也是创建这种混合动力的最大困难。

好消息是，苹果，谷歌和微软的三大制造商都在朝着三个不同的方向努力解决这个问题。 不好-他们每个人的中间结果。

苹果已经正式选择了iOS与Mac OS兼容的途径，并宣布了在桌面环境中启动移动应用程序的可能性。 这项承诺的实现始于2018年Mac OS Mojave的发布，其默认安装套件中出现了四个自己的iOS应用程序 。 这似乎是一个好主意，它将通过AppStore庞大的移动产品组合扩展可负担得起的笔记本电脑应用程序的范围，并使某些开发人员可以专注于针对一种平台而不是两种平台开发产品。 但是，从单个平台的角度来看，从理论上讲，这种运动应该朝相反的方向进行-适应桌面应用程序，包括用于移动环境的繁重且占用大量资源的工作工具。 一般来说，前进的道路看起来更长。

微软是创建统一操作系统的先驱-所有人都有不好的榜样 。 从移动电话的前iPhone时代的通信者时代到2010年的全新Windows Phone的古老变革，不仅是在Android和iOS之间获得市场立足点的一次稍晚的尝试，而且相反，它是一种创建通用操作系统的先进计划。系统。 在2012年发布的Windows 8中，基于用户的在线帐户，该用户帐户首次成为端到端帐户，并且从WinPhone移植的“平铺” Metro设计元素出现在界面中。 移植得太粗鲁了：没有想象力，屏幕上的简洁和时尚在5-6英寸“笔记本13-15上已经存在的移动界面元素”看起来很糟糕。 最高点是用覆盖整个屏幕的一块面板替代了通常的“开始”，这既浪费了空间，又保留了部分必要的功能。

Windows 8给人留下了双重印象-它既向前又向前迈进了一步。 根据移动操作系统的模式重新绘制桌面界面是一种架构上的疯狂。 -“ Windows 8有什么问题 ”

在Windows 8.1的更新中，回滚了以前的“开始”概念。 随着Windows Phone的关闭以及2015年Windows 10的发布，Microsoft的工作重点是逐步将桌面OS设计重新设计为触摸界面。 最近，当我不得不将具有触摸屏的旧版联想Carbon X1转移了一个星期时，我碰巧对MS在该领域的成功感到赞赏。 重绘的界面窗口的数量已显着增加，但仍远未完成。 重画的质量通常也令人沮丧-尽管具有触摸屏，但具有键盘控件的老式桌面模式却比平板电脑模式方便得多。

由于缺乏产品组合中的桌面操作系统，因此Google将Chrome浏览器变成了操作系统，这是它最适合桌面操作系统的产品，并且鉴于其跨平台的特性，我认为它是原则上最受欢迎的桌面应用程序。

这显然是有道理的，也有很多道理，但尚未听到任何大的突破。 Google的优势是三巨头从头开始重新发明操作系统的独特机会，而且实际上，在大多数情况下，计算机（主要是浏览器）需要大量的操作系统。

但是，对于需要其他应用程序（包括大量专业软件）的部分受众来说，Google显然并不光彩。 正是这一细分市场产生了对最大硬件功能的需求，从而阻止了MS和Apple简单地放弃其工作环境已经发展了多年的台式机操作系统的遗留物-并准备为此付出代价，这意味着Google设备类别可能会进行重大分层，其中一个派对，苹果与微软合作。
