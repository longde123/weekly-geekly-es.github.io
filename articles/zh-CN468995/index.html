开放性政策：用户如何影响项目

如果公司和受众不准备进行互动，那么信息公开有时弊大于利。 向玩家介绍所有产品，并给予他们影响开发的机会，就像走在冰上一样：失败的机会很大。 但是，如果您学会倾听用户并与他诚实，那么增加的开放度不仅会对声誉产生积极影响，还会对项目本身产生积极影响。



一年前，Plarium Krasnodar客户体验和支持部门的员工提议改变我们与用户交流并向他们提供信息的方式。 支持部门负责人Ivan Svintsov和社区与声誉部门主管Anna Zakharova谈到了新政策的出色之处，并展示了如何改善与受众互动的具体示例。

何时更改某些东西


开放的政策不是一天之内匆忙做出的决定，也不是“因为我想要这样做”。 在尝试新方法之前，请评估产品的状态，公司与用户进行公开对话的意愿以及反馈的质量。

“在Plarium，一切都有机地朝着采用新概念的方向发展，”伊万回忆说。 -玩家开始在社交网络上与我们交谈更多。 他们对只写评论而忘记它并不感兴趣。 他们想要更真诚的沟通。”

为什么要做这一切


我们公司信息透明的好处是什么？ 客户体验和支持部门的人员首先问了自己这个问题，并决定采用一种新方法将有所帮助：

  1. 提高用户服务质量。
  2. 增加在社交网络上发表正面评论的比例。
  3. 改善客户体验管理。
  4. 在游戏中以及与公司的沟通中增加观众的参与度。

“我们已经做了很多工作来收集玩家的报价，监控社交网络的情绪，创建有用的内容。 但是我们决定走得更远，开始考虑如何使我们的工作变得更好，”安娜说。

从哪里开始


在做出有关用户的任何决定之前，对目标受众进行分析非常重要。 因此，CE＆S部门的员工首先进行了大量研究，找出了玩家想听的内容以及公司准备说的话。

在回顾了其他开发人员的案例之后，他们得出了结论，并意识到了如何与受众互动，以及应该避免的事情。

新政策要求更改工作流程，因此准备工作的重要一步是重组公司团队，部门和部门之间的沟通。 毕竟，如果他们的工作不同步，结果将像是寓言中关于天鹅，派克和癌症的故事-手推车将留在原地。

Plarium发生了什么变化


“当管理层批准这一概念时，我们并没有一千个新闻震惊我们的参与者。 我们扩大了界限，但每个月都会扩大，”伊万说。

新政策已改变了与观众的所有交流-从社交网络上的帖子到支持回应。 因此，一年来已经完成了巨大的工作。

下表显示了与《维京传奇：部落战争》游戏用户的交流变化的示例。



怕什么


“我们不努力实现完全的信息透明性，”安娜说。 “在准备阶段，当我们分析国际经验时，我们意识到，无论看起来多么高贵，都不应尝试很多事情。”

伙计们检查了几起案件，以警告那些会遵循开放态度的人的错误。

情况一
开发人员鼓励用户在公司页面的讨论中写所有发现的错误。
结果：官方社区变成了鞭打磁带。 而且，在大多数情况下，玩家将机制或复杂功能误认为是游戏中的错误。 主持人无法为所有人提供详细的答案，因为他们无法跟上用户编写的所有内容。 反过来，那些人没有在正确的时间获得正确的信息，负面影响正在增加。 新玩家被大量的“臭虫”吓倒了。
解决方案：使用社交网络不是为了跟踪错误，而是为订户提供有关机制和新功能的有用信息。

情况二
开发人员报告游戏中的任何问题。
结果：“如果您发布有关每个本地故障的帖子，那么负面的增长是不可避免的。 玩家一看到问题就开始提出疑问，甚至不了解此案是否与他们有关，”安娜评论道。 结果，不仅用户的不满在增加，而且在检查每个请求时，支持专家的负担也越来越大。
解决方案：支持服务应根据当地情况而定。 “与每个用户分别沟通，” Ivan补充道，“我们可以收集有关该问题的更多信息，这意味着我们将更快，更高效地解决问题。”

情况3
即将在社交网络上详细描述即将到来的更新。
结果：“开发是一个复杂的过程，并且公司从一开始就对功能的看法到实施时就会改变，”安娜分享了她的经验。 -结果，如果我们在发布某个功能之前就谈论过该功能，然后又进行了一些更改，则用户会感到受骗。 对公司的信心正在下降。”
解决方案：与其他部门一起制定可以在外部资源上涵盖的主题列表，而不会冒着欺骗用户期望的风险。

铂金体验
“当我们在发布新功能之前很久就开始撰写更新时，我们遇到了一个有趣的现象，”安娜回忆道。 -用户以为，由于我们现在正在编写，因此该更新将在今天，明天或一周内最多发布。 他们的不耐烦变成了烦人，他们越来越多地向社区服务提出“好，何时？”的问题。 为了调整观众的期望，我们开始尽可能地告知您计划进行更新的时间。 对于公告，我们选择了对公司和用户都方便的时间。”

总结


基于2019年1月至5月的社交网络监控结果和支持服务的工作，CE＆S部门的人员总结了以下开放性政策结果：

  1. 对玩家的帮助达到了新的水平。 由于提交日志的格式发生了变化，与此相关的肯定调用次数增加了2倍，平均迭代次数从2.9减少到2.7。
  2. 观众对游戏中出现的问题变得更加宽容，并以更加中立的方式进行讨论。 在社交网络上，旨在提高开放度的帖子中负面评论的比例已大大降低（VK降低了22.5％，Facebook降低了15.4％）。
  3. 用户忠诚度不断提高。 开发者更开放的答案以及他对自己错误的认识引起了玩家的认可。 以积极的语调呼叫支持服务的总次数增加了近10倍，以消极的语调呼叫减少了7倍。 支持的平均评分从1.66增加到4.25。
  4. 观众越来越多地参与新功能的讨论。 VK和FB中一个帖子下的平均评论数增加了约57％。




CE＆S部门的员工对通过开放政策能够与用户建立建设性关系充满信心。 玩家感到了开发的一部分，公司从他们那里得到了有用的反馈。

小费


无论开放政策的结果如何令人鼓舞，有时都很难遵循。 Anna和Ivan分享了一些原则，这些原则将帮助那些正在引入新工作方法的人：

  1. 实验并坚持不懈。 要证明新用户通信策略的优势并据此重建工作流程，需要花费大量的毅力。
  2. 跟随趋势。 学习其他公司的正面和负面经验。
  3. 分析您的工作和方法。 不要仅仅依靠自己的感受。
  4. 考虑一下风险和机遇：受众准备做什么，产品处于何种状况，公司是否可以重建流程。
  5. 不要害怕您的用户。 他们确实有很多有用的想法。
  6. 开放自己，听听和听听反馈。
