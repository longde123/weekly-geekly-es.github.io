Überblick über die Arbeit mit Universal Links: Vor- und Nachteile

In diesem Artikel zeigen wir im Auftrag von Getloyal , einem automatisierten Retargeting-System für mobile Anwendungen, wie Universal Links in einer Remarketing-Kampagne funktionieren und wie Sie iOS und Android einrichten. Berücksichtigen Sie auch die wichtigsten Vor- und Nachteile dieser Technologie.


In einer Remarketing-Kampagne ist die Umstellung auf den Kauf mit Deeplinks um ein Vielfaches höher als bei herkömmlichen Links. Deeplink kann nicht nur die Anwendung öffnen, sondern auch einen bestimmten Bildschirm innerhalb der Anwendung. Damit der Link zum gewünschten Bildschirm in der Anwendung führt, müssen Sie das URL-Schema konfigurieren. Die Implementierung von URL-Schemata (die Struktur von Diplinks) ist ein wesentlicher Bestandteil des mobilen Entwicklungsprozesses.

Nur wenige wissen, dass Apple auf der WWDC15-Konferenz die Unterstützung von URL-Schemata für direkte Links eingestellt und eine neue Methode zur Implementierung auf mobilen Plattformen eingeführt hat - Seamless Linking. Über solche Links kann der Benutzer direkt zur Anwendung wechseln, ohne Safari öffnen zu müssen.

Ab iOS 9 führte Apple Universal Links ein. Der Hauptunterschied zwischen Universal Links und den üblichen benutzerdefinierten URL-Schemata besteht darin, dass der Link zur Instagram-Seite von Yegor Creed folgendermaßen aussah: instagram: // user? Username = egorkreed, und mit dem Aufkommen von Universal Links sieht er folgendermaßen aus: www.instagram.com / egorkreed .

Anleitungen zur Implementierung von URL-Schemata finden Sie weiterhin, jedoch nicht auf der Apple-Website. Was ist mit Android? Google ist auch für Datenschutz und UX, daher unterstützen Geräte seit Android 6.0 Android App Links. Dies ist fast dasselbe wie bei Universal Links in iOS.

Viele Entwickler haben bereits auf Universal Links umgestellt - dies vereinfacht die Konfiguration von Links und verbessert die Benutzererfahrung.

Wie funktioniert es


Wenn Sie auf Universal Link klicken, ermittelt das Betriebssystem automatisch, ob die Anwendung auf dem Gerät installiert ist. In diesem Fall öffnet der Benutzer die Anwendung. Wenn nicht, wird der Benutzer auf die Site umgeleitet. Dies kann eine Standardumleitung zum AppStore sein.

Durch das Herunterladen einer Anwendung, die diese Technologie implementiert, lädt der Benutzer auch eine Datei herunter, die die Routing-Regeln enthält, an die der Benutzer in dem einen oder anderen Fall gesendet werden soll.


Beispiele


Es gibt Unternehmen, die Universal Links bereits in ihren Anwendungen implementiert haben und das Richtige getan haben, weil alle davon profitiert haben:

  1. Benutzer: Das Teilen von Inhalten ist bequemer geworden. Das UX-Niveau hat sich deutlich erhöht.
  2. Partner: Promotion-Partner sind ebenfalls in einer erfolgreichen Position: CRE (Conversion to Re-Engagement) hat zugenommen, Sie können skalieren, der Werbetreibende wird sich freuen.
  3. Produkt: Die Umstellung auf Kauf steigt, das Retargeting funktioniert effizienter, die Kosten für die Gewinnung eines eindeutigen Benutzers werden reduziert.

Implementierung


Die Implementierung von Universal Links ist nicht schwierig. Die Ansätze unterscheiden sich jedoch je nach Plattform geringfügig.

iOS:

  1. Verknüpfen Sie eine Anwendung mit einer Domain (eine Domain kann entweder Ihre eigene oder eine Werbe-Tracker sein) in den Anwendungseinstellungen über Xcode.
  2. Erstellen Sie eine Zuordnungsdatei - eine Seite, die Daten im JSON-Format enthält, dh Inhalte, die in der Anwendung verfügbar sind.
  3. In der Appdelegate-Datei müssen Sie die Regeln für die Verarbeitung von Klicks auf Universal Link konfigurieren. Wenn die Anwendung installiert ist, wird die angeforderte Seite geöffnet. Wenn sie nicht installiert ist oder nicht den erforderlichen Inhalt bereitstellen kann, wird die Seite in Safari geöffnet.

Android:

  1. Implementieren Sie das Schema der Diplinks, die zu bestimmten Seiten in der Anwendung führen.
  2. Fügen Sie der Anwendung eine Regel hinzu, um App-Links zum Öffnen der Anwendung anzufordern.
  3. Veröffentlichen Sie die Zuordnungsdatei im JSON-Format in Ihrer Domain (es kann sich entweder um Ihre eigene Anwendungsdomäne oder um Werbe-Tracker-Domains handeln).

Nachteile von Universal Links:


  • Die Komplexität der Implementierung. Trotz der Tatsache, dass Universal Links mit wenigen Klicks konfiguriert wird, gibt es Nuancen im Setup-Prozess, bei denen Sie einen Fehler machen können.
  • Probleme verfolgen. Manchmal gibt es Probleme mit der Anzeige von Neuzuordnungen. Ein solcher Fehler ist schwer zu erkennen und die Veröffentlichung der Anwendung mit Universal Link wird verschoben.
  • Die Komplexität des Teilens. Wenn Sie Universal Link in iOS über Messenger freigeben oder den Link in das Safari-URL-Feld einfügen, wird die Site in Safari geöffnet. Wenn Sie den Link jedoch zu Facebook (App), Twitter (App), Mail (App) oder zum Banner auf der Site hinzufügen, wird die Anwendung geöffnet .

Vorteile von Universal Links:


Wir im Getloyal- Team haben festgestellt, dass die überwiegende Mehrheit der Mobilfunkunternehmen das URL-Link-Schema in der Android-Version der Anwendung implementiert hat. In iOS-Anwendungen funktionieren myapp: // links ebenfalls. Viele mögen sich fragen: "Warum brauchen wir Universal Links, wenn alles schon funktioniert?" Um diese Frage zu beantworten, haben wir nachfolgend eine Liste der Vorteile bereitgestellt:

  • Einzigartigkeit. Im Gegensatz zu URL-Schemata können Universal Links nicht von anderen Anwendungen belegt werden, da sie Standard-HTTP- oder HTTPS-Protokolle verwenden.
  • Flexibilität. Universal Links funktionieren auch dann, wenn die Anwendung nicht installiert ist. Wenn die Anwendung nicht installiert ist, öffnet der Benutzer den gewünschten Inhalt in Safari.
  • Super UX. Ein Link: sowohl auf der Website als auch in der Anwendung;
  • Unterstützung für Werbeplattformen. Auf vielen Websites, z. B. MyTarget, können Sie keine herkömmlichen Diplinks mehr in einen Link einfügen.
  • Durch die einfache Freigabe können Benutzer Links freigeben, die mit ... beginnen, da ein solcher Link viel vertrauter aussieht.
