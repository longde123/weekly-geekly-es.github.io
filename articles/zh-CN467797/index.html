“为……而笑”：为什么您需要没有“战斗”应用程序的软件工具

GitHub托管着超过1亿个存储库。 其中一些是大型公司使用的应用程序。 在其他情况下，开发人员可以在其项目中积极使用小型库。 但是有些存储库实际上没有用。

我们将讨论它们为何引起关注。 提示：星期三是一个星期五。

当你不想工作时


想象一种情况：您刚刚完成编写新模块并关闭了所有当前任务。 但是，直到工作日结束还有一个小时，而且您的公司通常比老板提早回家的习惯。 同时，管理层不喜欢员工闲置时的状态，而是时刻准备着执行其他任务。 特殊的应用程序将帮助您，这将有助于描述风暴活动。

这样的应用程序之一就是HackerTyper 。 它将任何键入的字符集转换为一种程序代码。 该实用程序的示例在项目网站上 。 HackerTyper也有一个替代选择-GeekTyper 。 当Reddit居民在他所在城市的当地电视台拍摄有关他在游戏开发公司的工作的故事时，便使用了此工具 ：


在GitHub上，您可以找到无法延长工作时间但可以帮助减少工作时间的工具，例如Volkswagen应用程序。 通过持续集成 （CI）服务器检查代码后，大众汽车减少了错误数量，从而使测试成功。 因此，您不必担心测试，而可以更加关注其他“更重要”的任务。 该实用程序以德国汽车制造商的名字命名，因为它是在围绕汽车制造商的丑闻爆发后立即开发的。 四年前，在柴油车中，公司发现了软件，该软件低估了测试过程中废气中有害气体的数量。

还值得注意的是“ 无代码”实用程序。 如果您根本不想触摸键盘，可以与她联系。 尽管很难将其称为实用程序-在存储库中您将找不到一行代码。 要开始使用无代码， 什么都不做 。 根据作者的说法，这是编写可靠和安全的应用程序的最佳方法。

何时保存数据


在GitHub存储库中，您可以找到πfs文件系统 ，该系统通过将数据保存在Pi编号中来节省磁盘空间。 文件系统的操作基于以下陈述：π中可以满足任何数字序列。 您可以使用以下命令保存数据： 

πfs -o mdd=<metadata directory> <mountpoint>

方括号中的第一个参数是πfs存储已保存文件的元数据的目录，第二个参数是标准文件系统的安装点。 如果您知道文件偏移及其长度（以π为单位），则可以使用Bailey – Borwain – Pluff公式将其提取，以找到数字Pi中必要的数字序列。

普遍认为，Pi号不仅可以用作已知信息的存储库，还可以用作未知数据的存储库。 从理论上讲，您可以从πfs中提取未写成的书籍，未完成的照片和未拍摄的电影-最重要的是，获取它们的元数据。

当您需要不寻常的服务时


在IT行业中，有XaaS（一切即服务）一词-“一切都作为服务”。 它描述了所有使用云计算的服务。 在网络上，您可以找到将这个定义提高到新水平的实用程序。 例如， booleans.io服务，它提供“布尔表达式作为服务”。 这是一个API集，可用于创建，删除和更新布尔对象。 例如，根据请求： 

 curl -X POST https://api.booleans.io --header "Content-Type: application/json" --data '{"value": true, "label": "Hello world!"}' --header "Authorization: Token [token]

该服务将生成以下响应： 
 { "id": "4ee4b8f7-8d31-4ae2-93b0-554e19af740d", "created_at": "2018-11-24 14:56:26", "updated_at": "2018-11-24 14:57:43", "value": true, "label": "Hello world!" }

另一个示例是shoutcloud.io 。 这是“全部大写作为服务”，它会自动将小写字母转换为大写字母。 使用特殊的API就足够了： 

 curl -X POST \ -d '{"INPUT": "hello world"}' \ -H 'Content-Type: application/json' \ HTTP://API.SHOUTCLOUD.IO/V1/SHOUT { "INPUT":"hello world", "OUTPUT":"HELLO WORLD" }
