Autorisation en EIES sur un serveur terminal avec signature numérique conformément à GOST-2012

Bonjour, Khabrovsk.

Collègues travaillant dans le domaine de l'Etat. les marchés publics ont déjà fait l'expérience de l'exigence obligatoire du gouvernement concernant l'utilisation de médias avec une signature numérique électronique GOST R 34.10-2012. L'utilisation d'EDS selon le nouveau GOST est obligatoire à partir du 1er janvier 2019, et pour travailler sur les portails zakupki.gov et gosuslugi.ru, il n'est plus possible d'émettre un EDS selon GOST 2001, et après le 1er janvier 2020, le support d'EDS selon l'ancien GOST sera complètement arrêté.

Combien de fois cela se produit-il. le site au 1er janvier 2018 était prêt mais pas complètement.

Pour travailler avec EDS selon GOST 2012, vous devez installer le «plug-in pour le travail
avec le portail des services publics ”version 3.0.3.0 ou 3.0.6.0, mais contrairement à la version du plug-in 2.0.6.0, les nouvelles versions ne prennent pas en charge le travail avec les chemins UNC (nous l'avons constaté lors de nombreuses expériences malsaines avec différentes versions des plug-ins), et si vous et nous utilisons des profils itinérants, alors l'autorisation sur le site ne fonctionnera pas pour vous; de plus, cela ne fonctionnera dans aucun navigateur: IE, Google Chrome, Mozilla Firefox et même Crypto Fox.

Longue correspondance avec le soutien de l'État. services, crypto pro et kontur.ekstern n'ont malheureusement pas aidé, les spécialistes de l'assistance technique du site Web des services publics se sont avérés extrêmement incompétents.

En fait, assez de mots, passons aux choses sérieuses.

Pour travailler avec le site Web de l'État Les services conformes à GOST 2012 avec des profils portables doivent essentiellement effectuer 3 actions.

  • Retirez complètement le plug-in de l'ancienne version et après avoir nettoyé les restes du système.
  • Installez le plug-in version 3.0.6.0 pour les systèmes 32 bits, même si vous utilisez un système d'exploitation 64 bits, en copiant le dossier du plug-in depuis appdata localement.
  • Modifiez manuellement le registre.

N'oubliez pas que pour effectuer les actions suivantes avec IFCPlugin, l'utilisateur doit disposer de droits d'administrateur de machine locale.

  • Suppression du plugin.

Nous supprimons le plugin de toutes les manières possibles: via l'installation, désinstallez les programmes.
En lançant msi de la version que vous avez installée (la version peut être consultée dans
Modules complémentaires IE ou dans le dossier du profil utilisateur.

Exemple: contoso.com\dfs\Profiles\AppData\Roaming\Rostelecom\IFCPlugin\3.0.6.0)
Utilisation de wmic. CMD → wmic → produit obtenir le nom → produit où nom = "nom du programme" appeler la désinstallation → Y

Nous supprimons l'intégralité du dossier «Rostelecom» du profil portable. Exemple: \\contoso.com\dfs\Profiles\%UserName%\AppData\Roaming\Rostelecom
Dans le registre, nous supprimons tous les soldes qui incluent «IFCPlugin» de la branche «HKCU».

Il est également conseillé de vider le cache d'Internet Exprorer. Ouvrez IE, appuyez sur Ctrl + Maj + Suppr, confirmez.

  • Téléchargez et installez la version 3.0.6.0 du plug-in IFCPlugin.msi 32 bits.

Après l'installation, vous devez copier le dossier du profil utilisateur portable localement sur le serveur, par exemple à partir de:
\\contoso.com\dfs\Profiles\%UserName%\AppData\Roaming\Rostelecom
Dans
C:\Users\%UserName%\AppData\Roaming\Rostelecom

  • Modification du registre.

Maintenant, la chose la plus intéressante est que vous devez trouver toutes les valeurs dans le registre qui incluent \\contoso.com\dfs\Profiles\%UserName%\AppData\Roaming\Rostelecom et les changer en C:\Users\%UserName%\AppData\Roaming\Rostelecom

Vous devriez avoir 6 à 9 remplacements.
