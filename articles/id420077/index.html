Peluang Perdagangan Online Baru

Manajemen


Pada artikel ini saya ingin berbicara tentang satu peningkatan yang secara teknis tidak terlalu sulit, tetapi, saya pikir, sangat berguna. Idenya adalah ketika menjual melalui internet, Anda dapat mengatur algoritma untuk mengubah harga barang. Artikel ini adalah tentang bagaimana saya datang kepadanya dan apa yang saya lakukan untuk mengimplementasikannya.

Jual komputer lama


Semuanya dimulai dengan komputer lama. Lebih tepatnya, saya memutuskan untuk menjualnya dan membeli yang baru. Saya ingin menjual semahal mungkin, sehingga saya harus membayar lebih sedikit untuk yang baru.

Setelah mencari-cari di internet, saya dengan cepat menentukan kisaran harga untuk komputer yang akan saya jual. Secara umum, dengan menempatkan iklan untuk dijual di situs web gratis, saya mulai menunggu pembeli saya. Saya menunggu seminggu .... kemudian satu minggu lagi ...., saya menunggu, menunggu sampai batas waktu pengumuman saya.

Batas waktu telah berakhir, tetapi saya belum menjual komputer. Tapi saya tidak putus asa, tidak berhasil menjual semuanya, Anda bisa menjualnya sedikit demi sedikit. Setelah melihat berapa banyak mereka menjual suku cadang di Internet, saya membongkar komputer dan menjual apa yang ada di dalamnya. Sejarah berulang, pengumuman digantung, tetapi tidak ada pembeli. Lalu saya mulai mengurangi harga ...

Tetapi mengurangi setiap hari adalah tugas yang sangat membosankan. Akibatnya, saya menetapkan harga minimum untuk komponen dan mulai menunggu. Dan pembeli pergi ... Saya bahkan akan mengatakan bahwa dia berlari. Orang-orang menelepon dan meminta mereka untuk tidak menjual kepada siapa pun, berjanji untuk datang secepat mungkin.

Akibatnya, semua komponen saya terjual dalam beberapa hari. Orang-orang bahagia, tetapi saya tidak terlalu bahagia. Tampaknya bagi saya bahwa di suatu tempat saya jauh lebih murah. Ada perasaan bahwa saya menurunkan harga sehingga saya benar-benar dibanjiri pelanggan. Algoritma penjualan jelas gagal di suatu tempat. Dan dia gagal pada saat saya lelah menurunkan harga sedikit setiap hari.
Dan kemudian saya berpikir ...

Ide


Tetapi bagaimana jika kita memperluas fungsi iklan situs. Pastikan harga barang yang dijual otomatis berkurang setiap hari !? Dalam hal ini, saya akan menemukan pembeli yang akan setuju untuk membayar paling untuk komponen saya. Selanjutnya, saya memutuskan untuk menggeneralisasi ide ini. Penjual, pada prinsipnya, harus dapat menetapkan algoritma untuk perilaku harga produknya. Di satu sisi, ini akan menyelamatkannya dari kebutuhan untuk menyesuaikan harga secara manual, dan di sisi lain, itu akan meningkatkan minat pada produk yang ditawarkan, karena pembeli potensial akan mulai melacak harga dengan harapan membeli dengan penawaran yang lebih baik. Algoritma perubahan harga membantu untuk menemukan barang untuk dijual pembeli. Akibatnya, ini harus menyederhanakan penjualan dan mendapatkan manfaat maksimal dari transaksi.

Implementasi


Langkah selanjutnya adalah prototipe. Saya menggunakan C #, JQuery dan Bootstrap, meskipun di tempat kerja saya menulis dalam Java dan Angular. Saya benar-benar ingin melakukan dengan cepat dan melihat apa yang terjadi, karena itu pilihan teknologi.
Dalam prosesnya, saya harus berkenalan dengan Google Maps API dan metode mengompresi dan mengubah ukuran foto di sisi klien saat memuatnya oleh pengguna.

  • Saya bertemu dengan Google Maps API di sini: developers.google.com/maps/documentation/javascript/tutorial
  • Saya mengubah ukuran foto dengan mengompresnya menggunakan proyek: github.com/rossturner/HTML5-ImageUploader

Waktu berlalu, pekerjaan berjalan lancar dan voila:

  • YouTube: www.youtube.com/watch?v=3eoCZW-3tlo
  • Kode sumber ada di sini: github.com/AlexKorole/BuyAndSell2

Algoritma


Situs web telah dibuat. Pada halaman penjual, pengguna dapat mengunduh informasi produk dan memilih algoritma untuk mengubah harga. Algoritme berikut tersedia dalam versi ini:

  • Pengurangan harga dari maksimum ke minimum

  • Kenaikan harga dari minimum ke maksimum

  • Perubahan harga secara acak, mulai dari minimum hingga maksimum

  • Harga tetap sama dengan maksimum
  • Harga tetap sama dengan minimum

Tangkapan layar menunjukkan pilihan algoritma:

Anda juga dapat menentukan langkah perubahan harga. Misalnya: 10 rubel setiap 3 detik atau 100 rubel setiap hari.

Pada halaman pembeli Anda dapat melihat barang-barang yang ditawarkan dan harga untuk mereka. Untuk melihat algoritma perubahan harga, tombol "Admin" ditambahkan.

Jika tombol "Admin" dinonaktifkan, kolom "Status Harga" tidak akan ditampilkan. Bahkan, ini akan menyembunyikan dari pembeli jenis algoritma yang dipilih untuk mengubah harga.

Kesimpulan


Dalam artikel ini, saya memutuskan untuk berbicara tentang ide yang lama menghantui saya. Mungkin itu masuk akal dan dapat membuat penjualan online menjadi alat yang lebih fleksibel dan nyaman, atau, sebaliknya, itu akan tetap menjadi ide lain yang tidak ditakdirkan untuk diterapkan. Bagaimanapun, saya menemukan itu menarik dan layak diterbitkan.
