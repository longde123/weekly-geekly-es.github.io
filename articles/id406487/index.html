Penyimpangan teknis kecil dalam gaya "Catatan Pemula Wordpress."

Terakhir kali, saya perhatikan bahwa beberapa pembaca menghargai blok informasi ini, jadi saya memutuskan untuk berbagi sedikit pengalaman lagi. Untuk pengguna berpengalaman, tidak ada yang baru atau istimewa - hanya catatan singkat saya tentang pengaturan blog Anda.

Pada bulan Agustus, saya menerima surat dalam surat saya yang memberitahukan saya tentang memperbarui mesin blog ke yang sekarang. Saya kagum pada intinya. Bukan berarti itu benar-benar mengejutkan di abad ke-21 kita, ketika perangkat lunak itu sendiri diperbarui tanpa masalah (atau masalah). Tetapi mesin situs! Saya entah bagaimana berpikir bahwa situs itu adalah hal yang cukup statis - saya menginstal, mengkonfigurasi dan berfungsi.

Sampai seseorang masuk ke file / database dengan tangan kecil mereka sendiri, sesuatu tidak akan terjadi di hoster atau peretas jahat Vasya akan datang. Inisiatif blog tidak cocok untuk saya. Setuju bahwa saya tidak ingin mendapatkan kegagalan selama pembaruan, yang akan dimasukkan situs, tetapi Anda akan mengetahuinya hanya setelah fakta dan jika Anda tidak akan dapat dengan cepat menyebarkan situs dari cadangan, maka itu akan berbohong untuk beberapa waktu. Saya akan senang dengan pemberitahuan sebelumnya tentang kemungkinan pembaruan. Ternyata untuk menonaktifkan pembaruan otomatis Wordpress, cukup menulis arahan: 

define('AUTOMATIC_UPDATER_DISABLED', true);

ke file wp-config.php. Arahan ini hanya menonaktifkan pembaruan otomatis, tetapi tidak menonaktifkan pemeriksaan berkala untuk pembaruan. Karena itu, jika ada pembaruan, dengan masuk ke panel admin di bawah admin, pemberitahuan akan segera terlihat dan pemasangannya untuk mesin, plugin dan / atau tema dapat diselesaikan secara manual.

Setelah beberapa waktu, tanda hubung yang berlimpah dari teks itu menarik perhatian saya. Dengan mereka, teks disejajarkan dengan baik secara vertikal, dengan lancar mengisi seluruh ruang halaman, tetapi sangat tidak biasa. Masalah ini diselesaikan melalui gaya - dalam style.css Anda perlu menonaktifkan tanda hubung.

Kami mencari sesuatu seperti ini: 

 /* Sidebar */ .widget-area .widget { -webkit-hyphens: auto; -moz-hyphens: auto; hyphens: auto; margin-bottom: 20px; word-wrap: break-word; } .site-content article { border-bottom: 2px double #F3F3F3; margin-bottom: 28px; padding-bottom: 24px; word-wrap: break-word; -webkit-hyphens: auto; -moz-hyphens: auto; hyphens: auto;

Dan kami membawanya ke formulir berikut: 

 /* Sidebar */ .widget-area .widget { -webkit-hyphens: none; -moz-hyphens: none; hyphens: none; margin-bottom: 20px; word-wrap: break-word; } .site-content article { border-bottom: 2px double #F3F3F3; margin-bottom: 28px; padding-bottom: 24px; word-wrap: break-word; -webkit-hyphens: none; -moz-hyphens: none; hyphens: none;

Poin lain adalah bahwa seseorang bernama Andrei secara curang menyerang blog saya dan secara sistematis mengirim surat kesalahan ketik mulai dari bab pertama. Terkadang saya merasa bahwa saya meletakkan teks cerita ini murni pada proofreading :) Andrey, terima kasih atas kerja kerasnya, dengan upaya Anda, sebagian besar bab telah diperbarui ke versi 1.0.1. Jadi, seorang teman menggunakan formulir umpan balik dari situs, yang disusun menggunakan plugin "Contact form 7". Untuk membuat form fungsional dengan plugin ini sangat sederhana - Anda perlu membuat form dan mengkonfigurasinya, form tersebut akan diberi semacam kode pendek seperti [contact-form-7 id = "101" title = "Feedback"]. Selanjutnya, buat halaman dengan alamat umpan balik dan tulis kode pendek ini ke dalamnya. Dan kemudian tautan ke halaman ini dapat ditulis di tempat yang nyaman di situs. Surat datang secara teratur dan itu menyenangkan bahwa peralatan bekerja tanpa kegagalan, tetapi menjadi sedikit tidak nyaman bagi seseorang karena kenyataan bahwa mungkin untuk mengimplementasikan fungsi ini dengan lebih nyaman.

Ya, maksud saya sistem Orphus, ketika cukup untuk memilih kata atau kombinasi kata dengan mouse dan tekan Ctrl + Enter - pesan pemberitahuan dikirim ke pemilik situs / artikel. Ada plugin khusus untuk Wordpress yang menambahkan fungsionalitas serupa ke situs. Saya terutama mencatat fungsionalitas otonom. Di abad kita, ketika diyakini bahwa "awan di celanamu" dan selalu bersamamu, hal-hal otonom memiliki nilai tertentu. Tidak seperti Orphus, tempat Anda perlu mendaftar, unduh skrip dan hubungkan ke situs, cukup instal plugin Mistape. Mengkonfigurasi pengaturan plug-in yang sangat sederhana dan pada akhir setiap artikel di situs teks pemberitahuan muncul tentang kemungkinan menggunakan Ctrl + Enter. Tapi ... ada beberapa poin.

Pertama, ketika memeriksa di browser yang berbeda, ternyata mengirimkan pemberitahuan untuk beberapa alasan secara stabil hanya bekerja ketika menggunakan Ctrl kiri. Kedua, ketika kalimat panjang dipilih dengan menekan tombol Ctrl + Enter, tidak ada yang terjadi sama sekali - itu secara eksperimental ditetapkan bahwa hanya sejumlah karakter di wilayah 10-15 kata yang dikirim tanpa masalah (tergantung pada panjang kata). Ketiga, di suatu tempat saya membaca bahwa pengembang membuat perlindungan terhadap spam (itu masuk akal, terutama jika plugin dikonfigurasi untuk menambahkan komentar ke notifikasi yang dikirim), yang tidak disebutkan dalam bantuan plugin untuk lebih dari lima pesan (walaupun saya mendapat lebih banyak ) Anda tidak dapat mengirimnya dalam waktu setengah jam - dengan menekan tombol Ctrl + Enter tidak ada yang terjadi sama sekali ketika batas tercapai. Saya menambahkan semua informasi yang tidak jelas ini ke teks notifikasi yang ditampilkan plug-in dalam artikel di situs.
