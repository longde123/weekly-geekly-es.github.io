WEB 3.0-射弹的第二种方法



首先，有一点历史。

Web 1.0是一个用于访问其所有者发布在网站上的内容的网络。 静态html页面，以只读方式访问信息，主要的乐趣是指向此站点和其他站点页面的超链接。 典型的网站格式是一种信息资源。 将离线内容传输到网络的时代：将书籍数字化，扫描图像（数码相机仍然很少见）。

Web 2.0-将人们聚集在一起的社交网络。 用户沉浸在Internet空间中，直接在网页上创建内容。 交互式动态站点，内容标记，Web联合，混搭技术，AJAX，Web服务。 信息资源让位于社交网络，博客托管，Wiki。 在线内容生成的时代。

显然，术语“ Web 1.0”仅在“ Web 2.0”出现之后才出现，指的是旧的Internet。 几乎立即，人们开始谈论未来的3.0版。 未来有多种愿景选择，所有这些选择都与克服Web 2.0的缺点和局限性有关。

Netscape.com首席执行官Jason Kalakanis主要担心用户生成内容的质量很差，他建议互联网的未来属于“会赠予他人”的人，他们将“创造高质量的内容”（Web 3.0，“官方”定义，2007年）。对于他自己而言，这是合理的，但是他没有说明如何在何处以及在哪些网站上做到这一点。

“ Web 2.0”一词的作者蒂姆·奥雷利（Tim O'Reilly）合理地建议，不需要像人这样的不可靠中介，就可以在网络上发布信息。 技术设备也可以向Internet提供数据。 同样的技术设备也可以直接从Web存储中读取数据。 实际上，蒂姆·奥雷利（Tim O'Reilly）提议将Web 3.0与我们已经熟悉的术语“物联网”相关联。

万维网的一位创始人蒂姆·伯纳斯·李（Tim Berners-Lee）在Internet的未来版本中看到了他长期（1998年）实现语义网的梦想。 他对这个术语的解释很成功-直到最近才宣布“ web 3.0”的大多数人都想到了语义网，即一个网络，其中站点页面的内容对计算机可读的机器有意义。 在2010-2012年左右的某个地方，有很多关于本体化的讨论，语义项目是成批诞生的，但是结果是众所周知的-我们仍然使用Internet版本2.0。 从本质上讲，只有Schema.org语义标记方案和Internet怪兽Google，Microsoft，Facebook，LinkedIn的知识图才能完全幸免。

强大的数字创新浪潮帮助掩盖了语义网的失败。 新闻界和普通百姓的兴趣已经转向大数据，物联网，深度培训，无人机，附加现实，当然还有区块链。 如果列表中的第一个主要是离线技术，那么从本质上讲，区块链是一个网络项目。 在2017-2018年受欢迎程度达到顶峰的时候，他甚至宣称了新互联网的作用（以太坊的约瑟夫·鲁宾的一位创始人反复表达了这个想法）。

但是时间过去了，“区块链”一词不再与对未来的突破联系在一起，而是与不合理的希望联系在一起。 重塑品牌的想法自然产生了：让我们不要将区块链视为一个自给自足的项目，而应将其包含在体现新事物和新事物的技术堆栈中。 就在这里为“新”找到了“ web 3.0”的名称（尽管不是新的）。 为了以某种方式证明名称的新颖性，我不得不在“ light”堆栈上包括一个语义网络。

因此，现在的趋势不是区块链，而是去中心化的Internet基础架构Web 3.0，它由几种基本技术组成：区块链，机器学习，语义Web和物联网。 在过去一年中出现的有关Web 3.0的新版本的许多文章中，您可以详细了解其各个组成部分，但不幸的是，没有自然问题的答案：这些技术如何集成到整体中，为什么神经网络需要物联网，以及区块链语义网？ 大多数团队只是继续参与区块链（可能希望创建一个可以克服母球的地穴，或者只是进行投资），但是使用了新的标语“ web 3.0”。 但是至少有一些关于未来的事情，而不是关于不合理的希望的事情。

但是，并非所有事情都如此悲伤。 现在，我将尝试简要回答上述问题。

为什么使用语义网络区块链？ 当然，在这里，我们不需要谈论区块链本身（一串加密链接的块），而是要谈论一种技术，该技术基于对等网络中的加密方法提供用户识别，共识验证和内容保护。 因此，像这样的网络的语义图会接收到可靠的分散式存储库，其中包含记录和用户的密码标识。 这不是您的免费托管页面的语义标记。

为什么要有条件的区块链语义？ 本体论通常是关于内容在主题领域和级别中的分布。 这意味着在对等网络上抛出的语义网（或更简单的是将网络数据组织到单个语义图中）可以确保网络的自然聚类，即其水平缩放。 图的级别组织使您可以并行处理语义独立的数据。 这是数据体系结构，不会将所有内容任意分散到块中并存储在所有节点上。

为什么物联网语义和区块链？ 有了区块链，一切似乎都是微不足道的-它需要作为具有内置系统的可靠存储，用于通过加密密钥识别参与者（包括IoT传感器）。 一方面，语义使您可以按主题集群隔离数据流，即，确保节点的卸载，另一方面，它使IoT设备发送的数据有意义，因此独立于应用程序。 您可能会忘记为应用程序API请求文档。

而且还有待发现，从机器学习和语义网络的交叉中获得的共同利益是什么？ 好吧，这里的一切都非常简单。 如果不在语义图中，那么哪里可以找到一种如此庞大的，经过验证的，结构化的，有意义地定义的数据，它们以单一格式存在，因此对于训练神经元而言是必要的吗？ 另一方面，有什么比神经网络图表更好地分析了有用或有害异常（例如，识别新概念，同义词或垃圾邮件）的神经网格呢？

我们需要这样的Web 3.0。 杰森·卡拉卡尼斯（Jason Kalakanis）会说：我说过，这将是一个由有才华的人创建高质量内容的工具。 Tim Berners-Lee将很高兴：语义规则。 蒂姆·奥雷利（Tim O'Reilly）也将是对的：当我们忘记了“进入网络”一词时，Web 3.0就是“互联网与物理世界的互动”，是消除在线和离线之间的界限。
