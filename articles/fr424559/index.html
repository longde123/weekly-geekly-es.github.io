Raison
Cookies-AutoDelete s'est avéré être, bien que complètement implicite, mais incompatible avec la technologie de niveau d'isolement de la première partie
Comme tout le monde le comprend - l'isolement est réalisé par l'isolement (ce n'est pas pour vous), y compris et les cookies, dans le cadre desquels FireFox met les «marques» appropriées dans la base de données cookies.sqlite. Et Cookies-AutoDelete l'interrompt avec son étiquette "CookiesAutoDelete" (vérifiée à l'aide de SQLiteStudio )

Pour le remplacement de «Cookies-AutoDelete», je peux offrir Cookiebro

- Bien qu'en anglais, il est facile à configurer
Il y a un éditeur de cookies
Pour les cookies préférés soigneusement stockés, il existe des paramètres de sécurité

Surligné - implique une preuve de compatibilité
Sécurisé - transmettez uniquement via https
Http uniquement - envoyer uniquement via http. Ce n'est pas dans le sens de https! voir ci-dessus. C'est dans le sens de ne pas donner à Java et à d'autres accès indésirables potentiellement dangereux, mais de les transmettre à l'ancienne, dans l'en-tête http
Hôte uniquement - ne donne pas accès aux sous-domaines au traitement (dans l'exemple - seul yandex.ru aurait accès, mais Yandex.Music, etc. - non)

Moins - pour que le cookie préféré soit conservé, vous devez ajouter le site à la liste blanche, et les cookies pour le site peuvent dépasser 9000. Très probablement, cela coûte aussi en quelque sorte, mais je n'ai pas cherché.
