O Google fala sobre uma nova iniciativa para acelerar as páginas da web em dispositivos móveis: páginas móveis aceleradas

Hoje, o Google blog oficial tem anunciou o anúncio de uma nova iniciativa aberta pela gigante de buscas acelerado Páginas Móvel (AMP), projetado para acelerar páginas da web em dispositivos móveis. Estamos falando sobre o uso por programadores da Web de uma biblioteca HTML especial de AMP, cujo código no status de uma visualização técnica está disponível no github . O símbolo não oficial da biblioteca é o relâmpago "⚡", implicando assim a velocidade do seu trabalho em dispositivos móveis.

O AMP HTML acredita que o desempenho de uma página da Web depende muito do número de código JavaScript para uma variedade de bibliotecas que implementam várias funcionalidades dinâmicas. Em vez de usar Javascript explícita ou implicitamente para elementos como vários votos ou vídeos, os desenvolvedores são incentivados a usar Web Components especificamente projetados para HTML AMP. Uma lista desses componentes pode ser encontrada aqui . O código JS desses componentes não deve afetar o desempenho da página da web.

Agora, a biblioteca HTML do AMP contém apenas 4 componentes e um validador especial, com o qual você pode verificar se a página foi criada de acordo com o padrão AMP. Se você adicionar "# development = 1" ao URL, a biblioteca executará uma série de métodos de afirmação para descobrir se o layout da página é adequado aos princípios AMP e exibir os resultados no console do navegador. Uma simples página da web deve ser algo como isto:

<!doctype html>
<html ⚡>
<head>
  <meta charset="utf-8">
  <link rel="canonical" href="hello-world.html" >
  <meta name="viewport" content="width=device-width,initial-scale=1,minimum-scale=1,maximum-scale=1,user-scalable=no,minimal-ui">
  <script src="https://cdn.ampproject.org/v0.js" async></script>
  <style>body {opacity: 0}</style><noscript><style>body {opacity: 1}</style></noscript>
</head>
<body>Hello World!</body>
</html>
