ODROID-C2是一台64位四处理器单板计算机，这是目前ARM世界中最有效的面向开发人员的64位计算机之一。
将于2016年3月2日上市，价格为40美元。批量生产将于2月15日开始，3月4日开始首批发货。

规格：

Amlogic S905（ARM®Cortex®-A53（ARMv8）2Ghz四核CPU）
2Gbyte DDR3 SDRAM
ARM Mali™-450 MP3 GPU（对于Linux和Android，OpenGL ES 2.0 / 1.1）
HDMI 2.0 4K / 60Hz显示
支持H.265 4K / 60FPS和H.264 4K / 30FPS的VPU
千兆以太网
40 + 7针GPIO端口
eMMC5.0 HS400闪存存储插槽/ UHS-1 SDR50 MicroSD插槽
USB 2.0主机x 4，USB 2.0 OTG x 1（电源+数据能力）
的红外线（红外线）接收器
的Ubuntu 16.04和所有的Android 5.1棒棒糖的基于内核3.14 LTS上
尺寸相同ODROID-C1 +

性能：

制造商在Ubuntu 16.04上的不同主板上启动了几种不同的测试，以测量计算能力和比较：Pi 2，ODROID-C1，ODROID-U3和ODROID- XU4。



或以下是Android的Antutu测试结果：



ODROID-C2产量约为33700pts，而ODROID-C1 +用于比较则约为21200pts。请注意，得益于更快的A15内核和具有6个内核的Mali-T628 MP6 GPU芯片，XU4达到了51000pts。但它的价格也贵了将近两倍半。因此，ODROID-C2在性价比方面是无可争议的领导者。

网络连接：

在C2板上有一个千兆位网络适配器。双向速度测试显示约为900Mbps。由于S905中的发送缓冲区加倍，因此订购速度提高了一倍。现在它是一个完整的千兆适配器。



HDMI 2.0和视频播放：

该开发板支持4K / 60Hz HDMI 2.0输出，并为您提供了以3840x2160超高清分辨率工作的机会。但是，不幸的是，没有我们想要的那么快。渲染明显很慢。
因此，Android使用了在FHD 1920x1080 GUI中对接口层和UHD 3840x2160中的视频层进行多路复用的功能，从而使您可以快速，高分辨率地工作。
将来可能会将相同的方法应用于Linux。
预先安装在Android中的最新Kodi Jarvis-RC2媒体处理器使用了相同的方法，现在可以实现相同的结果。

其他：

1.没有SPI总线。 S905芯片根本没有。不幸的是，其他SPI附件无法在C2上使用。
2.没有RTC。同样，S905根本没有它。也许公司将通过此类功能收取额外费用。
3.在S905中修复了不正确的Alpha混合。不再需要使用DDX混合作为解决方法。
4. S905规范的公共版本将于3月或4月发布。
5. Ubuntu 16.04（LTS）ARM64仍处于早期阶段，尚未为ARM64做好充分准备。它缺少某些程序包，例如Chromium。组装这种架构非常困难，而且到目前为止，显然没有人成功。
6. Ubuntu / Linux Mali GPU驱动程序仅采用fbdev的形式。 X11版本将在三月或四月下旬提供。
7. WiKi公司已提供操作系统映像和组装说明，网址为http://odroid.com/dokuwiki/doku.php?id=en:odroid-c2。8
.原型已发送给社区的发烧友，他们通过令人愉悦和有趣的设计证明了其有用性。用于公司产品。
