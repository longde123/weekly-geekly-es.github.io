CardInfo - API zur Bestimmung des Logos, der Farben der Bank und anderer Dinge anhand der Kartennummer



Hier ist eine Demo und Dokumentation: cardinfo.online ist eine API. Sie geben ihm die ersten 6 Ziffern einer Bankkarte, einen Link zum Banklogo, den Firmenfarben, der Marke (Visa, MasterCard usw.) usw. im JSON-Format. CardInfo wird für diejenigen benötigt, die ein Formular zum Akzeptieren von Bankkarten auf ihrer Website erstellen, um die UX (Komfort, Erfahrung, Sicherheitsgefühl) der Benutzer zu verbessern. Mit Cloudpayments.ru in Russland und der GUS können Sie Zahlungen direkt auf Ihrer Website und auf stripe.com im Rest der Welt akzeptieren.

Solche Formulare wurden von allen großen Unternehmen erstellt. Sie können ein lebendiges Beispiel sehen, wenn Sie etwas über Yandex.Cash oder in praktisch jeder mobilen Anwendung einer Bank bezahlen: Wenn Sie eine Kartennummer eingeben, wird das Formular in den Farben dieser Bank neu gestrichen und ein Logo wird daneben angezeigt. Jedes dieser großen Unternehmen hat eine eigene Lösung zur Bestimmung des Logos und der Farben implementiert. Wenn diese Entscheidung von allen großen Unternehmen umgesetzt wurde, hat sie Wert. Wenn große bereits gemacht wurden, kleine aber noch nicht, dann scheint dies ein Aufwärtstrend zu sein, der vernünftig zu satteln ist.

Große Unternehmen haben ihre Formulare jedoch schon lange so aussehen lassen, während kleine Unternehmen dies immer noch nicht tun. Warum? Das Erstellen einer solchen Lösung für Ihr Formular dauert etwa 7 Tage der Arbeit des Programmierers und 7 weitere Arbeiten des Designers (Auspumpen von Logos, Verarbeiten, Neuzeichnen). Große Unternehmen können es sich leisten, für die Erstellung eines solchen Formulars aufzuwenden. Kleine Unternehmen halten den Wert dieses Formulars für nicht ausreichend, um so viel dafür zu bezahlen.

Mit CardInfo können Sie ein vorhandenes Formular für die Annahme von Bankkarten innerhalb eines halben Arbeitstages des Programmierers in das gleiche wie Yandex.Cash umwandeln. Ich gehe davon aus, dass nach der Veröffentlichung von CardInfo die Definition des Banklogos und der Bankfarben für Zahlungsformulare zum Standard wird. Weil solche Formen besser sind. Denn jetzt können sich solche Formen alles leisten.

Übrigens, beachten Sie, dass alle Coolsten Zahlungen direkt auf ihrer Website akzeptieren und sie nicht an externe Dichtungen senden. Zum Beispiel Dodo Pizza dodopizza.ru

Im Allgemeinen ist das Akzeptieren von Karten direkt auf Ihrer Website cool, aber es ist noch cooler, Karten auf Ihrer Website über mit CardInfo erstellte Formulare zu akzeptieren.

Preispolitik


Nun, im Allgemeinen kostenlos. Dank unserer geschäftlichen Fähigkeiten dafür. Die API funktioniert für jeden Benutzer gleich und funktioniert auf der Vorderseite. Selbst wenn Sie einige Einschränkungen für die Verwendung der kostenlosen Version der API einführen, können Sie den bezahlten Schlüssel einer anderen Person einfach kopieren, und schon sind Sie fertig. Also wurden wir nicht weise, sondern frei gelassen. Das Ding ist nützlich, lass es sein.

Es gibt eine Hypothese, dass jemand mit der API nicht vertraut sein wird, was aufgrund seiner kostenlosen API nicht garantiert ist. Wir sind bereit, dass diese Personen die gesamte Datenbank und die Logos verkaufen, damit sie diese Lösung auf ihrer Seite implementieren können. Wen kümmert es, schreibe.

Gerne verkaufen wir das Projekt auch vollständig an jemanden, dessen Marke es in den Augen seiner Nutzer hervorheben kann. Zum Beispiel verkaufen wir uns gerne an CloudPayments.

Wenn jemand in der Community Ideen hat, wie dies sonst monetarisiert oder wie das Projekt entwickelt werden kann, freuen wir uns über Ihre Ideen in den Kommentaren.
