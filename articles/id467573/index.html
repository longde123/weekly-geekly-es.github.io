8 Filter Analytics Teratas yang Bermanfaat

Secara default, Google Analytics (GA) mengumpulkan semua statistik untuk pengunjung situs web, tetapi Anda mungkin tidak perlu menggunakan seluruh array data untuk analisis. Anda dapat menghapus dan memproses informasi menggunakan filter di tingkat presentasi. Kami akan menganalisis yang paling berguna di artikel ini.

Apa yang perlu Anda ketahui tentang filter?


Ada dua jenis filter di Google Analytics:

  • Kebiasaan
  • Tertanam.

Filter bawaan - alat sederhana dan intuitif yang memungkinkan Anda mengaktifkan dan mengecualikan lalu lintas.



Menggunakan lima parameter filter khusus, Anda dapat membuat kondisi pemfilteran lalu lintas yang lebih fleksibel:

  • Kecualikan
  • Aktifkan
  • Huruf kecil / Huruf besar
  • Temukan dan ganti
  • Mahir



Pengaturan dapat diterapkan ke sebagian besar parameter data sumber: sumber lalu lintas, karakteristik browser, nama halaman, dll.

Pelajari lebih lanjut tentang filter dalam Bantuan Google Analytics resmi.

Sebelum mulai bekerja, penting untuk membiarkan tampilan default tidak berubah dan tidak menerapkan filter apa pun padanya. Jadi, Anda akan selalu memiliki akses ke data sumber dan kemampuan untuk mengevaluasi efektivitas filter yang diterapkan.

Pemfilteran lalu lintas harus dikonfigurasikan dalam tampilan terpisah.

Pengaturan filter sangat individual untuk setiap proyek dan untuk setiap tugas. Berdasarkan praktik kami, kami telah menyusun daftar filter universal dan berguna, maka kami akan mempertimbangkannya dengan contoh.

Pengecualian lalu lintas IP


Filter penting, deskripsi yang dapat ditemukan bahkan di Bantuan GA, adalah pengecualian lalu lintas dari alamat IP perusahaan Anda.

Contoh: karyawan toko online sering pergi ke situs kerja tanpa melakukan pembelian. Dalam hal ini, konversi turun, data terdistorsi, kesimpulannya salah. Ini bisa dihindari dengan filter.

Pengaturan filter


Untuk mengecualikan lalu lintas internal yang mengubah data, kami membuat filter bawaan. Itu mengecualikan lalu lintas dari semua alamat IP yang dimulai dengan 209.185.108.1. Yaitu, filter tidak akan mengizinkan data dari alamat IP 209.185.108.100, 209.185.108.101, 209.185.108.102 dan seterusnya sebelum pengiriman.



Kriteria "Mulai dengan" hanyalah salah satu dari empat cara untuk menentukan alamat IP dalam filter.

Mereka juga dapat didefinisikan sebagai:

  • identik dengan ekspresi reguler
  • diakhiri dengan ekspresi reguler
  • mengandung regex

Pemisahan Acara TEST dan PROD


Bayangkan bahwa markup di situs Anda diimplementasikan melalui pengembang dan setelah seminggu yang sulit, pengembang peserta pelatihan secara tidak sengaja mengirimkan peristiwa kesalahan dua kali lipat ke toko online GA Anda ke GA. Sebelum Anda menemukan kesalahan pengembang, akhir pekan akan berlalu. Intinya - data terdistorsi, sulit untuk dianalisis. Distorsi data karena markup yang salah dapat dihindari dengan filter pengguna dan parameter pengguna.

Parameter pengguna h_HIT_TYPE dengan cakupan pada level hit (hit) mengambil nilai dari tipe acara. Secara default, setiap pengidentifikasi acara memiliki tipe [tes]. Setelah memeriksa kebenaran markup menggunakan Tabel Pencarian GTM, jenis acara secara manual diubah menjadi [prod].

Pengaturan filter


Filter khusus memungkinkan data disajikan hanya ketika parameter h_HIT_TYPE cocok dengan pola filter [prod]. Harap perhatikan: bidang "Pola Filter" berfungsi dalam mode ekspresi reguler.



Saat menerapkan filter seperti itu, peristiwa ganda tidak akan pernah masuk ke tampilan, mendistorsi data. Pemisahan ini relevan untuk semua acara kecuali tampilan halaman. Untuk itu, parameter h_HIT_TYPE harus selalu [prod].

Pemisahan lingkungan STAGE dan PROD


Lingkungan pengujian (atau tahap) adalah area pengembangan di situs Anda yang nyaman untuk menguji perubahan. Seringkali, pengujian semacam itu terjadi pada subdomain dengan akses terbatas. Data yang dihasilkan oleh penguji dan pengembang dapat memengaruhi semua statistik secara negatif, sehingga penting untuk memantau transisi situs dengan hati-hati dari lingkungan pengujian ke pertempuran (prod). Untuk melakukan ini, Anda dapat membuat filter berdasarkan jenis media dengan terlebih dahulu membuat parameter pengguna dengan cakupan di level hit h_ENVIRONMENT_TYPE. Cukup untuk mengubah nilainya dari [tahap] ke [prod] saat mentransfer situs dari lingkungan pengujian ke medan perang untuk mencegah data yang salah masuk ke tampilan.

Pengaturan filter


Mirip dengan pengaturan filter sebelumnya, kami mengizinkan data disajikan hanya ketika parameter h_ENVIRONMENT_TYPE cocok dengan templat filter [prod].



Dengan demikian, filter akan mulai meneruskan data ke tampilan hanya setelah mengubah tipe lingkungan dari [tahap] ke [prod].

Pengaturan filter


Di filter lanjutan pengguna, kami menggunakan dua bidang: Bidang A dan Bidang B, membuat Bidang Hasil berdasarkan pada mereka. Ekspresi "Ekstrak A" dan "Ekstrak B" digunakan untuk mencari kecocokan teks. Membawa URL ke satu tampilan

Ini adalah sumber yang sama, tetapi GA mengubah statistik karena kasus. Penting untuk mempertimbangkan ini saat membuat laporan. Untuk menggabungkan halaman dan menghindari kesalahan semacam ini dalam pengumpulan data, Anda dapat menggunakan filter, yang akan membawa URL situs ke satu register.

Bawa nama sumber, saluran, dan kampanye ke satu tampilan


Meskipun filter untuk mengubah huruf besar-kecil URL, sumber, saluran dan kampanye akan tetap dengan huruf asli. Ini disebabkan oleh fakta bahwa tag utm ditransfer ke profil Google Analytics sebagai parameter terpisah dari data mentah. Masing-masing membutuhkan filter terpisah untuk dikonversi menjadi satu register.

Pengaturan filter


Seperti pada pengaturan filter sebelumnya, pilih jenis register dan kolom filter yang diinginkan. Dalam kasus kami, kami menurunkan sumber kampanye ke huruf kecil.



Demikian pula, filter dikonfigurasikan untuk saluran dan kampanye itu sendiri.

Pengecualian www. dari nama host


Masalah pemisahan halaman yang identik dengan URL dapat timbul bukan hanya karena register yang berbeda. Misalnya, Google Analytics memisahkan data dari halaman seperti www.forexample.ru dan forexample.ru.

Masalah ini dapat diatasi dengan menggunakan filter untuk membawa URL ke bentuk yang sama.

Pengaturan filter


Di filter lanjutan pengguna, gunakan Bidang A dan Bidang Output. Dari nama host, kami mengekstrak semua yang ada setelah "www." menggunakan ungkapan reguler www \. (. *). Pilih parameter "Nama Host" di Bidang Output dan tulis nilai yang diterima ke dalamnya menggunakan ekspresi $ A1.



Koreksi Lalu Lintas SEO


Untuk mempromosikan situs di peringkat hasil mesin pencari, penting untuk menganalisis data konversi dari sistem ini. Tetapi Google Analytics tidak selalu menentukan lalu lintas SEO dengan benar dari pencarian Yandex: bagian dari lalu lintas organik masuk ke saluran rujukan. Anda dapat menyesuaikan data menggunakan filter.

Pengaturan filter


Di filter lanjutan pengguna, kami menggunakan dua bidang, Bidang A dan Bidang B, dan menulis ulang Bidang Output berdasarkan bidang tersebut. Jika sumber kampanye cocok dengan ekspresi reguler yandex, dan saluran kampanye cocok dengan ekspresi reguler rujukan, filter menimpa saluran pengguna sebagai organik.



Lalu lintas SEO Mail.ru juga memengaruhi masalah serupa, filter untuk mesin pencari ini dikonfigurasi dengan cara yang sama.

Filter di Google Analytics memiliki fungsi fleksibel. Dalam artikel ini, kami memeriksa pengaturan filter dasar, memungkinkan Anda untuk membawa data ke formulir yang mudah untuk analisis. Penting untuk diingat bahwa urutan filter juga memainkan peran penting dalam menyaring lalu lintas.
