Web- und Azure-Tool-Updates in Visual Studio 2019

Hoffentlich haben Sie inzwischen gesehen, dass Visual Studio 2019 jetzt allgemein verfügbar ist . Wie zu erwarten, haben wir Verbesserungen für die Web- und Azure-Entwicklung hinzugefügt. Als Ausgangspunkt bietet Visual Studio 2019 eine neue Erfahrung für den Einstieg in Ihren Code. Wir haben die Erfahrung für die Erstellung von ASP.NET- und ASP.NET Core-Projekten entsprechend aktualisiert:



Wenn Sie Ihre Anwendung in Azure veröffentlichen, können Sie den Azure App Service jetzt so konfigurieren, dass Azure Storage- und Azure SQL-Datenbankinstanzen direkt auf der Seite mit der Zusammenfassung des Veröffentlichungsprofils verwendet werden, ohne Visual Studio zu verlassen. Dies bedeutet, dass Sie für jede vorhandene Webanwendung, die im App Service ausgeführt wird, SQL und Speicher hinzufügen können. Dies ist nicht mehr nur auf die Erstellungszeit beschränkt.



Durch Klicken auf die Schaltfläche "Hinzufügen" können Sie zwischen Azure-Speicher und Azure SQL-Datenbank auswählen (weitere Azure-Dienste werden in Zukunft unterstützt):



Anschließend können Sie wählen, ob Sie eine vorhandene Instanz von Azure Storage verwenden möchten, die Sie in der Vergangenheit bereitgestellt haben, oder ob Sie sofort eine neue Instanz bereitstellen möchten:



Wenn Sie Ihren Azure App Service wie oben gezeigt über das Veröffentlichungsprofil konfigurieren, aktualisiert Visual Studio die Anwendungseinstellungen des Azure App Service so, dass sie die von Ihnen konfigurierten Verbindungszeichenfolgen enthalten (z. B. in diesem Fall azgist). Außerdem werden versteckte Tags auf die Instanzen in Azure angewendet, um zu ermitteln, wie sie für die Zusammenarbeit konfiguriert sind, damit diese Informationen nicht verloren gehen und später von anderen Instanzen von Visual Studio erneut erkannt werden können.
