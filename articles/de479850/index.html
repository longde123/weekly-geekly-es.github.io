Installieren Sie Exchange 2019 unter Windows Server Core 2019

Microsoft Exchange ist ein großer Prozessor, der das Empfangen und Verarbeiten von Briefen sowie eine Webschnittstelle für Ihren Mailserver und den Zugriff auf Unternehmenskalender und -aufgaben umfasst. Exchange ist in Active Directory integriert. Stellen Sie sich also vor, dass es bereits bereitgestellt wurde.

Nun, Windows Server 2019 Core ist eine Version von Windows Server ohne grafische Oberfläche.

In dieser Windows-Version gibt es kein traditionelles Windows, nichts zum Anklicken, kein Startmenü. Nur ein schwarzes Fenster und eine schwarze Befehlszeile. Gleichzeitig gibt es aber auch einen kleineren Angriffsbereich und ein höheres Einstiegsniveau, da wir nicht möchten, dass sich jemand mit kritischen Systemen befasst, oder?

Dieses Handbuch eignet sich auch für Server mit einer grafischen Benutzeroberfläche.


1. Stellen Sie eine Verbindung zum Server her


Öffne Powershell und gib den Befehl ein: 

Enter-PSSession 172.18.105.6 -Credential Administrator

Optional: Aktivieren Sie RDP. Dies vereinfacht die Installation, aber überhaupt nicht. 

 cscript C:\Windows\System32\Scregedit.wsf /ar 0

In dem Bild von Ultravds war RDP bereits enthalten.

2. Verbinden Sie den Server mit AD


Dies kann über das Windows Admin Center oder über Sconfig in RDP erfolgen.

2.1 Geben Sie DNS-Server oder Domänencontroller an



In Windows Admin Center stellen wir eine Verbindung zum Server her, gehen zum Netzwerkabschnitt und geben die IP-Adressen von Domänencontrollern oder DNS-Domänenservern an.


Geben Sie über RDP "Sconfig" in die Befehlszeile ein und rufen Sie das blaue Server-Konfigurationsfenster auf. Dort wählen wir Punkt 8) Netzwerkeinstellungen aus und geben den DNS-Domänenserver an.

2.2 Wir verbinden den Server mit der Domain



Klicken Sie in WAC auf "Computer-ID ändern". Daraufhin wird ein vertrautes Fenster zum Auswählen einer Arbeitsgruppe oder Domäne angezeigt. Alles ist wie gewohnt, Domain auswählen und beitreten.


Bei Verwendung von Sconfig müssen Sie zuerst Element 1 auswählen, auswählen, ob wir einer Arbeitsgruppe oder Domäne beitreten möchten, und die Domäne angeben, wenn wir der Domäne beitreten möchten. Und erst nach Abschluss des Vorgangs können wir den Servernamen ändern, aber auch dafür müssen Sie das Passwort erneut eingeben.

Powershell macht das noch einfacher: 

 Add-Computer -DomainName test.domain -NewName exchange -DomainCredential Administrator

3. Installieren Sie




Wenn Sie RDP verwenden, müssen Sie die erforderlichen Komponenten installieren, bevor Sie Exchange selbst installieren. 

 Install-WindowsFeature Server-Media-Foundation, RSAT-ADDS

Als Nächstes müssen wir das Datenträgerabbild mit dem Exchange-Installationsprogramm herunterladen. 

 Invoke-WebRequest -UseBasicParsing -Uri 'https://website.com/ ExchangeServer2019-x64.iso -OutFile C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

ISO einbinden: 

 Mount-DiskImage C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

Wenn Sie dies alles über die Befehlszeile tun, müssen Sie nur die heruntergeladene Diskette mounten und den Befehl eingeben: 

 D:\Setup.exe /m:install /roles:m /IAcceptExchangeServerLicenseTerms /InstallWindowsComponents

Fazit


Wie Sie sehen, ist das Installieren von Exchange auf Windows Server Core sowie das Eingeben der Domäne kein schmerzhafter Prozess. Angesichts der Sicherheitsgewinne hat sich dies gelohnt.

Es hat mich besonders gefreut, dass die Eingabe des Servers in AD über Powershell einfacher ist als über die Benutzeroberfläche oder das Windows Admin Center.

Schade, dass die Exchange-Installationsoption nur für Exchange 2019 hinzugefügt wurde, es war höchste Zeit.
