Substitusi impor dalam praktek. Bagian 3. Sistem Operasi



Kami melanjutkan serangkaian artikel tentang substitusi impor. Dalam publikasi sebelumnya, opsi untuk mengganti sistem yang digunakan dengan yang "domestik" , dan khususnya hypervisor "produksi dalam negeri", dipertimbangkan.

Sekarang giliran untuk berbicara tentang sistem operasi "domestik" yang termasuk dalam daftar Kementerian Komunikasi saat ini.

0. Titik awal


Saya mendapati diri saya berpikir bahwa saya tidak tahu bagaimana membandingkan distribusi LINUX. Saya naik ke Wikipedia , itu tidak menjadi lebih jelas. Kriteria apa yang harus dipertimbangkan? Apa yang harus diambil sebagai titik referensi? Bagi saya - kriteria yang paling indikatif untuk OS server adalah stabilitas. Namun dalam kerangka uji coba, kata "stabilitas" terdengar paling tidak aneh. Ya, seminggu saya mempelajari sistem yang diperluas ... Tapi seminggu bukanlah indikator di dunia di mana beberapa tahun uptime bahkan bukan nilai rata-rata. Tes stres? Bagaimana cara memuat sistem pada dudukan? Selain itu, itu adalah OS yang perlu dimuat, bukan aplikasi, dan itu harus dimuat agar crash ... Dan jika tidak ada yang crash - bagaimana membandingkan? ..

Tetapi kemudian saya sampai pada kesimpulan bahwa stabilitas dapat ditarik secara kondisional dari ayah distribusi OS "domestik". Untuk Astra, misalnya, seperti Debian, untuk ROSA - Mandriva, untuk Calculate - Gentoo, dll. Dan hanya untuk Alt, sudah begitu lama berpisah dari Mandrake, sehingga dapat dianggap sebagai distribusi independen (dalam kaitannya dengan semua "domestik" OS lainnya). Tetapi harap diingat bahwa ini semua sangat sewenang-wenang, karena tidak diketahui apa kode sumber diisi oleh pengembang, dan apa yang diubah sebagai bagian dari peningkatan keamanan OS.

Kriteria yang lebih dapat dilacak adalah komposisi paket pengiriman OS dan paket dalam repositori. Tetapi dalam hal ini kita harus melanjutkan dari persyaratan kebutuhan. Saya memiliki tugas saya sendiri yang perlu diselesaikan, Anda memiliki tugas Anda sendiri, dan pendekatan untuk memilih perangkat lunak harus seperti ini: "Tugasnya adalah memilih perangkat lunak", dan bukan sebaliknya, seperti yang sering terjadi dalam ...

Jadi, layanan yang perlu digunakan selama "pindah":

  • Server surat
  • Zabbix
  • DBMS
  • Server web
  • Server Jabber
  • Cadangkan
  • Paket kantor
  • Pelanggan SUFD dan Bank
  • Klien surat
  • Browser

AD, DNS, DHCP, CertService tetap di server Windows (penjelasan tentang ini diberikan pada artikel sebelumnya ). Tetapi dalam keadilan, saya perhatikan bahwa Layanan Direktori dapat dinaikkan pada SAMBA atau FreeIPA yang sama, dan beberapa distribusi menyatakan layanan direktori "mereka" (Direktori Astra Linux, ALT, Direktori ROSA, Direktori Lotos). DNS dan DHCP juga naik pada distribusi Linux apa pun, dan tidak semua orang membutuhkan server sertifikasi.

Server surat . Saya suka Zimbra . Saya bekerja dengannya, dia nyaman, tahu cara mengumpulkan data dari Exchange, tahu banyak hal. Tapi itu hanya bisa digunakan di ROSA Linux. Anda dapat meletakkannya di OS lainnya, tetapi itu tidak akan dianggap sah. Di sisi lain, masing-masing OS "domestik" memiliki server mail sendiri, saya bertemu Zimbra.

Zabbix Dia tidak memiliki pesaing. Dalam rangka substitusi impor - terlebih lagi. Zabbix adalah bagian dari Alt Linux, RED OS, Astra dan ROSA. Dalam Calculate, itu ditandai "tidak stabil".

DBMS PostgreSQL mendukung semua OS "domestik".

Server web . Apache ada di semua OS server.

Server Jabber . Secara umum, ini direncanakan untuk memperkenalkan Bitrix24 , tapi saya sudah terbiasa dengan fakta bahwa semuanya telah berlangsung sangat lama, dan karena itu saya mempertimbangkan opsi dengan obrolan korporat berdasarkan jabber. Saya sudah terbiasa dengan OpenFire . Ini adalah bagian dari Calculate , RED OS dan ROSA. Ada juga ejabberd sebagai bagian dari ROSA, Alt, RED OS dan Astra.

Cadangkan . Ada Bacula , bagian dari Astra, Rosa, Alt, Calculate, AlterOS.

Suite kantor . Suite office gratis Libre Office ada di semua OS "domestik" klien (dan seringkali server).

Klien email . Thunderbird ada di semua "domestik" OS klien (dan sering server).

Browser Setidaknya Mozilla Firefox tersedia di semua sistem operasi. Yandex.Browser juga dapat diinstal di semua OS.

Dengan SUFD dan Bank, pelanggan menjadi sedikit lebih rumit. Secara resmi, semua ini dapat bekerja di hampir semua OS "domestik". Dalam praktiknya, ini cukup sulit untuk diverifikasi, karena Anda perlu membawa pengguna, membawa ke mesin yang diuji dan berkata "coba". Itu penuh. Jadi untuk pertama kalinya saya akan meninggalkan skema lama - klien dengan Windows dan token yang dilemparkan ke dalamnya oleh mesin virtual untuk setiap Bank. Untungnya, Linux tahu cara meneruskan token dengan pasti. Dan itu akan terlihat.

Selanjutnya, kami melanjutkan ke pemilihan Sistem Operasi yang memenuhi kebutuhan kami. Tetapi untuk objektivitas, saya mencoba untuk mencakup sebanyak mungkin sistem operasi dari registri Kementerian Perdagangan dan Industri.

Daftar dalam daftar Kementerian Komunikasi cukup luas, tetapi setelah pertemuan dewan ahli tentang perangkat lunak Rusia di bawah Kementerian Komunikasi Rusia, diputuskan untuk memeriksa kembali Ulyanovsk.BSD , RED OS dan OS .

Sistem yang saya anggap perlu untuk "menyentuh" :

  • Astra linux
  • Alt
  • Hitung Linux
  • Rosa linux
  • OS MERAH
  • AlterOS
  • Wtware

Sistem yang menimbulkan lebih banyak pertanyaan daripada memberikan jawaban (untuk saya):

  • Ulyanovsk.BSD

  • Alpha OS
  • OS LOTOS
  • Halo OS

Pertama, saya ingin agar setiap OS mengunggah tangkapan layar, deskripsi, fitur ... Tapi semua ini sudah terjadi. Banyak screenshot di situs pengembang, deskripsi juga ada di ratusan artikel tentang topik ini di RuNet, deskripsi kemungkinan juga dapat ditemukan di situs resmi ... Tapi jika Anda tidak memberikan "latihan", maka semuanya akan menjadi teori lagi, seperti yang terjadi pada teori pertama dua artikel. Video? ada juga ... Piring ringkasan, tentu saja, tapi itu bukan praktik ...

Jadi pada akhirnya, saya memutuskan untuk hanya menulis pendapat dan pemikiran pribadi saya tentang setiap distribusi selama pengujian. Yah, dan sedikit lebih bermanfaat, dan tidak terlalu, informasi.

1.1. Astra linux


Situs web resmi

Versi saat ini:
Edisi Umum Astra Linux - 2.12
Edisi Khusus Astra Linux - 1.6

Distribusi induk adalah Debian.

Komposisi paket perangkat lunak dapat ditemukan di sini . (Tombol "Detail" yang tidak mencolok di bawah gambar perangkat lunak bermerek di bagian "KOMPOSISI SISTEM OPERASI".)

Butuh waktu yang sangat lama untuk menginstal. OS yang digunakan untuk mesin virtual hampir satu setengah jam ... Yaitu, jika ada kebutuhan untuk menyebarkannya ke 1500 PC domain, itu akan memakan banyak waktu.

Ini Debian. Ini adalah Debian yang sudah usang. Paket-paket Astra bahkan lebih tua dari induknya, baik dalam perakitan dan dalam repositori. Jika ada kebutuhan mendesak, adalah mungkin untuk menghubungkan repositori Debian, namun, ini secara otomatis membatalkan substitusi impor (dalam hal ini, Anda dapat memperbarui sistem dari repositori untuk pembaruan && penyimpanan repositori Deb, dan itu akan terus bekerja ... meskipun saya tidak yakin apa yang terjadi pada akhirnya, saya menembaknya karena ampun untuk berjaga-jaga ..).

Desktop "Terbang". Untuk server, GUI pada prinsipnya tidak diperlukan sama sekali, meskipun menyederhanakan beberapa tindakan. Tetapi untuk pengguna OS tanpa itu, tidak ada tempat. Secara umum, ia meninggalkan kesan yang baik, sementara sedekat mungkin dengan Windows, yang akan menyederhanakan transisi ke OS ini untuk pengguna. Ada banyak hal â€œ-Flyâ€ dalam sistem, dan semua ini dikembangkan oleh JSC â€œNPO RusBITechâ€. Tombol pintas untuk sebagian besar berfungsi sama seperti pada Windows. Win + E membuka "explorer", Win membuka menu taskbar, dll. Secara umum, kemungkinan besar, para pengembang mencoba membawa penampilan ke Windows sedekat mungkin.

OS bergabung dengan AD, memungkinkan Anda untuk mengkonfigurasi otorisasi, dll. Selama pengujian, ini terbukti stabil (sejauh yang dapat dinilai seseorang selama periode operasi uji), tidak berubah-ubah dan agak sederhana serta menyenangkan OS Debian.

Jika diinginkan, Anda dapat menginstal paket bukan dari repositori. Saya mencoba menggunakan OpenFire sebagai contoh. Anda mengunduh paket untuk Debian, dan semuanya terinstal dengan tenang.

Untuk mengatasi masalah saya, ini dapat digunakan sebagai platform untuk menyebarkan Zabbix, jabber-server, PosgreSQL, Apache. Sebagai OS pengguna, ia memenuhi semua persyaratan (Antarmuka yang bagus, LibreOffice, Thunderbird, Firefox). SUFD dan Klien tidak menguji bank.

Edisi Khusus berbeda dari Edisi Umum di mana Khusus itu cocok untuk bekerja dengan rahasia negara dan dokumen rahasia lainnya, itu disertifikasi untuk ini. Umum - OS "biasa", dapat digunakan di mana sertifikasi tidak diperlukan, dan Anda tidak perlu bekerja dengan rahasia itu.

Harga untuk 1 lisensi Edisi Khusus : 14 900 rubel.
Harga untuk 1 lisensi Edisi Umum : 3 900 rubel.

1.2. Alt


Situs web resmi

Distribusi induk - Alt Linux (MandrakeLinux diambil sebagai dasar pada tahun 2000)

Hal pertama yang sangat terkejut adalah installer. Sebelum menulis artikel ini, saya tidak berurusan dengan sistem ini, dan installer membuat saya sangat senang.

Fungsi utama

Repositori Sisyphus

Saya sangat menyukai OS server, di atasnya Anda dapat menggunakan semua yang saya butuhkan, kecuali Zimbra dalam kerangka substitusi impor, tentu saja. Termasuk Anda dapat menggunakan pengendali domain (ada implementasinya sendiri berdasarkan OpenLDAP dan MIT Kerberos).

Di server, desktop KDE. Mengenai yang asli, praktis tidak ada perubahan di dalamnya. Masalahnya adalah bahwa pada pengguna OS KDE belum mengalami perubahan, yang menunjukkan bahwa pengguna akan melolong dengan tidak terbiasa.

Keuntungan utama dari sistem ini adalah fakta bahwa ia telah dikembangkan di Rusia selama hampir 20 tahun. Dia memiliki seperangkat perangkat lunak yang luas dalam repositori dan akumulasi basis pengetahuan.

Saya ingin mencatat bahwa "Basalt SPO" adalah orang-orang hebat. Mereka melakukan hal mereka sendiri bahkan ketika itu belum menjadi arus utama, dan terus melakukannya. Dan mereka melakukannya dengan baik.

Harga untuk 1 lisensi server : 10.000 rubel.
OS Klien : 4 000 gosok.

1.3. Hitung Linux


Situs web resmi

Distribusi induk - Gentoo

Paket dapat dilihat di sini .

Ada edisi dengan implementasi GUI yang berbeda, ada banyak pilihan untuk kenyamanan pengguna. Edisi KDE, misalnya, sangat dekat dengan Windows.

Karena emerge digunakan untuk menginstal paket, pengaturan workstation membutuhkan waktu yang agak lama jika dilakukan secara manual. Mungkin di sini akan sangat tidak pada tempatnya, tetapi perlu mempertimbangkan semua opsi.

Sistem dapat secara otomatis memperbarui, dapat bekerja dalam domain AD.

Nilai tambah terbesar dari OS, menurut saya, adalah Calculate Console, suatu hal yang sangat nyaman dan berguna.

Hitung tidak memiliki dukungan.

Secara umum, sistem ini layak diperhatikan, dapat meningkatkan hampir semua layanan yang saya butuhkan: Zabbix (dipertanyakan, saya perlu mengujinya dalam lingkungan produk), server jabber, PosgreSQL, Apache. Sebagai OS pengguna, ia memenuhi semua persyaratan (Antarmuka yang bagus, LibreOffice, Thunderbird, Firefox). SUFD dan Klien tidak menguji bank.

1.6. AlterOS


Situs web resmi

Versi saat ini:

Distribusi induk - openSUSE

Setiap saat instalasi, serta penggunaan OS, saya memiliki perasaan yang kuat bahwa saya bekerja dengan CentOS, dan bukan dengan openSUSE.

Otentikasi pengguna memakan waktu sekitar 20 detik, yang menyebabkan setidaknya kebingungan.

Pada mesin virtual di lingkungan Hyper-V, kursor mouse tidak terlihat ... Ini berfungsi, menyorot tombol, mengkliknya, tapi saya tidak melihatnya. Reboot tidak membantu, saya tidak melihat kursor.

Desktop hot-key KDE cukup praktis. Desainnya bagus, dekat dengan Windows, yang bagus untuk pengguna akhir. Secara umum, GUI senang, jika bukan karena bug (atau fitur) dengan kursor tidak terlihat.

Itu bisa bekerja dengan AD, bisa login melaluinya. Ia juga tahu bagaimana menjadi server otorisasi.

Saya tidak memiliki masalah dengan AlterOS, kecuali kursornya, sehingga sistem ini beroperasi penuh.

Untuk mengatasi masalah saya, ini dapat digunakan sebagai platform untuk menggunakan PosgreSQL, Apache. Sebagai OS pengguna, ia memenuhi semua persyaratan (Antarmuka yang bagus, LibreOffice, Thunderbird, Firefox). SUFD dan Klien tidak menguji bank.

Roti yang berguna dalam bentuk gambar dan dokumentasi.

Harga untuk 1 lisensi : 11 500 rubel.

1.7. Wtware


Situs web resmi

WTware tidak dapat disebut OS dalam pengertian istilah yang biasa. Sistem ini adalah add-on untuk OS server, mengubahnya menjadi RDP untuk menghubungkan klien tipis, ini adalah paket yang memungkinkan klien tipis untuk boot melalui jaringan. Ini mendukung bekerja dengan Windows Server dari 2000 hingga 2016, Hyper-V VDI, manajemen Windows jarak jauh, xrdp di Linux, Server Terminal Mac.

Ini berisi server TFTP yang dirancang untuk memungkinkan klien mengunduh melalui jaringan, server HTTP yang bekerja dengan TFTP, dan server DHCP untuk menerbitkan alamat IP ke klien. Itu juga dapat mem-boot mesin klien dari hdd, CD-ROM atau flash-drive.
Perangkat lunak ini didokumentasikan dengan baik.

Biaya setiap lisensi:
1 - 9 lisensi: 1000 rubel
10 - 19 lisensi: 600 rubel
20 - 49 lisensi: 500 rubel
50 - 99 lisensi: 400 rubel
100 atau lebih lisensi: 350 rubel

Situs web resmi

Versi saat ini:
Ulyanovsk. BSD 12.0 RELEASE P3

Seperti yang ditulis di atas, Ulyanovsk.BSD memiliki setiap kesempatan untuk dihapus dari registri Kementerian Komunikasi, karena didasarkan pada FreeBSD, praktis tidak berbeda dari aslinya, dan menggunakan repositori, yang, sebagai bagian dari substitusi impor, menyebabkan kebingungan yang mengerikan mengenai apa yang dapat dipertimbangkan perangkat lunak yang sah.

Selain itu, distribusi yang diunduh tidak dimulai pada Hyper-V di lingkungan Windows 10 atau 2012R2. Hypervisor sama sekali tidak melihat harus mulai dari mana. Saya memutuskan bahwa saya tidak membutuhkannya saat ini ...

Saya belum melihat apa pun yang menulis, ada banyak ulasan tentang FreeBSD, jadi kami akan melanjutkan, kami tidak menunda.

OS MERAH - ya daripada tidak. Selain server surat dan sistem cadangan. Sebagai OS pengguna - mungkin tidak, hanya karena suite kantor yang sudah ketinggalan zaman. Tapi harga distribusi lebih tinggi dari harga pesaing ... Tapi ini adalah TOPI MERAH ... tapi ... tapi ...

AlterOS - baik Zabbix maupun server jabber tidak dapat dinaikkan. Sisanya adalah sistem yang cukup baik. Sebagai OS klien, masalahnya adalah suite kantor yang sudah usang, jika tidak untuk ini, maka itu adalah solusi yang cukup bagus.

WTware untuk klien tipis tidak masalah. Tapi ini bukan OS, jadi kemungkinan besar Anda tidak akan bisa menghitungnya dalam "hal-hal". Yaitu, dalam kasus saya, ketika ada 1.500 PC klien, tidak mungkin melaporkan substitusi impor dengan fakta bahwa kami mentransfer semua karyawan 1.5k ke klien tipis dan kami memiliki 300 jendela server lainnya, karena 1.5k ini bukan OS ...

Ulyanovsk.BSD - tidak. Untuk itu menimbulkan kekhawatiran karena kemungkinan akan dihapus dari registri Kementerian Komunikasi. Meskipun FreeBSD adalah hal yang baik dan terbukti, tetapi produk ini ...

OS- sampai masalah kelayakan perusahaan pengembangan dan dukungan diselesaikan, itu pasti tidak. Jika Anda memutuskan secara positif ... kemungkinan besar tidak baik ... Meskipun saya sudah terbiasa dengan CentOS, tetapi masih belum.

QP OS - jelas dan pasti tidak. Dengan spesialis dan sikap seperti itu ... Ini adalah pendapat subjektif saya, tetapi itu tidak akan berubah.

Alpha OS . Apa yang ditulis tentang dia di Internet dan ditampilkan dalam video terdengar menggoda. Masih akan ada sistem ini dalam kehidupan nyata ...

OS LOTOS . Membeli kucing di ladang? Tidak terima kasih Jika Anda tidak tertarik dalam pengujian, maka saya tidak tertarik untuk membeli perangkat lunak Anda untuk pengujian. Untuk alasan yang jelas,

Halo OS juga tidak ada di sana, karena saya tidak tahu apa itu dan apa yang dimakannya.

3. Ringkasan


Untuk menggunakan Zimbra Collaboration Suite OSE, saya memerlukan setidaknya 1 salinan ROSA Enterprise Linux Server , dan lebih baik 2, untuk mengonfigurasi proxy.

Untuk meningkatkan semua layanan lain, masuk akal untuk menggunakan Astra Common Edition atau EDIT OS , karena di masa depan harga sistem ini akan menjadi yang paling menguntungkan karena dukungan yang murah. Namun secara pribadi saya lebih terkesan dengan Astra.

Beberapa layanan non-kritis dapat digunakan berdasarkan Calculate Linux , karena gratis. Tetapi karena kurangnya dukungan, ini harus menjadi layanan seperti itu, downtime yang tidak penting untuk Enterprise, karena administrator sistem secara langsung bertanggung jawab atas kinerja mereka.

Custom OS - Saya lebih suka Astra CE yang sama . Dia memiliki office suite terbaru, GUI yang mudah digunakan, sistem dapat melakukan apa saja yang diperlukan. Ya, ini lebih murah daripada pesaing.

Jika ada kebutuhan untuk menyebarkan server direktori dan layanan infrastruktur lainnya, masuk akal untuk melihat OS dari keluarga yang sama yang akan digunakan untuk pengguna, setidaknya dari sudut pandang kompatibilitas. Dalam kasus saya, jika saya masih perlu melakukan ini, kemungkinan besar adalah Astra CE .
