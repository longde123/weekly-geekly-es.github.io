GitHub verrouille le référentiel du mot "retard" dans la description

Il y a quelques jours, une validation a été effectuée dans le référentiel WebMConverter , corrigeant le mot retard (frein, abruti) dans la description du projet. Auparavant, la description du projet était WebM for Retards, qui a été remplacée par WebM for Gits. Le propriétaire du référentiel a montré le texte de la lettre envoyée par le github:


Pour ceux qui ne parlent pas bien l'anglais, bref -
, ToS ( ) . , «retard» «retarded».
24 . , : ( )

Il est à noter que le référentiel a été bloqué pendant 3 jours pendant que le propriétaire tentait de contacter le support du github.
Une discussion animée s'est déroulée dans les commentaires sur l'engagement et dans le sujet correspondant sur le reddit concernant la légitimité des actions du github, en particulier, jusqu'où cette lutte pour les droits des minorités pourrait aller et si elle nuirait au développement de projets open source utilisant github. Beaucoup ont noté que le mot git lui-même (une personne stupide, incompétente et ennuyeuse) est abusif et a accusé le github de deux poids deux mesures. Il a été suggéré que sous le couvert de la lutte contre la discrimination, des projets répréhensibles pouvaient être bloqués.

Il me semble que cela n'a pas été beaucoup discuté dans RuNet (je peux me tromper). C'était intéressant de savoir ce qu'en pensent les utilisateurs du gigtime, en particulier ceux qui utilisent le github.
