Comment créer un beau schéma de base de données dans l'éditeur de graphiques yEd

Je veux vous montrer comment, à l'aide de l'application yEd désormais populaire, conçue pour construire divers diagrammes et schémas, vous pouvez créer un schéma de base de données complet (avec toutes les notations).

Donc, pour commencer, nous avons un conteneur standard avec des éléments pour construire un ERD (diagramme de relation d'entité). Leurs désignations sont représentées sur la figure.



Une petite digression, si vous avez besoin d'insérer juste du texte sur le diagramme, c'est-à-dire sans aucun bloc, les étapes simples suivantes d'un commentaire de jaiprakash résolvent rapidement ce problème. "Sélectionnez un lien -> ajouter une étiquette -> saisir du texte -> convertir en nœud".

Nous posons donc le problème conditionnel de la construction du schéma suivant.



Créez un cadre d'éléments de base.



Afin d'ajouter la notation des clés primaires et étrangères, vous devez un peu inventer. En sélectionnant l'élément souhaité, ajoutez-lui une nouvelle étiquette, que nous appellerons KEY. Parce que Il sera situé sous l'unité principale, puis sélectionnez-le à l'aide du Ctr serré.





Réglez son emplacement (placement) en sélectionnant Interne: en haut à gauche (à l'intérieur, en haut à gauche), la couleur comme vous le souhaitez, la distance (distance par rapport aux bords) à zéro, les encarts (insertion en pixels) à 20 2 0 0 afin qu'elle soit en face du champ d'attribut avec une petite distance, la famille de polices (police) définit le symbole d'interface utilisateur Segoe, car Il prend en charge avec précision les caractères nécessaires.



Les symboles de copie se trouvent, par exemple, dans la table des symboles Windows.





Faites de même avec les flèches.



Afin de combiner plusieurs tables en un bloc, vous pouvez utiliser le regroupement.
