Hintergrund: Was erwartet Sie von Fedora Silverblue?

Wir analysieren die Merkmale eines unveränderlichen Betriebssystems.


/ Foto Clem Onojeghuo Unsplash

Wie kam es zu Silverblue?


Fedora Silverblue ist ein unveränderliches Desktop-Betriebssystem. Darin werden alle Anwendungen in isolierten Containern gestartet und Updates atomar installiert.

Zuvor hieß das Projekt Fedora Atomic Workstation . Es wurde später in Silverblue umbenannt. Nach Angaben der Entwickler wurden mehr als 150 Namensvarianten berücksichtigt. Silverblue wurde einfach ausgewählt, weil es eine so kostenlose Domain und Konten in sozialen Netzwerken gab.

Das aktualisierte System ersetzte Fedora Workstation als Priority Build für Desktops in Fedora 30. Die Autoren sagen, dass Silverblue Fedora Workstation in Zukunft vollständig ersetzen könnte .

Ein Bewohner von Hacker News schlug vor, dass das Silverblue-Konzept die Entwicklung des Stateless Linux- Projekts sei. Er wurde vor ungefähr zehn Jahren zu Fedora befördert. Stateless Linux sollte die Verwaltung von Thin- und Thick-Clients vereinfachen. Auch darin wurden alle Systemkonfigurationsdateien im schreibgeschützten Modus geöffnet.

Was gibt "Unveränderlichkeit"


Der Begriff "unveränderliches Betriebssystem" bedeutet, dass das Stammverzeichnis und das Benutzerverzeichnis im schreibgeschützten Modus bereitgestellt werden. Alle veränderlichen Daten befinden sich im Verzeichnis / var. Eine ähnliche Methode wird von ChromeOS- und macOS Catalina- Entwicklern verwendet. Dieser Ansatz erhöht die Sicherheit des Betriebssystems und verhindert, dass Systemdateien gelöscht werden (z. B. aus Versehen).

Einer der Bewohner von Hacker News sagte in einem thematischen Thread , dass er versehentlich eine Reihe von Systemdateien gelöscht und das Thema von Ubuntu Yaru geändert habe. Gleichzeitig hatte er aufgrund eines Regex-Fehlers keine Backups. Ihm zufolge würde ein unveränderliches Betriebssystem helfen, Probleme zu vermeiden.

Die Installation von Updates wird ebenfalls vereinfacht. Starten Sie das System einfach von einem neuen Image aus neu. Darüber hinaus können Sie schnell zwischen mehreren Filialen wechseln (Fedora-Versionen). Zum Beispiel zwischen der aktuell entwickelten Version von Fedora Rawhide und dem Update-Test- Repository mit bevorstehenden Updates.

Was sind die Unterschiede zum klassischen Fedora?


Zur Installation der Basisumgebung (/ und / usr) wird die OSTree-Technologie verwendet. Wir können sagen, dass dies ein System zur "Versionierung" von RPM- Paketen ist. RPM-Pakete werden mit rpm-ostree in das OSTree-Repository übersetzt. Durch die Installation des Pakets wird ein Wiederherstellungspunkt gebildet, der im Fehlerfall zurückgesetzt werden kann.

Mit OSTree können Sie auch Anwendungen aus dnf / yum-Repositorys und Repositorys installieren, die von Fedora nicht unterstützt werden. Verwenden Sie dazu anstelle des Befehls dnf install rpm-ostree install. Das System erstellt ein neues Basis-Image des Betriebssystems und ersetzt es durch das installierte.

Flatpack wird als Mechanismus zum Aktualisieren von Anwendungen verwendet. Er startet sie in Containern. Ein Flatpack-Paket enthält nur anwendungsspezifische Abhängigkeiten. Alle Kernbibliotheken (wie die GNOME- und KDE-Bibliotheken) bleiben steckbare Laufzeitumgebungen. Mit diesem Ansatz können Sie die Größe von Paketen reduzieren, um doppelte Komponenten zu entfernen.


/ Foto Jonathan Larson Unsplash

Um Anwendungen zu installieren, die nicht in Flatpack gepackt sind, können Sie die Toolbox verwenden . Mit dem klassischen Fedora-Installationsprogramm können Sie einen Container erstellen.

Ähnliche Lösungen


Es gibt andere Distributionen, deren Aufgaben Silverblue ähneln. Ein Beispiel wäre das MicroOS von openOSUSE . Dies ist keine eigenständige Distribution, sondern Teil der openSUSE Kubic-Plattform für die Bereitstellung von CaaS (Container as a Service).

Das System arbeitet mit Docker-Containern. Ihre Bilder werden in Form von RPM-Paketen verteilt. Dies erleichtert die Installation von Befehlszeilenanwendungen, die nicht im Flatpack-Format verfügbar sind. Das Hostsystem zum Starten von Containern basiert auf dem offiziellen openSUSE Tumbleweed- Repository.

MicroOS wurde für die Bereitstellung in großen Umgebungen (z. B. in Rechenzentren) entwickelt, kann jedoch auch auf einzelnen Computern verwendet werden.

Ein Beispiel für eine andere ähnliche Entwicklung ist NixOS . Dies ist eine Linux-Distribution, die auf dem Nix-Paketmanager basiert. Das Hauptmerkmal ist eine deklarative Beschreibung der Konfigurationen. Der Administrator muss das System nicht installieren und manuell konfigurieren. Der Status wird in einer speziellen Datei registriert: Dort werden alle Pakete und Authentifizierungseinstellungen angezeigt. Außerdem bringt der Paketmanager das Betriebssystem automatisch in den angegebenen Zustand.

Dieses System wird von Cloud-Anbietern, Universitäten und IT-Unternehmen aktiv genutzt .

In jedem Fall hat Silverblue die Chance, seine Marktnische zu besetzen. Wird es gelingen - in der Zukunft gesehen zu werden.
