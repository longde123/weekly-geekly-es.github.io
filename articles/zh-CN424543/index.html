Java 11 / JDK 11：常规可用性

同志们，哈伯的Java战士！ 这一天，我们尽可能地靠近了。


今天，JDK 11的发行版本（Java 11的参考实现）以及以6个月的快速发行周期发行的Java的第一个LTS版本已经可用 。 现代最大，最重要的发行版，将带动整个Java社区转向最新和最先进的技术。




自第28版发布以来已经过去了四个星期，在此期间，没有一个P1级错误出现。 这意味着Java 11可以被认为已经过时并且可以完全用于工业用途。


→OpenJDK程序集可以并且应该从此处下载


这是一个大型且重要的版本，结合了Tracker中的17个主要功能和无数错误。 让我们回顾一下最大的功能：


  • JEP 181 ：基于嵌套的访问控制
  • JEP 309 ：动态类文件常量
  • JEP 315 ：改进Aarch64本征
  • JEP 318 ：Epsilon：无操作垃圾收集器（实验性）
  • JEP 320 ：删除Java EE和CORBA模块
  • JEP 321 ：HTTP客户端（标准）
  • JEP 323 ：Lambda参数的局部变量语法
  • JEP 324 ：与Curve25519和Curve448的关键协议
  • JEP 327 ：Unicode 10
  • JEP 328 ：飞行记录器
  • JEP 329 ：ChaCha20和Poly1305加密算法
  • JEP 330 ：启动单文件源代码程序
  • JEP 331 ：低开销堆分析
  • JEP 332 ：传输层安全性（TLS）1.3
  • JEP 333 ：ZGC：可扩展的低延迟垃圾收集器（实验性）
  • JEP 335 ：弃用Nashorn JavaScript引擎
  • JEP 336 ：弃用Pack200工具和API
