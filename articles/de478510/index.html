Wir laden Sie zum DINS QA EVENING 12/12/19 ein: Erstellen Sie eine Jenkins-Pipeline und lernen Sie, wie Sie mit deren Hilfe den Start von Tests parallelisieren können

Wir freuen uns, alle, die sich für Testautomatisierung interessieren, zum nächsten offenen DINS QA EVENING einzuladen, das in unserem Büro in Staro-Petergofsky, 19, stattfindet.

Heute Abend wird Alexander Mikhailov von Yandex.Money seine Erfahrungen beim Schreiben der Jenkins Pipeline teilen und zeigen, wie es beim Testen hilft. DINS Ilya Gulyaev erklärt Ihnen, wie Sie den Start von Tests auf mehreren Maschinen parallelisieren und mit derselben Jenkins-Pipeline so effizient wie möglich durchführen können.

Unter der Rubrik - Weitere Informationen zu Berichten und Referenten finden Sie einen Link zur Anmeldung für die Teilnahme an der Besprechung, Informationen zur Übertragung sowie Materialien der letzten Besprechung.



Berichte


„Jenkins Pipeline. Was ist das und wie verwenden wir es beim Testen? “(Alexander Mikhailov, Yandex.Money)

Jenkins Pipeline zu schreiben ist einfach! Zum ersten Mal sieht es kompliziert aus, aber die Syntax ist recht einfach und unkompliziert. Ich erkläre Ihnen, wie Sie mit minimalem Arbeitsaufwand eine Pipeline schreiben. Ich zeige Ihnen das Beispiel der Verwendung von Pipelines in unserem Team. Und was ist, wenn jemand alle Ihre 10+ Jobs gelöscht hat? Der Bericht ist für Junior Developer und QA Automation von Interesse.

In den letzten 4 Jahren hat Alexander Tests in Fintech-Projekten in Java und Kotlin automatisiert. In dieser Zeit habe ich sowohl das Backend als auch das Frontend getestet und nicht nur. Derzeit arbeitet er in einem Team, das Testautomatisierung als Dienstleistung anbietet.

"Distributed Test Launch" (Ilja Guljajew, DINS)

Jeder braucht einen verteilten Testlauf! Wir werden herausfinden, wie wir unsere Motorräder mit der Jenkins-Pipeline (nicht) schreiben können, und die Einführung von Tests für Hunderte von Autos skalieren. Ich erzähle Ihnen von den Fallstricken und wie Sie sie beseitigen können. Der Bericht ist für QA Automation auf jeder Ebene hilfreich. Wer schon einmal einen verteilten Testlauf hinter sich hat oder das Gefühl hat, bald konfrontiert zu sein.

Ilya beschäftigt sich mit Testautomatisierung - vom Schreiben von Tests bis zur Anpassung von Test-Frameworks an die Aufgaben des Projekts. Er liest gerne Code und versteht, wie alles von innen funktioniert. Er hat Erfahrung mit dem Schreiben von Plugins für Jenkins. Er glaubt, dass es sich lohnt, Fahrräder zu fahren, statt sie zu bauen.
