Resource Planner di HPE InfoSight



HPE InfoSight adalah layanan cloud HPE yang secara proaktif mengidentifikasi potensi masalah dengan keandalan dan kinerja array HPE Nimble dan HPE 3PAR. Pada saat yang sama, layanan juga dapat segera merekomendasikan cara untuk menghilangkan kemungkinan masalah, dan dalam beberapa kasus, pemecahan masalah dapat dilakukan secara proaktif, dalam mode otomatis.

Kita sudah bicara tentang HPE InfoSight di HABR, lihat, misalnya, di sini atau di sini .

Dalam posting ini saya ingin berbicara tentang satu fitur baru HPE InfoSight - Resource Planner (Resource Planner).

Perencana Sumber Daya Info HPE adalah alat baru yang kuat yang membantu pelanggan menentukan apakah mereka dapat menambahkan beban kerja atau aplikasi baru ke array mereka berdasarkan pada beban kerja yang ada. Apakah array akan menarik beban atau apakah array baru diperlukan? Jika Anda membutuhkan array baru, yang mana? Pemodelan prediktif dari Resource Planner membantu untuk secara akurat memahami kebutuhan dan dengan benar melakukan ukuran upgrade dari array yang ada atau ukuran array baru.

Penjadwal memungkinkan Anda untuk melakukan hal berikut:

  • Memodelkan perubahan potensial pada beban kerja yang ada
  • mengevaluasi dampak pada sumber daya array seperti: prosesor, kapasitas, dan cache;
  • lihat hasil untuk berbagai model array.

Dengan mengumpulkan statistik dan informasi parametrik tentang operasi array (di seluruh database array yang diinstal) dan menganalisis berbagai beban kerja di banyak lingkungan klien, kami dapat mengidentifikasi hubungan sebab akibat dan kuantitatif tertentu. Sebagai contoh, kita tahu bagaimana deduplikasi mempengaruhi pemanfaatan prosesor dalam model array yang berbeda. Kita tahu bahwa lingkungan Virtual Desktop terduplikasi dan dikompresi lebih baik daripada SQL. Kita tahu bahwa aplikasi Exchange biasanya memiliki persentase sekuensial (dibandingkan dengan acak) yang lebih tinggi daripada Virtual Desktop. Dengan menggunakan informasi ini, kita dapat mensimulasikan efek perubahan beban untuk memprediksi kebutuhan sumber daya untuk model array tertentu.

Bagaimana Penjadwal bekerja - mari kita lihat contoh berikut.

Resource Planner berjalan di portal HPE InfoSight di bagian LABS. Mari kita mulai dengan memilih beban baru - Tambahkan Beban Kerja Baru (selain yang sudah ada). Opsi lain yang akan kami pertimbangkan nanti adalah Tambah Beban Kerja yang Ada.



Pilih kategori pemuatan / aplikasi:



Jika perlu, Anda dapat membuat berbagai perubahan pada beban kerja baru: volume data, TIO, jenis beban kerja, dan mode deduplikasi.



Selanjutnya, kami memilih array (dari yang tersedia untuk pelanggan) yang kami ingin mensimulasikan beban kerja baru ini dan klik tombol Analisis.



Pada akhirnya, kami mendapatkan dampak dari beban kerja yang diusulkan ini (selain beban saat ini) pada sumber daya dan kapasitas CPU. Jika kami memilih array flash hybrid, kami juga akan melihat efek pada cache array, tetapi dalam hal ini kami memiliki array AF60 semua-flash, yang konsep memori cache (pada SSD) tidak berlaku.

Kita melihat (di sebelah kanan, pada diagram atas, kebutuhan CPU) bahwa array AF60, yang kami rencanakan beban baru, tidak memiliki sumber daya prosesor yang cukup untuk menangani beban kerja baru: ketika menambahkan beban baru, CPU akan digunakan 110%. Diagram bawah (Kebutuhan kapasitas) menunjukkan bahwa di bawah beban baru ada kapasitas yang cukup. Selain array AF60, kedua diagram juga menunjukkan model array lainnya - untuk perbandingan dengan apa yang akan terjadi jika kami memiliki array yang berbeda.



Gambar berikut ini menunjukkan apa yang terjadi ketika kami memeriksa kotak multi rak display (opsi saat memilih larik sumber). Opsi ini memungkinkan analisis untuk beberapa array identik. Dapat dilihat bahwa untuk muatan umum (baru dan yang sudah ada), satu array AF80, atau dua array AF60, atau tiga array AF40 sudah cukup.



Menggunakan penjadwal sumber daya, Anda juga dapat mensimulasikan perubahan hanya pada beban saat ini. Untuk melakukan ini, pada langkah pertama Anda harus memilih menambahkan beban kerja yang ada (alih-alih menambah beban kerja baru - seperti yang kami lakukan di awal). Selanjutnya, Anda dapat mensimulasikan perubahan pada beban yang ada dan melihat apa yang akan terjadi. Contoh di bawah ini mensimulasikan penggandaan beban dan penggandaan kapasitas untuk aplikasi seperti File Server (mis., Dalam contoh ini, kami tidak menambah seluruh beban pada larik, tetapi menambah beban hanya untuk jenis aplikasi tertentu).



Dalam hal ini, dapat dilihat bahwa sumber daya array dapat menggandakan beban untuk aplikasi File Server, tetapi tidak lebih dari dua kali lipat - karena sumber daya prosesor akan digunakan 99%.
