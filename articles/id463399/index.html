Memperkenalkan Pembaruan 3CX 16 3 Alpha - Advanced DNS dan Mobile Client Reconnection

Meskipun di halaman dan Agustus, kami tidak bersantai dan terus mempersiapkan musim bisnis baru. Bertemu 3CX v16 Perbarui 3 Alpha! Rilis ini menambahkan konfigurasi otomatis batang SIP berdasarkan informasi dari DNS, koneksi ulang otomatis klien seluler untuk Android dan iOS , pengenalan audio, dan menyeret lampiran ke jendela obrolan klien web.



Rilis baru ini menghadirkan fitur-fitur berikut:

Opsi DNS baru untuk batang SIP - opsi "Auto Discovery" memungkinkan Anda untuk secara otomatis menentukan jenis transportasi (UDP, TCP, TLS) dan jenis protokol trunk (IPv4 atau IPv6). Ini adalah peningkatan yang signifikan untuk teknologi trunk SIP di 3CX. Bagikan umpan balik Anda tentang seberapa baik deteksi otomatis berfungsi.



Penyambungan kembali otomatis aplikasi 3CX (dukungan sisi server) - aplikasi Android Beta dan aplikasi iOS yang belum dirilis dapat secara otomatis menyambung kembali ketika koneksi terputus, misalnya, ketika pengguna beralih dari WiFi ke jaringan 3G / 4G. Rekoneksi juga dapat digunakan dalam jaringan perusahaan besar tempat roaming pengguna digunakan. Sebagai aturan, saat roaming diperlukan, disarankan untuk menggunakan sistem DECT multi-sel yang mahal atau titik akses khusus yang mahal dengan pengontrol dan sertifikasi Wi-Fi Voice Enterprise . Namun, berkat implementasi perangkat lunak koneksi ulang di tingkat aplikasi, persyaratan ini tidak lagi diperlukan. Benar, masih ada kebutuhan untuk memprioritaskan lalu lintas suara, tetapi coba aktifkan dukungan codec OPUS - sangat mungkin bahwa semuanya akan bekerja "dan sebagainya."

Pembuat kode widget Obrolan Langsung dan Obrolan bawaan - di bagian "Opsi"> "Integrasi WordPress / Situs Web", kini Anda dapat menentukan opsi yang diperlukan untuk widget Obrolan Langsung dan Obrolan dan menghasilkan blok kode yang sudah jadi untuk situs Anda. Omong-omong, dalam waktu dekat kami akan menyajikan berita menarik terkait dengan integrasi teknologi teleponi kami dalam berbagai aplikasi pihak ketiga.



Ikon unduhan aplikasi 3CX - Ikon Google Play dan App Store untuk mengunduh aplikasi dengan cepat untuk iOS dan Android muncul di antarmuka klien web 3CX. Itu hanya nyaman.
Jembatan 3CX (Jembatan) telah dipindahkan ke bagian batang SIP - sekarang batang di antara berbagai PBX (Jembatan 3CX), batang SIP, gateway VoIP dan koneksi SBC dikonfigurasikan dalam satu bagian.

Dukungan untuk Ponsel IP Intelbras TIP 120 dan TIP 125 juga telah ditambahkan.

Uji rilis


Untuk menguji semua fitur rilis ini, instal versi beta terbaru dari aplikasi 3CX.



  • Aplikasi 3CX untuk Android Beta - dengan antarmuka pengguna baru, enkripsi panggilan melalui terowongan yang aman dan dukungan untuk koneksi ulang otomatis ke PBX. Sambungkan ke program uji beta 3CX dan unduh aplikasi dari Google Play.
  • Aplikasi 3CX untuk iOS - dengan dukungan protokol IPv6, enkripsi panggilan melalui terowongan dan obrolan fungsional dengan gaya klien web 3CX. Dalam waktu dekat, kemampuan aplikasi untuk iOS dan Android akan sama - sambungkan ke program TestFlight.

  • Pembaruan 3CX V16 3 Aplikasi Alpha dan 3CX memerlukan sertifikat SSL tepercaya di server PBX. Sertifikat yang ditandatangani sendiri tidak didukung.

Kami mengingatkan Anda bahwa instalasi rilis Alpha dan Beta 3CX secara default tidak dilakukan. Namun, Anda dapat menginstal rilis ini secara manual sesuai keinginan. Jangan menginstal rilis Alpha dan Beta di lingkungan kerja - mereka tidak tunduk pada peraturan dukungan teknis.
