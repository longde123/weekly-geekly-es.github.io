CodeOne 2018作为JavaOne，但仅在掩码中

Oracle代码一




Code One （JavaOne，直到2018年4月）是关于Oracle产品和Java编程语言的年度会议，在不同的城市举行。 许多Java开发人员都希望参加此活动并提前计划他们的参与。 我，Sbertech的测试自动化工程师，很幸运地参加了今年10月在旧金山举行的一次会议。 在猫之下，我对这次旅行的印象和想法。 在2019年Code One主持下的会议日期可在此处找到。

在Code One 2018上提交了600多份报告，其中约200份与Java有关，其余与容器，无服务器，云应用，数据库，大数据，数据科学，DevOps，现代Web等主题相关。 在同一日期（10月21日至10月25日），Oracle首次组织了另一场开放世界会议。 两项活动均在Moscone中心商务中心举行。 代码一在莫斯康中心西举行。

要注册会议，您需要一个Oracle帐户。 在注册过程中，系统会提示您选择一种方便的付款方式。 就我而言，是通过银行转帐付款的。 一周后，我收到了电子邀请。

组织细微差别


会议之前，组织者将一般会议计划发布在网站上 。

报名


在活动开始前一天进行认证最方便，否则在开幕当天您可能排成一排。



经鉴定合格后，发出带有姓名的塑料徽章（损失70美元的罚款）以及会议的每一天的塑料优惠券（午餐）。 在底楼的单独架子上，为参与者提供了Oracle纪念品（在2018年是背包）和Oracle CloudFest的门票。

行动计划


所有报告和讲习班都在二楼举行。 一个具有两个大厅的大区域，每个大厅有12个带区域的大厅：Java Magazine，Oracle Quiz，Oracle Shop和带表的区域。 同时，举行了20至24次会议。 他们从早上8-45或9-00开始，在17-00及以后结束。







包含报告和研讨会主题的程序已发布在“代码目录 ”网站“ 会话目录”模块中。

参加如此大规模的会议时，重要的是要制定自己的访问计划。 许多会议在建筑物的不同部分进行，因此不可能同时覆盖所有会议。 选择适合自己的最有趣的事物，制定时间表并继续了解Java（花了我大约30分钟的时间来了解第二天的报告并选择我感兴趣的报告）。 我建议您注意报告的作者，并熟悉他以前的演讲。 选择备份报告：如果在最初的几分钟内您发现讲话者不是您本人，则将有机会听到另一场演讲。

最好提前注册研讨会，否则您可能无法参加。 确保车间内的Wi-Fi工作稳定（如果您使用的是最新型号的笔记本电脑，例如Macbook，建议使用RJ-45适配器，因为Wi-Fi是间歇性的并且信号接收电平在任何地方都不稳定）。

报告书


总共，我在4天内参加了25场会议。 话题是最多样化的，甚至是Kito Mann遇到的好老硒 。

以下是我喜欢的报告列表。

1. Java 5、6、7、8、9、10、11：您错过了什么？ -Henri Tremblay，回顾了每个Java SE版本的主要功能，并讨论了新的Java发布策略。 建议阅读Java Champions的Java Is Is Free Free文章。



2. LinkedIn的整体测试 -Sajid Topiwala谈到了他们如何在LinkedIn上测试微服务，内部工具Epsilon以及整个开发如何进行基于Trunk的开发。





3. Java SE的未来 -Sharat Chander和Aurelio Garcia-Ribeyro的报告，有关新的Java＆OpenJDK版本模型，Java应该进一步期待哪些功能以及现在应该关注哪些项目（“不再有Java路线图指出了什么功能将在下一个版本中发布。”©）。



4.将Java UI客户端应用程序迁移到OpenJDK 11 -Philip Race和Sergey Bylokhov讲述了Java UI世界的巨大变化。


5.如何为Web测试自动化创建体系结构 -Elias Nogueira和Tatiane Aguirres Nogueira讨论了Web自动化期间应注意的要点。 我建议作者将Allure用于报告，将Selenoid用于自动测试基础结构。



6.作为现代化Java的Kotlin -Roman Elizarov的报告。 演讲者将所有注意力都集中在该语言的主要特征上。 材料的提交首先受到赞扬。



7.千分尺：它是SLF4J，但用于公制！ -乔恩·施耐德（Jon Schneider）审查了千分尺工具，该工具用于收集应用程序的指标。 从M4开始，此工具在SpringBoot 2.0中是开箱即用的。 千分尺类似于SLF4J，但仅用于收集度量。 作者解释了为什么我们应该在应用程序中使用指标，并给出了使用示例。

8.充分利用出色的IDEA：通过IntelliJ成为超级用户-Maxim Novak展示了IDEA领域的专业人员应该如何工作。 具有热键的源项目 。

9. Lambda编程实验室 -来自Jeff Dinkins，Stuart Marks，Maurice Naftalin和JoséPaumard的lambda研讨会。 您可以在github上找到原始任务和答案。

10.更多Java Community Insider的秘密 -Stephen Chin，Yolande Poirier和Sebastian Daschner的演讲。 该报告揭示了Java社区的秘密。 如何成为Java冠军，需要哪些步骤使您的文章在Java Twitter上转推 ，如何为Java Magazine撰写文章。 作者讨论了JavaOne4Kids，并鼓励积极分享经验。

我还建议关注布鲁诺·索扎 （ Bruno Souza） ，他在他的报告“开发人员设计自己的职业并打破职业高原的一种技能”中提出了有趣的开发人员职业问题。

总结


与我之前访问的会议相比，Code One因其规模和多方向性而给我留下了深刻的印象：SeleniumConf Chicago，Joker，Heisenbug，SQADays等。当然，我想一次涵盖所有内容，但不幸的是，同时发生了许多有趣的报告。

报告的质量各不相同，包括质量很低。 因此，有时我什至想知道如何选择这样的发言人参加会议。 毫无疑问，JUG.ru会议获胜：这些家伙努力只做出高质量的报告，而且他们做得很好。 但是，当然，有更多的世界领导者在Code One上。

除了午餐时间排长队和Wi-Fi信号差之外，我对活动的组织感到满意。 如果我们再次将它与JUG.ru进行比较：在Joker和JPoint上，我从未注意到网络有问题。 再加上JUG.ru，您就有机会购买任何房间的在线观看广播，因此无法远程观看Code One。

顺便说一句，很高兴在1-2位同事的陪同下参加这样的会议，讨论从报告中获得的信息并吸收更多的材料。
