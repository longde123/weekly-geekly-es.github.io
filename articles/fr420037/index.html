Stream avec plusieurs caméras à partir de matériaux improvisés

Tout a commencé lorsque, en m'utilisant, en utilisant OBS Studio et une sorte de programme pour animer des fonds d'écran, j'ai créé un logo vidéo à partir du logo (qui était également animé sous la musique en cours). À ce moment, j'ai réalisé que OBS Studio peut BEAUCOUP plus que de simples jeux en streaming. Après une connaissance plus étroite avec OBS Studio, elle prétend être le programme le plus cool que j'ai jamais rencontré .


J'ai dû diffuser un petit événement pour des amis, mais comme j'ai le matériel pour un bon son et l'envie de faire cool, j'ai été intrigué par l'organisation du stream avec un son séparé et plusieurs caméras. Pour diverses raisons, le flux s'est avéré être moyen, mais après cette expérience, je semble imaginer comment cela devrait être. Et je veux partager. Ici.



Quoi?


L'idée est de trouver 2-3 opérateurs sans fil qui se rendent sur différents sites de l'événement, communiquent avec les gens, décollent du trafic, etc. (enfin, comme des mecs sérieux, bref). Et quelqu'un s'assoit, communique avec lui sur le talkie-walkie et forme à partir de tout cela (et d'autre chose, par exemple, n'importe quel type de fenêtres / moniteurs / projecteurs restreints) un contenu vidéo intéressant qui n'est pas ennuyeux à regarder.


Vous pouvez également combiner les écrans de plusieurs ordinateurs en un seul flux, cela peut être utile pour les jeux. Bien qu'il existe très probablement d'autres technologies spécifiquement pour cette tâche.


Pourquoi?


Parce que cela vous permet de faire très cool et de déplacer considérablement le plafond technologique de la qualité du flux presque pour rien.


Lorsque vous utilisez un tas de flux vidéo, la fraîcheur du flux sera déterminée par la composante créative et organisationnelle (enfin, et la qualité des connexions).


Quoi?


  1. Ordinateur portable
    1.1. Obs studio
    1.2. nginx avec module rtmp
  2. Les opérateurs avec leurs smartphones
    2.1. Une sorte d'application émettant la demande "stream rtmp", par exemple Larix Broadcaster , ou (s'il y a un peu d'argent supplémentaire) Ace Live Streaming ou BitStream
  3. Wi-Fi stable, de préférence 5 GHz
  4. Liaison montante stable

S'il y a autre chose, vous pouvez faire un bon son, au moins sur des caméras fixes (qui peuvent être n'importe quelle toile de relativement haute qualité).


Et s'il y a un autre ordinateur Linux (ils disent que même le Raspberry Pi est correct pour ≤3 flux), vous pouvez légèrement décharger l'ordinateur principal et son réseau pour traiter et envoyer le flux.


Comment?


Le protocole reconnu pour le streaming vidéo est RTMP . Essayer d'utiliser autre chose mène à des retards sauvages, ce fut ma principale erreur.


En toute honnêteté, il convient de noter la technologie propriétaire NewTek NDI (merci, Alexsey ), qui semble plus cool que RTMP. Cependant, pour notre cas d'utilisation, l'utilisation de NDI nécessitera beaucoup d'argent supplémentaire.

Il s'avère que l'acceptation d'un flux RTMP et sa redirection vers OBS Studio est aussi simple que de pilonner des poires. Il vous suffit de compiler nginx avec un module personnalisé et d'écrire une config. Mais plus à ce sujet plus tard.


Dans le bon sens, vous devez faire quelque chose comme ceci:



Autrement dit, ne donnez pas aux téléphones mobiles Internet afin qu'ils ne dépensent pas de ressources pour autre chose qu'un flux.
Mais s'il existe déjà une bonne connexion Wi-Fi avec Internet (et qu'il n'y a pas de point d'accès supplémentaire à 5 GHz), alors vous ne pouvez pas dédaigner et utiliser les connexions existantes. Cependant, il est toujours souhaitable d'envoyer le flux final à partir d'une autre connexion, ou, dans des cas extrêmes, à partir de la même, mais via un câble.


S'il est nécessaire de recevoir des flux de caméras et d'envoyer le flux final via le même wifi, testez soigneusement la stabilité (et notez qu'elle diminuera considérablement si une foule d'appareils saute sur ce wifi).


Comment monter le serveur RTMP?


Il est préférable de le faire sous Linux, afin de ne pas rencontrer de problèmes avec toutes sortes de MINGW / MSYS. Et sur un matériel séparé (pas forcément puissant). Ou dans le menu fixe , vous pouvez ignorer cette section, car le fichier docker existe déjà . Grâce à WSL, il est également possible, mais il sera nécessaire de transférer manuellement le port TCP 1935 dans le pare-feu.


Voici un excellent manuel Comment configurer votre propre serveur RTMP privé à l'aide de nginx , et il est également écrit dans le dock . Un bref récit du freestyle:


  1. Nous mettons les dépendances: 


    sudo apt install build-essential libpcre3 libpcre3-dev libssl-dev

  2. Téléchargez la dernière version Mainline de la source ici : 


     wget http://nginx.org/download/nginx-1.15.2.tar.gz # Check for newer versions tar xzf nginx-* && cd !$

  3. Téléchargez la dernière version du module RTMP : 


     wget https://github.com/arut/nginx-rtmp-module/archive/master.zip -O rtmp-module.zip unzip !$ -d .

  4. Créez un Makefile: 


     ./configure --with-http_ssl_module --add-module=./nginx-rtmp-module-master

    Si vous tombez avant d'atteindre le Configuration summary la Configuration summary , nous résolvons les problèmes.
    Cette commande configurera votre streamer nginx pour qu'il vive dans /usr/local/nginx/ et s'exécutera avec la commande sudo /usr/local/nginx/sbin/nginx . Ainsi, nginx du référentiel ne ressent rien. Cela peut être changé en examinant ./configure --help .


  5. Nous compilons: 


     make -j4

  6. Installer: 


     echo "nginx with RTMP module" > description-pak sudo checkinstall --pkgname nginx-rtmp --provides nginx --nodoc --deldesc -y sudo mkdir /usr/local/nginx/logs/ # Doesn't start without it

    En principe, utiliser make install dans ce cas est sûr, mais ce n'est toujours pas nécessaire .


  7. Vérifier 


     $ /usr/local/nginx/sbin/nginx -v nginx version: nginx/1.15.2

  8. Personnaliser 


     sudo vim /usr/local/nginx/conf/nginx.conf

    À la fin, ajoutez la configuration du serveur RTMP: 


     rtmp { server { listen 1935; application live { live on; record off; } } }

    Si vous le souhaitez, vous pouvez configurer le serveur HTTP pour afficher des statistiques .
    Le dock décrit ce que vous pouvez configurer d'autre, il y a vraiment beaucoup de choses. Si vous savez comment le faire mieux, je serai heureux d'ajouter une section sur la configuration du serveur avec du matériel des commentaires.


  9. Nous le rendons pratique 


     alias rtmp-start="sudo /usr/local/nginx/sbin/nginx" alias rtmp-stop="sudo /usr/local/nginx/sbin/nginx -s stop" alias rtmp-status="cat /usr/local/nginx/logs/nginx.pid"


Que faire avec le serveur RTMP?


  1. Envoyez-lui un flux depuis l'application mobile à rtmp://< IP>:1935/live/habr où live est le nom de l'application RTMP dans la configuration nginx, et habr est la clé de flux , qui doit être modifiée pour différentes caméras.


  2. Si vous avez configuré pour afficher les statistiques (et que vous n'avez pas oublié de modifier le chemin d'accès au fichier stat.xsl ), vérifiez que le flux est arrivé (à http: // localhost: 8080 / stat ).


  3. Connectez-vous à tous les flux OBS Studio.




  4. PROFIT !!!



De toute évidence, le serveur peut être non seulement local, mais également accessible de l'extérieur, ce qui vous permettra de faire de même, mais pas via Wi-Fi, mais via Internet. Vous pouvez créer votre propre analogue de flux de groupe Instagram et, en général, des possibilités illimitées))

  • Il est nécessaire de prendre le code source EXACTEMENT auprès de Mercurial (c'est bien qu'il y ait un bouton "télécharger zip"), il n'y a pas de dossier src\os\win32 dans les archives du site. Si vous avez des codes sources sur le site, il y aura une erreur don't know how to make 'src/os/win32/ngx_win32_config.h' .
  • Besoin d'un ancien MinGW d'ici et non un peu moins ancien MSYS2.
  • Lors de l'installation de mingw-developer-toolkit vous devez supprimer la case à cocher de Perl. Perl doit être Windows.
  • Cette réponse a bien aidé.
  • Lors de la compilation, rc.exe utilisé à partir de %ProgramFiles(x86)%\Windows Kits\10\bin\10.0.16299.0\x64 , vous devez ajouter ce dossier manuellement dans PATH
  • Il est nécessaire de supprimer l'indicateur -WX de CFLAGS dans le Makefile pour que les vorings n'interrompent pas la compilation, il y a des vorings dans 1.15.3 ...
