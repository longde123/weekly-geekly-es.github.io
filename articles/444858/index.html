En busca del mejor remedio

En este artículo, le mostraré cómo conocí a Quest Netvault Backup. Acerca de Netvault Backup, cuando este software todavía era propiedad de Dell, ya había escuchado muchas críticas positivas, pero no había tenido que "tocarlo" con las manos.


Quest Software, también conocido como Quest, es una compañía de software con sede en California con 53 oficinas en 24 países. Fue fundado en 1987. La compañía es conocida por su software utilizado por profesionales en el campo de las bases de datos, en la gestión de la nube, en la seguridad de la información, para el análisis de datos, para el respaldo y la recuperación. Quest Software fue adquirido por Dell en 2012. Para el 1 de noviembre de 2016, la venta se completó y la compañía se reinició como Quest Software.

Conozca bien Quest Netvault no hace mucho tiempo. En uno de los proyectos, el Cliente solicitó encontrar una solución económica y óptima para proteger su infraestructura. El cliente estaba considerando varios programas de respaldo, una de las soluciones fue Quest Netvault Backup. En función de los resultados de la prueba, dados los parámetros importantes para el Cliente (algunos de los cuales se encuentran al final del artículo), se seleccionó Quest Netvault Backup.

Además de los requisitos básicos, el Cliente quería que el software se instalara en servidores con Linux. Dichos requisitos no se implementan en todos los software de respaldo, pero Quest Netvault Backup puede hacerlo.

Línea de base y requisitos


La tarea establecida por el Cliente fue diseñar un sistema que proporcione respaldo de datos en la cantidad de 62 TB. Estos datos estaban contenidos en sistemas de aplicaciones como SAP, Microsoft SQL, PostgreSQL, MariaDB, Microsoft Exchange, Microsoft SharePoint, etc. Estos sistemas de aplicación se ejecutan en servidores físicos y virtuales que ejecutan sistemas operativos de la familia Microsoft Windows Server, Linux y FreeBSD. El entorno virtual se creó sobre la plataforma de virtualización VMware vSphere. La infraestructura estaba ubicada en un sitio.

Dispositivos de almacenamiento

Quest Netvault implementa fácilmente la regla de almacenamiento 3-2-1, ya que también puede funcionar con dispositivos para almacenamiento de respaldo en línea (sistemas de almacenamiento en disco), así como con dispositivos para almacenamiento a largo plazo (dispositivos de deduplicación, bibliotecas de cintas físicas, cargadores automáticos, virtuales bibliotecas de cintas (VTL) y bibliotecas de cintas virtuales compartidas (SVTL)). Las copias de seguridad alienables se pueden almacenar en la nube, en un sitio remoto o en medios alienables (por ejemplo, una unidad de cinta).

Cuando se trabaja con dispositivos de deduplicación, se admiten protocolos especializados de RDA y DD Boost. Usando estos protocolos:

  • reduce la carga en la red y mejora el rendimiento de las tareas de respaldo, ya que los datos se deduplican en el cliente y solo se transfieren los bloques necesarios. Por ejemplo, trabajar en conjunto con Quest Qorestor a través del protocolo RDA puede lograr un rendimiento de hasta 20 terabytes por hora y una compresión de 20 a 1;
  • protege las copias de seguridad de los virus ransomware. Incluso si el servidor de respaldo está infectado y encriptado, las copias de seguridad permanecerán intactas. enlace .

Resultado de implementación


El resultado del proyecto fue un sistema de respaldo implementado por el Cliente basado en el software Quest Netvault Backup con la arquitectura que se muestra en la Figura 1.7.



Figura 1.7 - Estado del sistema de destino

Todos los componentes de Netvault Backup se implementaron en un servidor físico con las siguientes características:

  • dos procesadores con diez núcleos cada uno;
  • 64 GB de RAM;
  • dos discos duros SAS 300GB 10K (RAID1)
  • cuatro discos duros SAS 600GB 15K (RAID10);
  • HBA con dos puertos SAS externos;
  • dos puertos de 10 gbps;
  • OS CentOS.

Las copias de seguridad en línea se almacenaron en Quest Qorestor Standard (back end 150TB). El trabajo con Qorestor se realizó de acuerdo con el protocolo RDA. El coeficiente de deduplicación en Qorestor al final de la operación de prueba del sistema fue de 14.7 a 1.

Para el almacenamiento a largo plazo, se utilizó una biblioteca de cintas con cuatro unidades estándar LTO-7. La biblioteca de cintas conectada al servidor de respaldo a través de SAS. Periódicamente, los cartuchos se desechaban y se trasladaban para almacenarlos en una de las sucursales remotas.

Todos los complementos necesarios se descargaron y se colocaron en una carpeta de red para la instalación remota. El tiempo de implementación y configuración para este sistema fue de nueve días.

Conclusiones


De acuerdo con los resultados del proyecto, puedo decir que Quest Netvault Backup pudo implementar todos los requisitos del Cliente y esta solución es una de las herramientas para construir un sistema de respaldo para pequeñas empresas y Clientes de nivel empresarial.
