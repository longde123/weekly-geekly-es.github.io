Bagaimana saya mulai berbicara di konferensi dan tidak bisa berhenti

Dunia perkembangan modern, dengan caranya sendiri, indah. Praktik yang baik adalah aliran bebas dari pengetahuan dan pengembangan mereka. Mengejar pengetahuan menciptakan permintaan, dan habr, pemanggang roti (sekarang qna), github, pertemuan, konferensi dan banyak lagi adalah tawaran yang sangat baik. Hari ini saya ingin berbicara tentang pertemuan dan konferensi. Di bawah potongan, kisah tentang bagaimana saya, sebagai pengembang dan pemilik perusahaan IT, mulai berbicara di konferensi TI.


Pada tahun 2008, ketika saya berusia 14 tahun ketika saya sedang mengembangkan robot perdagangan di pasar FORTS, saya tidak dapat membayangkan bahwa saya akan menjadi bagian dari komunitas ini. Pengembangan bot perdagangan jauh dari open source - tidak ada yang berusaha membagikan penemuan dan cara mereka untuk mencapai tujuan. Pada semua acara tematik, sebagian besar orang berbagi kesuksesan finansial mereka dan menawarkan untuk berinvestasi dalam diri mereka sendiri. Tetapi pada beberapa titik, yaitu pada tahun 2011, saya mulai terjun ke dunia pengembangan WEB dan terkejut melihat bagaimana semuanya terbuka di dalamnya. Sejak itu, saya tetap berada di dunia "pengembangan browser".


Lambat laun, saya "maju" dan mulai membuat situs web dan "portal" di wordpress, modx, dan bahkan menulis sistem manajemen agensi real estat kecil di kota kami. Saya akui, pada saat itu saya pikir saya tahu "hampir semuanya" :) Saya belum pernah bertemu orang yang membuka kerangka kerja Yii2 untuk saya. Saya kemudian lulus dari Institut Fisika dan Teknologi Moskow dan menghabiskan seluruh musim panas untuk belajar - merekam konferensi, artikel tentang hub, GitHub - Saya mengumpulkan sepenuhnya semua informasi yang dapat saya temukan tentang masalah OOP, Yii2, Laravel, Zend, dan sebagainya.



Pada persiapan PHPRussia 2019


Beberapa tahun berlalu, di halaman pada Januari 2019 - Saya memiliki perusahaan sendiri, beberapa proyek serius di belakang saya, keinginan untuk bergerak maju dan memperluas wawasan saya. Dan di sini saya, secara kebetulan, menemukan pos Sasha Makarov ( SamDark ) tentang fakta bahwa sebuah konferensi tentang PHP di Moskow direncanakan! Saya memutuskan sendiri bahwa saya akan berakhir di sana! Benar, ketertarikan terbesar pada saya disebabkan oleh kesempatan untuk berbicara di konferensi ini. Sejak masa YiiConf di Moskow (Juni 2017), saya mulai berpikir untuk berbicara di konferensi pengembangan - sudah 3 tahun sejak rencana saya matang dan saya menyadari bahwa itu pertanda :) Dan di sini semua acara berputar yang membawaku ke 11 penampilan untuk 2019.


Atas alasan saya mengapa kinerja itu keren


Komunikasi dengan komite program dan kerja bersama dalam laporan


Untuk memunculkan topik laporan, tesis dan banding ke komite program adalah tugas yang agak sulit - terutama untuk pertama kalinya. Itu bukan tanpa keberuntungan: di perusahaan saya, saya mencoba untuk terus-menerus mencoba teknologi baru dan hanya pada saat itu kami mentransfer beberapa proyek dari REST API ke GraphQL - keuntungannya terlihat, masalah juga :) Oleh karena itu, topik pertama dari laporan saya disebut "Transisi dari REST API dalam GraphQL menggunakan proyek nyata sebagai contoh. "


Kirill Nesmeyanov ( SerafimArts ), penginjil GraphQL dan pengembang kerangka Railt, dulu dan sekarang di komite program PHP Rusia. Dia juga menerima laporan saya. Selama beberapa minggu, kami memanggilnya, berbicara tentang berbagai topik, kadang-kadang mereka suci dan, tentu saja, mengerjakan laporan saya. Cyril berhasil menunjukkan kepada saya beberapa fitur kerangka Laravel, fitur-fitur keren dan jahat dari GraphQL, yang saya temui hanya setelah beberapa bulan. Dan secara umum, komunikasi sangat berguna sehingga memungkinkan pada tahap ini untuk berhenti dan menikmati pengalaman berkomunikasi yang bermanfaat dengan orang seperti itu.


Berkomunikasi dengan PC, bahkan SEBELUM menerima aplikasi, sudah merupakan pekerjaan yang disengaja dan bergerak maju dengan pengisian yang tak terelakkan dari bagasi pengetahuan, dan perasaan "mengisi bahan bakar otak" dengan cepat menjadi kecanduan dan saya menyadari bahwa saya bergerak ke arah yang benar.


Pendarahan dalam berbicara di depan umum dan pandangan samping


Suatu hari saya pergi tidur dan hampir tertidur. "Bzz-bzz" kata ponsel saya dan di layar saya melihat - "Laporan Anda telah disetujui." Saya harus mengatakan - tidak mudah tertidur - saya langsung bangun, jadi saya juga membangunkan istri saya, yang baru saja kembali dari rumah sakit).


Konferensi itu sebulan kemudian dan, untuk bulan ini, saya harus menyiapkan presentasi dan berlatih presentasi itu sendiri. Dan kemudian menelepon dengan pelatih komite program Roman Poborchim ( p0b0rchy ) banyak membantu saya - saya belajar banyak informasi yang berguna hanya dalam satu jam percakapan.


Banyak hal yang sekarang jelas bagi saya tidak tersedia bagi saya sebelum berbicara dengan Roman. Tampaknya huruf-huruf kecil tidak terlihat dengan baik dan lebih baik tidak menggunakannya dalam presentasi, tetapi bagaimana lagi untuk menunjukkan sepotong kode sehingga konteksnya jelas? Tetapi mudah untuk menyederhanakan kode agar hanya menyampaikan esensi dan agar sesuai dengan slide tidak lebih dari 5-7 baris kode. Saya masih berpikir, "apa yang bisa dimengerti di sini," dan kemudian saya melihat beberapa laporan lain, di mana "mereka berbagi kode besar" dan menyadari bahwa kebenaran ada di sana - tidak mudah untuk mengetahuinya di kepala orang lain, seperti dalam kode orang lain :) Atau , misalnya, Anda dapat memecah satu slide dengan sekelompok informasi menjadi 4 slide dengan blok informasi kecil dan menjelaskan hubungan di antara mereka dalam kata-kata. Saya berbicara dengan Roman tentang setiap pidato saya di konferensi ONTIKO dan setiap kali itu sangat berguna!


Persiapan laporan


Berapa yang dibutuhkan untuk membaca kembali dan mengulas sebelum menyiapkan laporan?
Jika Anda berbicara tentang sesuatu dari panggung, bersiaplah untuk menjawab maksimal pertanyaan tentang topik tersebut. Oleh karena itu, sering menyiapkan laporan adalah pemicu nyata pada topik. Bukan tanpa alasan mereka mengatakan bahwa jika Anda ingin mencapai kesempurnaan dengan sesuatu, ajarkan orang lain itu. Setelah saya berbicara tentang GraphQL, saya berbicara dengan berbagai topik. PWA, migrasi dari PHP ke GoLang, organisasi penyimpanan file adalah topik utama yang saya bicarakan pada tahun 2019, di samping laporan dengan GraphQL dan variasinya. Sementara saya sedang mempersiapkan untuk setiap laporan tentang topik yang tampaknya sudah dipelajari, saya menemukan kasus atau teknologi yang tidak terduga sehingga saya segera memperkenalkan mereka ke dalam proses perusahaan atau setidaknya membawanya ke diskusi.


Sejalan dengan PHP Rusia, saya mengirimkan laporan serupa tentang GraphQL, tetapi dengan arah yang berbeda ke RITFest dan Highload Siberia. Laporan disetujui di kedua konferensi, yang membuat saya sangat senang! Saya terutama terkesan oleh fakta bahwa saya akan mengunjungi Siberia, tempat yang sudah lama saya inginkan.


Kencan yang berguna dan dorongan energi


Waktu berlalu dan hari itu tiba - 17 Mei dan konferensi PHP Russia. Sehari sebelumnya, panitia mengatur persiapan untuk para pembicara, di mana saya bertemu secara pribadi dengan pengembang Yii2 , di mana sebagian besar proyek kami, Alexander Makarov. Secara umum, tentu saja, menghadiri konferensi sebagai pembicara memungkinkan Anda untuk secara pribadi mengenal banyak orang tentang siapa yang baru saja saya baca sebelumnya. Dan ini membuat ketagihan.



Penampilan pertama sangat gugup dan mengkhawatirkan. Saya tampil di penampilan pertama di Hall C (Aula itu disebut M-VC). Saya senang menjadi yang pertama berbicara - saya membalas dan bebas :) Setelah itu saya hanya menikmati mengobrol dan mengenal orang-orang baru yang menarik dari dunia PHP - Konstantin Burkalev ( ksdaemon ), yang podcastnya saya dengarkan sebelumnya, Dmitry Eliseev ( ElisDN ) - berdasarkan pelajarannya Saya belajar dunia OOP dan kerangka kerja dan orang lain yang sebelumnya saya hanya melihat di layar monitor.



Saya dan ksdaemon di latar belakang ksdaemon


Pada akhirnya, pada berbagai konferensi, pada tahun 2019, saya berhasil berkomunikasi secara pribadi dengan pengembang Sphinx Andrey Aksyonov ( shodan ), ClickHouse Alexey Milovidov ( o6CuFl2Q ), penulis buku dan blog di ReactPHP Sergey Zhuk ( seregazhuk ).
Secara umum, saya mendengar pendapat bahwa obrolan kosong di konferensi tidak lebih dari buang-buang waktu. Tapi saya punya pandangan sendiri tentang ini. Komunikasi di konferensi memberi saya:


  • biasa untuk mempelajari sesuatu yang baru, tentang yang saya belum dengar;
  • untuk mendapatkan pendapat para profesional tentang masalah-masalah yang menjadi perhatian saya sangat berharga;
  • penemuan, wawasan, kadang-kadang beberapa ide gila, yang kemudian dapat berkembang menjadi sesuatu yang bermanfaat (atau mungkin tidak));
  • tautan yang berguna untuk proyek yang sedang berjalan;
  • kenalan pribadi dengan orang-orang ikonik di bidang aktivitas saya;

Dan, tentu saja, ini adalah energi. Di konferensi, orang-orang pintar, bersemangat, dan antusias yang gila - di satu sisi menginspirasi, di sisi lain, ini memperjelas bahwa persaingannya tinggi dan tidak ada waktu untuk bersantai :)



Naik taksi tak berawak di pemogokan di Innopolis


Tentu saja, ada orang yang datang ke konferensi untuk bekerja, berburu, mencari klien, dll., Tetapi orang-orang seperti itu biasanya cepat berhenti berbicara sendiri jika mereka mengerti bahwa saya bukan target audiens. Karena itu, memusatkan perhatian pada mereka tidak terlalu berarti.


Pertunjukan - sebagai hobi yang bermanfaat


Sejak saat itu saya mulai masuk lebih dalam dan lebih dalam ke masalah berbicara di konferensi TI dan berencana untuk berbicara, jika mungkin, lebih jauh. Misalnya, tahun ini saya akan berbicara di http://teamleadconf.ru/moscow/2020


Tetapi hanya ingin berbicara saja tidak cukup. Sejauh ini, saya bukan pembicara terkenal yang diundang untuk tampil di suatu tempat setiap akhir pekan, saya sendiri mencari acara dan menyarankan pencalonan saya. Dan kemudian saya menemukan pertanyaan - bagaimana dan di mana menemukan peristiwa di mana Anda ingin dilihat sebagai pembicara? Saya menemukan sendiri beberapa sumber yang secara rutin saya ulas:


  • https://papercall.io - konferensi asing dan langsung di antarmuka yang sama Anda dapat mengirimkan laporan;
  • https://devfest.withgoogle.com/ - semua devfest didukung oleh Google. Saya tidak tahu apakah situs itu akan berfungsi pada tahun 2020;
  • http://it-events.com/ - berbagai acara Rusia, tetapi tidak segera jelas apakah mereka menunggu pembicara atau tidak. Anda harus membuka setiap halaman dan mempelajari masalah ini;
  • https://bit.ly/3715u4k - tabel konferensi dengan CFP. Sumber daya yang agak berguna dan nyaman, tetapi tidak semua konferensi dan sangat tidak nyaman ketika orang lain menggunakan tabel secara paralel
    Pembicara lain yang akrab - kami mencoba saling mengirim informasi tentang penerimaan terbuka laporan;

Mempelajari semua sumber daya ini, saya tidak menemukan tempat yang nyaman dari mana dimungkinkan untuk secara teratur menerima informasi terkini tentang konferensi dan tanggal untuk menerima laporan. Setelah menghabiskan beberapa waktu, saya membuat sendiri sebuah kalender dengan filter dan saluran telegram di mana sekarang saya menambahkan informasi tentang acara yang dapat Anda kirimi laporan Anda.



Kalender untuk mereka yang ingin berbicara


Saya ingin alat ini dapat diakses dan bermanfaat tidak hanya untuk saya, tetapi juga untuk pembicara lain. Sekarang saya sendiri mengisinya dengan informasi tentang acara-acara dari sumber terbuka, tetapi saya mungkin tidak tahu tentang beberapa peristiwa - saya akan berterima kasih jika Anda membaginya dengan saya dan saya akan menambahkannya ke kalender.


Jadi kehausan akan pengetahuan secara bertahap mengubah saya menjadi bagian kehidupan yang bermanfaat dan menarik, tetapi apa yang Anda pikirkan tentang manfaat berbicara di konferensi?
