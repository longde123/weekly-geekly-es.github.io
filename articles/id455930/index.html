19 Juni pukul 18:00 (Moskow) di kantor mitap IBM untuk teknologi Java
Tamu kami akan menjadi Java Champion Sebastian Daschner pada 19 Juni pukul 18:00 di kantor IBM.

Akankah kita berbicara tentang apa yang terjadi dengan Java dan server aplikasi di era cloud? Oracle memperkenalkan biaya untuk menggunakan Java di server dan workstation. Java EE berubah menjadi Jakarta EE. Seringkali, pengembang menggunakan solusi berbasis jvm murni untuk digunakan dalam kontainer di cloud pribadi dan publik, menghemat sumber daya dengan menghilangkan perpustakaan JEE biasa dari server aplikasi.

Kali ini tamu kami akan memiliki Java Champion nyata, yang dicatat di situs web Oracle Java - Sebastian Dashner. Dia akan berbicara tentang bagaimana membangun aplikasi menggunakan wadah berdasarkan server aplikasi terbuka OpenLiberty, serta struktur yang menjanjikan dari komunitas Java (OpenJDK dan AdoptOpenJDK, ...) dan Jakarta EE, serta standar MicroProfile baru untuk membuat aplikasi layanan mikro .
Sebastian Dashner akan memberi tahu Anda bagaimana membangun aplikasi dalam hal menggunakan kontainer berdasarkan pada server aplikasi terbuka OpenLiberty, serta tentang struktur yang menjanjikan dari komunitas Java (OpenJDK dan AdoptOpenJDK, ...) dan Jakarta EE, dan standar MicroProfile baru untuk membuat aplikasi layanan microservice.

Sebastian Daschner
Java Champion , penulis dan instruktur, profesional pengembangan Java (termasuk EE). Ia menulis buku Arsitektur Aplikasi Java EE Modern . Sebastian berpartisipasi dalam JCP, membantu membentuk standar Java EE di masa depan, bekerja di JAX-RS, JSON-P, dan grup pakar Config dan berkolaborasi dalam berbagai proyek sumber terbuka. Untuk kontribusinya pada komunitas dan ekosistem Jawa, ia dikenal sebagai Java Champion, Oracle dan JavaOne Rockstar Developer Champion.
Selain Java, Sebastian juga merupakan pengguna aktif Linux dan teknologi wadah seperti Docker. Dia adalah penulis blog , dia dapat ditemukan di Twitter: @DaschnerS .

20 Juni pukul 18:00 (Moskow) di kantor mitaps IBM di Service Mesh - Istio
Kami berkumpul, berkumpul, dan akhirnya berkumpul! Mitap pertama pada Istio (seperti yang dilakukan orang lain?) 20 Juni di Moskow!

Mengapa Anda harus menghabiskan waktu dan datang?


  • Kami akan memiliki orang-orang dari tim pengelola Istio! Lab Riset IBM di Haifa pernah mengembangkan proyek amalgam8, yang kemudian menjadi Istio. Dan sekarang salah satu karyawan Laboratorium (Vadim Eisenberg) berada di 5 kontributor teratas dari seluruh proyek Istio!
  • Sebenarnya, kehadiran para ahli dari Haifa sudah cukup, tetapi selain itu kami masih memiliki Phil Estes (kapten Docker, IBM Distinguished Engineer).
  • Dan kita akan memiliki cerita tentang proses mengadaptasi Istio ke "perusahaan berdarah", setidaknya dari orang-orang dari Sberbank.

Apa yang akan terjadi di mitap:


  • Mari kita bicara tentang bagaimana Istio dibuat dan mengapa arah service mesh muncul.
  • Kami akan memberi tahu Anda apa itu Istio / service mesh.
  • Mari kita bahas kapan harus menggunakan service mesh, dan kapan tidak.
  • Mari kita lihat bagaimana Istio dan Kubernet membandingkan.
  • Tampilkan demo langsung.

20 Juni pukul 19.00 (St. Petersburg) - Java Guru meetup - di platform DataArt tentang topik Jawa dan arsitektur layanan mikro


IBM dan DataArt membawa dua guru Jawa ke St. Petersburg. Sebastian Daschner , Java Champion, seorang profesional pengembangan Java, dan Denis Tsyplakov , Solution Architect, DataArt Voronezh akan berbicara pada pertemuan khusus tentang pengembangan Java.

OpenLiberty adalah jaguar yang tidak dikenal di antara server aplikasi OpenSource


Laporan ini dalam bahasa Inggris.

OpenSource menjadi semakin menjadi bagian dari komponen kehidupan kita sehari-hari dan tak tergantikan. Proses ini terjadi di seluruh dunia, termasuk Rusia. Mengapa Vendor besar bergerak menuju OpenSource untuk kesederhanaan dan penyatuan penyebaran solusi di awan.
Apa yang terjadi dengan Java dan server aplikasi di era cloud? Oracle memperkenalkan biaya untuk menggunakan Java di server dan workstation. Java EE berubah menjadi Jakarta EE. Seringkali, pengembang menggunakan solusi berbasis jvm murni untuk digunakan dalam kontainer di cloud pribadi dan publik, menghemat sumber daya dengan menghilangkan perpustakaan JEE biasa dari server aplikasi.

Tetapi bagaimana jika server aplikasi bisa sangat ringan dan fleksibel sehingga akan memungkinkan Anda untuk mengambil keuntungan dari platform Edisi Enterprise dalam wadah dengan dampak minimal pada sumber daya yang dikonsumsi? Tetapi bagaimana jika kita dapat menjadikan server aplikasi platform dasar untuk arsitektur layanan mikro?

Saya akan memberi tahu Anda cara membuat aplikasi menggunakan wadah berdasarkan server aplikasi terbuka OpenLiberty, serta tentang struktur yang menjanjikan dari komunitas Java (OpenJDK, AdoptOpenJDK dan lainnya), tentang Jakarta EE dan standar MicroProfile baru untuk membuat aplikasi layanan microser.

Sebastian Daschner


Java Champion , penulis dan instruktur, profesional pengembangan Java (termasuk EE). Ia menulis buku Arsitektur Aplikasi Java EE Modern . Sebastian berpartisipasi dalam JCP, membantu membentuk standar Java EE di masa depan, bekerja di JAX-RS, JSON-P, dan grup pakar Config dan berkolaborasi dalam berbagai proyek sumber terbuka. Untuk kontribusinya pada komunitas dan ekosistem Jawa, ia telah diakui sebagai Java Champion, Oracle dan JavaOne Rockstar Development Champion.

Selain Java, Sebastian juga merupakan pengguna aktif Linux dan teknologi wadah seperti Docker. Dia adalah penulis blog dan dapat ditemukan di Twitter melalui @DaschnerS.

Facebook dalam kiamat zombie


Layanan online modern memiliki kelemahan yang signifikan. Anda tidak memiliki data yang Anda unggah ke mereka, jangan mengontrol distribusi data ini. Setiap saat, akun Anda, dalam pengembangan yang telah Anda investasikan selama bertahun-tahun, dapat terputus dari layanan tanpa memberikan alasan dan harapan untuk pemulihan.

Kami akan membahas seperti apa Internet itu jika pengembangan berjalan sesuai dengan prinsip "data milik pengguna yang membuatnya, layanan milik pengguna yang menggunakannya".

Karena saya bukan pengacara atau politisi, tetapi arsitek Jawa, saya akan melihat masalah dari sisi teknis. Apa yang bisa menjadi alternatif untuk skema klasik "browser - situs web - database" di dunia cloud modern. Lima tahun lalu, semua alternatif tampak lemah secara teknis, tetapi sekarang dengan pengembangan layanan cloud dan teknologi Docker, Kubernetes, Helm, tampaknya setidaknya secara teknis ada alternatif.

Denis Tsyplakov, Arsitek Solusi


Dia mulai menulis program pada akhir 1980-an, dan telah pemrograman secara profesional sejak pertengahan 1990-an. Dia menulis program dalam lebih dari 10 bahasa, tetapi Jawa tetap menjadi favoritnya. Sejak 2006 ia telah bekerja di DataArt. Minat utama dalam TI: penciptaan layanan yang toleran terhadap kesalahan, arsitektur sistem pragmatis, solusi kreatif untuk tugas-tugas non-sepele.
