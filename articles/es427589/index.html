Escribimos un chat en línea en Websockets usando Swoole



El tema de Websockets ya se tocó repetidamente en Habré, en particular se consideraron opciones para la implementación en PHP. Sin embargo, ha pasado más de un año desde el último artículo con una descripción general de las diferentes tecnologías, y el mundo PHP tiene algo de qué alardear con el tiempo.

En este artículo quiero presentar a Swoole a la comunidad de habla rusa: el marco asíncrono de código abierto para PHP, escrito en C y entregado como una extensión pecl.

Fuentes en github .

¿Por qué desmayarse?


Seguramente habrá personas que en principio estarán en contra del uso de PHP para tales fines, sin embargo, a menudo pueden jugar a favor de PHP:

  • Renuencia a criar un zoológico de varios idiomas en el proyecto
  • Posibilidad de utilizar una base de código ya desarrollada (si el proyecto está en PHP).

Sin embargo, incluso comparando con node.js / go / erlang y otros lenguajes que ofrecen de forma nativa un modelo asíncrono, Swoole, un marco escrito en C y que combina un umbral de entrada bajo y una funcionalidad potente puede ser un buen candidato.

Características del marco:

  • Evento, modelo de programación asíncrono
  • TCP / UDP / HTTP / HTTP / Websocket / HTTP2 API / cliente asíncrono
  • Compatible con IPv4 / IPv6 / Unixsocket / TCP / UDP y SSL / TLS
  • Rápida serialización / deserialización de datos
  • Alto rendimiento, extensibilidad, soporte para hasta 1 millón de conexiones simultáneas
  • Planificador de tareas de milisegundos
  • Código abierto
  • Soporte de corutinas

Posibles casos de uso:

  • Microservicios
  • Servidores de juegos
  • Internet de las cosas
  • Sistemas de comunicación en vivo
  • API WEB
  • Cualquier otro servicio que requiera respuesta instantánea / alta velocidad / ejecución asincrónica

Se pueden ver ejemplos de código en la página principal del sitio . En la sección de documentación, información más detallada sobre todas las funcionalidades del framework.

Plantilla mínima del servidor Websocket: 

 <?php $server = new swoole_websocket_server("127.0.0.1", 9502); $server->on('open', function($server, $req) { echo "connection open: {$req->fd}\n"; }); $server->on('message', function($server, $frame) { echo "received message: {$frame->data}\n"; $server->push($frame->fd, json_encode(["hello", "world"])); }); $server->on('close', function($server, $fd) { echo "connection close: {$fd}\n"; }); $server->start();

$ fd es el identificador de conexión.
Obtén conexiones actuales: 

 $server->connections;

$ Frame contiene todos los datos enviados. Aquí hay un ejemplo de un objeto que entró en la función onMessage: 

 Swoole\WebSocket\Frame Object ( [fd] => 20 [data] => {"type":"login","username":"new user"} [opcode] => 1 [finish] => 1 )

Los datos se envían al cliente usando la función 

 Server::push($fd, $data, $opcode=null, $finish=null)

Servidor Websocket, se decidió emitir en forma de clase e iniciarlo en el constructor:

Constructor 
 public function __construct() { $this->ws = new swoole_websocket_server('0.0.0.0', 9502); $this->ws->on('open', function ($ws, $request) { $this->onConnection($request); }); $this->ws->on('message', function ($ws, $frame) { $this->onMessage($frame); }); $this->ws->on('close', function ($ws, $id) { $this->onClose($id); }); $this->ws->on('workerStart', function (swoole_websocket_server $ws) { $this->onWorkerStart($ws); }); $this->ws->start(); }


Problemas encontrados:

  1. Un usuario conectado al chat se desconecta después de 60 segundos si no hay intercambio de paquetes (es decir, el usuario no envió ni recibió nada)
  2. El servidor web pierde la conexión con MySQL si no se produce interacción durante mucho tiempo

Solución:

En ambos casos, necesitamos una implementación de la función ping, que constantemente hará ping al cliente cada n segundos en el primer caso, y la base de datos MySQL en el segundo.

Dado que ambas funciones deben funcionar de forma asíncrona, se deben invocar en los procesos secundarios del servidor.

Para hacer esto, se pueden inicializar con el evento "workerStart". Ya lo definimos en el constructor, y con este evento el método $ this-> onWorkerStart ya se llama:
El protocolo Websocket admite ping-pong fuera de la caja. A continuación puede ver la implementación en Swoole.

onWorkerStart 
 private function onWorkerStart(swoole_websocket_server $ws) { $this->messagesRepository = new MessagesRepository(); $ws->tick(self::PING_DELAY_MS, function () use ($ws) { foreach ($ws->connections as $id) { $ws->push($id, 'ping', WEBSOCKET_OPCODE_PING); } }); }


Luego, implementé una función simple para hacer ping a un servidor MySQL cada N segundos usando swoole \ Timer:

El temporizador se inicia en initPdo si aún no está habilitado: 

  /** * Init new Connection, and ping DB timer function */ private static function initPdo() { if (self::$timerId === null || (!Timer::exists(self::$timerId))) { self::$timerId = Timer::tick(self::MySQL_PING_INTERVAL, function () { self::ping(); }); } self::$pdo = new PDO(self::DSN, DBConfig::USER, DBConfig::PASSWORD, self::OPT); } /** * Ping database to maintain the connection */ private static function ping() { try { self::$pdo->query('SELECT 1'); } catch (PDOException $e) { self::initPdo(); } }
