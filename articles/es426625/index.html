Árboles multilingües en Yii2 como ejemplo de creación de un módulo de menú

Entrada


Muchos desarrolladores web novatos se enfrentan a la necesidad de crear menús, directorios o categorías para su proyecto Yii2, que tendría una estructura jerárquica, pero al mismo tiempo admitiría el multilingüismo. La tarea es bastante simple, pero no del todo obvia dentro del marco de este marco. Hay una gran cantidad de extensiones listas para crear estructuras de árbol (menús, directorios, etc.), pero es bastante difícil encontrar una solución que admita el trabajo completo con varios idiomas. Además, no estamos hablando de traducir la interfaz utilizando herramientas marco estándar, sino de almacenar datos en una base de datos en varios idiomas. También es bastante difícil encontrar un widget conveniente y totalmente funcional para administrar el árbol, que también podría funcionar con contenido multilingüe sin manipulaciones de código complicadas.


Me gustaría compartir una receta sobre cómo crear dichos módulos utilizando el ejemplo de implementación del módulo de menú. Por ejemplo, utilizaré la plantilla de aplicación Yii2 App Basic, pero puede adaptar todo a su plantilla si difiere de la base.


Preparación


Para llevar a cabo la tarea, necesitaremos algunas extensiones maravillosas, a saber:


  • Lista de adyacencia : para almacenar una estructura de árbol
    menú en la base de datos;
  • Yii2 Bootstrap Treeview - widget
    para una visualización conveniente del menú en forma de árbol;
  • Comportamiento traducible - comportamiento para
    soporte multilingüe en modelos;

Instale los datos de extensión a través del compositor: 


composer require paulzi/yii2-adjacency-list composer require execut/yii2-widget-bootstraptreeview composer require creocoder/yii2-translateable

Para implementar el menú como un módulo, utilizando el generador Gii (o manualmente) cree un nuevo módulo de menú y conéctelo en la configuración de la aplicación.


El proyecto también debe tener un mecanismo de cambio de idioma configurado. Prefiero usar esta extensión para Yii2 .


Creación de modelos


Para almacenar el menú (u otra entidad que tenga multilingüismo) en la base de datos, necesitamos crear dos tablas. De hecho, se pueden usar diferentes métodos para almacenar datos multilingües, pero me gusta la opción con dos tablas, una de las cuales almacena la esencia en sí, y la segunda, sus variaciones de idioma, más que las otras. Es conveniente usar migraciones para crear tablas. Aquí hay un ejemplo de tal migración:

Después de crear las tablas necesarias y agregarles un nuevo menú mediante la migración, necesitamos crear modelos. Dado que el multilingüismo y los árboles se pueden encontrar no solo en el menú, sino también en otras entidades (por ejemplo, la página del sitio) en el proyecto, sugiero que los métodos que implementan el mecanismo de multilingüismo y la organización del árbol se pongan en rasgos separados para que podamos usarlos fácilmente más adelante ellos en otros modelos sin duplicación de código. Cree una carpeta de rasgos en la raíz de la aplicación (si aún no está allí) y coloque dos archivos allí:


LangTrait.php 
 <?php namespace app\traits; use Yii; use yii\behaviors\SluggableBehavior; use creocoder\translateable\TranslateableBehavior; trait LangTrait { public static function langClass() { return self::class . 'Lang'; } public static function langTableName() { return self::tableName() . '_lang'; } public function langBehaviors($translationAttributes) { return [ 'translateable' => [ 'class' => TranslateableBehavior::class, 'translationAttributes' => $translationAttributes, 'translationRelation' => 'translations', 'translationLanguageAttribute' => 'language', ], ]; } public function transactions() { return [ self::SCENARIO_DEFAULT => self::OP_INSERT | self::OP_UPDATE, ]; } public function getLang() { return $this->hasOne(self::langClass(), ['owner_id' => 'id'])->where([self::langTableName() . Ahora en el módulo, cree el archivo /modules/menu/controllers/AdminController.php. Este será el controlador principal para administrar el menú, y dado que implementa el árbol y el multilingüismo, heredará de la base que ya creamos en el paso anterior:


AdminController.php 
 <?php namespace app\modules\menu\controllers; use app\base\controllers\AdminLangTreeController as BaseController; class AdminController extends BaseController { public $modelClass = \app\modules\menu\models\Menu::class; public $modelClassSearch = \app\modules\menu\models\MenuSearch::class; public $modelName = 'Menu'; public $modelNameLang = 'MenuLang'; }

Como puede ver, el código de este controlador contiene solo los nombres de los modelos y sus clases. Es decir, para crear controladores CRUD para otros módulos (catálogo, rubricator, etc.), que también utilizarán el árbol y el multilingüismo, puede hacer lo mismo de la misma manera: expanda el controlador base.


Crear una interfaz para administrar el menú


La etapa final es la creación de una interfaz para administrar un árbol multilingüe. La extensión Bootstrap Treeview maneja la tarea de mostrar un árbol, que se puede configurar de manera bastante flexible y admite muchas funciones convenientes (por ejemplo, buscar en un árbol). Conclusión


Como resultado, deberíamos obtener lo siguiente:


  • Un módulo listo para usar para un menú multilingüe en forma de árbol de un sitio con una interfaz conveniente y personalizable;
  • Un controlador CRUD básico que se puede heredar al crear otros módulos que usan madera y multilingüismo;
  • Dos rasgos (multilenguaje y árbol) que pueden conectarse a modelos para implementar las funciones correspondientes.
