Ulasan SSD Solid State untuk Pengguna Perusahaan Kingston DC500R

Kingston baru-baru ini merilis solid state drive Kingston DC500R kelas perusahaan, yang dirancang untuk beban konstan tinggi. Sekarang banyak jurnalis secara aktif menguji produk baru dan merilis materi yang menarik. Kami ingin berbagi dengan Habr salah satu ulasan terperinci dari Kingston DC500R, yang pengujiannya akan menyenangkan pembaca. Yang asli ada di situs web Storagereview dan diterbitkan dalam bahasa Inggris. Untuk kenyamanan Anda, kami telah menerjemahkan materi ke dalam bahasa Rusia dan menempatkannya di bawah potongan. Selamat membaca!



Drive Kingston DC500R didasarkan pada teknologi memori flash TLC NAND 3D. Opsi tersedia dalam kapasitas 480 GB, 960 GB, 1,92 TB dan 3,84 TB, yang menyediakan pilihan tambahan untuk perusahaan yang ingin menghemat uang, atau bagi mereka yang tidak membutuhkan drive berkapasitas tinggi. Tinjauan ini membahas versi 3,48 TB dengan kecepatan baca dan tulis berurutan yang dinyatakan masing-masing sebesar 555 MB / s dan 520 MB / s, dan kecepatan baca dan tulis blok 4 KB pada beban konstan 98.000 dan 28.000 operasi input -Output per detik (IOPS), masing-masing. Sebagai bagian dari rangkaian produk ini, Kingston juga menawarkan DC500M, yang dioptimalkan untuk aplikasi serba guna.

Menguji Informasi

Laboratorium uji tingkat perusahaan StorageReview memberikan banyak peluang untuk menguji perangkat penyimpanan di lingkungan yang dekat dengan kondisi nyata. Laboratorium ini mencakup berbagai server, perangkat jaringan, sistem daya, dan infrastruktur jaringan lainnya. Ini memungkinkan karyawan kami untuk membuat kondisi yang mendekati kondisi sebenarnya untuk penilaian kinerja peralatan yang akurat.
Informasi tentang lingkungan dan protokol dimasukkan dalam ulasan sehingga profesional TI dan mereka yang bertanggung jawab untuk pembelian sistem penyimpanan dapat mengevaluasi kondisi di mana hasil yang sesuai dicapai. Produsen alat uji tidak membayar untuk ulasan dan tidak mengendalikannya.

Analisis Beban Kerja Aplikasi

Untuk mengevaluasi dengan benar parameter kinerja perangkat penyimpanan kelas perusahaan, penting untuk memodelkan infrastruktur dan beban kerja aplikasi yang sesuai untuk lingkungan dunia nyata. Oleh karena itu, untuk mengevaluasi Samsung 883 DCT SSD, kami mengukur kinerja database MySQL OLTP menggunakan utilitas SysBench dan kinerja database Microsoft SQL Server OLTP dengan meniru beban kerja TCP-C. Dalam hal ini, untuk aplikasi, setiap drive akan memproses dari 2 hingga 4 mesin virtual yang dikonfigurasi secara sama.

Kinerja SQL Server

Dua disk virtual dikonfigurasi untuk setiap mesin virtual SQL Server: disk boot dengan kapasitas 100 GB, serta disk dengan kapasitas 500 GB yang ditujukan untuk hosting database dan file log. Dalam hal sumber daya sistem, setiap mesin virtual dilengkapi dengan 16 prosesor virtual, DRAM 64 GB, dan pengontrol LSI Logic SAS SCSI. Sebelumnya, dengan beban kerja Sysbench, kami menguji kecepatan I / O dan pemanfaatan kapasitas drive. Tes SQL, pada gilirannya, membantu mengevaluasi latensi.

Sebagai bagian dari pengujian, SQL Server 2014 digunakan untuk mesin virtual tamu yang menjalankan Windows Server 2012 R2. Banyak beban dibuat menggunakan perangkat lunak Benchmark Factory Quest for Databases. StorageReview's Microsoft SQL Server OLTP Database Testing Protocol menggunakan versi saat ini dari Dewan Kinerja Pemrosesan Transaksi yang Benchmark C Program Penilaian Kinerja (TPC-C). Program evaluasi kinerja pemrosesan transaksi waktu-nyata ini mensimulasikan proses lingkungan aplikasi yang kompleks. Pengujian TPC-C memungkinkan Anda untuk secara lebih akurat menentukan kekuatan dan kelemahan infrastruktur penyimpanan di lingkungan basis data, daripada pengujian kinerja buatan. Sebagai bagian dari pengujian kami, setiap mesin virtual SQL Server bekerja dengan database SQL Server 333 GB (skala 1500). Pengukuran kinerja dan latensi selama pemrosesan transaksi dilakukan di bawah beban 15.000 pengguna virtual.

Konfigurasi tes SQL Server (untuk setiap VM):
• Windows Server 2012 R2
• Ruang disk: dialokasikan 600 GB, digunakan 500 GB
• SQL Server 2014
- Ukuran basis data: skala 1.500
- Jumlah klien virtual: 15.000
- Buffer memori RAM: 48 GB
• Durasi tes: 3 jam
- 2,5 jam - tahap awal
- 30 menit - pengujian langsung

Berdasarkan kinerja pemrosesan transaksi SQL Server, DC500R Kingston hanya sedikit di belakang Samsung 883 DCT, menunjukkan kinerja total 6.290,6 transaksi per detik (TPS).



Bahkan lebih baik daripada metrik TPS, mengevaluasi kinerja SQL Server adalah ukuran latensi. Di sini, kedua drive - Samsung 860 DCT dan Kingston DC500R - menunjukkan waktu yang sama: 26,5 ms.



Performa Menggunakan Sysbench

Tes selanjutnya menggunakan database MySQL Percona . Kinerja OLTP dievaluasi menggunakan utilitas SysBench. Dalam hal ini, TPS rata-rata dan penundaan diukur, serta waktu tunda rata-rata dalam skenario yang paling tidak menguntungkan.

Setiap mesin virtual Sysbench menggunakan tiga disk virtual: disk boot sekitar 92 GB, disk dengan basis data pra-instal sekitar 447 GB, dan disk dengan basis data uji 270 GB. Dalam hal sumber daya sistem, setiap mesin virtual dilengkapi dengan 16 prosesor virtual, DRAM 60 GB, dan pengontrol LSI Logic SAS SCSI.

Konfigurasi tes Sysbench (untuk setiap VM):

• CentOS 6.3 64-bit
• Percona XtraDB 5.5.30-rel30.1
- Jumlah tabel database: 100
- Ukuran basis data: 10.000.000
- Jumlah utas basis data: 32
- Buffer memori RAM: 24 GB
• Durasi tes: 3 jam
- 2 jam - tahap awal, 32 utas
- 1 jam - pengujian langsung, 32 utas

Berdasarkan peringkat kinerja pemrosesan transaksi Sysbench, DC500R berada di belakang kompetisi, memberikan kinerja 1.680,47 transaksi per detik.



Dalam hal latensi rata-rata, DC500R juga berada di peringkat terakhir dalam peringkat dengan angka 76,2 ms.



Akhirnya, setelah menguji waktu tunda dalam skenario kasus terburuk (persentil ke-99), DC500R kembali berada di akhir daftar dengan hasil 134,9 ms.



Analisis beban kerja VDBench

Saat menguji perangkat penyimpanan, pengujian aplikasi lebih disukai daripada tes sintetis. Namun, meskipun hasilnya tidak sesuai dengan kondisi nyata, tes sintetis karena pengulangan tugas nyaman untuk menentukan indikator dasar dan membandingkan solusi yang bersaing. Tes ini menawarkan berbagai profil, dari empat tes sudut dan sampel uji migrasi database hingga pelacakan tangkapan dari berbagai lingkungan VDI. Dalam semua kasus ini, generator beban kerja vdBench tunggal dengan prosesor skrip digunakan untuk mengotomatisasi dan mengumpulkan hasil dalam sekelompok besar tes komputasi. Ini memungkinkan untuk menggunakan beban kerja yang sama untuk berbagai drive, termasuk flash array dan drive individual. Sebagai bagian dari pengujian, kami mengisi drive sepenuhnya dengan data, dan kemudian membaginya menjadi partisi dengan kapasitas 25% dari aslinya, untuk mensimulasikan beban aplikasi dan mengevaluasi perilaku drive. Pendekatan ini berbeda dari tes entropi sepenuhnya, di mana, di bawah beban konstan, seluruh disk segera digunakan. Karena alasan ini, hasil berikut mencerminkan kecepatan perekaman yang lebih stabil.

Profil:
• Baca 4k acak: hanya baca, 128 utas, kecepatan input / output dari 0 hingga 120%
• Tulis 4K acak: hanya menulis, 64 stream, kecepatan input / output dari 0 hingga 120%
• Pembacaan berurutan 64 KB: hanya baca, 128 utas, kecepatan input / output dari 0 hingga 120%
• Penulisan berurutan 64 KB: hanya menulis, 64 stream, kecepatan input / output dari 0 hingga 120%
• Basis data sintetik: SQL dan Oracle
• VDI Salin (salinan lengkap dan buat salinan tertaut)

Dalam tes beban kerja VDBench pertama (pembacaan acak 4K), Kingston DC500R menunjukkan hasil yang mengesankan: penundaan 1 ms hingga mencapai tingkat kecepatan 80.000 IOPS dan kecepatan maksimum 80.209 IOPS dengan penundaan 1,59 ms.



Semua drive yang diuji menunjukkan hasil yang hampir sama pada tes kedua (Random Write 4 KB): kecepatannya sedikit lebih tinggi dari 63.000 IOPS dengan penundaan 2 ms.



Pindah ke beban berurutan, kami pertama kali melihat 64K dibaca. Dalam hal ini, drive Kingston mempertahankan penundaan dalam milidetik hingga mencapai 5200 IOPS (325 MB / s). Angka maksimum 7183 IOPS (449 MB / s) dengan penundaan 2,22 ms membawa disk ini ke tempat kedua dalam keseluruhan klasemen.



Saat menguji operasi perekaman berurutan, perangkat Kingston mengungguli semua pesaing dengan menahan penundaan di bawah 1 ms hingga kecepatan 5700 IOPS (356 MB / s). Kecepatan maksimum adalah 6291 IOPS (395 MB / s) dengan penundaan 2,51 ms.



Setelah itu, kami beralih ke tugas SQL, di mana DC500R Kingston adalah satu-satunya perangkat yang tingkat keterlambatannya melampaui milidetik dalam ketiga tes. Dalam kasus pertama, disk menunjukkan kecepatan maksimum 26411 IOPS dengan penundaan 1,2 ms.



Dalam uji SQL 90-10, Kingston mengambil tempat terakhir dengan kecepatan maksimum 27339 IOPS dengan penundaan 1,17 ms.



Hal yang sama terjadi pada tes SQL 80-20. Perangkat Kingston dalam hal ini menunjukkan kecepatan maksimum 29576 IOPS dengan penundaan 1,08 ms.



Hasil uji beban kerja Oracle menempatkan DC500R kembali di tempat terakhir, tetapi dalam dua tes perangkat masih menunjukkan keterlambatan dalam milidetik. Dalam kasus pertama, kecepatan disk maksimum Kingston adalah 29098 IOPS dengan penundaan 1,18 ms.



Dalam tes kedua (Oracle 90-10), DC500R mencapai 24555 IOPS dengan penundaan 894,3 μs.



Pada pengujian ketiga (Oracle 80-20), kecepatan maksimum perangkat dari Kingston adalah 26401 IOPS dengan tingkat penundaan 831,9 μs.



Kemudian kami beralih ke menyalin VDI - membuat salinan lengkap dan tertaut. Dalam menguji unduhan salinan lengkap VDI, drive Kingston kembali gagal mengungguli pesaingnya. Perangkat mendukung penundaan di bawah 1 ms hingga kecepatan sekitar 12000 IOPS, dan kecepatan maksimum adalah 16203 IOPS dengan penundaan 2,14 ms.



Saat menguji salinan Initial Login VDI, perangkat Kingston berkinerja lebih baik, akhirnya mengambil (dengan sedikit margin) tempat kedua. Drive mendukung penundaan dalam milidetik hingga kecepatan 11000 IOPS tercapai, dan kecepatan maksimum adalah 13652 IOPS dengan penundaan 2,18 ms.



Juga dengan sedikit margin, drive Kingston menempati posisi kedua pada hasil tes Senin Login untuk salinan lengkap VDI. Hard disk Seagate Nytro 1351 menunjukkan kecepatan maksimum yang sedikit lebih tinggi, tetapi perangkat Kingston umumnya menunjukkan tingkat latensi yang lebih rendah selama pengujian. Kecepatan maksimum DC500R adalah 11897 IOPS dengan penundaan 1,31 ms.



Dalam menguji unduhan salinan tertaut VDI, Kingston berada di posisi terakhir. Penundaan melampaui 1 ms sudah pada kecepatan kurang dari 6000 IOPS. Kecepatan maksimum DC500R adalah 7861 IOPS dengan penundaan 2,03 ms.



Namun, berdasarkan hasil uji Login Awal, drive kembali menempati posisi kedua: keterlambatan melampaui batas milidetik hanya setelah mencapai kinerja puncak, yang akhirnya berjumlah 7950 IOPS dengan penundaan 1,001 ms.



Dalam tes terakhir dari salinan VDI yang terhubung - Monday Login - disk juga menunjukkan hasil kedua: kecepatan maksimum pada 9205 IOPS dengan penundaan 1,72 ms. Di luar milidetik, penundaan melebihi ketika kecepatan 6400 IOPS tercapai.



Kesimpulan


DC500R adalah solid state drive terbaru Kingston untuk pengguna perusahaan. DC500R tersedia dalam form factor 2,5 inci. Kapasitas yang tersedia mulai dari 480 GB hingga 3,84 TB. Drive ini didasarkan pada teknologi memori flash TLC NAND 3D dan menggabungkan sumber daya yang besar dan tingkat kinerja yang tinggi. Untuk disk 3,48 TB, kecepatan baca dan tulis berurutan masing-masing sebesar 555 dan 520 MB / s, masing-masing, membaca dan menulis dengan beban konstan masing-masing 98.000 dan 28.000 IOPS, dan kapasitas sumber daya 3504 TBW diklaim.

Untuk mengevaluasi kinerja Kingston DC500R, kami membandingkannya dengan SSD SATA populer lainnya, termasuk drive Samsung 860 DCT dan 883 DCT , serta drive Seagate Nytro 3530 . Kington DC500R mampu bertahan di level pesaing, dan dalam beberapa kasus bahkan melampaui mereka. Saat menguji beban kerja aplikasi, Kingston DC500R berkinerja cukup baik saat memproses tugas SQL, mengambil tempat kedua secara keseluruhan dalam hal transaksi per detik (6291,8 TPS) dan latensi (26,5 ms). Dalam tes Sysbench dengan kinerja penulisan yang lebih intens, DC500R berakhir di akhir daftar dengan kinerja 1.680,5 TPS dengan latensi rata-rata 76,2 ms dan latensi skenario paling tidak menguntungkan di 134,9 ms.

Sebagai bagian dari pengujian blok baca dan tulis acak 4 KB, Kingston DC500R menunjukkan kecepatan 80209 IOPS dan penundaan 1,59 ms saat membaca, serta kecepatan 63000 IOPS dan penundaan 2 ms saat menulis. Sebagai bagian dari uji baca dan tulis 64 KB, DC500R mencapai kecepatan 7183 IOPS (449 MB / s) dengan penundaan 2,22 ms dan 6291 IOPS (395 MB / s) dengan penundaan masing-masing 2,51 ms. Dalam konteks pengujian sintetik menggunakan database SQL dan Oracle dan peningkatan persyaratan kecepatan tulis, kinerja DC500R buruk. Dalam hal beban kerja SQL, Kingston DC500R mengambil tempat terakhir di ketiga tes dan ternyata menjadi satu-satunya pengandar yang tingkat keterlambatannya melampaui milidetik. Namun, dalam pengujian Oracle gambarnya jauh lebih baik. Dalam dua dari tiga tes, drive mendukung penundaan kurang dari 1 ms, yang menjadikannya tempat kedua. Kingston DC500R menunjukkan tingkat kinerja yang layak ketika menguji menggunakan salinan VDI - baik penuh dan terkait.

Secara umum, Kingston DC500R SSD adalah perangkat berkualitas tinggi di kelasnya yang patut mendapat perhatian lebih. Tidak peduli seberapa besar kita menyukai teknologi berkinerja tinggi (NVMe dan analog), itu adalah drive SATA yang tetap menjadi solusi yang paling disukai untuk memproses tugas-tugas di mana keandalan memainkan peran penting, misalnya, memuat server atau pengontrol penyimpanan. Drive tersebut juga merupakan solusi ekonomis untuk menyimpan data server dalam situasi di mana nilai uang sangat penting. Selain itu, mereka menawarkan semua manfaat dalam hal total biaya kepemilikan yang solid state drive menonjol dari latar belakang hard drive. Kinerja DC500R memungkinkan perangkat untuk menempati baris teratas sesuai dengan hasil dari banyak pengujian yang kami lakukan dibandingkan dengan drive yang layak lainnya. DC500R adalah drive SATA yang sangat baik untuk skenario yang membutuhkan drive yang andal dan produktif dengan kapasitas sumber daya yang besar dan berbagai pilihan kapasitas.

Model Seri DC500 tersedia dari distributor resmi Kingston.
