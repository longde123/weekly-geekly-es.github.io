开源基金在做什么-谈论最近的OpenStack和Linux Foundation项目

我们决定谈论最近加入这两个大型基金的项目（Kata Containers，Zuul，FATE和CommunityBridge），以及它们的发展方向。


照片-Alex Holyoake-不飞溅

OpenStack Foundation的情况如何


OpenStack Foundation（OSF）成立于2012年，旨在支持 OpenStack开放云平台的开发。 而且，该组织很快与其社区“杂草丛生”。 迄今为止，OpenStack基金会已有 500多个成员。 其中包括电信，云提供商，硬件制造商，甚至是域名注册商。

长期以来，OpenStack基金会一直致力于其同名项目的开发。 但是在今年年初，该基金改变了它的计算方式 。 该组织开始支持与机器学习，CI / CD，外围计算和容器化有关的项目。

在这方面，有几个新项目加入了该基金。

什么样的项目。 在5月举行的开放基础架构峰会上，OSF代表谈到了第一个“新来者” ，即 Kata Containers和Zuul 。

第一个项目是开发安全虚拟机，其性能可与Kubernetes和Docker容器相媲美。 VM的加载速度不超过100毫秒，因此它们将在云中找到用于动态部署计算资源的应用程序。 顺便说一下，几家大型IaaS提供商已经参与了Kata的开发。

第二个项目Zuul是CI / CD系统。 它对代码修改进行并行测试，并防止潜在的崩溃。

基金的前景。 OpenStack基金会表示，通过改变发展方向，他们将能够与有才华的开发人员一起加强社区。 但是，并非所有人都这么认为-在5月的会议上，Canonical创始人马克·沙特尔沃思（ Mark Shuttleworth） 称扩大基金的投资组合是“错误的”。 他认为，OpenStack基金会对资源的利用效率低下，最终将影响其主要产品OpenStack云平台的质量。 将来是否会如此仍然有待观察。

Linux Foundation的作用


该基金会致力于 Linux 的推广和标准化，以及整个开源软件生态系统的开发。 该基金的投资组合会定期更新以新项目-其中一些确实在本周出现。

什么样的项目。 6月25日，FATE框架成为 Linux Foundation的一部分。 它已转移到开源中国银行WeBank和腾讯。 新解决方案的目标是帮助公司开发符合GDPR要求的安全人工智能系统。 它包括用于实施深度学习， 逻辑回归和“ 转移学习 ”方法的工具（在这种情况下，使用已受过训练的模型，适用于解决其他问题）。 该项目的源代码可以在GitHub上找到 。


照片-Cassidy Mills-不飞溅

同样在今年年初，Linux基金会宣布了CommunityBridge平台。 它充当准备赞助开放项目的开发商和投资者之间的桥梁。 该平台应有助于吸引新的开发人员开源。

尽管如此，她已经受到批评。 业内专家指出 ，Linux基金会将仅提供最少的财务服务，而合同和许可等问题仍然存在。 也许将来，CommunityBridge的功能将得到扩展。

基金的前景。 去年下半年 ，Linux基金会为GraphQL和Ceph建立了两个新的基金会。 该组织计划继续开发开源软件生态系统。

例如，Linux Foundation和Facebook 计划开设一个专门用于osquery项目的新基金。 Osquery是社交网络开发人员以及Airbnb，Netflix和Uber使用的操作系统监视框架。 该工具使您可以优化获取有关正在运行的进程，已加载的内核模块和网络连接的数据的过程。

可以预见，在不久的将来Linux基金会将再次扩大其产品组合。 也许他们在等待成功的Cloud Native Computing Foundation的命运，Kubernetes和CoreDNS就是从该基金会诞生的。 也许他们会跟随Tizen基金的脚步，由于同名操作系统的不受欢迎 ，其前景仍然不清楚。
