Raspberry Pi 3B +和WebMeeting更新上的3CX v16测试

在Raspberry Pi 3B +上测试Micro PBX


当然，当我们开发企业IP-PBX时，为大中型企业增加更多功能是很有趣的。 但是，您必须承认，强壮的肌肉只能在强壮的骨骼上运作良好。 产品架构的质量比在线报告中的漂亮颜色更为重要。 这种质量仅在硬件功能最少的系统上体现出来。

我们决定找出3CX v16对硬件的要求不高，并在Raspberry Pi上启动了它。 该系统已通过各种测试，我们急于分享其结果。



v16 Raspberry Pi的安装非常简单，本指南对此进行了介绍。 如前所述，您将需要：

  • 设备-Raspberry Pi 3B +
  • 该存储卡为32 + GB Class 10 Micro SD，访问速度至少为10 Mb / s。
  • 该外壳与Raspberry Pi兼容，最好与散热器和风扇冷却兼容。
  • 电源-与Raspberry Pi，2.5 A，Micro USB兼容。

例如，在Aliexpress上以54美元的价格提供类似的工具包



为什么我们推荐风扇套件？ 事实证明，在负载适当的情况下，PBX的长期运行会大大加热设备，尤其是在使用“大量”编解码器的情况下。 因此，如果您的组织希望PBX能够正常工作，或者许多使用G729和OPUS编解码器的移动用户可以正常工作，请确保安装散热器和/或风扇。

尽管起初我们担心Raspberry Pi不会“拉”出或多或少的严重安装，但是3CX对于多达32位用户的系统显示出了出色的效果。



  • 处理器负载从无动作的4％到峰值的46％，平均负载在23.7％左右。 这表明优化的3CX代码。
  • 内存消耗-从空闲时的32％到负载下的38％。 再次确认了代码的卓越品质。
  • 网络负载-Pi 3B +具有改进的千兆位以太网控制器，有效速度约为224 Mbps。 具有8个同时呼叫的平均流约为1.5 Mbit / s，即 负载小于1％。 当然，很大程度上取决于您的网络质量和所使用的语音编解码器。
  • 功耗-如果X86-64架构上的普通服务器每天消耗大约20.4 kWh，则Raspberry Pi每年消耗相同的功率！
  • 温度模式-在不使用散热器或风扇的情况下，Pi温度传感器在闲置时显示为47.2°C，在满载时显示为59.5°C。 平均工作温度为52.5°C。

在哪里可以使用PBX，价格为54美元


经过测试，我们可以说，适当的冷却，这个婴儿将使您持续多年。 当前，推荐的用例是具有8个同时呼叫和16-32个用户的系统。 在每个用户的IP电话上，平均最多可以配置8个BLF指示灯（它们的数量会影响3CX SIP堆栈和内置Web服务器上的负载）。 想象一下Raspberry Pi +上的3CX可以派上用场的地方：

  • 一家小公司的企业级电话交换机，它不仅允许员工在世界任何地方彼此通信，而且还允许他们的家人通信。
  • 一家大公司的分支机构，每个分支机构都需要一个成熟的本地电话交换机，以防与父级电话交换机的连接中断。
  • 住宅建筑物的ATS，其居民可以使用iOS和Android应用程序，并将呼叫面板（SIP-intercom）连接到系统。
  • 教育机构的自动电话交换，用于员工，教师甚至学生学习之间的通信。
  • 小旅馆，提供旅馆电话交换机的有趣功能。

红利 如果出现问题，您可以快速将PBX放进口袋中并随身携带！

3CX WebMeeting更新-现在快50倍


改善会议机会


3CX WebMeeting网络会议平台的最新更新提供了新的会议功能，增强的语音质量和其他MCU服务器，为您提供方便。

现在，会议活动挂图（白板）可以根据压力程度提供更平滑，更准确的绘图。 带有触摸屏的平板电脑和笔记本电脑的用户会喜欢此功能。 另外，当您将鼠标悬停在某个对象上时，例如，在会议参与者面板中，该对象将突出显示-方便。



信号强度从WebMeeting界面消失了。 当通讯频道的速度过慢导致视频质量下降时，显示的小通知将代替它。 通过通知，您可以了解问题的原因并采取措施。 如果您对有关连接质量的更多详细信息感兴趣，请转到3CX WebMeeting部分的“设置”>“网络”以查看高级统计信息。



改善服务质量


现在，我们每个月大约举行100,000个网络会议，有150,000人参加。 为了扩大服务范围并提高可靠性，我们将3CX WebMeeting服务器移至Google和Amazon的基础架构。

短时间存储的会议数据（记录，统计数据）放置在欧洲数据中心Google中。 这提供了可靠性，安全性和其他技术优势。 网络会议服务器（MCU）本身托管在全球Amazon的基础架构中。 这样可以确保与最终用户的距离和出色的性能。 MCU WebMeeting当前位于澳大利亚，巴西，加拿大，法国，德国，俄罗斯，南非，新加坡，英国，美国，越南和印度。 无论您身在何处，3CX都会自动尝试提供离您最近的服务器。

语音会议参与者的OPUS编解码器


WebMeeting普及的客观原因之一是其出色的音质。 在最新的更新中，我们为仅通过语音参加会议的用户连接了自适应的高质量OPUS编解码器。 以前使用的是G.722编解码器，在移动网络上无法很好地工作。 当然，如果与会者通过IP电话连接，则支持其他语音编解码器。

记录和报告速度提高50倍


在此更新中，我们还加速了视频会议录像的处理和最终报告的生成。 我们的统计数据显示，在完成大约50分钟之后，还会有4位参与者使用音频和视频进行为时1小时的会议。 只有在此之后，会议记录的链接（.mp4文件）才会出现在组织者的电子邮件中。 现在，后处理需要1-3分钟。 取决于安装的选件和视频质量。
