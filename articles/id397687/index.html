Pada beberapa laptop dengan Windows 10 Anda tidak dapat mengirimkan Linux



Microsoft secara resmi menyatakan kebijakan "cinta" untuk Linux dan perangkat lunak bebas. Perusahaan secara berkala menerbitkan kode sumber programnya, telah menerapkan subsistem Linux di kernel Windows . Microsoft memiliki catatan jumlah asisten di Github, dalam indikator ini berada di tempat pertama , di depan Facebook, Docker dan Google.

Namun, untuk membersihkan reputasi Anda dan memulai hidup dari awal tidak begitu sederhana. Dari ingatan orang-orang, ingatan akan tindakan mengerikan perusahaan pada masa ketika Steve Ballmer berada di pucuk pimpinan tidak terhapus . Dan sekarang, tidak semuanya lancar: Microsoft terus memeras biaya lisensi dari produsen perangkat untuk Android , menghadirkan paten "meragukan" untuk Linux”, Memaksa produsen untuk menginstal perangkat lunak berpemilik dari Microsoft. Korban terakhir dari kebijakan semacam itu sebulan yang lalu adalah Lenovo , yang berjanji untuk melakukan pra-instal program Microsoft Office, OneDrive dan Skype pada ponsel cerdas.

Tidak mengherankan, dalam situasi ini, beberapa pengguna mencurigai Microsoft "tidak adil bermain."



Baru-baru ini , muncul rumor di forum-forum bahwa beberapa model komputer pribadi dengan sistem operasi Windows 10 yang sudah diinstal sebelumnya tidak begitu bersahabat dengan Linux. Secara khusus, tidak mungkin untuk menginstal OS gratis di sejumlah laptop Lenovo yang datang dengan beberapa versi Windows 10 Signature Edition. Menurut asumsi pengguna, penunjukan "Edisi Tanda Tangan" berarti bahwa Microsoft memiliki perjanjian dengan pabrikan mengenai komputer ini bahwa komputer dikunci ke sistem operasi Windows.

Ternyata, misalnya, bahwa tidak mungkin untuk menginstal Linux dalam mode RAID eksklusif, yang dikunci melalui UEFI / BIOS, pada laptop Lenovo Yoga 900 ISK2 UltraBook. Linux tidak melihat SSD ini.

Masalah serupa menginstal Linuxtelah muncul dari pembeli laptop Ideapad 710S .


Tangkapan layar dari forum di mana pengguna mengeluh tentang ketidakmungkinan menginstal Linux pada laptop Lenovo

Ini adalah masalah yang diketahui : pada platform Intel, drive dikonfigurasi dalam UEFI / BIOS dalam dua cara: sebagai "standar" atau sebagai RAID. Dalam kasus kedua, ID PCI hanya diubah untuk menonaktifkan driver standar dan menjamin penggunaan driver Intel tertentu, yang pasti memiliki dukungan RAID. Intel tidak menyiapkan komitmen terhadap kernel Linux untuk mendukung mode RAID untuk driver ini, oleh karena itu, dengan pengaturan UEFI / BIOS, sistem operasi Linux “tidak melihat” drive.

Dalam hal ini, laptop Lenovo tidak memiliki kemampuan untuk mengubah konfigurasi drive, itu dipasang dengan kuat sebagai RAID.

Pengguna yang tidak puas mengeluh kepada perwakilan Lenovo. Dia menjelaskan bahwa dalam kasus ini, laptop memiliki versi Signature Edition dari sistem operasi Windows 10 Home yang diinstal. "Terkunci oleh perjanjian kami dengan Microsoft," kata karyawan itu.

Tidak jelas apa artinya ini. Mungkin saja tidak ada yang mau menindas sistem operasi Linux. Mungkin pengaturan hard RAID disebabkan oleh kebutuhan untuk menggunakan driver dari Intel, dan bukan driver dari Microsoft. Kemungkinan besar, karena peralatan Intel terbaru memerlukan pengaturan khusus untuk manajemen daya , dan driver standar dari Microsoft tidak menyediakan ini.

Namun, masalahnya ada secara de facto. Anda dapat menginstal Linux di laptop ini, tetapi Linux tidak akan dapat bekerja dengan SSD. Pada saat yang sama, pertanyaannya tetap apa arti sebenarnya “Signature Edition” dari sistem operasi Windows 10 Home. Menurut definisi Microsoft sendiri, ini adalah komputer tanpa bloatware pra-instal yang menurunkan kinerja PC, dengan Windows Defender diaktifkan secara default, dengan keyboard berkualitas tinggi, dengan touchpad yang memenuhi standar Microsoft dan peralatan lainnya, yang sesuai dengan standar sertifikasi perangkat keras Windows.

Jika Anda yakin karyawan Lenovo, maka Anda perlu menambahkan "kunci" ke sistem operasi Windows ke daftar ini, tetapi persyaratan seperti itu tidak dinyatakan secara resmi.

Microsoft Store saat ini memiliki daftar desktop, laptop, dan tablet dalam kategori “Signature Edition”.

Tentu saja, Lenovo dapat memberi pengguna opsi untuk memilih pengaturan di UEFI / BIOS, tetapi karena potensi masalah yang disebutkan dengan manajemen daya, ini akan menimbulkan pertanyaan yang tidak perlu terhadap dukungan teknis Lenovo. Kemungkinan besar, manajemen perusahaan Cina memutuskan bahwa jumlah pengguna Linux sangat kecil sehingga mereka dapat diabaikan untuk mengurangi jumlah panggilan ke dukungan teknis dari pengguna biasa.

menurut pendapatMatthew Garrett, pengembang subsistem manajemen daya Linux, subsistem keamanan CoreOS dan anggota Free Software Foundation, masalah utama adalah bahwa Intel tidak membuat upaya yang cukup untuk memastikan bahwa OS gratis bekerja dengan baik pada versi terbaru perangkat kerasnya - para pengembang kekurangan informasi dari Intel tentang cara menyediakan manajemen daya yang paling efisien, tidak ada dukungan untuk perangkat dalam mode RAID dan tidak ada kepastian bahwa situasi akan membaik di masa mendatang. Jika Intel memberikan pengembang semua informasi yang diperlukan, maka masalah dengan laptop Lenovo tidak akan pernah muncul sama sekali, kata Matthew Garrett.
