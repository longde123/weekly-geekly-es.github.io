PHP internos


  • [RFC] Cambiar la precedencia del operador de concatenación : propuesta aceptada para reducir la prioridad del operador de concatenación . . El comportamiento se cambiará en PHP 8, y en PHP 7.4 se lanzará una advertencia al usar expresiones con . antes + o - sin paréntesis. 
     echo "sum: " . $a + $b; //   echo ("sum: " . $a) + $b; //   PHP 8 echo "sum :" . ($a + $b);
  • [RFC] Separador literal numérico : se propone agregar la posibilidad de usar el símbolo _ como separador en números, para aumentar la legibilidad: 
     $i = 197_823_459; // 197823459
    La propuesta ya había fallado en la votación hace dos años, pero desde entonces se ha agregado una característica similar en Python, JavaScript y TypeScript.
