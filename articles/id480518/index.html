Tutor pemrograman: mengapa Anda harus menjadi guru dan bagaimana melakukannya

Selamat siang, Habr. Saya mempersembahkan kepada Anda terjemahan artikel: "Mentor Pengodean: Mengapa Anda Harus Menjadi Satu dan Cara Melakukannya" oleh Oleg Sklyarov (Oleg Sklyarov).



Hai Nama saya Oleg Sklyarov, saya bekerja sebagai pemimpin kelompok untuk pengembangan perangkat seluler untuk anak-anak di Skyeng . Di waktu luang saya, saya bekerja sebagai tutor untuk siswa IT. Itu adalah pengalaman hebat bagi saya, jadi saya ingin berbagi cerita dan ide-ide yang saya terima selama kelas-kelas ini.



Bagaimana saya menjadi seorang mentor


Beberapa tahun yang lalu, hari saya yang biasa terlihat sangat membosankan. Saya biasanya meninggalkan rumah pada jam 9 pagi, mulai bekerja sekitar jam 10 pagi, mengobrol dengan teman satu tim sambil minum kopi sebelum makan siang, makan siang, menonton YouTube dan benar-benar mulai bekerja sekitar jam 3 pagi. Untuk melakukan sesuatu, saya harus tetap di kantor sampai jam 9 malam.

Pada titik tertentu saya sadar - di mana semua waktu saya pergi? Saya meninggalkan rumah selama 12 jam, tetapi saya dibayar hanya untuk 8. Karena itu, saya membuat beberapa perhitungan. Hasilnya menakutkan - 33% dari waktu saya terbuang sia-sia.

Wahyu # 1: Pahami ke mana perginya waktu Anda


Saya tidak puas dengan hasil ini. Selain itu, saya merasa bahwa saya tidak tumbuh secepat mungkin. Saya memiliki prospek untuk menjadi pemimpin tim dalam lima tahun, mungkin berubah menjadi arsitek dalam sepuluh tahun. Tapi bagi saya itu sangat lambat. Sebelum saya adalah contoh dari Bill Gates dan Steve Jobs - mereka mencapai lebih banyak ketika mereka di usia saya.

Penyingkapan No. 2: Bukan karena orang-orang ini keren, tetapi bahwa saya melakukan sesuatu yang salah


Pada saat itu, saya menyadari bahwa jika saya ingin menjadi setara dengan Gates atau Jobs, saya perlu mengubah sesuatu dalam diri saya. Saya bisa menjadi lebih produktif dan efektif jika saya membuat beberapa penyesuaian pada hidup saya.

Suatu hari, menonton YouTube setelah makan malam, saya melihat Steve Jobs membuka pidato di Stanford. Dan itu benar-benar mengubah sesuatu dalam diri saya.

Saya menyadari bahwa semua orang ini - Steve, Bill, Mark Zuckerberg - berbagi. Steve Jobs membagikan video ini. Saya yakin bahwa ini bukan satu-satunya hal yang telah ia lakukan atas kehendaknya sendiri. Bill Gates menyumbangkan setengah kekayaannya untuk amal dan meyakinkan setengah Forbes untuk melakukan hal yang sama. Mereka berbagi apa yang mereka bisa, tetapi saya tidak pernah memiliki ide seperti itu.

Wahyu No. 3: Jika Anda melakukan semuanya dengan cara lama, semuanya akan tetap sama


Saya ingin mengubah sesuatu. Pertama, saya mengubah pekerjaan kantor saya menjadi pekerjaan jarak jauh di Skyeng. Saya tidak mendapatkan kembali empat jam - hanya dua. Tapi itu perlu dimulai di suatu tempat.
Saya melihat semua orang ini dan berpikir - â€œmungkin saya harus mencoba les.â€ Saya menghubungi universitas lokal saya, lalu mencari platform untuk guru. Begitulah cara saya menemukan tempat di mana saya bekerja sekarang - platform online dengan kursus pemrograman. Semuanya datang bersama dan saya menjadi tutor.

Apa yang saya lakukan sekarang sebagai tutor



Pekerjaan saya adalah memeriksa pekerjaan rumah siswa di Github dan memberi mereka nasihat. Kami melakukan konsultasi seminggu sekali melalui obrolan video. Mereka mengajukan pertanyaan, dan saya menjawabnya. Seringkali mereka mendemonstrasikan layar mereka dan kami menulis kode bersama. Terkadang siswa mengirimkan saya pertanyaan mereka. Misalnya, "cara menggabungkan dua array." Dan saya menjelaskan cara menggabungkan array. Saya seperti Google, tetapi lebih maju :-)

Tentu saja, tidak semuanya berjalan lancar. Berikut adalah beberapa episode yang paling menyakitkan:

  • Saya mengambil terlalu banyak siswa. Pada satu titik, saya punya lima siswa - dan itu terlalu banyak. Menilai pekerjaan rumah dan menjawab pertanyaan membutuhkan banyak waktu, tetapi les tidak dibayar sebaik pemrograman. Akibatnya, saya menghabiskan lebih banyak waktu untuk bekerja, tetapi mendapat lebih sedikit. Istri saya tidak terlalu senang tentang itu.
  • Saya menyadari bahwa pengetahuan saya tentang PHP tidak cukup. Saya telah memprogram dalam Yii Framework selama bertahun-tahun dan di Symfony selama beberapa tahun. Karena itu, ketika seseorang bertanya, "Bagaimana saya bisa menggabungkan dua array dengan plus, daripada menggabungkan array?" Saya berkata: "Mari kita istirahat." Saya harus menyegarkan pengetahuan saya untuk menjelaskan hal ini kepada mereka.
  • Saya tidak menetapkan batasan sejak awal. Dan siswa bisa sangat mengganggu. Mereka tidak dapat melakukan apa-apa sepanjang minggu, dan kemudian muncul bertanya "Saya mengirim solusi saya, apakah Anda memeriksanya? Dan sekarang? Apakah Anda memeriksanya sekarang?"

Semua ini agak tidak menyenangkan; Saya merasa harapan saya tidak terpenuhi. Saya berpikir untuk kembali ke pekerjaan kantor, tetapi memutuskan untuk memberikan bimbingan terakhir.

Wahyu No. 4: Jika Anda tidak segera berhasil, jangan menyerah. Kerjakan kesalahan Anda dan semuanya bisa menjadi lebih baik.


Saya membuat beberapa penyesuaian dan menerima buah pertama dari pekerjaan saya. Masa sulit akan berakhir.

Manfaat menjadi tutor

  • Meningkatkan ego. Ketika Anda melakukan proyek dengan siswa Anda, dan kemudian seorang pakar independen mengevaluasinya 90 dari 100 ke atas, itu menginspirasi. Saya merasa seperti ini: "Wow, lihat mereka, mereka benar-benar belajar sesuatu."
  • Stok kepercayaan diri. Bagi saya, ulasan ini dari murid saya yang paling menyebalkan. Pada awalnya, dia mengkritik setiap langkah saya dan selalu tidak puas dengan sesuatu. Tetapi pada akhir kursus, dia memberi saya 9 dari 10 dan berkomentar: "Dulu payah, tapi sekarang semuanya baik-baik saja dengan bantuan Anda." Saya membaca kembali komentar ini setiap kali saya merasa tertekan - itu mengilhami saya untuk melanjutkan.
  • Pelatihan lanjutan. Saya benar-benar meningkatkan keterampilan teknis saya. Di tempat kerja, Anda terkadang menggunakan hal-hal tanpa memahami cara kerjanya. Tetapi begitu Anda mempelajarinya sejak awal, akan lebih mudah bagi Anda untuk menemukan dan memperbaiki kesalahan.
  • Meningkatkan soft skill. Salah satu yang paling dicintai. Di tempat kerja saya, saya mengelola tim yang terdiri dari tujuh pengembang. Bimbingan belajar telah membantu saya menjadi manajer yang lebih baik. Ketika saya berkomunikasi dengan banyak orang, saya telah mempelajari berbagai pendekatan dan jenis motivasi. Sekarang saya tahu persis bagaimana memotivasi programmer saya.

Mengapa Anda harus menjadi tutor?



Suatu kali saya bertemu dengan seorang pria dari pekerjaan lama saya. Saya memberi tahu dia tentang pendampingan saya - dan dia tidak begitu mengerti mengapa saya melakukan ini. Saya menjelaskan ini kepadanya dan akan menjelaskan kepada Anda:

  • Ubah Bagi saya, menjadi seorang mentor berarti menjadi aktif. Hari ini Anda tidak perlu PhD untuk mengajar, Anda dapat membagikan apa yang Anda ketahui dan gunakan dalam pekerjaan Anda. Jika Anda tidak nyaman dengan sistem pendidikan di negara Anda, Anda memiliki setiap kesempatan untuk mengubahnya. Saya senang menjadi bagian dari perubahan ini.
  • Tingkatkan keterampilan Anda. Seperti yang saya katakan, mengajar sangat mengembangkan keterampilan Anda sendiri. Anda akan mendapatkan banyak informasi tentang pemrograman dan teknik. Selain itu, Anda akan menjadi lebih baik dalam berkomunikasi dengan orang dan mengelola mereka. Dua poin ini akan membantu Anda dalam karier utama Anda.
  • Investasikan di masa depan. Banyak orang bertanya-tanya mengapa saya mentoring daripada mengambil dua jam kerja ekstra. Bagi saya itu seperti berinvestasi - saya berinvestasi dalam diri saya sendiri, mengembangkan keterampilan saya, dan pada orang lain, membantu mereka tumbuh. Saya pikir ini lebih berguna daripada menumpuk kertas.

Jika kisah saya selaras dengan Anda, cobalah les. Ini sangat sederhana, dan memiliki dampak nyata nyata pada orang-orang di sekitar Anda dan komunitas profesional.

Ayo tumbuh bersama!
