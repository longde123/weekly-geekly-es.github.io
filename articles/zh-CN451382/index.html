二合一：英特尔傲腾内存H10（第1部分）

第1 部分 >> 第2部分

SSD缓存已经存在了很长时间，可以让您从快速存储设备中获得最大性能。 近年来，使用非易失性3D XPoint内存的Intel Optane产品已成为小型，昂贵和非常快速的驱动器领域的主导。 借助英特尔的第三代Optane Memory高速缓存存储器，Optane的性能将符合新产品领域的要求。



第一个Optane Memory驱动器是微型NVMe SSD，旨在加快对速度较慢的SATA驱动器（尤其是机械硬盘）的访问。 英特尔目前正在继续使用Optane内存SSD来缓存其他NVMe SSM，重点是Optane闪存和QLC NAND的组合。 现在，他们已将两种类型的SSD放在同一M.2模块上，以创建新的Optane Memory H10。

英特尔Optane Memory H10首次允许英特尔在超极本中实现其Optane Memory缓存技术，在超极本中，只有一个SSD的空间，没有什么可谈论SATA的。 尤其是由于英特尔四位闪存QLC NAND的高密度，可以将两个磁盘压缩到一个单面80 mm M.2模块中。 英特尔660p QLC SSD在主板上具有大量的1 TB和512 GB版本的可用空间，Optane缓存可以弥补QLC NAND在性能和耐用性方面的不足。 即使在一个模块上放置两种类型的内存，H10的设计也非常简单。



Optane存储器H10不使用任何新的ASIC或其他硬件来使Optane和QLC驱动器模块看起来像一个设备。 缓存完全由软件控制，并且主机系统独立访问Optane和QLC H10。 驱动器的每一半都有两个专用的PCIe通道。 以前，所有Optane Memory SSD都是PCIe x2设备，因此它们在新设备中不会丢失任何东西。 但是英特尔660p使用4倍控制器Silicon Motion NVMe，现在仅限于两行PCIe。 坦率地说，尽管660p几乎从来不需要比x2通道所能提供的带宽更多的带宽，所以这并不是一个瓶颈。



在同一台英特尔设备上使用慢速QLC SSD和Optane快速SSD，很难对驱动器性能做出困难的决定。 对于两个较大的H10容量（512 GB和1 TB），声明的连续读取速度超过2 GB / s。 这反映了Intel Optane Memory软件同时从QLC和Optane H10提取数据的能力。 记录也可以在不同类型的内存之间交替，但是最大额定值不会超过任何设备性能的明显限制。 H10的随机I / O规格介于Optane内存和660p SSD设备之间，但更接近Optane的性能。 英特尔并未试图宣传理想的高速缓存命中率，但他们希望它足以用于实际使用。

Optane缓存应减少QLC H10写入的负载，但英特尔仍估计整个设备的耐用性与每天相同的0.16磁盘重写相同。 这与660p QLC SSD具有相同的耐用性。







英特尔Optane Memory H10的市场照片采用了带有双色PCB的SSD，以强调驱动器的双重特性，但实际上，驱动器板是单色的。 得益于两个控制器和三种类型的内存，该板的布局是独特的，但它也清楚地显示了其所基于的两个分立产品。 使用QLC NAND的驱动器有一半靠近M.2连接器，并配备了SM2263控制器，以及一个DRAM和NAND封装。 熟悉的Silicon Motion测试/调试连接位于NAND与Optane之间的边界。 Optane侧包含一个小型Intel Optane控制器，一个3D XPoint内存包和大多数电源管理组件。 带有Optane内存的Intel SSD 660p和更早的固态驱动器在印刷电路板上有很多可用空间。 Optane Memory H10已满，并且可能是市场上所有M.2 SSD中最多的部分。



乍一看，Optane Memory软件几乎没有变化； 除非现在在选择缓存设备方面有了更大的灵活性。 （英特尔已经宣布在Core处理器已经支持的平台上增强对Pentium和Celeron处理器的Optane Memory支持。）如果缓存了启动卷，则Intel软件允许用户缓存所选的文件和应用程序，以防止将其从缓存中删除。 除此选项外，没有缓存行为设置。

一些配备Optane Memory的OEM销售系统报告的内存大小为DRAM和Optane的总和。 当我们谈论连接到处理器的内存控制器的Optane DC只读内存模块时，这将很有意义。 令人误解的是，有关的Optane产品是SSD。

最初，Optane Memory H10将只是OEM的一部分。 新的SSD仅作为新系统（主要是笔记本电脑）的组成部分向消费者提供。 英特尔目前正在考虑以零售形式发布H10的可能性-作为独立产品还是NUC套件的一部分，尚未宣布任何时间表。 他们的主板合作伙伴已经为H10支持奠定了基础，并且许多台式机300系列主板已经通过最新的公开固件支持H10。

平台兼容性


至少可以说，将两个PCIe设备放置在同一M.2卡上是新的。 英特尔已经在一块电路板上安装了两个SSD控制器-在企业高端SSD中，例如P3608和P4608，但是这些驱动器使用PCIe交换芯片将x8主机连接分成板上的两个NVMe控制器中的每个x2主机。 。 这种方法将设备的TDP增大到40瓦，这在M.2的限制下根本没有用。

有多个PCIe扩展卡，可让您通过一个PCIe x16插槽连接四个M.2 PCIe SSD。 这些卡中的一些卡包括PCIe交换机，但是大多数卡使用支持双PCIe端口的主机系统，将一个x16端口拆分为四个独立的x4端口。 常规的消费类CPU通常不支持此功能，并且仅限于拆分x8 + x4 + x4或仅x8 + x8，并且仅在将行重定向到不同插槽以支持使用多个GPU的情况下。 用于服务器和工作站的最新处理器更可能支持最大x4的端口分离，但是支持此功能的主板支持并非一无是处。

即使在可以将x16插槽划分为四个x4端口的处理器上，也很少或根本不可能进一步分隔多达x2个端口。 支持大多数PCIe通道的芯片（例如狭窄的x2或x1端口）是大多数主板上的南桥/ PCH芯片。 通常，它们不支持大于x4的端口，因为这是与处理器的连接的正常宽度。

测试系统


我们测试消费者SSD的主要系统是Skylake台式计算机。 它具有Quarch XLC可编程电源模块，可进行详细的功率测量，并用于IO ATSB跟踪测试和使用FIO的综合测试。 但是我们的系统比Optane Memory更旧，并且Intel及其主板合作伙伴不希望发布固件更新来支持Skylake一代系统中的Optane Memory缓存。 结果，在我们的测试平台的帮助下，我们只能访问一半的磁盘-QLC NAND。



与发布新的Optane内存一样，英特尔向我们发送了一个完整的系统，其中已安装并配置了新的Optane内存H10。 因此，现在用于测试的系统是带有Intel Core i7-8565U Whiskey Lake处理器和16 GB DDR4内存的HP Spectre x360 13t笔记本电脑。 在过去的几年中，英特尔提供了用于测试Optane Memory产品的台式机系统，但是H10的主要优点是它是一个适用于小型系统的M.2模块，因此选择了13英寸的笔记本电脑。 英特尔已经确认，Spectre x360即将与Optane Memory H10一起出售，作为其存储选项之一。

HP Spectre x360 13t仅具有一个M.2 M型插槽，因此，为了测试使用SATA的多个磁盘或选件上的缓存配置，我们使用了Intel为以前版本的Optane内存提供的Coffee Lake和Kaby Lake系统。 这些机器之间的处理器能力和RAM的差异极大地影响了诸如SYSmark和PCMark之类的应用程序测试结果，因此我们必须为每种测试的磁盘配置列出三组等级。 但是，我们的AnandTech Storage Bench I / O测试和使用FIO的综合测试对于这三个系统给出的结果几乎相同，因此我们可以进行直接比较，对于每个此类测试，我们为每种存储配置列出一组要点。



英特尔傲腾内存缓存软件仅适用于Windows，因此我们基于Linux的合成FIO测试必须适应于在Windows上运行。 配置和测试过程与我们通常的方法尽可能接近，但是一些重要的区别意味着该评论的结果不能与我们通常的SSD评论和在Bench上发布的结果直接进行比较。 特别是在某些情况下，无法从Windows安全删除或格式化NVMe。 我们的测试通常涉及在主要阶段之间擦除磁盘以恢复性能，而无需等待后台SSD垃圾收集完成SLC缓存的清除和清除。 在对基于Windows的综合测试进行的回顾中，记录最小数据量的测试首先运行，而需要填满整个磁盘的测试则最后运行。

Optane内存缓存需要使用Intel驱动程序。 基于Windows的测试的通常过程是使用Microsoft自己的NVMe驱动程序，而不使用供应商特定的驱动程序。 本次评估中的Optane缓存配置测试是使用Intel驱动程序进行的，但默认情况下，Windows驱动程序在一个驱动器的所有测试中均使用Windows驱动程序（包括仅在Optane Memory H10的一侧进行的测试）。

我们的标准Skylake测试平台旨在测试连接到处理器的主PCIe x16插槽中的NVMe SSD。 Optane内存缓存要求通过芯片组连接驱动器，因此x4 DMI通道上的过载几乎不可能影响最快的驱动器，但H10不太可能使该连接饱和。

我们试图在几乎所有性能测试中都包含详细的功率测量，但是在本次评测中，我们被迫跳过其中的许多功能。 我们最先进的功率测量设备无法为笔记本电脑的M.2插槽供电；它需要常规的PCIe x4插槽。

应用测试


使用像Intel Optane Memory H10这样的复杂的分层存储系统，最准确的基准测试将是使用实际应用程序的测试。 BAPCo的SYSmark 2018和UL的PCMark 10是两组相互竞争的自动化应用程序测试。 两者都有一个共同的目标-提供对系统总体性能的评估，以及涵盖各种常见用例的若干其他评估。 PCMark 10是一个较短的测试，它提供了各种参数的更详细细分。 由于3D渲染已包含在标准测试套件中，而某些3DMark测试也包含在高级测试中，因此它也使GPU的负担更加沉重。 SYSmark 2018的优势是使用了流行的应用程序的完整商业版，包括Microsoft Office和Adobe Creative Suite，并且它具有在测试期间测量系统总能耗的能力。 这些测试的缺点是它们仅涵盖日常使用中最常见的情况，并且不会模拟繁重的多任务负载。 他们的子测试都没有密集使用存储，因此在比较快速和慢速SSD时，分数通常会略有不同。

BAPCo SYSmark 2018


BAPCo SYSmark 2018是基于应用程序的基准测试，它使用真实世界的应用程序来模拟业务用户的行为，并具有生产率，创造力和响应能力的其他指标。 结果反映了总体系统性能，并针对在每种情况下性能为1000点的参考系统进行了校准。 例如，如果评估为2000，则表示被测系统的速度是参考系统的两倍。









Kaby Lake台式机和Whiskey Lake笔记本电脑会根据子测试更改位置。 有时是由于其额外的RAM而使笔记本电脑位于前面，而由于TDP较高，有时是台式计算机在前面。 这些差异通常会比选择存储产生更大的影响，尽管响应性测试表明仅硬盘驱动器的参数还不够。 评估启用了缓存的Optane内存H10并不比仅使用一部分QLC更好，并且使用Optane缓存，它可以在其他更简单的SSD配置级别上工作。

能源消耗


SYSmark能源额定值显示了系统总消耗量，不包括显示屏。 根据我们的结果，我们的Kaby Lake测试系统在闲置时消耗约26瓦特的功率，峰值功率超过60瓦特。 SATA SSD很少消耗超过5瓦的功率，并且闲置时只有几瓦的功率；此外，SSD在空闲模式下执行大部分测试。 这意味着能源使用将非常相似。 尽管还考虑了显示器的功耗，但笔记本电脑的功耗大大降低。 所有高能耗存储选项（硬盘驱动器，Optane 900P）都无法装入该系统，因此能耗数据也非常接近。



Optane Memory H10是M.2中最贪吃的版本。 禁用Optane高速缓存可以节省一些电量，但不足以赶上基于TLC的优质驱动器。 Optane SSD 800P具有比大多数闪存驱动器更高的能源效率，但是其低容量阻碍了其实际使用。
