Administratoren werden nicht mehr benötigt?


Als Systemadministrator / -ingenieur, der seit einiger Zeit mit Microsoft- und VMware-Produkten arbeitet, stellte ich fest, dass in den letzten Jahren regelmäßig die Rede davon war, dass Systemadministratoren bald von niemandem benötigt werden, weil:

  1. Die gesamte Infrastruktur wird sich ändern und in IaaC (Infrastruktur als Code) verwandeln. Jetzt gibt es keine Benutzeroberfläche mit Schaltflächen, sondern nur PowerShell, Yaml-Dateien, Konfigurationen und mehr. Wenn ein Service oder seine Komponente defekt ist, muss er nicht mehr repariert werden, weil Stellen Sie eine neue Kopie aus dem letzten Arbeitszustand schneller bereit.
  2. Die gesamte IT-Infrastruktur wird in Kürze in die Cloud verlagert, und lokal (vor Ort) gibt es nur Netzwerkkabel zum nächstgelegenen Router, die uns mit allen anderen Unternehmensressourcen in der Cloud verbinden. Nun, der maximale Drucker bleibt immer noch lokal, so dass Buchhalterinnen Bilder von Katzen aus dem Internet darauf drucken können. Alles andere sollte in der Cloud sein.
  3. Ein DevOps-Guru wird kommen und alles automatisieren, sodass sich Administratoren nur noch genau daran erinnern müssen, wie sie früher Pings und Traces gestartet haben, um grundlegende Probleme im Netzwerk und auf den Servern zu diagnostizieren.
  4. Ich habe auch von so einem Phänomen wie „Vendekapets“ gehört, aber es ist lange her, als ich gerade anfing, die ersten Schritte in Richtung Systemadministration zu machen. Microsoft Exchange Server ist ein multifunktionaler Mailserver, der nicht nur die Arbeit mit E-Mails, sondern auch mit Kontakten, Kalendern, Aufgaben und vielem mehr umfasst. Exchange Server ist eines der Vorzeigeprodukte von Microsoft, das in vielen Unternehmen de facto zum Unternehmensstandard geworden ist. Es ist nicht nur eng mit Microsoft-Produkten integriert, sondern auch mit Lösungen von Drittanbietern. Der Austausch ist sowohl bei mittleren (ab 100 Personen) als auch bei großen Unternehmen beliebt.

Derzeit wird die aktuelle Version als Exchange Server 2019 betrachtet. Zuvor wurde das Produkt recht aktiv entwickelt. Seit der Veröffentlichung von Exchange 2013 hat sich diese Entwicklung jedoch erheblich verlangsamt. Daher kann Exchange 2016 als Service Pack 1 (SP1) für Exchange 2013 bezeichnet werden, und Exchange 2019 ist daher Dienst Pack 2 (SP2) für Exchange 2013. Das Schicksal der nächsten On-Premise-Version (Exchange 2022) ist noch nicht geklärt.

Jetzt fördert Microsoft Exchange Online aktiv als Teil des Office 365-Clouddienstes, sodass alle neuen Funktionen hauptsächlich dort angezeigt werden. Exchange Online ist nicht nur der erste, der neue Funktionen erhält, sondern auch zusätzliche Funktionen, die in naher Zukunft nicht auf lokale Installationen übertragen werden. Dies geschieht, um den Übergang einer Reihe von Unternehmen in die Cloud zu beschleunigen Das Abonnementmodell ist für Microsoft finanziell vorteilhafter als ein einmaliger Verkauf.

Wenn Sie derzeit eine lokale Installation von Exchange Server (2013 - 2019) warten, können Sie dies für die nächsten 3-5 Jahre fortsetzen. Auf diesem Weg sollten Sie die Möglichkeiten erkunden, die Exchange Online bietet. Bei Hybridkonfigurationen sind die lokale und die Cloud-Version gleichzeitig vorhanden. Auch wenn wir davon ausgehen, dass die nächste lokale (lokale) Version von Exchange nicht mehr verfügbar ist, sind die in Exchange Server gewonnenen Erkenntnisse aus verschiedenen Gründen für einige Zeit weiterhin relevant:

  • Die Anzahl der lokalen Installationen ist derzeit recht groß, daher werden qualifizierte Administratoren benötigt, die sie begleiten können. Nicht alle Organisationen werden in naher Zukunft aus dem einen oder anderen Grund in der Lage sein, ihre E-Mails in die Cloud zu übertragen.
  • Cloud-Migrationsprojekte sind noch nicht trivial. Daher sind Kenntnisse über die Besonderheiten lokaler und Cloud-Lösungen erforderlich, um die meisten Fallstricke zu umgehen und die Migration erfolgreich abzuschließen.
  • Die Kenntnisse in den Protokollen smtp \ imap \ mapi \ pop3, mail flow, dkim, dmark, spf, antivirus und antispam sind universell und gelten für jedes Mailsystem.
  • Dank der Erfahrungen, die Sie bei der Arbeit mit dem lokalen Exchange Server gesammelt haben, können Sie Exchange Online schnell verstehen und die gewünschte Konfiguration konfigurieren.
  • E-Mail ist einer der wichtigsten Kommunikationskanäle mit der Außenwelt, daher bleibt der Bedarf bestehen. Anhänger "Messenger und Chat-Bots ersetzen Mail" können nicht abgehört werden, weil sie "begruben" die Post viele Male und bis jetzt ohne Erfolg.

System Center ist eine Produktfamilie zum Bereitstellen, Konfigurieren, Verwalten und Überwachen großer Windows-Infrastrukturen. Nach folgenden Kriterien zu urteilen: System Center Configuration Manager (SCCM), System Center Virtual Machine Manager (SCVMM), System Center Operations Manager (SCOM), System Center Data Protection Manager (SCDPM), System Center Service Manager (SCSM), System Center Orchestrator (SCORCH) )



Die gesamte Palette der System Center-Produkte ist in der Regel nur in großen Unternehmen gefragt, während mittelständische Unternehmen in der Regel nur ein oder zwei Produkte einsetzen.

Da System Center-Produkte nur schwer zu erlernen sind und normalerweise nur in großen Infrastrukturen verwendet werden, ist es üblich, einzelne Personen für die Arbeit mit ihnen auszuwählen, z. B. einen Überwachungssystemadministrator (SCOM), einen Arbeitsstationsdienstadministrator (SCCM) und einen Virtualisierungssystemadministrator (Hyper) -V + SCVMM), Administrator für Infrastrukturautomatisierung (SCORCH + SCSM).

Microsoft entwickelt seine Cloud-Dienste rasant weiter, sodass die Funktionalität von System Center allmählich in die Cloud fließt. All dies wird in naher Zukunft erhebliche Auswirkungen auf die On-Premise-Produkte von System Center haben.

Die künftige Funktionalität von System Center Orchestrator (SCORCH) ersetzt den Azure-Automatisierungsdienst ( https://docs.microsoft.com/en-us/azure/automation/automation-intro ).

Die künftige Funktionalität von System Center Operations Manager (SCOM) ersetzt den Azure Monitor-Dienst ( https://docs.microsoft.com/en-us/azure/azure-monitor/overview ).

Die Funktionalität von System Center Data Protection Manager (SCDPM) ersetzt künftig den Azure-Sicherungsdienst ( https://docs.microsoft.com/en-us/azure/backup/backup-overview ).

Die Funktionalität von System Center Service Manager (SCSM) ist nicht mehr gefragt oder wird durch ein anderes Ticketsystem, z. B. Jira, ersetzt.

Der System Center Virtual Machine Manager (SCVMM) verbleibt bei Unternehmen, die Hyper-V-Virtualisierung lokal verwenden. Kleine Hyper-V-Installationen (10-15 Server) können ohne SCVMM nur mit den Standardtools Failover Cluster Manager, Hyper-V Manager und Windows Admin Center erfolgreich verwaltet werden.

System Center Configuration Manager (SCCM) - wird für die Massenbereitstellung von Betriebssystemen, die Installation von Unternehmensanwendungen aus einem einzigen Verzeichnis, die Installation von Windows-Updates auf Servern und Endarbeitsstationen, die Bestandsaufnahme von Anwendungen und die Lizenzzählung verwendet. Es scheint, dass dies das einzige Produkt aus der gesamten System Center-Reihe ist, das in der On-Premise-Infrastruktur as verbleiben wird es im Moment nicht vollständig durch etwas Trübes zu ersetzen.

Die Rolle der Administratoren der meisten anderen System Center-Produkte wird eingestellt. Azure-Dienste vereinfachen ihre Arbeit erheblich und verbergen die Komplexität vor neugierigen Blicken. Nehmen Sie zum Beispiel den Automatisierungsadministrator (SCORCH + SCSM). SCORCH wird durch Azure Automation ersetzt. Kenntnisse über den Automatisierungsprozess, PowerShell und SQL bleiben erhalten und sind für Azure Automation von Nutzen. Kenntnisse über die Erstellung von SCORCH-Clustern, die Gewährleistung ihrer hohen Verfügbarkeit, die Größenanpassung von Ressourcen, die Aktualisierung, die Migration auf neue Versionen sowie die Sicherung und Überwachung verlieren jedoch an Relevanz All diese Arbeiten werden von der Azure-Cloud ausgeführt.
