Microsoft bloquera les anciennes versions de Skype

Ces dernières années, Microsoft a migré son infrastructure Skype de l'ancien poste à poste (avec supernode) vers la nouvelle infrastructure cloud dans son cloud Azure. Dans le cadre de cette migration, nous devons faire des compromis techniques, notamment sacrifier la rétrocompatibilité avec les anciennes versions P2P des logiciels clients qui ne peuvent pas fonctionner dans le cloud.

La société a annoncé qu'à partir du 1er mars 2017, la prise en charge des versions P2P du programme pour Windows (7.16 et antérieures) et pour Mac (7.0 à 7.18) cessera . Les utilisateurs de ces versions ne pourront plus se connecter au programme. Apparemment, le client natif pour Linux (4.3) cessera également de fonctionner, seule une version Web compactée restera sur ce système d'exploitation.

Selon Microsoft, le refus de prendre en charge les anciennes versions P2P est un petit supplément pour les avantages que la transition vers l'infrastructure cloud offre: appels vidéo haute définition, qualité sonore améliorée, communications fiables et prise en charge de la synchronisation avec les appareils mobiles. De plus, maintenant il n'est plus nécessaire d'installer la version native du client, Skype pour Web fonctionne très bien via le navigateur.

Avec la dernière mise à niveau, les utilisateurs se voient promettre un support pour de nouvelles fonctionnalités et diverses améliorations, notamment:

  • Appels vidéo de groupe mobiles
  • Messagerie vidéo de groupe
  • Enregistrement de messages vidéo
  • Partage de fichiers dans le cloud (fichiers jusqu'à 300 Mo)

De plus, Skype dans la nouvelle version deviendra «plus léger et avec un temps de réponse minimum que jamais». En ce qui concerne la «légèreté record», Microsoft a peut-être été excité, mais cela ne ferait vraiment pas de mal d'optimiser le lourd client Skype. Sur les ordinateurs de mauvaise configuration et sur les téléphones mobiles, ce géant lent prend beaucoup de temps à charger, consomme une grande quantité de RAM, consomme une partie considérable des ressources CPU.

Skype officiel pour Android est si terrible que les utilisateurs ont dû installer massivement une version modifiée de Skype avec les développeurs xda , sinon il était presque impossible à utiliser en raison de plantages constants.

Il est conseillé aux utilisateurs de ces anciennes versions de Skype de télécharger la mise à jour avant le 1er mars . Après cela, le compte sera automatiquement transféré vers la nouvelle infrastructure.

Dès le début de son existence, Skype a travaillé sur le principe du P2P, c'est-à-dire qu'il a établi des connexions directement entre les utilisateurs. L'architecture Skype prévoyait la présence de "super nœuds" (supernode), qui étaient des utilisateurs avec des ordinateurs et des canaux de communication assez puissants. Grâce à eux, le trafic des autres participants du réseau P2P a été transmis (la possibilité de désactiver le supernode sur votre PC n'est apparue qu'avec Skype 3.0). En décembre 2010, Skype a commencé à héberger certains d'entre eux dans le cloud Amazon EC2.

Après avoir acheté Skype, Microsoft a poursuivi ses travaux. En 2012, elle a presque entièrement migré les supernodes vers ses serveurs, qui sont hébergés dans le cloud Azure. Ainsi, en 2012, l'infrastructure était encore plus centralisée: au lieu de 48+ mille supernode, le réseau P2P de Skype a commencé à faire passer le trafic à travers plus de 10,000 méga supernode sur l'hébergement Microsoft. La société a expliqué que cela "améliorait considérablement la sécurité du système et la qualité du service, compte tenu de l'augmentation spectaculaire du nombre d'utilisateurs." Chaque supernode de l'ancien système pouvait desservir environ 800 utilisateurs, et les nouveaux nœuds Microsoft peuvent contenir jusqu'à 100 000 connexions.

Les changements dans l'infrastructure Skype de Microsoft - le transfert complet des supernodes au cloud Azure et leur transformation en méga-supernodes - sont devenus le changement le plus spectaculaire de l'infrastructure Skype dans l'histoire de ce service.

En juillet 2016, il a été annoncé que Skype passait complètement au cloud. Depuis le 1er novembre 2016, Skype pour Windows Phone 8, Windows Phone 8.1, Windows RT et les anciennes versions d'OS X a été abandonné. Le nouveau client ne fonctionne que sur Mac OS X 10.9 ou supérieur. Il a été recommandé à d'autres utilisateurs d'utiliser le client Web.

La société a alors admis que le transfert de Skype vers le cloud nécessite un effort technique considérable, mais c'est une étape absolument nécessaire pour assurer le développement du service à l'avenir.

Ce n'est pas la première fois que Microsoft refuse la prise en charge des versions précédentes du logiciel client en raison d'une mise à niveau de l'infrastructure. La même chose s'est produite à l'automne 2014 lorsque les clients Skype pour Windows (6.13 et versions antérieures) et Skype pour Mac (6.14 et versions antérieures) ont cessé de fonctionner.

Il arrive très rarement qu'une nouvelle version d'un programme devienne plus légère, moins gourmande en ressources. Par conséquent, de nombreux utilisateurs sont habitués à vivre selon le principe «Ne touchez pas à ce qui fonctionne», sans installer de mises à jour logicielles, sauf si cela est absolument nécessaire. Le 1er mars 2017, ce "besoin extrême" viendra pour les utilisateurs de Skype.
