PHP Composer-PHP依赖管理器

Web开发人员面临的众多问题之一与依赖关系有关。 将新软件包集成到项目中通常带来的麻烦多于收益。 对于PHP开发人员而言尤其如此。 程序员经常从头开始构建模块，而实现网页之间数据的一致性是一件痛苦的事情，变量越大，痛苦就越大。 因此，优秀的开发人员会在冗余任务上花费大量时间和资源，同时他们希望提高工作效率并快速发布新代码。 这就是Plesk Obsidian拥有Composer的原因，Composer是一个优雅而简单的PHP依赖管理器，它使管理PHP项目的依赖关系变得容易（扩展是手动安装的）。

Docker NextGen-Docker便捷服务交付功能

在容器而不是虚拟机中运行应用程序在IT世界中获得了动力。 技术被认为是最近软件行业发展最快的技术之一。 它基于Docker，该平台使用户可以轻松打包，分发和管理容器化的应用程序。 借助Docker NextGen功能，使用基于交钥匙的基于Docker的解决方案（Redis，Memcached，MongoDB，Varnish等）变得更加容易，而不是公开Docker技术本身。 一键式部署网站支持服务。 Plesk配置服务，然后轻松地自动将它们与您的网站集成。 （即将推出）。

根据2018年Stack Overflow开发者调查 ，这是需求最大的调查，这是世界上最大的开发者调查，有超过100,000名受访者。 Plesk Obsidian配置MongoDB服务。 像任何其他数据库一样，MongoDB实例可以在本地或远程进行管理。 并将它们无缝集成到开发工作流程中。 （将很快可用）。

更安全，有用和可靠的开发人员工具

  • 对PHP-FPM和Apache服务的一些改进。 现在，重新启动Apache足够可靠，可以将其设置为默认值，以最大程度地减少站点停机时间。
  • 从存储在远程存储中的备份还原单个对象所需的磁盘空间已减少。
  • Plesk Obsidian随附的PHP引擎包含流行的PHP扩展（odium，exif，fileinfo等）。
  • 现在，PageSpeed模块已与NGINX预编译。

全面的安全性Plesk安全性核心


默认情况下，一流的服务器到站点保护功能可针对网站上最常见的攻击和恶意用户提供保护。


良好的默认托管

  • 开箱即用，Mod_security（WAF）和fail2ban处于活动状态。
  • 默认情况下，systemd现在将在5秒后自动重新启动失败的Plesk服务。
  • 在新创建的网站上，默认情况下启用针对SEO优化的HTTP> HTTPS。
  • 在基于系统的Linux（CentOS 7，RHEL 7，Ubuntu 16.04 / 18.04和Debian 8/9）上，Plesk紧急服务现在会自动重新启动。
  • PHP-FPM限制（通常称为max_children）是配置服务器上运行的并行PHP-FPM进程的最大数量（之前有5个）。
  • 现在默认情况下对传入和传出电子邮件启用SPF，DKIM和DMARC。

邮件增强

  • 现在，当超过95％的邮箱磁盘空间被占用时，邮件用户会收到电子邮件通知。 更多细节。
  • 邮件用户还可以在Horde和Roundcube Webmail客户端中查看有关邮箱磁盘空间，其使用和限制的信息。
  • 现在默认情况下可以通过HTTPS访问Plesk邮件服务器和Webmail：它们受Plesk自身保护的标准SSL / TLS证书保护。 更多细节。
  • Plesk管理员现在可以通过向客户自动发送一封电子邮件，其中包含用于重置密码的链接，从而更改客户，代理商和其他用户的密码。 邮件用户和其他用户现在可以指定一个外部电子邮件地址，如果他们失去对主要电子邮件地址的访问权限，则将用于重置密码。 更多细节。
  • 默认情况下，在panel.ini中启用了邮件自动检测功能，因此Plesk可以轻松支持最受欢迎的电子邮件，桌面和移动电子邮件客户端。 此新功能使您可以自动为Exchange Outlook和Thunderbird电子邮件客户端配置邮件。 更多细节 。

备份优化

  • 显着减少了创建备份并将备份恢复到云存储（Google Drive，Amazon S3，FTP，Microsoft One Drive等）所需的服务器上的可用磁盘空间。 这样可以节省存储成本。
  • 远程备份操作时间减少了。 例如，现在可以比以前快四倍地删除存储在云存储中的备份。
  • 要从完整服务器备份中还原一个预订，现在只需要等于此特定预订占用的额外可用磁盘空间，就不需要完整服务器备份。
  • 现在要将服务器备份到云存储需要额外的可用磁盘空间，该空间等于两个预订而不是整个服务器占用的空间。
  • Plesk Obsidian随附有修理工具包，这是一个功能强大的诊断和自我修复工具，可让您的客户随时随地甚至在无法使用Plesk的情况下修复潜在的问题。 它解决了以下问题：邮件服务器，Web服务器，DNS服务器，FTP服务器，Plesk Microsoft SQL Server数据库或Plesk文件系统本身-MySQL。
