支持vSphere 6.7和最新Veeam Backup＆Replication 9.5 Update 3a的其他功能

您记得之前的Veeam Backup＆Replication 9.5 Update 3是在2017年12月发布的。 从那时起，与该解决方案一起使用的平台和应用程序的制造商也发布了许多新版本-当然，Veeam必须支持它们，而我们的工程师成功地对其进行了管理。

今天，我将向您介绍一个新颖性- 更新3a （已分配此数字，因为计划在今年年底之前进行另一项重大更新，它已经是更新4）。
有关详细信息，欢迎猫。



因此，在Update 3a中有什么可以讨好您：

  • 支持VMware vSphere 6.7 。 请注意，现在支持异步NBD / NBDSSL-如果您使用的是ESXi 6.7，则可以提高网络传输模式的读取性能。 此外，还发布了用于Veeam备份和复制的 vSphere Web Client 插件的新（HTML5）版本。
  • 预先声明了对VMware vSphere 6.5 U2的支持 。
    重要！ 尽管Veeam尝试支持此版本的所有声明的功能，但应记住，在ESXi 6.5 U2代码中发现了严重错误，由于该错误，如果CPU和其他资源过载，vSphere API将停止工作。足够高。 目前，该问题已在VMware方面得到确认，正在进行研究和消除。 如果您是英语的朋友，明智的做法是订阅我们论坛的相应主题，以便与时俱进。
  • 支持VMware vCloud Director 9.1。
  • 支持VMware Cloud on AWS版本1.3 ，包括预先宣布的与将来更新的兼容性（因为它们取消了对匹配API版本的要求）。 此外，现在支持带有动态磁盘的虚拟机。

还实现了：

  • 支持Microsoft Windows Server 1803作为虚拟机的来宾OS。 此外，此类服务器可用于安装远程Veeam备份和复制组件，以及应该使用Veeam Explorer for Microsoft Active Directory还原其AD数据库（ .dit ） 文件的域控制器。
  • 支持Microsoft Windows Hyper-V Server 1803作为虚拟机监控程序主机。 虚拟机版本8.3（虚拟硬件版本8.3）支持VM。
  • 支持将Microsoft Windows 2018年4月10日更新为虚拟机的来宾OS。 此外，此类服务器可用于安装Veeam Backup＆Replication服务器和任何解决方案组件。

...和其他改进


  1. 通过优化与系统内存的交互，可以提高直接存储访问（DirectSAN）和虚拟设备（热添加）模式的性能。
  2. 在Microsoft Azure中为通过ExpressRoute和站点到站点VPN进行连接的基础结构实现了对直接还原恢复的支持（在这种情况下，代理和帮助设备只能具有私有IP地址）。 要激活此功能，需要在Veeam备份服务器上的HKLM \ SOFTWARE \ Veeam \ Veeam备份和复制分支中创建AzurePreferPrivateIpAddressesForProxyandLinuxAppliance（DWORD，1）注册表项。
  3. 恢复Linux文件时，现在如果选择复制到 （将还原的文件复制到指定位置），则将保存SUID和SGID。 还增加了对位于LVM上的Btrfs卷的支持。
  4. 他们简化了通用存储API的工作-现在您无需使用Veeam备份控制台在每台计算机上安装插件，只需将其安装在服务器上即可。

您可以在此处 （英语）阅读有关以前版本的其他改进和已解决问题的信息。

重要！ 在安装更新之前，检查正在使用的Veeam Backup＆Replication内部版本号（在主菜单中，选择“ 帮助” |“关于” ）。 这应该是以下数字之一：9.5.0.580、9.5.0.711、9.5.0802、9.5.028.3、9.5.0.1038、9.5.0.1335、9.5.0.1536。
安装此更新后，内部版本号将更改为9.5.0.1922。

如果使用的是Veeam Backup＆Replication 9.0，则首先需要从此处下载9.5 ISO并按照说明进行升级，以升级到9.5。
