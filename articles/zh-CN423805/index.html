杰夫·贝索斯（Jeff Bezos）哲学：“第一天”

9月13日，杰夫·贝佐斯（Jeff Bezos）启动了慈善项目第一天 。 我们将挖掘该名称的含义。



杰夫·贝佐斯（Jeff Bezos）的“第一天”哲学是什么？

这个问题最初出现在“ Quora”上：一个学习和共享知识的地方，人们可以从中学到东西并更好地了解世界。

Read Multiplex的创始人和编辑Brian Romemele就是这样回答的：
如果您不熟悉或无法迅速接受强大的趋势，“外部世界可以将您带入“第2天”状态。 如果您在与他们作战，那么您可能正在与未来作战。 使用它们并顺风顺水。”
-杰夫·贝佐斯

第一天


最近的证券交易委员会发现Jeff Bezos（EX-99.1）的演讲非常有趣。 杰夫（Jeff）在2016年发布给股东的亚马逊信息中描述了一种被称为第一天和第二天的哲学。 这个想法在亚马逊成立之初就传给了他。 他在名为“第1天”的大楼里工作，这起了提醒我们公司应始终处于“第1天”的模式。 Jeff的“第2天”是“第1天”概念的静态或完整变化，其后是无关紧要的。 其背后是痛苦，痛苦的衰落。 然后死亡。

在公司内部，这个想法成了民间传说。 亚马逊的成功很大程度上归功于其第一天的概念。 她有点传奇。

这是此概念的摘要：

专注于结果，而不是过程


Jeff解释说，随着公司的成长，依靠流程而不是结果变得更加容易。 在这种情况下，该过程成为“事物”。 发生这种情况时，公司有时会停止查看结果，而将注意力集中在他们是否正确地遵循了流程上，而不是在结果是否实现上。

快速做出决定


杰夫说，遵循第2天概念的公司实际上在做出正确的决策，但他们所做的只是缓慢的决策。 事实是，初创公司可以毫无问题地做出高质量，快速的决策，但这对大公司而言是困难的。 “亚马逊高管打算保持较高的决策速度。 速度在企业中很重要，也使环境变得更加有趣。”

为此，“第一天”流程为员工提供了一个系统：“不同意，但一定要做到”。 这样做的想法是，并非每个人都可以做出相同的决定，但是即使如此，仍然可以共同努力实现一个共同的目标。 他提到，他并不完全确定有关Amazon Prime的一个建议系列：部分是因为有点兴趣，还因为先进的商业条件。 他说：“我有完全不同的看法，但他们想借此机会。” 因此，我立即回答了他们：“我不同意，但是我开了绿灯，希望这个系列能够成为我们之前所做的最受关注的节目。” “想象一下，如果团队确实必须说服我而不是仅仅获得我的信任，我们会在多大程度上减慢决策周期。”

不要只专注于公司


对“第一天”的哲学的最雄辩的理解如下：“如果您不或者不能迅速接受强大的趋势，那么外界可以将您带入“第二天”的状态。 如果您在与他们作战，那么您可能正在与未来作战。 使用它们并顺风顺水。” 这些强大的趋势很难错过，但是在大多数情况下，大公司很难接受它们。 他指出，当今世界正在参与最重要的趋势-机器学习和人工智能。 因此，Echo / Alexa系统是亚马逊历史上增长最快，最畅销的产品。 杰夫说：“无论我们多么努力，我们仍然难以管理不断增加的Echo库存。 这是一个高质量的问题，但是是一个问题。 我们正在努力。”

但这还不是全部。 亚马逊正在尝试使用无人机和Amazon Go杂货店，后者使用机器视觉消除结账队列。 Jeff说：“ Machine Learning管理着我们的算法，用于预测需求，对产品搜索进行排名，产品和交易建议，商品销售，欺诈检测，转移等； “尽管不那么引人注目，但是机器学习的大部分工作将只是-不显眼，但改善了关键操作。”

市场分析


杰夫写道：“这是另一个例子：市场研究和客户调查可以成为客户的模型，当您发明和开发产品时，这尤其危险。 55％的Beta测试人员表示喜欢此功能。 与第一次调查相比，这个数字高出47％。 “这类信息很难解释，并且可能会在无意中误导人们。” 这个前提在某种程度上影响了苹果的史蒂夫·乔布斯。 这无疑表明亚马逊是如何创造第一次革命的声音的。 我写了杰夫只为Alexa创建的第1000支军队。

宣言“第一天”


第一天的概念构成了亚马逊清单的基础。 我相信这是我所读过的对股东最好的信息之一（几十年来我已经读够了）。 动态目标不仅是文字，而且是公司每天使用的完全涉及的概念。 这是二十年来一直在创造这些概念的内心罕见的外观。

以下是向美国证券交易委员会提交的Amazon Exhibit 99.1声明，该声明与1997年首次公开募股期间提交的原始Exhibit 99相互关联：

杰夫，第二天看起来像什么？

在最近的一次会议上我问了这个问题。 我提醒人们，几十年来我们拥有“第一天”。 我在名为Day 1的建筑物中工作，当我搬到另一座建筑物时，我决定保留这个名字。 我花了很多时间思考它。

“第二天”是静态的，后面是无关紧要的。 其背后是痛苦，痛苦的衰落。 然后死亡。 这就是为什么我们总是有“第一天”的原因。

不要犹豫-这样的下降将尽可能缓慢地发生。 某家公司可以在第二天收获数十年的收益，但最终结果将显而易见。
问题是我们如何应对第二天。 我们的招数和策略是什么？ 即使在大型组织中，如何保持“第一天”的生存能力？

没有简单的答案。 但是有很多组件，路径和陷阱。 我不知道全部答案，但我可能知道粒子。 这是保护第1天所必需的入门包：客户的痴迷，对模型的怀疑，对外部趋势的强烈渴望以及高速决策。

真正的客户痴迷


建立业务的方法有很多。 您可以将重点放在竞争对手，产品，技术，业务模型等上。 但是，我认为，客户的痴迷是保护第一天生存能力的最佳方法。

怎么了 以客户为中心的方法有很多优点，但最大的优点是：即使客户说自己很高兴并且生意很好，客户也总是完美而出乎意料地不满意。 他们甚至想要更多的东西，即使他们自己不知道，而您取悦顾客的强烈愿望也会引导您进行发明。 从来没有人要求我们创建Amazon Prime，但是事实证明，每个人都想要这项服务，并且有很多这样的例子。

第1天要求您冷静地进行实验，接受失败，种下种子，保护幼苗，并通过客户满意度使所有这些加倍。 痴迷于客户的文化创造了良好的环境。

不要屈服于现成的模型


随着公司开始变得更大，越来越复杂，它倾向于依赖经过验证的模型。 它可以以不同的方式表现出来，但是很危险，很狡猾，并且具有“第二天”的风格。

一个常见的示例是将流程作为模型。 一个好的流程可以为您服务于客户。 但是，如果您不够警惕，则该过程可能成为不可或缺的一部分。 这在大型组织中很常见。 该过程成为您要实现的结果的模型。 您将停止监视结果，并专注于过程的正确性。 您经常会听到一位年轻的领导者用这样的话辩护一个不好的结果：“好吧，我们只是按照指示进行。” 经验更丰富的领导者将利用这个机会来研究和改进流程。 该过程不能单靠自己。 您应该经常问自己：是流程的所有者还是流程的所有者？ 在第2天，第二种选择很有可能。

这是另一个示例：市场研究和客户调查可以成为客户的模型，当您发明和开发产品时，这尤其危险。 “ 55％的Beta测试人员表示他们喜欢此功能。 与第一次调查相比，它高出47％。” “这类信息很难解释，并且可能会在无意中误导人们。”

优秀的发明家和设计师会更深入地了解他们的客户。 他们需要大量时间和精力来发展这种直觉。 他们学习和理解大量的故事，而不仅仅是问卷的平均值。 他们是按设计生活的。

我不反对Beta测试或民意调查。 但是，您（产品或服务的所有者）必须了解客户，有远见并真正喜欢此报价。 在这种情况下，beta测试和研究将帮助您找到盲点。 真正令人难以置信的客户体验始于内心，直觉，好奇心，娱乐，勇气和品味。 调查不会为您提供类似的信息。

涵盖外部趋势


如果您不熟悉或无法迅速接受强大的趋势，外界可能会将您带入“第2天”状态。 如果您在与他们作战，那么您可能正在与未来作战。 用它们，迎着风。

很难不注意到这些趋势（他们谈论并写了很多关于它们的趋势），但是大公司很难接受它们。 现在我们有一个明显的趋势-机器学习和人工智能。

在过去的几十年中，计算机在很大程度上具有自动化任务，程序员可以使用明确的规则和算法对其进行描述。 现代机器学习方法使我们能够处理难以描述清晰规则的任务。

亚马逊多年来一直在练习机器学习。 值得一看的东西：我们用于Prime Air交付的无人驾驶无人机； Amazon Go杂货店，它使用机器视觉消除结帐队列； Alexa是我们的云AI助手。 （无论我们多么努力，我们仍然很难不断管理Echo库存。这是一个高品质的问题，但是是一个问题。我们正在努力。）

但是很多机器学习并不明显。 机器学习管理我们的算法，用于需求预测，产品搜索排名，产品和交易建议，商品推销，欺诈检测，转移等； 尽管不那么引人注目，但是机器学习的大部分工作只是这样-不可见，但可以改善关键操作。

借助AWS（Amazon Web Services）云平台，我们降低了成本，降低了其他机器学习和AI壁垒，因此所有组织和公司都可以利用先进技术。

使用根据P2计算模型（针对此类工作负载进行了优化）运行的流行深度学习系统的预先准备好的版本，客户已经在各个领域开发了功能强大的系统：从疾病的早期发现到提高产量。 我们还以方便的方式使Amazon的更高级别的服务更易于访问。 Amazon Lex（位于Alexa内部），Amazon Polly和Amazon Rekognition消除了自然语言识别，语音生成和图像分析的繁琐工作。 通过简单的API调用提供对它们的访问，该调用不需要机器学习方面的特殊知识。 密切注意该区域。 更多就是更多。

决策速度快


第2天的公司做出真正高质量的决策，但要慢慢做。 为了节省“第一天”的能源和动力，您必须能够在解决方案中兼顾高速和高质量。 与大型公司不同，这对初创公司来说很容易。 亚马逊团队致力于支持快速决策。 速度对业务至关重要，也使环境变得更加有趣。 我们不知道所有的答案，但这是我们的一些想法。

首先 ，永远不要使用通用的决策过程。 许多决定有两个方面。 轻量级过程可用于这些决策。 如果输入错误会怎样？ 我在去年的演讲中谈到了这一点。

其次 ，大多数决策应使用您希望拥有的大约70％的信息来做出。 如果您等待多达90％的时间，很可能您会变慢。 此外，在任何情况下，您都必须能够快速识别并纠正错误的决定。 如果您可以轻松地纠正解决方案的方向，那么错误将不会像您想的那样昂贵（但是延迟总是很昂贵）。

第三 ，运用“不同意，但是做”的原则。 他将节省大量时间。 如果您对某个方向有信念，但没有一致决定，则最好说：“听着，我了解我们尚未达成一致决定，但也许我们会冒险吗？ 不同意吗？” 在此阶段，没有人知道正确的答案，因此您可能很快会听到肯定的答案。

这不是单方面起作用的 。 如果您是老板，则还需要遵守这一原则。 我总是不同意，但是我同意。 最近，我们为Amazon Studios的一个系列开了绿灯。 我告诉我的团队，我不确定它会很有趣，它是否容易发布，业务条件不是很好，并且我们还有很多建议。 他们的面貌完全不同，他们想借此机会。 我立即回答他们：“我不同意，但我开了绿灯，我希望本系列将是我们之前所做的最受关注的节目。” 想象一下，如果团队确实必须说服我而不是仅仅获得我的信任，我们将减慢决策周期的速度。

请注意，在此示例中，我没有想到：“好吧，伙计们完全错了，没有掌握本质，这不值得我付出努力。” 这是真正的观点差异，是我观点的公开表达，是团队衡量我观点并坚持自己的方式的机会。 鉴于该团队已经获得了11项艾美奖，6项金球奖和3项奥斯卡奖，所以我很高兴他们总体上让我进入了房间！

第四 ，尽快发现不一致的真正问题，并立即将它们提高到一个新的水平。 有时团队有不同的目标和完全不同的观点。 他们不同意。 没有讨论也没有开会将解决彼此之间的这种深刻误解。 如果不升级，这种情况下的默认争议解决机制将被用尽。 最终决定由具有较高耐力的人做出。

多年来，在亚马逊，我看到了许多真正分歧的例子。 当我们邀请应该在我们自己的产品中直接与我们竞争的第三方卖家时，这一点尤其明显。 许多聪明的亚马逊员工根本不同意所选择的方向。 一个重大的决定需要一堆小的决定，其中许多必须提升为更高级别的高级管理人员。

“您完全让我累了”是一个糟糕的决策过程。 这很慢，而且精力不足。 最好将问题提高到另一个层次。

那么，您是专注于决策质量还是考虑速度？ 世界帆是否使您的帆充满风？ 您会成为模特的猎物还是模特为您效劳？ 而且，最重要的是，您能否取悦客户？ 您可以拥有大型公司的规模和能力，以及小型公司的精神和心脏。 选择是您的。
非常感谢每个客户让我们为您提供服务，感谢股东们的支持，也感谢每个员工的辛勤工作，独创性和热情。

同往常一样，我随函附上1997年原始信件的副本。 我们还有第一天。
