Atlassian会议回顾



这是9月5日至8日在巴塞罗那举行的Atlas Camp会议的概述。 Atlassian公司组织了此活动，以介绍其产品的创新以及Atlassian开发人员与访客的直接交流。 演讲者讨论了API即将发生的变化，市场将如何发展，界面的变化以及开发中的最佳实践。 此外，会议计划还包括开发最成功和最重要插件的公司代表的演讲。

由于我的自动化任务中有很大一部分与Atlassian产品相关（目前是Confluence和Jira，在我不得不处理Crucible，Fisheye，Bitbucket之前），我想知道在不久的将来会在其API中出现哪些创新。 此外，我想听听有趣的插件，并了解有关开发自己的插件的更多信息。 在削减的范围内，有一份关于我最终看到和发现的内容的详细报告，以及其他重要的组织问题。

第一件事


关于门票 。 门票只有一种类型（768欧元，含增值税）。 他们有权参观所有大厅，自助餐桌和小摊（他们免费提供了贴纸和T恤，其他“ nishtyaks”也都卖了）。



关于网站 。 会议在地铁Fira附近的酒店举行。 相对于机场和城市而言，地理位置十分便利。 会议室宽敞，舒适，一切都可以听到和看到。 没错，在会议的第一天，将一个大礼堂变成两个小礼堂就出现了技术问题，这就是为什么有些报告推迟到第二天早上的原因。 因此，在非常早的早晨，只有少数人拜访了他们，他们非常需要它。 安排食物（自助餐）的大厅实际上是一个大厅，学生可通过该大厅进入报告室。 由于空间的组织，那里的人群是永久性的。



关于酒店 。 我定居在费拉国会酒店。 距机场，地铁站和会议场地500米，地理位置优越。 良好的早餐，体面的房间。 但是我个人没有足够的毯子。 我了解的一切：这个南部的国家总是很温暖，但是在外表之下还是比较舒适的。 而且双层玻璃的窗户不够紧（如果下大雨，水可能会漏出）。



关于导航 。 会议的第一天，一位女士在入口处遇到了我们，她从字面上告诉了她的手指去哪里注册。 每个房间的前面是一台平板电脑，上面有一些活动时间表。 此外，会议还有一个移动应用程序，您可以在其中制定自己的行动计划，然后在选定报告开始前10分钟，电话上会出现提醒，指出正在召开哪个会议厅。 该应用程序还发布事件新闻和其他有用的信息。 它还充当Atlassian社交网络：在其中，会议参与者交换印象并发布照片。



关于日程安排 。 会议于上午9点开幕。 这时候，咖啡点正在现场工作，组织者的开发人员正在值班，因此可以与他们交谈。 报告本身从上午10点开始，演讲在下午6点结束。



关于报告 。 报告分为4个流。 此外，在一个特别的会议室中为插件开发人员组织了一次圆桌讨论会，并分配了另一个会议室以一对一的形式与Atlassian开发人员进行通信。 每个人都可以预先与一个或另一个开发人员预约约会，之前他们已经通过一种特殊的表格（可以自动设置会议时间）发送问题。

总的来说，这些报告甚至对那些不开发插件的人也很有趣，因为它们大多数都是关于开发中的各种最佳实践的。

Atlassian技术总监Sri Viswanath在主题演讲中发言。 他谈到了Atlassian应用程序开发人员的新增功能以及计划要做的其他事情。

Atlassian系统架构师Patrick Streule在他的报告《 Atlassian文档格式即将应用到您附近的API》中，谈到了新的文档格式-Atlassian文档格式（ADF）。 如果您的应用程序从Atlassian产品读取/写入内容，了解这一点很有用，因为现在它是所有产品的单一格式。 帕特里克（Patrick）还提到了开始使用新格式所需要的内容（库，技巧，技术等）。

K15t软件的联合创始人兼首席执行官Stefan Kleineikenscheidt就我开发了一个应用程序和OMG（现在我正在经营一家公司 ）作了非常有趣的演讲！ 他告诉了一个由50个成员组成的公司，该公司是如何从一个插件发展起来的，该插件开发并支持Confluence的前10个插件中的3个。 该报告分为几个部分，分别介绍了公司生命周期的不同阶段（他如何提出要编写的应用程序；他如何组建团队；组建团队并全力以赴工作时该怎么做）。

我不喜欢Atlassian开发人员Ian Buchanan的报告“ Getting Git Right” 。 他谈到了Git的内部结构和一些技巧，但是所有这些似乎已经过时了，无法从报告中收集到一些新的东西。

Atlassian首席用户体验设计师Kelly Snow也发表了一份有趣的报告，说明使用户方便安装和配置应用程序有多重要。 她还提供了一些有关如何实施此建议的实用建议。

我的个人观点 ：仅当您是Atlassian产品的插件开发人员或即将成为一体时，才需要参加会议。















关于商品 。 在注册期间，除了广告和信息手册外，他们还发行了T恤。 支付了另一种商品（包，马克杯，印有不同图案的T恤）。 我什么都没买，因为我不喜欢视觉上的东西。



关于食物 。 一点钟，午餐以自助餐形式提供。 从热的只有迷你汉堡。 但是有足够的食物，所以我不必为了金钱而寻找其他食物。 如果您看的话，所有食物都是西班牙人熟悉的传统西班牙小吃。 甚至汤（冷汤）也可以当点心吃-将它们倒入非常类似于杯子的东西中。 第一天，预留了两个大厅供午餐使用，第二道自助餐线被转移到大堂，并在会议室入口处进食。 实际上，在这个大厅中，小吃和饮料在不断更新，因此没有必要挨饿，也不需要排队等候食物。



组织会议记录 。 每个房间的前面都有一个带有QR码的大架子，您可以立即访问该幻灯片。 这非常方便，尤其是当您坐在后排，而前面是斯堪的纳维亚人时。

关于翻译的难点 。 所有发言者都说英语，尽管我并不完全会说流利的英语，但理解上没有困难。 对于那些讲得不太好，但精通技术术语的人来说，没有任何困难。

关于听众 。 听众中大多数是插件开发人员。 评估他们的技能水平很困难，但是我没有听到愚蠢的问题。 他们中的许多人都已经提出了有关疼痛主题的问题。

组织者没有报告他们设法召集了多少听众参加会议，但是根据我的感觉，人数不超过500人。 人们来自世界各地。 Atlassian竭尽全力向第三方开发人员展示其开放性，沟通和帮助的愿望。 通常，导航很方便，应用程序通常是会议最方便的功能，大大简化了生活。



提示经验丰富 。 在Atlas Camp会议上，我很感兴趣，我学到了很多有关Atlassian插件开发的知识，也许我会自己尝试。 但是，听到的创新不太可能对我当前的工作有所帮助。

我认为这次会议主要是为了在插件开发人员和Atlassian之间进行交流，以交流经验。 如果您使用他们的产品从事自动化工作，那么在官方程序的框架中您将几乎无法学习新知识，但是在与开发人员的非正式（且组织良好）沟通的框架中，您几乎可以解决所有问题。 但是值得为此支付768欧元吗？ 在我看来，这就是“那个马赫”。 尽管，正如他们所说的那样，“哼哼”。
