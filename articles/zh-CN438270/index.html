“在Booking.com的Kubernetes”
Ivan Kruglov，Booking.com，首席开发人员

在过去的两年中，Booking.com一直在应对加速新产品推向市场的挑战。 新方法的一部分是构建内部云。 它的基础是15个Kubernetes集群。 创建它们时，Booking.com脱离了公认的方法：这里的设置包括共享集群，扁平网络，为每个集群巧妙地计算出的SLO / SLI，一组用于实时验证集群功能及其集成的测试。

Booking.com不仅运营Kubernetes，而且还积极开发旨在改善生态系统的系统应用程序。 例如，它是在开放源代码Shipper中创建和布局的， Shipper是一组控制器，可以同时向多个集群提供Canary和Blue-Green部署的Kubernetes本地编排。

“ Kubernetes的安全性。 如何停止担忧并开始生活”
Dmitry Lazarenko，Mail.Ru Cloud Solutions，PaaS负责人

Kubernetes既强大又复杂。 安全问题也不例外：这里的一切都不像在虚拟环境中甚至基于Docker运行的应用程序的“经典”安全性情况那么琐碎。

在过去的一年中，在公共Mail.ru云中提供Kubernetes作为服务，Mail.Ru云解决方案团队面临着许多关于如何在所有级别实现Kubernetes中最大（有时是偏执的）应用程序安全的要求，这甚至适用于苛刻的企业，并且如何在Kubernetes中构建正确的安全开发生命周期/ DevSecOps。

演讲者将分享他在Kubernetes中实现典型安全模式的经验，该模式可应用于公共云和私有云。

“您需要在Bare Metal上了解Kubernetes的所有知识”
Andrey Kvapil，WEDOS Internet as，云架构师/ DevOps

Andrey Kvapil将在最大的捷克托管WEDOS上分享他在云解决方案体系结构和DevOps方面的经验，Kubernetes在该站点上积极使用Kubernetes来部署服务和服务器-现在可用于500多个节点，并且计划扩展到数千个。

将特别注意具有网络启动和存储选择功能的服务器场的组织。 演讲者在决定使用Linstor之前测试了许多免费的SDS解决方案，这是一个相当年轻的项目，尽管如此，该项目已经被证明具有出色的操作性。

随之而来的还有其他紧迫的问题。 在程序中：

  • 什么是裸金属，它与什么一起吃。 构建裸机集群时将面临哪些问题。
  • 使用LTSP的无盘节点和服务器部署自动化。
  • 存储选择。 考虑存储类型以及何时使用。 本地卷预配器如何工作
  • 关于网络，IPVS的功能和设置MetalLB的几句话。 尽可能摆脱覆盖。
  • 配置集群访问，与LDAP通信以及无缝管理多个集群。

小组讨论：“ Vanilla Kubernetes或供应商分布：未来是什么？”
主持人：Mikhail Zhuchkov

Kubernetes的发展不会停滞不前：每个季度我们都会收到新版本，通常会有重大改进。 Cloud Native Computing Foundation的生态系统也在积极向前发展，刺激了Kubernetes的快速增长并扩展了技术能力。

尽管大多数云提供商都基于原始分布提供Kubernetes作为服务，但现在市场上还提供了许多Kubernetes组件，甚至基于它们的单个产品。 其中一些只是稍微改善了非最佳行为，还有一些（例如OpenShift）几乎无法识别地改变了Kubernetes。 这与几年前Android设备的碎片化非常相似。

为了了解每种选择的利弊，并了解哪种方法能够生存，在我们的讨论中，我们收集了此类发行版的供应商以及运营香草Kubernetes的公司代表以及供应商解决方案。
