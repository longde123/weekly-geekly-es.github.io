Konfigurieren von UFW unter Ubuntu Server 16.04 mit dem BigBlueButton-Beispiel

UFW (Uncomplicated Firewall) ist eine iptables-Schnittstelle, die die Konfiguration einer Firewall vereinfacht. Das iptables-Tool ist zuverlässig und flexibel, aber für Neulinge ist es nicht einfach, es selbst zu konfigurieren. Ich bin gerade neu in diesem Geschäft.

Standardmäßig ist die ACCEPT-Richtlinie in iptables aktiviert und sieht folgendermaßen aus:



In unserem Beispiel gibt es keine Einschränkungen. Die Standardrichtlinie ist AKZEPTIEREN.

EINGANG - EINGANGSANSCHLÜSSE.
AUSGANG - AUSGANGSVERBINDUNGEN.
FORWARD - ROUTING (ZB IST ES ERFORDERLICH, EINE ROUTE an den Computer hinter dem Gateway zu schreiben).

Sie können ufw mit dem Befehl sudo ufw enable
Unmittelbar nach diesem Befehl ändert sich die iptables-Richtlinie.

Der ausgewählte Bereich ist der Ort für Benutzerregeln, sie werden mit den Befehlen registriert:

Für Hafen:

sudo ufw allow ssh (nach sudo ufw allow ssh )
sudo ufw allow 22 (Portnummer speziell angeben)
sudo ufw allow 80/udp (unter Angabe der Portnummer und des TCP- oder UDP-Protokolls)

Für Portbereich: 

 sudo ufw allow 16384:32768/udp

Regeln entfernen: 

 sudo ufw delete allow 80

Wir sind an Leitungen mit xxx.xxx.xxx.x interessiert (xxx.xxx.xxx.x ist die IP-Adresse Ihres Servers), da dies anzeigt, dass der Dienst auf der Netzwerkkarte auf eingehende Anforderungen aus allen Netzwerken wartet (0.0.0.0:*). . Zeilen mit 127.0.0.1 interessieren uns nicht, da dies die interne Schnittstelle ist. Daher interessieren uns auch Zeilen mit 0.0.0.0 in der 4. Spalte (fett hervorgehoben)

tcp 0 0 0.0.0.0:80 0.0.0.0:* LISTEN 1359/nginx -g daemo

Wie es heißt, ist Port 80 für alle Serverschnittstellen offen und akzeptiert Anforderungen von allen Netzwerken.

Demnach machen wir die Regeln:

sudo ufw allow 22 für ssh
sudo ufw allow 80 für sudo ufw allow 80 oder Apache-Webserver

Eine Reihe von Regeln für den Freischalter: 

 sudo ufw allow 5060 sudo ufw allow 5090 sudo ufw allow 2855 sudo ufw allow 2856 sudo ufw allow 5066 sudo ufw allow 8081 sudo ufw allow 8082

für den Knotendienst:

sudo ufw allow 3000
Als nächstes müssen Sie sich die Dokumentation ansehen, in der wir sehen werden, dass die folgenden Ports geöffnet werden müssen:

sudo ufw allow 16384:32768/udp - für das Arbeiten mit WebRTC
sudo ufw allow 443
sudo ufw allow 1935

Damit ist die grundlegende ufw-Einrichtung für den BigBlueButton 2.0-Server abgeschlossen.

Ich möchte nur hinzufügen, dass es wünschenswert ist, entweder ssh zu deaktivieren oder den Port zu ändern oder diesen Port nur für ein bestimmtes Netzwerk zuzulassen, beispielsweise für das Netzwerk Ihres Unternehmens.

Beispielsweise verfügt Ihr Gateway über zwei Netzwerkschnittstellen, eine virtuelle (für Benutzer) 192.168.3.2 und eine echte 105.xxx.xxx.xxx (für den Anbieter). Für die ufw-Regel verwenden wir hier eine echte IP-Adresse oder geben das Netzwerk an, zu dem diese gehört ip.

Ändern Sie zunächst den Port in der Konfigurationsdatei des SSH-Servers. Sie können einen beliebigen Texteditor verwenden: 

 vim /etc/ssh/sshd_config

Dann löschen wir die Regel, die Port 22 zulässt:

Sudo ufw delete allow 22 - manchmal auch OpenSSH genannt.

Wir erstellen eine Regel, die die Verbindung zum neuen Port nur über das Unternehmensnetzwerk zulässt: 

 sudo ufw allow from 105.../26 to any port 4321

Dann starten wir den SSH-Dienst neu: 

 /etc/init.d/ssh restart

Und sehen Sie das Ergebnis 

 netstat -ntlp | grep LISTEN 

 tcp 0 0.0.0.0:4321 0.0.0.0:* listen 5676/sshd

und in den ufw-Regeln werden wir sehen, wer Zugang zum Hafen hat: 

 ufw status 

 4321 ALLOW 105.xxx.xxx.xxx/26
