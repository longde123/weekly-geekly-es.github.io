Baru di Musim Semi 5. Migrasi Proyek ke Musim Semi 5

Halo Melanjutkan serangkaian migrasi ke versi kerangka kerja baru.
Terakhir kali adalah JUnit 5 dalam 10 menit. , kali ini adalah Spring 5.

Ketika mereka berbicara tentang hal-hal baru di Spring 5, mereka kebanyakan mulai berbicara tentang tumpukan jet, menghilangkan banyak hal baru yang muncul di dalamnya selain itu. Ada sangat sedikit driver reaktif untuk database, jadi video review saya tentang pembaruan penting lainnya, saya harap, akan relevan untuk sebagian besar pengguna Spring. Selain itu, semua dependensinya diperbarui bersama dengan Spring 5.x: spring-data 2.x, spring-security 5.x. Pada bulan Januari tahun ini, proyek pelatihan Spring 5 / JPA Enterprise (Topjava) kami bermigrasi dari Spring 4.3.x ke Spring 5. Prosesnya cukup mudah dan pada akhir video singkat selama 10 menit, saya juga akan membuat daftar semua perubahan dalam kode proyek.


Baru di Musim Semi 5


  • Menggunakan JDK 8+ (Opsional, CompletableFuture, Time API, java.util.function, metode default)
  • Dukungan Java 9 (Automatic-Module-Name in 5.0, module-info in 6.0+ , ASM 6)
  • Mendukung HTTP / 2 ( TLS , Push ), NIO / NIO.2, Kotlin

  • Perubahan lainnya

    • Anotasi keamanan- @Nullable ( @Nullable ), dokumentasi baru
    • Indeks komponen pada waktu kompilasi (alternatif untuk pemindaian classpath)
    • Kompatibel dengan Java EE 8 (Servlet 4.0, Bean Validation 2.0, JPA 2.2 , JSON Binding API 1.0)
    • Jembatan logging umum baru - spring-jcl
    • Dukungan untuk JUnit 5 + Peningkatan Pengujian (bersyarat dan bersamaan)
    • Dukungan yang dihapus: Portlet, Velocity, JasperReports, XMLBeans, JDO, Guava
  • Spring-Data-JPA 2.x, Spring-Security 5.x
  • Reaktivitas ( Web pada Stack Reaktif )
