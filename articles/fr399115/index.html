Lenovo a autorisé Linux à être installé sur les ordinateurs portables exécutant Windows 10, mais avec l'abandon du support technique


Lenovo Yoga 900

En septembre 2016, Lenovo a été fortement critiqué lorsque des militants Open Source ont suspecté un «accord secret» avec Microsoft pour installer exclusivement Windows 10 sur certains ordinateurs Lenovo. La raison de ces soupçons était le fait qu'il est impossible d'installer Linux sur certains modèles d'ordinateurs portables avec Windows 10 .

Après de vives protestations publiques, Lenovo a fait marche arrière et a publié un nouveau micrologiciel BIOS pour ces ordinateurs portables.

En septembre, des messages sont apparus sur les forums indiquant que Linux ne pouvait pas être installé sur un certain nombre d'ordinateurs portables Lenovo livrés avec une version de Windows 10 Signature Edition. Selon les hypothèses des utilisateurs, la désignation «Signature Edition» signifie que Microsoft a un accord avec le fabricant concernant ces ordinateurs que l'ordinateur est verrouillé sur le système d'exploitation Windows. Par exemple, il n'est pas possible d'installer Linux en mode RAID propriétaire, qui est verrouillé via UEFI / BIOS, sur un ordinateur portable Lenovo Yoga 900 ISK2 UltraBook.

L'enquête a révéléque sur les plateformes Intel, le disque est configuré en UEFI / BIOS de deux manières: en "standard" ou en RAID. Dans le deuxième cas, les identifiants PCI sont simplement modifiés pour désactiver les pilotes standard et garantir l'utilisation de pilotes Intel spécifiques, qui prennent définitivement en charge le RAID. Intel n'a pas préparé de commit sur le noyau Linux pour prendre en charge le mode RAID pour ces pilotes. Par conséquent, avec de tels paramètres UEFI / BIOS, le système d'exploitation Linux «ne voit pas» le lecteur. Dans ce cas, les ordinateurs portables Lenovo n'ont pas la possibilité de modifier la configuration du disque, il est fermement installé en RAID.

Un porte-parole de Lenovo a ensuite déclaré que dans ce cas, la version Signature Edition du système d'exploitation Windows 10 Home était installée sur l'ordinateur portable: «Elle est verrouillée par notre accord avec Microsoft».

Le fait d'un accord secret n'a pas été confirmé par la suite. Peut-être que le problème était précisément dans les pilotes Intel. D'une manière ou d'une autre, mais Lenovo a finalement reconnu son erreur. Le site officiel du support technique a publié de nouvelles versions de firmware pour les ordinateurs portables Yoga 900 et IdeaPad 710, ce qui a provoqué l'insatisfaction des utilisateurs.

  • YOGA 900-13ISK2:
  • YOGA 900-13ISK pour BIZ
  • YOGA 900S-12ISK
  • Ideapad 710S Plus-13ISK
  • Ideapad 710S-13IKB
  • Ideapad 710S-13ISK

Dans le nouveau firmware du BIOS, il y a une option AHCI, qui vous permet toujours de changer le mode de lecteur et d'installer Linux.

Certes, il y a une mise en garde. Le forum souligne que ces versions du BIOS "ne sont pas destinées à être utilisées sur des ordinateurs exécutant le système d'exploitation Windows". De plus, Lenovo ne fournit pas de support officiel pour ce BIOS, et tous les utilisateurs sont avertis des risques de «dégradation des performances pouvant être associés à l'utilisation de ce BIOS».

Les fans de Linux ont été autorisés à installer un système d'exploitation gratuit, mais à leurs risques et périls, avec la perte du support technique officiel.

Ainsi, Lenovo a trouvé un moyen de résoudre le problème. Après tout, si nous omettons la version théologique du complot de l'accord secret avec Microsoft, alors pourquoi a-t-il initialement «verrouillé» le BIOS? Afin d'exclure l'utilisation de pilotes Intel non vérifiés, qui nécessitent des paramètres spéciaux pour la gestion de l'alimentation. Comme mentionné ci-dessus, jusqu'à récemment, Intel ne fournissait pas de commit au noyau Linux pour prendre en charge le mode RAID pour ces pilotes. Ce n'est que le 21 octobre que les développeurs Intel ont toujours introduit le patch correspondant .

Lenovo a décidé que cela entraînerait des questions inutiles au support technique de Lenovo. La direction de la société chinoise a décidé que le nombre d'utilisateurs de Linux est si petit qu'ils peuvent être négligés afin de réduire le nombre d'appels au support technique.

Après que l'histoire est devenue scandaleuse, ils ont décidé de publier un nouveau firmware. Mais les utilisateurs qui l'installent n'ont pas été autorisés à appeler le support technique . Tout est simple et logique.

Lenovo a une liste d'ordinateurs portables certifiés pour installer Linux, mais le Yoga 900 et l'IdeaPad 710 ne figurent toujours pas sur cette liste.

Il est plus difficile pour les utilisateurs qui installent Linux sur les ordinateurs portables Lenovo de s'appuyer sur le support technique car le personnel du support technique est formé pour répondre aux questions principalement sous Windows. Dans ce cas, les utilisateurs sont généralement privés de support. Cependant, ce n'est pas un gros problème - la plupart des utilisateurs de Linux n'ont pas besoin des conseils des filles du support.
