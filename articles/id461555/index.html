Google Spreadsheets Visualisasi

Sejak awal, Excel telah menjadi alat otomatisasi perhitungan paling populer dan serbaguna untuk non-programmer dan semi-programmer (seperti saya). Pada suatu waktu, saya menulis banyak hal di VBA, membuat skrip untuk penulisan kursus mahasiswa di perguruan tinggi, dan bahkan menulis perhitungan berulang yang kuat dari distribusi panas dalam bahan bakar padat selama pembakaran, yang melumpuhkan kompas selama 20 menit. Saat ini, di era cloud dan solusi web, Google Sheets mengambil alih ceruk ini.

Baru-baru ini, saya melakukan studi kecil tabel Google untuk bekerja dengan visualisasi. Dalam proses belajar, beberapa ide menarik dan satu naskah lahir. Di bawah potongan, hasil penelitian saya.


Saya bekerja di bidang manajemen dan perencanaan konstruksi, tetapi di lokasi konstruksi sesuatu yang lebih rumit daripada Ixel berakar dengan sangat buruk. Oleh karena itu, tabel Google memiliki setiap kesempatan untuk memenangkan hati (atau mungkin bahkan pikiran) pembangun. Beberapa metode yang sudah menjadi standar kerja di spreadsheet diadopsi oleh Google dari pendahulunya, jadi pada pandangan pertama perbedaan dengan isxel tidak signifikan. Pada saat yang sama, beberapa fitur sedang diimplementasikan dalam bentuk baru, dan salah satu implementasi yang sukses menurut saya adalah rumus array. Mereka juga berada di Iksel, tetapi ketika memandang mereka, saya ingin mengatakan "terima kasih Tuhan bahwa saya tidak tahu di mana saya membutuhkannya." Di Google, senang bekerja dengan mereka, dan saya pikir mereka akan tetap menunjukkan kegunaannya dalam banyak tugas. Saya akan menunjukkan bagaimana mereka dapat digunakan untuk bekerja dengan membangun visualisasi.

Mendemonstrasikan semua ini dalam format video lebih mudah, jadi saya membuat add-on kecil:


Dan di sini adalah kode skrip yang memungkinkan Anda untuk menghubungkan visualisasi dengan notasi yang berbeda dengan referensi parameter: 

/** *         * @param {"A5:D20"} arr     * @param {"A21:D25"} sprav  () * @param {""} param     * @customfunction */ function (arr, sprav, param) { if(typeof sprav=="object"&&sprav.length!=undefined) { if(typeof arr=="object"&&arr.length!=undefined) { for (var i = 0; i<arr.length; i++){ for (var j= 0; j<arr[i].length; j++){ if (arr[i][j] != ""){ var r = sprav.map(function(value){return value[0]}).indexOf(arr[i][j]); var c = sprav[0].indexOf(param); if (r!=-1 && c!=-1) {arr[i][j] = sprav[r][c]} } } } } } return arr; }

Mungkin optimasi kode tidak sesuai dengan standar, saya masih bukan programmer PRO, tetapi berhasil.

Tentu saja, mereka yang ada di subjek, mengatakan bahwa ini semua sampah, sudah lama ada BIM. Ya ada. Tapi kami akan dengan cepat mengencangkan tabel ke model di SketchUp (juga Google sehingga), daripada kami akan menerapkan BIM. Karena sementara semuanya mulai dari produsen dan perancang bahan bangunan hingga pembangun dan pemeliharaan tidak bekerja di BIM, dibutuhkan lebih banyak pekerjaan daripada yang baik. Dan di negara kita ini tidak akan terjadi segera.

Akhirnya tentang Google. Tampaknya oleh kualitas produk itu jauh di depan analog, di mana permukaan googling memberikan seluruh galaksi. Tapi tidak ada tempat saya bertemu dengan logika kerja yang dipikirkan dengan baik, lintas-perangkat berkualitas tinggi dan kemampuan untuk memperluas fungsionalitas dengan skrip. Tapi ini hanya pendapat saya, dan jika saya salah, bagikan pendapat Anda.
