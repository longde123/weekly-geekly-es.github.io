3CX WebMeeting服务更新，Elastix在线转换器和新视频教程

上周，我们为3CX系统的开发人员发布了一些新产品和培训视频课程。 但是首先要注意-确实有很多新闻！

3CX WebMeeting更新


我们的免费视频会议服务的下一个版本3CX WebMeeting包括适用于Android和iOS的新移动应用程序，增强的安全性，许多新的服务器平台功能以及其他改进。 更新的WebMeeting变得更安全，更方便。 简要介绍新功能：

  • 加强TLS / SSL。 WebMeeting微服务的增强TLS / SSL加密-转换器，MCU，门户和文件传输。
  • 漫长的会议。 一次网络会议的持续时间增加到90小时。 这意味着您可以根据业务需要在不同时区或用户组举行马拉松会议。 另外，远程医疗中可能需要此解决方案。
  • 先进的遥控器。 在此版本中，按需在“快速”会议中显示了参与者PC的远程控制（以前仅在预定会议中可用）。
  • 更新的用户界面。 熟悉的视频会议界面也已更新和优化。

我们还努力开发了适用于Android和iOS的移动视频会议应用程序。 主要变化：

  • 支持最新版本的Google WebRTC。 应用程序支持最新的WebRTC SDP方案，该方案在最新版本的Chrome和Firefox中实现。
  • 节省电池电量。 在与多位与会者进行视频会议时，我们将智能手机电池电量节省了三倍！

请注意-要使用更新的3CX WebMeeting服务，必须对Android和iOS应用程序进行强制性更新-旧应用程序将无法使用。

版本中的完整变更日志 。

MCU WebMeeting可用性监控


现在，视频会议服务器的分布式系统（MCU-多点控制单元）3CX WebMeeting每月可以服务超过10万次视频会议会话，参与者总数高达20万！ 服务器位于世界各地的数据中心，不幸的是，有时其中一些服务器可能不可用。 为了让您提前知道该问题，或者只是选择ping效果最好的MCU，我们创建了一个WebMeeting服务可用性监视页面 。



使用监视非常简单。 将鼠标悬停在地图上您感兴趣的服务器上以查看其可用性。 下面是所有MCU WebMeeting的列表，按您附近的最小ping排序，在右侧-与某些功能的可用性相关的重要新闻。

某项服务的云基础架构可以保证100％的正常运行时间，这是不公平的。 因此，我们为管理员提供了手动验证节点操作的机会，如果没有最近的节点，则只需选择另一个即可：

  • 在3CX Web客户端中创建视频会议时，单击扳手图标并选择适当的MCU（如果建议的MCU不可用或ping过多）。
  • 在3CX服务器界面中，转到“设置”>“会议”，然后选择首选的MCU。 在这种情况下，默认情况下将在此MCU上创建所有用户会议。



用于FreePBX，Elastix和Askozia的在线配置转换器


也许你们中的许多人知道3CX在不同时期收购了其他软件PBX制造商-FreePBX，Elastix和Askozia。 为了使这些制造商的老式电话交换机的用户可以轻松切换到3CX（其中包括一些已接收的技术），我们发布了一个简单的在线配置转换器 。



要切换到3CX，只需备份旧PBX的配置，将其加载到转换器中，在3CX中指示所需的分机号，然后下载新配置。 然后在3CX初始安装向导中指定此配置文件。 填写此表格，以获取免费的3CX许可证。

支持以下PBX：

  • Askozia-v2.x / v3.x / v4.x / v5.x（.xml文件）
  • Elastix-v2.5 / 4.0（.tar文件）
  • FreePBX-v12 / v13 / v14（.tgz文件）

转换时，将传输以下PBX对象：

  • 分机号码（用户）
  • SIP中继
  • 入站和出站路由规则
  • 语音菜单（IVR）
  • 封锁清单
  • PBX系统代码

如果旧的电话交换机使用模拟或数字电话卡连接到PSTN，我们建议您将物理的“铜”线传输到SIP中继。 但是，如果由于某种原因您不能放弃铜缆，则有两种解决方案：

  • 将Elastix作为VoIP网关连接到3CX
  • 收购现成的模拟/数字VoIP网关

3CX实例管理器和呼叫流程设计器视频教程


3CX实例管理器是在3CX v16版本中引入的。 这是功能相当强大的3CX服务器机队管理门户。 如果您为客户支持许多自动电话交换或在大型组织中管理公司电话，则建议使用它。 实例管理器立即显示所有受监视系统的状态。 在许多情况下，您可以立即从中开始维护操作。

要将客户端PBX连接到实例管理器：

  1. 在PBX界面的主页上，将您的会员ID绑定到此3CX实例。
  2. 在3CX管理界面中，转到“设置”>“ 3CX实例管理器”并启用所需的选项。 ATS受到监控！


您当然知道3CX呼叫流程设计器（CFD）是用于语音应用程序的强大开发环境，可让您自动处理传入和传出的呼叫。 通过拖放现成的组件来创建应用程序-您可以完全不编程，也可以很少使用代码。 3CX PRO中免费包含3CX CFD 。

现在，连同文档一起 ，开发人员可以观看CFD创建者自己提供的有关开发环境的视频教程。

  1. 创建您的第一个语音应用程序（呼叫流程）并将其上传到3CX服务器。
  2. 修改现有语音应用程序，例如，在PBX中安装非工作时间并发送电子邮件。
  3. 用新组件扩展现有项目（应用程序）。
  4. 向呼叫流添加复杂的逻辑-通过ID在数据库中进行用户身份验证。
  5. 使用验证算法检查用户输入并向订户播放一组号码。
