Menghubungkan kotak surat eksternal ke Zimbra

Hanya sedikit perusahaan besar yang bisa membanggakan diri pada awalnya. Banyak rantai ritel besar, integrator, pengembang, dan ISP memulai bisnis mereka dengan kantor-kantor kecil dan beberapa lusin karyawan yang bekerja keras untuk kesuksesan masa depan perusahaan mereka. Tentu saja, di perusahaan kecil seperti itu, pertanyaan tentang penerapan server mail atau sistem kolaborasi Anda sendiri sangat jarang, dan sebagai aturan, kotak surat karyawan dibuat dalam layanan email publik seperti surat Yandex atau gmail. Namun, ketika suatu perusahaan mencapai tahap tertentu dalam pengembangannya dan sampai pada kebutuhan untuk membuat server surat sendiri berdasarkan Zimbra, muncul pertanyaan tentang menghubungkan kotak surat lama ke akun baru.



Karena pengembangan suatu perusahaan hampir selalu menyiratkan kampanye iklan aktif di Internet, distribusi kartu nama yang menunjukkan e-mail, serta menambahkan alamat surat ke berbagai direktori, aliran surat dari pelanggan potensial atau aktif biasanya tidak melemah bahkan bertahun-tahun setelah mengubah alamat surat yang ada. Tentu saja, Anda dapat mewajibkan karyawan untuk memeriksa kotak surat lama setiap hari, tetapi menggabungkan kedua kotak surat itu menjadi satu masih akan menjadi keputusan yang lebih cerdas. Selain menghemat waktu dan tenaga staf, solusi seperti itu akan membantu Anda dengan cepat pindah ke kotak surat baru, karena sekarang jawaban untuk setiap surat akan datang dari alamat email baru.

Opsi termudah untuk menghubungkan kotak surat eksternal ke Zimbra pada pandangan pertama tampaknya adalah mengatur penerusan surat dari akun lama ke yang baru. Namun, metode ini memiliki sejumlah masalah. Khususnya, karena filter spam dan antivirus yang ada di Zimbra, sebuah surat atau beberapa lampiran mungkin tidak mencapai alamat baru. Selain itu, beberapa layanan mungkin tidak mendukung pengiriman email, dan sepenuhnya bergantung pada mereka dalam masalah yang penting juga bukan ide yang baik.

Itulah mengapa akan lebih tepat untuk menetapkan tugas menerima email dari kotak surat eksternal ke server yang dikendalikan oleh Anda. Namun, ada masalah kecil, karena tidak ada utilitas bawaan untuk bekerja dengan kotak surat eksternal di Zimbra Collaboration Suite. Namun, ini tidak akan menjadi masalah besar, karena Zimbra dapat bekerja dengan sempurna dengan program dan utilitas pihak ketiga. Itulah sebabnya untuk menerima email dari alamat eksternal, Anda dapat menggunakan program fetchmail , yang tersedia di repositori pada sebagian besar distribusi Linux.

Setelah manajer paket menyelesaikan instalasi fetchmail, akan diperlukan untuk menghasilkan formulir berikut di file / etc / default / fetchmail dan, jika perlu, batalkan komentar pada baris START_DAEMON = ya dan set daemon 600 . Pada baris pertama, kami mengizinkan fetchmail untuk dijalankan sebagai layanan sistem, dan pada baris kedua, kami mengatur interval untuk mengakses kotak surat eksternal dalam hitungan detik. Dalam hal ini, parameter ini adalah 600 detik dan ini berarti bahwa kotak surat akan diperiksa secara otomatis setiap 10 menit. Interval ini dapat ditingkatkan untuk mengurangi beban di server, atau dapat dikurangi jika Anda ingin surat-surat tiba secepat mungkin.

Setelah itu, tetap hanya menentukan dalam mengambil data untuk menghubungkan ke kotak surat eksternal. Misalnya, misalkan utilitas tersebut akan terhubung ke kotak surat manager@gmail.com , dan kemudian meneruskan email ke manager@company.ru . Untuk melakukan ini, kita harus terlebih dahulu membuat file konfigurasi fetchmail, dan kemudian membuat sejumlah perubahan padanya. Anda dapat melakukan ini dengan perintah cp /usr/share/doc/fetchmail/examples/fetchmailrc.example / etc / fetchmailrc

Setelah itu, buka file / etc / fetchmailrc di editor teks apa pun dan tentukan parameter untuk menghubungkan ke kotak surat eksternal di akhir file, data otorisasi untuk kotak surat eksternal kami: 

poll imap.gmail.com protocol IMAP user manager@gmail.com is manager@company.ru here password ********* keep ssl
Setelah itu, tetap hanya untuk me-restart fetchmail dengan secara berurutan memasukkan service fetchmail stop dan service fetchmail memulai perintah, dan kemudian menguji layanan dengan mengirimkan surat uji ke kotak surat eksternal. Dengan demikian, bahkan setelah memperkenalkan server email Zimbra Anda sendiri di perusahaan, Anda dapat menyelamatkan perusahaan dari keharusan memberi tahu pelanggan tentang perubahan alamat email.
