Manajemen file salah dilakukan - Bagian 1: Awalnya dari tahun 90-an

Hai, saya Gregory, saya telah menggunakan manajer file selama hampir 25 tahun, dan saya memiliki sesuatu untuk dikatakan tentang mereka - manajemen file saat ini tidak dilakukan dengan baik . Saya akan mencoba menunjukkan masalah pada program dan layanan yang paling umum, dan jika Anda mau, baca artikel whitepaper ini.


Lihatlah screenshot ini dan pikirkan apa yang telah berubah selama 20 tahun terakhir .



Pernyataan pengantar utama:


  1. Saat ini, manajemen file tidak dapat dipisahkan pada desktop, seluler, dan web - secara pribadi di depan umum dan dalam kolaborasi. Manajer file yang dibuat dengan baik harus menghapus batasan antar platform, membuat file disinkronkan dan siap untuk dibagikan dan dibagikan.
  2. Ini harus menyediakan pratinjau informatif tentang semua jenis konten, metadata yang relevan, dan alat manajemen tampilan konten yang cocok untuk setiap platform.
  3. Seharusnya alat yang relevan untuk operasi file, sehingga Anda dapat melakukan hal-hal sederhana dan kompleks tanpa terlalu banyak cuci otak.

Manajer file mainstream tidak dapat melakukan ini , secara umum mereka menyedot ketiga poin. Inilah alasannya:


  1. Desainer tidak peduli apa yang mereka desain, mereka tidak menggunakan produk. Mereka hanya melakukan pekerjaan mereka.
  2. Atasan mereka tidak peduli dengan desain. Haruskah begitu?
  3. Pengguna biasa tidak melihat kekurangan "minor". Mereka hanya menggunakannya.

Ketika orang bertanya kepada saya apa yang saya lakukan, saya menjawab dengan "masalah manajemen file", sebagian besar tidak mengerti apa yang saya maksud, yang lain mengatakan "apakah ada masalah?"

Saya ingin menunjukkan bagaimana manajer file menghabiskan waktu saya - berbagi rasa sakit . Ketika saya mencoba menemukan sesuatu. Menunggu generasi pratinjau. Menunggu pencarian. Saya mencari gambar yang tepat. Saya menggabungkan dua folder. Saya berbagi folder. Menunggu pembuatan tautan. Saya memperkenalkan tautan tidak-get-beberapa-lama dari Google Drive. Saya menggunakan manajer file, penampil gambar, penyimpanan cloud, dan alat penerbitan untuk membuat satu hal sederhana!


Bagian pertama mungkin sedikit membosankan, tetapi Anda harus memulainya. Akan ada banyak tangkapan layar. Dan banyak kebencian . Lanjutkan dengan risiko Anda sendiri.



Presentasi data


Dasar dari manajer file adalah presentasi file dan folder - presentasi grafis konten, pratinjau bawaan, ikon dan metadata yang relevan - ini harus menyediakan pencarian visual sederhana untuk file pada semua platform.


Desktop dan web


Mari kita mulai dari awal dan mempertimbangkan manajer yang paling umum - Windows File Explorer .



Tunggu, apakah ini manajer file # 1 nyata di dunia? Hampir terlihat seperti Explorer dari tahun 90an!

Pratinjau sangat kecil, meskipun ada banyak ruang di sekitar. Rasio aspek gambar diperhitungkan, tetapi ini tidak banyak membantu karena ukuran pratinjau. Tidak ada metadata.


Sebagai seorang fotografer, saya harus menyatakan bahwa rasio aspek gambar harus diperhitungkan.

Berikutnya datang Google Drive - folder di sini terlihat konyol dan benar-benar tidak informatif. Rasio aspek gambar tidak diperhitungkan. Di sini dimungkinkan tanpa metadata.



Mari kita lihat Google Sheets - walaupun ini bukan file manager, ini masih contoh yang bagus - berikut ini adalah preview dan metadata yang bagus.



Contoh bagus lainnya, Foto Google, adalah pratinjau yang baik.



Yandex.Disk di web hanya menyalin Explorer lama. Mengapa mereka melakukan itu? Saya pikir mereka tidak bisa membuat sendiri.



Dengan contoh-contoh di atas, kelebihan dan kekurangannya, saya merancang versi saya sendiri. Saya kehilangan urutan abjad dan pratinjau PDF, tetapi saya harap Anda mengerti maksudnya.



Tampilan daftar


Dalam folder dengan file audio Explorer tidak menunjukkan ukuran atau panjang trek - sangat nyaman.




Tentang folder dengan foto, saya bahkan tidak ingin menulis komentar, terima kasih, orang-orang dari Microsoft . Saya harap Anda menginginkan yang terbaik, tetapi ternyata Anda lupa untuk menguji ini pada file nyata. Lihat versi saya.



Aplikasi seluler


Google Drive memiliki 2 versi - daftar dan ubin, keduanya terlihat OK, tetapi sekali lagi dengan folder aneh dan tanpa menghormati rasio aspek foto. Dan belum ada pembesaran, jadi sulit untuk menemukan 10 file yang diperlukan di antara 256.



Aturan Foto Google lagi, mereka memiliki zoom dan metadata. Omong-omong, aplikasi Album Sony yang lama juga keren.



Saya menggambar versi saya lagi - sekarang dari tunggul kambing, dengan (imajiner) zoom, daftar dan ubin, sehingga Anda dapat dengan mudah menemukan gambar Anda! Saya tidak ingin menggambar kotak persegi dengan 24 foto, jadi bayangkan saja, tolong.



Satu Klien / Disk Virtual


Yandex.Disk melakukan sesuatu yang menarik - klien unduh-permintaan, sehingga Anda dapat mengunduh file hanya saat dibutuhkan. Tetapi mengapa klien individu? Mengapa folder virtual tidak dapat dikendarai? Tuhan tahu. Saya tidak ingat klien standalon lainnya. Mungkin ini pelanggan yang keren? Tidak. Dia mengerikan dalam penampilan dan bekerja stormno.


Ayah saya bilang dia mirip Norton Commander, sama sekali tidak seperti mereka di pendaratan .

Mereka hanya menyalin Explorer lama. Mengapa mereka tidak melakukan sesuatu yang sangat baik ? Karena mereka tidak bisa.



Tapi saya pikir ide disk virtual itu bagus, pertama kali saya melihatnya di ZumoDrive . Hari ini ia mengimplementasikan Disk-O dari Mail.ru (waspadalah).

Unduh dan pramuat data


File Explorer tidak membuat HDD tetap aktif, dan ketika Anda mencoba masuk ke folder setelah downtime, memutar disk memerlukan beberapa detik. Mengapa tidak tetap aktif atau membuka pratinjau dan metadata yang di-cache? HZ.


Mengunduh foto pratinjau di Explorer membutuhkan banyak waktu, folder terbuka sangat lambat. Penampil foto seperti FastStone melakukan ini lebih cepat dan tidak memblokir antarmuka saat boot.


Cari


Pencarian di File Explorer pada Windows 10 tidak dapat menemukan file. Saya tidak tahu apa yang menghentikannya. Mereka ada di sana , tetapi pencarian mengatakan tidak. Pencarian sangat lambat , mereka butuh beberapa menit. Opsi pencarian sangat tersembunyi sehingga tidak ada yang dapat menemukannya. Dan mereka terlihat seperti neraka.



Hasil pencarian tidak terlalu informatif, dan ketika Anda mencoba untuk pergi ke lokasi salah satu file yang ditemukan, Anda tidak dapat menggunakan navbar secara normal, Anda tidak dapat melihat lokasi folder yang sebenarnya dan Anda bahkan tidak bisa kembali ke hasil pencarian - itu akan mulai lagi! Oh, betapa gilanya itu.



Saya ingin menunjukkan contoh pencarian yang bagus di desktop, tapi saya tidak bisa - mereka tidak .
