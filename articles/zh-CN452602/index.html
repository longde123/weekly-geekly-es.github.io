引言


当前，有两种方法来组织超融合解决方案。 第一种方法基于以软件形式提供的软件定义的解决方案，客户可以自行选择设备。 第二种方法基于交钥匙解决方案，即包含软件，硬件和技术支持。 思科我们坚持第二种方法，并向客户提供现成的解决方案，因为只有这样，我们才能保证稳定的系统性能，一个制造商的高质量技术支持和高性能。
高系统性能是决定在关键任务中使用产品的关键因素之一。

如今，组织通常在关键的三层体系结构解决方案（DSS>存储网络>服务器）上部署关键任务。 同时，大多数组织都在努力简化和降低IT基础架构的成本，同时又不降低其稳定性和生产率。 因此，客户越来越多地转向超融合解决方案。

在本文的框架中，我们将讨论由独立实验室ESG（企业策略组）进行的最新测试（2019年2月）。 在测试过程中，模拟了高负载的Oracle和MS SQL DBMS（OLTP测试）的工作，这是实际生产环境中IT基础结构最关键的组件之一。

此负载是在以下三个解决方案上执行的：Cisco Hyperflex，以及两个安装在Hyperflex中使用的同一服务器上的软件定义的解决方案，即Cisco UCS服务器上。

测试配置




供应商A不使用缓存，因为解决方案开发人员不支持缓存配置。 因此，使用了更大的存储磁盘。

测试方法


OLTP测试使用四个虚拟机和一个3.2 TB的工作数据集进行。 在每次测试之前，每个虚拟机都使用测试工具填充了记录的数据。 这样可以确保测试读取“真实”数据并将其写入现有块，而不仅仅是直接从内存返回零块或零值。 当数据未满时会发生这种情况，因此确保测试准确反映应用程序环境中数据的读写方式非常重要。 完成这个庞大的工作集需要很长时间，但是在我们看来，这是对时间的有效投资，因为它可以使我们获得更准确的性能数据。

测试是使用HCI Bench工具（基于Oracle Vdbench）和I / O配置文件进行的，这些I / O配置文件旨在使用Oracle和SQL Server后端来模拟复杂的关键OLTP工作负载。 根据具有100％随机访问数据（完全随机）的模拟应用程序分配块大小。

Oracle数据库工作量


第一个是旨在模拟Oracle环境的OLTP测试。 Vdbench用于创建具有不同读/写比率的工作负载。 该测试是在四个虚拟机上进行的。 在四个小时的测试过程中，HyperFlex设法显示了超过420,000 IOPS，而延迟仅为4.4毫秒。 软件解决方案A和B分别只能显示238,000和251,000 IOPS。




除了供应商B的平均平均延迟为26.49 ms，读取延迟为2.9 ms以外，不同系统中的延迟级别大致相同。 压缩和重复数据删除在所有系统中都很活跃。

Microsoft SQL Server工作负载


然后，检查了旨在模拟Microsoft SQL Server DBMS的OLTP工作负载。


测试的结果是，Cisco HyperFlex集群的竞争优势超过了竞争对手A和B两次，思科的IOPS为490,000 IOPS，而制造商A和B的为200,000和260,000。


Cisco HyperFlex中的延迟结果与Oracle测试相差无几，也就是说，处于4.4 ms的良好水平。 同时，制造商A和B的结果要比对Oracle的测试明显差。 对于竞争解决方案B来说，唯一有利的事情是2.9 ms的持续低读取延迟；在所有其他指标中，Hyperflex比竞争解决方案领先两倍或更多。

结论


由独立的ESG实验室进行的测试不仅再次证明了Cisco Hyperflex解决方案的性能水平，而且还证明了超融合系统已准备好广泛用于关键任务。

长期以来，人们一直认为超融合系统更适合于非关键负载。 2016年，ESG对大公司进行了一项调查。 他们被问到为什么他们偏爱传统基础架构而不是超融合。 54％的受访者表示原因是生产力。

快进到2018年。 情况已经发生了变化：ESG的重复调查已经显示，只有24％的受访者仍然认为传统方法的效果仍然更好。

当技术的发展改变了行业决策标准时，客户想要的和他们能得到的往往不匹配。 能够看到缺失并填补这一空白的制造商将获得优势。 思科提供了一种超融合解决方案，该解决方案可提供简单性和成本效益以及始终如一的高性能，而这正是客户所缺乏的，并且客户需要执行关键任务的工作负载。

思科在超融合系统领域正在向前发展，不仅因为思科Hyperflex解决方案的出色功能，而且其在市场中的地位也得到了证实。 因此，根据Gartner的说法，在2018年秋季，思科当之无愧地跻身于HCI市场领导者之列。


现在，您已经可以通过访问我们在莫斯科和克拉斯诺达尔举行的演示来确保Hyperflex是解决最复杂和最苛刻业务任务的出色解决方案。
