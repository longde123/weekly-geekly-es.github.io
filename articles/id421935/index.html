Kesan pertama transisi dari Ubuntu 16.04 LTS ke Ubuntu 18.04 LTS

Seperti yang Anda tahu, perubahan visual utama dalam rilis 18 adalah ditinggalkannya Persatuan dan transisi ke Gnome 3. Di sini saya ingin berbagi kesan saya tentang transisi dari 16 ke 18

Pertama, tentang kebaikan ...


Rilis 18 membawa kita kernel Linux baru (4.15) dengan tambalan dari Meltdown dan Specter, yang diaktifkan secara default dan, menurut para ahli, harus sangat menurunkan kinerja komputer. Namun, terlepas dari kenyataan bahwa prosesor saya (Intel Core i5) termasuk dalam daftar yang rentan, yang berarti tambalan untuk itu disertakan, yang dikonfirmasi oleh log unduhan, dalam tugas-tugas biasa saya, saya tidak melihat adanya penurunan kinerja yang nyata. Oleh karena itu, jika komputer Anda bukan server yang berjalan di bawah beban tinggi yang konstan pada batas kemampuannya, sangat tidak layak untuk segera mematikan patch ini hanya karena takut akan â€œremâ€.

Antarmuka grafis gnome itu sendiri, menurut perasaan saya, bekerja jauh lebih cepat daripada sebuah unit. Saya percaya bahwa ini disebabkan oleh fakta bahwa ini berisi windows untuk membuka windows, dll. dll. hanya ada sedikit efek yang dapat dinonaktifkan di unit. Tetapi karena pekerjaan saya bukan untuk mempelajari antarmuka ubunta, tetapi atas inisiatif saya sendiri saya terlalu malas untuk melakukan ini, saya harus menggunakan apa yang â€œout of the boxâ€. Di luar kotak, gnome bekerja lebih cepat dari unit (atau, lebih tepatnya, gnome 3 di ubunt pada 04/18/01 melawan unit di ubunt pada 04/16). Namun, ini "secara signifikan lebih cepat" tidak berarti bahwa antarmuka unit mengerikan dan pengereman. Tidak semuanya. Kecepatan karyanya cukup bisa diterima. Hanya sedikit lebih cepat.

Saya juga sangat menyukai analog dari menu Start (kotak sembilan lingkaran di sudut kiri bawah). Terlepas dari kenyataan bahwa itu juga ubin, itu terbuka dengan cepat dan menunjukkan semua aplikasi yang diinstal dalam sistem. Bekerja dengannya sederhana dan nyaman. Di unit ini, saya praktis tidak menggunakannya sama sekali karena komponen khusus ini benar-benar sangat menghambat dan entah bagaimana menunjukkan tidak cukup apa yang saya harapkan untuk dilihat.

Setelah plus selesai, buka minus ...


1. Repositori pihak ketiga

Saat beralih ke versi baru, ubuntu biasanya menonaktifkan repositori pihak ketiga, dan kemudian harus dinyalakan kembali. Ini biasanya dilakukan dari antarmuka jendela. Namun, kali ini, tidak peduli berapa banyak saya menekan tanda centang, jendela hanya berubah abu-abu dan tidak ada yang terjadi. Saya harus mengedit file secara manual di / etc / apt /. Agak sepele, tentu saja ...

2. Pemberitahuan aplikasi telah hilang

Ketika di unit saya menerima surat baru dalam surat atau pesan dalam slack, ikon menggantung terus-menerus muncul di sudut kanan atas, ketika diklik, aplikasi yang sesuai dibuka. Baki sistem analog di windows. Saat beralih ke katai, semua pemberitahuan seperti itu menghilang. Yaitu, gnome memiliki mekanisme notifikasi tertentu ketika, ketika sebuah surat baru tiba selama 10 detik, sebuah kotak pesan muncul hampir di tengah layar. Tetapi bagaimana jika saat ini saya tidak di depan komputer? Atau bahkan hanya selama 10 detik ini tampak sebaliknya? Ya, dan mereka datang juga, tidak jelas bagaimana caranya. Tampaknya hanya memberi tahu tentang huruf pertama, dan semua yang berikutnya diabaikan.

Setelah beberapa menari dengan rebana, saya berhasil membuat Thunderbird untuk menampilkan lingkaran merah dengan jumlah huruf baru pada ikonnya di taskbar. Meskipun ini bukan baki sistem, ini adalah pengganti yang sepenuhnya setara. Dan saya punya alasan untuk percaya bahwa lingkaran merah ini seharusnya didapat tepat setelah instalasi, karena saya tidak melakukan sesuatu yang istimewa dalam tarian saya dengan rebana, dan pada akhirnya saya kembali ke tempat saya mulai. Dan klien email bukan pihak ketiga, tetapi yang default, itu harus dijilat di sepanjang dan di seberang. Tetapi untuk beberapa alasan, itu tidak berhasil untuk saya segera.

Pemberitahuan untuk slack tidak dapat dikonfigurasi. Dan mengingat ini adalah sarana utama komunikasi korporat kami, maka masalah adalah masalah (Jika seseorang tahu bagaimana menghadapi masalah ini, saya akan berterima kasih atas bantuannya)

Secara umum, untuk notifikasi minus besar

3. "Trik" berikutnya di terminal gnome

gnome-terminal adalah terminal default di unit, jadi ini bukan masalah pindah dari unit ke gnome. Ini adalah masalah beralih dari ubunt 16 ke 18.

Intinya adalah jika sebelumnya saya dapat membuat skrip yang menciptakan kembali lingkungan kerja saya, dari formulir 

gnome-terminal --maximize \ --tab --working-directory=$HOME/workspace/project1 -e "script" \ --tab -e "top" \ --tab -e "ssh -t user@host.ru" \ ... gnome-terminal --maximize \ --tab ...

sekarang desain ini bekerja secara acak, karena tab dari kedua terminal akan terbuka di tab jendela panggilan, yaitu, mereka akan ditumpuk semua dalam satu tumpukan. Plus, terminal akan membombardir Anda dengan pesan bahwa opsi -e sudah usang dan akan segera dihapus. Bahwa alih-alih "perintah" harus menggunakan konstruksi - perintah. Artinya, satu panggilan untuk membuka dua tab dengan koneksi ssh ke dua host yang berbeda akan menjadi masalah. Dan secara umum itu akan menjadi masalah untuk membuat jendela baru dari baris perintah dengan satu set tab sendiri (pada kenyataannya, saya pikir Anda dapat membuat jendela baru dan mendaftarkan serangkaian panggilan ke terminal dengan tab terpisah di perintah untuk itu, tetapi menurut saya ini ...)

Bagi sebagian besar pengguna, masalah ini, tentu saja, sepele, tetapi bagi mereka yang menggunakan terminal gnome secara langsung, itu sudah tidak menyenangkan ...

4. Taskbar dengan dua monitor

Ada dua monitor. Mereka bekerja dalam mode ketika masing-masing memiliki kontennya sendiri. Bilah tugas dikonfigurasikan untuk ditampilkan pada keduanya. Sekarang kita membuka dua jendela, misalnya, browser atau lingkungan pengembangan, dan membawanya ke layar yang berbeda. Penting bahwa kedua jendela memiliki aplikasi yang sama. Biarkan itu menjadi browser chrome. Di bilah tugas di sebelah kanan ikon krom, kita akan melihat dua lingkaran kecil yang menunjukkan bahwa dua salinan krom terbuka.

Bagaimana itu dalam kesatuan?

Jika saya mengeklik ikon chrome di bilah tugas kiri, sistem akan secara otomatis mengalihkan saya ke jendela chrome di layar kiri. Jika saya mengklik ikon di sebelah kanan, maka sesuai apa yang ada di sebelah kanan. Jika di jendela kiri saya membuka jendela lain dari chrome, yang ketiga, maka ketika saya mengklik ikon di panel kiri, sistem akan memberi saya pilihan jendela mana yang saya inginkan. Panel kanan masih akan segera beralih ke satu-satunya jendela, tanpa ada pertanyaan lebih lanjut. Sangat nyaman!

Bagaimana gnome sekarang?

Panel pada kedua layar benar-benar identik dan selalu menampilkan permintaan dalam bentuk thumbnail untuk memilih jendela mana. Selain itu, dari permintaan ini bermasalah untuk memahami jendela mana yang menjadi milik layar mana. Dan ketika bekerja dengan banyak jendela dari aplikasi yang sama, Anda harus membuat banyak klik yang tidak perlu.

Dengan demikian, jika tugas bekerja secara aktif pada dua (atau lebih) monitor relevan untuk Anda, saya akan merekomendasikan berpikir dua kali sebelum meninggalkan unit demi gnome.

Ringkasan


Secara umum, unit dan gnome cukup sebanding dalam kemampuan dan fungsinya. Apa yang harus dipilih lebih merupakan masalah preferensi pribadi. Saya membawa sesuatu yang segera menarik perhatian saya dan â€œmempersulitâ€ hidup saya secara pribadi ketika saya beralih ke katai. Manakah dari berikut ini yang relevan untuk Anda dan apakah relevan secara umum, terserah Anda untuk memutuskan.
