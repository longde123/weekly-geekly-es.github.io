Paquete acumulativo: ya puede crear aplicaciones

Rollup es una aplicación javascript de nueva generación y un generador de bibliotecas. Muchas personas lo conocen desde hace mucho tiempo como un generador prometedor que es muy adecuado para crear bibliotecas, pero poco adecuado para crear aplicaciones. Sin embargo, a medida que pasa el tiempo, el producto se está desarrollando activamente.

Lo probé por primera vez a principios de 2017. Inmediatamente me gustó por admitir la compilación en ES2015, los árboles, la falta de módulos en el ensamblaje y, por supuesto, una configuración simple. Pero luego era un producto en bruto, con una pequeña cantidad de complementos y una funcionalidad muy limitada, y decidí dejarlo para más adelante y seguí construyendo a través de browserify. El segundo intento fue en 2018, luego ya había superado significativamente la comunidad, los complementos y la funcionalidad, pero aún no había suficiente calidad en algunas funciones, incluido el observador. Y finalmente, a principios de 2019, podemos decir con seguridad: con la ayuda de Rollup, puede crear aplicaciones modernas de manera simple y conveniente.

Para comprender los beneficios, repasemos las características clave y comparemos con Webpack (la situación es la misma para Browserify).

Configuración simple


Inmediatamente lo que llama la atención es una configuración muy simple y comprensible: 

export default [{ input: 'src/index.ts', output: [{ file: 'dist/index.min.js', format: 'iife' }], plugins: [ // todo:     ], }];

Ingrese rollup -c en el cosnol y su paquete comenzará a ensamblarse. Para la exportación, puede proporcionar una variedad de paquetes para el ensamblaje, por ejemplo, si recolecta por separado los polífilos, varios programas, trabajadores, etc. En la entrada, puede enviar una matriz de archivos, luego se recopilarán fragmentos. En la salida, puede alimentar una matriz de archivos de salida y ensamblarlos en diferentes sistemas modulares: iife, commonjs, umd.

Soporte de vida


Soporte para ensamblar en la propia función llamada sin módulos. Para comprender, tomemos el programa más famoso: 

 console.log("Hello, world!");

conduzca a través de Rollup a formato iife y vea el resultado: 

 (function () { 'use strict'; console.log("Hello, world!"); }());

La salida es un código muy compacto, solo 69 bytes. Si aún no comprende la ventaja, Webpack / Browserify compilará el siguiente código:

Compilación en ES2015


El paquete acumulativo de nombre "coleccionista de próxima generación" en 2016 recibió la capacidad de ensamblar en ES2015. Y hasta finales de 2018, fue el único coleccionista que sabía cómo hacer esto.
Por ejemplo, si toma el código: 

 export class TestA { getData(){return "A"} } console.log("Hello, world!", new TestB().getData());

y conducir a través de Rollup, luego en la salida obtenemos lo mismo. Y si! A principios de 2019, ya el 87% de los navegadores pueden ejecutarlo de forma nativa.

Luego, en 2016, parecía un gran avance, porque había una gran cantidad de aplicaciones que no necesitaban soporte para navegadores antiguos: administradores, quioscos, no aplicaciones web, y no había herramientas de compilación para ellos. Y ahora con Rollup podemos recopilar varios paquetes en una sola pasada, en es3, es5, es2015, exnext y, según el navegador, descargue el necesario.

También una gran ventaja de ES2015 es su tamaño y velocidad de ejecución. Debido a la falta de transpiración a una capa inferior, el código resulta ser mucho más compacto, y debido a la falta de un código auxiliar generado por los transpiladores, este código también funciona 3 veces más rápido (según mis pruebas subjetivas).

Sacudida del árbol


Este es un chip Rollup, ¡él lo inventó! Webpack ha estado intentando implementarlo durante muchos años seguidos, pero solo con la versión 4 comenzó a funcionar algo. A Browserify le está yendo muy mal.
¿Qué tipo de animal es este? Tomemos los siguientes dos archivos como ejemplo: 

 // module.ts export class TestA { getData(){return "A"} } export class TestB { getData(){return "B"} } // index.ts import { TestB } from './module'; const test = new TestB(); console.log("Hello, world!", test.getData());

ejecutar a través de Rollup y obtener: 

 (function () { 'use strict'; class TestB { getData() { return "B"; } } const test = new TestB(); console.log("Hello, world!", test.getData()); }());

Como resultado de TreeShaking, el código muerto se descartó incluso en la etapa de resolución de dependencias. Gracias a lo que los conjuntos de rollup resultan mucho más compactos. Ahora veamos qué genera Webpack:

Complementos


Solo el ES2015 + desnudo puede ensamblarse desde una caja de Rollup. Para enseñarle funcionalidades adicionales, como conectar commonjs, módulos de mecanografía, cargar html y scss, etc., debe conectar complementos.

Esto se hace de manera muy simple: 
 import nodeResolve from 'rollup-plugin-node-resolve'; import commonJs from 'rollup-plugin-commonjs'; import typeScript from 'rollup-plugin-typescript2'; import postcss from 'rollup-plugin-postcss'; import html from 'rollup-plugin-html'; import visualizer from 'rollup-plugin-visualizer'; import {sizeSnapshot} from "rollup-plugin-size-snapshot"; import {terser} from 'rollup-plugin-terser'; export default [{ input: 'src/index.ts', output: [{ file: 'dist/index.r.min.js', format: 'iife' }], plugins: [ nodeResolve(), //   node commonJs(), //   commonjs postcss(), //   postcc,    scss  less html(), //  html  typeScript({tsconfig: "tsconfig.json"}), //  typescript sizeSnapshot(), //      terser(), //    ES2015+,    UglifyES visualizer() //   ] }];

Tan simple, un complemento está conectado en una palabra. Esta configuración puede recopilar cualquier aplicación compleja e incluso genera un análisis de paquete en la salida.

Resumen


Y ahora, armados con nuevos conocimientos, hagamos una configuración que recopile por separado los polífilos, una aplicación separada para navegadores antiguos y nuevos, un trabajador de servicios para pwa y un trabajador web para cálculos complejos en general. 

 import nodeResolve from 'rollup-plugin-node-resolve'; import commonJs from 'rollup-plugin-commonjs'; import typeScript from 'rollup-plugin-typescript2'; import postcss from 'rollup-plugin-postcss'; import html from 'rollup-plugin-html'; import visualizer from 'rollup-plugin-visualizer'; import { sizeSnapshot } from "rollup-plugin-size-snapshot"; import { terser } from 'rollup-plugin-terser'; const getPlugins = (options) => [ nodeResolve(), commonJs(), postcss(), html(), typeScript({ tsconfig: "tsconfig.json", tsconfigOverride: { compilerOptions: { "target": options.target } } }), sizeSnapshot(), terser(), visualizer() ]; export default [{ input: 'src/polyfills.ts', output: [{ file: 'dist/polyfills.min.js', format: 'iife' }], plugins: getPlugins({ target: "es5" }) },{ input: 'src/index.ts', output: [{ file: 'dist/index.next.min.js', format: 'iife' }], plugins: getPlugins({ target: "esnext" }) },{ input: 'src/index.ts', output: [{ file: 'dist/index.es5.min.js', format: 'iife' }], plugins: getPlugins({ target: "es5" }) },{ input: 'src/index.ts', output: [{ file: 'dist/index.es3.min.js', format: 'iife' }], plugins: getPlugins({ target: "es3" }) },{ input: 'src/serviceworker.ts', output: [{ file: 'dist/serviceworker.min.js', format: 'iife' }], plugins: getPlugins({ target: "es5" }) },{ input: 'src/webworker.ts', output: [{ file: 'dist/webworker.min.js', format: 'iife' }], plugins: getPlugins({ target: "es5" }) }];
