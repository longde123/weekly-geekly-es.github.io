Atlassian Jira Software的常规冲刺

大家好！

在本文中，我想谈一谈Atlassian Jira Software中的Shared Sprint。

例如，如果您在此处阅读官方文档，则不会有一般的冲刺之类的事情。

我首先在这里遇到了“通用冲刺”一词。 本文很好地解释了什么是常见的冲刺以及如何与它们一起生活。

共享sprint是Atlassian Jira Software中的一个重要概念，因为对共享sprint的了解通常是意料之外的，而且似乎出了点问题。 但是事实并非如此，如果您知道常见的sprint是如何工作的，则可以根据需要使用它们。

在本文中，我将说明常见的Sprint外观以及如何将它们与常规Sprint区分开。 我还将介绍它们的功能。

我在Jira Software Cloud和Jira Software Server 7.12.3上尝试了本文中的所有示例。

什么是常见的冲刺？


常见的冲刺是在多个板上可见的冲刺。

例如，以下是木板的屏幕截图：

SCRUM板 ：



SCRUM2板 ：



您可以看到SCRUM和SCRUM2板上有一个名为SCRUM Sprint 3的冲刺。该冲刺在两个板上都可见。 这是否意味着我们看到了一个共同的冲刺？ 不行 Jira Software可以具有相同名称的两个不同的Sprint。

如何查看Sprint ID？


为了了解常见的Sprint是摆在我们面前还是普通的Sprint，我们必须查看这两个Sprint的ID。

您可以看到这样的Sprint ID：



如果将鼠标悬停在上方红色突出显示的按钮上，我们将看到一个以sprintId = <number>结尾的网址。 此数字是冲刺ID。 在我们的例子中，两个Sprint的ID不同，这意味着我们面前不是一个普通的Sprint，而是两个普通的Sprint。

一般冲刺


现在让我们看一下这个屏幕截图：



在屏幕截图中，我们看到了SCRUM板，上面有两个具有相同名称的sprint。 并且以红色突出显示的sprint包含与SCRUM2板上的sprint相同的票证。 如果我们在SCRUM和SCRUM2板上检查此sprint的ID，则该ID将重合，这意味着我们有一个共同的sprint。

为什么我们在两个板上的冲刺相同？


板上可见的票证是通过为板上定义的过滤器选择的。 可以通过转到董事会设置->常规来查看过滤器。 这是板过滤器的外观：

SCRUM板 ：

project = SCRUM OR priority is not EMPTY ORDER BY Rank ASC

SCRUM2板 ：

project = SCRUM2 ORDER BY Rank ASC

我们看到SCRUM过滤器不仅从SCRUM项目中选择票证，而且还选择了Jira实例中所有优先级已填充的票证，这意味着它从SCRUM2项目中选择了票证。 因此，SCRUM2项目的票证在SCRUM板上和SCRUM2板上均可见。 因此，如果我们在票证之一中填写“ Sprint”字段，则此Sprint将出现在两个面板上。

冲刺是从电路板上创建的，并且包含指向创建它的电路板的链接。 为此，您可以执行rest / agile / 1.0 / sprint / sprintId，并查看创建sprint的电路板。 在我们的例子中，我们得到以下结果： 

 {"id":5,"self":"https://alex.atlassian.net/rest/agile/1.0/sprint/5","state":"active","name":"SCRUM Sprint 3","startDate":"2018-09-23T06:33:01.266Z","endDate":"2018-10-07T06:33:00.000Z","originBoardId":3,"goal":""}

originBoardId = 3，这是SCRUM2。 这意味着该sprint最初是在SCRUM2板上创建的，并出现在SCRUM板上，因为在SCRUM2板上和SCRUM板上都有来自该sprint的票证。

常见的冲刺表现如何？


如果您在可见该冲刺的任何板上对常规冲刺进行了任何更改，则这些更改将在所有板上均可见。

例如，如果我们在SCRUM板上重命名sprint，那么它也将在SCRUM2板上重命名。 如果我们关闭SCRUM板上的sprint，则它将关闭SCRUM板上。

用户通常会遇到这种行为。 他们与冲刺一起工作，突然冲刺关闭。 没有团队关闭它。 结果，事实证明该冲刺是从另一个板关闭的，其中筛选器从该项目中选择票证。

如何使用常见的冲刺？


假设我们有几个团队，每个团队都有自己的项目。 每个团队都创建一个董事会并在此董事会上工作。 我们希望看到所有团队的冲刺。 我们该怎么做？

  • 我们可以要求为我们共享所有板，然后如果转到正确的板，则可以看到冲刺。
  • 我们可以创建一个董事会，从所有团队项目中选择票证。 在这种情况下，我们将在一个板上查看所有冲刺的信息，这将使我们减少查看所有团队冲刺的工作量。 但是我们不能忘记，董事会上任何带有冲刺的动作都将导致团队董事会进行相同的更改。

这就是我在这篇文章中想要说的。 我希望这些信息能派上用场。
