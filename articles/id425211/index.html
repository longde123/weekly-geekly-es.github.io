286 dan jaringan



Saya sudah tua. Saya memiliki lemari penuh besi tua. Dari chip logika Boolean dalam paket DIP ke Voodoo5. Tentu saja, semua ini tidak mewakili nilai praktis apa pun, tetapi menyenangkan bagi sebagian orang untuk mengacaukan potongan-potongan besi lama. Jika Anda salah satu dari mereka, saya mengundang Anda ke kucing, di mana saya akan memberi tahu Anda bagaimana komputer yang didasarkan pada prosesor AMD 286 adalah "teman" dengan jaringan modern, dan apa yang terjadi.

Gagasan untuk mengencangkan jaringan ke 286m telah saya miliki sejak lama, karena mentransfer data ke komputer adalah cara termudah melalui jaringan, dan memang, ini menarik. Apakah TCP / IP berfungsi? Apakah browser web berfungsi? Meskipun orang yang menjalankan dos 286m sudah meluncurkan browser di bawah dos, saya belum pernah melihat yang ke 286 di bawah Windows di Internet. Nah, tujuan utamanya adalah untuk terhubung ke NAS rumah, yang menyimpan semua perangkat lunak untuk besi lama saya. Dan NAS lebih atau kurang modern, dan hanya TCP / IP yang mampu.

Saya sudah memiliki pengalaman menghubungkan komputer lama ke drive ini. Termasuk pada saat menulis artikel sebelumnya tentang besi tua. Tetapi dalam artikel sebelumnya saya menganggap hanya prosesor 32-bit, dan semuanya cukup mudah di sana: bahkan Windows 386 menjalankan Windows 95 (ya, sangat lambat, ya, instalasi membutuhkan waktu lebih dari 9 jam, tetapi berfungsi), yang memiliki tumpukan TCP asli / IP dan banyak driver untuk semua jenis kartu jaringan yang disertakan. Jika Anda ingin lebih cepat, yaitu, Windows for Workgroups 3.11, yang, setelah menginstal Win32s, Anda dapat menggulung tumpukan TCP / IP Anda sendiri. Dan, meskipun tidak dapat masuk ke server file modern, folder terbuka tanpa kata sandi menjadi cukup mudah diakses untuk PC, bahkan dalam konfigurasi ini. Dan Internet Explorer 5 bahkan berjalan pada 386SX, cukup memori!


Saya mencuri gambar dari Google

Namun sayang, untuk 286 semua ini tidak berlaku. Kesulitan utama adalah bahwa prosesor 286 adalah 16-bit, dan Windows for Workgroups 3.11 tidak tersedia untuk itu, karena mode Standar yang memungkinkan pendahulunya 3.1 dijalankan pada prosesor 16-bit terputus dari 3.11. Ekstensi Win32s juga tidak tersedia (untuk alasan yang jelas), dan karenanya, tidak mungkin untuk menginstal tumpukan Microsoft TCP / IP asli.

Jadi, kita kembali ke DOS.

Setelah google berpikir singkat, gambar dengan TCP / IP untuk DOS menjadi jauh lebih jelas daripada sebelumnya. Pada dasarnya, itu semua bermuara pada kenyataan bahwa aplikasi yang berfungsi baik di bawah DOS itu sendiri memiliki tumpukan sendiri, dan semua yang dibutuhkan adalah driver paket untuk kartu jaringan. Tetapi saya tidak dapat menemukan aplikasi yang memungkinkan Anda memasang drive jarak jauh.
Namun, saya telah menemukan 2 klien jaringan yang berbeda untuk DOS dari Microsoft itu sendiri (MS Client dan MS LAN Manager), dan binatang buas yang sangat aneh bernama Windows for Workgroups 3.1. Ya, itu 3.1, bukan 3.11, dan itu berarti bahwa mungkin semuanya akan bekerja dalam mode standar.
Itu akan sangat menarik. Tentu saja, saya tidak berharap melihat tumpukan TCP / IP di 3.1, tetapi juga di Google saya menemukan deskripsi tumpukan TCP / IP 16-bit dari Microsoft. Menarik.
Setelah membeli kit distribusi 3.1 di eBay, saya memulai instalasi.


Saya mencuri gambar dari Google

Sistem tersebut ternyata sedikit dimodifikasi 3.1 dengan antarmuka jaringan built-in untuk DOS, sangat mirip dengan MS LAN Manager yang dilucuti. Semua perbedaan dari 3.1 tradisional dikurangi menjadi adanya tombol "connect network drive" di File Manager dan kotak dialog yang sesuai. Menyiapkan jaringan dari bawah Windows sendiri tidak dimungkinkan. Cara mengkonfigurasi dari DOS tidak jelas. Meskipun, setelah menginstal sistem, dia mengidentifikasi kartu jaringan saya sebagai NE1000 atau kompatibel, yang tidak jauh dari kebenaran, karena kartu Genius GR1222 yang saya gunakan benar-benar sama. Tapi saya punya disk driver asli untuk kartu ini, termasuk di bawah DOS. Namun tetap tidak dapat dipahami, bagaimana mengkonfigurasi driver untuk gangguan dan alamat I / O kartu.

Selain itu, setelah membaca Google, ternyata driver untuk MS LAN Manager dan paket driver adalah dua hal yang berbeda.

Situasi dengan driver di bawah DOS mulai menjernihkan sedikit.

Jadi ada 3 standar besar:

  1. ODI Novell menggunakan model driver ini untuk berkomunikasi dengan server Netware-nya. Saya tidak punya makhluk, jadi tidak akan berhasil.
  2. NDIS Ini adalah model Microsoft untuk LAN Manager dan Windows mereka. Secara teori, apa yang Anda butuhkan.
  3. Driver batch yang banyak digunakan aplikasi DOS.
