Mengumumkan .NET Core 3.1 Pratinjau 2

Kami mengumumkan .NET Core 3.1 Pratinjau 2 . .NET Core 3.1 akan menjadi rilis kecil dan pendek yang berfokus pada peningkatan utama pada Blazor dan desktop Windows, dua tambahan besar dalam .NET Core 3.0. . Ini akan menjadi rilis dukungan jangka panjang (LTS) dengan perkiraan tanggal pengiriman akhir Desember 2019.

Anda dapat mengunduh .NET Core 3.1 Preview 2 di Windows, macOS, dan Linux.

  • .NET Core 3.1 Pratinjau 2 SDK dan Runtime
  • Gambar Docker

ASP.NET Core dan EF Core juga merilis pembaruan hari ini.

Visual Studio 16.4 Pratinjau 3 dan Visual Studio untuk Mac 8.4 Pratinjau 3 juga dirilis hari ini. Mereka diperlukan pembaruan untuk menggunakan .NET Core 3.1 Pratinjau 2. Visual Studio 16.4 termasuk .NET Core 3.1, jadi hanya memperbarui Visual Studio akan memberikan Anda berdua rilis.

Detail:

  • Catatan rilis .NET Core 3.1
  • Rilis Github
  • Masalah Github untuk pratinjau 2



Perbaikan


Peningkatan terbesar dalam rilis ini adalah dukungan untuk C ++ / CLI (AKA "managed C ++"). C ++ / CLI hanya diaktifkan di Windows. Perubahan untuk C ++ / CLI terutama di Visual Studio. Anda perlu menginstal beban kerja "Pengembangan desktop dengan C ++" dan komponen "Dukungan C ++ / CLI" untuk menggunakan C ++ / CLI. Anda dapat melihat komponen ini dipilih (ini adalah yang terakhir ditampilkan) pada gambar di bawah ini.



Komponen ini menambahkan beberapa templat yang dapat Anda gunakan:

  • Perpustakaan Kelas CLR (.NET Core)
  • Proyek Kosong CLR (.NET Core)

Jika Anda tidak dapat menemukannya, cukup cari di dialog Proyek Baru.

Penutupan


Tujuan utama .NET Core 3.1 adalah untuk memoles fitur dan skenario yang kami kirimkan di .NET Core 3.0. .NET Core 3.1 akan menjadi rilis dukungan jangka panjang (LTS) , didukung setidaknya selama 3 tahun.

Silakan instal dan uji .NET Core 3.1 Pratinjau 2 dan beri kami umpan balik. Itu belum didukung atau direkomendasikan untuk digunakan dalam produksi.

Jika Anda melewatkannya, lihat pengumuman .NET Core 3.0 dari bulan lalu.
