Idiomas transcompilados: proyectos de conversión de código a código


Fuente: Ward Cunningham

La transpilación es la conversión de código de un idioma a otro. Usando un transpilador especial (transcompilador), un lenguaje de programación de propósito general se puede transferir a cualquier otro lenguaje de programación de propósito general.

Si necesita cambiar a otro idioma, los transpiladores ayudarán a los desarrolladores a guardar la mayor parte del código existente sin tener que volver a escribir manualmente el código completo. Por ejemplo, al convertir programas de Python 2 a Python 3, o al mover software de la API anterior a la nueva.

Los términos transpilador y transcompilador a menudo se usan indistintamente, pero aún se cree que existen diferencias. Por ejemplo, para convertir el código C ++ a C, necesita un transpilador, y para convertir Python-Ruby, necesita un transcompilador. Babel para JavaScript es un transpilador, y TypeScript es un lenguaje transcompilado.

A veces, después de la conversión por el transportista, para el correcto funcionamiento del código, se requiere una configuración manual, mientras que en los idiomas transcompilados el código debe funcionar sin cambios. Hoy hablaremos sobre esos y otros, pero lo más importante, hablaremos sobre proyectos trans populares que los desarrolladores pueden usar en su trabajo diario.

Transpilación frontal


La conversión es una parte importante del desarrollo front-end: a medida que las nuevas características aparecen lentamente en los navegadores, se han creado idiomas con capacidades experimentales que se traducen a idiomas compatibles con el navegador.

Los desarrolladores de navegadores que usan su propio motor de JavaScript en cada proyecto también son responsables del auge de los idiomas trans para JavaScript. Como resultado, los navegadores tienen un rendimiento diferente, implementan su propio conjunto de funciones de JavaScript y se acercan al cumplimiento total de las especificaciones a diferentes velocidades.



Sass , Less y Stylus facilitan la creación de código CSS. Por lo tanto, Sass tiene una sintaxis flexible, lógica (@if, cada una), matemáticas (puede agregar números, líneas y colores), por lo que toda la riqueza del código se transforma en un programa CSS regular de rubíes.

Less (Leaner Style Sheets) tiene el mismo aspecto que CSS, pero tiene adiciones convenientes: variables, mixins, operaciones aritméticas (como Sass), etc.

Stylus está escrito en Node.js y difiere de Sass y Less en una sintaxis más limpia. Pero, de hecho, estos tres idiomas son muy similares entre sí, por lo que no se equivocará si selecciona alguno de ellos.

TypeScript es un proyecto competitivo para JavaScript, un lenguaje de secuencias de comandos con la adición de escritura estática opcional. Le permite describir de manera más completa las propiedades y métodos de objetos y clases, a diferencia de JS, eliminando la necesidad de verificar todos los argumentos que se incluyen en un método o función.

Existe la opinión de que JavaScript es un lenguaje con fallas fundamentales. De lo contrario, es difícil explicar la popularidad de los trans-idiomas que trabajan específicamente para corregir sus deficiencias. Además de los proyectos anteriores, también hay un lenguaje de programación web flexible Dart , un compilador de JavaScript en el compilador de cierre de JavaScript y muchas otras soluciones, que puede encontrar en la página " Lista de idiomas que compilan en JS ".

Convierte a C




En 2006, apareció el proyecto Vala , que consta de dos lenguajes: Vala y Genie, que traducen el código C más común, que, a su vez, se compila en un archivo o biblioteca ejecutable.

La sintaxis de Vala es muy similar a C #, pero está adaptada para una mejor compatibilidad con los sistemas GObject, mientras que Genie tiene mucho en común con Python. El primer idioma ahora es más común que Genie, así que echemos un vistazo más de cerca.

¿Por qué necesitabas tal proyecto? Muchos desarrolladores querían escribir aplicaciones y bibliotecas para GNOME en lenguajes de alto nivel, pero por diversas razones no podían o no querían usar C # o Java. Vala le permite crear rápidamente código complejo orientado a objetos, que admite C API estándar y ABI con bajos requisitos de memoria.

Las bibliotecas C # y Java no pueden usarse de la misma manera que las bibliotecas nativas de GObject de C y otros lenguajes, y no pueden considerarse parte de la plataforma GNOME. Valac, el transcompilador de Vala, crea archivos C a partir de archivos fuente de Vala como si escribiera su biblioteca o aplicación directamente en C. Usar la biblioteca Vala de la aplicación C no es diferente de usar cualquier otra biblioteca basada en GObject.

De Flash a multiplataforma




Haxe se creó al mismo tiempo que Vala, pero está destinado a transpilarse en Flash, JavaScript y Neko. Con tal conjunto de habilidades, el lenguaje estaba condenado a la extinción, pero no solo sobrevivió, sino que también ganó gran popularidad.

El secreto del éxito es multiplataforma. Con el tiempo, Haxe se ha convertido en un poderoso conjunto de herramientas que admiten la transcompilación en varios lenguajes y plataformas, incluidos JavaScript, C ++, C #, Java, JVM, Python, Lua, PHP y Flash.

Haxe es un lenguaje de programación moderno, de alto nivel y fuertemente tipado. Al mismo tiempo, es bastante simple y práctico, por lo que la compilación se puede adaptar a varias plataformas de destino. Su sintaxis es básicamente el estándar ECMAScript, pero se cambia si es necesario.

"Hola mundo" en Haxe: 

 class Main { static public function main():Void { trace("Hello World"); } }

En JavaScript, estas líneas de código se verán así: 

 haxe -main HelloWorld -js HelloWorld.js

Y en Java así: 

 haxe -main HelloWorld -java path/to/java/out

La mayoría del código Haxe está organizado en clases y funciones, lo que lo convierte en un lenguaje OOP que se parece a Java, ActionScript 3 y C #. Sin embargo, Haxe fue diseñado con énfasis en el pragmatismo y la facilidad de sintaxis.

Y sobre los transpiladores


  • C2Rust y Corrode son dos oportunidades no impecables para migrar de C a Rust.
  • VOC es un transportador de Python 3.4+ a Java que lo ayudará a crear aplicaciones de Android basadas en el código de Python.
  • Lombok es una solución interesante para quienes no les gusta Java, o no les gusta nada más que Java. Este es un complemento de compilación que agrega nuevas "palabras clave" a Java y convierte las anotaciones en código Java, lo que reduce los esfuerzos de desarrollo y proporciona funcionalidad adicional. Por ejemplo, permite el uso de la palabra clave val (un análogo de var de C #) en el código Java normal.
  • Bridge.NET le permite usar el rendimiento de C # en JavaScript, así como el potente IDE de VisualStudio y las herramientas estándar de .NET (como msbuild, refactorización, pruebas unitarias, análisis estático, visualización de código, FxCop).
  • Grumpy traduce el código de Python en una representación de Go y le permite ejecutar sin problemas programas de Python en entornos de tiempo de ejecución de Go. El proyecto elimina el problema del bloqueo global del intérprete, que no permite la ejecución paralela de varios hilos de código.

Por supuesto, hay muchos otros transpiladores. Tradicionalmente, puedes hablar sobre tus herramientas favoritas en los comentarios.

Conclusión


Hay una gran cantidad de otros idiomas, por lo que para su proyecto siempre puede elegir el más adecuado. Pero las situaciones en las que se necesita la conversión de código a código no son infrecuentes, ya que las transcripciones de C ++ a C, PHP a C ++ y "cualquier cosa en JavaScript" ocurren con bastante frecuencia.

Un "traductor" de un idioma a otro es una buena idea, pero solo mientras el resultado pueda predecirse claramente y usarse sin riesgo. Entonces, la transición de COBOL a Java fue difícil, porque en Java antes de JDK 1.4 no había archivos mapeados en memoria . Y el C2Rust y Corrode mencionados en el artículo dan un código no idiomático, marcado como inseguro y lleno de punteros en bruto.
