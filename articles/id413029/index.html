Solusi OpenShift Terkelola pertama di cloud

Halo, Habr! Pada Red Hat Summit 2018 di San Francisco, bersama dengan Red Hat, kami memperkenalkan OpenShift pada platform Azure, platform kontainer OpenShift pertama yang dikelola sepenuhnya dan mudah digunakan sebagai layanan cloud. Hari ini kami akan membicarakannya secara lebih rinci dan memberi tahu Anda cara mendapatkan lebih banyak orang dalam. Lihat di bawah kucing!



Setelah OpenShift Dedicated on Azure diumumkan tahun lalu, Microsoft dan Red Hat menciptakan solusi baru berdasarkan umpan balik pengguna. Ini menawarkan pelanggan lebih dari versi awal produk. OpenShift Dedicated adalah solusi yang sepenuhnya dikelola dan dapat disesuaikan yang membuatnya mudah dan nyaman untuk digunakan pada Azure.

Apa artinya "sepenuhnya dapat dikendalikan"? Red Hat dan Microsoft akan bersama-sama mengembangkan, mengelola, dan mendukung OpenShift on Azure. Menjadi layanan yang dikelola sepenuhnya, itu akan menerima dukungan berkelanjutan sebagai proyek tunggal, yang akan memberikan dukungan pengguna yang komprehensif yang tidak kalah kualitasnya dengan dukungan Azure. Menurut pelanggan kami, saat menggunakan solusi cloud OpenShift lainnya, mereka tidak memiliki keuntungan yang timbul dari penggunaan layanan dalam satu penyedia. Kami senang dapat memenuhi kebutuhan penting ini.

Terlepas dari kenyataan bahwa hari ini OpenShift digunakan di banyak lingkungan cloud, OpenShift on Azure yang baru-baru ini diumumkan telah menjadi lebih mudah dikonfigurasi dan digunakan dengan mekanisme integrasi Azure asli, Anda juga dapat menggunakan Kubernetes, Layanan Azure Kubernetes yang mendasarinya.

Jika Anda ingin membuat cluster OpenShift baru, Anda tidak perlu membuat permintaan layanan dan menghabiskan beberapa jam menunggu komponen jaringan untuk terhubung (seperti yang terjadi di cloud lain). Sebagai gantinya, Anda bisa menggunakan Azure CLI untuk menjalankan berbagai perintah:



Mendapatkan cluster baru dalam hitungan menit lebih cepat dan lebih mudah daripada dengan solusi cloud publik lainnya yang tersedia sekarang.

Broker Layanan Terbuka untuk Azure (OSBA) juga bekerja dengan OpenShift untuk Azure, termasuk Azure Stack, dan secara otomatis mendeteksi dan menampilkan daftar layanan Azure yang tersedia seperti Cosmos DB, Azure KeyVault, dll. OSBA meningkatkan efektivitas semua yang Anda gunakan menggunakan OpenShift on Azure, memungkinkan Anda untuk dengan mudah mengintegrasikan objek lain yang dipindahkan ke Azure. Ini adalah fitur lain yang nyaman yang memungkinkan Anda meluncurkan proyek dengan cepat, serta menghubungkannya dengan proyek lain yang sudah ada atau yang direncanakan untuk diluncurkan di Azure.



Apa yang lebih menyenangkan bagi pelanggan, semua ini hadir dengan dukungan yang diumumkan baru-baru ini untuk Azure Stack dan Windows Container untuk platform OpenShift Container lokal, yang sangat cocok untuk pelanggan korporat yang ingin menggunakan platform hybrid untuk wadah cloud, yang karyanya secara simultan didukung baik di RHEL maupun di Windows Pelajari cara menggunakan Platform OpenShift Container di Azure Stack sekarang.

Ketersediaan


Solusi Red Hat OpenShift Azure diharapkan akan tersedia dalam beberapa bulan mendatang sebagai demo terbuka. Platform Red Hat OpenShift Container dan Red Hat Enterprise Linux pada Azure dan Azure Stack saat ini tersedia.

Jika Anda ingin wawasan lebih lanjut


Pada 16 Juni, di Moskow pada konferensi DevOps Days , Microsoft, bersama Red Hat, akan mengadakan kelas master "DevOps, Red Hat, Microsoft, dan semuanya." Di kelas master, kami akan menunjukkan seberapa cepat dan indah tugas-tugas harian Anda diselesaikan di Platform Red Hat OpenShift Container di Microsoft Azure.

Apa yang akan terjadi


  • Mekanisme source2image di OpenShift. Anda (dan kolega Anda di tim DevOps) cukup mengambil tautan ke git dan memberikannya OpenShift, lalu dia akan melakukan semuanya sendiri: ia akan mengumpulkan dan menggunakan aplikasi sendiri, menunjukkan metrik, log, dan skala pada node yang diberikan ke nilai yang ditentukan. Pengembang tidak perlu repot dengan apa dan bagaimana melakukannya di tingkat infrastruktur, mereka hanya terus melakukan pekerjaan biasa, menggunakan cara biasa - mereka menulis kode, mengkloning repositori mereka - dan menunggu sampai mereka berkumpul.
  • Akuntansi dan kontrol. Menyebarkan WordPress dari gambar wadah untuk membuat selebaran untuk kucing Anda. Kami menunjukkan bagaimana wadah kami tidak memulai begitu saja, karena itu membutuhkan hak istimewa. Kami akan memberi tahu dan menunjukkan bagaimana situasi keamanan dengan contoh ini dan memberikan wadah hak istimewa khusus, kemudian kita akan melihat bagaimana itu dimulai, mengikat database untuk itu dan membuat posting blog super pertama.
  • Kotak Schrödinger berwarna atau Penempatan Biru / Hijau melalui pipa Jenkins yang kami kembangkan dalam satu (tapi tidak persis) klik di OpenShift - skenario bulat dalam ruang hampa udara yang menunjukkan bagaimana membuat versi baru aplikasi tersedia untuk pengguna dengan downtime nol dan tanpa kehilangan saraf baik Dev dan Ops.

Presenter


Ivan Fateev adalah penginjil teknologi di Microsoft Russia. Ivan berspesialisasi dalam arsitektur aplikasi modern dan praktik terbaik dalam pengembangan perangkat lunak. Ivan memiliki pengalaman luas dalam pengembangan server dan pengembangan klien. Dia memulai karirnya sebagai pengembang web pada tahun 2008. Kemudian pada 2010, ia beralih ke pengembangan aplikasi iOS di sisi klien, lalu game. Saat ini, Ivan berspesialisasi dalam mengembangkan aplikasi menggunakan teknologi cloud.
