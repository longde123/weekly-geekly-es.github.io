"Generación de código sin kapt", Mikhail Rozumyansky, Joom


Sobre el informe

Hace unos años, comenzamos a trabajar con bytecode, generándolo y modificándolo en tiempo de compilación, ya que APT para Kotlin no era compatible. Después del advenimiento de kapt, lo agregamos a nuestro proyecto para usar DataBinding y, como resultado, encontramos muchos problemas: ensamblaje largo, errores de compilación extraños, pérdidas de memoria. Rápidamente, quedó claro que era mejor no contactar a APT, por lo que continuamos desarrollando nuestros proyectos que funcionan con bytecode. Y no hace mucho tiempo, logramos ejecutar DataBinding sin kapt, y así eliminamos completamente kapt en el proyecto. En este informe, hablaremos sobre cómo vivir sin kapt, qué tareas se pueden resolver a nivel de bytecode, y también consideraremos los pros y los contras de los diferentes enfoques para la generación de código.

"Configuración de aplicación controlada por modelo en Kotlin DSL", Andrey Govorovsky, Grupo Mail.ru


Sobre el informe

Es difícil imaginar una aplicación móvil moderna sin una configuración remota: pruebas A / B, lanzamiento de nuevas funciones, reemplazo de cadenas, recursos gráficos.

En el curso del informe, veremos cómo creamos DSL en Kotlin en la aplicación Mail.ru Mail para describir la configuración de la aplicación, cómo se generaron los analizadores con validación para reducir el código repetitivo, la documentación sobre las características de las pruebas A / B, la configuración del desarrollador para los probadores y cómo está todo integrado en el proceso de ensamblaje.

¿Por qué se necesita todo esto? Como llevamos a cabo muchas pruebas A / B, necesitábamos una única fuente de información sobre las funciones en una versión específica de la aplicación.

Te diré cómo escribimos un DSL Kotlin simple para describir el modelo de configuración de la aplicación.

Como más tarde a partir de este DSL, generamos configuraciones de modelos / analizadores / base / desarrollador (para cambiar la configuración sobre la marcha) y lo incorporamos a la aplicación.

Cómo se usa el dock para la configuración remota (el dock genera ejemplos json para características que se pueden enviar a la aplicación).

Le diré cómo incrustarlo todo en el proceso de compilación a través de la tarea gradle y hacerlo incremental (@InputFile, @OutputDirectory).
