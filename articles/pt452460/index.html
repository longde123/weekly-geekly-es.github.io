Apresentando o Terminal do Windows

Estamos muito animados para anunciar o Windows Terminal! O Windows Terminal é um aplicativo de terminal novo, moderno, rápido, eficiente, poderoso e produtivo para usuários de ferramentas e shells de linha de comando, como prompt de comando, PowerShell e WSL.



O Terminal do Windows será entregue via Microsoft Store no Windows 10 e será atualizado regularmente, garantindo que você esteja sempre atualizado e possa aproveitar os recursos mais recentes e as melhorias mais recentes com o mínimo esforço.



Principais recursos do Terminal do Windows


Múltiplas guias


Você pediu e nós ouvimos! O recurso solicitado com mais freqüência para o Terminal é o suporte a várias guias e estamos SUPER empolgados por FINALMENTE podermos oferecer esse recurso essencial. Agora você poderá abrir qualquer número de guias, cada uma conectada a um shell ou aplicativo de linha de comando de sua escolha, por exemplo, Prompt de Comando, PowerShell, Ubuntu na WSL, um Raspberry Pi via SSH, etc.



Belo texto


O Windows Terminal usa um mecanismo de renderização de texto baseado em DirectWrite / DirectX, acelerado por GPU. Este novo mecanismo de renderização de texto exibirá caracteres de texto, glifos e símbolos presentes nas fontes do seu PC, incluindo ideogramas CJK, emoji, símbolos de linha de força, ícones, ligaduras de programação etc. Esse mecanismo também renderiza o texto muito mais rapidamente que o mecanismo GDI do console anterior!



Você também terá a opção de usar nossa nova fonte! Queríamos criar uma fonte divertida, nova e monoespaçada, para aprimorar a aparência moderna do Terminal. Essa fonte não apenas inclui ligaduras de programação, mas também é de código aberto e possui seu próprio repositório. Fique ligado para obter mais informações sobre o novo projeto de fonte!



Definições e configurabilidade


Nós nos conectamos com tantos usuários de linha de comando que adoram personalizar seus terminais e aplicativos de linha de comando. O Terminal do Windows fornece várias opções de configuração que oferecem grande controle sobre a aparência do Terminal e sobre cada um dos shells / perfis que você pode abrir como novas guias. As configurações são armazenadas em um arquivo de texto estruturado, facilitando a configuração dos usuários e / ou ferramentas.


Usando o mecanismo de configuração do Terminal, você poderá criar vários "perfis" para cada shell / aplicativo / ferramenta que deseja usar, seja PowerShell, Prompt de Comando, Ubuntu ou até conexões SSH com dispositivos Azure ou IoT. Esses perfis podem ter sua própria combinação de estilos e tamanhos de fonte, temas de cores, níveis de desfoque / transparência de fundo, etc. Agora você pode criar seu próprio terminal personalizado, personalizado ao seu gosto!


Mais!


Depois de enviar o Windows Terminal 1.0, planejamos começar muitos dos recursos que já estão em nossa lista de pendências, além dos muitos recursos que você, como a comunidade, provavelmente adicionará!


Quando posso colocar minhas mãos nele?


A partir de hoje, o Terminal do Windows e o Windows Console foram criados de código aberto e você pode clonar, criar, executar e testar o código do repositório no GitHub: https://github.com/Microsoft/Terminal


Neste verão de 2019, as visualizações do Terminal do Windows serão lançadas na Microsoft Store para os usuários adotarem e fornecerem feedback.


Neste inverno de 2019, nosso objetivo é lançar o Windows Terminal 1.0 e trabalharemos com a comunidade para garantir que esteja pronto antes do lançamento!

Espere ... você disse open source?


Sim nós fizemos! Temos o prazer de anunciar que estamos fornecendo código aberto não apenas o Windows Terminal, mas também o Windows Console, que hospeda a infraestrutura de linha de comando no Windows e fornece o tradicional Console UX.


Mal podemos esperar para trabalhar com você para melhorar e aprimorar a experiência da linha de comando do Windows!


Isso parece incrível, mas por que você não pode simplesmente melhorar o Console do Windows existente?


O objetivo principal do Windows Console é preservar a compatibilidade com versões anteriores das ferramentas de linha de comando, scripts, etc. Embora tenhamos conseguido introduzir muitas melhorias importantes nos recursos do console (por exemplo, adicionando suporte a cores de 24 bits e VT, etc. consulte esta postagem do blog ), não podemos introduzir melhorias significativas na interface do usuário do console sem “quebrar o mundo” . ”


Portanto, chegou a hora de uma nova abordagem.


O Windows Terminal é instalado e executado ao lado do aplicativo existente do Windows Console. Se você executar o Cmd / PowerShell / etc. diretamente, eles começarão a anexar a uma instância tradicional do console da mesma maneira que fazem hoje. Dessa forma, a compatibilidade com versões anteriores permanece intacta, oferecendo a opção de experimentar o Terminal do Windows, se / quando você desejar. O Windows Console continuará a ser fornecido no Windows nas próximas décadas para oferecer suporte a aplicativos e sistemas existentes / herdados.


Ok, mas que tal contribuir para um projeto de terminal / aplicativo de código aberto existente?


Exploramos cuidadosamente essa opção durante o planejamento e determinamos que nosso envolvimento em um projeto existente exigiria a alteração dos requisitos e da arquitetura do projeto de maneiras muito perturbadoras.


Em vez disso, criando um novo aplicativo de terminal de código aberto e o Windows Console de código aberto, agora podemos convidar a comunidade a colaborar conosco para melhorar o código e aproveitá-lo em seus respectivos projetos.


Acreditamos que há muito espaço no mercado para idéias novas / diferentes sobre o que um terminal pode e deve fazer e nosso objetivo é ajudar o ecossistema de aplicativos de terminais (e relacionados) a florescer e crescer com a introdução de novas idéias, abordagens interessantes, e inovações emocionantes neste espaço.


Eu estou vendido! Como posso me envolver?


Visite o repositório em https://github.com/Microsoft/Terminal para clonar, criar, testar e executar o Terminal! Você pode registrar bugs e compartilhar comentários conosco e com a comunidade, além de corrigir problemas e fazer melhorias no GitHub.


A partir deste verão, tente instalar e executar o Windows Terminal na Microsoft Store. Se você encontrar algum erro, compartilhe seu feedback através dos problemas do Hub de Feedback ou do GitHub para obter detalhes sobre questões / discussões.
