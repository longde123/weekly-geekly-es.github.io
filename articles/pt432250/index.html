AWS Firecracker (Amazon Web Services)



"O AWC introduziu o Firecracker -" micro-virtualização "para Linux" :


Na AWS re: Invent 2018 , que acontece atualmente em Las Vegas, ocorreu o anúncio do Firecracker , uma nova tecnologia de virtualização de código aberto baseada no Linux KVM. Os autores prometem que, com isso, "em uma fração de segundo, você pode executar máquinas micro-virtuais leves (microVMs) em um ambiente não virtualizado, obtendo as vantagens das VMs tradicionais - na forma de segurança e isolamento de cargas de trabalho e contêineres - na forma de uso eficiente dos recursos".

O projeto foi baseado no desenvolvimento de código aberto do Google - crosvm do Chromium OS, escrito em Rust e responsável por lançar sistemas operacionais com virtualização de dispositivo (mas sem emular hardware real). Portanto, o código do Firecracker também é escrito na linguagem Rust, e seus autores prometem retornar suas correções à base de código do projeto pai, embora os próprios projetos tenham divergido bastante de propósito.

RFC


De acordo com a RFC (solicitação de comentários), é bastante conveniente observar em que direção o idioma está se movendo; portanto, aqui estão algumas interessantes. Há um link "Renderizado" no cabeçalho dos PRs, pelo qual o RFC está disponível de forma legível.


Este mês foi aceito:


  • RFC # 2561 Possibilidades futuras - nas novas RFCs, haverá uma seção com as perspectivas para o desenvolvimento da ideia proposta (meta-RFC);
  • RFC # 2591 Estabilizar correspondência exaustiva de padrões inteiros - estabilização de comparações numéricas exaustivas, ou seja, por exemplo, se todos os 255 valores estiverem listados no mapeamento de variáveis u8 , o ramo _ não será necessário;

RFC em consideração:


  • RFC # 2592 Estabilizar 'std :: task' e 'std :: future :: Future' - oferece para estabilizar futuros (isso é separado da sintaxe RFC assíncrona / em espera );
  • RFC # 2593 Tipos de variantes enum - oferece a expansão das possibilidades de uso das opções de enumeração, aproximando-as das estruturas comuns;
  • Registros Estruturais RFC # 2584 - sugere adicionar estruturas anônimas (ou podem ser chamadas de tuplas com campos nomeados);
  • RFC # 2602 # [attribute] s galore - sugere permitir que atributos sejam pendurados no VL, tipos, limites e um monte de outras coisas;
  • RFC # 2603 Symbol Mangling v2 - sugere pentear o esquema dos símbolos de decoração ;
  • A API RFC # 2500 Needle propõe adicionar std::needle com abstrações sobre &str , &mut str , &[T] , &mut [T] , Vec<T> e &OsStr ;

Pré-RFC:


  • Pré-RFC para tipos de variantes anônimas, uma proposta de tipo de soma mínima anônima - sugere adicionar jumentos (_|_) enumerações anônimas com tipos como (NoneError|ParseIntError) ;
  • Pré-RFC: sum-enums - oferece a introdução de um novo tipo de enumeração com a sintaxe enum(A, B, C, ..) e enum Foo(A, B, C, ..) ;
  • Características de ferrugem e sua (falta de) privacidade - o phaazon sugere adicionar a capacidade de tipos privados terem métodos privados ( aqui, de alguma forma, sou muito cético );
