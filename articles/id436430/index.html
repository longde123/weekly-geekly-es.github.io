Pengakuan seorang graphomaniac

Saya sering ditanya tentang tulisan saya. Bukan tentang peristiwa tertentu, atau fakta, atau fiksi yang dituangkan dalam teks tertentu, tetapi tentang proses pembuatan teks. Bagaimana saya melakukan ini, kapan, mengapa, pada kecepatan apa, bagaimana saya menggabungkan dengan pekerjaan, dan apakah saya memiliki pekerjaan sama sekali, dll.

Saya, seperti yang diharapkan, menjawab pertanyaan-pertanyaan ini, tetapi ada begitu banyak dari mereka sehingga saya memutuskan untuk menyajikan pengalaman saya dalam bentuk artikel. Tujuannya persis sama - untuk berbicara secara terpusat tentang bagaimana saya melakukannya. Mungkin pengalaman ini akan membantu seseorang. Dan saya hanya akan memberikan tautan ke artikel ini ketika saya mendapatkan pertanyaan lain tentang topik "Bagaimana Anda menulis semua omong kosong ini?"

Mungkin ini adalah teks terakhir saya, karena kemarin saya memiliki perubahan tajam dalam hidup saya.

Kecepatan


Saya tidak tahu seberapa cepat saya menulis teks. "Cepat" adalah perkiraan kecepatan aktual, jadi saya usulkan ini: Saya hanya akan mengukur kecepatan artikel ini, dan Anda memutuskan apakah saya dengan cepat membuatnya atau tidak.

Jadi, saya membuat file dengan artikel ini pada 8-10 waktu Chelyabinsk. Waktu akan menjadi "kotor" - Saya akan terganggu untuk menuangkan kopi, teh, dan asap. Singkatnya, proses yang sepenuhnya alami. Pada akhirnya, rangkum pengukurannya.

Sedikit sejarah


Di mana tanpa sejarah, lalu ... Sejak 2005, saya bekerja sebagai programmer 1C. Pertama di waralaba, lalu di pabrik, lalu di peternakan unggas, lalu di pabrik lain. Posisi kepemimpinan yang diduduki secara berkala, secara berkala mundur dari mereka.

Pada 2017, ia pindah kerja di Oksnosoft - kami menulis di sini di js dan 1C. Hanya ada kita berdua di sini, perusahaan itu memerintah sendiri - mereka hanya berbagi tanggung jawab. Kami berdua memprogram, kami memperkenalkan sesuatu kepada pelanggan, kami berdua menciptakan produk dan layanan, keduanya mencoba membuat hidup lebih baik. Teks yang saya tulis terutama saya.

Jadi, saya mulai menulis di pabrik terakhir. Saya ingat persis - itu adalah buku harian tentang rancangan perubahan strategis yang saya pimpin. Itu adalah hal yang besar dan menarik, dan untuk beberapa alasan saya ingin menuliskan peristiwa dan pemikiran saya tentang topik ini - dengan siapa saya berbicara, metode apa yang saya gunakan, apa yang terjadi, apa yang terjadi, dll. Sayangnya, buku harian ini kemudian hilang, yah, dan persetan dengan itu.

Ketika saya memperkenalkan perubahan itu sendiri, dengan tangan saya sendiri, tidak perlu mentransfer pengetahuan apa pun - namun, itu ada di kepala saya. Tetapi ketika sebuah tim besar muncul, pengetahuan harus ditransfer. Pertama, secara verbal, kemudian mulai merekam. Jadi teks pertama yang terkait dengan pemrograman bisnis muncul. Pertama di portal internal, lalu di blog.

Ketika dia meninggalkan pabrik, dia mulai menerbitkan teks-teks ini di Internet. Maka kisah dari teks saya bersifat publik.

Hudlite


Hoodlite muncul dalam daftar lagu saya di Februari 2017. Tentu saja, saya mengambil ide dari Goldratt. Khudlit, sebagai format untuk penyajian informasi teknis dan metodologis, memiliki banyak keuntungan.

Pertama, ini memungkinkan Anda untuk menghilangkan diri dari cerita. Ini sangat penting karena pertanyaan selalu muncul untuk artikel "biasa" - siapa Anda? Seseorang menulis tentang pengalamannya - dan siapa Anda untuk menceritakan pengalaman Anda di sini? Seseorang menulis tentang fitur pemrograman dalam bahasa apa pun - dan siapa Anda? Di mana profil github Anda? Di mana layanan yang Anda buat?

Itu menghilangkan semua ini. Anda berbicara tentang orang, peristiwa, situasi, berdiri di samping. Tidak satu pun dari karakter Anda yang diwajibkan untuk menjadi baik, buruk, pintar, bodoh, dll.

Kedua, melambat memungkinkan Anda untuk menyampaikan sejarah, proses, dan emosi dari proses apa pun. Jadi, misalnya, ada dalam novel bisnis "The Goal" oleh Goldratt. Dia hanya bisa menulis manual "Teori Kendala", tetapi tidak ada yang membacanya. Dan unit-unit yang membaca akan mengajukan banyak pertanyaan - tetapi bagaimana menerapkannya dalam situasi seperti ini? Dan bagaimana meyakinkan semua orang bahwa teknik khusus ini harus digunakan? Dan kesulitan apa yang akan ditemui dalam hal implementasi?

Khudlit menjawab semua pertanyaan ini. Tidak perlu tersiksa, sistematiskan sesuatu seperti "opsi untuk mengatasi resistensi selama implementasi" - cukup tulis cerita tentang bagaimana resistensi muncul dan bagaimana para pahlawan mengatasinya.

Sederhananya, lebih tipis memungkinkan Anda menulis sebuah cerita, bukan sebuah cerita.

Metrik


Jika Anda membaca artikel non-fiksi saya, Anda tahu bahwa saya suka mengukur semuanya untuk mengevaluasi efisiensi dan produktivitas. Hal yang sama berlaku untuk artikel.

Saya mengukur kinerja secara sederhana - saya memperkirakan jumlah teks tertulis dengan poin sesuai dengan teknik perencanaan poker. Untuk teks "out of my head" tandanya adalah 13 poin untuk 10.000 karakter. Untuk teks “sedang bekerja” - 8 poin untuk 5.000 karakter. Untuk membuatnya jelas: 10.000 karakter adalah ukuran minimum satu artikel. Rata-rata adalah 15.000 karakter, tetapi kadang-kadang 30.000, dan bahkan 55.000.

Saya memasukkan tugas menulis ke dalam Flowcon, bersama dengan sisa tugas kerja yang biasa. Kinerja rata-rata untuk semua tugas, saya punya sekitar 600 poin per bulan. Dengan demikian, jika saya hanya menulis teks, saya akan mendapat 460.000 karakter sebulan - ini adalah sekitar 30 artikel berukuran sedang, atau 10 lembar hak cipta dalam hal pencetakan, atau 50-70% dari rata-rata buku.

Tapi ini persis "jika saya hanya menulis teks", yang tidak saya lakukan.

Pengukuran membantu mengatur proses penulisan. Saya melihat saat-saat ketika saya terbawa suasana dan mulai menulis terlalu banyak, dan beralih ke pemrograman. Demikian pula, saya terkadang melihat bahwa saya mulai menulis kurang dari rata-rata, dan sekali lagi saya membuka editor teks.

Jika saya menulis tentang para pekerja di bengkel, itu akan lebih mudah - karena ada sesuatu yang terjadi setiap saat. Ada mesin yang bisa pecah, ada potongan besi yang jatuh di kaki Anda, balok balok, dll.

Dan apa yang ada di kantor? Meja, kursi, komputer, dan manusia. Oleh karena itu - "Sergey duduk di kursi, membuka kunci komputer, menoleh ke Stas dan berkata ...".

Sekarang tentang mengapa dialog itu penting. Jawabannya ada di bagian cerita, tapi saya ulangi: karakter itu sendiri yang menciptakan isyarat.

Jika menurut Anda saya berbicara dengan suara pahlawan, maka Anda salah. Saya hanya membayangkan situasi, orang-orang dan karakter mereka, dan muncul dengan frasa pertama. Misalnya, "Gagasan dengan iPhone benar-benar sial." Apa yang akan terjadi selanjutnya, saya tidak tahu diri. Aku hanya menempatkan diriku, secara mental, di sudut ruangan tempat aksi itu terjadi, dan menunggu seseorang mengatakan apa. Dan tulis.

Secara pribadi, pendekatan ini mengganggu saya. Kebetulan saya benar-benar ingin menyampaikan beberapa ide atau frasa melalui ketipisan, tetapi tidak berhasil bagi saya. Percakapan para pahlawan, seolah-olah, sendiri. Pada titik tertentu, saya kira begitu, hentikan, Sergey harus mengatakan ini. Tapi dia tidak mau. Percakapan pada umumnya pergi ke pesawat lain, dan tidak ada tempat untuk mengacaukan frasa saya. Kita harus dengan rendah hati menunggu sampai pembicaraan mendekati topik yang diinginkan untuk setidaknya memengaruhi topik tersebut.

Dan ketika saya tidak tahan, dan dengan paksa saya mengubah plot ke arah yang benar, ternyata buruk. Komentator sering menulis tentang hal ini - intervensi saya terlihat dengan mata telanjang, seolah-olah tersingkir dari arus peristiwa atau dialog yang normal.

Teknik


Tidak ada yang supranatural. Awalnya, saya menulis di Word. Lalu saya memformatnya dengan persyaratan situs.

Pada sebagian orang, editornya sangat kuat, dapat menelan format apa pun, mengambil tajuk, tautan, dan gambar.

Beberapa harus ditulis dalam penurunan harga, yang menyebabkan kesulitan tertentu, tetapi saya sudah terbiasa.

Agar tidak menderita dengan pemformatan, saya menulis teks kosong di Word, dan saya sudah memformatnya di situs tertentu.

Kenapa saya menulis?


Hanya menulis, itu saja. Saya dulu mencoba berfantasi bahwa saya ingin menyampaikan beberapa pemikiran atau pengetahuan. Kemudian saya menyadari bahwa menyampaikan adalah bentuk aktif, intervensi dalam kehidupan seseorang, upaya untuk mengubahnya.

Karena itu, saya mulai menulis dalam kehampaan. Jika seseorang menemukan sesuatu yang bermanfaat untuk diri mereka sendiri, bagus. Jika Anda tidak menemukannya - tidak peduli.

Saya akan menggambar analogi dengan pemrograman. Produk dan solusi terbaik yang saya buat dalam praktik pemrograman saya muncul tanpa tujuan dan pesanan. Itu hanya diminta oleh jiwa - mari kita lakukan hal seperti itu! Saya lakukan, dan semuanya berjalan lancar - keputusan-keputusan ini menyimpang dalam pelaksanaan massal, berhasil diterapkan dalam praktik, dan mengumpulkan sepasukan penggemar dan pengikut.

Dan ketika saya memprogram "custom-made", ternyata ... Yah, sesuatu terjadi. Tampaknya bahkan berhasil. Dan sekarang berhasil. Mungkin itu tidak memalukan untuk diceritakan. Tapi saya tidak mau.

Dalam coretan, pendekatannya sama. Apa yang diminta jiwa, lalu saya jelaskan. Kebebasan penuh. Saya tidak berhutang kepada siapa pun, nikmati saja prosesnya. Tidak ada tujuan, seperti peringkat, penjualan, ulasan. Dan jika tidak ada tujuan, maka tidak ada penilaian pencapaiannya. Tidak ada yang perlu suka, cocok dengan apa pun, menjual apa pun.

Apakah saya seorang graphomaniac?


Dengan beberapa kriteria, ya. Kutipan dari Wikipedia:

"Graphomania adalah keinginan patologis untuk penulisan multi, untuk menulis karya yang berpura-pura diterbitkan dalam publikasi sastra, risalah ilmiah semu, dll. Seorang penderita graphomania dapat menulis" karya ilmiah "di daerah di mana ia tidak mengerti, menulis karya seni di tidak adanya kemampuan kreatif sama sekali. Ditulis oleh graphomaniacs sebagian besar dangkal atau bahkan tidak ada artinya dalam konten. "

Benar, ada penyimpangan dari definisi tersebut. Misalnya, saya tidak punya patologi. Saya bisa menulis, saya tidak bisa menulis. Saya juga tidak mengaku diterbitkan dalam publikasi sastra. Dan saya tidak sedang menulis makalah ilmiah.

Saya sering disebut seorang graphomaniac. Awalnya saya mencoba berdebat dan membuat alasan, lalu saya mulai mengabaikannya, dan sekarang saya memutuskan untuk setuju. Apa bedanya?

Dalam definisi seorang graphomaniac adalah penilaian tersembunyi. Diasumsikan bahwa ada beberapa orang yang dapat mengevaluasi teks dan memutuskan apakah seseorang memiliki kemampuan, apakah karya-karyanya ilmiah, apakah ia memahami di beberapa bidang. Dan orang-orang ini memberi tanda - yah, di sana, saya tidak tahu, pada skala tertentu. Penulis dalam hal ini menjadi posisi pemohon - membawa teks, dan ingin dihargai. Plus menempatkan, atau menulis ulasan, atau dikirim ke sirkulasi.

Dan apa yang akan terjadi jika prosedur ini tidak berjalan? Prosedur evaluasi, dalam arti. Apakah seseorang yang menulis teks dan tidak memperlihatkannya kepada siapa pun yang dianggap sebagai ahli grafik? Atau orang yang menerbitkan teks, tetapi tidak datang untuk “pendapat ahli”? Seluruh komisi mengejarnya, berteriak - Tunggu! Anda seorang graphomaniac! Teks Anda sial! Dan dia pergi ke suatu tempat, sesuai dengan urusannya sendiri, dan tidak meniup kumis.
Seperti ini Tapi, sejak pengakuan dosa, izinkan saya menjadi seorang graphomaniac.

Apakah ada, yang menarik, istilah yang sama untuk pemrograman? Jadi ketika saya menyandikan sesuatu yang tidak ada yang meminta, dan saya melakukannya karena satu alasan - apakah saya suka memprogram?

Dampak coretan pada kehidupan


Scribble, terutama yang lebih tipis, mempengaruhi kehidupan dengan cukup signifikan, dan terutama secara positif. Ini memperkaya persepsi informasi eksternal. Anda mulai memperhatikan sesuatu yang Anda tidak melihat titik kosong.

Misalnya, membaca fiksi, saya sekarang memperhatikan teknik pertunjukannya. Bagaimana penulis membuat dialog - padat atau masukkan poin seperti "John menjawab"? Seberapa sering ia menggunakan poin-poin ini? Kalimat menulis panjang atau pendek? Seberapa sering menggunakan frasa partisipatif dan partisipatif, kata benda verbal, ellipsis, dan monolog internal karakter?

Anda juga mulai memperhatikan secara otomatis dalam hidup Anda titik awal plot - situasi. Anda melihat suatu peristiwa, dan ide sebuah cerita segera lahir di kepala Anda, dan alur ceritanya muncul dalam proses presentasi.

Anda melihat situasi dalam kehidupan dan di tempat kerja. Misalnya, Anda memperhatikan fakta bahwa klien memiliki basis 1C dari 1 TB. Terlepas dari kenyataan bahwa intensitas entri data, jumlah objek dan loop pengukuran hampir sama dengan yang saya miliki di pabrik saya. Tapi basis saya adalah 15 GB. Kepala itu sendiri mulai berpikir apa alasannya. Lalu saya menangkap frasa, saya mendengar komentar di sana - ya, inilah alurnya.

Pengungkapan liburan terakhir adalah, misalnya, seri "Dokter Rumah". Saya dulu hanya menonton dan menikmatinya. Dan sekarang saya melihat karya penulis naskah, upaya mereka untuk mengisi plot dengan garis tambahan yang kadang-kadang lebih penting daripada yang utama. Saya menarik perhatian karakter karakter, yang dari musim ke musim menjadi lebih beragam, dan saya juga ingin belajar bagaimana melakukan ini.

Begini, pada hal-hal sepele, di tempat kerja itu tercermin. Jika sebelum saya menulis teks tentang layanan baru, katakanlah, selama beberapa hari, sekarang butuh beberapa jam. Ini umumnya tidak lagi menjadi masalah. Hanya menguasai teknologi, atau mengembangkan kompetensi, yang selalu berguna. Nah, kecepatan mengetik huruf Rusia telah meningkat secara signifikan.

Penerbitan buku


Saya sering ditanya apakah ada teks yang akan diterbitkan dalam bentuk kertas. Awalnya, tujuan ini menarik bagi saya, dan saya bergegas menulis surat kepada penerbit beberapa kali. Ada dua jawaban: "kita tidak melihat potensi komersial" dan "tidak cukup, kita perlu menyelesaikan".

Tidak ada keraguan bahwa tidak ada potensi komersial - saya tidak menetapkan tujuan seperti itu.

Dan "perlu untuk memodifikasi" bertentangan dengan prinsip saya - saya tidak menulis sesuai pesanan. Nah, Anda perlu mengambil teks, memasukkannya ke dalam semacam kerangka kerja sehingga cocok dengan kriteria seseorang, dll. Untuk apa ini bagiku?

Anda tidak akan menghasilkan banyak uang di sana, tetapi rewel selama enam bulan. Saya tidak memiliki tujuan "menjadi orang yang menerbitkan buku kertas," jadi setelah berpikir beberapa jam, saya memutuskan bahwa saya tidak membutuhkannya. Di sini kami memiliki harmoni penuh dengan penerbit: mereka tidak membutuhkan saya, saya tidak membutuhkan mereka.

Jika Anda pernah terjebak di ekor, dan memutuskan untuk menerbitkan sesuatu, saya akan melakukannya dengan biaya sendiri. Sepertinya tidak mahal.

Kualitas Teks


Kualitas teks saya agak rendah, banyak yang memperhatikannya. Saya sering membuat kesalahan, baik ejaan dan tanda baca, dan apa lagi yang ada di sana.

Alasannya sederhana: Saya tidak mengedit teks. Saya bahkan tidak membacanya. Saya menulis, menulis, menulis, dan ketika saya selesai, saya menyebarkannya. Itulah yang ditekankan Firman kepada saya dalam warna merah atau biru, kemudian saya memperbaikinya.

Awalnya saya khawatir tentang ini. Saya pikir itu perlu diedit, perlu untuk memeriksa, baik, seperti apa stan itu. Dan kemudian dia membuat satu penemuan luar biasa, dan berhenti mengkhawatirkan.

Kuncinya adalah bahwa, tanpa membaca teks saya, saya melupakannya dengan sangat cepat. Saya benar-benar tidak ingat apa yang ditulis artikel itu enam bulan lalu. Jadi, secara umum, secara umum.

Dan kemudian, sekali, secara kebetulan, tanpa ada hubungannya, saya membaca beberapa artikel lama saya. Dan aku menyukainya! Yang, secara umum, logis - jika saya menulis sesuka saya, maka saya akan suka membaca.
