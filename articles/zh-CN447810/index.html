Azure DevOps Services的分析现已公开可用

对于依靠Analytics （Azure Analytics Service）进行数据驱动决策的Azure DevOps用户而言，报告是一个重要的机会。

今天，我们很高兴地宣布，Azure DevOps Services将包含以下Analytics（分析）功能，无需额外付费。 在不久的将来，客户将在帐户中看到这些更改。



Azure DevOps Services中现已提供分析功能


  • Analytics Widgets-自定义模块，可在仪表板上显示数据并帮助跟踪进度。
    1. Burndown and Burnup-在特定时间段内监视某些区域的工作进度。
    2. 周期时间和提前期 -可视化团队中的产品开发周期。
    3. 累积流程图（CFD） -在工作项目经过各种状态时对其进行跟踪。
    4. 速度 -跟踪团队如何在多个冲刺中增加价值。
    5. 测试结果趋势 -跟踪测试趋势，识别一条或多条管道（Azure管道）的测试失败和持续时间。

  • 在“产品体验”中-分析在DevOps Azure中运行，并且在显示数据和分析的仪表板之外。
    1. 顶级失败测试报告 -了解管道中最严重的失败测试。


我们将继续为所有Azure DevOps Services客户通过Analytics（分析）视图提供Power BI集成，并以预览模式直接访问OData Endpoint 。 预计将于2019年6月获得有关集成Power BI和OData的定价模型的更多信息。

当前在市场中安装了Analytics扩展的Azure DevOps Services客户可以继续使用Analytics（分析），而无需采取任何其他步骤来获取Analytics。 因此，我们将谴责托管客户的Marketplace分析扩展 。

Azure DevOps服务器2019


对于Azure DevOps Server，Analytics（分析）功能将保留在预览模式下，作为可在本地市场上安装的扩展，并将在下一个主要版本中公开提供。

Azure DevOps Analytics是报告的未来，我们将继续投资于Analytics提供的新功能。 要了解有关Analytics（分析）及其当前提供的机会的更多信息，请执行以下操作：

  • 查看我们的分析概述文档 。
  • 查看主要功能： Analytics Widgets ， Top Failing Test Report ， Power BI Integration和OData Endpoint 。
  • 在Channel 9上观看有关Azure DevOps Analytics的视频 。
