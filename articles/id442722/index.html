Berhenti mengunduh dan menyimpan angka nol


Yang dimaksud dengan "blok nol" adalah blok (bagian dari file) yang diisi dengan byte nol. Anda dapat pra-menghitung hash dan tidak meminta blok ini dari sumber, tetapi segera tandai mereka sudah dimuat.


Nol blok tidak perlu disimpan di disk. Berkat bendera yang jarang, sistem operasi hanya menandai bagian file ini sebagai yang diisi dengan nol dan tidak menyimpan angka nol ini di disk.


Properti file diisi dengan nol


Juga, menunjukkan bahwa bagian dari file yang diunduh diisi dengan nol, Anda dapat memotivasi pengguna untuk menolak mengunduh dan mendistribusikan file yang rusak. Di Shareaza versi saya, bagian ini ditandai dengan bilah merah di atas bilah progres dari unggahan file.


Dari mana asalnya blok nol dalam file?


  1. Distributor tidak menunggu unduhan penuh dan verifikasi file dan mengunggah sebagian file.
  2. Hasil kerusakan pada sektor disk distributor.

Ini adalah opsi yang muncul di benak saya.


Mengapa file-file ini terus menyebar


  1. File video / audio mungkin memiliki blok nol di tengah dan diputar dengan tenang, cukup lewati tempat di mana blok nol ditangkap. Jadi, itu mungkin tampak utuh.
  2. Gambar disk juga dapat memiliki blok nol di tempat yang berbeda dan ini hanya akan memanifestasikan dirinya ketika mencoba membaca file dari blok ini.

Bagaimana cara kerjanya


  1. Sebelum mengunduh file p2p , klien menerima daftar hash blok dari sumber.
  2. Setiap hash dari daftar dibandingkan dengan hash pra-hitung dari blok nol.
  3. Blok-blok yang hash cocok dengan hash dari blok nol ditandai sudah dimuat. Pengguna juga ditunjukkan bahwa hanya ada nol di tempat ini.

Alat untuk menghitung blok nol


  1. RHash - itu akan langsung menghitung hash.
  2. Lua - akan memuat jumlah nol yang diperlukan ke dalam RHash dan menampilkan hasilnya.

  3. Fungsi skrip umum: 


    local lua = "lua5.1" --  local script_name = "zero-block-hash" --   (   ) --    hex    -- https://stackoverflow.com/a/9140231 function string.fromhex(str) return (str:gsub('..', function (cc) return string.char(tonumber(cc, 16)) end)) end --   io.stdout    function std_write(size) index = index or 0; local buffer_size = 1024*1024*16; if ( size <= buffer_size ) then io.stdout:write( ('\0'):rep( size ) ); else local zero_buffer = ('\0'):rep( buffer_size ); local count = math.floor( size / #zero_buffer ); local tail = math.fmod( size, #zero_buffer ); for i = 1, count do io.stdout:write( zero_buffer ); end if ( tail > 0 ) then io.stdout:write( zero_buffer:sub( 1, tail ) ); end end end

Jangan mengunduh nol dari jaringan DirectConnect, Gnutella dan Gnutella2


Ketiga jaringan ini menggunakan Tree Tiger Hash (TTH). Berdasarkan namanya, ini adalah pohon hash , dan fungsi Tiger digunakan untuk perhitungan. Saya menyebut jenis hash ini "kayu." TTH, menurut saya, adalah pohon yang paling sederhana dan, berkat propertinya, kami dapat dengan cepat menghitung hash untuk ukuran blok yang berbeda.


Ukuran blok minimum untuk TTH adalah 1024 byte. Selanjutnya, ukuran blok berlipat ganda di setiap level.


Itu dihitung sebagai berikut: 


 Tiger(0x00 + block_data)

Awalan 0x00 -byte blok daun
block_data - blokir data (dalam kasus kami, ini adalah 1024 nol byte)
+ - Rangkaian
Tiger - fungsi hash


Selanjutnya, untuk menghitung blok nol yang lebih besar, kami menggunakan hash dari blok nol yang lebih kecil. 


 Tiger(0x01 + hash + hash)

Awalan 0x01 - byte untuk sepasang hash
hash - hash dari blok nol yang kami terima di level sebelumnya.
Tiger - fungsi hash


Kami menulis fungsi untuk perhitungan: 


 --Tiger Tree Hash Leaf block local leaf_hash_cmd = lua..' -l"'..script_name..'" -e"std_write_leaf_hash()"'; --Tiger Tree Hash Internal block local internal_hash_cmd = lua..' -l"'..script_name..'" -e"std_write_internal_hash(\'%s\')"'; --   rhash -   '\0' function std_write_leaf_hash() local tiger = io.popen('rhash -p"%x{tiger}" -', "wb") tiger:write('\0'..('\0'):rep(1024)) tiger:close() end --   rhash      '\1' function std_write_internal_hash(hash) local tiger = io.popen('rhash -p"%x{tiger}" -', "wb"); hash = hash:fromhex(); tiger:write('\1'..hash..hash) tiger:close() end --   - function tth_leaf() local rhash = io.popen(leaf_hash_cmd, "rb"); local hash = rhash:read("*a"); rhash:close(); return hash; end --      function tth_root(hash_hex) local rhash = io.popen(internal_hash_cmd:format(hash_hex), "rb"); local hash = rhash:read("*a"); rhash:close(); return hash; end --     function gen_tth_hashes(hash_count) --    - local hash_hex = tth_leaf():upper(); hash_count = hash_count or 37; hash_count = hash_count - 1; print("") print("// tth_hashes") for i = 0, hash_count do print("// Hash: "..hash_hex, " Size: "..math.floor(1024*2^i)); --    -    hash_hex = tth_root(hash_hex):upper(); end end
