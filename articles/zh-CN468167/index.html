“我会教你，你会离开。” 我是什么，一个傻瓜？ 去工作吧-预期的答案会响起。

系统管理员到位，打开一个论坛Toster，Habr，并阅读如何在几乎博物馆设备的粪便和棍子网络上配置路由。 双手掉了一点，但是很好-您可以省下训练钱，自己掏钱。 或者也许是真的离开？ 邻居们带来了新的Tsiska ...

陷入这种情况？ 在我看来，由公司组织或由员工主动进行的工作过程培训是最有成效的形式之一：员工已经确切地知道他希望从课程中获得什么，如何评估信息以及如何使用信息。 当为期六个月的课程可以带来比整个大学加起来更多的收益时，就是这种情况。 今天，我们将讨论课程，企业大学，指导和最无用的培训形式。 倒热茶，坐在显示器前，让我们一起选择培训的形式和/或形式。

因此，在大学，研究生院，甚至后面的研究生院，您都在工作。 工作日程已经被推迟，任务的处理方法已经形成，工资是每月两次，并且近期前景或多或少是显而易见的。 为了认真开始再次学习会有什么动机？ 有足够的动机。

  • 渴望改变活动范围以求更好的工作，赚更多的钱，学习新的职业等。
  • 需要提高当前工作的技能以垂直增长或水平转移； 换工作。
  • 需要获得新知识，尝试不同的领域-例如，当错误的大学结束，选择了错误的工作，职业和智力停滞的感觉等时。
  • 情感上的原因（出于公司的利益，出于兴趣，出于无聊等）。 最有争议的动机，因为在这种情况下，永恒的学生没有目标和具体的计划。 为了捍卫这群学生，可以说他们在学习过程中经常渗透并以同样的热情进入新的专业。

我们已经确定了是否值得再次接受高等教育 ，现在我们将讨论节省时间（但不节省金钱）并允许您在最短的时间内学习新知识的替代选择。

与工作有关但不在内部的培训


m校外晚间课程


与大学最相似的学习形式：晚上您参加3-3.5个小时的讲座和练习，老师们在其中帮助您掌握新材料。 同时，课程中没有不必要的非核心科目，学生与您是同一个人，也就是说，除了培训之外，您还可以结识新朋友，有时是有用的朋友。

优点


  • 通常，此类课程的老师是从业人员，这意味着他们提供的材料对您在实际工作中有用。 从第一天开始就可以应用一些技能。
  • 上课时间每周晚上2-3次，不会干扰工作（如果您必须去堵车，请同意在上学日您会早一点上班，然后分别离开学校）。
  • 您可以在与您同等的专业人员的陪伴下解决实际问题，因此，还可以感知思维方式，运用团队合作技能并从同学那里获得更多信息。
  • 课程中的小组通常数量很少，每位学生在回答问题和实际工作方面都得到老师的极大关注。
  • 如果课程与公司有任何隶属关系，那么最后您可以获得一份可以从事自己专业的工作的机会-如果您只是踏入IT领域，这将是一个非常酷的机会（例如，我们的9人小组立即收到了一份提议，其中三人同意参加到公司毕业后，又收到了三份报价，但被拒绝了）。

缺点


  • 课程非常昂贵。
  • 大学课程可以充斥非核心科目，并由定期演讲后兼职的理论家教授。
  • 您可能会非常想念其教育背景（例如，在软件开发计划的培训期间，我缺乏数学知识，因此我必须首先对问题进行数学分析，然后以编程方式解决问题）。
  • 您可能会遇到过时的物质基础（例如，正在开发Windows Server 2008和用于XP的PC在2018年？），因此，一台笔记本电脑，一笔许可费用或为培训目的找到一些盗版物品的能力非常有用，但新鲜:-)

寻找什么


  • 仔细研究课程计划和学时数，找出培训中包含的内容以及最终要等待的最终证书形式（从无内容到捍卫完整的英语文凭项目）。
  • 询问方法学家您的老师是谁，他有什么经验，以及他是否执业。
  • 了解分期付款计划或按期划分付款-通常，这种付款方式负担较小。
  • 如果您有入学考试或入学面试，请不要尝试绕开它，一定要通过它-这样您将评估您的准备水平并能够提出对您重要的问题。
  • 如果课程框架中有英语，请不要尝试从培训成本中减去其成本（因为您已经知道）。 是在外国班上与小组的熟人，这很重要-经常有同学互相打电话来上班。
  • 查明是否提供了课程结业证书以及采用什么格式（需要带有盖章和签名的任何论文）。

universities企业大学


一种有趣的培训格式，公司内部员工和外部学生均可使用。 您可以在公司本身，其授权的培训中心或基础大学的合作部门（例如HSE或您的州立大学）学习，并且还可以在所选专业领域（信息安全，通信系统，软件开发，项目管理，1C编程等）。

优点


  • 这是结识公司的好方法，可以与老师（通常不低于中级职位的老师）尝试工作。 而且，有时候这是进入公司，在培训期间展示自己的唯一简单方法。
  • 公司大学教授中有90％是从业人员。 您不仅要学习，而且要解决老师作为经理或技术人员必须解决的实际战斗任务。
  • 舒适的学习环境-实际上，您与老师处于平等地位，因为这两个经理都是公司，但公司不同。

缺点


  • 在您的公司中，管理人员可能不欣赏在其他公司的大学内部进行培训的前景。
  • 教师可以根据公司的模式和问题提供量身定制的信息； 也许某些东西对您不相关或不适用。

但是，如果拥有该课程的公司的一名员工正在公司大学学习，那么会有更多的好处（培训的好处，在桌面附近，同事和管理人员的关注，易于应用的知识，可理解的职业晋升/运动模型），有时很难做到减负认为他们的同事是老师。

▍远程课程和在线培训


您可以访问教育资源（视频，讲座，摘要，书籍，有时甚至是整个图书馆，代码存储库等），并且可以在方便或约定的时间学习，而无需离开工作场所（在家中或从个人PC上）。 您的工作很“酷”，能够与老师交流（聊天或Skype），做家庭作业，但是大多数情况下，您不知道自己在课程中有多少，与谁在一起以及与“同伴”的交流会变成一场漫无目的的洪水。

优点


  • 节省能源和时间，并节省费用。
  • 方便且熟悉的培训格式。
  • 您可以直接在工作中学习，也可以在办公室后立即学习（如果没有严厉的系统来控制工作时间，行动，日志记录，强大的安全服务和告密者。
  • 您可以选择舒适的工作节奏，并在互联网，Toster，Habré，StackOverflow等上处理难以理解的时刻。

缺点


  • 需要高的动力和自组织能力，因为这比起接受经典导师的培训更能自我教育。
  • 学习过程中没有实时交流。
  • 检查老师并弄清楚这是否是课程说明中宣布的老师是非常困难的。
  • 做出错误选择的风险-现在有太多这样的选择，很难不错过并进入一所真正高质量的在线学校（即使一家公司也很容易）。
  • 最低限度的就业机会-如果您没有表现出出色的能力（以及如何在线上工作？），您唯一可以依靠的就是在合作伙伴公司的HR数据库中获得一份简历，必要时可以致电。

寻找什么


  • 在证明书和获得带有加盖印章的纸质证书的条件上（通常需要为此多付钱）。
  • 付款条件和访问课程资料的紧迫性（理想情况下，这应该是不受限制的访问）。
  • 在社交网络和独立站点上进行听众评论（通常在站点上进行审核）。
  • 关于与老师互动的形式（理想情况下，应该是聊天+与学生进行功课分析，最好是预先发送功课）。

自从“生活，学习”系列开始以来，我们就评论的主观性达成了一致，我要说我对在线教育形式持谨慎态度。 有时为未知的内容付出巨额资金令人恐惧。 在Internet上，所有IT知识领域都有很多很酷且真正可理解的课程，在我看来，这是最好的选择，让他们对这些知识有偏好和时间。 此外，大多数雇主都希望对怀疑态度很高的在线学校的纸片吐口水 ，但是真正的技能和理论技能并没有打扰任何人。 例如，由于我对OSI网络模型具有非凡的理论知识，我设法在IT领域找到了第一份工作-成为一名测试工程师（27岁，没有技术背景）。 当然，这取决于您自己决定，但我更支持0.5-1.1.5年课程（具有脱机状态）的支持。

▍培训和讲习班


当然，如果我们不谈论个人成长培训和其他商业青年，那么这是一种很好的培训形式。 这些是短期的密集课程，在该课程中，老师可以帮助您加深您熟悉的领域的知识并通过短期实践。

持续3个小时到几天。 我不会谈论其优缺点-最主要的是，这不是某些下一产品的广告。 查看赞助商，检查组织者和演讲者的评论并继续。 有时候，去不在您所在地区的培训或讲习班很有意思-例如，您可以更好地了解同事。

工作流程中的学习形式


这是一个非常重要的块，无法绕开。 我在公司内部有混合的培训经验，我认为值得一提，因为公司本身将其定位为人力资源PR的竞争优势，而员工则希望取得这一结果。

▍指导与指导


新人在第一工作日对贵公司的感觉如何？ 坐在一张空桌子旁并在等待一台工作正常的PC时紧张地摆弄欢迎包？ 戳电话，只是不引起同事的注意？ 还是轻松舒适地阅读有关他们工作的信息？ las，我的经验表明，后者是最低要求。 同时，在俄罗斯IT领域，有很多公司（甚至很小的公司）值得向他们学习：新员工被分配了一位导师，作为其工作时间的一部分，它为新手提供了有关基本任务的培训，同时显示了基础架构（访问，服务器，设备，错误跟踪器，服务台，项目管理系统等），介绍同事等。 因此，新员工立即与导师一起加入团队，知道与谁联系并快速学习工作材料。 有时，辅导伴随着活动领域的模块化或期末考试，尽管这虽然有点压力，但是却为员工和公司提供了某种保证。

在组织工作指导时，您需要了解/理解几点。

  • 导师的工作必须以奖金或KPI的形式支付。 付款不应该取决于初学者的工作时间，但是根据试用期的结果，您仍然可以获得更多的报酬，这意味着他训练有素并且带走了高质量。
  • 导师应该有经验并且善于交际-las，如果超级天才DevOps将手册扔在桌子上并提供指向内部Wiki的链接，那将没有任何用处。 新员工和导师应具有沟通方式和对话方式。
  • 指导员应对培训期间病房的工作负责-例如，如果经验不足的测试员通过DHCP给每个人127.0.0.0，则是指导者需要解决此问题，同时还要自己了解需要在测试环境中学习的知识（嗯，是的，根据实际事件，我们受过训练，我们受过训练-总的来说，我们不会感到无聊）。
  • 指导者应充当公司的指导，提供访问权限，与系统管理员沟通，介绍其他部门的同事等。
  • 如果出现个人敌对或冲突情况，应立即更换导师。
  • 培训期间指导者的工作量应减少，并在合理的范围内重新分配给其他同事。
  • 从实习生到大四生，每个新来者都应该有一位导师，区别仅在于提供信息的方式，时间和信息量。 人事部门应注意每个员工适应过程的正常进行，否则工作过程中不可避免的问题是有问题的，因为每个公司都有自己的工作特点。

无论如何，如果您没有尝试过公司内部的指导机构，请在下个月为自己设置这项任务-与新员工一起工作的结果会让您感到惊讶。

▍会议，讲座，会议


在工作框架内，也许是最富有成效的培训形式之一：员工相互交流成就，分享技能，举行产品会议和演示文稿，邀请其他公司的同事交流经验（有时是为了狩猎）。 这样的会议有很多优点：

  • 员工学会彼此了解，并在一个协调良好的团队中工作；
  • 开发人员以一种语言进行交流并交换可以安全采用和应用的解决方案；
  • 您可以了解另一家公司的文化并展示您的优势；
  • mitaps是免费的。

出色的mitap的关键是准备：与演讲者一起工作，准备演讲，布置大厅并关注主题。 结果将是令人愉快和有用的。

如何在工作中学习？


工作时，对您来说最宝贵的资源就是时间。 这是生活中的艰难时期，当您需要工作，建立事业，不错过机会，建立家庭，帮助父母，实现自己对爱好和爱好的渴望时。 因此，最大的问题是寻找时间进行培训，从而证明培训是紧张而有效的。

  • 停止浪费时间在茶水，咖啡或与同事闲聊的无关紧要的话题上-这次利用理论和对研究产生的问题进行分析。
  • 在午餐时间和吸烟室与同事进行工作讨论-人们通常很高兴在轻松的氛围中分享他们的知识。
  • 如果您有任何阻碍，请阅读并收听有关交通堵塞和车辆的讲座。
  • 请务必在笔记本上概述讲座和练习，不要依赖记忆。 如果您在听课时不清楚，请将笔记放在空白处。 例如，NB对于需要重复和加深的内容和“？” 需要澄清，询问，独立研究的内容。
  • 永远不要在晚上教书或学习-首先，您将长时间入睡，其次，所有事情都会在早晨被遗忘。
  • 在安静的环境中学习。 如果公司的政策允许（并且在IT领域几乎可以在任何地方允许），请在办公室多呆一个半小时来完成教育任务。
  • 不要学习有害的工作-这种故意的欺骗不会使任何人受益。
  • 如果您学习编程或系统管理，还不够记住理论并阅读Habr，那么您需要在实践中做所有事情：编写和测试代码，使用操作系统以及使用“笔”进行尝试。

而且，可能的主要建议是：不要像在学生时期那样对待学习。 通过为自己付费并针对实践的学习打分，就在欺骗自己。

如何与管理层协商？


如果要支付有偿学费，最好自己付钱-这样您就可以独立于雇主。 如果公司付款，您很可能必须制定一定的强制期限或在解雇后退还部分款项。 如果您没有解雇的计划，请务必与经理讨论部分或全额付款，并说明您的培训将如何有用。

在培训之前（实际上不是！），讨论时间表的更改或过渡为浮动时间表-通常，在IT部门中，他们最经常会与他们见面。

好吧，最主要的是，如果您了解自己还没有准备好投入适当的时间进行培训，并且会做好工作，由于诸如此类而跳过课程等，那么最好不要开始。 也许您已经是一个很酷的专家，而您根本没有足够的食物供头脑使用。 由您决定。
