Utilizamos Amazon Dash Button para nuestros propios fines.



Probablemente ya haya visto este video promocional sobre botones para ordenar productos con Amazon y probablemente también haya querido automatizar algo.

Debajo del gato, el caso del uso del botón amazon dash en las realidades rusas y su implementación en nodejs.

Introducción
Mientras leía artículos sobre IoT en las vacaciones de Año Nuevo, se me peinaron las manos y recién llegué para revisar el botón de Amazon ordenado para la prueba, después de haber jugado lo suficiente, comencé a pensar en cómo usarlo con al menos algún beneficio.
Recordé que en Flexby a menudo olvidamos preordenar agua para el refrigerador. Parece que ves, termina, pero mientras tomas té / café ya te olvidas. En consecuencia, la orden se envió solo cuando no había agua en absoluto.
Ahora es suficiente hacer clic en el botón "Comprar agua" y la notificación en holgura lo ayudará a no olvidarlo, pero fuimos un poco más allá y escribimos un guión que inmediatamente hace un pedido en el sitio de una compañía local de suministro de agua.
No, el pedido no se envía después de cada clic.
, . , .


Como funciona
0. El botón está "atado" a wi-fi de acuerdo con las instrucciones, pero no está completamente configurado, el producto no está especificado para que no se cree un pedido para Amazon.

1. Cuando hace clic en el botón:
1.1 se conecta a la red wi-fi para enviar una solicitud a Amazon.
1.2 El pedido no se forma porque El proceso de configuración del comportamiento del botón no se ha completado.

2. El script, en un servidor local conectado a la misma red wi-fi, monitorea la apariencia de la dirección mac del botón y concluye que fue presionado.
2.1 Se realizan las acciones que necesita.
en nuestro caso, esto es enviar un pedido de agua para el enfriador y notificarlo con holgura

ahora sobre todo en orden.

Vale la pena mencionar que hayesta es una versión para desarrolladores sobre cómo usarla, escribieron en geektimes con ella, por supuesto, es mucho más fácil y hermoso integrar lo que quieras, pero dónde obtener estos botones de iot sigue siendo un misterio para mí.

"Servidor" para un botón en NodeJs
Hay un módulo fácil de usar .
Puntos clave:
  • Funciona bajo Linux, posiblemente en amapolas (no probado).
  • Requiere libpcap-dev para funcionar.
  • El script debe ejecutarse con privilegios de superusuario.
  • Y, por supuesto, en una máquina conectada a la misma red wi-fi que el botón .


npm install node-dash-button

Si tiene dificultades para instalar, intente actualizar npm y node-gyp
sudo npm install -g npm
sudo npm install -g node-gyp

La dirección mac del botón se puede encontrar ejecutando el siguiente script: 
sudo node node_modules/node-dash-button/bin/findbutton

Bueno, o solo en el panel de administración del enrutador, determine el método de excepción.

Aquí de una manera simple, haga clic en el botón de captura:
sniffer = require('node-dash-button');

// mac-    
buttons = {
	'8f:3f:20:33:54:44':{
		action: function(){
			console.log('Press button 1')
		}
	},
	
	'2e:3f:20:33:54:22': {
		action: function(){
			console.log('Press button 2')
		}
	}
	// ...
};

//  
// Object.keys(buttons) -  mac-  
sniffer( Object.keys(buttons) ).on("detected", function (catched_mac){
	console.log(' mac-: '+catched_mac+'\n');

	//    
	buttons[catched_mac].action(); 
});

Es decir Decimos qué direcciones mac necesitamos cuando se iluminan. Llamamos a la función que necesitamos.

UPD : Un par de ideas para usar.
De todos modos, el caso principal, como se muestra en el comercial, es un recordatorio sobre la necesidad de reponer las reservas de algún recurso.

En la oficina :
  • El botón "Comprar papel", al lado de la impresora o el lugar donde se almacena el papel de impresión, envía un correo electrónico a la persona a cargo o hace un pedido de inmediato.
  • "Se acaban las cookies" - envíe un correo electrónico a quien debe reponerlas.


En casa :
  • "Compre papel higiénico", enviándose recordatorios a usted mismo, correo electrónico / tarea en wunderlist.
    Quizás este es el único recurso crítico :) El resto, por analogía, es fácil de pensar dependiendo de sus necesidades, pueden ser pañales, pasta de dientes, tabletas para lavavajillas, detergente, etc.

Por cierto, no encontré ninguna aplicación para la lista de compras con la API.

Algunas opciones no estándar :
  • Timbre de la puerta.
    Para que, por ejemplo, una llamada no perturbe el sueño del niño, permítale que envíe una notificación en silencio al teléfono. En otras ocasiones, dicha llamada puede incluir cualquier melodía, de nuevo será posible ajustar el volumen.
  • Contador de algo. El botón se puede usar como un llavero y si cuenta algo que necesita en el área de cobertura de su red wifi, es perfecto para esto.
    Por ejemplo, de 7:00 a 9:00 contamos el número de asociados que vinieron antes. De 9:00 a 13:00 tarde. De 13:00 a 15:00 detenidos a la hora del almuerzo, de 15:00 a 18:00 partieron temprano, de 18:00 a 21:00 detenidos en el trabajo. Pueden salir estadísticas interesantes.
    O puede considerar la cantidad de apelaciones orales de ayuda al departamento de TI.
