Botones de forma personalizada en la interfaz de usuario de Unity

Recientemente, tuve la necesidad de usar botones de forma no estándar en mi proyecto. Sin embargo, esta tarea aparentemente trivial me causó algunas dificultades.

El problema es que el botón de interfaz de usuario estándar maneja los clics incluso en áreas transparentes, lo que en este caso es completamente indeseable. Al mismo tiempo, el componente del botón en sí no tiene ningún parámetro (visible en el inspector) que pueda ser responsable de qué áreas del botón podrían manejar el desplazamiento / clic.


Después de un tiempo de búsqueda en Internet y documentación de fumar, se me ocurrió lo siguiente, con esta solución bastante simple:

(Diré de inmediato que hay un ejemplo similar en la documentación de Unity , pero en este momento su código ya está marcado como obsoleto, así que decidí actualizarlo y modificarlo un poco) 

using UnityEngine; using UnityEngine.UI; public class ButtonClickMask : MonoBehaviour { [Range(0f, 1f)] //1 public float AlphaLevel = 1f; //2 private Image bt; //3 void Start() { bt = gameObject.GetComponent<Image>(); bt.alphaHitTestMinimumThreshold = AlphaLevel; //4 } }

  1. Un atributo que muestra en el inspector un control deslizante conveniente con valores de 0 a 1.
  2. El valor mínimo del canal alfa que debe tener la parte de la textura sobre la que se encuentra el cursor para manejar los clics.
  3. El componente de imagen de un botón (debe trabajar con él y no con el botón).
  4. El parámetro alphaHitTestMinimumThreshold es exactamente cuál debe ser el nivel mínimo de transparencia para una parte de la textura para que pueda manejar el clic.

Entonces el guión está listo. Ahora pasemos a ajustar la textura del botón. Para que el código funcione y no produzca errores, debe habilitar la capacidad de lectura / escritura en la textura Importar configuración. Recuerde aceptar los cambios haciendo clic en Aplicar.



Ahora colgamos el script en el botón deseado, establecemos el Nivel Alfa en el valor deseado (en mi caso - 1) ...



... lanzamos y nos alegramos de que el botón ya no maneje clics en áreas transparentes.
