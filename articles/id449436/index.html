RHEL 8 Beta Workshop: Menginstal Microsoft SQL Server

Microsoft SQL Server 2017 telah tersedia untuk penggunaan penuh di bawah RHEL 7 sejak Oktober 2017, dan sambil bekerja pada RHEL 8 Beta Red Hat bekerja sama dengan Microsoft untuk meningkatkan produktivitas dan mendukung lebih banyak bahasa pemrograman dan kerangka kerja aplikasi, untuk menawarkan pengembang pilihan yang lebih luas alat yang tersedia untuk mengerjakan aplikasi mereka berikutnya.



Cara terbaik untuk memahami perubahan yang dibuat dan dampaknya pada pekerjaan Anda adalah dengan mencobanya, tetapi RHEL 8 masih dalam versi beta dan Microsoft SQL Server 2017 tidak didukung untuk digunakan dalam aplikasi dunia nyata. Apa yang harus dilakukan

Jika Anda ingin menguji SQL Server pada versi beta dari RHEL 8, posting ini akan membantu Anda meluncurkannya, tetapi Anda tidak boleh menggunakannya dalam lingkungan produksi sampai Red Hat Enterprise Linux 8 tersedia untuk umum dan Microsoft menyediakan paket yang didukung secara resmi untuk instalasi.

Salah satu tujuan utama Red Hat Enterprise Linux adalah untuk menciptakan lingkungan yang stabil dan konsisten untuk menjalankan aplikasi pihak ketiga . Untuk melakukan ini, RHEL mengimplementasikan kompatibilitas aplikasi pada tingkat API individu dan antarmuka kernel. Ketika kita beralih ke rilis besar baru, biasanya ada perbedaan khusus dalam nama paket, versi baru perpustakaan dan utilitas baru, yang dapat menyebabkan kesulitan ketika memulai aplikasi yang sudah ada yang dibangun untuk rilis sebelumnya. Vendor perangkat lunak dapat, mengikuti rekomendasi Red Hat, membuat binari di Red Hat Enterprise Linux 7 yang akan berjalan di Red Hat Enterprise Linux 8, tetapi bekerja dengan paket adalah masalah yang berbeda. Paket perangkat lunak yang dibuat untuk Red Hat Enterprise Linux 7 tidak akan didukung di Red Hat Enterprise Linux 8.

SQL Server 2017 di Red Hat Enterprise Linux 7 menggunakan python2 dan OpenSSL 1.0. Langkah-langkah berikut akan menyediakan lingkungan kerja yang kompatibel dengan dua komponen ini yang telah bermigrasi ke versi yang lebih baru di RHEL 8 Beta. Dimasukkannya versi yang lebih lama dibuat oleh Red Hat khusus untuk menjaga kompatibilitas. 

sudo yum install python2 sudo yum install compat-openssl10

Sekarang Anda perlu memahami pengaturan python awal pada sistem ini. Red Hat Enterprise Linux 8 dapat bekerja secara bersamaan dengan python2 dan python3 , tetapi secara default sistem tidak memiliki / usr / bin / python. Kita perlu menjadikan python2 sebagai penerjemah default sehingga SQL Server 2017 dapat melihat / usr / bin / python di mana ia mengharapkan untuk melihatnya. Untuk melakukan ini, jalankan perintah berikut: 

 sudo alternatives —config python

Anda akan diminta untuk memilih versi Python, setelah itu tautan simbolik akan dibuat, yang akan tetap ada setelah pembaruan sistem.

Ada tiga executable berbeda untuk bekerja dengan python: 

  Selection Command ———————————————————————- * 1 /usr/libexec/no-python + 2 /usr/bin/python2 3 /usr/bin/python3 Enter to keep the current selection[+], or type selection number:

Di sini Anda perlu memilih opsi kedua, setelah itu tautan simbolik akan dibuat dari / usr / bin / python2 ke / usr / bin / python.

Sekarang Anda dapat terus mengkonfigurasi sistem agar berfungsi dengan repositori perangkat lunak Microsoft SQL Server 2017 menggunakan perintah curl: 

 sudo curl -o /etc/yum.repos.d/mssql-server.repo https://packages.microsoft.com/config/rhel/7/mssql-server-2017.repo

Selanjutnya, unduh file instalasi SQL Server 2017 menggunakan fungsi boot baru di yum. Anda perlu melakukan ini sedemikian rupa sehingga Anda dapat melakukan instalasi tanpa harus menyelesaikan dependensi: 

 sudo yum download mssql-server

Sekarang instal server tanpa menyelesaikan dependensi menggunakan perintah rpm: 

 sudo rpm -Uvh —nodeps mssql-server*rpm

Setelah itu, Anda dapat melanjutkan instalasi normal SQL Server, seperti yang dijelaskan dalam Microsoft Quick Start: Instal SQL Server dan Buat Database di Red Hat panduan dari langkah # 3: 

 3.       mssql-conf setup         (SA)    . 


 sudo /opt/mssql/bin/mssql-conf setup

Setelah instalasi selesai, Anda dapat memeriksa versi server SQL yang diinstal menggunakan perintah: 

 # yum list —installed | grep mssql-server

Akan mendukung wadah


Dengan rilis SQL Server 2019, instalasi menjanjikan akan lebih mudah, karena versi ini diharapkan akan tersedia di RHEL sebagai wadah. SQL Server 2019 sudah tersedia dalam versi beta. Untuk mencobanya di RHEL 8 Beta, Anda hanya perlu tiga langkah:

Pertama, buat direktori basis data tempat semua data SQL kami akan disimpan. Untuk contoh ini, kita akan menggunakan direktori / var / mssql. 

 sudo mkdir /var/mssql sudo chmod 755 /var/mssql

Sekarang Anda perlu memuat wadah dengan SQL 2019 Beta dari repositori Microsoft Container repository dengan perintah: 

 sudo podman pull mcr.microsoft.com/mssql/rhel/server:2019-CTP2.2

Akhirnya, Anda perlu mengkonfigurasi server SQL. Dalam hal ini, kami akan menetapkan kata sandi administrator (SA) untuk database yang disebut sql1, yang berfungsi dengan port 1401 - 1433. 

 sudo podman run -e 'ACCEPT_EULA=Y' -e \ 'MSSQL_SA_PASSWORD=<YourStrong!Passw0rd>' \ —name 'sql1' -p 1401:1433 -v /var/mssql:/var/opt/mssql:Z -d \ mcr.microsoft.com/mssql/rhel/server:2019-CTP2.2

Informasi lebih lanjut tentang podman dan wadah di Red Hat Enterprise Linux 8 Beta dapat ditemukan di sini .

Bekerja untuk dua orang


Anda dapat mencoba bundel RHEL 8 Beta dan SQL Server 2017 menggunakan instalasi tradisional atau menginstal aplikasi kontainer. Dalam hal apa pun, Anda sekarang akan memiliki instance SQL Server yang berfungsi yang Anda inginkan, dan Anda dapat mulai mengisi database atau mempelajari alat yang tersedia di RHEL 8 Beta untuk membuat tumpukan aplikasi, mengotomatiskan proses tuning atau mengoptimalkan kinerja.

Pastikan untuk mendengarkan Bob Ward, arsitek senior di Microsoft Database Systems Group, di KTT Red Hat Summit 2019 pada awal Mei untuk membahas penyebaran platform data modern berdasarkan SQL Server 2019 dan Red Hat Enterprise Linux 8 Beta

Dan pada 8 Mei, rilis resmi diharapkan, membuka penggunaan SQL Server dalam aplikasi nyata.
