从IBM Notes / Domino迁移到Zimbra Collaboration Suite

在上一篇文章的评论中，有人提出了从IBM Notes / Domino迁移到Zimbra Collaboration Suite的可能性。 实际上，与从MS Exchange过渡到Zimbra Collaboration Suite的过程不同，在上一篇文章中已详细介绍了该过程，而从IBM Notes / Domino进行迁移要困难得多。 但是，正如我们所知，没有绝望的情况，现在我们将讨论从IBM Notes / Domino服务器转移到Zimbra Collaboration Suite服务器的内容和方法。



首先，值得注意的是，由于该过程非常复杂，因此最好将迁移委托给在IBM Notes / Domino中具有丰富经验的人员进行，以便他可以快速解决在数据传输过程中出现的问题。 可以在官方的Zimbra论坛以及Zimbra Wiki中找到可能出现的问题的解决方案。

对于直接迁移，我们将使用Lotus Domino的Zimbra Collaboration Server迁移向导 。 该用于MS Windows操作系统的应用程序将帮助将必要的用户帐户转移到IBM Notes / Domino，其邮箱的内容以及日历数据和联系人列表（从通讯簿到使用Zimbra的服务器）。 要使其正常工作，您将需要输入IBM Notes / Domino管理员帐户信息，以获得用于导入的用户数据。 请注意，如果以前从未创建过，则用于Lotus Domino的ZCS迁移向导会为从IBM Notes / Domino服务器转移的帐户单独创建帐户。 我们将在安装IBM Notes / Domino的计算机上启动Lotus Domino的ZCS迁移向导。 为了使其正常工作，它将需要IBM Notes / Domino随附的最新版本的.dll文件。

在迁移之前，与员工进行信息工作非常重要。 有必要通知每个将其帐户转移到新服务器的员工，他即将使用具有新界面的新电子邮件应用程序。 此外，有必要向每个人提供有关Zimbra Collaboration Suite将在其上运行的域名的信息，以及将用于访问电子邮件和协作工具的用户名/密码对。 迁移本身最好在周末或假日计划，这时电子邮件的负担将很小。

我们还建议您一小部分迁移邮箱，这样过程就不会花费很多时间，并且您有机会快速更改已迁移邮箱的记录，以便新信件将已经到达Zimbra服务器上而不会保留在IBM Notes / Domino上。

启动Lotus Domino的ZCS迁移向导后，将提示您在几个对话框中输入某些信息。 在标题为“ 导入目标 ”的第一个窗口中， 您应该指示：

  • 主机名-Zimbra的服务器域名
  • 管理员端口 -访问Zimbra管理服务的端口（默认为7071）
  • 使用安全连接 -用于建立与服务器的安全连接
  • 用户名和密码 -Zimbra服务器上的管理员帐户数据。

在标题为“ 目标域”的窗口中，您必须使用Zimbra输入服务器上的域，IBM Notes / Domino的帐户将被转移到该域。

在“ LDAP服务器信息”窗口中，我们必须输入必要的信息以连接到LDAP服务器：

  • LDAP服务器 -内部或外部LDAP服务器的地址
  • LDAP端口-LDAP访问端口（默认为389）
  • 用户DN和密码 -输入专有名称和用户密码以访问LDAP
  • BasDN-输入基本专有名称进行搜索

如果Lotus Domino的ZCS迁移向导可以检测到IBM Notes / Domino服务器，则通常独立地填充“ 导入源”窗口中的信息。 如果没有发生，则必须输入：

  • Domino服务器名称 -具有IBM Notes / Domino的服务器域名
  • 管理员ID文件路径 -带有管理员数据的ID文件的路径
  • Notes INI文件路径-Notes INI文件的路径
  • 迁移专用文件夹 -选中此框可将用户专用文件夹传输到Zimbra服务器。 为此，您需要告诉程序要转移其帐户的用户的ID文件的路径，以及带有密码的文件的路径。

之后，在“ 源和目标邮箱”窗口中，我们必须选择将从Zimbra传输到服务器的邮箱。 该窗口有几个打开不同窗口的按钮。 因此，例如，在“ 对象选择器”窗口中，您可以找到并选择要迁移到新服务器的帐户。 为此，您需要开始输入邮箱的名称，自动完成功能本身将为您提供合适的选项。 在“ 查询生成器”窗口中，您应该创建一个查询以按用户在LDAP数据库中进行搜索。

之后，先前选择的帐户将出现在先前为空的“ 源和目标邮箱”窗口中，该窗口将被转移到Zimbra服务器。 如果您改变主意来移动邮箱，则可以将其从列表中删除。 如果列表正确编译，则可以继续。

在“ 目标帐户验证”阶段，Lotus Domino的ZCS迁移向导将检查Zimbra服务器上是否有与从IBM Notes / Domino服务器迁移的帐户名称相同的帐户。 如果没有名称相同的帐户，则将创建该帐户。

在“ 帐户设置”对话框中，管理员将需要配置在Zimbra服务器上迁移后创建的帐户。 特别是，您必须输入默认情况下此类用户将使用的密码，并且还必须指定所迁移帐户所属的用户组。

创建所有帐户后，您可以开始导入帐户的内容。 为此，在“ 导入项目”对话框中，选择应转移的组件：

  • 导入邮件 -转移邮件对应
  • 导入联系人 -转移联系人
  • 导入任务 -任务转移
  • 导入日历 -传输日历数据

在下一个称为“ 导入选项”的窗口中，将要求您选择几个选项：

  • 导入垃圾邮件文件夹 -是否转移垃圾邮件文件夹
  • 导入已删除邮件文件夹 -是否移动带有已删除邮件的“回收站”
  • 导入之后接收的电子邮件 -指示应从中传输消息的日期
  • 要同时导入的邮箱数-指定同时导入的邮箱数

成功完成一个迁移后，系统将提示您进行另一个迁移。 如果您利用我们的建议并将迁移分为几个部分，则可以稍作休息以更改电子邮件中继中的条目，以使发给新迁移用户的信件到达Zimbra服务器。

请注意，如果您面临仅将邮箱内容从IBM Notes / Domino传输到Zimbra的任务，则可以使用imapsync实用程序进行迁移。 先前的文章中介绍了这种迁移的过程。
