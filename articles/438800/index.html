PHP Pear pirateado y no disponible

Según el canal oficial de Twitter del popular repositorio de código fuente para PHP, se detectó un hack y el servicio se suspendió temporalmente. Pear ha estado distribuyendo código malicioso durante al menos medio año.

Ataque a PHP Pear: este ataque a la llamada cadena de suministro. Este tipo de ataque infecta un repositorio de distribución de software en el que confían los usuarios y, más adelante, se distribuye una cadena de código malicioso entre los usuarios del servicio. Dada la popularidad de Pear y la cantidad de tiempo que ha pasado desde la infección, se puede suponer que el código malicioso ha llegado a un gran número de usuarios del servicio.

Para todos los que han usado go-pear.phar durante el último medio año, se recomienda verificar las sumas de verificación de los paquetes con los de github. Si las sumas de verificación son diferentes, esto significa que el paquete está infectado. Algunos de los troyanos detectados son: Backdoor.PHP.AZI, Backdoor.PHP.Webshell, HEUR: Trojan.Script.Generic. Le permiten tomar el control completo del servicio infectado.
