“ Beginners Wordpress Note”风格的一个小技术题。

上次，我注意到一些读者喜欢此信息块，因此我决定再次分享一点经验。 对于经验丰富的用户，没有什么新鲜或特别的地方-仅是我关于设置博客的简短说明。

8月，我收到了一封邮件，通知我将博客引擎更新为当前版本。 我对核心感到惊讶。 在软件本身没有问题（或问题）的情况下进行更新的21世纪，这并不是完全令人惊讶的事情。 但是网站引擎！ 我以某种方式认为该站点是一个相当静态的东西-我安装，配置并且可以正常工作。

除非有人用自己的双手进入文件/数据库，否则托管者将不会发生任何事情，否则邪恶的黑客Vasya将会出现。 博客的主动性不太适合我。 同意我不希望站点在更新过程中失败，但是只有在事实发生后您才能知道它，并且如果您不能从备份中快速部署站点，那么它将花费一段时间。 我对可能进行更新的预先通知感到满意。 事实证明，要禁用Wordpress自动更新，只需编写一条指令即可： 

define('AUTOMATIC_UPDATER_DISABLED', true);

到wp-config.php文件。 该指令仅禁用自动更新，但不禁用定期检查更新。 因此，如果有更新，请转到管理员下的管理面板，通知将立即可见，并且可以手动解决它们针对引擎，插件和/或主题的安装。

一段时间后，文本的大量连字符引起了我的注意。 使用它们，文本可以很好地垂直对齐，可以平滑地填充页面的整个空间，但这是非常不寻常的。 此问题已通过样式解决-在style.css中，您需要禁用连字符。

我们正在寻找这样的东西： 

 /* Sidebar */ .widget-area .widget { -webkit-hyphens: auto; -moz-hyphens: auto; hyphens: auto; margin-bottom: 20px; word-wrap: break-word; } .site-content article { border-bottom: 2px double #F3F3F3; margin-bottom: 28px; padding-bottom: 24px; word-wrap: break-word; -webkit-hyphens: auto; -moz-hyphens: auto; hyphens: auto;

我们将其转换为以下形式： 

 /* Sidebar */ .widget-area .widget { -webkit-hyphens: none; -moz-hyphens: none; hyphens: none; margin-bottom: 20px; word-wrap: break-word; } .site-content article { border-bottom: 2px double #F3F3F3; margin-bottom: 28px; padding-bottom: 24px; word-wrap: break-word; -webkit-hyphens: none; -moz-hyphens: none; hyphens: none;

是的，我的意思是Orphus系统，当足够用鼠标选择一个单词或单词组合并按Ctrl + Enter时，就会向网站/文章所有者发送一条通知消息。 有一个特殊的Wordpress插件，可以为网站添加类似的功能。 我特别注意自主功能。 在我们的世纪中，当人们相信“乌云密布”并始终伴随着您时，自主的事物特别有价值。 与Orphus不同，在Orphus中需要注册，下载脚本并将其连接到站点，只需安装Mistape插件即可。 配置很少的插件设置非常简单，并且该站点上每篇文章的末尾都会出现一条有关使用Ctrl + Enter的通知文本。 但是...有几点。
