AWS Firecracker (Amazon Web Services)



"AWC introdujo Firecracker -" micro-virtualización "para Linux" :


En AWS re: Invent 2018 , que se está llevando a cabo estos días en Las Vegas, se realizó el anuncio de Firecracker , una nueva tecnología de virtualización de código abierto basada en Linux KVM. Los autores prometen que con él, "en una fracción de segundo, puede ejecutar máquinas micro-virtuales livianas (microVM) en un entorno no virtualizado, obteniendo las ventajas de las máquinas virtuales tradicionales, en forma de seguridad y aislamiento de cargas de trabajo y contenedores, en forma de uso eficiente de los recursos".

El proyecto se basó en el desarrollo de código abierto de Google, crosvm de Chromium OS, que está escrito en Rust y es responsable de lanzar sistemas operativos con virtualización de dispositivos (pero sin emular hardware real). Por lo tanto, el código de Firecracker también está escrito en el lenguaje Rust, y sus autores prometen devolver sus correcciones a la base del código del proyecto principal, aunque los proyectos en sí mismos han diferido mucho en su propósito.

RFC


De acuerdo con RFC (Solicitud de comentarios), es bastante conveniente observar en qué dirección se mueve el idioma, así que aquí hay algunos interesantes. Hay un enlace "Representado" en el encabezado de los RP, por el cual el RFC está disponible en forma legible.


Este mes han sido aceptados:


  • RFC # 2561 Posibilidades futuras : en los nuevos RFC habrá una sección con las perspectivas para el desarrollo de la idea propuesta (meta-RFC);
  • RFC # 2591 Estabilizar la concordancia exhaustiva de patrones enteros : estabilización de comparaciones numéricas exhaustivas, es decir, por ejemplo, si todos los 255 valores se enumeran en el u8 variables u8 , entonces no será necesaria la rama _ ;

RFC en consideración:


  • RFC # 2592 Estabilizar 'std :: task' y 'std :: future :: Future' - ofrece estabilizar futuros (esto es independiente de la sintaxis asíncrona / espera RFC);
  • RFC # 2593 Tipos de variantes de enumeración: ofrece ampliar las posibilidades de usar opciones de enumeración, acercándolas a las estructuras ordinarias;
  • RFC # 2584 Registros estructurales : sugiere agregar estructuras anónimas (o pueden llamarse tuplas con campos con nombre);
  • RFC # 2602 # [atributo] s en abundancia : sugiere permitir que los atributos se cuelguen en el VL, los tipos, los límites y un montón de otras cosas;
  • RFC # 2603 Symbol Mangling v2 : sugiere combinar el esquema de símbolos de decoración ;
  • RFC # 2500 Needle API propone agregar std::needle con abstracciones sobre &str , &mut str , &[T] , &mut [T] , Vec<T> y &OsStr ;

Pre-RFC:


  • Pre-RFC para tipos de variantes anónimos, una propuesta de tipo de suma anónima mínima : sugiere agregar asnos (_|_) enumeraciones anónimas con tipos como (NoneError|ParseIntError) ;
  • Pre-RFC: sum-enums : ofrece introducir un nuevo tipo de enumeración con la sintaxis enum(A, B, C, ..) y enum Foo(A, B, C, ..) ;
  • Rasgos de óxido y su (falta de) privacidad : Phaazon sugiere agregar la capacidad de tener métodos privados a los tipos ( aquí de alguna manera soy muy escéptico );

El RFC # 2328 que adoptó oficialmente a Ferris fue rechazado: decidieron que Ferris no tenía mucho sentido para hacer una mascota oficial;
