如何根据Google Data Studio中的方案预测为SEO选择有前途的密钥（+模板）



典型的情况：您需要将该网站升级为800个请求，但仅100个就够了，如何从总的量词中选择一个能在预算范围内增加流量和转化次数的词组？


一种选择是基于场景的模型，用于使用Data Studio预测流量和转化。 我们告诉您如何收集预测数据，为Yandex / Google建立模型并选择要提升的查询。


John McAlpin在SEJ的一篇文章中提出了一种预测模型。 该模型回答了两个问题：


  • 如果按键位置增加了1、2、3、4、5和10行，您在上一个时期（在设备方面）将获得多少流量和转换；
  • 哪一个特定的键就能很好地增加点击量和点击量，从而增加点击量和转化次数（在预算较低的情况下就是这种情况）。

重要！ 该模型未考虑季节性因素，也未建立每月预测。 这是一个方案预测，用于了解关键升级的前景。


使用Data Studio的预测模型如何工作


Data Studio允许您设置一个交互式仪表板，以评估单个关键字的订单项增长的潜在影响。 更新Google表格中的源数据时，报告和图形会自动更新。


我们将举一个具体的例子。 立即复制SEO预测数据文档。 我们将在这里介绍初始数据。




Google表格中的数据收集和模板填写


对于预测，您需要以下数据：


  • 关键字及其上的位置；
  • 频率
  • 有机桌面和移动流量的平均百分比；
  • 台式机和移动设备的平均转换率；
  • 自然搜索Yandex / Google中搜索结果的点击率。

键及其上的位置


可以从Yandex.Webmaster或Google Search Console下载密钥（取决于您需要预测的PS）。


要从Google Search Console卸载，请转到“效率”部分，打开平均排名的显示，将时间段设置为“过去28天”，然后以表格形式加载数据。




要从Yandex.Webmaster卸载密钥，请转到“搜索查询” /“查询统计信息”部分。 我们设置了上个月的数据显示，并添加了指标“平均”。 位置”并以XLSX格式加载数据。




问题在于从网站管理员面板上卸载密钥后，位置以十进制表示。




我们需要一个整数。 必须四舍五入。


下载密钥的另一种方法是使用PromoPult SEO模块中的免费工具。 它收集网站在Yandex和/或Google的TOP 50中的短语。


我们将网站添加到系统中并运行搜索引擎促销。 除了此处的位置之外，还将针对每个单词自动计算点击次数的预测-如果您需要快速估算已提升网站上的点击量，则此数据很有用。




使用Excel中的键手动复制表。




回到我们的模板。


SEO预测数据模板提供150个交钥匙的“座位”。 怎么了 向John McAlpin解释：


这个预测模型的目的是显示SEO可以提供什么效果，即使在处理数量有限的请求时也是如此。


该模型的优势在于，它评估的是提升特定密钥的潜在影响，而不是评估网站流量到达的整个请求集合。 您只分析了150个最重要的键，据估计，最有效的升级。


这是逻辑。 在TOP-20之外，尤其是TOP-30之外，流量趋于零。 例如，将位置从35增加到30不会对访问者的涌入产生最大的影响，例如，从13增加到8（甚至从8到3更大）。


我们的文件中有896个密钥。 选择150件，像这样：


  • 我们会根据有趣的PS（按Yandex）的增长按位置对密钥进行排序；



  • 我们在TOP-30中选择请求（我们不在第一位置考虑按键）；
  • 实际上，在TOP-30中，我们有150多个请求-我们以较低的频率（低于20）过滤密钥。



如果TOP-30中的键数少于150个，请使用几个键，并将其余模板单元格留空。


重要！ 如果要向模型添加更多键或更改脚本（例如，要预测不是按10个位置而是按15个位置增长），请对模板进行更改。


频次


选择按键后，您需要收集其频率。 如果您使用PromoPult解析密钥，那么您将已经具有Wordstat的频率。 使用Google 关键字规划师 （一种用于印象和点击的预测工具）在Google上收集频率。 也适用于Google，例如Keywordtool.io和KeyCollector。


因此，我们有按键，按键位置和频率。 将它们放在主工作表上复制的SEO预测数据模板中。 填充列A（键），B（位置），C（频率）和第2-151行（该区域以蓝色突出显示）。 在151行以下还有数据，将在填充2-151行后自动更新。 请勿更改其他任何内容。




接下来，填写模板中的第二张纸。


设备之间的自然流量分配


在Google Analytics（分析）中，此信息可用（“受众群体” /“移动设备” /“概述”）。 我们创建一个细分“搜索点击量”。 设备“平板电脑”的类型是移动的。




要在Yandex.Metrica中获得类似的信息，请转到“报告” /“标准报告” /“技术” /“设备”部分。 我们将细分设置为“搜索流量”。 在计算利息时，类别“平板电脑”与“智能手机”组合在一起。




Yandex.Metrica和Google Analytics（分析）中的数字略有不同，但并不重要。 自己采取任何选择。


台式机和移动设备的平均转化率


查看设置了目标或电子商务的转化。


例如，在我们的网站上，主要目标是下订单。 我们已经在Yandex.Metrica中设定了一个目标。 转到“报告” /“标准报告” /“转化”部分，设置期间-上个月（尽管您可以做更多）。 我们设置条件：“第一个来源的类型：搜索引擎的转换”和“设备：智能手机或平板电脑”。 我们将手机的转化率定为1.24％。




要在台式机上查看数据，只需将“设备：”条件从“智能手机或平板电脑”更改为“ PC”。 在我们的示例中，此处的转化率为1.68％。
我们在SEO预测数据模板的“平均CTR数据和输入”表上输入有关设备之间流量分配和转换的数据。




Yandex / Google自然搜索结果的点击率


点击次数在搜索结果之间分布不均。 为了进行预测，您需要知道自然搜索结果中每个位置获得的点击次数所占的比例。


Google没有任何问题。 AdvancedWebRanking服务按设备，地区（美国/全球），查询类型（品牌/非品牌），搜索意图（商业，信息等）和类别（家庭和花园，金融，体育和健身等）收集数据。 ）。




很明显，可点击性在每个地区和每个请求之间都不同，但是Google的点击率分布的总体情况是清晰的。


使用Yandex更加困难。 没有类似的在线监视服务。 有几种获取数据的方法：


  • 利用第三方点击率研究的结果（这是最后一项研究 -但此处的点击率仅在TOP-10中标明）。



Yandex中的点击率研究结果（在Cyber​​Marketing 2018大会上发表）


  • 使用您自己的站点数据库分析点击率。 但是此方法仅适用于大型代理和在线服务，这些代理和在线服务在数据库中具有数百个（甚至更好的是-数千个）站点。 获得或多或少可靠结果的唯一方法。
  • 接受有关Yandex中CTR的分布与Google中大致相同的假设，并使用AdvancedWebRanking数据。 由于缺少其他数据，这不是一个坏决定。

在“ SEO预测数据”模板的“平均CTR数据和输入”表上输入点击率数据（在流量和转化的表格右侧）。 对于位置1到20，我们使用AdvancedWebRanking数据。 对于位置21-29-0.005，对于位置30及以下位置-0.002。




因此，收集了数据并填充了Google表格中的模板。 剩下的工作就是下载Google Data Studio模板并将数据源连接到它。


将数据连接到Google Data Studio模板


打开Data Studio SEO Impact Analysis Scenarios模板并创建一个副本（原始模板包含错误-我们已将其修复）。 复制模板时，请指定已加载并填充数据的SEO预测数据-主工作表模板作为数据源。




如果您正确指定了源并且其中不包含错误，则Data Studio中的报告将自动填充。


在报告的底部，您将看到一个表，其中包含有关头寸增长情况的数据。 每个指标可用的升序/降序排序。




模板的顶部是两个图块：


  • 每个增长场景的流量和转化；
  • 按设备分配流量和转化。



在我们的示例中，一般情况下1-4个职位的增长不会带来明显的流量增长。 增加5个职位更为明显，但最显着-增加10个职位。 一切似乎都是合乎逻辑的，但是在单个键的上下文中，图片却完全不同-这是最有趣的。


让我们看一下纸尿裤的流量和转化动态。 为此，请单击报告标题中的“关键字”按钮，然后单击请求对面的“仅”按钮。




事实证明，按需增加10个职位将导致流量和转化次数减少。 这是由于以下事实：根据AdvancedWebRanking统计，位置14-18处的投放结果的点击率高于位置10-13处的点击率。 也就是说，如果您将请求从位置18“踢”到位置13，我们的流量就会减少。


如何处理此类请求？ 这里的一切都很简单。 如果预算非常有限，请将其推迟到更好的时间。 如果资源允许，我们将其投入使用，并将其放入TOP-5，甚至更好的TOP-3。


一个相反的例子是“尿布过敏”查询。 此处，随着10个职位的增加，流量和转化次数成倍增加。




应特别注意诸如“ Libero尿布”之类的查询。 即使头寸增加了3点，也开始出现明显的增长。 我们会在促销期间优先处理此类请求。




另一个类似的示例是查询“俄罗斯尿布”。




当然，在最后选择请求时，必须考虑到请求的意图，竞争程度和促销费用。


不过，通过我们进行的情景分析，我们可以识别出流量和转化次数都有所增长且排名略有上升的查询。 在它们上，您可以获得快速的结果。


当然，这不是选择预期密钥的唯一真正方法。 但是，将其与其他行之有效的方法结合使用，可以以较低的成本进行预测并获得结果。
