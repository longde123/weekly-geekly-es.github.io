1. Syarat Masuk dan Persyaratan Analis




Kekuatan BI


Saya melihat banyak pengguna Power BI yang bukan spesialis IT, tetapi yang dapat membuat laporan yang bagus. Power BI menggunakan bahasa kueri yang sama dengan Excel - Power Query dan bahasa rumus DAX. Banyak analis mengenal Excel dengan baik, sehingga beralih ke sistem BI ini cukup mudah bagi mereka.

Sebagian besar tindakan tidak cukup rumit di editor kueri. Plus ada editor canggih dengan bahasa M untuk para profesional.

Gambar 2. Perancang kueri BI Power

Qlik Sense


Qlik Sense terlihat sangat ramah - sejumlah kecil pengaturan, kemampuan cepat untuk membuat laporan, Anda dapat menggunakan konstruktor pemuatan data.

Pada awalnya, tampaknya lebih sederhana daripada Power BI dan Tableau. Tetapi dari pengalaman, saya akan mengatakan bahwa setelah beberapa saat, ketika analis membuat beberapa laporan sederhana dan membutuhkan sesuatu yang lebih kompleks, ia akan dihadapkan dengan kebutuhan untuk memprogram.

Qlik memiliki bahasa yang sangat kuat untuk memuat dan memproses data. Ada bahasa rumus Analisis Set milik. Oleh karena itu, analis harus dapat menulis kueri dan koneksi, memasukkan data dalam tabel virtual, dan secara aktif menggunakan variabel. Kemungkinan bahasanya sangat luas, tetapi akan membutuhkan studi. Mungkin semua analis Qlik yang saya kenal memiliki semacam latar belakang IT yang serius.

Qlik integrator, seperti kita, sering suka berbicara tentang model asosiatif, ketika ketika memuat data mereka semua ditempatkan dalam RAM, dan koneksi antara data dilakukan oleh mekanisme internal platform. Bahwa pada pilihan nilai, subqueries internal tidak dibuat, seperti dalam database klasik. Data dihasilkan hampir secara instan karena nilai dan hubungan pra-indeks.

Benar dalam praktiknya, ini mengarah pada pembuatan tabel otomatis bergabung ketika nama bidang cocok. Misalnya, Anda tidak dapat memiliki tabel berbeda tanpa hubungan yang memiliki bidang yang sama. Anda harus terbiasa dengannya. Anda harus mengganti nama kolom dan memastikan bahwa nama tidak cocok, atau menggabungkan semua tabel fakta menjadi satu dan mengelilinginya dengan direktori dari jenis "bintang". Mungkin nyaman untuk pemula, tetapi untuk analis berpengalaman tidak ada bedanya.

Antarmuka pemuatan dan pemrosesan data analitik khas terlihat seperti ini.

Gambar 3. Editor beban data Qlik Sense, tabel Kalender

Catatan: Di Power BI, situasinya biasanya terlihat berbeda, Anda meninggalkan tabel fakta dan buku referensi yang berbeda, Anda dapat secara manual bergabung dengan tabel dengan cara klasik, mis. Saya memetakan kolom satu sama lain secara manual.

Tablo


Pengembang Tableau memposisikan diri sebagai BI dengan antarmuka yang nyaman dan ramah pengguna yang akan memungkinkan analis untuk mempelajari data mereka secara independen. Ya, di perusahaan kami ada analis yang bisa membuat laporan tanpa pengalaman IT. Tapi saya akan menurunkan skor Tableau karena beberapa alasan:
- Lemah lokalisasi dengan bahasa Rusia
- Server Tableau Online tidak berada di Federasi Rusia
- Konstruktor beban yang cukup sederhana mulai menimbulkan masalah ketika Anda perlu membangun model data yang agak rumit.

Gambar 4. Tableau konstruktor pemuatan data

Salah satu pertanyaan yang kami ajukan pada wawancara dengan analis Tableau adalah "Bagaimana membangun model tabel fakta dengan tabel referensi tanpa membawa semuanya ke dalam satu tabel?!" Pencampuran data membutuhkan penggunaan yang bijaksana. Saya telah memperbaiki beberapa kali kesalahan duplikasi data analitik setelah asosiasi tersebut.

Plus, Tableau memiliki sistem yang agak aneh, ketika Anda membuat setiap bagan pada lembar terpisah, dan kemudian membuat Dashboard, di mana Anda mulai menempatkan lembar yang dibuat. Kemudian Anda dapat membuat Story, ini adalah kombinasi dari Dashboard yang berbeda. Pengembangan dalam Qlik dan Power BI lebih mudah dalam hal ini, Anda segera melempar template grafik pada sheet, mengatur ukuran dan dimensi, dan Dashboard sudah siap. Tampaknya bagi saya bahwa biaya tenaga kerja untuk persiapan di Tableau meningkat.

2. Sumber dan Unduhan Data




Tidak ada pemimpin yang jelas di bagian ini, tetapi kami akan menyoroti Qlik karena beberapa fitur yang bagus.

Tablo dalam versi gratis terbatas dalam sumber, tetapi dalam artikel kami, kami lebih fokus pada bisnis, dan bisnis dapat membeli produk komersial dan analis. Oleh karena itu, Tableau tidak menurunkan skor untuk parameter ini.

Gambar 5. Daftar kemungkinan sumber Tableau

Kalau tidak, daftar sumber sangat mengesankan di mana-mana - semua file tabel, semua database standar, koneksi web, semuanya bekerja di mana-mana. Saya tidak menemukan penyimpanan data yang tidak standar, mungkin ada beberapa nuansa di sana, tetapi dalam kebanyakan kasus Anda tidak akan mengalami masalah saat memuat data. Pengecualiannya mungkin hanya 1C. Tidak ada konektor langsung ke 1C.

Mitra Qlik di Rusia menjual konektor mereka sendiri seharga 100.000 - 200.000 rubel, tetapi dalam kebanyakan kasus lebih murah mengunggah dari 1C ke FTP ke Excel atau database SQL. Dan Anda dapat menerbitkan basis data 1C di web dan menyambungkannya menggunakan protokol Odata.

PowerBI dan Tableau dapat melakukan ini sebagai standar, tetapi Qlik akan meminta konektor berbayar, sehingga juga lebih mudah untuk mengunggah ke database perantara. Dalam hal apa pun, semua masalah koneksi dapat diselesaikan.

Gambar 8. Kekuatan sumber data BI

Tapi mengapa Qlik menonjol? Saya sangat suka 3 hal:
- File QVD
Format penyimpanan data sendiri. Terkadang Anda dapat membangun proyek komersial yang serius hanya pada file QVD. Misalnya, level pertama adalah data mentah. Level kedua adalah file yang diproses. Tingkat ketiga adalah data agregat, dll. File-file ini dapat digunakan dalam aplikasi yang berbeda, karyawan dan layanan yang berbeda dapat bertanggung jawab untuknya. Kecepatan unduhan dari file tersebut sepuluh kali lebih cepat daripada dari sumber data konvensional. Ini memungkinkan Anda untuk menghemat basis data dan bertukar informasi antara berbagai aplikasi Qlik.

- Pemuatan data tambahan
Ya, Power BI dan Tableau juga bisa melakukan ini. Tetapi Power BI akan membutuhkan versi Premium yang mahal, dan Tableau tidak memiliki fleksibilitas yang sama dengan Qlik. Dalam Qlik pada file QVD, Anda dapat mengambil snapshot sistem pada waktu yang berbeda dan kemudian memproses data ini sesuka Anda

- Koneksi skrip eksternal
Selain file QVD untuk menyimpan data, dalam kode skrip Qlik juga dapat dipindahkan ke luar aplikasi dan terhubung menggunakan perintah Sertakan. Ini sudah memungkinkan Anda untuk mengatur kerja tim, menggunakan sistem kontrol versi, mengelola satu kode untuk berbagai aplikasi. Power BI memiliki editor kueri lanjutan, tetapi kami tidak dapat mengonfigurasi kerja tim seperti di Qlik. Secara umum, semua BI memiliki masalah dengan ini, tidak mungkin untuk mengelola data dan kode dan visualisasi di semua aplikasi dari satu tempat. Maksimum yang dapat kami lakukan adalah membuat file QVD dan kode skrip. Elemen visual harus diedit di dalam laporan itu sendiri, yang tidak memungkinkan kami mengubah secara besar-besaran visualisasi semua klien pada saat yang sama.

Tapi bagaimana dengan mekanisme seperti koneksi langsung? Tablo dan Power BI mendukung koneksi LANGSUNG ke sejumlah sumber, tidak seperti Qlik. Kami agak tidak peduli dengan fitur ini, karena latihan menunjukkan bahwa ketika menyangkut data besar, bekerja dengan koneksi LANGSUNG menjadi sangat mustahil. Dan dalam kebanyakan kasus, BI diperlukan untuk data besar.

5. Lingkungan perusahaan - server, laporan




Semua produk memiliki versi server perusahaan. Saya bekerja dengan semua edisi dan saya dapat mengatakan bahwa setiap orang memiliki kekuatan dan kelemahan. Pilihan produk harus dipilih dari persyaratan perangkat lunak Anda, dengan mempertimbangkan nuansa mereka. Semua vendor dapat diberikan hak di tingkat akun dan grup, dan di tingkat Keamanan Tingkat Baris Data. Laporan yang dijadwalkan diperbarui secara otomatis.

Qlik Sense Enterprise adalah peluang besar untuk membangun analitik dalam organisasi Anda untuk bisnis skala menengah. Ini mungkin tampak lebih mahal daripada Power BI Pro, tetapi jangan lupa bahwa server Power BI Pro terletak di cloud di wilayah Microsoft dan Anda tidak dapat memengaruhi kinerja, dan ketika Anda membutuhkan Power BI Premium, yang dapat digunakan pada server Anda, maka Harganya mulai dari $ 5000 per bulan.



Qlik Sense Enterprise mulai dari 230.000 rubel. untuk 5 lisensi (biaya tahunan, maka hanya dukungan teknis), yang jauh lebih terjangkau daripada Power BI Premium. Dan Qlik Sense Enterprise akan memungkinkan Anda untuk memanfaatkan Qlik sepenuhnya. Mungkin kecuali satu. Untuk beberapa alasan, Qlik memutuskan bahwa fungsi seperti kemampuan untuk mengirim laporan ke PDF melalui email harus dikirimkan sebagai layanan NPrinting terpisah.

Tetapi Qlik Sense Enterprise lebih kuat dari Power BI Pro, dan karenanya perbandingan berikut dapat dilakukan di sini.

Qlik Sense Enterprise = Power BI Premium, dengan kemampuan yang sama, lebih murah dengan penerapan rata-rata. Implementasi besar biasanya sudah dihitung di sisi vendor, di mana mereka dapat memberikan kondisi individu untuk perusahaan Anda.

Dalam hal ini, kami akan memberikan preferensi kepada Qlik Sense Enterprise, ada setiap peluang untuk membangun analitik serius pada data yang sangat besar. Menurut pendapat kami, Qlik akan berjalan lebih cepat dari Power BI pada array besar, di konferensi Qlik ada klien yang pada awalnya menguji data mereka dalam miliaran catatan dan Power BI menunjukkan hasil yang lebih buruk.

7. Analisis tertanam di aplikasi / situs pihak ketiga




Menggunakan analitik sebagai layanan pihak ketiga tidak selalu nyaman. Mungkin Anda sedang mengembangkan produk Anda, tetapi Anda belum siap untuk mulai mengembangkan mesin visualisasi dan analisis dari awal. Mungkin Anda ingin menggunakan analitik di situs web Anda sehingga klien mendaftar, mengunggah datanya, dan menganalisisnya di dalam akun pribadinya. Untuk melakukan ini, Anda memerlukan Analisis Tertanam.
Semua produk memungkinkan Anda untuk melakukan ini, tetapi dalam kategori ini kami menyoroti Qlik.

Power Bi dan Tableau dengan jelas mengatakan bahwa untuk tujuan seperti itu Anda perlu membeli produk terpisah Tableau Embedded Analytics atau Power BI Embedded. Ini bukan solusi murah dalam ribuan dolar per bulan, yang segera membatasi penggunaannya. Sebagian besar proyek segera menjadi tidak menguntungkan bagi pelanggan kami. Ini berarti bahwa Anda tidak hanya perlu mempublikasikan laporan seluruh Internet, tetapi untuk memastikan publikasi laporan sesuai dengan akses tertentu, dengan perlindungan data, otorisasi pengguna, dll.

Dan Qlik akan membiarkanmu keluar. Tentu saja, mereka juga memiliki Platform Analisis Qlik yang dilisensikan ke server dan mengatur jumlah koneksi yang tidak terbatas. Ini juga akan mahal seperti pesaing Tableau dan Power Bi. Dan dalam hal koneksi tanpa batas, tidak ada opsi khusus.

Tapi Qlik memiliki konsep seperti Mashup. Katakanlah Anda memiliki Qlik Sense Enterprise dan 10 lisensi. Analitik standar, penampilan, semuanya sudah lelah. Anda meningkatkan situs web atau aplikasi Anda sendiri, dan Anda dapat menerapkan semua analitik Anda di sana. Caranya adalah, sederhananya, Mashup adalah visualisasi dalam kode program. Dengan API, Anda dapat secara terprogram mengatur pembuatan visualisasi di dalam aplikasi atau situs Anda. Anda masih memerlukan Qlik Sense Enterprise untuk lisensi (lisensi untuk menghubungkan ke situs = lisensi untuk terhubung ke BI), untuk mengunduh data, dll., Tetapi visualisasi tidak akan ditampilkan di sisi server, tetapi akan dimasukkan ke dalam aplikasi Anda atau situs. Anda dapat menggunakan gaya CSS, mengatur font baru, warna. 10 pengguna Anda tidak akan lagi masuk ke server analytics,Mereka akan menggunakan portal atau aplikasi perusahaan Anda. Analisis akan mencapai level baru.
