进行成功的移动应用A / B测试的提示和技巧


根据最新统计数据，全球有超过27亿智能手机用户，Google Play商店中有超过280万应用程序。


现在，事实是，移动用户的数量正以惊人的速度增长，Google Play商店中的应用程序名称也是如此。


但是，所有应用程序都能成功发挥作用吗？ 不，但只有少数能脱颖而出并获得人气。


例如，Play商店中有几款游戏，但为什么只有《糖果粉碎》，《地铁冲浪者》和《愤怒的小鸟》位居榜首，而其他游戏却连10次下载都挣扎。


诀窍是为玩家提供他们想要的东西，反过来，此类应用程序的反应巨大。


但是，在这一点上，当竞争非常激烈时，要想出一些能在人群中脱颖而出的新事物是具有挑战性的。


建立品牌名称以使人们能够谈论它至关重要。


因此，如果您正努力使自己的应用程序受到关注，但仍然滞后，那么该是重新审视您的应用程序策略并对其进行尽可能多的修改的时候了。


但是你该怎么办？


让自己陷入客户的困境，并尝试找出为什么您的应用无法正常工作。 您的客户需要什么？ 在这种方法中，对您有帮助的是A / B测试。


什么是A / B测试？


大多数人误以为A / B测试只是对标题，颜色或号召性用语进行一些细微调整的一种方法，但实际上，它比这更深刻。


简而言之，A / B测试是一种了解应用程序访问者中哪种设计，功能或内容更成功的方法。 它使您可以测试可能影响客户行为的页面变体。


这也称为“拆分”测试，因为您可以向一半用户显示应用程序版本A，而向另一用户显示版本B。


为什么需要A / B测试？


为移动应用程序运行A / B测试的好处类似于在网络上运行测试的好处。


它允许您测试各种假设以了解移动应用程序中的体验，并根据实际数据（而非直觉）进行修改。


A / B测试将使您能够对所做更改将产生的影响产生统计上的信心，从而确定正确的选项，并准确地衡量影响的程度。


通过在移动应用程序上定期运行测试，您将能够不断修改该应用程序并改善其使用体验以及转化指标。


这就是Google，Facebook和Amazon等巨头继续对其移动应用程序进行A / B测试的原因。


进行成功的移动应用程序A / B测试的提示和技巧



只选择一个变量进行测试


在开始优化应用程序的元素时，您可能会发现有几个要测试的变量。


但是要评估更改的有效性，您需要隔离一个“独立变量”并衡量性能。


如果一次优化多个元素，则可能无法理解是哪个元素导致了性能变化。


您可以为一个应用程序测试多个元素，但请确保一次仅测试一个元素。


查看不同的元素及其在设计，布局和措辞上的可能替代方案。


请记住，即使是很小的更改，例如更改颜色，号召性用语按钮，按钮位置，页面内容，也可以推动显着改善。


确定目标


尽管您将为每个测试测量多个指标，但是在开始测试之前，需要选择一个重点关注的指标。


要选择目标，以考虑您希望此变量在拆分测试结束时位于何处。 您可以陈述正确的假设并根据这些预测检查结果。


如果您没有在开始之前设定目标，则可能无法最有效地进行测试。


测试您的App Store说明


您的应用商店说明是用户要检查的最重要的内容之一。 通过这种方式，您可以使用文字来描述您的应用，从而在转化方面产生重大影响。


您可以从营销信息或客户评论和评级开始对应用程序的描述。 时不时尝试不同的方法，并检查哪种方法最适合您的应用。


征求用户的反馈


您的用户是您的首要任务，您需要知道他们喜欢什么，什么不是，毕竟这是A / B测试的全部内容。


A / B测试与定量数据有很大关系，但这并不能告诉您为什么用户随后采取特定措施。 在运行A / B测试时，为什么不要求用户反馈？


最好的方法是进行调查和民意调查。 您可以在应用程序上添加退出调查，询问用户为什么他们没有点击某些特定的CTA。


这一小步骤将使您了解用户的喜好以及可以在应用程序中进行更改的内容。


继续努力以实现长期目标


每当我们寻求改变时，我们往往倾向于忽略整体结果。 我们倾向于将重点放在短期结果上而不是长期结果上。


显然，您的主要目标是立即提高应用程序的转换率，但您可能无法在首次运行测试后立即实现所有转换。


有时，花费比预期时间更多的时间才能获得预期的结果。 因此，您必须保持良好的工作状态，因为在测试移动应用程序时总会有优化的余地。


根据结果​​采取行动


如果一项更改在统计上比其他更改要好，那您就赢了。 如果两种更改的统计结果都没有更好，则需要将该测试标记为不确定的。


在这种情况下，请保留较旧的版本或运行另一项测试以探索机会。 您可以使用失败的数据找出新的修改，以进行进一步的分析。


虽然A / B测试可以帮助任何移动应用程序开发公司逐案影响结果，但您也可以应用从以前的标准中学到的经验教训，并将其应用于将来的标准。


结论


当应用程序开发人员构建应用程序，添加功能，添加各种颜色或设计号召性用语按钮时，可能会很想利用他们的直觉来预测促使用户参与，转换和重新访问的因素。


但是，根据“本能”做出发展决定可能对结果不利。 建议不要进行猜测，而应该进行A / B测试并确定最适合该应用程序的方法。


刚开始时，这似乎是一项艰巨的任务，但是随着时间的流逝，您将开始探索改善用户体验，获得更多潜在客户和转换的可能方法。
