RHEL 8 Beta Workshop: Installieren von Microsoft SQL Server

Microsoft SQL Server 2017 ist seit Oktober 2017 für die vollständige Nutzung unter RHEL 7 verfügbar. Während der Arbeit an RHEL 8 Beta arbeitete Red Hat eng mit Microsoft zusammen, um die Produktivität zu steigern und mehr Programmiersprachen und Anwendungsframeworks zu unterstützen, um Entwicklern eine größere Auswahl zu bieten verfügbare Tools, um an ihrer nächsten Anwendung zu arbeiten.



Der beste Weg, um die vorgenommenen Änderungen und ihre Auswirkungen auf Ihre Arbeit zu verstehen, besteht darin, sie auszuprobieren. RHEL 8 befindet sich jedoch noch in der Beta-Phase und Microsoft SQL Server 2017 wird für die Verwendung in realen Anwendungen nicht unterstützt. Was tun?

Wenn Sie SQL Server auf der Beta-Version von RHEL 8 testen möchten, hilft Ihnen dieser Beitrag beim Starten. Sie sollten es jedoch nicht in einer Produktionsumgebung verwenden, bis Red Hat Enterprise Linux 8 öffentlich verfügbar ist und Microsoft das offiziell unterstützte Paket für bereitstellt Installation.

Eines der Hauptziele von Red Hat Enterprise Linux ist die Schaffung einer stabilen, konsistenten Umgebung für die Ausführung von Anwendungen von Drittanbietern . Zu diesem Zweck implementiert RHEL die Anwendungskompatibilität auf der Ebene einzelner APIs und Kernelschnittstellen. Wenn wir zu einer neuen großen Version übergehen, gibt es normalerweise spezielle Unterschiede in den Namen von Paketen, neuen Versionen von Bibliotheken und neuen Dienstprogrammen, die beim Starten vorhandener Anwendungen, die für die vorherige Version erstellt wurden, zu Schwierigkeiten führen können. Softwareanbieter können gemäß den Empfehlungen von Red Hat Binärdateien in Red Hat Enterprise Linux 7 erstellen, die unter Red Hat Enterprise Linux 8 ausgeführt werden. Die Arbeit mit Paketen ist jedoch eine andere Sache. Das für Red Hat Enterprise Linux 7 erstellte Softwarepaket wird unter Red Hat Enterprise Linux 8 nicht unterstützt.

SQL Server 2017 unter Red Hat Enterprise Linux 7 verwendet Python2 und OpenSSL 1.0. Die folgenden Schritte bieten eine Arbeitsumgebung, die mit diesen beiden Komponenten kompatibel ist, die bereits auf neuere Versionen in RHEL 8 Beta migriert wurden. Die Aufnahme älterer Versionen wurde von Red Hat speziell vorgenommen, um die Abwärtskompatibilität zu gewährleisten. 

sudo yum install python2 sudo yum install compat-openssl10

Jetzt müssen Sie die anfänglichen Python-Einstellungen auf diesem System verstehen. Red Hat Enterprise Linux 8 kann gleichzeitig mit Python2 und Python3 arbeiten , aber standardmäßig verfügt das System nicht über / usr / bin / python. Wir müssen python2 zum Standardinterpreter machen, damit SQL Server 2017 / usr / bin / python dort sehen kann, wo es es erwartet. Führen Sie dazu den folgenden Befehl aus: 

 sudo alternatives —config python

Sie werden aufgefordert, eine Version von Python auszuwählen. Anschließend wird ein symbolischer Link erstellt, der nach dem Systemupdate erhalten bleibt.

Es gibt drei verschiedene ausführbare Dateien für die Arbeit mit Python: 

  Selection Command ———————————————————————- * 1 /usr/libexec/no-python + 2 /usr/bin/python2 3 /usr/bin/python3 Enter to keep the current selection[+], or type selection number:

Hier müssen Sie die zweite Option auswählen, nach der eine symbolische Verknüpfung von / usr / bin / python2 zu / usr / bin / python erstellt wird.

Jetzt können Sie das System mit dem Befehl curl weiter für die Arbeit mit dem Microsoft SQL Server 2017-Software-Repository konfigurieren: 

 sudo curl -o /etc/yum.repos.d/mssql-server.repo https://packages.microsoft.com/config/rhel/7/mssql-server-2017.repo

Laden Sie als Nächstes die SQL Server 2017-Installationsdateien mit der neuen Startfunktion in yum herunter. Sie müssen dies so tun, dass Sie die Installation durchführen können, ohne die Abhängigkeiten auflösen zu müssen: 

 sudo yum download mssql-server

Installieren Sie nun den Server, ohne Abhängigkeiten mit dem Befehl rpm aufzulösen: 

 sudo rpm -Uvh —nodeps mssql-server*rpm

Danach können Sie die normale Installation von SQL Server fortsetzen, wie im Microsoft-Schnellstart beschrieben: Installieren von SQL Server und Erstellen einer Datenbank in Red Hat - Handbuch ab Schritt 3: 

 3.       mssql-conf setup         (SA)    . 


 sudo /opt/mssql/bin/mssql-conf setup

Nach Abschluss der Installation können Sie die Version des installierten SQL-Servers mit dem folgenden Befehl überprüfen: 

 # yum list —installed | grep mssql-server

Unterstützt Container


Mit der Veröffentlichung von SQL Server 2019 verspricht die Installation noch einfacher zu werden, da diese Version voraussichtlich in RHEL als Container verfügbar sein wird. SQL Server 2019 ist bereits in der Beta verfügbar. Um es in RHEL 8 Beta zu versuchen, benötigen Sie nur drei Schritte:

Erstellen Sie zunächst ein Datenbankverzeichnis, in dem alle unsere SQL-Daten gespeichert werden. In diesem Beispiel verwenden wir das Verzeichnis / var / mssql. 

 sudo mkdir /var/mssql sudo chmod 755 /var/mssql

Jetzt müssen Sie den Container mit SQL 2019 Beta aus dem Microsoft Container Repository-Repository mit dem folgenden Befehl laden: 

 sudo podman pull mcr.microsoft.com/mssql/rhel/server:2019-CTP2.2

Schließlich müssen Sie den SQL Server konfigurieren. In diesem Fall legen wir ein Administratorkennwort (SA) für die Datenbank sql1 fest, das mit den Ports 1401 - 1433 funktioniert. 

 sudo podman run -e 'ACCEPT_EULA=Y' -e \ 'MSSQL_SA_PASSWORD=<YourStrong!Passw0rd>' \ —name 'sql1' -p 1401:1433 -v /var/mssql:/var/opt/mssql:Z -d \ mcr.microsoft.com/mssql/rhel/server:2019-CTP2.2

Weitere Informationen zu Podman und Containern in Red Hat Enterprise Linux 8 Beta finden Sie hier .

Funktioniert für zwei


Sie können das RHEL 8 Beta und SQL Server 2017-Bundle entweder mit einer herkömmlichen Installation oder mit einer Containeranwendung testen. In jedem Fall steht Ihnen jetzt eine funktionierende Instanz von SQL Server zur Verfügung. Sie können die Datenbank füllen oder die in RHEL 8 Beta verfügbaren Tools untersuchen, um einen Anwendungsstapel zu erstellen, den Optimierungsprozess zu automatisieren oder die Leistung zu optimieren.

Hören Sie sich Bob Ward, Senior Architect bei der Microsoft Database Systems Group, auf dem Red Hat Summit 2019 Anfang Mai an, um die Bereitstellung einer modernen Datenplattform auf Basis von SQL Server 2019 und Red Hat Enterprise Linux 8 zu erörtern Beta

Am 8. Mai wird eine offizielle Version erwartet, die die Verwendung von SQL Server in realen Anwendungen ermöglicht.
