Cómo se sincronizan las aplicaciones de finanzas personales con los bancos

Hoy, cada persona tiene tres o cuatro cuentas en diferentes bancos. El seguimiento de cada uno individualmente es difícil. Las aplicaciones para contabilizar las finanzas con la función de sincronización con los bancos ayudan a simplificar la tarea.

¿Por qué necesito sincronización?


La sincronización de las cuentas bancarias es necesaria, en primer lugar, para no perder el tiempo registrando gastos, y en segundo lugar, para combinar cuentas de diferentes bancos en un solo lugar.

Muchos de nosotros dejamos de tener en cuenta los ingresos y los gastos solo porque nos olvidamos de hacerlos diariamente en una computadora portátil, archivo o programa. Esto puede ser muy desconcertante. Cuando se conecta la sincronización, puramente psicológicamente se vuelve más fácil.

Además, hay tiempo para un análisis más profundo de su situación financiera, conclusiones y planificación.

Formas de sincronizar con bancos


Hoy en día, hay tres formas de sincronizar aplicaciones con bancos:

  1. Procesamiento de SMS desde bancos.
  2. Conexión a bancos mediante servicios de terceros.
  3. Sincronización con complementos de aplicaciones.

Consideremos cada uno de ellos.

Procesamiento de SMS desde bancos


Este método de sincronización se está volviendo gradualmente obsoleto. Aunque a muchos usuarios les gusta porque lo encuentran más seguro que otros métodos.

¿Por qué no se usará el método pronto?

1. Debido a limitaciones del sistema operativo. En iOS, las aplicaciones no tienen acceso a SMS. Lo mismo sucederá pronto con Android. En el "Centro para Reglas de Desarrolladores" hay una sección sobre confidencialidad, que prohíbe el uso de SMS por aquellas aplicaciones que no son ayudantes o procesadores de mensajes o llamadas en blanco y negro.
Las aplicaciones que no pueden asignarse como asistente o controlador predeterminado para SMS o llamadas tienen prohibido solicitar dichos permisos. Incluso para ellos es imposible especificar las etiquetas correspondientes en el manifiesto.

Para solicitar los permisos anteriores, la aplicación debe estar registrada como asistente o manejador de SMS o llamadas de forma predeterminada. Si se asigna otra aplicación como asistente o controlador para SMS o llamadas, debe dejar de usar estos permisos de inmediato. Fuente
2. Los bancos también recientemente rechazan cada vez más los SMS y cambian a notificaciones push. Después de instalar la aplicación bancaria en un teléfono inteligente, es muy posible desactivar las alertas por SMS incluso sin el conocimiento del usuario. Las notificaciones push solo pueden ser aceptadas por la aplicación del banco, y es posible que la aplicación de finanzas no reconozca dicho mensaje.

Conectarse a bancos utilizando servicios de terceros


Para no molestarse con la sincronización, los desarrolladores de algunas aplicaciones de contabilidad para finanzas utilizan soluciones desarrolladas por terceros.

Dichos servicios utilizan API de banca abierta. Las aplicaciones conectadas obtienen acceso al historial de transacciones del usuario.

A partir de las solicitudes de contabilidad para las finanzas, Wallet, Spendee, Fentury utiliza este sistema. Según las opiniones de los usuarios de estas aplicaciones, el intercambio es lento. La actualización de la sincronización con el banco lleva unos cinco minutos. Primero, un minuto y medio es la conexión. Luego, el sistema solicita la contraseña que vino en SMS. Solo entonces comienza la sincronización.

Además de este inconveniente, hay tres más:

  1. Debe pagar por la sincronización, y esto afectará el costo de la suscripción.
  2. No es seguro transferir inicios de sesión y contraseñas, porque los datos se transfieren a servidores de terceros.
  3. Si se producen errores de sincronización (y los usuarios están muy nerviosos), no se sabe qué tan rápido se reparará.

Sincronización con complementos de aplicaciones


La sincronización desde la aplicación funciona según el principio de los complementos que están escritos en API de aplicaciones abiertas y utilizan datos abiertos de bancos en línea. Esto permite a los usuarios que tienen conocimientos profesionales y desean escribir un código por su cuenta y conectarse a un banco antes de que aparezca la sincronización en la aplicación. Este método se usa en una aplicación Zen mani.

La principal desventaja del método, en comparación con el anterior, es la velocidad de desarrollo. Por lo general, los desarrolladores eligen bancos que son populares entre los usuarios para la sincronización. Todos los demás tienen que esperar mucho tiempo o conectarse para informar por SMS.

Este método es seguro. Todos los datos se almacenan en forma cifrada en el teléfono del usuario. Se transfieren solo al banco. Cuando se elimina la aplicación, se cambia la clave de cifrado o el usuario abandona el perfil, todos los datos sobre la conexión al banco se eliminan automáticamente.

Increíble, pero cierto!


A pesar de la seguridad y todas las comodidades que aparecen cuando sincroniza la aplicación con el banco, la mayoría de los usuarios aún intentan usar algunas formas "manuales" para obtener información del banco. Es decir, descargue y cargue el archivo CSV. A veces esto no es una experiencia agradable, porque se producen errores.

De los tres métodos, el primero es el más confiable, porque es el más familiar y se considera el más seguro después de importar archivos. La gente sabe menos sobre sincronización. Es difícil entender la parte técnica. Hasta que lo intente, no está claro si vale la pena contar con confiabilidad y seguridad.
