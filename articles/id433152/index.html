Kami mentransfer Zimbra dari server tunggal ke infrastruktur multi-server

Dalam artikel sebelumnya, kami berbicara tentang cara menginstal Zimbra pada infrastruktur multi-server. Selain peningkatan ketahanan, infrastruktur ini mudah disesuaikan dan ideal untuk penyedia SaaS yang menjual email dan alat kolaborasi untuk layanan dan perusahaan besar. Tetapi bagaimana jika Zimbra pada awalnya diinstal pada satu server di perusahaan Anda, tetapi ketika perusahaan tumbuh dan berkembang, kecepatan dan ketersediaannya mulai menurun secara signifikan? Solusi yang jelas adalah dengan memigrasi Zimbra Collaboration Suite ke infrastruktur multi-server, tetapi apakah mungkin untuk melakukan ini tanpa downtime yang lama dan tanpa kehilangan data berharga?



Ternyata ini mungkin. Saat menggunakan solusi dari Zextras, proses migrasi ke infrastruktur baru akan berlangsung dalam beberapa langkah:

  1. Instalasi bersih Zimbra pada infrastruktur multiserver yang Anda inginkan
  2. Menginstal Winterlet Cadangan Zextras di Server Sumber
  3. Instal Cadangan Zextras di server tujuan dengan Penyimpanan Kotak Pesan
  4. Membuat cadangan dengan konten server sumber
  5. Menyebarkan Cadangan Zimbra pada Infrastruktur Target
  6. Commissioning infrastruktur multi-server dengan Zimbra

Proses instalasi multi-server Edisi Zimbra Edisi Terbuka dijelaskan secara rinci dalam artikel sebelumnya . Setelah elemen Zimbra yang diinstal pada server yang berbeda mulai bekerja secara keseluruhan, dimungkinkan untuk memulai proses "pindah" dari server lama ke infrastruktur baru. Zextras Backup musim dingin yang dirancang oleh perusahaan IT Italia Zextras dan termasuk dalam paket ekstensi modular Zextras Suite akan baik-baik saja dengan tugas ini.



Setelah menginstal zimblet di server sumber, jalankan fungsi Ekspor Cadangan. Zextras Backup akan memindai server Anda dan membuat salinan cadangan dari isi kotak surat, pengaturan dan parameter pengguna, data kalender, file bersama, serta informasi tentang grup dan domain pengguna. Setelah cadangan dibuat, tidak akan berlebihan untuk menyimpannya di tempat lain untuk keandalan. Untuk keperluan ini, hard drive eksternal dan penyimpanan cloud cocok.

Perhatikan bahwa karena masa uji coba 30 hari, menggunakan Zextras Backup untuk menggunakan salinan cadangan server Zimbra asli pada infrastruktur baru dapat benar-benar gratis dan pada saat yang sama mengevaluasi kesederhanaan dan kemudahan penggunaan Zextras Suite.

Instal Zextras Suite di satu atau lebih server Mailbox dari infrastruktur target. Setelah instalasi selesai, dimungkinkan untuk menggunakan fungsi Impor Cadangan yang disediakan oleh Cadangan Zextras untuk memulihkan dari cadangan. Di kotak dialog yang muncul, Anda dapat memilih mana dari item yang dipesan harus dikembalikan ke server ini. Berkat ini, administrator infrastruktur Zimbra dapat menyebar berbagai domain atau bahkan akun di berbagai server Kotak Pesan.



Setelah pemulihan cadangan server sumber pada infrastruktur target selesai, infrastruktur multiserver Zimbra dapat ditugaskan.

Akan tetapi, pembaca yang penuh perhatian akan menarik perhatian pada fakta bahwa metode ini masih menyiratkan penghentian dalam pekerjaan server sumber Zimbra saat cadangan diambil dan digunakan untuk infrastruktur target. Bergantung pada jumlah data, pemutusan tersebut dapat bervariasi dari beberapa menit hingga nilai yang sepenuhnya tidak dapat diterima. Jika durasi pemadaman server tidak sesuai dengan Anda, Anda dapat menggunakan "Migrasi tambahan" - metode yang memungkinkan Anda untuk bermigrasi tanpa menghentikan server, terlepas dari jumlah data yang ditransfer.

Untuk melakukan migrasi inkremental, pengguna harus menginstal Zextras Backup di server sumber, serta pada satu atau beberapa server kotak surat dari infrastruktur target. Untuk mengurangi beban pada server yang sedang berjalan, disarankan untuk menghubungkan hard drive eksternal dan pasang ke folder / opt / zimbra / backup / zextras / . Juga, sebelum memulai migrasi, disarankan untuk mengatur nilai TTL ke 300 dalam catatan MX DNS North untuk memaksimalkan kecepatan proses transfer data.



Mari kita mulai migrasi dengan menonaktifkan semua server Real Time Scanner menggunakan cadangan zxsuite setProperty ZxBackup_RealTimeScanner perintah palsu . Kemudian, pada server sumber, luncurkan SmartScan menggunakan perintah doSmartScan cadangan zxsuite . Berkat ini, semua data kami diekspor ke folder / opt / zimbra / backup / zextras / , dan jika sebelumnya Anda memasang hard drive eksternal di dalamnya, Anda juga akan mendapatkan media cadangan. Perhatikan bahwa jika Anda berniat untuk menyebarkan domain dari server sumber untuk server Kotak Surat yang berbeda pada infrastruktur target, maka Anda juga dapat melakukan ini dengan menggunakan perintah form zxsuite backup doExport / path / ke / export / folder / domains domain1.com, domain2.com dan akhirnya mendapatkan cadangan dari berbagai domain.

Setelah itu, Anda dapat melanjutkan untuk menggunakan cadangan pada infrastruktur target. Untuk melakukan ini, kami sarankan menggunakan opsi Provisioning Only , yang memungkinkan Anda untuk mengimpor hanya domain, grup pengguna dan data akun, mengabaikan konten kotak surat. Untuk menggunakan opsi ini, masukkan perintah backup zxsuite doExternalRestore / path / untuk / the / data / provisioning_only TRUE pada server target. Ulangi prosedur ini dengan setiap server dan buat cadangan domain yang sesuai jika Anda ingin mendistribusikannya di berbagai server Kotak Pesan

Sebagai hasilnya, Anda akan mendapatkan salinan utara yang berfungsi tanpa isi kotak surat. Setelah itu, Anda dapat segera mengalihkan aliran huruf ke infrastruktur target dan kemudian mulai memindahkan isi kotak surat dari server lama menggunakan cadangan zxsuite doExternalRestore / path / untuk / the / data / perintah. Kelemahan yang jelas dari pendekatan ini adalah bahwa pengguna Zimbra tidak akan memiliki akses ke email yang dikirim dan diterima selama beberapa waktu, meskipun mereka akan terus menerima dan mengirim email secara normal. Selain itu, selama pemulihan langsung isi kotak surat, kinerja server dapat turun. Setelah pemulihan konten kotak surat selesai, kita dapat berbicara tentang keberhasilan penyelesaian migrasi tanpa downtime.
