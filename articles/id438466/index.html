Kruk untuk laser Cina

Halo

Saya bekerja sebagai operator penanda laser. Saya menempatkan gambar pada papan nama logam. Pada artikel ini saya ingin membagikan sedikit trik, seperti yang biasa dikatakan - "hack hidup".

Pendahuluan


Saya akan mulai sedikit dari jauh. Saya ditugaskan di instalasi teknologi kecil, laser serat solid-state Unilaser, diproduksi di Cina. Mesin ini dikendalikan oleh perangkat lunak khusus, EzCad2 [Unilaser] . Program ini menyediakan sejumlah kemungkinan untuk merender gambar untuk menggambar, mengontrol mode pemotongan. Namun, memposisikan teks di dalamnya secara akurat sangat sulit, oleh karena itu, editor vektor tambahan digunakan. Dengan demikian, hingga saat ini, Corel Draw X8.

Secara umum, algoritma pelabelan label terlihat seperti ini:



Diagram menunjukkan bahwa pada tahap mempersiapkan gambar untuk aplikasi, template yang disetujui sebelumnya digunakan dan dimodifikasi, atau gambar dibuat dari awal di bawah persyaratan tertentu. Bergantung pada jumlah dan kerumitan gambar, EzCad2 atau editor vektor dapat digunakan secara langsung selama pembuatan. Dalam kasus saya, Corel Draw X8. Juga situasi yang sering terjadi adalah ketika templat tertentu dibuat dalam vektor, dan teks disesuaikan dengan templat dan sudah diubah di EzCad2.

Keunikannya adalah EzCad2 tidak mengerti banyak format vektor. Lebih khusus, jenis file berikut:



Sampai saat ini, jenis file yang disukai yang diekspor dari editor vektor adalah * .svg.

Namun, dalam tata letak yang rumit, ketika mengimpor corel yang dibentuk svg ke EzCad2, hasilnya adalah kekacauan yang tidak dapat dipahami. Atau berlaku secara kondisional, membutuhkan file fine-tuning yang serius. Di bawah ini saya akan menunjukkan contoh.

Pada titik ini, saya mencoba menggunakan Inkscape sebagai editor yang memanfaatkan sebagian besar kemampuan format svg. Namun, EzCad2 benar-benar tidak ingin bekerja dengan SVG dari Inkscape. Mendengar ini, dia menghentikan tes, dan terus bekerja di Corel. Juga, saya secara eksperimen menemukan bahwa file DXF yang diekspor dari Corel Draw diterima oleh EzCad2 tanpa masalah. Dengan demikian, kebutuhan untuk menggunakan SVG dalam alur kerja diminimalkan.

Semua kemegahan berfungsi cukup baik sampai awal minggu ini, ketika Corel Draw, pada saat startup, dengan gigih meminta pena emas $ 245.

Saya mulai mencari beberapa cara kerja untuk terus bekerja, tanpa kehilangan arsip file dari format CDR v.18.

Sebagai hasil dari sejumlah percobaan, metode berikut telah diidentifikasi. Metode ini tidak optimal, jelas membutuhkan optimasi dan otomatisasi proses. Namun, saya tidak terlalu terlatih dalam otomatisasi kung fu, dan saya harus bekerja. Karena itu, saya akan menundanya untuk nanti.

Solusi


Dalam mencari cara cepat dan bekerja untuk membuka CDR, ia kembali ke Inkscape. Versi terbaru membuka file tata letak dengan benar. Namun, masalah tidak dapat dioperasikannya file Inkscape SVG belum hilang. Juga, file DXF, PLT, PS, AI yang diterima dari Inkscape tidak berfungsi. EzCad2 menjatuhkan semua vektor dalam satu kelompok. Selama pengujian konversi SVG ke PLT, program diminta untuk menginstal Uniconverter. Melakukan operasi ini, tetapi hasilnya menerima kekacauan file dan banyak kesalahan.

Dalam serangkaian percobaan yang gagal, Uniconverter diluncurkan secara terpisah, di luar Inkscape. Atur dia pada file yang diinginkan. Dan tiba-tiba saya mendapat file plt yang berfungsi di EzCad2!

Saya mencoba mengulanginya. Itu tidak bekerja segera.

Poin berikut harus diperhatikan di sini:



Inkscape memungkinkan Anda untuk menyimpan jenis file berikut. Untuk kejelasan, hijau menyoroti format yang seharusnya dirasakan oleh EzCad2. Namun, seperti disebutkan di atas, tidak semua file benar-benar terbuka. Dari file yang dipilih, satu-satunya opsi yang berfungsi adalah Dioptimalkan SVG

Saya akan menggambarkan hal tersebut di atas.

Latihan:


1) Luncurkan Inkscape



2) Jalankan Uniconverter



3) di Inkscape kita membuka file yang diperlukan.



4) Pastikan (!) Salin fragmen yang ditujukan untuk ekspor

5) buat file baru, masukkan isi buffer.



6) dikonversi ke kurva



7) Simpan dokumen sebagai "Dioptimalkan SVG". Kami mengklik tombol dalam dialog simpan.





8) Buka Uniconverter, pada dialog atas pilih file yang diperlukan, di bawah pilih jenis file * .plt

Konversi.





9) buka EzCad2, impor file vektor (Ctrl + B), pilih file PLT yang dihasilkan





Kami sedang mempersiapkan tata letak untuk dicetak.

10) kami bekerja lebih lanjut :-)

Kesimpulan


Saya setuju bahwa "tongkat." Saya setuju bahwa itu membutuhkan gerakan ekstra. Namun, itu berhasil !!!

Sekarang akan seperti mengotomatisasi sedikit. Seperti, saya memilih sebuah fragmen di inscape, itu semua "figax" - dan dihitung. Tapi sejauh ini tidak sampai di situ.

Terima kasih atas perhatian Anda, semoga menarik dan / atau bermanfaat.
