Mengkonfigurasi peta OsmAnd dari lapisan peta panas Strava

Aplikasi OsmAnd untuk ponsel Android memiliki kemampuan untuk menambahkan lapisan tambahan ke peta OSM (Open Street Map). Tambahan yang berguna untuk sistem navigasi Anda akan menjadi lapisan peta panas Strava, hasil dari jejak pelari, pengendara sepeda, perenang yang direkam selama perjalanan dan pelatihan. Visualisasi data ini memungkinkan seseorang untuk menemukan rute yang paling populer disorot oleh jalur yang mengesankan di satu sisi, dan untuk menentukan patensi atau kelaikan suatu situs di tanah dengan benang tipis dari jalur tunggal.

Sejak 2018, laboratorium Strava hanya menyediakan ekor resolusi rendah di domain publik. Ini terjadi, tampaknya, untuk "memastikan keamanan" dalam menanggapi publikasi tentang deklasifikasi massal lokasi pelatihan pengguna.

Untuk mencari rute populer, perincian rendah sudah cukup, tetapi untuk menilai apakah kami melewati bagian medan, kami membutuhkan catatan lintasan tunggal.

Saya akan memberi contoh. Tangkapan layar layar ponsel menunjukkan bahwa keturunan di sepanjang jalan turun dari Gunung Akhun ke Agursky Falls (1) adalah rute berjalan dan bersepeda yang cukup populer (perhatian, hanya untuk pengendara sepeda ekstrim!). Tetapi untuk mengatasi kemiringan 1,5 km dari jalan ke puncak Akhuna ke ngarai Gerbang Setan (2) di sungai Khosta, titik akhir dari jalur kayu Yew-boxwood (3), dengan semua godaan dari jalan seperti itu, sepertinya tidak akan berhasil. Saat peta mendekat, akan terlihat dari jalur tunggal bahwa ada upaya tantangan yang sangat jelas, tetapi semuanya berakhir tidak berhasil.



Hubungkan OsmAnd Strava heatmap dalam resolusi rendah


Mari kita cari tahu cara mengatur lapisan peta panas Strava pada ponsel cerdas Anda. Untuk menghubungkan peta panas di OsmAnd dalam resolusi rendah, buat sumber baru untuk layer (Menu -> Configure Map -> Source Map -> Set / Edit ...)



Panggil layer baru, misalnya, siklus HeatMap (tidak dirinci)
Tetapkan URL untuk mengunduh ubin: 
https://heatmap-external-{switch:a,b,c}.strava.com/tiles/cycling/bluered/{0}/{1}/{2}.png?px=256 tiles/ -     cycling/ -  ,     ride, both, all bluered/ -  ,    hot, grey ?px=256 -    OsmAnd


Anda dapat memilih tampilan layer yang sesuai di OsmAnd dengan mengalihkan pengaturan pada peta Strava.



Untuk hiking, misalnya, siapkan palet berwarna kebiruan, dan untuk sepeda yang panas dan ganti tergantung pada perjalanan Anda. Perbedaan antara "hitmaps" ini minimal, tetapi kadang-kadang perbedaan ini penting. Di mana ada jejak gunung dan Anda bisa pergi, tidak selalu memungkinkan untuk mengemudi.

Mengatur lapisan detail peta panas Strava


Untuk menampilkan ubin detail di OsmAnd Anda, Anda perlu menentukan jalur gambar yang berbeda (/ tiles-auth /) dan menambahkan parameter otorisasi ke tautan, yang akan kami ambil dari cookie browser.

Contoh tautan: 
 https://heatmap-external-{switch:a,b,c}.strava.com/tiles-auth/running/bluered/{0}/{1}/{3}.png?px=256&Key-Pair-Id={ID}&Signature={Sig}&Policy={P} : {0} = zoom {1} = x* {2} = y* {ID} = CloudFront-Key-Pair-Id {Sig} = CloudFront-Signature {P} = CloudFront-Policy

* Nilai zoom, x, dan y secara otomatis ditetapkan saat menggunakan kartu OsmAnd.
Pelajari lebih lanjut tentang koordinat petak peta Google di sini .

Untuk mendapatkan parameter otorisasi, masuk ke akun Strava.com Anda dan salin nilai cookie yang sesuai. Di Chrome, ketikkan bilah alamat: chrome: // setting / cookies / detail? Site = strava.com. Untuk melihat cookie di Firefox, tekan Shift + F9.

Contoh nilai bidang Kebijakan Depan untuk ditambahkan ke URL.



Saat menyalin cookie, perhatikan periode validitas untuk menggantinya tepat waktu.

Selain itu, ada opsi yang menghilangkan kebutuhan untuk mengganti cookie dalam aplikasi setiap 2 minggu sekali. Artikel ini menjelaskan solusi dengan tautan untuk secara otomatis menghasilkan permintaan resmi untuk lapisan terperinci. Otorisasi otomatis pada kartu Strava Heatmap . Omong-omong, saya tidak menemukan hal seperti itu di Internet berbahasa Inggris.

Mungkin seseorang akan menawarkan opsi lain untuk permintaan resmi ke kartu Strava Heatmap, misalnya, berdasarkan OSMAnd itu sendiri. Tulis tentang ini di komentar.

Atur transparansi peta cakupan, menggunakan bilah geser ini, Anda dapat menyesuaikan tampilan peta yang optimal.

Tonton ukuran folder data / net.osmand / file / tiles tempat peta yang di-cache di-cache. Saya telah mengumpulkan 0,5 Gb dalam 2 minggu penggunaan. Saat menghapus folder dengan gambar peta dengan ukuran berbeda, hapus file .metainfo, tetapi simpan direktori (nama layer Anda).



Kesimpulannya


Sesuaikan peta Anda di OsmAnd, nikmati rencana perjalanan yang populer, cari yang sulit, tetapi menarik, rencanakan dengan hati-hati untuk mengatasi tempat-tempat yang belum pernah dikunjungi atau dikunjungi orang sebelumnya. Perjalanan yang menarik dan aman untuk Anda!
