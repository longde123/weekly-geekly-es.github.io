Ivideon Bridge：如何将旧式CCTV系统连接到云



一旦部署了视频监控系统，然后对其进行扩展，用户通常会成为已安装设备的“人质”。 从一家铁和服务提供商转移到另一家是昂贵的。

市场上有许多制造商创建了自己的服务，在大型解决方案动物园中放置了另一只宠物。 很难在一个项目中“结交朋友”使用不同的设备，但是我们想出了方法。

今天，我们将向您介绍Ivideon Bridge项目，该项目无需额外的设备（PC和DVR）就能将您的视频监控系统与Ivideon集成在一起。

Ivideon Bridge是一种设备，可让您最多将16个摄像机（或DVR通道）连接到我们的服务。 其主要目的是使本地系统的用户能够以最低的成本访问现代的基于云的视频监视和视频分析服务，而无需更换已安装的设备。

当然，这样的机会以前就存在过。 根据任务，我们将使用DVR和固件的NVR或安装Ivideon Server的NVR的客户端连接起来。 我们还允许从任何计算机（企业服务器或家用PC）连接到服务。

这些解决方案具有缺点-便利性有限且成本较高。 评估客户需求，权衡现有连接到我们云的方法的所有优点和缺点，我们开发了一种从根本上将低价格和高功能相结合的新设备。 现在，您可以买得起廉价的小型设备，轻松将“视频监控”打包到云中。 在许多情况下，紧凑的设备将用Ivideon Server或注册器代替PC。

除了访问Ivideon服务的明显优势外，该小工具还有7个主要优势 ：

  • 成本低
  • 可与任意组合连接的16个摄像机或DVR通道一起使用；
  • 易于连接-Bridge自动查找网络上的设备；
  • 易于设置-用户首次获得实际上是从您的帐户远程配置的Ivideon服务器
  • 广泛支持的设备：摄像机，DVR以及支持RTSP或ONVIF的任何其他设备或VMS；
  • ONVIF的事件记录； 将RTSP永久记录到云中，以记录业务资费；
  • 通过使用本机协议，海康威视和大华设备具有多种功能-与终端设备上的本地存档一起使用。

Ivideon Bridge不仅是另一台录像机，而且是即插即用的设备，可大大简化远程管理。 现在，首次在Ivideon Server类解决方案中可以通过云进行管理。

何时使用Ivideon Bridge




例如，几年前，您在商店中安装了市场上最便宜的摄像机提供的视频监控系统。 系统仅显示和存储来自交易大厅和售票柜台的视频。 随着时间的流逝，您意识到摄像头没有足够的机会来提高业务效率-竞争对手已经开发了带有员工时间监控，队列检测和智能通知的系统。

安装新系统并拆除旧系统可能对您没有帮助，但是Ivideon Bridge无需更换所有摄像机即可轻松访问视频分析和新功能。 您甚至可以通过连接到Ivideon Bridge的DVR将模拟摄像机添加到云中。

如果我们在基于云的摄像头连接和对EDGE存档进行远程访问的情况下考虑该设备，那么您只需要设置一个较低的资费计划-例如，云中8台摄像头的最低年度成本仅为1600卢布/月或19200卢布/年。

需要设备的5种情况：

  1. 您想扩展现有的视频监控系统并购买新的Ivideon摄像机。 您将获得Ivideon Bridge来选择云中的旧系统，并将其与新系统组合到一个个人帐户中。
  2. 该设施的摄像头无法远程访问。 Ivideon Bridge将帮助您获得对实时流和存档，便捷的客户端应用程序，远程通知和设置的多重访问。
  3. 您有几家商店使用不同的视频监视系统（使用不同的应用程序登录）。 您想将它们组合到一个系统中，以一个个人帐户集中配置和监视设备的性能。
  4. 维护服务计划优化其工作：由于通过云进行的配置，您不需要对设备进行物理或远程网络访问，可以通过个人帐户将摄像机添加到客户端-客户端将设备连接到网络，服务组织的专家可以远程执行所有其他操作。
  5. Ivideon合作伙伴简化了用户支持。 最终客户只需在Ivideon Bridge中物理连接两根电缆就足够了，所有必需的设置将由供应商进行远程设置。

桥接功能和功能




从外部看，该设备是一个紧凑的盒子，带有端口，可轻松放入您的手掌。 桥接器的“心脏”是基于四核64位ARM CPU的单板微型计算机，频率为2 GHz，板载RAM为2 GB。

要连接网桥，需要一个220伏插座和网络设备上具有Internet访问功能的空闲端口。 我们已尽力使设备默认开箱即用，但在极少数情况下，您将需要连接显示器和键盘-例如，如果网络中没有DHCP服务器，则需要配置网络连接。

Ivideon Bridge支持任意组合使用16个摄像机或DVR的通道。 所有连接的摄像机的总比特率不应超过32 Mbit / s：您可以连接16个2 Mbit / s的摄像机或8个比特率为4 Mbit / s的摄像机。 允许其他组合。

Ivideon Bridge软件包括Ivideon Server的专用版本，该版本与Ivideon Backend紧密集成-API功能可通过云进行管理和配置。

所有设备管理功能都在Ivideon云中实现，设备上没有本地图形界面。 在Ivideon服务器类解决方案中，第一次实现了通过云的远程管理-完全通过Ivideon的个人帐户执行将摄像机连接和配置到设备的操作。

桥接器还具有许多辅助系统服务，旨在确保设备和OTA（空中）更新的稳定运行。 设备的软件（包括安全补丁和新功能）使用户可以轻松地从其个人帐户免费更新。

在网络上自动搜索摄像机


连接的摄像机至少应在RTSP协议上运行，或者更好地支持ONVIF，这将提供更多的Ivideon功能。 与Ivideon Server不同，网桥上没有内置的运动检测器，因此，运动记录不适用于RTSP摄像机，只能进行永久性的商业收费记录存档。

锦上添花的是Ivideon Bridge中原生的Dahua和海康威视ISAPI协议的支持：这些制造商的设备的用户将可以直接在摄像机或DVR上访问本地存档，以及OSD和运动检测器设置，包括设置检测区域。

进入设置后，Ivideon Bridge会立即在网络上搜索摄像机和录像机。 要连接检测到的设备，请单击它，输入登录名和密码，并在必要时输入其他设置，请确保正确显示预览中的视频并确认相机的连接。

连接检测到的相机


我们提供了专业配置摄像机连接的功能。 如果计划将无法自动检测到的旧录像机或摄像机连接到云，则需要此模式。 当网桥位于不同的网段中并且基于分组广播的发现协议无法自动找到设备时，可能会出现这种情况。

手动连接模式将在必要时帮助设置视频流的“自定义”设置-例如，将摄像机的低端流指定为主端口，或为RTSP流设置非标准端口。

由于桥接设置的灵活性，用户可以轻松应对摄像机的IP地址，登录名/密码更改或设备更换的情况。 通过更换摄像机，您将不会丢失以前在云中记录的视频存档以及对该服务的已付费订阅。 反过来，我们将迅速扩展对新相机和录像机的支持。

总结：桥接器在连接设备时提供了多种选择，并允许访问来自不同制造商的Ivideon设备。 同时，我们根据连接设备的实际特性“挤压”其最大功能。

根据受支持的协议，在Ivideon中连接的摄像机上将提供一组或另一组功能。 功能最强大的案例是Dahua和Hikvision设备，Bridge通过本机协议与之通信。 连接这些制造商的型号时，用户将可以访问设备本身上的视频存档。
