CSS iOS de la mort


Un code a été publié sur github qui conduit à forcer le redémarrage des appareils iOS (11/12 GM) lors de la visite d'une page html. Mac OS High Sierra / Mojave se bloque également lors de l'utilisation de Safari.


Le code conduisant à forcer le redémarrage est un code html avec un grand nombre de divs imbriqués et une fonction de flou d'arrière-plan "fou": 


... div { backdrop-filter: blur(10px); -webkit-backdrop-filter: blur(10px); width:10000px; height:10000px; } </style> </head> <body> <div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div> ...

Également publié une page html PoC qui implémente ce bug (les utilisateurs de Safari vont à leurs risques et périls).



On suppose que le bogue s'est glissé à un niveau inférieur à celui du webkit et peut entraîner des conséquences plus graves. De plus, l'utilisation de ce bogue peut être applicable dans les entreprises sociotechniques et les blagues pratiques stupides, donc je vous conseille d'être extrêmement méfiant envers tous les liens, d'autant plus que sur iOS tous les navigateurs, en fait, sont un add-on pour Safari.
