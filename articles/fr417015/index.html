Nous comprenons ce qui enregistre et ce qui n'enregistre pas l'application Burger King


Cet article est une réponse à l 'article de fennikami sur la façon dont un mauvais Burger King enregistre des vidéos depuis un écran de smartphone. L'article original prétend que de cette manière Burger King peut accéder aux informations sur les cartes d'entrée, les compromettant ainsi. Je vais essayer de comprendre si c'est vraiment le cas.
À l'aide d'une petite instruction publiée dans les commentaires, vous pouvez recevoir indépendamment la vidéo envoyée par Appsee. Et assurez-vous quelles données sont retouchées et lesquelles ne le sont pas.

Une application Android version 2.2.2 avec une date de mise à jour du 9 juillet 2018 a subi mon analyse. Les résultats sur d'autres plates-formes et dans différentes versions de l'application peuvent différer de ceux donnés dans cet article.

Initialisation Appsee


Appsee est l'un des systèmes d'analyse utilisés pour améliorer l'expérience utilisateur avec l'application cliente. Il enregistre l'écran et vos actions lorsque vous êtes dans l'application. En général, il n'y a rien de mal à cela; la technologie WebVizor de Yandex est largement répandue sur le Web, dans le même but.

Au stade initial de son travail, Appsee demande une configuration pour une application spécifique du serveur, c'est cette demande que l'auteur a prise dans l'article d'origine, dans mon cas, la réponse ressemblait à ceci:

Vous pouvez remarquer que les paramètres HideInput , HideSensitiveViews sont vrais, masquant ainsi des informations importantes dans la vidéo. La question se pose immédiatement de savoir comment l'importance de l'information est déterminée, pour une réponse nous nous tournons vers la documentation Appsee . En bref, par défaut, tous les champs de saisie de données sont masqués; en outre, vous pouvez configurer le masquage pour d'autres champs ou parties de l'interface.

Nous prêtons également attention à la qualité de l'enregistrement vidéo, dans mon cas c'était de la vidéo avec une résolution de 320 par 160 pixels, avec une fréquence de trois images par seconde.

Données réelles collectées par Appsee


J'ai décidé de vérifier quelles données sont réellement envoyées au serveur Appsee, pour cela j'ai utilisé Proxy de Burp Suite. Au début, le système d'analyse transfère le fichier avec les actions de l'utilisateur à l'intérieur de l'application, y compris les coordonnées des clics de l'utilisateur, les frappes ne sont pas enregistrées.

Nous regardons ce qui se passe ensuite


Il semblerait que l'appsee soit sûr et a probablement aidé les développeurs à améliorer plusieurs fois l'application. Mais vérifions ensuite ce qui se passe avec les données cartographiques. Vous devez d'abord lire le commentaire du représentant de Burger King dans les médias sur la situation:
«L'application mobile Burger King avec une commande à distance ne collecte pas les données personnelles de ses abonnés. Toutes les transactions sont protégées de manière fiable par l'un des meilleurs acquéreurs du pays - Yandex.Kassa. Les données sur le comportement des utilisateurs dans l'application ne contiennent pas d'informations sur les cartes bancaires et ne sont pas stockées sur le serveur », a déclaré la société.
Cela semble assez convaincant, et maintenant nous regardons le trafic réel. Pour le paiement, le traitement de Yandex est réellement utilisé, cela peut être déterminé par demande avec l'adresse

https://payment.yandex.net/api/v2/payments/bankCard
