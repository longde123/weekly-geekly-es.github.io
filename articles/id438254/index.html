Eclipse Meluncurkan GlassFish 5.1 untuk Java EE 8

Berita asli



Bergerak maju dengan perusahaan Java, Eclipse Foundation memperkenalkan versinya sendiri dari server aplikasi GlassFish, yang secara tradisional berfungsi sebagai referensi implementasi platform Java EE (Java Enterprise Edition).


Eclipse GlassFish 5.1 kompatibel dengan spesifikasi Java EE 8 dan merupakan migrasi lengkap GlassFish ke Eclipse Foundation open source. Server aplikasi GlassFish mendukung teknologi perusahaan, termasuk JavaServer Faces, Enterprise JavaBeans, dan Java Message Service (JMS).


Dari Oracle ke Eclipse Foundation


Eclipse, yang telah mengambil evolusi perusahaan Java dari Oracle sejak 2017, mengatakan rilis ini adalah langkah menuju kompatibilitas dengan Jakarta EE, yang merupakan penerus yang direncanakan untuk Eclipse untuk Java EE. Versi berikutnya dari Eclipse GlassFish, Eclipse GlassFish 5.2, akan berfungsi sebagai implementasi referensi yang kompatibel dengan Jakarta EE 8.


Menurut yayasan tersebut, migrasi GlassFish ke Eclipse adalah masalah teknis dan hukum yang â€œsangat besarâ€. Kontribusi GlassFish dan Oracle Java EE API ke Jakarta EE selesai. Java EE TCK (kit kompatibilitas uji), yang sebelumnya bersifat rahasia dan eksklusif, sekarang bersifat open source dan di-host di Eclipse. Selain itu, basis kode Eclipse GlassFish telah dilisensikan ulang berdasarkan CDDL-GPL (Pengembangan Umum dan Lisensi Distribusi, GNU General Public License) dan Classpath pada Eclipse Public License 2.0 plus GPL dengan pengecualian Classpath.


Dari Java EE ke Jakarta EE


EE Jakarta adalah merek dan serangkaian spesifikasi, sama seperti Java EE adalah merek dan serangkaian spesifikasi. Server aplikasi Java akan berpindah dari Java EE ke Jakarta EE. Namun, proses spesifikasi EE Jakarta masih dalam pengembangan. Rilis pertama Jakarta EE akan menjadi Jakarta EE 8, setara dengan Java EE 8. Eclipse berharap untuk merilis Jakarta EE 8 pada pertengahan tahun. Setelah itu, berencana untuk mempertimbangkan penambahan fitur seperti modularitas, layanan microser dan model non-blocking reaktif ke Jakarta EE. Modularisasi akan menyinkronkan Java perusahaan dengan Java SE (Edisi Standar). EE Jakarta akan fokus pada penyebaran cloud. Eclipse juga membutuhkan beberapa implementasi referensi EE Jakarta yang kompatibel.
