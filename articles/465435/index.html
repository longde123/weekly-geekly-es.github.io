¿Qué distribución es mejor para su sistema integrado?

Qué distribución es mejor usar para sistemas embebidos. Este tema es relevante hoy.

Existen tres enfoques relevantes para resolver este problema:

  1. Use una distribución preparada para su computadora de placa única (Armbian, Openwrt, etc.)
  2. Compile y configure su distribución utilizando sistemas de compilación (Buildroot / Yocto, etc.).
  3. Use su cargador de arranque de paquete (u-boot) + kernel (kernel) + rootfs (busybox).

Distribuciones listas para usar


Creo que usar una distribución preparada es la forma más fácil y más fácil. Puede descargar la imagen terminada o crear la suya propia. El ensamblaje de su imagen Armbain no siempre es flexible, porque no puede seleccionar ninguna versión del kernel de Linux, sino utilizar solo las que se proporcionan.

La principal ventaja de las distribuciones listas para usar también debe ser respondida: su estabilidad.
Para empezar, recomendaría estas distribuciones.


Armbian es una distribución popular de Linux disponible para una amplia variedad de dispositivos ARM: Orange Pi, Banana Pi, Odroid, etc. Está basada en Ubuntu y / o Debian.
www.armbian.com



OpenWrt es un sistema operativo integrado basado en el kernel de Linux y diseñado principalmente para enrutadores domésticos. Los componentes clave incluyen el kernel de Linux, util-linux, uClibc o musl y BusyBox. El tamaño de todos los componentes está optimizado debido al hecho de que la mayoría de los enrutadores domésticos tienen una cantidad muy limitada de memoria.

Agregaría por mi cuenta, muy adecuado para placas con wi-fi a bordo (por ejemplo, Orange Pi Zero).
openwrt.org

Sistemas de montaje


La principal ventaja de los sistemas de ensamblaje es que puede construir Linux mínimo y flexible para sistemas embebidos.

Buildroot



Buildroot es un sistema de compilación de distribución para sistemas embebidos. Es compatible con un montón de placas y el resultado de su trabajo es el gestor de arranque ensamblado, el núcleo y la imagen.
sistema de archivos

Lo que permite construir buildroot para su sistema:

  • imagen del sistema;
  • Este sistema le permite seleccionar la versión del núcleo, cualquiera.
  • Puede agregar parches e instalar cualquier programa.
  • configuración flexible de utilidades (por ejemplo, busyBox, bash, etc.)

Cabe señalar que todas las fuentes adicionales se extraen de la red.

Distinguir tal asamblea, por ejemplo, de la asamblea armbiana:

  • la distribución no se basa en (Debian o Ubuntu), sino en un Linux flexible y personalizado.
  • Tamaño mínimo de montaje.
  • selección de cualquier versión del kernel.

buildroot.org

3. Usando su paquete:

Para esto necesitamos:

  • Compilador cruzado (por ejemplo, Linaro);
  • Gestor de arranque (por ejemplo, U-boot);
  • Kernel;
  • RootFs (por ejemplo, Busybox).

Entonces vamos.
Por lo tanto, recopilaremos localmente en nuestra máquina, necesitamos un compilador cruzado, por ejemplo, linaro. El compilador cruzado ayudará en la arquitectura x86 para ensamblar nuestra distribución para la plataforma ARM de destino.
www.linaro.org

A continuación, debe crear un gestor de arranque.

Bootloader (U-Boot) : el gestor de arranque más popular para ARM es U-boot. La tarea principal del gestor de arranque es iniciar el kernel de Linux Kernel. También puede usar, por ejemplo, barebox u otro.
www.denx.de/wiki/U-Boot

RootFs es el sistema de archivos raíz que Kernel montará después del arranque. Recomiendo usar Busybox.


busybox.net
