Pemantauan Cuaca di Grafana

Pada artikel ini, kita akan berbicara tentang cara mendapatkan untuk memonitor indikator sistem yang diperlukan pada grafik, tanpa menulis satu baris kode. Perangkat lunak yang digunakan didistribusikan secara bebas kecuali untuk server basis data - ini adalah server MS SQL yang digunakan dalam layanan cloud MS Azure, yang dapat Anda gantikan dengan mudah dengan yang lain.

Stasiun cuaca mengambil indikator suhu dan kelembaban dan mengirimkannya secara real time ke server MS Azure, yang menempatkan data ini dalam database. Dengan demikian, serangkaian waktu tertentu dihasilkan, yang diusulkan untuk dipantau menggunakan Grafana dan ditampilkan di dasbor administrator.



Tugas stasiun cuaca tidak dipertimbangkan dalam artikel ini, kami akan menganalisis data yang diterima, untuk ini kami akan menggunakan FastReport.




Ini adalah bagaimana ini disimpan dalam basis data.



Unduh dari grafana.com dan instal server Grafana, lalu sambungkan ke database dengan data.



Selanjutnya, kami membuat kueri untuk mencerminkan seri waktu suhu: 

SELECT regdate as time, Temperature as value, 'Temperature' as metric FROM weather WHERE $__timeFilter(regdate) ORDER BY regdate ASC

Pada awal artikel dikatakan bahwa tidak perlu memprogram, ini benar-benar terjadi karena permintaan sampel yang diajukan dihasilkan oleh sistem itu sendiri. Di satu sisi, Anda tidak perlu memprogram apa pun, dan di sisi lain, untuk dengan cepat membangun kueri yang kompleks, mudah untuk memodifikasi kueri dan "menjelaskan" ke sistem bahasa apa yang perlu dibuat.
