Descripción general de las herramientas de seguridad del repositorio de GitHub



Introduccion


Cuando comienzas a crear un repositorio en GitHub, una de las primeras cosas en las que debes pensar es en la seguridad.

En caso de que esté creando su propio repositorio de GitHub o que a menudo contribuya al repositorio, debe saber si su código contiene alguna vulnerabilidad. Vulnerabilidades en repositorios en el pasado causaron problemas de seguridad. Esto fue enfatizado por el hecho de que dos de las mayores filtraciones de datos en los últimos tiempos, Equifax y Heartbleed SSL Exploit , comenzaron con vulnerabilidades con los componentes de código abierto correspondientes que podrían usarse en el futuro.

En esta publicación, analizaremos y analizaremos cuatro herramientas separadas que puede usar para identificar vulnerabilidades en su repositorio de GitHub. Cada una de estas cuatro herramientas tiene sus propios superpoderes, pero cada uno tiene sus debilidades. Este artículo lo ayudará a elegir la herramienta adecuada para su proyecto de código abierto.

Este artículo fue traducido con el apoyo de EDISON Software, que brinda consejos prácticos a los jóvenes , además de diseñar software y escribir TK en ruso e inglés .


Barandillas




GuardRails es una aplicación de seguridad gratuita disponible en el mercado de GitHub . GuardRails puede proporcionar análisis de código estático, así como identificar dependencias vulnerables. Escribe comentarios al grupo de solicitudes con vulnerabilidades.

La aplicación en sí escaneará nuevas entradas en el código de usuario en tiempo real, lo que permite a los usuarios tomar medidas rápidamente para eliminar vulnerabilidades casi inmediatamente después de que aparezcan. Esto ayuda a proteger el repositorio y el código de intrusos. Con respecto al grupo de solicitudes, GuardRails escribirá comentarios sobre cada solicitud cuando detecte un problema de seguridad, y con las ramas esta información se mostrará en su panel de GuardRails.

El principio rector del servicio GuardRails es su configuración completa y rápida, en la que los usuarios pueden integrar GuardRails con todos sus repositorios en minutos. También puedes integrar GuardRails con Slack para que las notificaciones te lleguen de manera más eficiente.

GuardRails actualmente es compatible con Python, Ruby, JavaScript, Solidity, Go, Java y PHP.

WhiteSource Bolt




WhiteSource Bolt ayuda a los usuarios de GitHub a crear escaneos de sus repositorios, lo que les permite identificar vulnerabilidades de código abierto que pueden aparecer en el código. Lo proporciona WhiteSource, un especialista en el campo de la seguridad, las licencias y los informes en el campo del código abierto. Han estado operando en el mercado desde 2011 y pueden contar con la ayuda de más de 2.1 millones de desarrolladores diferentes.

Su servicio funciona de tal manera que cada vez que ocurre una acción de inserción, Bolt comienza a escanear su repositorio y luego crea un problema para cada vulnerabilidad detectada. También creará problemas para las nuevas vulnerabilidades que se han descubierto con los componentes de código abierto existentes. Además, puede evitar que los componentes vulnerables ingresen al código al cancelar automáticamente un conjunto de solicitudes que contienen vulnerabilidades.

Bolt también proporciona a sus usuarios acceso a su propia base de datos de vulnerabilidad WhiteSource, que es extensa y que muchos consideran el mercado de seguridad de código abierto más costoso. Recibirá información sobre las vulnerabilidades detectadas, incluidos los datos de CVE y CVSS, las correcciones sugeridas, las rutas a los componentes vulnerables y los enlaces para obtener ayuda.

Actualmente, Bolt admite más de 200 programas diferentes, incluidos Java, Python, PHP, C #, C ++ y otros.

LGTM




LGTM es una aplicación de proyecto de código abierto que ayuda a los usuarios a detectar posibles vulnerabilidades en su código y también evita que ocurran en primer lugar. En particular, LGTM utiliza datos recopilados por un equipo de investigación de seguridad que se enfoca en encontrar vulnerabilidades de día cero. Más de 700,000 desarrolladores y más de 135,000 proyectos de código abierto se han beneficiado de los servicios LGTM, y este nivel de experiencia da fe de la calidad de sus servicios. La aplicación LGTM GitHub está disponible en el mercado de GitHub.

Al trabajar en su repositorio, LGTM puede escanear automáticamente su código, buscando vulnerabilidades y CVE que puedan aparecer. Gracias a la gran comunidad de desarrolladores e investigadores LGTM experimentados, comprende que los servicios que brindan pueden ser de gran beneficio para la seguridad de su repositorio. Esto hace que sea aún más fácil que mantener un registro de consultas y con él, puede detectar posibles vulnerabilidades antes de que entren en la base del código.

LGTM actualmente tiene una gran cantidad de lenguajes de programación compatibles, cuyo soporte se extiende a C, C ++. COBOL, Python, Javascript y Java.

Alerta de seguridad de Github




GitHub Security Alerts es un servicio gratuito proporcionado a propietarios y miembros de repositorios de GitHub con dependencias. Usando su propio gráfico de dependencia, los usuarios podrán ver cuándo hay vulnerabilidades en sus dependencias, y proporcionarán a los usuarios sugerencias para corregir estas vulnerabilidades.

Cuando GitHub le notifique sobre una posible vulnerabilidad, recibirá una actualización en la que GitHub le informará sobre cuáles de sus dependencias deben actualizarse. Si se conoce una versión segura de la dependencia, GitHub elegirá una para usted mediante el aprendizaje automático y se incluirá en la recomendación.
Cuando se trata de información sobre cada vulnerabilidad, GitHub le dice a qué vulnerabilidad afecta, el rango de versiones que afecta, el ID de CVE y cualquier corrección propuesta contenida en la base de datos de vulnerabilidades.

El servicio actualmente admite JavaScript, Ruby y Python.
