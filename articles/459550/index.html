Backup, Parte 5: Prueba de Bacula y Veeam Backup para Linux


Este artículo discutirá varias herramientas de software de copia de seguridad "grandes", incluidas las comerciales. Lista de candidatos: Veeam Agent para Linux, Bacula.


Se verificará el trabajo con el sistema de archivos para que sea conveniente compararlo con candidatos anteriores.


Resultados esperados


Dado que ambos candidatos son soluciones universales listas para usar, el resultado más importante será la previsibilidad del trabajo, es decir, el mismo tiempo de trabajo al procesar la misma cantidad de datos, así como la misma carga.


Descripción general de Veeam Agent para Linux


Este programa de respaldo funciona con dispositivos de bloque, para lo cual tiene un módulo para el kernel de Linux, que garantiza la integridad del respaldo al rastrear los bloques de datos modificados. Una descripción más detallada está aquí .


El proceso de creación de una copia de seguridad de archivos funciona sobre la base del mismo módulo del kernel: se crea una instantánea del dispositivo de bloque, que se monta en un directorio temporal, después de lo cual los datos se sincronizan archivo por archivo desde el nugget a otro directorio local, o se eliminan mediante el protocolo smb o nfs, donde se crean varios archivos en formato propietario


El proceso de creación de una copia de seguridad de archivos no funcionó. Aproximadamente el 15-16% de la velocidad de ejecución se redujo a 600 kb / sy menos, con un 50% de uso de CPU, lo que potencialmente llevó a 6-7 horas del proceso de copia de seguridad, por lo que el proceso se detuvo.


Se creó una aplicación para el soporte técnico de Veeam, cuyo personal propuso usar el modo de bloque como solución.

El tiempo de ejecución del programa en este modo es de 6 minutos por 20 GB de datos.


En general, las impresiones bastante buenas del programa funcionan, pero en la revisión general no se tendrán en cuenta debido al modo muy lento de la operación del archivo.


Revisión Bacula


Bacula: software cliente-servidor para crear copias de seguridad, lógicamente consta de varias partes, cada una de las cuales realiza su propia parte del trabajo. Hay Director, que se utiliza para la administración, FileDaemon es el servicio responsable de las copias de seguridad, StorageDaemon es el servicio de almacenamiento de copias de seguridad, la consola es la interfaz con el Director (hay opciones TUI, GUI, Web). Este complejo también se incluye en la revisión porque, a pesar del umbral de entrada significativamente grande, es un medio bastante popular para organizar copias de seguridad.

Resultados
