Thinkpad T22, Windows 98 dan kecukupan

Laptop ini dalam kondisi sangat baik sehingga saya bahkan tidak membukanya dan membersihkannya. Semuanya berfungsi, bisik kipas, hard drive dengan riang mengklik kepalanya. Bahkan baterainya sedikit hidup. Sebuah keajaiban, bukan laptop, meski usianya 17 tahun. Tidak seperti model sebelumnya yang masih kompatibel dengan DOS , komputer pada awal tahun 2000 sangat mirip dengan yang modern. Ini kesan yang menyesatkan: jika Anda tidak mengharapkan prestasi apa pun dari mesin dengan Windows 95 atau MS-DOS, maka dalam hal ini tampaknya ada beberapa driver lagi, dan itu akan berfungsi sebagaimana mestinya, memuat Facebook, telegram dan dropbox, photoshop, dan Office 365.

Tidak ada yang seperti itu. Perbatasan bersyarat antara besi yang bekerja dan barang antik sekarang melewati suatu tempat di tahun 2005. ThinkPad T22, penerus cita-cita ThinkPad 600, diluncurkan pada tahun 2001. Tidak akan ada pembongkaran dalam posting ini, tetapi akan ada cerita tentang menginstal Windows 98 SE dan tentang upaya untuk terhubung ke setidaknya beberapa jaringan - setidaknya di seluruh dunia, setidaknya lokal.

Saya akan membagikan hack seumur hidup: jika Anda ingin tahu lebih banyak tentang ThinkPad lama-kuno, Anda harus pergi ke sini . Di sana, misalnya, Anda dapat mengetahui bahwa panel atas dan bawah dari model khusus ini terbuat dari titanium - untuk kekuatan dan cahaya.



Mungkin Anda akan menemukan di sana konfigurasi asli model Anda, tetapi hanya jika itu dirilis untuk pasar AS. Laptop saya adalah Eropa.



RAM, dua strip 128 megabyte. Pada titik tertentu, saya memutuskan untuk bereksperimen dengan kompatibilitas dengan permainan DOS dan meresepkan driver himem.sys dalam unduhan. Akibatnya, Windows berhenti melihat memori melebihi 65 megabita (yang merupakan batas untuk DOS). Jika memungkinkan, jangan lakukan itu. Di sebelah kiri Anda dapat melihat baterai pertanian kolektif untuk BIOS - benar-benar semua laptop yang lebih dari 15 tahun yang telah menganggur untuk waktu yang lama membutuhkan perbaikan ini.

Banyak konektor! Satu-satunya hal yang hilang adalah antarmuka FireWire, yang menghubungkan Apple iPod yang dirilis pada tahun yang sama. Benar-benar semua pengamat di semua majalah di semua ulasan Thinkpad menulis ungkapan â€œApakah kekurangan FireWireâ€ sejak saat standar muncul sampai USB akhirnya menang.

Menginstal Windows 98 SE


T22 datang dengan Windows 98 atau dengan Windows 2000 yang baru. Dengan Win2K saya akan bekerja pada laptop lain, tapi di sini saya akan menempatkan yang ke 98. Saya punya stiker dengan nomor seri.



Tidak seperti Thinkpad T43, T22 tidak dapat boot dari perangkat USB, sehingga emulator drive optik saya tidak dapat digunakan. Cara tercepat adalah mem-boot ke DOS / Linux, memformat hard drive, mentransfer file sistem dasar dari disk boot Win98 dan menulis ulang paket distribusi. Kemudian Anda dapat menginstal sistem dengan menjalankan setup.exe dari DOS. Untuk beberapa alasan, Scandisk dari Win98 bersumpah jika saya memformat hard drive menggunakan Linux. Memformat secara asli, semakin banyak orang mengingat perintah .



Seiring dengan distribusinya, ada baiknya menulis ulang driver yang bisa Anda dapatkan di sini . Ada beberapa nuansa yang bagus. Misalnya, ada dua driver untuk sistem suara yang dapat dipilih. Yang lebih tua menyediakan kompatibilitas saat boot ke DOS bersih. Yang muda tidak. Driver jaringan tidak memiliki installer dan menyarankan memperbarui driver menggunakan Device Manager. Dari perangkat lunak Anda harus menginstal Integrasi Fitur Hotkey, On-Screen Display (tombol khusus akan mulai muncul dan indikasi apa yang mereka lakukan akan muncul di layar), Battery Maximizer Utility (status baterai dan widget untuk kontrol pembuangan), opsional ThinkPad Configuration Utility, walaupun BIOS di sini tanpa burung dan menunjukkan semua pengaturan dan sebagainya.



Baterai menunjukkan sekitar setengah kapasitas dan bertahan sedikit lebih dari satu jam. Cukup memindahkan laptop dari stopkontak ke stopkontak.

Membuat Windows 98 melakukan sesuatu yang bermanfaat


Ok, kami punya sistem kerja di mana semuanya dan tidak ada yang bekerja secara bersamaan. Peramban - Internet Explorer 5 - hanya membuka situs lama (tapi dengan cepat), tidak bersahabat dengan https, dan sering macet bahkan di http minimal kompleks. Tidak ada gunanya meletakkan browser baru di mesin dengan 256 megabyte RAM. Untuk bersenang-senang, saya menginstal versi baru Opera (10xx) dan setelah sekitar tiga hingga empat menit saya dapat mengunduh situs berita yang relatif sederhana. Mesin ini bukan untuk web modern.



Oke, mungkin kemudian kita akan pergi ke berbagi file rumah untuk bertukar file? Tidak, Windows 98 tidak tahu bahwa akses tidak hanya memerlukan kata sandi, tetapi juga nama pengguna. Saya tidak bisa mengatakan bahwa saya mencari jalan ini sampai akhir, jika Anda tahu tentang penyelesaian masalah - beri tahu saya. Jelas bahwa Anda dapat berbagi file tanpa kata sandi, berbagi folder di Windows 98 sendiri dan mengaksesnya dari komputer lain, tetapi saya lebih suka membangun server terpisah untuk Windows 98 yang akan sepenuhnya memadai untuk era ini.


Pada saat yang sama, inilah kesalahan antarmuka nostalgia untuk Anda.

Ya, kami memiliki USB, kami akan mengarahkan file pada flash drive, meskipun lambat (tidak lebih dari 1 megabyte per detik). Ya, tapi tidak segera. Windows 98 adalah OS pertama Microsoft yang mendukung USB. Dalam praktiknya, ini berarti mampu mengenali perangkat USB. Dan kemudian, Nyonya, Anda membutuhkan sopir. Tidak ada driver universal dari pabrikan (IBM), hanya untuk perangkat tertentu. Untungnya, ada paket driver pihak ketiga yang menyediakan dukungan untuk flash drive, drive optik, keyboard USB, mouse, dan periferal standar lainnya. Anda dapat mengambilnya, misalnya, di sini (perhatikan bahwa untuk Windows 98 rilis asli dan versi SE drivernya berbeda).



Semua ini bekerja dengan sangat baik, disesuaikan dengan karakter absurd ke-98: pengecualian fatal, layar biru dan bencana lainnya tidak sering terjadi, tetapi ternyata terjadi. Di laptop lain, urutan pemasangan driver yang salah menyebabkan layar biru yang tidak dapat disembuhkan, lebih mudah dihancurkan daripada dipahami. Tidak ada perlindungan Windows yang biasa dari pengguna. Segalanya mungkin, tetapi Anda sendiri yang bertanggung jawab.

Biarkan saya mengingatkan Anda bahwa Windows 98 hanya mendukung drive yang diformat FAT32. Dukungan NTFS dapat diberikan dengan perangkat lunak pihak ketiga dari Paragon (gratis!). Dia membaca flash drive 128 GB tanpa masalah, tetapi dia tidak dapat mengatasi hard drive (ukurannya mungkin telah terpengaruh). Saya lebih suka menggunakan FAT32, jadi lebih cepat dan lebih dapat diandalkan.

Lebih jauh Dari program modern yang saya butuhkan, saya hanya dapat menginstal Total Commander. Untuk memutar video dan DVD, Anda dapat menggunakan VLC player, untuk melihat foto - IrfanView, tetapi Anda harus mencari versi lama sekitar lima tahun yang lalu.



Saya akan kembali ke pernyataan saya di awal posting. Laptop 2001 dengan Windows 98 adalah troli roti. Ini sangat mirip dengan laptop modern dalam tampilan, antarmuka Windows 98, secara umum, juga tidak jauh berbeda dari kenyataan, terutama dengan penolakan besar-besaran busur gaya XP mengkilap. Sangat mirip, tetapi tidak berfungsi. Saya bisa mengakhiri posting dengan bagian khas tentang topik lama seperti â€œdan sekarang saya kadang menggunakan laptop X untuk game favorit sayaâ€. Dan ya, C&C untuk Windows (dan DOS), MDK, Duke Nukem 3D, dan Quake semuanya berfungsi dengan baik. Tapi troli ini bagi saya adalah tahap yang sudah lewat, dan saya ingin membangun yang lain.

Tentang kecukupan


Jika Thinkpad T43, dengan beberapa batasan, dapat ditarik ke lingkungan digital modern, maka T22 sama sekali tidak cocok untuk ini. Alih-alih menikmati sepotong besi tua yang bekerja, frustrasi dan frustrasi diperoleh. T22 ingin ditempatkan di lingkungan di mana ia merasa baik. Untuk mesin DOS, mudah, cukup jalankan game dan program yang tepat. Windows 98 termasuk dalam era Internet awal, dan lingkungan ini harus diciptakan kembali secara khusus untuk itu. Dan tugas yang sangat terhormat!

Anda dapat mulai dengan mengatur lingkungan jaringan lokal yang sesuai, yang saya rencanakan untuk menginstal Windows 2000 pada laptop lain (saya memilih T40, lebih kuat, server setelah semua). Dan kita akan lihat nanti. Proxy yang menerjemahkan situs modern menjadi gambar dan bingkai 256-warna? Antarmuka arsip web dengan akses realistis ke situs versi kuno? Atau buat salinan lokal? Emulator jaringan telepon dengan akses modem? BBS dengan FIDO gash? Beberapa laptop lama sedang menunggu pemulihan dan penyesuaian yang biasa, tetapi kemudian saya bermaksud untuk membangun internet sendiri untuk mereka, dengan preferensi dan penyair. Ini memungkinkan untuk memberikan hobi saya sedikit lebih berarti. Dan ini menarik.



Tapi pertama-tama saya akan mencoba menyiapkan Wi-Fi di Windows sembilan puluh delapan.
