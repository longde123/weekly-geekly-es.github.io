286和网络



我是老人 我有一个满满的旧铁柜子。 从DIP封装中的布尔逻辑芯片到Voodoo5。 当然，所有这些都不代表任何实用价值，但是使某些人将旧铁片弄得一团糟，这是令人高兴的。 如果您是其中之一，我邀请您加入Cat，在这里，我将告诉您基于AMD 286处理器的计算机如何与现代网络“融为一体”，以及它的来龙去脉。

将网络固定到286m的想法对我来说很长一段时间，因为通过网络将数据传输到计算机最容易，这确实很有趣。 TCP / IP是否有效？ 网络浏览器可以工作吗？ 尽管运行286m dos的人已经在dos下启动了浏览器，但我从未见过Internet上Windows下的286th。 好吧，主要目标是连接到家用NAS，该NAS存储所有用于旧铁件的软件。 NAS或多或少是现代的，只有TCP / IP才能够。

我已经有将旧计算机连接到该驱动器的经验。 包括在撰写先前有关旧铁的文章时。 但是在以前的文章中，我只考虑了32位处理器，并且那里的一切都非常简单：即使Windows 95也在386SX-16上运行（是的，非常慢，是的，安装过程要花费9个小时以上，但是可以工作），它具有本机TCP堆栈/ IP和一堆用于各种网卡的驱动程序。 如果您想更快一点，那就是Windows for Workgroups 3.11，在Windows上安装Win32后，您可以滚动自己的TCP / IP堆栈。 而且，尽管它无法登录到现代文件服务器，但是即使使用此配置，PC也可以轻松访问没有密码的打开的文件夹。 Internet Explorer 5甚至可以在386SX上运行，并且有足够的内存！


我从Google偷了一张照片

但是可惜的是，对于286，这都不适用。 主要困难在于286处理器是16位的，而Windows for Workgroups 3.11则不可用，因为允许其前代3.1在16位处理器上运行的标准模式已从3.11中删除。 Win32s扩展也不可用（由于明显的原因），因此，将无法安装本机Microsoft TCP / IP堆栈。

这样，我们回到DOS。

经过简短周到的google搜索后，带有TCP / IP for DOS的图片变得比以前清晰得多。 基本上，所有这些都归结为以下事实：在DOS下正常工作的应用程序本身具有自己的堆栈，而它所需要的只是一个网卡的数据包驱动程序。 但是我找不到允许您挂载远程驱动器的应用程序。
但是，我遇到了来自Microsoft本身的两个不同的DOS网络客户端（MS客户端和MS LAN管理器），以及另一个非常奇怪的野兽，称为Windows for Workgroups 3.1。 是的，它是3.1，而不是3.11，这意味着也许所有内容都可以在标准模式下运行。
这将是非常有趣的。 当然，我没想到在3.1中会看到TCP / IP堆栈，但是在Google中，我遇到了Microsoft对16位TCP / IP堆栈的描述。 有趣
在eBay上购买了分发工具包3.1之后，我开始安装。


我从Google偷了一张照片

事实证明，该系统经过了3.1的稍微修改，带有用于DOS的内置网络接口，与精简的MS LAN Manager非常相似。 与传统3.1的所有区别都减少到文件管理器和相应对话框中都存在“连接网络驱动器”按钮。 无法从Windows本身设置网络。 不清楚如何从DOS配置。 尽管在安装系统后，她将我的网卡标识为NE1000或兼容网卡，但事实并非如此，因为我使用的Genius GR1222卡实际上是相同的。 但是我有一张用于此卡的本机驱动程序磁盘，包括在DOS下。 但是，如何配置用于中断的驱动程序和卡的I / O地址仍然难以理解。

而且，在阅读Google之后，事实证明MS LAN Manager的驱动程序和数据包驱动程序是两回事。

DOS下的驱动程序情况开始有所改善。

  1. 对外直接投资 Novell使用此驱动程序模型与其Netware服务器进行通信。 我没有生物，所以不会。
  2. NDIS 这是用于其LAN Manager和Windows的Microsoft模型。 从理论上讲，您需要什么。
  3. 许多DOS应用程序使用的批处理驱动程序。
