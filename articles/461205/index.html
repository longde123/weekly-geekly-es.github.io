Mejores sistemas de gestión de pruebas 2019

Cada proyecto es único y cada equipo tiene sus propias solicitudes. Pero todos estamos unidos por el deseo de trabajar con herramientas de calidad que ahorren tiempo.

Analizamos los nuevos y probados sistemas de gestión de pruebas que ahora son populares en el mercado. Elegimos las funciones que deberían estar en el sistema de gestión de pruebas ideal, comparamos las capacidades de los productos y estudiamos las opiniones de los usuarios.
Como resultado, aquí hay una lista de herramientas, una de las cuales definitivamente se adaptará a su equipo.

No hay calificación, cada herramienta tiene sus ventajas y desventajas. La mayoría de las herramientas de administración de pruebas se pagan, pero casi todas tienen una prueba gratuita.



¿Qué queremos de un sistema de gestión de pruebas?


El usuario del sistema de gestión de pruebas espera ver lo siguiente:

  • Conveniente instalación y soporte.
  • Creación y gestión de proyectos.
  • Crear usuarios y roles de usuario.
  • Integración conveniente con pruebas automáticas.
  • Crea un plan de prueba.
  • Creación de un caso de prueba.
  • Ejecutar caso de prueba.
  • Sistema de informes claro.
  • Sistema de seguimiento de errores incorporado.
  • Capacidad para integrarse con otras herramientas.

¿Por qué es necesario?
Hay varias formas de resolver el problema de crear un sistema único para trabajar con toda la documentación del proyecto:

  • La forma más económica es no molestarse y elegir Google Docs para la matriz de trazabilidad, y mantener los defectos en el rastreador de errores de código abierto.
  • Otra forma es usar uno de los populares TMS'ok, integrado con el rastreador de errores de la compañía.
  • El siguiente nivel es elegir el Sistema de gestión de pruebas, en función de los detalles de los proyectos, el volumen de tareas, los tipos de documentación y los tipos de pruebas utilizados.

Es muy importante abordar el problema de elegir TMS de manera responsable, porque para una empresa, el precio de un error puede ser alto.

1. TestLink
Una de las pocas herramientas de gestión de pruebas de código abierto disponibles en el mercado. Esta es una herramienta web con funciones típicas, como gestión de requisitos, creación y mantenimiento de conjuntos de pruebas, ejecuciones de pruebas, seguimiento de errores, integración con sistemas de seguimiento de errores conocidos.

Los informes y gráficos están disponibles para rastrear el progreso del proyecto, y las características adicionales incluyen el etiquetado de palabras clave, la especificación de requisitos y un registro de eventos.
El código del proyecto a menudo se actualiza y complementa.

Capacidades:

  • Gestión de requisitos
  • Especificación: definición de casos de prueba mediante agrupación en diferentes conjuntos de pruebas
  • Asignación para ejecutar conjuntos de pruebas de ensamblaje
  • Usuario centralizado y gestión de roles
  • Personalización de campos personalizados.

2. Probarlo

Test IT: TMS creado por probadores para probadores. Esta herramienta presenta una interfaz reflexiva e intuitiva. Dentro del sistema, puede crear proyectos y mantener una biblioteca estructurada de casos de prueba y listas de verificación para cada uno, a menudo las operaciones repetidas se resaltan en pasos generales. La herramienta es flexible: en cada proyecto se crean atributos de usuario adicionales, se distribuyen roles y derechos, lo que simplifica la configuración de TMS para los procesos de su empresa. Test IT ayuda a los líderes de equipos de prueba a distribuir la carga de trabajo entre los probadores de manera uniforme y controlar la ejecución del trabajo mediante consultas e informes personalizados.

Los desarrolladores de aplicaciones prestan gran atención a las pruebas automatizadas; cada caso de prueba en la biblioteca de pruebas se puede integrar con las pruebas automáticas por API. La integración configurada correctamente con las pruebas automáticas le permite controlar las ejecuciones y sus resultados directamente desde TMS en tiempo real. Podrá ver qué pruebas automatizadas están en progreso, analizar sus resultados y ver el código fuente directamente desde Test IT.

La aplicación se está desarrollando activamente, en un futuro próximo se anuncian muchas características útiles.

Capacidades:

  • Gestionar planes de prueba, casos de prueba y listas de verificación
  • Pasos comunes para acciones repetitivas
  • Distribución automática de pruebas para ingenieros de control de calidad.
  • Integre pruebas automatizadas con la API
  • Análisis para pruebas automáticas y manuales.
  • Modelo a seguir y personalización
  • Gamificación
  • Integración bidireccional con JIRA
  • Importar desde otros sistemas de gestión de pruebas

Prueba gratuita: versión de demostración abierta en el sitio

3. Zephyr

Zephyr es un complemento para el conocido JIRA que integra las pruebas en el ciclo del proyecto, lo que le permite monitorear la calidad del software y tomar decisiones de ir / no ir. Los casos de prueba se pueden crear, ejecutar y rastrear como cualquier otra tarea en JIRA. Para una fijación más óptima del proceso de prueba, hay integración con herramientas para la gestión de calidad, automatización, integración continua y análisis.
Además, el producto tiene un soporte técnico rápido.

Capacidades:

  • Enlace a historias de usuarios, tareas, requisitos, defectos
  • Configuraciones de implementación: en la nube, en el servidor, en el centro de datos
  • Información ampliada sobre análisis y paneles de DevOps
  • Integración con JIRA, Confluence, Selenium, Jenkins y Bamboo.

Prueba gratuita: 30 días

4. qTest

La herramienta es útil no solo para los evaluadores, sino también para todo el equipo. La interfaz qTest es nativamente clara, los manuales son fáciles de aprender. Esto le permite crear, organizar y administrar de manera rápida y eficiente los casos de prueba.

Los desarrolladores afirman sin moderación que su herramienta de administración de pruebas No. 1
Según el análisis de mercado, qTest es una de las soluciones de gestión de pruebas de más rápido crecimiento entre los equipos de desarrollo ágil.

Capacidades:

  • Prueba de planificación
  • Crear y gestionar requisitos.
  • Interfaz intuitiva de arrastrar y soltar
  • Matriz de trazabilidad integrada
  • Informes visuales con gráficos detallados.
  • Integración con herramientas de seguimiento de errores de terceros.
  • Control detallado de acceso de usuario
  • Herramienta de integración en la nube JIRA

Prueba gratuita: 30 días

5. PractiTest

PractiTest es una herramienta integral de gestión de pruebas. Proporciona visibilidad completa del proceso de prueba y una comprensión más profunda de los resultados de la prueba. Esta herramienta ayudará a organizar suites de pruebas de acuerdo con sus ciclos y sprints. Los conjuntos de prueba se pueden formar de acuerdo con varios criterios, como componentes, versiones o tipos. La herramienta se centra en pruebas ágiles, pruebas de regresión, pruebas de microservicios y DevOps.

Y en ese trabajo de apoyo, empleados capacitados de control de calidad que pueden entender rápidamente su problema.

Capacidades:

  • Agregue fácilmente nuevas pruebas de características a las pruebas de regresión
  • Agrupación de pruebas basada en microservicios que cubren, incluso servicios cruzados
  • Visualización diferente de información para diferentes grupos de usuarios.
  • Los paneles en tiempo real muestran el estado de las pruebas, se ejecutan en las etapas de desarrollo y durante la implementación en producción
  • Integración con JIRA, Redmine, Jenkins, GitLab y Slack

Prueba gratuita: 14 días

Capacidades:

  • Conjunto básico de creación, edición de un plan de prueba y suites de prueba

La herramienta le permite tener una idea del progreso de las pruebas en tiempo real.

Es complicado en el desarrollo inicial, pero indispensable para una empresa a gran escala, donde se presta especial atención a los detalles de producción.
Precisamente porque el producto ya ha sido instalado, hay muchos manuales y guías de video para configurar y usar en Internet.

Capacidades:

12. Testuff

Testuff es una potente plataforma de administración de pruebas basada en la web que facilita el diseño, la ejecución y la administración de un número ilimitado de pruebas. La herramienta se puede personalizar para cualquier formato de prueba: desde la popular metodología ágil y TDD, hasta White-Box o Black-Box; De arriba hacia abajo o de abajo hacia arriba.

También hay una cola de gestión de tareas optimizada para cada probador que utiliza la gestión del tiempo. Y la organización de pruebas en proyectos, ramas de código y conjuntos de pruebas típicos. Además del bono de exportación en HTML / Excel, hay pequeños bollos en forma de herramienta integrada para crear capturas de pantalla y videos con un puntero y pulsaciones de teclas.

Capacidades:

  • Dos clientes disponibles: web y escritorio

  • Conexión de resultados de pruebas automatizadas a través de API
