Fasilitas lainnya untuk pengikut JSON-RPC

Hai

Suatu ketika, pada satu proyek di mana saya memiliki kesempatan untuk bekerja hanya di bagian depan, saya tidak harus berurusan dengan REST di mana-mana, dan tidak dengan GraphS Hyperscript, tetapi dengan RPC lama, yaitu, versi JSON-RPC 2.0.

Dan Anda tahu, sejak saat itu (ketika saya mendukung) saya hampir selalu menggunakannya.

Kami tidak akan melakukan holivarit, tetapi jika Anda juga seorang pendukung penggunaan JSON-RPC, maka Anda tahu bahwa dalam hal HTTP, menonton permintaan, misalnya di Chrome DevTools / Network, sangat tidak nyaman. Kami hampir selalu memiliki satu titik akhir, statusnya selalu 200 (well, jika server tidak gagal sama sekali). Kesimpulan - Anda membutuhkan ekstensi ke devtools. Jadi, sketsa kecil (pull-quests are welcome): github.com/kityan/json-rpc-viewer
Berguna dan bagus akan menghentikan sementara dan memperbarui versi di toko, tetapi secara umum, garpu dan sekrup preferensi dan pelacur sesuka hati.

Direncanakan:

  • dukungan permintaan batch
  • penyaringan ekspresi reguler dengan nama metode
  • konfigurasi kode kesalahan Anda untuk lebih menyesuaikan visualisasi Anda
  • mungkin menampilkan permintaan dalam status tertunda

Ekstensi itu sendiri dapat diinstal baik dibongkar dari folder dist lobak dan dari Toko Web Chrome .
