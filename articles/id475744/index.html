Model Administrator Sistem Empat Tingkat

Pendahuluan


SDM perusahaan manufaktur diminta menulis, apa yang harus dilakukan administrator sistem? Untuk organisasi dengan staf TI hanya pada staf, ini adalah pertanyaan jebakan. Saya mencoba dengan kata-kata sederhana untuk menggambarkan tingkat fungsional seorang spesialis. Saya harap ini membantu seseorang dalam berkomunikasi dengan orang-orang non-IT Muggle. Jika saya melewatkan sesuatu, maka teman lama saya akan memperbaiki saya.



Level 1: Teknisi


Tugas Di sini masalah ekonomi diselesaikan. Untuk mengerjakan apa yang bisa Anda sentuh dengan tangan Anda. Pada tingkat ini: audit, inventaris, sistem akuntansi, bor, obeng. Hapus kabel dari bawah tabel. Ganti kipas atau catu daya. Temukan kontrak TI, kartu garansi, dan masukkan ke dalam folder. Tuliskan telepon 1C-julukan, master peralatan kantor, penyedia. Kenali wanita pembersih. Wanita pembersih adalah teman dan penolong Anda.

Inilah yayasan. Anda tidak akan dapat bertindak pada level berikut jika Anda terganggu oleh panggilan untuk cetakan yang samar atau baterai mati. Kartrid cadangan harus berada di meja samping tempat tidur untuk MFP, baterai cadangan untuk tikus - pada manajer kantor. Dan Anda harus mengurus ini.

Pada level ini, Anda hampir tidak bekerja di depan komputer. Yang penting bagi Anda bukanlah versi build dari sistem operasi, tetapi apakah perusahaan memiliki penyedot debu biasa.

Interaksi Di tingkat ini, selain mitra TI, Anda berkomunikasi dengan manajer, insinyur bangunan, pembersih, tukang listrik. Berkomunikasi dengan penuh hormat. Anda adalah rekan kerja dengan mereka. Anda memiliki banyak tugas umum. Anda harus saling membantu.

Kualitas . Lengan lurus, akurasi, cinta ketertiban.

Level 2: Enikey


Tugas Bekerja dengan program pengguna. 80% dukungan teknis jatuh pada Enikey.

Kami duduk di depan komputer. Anda tahu, setidaknya, tiga cara untuk menyelesaikan sebagian besar tugas yang dihadapi pengguna. Ini menyebabkan kesenangan tertentu. Tapi ingat, mereka menghasilkan uang untuk perusahaan. Dan Anda baru tahu cara menginstal ulang Windows dengan cepat dan Anda tahu bahwa beberapa jenis driver cetak tidak boleh digunakan. Bahkan, Anda hanyalah pengguna yang sangat maju. Anda bisa memecahkan masalah dengan tabel di Excel dan dokumen di Word. Instal dan konfigurasikan program apa pun.

Pada level ini, Anda bekerja di komputer. Untuk sebagian besar orang asing. Penting bagi Anda untuk mengetahui perangkat lunak spesifik yang digunakan perusahaan. Pembukuan ada di mana-mana, jadi fitur pengaturan 1C di sisi klien dalam konfigurasi apa pun adalah roti Anda. Tetapi masih ada desainer, pengacara, departemen produksi. Dan mereka juga memiliki program dengan karakteristik mereka sendiri. Masih ada programmer. Berita baiknya: mereka akan mengatur semuanya sendiri.

Interaksi Anda menebaknya. Dengan pengguna. Tapi tidak hanya itu. Layanan online semakin merambah aplikasi konvensional. Di situs mereka membuat aplikasi, mengontrol pengiriman, menulis pass, bekerja dengan kontrak negara. Layanan ini tidak ditulis oleh Anda. Tapi mereka akan bertanya padamu. Mengapa faktur dalam Excel tidak dicetak dari situs ini? Kemarin berhasil. Anda akan membutuhkan telepon dukungan teknis dan sepasang rebana perdukunan.

Kualitas . Paparan, kemampuan untuk dengan cepat memecahkan masalah, ketekunan.

Level 3: Sysadmin


Tugas Layanan, server, jaringan, cadangan, dokumentasi.

Tanyakan enikey: apakah server berfungsi? Dia akan menjawab: Anda perlu mengambil kunci ke ruang server dan memeriksa apakah kotak hitam dengan lampu hijau berdengung.

Dan administrator sistem tidak dapat menjawab pertanyaan ini.
Dia harus terlebih dahulu memahami apa yang dimaksud. Mungkin, kita berbicara tentang server 1C: Enterprise. Tapi bukan fakta. Mungkin tentang database Microsoft SQL Server di mana 1C ini menyimpan data? Atau tentang sistem operasi virtual Windows Server 2019 yang menjalankan SQL Server ini? Windows Server 2019, pada gilirannya (jangan khawatir, ini akan segera berakhir), berjalan di VMware ESX Server, yang menjalankan selusin server virtual lainnya. Dan sekarang VMware ESX bekerja pada server hitam dengan bola lampu yang indah.

Pada tingkat ini, Anda memiliki komputer yang layak dengan dua monitor, di satu, artikel "Cara mengatur XXX di YYY " terbuka di sisi lain - konsol server jarak jauh dengan YYY , tempat Anda mencoba membuat XXX . Dan Anda selesai dengan baik, dan semuanya baik-baik saja dengan Anda jika server jauh ini berada dalam lingkungan pengujian.

Pemrograman skrip, cadangan, sistem pemantauan, basis data, virtualisasi server - ini adalah tugas Administrator Sistem. Pengguna lelah, mereka mengalihkan perhatiannya dari dunia indah perintah konsol, penyimpanan file dan server cloud. Dia juga tidak suka berkomunikasi dengan atasan, karena sulit bagi mereka untuk menjelaskan apa sebenarnya yang dia lakukan di sini dan mengapa membeli server lain seharga 300 ribu.

Semua karena administrator sistem terlibat dalam layanan infrastruktur.
Tanyakan kepada Google bahwa itu ... tidak akan menjadi lebih jelas. Padahal, semuanya sederhana.
Ini adalah sistem yang tidak diperlukan sendiri. Tetapi hanya untuk pengoperasian sistem lain.

Ini laptopnya. Anda harus bekerja. Untuk mengkonfigurasi banyak laptop, Anda memerlukan layanan direktori Direktori Aktif. AD - layanan infrastruktur. Dapatkah saya melakukannya tanpa Direktori Aktif? Kamu bisa. Tapi itu lebih nyaman baginya. Di mana dibutuhkan lima admin, satu sekarang mengelola.

Interaksi Administrator sistem tetap harus berkomunikasi. Dan banyak. Dengan sysadmin lainnya. Dengan admin klien Anda akan memutuskan mengapa email tidak masuk di antara server email perusahaan Anda. Dengan penyedia telepon IP, mengapa ekstensi tidak berfungsi. Dengan Diadoc, mengapa tanda tangan elektronik dokumen tidak berfungsi. Dengan franchisee 1C, Anda akan mengetahui batas-batas area tanggung jawab. Dan pengembang perlu menyediakan lingkungan virtual untuk server web dan akses ke database.

Kualitas . Kemampuan untuk memecah tugas yang sulit menjadi beberapa yang sederhana, ketekunan, perhatian. Kemampuan untuk memprioritaskan.

3.1 sublevel: Networker


Administrator Jaringan. Ini adalah spesialis dalam jaringan komputer. Ini dunia yang besar. Baik penyedia, maupun operator telekomunikasi, atau bank dapat melakukannya tanpa Network Engineer. Di perusahaan besar dengan jaringan cabang, Networker juga memiliki pekerjaan yang cukup. Dasar-dasar profesi ini harus dimiliki oleh administrator sistem apa pun.

3.2 sublevel: Pengembang


Ini adalah programmer. Kasta sendiri, bahkan beberapa. Beberapa menulis toko online, yang lain - memproses dalam 1C. Pekerjaannya menarik, tetapi jika perusahaan non-teknologi biasa memiliki posisi sebagai programmer, mungkin ada yang salah dengan proses bisnis. Sysadmin menulis kode untuk mengotomatisasi tugas mereka, tetapi tetap saja, Sysadmin dan Pengembang profesi berbeda.

Level 4: Manajer


Tugas Kepemimpinan TI. Manajemen risiko. Mengelola harapan bisnis. Perhitungan efisiensi ekonomi.

Anda sedang mempertimbangkan strategi pengembangan TI. Anda mengendalikan proses ini. Anda menyampaikan visi Anda kepada kepemimpinan. Atur proyek sebagai bagian dari strategi ini. Anda mengalokasikan sumber daya waktu tim Anda dan anggaran departemen.

Anda tidak peduli bagaimana Linux bekerja, tetapi Anda sangat tertarik pada apakah menguntungkan untuk beralih dari Windows ke Linux, dengan mempertimbangkan biaya operasi akun.

Anda tidak akan mengerti mengapa situs tidak berfungsi, tetapi Anda tahu berapa banyak perusahaan menghabiskan waktu henti per jam untuk layanan ini.

Dan jika administrator sistem membuat semuanya berfungsi dan menjadi stabil, maka Anda, sebagai manajer, sebaliknya, mengganggunya. Karena Anda membuat perubahan. Dan perubahan adalah risiko downtime untuk bisnis, pekerjaan tambahan untuk administrator sistem dan perlambatan dalam fase pelatihan bagi pengguna.

Interaksi Manajemen, manajemen puncak, perusahaan outsourcing. Anda berada dalam bisnis. Penting untuk memahami proses bisnis, dan bagaimana TI dapat memengaruhi bisnis secara keseluruhan.

Kualitas . Kemampuan untuk berkomunikasi dengan manajemen, bernegosiasi dengan manajer lain, menetapkan tujuan dan mencapai pemenuhannya. Pendekatan sistematis.

Kesimpulan


Pengguna mengharapkan Anda untuk mengganti kartrid untuk mereka. Manajemen ingin melihat beberapa inisiatif strategis di pihak Anda. Keduanya benar dengan caranya sendiri. Menemukan keseimbangan antara persyaratan ini dan membangun hubungan dalam tim adalah tugas yang menarik. Dan bagaimana Anda memutuskannya?
