Einführung in Windows Terminal

Wir freuen uns sehr, Windows Terminal ankündigen zu können! Windows Terminal ist eine neue, moderne, schnelle, effiziente, leistungsstarke und produktive Terminalanwendung für Benutzer von Befehlszeilentools und Shells wie Eingabeaufforderung, PowerShell und WSL.



Windows Terminal wird über den Microsoft Store in Windows 10 bereitgestellt und regelmäßig aktualisiert, um sicherzustellen, dass Sie immer auf dem neuesten Stand sind und die neuesten Funktionen und Verbesserungen mit minimalem Aufwand nutzen können.



Hauptfunktionen des Windows-Terminals


Mehrere Registerkarten


Sie haben gefragt und wir haben zugehört! Die am häufigsten nachgefragte Funktion für das Terminal ist die Unterstützung mehrerer Registerkarten. Wir freuen uns sehr, dass wir diese Schlüsselfunktion endlich bereitstellen können. Sie können jetzt eine beliebige Anzahl von Registerkarten öffnen, die jeweils mit einer Befehlszeilen-Shell oder App Ihrer Wahl verbunden sind, z. B. Eingabeaufforderung, PowerShell, Ubuntu unter WSL, ein Raspberry Pi über SSH usw.



Schöner Text


Das Windows-Terminal verwendet eine GPU-beschleunigte DirectWrite / DirectX-basierte Text-Rendering-Engine. Diese neue Text-Rendering-Engine zeigt Textzeichen, Glyphen und Symbole an, die in Schriftarten auf Ihrem PC vorhanden sind, einschließlich CJK-Ideogrammen, Emoji, Powerline-Symbolen, Symbolen, Programmierligaturen usw. Diese Engine rendert Text auch viel schneller als die GDI-Engine der vorherigen Konsole!



Sie haben auch die Möglichkeit, unsere neue Schriftart zu verwenden! Wir wollten eine unterhaltsame, neue, monospaced Schriftart erstellen, um das moderne Erscheinungsbild des Terminals zu verbessern. Diese Schriftart enthält nicht nur Programmierligaturen, sondern ist auch Open Source und verfügt über ein eigenes Repository. Weitere Informationen zum neuen Schriftprojekt finden Sie hier!



Einstellungen und Konfigurierbarkeit


Wir haben uns mit so vielen Befehlszeilenbenutzern verbunden, die es LIEBEN, ihre Terminals und Befehlszeilenanwendungen anzupassen. Windows Terminal bietet viele Einstellungen und Konfigurationsoptionen, mit denen Sie das Erscheinungsbild des Terminals und alle Shells / Profile, die Sie als neue Registerkarten öffnen können, weitgehend steuern können. Die Einstellungen werden in einer strukturierten Textdatei gespeichert, die Benutzern und / oder Tools die Konfiguration erleichtert.


Mithilfe des Konfigurationsmechanismus von Terminal können Sie für jede Shell / App / Tool, die Sie verwenden möchten, mehrere „Profile“ erstellen, unabhängig davon, ob es sich um PowerShell-, Eingabeaufforderungs-, Ubuntu- oder sogar SSH-Verbindungen zu Azure- oder IoT-Geräten handelt. Diese Profile können eine eigene Kombination aus Schriftstilen und -größen, Farbthemen, Hintergrundunschärfe- / Transparenzstufen usw. aufweisen. Sie können jetzt Ihr eigenes Terminal im benutzerdefinierten Stil erstellen, das auf Ihren individuellen Geschmack zugeschnitten ist!


Mehr!


Nachdem wir Windows Terminal 1.0 ausgeliefert haben, planen wir, mit vielen der Funktionen, die bereits in unserem Backlog enthalten sind, zu beginnen, zusätzlich zu den vielen Funktionen, die Sie als Community wahrscheinlich hinzufügen werden!


Wann kann ich es in die Hände bekommen?


Ab heute sind das Windows-Terminal und die Windows-Konsole Open Source und Sie können den Code aus dem Repository auf GitHub klonen, erstellen, ausführen und testen: https://github.com/Microsoft/Terminal


In diesem Sommer im Jahr 2019 werden Windows Terminal-Vorschauen im Microsoft Store veröffentlicht, damit Early Adopters sie verwenden und Feedback geben können.


In diesem Winter im Jahr 2019 ist es unser Ziel, Windows Terminal 1.0 zu starten, und wir werden mit der Community zusammenarbeiten, um sicherzustellen, dass es bereit ist, bevor wir es veröffentlichen!

Warten Sie ... haben Sie Open Source gesagt?


Ja, wir haben es getan! Wir freuen uns, Ihnen mitteilen zu können, dass wir nicht nur Windows Terminal, sondern auch die Windows-Konsole als Open-Sourcing-Lösung anbieten, die die Befehlszeileninfrastruktur in Windows hostet und die traditionelle Konsolen-UX bereitstellt.


Wir können es kaum erwarten, mit Ihnen an der Verbesserung und Verbesserung der Windows-Befehlszeilenerfahrung zu arbeiten!


Das klingt großartig, aber warum können Sie die vorhandene Windows-Konsole nicht einfach verbessern?


Das Hauptziel der Windows-Konsole besteht darin, die Abwärtskompatibilität mit vorhandenen Befehlszeilentools, Skripten usw. zu gewährleisten. Obwohl wir es geschafft haben, viele wichtige Verbesserungen an den Funktionen der Konsole einzuführen (z. B. Hinzufügen von VT- und 24-Bit-Farbunterstützung usw., siehe diesen Blog-Beitrag ), können wir die Benutzeroberfläche der Konsole nicht weiter verbessern, ohne die Welt zu brechen . ”


Daher ist die Zeit für einen neuen, frischen Ansatz gekommen.


Windows Terminal wird zusammen mit der vorhandenen Windows Console-Anwendung installiert und ausgeführt. Wenn Sie Cmd / PowerShell / etc. ausführen. Direkt werden sie genauso wie heute an eine traditionelle Konsoleninstanz angehängt. Auf diese Weise bleibt die Abwärtskompatibilität erhalten, und Sie haben die Möglichkeit, Windows Terminal zu verwenden, wenn Sie dies möchten. Die Windows-Konsole wird noch Jahrzehnte in Windows ausgeliefert, um vorhandene / ältere Anwendungen und Systeme zu unterstützen.


Okay, aber wie wäre es stattdessen mit einem Beitrag zu einem bestehenden Open Source Terminal / App-Projekt?


Wir haben diese Option während der Planung sorgfältig geprüft und festgestellt, dass unsere Beteiligung an einem bestehenden Projekt eine Änderung der Projektanforderungen und der Architektur auf eine Weise erfordern würde, die zu störend wäre.


Stattdessen können wir durch Erstellen einer neuen Open-Source-Terminalanwendung und Open-Sourcing-Windows-Konsole die Community jetzt einladen, mit uns zusammenzuarbeiten, um den Code zu verbessern und ihn in ihren jeweiligen Projekten zu nutzen.


Wir glauben, dass es auf dem Markt viel Raum für neue / unterschiedliche Ideen gibt, was ein Terminal tun kann und sollte, und wir möchten dem Ökosystem von Terminal- (und verwandten) Anwendungen helfen, durch die Einführung neuer Ideen, interessanter Ansätze, zu gedeihen und zu wachsen. und aufregende Innovationen in diesem Bereich.


Ich bin verkauft! Wie kann ich mich engagieren?


Besuchen Sie das Repo unter https://github.com/Microsoft/Terminal , um das Terminal zu klonen, zu erstellen, zu testen und auszuführen! Sie können Fehler melden und Feedback mit uns und der Community teilen sowie Probleme beheben und Verbesserungen an GitHub vornehmen.


Versuchen Sie ab diesem Sommer, Windows Terminal im Microsoft Store zu installieren und auszuführen. Wenn Sie auf Fehler stoßen, teilen Sie Feedback entweder über den Feedback Hub oder GitHub-Probleme mit, um detaillierte Probleme / Diskussionen zu erhalten.


Wir freuen uns sehr, mit Ihnen zusammenzuarbeiten! Wenn Sie Fragen oder Feedback haben, wenden Sie sich bitte an Kayla @cinnamon_msft und / oder Rich @richturn_ms auf Twitter. Wir können es kaum erwarten zu sehen, welche aufregenden Verbesserungen und Funktionen Sie an Windows Terminal und Windows Console vornehmen.
