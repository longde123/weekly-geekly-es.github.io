WISE-PaaS-工业物联网的云平台



WISE-PaaS-（平台即服务），用于工业物联网的研华云平台，结合了各种工具来收集，处理和可视化数据，自动化，设备管理以及人工智能系统和机器学习。 该平台结合了几套现成的软件解决方案，用于构建工业，医疗保健，制造，物流等领域的复杂系统。

WISE / PaaS平台可以在云提供商Amazon Web Services（AWS），Google Cloud Platform（GCP）和Microsoft Azure以及OpenStack本地上运行。

本文讨论了一些WISE / PaaS软件产品，这些产品使您可以快速开发应用程序而不会因构建基础结构而分散注意力。 它支持以流行语言运行的应用程序：Java，.NET，Ruby on Rails，Node.js，Grails，Scala，Lift，Python PHP以及主要的MySQL数据库引擎，MongoDB，PostgreSQL，Redis，RabbitMQ。 为了可视化数据，可使用Grafana框架。 用于嵌入式系统的一套单独的软件工具简化了设备的初始设置，更新固件以及从IoT设备收集数据。

应用市场


该平台是许多不同工具的构造函数，可以根据需要在应用程序商店中购买这些工具。 在提出的解决方案中，研华自己的产品和合作伙伴产品均可提供。 某些研华产品有免费试用期。

市场上的软件解决方案根据目的分为几大类：

  • 云应用管理系统 -编排，监视，备份
  • 终端设备的管理 -更新固件，从设备收集数据，监视，自动配置
  • 安全性 -入侵检测和防御系统（IPS / IDS），防病毒
  • WISE-Paas / AFS-人工智能，机器学习和机器视觉系统。



为了免费测试，您必须在WISE / PaaS门户上注册并申请试用订阅。 为此，请选择市场上感兴趣的产品，然后单击“开始试用”。



进入系统的门户地址将取决于注册期间选择的数据中心。 当前可用的Azure数据中心（香港，北京），阿里云（杭州）。

WISE-PaaS /仪表板


WISE-PaaS /仪表板-一套用于基于Grafana框架可视化数据的工具。 它通常用于构建长时间出现的过程的图形，图表和可视化显示。 除工业用途外，它还可用于监视智能家居和医疗保健系统中的气候过程。


WISE-PaaS /仪表盘数据可视化平台

小部件


Grafana框架有许多显示数据的选项：表格，图形，图表，热图等等。 您可以创建一个完整的仪表板来显示各种数据，而几乎没有编程技能，也可以使用鼠标添加小部件。


用于将grafana小部件添加到仪表板的界面

除了内置窗口小部件，您还可以安装第三方插件以与其他系统集成。 例如，Zabbix监视系统的插件允许您从其导入数据并显示监视系统通知。

因此，WISE-PaaS /仪表板允许您组合来自各种来源的数据以在单个面板中显示它们。


Grafana界面中来自Zabbix监控系统的数据

资料来源


仪表板可以从各种来源接收数据以进行显示。 当前支持的数据库：CloudWatch，Elasticsearch，Graphite，InfluxDB，MySQL，OpenTSDB，PostgreSQL，Prometheus，RMM-SimpleJson，SCADA-SimpleJson，SimpleJson。 除了这些数据库之外，您还可以配置任何查询格式以从不同来源收集数据。 测试数据集也可用于浏览系统。


Grafana支持各种数据源

异常通知


为了响应某些事件，仪表板允许您配置各种通知。 这些可以是自动API调用，也可以是操作员通知。 这在创建控制台以通知值班操作员异常更改时特别有用。 作为触发条件，可以设置一定水平的过量或降低，特定时间段的平均值，数据不足等。


创建一个新触发器并添加一个通知栏

要显示通知，有一个单独的小部件“警报”，可以将它们显示在与图表相同的面板上。

WISE-PaaS / SaaS作曲家


SaaS Composer是用于构建技术流程的二维和三维方案的框架。 与经典的过时方案不同，它允许您为正在进行的过程创建更多信息和可视化模型。 以3D可视化整个生产线和建筑物，以三维模型显示实时事件。



主要功能：

  • HTML 5 Canvas上的Web界面。 良好的性能，无需安装其他软件即可查看图表。
  • 创建2D和3D模型。 以.OBJ + .MTL格式导入3D模型
  • 支持JPEG，PNG，SVG，OBJ，MTL图像。 在构建3D模型时支持矢量图形SVG。 您可以从现有方案中导入图形。
  • 在3D对象之上添加动画并显示数据
  • 脚本语言支持
  • 与其他WISE-Paas产品集成，尤其是WISE-PaaS /仪表板

3D模型的设计者拥有一个主要组件库：管道，阀门，电线，电动机，机床，格栅等。 在其中，您可以创建真实对象的逼真的模型，并添加带有数据的小部件。



研华大楼的演示图，实时显示能源系统的状态，不同位置的能耗水平，空调：CO2水平，空气中的细颗粒水平等。


使用SaaS Composer创建的演示显示了建筑物中各种参数的状态。

WISE-PaaS / APM


资产绩效管理-旨在增强对生产线效率的控制，以进行更准确的预测，风险评估和生产量控制。

WISE-PaaS / APM具有用于分析生产过程的内置算法，可让您跟踪哪些机器工作效率不高，预测产量，可能出现的问题以及对这些机器的需求。 服务。


WISE-PaaS / APM使您可以控制生产线的效率

WISE-PaaS / EnSaaS-使用设备（边缘到云）


为了方便地将终端设备集成到云基础架构中，WISE-PaaS提供了一套用于与嵌入式系统和物联网一起工作的工具。

WISE-Paas / DeviceOn是一个用于管理和配置大量终端设备（例如传感器，终端，嵌入式系统等）的平台。


主要功能：

  • 零接触配置 -自动配置终端设备并将其添加到系统
  • 访问限制 -确保设备的安全性并防止未经授权的访问
  • 更新（OTA）-自动更新终端设备上的软件和固件
  • 监视 -监视设备状态并通过推送通知，SMS或电子邮件通知问题
  • 备份和存档 -创建设备配置及其数据的备份
  • 构建设备地图 -用于在建筑平面图和地图上构建设备布局的构造函数

WISE-Paas / WISE-Agent


WISE-Agent-安装在终端设备上以与WISE-PaaS / DeviceOn交互的软件。 支持所有主要操作系统。 编译的软件包可用于Windows，Ubuntu，Android（RISC），OpenWRT（RISC）。
与云平台的交互根据MQTT协议进行。

[案例]在智能船的开发中使用WISE-PaaS平台


SaierNico公司-正在为运输公司开发智能系统并为船舶开发设备。 通过使用Wise-PaaS平台，SaierNico开发了实时远程船舶监控和事件响应系统。



传感器从船舶的各个组成部分收集数据：发动机转速，压力，空调系统状况，泵等。 对于数据传输，使用RabbitMQ代理，由于与容器的通信不稳定，因此增加了消息传递的可靠性。 数据流入WebAccess / SCADA系统。

系统架构


WISE-PaaS / APM用于跟踪船舶部件的性能。
控制中心的数据可视化基于WISE-PaaS / Dashboard和WISE-PaaS / SaaS Composer 。

船舶系统中终端设备的固件升级是使用WISE-PaaS / OTA进行的 。
