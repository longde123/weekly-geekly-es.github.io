AMP：电子邮件的未来


俄罗斯开发人员从ecwid.ru使用AMP技术创建的动态电子邮件


AMP项目旨在改善网络上的用户体验，这意味着在网络上处理电子邮件时可以使用它。 对于我们大多数人来说，电子邮件的功能自诞生以来就没有太大变化（而我们大多数人显然都没有发现这种现象）。 嗯，AMP的本质是确保速度和安全性，因此，不可能不为电子邮件开发AMP 。 邮件中的JavaScript似乎是一个好主意，但是由于有了AMP框架，用户可以实时与电子邮件进行交互，而不必担心数据的安全性。


“怎么样？” -你问？ 切下答案


多亏了AMP，用户有了与邮件互动的全新机会：现在，您可以回复邀请，填写调查表，浏览目录或直接在信函正文中留下评论。 此外，现在无论是主题中的新评论还是最新建议，您都可以借助信件来跟踪更新。

立即创建AMP字母


创建AMP字母的测试站点不止一个。 如前所述，Mail.ru提供了Playground 。 好吧，我们将在amp.dev上的AMP项目本身中更多地介绍测试站点 。 允许您编辑标记并实时跟踪更改。 有关支持的功能和语法的更多信息，请参见链接 。 但是这个过程并不复杂： 


<!doctype html> <html amp4email> <head> <meta charset="utf-8"> <script async src="https://cdn.ampproject.org/v0.js"></script> <style amp4email-boilerplate>body{visibility:hidden}</style> </head> <body> Hello, AMP4EMAIL world. </body> </html>

如您所见，这完全是普通的HTML，但是amp4email必须添加到html标记中，当然，不要忘记AMP库本身- <script async src =“ https://cdn.ampproject.org/v0.js ”> </ script> 。 然后一切都很简单。 如果您已经熟悉AMP，那么请满足您所有喜欢的标签，例如amp-img和amp-carousel 。 而且，如果该框架对您来说是新的或知识不完整，那么您来了 。


“但是所有承诺的AMP邮件都是动态的，它在哪里？” -你问。 在amp-bind中 ，还有其他地方！ 我们不会在这里重写所有文档，但是如果您有任何疑问，我们正在等待评论！


如果第三方公司正在为您发送电子邮件，则需要确保它支持AMP。 以下服务已经支持AMP规范，或者正在积极致力于该技术的实现：


  • SparkPost-电子邮件服务和分析平台
  • Litmus-一套用于营销和设计电子邮件的工具
  • Twilio Sendgrid是一个用于发送交易和营销邮件的平台，已有超过80,000家公司使用该平台。
  • Amazon SES和Amazon Pinpoint-基于云的电子邮件和分析服务
  • 和其他支持该技术的服务。 请与您的电子邮件服务联系以获取详细信息。

借助新的MIME标准， AMP电子邮件与当前的电子邮件生态系统兼容。 因此，如果邮件客户端尚不支持AMP字母，则此类字母将以HTML格式显示。 有关规格的更多信息以及说明，请参见此处 。 我们非常期待用户和开发人员的反馈。


新的AMP工作组


为了使AMP不断发展以造福用户，我们需要您的帮助。 因此，在AMP管理模型的框架内，并作为接收来自发件人和邮件服务的反馈的渠道，创建了一个专门的工作组来开发用于电子邮件的AMP技术。 该小组有两个任务：


  1. 鼓励在电子邮件服务之间交换有关AMP电子邮件的知识，以确保各种电子邮件服务的互操作性。
  2. 组织用于将用户和社区反馈发送给发件人和邮件服务的渠道，以开发和改进技术。

如果您代表电子邮件服务，发送电子邮件，或者只是想加入AMP技术开发团队 ，则可以在ampproject / wg（amp4email存储库）页面上参与问题讨论，或在Slack上加入＃wg-amp4email频道 。


开源软件，来自多个供应商的支持，逐步的迁移策略以及一个倾听每个人的开放工作组将帮助我们创建可被每个人访问的新一代电子邮件。
