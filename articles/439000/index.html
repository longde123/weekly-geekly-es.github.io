Reflejo del espejo


Una de las operaciones m√°s simples es reflejar el mundo sobre un eje. Para reflejarlo alrededor del eje y, tomamos el valor absoluto del componente x de nuestra posici√≥n. (-1, 1) convierte en (1, 1) y resulta estar dentro de un c√≠rculo usando (1, 1) como origen de coordenadas y con un radio mayor que 0.

Muy a menudo, el c√≥digo que usa esta funci√≥n se ver√° como position = mirror(position); para que podamos simplificarlo un poco. Simplemente declararemos el argumento de posici√≥n como inout. Result√≥ bastante bien, pero de esta manera solo obtenemos un eje para la duplicaci√≥n. Podemos expandir la funci√≥n girando el espacio como lo hicimos al girar las figuras. Si sabe c√≥mo funciona la generaci√≥n de ruido , entonces comprende que para la generaci√≥n de procedimientos a menudo repetimos la posici√≥n y obtenemos celdas peque√±as que son esencialmente las mismas, que difieren solo en par√°metros insignificantes. Dado que la funci√≥n fmod (adem√°s de usar% para dividir con el resto) nos da el resto, no la definici√≥n del resto, tendremos que usar un truco. Para n√∫meros positivos, esto es exactamente lo que necesitamos, y para n√∫meros negativos, este es el resultado que necesitamos menos el per√≠odo. Puede arreglar esto agregando un punto y nuevamente tomando el resto de la divisi√≥n. Agregar un per√≠odo dar√° el resultado deseado para valores de entrada negativos, y para valores de entrada positivos, el valor es un per√≠odo m√°s alto. El segundo resto de la divisi√≥n no har√° nada con los valores para los valores de entrada negativos, porque ya est√°n en el rango de 0 al per√≠odo, y para los valores de entrada positivos, esencialmente restamos un per√≠odo.

El problema con las c√©lulas es que perdemos la continuidad por la que amamos los campos de distancia.
