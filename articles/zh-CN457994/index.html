Visual Studio提示和技巧

无论您是新手还是使用Visual Studio多年，都有许多技巧可以使您的工作效率更高。 一段时间以来，我们一直在Twitter上使用#vstip标签分享提示，这是迄今为止最好的提示的集合。

调试器


在您第一次执行自己的代码时，按F10键来构建，运行和附加调试器而不是F5会自动中断。 无需断点。



受Visual Studio 2005支持

当您不得不一次又一次地附加到同一过程时，重新附加到过程（Shift + Alt + P）极为有用。



从Visual Studio 2017 v15.8支持

页边的蓝点表示在调试过程中线程切换。



受Visual Studio 2013支持

解决方案


通过在解决方案资源管理器中禁用节点扩展的还原以及在解决方案负载时重新打开文档来提高解决方案负载的性能并减少视觉噪声。



从Visual Studio 2019支持

为了快速进行键盘导航，请使用Ctrl + T查找解决方案中的所有内容-文件，类等。



受Visual Studio 2017支持

分配键盘快捷方式以执行“ git pull”，这样您就不必使用CLI或Team Explorer来确保您的存储库是最新的。



在Visual Studio 2019中受支持

使解决方案资源管理器自动选择当前的活动文档，因此您永远不会丢失其在项目中的位置。



受Visual Studio 2010支持

主编


使用Shift + Alt + W轻松围绕HTML元素。 已选择插入项，因此您可以轻松地将其编辑为所需的任何标签，并且结束标签会自动匹配。



受Visual Studio 2017支持

将任何JSON片段复制到剪贴板，并将其作为强类型.NET类粘贴到任何C＃或VB代码文件中。



受Visual Studio 2013支持

您无需在JSON属性名称周围写引号，只需键入冒号，Visual Studio就会自动插入引号。



在Visual Studio 2015中受支持

在按住Control键的时间内，使IntelliSense和工具提示变为半透明。



受Visual Studio 2010支持

不必重新键入'（'以在方法签名中显示参数信息，而是使用Ctrl + Shift +空格键显示当前使用的重载。



受Visual Studio 2010支持

杂项


在Visual Studio中发生某些事件时播放声音。



受Visual Studio 2010支持

为特定的开发方案创建自定义窗口布局，或监视设置并在它们之间轻松切换。



受Visual Studio 2017支持

指定任何解决方案都需要哪些Visual Studio组件，如果缺少，Visual Studio会提示用户安装它们。 在博客文章中使用.vsconfig配置整个组织中的Visual Studio ，以了解更多信息 。



从Visual Studio 2019支持

扩展名


Visual Studio拼写检查器 。 一个编辑器扩展程序，用于在您键入或与工具窗口交互时检查注释，字符串和纯文本的拼写。 它还可以拼写检查整个解决方案，项目或选定的项目。 选项可用于定义多种语言以进行拼写检查。



受Visual Studio 2013支持

添加新文件 。 一个Visual Studio扩展，可轻松将新文件添加到任何项目。 只需按Shift + F2即可在所选文件夹或与所选文件相同的文件夹中创建一个空文件。



受Visual Studio 2015支持

Git Diff保证金 。 Git Diff Margin在Visual Studio页边距和滚动条上显示当前编辑文件的实时Git更改。



受Visual Studio 2012支持

这只是数千个可用扩展中的几个。 要查看更多扩展，请转到Visual Studio市场 。
