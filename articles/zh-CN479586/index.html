Efros Config Inspector甚至使那些不使用它的人受益

Efros Config Inspector软件包旨在监视活动的网络设备配置，虚拟化环境，操作系统的安全性-最近，业务流程自动化工具（有关更多详细信息，请参见此处 ）已被各种用户成功使用了八年以上。 鉴于信息技术和网络防御工具的发展趋势，开发人员特别注意跨平台解决方案。

为了实现用户在我们的下一个冬季版本中会喜欢的配置管理功能，选择了OpenSSH跨平台软件包。 大多数Linux发行版都支持此功能，最近Windows操作系统家族也支持此功能。

对软件包功能的分析发现一个错误：客户端（即Cisco路由器）与服务器断开连接。 在OpenSSH软件包中的服务器代码中，存在与各种设备（特别是与Cisco设备）的所谓兼容性支持。 它使您可以指定客户端和服务器之间的交互协议，具体取决于所用设备的类型。 经过详尽的研究，很明显，在将OpenSSH软件包移植到Windows的过程中，由于移植的特殊性，禁用了此兼容性支持。

为了消除此漏洞，分析了OpenSSH的SSH服务器程序的源代码。 为了调试客户端-服务器交互的机制，考虑到移植到Windows的特殊性，进行了相应的修订。 这样做的方式不仅要解决Cisco路由器的问题，而且还可能要解决将与SSH服务器交互的所有设备的问题。 在成功完成工作和内部测试之后，Gazinformservice决定将源代码更改转移给Windows的OpenSSH开发人员。

开发社区赞赏软件包开发的贡献，并接受了编辑。 因此，Efros Config Inspector开发团队不仅根据自己的利益修改了开源解决方案，而且还为所有感兴趣的OpenSSH软件包用户提供了一个固定版本。

使用开源软件可以使您更快地解决业务问题-如今，没有人对此表示怀疑。 但是，为了满足整个IT社区的利益，需要为响应“免费”软件的开发而做一些事情-感谢和支持。

谢尔盖·尼基丁，
Gazinformservice LLC产品经理
