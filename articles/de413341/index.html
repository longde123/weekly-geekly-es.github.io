SberTech Oka Day in Rjasan am 16. Juni:


Formel eines idealen Sommer-Mitaps: nützliche Berichte + leckeres Grillen + Blick auf die Oka

- „Wie man für Habr und technische Blogs schreibt“, Nikolay Zemlyansky, Herausgeber von www.habr.com

- „Reaktive Programmierung auf Vert.x“, Anton Lenok, Software Engineer bei SberTech
Sind Sie es leid, zum ersten Mal einen kritischen Abschnitt zu debuggen? Möchten Sie nützlichen Code schreiben und das Problem mit Thread Racing nicht lösen? Vielleicht solltest du etwas Neues ausprobieren?
Vert.x unterscheidet sich grundlegend von dem guten alten Spring und Java EE. Dies ist seine Stärke, aber sie ist voller Komplexität. Der Bericht ist für alle nützlich, die Vert.x verwenden möchten, aber nicht wissen, wie sie vorgehen sollen.

- „Vom Monolithen zum Mikrodienst“, Nikolai Archakov, Senior Architect, SberTech
Es ist allgemein anerkannt, dass die moderne Architektur eines automatisierten Systems notwendigerweise auf Microservices aufbauen muss. Wenn Sie jedoch mit einer monolithischen Architektur beginnen, wird das System für die Emission geschrieben. In dem Bericht werde ich über den Weg sprechen, den die Entwickler der "Credit Factory" der Sberbank von einer monolithischen Architektur zu einer Microservice-Architektur eingeschlagen haben, über die Hauptprobleme, die auftreten mussten, und über die Praktiken, die für die erfolgreiche Migration von einem Monolithen zu Microservices verwendet wurden.

MOSDROID # 9 Fluor in Moskau am 23. Juni:


- „Mach die Benutzeroberfläche wieder großartig“, Alexander Smirnov, PapaJobs
Die Benutzeroberfläche ist ein ausreichend großer und wichtiger Bestandteil jeder Anwendung. In diesem Vortrag erfahren Sie, wie Sie mithilfe des Contraint-Layouts einfach und einfach eine Benutzeroberfläche erstellen. Warum so schnell eine völlig neue Version von Constraint Layout 2.0 veröffentlicht wurde und was eigentlich neu ist

- „Die Ordnungen des Codes - wir überwachen die Sauberkeit durch die Kräfte der Automatisierung“, Alena Manyukhina, Auto.ru
Wie kann man den Code sauber halten und gleichzeitig den Kollegen-Programmierer nicht mit geringfügigen Änderungen nervös machen? Wie misst man "Code-Sauberkeit"? Wie kann ein Zerfall der Codebasis verhindert werden? Wie bringe ich Ordnung in den Code eines alten Projekts? Und wie geht das in einem messbaren Zeitrahmen? Solche Fragen werden von jedem Programmierer mindestens einmal gestellt, wenn er an Projekten arbeitet, insbesondere an großen und langlebigen. Glücklicherweise steht die Technologie nicht still und es erscheinen ständig neue Tools, mit denen Sie Code schreiben und riesige Projekte schneller und besser entwickeln können. Einige sind in Ihre tägliche IDE integriert, andere müssen auf Remote-Servern ausgeführt werden, andere können einfach über die Konsole ausgeführt werden. Lassen Sie uns über diese Tools sprechen und uns insbesondere auf diejenigen konzentrieren, die unseren jüngsten Favoriten Kotlin unterstützen.

- "Android JetPack oder was ist neu in 2018", Gevorg Safaryan, SberTech
Lassen Sie uns darüber sprechen, welche neuen Komponenten zu Jetpack hinzugefügt wurden. Betrachten wir sie separat. Wir diskutieren, was sie unter der Haube haben. Und werden sie davon profitieren?
