Pengembangan Linux pada Windows dengan WSL dan Visual Studio Code Remote

Bekerja dengan VS Code Remote dan Windows Subsystem untuk Linux (WSL) memungkinkan untuk menggunakan lingkungan pengembangan Linux yang berfungsi penuh pada laptop atau desktop dengan Windows yang sudah diinstal. Di artikel ini, kita akan melihat cara menggunakan alat-alat ini untuk mengembangkan aplikasi Python di Linux.



Hal pertama: menginstal komponen yang diperlukan


Untuk menyelesaikan semua langkah yang dijelaskan dalam artikel ini, Anda akan membutuhkan WSL dan VS Code dengan ekstensi Remote VS Code yang terinstal - WSL. Mari kita cepat melihat apa itu dan bagaimana cara menginstalnya.

Subsistem Windows untuk Linux


Alat Subsistem Windows untuk Linux, disingkat WSL, memungkinkan Anda menjalankan binari Linux yang tidak dimodifikasi secara langsung di Windows. Ini berarti Anda dapat menggunakan alat Linux favorit Anda, aplikasi baris perintah, dan alur kerja di komputer Windows Anda tanpa perlu pengaturan rumit seperti Dual Boot. Dan yang paling penting, WSL adalah solusi terintegrasi yang erat antara Windows dan Linux. Dengan kata lain, jika Anda menyukai Linux, maka Anda dapat mengakses semua file Windows dan file executable langsung dari bash shell, dan jika Anda menyukai Windows, Anda dapat dengan mudah mengakses binari dan file Linux dari PowerShell atau CMD.

Jika Anda baru mengenal WSL dan ingin mempelajari lebih lanjut, Anda dapat mengambil kursus lengkap kami: Memulai dengan subsistem Windows untuk Linux , atau hanya membaca instruksi penginstalan dalam dokumentasi WSL .

Kode studio visual


Visual Studio Code adalah editor kode yang ringan namun kuat yang berjalan di desktop dan tersedia di Windows, macOS, dan Linux. Muncul dengan dukungan bawaan untuk JavaScript, TypeScript dan Node.js, dan juga memiliki ekosistem ekstensi yang kaya untuk bahasa lain (seperti C ++, C #, Java, Python, PHP, Go) dan runtimes (seperti .NET dan Unity). Unduh dan mulai perjalanan Anda dengan VS Code .

Anda juga membutuhkan ekstensi VS Code Remote - WSL, yang memungkinkan Anda untuk menggunakan wadah, komputer jarak jauh, atau Windows Subsystem untuk Linux (WSL) sebagai lingkungan pengembangan yang berfungsi penuh. Untuk mempelajari lebih lanjut dan memulai, kunjungi halaman dokumentasi VS Code Remote .

Sebagai permulaan, perhatian, Linux!


Alih-alih menulis bagian panjang tentang memulai dengan VS Code Remote di WSL, kami memutuskan untuk merekam video pendek ini yang menunjukkan pengembangan Python di WSL menggunakan VS Code Remote.


Ini bukan hanya Python terisolasi, Anda dapat menggunakan VSCode Remote dan WSL di proyek Linux apa pun


Saat menggunakan VS Code Remote di WSL, perintah dan ekstensi berjalan langsung pada distribusi Linux, jadi Anda tidak perlu khawatir tentang masalah kompatibilitas biner atau masalah lintas-OS lainnya. Anda dapat menggunakan Kode VS di WSL dengan cara yang sama seperti di Windows.

VS Code Remote dan WSL mengubah aturan main dalam hal pengembangan Linux di Windows. Alat-alat ini memungkinkan Anda mengembangkan, menjalankan, dan men-debug banyak aplikasi Linux secara langsung di komputer Windows Anda, memberi Anda yang terbaik dari Windows dan Linux.

Jika Anda ingin tahu lebih banyak tentang Subsistem Windows untuk Linux dan semua tentang baris perintah Windows, tetap ikuti perkembangan pembaruan di blog ini! Dan jika Anda tertarik untuk mempelajari lebih lanjut tentang VS Code Remote, Anda dapat membaca artikel blog di sini dan dokumentasi lengkap yang membahas arsitektur di sini .

Kami senang mempersembahkan kombinasi yang kuat untuk VS Code Remote dan WSL!
