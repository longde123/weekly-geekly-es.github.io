OpenCV 4.0 y 4.1: ¿qué hay de nuevo?



Volvemos a la historia sobre la biblioteca de visión por computadora de código abierto OpenCV . El proyecto vive y se desarrolla, impulsado por un equipo de desarrolladores que trabajan en Intel, así como por un soporte comunitario duradero. A finales de 2018, se lanzó la primera versión estable de la rama 4.x, y hace solo un mes se lanzó una nueva actualización: la versión 4.1. Les pedimos a los autores de la biblioteca que enumeraran brevemente qué novedades traían estas dos versiones a la funcionalidad OpenCV.

Opencv 4.0


El lanzamiento de OpenCV 4.0 completó el ciclo de vida de la versión 3.x: para corregir errores y mejoras menores, se creó una rama 3.4, a partir de la cual ya se crearán versiones menores 3.4.x (similar a 2.4.x).

Opencv 4.0 final

  • OpenCV ahora es una biblioteca C ++ 11 y requiere un compilador compatible con C ++ 11;
  • Se han eliminado muchas funciones de la API C obsoleta (de OpenCV 1.0), las constantes antiguas y las declaraciones de funciones se han movido a archivos de encabezado separados ( imgproc_c.h ) y ahora el usuario debe incluirlas explícitamente ( #include <opencv / imgproc / imgproc_c.h> );
  • Todos los módulos CUDA se han portado al repositorio opencv_contrib ;
  • La API de persistencia para escribir y leer datos en un archivo se ha reescrito en C ++, se han eliminado las funciones antiguas;
  • Se ha agregado un nuevo módulo G-API, que le permite crear gráficos a partir de operaciones en imágenes y aplicarles varias optimizaciones;
  • Se agregó soporte para el Kit de herramientas de implementación de Deep Learning (incluidas las versiones de código abierto ) al módulo dnn , incluido el uso de Intel Movidius Neural Compute Stick o Intel Neural Compute Stick 2 en Raspberri Pi 3 ;
  • El soporte para redes en el formato ONNX (Open Neural Network Exchange) se ha agregado al módulo dnn ;
  • Se agregó soporte experimental para cálculos a través de Vulkan al módulo dnn;
  • Se ha agregado la implementación del algoritmo en tiempo real para procesar escenas / modelos 3D de KinectFusion (con optimización para CPU y GPU / OpenCL);
  • Se agregó soporte para detectar y decodificar códigos QR al módulo objdetect (el decodificador usa la biblioteca QUirc): este verano, como parte de la pasantía de verano, se trabajará para mejorar la calidad y, posiblemente, se agregará el modo de detección-decodificación simultánea de más de un código QR en la imagen;
  • Se transfirió un algoritmo de flujo óptico DIS muy eficiente y al mismo tiempo de alta precisión desde opencv_contrib al módulo de video del repositorio principal.

Opencv 4.1


  • Se agregaron implementaciones optimizadas de despacho de muchos algoritmos en los módulos core e imgproc ;
  • Mejoras en el módulo dnn :
    • Soporte implementado para el lanzamiento de redes en el Intel Neural Compute Stick 2 (usando DLDT);
    • Consumo de memoria máximo reducido, soporte introducido para muchas redes nuevas de TensorFlow
  • El módulo Videoio agrega soporte para Android Media NDK API para leer archivos / transmisiones de video en dispositivos Android desde código C ++ (útil para probar algoritmos);
  • Se ha agregado un nuevo módulo para analizar la calidad de la imagen ( opencv_contrib / quality ). Implementa tanto algoritmos básicos (PSNR, SSIM) como nuevos algoritmos especializados (como el algoritmo de evaluación de calidad sin usar las imágenes BRISQUE originales: Evaluador de calidad espacial de imágenes ciegas / sin referencia);
  • Se implementaron varios algoritmos nuevos: flujo óptico local robusto, estéreo casi denso, calibración de la cámara conectada al manipulador (Hand-Eye);

Se puede encontrar más información sobre la biblioteca en el sitio del proyecto, que ha cambiado más allá del reconocimiento.

El número de parches de 4.0.0 a 4.1.0: 462 (aproximadamente 5.3 parches por día, excluyendo fines de semana y feriados). Hay muchos cambios, como puede ver, y son significativos. Si tiene preguntas sobre la funcionalidad incorporada o, por el contrario, la no implementada, bienvenido a los comentarios, los desarrolladores de OpenCV intentarán responderlos.
