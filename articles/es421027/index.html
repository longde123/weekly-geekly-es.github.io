SDMX (intercambio de datos estadísticos y metadatos)


Hay muy poca información sobre SDMX en Internet en ruso, a pesar de que este estándar ha sido utilizado durante mucho tiempo para la publicación e intercambio de datos por parte de muchos países y organizaciones internacionales. La iniciativa de desarrollo estándar fue lanzada por siete organizaciones internacionales que trabajan con estadísticas que patrocinaron el desarrollo. El objetivo principal era simplificar el intercambio de datos estadísticos entre dichas organizaciones, crear un estándar para dicho intercambio y describir el proceso comercial para implementar este estándar. Un enfoque único no solo le permite simplificar el acceso a datos estadísticos, sino que también mediante el uso de metadatos (datos sobre datos) hace que la comprensión de su significado y contenido sea más accesible.


El sitio principal de la iniciativa SDMX es sdmx.org , que también contiene una lista de conceptos aprobados entre dominios, libros de referencia y clasificadores. Cada organización que se une al estándar puede ampliarlo y complementarlo mediante un procedimiento de registro administrativo especial.


El estándar no es una guía estricta de acción; las organizaciones mismas eligen qué elementos SDMX usarán y para qué fines.


Un poco de historia
La primera versión del estándar permitió el intercambio de datos estadísticos y metadatos en los formatos GESMES / TS (su propio formato SDMX, que recuerda un poco a CSV con un separador, más tarde pasó a llamarse SDMX-EDI) y el formato de datos XML SDML-ML.
La versión 1.0 se aprobó en septiembre de 2004 y se adoptó como la especificación técnica de ISO (ISO / TS 17369: 2005) en abril de 2005.
En noviembre de 2005, la versión 2.0 estaba lista y aprobada, que es totalmente compatible con la versión 1.0, pero agregó la capacidad de intercambiar metadatos de enlaces (descriptivos).
La versión 2.1 (actual para 2018) se lanzó en mayo de 2011 y en 2013 se publicó como la norma internacional ISO 17369.
Más tarde, se describieron los estándares de intercambio en formatos CSV y JSON.


La descripción estándar de SDMX contiene los siguientes componentes:

  • Modelo de información
  • Esquemas XSD para describir la estructura, el modelo de contenido y los tipos de datos.
  • Pautas orientadas al contenido
  • Un conjunto de programas y herramientas para trabajar con SDMX

El modelo de información SDMX es la base del estándar. Está representado por conceptos ( CONCEPTO ), restricciones ( CONSTRAIN ), reglas, operaciones para determinar el formato y la composición de los datos estadísticos divulgados por la organización. En el marco de este artículo, no está previsto describir completamente todas las entidades SDMX, solo los componentes principales.


Modelo de información SDMX


¿En qué se diferencian las estadísticas de los datos regulares? Sí, en general, nada.


Datos estadísticos : un conjunto de datos ordenados y clasificados sobre un fenómeno o proceso de masas. Se caracterizan por un conjunto de medidas (conceptos, en términos de SDMX), una de las cuales suele ser un período de tiempo. Las herramientas de BI generalmente se utilizan para procesar y analizar dichos datos.


La observación estadística es un conjunto de valores específicos de conceptos que caracterizan de manera única cada unidad de la totalidad de la matriz de datos.



El número "208.36" es una observación estadística definida por un conjunto de conceptos (todos los datos son ficticios)


En SDMX, el concepto es el objeto básico de la estructura y representa una característica cualitativa de las observaciones estadísticas. Los valores para un concepto pueden ser un número, una cadena, una fecha o valores de directorios de código ( CODELIST ). Esta representación se puede redefinir en la definición de estructura de datos cuando el concepto se utiliza como una dimensión o atributo.


Las referencias de código son una simple lista de valores clave. La lista enumera los muchos valores que se utilizarán en la vista: indicadores, atributos y otros elementos de la parte estructural SDMX. Se complementan con otros metadatos estructurales, que pueden reflejar la descripción específica de diferentes idiomas y la organización jerárquica de los códigos.


El principio de codificación de objetos estructurales SDMX se define en el estándar: se permiten letras latinas en mayúsculas, números y guiones bajos. Además, se admiten estructuras versionadas.


Descripción de la estructura de datos La definición de estructura de datos ( DSD ) determina la composición adecuada y el orden de los conceptos para la formación del conjunto de datos final ( DATASET ). Cada concepto incluido en la estructura recibe una definición de su rol en el conjunto de datos:


  • Dimensión ( DIMENSION ): el identificador de datos principal. El conjunto de valores de todas las dimensiones, excepto la temporal, forma un código único ( CÓDIGO ) de la serie dentro del marco de una estructura de datos.
  • Atributo ( ATRIBUTO ): proporciona una descripción adicional para un conjunto de datos o para una observación específica. Un ejemplo de un atributo puede ser una unidad de medida o un estado de observación (preliminar, pronosticado, revisado, etc.).
  • El valor inmediato es la observación ( MEDIDA ).

Metadatos


En SDMX, los metadatos se dividen en dos grupos:


  • Los metadatos estructurales son un conjunto de conceptos utilizados para describir e identificar datos y metadatos estadísticos.
  • Los metadatos de referencia son un amplio conjunto de conceptos que definen y califican los conjuntos de datos y que generalmente no describen una observación o una serie de datos, sino todo el conjunto de datos o incluso la organización que proporciona los datos. Los metadatos de referencia generalmente están en formato de texto o HTML y utilizan conceptos que describen el contenido, la metodología y la calidad de los datos.

La descripción de la estructura de metadatos Metadata Structure Definition ( MSD ) incluye información sobre cómo se organizan los conjuntos de metadatos que contienen valores de referencia (similares a los DSD). En particular, MSD describe qué se incluye en el intercambio de metadatos y cómo se relacionan los conceptos entre sí, cómo se mostrarán (en forma de texto o valores del directorio) y con qué tipo de objeto (agencia, flujo de datos, proveedor de datos, conjunto de datos, etc.) Están conectados.


El conjunto de metadatos de referencia ( METADATASET ) es información que describe directamente el enfoque estadístico, la organización que proporciona los datos o la estructura de datos, el calendario de publicación, la calidad de los datos, etc., de acuerdo con la estructura de metadatos.

Pautas de contenido


Las pautas orientadas al contenido son un conjunto de pautas dentro del estándar SDMX. Su objetivo es la máxima compatibilidad en el intercambio de datos y metadatos entre organizaciones. Se alienta su uso entre organizaciones estadísticas tanto como sea posible. Los documentos principales son:


  • Lista de conceptos entre dominios
  • Áreas temáticas estadísticas
  • Diccionario general de metadatos

La lista de conceptos de dominio cruzado contiene una lista de conceptos estadísticos que se relacionan con el proceso estadístico y la calidad de los datos. Esta lista se basa en conceptos utilizados por organizaciones patrocinadoras internacionales. No es exhaustivo y se complementará en el futuro.

Los conceptos se pueden usar tanto para datos como para metadatos. Cada concepto tiene un código único y una descripción del contexto en el que se puede usar este concepto, así como una presentación en el estándar SDMX.


Los dominios estadísticos orientados a temas ( dominios orientados a contenido ) son una clasificación de alto nivel basada en el trabajo de la Comisión Económica de las Naciones Unidas para Europa (CEPE) sobre dominios estadísticos. La clasificación ofrece un punto de partida en la organización del intercambio de datos estadísticos y metadatos.


El Metadata Common Vocabulary Metadata Dictionary ( MCV ) contiene los conceptos y dimensiones asociadas utilizados en los metadatos estructurales y de referencia de las organizaciones internacionales y agencias nacionales. MCV es un vocabulario que recomienda el uso de terminología común para simplificar la comunicación y la comprensión. MCV está estrechamente relacionado con los conceptos de dominio cruzado y también contiene todos estos conceptos, lo que indica su definición y descripción del contexto.


Herramientas informáticas para trabajar con SDMX


Una lista de herramientas para trabajar con SDMX está disponible en sdmx.org .

La herramienta principal para trabajar con metadatos estructurales es el desarrollo de Metadata Technology - Fusion Registry . Funciona como una aplicación web. Hay dos versiones: Community (versión gratuita con funciones limitadas) y Enterprise Edition (de pago). Este software utiliza el Fondo Monetario Internacional sdmxcentral.imf.org como un único registrador (un único punto de recopilación y difusión de datos y metadatos). Este producto de software también utiliza la comunidad SDMX: Registry.sdmx.org .


Las últimas versiones de Fusion Registry han implementado casi por completo toda la funcionalidad del estándar. La aplicación también puede funcionar como un registrador SDMX. Desafortunadamente, no hay posibilidad de generar datos y metadatos en formato SDMX.


Asistente de estructura de datos : la aplicación Java para crear metadatos estructurales versiones 2.0 y 2.1, admite la creación de todas las entidades SDMX básicas.


El convertidor SDMX es la herramienta principal para trabajar con datos SDMX creados por Eurostat. Le permite crear un conjunto de datos (pero no metadatos) a partir de archivos en formato Excel, CSV, FLR, así como convertir datos entre diferentes formatos SDMX.


En lugar de una conclusión


La estandarización de la información estadística dentro del estándar SDMX simplifica enormemente la difusión y el análisis de datos. El uso de servicios web nos permite simplificar el procesamiento de matrices de información y garantizar la conexión de los sistemas relacionados, brindando a cualquier usuario la oportunidad de obtener y comparar indicadores macroeconómicos que le interesan en diferentes países del mundo. Las ventajas indicadas del estándar SDMX subyacen al proyecto interdepartamental que se está implementando actualmente en Rusia para introducir el estándar en la práctica de diseminar datos estadísticos tanto a través del intercambio de información con organizaciones internacionales como en el marco de proporcionar datos a un círculo ilimitado de usuarios que utilizan tecnologías de portal.
