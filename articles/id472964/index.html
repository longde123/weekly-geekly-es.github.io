Situasi: semakin banyak perusahaan open source mengubah lisensi - kami membahas pendapat para ahli

Selama 18 bulan terakhir, lisensi perangkat lunak telah memodifikasi setidaknya 12 pengembang solusi open source. Tujuannya adalah untuk melarang perusahaan IT besar untuk mengkomersilkan perangkat lunak mereka dalam satu atau lain bentuk. Inisiatif serupa membagi komunitas menjadi dua kubu.


Foto - Serenity Mitchell - Unsplash

Perusahaan Berlisensi


Pada akhir September , Open Core Summit diadakan di San Francisco. Di sana, peserta dalam komunitas IT mendiskusikan topik menjual kembali solusi open source dalam bentuk SaaS oleh perusahaan pihak ketiga. Banyak yang tidak senang bahwa dalam hal ini, pengembang perpustakaan asli tetap tidak bekerja dalam hal distribusi pendapatan. Karena itu, mereka pindah ke jenis lisensi baru.

Desember lalu, lisensi diubah di Confluent. Mesin KSQL untuk pemrosesan data real-time sekarang didistribusikan di bawah Lisensi Komunitas Confluent bukannya Apache 2.0. Lisensi baru masih memungkinkan Anda untuk mengunduh, memodifikasi, dan mendistribusikan kode sistem, tetapi melarang menawarkannya sebagai layanan berdasarkan pembayaran.

Jalur serupa dipilih oleh penulis Redis. Mereka mengembangkan Lisensi Sumber Tersedia Redis ( RSAL ). Kode dan solusi perusahaan dapat digunakan untuk aplikasi pihak ketiga. Tapi ada syarat - tidak bisa berupa database, sistem pemrosesan data streaming, atau mesin pencarian / cache. Hal serupa diterapkan di MongoDB. Lisensi mereka - Server Side Public License ( SSPL ) - mewajibkan pengembang untuk membayar jika perusahaan menjual versi DBMS yang dimodifikasi.

Para penulis Neo4j, sistem manajemen basis data grafik, umumnya memutuskan untuk menghapus kode untuk versi layanan perusahaan dari open source. Dan beralih ke lisensi komersial tahun lalu.

Siapa yang berpikir itu bagus


Setiap proyek open source membutuhkan model bisnis. Tanpa pembiayaan, tidak akan ada, berkembang, dan berkembang. Komersialisasi kode secara parsial membuka sumber pendapatan tambahan dan memungkinkan untuk menarik pengembang baru. Praktek ini akan membantu menghindari nasib manajer kalender terbuka Chandler . Itu berlangsung setahun - kemudian pengembang kehabisan uang untuk mendukungnya.

Beberapa anggota komunitas TI telah melangkah lebih jauh. Mereka menyarankan untuk memperkenalkan lisensi yang tidak hanya akan melarang penjualan kembali kode sumber terbuka, tetapi juga akan membatasi penggunaannya untuk perusahaan tertentu. Sebulan yang lalu, pengembang perpustakaan Ruby yang menyederhanakan bekerja dengan Chef, sebagai protes, menghapus proyeknya sendiri dari repositori di GitHub ketika ia mengetahui bahwa para insinyur dari Layanan Imigrasi AS menggunakannya. Insiden ini secara singkat mengganggu pekerjaan agensi. Para ahli mengatakan bahwa lisensi baru akan mencegah munculnya situasi seperti itu.

Dan pekerjaan ke arah ini sudah berlangsung. Di musim panas , proyek PolyForm muncul, dalam kerangka di mana lima lisensi baru untuk open source dikembangkan. Misalnya, salah satunya melarang bekerja dengan perangkat lunak sumber terbuka untuk perusahaan dengan lebih dari seratus karyawan.

Siapa yang meyakini sebaliknya


Namun, sebagian besar komunitas TI menentang komersialisasi proyek open source. Bruce Perens, penulis definisi sumber terbuka dan Kontrak Sosial Debian, mengatakan lisensi baru Redis, Confluent, MongoDB, dan perusahaan lain bertentangan dengan konsep perangkat lunak sumber terbuka. Paragraf kesembilan dari definisi tersebut menyatakan bahwa "lisensi tidak akan membatasi produk perangkat lunak lain". Salah satu penghuni Hacker News bahkan mengusulkan untuk menunjuk proyek dengan lisensi terbatas dengan istilah baru - Source Visible.

Pengguna HN lain juga mencatat bahwa posisi organisasi yang telah mengubah ketentuan lisensi dapat disebut munafik. Mereka sendiri menawarkan layanan perusahaan berbayar, menggunakan Linux, GNU, dan aplikasi open source lainnya secara paralel. Namun, mereka tidak memberikan kontribusi finansial apa pun untuk mereka.


Foto - Bebas Menggunakan Suara - Unsplash

Gordon Haff (Gordon Haff), kepala divisi cloud Red Hat, percaya bahwa ini adalah bagaimana perusahaan ingin "duduk di atas dua kursi" - untuk mendapatkan keuntungan dari penjualan modul dan menjadi perusahaan sumber terbuka. Pandangannya didukung oleh Thomas Dinsmore, penulis Disruptive Analytics . Dia mengatakan bahwa pengembang memiliki hak untuk memonetisasi produk mereka, tetapi dalam hal ini mereka pada awalnya harus memilih lisensi komersial.

Diyakini bahwa perubahan besar-besaran dari lisensi proyek-proyek besar akan menghancurkan komunitas terbuka. Sebagai contoh, pada 2015, FoundationDB dibeli oleh Apple. Perusahaan membatasi unduhan multimodel DBMS dan menutup semua repositori di GitHub. Akibatnya, komunitas, yang terbentuk selama lima tahun, mulai hancur.

Perusahaan IT besar juga merespons negatif terhadap praktik pergantian lisensi. Mereka menyebut keputusan semacam itu memusuhi pengguna, karena menjadi sulit bagi mereka untuk memahami lisensi. Dan mereka menuduh para penulis inisiatif bahwa proposal mereka dapat menyebabkan kehancuran total dunia open source.
