生活会更轻松。Windows 7和8.1切换到累积更新模型

每月的第一个累积更新将在10月发布




2016年5月，Microsoft做得很好，并将 Windows 7操作系统的所有更新自2011年SP1发行以来都放在一个“便捷汇总”软件包中。累积软件包发布在官方的Microsoft Update目录网站上。

然后在5月，该公司宣布Windows 7 SP1，Windows 8.1，Windows Server 2008 R2 SP1，Windows Server 2012和Windows Server 2012 R2的非安全更新将每月作为一个软件包发布。现在，由于用户的大量要求，Microsoft已决定对该程序进行一些更改。决定更新将不仅是每月一次，而且是累积的，也就是说，将在每时每刻可用仅是最相关的更新之一，其中包括所有先前的更新。这将极大地便利系统管理员和普通用户的生活。

通过切换到累积每月更新模型，Microsoft有望解决一些问题。其中最重要的是碎片化，即用户计算机上的所有操作系统彼此都不相同时，即，在所有操作系统上都安装了不同的更新集。由于已安装系统文件的组合不同，因此会发生依赖性错误以及安装新更新的问题。通常，在某些情况下，要安装一个更新，必须安装前一个更新，而对于前一个更新，则必须安装另一个更新。现在，用户将不会有这样的头痛。

消除碎片和累积更新可简化系统管理，并减少安装更新所需的工作。

微软将从2016年10月开始发布每月累积的每月更新每月汇总。更新将发布在Windows Update（WU），WSUS，SCCM以及Microsoft Update Catalog网站上。顺便说一句，ActiveX组件将很快从该站点中删除，因此它将在任何浏览器上运行，而不仅仅是IE。

10月Service Pack将包括上个月的所有更新。下一个11月的软件包将替换它，也就是说，它将同时包含10月的软件包和所有最新更新。通过Windows Update（WU）和WSUS进行的新的累积更新将通过快速软件包分发，即作为以前软件包的补充，以节省流量和磁盘空间。

逐渐地，Microsoft将在累积软件包中包含前几年发布的那些更新，以使其真正成为累积的。

Microsoft承诺与Windows 10一样，详细报告每个软件包中包含哪些特定的KB，以及发行说明。

尚不清楚用户是否必须选择安装哪些更新以及拒绝哪些更新。如您所知，除了安全更新外，Microsoft有时还会引入不必要的更新。我们希望删除不必要更新的标准命令能够像以前一样工作。

wusa.exe /kb:3080149 /uninstall /quiet /norestart

而是/kb:3080149应指定要​​删除的更新号。

除了“每月汇总”汇总包（安全更新+定期更新）之外，Microsoft将从10月开始单独发布每月安全更新，其中包括所有每月安全补丁。这些补丁将不再一次分发。单个安全更新将无法通过Windows Update获得，而只能通过WSUS，SCCM和Microsoft Update Catalog获得。

最后，更新.NET Framework也将移至月度系统。.NET Framework月度汇总更新将包括常规修补程序和安全更新。Microsoft强调此更新仅适用于当前版本的.NET Framework，不会自动升级到较新版本。除了常规软件包之外，开发人员现在仅使用.NET Framework安全更新来准备单独的软件包。

通过累积更新，Windows 7和8.1用户的生活将变得更加轻松。新系统应与Windows 10中的工作原理相同。
