Como configurar os backups da Veeam no armazenamento de blob do Microsoft Azure usando o StarWind VTL

Chamo a atenção para a tradução de um artigo do meu colega Andrew Zhelezko sobre o uso de uma solução integrada de armazenamento de backup baseada nos produtos Veeam, StarWind e Azure.

Até hoje, muitas empresas usam bibliotecas de fita para seus serviços, no entanto, o armazenamento em nuvem, que garante uma operação confiável e estável dos sistemas de backup, está ganhando popularidade. Além do suporte a bibliotecas de fitas virtuais comuns, o Veeam agora permite trabalhar com a biblioteca StarWind VTL para armazenamento de blob do Microsoft Azure . Essa é uma ótima oportunidade para quem precisa de um armazenamento em nuvem barato e confiável para armazenar com segurança backups de dados nele. Os usuários de uma solução integrada poderão implementar políticas flexíveis de armazenamento de dados: por exemplo, mantenha os backups em sua infraestrutura usual por uma a duas semanas e depois os mova para o Armazenamento de Blob do Microsoft Azure a longo prazo. Neste artigo, falarei brevemente sobre como configurar a integração.



Observo que o esforço que um usuário da Veeam precisará gastar na configuração da integração é mínimo. No entanto, deve-se ter em mente que existem muitos cenários para implantar componentes de uma solução integrada; portanto, você precisa ter cuidado e escolher a configuração certa para você. Os cenários podem ser muito diferentes - desde a instalação das soluções Veeam e StarWind no mesmo servidor na infraestrutura local até a instalação em VMs diferentes no ambiente de nuvem do Azure.

No processo geral de configuração de uma solução integrada, três estágios principais podem ser distinguidos.

Etapa 1: Preparando a infraestrutura em nuvem do Azure


Para preparar a infraestrutura de nuvem, você precisa fazer o seguinte:

  1. Vá para o portal do Azure, selecione Contas de armazenamento , adicione um novo armazenamento (ou armazenamento existente de Blob).
  2. Digite todas as informações necessárias, em Tipo de conta, selecione Armazenamento de blob .
  3. Vá para o novo repositório, copie o nome e a chave de acesso (localizada em Configurações > Teclas de acesso ).
  4. Crie um contêiner que será usado para armazenar dados ( serviço Blob > Contêineres > Novo ).

Você precisará dessas configurações ao configurar a replicação em nuvem no StarWind VTL.


Etapa 2: Configurar o StarWind VTL


Nesse ponto, emularemos a instalação de uma biblioteca de fitas no servidor selecionado. O Veeam Backup & Replication gravará dados na mídia nesta biblioteca e depois serão processados ​​e preparados para transferência para a nuvem. Este é o esquema clássico do D2D2C (disco para disco para nuvem) em ação.

As etapas a seguir devem ser concluídas:

  1. Instale a versão mais recente do StarWind VTL (no momento da redação, era 8.0.0.12248) no servidor de destino - pode ser uma máquina física ou virtual, um servidor em nuvem ou até um servidor Veeam.
  2. Durante a instalação, selecione a opção VTL e replicação em nuvem - os componentes StarWind correspondentes serão instalados automaticamente.
  3. Especifique o caminho para o pool de armazenamento (pode ser deixado por padrão na unidade C ).
  4. Em seguida, acesse o console StarWind, conecte-se ao servidor desejado (se você tiver tudo em uma máquina, selecione host local ou 127.0.0.1 para conectar) e crie um dispositivo de fita virtual com o número necessário de cassetes.

O StarWind VTL emula a biblioteca de fitas HPE MSL8096 real, com princípios operacionais típicos e um conjunto de operações.

Nota: Pode ser necessário instalar os drivers mais recentes para o servidor reconhecer corretamente a biblioteca especificada. Você pode levá-los aqui .

Agora você pode conectar a biblioteca ao servidor usando o kit de ferramentas iSCSI padrão do Windows ( Painel de Controle > Ferramentas Administrativas > Iniciador iSCSI ). Acesse Discovery > Discovery Portal para inicializar e conectar-se à biblioteca (guia Destinos ).


Depois disso, você precisa configurar o arquivamento na nuvem usando replicação. Para fazer isso, selecione Microsoft Azure Cloud Storage e especifique os dados que você salvou no estágio

1. Depois disso, insira as configurações do período de armazenamento desejado.



Etapa 3: configurar no lado do Veeam Backup & Replication


Quanto ao Veeam Backup & Replication, você precisa adicionar o servidor configurado na etapa anterior à infraestrutura Veeam como um servidor de fita. Você pode fazer isso no Veeam Backup & Replication; você precisará especificar IP / DNS e uma conta para acesso.
Ao adicionar um servidor nessa função, o Serviço de Transporte e o Serviço Proxy de Fita serão instalados automaticamente nele. Você também pode selecionar a opção de inventário automático de fita.

Depois que todos os cassetes são reconhecidos e colocados no Free Media Pool , faz sentido criar um pool de destino de vários cassetes. Vamos precisar um pouco mais tarde.

Para isso, basta criar uma tarefa de arquivamento na fita Backup to Tape (ou File to Tape ) e especificar o que arquivar (para Backup to Tape , você já deve ter cópias de backup das máquinas). Um pouco sobre como restaurar máquinas virtuais de uma biblioteca VTL para o Azure


Os usuários do StarWind podem iniciar a operação de restauração no armazenamento em nuvem do Azure - para isso, você pode abrir o console do Veeam Backup & Replication e seguir o caminho padrão com o assistente de recuperação de arquivos ou a VM. No entanto, por que não executar a recuperação diretamente na sua infraestrutura do Azure? De fato, no Azure Marketplace, há uma máquina virtual do dispositivo StarWind - como Veeam Backup & Replication, pode ser implantada no Azure e depois restaurada do armazenamento do arquivo morto diretamente para a nova VM no Azure. Essa abordagem economizará tempo e permitirá que os aplicativos funcionem em diferentes "compartimentos" da infraestrutura de backup.
Além de tudo, você pode fornecer acesso a essas novas VMs restauradas no Azure através do VeeamPN (Veeam-Powered Network) - por exemplo, para fornecer uma conexão segura ao principal data center da sua organização.
