ABBYY Mobile Web Capture: Foto dokumen berkualitas tinggi tepat di browser ponsel cerdas Anda



Pelanggan kami sering menggunakan ponsel untuk memotret dokumen dan mengirimkannya ke perusahaan berbagi mobil, pusat visa, operator telekomunikasi, perusahaan keuangan dan lainnya. Foto dokumen sudah cukup untuk mendapatkan mobil untuk disewa, mengaktifkan kartu SIM, mengajukan pinjaman. Tetapi terkadang mendapatkan gambar berkualitas baik dari smartphone bisa jadi sulit. Namun demikian, kami berhasil menyelesaikan masalah ini.

Sekarang ada banyak aplikasi di iOS dan Android untuk "pemindaian seluler" dokumen. Tetapi berapa banyak aplikasi seluler yang Anda miliki di ponsel Anda? Mengapa membuang waktu dan menginstal lebih banyak yang baru jika Anda tidak dapat melakukan ini?

Jauh lebih mudah untuk memotret dokumen secara langsung di peramban seluler, yang sudah ada di smartphone mana pun. Itu sebabnya kami membuat ABBYY Mobile Web Capture . Ini adalah JavaScript API, yaitu SDK yang kami tawarkan kepada pelanggan kami untuk disematkan di halaman web mereka dan aplikasi berbasis web. Ini memungkinkan Anda untuk mengambil gambar yang bagus secara langsung di browser web pada OS seluler paling populer dan mengirimkannya untuk diproses lebih lanjut ke server atau ke cloud. Hari ini, kita akan berbicara tentang cara kerja teknologi ini.

ABBYY Mobile Web Capture menawarkan untuk mengambil foto dokumen yang diinginkan dalam aliran video dari peramban seluler. Akun, SIM, paspor, kontrak, formulir aplikasi, aplikasi - dokumen apa pun dapat diproses.

Produk baru menggunakan teknologi seluler kami untuk pengambilan gambar secara otomatis Image Capture, yang ditransfer ke JavaScript. Inti dari algoritma ini ditulis dalam C ++, jadi kami menggunakan teknologi WebAssembly untuk mentransfernya ke browser web. Selain itu, kami membuat komponen UI yang bertanggung jawab untuk bekerja dengan kamera. Mereka ditambahkan ke API JS sehingga pengembang dapat dengan mudah dan mudah menanamkan tangkapan dari aliran video di aplikasi atau situs web berbasis web mereka. Untuk membuat integrasi menjadi sangat sederhana, kami menambahkan kode sumber dari kode halaman web sampel ke paket distribusi produk dan menunjukkan cara menggunakan API kami dengan benar. Bahkan, pengembang hanya perlu menyalin kode ini ke situs webnya dan itu saja - tidak lebih rumit daripada, misalnya, memasukkan metrik untuk analisis lalu lintas.

Setelah itu, tepat di browser, Anda dapat menghidupkan camcorder dan arahkan ke dokumen. Dan kemudian ABBYY Mobile Web Capture bertindak:

1. Ketika dokumen masuk ke dalam bingkai, kami menemukan batas-batasnya dalam aliran video.

2. Selanjutnya, Anda akan secara otomatis mengambil gambar dokumen yang jelas. Untuk melakukan ini, Anda harus memastikan bahwa orang tersebut telah berhenti menggerakkan kamera dan telah "mengarahkan" pada dokumen yang diinginkan. Kami tidak bergantung pada sensor telepon, karena selalu ada kemungkinan seseorang memegang telepon tidak bergerak, tetapi di sisi lain dokumen bergerak. Untuk menghindari ini, kami mengevaluasi apakah gambar bergerak, mis. pertimbangkan offset antara objek dari bingkai ke bingkai. Jika minimal, maka Anda dapat mulai menangkap. Selain itu, kami melihat ketajaman. Dengan demikian, SDK secara otomatis menangkap momen ketika Anda perlu mengambil gambar dokumen untuk mendapatkan gambar berkualitas tinggi. ABBYY Mobile Web Capture memberikan gambar yang bagus, yang kemudian dapat dikirim untuk pengakuan, misalnya, di ABBYY FlexiCapture , dan itu pasti akan diproses. Pada contoh proyek kami dengan ABBYY FlexiCapture, kami melihat bahwa seringkali lebih mudah bagi pelanggan untuk memasukkan dokumen melalui smartphone daripada pemindai. Tetapi gambar yang diperoleh dengan cara ini seringkali menjadi buram - maka mereka tidak dapat diproses dengan kualitas tinggi. Kemudian orang yang mengirim foto dari telepon diminta untuk "mengambil gambar". Tetapi tidak selalu nyaman bagi klien untuk melakukan ini.

Ketika kami mengembangkan ABBYY Mobile Web Capture, kami menyadari bahwa memotret dokumen di browser dalam aliran video tidak sesederhana itu. Pertama, mencari batas-batas dokumen, estimasi offset dan ketajaman frame membutuhkan sumber daya komputasi - saya harus mengoptimalkan sehingga aliran video di browser tidak melambat. Kedua, di iOS, kami dihadapkan dengan fakta bahwa di Safari tidak mungkin mendapatkan video dengan resolusi lebih tinggi dari HD. Gambar yang kami ambil di iOS, bahkan pada iPhone XS yang keren, tidak terlalu bagus. Mereka tidak bisa dikenali, karena untuk mengenali dokumen A4 yang diketik dalam font ke-10, Anda memerlukan gambar full HD dan di atasnya. Kami menulis laporan bug ke Apple dan meminta untuk memungkinkan bekerja dengan kamera dalam resolusi tinggi dari Safari. Dan mereka memperbaikinya di iOS 12.2! Tanpa ini, produk kami tidak akan berfungsi seperti sekarang. Dan sekarang - SDK memberi Anda gambar yang bagus, dan kemudian Anda dapat melakukan apa pun yang Anda inginkan dengannya.

Tentu saja, ketika kami melakukan SDK, kami mengevaluasi tugas apa yang harus diselesaikan pengguna akhir. Ceritakan sedikit tentang mereka.

Anda perlu membuka akun untuk pertama kalinya atau mengambil pinjaman dari bank, memesan asuransi, menyewa mobil atau layanan lainnya


Bayangkan Anda datang ke bank. Meski tidak, jika Anda datang, maka semuanya sudah salah. Banyak orang benci pergi ke bank. Anda berpikir seperti itu: "Eh, sekarang setidaknya setengah jam sekarang." Karena itu, jika Anda perlu mendapatkan semacam pinjaman tunai secepat mungkin, Anda kemungkinan besar akan mengambil telepon dan google opsi apa yang tersedia. Misalkan Anda menemukan informasi tentang pinjaman, dan situs menawarkan untuk mengisi aplikasi online. Lebih mudah untuk mengaturnya hanya di situs, dan tidak di dalam aplikasi - Anda perlu menginstal aplikasi, tetapi Anda masih tidak tahu apakah mereka akan memberi Anda pinjaman atau tidak. Mengapa pra-menyumbat ponsel dengan aplikasi yang tidak perlu? Jadi, Anda menekan tombol, dan kemudian Anda perlu mengisi data pribadi Anda.

Dulu seperti ini: bank meminta untuk membawa dokumen ke kamera atau mengunggah foto yang sudah selesai dari galeri. Orang-orang melakukan ini, tetapi gambar mungkin buram atau tanpa dokumen sama sekali - menurut pengalaman proyek kami, orang sering bingung dan mengunggah file yang salah sama sekali. Ini semua dikirim ke bank, tetapi gambar tidak cocok, dan semuanya sia-sia.

Beberapa bank mencoba untuk mengatasi masalah dengan cara ini: mereka memberikan pengakuan "instan". Tapi di sini ada cerita lain: klien memotret dokumen itu, mengirimkannya ke kantor belakang untuk pengakuan. Dibutuhkan sekitar satu menit. Tetapi ketika Anda melihat layanan dari ponsel, maka satu menit adalah waktu yang sangat lama. Selama waktu ini, Anda kemungkinan besar akan memutuskan bahwa semuanya macet, tutup halaman dan hubungi dukungan teknis atau tinggalkan semuanya dan pergi ke situs lain untuk mengajukan pinjaman.

Dengan ABBYY Mobile Web Capture, klien tidak perlu menginstal apa pun. Dia mengambil dokumen dalam aliran video. Teknologi memproses foto dan meningkatkan gambar, yang sekarang dapat dengan mudah dikirim ke bank.

Anda perlu memproses foto dari ponsel Anda untuk dikirim ke agen pemerintah, pusat visa, bank


Anda punya paspor? Kebetulan tidak ada dokumen di tangan, tetapi Anda memotretnya dan gambar disimpan di galeri ponsel cerdas Anda. Itu selalu berguna jika Anda perlu mendaftar di suatu tempat atau membeli tiket di situs web maskapai. Terkadang Anda mungkin diminta untuk mengirim foto U-turn paspor Anda. Bagaimana jika foto itu tidak begitu jelas atau diambil di atas karpet? Apakah mereka akan mengerti hal ini di pusat visa? Mungkin, tapi lebih baik tidak mengambil risiko.

Anda dapat mengunggah foto ini ke situs Anda menggunakan ABBYY Mobile Web Capture, dan teknologinya akan menemukan batas-batas dokumen pada gambar. Jika tidak ada batasan di atasnya, kami akan mengeluarkan peringatan dan menarik perhatian klien pada fakta bahwa ia mungkin mengunduh sesuatu yang salah. Plus, kami akan mencoba untuk mengevaluasi kualitas dokumen untuk memahami apakah itu cocok untuk pengakuan lebih lanjut. ("Kami akan mencoba" karena fitur ini sekarang dalam mode pratinjau teknologi, tetapi kami secara aktif bekerja untuk memperbaikinya.)

Perlu mengisi kartu dalam CRM tanpa siksaan yang tidak perlu


Berikut ini sebuah contoh: kami memiliki pelanggan potensial - dealer mobil, dan ia perlu tahu segalanya tentang orang-orang yang ia jual mobil. Untuk melakukan ini, karyawan memasukkan data pelanggan di portal web, seperti sistem CRM. Mereka meminta klien untuk hak, memindai mereka, dan kemudian mengetik ulang data ke komputer. Mengapa karyawan menggunakan portal web dan bukan aplikasi seluler? Sangat sederhana: tugas utama dealer adalah menjual mobil dan melayani pelanggan, daripada menulis banyak kode untuk aplikasi internal. Karena itu, penting bagi perusahaan untuk cepat menciptakan solusi yang akan bekerja pada semua platform.

Dengan ABBYY Mobile Web Capture, proses bisnis ini dapat disederhanakan: seorang karyawan hanya perlu mengambil gambar dokumen pada smartphone, dan kemudian mengirim gambar untuk pengakuan dan pemrosesan menggunakan produk kami yang lain - ABBYY FlexiCapture. Jadi pada akhirnya, kami akan menghemat waktu, dan kami akan meningkatkan kualitas data.

Sejujurnya, saya sendiri telah menemukan beberapa situasi di mana akan sangat berguna bagi saya untuk memindai dokumen langsung di browser. Misalnya, ketika saya mengajukan permohonan visa setahun yang lalu, saya menghabiskan sekitar satu jam untuk mengambil semua dokumen yang diperlukan, mentransfernya ke komputer, menyimpan dalam format yang diinginkan dan mengunggah ke situs. Jadi dalam 15 menit semuanya bisa difoto.

Saya sangat berharap bahwa ABBYY Mobile Web Capture akan segera digunakan di banyak situs dan akan membantu menyederhanakan tugas-tugas yang memerlukan memotret dokumen!

Olga Titova, pemilik Produk Mobile SDK
