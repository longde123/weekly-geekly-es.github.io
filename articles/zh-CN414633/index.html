部分有S.L.O.N.a。 介绍ITAM而不是扼杀（第2部分）



不存在ITAM实施步骤的通用顺序。 但是骨干网很突出，可以适合大多数公司。 项目“框架”中未包含的流程成为实施的各个功能。


在上一篇文章中，我分享了我的经验：


  • 从哪里开始实施ITAM解决方案；
  • 选择IT资产管理模型时要依赖什么？
  • 基本实施阶段的特点是什么？

在本文中，我想继续讲这个话题并告诉：


  • 如何建立用于IT资产核算的资源服务和资源财务模型；
  • 管理许可证和供应商的细节是什么？
  • 增长点是什么

IT资产管理包括哪些部分？


我有条件地将构成ITAM实践的各个组成部分的总和称为“ S.L.O.N.”。 这些组件包括：


  • 协议
  • 执照
  • 设备
  • 无形资产。

S.L.O.N.a我分为很多部分：


  • 识别和跟踪IT资产的变化；
  • 采购计划和IT资产的注销；
  • 建立用于IT资产核算的资源服务和资源财务模型；
  • 许可证和供应商管理。

前一篇文章分析了前两部分的细节。


今天通过目标→步骤→结果的棱镜， 我将考虑剩余的结果 。


建立用于会计IT资产的资源服务和资源财务模型




当在IT系统中积累有关IT资产的信息时，可以扩展此信息的使用载体。 我们开始考虑与承包商的所有合同，在这些合同下，我们承担IT资产的成本，并将IT资产与固定资产相关联。


我们要实现的目标：


  1. 获取有关使用IT资产的目标的信息（它们支持的服务工作）。
  2. 收集有关IT资产拥有成本和提供服务成本的信息。
  3. 在预算范围内制定成本计划机制。

我们采取什么步骤：


  1. 我们从会计系统（通过固定资产进行连接）加载有关IT资产的合同，成本，折旧和残值的信息。
  2. 我们输入有关IT资产之间关系的信息，从而创建资源服务模型。
  3. 我们使用资源服务模型和会计系统信息的链接来形成用于IT资产成本分配的财务模型。
  4. 我们正在计划IT预算。



监视使用IT资产的目标的结果是：

  1. 减少闲置的IT资产。
  2. 监视关键的IT资产。
  3. 改善资产回收率。

作为优化IT成本的结果：


  1. 强调昂贵的服务，活动和运营。
  2. 有关外包的决策。
  3. 更好的预算计划。
  4. 预算执行控制。

管理许可证和IT资产提供者




下一个重要步骤是与IT资产供应商合作。 这是ITAM“大象”的一个单独部分，在该部分中，我们考虑了供应商的所有义务，与他们的合同，包括支持条件。


重新定义目标：


  1. 优化许可成本。
  2. 组织供应商控制流程。

我们该怎么做：


  1. 我们收集有关许可证的数据。
  2. 我们形成一个许可的模型。
  3. 我们统一与供应商互动的规则。
  4. 我们对与供应商的合作质量进行分类和衡量。



优化许可成本的结果将是：


  1. 合并许可证，寻找更有利可图的许可证计划。
  2. 减少许可产品的数量，使其统一。
  3. 通过合并信息改善许可条件。
  4. 提高许可证利用率。

在组织供应商控制时，我们得到：


  1. 与供应商的工作流程的可衡量性和透明度。
  2. 通过整合和标准化节省成本。
  3. 简化供应商参与操作。
  4. 将合同作为独立资产运行。

如何开发IT资产管理系统




当S.L.O.N.的大部分内容被引入并实现自动化时，您可以考虑IT系统的开发。


增长点包括：


  1. 政策制定 -从供应商要求到内部要求。 支持和制定IT资产的企业标准。
  2. 在项目活动中使用IT资产 -服务，安装，许可证。
  3. IT资产中的知识管理 -包含有关事件和服务请求，文档，说明，常见问题的典型解决方案的文章数据库

结果，我们获得了组装在一起并相互连接的ITAM（大象）的多组件图片。




结果如何


我们自己引入ITAM解决方案的实践得出以下结论：


  1. IT资产管理过程是多部分的。
  2. S.L.O.N.a可以被“吃掉”，但不能一entirely而就。
  3. 在实施开始时，就有必要确定我们追求的目标以及在每个阶段执行的步骤。
  4. 总有增长点，因为 ITAM不仅限于收集有关IT资产的信息并跟踪其更改。
