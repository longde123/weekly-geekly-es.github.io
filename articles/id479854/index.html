  • Akhir dari Spring Boot 2.1.x dijadwalkan 1 November 2020. Beberapa pembaruan masih akan ada sebelum tanggal ini, dan setelah itu mereka akan hilang. Perlu diperbarui ke 2.2. Prosedur pembaruan itu sendiri tampaknya tidak rumit, petunjuknya ada di wiki . Ada juga daftar versi yang didukung .
  • Versi pertama dari toolkit Kilt , yang dirancang untuk menangani bundel internasionalisasi, telah dirilis. Sebenarnya, dalam nama Kilt, huruf i adalah i18n, l adalah l10n, t adalah t9n. Perpustakaan dapat mengonversi bundel dari / ke XLS (X) (berguna untuk tim penerjemah kecil), menyediakan fasad jenis-aman untuk bekerja dengan mereka di Jawa (tidak, huruf K atas nama bukan Kotlin) dan reforasi sederhana di dalam bundel, seperti memformat ulang teks. Ini dapat berfungsi sebagai aplikasi baris perintah mandiri, sebagai tugas untuk Ant, sebagai plugin untuk Maven. Hampir 100% kode ditulis di Jawa. 198 berkomitmen pada GitHub, hanya sepuluh bintang (tidak mengherankan, penulis mengumumkannya hanya hari ini).
  • Payara Server sekarang secara resmi kompatibel dengan Jakarta EE 8. Saya mengingatkan Anda bahwa ini adalah middleware Java EE open source dengan fokus pada cloud hybrid. Hadir di GitHub . Jakarta EE (jika Anda telah dalam animasi ditangguhkan selama beberapa tahun terakhir) adalah penerus Java EE, melalui Partai - masa depan Cloud Native Java. Dimulai dengan versi 5.193.1 , produk ini bergabung dengan keluarga bahagia Glassfish (Eclipse Foundation), OpenLiberty (IBM) dan Wildfly (RedHat). Jika Anda khawatir tentang masa depan Wildfly di sana, sekarang kami memiliki dua implementasi alternatif.
  • Versi terbaru dari Spring Cloud Gateway di GitHub adalah 2.2.0. Di versi 2.2.1 yang akan datang, kita bisa menerapkan pemutus arus baru di atas pustaka resilience4j yang terkenal (lebih dari seribu komit dan lima ribu bintang di GitHub). Di belakangnya ada kisah tragis kecil. Penggantian ini tidak terjadi atas perintah tumit kiri hipsters, tetapi karena Netflix membekukan Hystrix yang indah saat perusahaan mengubah tumpukan teknologinya. Gibb Spencer (pendiri Netflix) berjanji kepada kami bahwa Hystrix dan Ribbon akan terus mendukung sebagai bagian dari Spring Cloud, tetapi semua orang yang membutuhkannya telah mundur dari kapal dengan ketakutan. (Tweet ini bahkan sempat disebutkan di Josh Long 's Week in in Spring. ) Saya mengingatkan Anda bahwa alternatif kedua untuk resilience4j adalah Sentinel dari Grup Alibaba. Resilience 4j, pada gilirannya, dibuat oleh Robert Winkler dari Deutsche Telekom, Mahmoud Romeh dari IBM, Dan Maas dari Target Corporation (ini adalah jaringan toko ritel Amerika dengan omset 72 miliar dolar) dan Bogdan Storozhuk . Orang-orang yang serius, Anda harus menerimanya. Proses adaptasi dapat diamati pada hal-hal yang membentuk sistem seperti Spring Cloud Gateway, dan Anda dapat membaca lebih lanjut tentang ini di blog Pyotr Minkovsky (ini adalah penulis buku Mastering Spring Cloud ).
