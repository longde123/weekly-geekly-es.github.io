我们很高兴地宣布Mail.Ru Cloud Solutions同事的报告：

Mail.Ru云容器的开发人员Sergey Filatov将讨论使用OpenStack Magnum创建和管理Kubernetes集群的内容 ，其中包括：

  • 万能模块架构。
  • Heat的已使用功能，包括软件部署和生命周期群集的组织。
  • 团队在将OpenStack Magnum与Mail.Ru Cloud Solutions云基础架构集成时面临的陷阱。

Mail.Ru云解决方案的Victor Chembaev将分享从基于HAProxy的Neutron LBaaS v2切换到Octavia的经验， Octavia提供了用户需要的许多新选项：例如，开箱即用的高可用性和SSL流量代理。 您将了解：

  • 决策体系结构上的巨大差异。
  • HAProxy和Octavia在OpenStack的一个实例中同时共存的细微差别。
  • 无缝集成到现有公共云基础架构中的功能。

其他公司的代表也将在会议上发言：

1C系统架构师Basel Darvish将告诉您1C如何：Ready工作站平台是基于OpenStack构建的。 Hystax的 CTO Maxim Bozhenko将展示从手动集成到将应用程序迁移到OpenStack的盒装解决方案的方法。 SberCloud的Alexander Chadin将发表有关OpenStack Watcher的演讲，而Arista Networks的Andrey Nushtaev将谈论使用Arista设备在OpenStack中组织底层网络和覆盖网络。
