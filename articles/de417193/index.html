Überblick über den IT-Markt für Cloud-Business-Lösungen

Kürzlich wurde ich vom Leiter eines kleinen neu gegründeten Unternehmens mit der Bitte angesprochen, ihn am modernen Cloud-Computing-Markt oder in einfachen Clouds zu orientieren. Wie jeder von den Wolken gehört hat, wissen sie, dass sie modisch und nützlich sind. Aber wie benutzt man Wolken in der täglichen Arbeit eines gewöhnlichen russischen Unternehmens? Wann und warum Wolken verwenden?

Die Essenz des Cloud Computing


Wolken tauchten in der IT als Spiegelbild des allgemeinen Trends der Wirtschaft des gemeinsamen Konsums auf, der sich auf Nutzung statt Eigentum konzentriert und uns bereits Wohnungsaustausch , Carsharing und sogar Brautkleider zur Miete gegeben hat. Es gibt immer mehr Dinge und Dienstleistungen, die nur für ihre tatsächliche Nutzung genutzt und bezahlt werden können, ohne große Kapitalkosten für den Erwerb von Immobilien.

Im Falle einer IT-Infrastruktur können die Investitionsausgaben erheblich sein, sie werden auf verräterische Weise schnell veraltet und verlieren an Wert. Daher ist ihre Verwendung in einem Mietsystem häufig gerechtfertigt.

Es gibt eine erhebliche Einschränkung: Eine stabile Internetverbindung an dem Ort, an dem Sie die Cloud nutzen möchten, weil Alle „Cloud-Magie“ wird über das Internet geliefert. Wenn es kein Internet gibt, gibt es keine Magie.

Neben möglichen Einsparungen bei den Kapitalkosten gibt es weitere wesentliche Vorteile:

  • Geschwindigkeit. Führende Lieferanten können in wenigen Minuten / Stunden eine Cloud erhalten.
  • Flexibilität. Dank der Visualisierungstechnologien können Sie hier und jetzt so viele Computerressourcen abrufen, wie Sie benötigen. Einen zu langsamen Server zu verpassen und zu kaufen, ist keine große Sache.
  • Einsparungen bei den damit verbundenen Kosten. Unterbrechungsfreie Stromversorgung des Servers, Einschränkung des physischen Zugriffs auf den Server (+ die erste Geschäftsregel in Russland * ist einfach zu implementieren) usw.
  • In einigen Fällen - Einsparungen beim Personal (unten mehr)

Was sind die Wolken?


Um eine Cloud-Lösung auszuwählen, müssen Sie entscheiden, wie stark Sie die Cloud selbst steuern möchten und was Sie auslagern, an eine Drittanbieterorganisation auslagern (und davon abhängig sind).

Es gibt drei Hauptformate von Cloud-Lösungen gemäß dem Servicemodell.

SaaS - Software als Service


Die benutzerfreundlichsten Clouds. Sie arbeiten einfach in den erforderlichen Programmen über das Internet (ohne sie auf Ihrem Gerät zu installieren).

Vorteile:

+ technische Details sind komplett verborgen. Wir verbinden uns über das Internet und verwenden das fertige Programm. Update, Verfügbarkeit und andere technische Probleme werden vom Dienstanbieter behoben.
+ In der Regel die Mindestverfügbarkeitsbedingungen. Es reicht aus, den Service zu bestellen und Sie können ihn in wenigen Minuten nutzen.

Nachteile:

- Aus technischen Gründen sind nicht alle Programme in diesem Format verfügbar.
- Die Möglichkeiten zum Anpassen und Ändern des Programms an Ihre Anforderungen sind begrenzt.
- Starke Abhängigkeit vom Dienstleister und der Qualität seiner Arbeit.
- Oft haben Sie nur begrenzte Kontrolle über Ihre eigenen Daten.
- Wird häufig teurer als andere Arten von Clouds / klassischen Anwendungen, insbesondere für Großkunden.

PaaS - Plattform als Service


Sie mieten eine bestimmte Computerplattform, die aus virtuellen Computerdiensten (ein Analogon des Zentralprozessors und des Server-RAM), Informationsspeicherdiensten (ein Analogon der Festplatte des Servers), Datenübertragungsdiensten (verbrauchter Datenverkehr) usw. besteht. Das goldene Mittel der Cloud-Dienste. Sie können beliebige virtuelle Computer mit einer Vielzahl von System- und Benutzersoftware entwerfen. Lieferanten - führende IT-Unternehmen von Weltrang - sorgen für einen stabilen Betrieb dieser Computer und Programme. Lösungen in diesem Bereich sind bei Firmenkunden und Entwicklern beliebt.

Für kleine Unternehmen sind diese Lösungen schlecht geschärft und es ist unpraktisch, sie in diesem Zusammenhang zu verwenden.

Vorteile:

+ erstaunliche Flexibilität - Sie können einen Computer mit beliebiger Kapazität bauen (von einem Mikroserver mit vergleichbarer Leistung über ein Smartphone bis hin zu einem Cluster von Hunderttausenden von Servern) und eine Vielzahl von Anwendungen darauf installieren.
+ Hauptanbieter verfügen über riesige geografisch verteilte Computernetzwerke, die die Bereitstellung schneller, massenhafter und fehlertoleranter Anwendungen vereinfachen
+ Es ist möglich, erweiterte Dienste zu verbinden, bei denen Oracle, Microsoft und Google über einzigartige Kompetenzen verfügen - die Verwendung künstlicher Intelligenz, die Analyse großer Informationsmengen usw.
+ Es ist möglich, nur tatsächlich verbrauchte Ressourcen zu bezahlen.

Nachteile:

- Zur Verwendung müssen Sie das System aus virtuellen Komponenten-Komponenten "zusammenbauen". Dies erfordert spezifische Kenntnisse und Fähigkeiten.
- Jede Plattform schränkt die Implementierung ein, es gibt keine vollständige Freiheit bei der Implementierung
- ziemlich hohe Grundkosten

IaaS - Infrastruktur als Service


Sie mieten Server- / Computerressourcen. Dem Dienstanbieter wird die Funktionsfähigkeit auf Serverebene (elektronische Serverfüllung, Internet, Stromversorgung usw.) / Virtualisierungstechnologien (siehe unten) garantiert.

Es gibt drei Formate für die Erbringung von Dienstleistungen:

  1. Dedizierter Server - ein dedizierter Server. Mieten Sie einen separaten physischen Server, den nur Sie verwenden können (daher der Name - dediziert). Es hat relativ hohe Kosten, stellt jedoch sicher, dass alle Computerressourcen des Servers vollständig zu Ihrer Verfügung stehen. Der Anbieter bietet vollständigen Administratorzugriff auf den Server und wird von Ihrem Spezialisten nach Ihren Wünschen konfiguriert.
  2. IaaS in seiner reinsten Form ist ein virtuelles Rechenzentrum. Der Lieferant installiert spezielle Virtualisierungsprogramme auf einem oder mehreren Servern und überträgt Ihnen dann den Zugriff auf die Verwaltungsschnittstelle dieser Programme. Dieser Ansatz ist sehr flexibel, da Sie eine beliebige Anzahl von virtuellen Servern unabhängig voneinander erstellen und konfigurieren können.
  3. VPS / VDS - virtueller privater / dedizierter Server. In diesem Fall hat der Anbieter IaaS aus Klausel 2 für Sie konfiguriert und den Zugriff nur auf virtuelle Server übertragen, deren Parameter Sie nicht ändern können. Wichtige Unterschiede sind hier die verwendete Virtualisierungstechnologie (KVM / XEN / OpenVZ / Hyper-V / VMware) sowie die garantierte Leistung. VPS / VDS sind kostengünstig, können jedoch aufgrund von Überverkauf eine geringe / instabile Leistung aufweisen. Die VDS-Leistung ist höher als die von VPS, alle anderen Dinge sind gleich. Der VPS / VDS-Markt ist ein gesättigter und wettbewerbsintensiver VPS-Suchdienst

Vorteile:

+ Geld beim Kauf eines Servers zu "vergraben" ist nicht immer vernünftig, weil es wird mit der Zeit veraltet. Wenn Sie einen Server mieten, können Sie ihn jederzeit mit minimalen Kosten ändern oder aktualisieren.
+ Sie können die Leistung und die Serverparameter nach Ihren Wünschen auswählen und verwalten
+ völlige Freiheit bei der Umsetzung des gewünschten. Sie können alle Betriebssysteme und Programme installieren. Anpassen und verwenden, wie Sie möchten.
+ niedrige Cent, gutes Preis-Leistungs-Verhältnis

Nachteile:

- Sie benötigen einen Spezialisten für die Konfiguration und Wartung der Software.
- Oft besteht eine Bindung an die spezifischen Merkmale der Server- / Virtualisierungstechnologie, und die Flexibilität geht ein wenig verloren.
