Perbandingan pencarian internal untuk produk Yandex (beta)

Kemampuan untuk menanamkan pencarian barang (beta) dari Yandex muncul pada musim panas 2016. Masih ada sedikit informasi tentang integrasi fungsinya di situs web di Internet. Memperbaiki ...

Beberapa statistik untuk situs dengan katalog barang: 78% pengunjung situs pergi ke "katalog dengan barang", 45% di antaranya akan menggunakan pencarian berdasarkan kategori di menu, 40% akan menggunakan bilah pencarian saat mencari barang . Statistik diambil dari metrik Yandex untuk hypermarket Domostroy (23 toko di Distrik Federal Ural untuk 35.000 - 40.000 item, masing-masing 7 hingga 8 ribu meter persegi). Di situs yang sama, saya menginstal dan menguji kemampuan pencarian dari Yandex (beta).

Perbandingan pencarian produk Yandex dengan toko barang reparasi dan rumah yang paling banyak dikunjungi


Pada gambar di bawah ini: Situs 1 = domostroy-shop.ru dengan pencarian Yandex (beta) untuk toko online. Situs 2-9 dari peringkat TOP 15 dari radar.yandex.ru (Topik: bahan dan peralatan bangunan, ketik: semua).



Selain relevansi, pencarian tidak boleh memperlambat kecepatan pemuatan situs:



Anda dapat melihat kode JavaScript untuk disematkan dengan penjelasan di situs dalam direktori .

Pro:


  • Kecepatan (pencarian itu sendiri membutuhkan 0,2-0,3 detik)
  • Relevansi
  • Cari SEMUA produk toko dari file YML (bahkan jika produk tidak ada dalam indeks)

Cons:


  • Pengindeksan penuh dari direktori baru membutuhkan waktu sekitar 30 menit. Saya memecahkan masalah seperti ini: sebenarnya 2 pencarian (2 kunci API), cukup ubah kunci + id setelah pencarian diindeks.
  • Tanpa mengunggah ke JSON, hasilnya dalam Iframe tanpa kemampuan untuk mengubah gaya.
  • Hasil dihasilkan menggunakan JavaScript, jadi Yandex tidak melihatnya. Jika pengindeksan kategori diperlukan, maka mudah dipecahkan jika diinginkan.
  • Sulit untuk membuat halaman "semua produk" karena harus selalu ada kata pencarian dalam permintaan API. Saya memutuskan untuk menambahkan "kata" tambahan ke nama produk, yang dihapus sebelum dipublikasikan pada halaman.
