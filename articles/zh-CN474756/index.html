VMworld Europe 2019：传奇背包的重要公告和包装

在巴塞罗那举行的VMworld Europe 2019开幕式上，VMware首席执行官Pat Gelsinger在会议和新闻发布会上致辞。 在他的致欢迎辞中，他概述了IT专业人员的现代代表的关键概念：任何时代的技术，任何事物和所有人的时代的技术。 今年，会议聚集了来自100多个国家的近5,000家公司的代表。 在今天的文章中，我们想与您分享VMworld Europe 2019大会第一天的主要公告，最后-漫步VMworld Europe 2019并打开传奇背包！



帕特在讲话中指出，以前只有一小群发烧友的命运的技术，如今已捕捉到普通人生活的方方面面。 它们成为生活的一部分，就像检查您的早间邮件，测量睡眠，血压和……等等一样自然，这一切都是技术！

当然，如此紧密的生活与技术融合不会影响经济：根据Pat Gelsinger的说法，消费者在电影市场上的支出高达360亿欧元，而在移动应用市场上的支出几乎是其三倍-高达1,080亿欧元。 数字市场的这种资本化使IT公司不仅可以创建新技术，而且可以独立确定和增加自身的功能限制，从而为用户创建越来越强大的产品。

Pat尤其与听众分享了VMware对未来技术的现代观点：在任何云中，从公共云，混合云到私有云，都可以从任何数字设备安全使用任何应用程序。
掌握多云的技术人员将拥有下一个十年。

另一个好消息是扩大VMware与Amazon，Google，IBM，Oracle之间现有合作关系的功能。 这使我们可以谈论基于合作伙伴公司产品的全新，成熟的体系结构的出现。
Kubernetes通常被称为容器编排。 但这实际上更像是即兴爵士乐。

在鼓舞人心的开场白之后，VMware首席工程师，Kubernetes的原始作者之一乔·贝达（Joe Beda）出现在舞台上，帕特（Pat）亲自称其为社区的摇滚明星 。

VMware Tanzu


Joe表示，目前该项目的任务之一是保持Kubernetes的相同灵活性，但要克服随着功能增加而增加的配置复杂性阈值。 这正是出现新项目-用于Kubernetes容器化应用程序基础架构的VMware Tanzu的原因。 实际上，Tanzu被认为是一个多合一的套件，该套件的产品组合将改变公司构建现代应用程序的方法。 它将结合Kubernetes中应用程序的生命周期（包括组成）的三条鲸鱼：创建，操作和管理。

Tanzu的工作遍布全球500万程序员和多家公司。 特别是，Bitnami，Pivotal和Spring参与了该项目。

Bitnami带来了可用于在容器和虚拟机中部署的组件的开源目录。 Pivotal专注于构建本机云应用程序，而Spring的基础工作可以显着提高开发人员的生产力。

VMware Tanzu任务控制


允许您从一个“窗口”有效地管理Kubernetes的所有集群，包括VMware本地，公共云中的环境，EKS，IKS，GKE。 保证了从VMware PKS到Tanzu的超顺畅过渡。

大帆船项目


另一个公告是Project Galleon的beta版，这是Bitnami的应用程序市场，它实际上是Bitnami目录的逻辑扩展。 Galleon将允许您组装最适合公司标准的一组组件，对其进行自定义，并创建自己易于管理和修改的目录，这将使您更快地开发安全可靠的应用程序。

太平洋项目


Project Pacific是VMware的另一项公告。 该项目的任务是完全连接Kubernetes和vSphere。 这与在vSphere之上启动Kubernetes无关，而与绝对集成有关。 Project Pacific是一套用于将VMware vSphere转换为Kubernetes集群的本机平台的工具。 通过单个API，将有可能同时管理Kubernetes虚拟机和资产。 集成不仅会影响管理级别，还会影响虚拟化。 根据Joe Beda的说法，与Linux上的传统虚拟机相比，使用Kubernetes + vSphere时的性能提高将达到30％，与裸机服务器相比，性能将提高8％。

NSX Advanced Load Balancer和NSX Intelligence


在VMworld 2019上，VMware推出了NSX Advanced Load Balancer和NSX Intelligence，这是NSX-T 2.5的主要亮点，这对VMworld 2019产生了重大影响。

VMware NSX Advanced Load Balancer （以前称为Avi Networks Platform）-用于云的分布式应用程序交付控制器（ADC）。 结合了软件负载平衡器，智能Web应用程序防火墙（iWAF），监视和高级分析功能。

VMware NSX Intelligence是NSX-T内置的新的分布式分析引擎，可为安全管理员提供数据中心共享。 NSX Intelligence会验证各个虚拟化的工作负载包，从而提供极其详细的网络策略。 NSX-T还获得了2.5版，其中包括本机云强制实施模式，该模式允许客户使用自己的云安全控制来保护公共云工作负载。

适用于云提供商的VMware Cloud Foundation


会议的另一个重要公告是VMware Cloud Foundation for Cloud Providers，其中包括：

  • 服务提供商VCF（VCPP）-为大型客户构建专用云基础架构的产品的基础（专用私有云）；
  • HCX混合云迁移和部署解决方案。 允许您快速轻松地将客户端工作负载从本地环境转移到专用私有云；
  • SDDC Manager自动化平台：SDDC Manager除了进行部署外，还提供VMware Cloud Foundation的生命周期管理。







但是回到主要场景。 Pat Gelsinger在介绍性演讲结束时回顾了VMware Cloud Foundation和VMware vCloud Director的功能，之后邀请Sanjay Punen进入COO阶段，与SAP SE执行董事会成员Thomas Saueressig进行了公开讨论。 Saueressig与听众分享了公司关于云技术的未来以及与公司市场领域的互动的战略和实践观点。 Sanjay反过来指出，SAP是使用软件定义的解决方案的先驱，这些解决方案目前正在演变为SDx。 在对话中，与会者谈到了会议的主要议程之一-多云。 该公司正在与AWS，Microsoft Azure，Google Cloud Platform，AliCloud合作积极实施这一战略方向，但同时也关注SAP客户自己的云：
“我们生活在一个多云混合云解决方案的世界中，这使消费者可以选择S / 4HANA的资源，这是一个巨大的优势。”

VMware愿景


Pat Gelsinger还分享了VMware的最新愿景，称其​​为公司合作伙伴云业务的基石，并指出VMware的任务之一不仅是建立合作伙伴和服务提供商社区，而且要建立一个真正的星系。 Pat认为，正是这一步将使我们迈入真正的全球多云时代。


“在我们打造未来银河云的过程中，愿力量与您同在”

作为向多云过渡的一部分，Pat Gelsinger谈到了VMware在全球的存在的主题。 目前：

  • 全球超过128个国家/地区的10,000多个数据中心在使用VMWare；
  • VMware上有7000万虚拟机，其中有1000万在云中。

他还指出，VMware战略的关键要素是通过与全球云提供商领先者亚马逊的合作关系来扩展技术能力。



新的Microsoft集成功能


在与Sanjay Punen发生另一个用户案例之后，Pat Gelsinger返回舞台，讨论与Microsoft解决方案的新集成选项。 通过两家公司的合作，他们可以宣布新的解决方案和产品更新，以加速公司客户的数字化转型。 除其他外，强调了以下举措：

  • 使用VMware Workspace ONE和Microsoft Endpoint Manager加快Windows 10现代化管理的新解决方案；
  • 使用VMware HCX通过新的增强型云迁移功能进一步扩展Azure VMware解决方案混合云服务的全球范围；
  • 客户端数据中心在VMware vSphere中对Microsoft SQL Server 2019的认证和支持
  • 使用VeloCloud的VMo SD-WAN进行Azure外围设备扩展。

两家公司都致力于进行富有成效的协作，这将使Azure VMware解决方案在全球范围内可用。 帕特·基辛格（Pat Gelsinger）承诺将增加提供这些解决方案的区域数量-特别是日本，东南亚和澳大利亚。

Dell EMC上的VMware Cloud


戴尔EMC上的VMware Cloud宣布以及在巴塞罗那发布的其他一些宣布，很难被称为“热门”-它已于今年4月初宣布，但我们无法绕开这一决定。 该解决方案结合了公共云的灵活性和保护本地基础架构的能力。 戴尔EMC上的VMware Cloud是一项服务，客户可以在其中获得用于本地数据中心和客户网络外围的便捷，安全和可扩展的基础架构。 该系统基于“云数据中心即服务”解决方案，可以在客户自己的设备上运行，为客户提供所有云基础架构功能即服务。 对于那些由于某些原因（例如合法）无法将其IT系统发送到云的公司和组织而言，这一优势可能至关重要。 这种独特的方法使专注于创新和获得竞争优势成为可能。



5G和大师计划


帕特·基辛格（Pat Gelsinger）演讲的一部分专门讨论了一个话题，这个话题已经在世界范围内讨论了一个多月，即5G。



作为主题的一部分，该公司对Project Maestro（一种新的VMware解决方案）进行了初步评估，其目的是帮助提供商缩短现代云和网络解决方案的上市时间。 Project Maestro提供了一种用于建模，组织和管理虚拟网络的统一方法。 借助其帮助，通信服务提供商将能够创建和自动化网络服务，从而在NFV和电信云架构的每个级别上提供交互和优化操作。 通过自动化任何网络和任何云中的生命周期管理。
