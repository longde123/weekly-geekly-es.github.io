让我们活到星期一或如何在黑色星期五生存

明天是“黑色星期五”-对于Internet项目，这意味着网站上的负载将达到峰值。 甚至巨头也无法忍受它们，例如2017年Prime Day亚马逊发生的事情 。



我们决定给出一些使用虚拟服务器的简单示例，以避免出现错误并且与503页或更糟糕的人见面：About（空白）和ERR_CONNECTION_TIMED_OUT。 保留一天的准备时间。

资源扩展


网站通常由不同的模块组成-数据库，Web服务器，缓存系统。 每个模块都需要不同类型和数量的资源。 有必要使用压力测试预先分析消耗的资源量，并评估磁盘I / O的速度，处理器时间，内存以及站点Internet地带的宽度。

压力测试可以帮助您确定系统中的瓶颈并提前进行扩展。 因此，例如，您可以通过在操作期间增加硬盘驱动器上的空间，扩大站点的范围或增加虚拟服务器的RAM来提高服务器的功能。 完成此操作后，您可以按原样返回所有内容，无需您联系技术支持即可在您的个人帐户中完成，这需要花费几分钟，但是最好提前进行此操作，并且在网站上最少的客户活动期间进行。

预先保护自己免受DDoS攻击


销售日的站点数量下降不仅是由于客户流量的增加，还因为DDoS攻击。 它们可以由希望将您的流量重定向到其网络钓鱼资源的攻击者进行组织。

DDoS攻击每天都在变得越来越复杂。 黑客使用不同的方法，同时利用DDoS攻击和对应用程序漏洞的攻击。 在大多数情况下，攻击伴随着对网站的黑客攻击。

事先准备好并将受攻击保护的IP地址连接到服务器也很重要。 在UltraVDS中，我们不是在遭受攻击后保护服务器，而是可以承受高达1.5 Tbps的全天候攻击！ 为了保护服务器免受DDoS攻击，使用了一系列筛选器，这些筛选器以足够大的带宽连接到Internet通道。 筛选器顺序分析通过的流量，揭示异常和非标准网络活动。 分析的非标准流量模式包括所有当前已知的攻击方法，包括使用分布式僵尸网络实施的攻击方法。

要将安全地址连接到虚拟服务器，您必须事先向提供商的支持服务发送请求。

加快网站加载


在活动期间，服务器上的负载增加，在网站上照片和产品卡开始长时间加载。 同样，页面的加载由各种框架，JS库，CSS模块等来加权。 潜在的客户可以离开页面而不会收到站点的响应，即使在报价比竞争对手更好的情况下也是如此。 要检查页面加载速度，建议使用Google DevTools。

内容交付网络（CDN）可以帮助加快页面加载速度。 CDN是一个地理分布的网络，由缓存节点（存在点）组成，它们可以位于世界各地。 进入站点时，客户端将不会从您的服务器而是从进入CDN网络且距离它更近的服务器接收静态内容。 通过减少服务器和客户端之间的路由，站点上的数据加载速度更快。

如果您在Windows Server Core 2019上具有VDS，则可以自己配置CDN网络，为此，请使用操作系统内置的工具，例如Active Directory，DFS，IIS，WinAcme，RSAT。 您还可以使用现成的解决方案，例如Cloudflare的CDN可以更快，更便宜地解决问题。 此外，该系统还具有其他功能：DNS，HTML，CSS，JS压缩，许多存在点。
