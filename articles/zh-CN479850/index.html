在Windows Server Core 2019上安装Exchange 2019

Microsoft Exchange是一个大型处理器，它包括接收和处理信件以及邮件服务器的Web界面，对公司日历和任务的访问。 Exchange已集成到Active Directory中，因此可以想象它已经被部署。

好吧，Windows Server 2019 Core是没有图形界面的Windows Server版本。

此版本的Windows中没有传统的Windows，无需单击任何内容，也没有“开始”菜单。 只有黑色的窗口和黑色的命令行。 但是同时，由于我们不想让任何人深入研究关键系统，因此攻击区域更小，进入级别也有所增加，对吗？

本指南也适用于具有GUI的服务器。


1.连接到服务器


打开Powershell并输入命令： 

Enter-PSSession 172.18.105.6 -Credential Administrator

可选：打开RDP。 这简化了安装，但根本没有。 

 cscript C:\Windows\System32\Scregedit.wsf /ar 0

在Ultravds的映像中，已经包含了RDP。

2.将服务器连接到AD


这可以通过Windows管理中心或RDP中的Sconfig完成。

2.1指定DNS服务器或域控制器



在Windows管理中心中，我们连接到服务器，转到“网络”部分，然后指定域控制器或DNS域服务器的IP地址。


通过RDP，在命令行中输入“ Sconfig”并进入蓝色服务器配置窗口。 在这里，我们选择点8）网络设置，然后通过指定DNS域服务器来执行相同的操作。

2.2我们将服务器加入域



在WAC中，单击“更改计算机ID”，我们将看到一个熟悉的窗口，用于选择工作组或域。 一切照常，选择一个域并加入。


使用Sconfig，必须首先选择第1项，选择是否要加入工作组或域，如果要加入域，请指定域。 并且只有在完成该过程之后，我们才可以更改服务器名称，但是即使如此，您也需要再次输入密码。

Powershell使这变得更加容易： 

 Add-Computer -DomainName test.domain -NewName exchange -DomainCredential Administrator

3.安装




如果使用RDP，则必须在安装Exchange本身之前安装必要的组件。 

 Install-WindowsFeature Server-Media-Foundation, RSAT-ADDS

接下来，我们需要使用Exchange安装程序下载磁盘映像。 

 Invoke-WebRequest -UseBasicParsing -Uri 'https://website.com/ ExchangeServer2019-x64.iso -OutFile C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

挂载ISO： 

 Mount-DiskImage C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

如果要通过命令行全部执行此操作，则只需挂载下载的磁盘并输入以下命令： 

 D:\Setup.exe /m:install /roles:m /IAcceptExchangeServerLicenseTerms /InstallWindowsComponents

结论


如您所见，在Windows Server Core上安装Exchange以及进入域并不是一个痛苦的过程，考虑到我们如何赢得安全性，这是值得的。

我特别高兴的是，使用Powershell将服务器输入到AD中比使用GUI或Windows管理中心更容易。

遗憾的是，仅为Exchange 2019添加了Exchange安装选项，时间太长了。
