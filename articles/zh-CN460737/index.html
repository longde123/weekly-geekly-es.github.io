听背景：项目管理播客

世界上的播客正变得越来越多样化。 广播主题从小工具到学术类不一而足，但是我们决定选择在IT协会每天遇到的主题播客。 它们是ITSM，ITIL，DevOps和IT项目管理。


/不飞溅/ 杰森·罗斯威尔


CIO Talk网络播客


[ iTunes ] [ SoundCloud ]

播客的历史可以追溯到2003年。 然后它被称为“管理技术：正确的方法！” 并进行了现场直播 IT咨询师兼企业家Sanjog Aul一直是并将继续保持其领导地位，该公司被《多样性MBA》评为“ 50岁以下百强领导者”之一。 今天，播客是对《财富》 500强和《财富》 1000强大公司高管的一系列采访。

他们在说什么：没有专门的专业知识-他们在谈论大数据，云技术和物联网。 但是，通常会解决IT管理主题。 例如，在最新一期中，它涉及使用ITSM方法提高客户服务质量。 在另一场会议中， 与会人员讨论了人工智能系统在技术支持方面的前景。 人们认为 ，播客非常适合作为这些领域中陌生主题的“初学者”。

出现的频率：新剧集大约每十天出现一次。

时长： 40-60分钟。


DevOps电台


[ iTunes ]

展会的主持人是DevOps传播者Brian Dawson，以及CloudBees的营销副总裁Andre Pino。 该公司从事持续交付工具。

他们在说什么：一系列播客，专门介绍与软件开发和部署相关的所有内容：从DevOps到Docker。 来宾中有来自博世 ， Forrester ， Electronic Arts和许多其他公司的专家。 在展会的整个过程中，演讲者都是IT行业的众多标志性人物。

我们与乐器作者川口浩辅（Kohsuke Kawaguchi）讨论了詹金斯的历史（ 在这里和这里 ）。 我们与Jenkins X开发人员讨论了出现这种基于Kubernetes的系统的原因以及其哪些功能对DevOps工程师有用。 我们还就“大规模”进行DevOps转型采访了资本控股银行（Capital One）的代表。

多久发布一次：每两周发布一次新版本。

时长： 30-50分钟。


粉红象


[ iTunes ] [ Player.fm ]

由其两名副总裁主持的粉红大象播客。 自1980年以来，该组织本身就一直从事IT管理领域的培训和专业发展。

他们正在谈论的内容：问题主要集中在DevOps，敏捷和ITSM最佳实践上。 例如， 其中一位研究了看板方法，以及如何在IT服务管理领域中使用它。 换句话说 ，什么人工智能系统可以帮助资产管理。 传播版本被有趣的实际案例和幽默所稀释。

发布频率：直到2016年是每月一次，现在发布频率则大大降低。

时长： 30分钟。


ServiceNow TechBytes


[ iTunes ] [ SoundCloud ]

多功能ITSM自动化平台ServiceNow的播客。 Gartner多次称她为该领域的领导者。 该程序已于2015年发布。

他们在说什么： ServiceNow专家和工程师在讨论平台功能，新版本并提供实际案例。 例如，在其中一个问题中， 他们谈到了平台与IBM Watson超级计算机的配合。 另一个问题涉及组织技术支持的Walk-Up方法 -当用户或客户可以去找专家并亲自讨论他们的问题时。 这种方法的一个示例是Apple Genius Bars 。

出现的频率：每月两次到三次。

时长： 20-30分钟。


ITAM评论播客


[ iTunes ] [ SoundCloud ]

成立于2008年的门户ITAM Review的音频广播。 这是一个私人平台，可以对 ITSM解决方案和服务提供商进行独立评估。


/不溅水/ 尼尔色

他们在说什么：参与者讨论ITSM新闻。 关于新的SAP解决方案以及ITAM中的云技术前景存在问题。 还有一些实用指南，例如，降低IT基础架构成本的指南 。

发出频率：一个月几次。

时长： 30分钟至一个小时。


5'PM播客


[ iTunes ] [ SoundCloud ]

播客的作者是Ricardo Vargas。 自2007年以来，他一直在发布他的程序。 里卡多（Ricardo）从事天然气，能源和电信领域的大型公司的业务流程转型已有20年。 他还撰写了许多有关项目管理的书籍，在全球范围内分发了超过200,000册。 他的最新作品之一“分析层次结构过程，挣值和其他项目管理主题”可以在官方网站上免费下载 。

播客的所有剧集都很短-平均为5分钟（因此节目的名称）。 但是听众说 ，尽管框架很“严格”，但里卡多还是设法为PM-s提供了一些全面而有用的技巧。

他谈论的是：这里有两个主题：如果开发团队没有时间在截止日期之前完成项目该怎么 办 ，以及如何在混乱的环境中进行项目。 里卡多还回答了诸如“ PM应该为他人做的工作吗？”之类的问题。 “并且” 年龄对PM重要吗？ ”。 您还可以从里卡多（Ricardo）在TED大会上的演讲列表中找到里卡多（Ricardo）的演讲录音，其中涵盖了从艺术到机器学习的各种主题。

发出频率：这是每月一次的传送。

时长： 5分钟。


软件工程学院播客


[ iTunes ]

演讲者是卡内基梅隆大学的CERT 。

他们说的是：主要方向是网络安全。 因此，关于DevOps和ITSM的大多数讨论都以某种方式与IT基础架构的保护有关。 在一个问题中，专家们讨论了如何在医药和金融等高度管制的行业中建立IT管理流程。

有一些保护基础架构的实用建议。 专家分析了去年的历史，当时美国和其他国家/地区的警察逮捕了 74起涉嫌破坏大公司的公司电子邮件的案件。 除了历史信息外，该问题还提供了有关如何保护组织免受此类攻击的建议。

出版频率：每月两到三期。

时长： 30–40分钟。
