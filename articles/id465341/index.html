Cara membuat cloud pribadi untuk pengawasan video



8 tahun yang lalu, cloud dengan cepat mengubah dunia - iklan mulai berlomba-lomba menjanjikan untuk menghubungkan bisnis apa pun dengan server jarak jauh, penyimpanan di pusat data dan sumber daya komputasi terdistribusi. Saat ini, perusahaan besar tidak hanya dapat bermigrasi ke server virtual, tetapi juga menjalankan cloud pribadi di batas keamanan mereka.

Ivideon menyebarkan dan mengonfigurasi sistem pengawasan video di mana saja, secara efektif menyediakan "layanan cloud pribadi turnkey" kepada pelanggan. Dan bayangkan: beberapa solusi yang pelanggan anggap sebagai pesaing kami sebenarnya adalah Ivideon dengan merek yang berbeda ... Hari ini Anda akan belajar bagaimana semuanya bekerja dan bagaimana hal itu menguntungkan bisnis.

Teknologi Cloud Baru




Semua teknologi cloud modern berasal dari abad terakhir. Downtime dari sistem mahal dan kuat pertama, menempati beberapa kamar dan membutuhkan energi besar dan sumber daya manusia untuk bekerja, terpukul dengan anggaran terbatas. Saat itulah praktik mulai menyewakan "waktu komputer" dan ruang disk. Untuk ini, terminal jarak jauh digunakan, dan tidak berlebihan untuk mengatakan bahwa pengguna mereka telah bekerja dengan layanan cloud.

Istilah "solusi cloud" hari ini mengacu pada berbagai layanan penyimpanan jarak jauh dan pemrosesan data yang diberikan kepada pengguna sebagai layanan Internet. Mereka dapat diklasifikasikan sebagai berikut: publik, pribadi dan hibrida. Ivideon menggunakan semua model. Anda sudah tahu segalanya tentang cloud biasa (kami harap) - mari kita memikirkan solusi "eksotis": Private Cloud (PC), White Label dan Enterprise Cloud (EC).

Ivideon cloud pribadi


Produk Cloud Private Cloud dipilih oleh perusahaan dan bank. Ada banyak perusahaan yang, karena berbagai alasan (persyaratan keamanan internal, ketersediaan pusat data pribadi), ingin sepenuhnya mentransfer fungsi Ivideon ke fasilitas mereka sendiri.

Solusi PC hanyalah versi produk Ivideon di mana semua server (kontrol, arsip, dan streaming) berada di fasilitas pelanggan. Solusi ini dimaksudkan hanya untuk penggunaan internal oleh pelanggan akhir.

Untuk Ivideon Private Cloud, kami telah mengidentifikasi dua model koneksi:

  • permintaan khusus untuk perangkat lunak, yaitu, pelanggan melakukan instalasi, instalasi dan pemeliharaan komponen yang tersisa dari sistem (kamera, server, infrastruktur switching);
  • perangkat keras dan perangkat lunak yang lengkap, turnkey turnkey sistem pengawasan video Pelanggan membeli sistem yang sudah selesai, yang meliputi perangkat lunak, perangkat keras server, kamera, serta instalasi, konfigurasi, dan pekerjaan administrasi.

Private Cloud adalah otomatisasi tingkat tertinggi yang dengannya bisnis tidak bergantung pada penyedia cloud, tidak dibatasi oleh lebar saluran Internet, tidak membayar untuk lalu lintas, memantau proses enkripsi itu sendiri dan mencapai otonomi penuh dalam bunker rahasia.

Berkat cloud pribadi, Anda juga bisa mendapatkan beberapa fitur berguna:

  • penyimpanan data di server Anda sendiri atau di pusat data Anda;
  • kemampuan untuk menyalin dan mengembalikan konfigurasi dengan cepat;
  • backup data multi-level, sistem toleran-kesalahan yang stabil.

Formula aplikasi cloud pribadi sebenarnya tidak terbatas. Ada perusahaan yang menggunakan PC untuk mengontrol "anak perempuan" mereka dan cabang - lusinan, ratusan dan ribuan objek beroperasi dalam jaringan tertutup raksasa. Pemilik PC mengelola semua objek pemantauan secara terpusat dan dapat menambahkan jumlah kamera dan pengguna yang tidak terbatas ke sistem.

Tentu saja, cloud pribadi lebih sulit dikelola - layanan TI lokal menangani pembaruan yang aman dari semua server jaringan dengan tambalan penting. Nasib layanan tergantung pada profesionalisme mereka - untuk bertanya, dalam hal ini, Anda harus melakukannya sendiri. Namun, minus ini diratakan - Ivideon dapat menyediakan layanan tersebut tambahan.

Program Afiliasi White Label


Program afiliasi cloud ditawarkan kepada mereka yang menginginkan logo dan warna korporat mereka muncul di akun pribadi dan aplikasi seluler mereka - ini dapat berupa operator telekomunikasi, penyedia Internet, penyedia sistem keamanan, dan perusahaan lain.

Sebagai bagian dari program, mitra kami menyediakan layanan pengawasan video berbasis cloud sendiri, baik menggunakan produk Ivideon yang sudah selesai maupun menciptakan kondisi mereka sendiri untuk pelanggan. Mitra membentuk tarif individu dan grup untuk pelanggan mereka, dan mempertahankan layanan menggunakan sumber daya teknis mereka sendiri.

Pada WL, Anda dapat mengontrol aliran video secara adaptif untuk mengurangi beban pada saluran operator, dan menggunakan tiga cadangan data dan saluran komunikasi untuk memastikan tingkat keandalan yang tinggi.

Sebuah sistem di bawah merek sendiri adalah solusi turnkey di mana Ivideon menyediakan perangkat lunak, shell layanan bermerek, peralatan, dan materi pemasaran. Selanjutnya, penyedia, operator atau penyedia layanan Internet yang tidak pernah terlibat dalam pengawasan video, itu sendiri menjadi penyedia layanan cloud.

Solusi utama Ivideon tidak berbeda dengan WL dalam hal kualitas layanan, tetapi pemilik label dapat menawarkan hampir semua tarif kepada pelanggan, termasuk layanan yang tidak kami miliki dan tidak akan pernah miliki. Model interaksi fleksibel ini membantu memberi akses ke pengawasan video berbasis cloud bahkan bagi mereka yang belum pernah tertarik pada Ivideon.

Cloud perusahaan


Produk server yang jarang digunakan di mana Anda dapat meng-host server penyimpanan dan streaming secara lokal. Pada saat yang sama, server manajemen yang termasuk dalam UE tetap berada di pihak kami.

Di UE, server berikut digunakan di fasilitas klien:

  • server penyimpanan - jenis server Cloud Pribadi yang menerima video dari kamera yang terhubung, menyiarkan video ini ke klien dan merekamnya di arsip;
  • streaming server - jenis server Cloud Pribadi yang menerima video dari kamera yang terhubung, menyiarkan video ini ke klien dan tidak merekamnya dalam arsip.


Besi Awan Pribadi




Pengguna Ivideon biasa hanya perlu mengunduh aplikasi seluler atau menginstal Ivideon Server atau Klien di hampir semua PC, tetapi pemilik cloud-nya sendiri, selain perangkat lunak kami, akan membutuhkan sumber daya perangkat keras yang serius. Ini adalah konfigurasi khas server penyimpanan yang melayani 80 kamera dengan bitrate 1 Mbit / dtk, mengatur pergerakan 30% dalam bingkai dan dengan kedalaman arsip 30 hari:

Prosesor: Intel Xeon E1230v5
Memori: 16 GB
Drive: empat 4TB SATA 7200
Jaringan: 100 Mbps.
Ukuran: 1U

Contoh konfigurasi perangkat keras untuk server streaming untuk 2.000 kamera dan 100-500 tayangan video aktif:

Prosesor: Intel Xeon E1230v5
Memori: 32 GB
Drive: dua 1TB SATA 7200
Jaringan: 1000 Mbps.
Ukuran: 1U

Perhatikan bahwa lalu lintas antara server penyimpanan, streaming, dan manajemen sekitar 5% dari stream video dari kamera.

Agar Private Cloud dapat bekerja dengan benar, setidaknya tiga server manajemen tambahan harus ditempatkan di situs pelanggan. Bahkan jika Anda hanya menggunakan satu kamera, Anda memerlukan tiga server manajemen.

Konfigurasi khas server manajemen:

Ukuran: 1 / 2U dengan "keranjang" 8 rongga disk
Prosesor: dua 12-core (mis. E5-2650v4)
Memori: 128 GB (64 GB per prosesor)

Disk:

  • empat SATA 6G eMLC NAND Enterprise SSD masing-masing 480 GB (raid-10 untuk database);
  • dua SATA / SAS masing-masing 2 TB (untuk data yang tidak penting, seperti log dan cadangan);
  • Pengontrol RAID: perangkat keras yang mendukung RAID-10.

Jaringan: 1 Gbps.

Sekelompok tiga server seperti itu mampu melayani 10 ribu pelanggan. Jika ini tidak cukup, maka Anda harus menambahkan tiga server serupa yang Anda dapat mentransfer database dalam arsip. Dalam hal ini, mudah untuk melayani 20-30 ribu pelanggan. Diperbolehkan untuk menambah jumlah server lebih lanjut - skala sistem tergantung pada tugas.

Untuk memastikan toleransi kesalahan layanan, jumlah server harus digandakan. Pelestarian catatan dipastikan dengan duplikasi di tingkat server arsip.

Jika Anda perlu mengakses rekaman dari kamera melalui portal web, maka untuk pekerjaannya Anda juga akan memerlukan server terpisah. Muatan di dalamnya akan bervariasi tergantung pada jenis panggilan - menonton video di browser atau meminta API.

Kami merekomendasikan konfigurasi server web berikut:

  • prosesor dual core;
  • 8 GB RAM = 30 rps (yang paling gemuk);
  • SSD mulai dari 400 GB;
  • LAN 100 Mbps. atau 1 Gbit / s.

Dunia satu awan


Mungkin di masa depan semua orang akan bergabung dalam satu cloud virtual (di server Amazon), tetapi dunia tidak akan mengetahuinya, karena jumlah penyedia layanan cloud tidak akan berkurang, tetapi bahkan bertambah. Fitur khas Private Cloud, Enterprise Cloud, dan White Label adalah privasi. Menurut NDA, kami tidak dapat menyebutkan nama klien dan mengungkapkan konfigurasi server mereka.

Pemilik cloud pribadi, setelah menarik garis yang tidak terlihat antara mereka dan merek kami, dapat melakukan apa saja :) sebagai bagian dari kebaikan bersama, tentu saja. Desain Anda, tarif Anda, pola penggunaan Anda.

Ada beberapa batasan: Anda tidak dapat mengubah arsitektur dasar layanan, setelah melakukan pekerjaan yang salah di mana ada peluang untuk mendapatkan labu alih-alih gerbong; Dilarang menyalin kode sumber dan mencoba menerapkan kode di tempat lain selain cloud kami.

Untuk menyelesaikan tugas-tugas sederhana di tingkat "jangan sentuh", kami menarik dukungan teknis, tetapi seringkali ini tidak cukup, oleh karena itu, dalam kerja sama yang erat dengan klien cloud pribadi, tim pengembangan beroperasi. Beberapa fungsi yang kemudian diperkenalkan ke layanan umum, pernah muncul sebagai add-on yang dikembangkan atas permintaan pelanggan tersebut.

Kontak untuk kontak jika Anda ingin mempelajari lebih lanjut tentang ketentuan untuk menghubungkan Cloud Privat dan peluang bisnis Ivideon lainnya.
