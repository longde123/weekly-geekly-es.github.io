Analytics für Azure DevOps Services ist jetzt allgemein verfügbar

Die Berichterstellung war eine wichtige Funktion für Azure DevOps-Kunden, die sich bei datengesteuerten Entscheidungen auf Analytics verlassen.


Wir freuen uns, Ihnen heute mitteilen zu können, dass die folgenden unten aufgeführten Analytics-Funktionen ohne zusätzliche Kosten in unser Azure DevOps Services- Angebot aufgenommen werden. Kunden werden diese Änderungen bald auf ihren Konten sehen.




Analytics-Funktionen, die allgemein für Azure DevOps-Dienste verfügbar sind


  • Analytics Widgets - konfigurierbare Module, die Daten in einem Dashboard anzeigen und Ihnen helfen, den Fortschritt Ihrer Arbeit zu überwachen.


    • Burndown und Burnup - Überwachen Sie den Fortschritt einer Reihe von Arbeiten über einen bestimmten Zeitraum.


    • Zykluszeit und Vorlaufzeit - Visualisieren Sie, wie sich die Arbeit durch den Entwicklungszyklus Ihres Teams bewegt.


    • Kumulatives Flussdiagramm (CFD) - Verfolgen Sie Arbeitselemente, während sie verschiedene Zustände durchlaufen.


    • Geschwindigkeit - Verfolgen Sie, wie ein Team über mehrere Sprints hinweg Wert liefert.


    • Testergebnis-Trend - Überwachen Sie Testtrends, erkennen Sie Fehler- und Dauer-Muster für Tests über einzelne oder mehrere Pipelines.


  • In Produkterfahrungen - Analytics-basierte Erlebnisse in Azure DevOps und außerhalb eines Dashboards, in dem Daten und Erkenntnisse angezeigt werden.


    • Top Failing Test Report - Erhalten Sie Einblicke in Top Failing Tests in Ihrer Pipeline, um die Zuverlässigkeit der Pipeline zu verbessern und die Testverschuldung zu reduzieren.



Wir werden weiterhin Power BI-Integration über Analytics Views und direkten Zugriff auf unseren OData-Endpunkt in der Vorschau für alle Azure DevOps Services-Kunden anbieten. Weitere Informationen zum Preismodell für die Power BI-Integration und OData bis Juni 2019 finden Sie hier.


Aktuelle Azure DevOps Services-Kunden, auf denen die Analytics-Marktplatzerweiterung installiert ist, können Analytics weiterhin wie zuvor verwenden und müssen keine zusätzlichen Schritte ausführen, um Analytics abzurufen. Aus diesem Grund werden wir die Erweiterung des Analytics-Marktplatzes für gehostete Kunden ablehnen.


Azure DevOps Server 2019


Für Azure DevOps Server bleibt Analytics als installierbare Erweiterung auf dem lokalen Marktplatz in der Vorschau und wird in der nächsten Hauptversion allgemein verfügbar sein.


Das Azure DevOps Analytics-Angebot ist die Zukunft der Berichterstellung. Wir werden weiterhin in neue Funktionen investieren, die von Analytics gesteuert werden. Um mehr über Analytics und die Erfahrungen zu erfahren, die es derzeit ermöglicht:

  • Informieren Sie sich über die wichtigsten Funktionen: Analytics-Widgets , Top Failing-Testbericht , Power BI-Integration und OData-Endpunkt .
