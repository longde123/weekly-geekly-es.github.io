Übersicht über Update 4 für Veeam Cloud Connect

Das neue Jahr begann mit neuen Produkten von Veeam. Letzte Woche wurde Update 4 für Veeam Cloud Connect für Dienstanbieter verfügbar. Ich möchte Sie daran erinnern, dass Veeam Cloud Connect dabei hilft, die Replikation virtueller Maschinen zu konfigurieren, Replikate in der Cloud eines Dienstanbieters zu starten und ein Remote-Repository für Sicherungen auf der Site des Anbieters zu organisieren. Ich habe hier bereits über dieses Tool geschrieben. Heute zeige ich Ihnen, was es Neues gibt.

Hier sind die wichtigsten Änderungen:

  • Bindung von Cloud Gateway an bestimmte Clients;
  • Replikation auf VMware vCloud Director. Bisher konnten Sie nur Replikate auf vCenter-Ebene erstellen. Clients konnten ohne Failover keine Verbindung zum Replikat der virtuellen Maschine über die Konsole herstellen.
  • Backup auf Bändern mit Client-Bindung an ein bestimmtes Band.

Details unter dem Schnitt.





Cloud-Gateway-Pool


Jetzt können dem Client separate Pools von Cloud-Gateways zugewiesen werden (Cloud Gateway Pool). Wenn der Client bestimmte Kanalanforderungen für Replikationsaufgaben hat (z. B. Kanalverschlüsselung gemäß GOST oder garantierte Geschwindigkeit), kann dies im neuen Veeam Cloud Connect arrangiert werden. Zu diesem Zweck wird ein Cloud-Pool für den Client erstellt. Es kann mehrere geben - für Fehlertoleranz oder Lastausgleich. Ich werde ein Beispiel zeigen.



Erstellen Sie als Nächstes einen Gateway-Pool und fügen Sie ihm die erforderlichen Cloud-Gateways hinzu.









Wenn der Gateway-Pool erstellt wird, weisen Sie ihn dem entsprechenden Client zu. Es gibt eine Failover-Option für andere Cloud-Gateways, wenn mit dem dedizierten Pool etwas passiert.



Replikation in vCloud Director


Jetzt können Sie die Replikation virtueller Maschinen nicht nur für die VMware-Infrastruktur des Cloud-Anbieters, sondern auch für die ausgewählte vCloud Director-Organisation konfigurieren. Die Versionen 8.10, 8.20, 9.0, 9.0.0.1, 9.0.0.2 und auch 9.1 werden unterstützt. Beachten Sie, dass die Hardwareversion der replizierten virtuellen Maschine nicht höher sein sollte als die des Anbieters. So wird es in der Praxis aussehen.

In Cloud Connect erstellen wir ein Kundenprofil basierend auf dem Konto in vCloud Director.



Wir wählen die Organisation aus, in der Replikate hinzugefügt werden sollen.



Wir wählen Gateway Pool für den angegebenen Client.



Auf der Registerkarte Replikatressourcen wird die ausgewählte Organisation in der Cloud angezeigt. Klicken Sie auf Übernehmen, und der Client kann eine Verbindung zu Cloud Connect herstellen.



Für die Netzwerkkonnektivität zwischen Ressourcen in der Cloud des Dienstanbieters und seiner Infrastruktur wird die Verwendung der Veeam-Netzwerkerweiterung vorgeschlagen. Damit können Sie teilweise (einzelne Computer aus Replikaten wiederherstellen) und das Failover abschließen (den gesamten Standort wiederherstellen). Sie können die NSX Edge-Funktionalität für diese Zwecke verwenden. Es ist viel bequemer.

Nun wollen wir sehen, wie die Konfiguration der Replikation auf der Clientseite aussieht.

Um eine Verbindung zu Cloud Connect in der Veeam Backup and Replication-Konsole herzustellen, wählt der Client einen Dienstanbieter aus.





Der Client meldet sich mit dem vCloud Director-Konto bei Cloud Connect an, zu dem in Zukunft Replikate hinzugefügt werden.



Nach der Autorisierung sieht der Client die für das Replikat zugewiesenen Ressourcen: Name der Organisation, CPU, Speicher, Speicher usw.).



Als Nächstes können Sie Replikationsjobs erstellen. Hier befindet sich wie in früheren Versionen alles auf der Registerkarte Ziel.





Auf der Registerkarte Ziel wählt der Client die vApp aus, auf die die virtuelle Maschine repliziert werden soll, sowie die Speicherrichtlinie.



Dann wird die Aufgabe wie in der vorherigen Version konfiguriert. So sieht der fertige Replikationsjob aus.



Und dies sind fertige Replikate in der Cloud des Dienstanbieters.



Auf alle Replikate kann über das vCloud Director-Kontrollfeld zugegriffen werden. Jede virtuelle Maschine kann ein- und ausgeschaltet, getestet, geklont und alles, was mit normalen virtuellen Maschinen möglich ist. Wenn das Thema interessant ist, kann ich Ihnen im nächsten Beitrag mehr über das Einrichten dieser ganzen Sache erzählen.





Aufzeichnung auf Band aufnehmen


Mit Update 4 wurde die Möglichkeit hinzugefügt, einem bestimmten Client bestimmte Bänder zuzuweisen. Zuvor musste ich mich im Repository umsehen, um zu verstehen, was und wo es ist. Dies ist mehr für das Sparschwein der Annehmlichkeiten für den Anbieter.





Für die breite Öffentlichkeit wird Update 4 am 22. Januar geöffnet sein. In der CloudLine- Cloud können Sie einen Testressourcenpool erstellen, um die neuen Funktionen von Veeam Cloud Connect auszuprobieren.
