Mozilla将于11月3日发布适用于Windows的64位Firefox

到2016年底，将从Firefox中删除对NPAPI的支持


Mozilla浏览器的下一个版本Firefox 42将以32位应用程序和64位程序的形式提供。仅在FTP服务器上可用的最新版本将利用x86-64架构（AMD64 / Intel 64）。例如，它将能够寻址超过4 GB的内存。当前，在Windows下，只有32位版本的Firefox。对于Linux和Mac OS的用户，很长一段时间以来一直使用64位Firefox。Firefox 42将于今年11月3日发布。

在过去的五年中，Mozilla开始开发64位Firefox的过程，然后将其删除，然后再次开始。自2014年以来，在优先考虑64位版本Firefox for Windows的开发过程之后，输出延迟了数次。与往常一样，由于开发过程是开放的，因此最好从bug跟踪器中获取第一个消息。

11月3日，将发布新版本的Firefox 42，但将不提供安装64位版本的选项。而且，即使在下载页面上也没有链接。大会将仅在FTP服务器上进行测试。 Mozilla发言人表示，预计会出现某种“来自合作伙伴的变更”，因此不会有正式的公告。 64位版本将不支持使用NPAPI的插件。

这里的64位是指具有中央处理器x86-64架构的系统在其上安装了64位OS。 x86-64是32位x86的64位扩展，具有向后兼容性。几乎所有现代个人计算机的处理器都使用x86-64体系结构。 32位操作系统不允许寻址超过4 GB的RAM。此外，该数字包括RAM本身和视频加速器的内存。 64位系统具有更高的限制，例如，Windows 10最多可以使用2 TiB的RAM。 64位操作系统允许您运行32位应用程序。 32位操作系统无法运行64位程序。 64位代码的优点并不仅仅在于可寻址的RAM数量：程序可以更快。

不像谷歌浏览器，它采用了多线程架构，Firefox使用一个单一的过程中，虽然引进的多线程架构的工作发生了。现在，适用于Windows的Firefox发行版是32位，并且包含在一个进程中，在32位系统中，它不能处理超过3 GiB的内容，而在64位系统中，不能解决4 GiB的问题。Windows 有每晚64位版本。 11月3日，Windows将会首次发布64位Firefox的发行版本。

Firefox竞争对手长期以来一直在其武器库中使用64位版本。 Chrome的功能强大，可以超越32位的地址空间并轻松使用计算机的所有资源，已为用户所熟知，并成为玩笑的主题，尽管后来它由 Google进行了优化处理内存。即使基于免费Firefox源代码的项目在这一点上也超越了祖先：Pale Moon和Waterfox长期以来一直在Windows的64位版本中可用。

将于11月3日发布的Firefox 42的Win64版本将不支持NPAPI插件。通常，Mozilla 会删除到明年年底，Firefox将支持NPAPI。这意味着流行的插件（例如Java或Unity Web Player）将无法使用。决定取消NPAPI支持的原因是浏览器本身已本身支持插件功能：在Firefox中，您可以观看视频，这要归功于HTML5和必要的编解码器，并可以使用WebGL玩3D游戏。 NPAPI插件以性能问题（影响移动设备的电池寿命），崩溃和信息安全事件的形式越来越多地显示技术的时代。

Chrome已经删除了对NPAPI插件的支持。对于Flash，Mozilla的开发人员会例外，因为它已成为Web用户日常生活的重要组成部分。大概在某个发展时期，他也将被遣散。 Oracle了解Java的拒绝，并提供了自己的解决方案。 Unity 放弃了在Unity Web Player上的工作。相反，建议使用导出到WebGL。在明年3月发布的Unity 5.4版本中，浏览器插件支持将消失。

发布后（11月3日），适用于Windows的Firefox 42 64位版本将在https://ftp.mozilla.org/pub/mozilla.org/firefox/releases/latest/提供。。Firefox的开发周期包括依次将每晚构建的状态更改为开发人员版本，测试版和发布版本。64位浏览器的测试版已经可用。
