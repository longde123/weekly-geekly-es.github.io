Empfohlenes Update KB3133977 stürzt einige Windows 7 x64-Computer ab



Jeder weiß, welche Anstrengungen Microsoft unternimmt, um die Benutzerbasis auf das neue Windows 10-Betriebssystem zu übertragen. Was es jetzt getan hat, ist absolut unerklärlich.

Kurz gesagt: Microsoft hat eine kleine, aber sehr merkwürdige Änderung an Windows Update unter Windows 7 vorgenommen, die Computer mit Asus-Motherboards zum Absturz bringt. Microsoft bestätigte dann, dass Computer mit diesem Update nicht wirklich gestartet wurden, und empfahl, ein Upgrade auf Windows 10 durchzuführen.

Das Wichtigste zuerst.

Am 15. März hat Microsoft ein neues Update für KB3133977 für Windows 7 eingeführt. Das Update für KB3133977 ist völlig geringfügig. Wenn jemand interessiert ist, wird ein Fehler im Update KB2990184 behoben , der wiederum die Sicherheitsanfälligkeit durch falsche Speicherung des FIPS-Kennworts (Federal Information Processing Standard) für Active Directory im FIPS-Kompatibilitätsmodus beseitigt.

Es wurde sofort klar, dass das Update nicht mit Computern kompatibel ist, die auf Asus-Motherboards mit Secure Boot-Unterstützung in UEFI ausgeführt werden. Unter bestimmten Umständen weigerten sich diese Computer zu booten und gaben eine Fehlermeldung aus (siehe Abbildung oben). Nur wenige haben das Problem betroffen, da dieses Update für Windows 7 optional war und nur ein kleiner Teil der Benutzer es installiert hat. Asus war sich der

Inkompatibilität des Patches bewusst , und Microsoft wusste es auch . Besitzer von Asus-Motherboards mussten nichts tun - installieren Sie dieses Update einfach nicht. Im Allgemeinen nichts Besonderes. Aus irgendeinem Grund hat Microsoft am 12. April den Status des KB3133977-Updates von "Optional" in "Empfohlen" geändert. Gleichzeitig habe das Unternehmen im Voraus verstanden, zu welchen Problemen dies auf Computern mit Asus-Motherboards führen würde, schreibt



Forbes

Asus ist einer der weltweit größten Hersteller von Computerkomponenten. Asus-Motherboards sind auf vielen Computern installiert, und in den letzten Tagen ist eine Reihe von Beschwerden von Benutzern heruntergekommen .

Asus veröffentlichte eine FAQ, in der Schritt für Schritt gezeigt wurde, wie UEFI Secure Boot im BIOS deaktiviert wird.





Nun, Microsoft auf der Support-Website erklärte, dass der Fehler durch Asus-Motherboards verursacht wurde, die Secure Boot standardmäßig aktivieren, obwohl Windows 7 dies nicht unterstützt.

Microsoft fügte hinzu, dass Secure Boot unter Windows 10 unterstützt wird, und hat einen Link zu einer Seite mit Informationen zum Upgrade von Windows 7 auf Windows 10 veröffentlicht .

Was kann ich sagen Angenommen, Microsoft wollte solche Probleme nicht absichtlich für Millionen von Benutzern erstellen. Aber wenn Probleme sogar durch die Schuld eines anderen entstanden sind, ist es irgendwie hässlich, diese Gelegenheit zu nutzen, um für Windows 10 zu werben.
