发布了3CX V16 Update 3和适用于Android的新3CX移动应用

上周，我们完成了大部分工作，并发布了3CX V16 Update 3的最终版本。它包含新的安全技术，与HubSpot CRM的集成模块以及其他有趣的新项目。 让我们依次讨论所有内容。

安全技术


在更新3中，我们集中于系统各个模块中对TLS协议的更完整支持。

  • TLS协议级别-“参数”→“安全”部分中的新参数SSL / SecureSIP传输和加密算法将PBX服务器设置为与TLS v1.2兼容。在Update 3中，默认情况下启用此参数-禁用与TLS v1的兼容性。 0.如果在连接旧版SIP设备时遇到问题，请禁用此选项。
  • 通过TLS连接SIP中继是中继参数-“传输协议” -TLS（传输层安全性）中的新选项。 要通过TLS连接加密的中继，请启用它并将SIP操作员安全证书（.pem）上传到PBX。 通常，您还需要在中继上启用SRTP支持。 之后，PBX和提供商之间的加密通信通道将起作用。



更新了3CX Live Chat＆Talk的小部件


3CX V16 Update 3随附了用于3CX Live Chat＆Talk的小部件的新版本。 它添加了其他选项，例如，设置到Facebook和Twitter帐户的链接。 此外，现在您可以自动生成用于在网站上放置的窗口小部件代码（如果您的网站无法在WordPress CMS上运行）。



如您所见，现在您无需手动创建小部件HTML代码。 它是在“选项”→“与站点/ WordPress集成”部分中生成的。 小部件参数的详细信息在文档中进行了讨论。

HubSpot CRM集成




Update 3引入了与另一个著名的CRM系统HubSpot CRM的集成。 与其他CRM一样，集成支持以下功能：

  • 点击通话-通过3CX Diver从CRM界面直接通话。
  • 打开联系人卡-CRM中的联系人或引导卡会在来电时打开。
  • 交互日志-与客户的所有对话都记录在CRM交互的历史记录中。
  • 如果找不到呼叫者号码，则系统可以在CRM中创建新联系人。

改善用户体验


  • PBX Web服务器的初始化-更新PBX Web服务器的SSL证书（如果您具有3CX颁发的服务器的FQDN）时，nginx服务器不会像以前那样重新启动。 PBX只需下载并初始化新证书。 重要的是，请勿中断当前通话。
  • 自动重新连接-在3CX And​​roid移动应用中，当连接断开时（例如，当用户从Wi-Fi切换到3G / 4G网络时），就会出现重新连接。 仅当安装了最新版本的3CX And​​roid应用程序时，重新连接才有效（请参见下文）。
  • 状态的PUSH通知-现在，您可以分别为每个用户状态启用或禁用PUSH通知。 除了实际的应用程序之外，还可以在3CX管理界面中为用户配置通知。

新的Web客户端功能




  • 群组对话的名称-现在您可以指定群组对话的名称，它将显示给Web客户端以及适用于Android和iOS的应用程序中的所有聊天参与者。
  • 将附件拖放到聊天中-您现在可以将受支持类型的文件拖放到聊天窗口中，这些文件将被发送给其他参与者。
  • 智能手机自动调谐-Web客户端界面中已出现个人QR码，可快速设置3CX移动应用程序。

其他SIP中继参数


  • 备用SIP代理-新选项“备用代理”允许您添加备用SIP服务器（如果VoIP提供商提供了此选项）。 这简化了容错SIP中继的配置，无需额外的备份中继。
  • 改进了DNS的工作-自动检测，传输协议和IP模式参数自动适应VoIP运营商的各种要求，从DNS区域接收信息。
  • 结合3CX桥接器和中继配置-为了简化管理界面，桥接，SIP中继和VoIP网关的配置按钮现在位于一个区域中。

支持新的IP电话


我们增加了对新IP电话的支持（固件自动配置模板）：

  • Intelbras TIP 120/125（用于酒店房间的设备）。
  • Fanvil X1S / X1SP / X3U / X3SG / X4U / X5U / X6U 。
  • 亿联T41U / T43U / T46U / T48U 。

新的3CX And​​roid应用


我们与3CX v16 Update 3一起发布了适用于Android的新3CX应用程序。 它已经针对Android 10进行了优化（还支持Android 7 Nougat，Android 8 Oreo和Android 9 Pie），并且旨在与3CX v16 Update 3和更高版本配合使用。 此应用程序替代了当前的Android客户端。

该应用程序具有一个新的界面，该界面可提供高速和可扩展的功能。 添加了高级功能，例如，根据用户状态的PUSH通知，默认情况下GSM呼叫相对于VoIP呼叫的优先级以及对话的加密。



设计应用程序界面的新方法可确保与最新版本的Android兼容-而不会使设计复杂化。 界面变得可扩展，呼叫控制屏幕包含更多功能，并且状态设置更加轻松。

如上所述，当连接断开时，例如在办公室Wi-Fi和公共4G网络之间切换时，应用程序会自动重新连接电话对话。 这是无缝进行的-您不会注意到任何东西，也不会听到短暂的停顿。

适用于Android的3CX集成了3CX v16服务器中出现的新隧道。 它提供了从应用程序到服务器的语音流量加密。 对话过程中，屏幕上的黄色锁表示对话已加密。


现在，只需单击一下即可设置您的当前状态（可用，丢失等）。 您还可以指定是否要接收PUSH通知。 例如，您可以配置为，当状态为“可用”时，应仅在台式电话上接收呼叫，而不在移动应用程序上接收。

我们简要列出此版本中的其他一些小但重要的改进：

  • 新的聊天菜单-您可以将聊天转移给自己或从界面隐藏。
  • 更快地下载对话和联系历史。
  • 所有传输的附件都存储在设备的“ 3CXPhone \ 3CX”文件夹中。
  • 通过公司名称搜索联系人。
  • GSM呼叫始终优先于VoIP呼叫。
  • 来电时有一个快速断开呼叫（静音）。

如果您使用的是3CX的早期版本，建议升级到v16-它更安全并且具有许多新功能。 如果您具有有效的更新订阅或年度订阅 ，则免费提供更新 。 如果您不打算更新3CX，请在设备上禁用自动更新应用程序 。

如果您使用的是旧版本的Android（Android 7 Nougat之前的版本），或者您不打算从3CX v15.5升级，请使用移动应用程序的先前版本 。 请注意，旧版应用程序“按原样”提供，并且3CX不再支持。

安装更新


在3CX管理界面中，转到“更新”部分，选择“ v16更新3”，然后单击“下载所选内容”或安装发行版：

  • Windows： 3CX v16 Update 3 。
  • Linux： Debian 9 ISO上的3CX v16-在“ 3CX更新频道”屏幕上，选择“稳定”。
  • 新的3CX And​​roid应用程序已在Google Play商店中提供 。
