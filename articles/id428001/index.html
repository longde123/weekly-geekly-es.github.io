Masalah yang diketahui, bug, dan perbaikan pada pembaruan Windows 10 untuk Oktober 2018



Hampir dua minggu telah berlalu sejak Microsoft merilis pembaruan untuk Windows 10 sebagai bagian dari Patch Selasa 2018 Oktober - dan itu sudah lama dua minggu. Set tambalan ini dapat disebut sebagai tambalan terburuk Selasa dari Microsoft dalam waktu yang sangat lama, karena ia menemukan banyak kesalahan yang tidak hanya membuat PC Windows 10 tidak dapat digunakan, tetapi juga menyebabkan (dalam beberapa kasus) untuk mengajukan kehilangan oleh pengguna.

Semuanya sangat menyedihkan bahwa Microsoft harus mengingat pembaruan untuk Windows 10 dari Oktober 2018 untuk mengetahui dengan tepat apa yang sebenarnya menyebabkan masalah, memperbaikinya, dan kemudian memeriksa set yang diperbarui menggunakan Windows Insiders.

Pada artikel ini, kita akan melihat bug terburuk yang terkait dengan pembaruan ini dan solusi untuk memperbaikinya.

Pembaruan Oktober 2018 Menghapus Beberapa File Pengguna


Kesalahan paling menghancurkan dalam paket layanan ini adalah bug yang menghapus data pengguna selama proses pembaruan. Pengguna seperti itu tidak punya pilihan selain mengembalikan data dari cadangan, salinan bayangan, melalui pemulihan sistem atau melalui perangkat lunak pemulihan file.

Itu sampai pada titik bahwa Microsoft merekomendasikan untuk tidak menggunakan komputer yang menderita dari kesalahan ini, sehingga perangkat lunak pemulihan file memiliki peluang yang lebih baik untuk memulihkannya.

Seminggu kemudian, Microsoft mengumumkan bahwa penghapusan file disebabkan oleh fakta bahwa pengaturan Pengalihan Folder Dikenal [mengarahkan folder yang dikenal] diaktifkan di komputer pengguna.

"Sebelum rilis ulang pembaruan Oktober, penyelidikan kami menemukan bahwa sangat sedikit pengguna kehilangan file mereka selama pembaruan paket Oktober," kata pengumuman dari Microsoft. - Ini terjadi jika fungsi Pengalihan Folder Dikenal (KFR) dinyalakan sebelumnya dan pengguna meninggalkan file di folder "lama" yang asli, alih-alih memindahkannya ke yang baru, yang dialihkan. KFR adalah proses mengarahkan ulang folder Windows yang diketahui, termasuk Desktop, Dokumen, Gambar, Screenshot, Video, Rol Kamera, dll. dari lokasi default mereka, c: \ users \ username \, ke lokasi baru. Sebagai bagian dari umpan balik dari paket layanan April sebelumnya, pengguna melaporkan kemunculan salinan kosong “folder yang dikenal” pada perangkat, ”jelas John Cable, Direktur Manajemen Program, Pemeliharaan dan Pengiriman Windows. “Berdasarkan umpan balik pengguna, kami memperkenalkan kode pada pembaruan Oktober yang menghapus salinan kosong folder yang dikenal ini. Perubahan ini, bersama dengan perubahan lain yang memperbarui urutan penerapan pembaruan, menyebabkan fakta bahwa folder asli, "lama" dihapus beserta isinya, dan hanya "baru" yang tidak tersentuh.

Sayangnya, pengguna yang tidak memiliki salinan cadangan atau bayangan untuk memulihkan file hanya akan dapat memulihkannya menggunakan perangkat lunak khusus untuk memulihkan file yang dihapus, seperti Recuva.

Driver keyboard yang tidak kompatibel menjatuhkan komputer HP


Setelah menginstal paket layanan terbaru, pengguna komputer HP menemukan beberapa kegagalan OS dalam bentuk "layar biru kematian", yang menunjukkan kesalahan WDF_VIOLATION. Kegagalan ini disebabkan oleh driver keyboard HP yang tidak kompatibel (versi 11.0.3.1) diinstal dengan pembaruan



Menurut artikel pendamping, Microsoft mendeteksi driver yang menyebabkan masalah dan menghapus instalannya dari Pembaruan Windows. Untuk pengguna yang telah menginstal driver dan belum me-restart komputer, Microsoft telah merilis pembaruan lain yang menghapus driver yang tidak kompatibel.

»Bekerja bersama dengan HP, Microsoft mengidentifikasi driver dari HP yang diketahui tidak kompatibel dengan perangkat HP tertentu ketika menjalankan Windows 10 versi 1803 dan 1809. Pada 11 Oktober, Microsoft menghapus driver dari pembaruan untuk mengurangi jumlah perangkat yang terpengaruh. Kami juga merilis pembaruan KB 4468304, yang menghapus driver yang tidak kompatibel dari perangkat yang membutuhkan reboot. HP secara aktif menangani masalah ini. "

Pengguna yang menginstal driver, reboot komputer, dan dihadapkan dengan crash OS dapat menggunakan instruksi pemulihan .

Masalah audio setelah menginstal pembaruan


Untuk beberapa waktu, Microsoft mengirimi Intel driver audio kepada pengguna secara tidak sengaja, karena itu audio di komputer mereka berhenti bekerja di versi Windows 10 1803 dan 1809. Pengguna yang menderita driver mengalami kekurangan suara di Windows 10, dan sistem operasi memberi pesan "Tidak ada Audio Perangkat Keluaran dipasang ”[perangkat keluaran audio hilang].



Microsoft sudah berhenti mendistribusikan driver ini, tetapi jika Anda menderita ini, Anda dapat menggunakan instruksi pemulihan .

Masalah kecerahan layar


Setelah menginstal pembaruan, beberapa pengguna mengeluh tentang kecerahan layar yang sangat rendah setelah setiap kali restart komputer. Mereka harus mengembalikan levelnya secara manual. Mungkin ini dapat diperbaiki setelah menginstal driver kartu video yang diperbarui, tetapi kebanyakan orang dengan masalah ini mengatakan bahwa itu tidak membantu.

Sayangnya, saat ini masalah ini tidak memiliki solusi yang diketahui.

Aplikasi Microsoft Edge dan UWP tidak dapat terhubung ke Internet

Setelah merilis Windows 10 1809, aplikasi Microsoft Edge dan Microsoft Store memerlukan dukungan TCP / IPv6, yang tanpanya mereka tidak dapat online. Untuk mengaktifkan dukungan, Anda harus:
  • Buka pengaturan.
  • Pilih "Internet dan Jaringan."
  • Pilih "Ubah pengaturan adaptor."
  • Klik kanan pada adaptor yang digunakan untuk mengakses Internet, dan pilih "Properties".
  • Pastikan kotak centang di sebelah Internet Protocol Version 6 (TCP / IPv6) dicentang.
  • Klik OK untuk keluar dari properti.


Setelah itu, Anda dapat memeriksa apakah Edge dan aplikasi online. Jika ini tidak membantu, Anda dapat mengedit registri. Untuk melakukan ini, jalankan regedit.exe dan pergi ke cabang

"HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ Tcpip6 \ Parameter"
Di sana Anda perlu menemukan variabel DisabledComponents dan menetapkan nilainya ke 0.



Perhatian Anda Diperlukan Peringatan Terkait dengan Driver Audio Intel


Akhirnya, jika Anda mencoba menginstal pembaruan terbaru, Anda melihat pemberitahuan "Perhatian Anda diperlukan", di mana driver "Intel Audio Display (intcdaud.sys), KB4465877" ditampilkan, Anda tidak dapat menginstal pembaruan Windows 10 terbaru.



Anda melihat pesan ini karena Microsoft telah mendeteksi ketidakcocokan dengan driver Intel yang dapat menyebabkan masalah dengan pembaruan Oktober. Microsoft mengatakan bahwa jika Anda melihat pesan ini, Anda perlu menekan "kembali" dan menunggu hingga driver yang diperbarui keluar.

Kesalahan saat membongkar arsip Zip


Kesalahan lain dalam memutakhirkan ke Build 1809 mengarah pada fakta bahwa pengguna tidak diperlihatkan konfirmasi menimpa file dari atas saat mengekstraksi file dari arsip zip menggunakan fungsionalitas Windows bawaan. Kesalahan ini berlaku tidak konsisten, karena bagi sebagian pengguna file ditimpa tanpa konfirmasi, dan bagi sebagian orang, tidak ada yang terjadi sama sekali. Bug ini diperbaiki pada pembaruan Windows 10 19H1 terbaru.
