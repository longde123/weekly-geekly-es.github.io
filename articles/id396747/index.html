Hidup akan lebih mudah. Windows 7 dan 8.1 beralih ke model pembaruan kumulatif

Pembaruan kumulatif bulanan pertama akan dirilis pada bulan Oktober




Pada bulan Mei 2016, Microsoft melakukan pekerjaan dengan baik dan menempatkan semua pembaruan untuk sistem operasi Windows 7 ke dalam paket Convenience Rollup tunggal sejak rilis SP1 pada tahun 2011. Paket kumulatif diterbitkan di situs web resmi Katalog Pembaruan Microsoft.

Kemudian pada bulan Mei, perusahaan mengumumkan bahwa pembaruan non-keamanan untuk Windows 7 SP1, Windows 8.1, Windows Server 2008 R2 SP1, Windows Server 2012 dan Windows Server 2012 R2 akan dirilis sebagai satu paket setiap bulan. Sekarang, karena banyaknya permintaan dari pengguna, Microsoft telah memutuskan untuk membuat beberapa perubahan pada prosedur. Diputuskan bahwa pembaruan tidak hanya bulanan, tetapi kumulatif , yaitu, pada setiap saat waktu itu akan tersediaHanya satu dari pembaruan yang paling relevan , yang mencakup semua yang sebelumnya. Ini akan sangat memudahkan kehidupan administrator sistem dan pengguna biasa.

Dengan beralih ke model pembaruan bulanan kumulatif, Microsoft berharap dapat memecahkan beberapa masalah. Yang paling penting dari mereka adalah fragmentasi, ketika semua sistem operasi pada komputer pengguna berbeda satu sama lain, yaitu, set pembaruan yang berbeda diinstal pada semua. Karena kombinasi berbeda dari file sistem yang diinstal, kesalahan ketergantungan dan masalah menginstal pembaruan baru terjadi. Seringkali ada situasi ketika, untuk menginstal satu pembaruan, perlu untuk menginstal yang sebelumnya, dan untuk yang sebelumnya, pada gilirannya, perlu untuk menginstal yang lain. Sekarang pengguna tidak akan mengalami sakit kepala seperti itu.

Penghapusan fragmentasi dan pembaruan kumulatif menyederhanakan administrasi sistem dan mengurangi upaya yang diperlukan untuk menginstal pembaruan.

Microsoft akan mulai merilis pembaruan bulanan kumulatif bulanan Rollup Bulanan dari Oktober 2016. Pembaruan akan dipublikasikan pada Pembaruan Windows (WU), WSUS, SCCM, serta di situs web Katalog Pembaruan Microsoft. Ngomong-ngomong, komponen ActiveX akan segera dihapus dari situs ini, sehingga akan berfungsi di browser apa pun, bukan hanya IE.

Paket layanan Oktober akan mencakup semua pembaruan untuk bulan lalu. Paket November mendatang akan menggantinya, yaitu akan mencakup paket Oktober dan semua pembaruan terbaru. Pembaruan kumulatif baru melalui Pembaruan Windows (WU) dan WSUS akan didistribusikan melalui paket ekspres, yaitu, sebagai tambahan pada paket sebelumnya, untuk menghemat lalu lintas dan ruang disk.

Secara bertahap, Microsoft akan memasukkan dalam paket kumulatif pembaruan yang dirilis pada tahun-tahun sebelumnya, sehingga akan menjadi benar-benar kumulatif.

Microsoft berjanji untuk melaporkan secara rinci KB mana yang termasuk dalam setiap paket, dan merilis catatan rilis, seperti pada Windows 10.

Tidak sepenuhnya jelas apakah pengguna harus memilih pembaruan mana yang akan dipasang dan mana yang harus ditolak. Seperti yang Anda ketahui, bersama dengan pembaruan keamanan, Microsoft terkadang memasukkan pembaruan yang tidak perlu . Mari kita berharap bahwa perintah standar untuk menghapus pembaruan yang tidak perlu akan berfungsi seperti sebelumnya.

wusa.exe /kb:3080149 /uninstall /quiet /norestart

Sebagai gantinya /kb:3080149, Anda harus menentukan nomor pembaruan yang akan dihapus.

Selain paket rollup Bulanan (pembaruan keamanan + pembaruan rutin), pada bulan Oktober Microsoft akan mulai merilis pembaruan keamanan bulanan secara terpisah, yang mencakup semua tambalan keamanan bulanan. Tambalan ini tidak lagi dibagikan satu per satu . Dan pembaruan keamanan individu tidak akan tersedia melalui Pembaruan Windows, tetapi hanya melalui WSUS, SCCM dan Katalog Pembaruan Microsoft.

Akhirnya, memperbarui .NET Framework juga pindah ke sistem bulanan. Pembaruan Rollup Bulanan .NET Framework akan mencakup pembaruan rutin dan keamanan. Microsoft menekankan bahwa pembaruan akan ditujukan hanya untuk versi .NET Framework saat ini, dan tidak akan secara otomatis ditingkatkan ke versi yang lebih baru. Selain paket reguler, pengembang sekarang menyiapkan paket terpisah hanya dengan pembaruan keamanan .NET Framework.

Dengan pembaruan kumulatif, masa pakai untuk pengguna Windows 7 dan 8.1 akan menjadi lebih mudah. Sistem baru harus bekerja sama seperti pada Windows 10.
