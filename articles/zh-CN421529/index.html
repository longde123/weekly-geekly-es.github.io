Netflix，Uber，Google和您参加MBLT DEV 2018

MBLT DEV 2018移动开发者大会还剩下一个月。 如果您尚未加入我们，请注册 ，因为9月1日，门票价格将上涨。

根据削减，我们讨论演讲者和报告。 阅读说明并开始准备问题：我们将为最酷，最无畏的人提供奖品

iOS轨道：





一位最受期待的嘉宾-来自Netflix的 John C. Fox将谈论应用程序的高质量本地化，以及如何应对恶劣的网络条件和A / B测试。 也许他仍然知道新的陌生事物季节何时到来？


《纽约时报》的KrzysztofZabłocki是一位iOS开发人员，以创建Objective-C游乐场和Foldify和3D游戏引擎等应用而闻名。 他将发表有关iOS体系结构的演讲。 Krzysztof将讨论iOS中的模式，并探讨开发人员在使用MVC和MVVM时所犯的错误。


优步立即派了两名发言人： 本·皮蒂和埃利·申 。 来自Uber移动平台团队的Ben。 他之前曾在世界上最快的键盘扩展程序Fleksy工作。 在会议上，Ben将讨论Uber团队如何解决在庞大的代码库中使用Swift的局限性。


在Uber之前的Ellie Shin曾在Spotify和Microsoft工作。 在MBLT DEV，她将告诉您如何节制移动应用程序。 了解如何减小应用程序的大小，提高其性能并使它对用户有吸引力。 我们还将讨论二进制分析，优化和向Uber学习生活技巧。



Android曲目：




尚未确定您要在哪个平台上书写？ 为什么不使用Flutter开始在两者上书写？ Google开发人员专家Laura Morinigo一起将学习使用Flutter的复杂性以及与Firebase集成以改进您的应用程序的功能。



Kastak Gopal，Instacart ，已经开发了12年，并以其Fragmented Podcast在专业社区中闻名。 从该报告中，我们学习如何使用功能强大且稳定的体系结构将旧应用程序重构为新应用程序。



去年，来自e-Legion的 Misha Funikov是MBLT DEV礼堂之一的负责人，今年他已经在表演！ 在他的报告中，他将讨论e-Legion如何在生产项目中使用协程。 我们将分析使用这种方法的困难和利弊。



Google的 Dina Sidorova正在准备有关Google Assistant的报告。 为了使您尽可能有趣，我们建议您参与报告的创建。 调查中的主题列表，选择！



在2018年I / O大会上，谷歌宣布了Firebase ML Kit，这使得利用机器学习变得容易。 在MBLT DEV 2018上，来自Coding Blocks的 Harshit Dwivedi将告诉您如何使用Firebase ML Kit中的API。



Badoo赞赏开发速度和功能质量。 在这两个标准之间保持平衡并不容易。 Artyom Rudoy将讨论Badoo在引入统一设计系统和加速UI开发方面的经验。 我们将学习如何开发该系统，它们遇到了什么问题以及集成后开发过程如何发生变化。



来自Alfa Bank的Android开发人员，自动化，扩展和改进开发阶段的爱好者Alexander Simonenko将在MBLT DEV 2018上发表演讲，并提供有关Android应用程序CI \ CD流程的发展报告。
