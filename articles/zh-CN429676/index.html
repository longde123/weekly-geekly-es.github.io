备份到磁带

俄罗斯有庞大的商店网络。 每个存储都备份到一个磁带库（照片中的下方-备件）。 然后他们拿走磁带，然后把它们带到汽车中的存档中。



机械设备：它们发生故障，失败后，我们去维修。 然后他们离开了延长保修期，这激怒了所有人。

在某些时候，它们已经过时了。 但是预算恰好是新版本的磁带库。 那时，客户出现在我们家门口，有一定数量，问是否可以在其框架内发明一些东西。

我们曾考虑过集中安装一个大铁块，但是由于商店的通道限制为5 Mbps（距离最远），这一事实使情况变得复杂。

找到的数据域


在研究基础架构时，事实证明中央办公室已经安装了Dell EMC Data Domain。 是的，对于他们的办公室任务。 在办公室，他们已经使用了很长时间并且成功了。 他们知道如何使用它，它与备份软件兼容，并且牢固地集成到基础架构中。 现在-解决问题：这里有商店，缎带，钱。 在本地，它们必须在商店中存储最多三个月，然后根据许多指标，商店中不可用的数据最多可以存储10年。 根据监管机构的要求和国内政策，这是必要的。

有希望不要打扰丝带。 没有特别的速度要求。 但是我非常想尽量减少管理员的手动操作次数，并减少无法恢复的可能性。 las，这是在磁带上发生的。 一盘磁带坏了-可汗的副本，因为没有冗余。

我们的选择


我们决定提议考虑替代磁带的方法：将备份转移到兼容S3的存储中。 当然，您可以进入Amazon，MS-cloud，也可以进入我们的公共云 -在任何具有某些SLA的对象存储的地方。 您可以在办公室中安装并安装小型私有云。 这正是Dell EMC拥有的解决方案：您可以将硬件带到办公室并进行云安装。 而且Dell EMC已经是熟悉的供应商，因此集成问题比其他所有情况都容易得多。

另外，已经有一个可以进行重复数据删除的数据域。 而且，将重复数据消除后的数据传输到其中可以大大减少流量。

根据客户的要求，我们进行了比较成本分析：现场的Dell EMC，我们的CROC和MS云。

Dell EMC ECS-大量购买，有必要扩展支持，在服务器和数据中心中进行部署。 我们观察了10年的时间，结果发现它的成本更高，原因是最低配置非常冗余，因此像飞机的机翼一样花费，您必须立即付款，然后再延长支持费用（以美元计，目前尚不清楚它们在3-5中的价格是多少–7年），并记住销售终止和支持终止的日期。 具有相同冗余度的MS存储更加昂贵。 S3的一个功能是自动将数据分散到三个地理上分离的站点。 MS的地理位置较高。

结果，客户在我们的S3中寻找并询问了飞行员。 假设，让我们看看它是否有效。 因为供应商说：我们得到了Amazon，Azhur和Google Cloud的支持，没有人知道这种解决方案是否可以与我们一起使用。

关键是将“热”数据放在存储上，然后以与存储上相同的格式将旧副本从存储上载到云中。



我们对Dell EMC Data Domain进行了测试。 他们表示可以将其副本从自己转移到S3。 Dell EMC DD已获得了成功，他们的支持极大地帮助了我们，因为对方的工程师对此任务直接感到高兴，他说：Veeam的故事很酷！



此外，我们还面临着一个事实，那就是Dell EMC具有一项功能：这样做是将数据首先添加到铁片中14天，然后才可以上传到云中。 工程师说这是硬连线的，这是开发人员的逻辑：这两个存储周期几乎都用硬代码编写。 更多的可能，更少的是不可能的。 据信，用户要恢复时的存储时间为两周。

如果数据已经传输到云中，我们将其返回，并从中恢复，它们会再次放置14天，然后再离开。

我们想定一个星期，但是整个皇家军和所有骑兵都帮不了我们。

总结


  • Veeam照常从存储的所有对象中收集备份，并将其提供给Data Domain。 他对S3一无所知。
  • Data Domain实例对本地流量进行重复数据删除，并且可以在必要时将副本传输到中心办公室。
  • Cloud Tier内置在数据域中，它会自动将数据传输到我们数据中心的S3。
  • 当用户需要从备份中取出内容时，他会敲打Veeam。 Veeam敲响了其记录系统，录音系统敲了其“磁盘”（物理磁盘或S3）并取出了副本。 一切都很好地集成在一起。

结果-它们适合无预算的设定预算，考虑了维护工作的所有成本，并实施了更可靠的地理分布解决方案。 好吧，他们让供应商方面的一名工程师感到高兴，他很高兴这个世界上有人可以思考：这是我现在的连锁店管理人员团队。
