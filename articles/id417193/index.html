Tinjauan pasar TI untuk solusi bisnis cloud

Baru-baru ini, saya didekati oleh kepala perusahaan kecil yang baru dibuat dengan permintaan untuk mengarahkannya ke pasar komputasi awan modern (bahasa Inggris. Cloud computing) atau, dengan cara sederhana, cloud. Seperti semua orang mendengar tentang awan, mereka tahu bahwa itu modis dan bermanfaat. Tetapi bagaimana cara menggunakan cloud dalam pekerjaan sehari-hari perusahaan Rusia biasa? Kapan menggunakan cloud dan mengapa?

Inti dari komputasi awan


Awan muncul di IT sebagai refleksi dari tren umum ekonomi konsumsi bersama, yang difokuskan pada penggunaan alih-alih kepemilikan dan yang telah memberi kami pertukaran perumahan , berbagi mobil , dan bahkan gaun pengantin untuk disewakan. Ada semakin banyak hal dan layanan yang dapat digunakan dan dibayar hanya untuk penggunaan aktualnya, tanpa biaya modal besar untuk akuisisi properti.

Dalam hal infrastruktur-IT, pengeluaran modal dapat menjadi signifikan, dengan cepat menjadi usang dan kehilangan nilainya, oleh karena itu, penggunaannya pada skema sewa seringkali dibenarkan.

Ada batasan yang signifikan: koneksi Internet yang stabil di tempat Anda ingin menggunakan cloud, karena semua "cloud magic" dikirimkan melalui Internet. Jika tidak ada Internet, tidak ada keajaiban.

Selain kemungkinan penghematan pada biaya modal, ada keuntungan signifikan lainnya:

  • Kecepatan. Pemasok terkemuka bisa mendapatkan cloud dalam beberapa menit / jam.
  • Fleksibilitas. Karena teknologi visualisasi, Anda bisa mendapatkan sumber daya komputasi sebanyak yang Anda butuhkan di sini dan sekarang. Melewatkan dan membeli server yang terlalu lambat bukanlah masalah besar.
  • Penghematan biaya terkait. Catu daya tanpa gangguan ke server, pembatasan akses fisik ke server (+ aturan pertama dalam berbisnis di Rusia * mudah diterapkan), dll.
  • Dalam beberapa kasus - penghematan staf (di bawah lebih banyak)

Menarik? Kemudian lanjutkan.

SaaS - Perangkat Lunak sebagai Layanan


Awan paling ramah pengguna. Anda cukup bekerja di program yang diperlukan melalui Internet (tanpa menginstalnya di perangkat Anda).

Keuntungan:

+ Rincian teknis sepenuhnya tersembunyi. Kami terhubung melalui Internet dan menggunakan program yang sudah jadi. Pembaruan, ketersediaan, dan masalah teknis lainnya serta masalah ditangani oleh penyedia layanan.
+ Sebagai aturan, ketentuan ketersediaan minimum. Sudah cukup untuk memesan layanan dan Anda dapat menggunakannya dalam beberapa menit.

Kekurangan:

- Tidak semua program tersedia dalam format ini karena alasan teknis.
- Opsi untuk menyesuaikan dan mengubah program sesuai kebutuhan Anda terbatas.
- Ketergantungan yang kuat pada penyedia layanan dan kualitas pekerjaannya.
- Seringkali Anda memiliki kontrol terbatas atas data Anda sendiri.
- Seringkali keluar lebih mahal daripada jenis cloud / aplikasi klasik lainnya, terutama untuk pelanggan besar.

Contoh SaaS


Sebagai aturan, pengembang aplikasi ini menyediakan SaaS. Menemukan solusi komprehensif untuk semua program yang diperlukan dari satu pemasok bermasalah.

PaaS - Platform sebagai Layanan


Anda menyewa platform komputasi tertentu yang terdiri dari layanan komputasi virtual (analog dari prosesor pusat dan RAM server), layanan penyimpanan informasi (analog dari hard drive server), layanan transfer data (lalu lintas yang dikonsumsi), dll. Mean emas dari layanan cloud. Anda dapat merancang komputer virtual sewenang-wenang dengan beragam perangkat lunak sistem dan pengguna. Pemasok - perusahaan IT terkemuka yang terkenal di dunia - memastikan pengoperasian yang stabil dari komputer dan program ini. Solusi di bidang ini populer di kalangan pelanggan korporat dan pengembang.

Untuk usaha kecil, solusi ini dipertajam dengan buruk dan tidak nyaman untuk menggunakannya dalam konteks ini.

Keuntungan:

+ fleksibilitas luar biasa - Anda dapat membangun komputer dengan kapasitas apa pun (dari server mikro yang sebanding dalam kapasitasnya dengan smartphone hingga sekelompok ratusan ribu server), dan menginstal berbagai aplikasi di dalamnya.
+ Pemasok utama memiliki jaringan komputasi yang tersebar luas secara geografis, yang membuatnya mudah untuk menyebarkan aplikasi yang cepat, massal, dan toleran terhadap kesalahan
+ adalah mungkin untuk menghubungkan layanan lanjutan di mana Oracle, Microsoft, Google memiliki kompetensi unik - penggunaan kecerdasan buatan, analisis sejumlah besar informasi, dll.
+ adalah mungkin untuk membayar hanya sumber daya yang benar-benar dikonsumsi.

Kekurangan:

- untuk digunakan, Anda perlu "merakit" sistem dari komponen-komponen virtual. Ini membutuhkan pengetahuan dan keterampilan khusus.
- setiap platform memaksakan batasannya pada implementasi, tidak ada kebebasan penuh dalam implementasi
- Biaya dasar yang cukup tinggi

IaaS - Infrastruktur sebagai Layanan


Anda menyewa sumber daya server / komputasi. Penyedia layanan dijamin operabilitas di tingkat server (pengisian server elektronik, Internet, catu daya, dll.) / Teknologi virtualisasi (lihat di bawah).

Ada tiga format untuk penyediaan layanan:

  1. Server khusus - server khusus. Bahkan, menyewa server fisik terpisah yang hanya dapat Anda gunakan (karenanya namanya - didedikasikan). Ini memiliki biaya yang relatif tinggi, tetapi memastikan bahwa semua sumber daya komputasi server sepenuhnya siap membantu Anda. Penyedia menyediakan akses administratif penuh ke server dan spesialis Anda mengonfigurasinya seperti yang Anda inginkan.
  2. IaaS dalam bentuk paling murni adalah pusat data virtual. Pemasok menginstal program virtualisasi khusus pada satu atau beberapa server, dan kemudian mentransfer akses ke antarmuka manajemen program ini kepada Anda. Pendekatan ini sangat fleksibel, karena Anda dapat secara mandiri membuat dan mengonfigurasi sendiri sejumlah server virtual.
  3. VPS / VDS - virtual private / dedicated server. Ini terjadi ketika penyedia mengkonfigurasi IaaS dari klausa 2 untuk Anda dan mentransfer akses hanya ke server virtual, yang parameternya tidak dapat Anda ubah. Perbedaan penting di sini adalah teknologi virtualisasi yang digunakan (KVM / XEN / OpenVZ / Hyper-V / VMware), serta kinerja yang dijamin. VPS / VDS berbiaya rendah, tetapi karena oversellig mungkin memiliki kinerja rendah / tidak stabil. Kinerja VDS lebih tinggi dari VPS, semua hal lain dianggap sama. Pasar VPS / VDS jenuh dan sangat kompetitif, layanan pencarian VPS

Keuntungan:

+ "Mengubur" uang dalam pembelian server tidak selalu masuk akal, karena itu menjadi usang seiring waktu. Saat menyewa server, Anda selalu dapat mengubah atau memperbaruinya dengan biaya minimal.
+ Anda dapat memilih dan mengelola parameter kinerja dan server seperti yang Anda inginkan
+ kebebasan penuh dalam implementasi yang diinginkan. Anda dapat menginstal semua sistem operasi, program. Kustomisasi dan gunakan sesuka Anda.
+ sen rendah, bernilai baik untuk uang

Kekurangan:

- Anda memerlukan spesialis untuk mengkonfigurasi dan memelihara perangkat lunak.
- Seringkali ada ikatan dengan karakteristik spesifik dari server / teknologi virtualisasi dan fleksibilitas hilang sedikit.
