" Kami membutuhkan hosting baru ," kata bos kami di awal musim semi. → Ringkasan

Tujuan pengujian adalah untuk mengukur kinerja disk, jaringan, dan benchmark di AIDA64. AIDA64 tidak dipilih secara acak, karena Anda dapat mengunduhnya sendiri dan membandingkan hosting dengan perangkat keras Anda, melihat seberapa tepat menempatkan aplikasi Anda di sana.

Perbandingan akan terjadi hanya dalam 3 parameter dari seluruh taman parrot AIDA64. Yaitu:

  1. Tulis ke memori - tulis memori ke data acak. Semua instruksi modern digunakan. Termasuk AVX-512. Indikator seberapa bebas prosesor untuk pekerjaan ini.
  2. VP8 - pengkodean video dengan codec VP8, menunjukkan seberapa cepat prosesor dalam kinerja single-threaded. Tes ini menggunakan instruksi: MMX, SSE2, SSSE3 atau SSE4.1
  3. FP64 - penelusuran sinar. Seluruh armada instruksi terlibat dalam pengujian ini: x87, SSE, SSE2, SSE3, SSSE3, SSE4.1, AVX, AVX2, XOP, FMA, FMA4, dan AVX-512. Prosesor baru harus terbukti paling produktif dalam pengujian ini.

Untuk pengujian, dua server disewa:

Server 1: 1 atau 2 core CPU, 2 GB RAM, 20 hingga n SSD
Server 2: 2 core CPU, 4 GB RAM, 20 to n SSD

Untuk mendapatkan nilai kinerja absolut yang benar, asalkan hosting tidak menyediakan 2 core CPU dengan 4 GB RAM, core tambahan dinonaktifkan melalui msconfig.

Jaringan diukur pada server 1. Bahkan, konfigurasi paling minimal dengan Windows Server yang disediakan oleh hosting. Setelah itu, server dimatikan. Pengukuran diulang lagi, setelah 27 hari.

Disk diukur dengan menyalin file VHDX 6 GB dari drive C: ke drive C:, setelah itu server dimatikan. Pengukuran diulang lagi, setelah 27 hari. Tangkapan layar dari kedua pengukuran hanya diterapkan jika hasilnya berbeda secara signifikan.

Pengukuran kinerja berlangsung di server 2 dalam dua tahap. 14 hari pertama, garis dasar produktivitas telah ditetapkan. Setelah itu, stress test AIDA64 dinyalakan selama 7 hari, dan dimatikan hanya untuk pengukuran kinerja. Ini diperlukan untuk mengidentifikasi pembatasan tersembunyi.

Menimbang bahwa tes minggu terakhir pada server virtual 2 adalah skenario ekstrem yang tidak seorang pun akan menggunakan, tidak ada yang akan menambang di mesin virtual, data pengukuran akhir dihitung menurut median. Hasil dalam parrot cincang disajikan seperti itu. Jumlah beo cincang dihitung dengan rumus: "Beo dibagi dengan total biaya sewa bulanan."

Selain pengukuran dan tes, saya akan membuat catatan kecil tentang pendaftaran dan proses terkait, jika ada sesuatu yang harus diperhatikan. Tentunya seseorang di suatu tempat akan melihat iklan atau "urutan pesaing" di bawah, tetapi untuk berjaga-jaga jika saya akan menghapus semua tautan, dan mengurutkan hoster dalam urutan abjad.

Untuk mengambil bagian dalam misteri Yandex Cloud, Anda harus mendaftarkan Yandex.Passport terlebih dahulu, mengaktifkan otentikasi SMS, dan mengonfirmasi nomor telepon. Untuk memulai pengguna, Anda harus mendaftarkan pembayar baru, mengkonfirmasi kartu dan mentransfer nama. Untuk mengkonfirmasi kartu debit, mereka menembak dan segera mengembalikan rekor terendah 2 ₽.

Memasang server baru memerlukan waktu 5 menit. Pada gambar standar, pengguna dan layanan tambahan (Cloudbase-init) terdeteksi. Tapi tidak ada yang perlu dikhawatirkan.

Kecepatan jaringan adalah 1 gigabit pada tingkat serendah mungkin.

Saya tidak menemukan informasi tentang pengembalian dana. Dukungan teknis dibayar, tidak menghubungi.

VNC atau KVM tidak. Mesin virtual hanya dapat dihentikan atau dimulai. Namun, Anda dapat mengelola disk virtual dan jaringan. Anda dapat membuat jaringan lokal, menghubungkan disk dan mem-boot gambar Anda. OS ini menempati 12,3 GB (dibandingkan 9,87 referensi).

Karena tagihan yang telah mereka pilih, perhitungan biaya yang benar dalam kondisi yang sama seperti untuk hosting lainnya tidak dimungkinkan. Total biaya server saya selama 3 minggu penggunaan adalah 5104,78 ₽. Dari jumlah ini saya akan melanjutkan. Saya telah memilih cadangan 5%.

Inoventica (tidak diberi peringkat)


Saat mendaftar, Anda perlu mengkonfirmasi email. Itu saja.

Memasang server baru memerlukan waktu 21 menit.

OS menempati 10,3 GB (dibandingkan 9,87 referensi).

Kenalan dengan hosting dimulai dengan musim gugur. Segera setelah crash ini, 2 virus muncul di server yang menyamar sebagai anydesk dan sqlserver, saya harus menginstal ulang Windows. Dan bahkan setelah menginstal ulang OS dan memperbaruinya, masih macet dan menjadi terinfeksi. Itu crash terus-menerus, dan karena penjualan liar, memori OS dimuat tidak lebih dari layar pemulihan.

Tes stres memori Aida yang diluncurkan, menulis ke memori, tidak menerima semua 4 GB RAM yang saya pesan, tetapi hanya menerima 2,4 GB.

Kecepatan jaringan adalah 420 megabit per detik untuk pengunduhan dan 62,93 untuk pengunggahan.

Untuk mengembalikan uang, Anda perlu menulis aplikasi yang kompleks secara tertulis dan melampirkan pemindaian paspor.

Tinjauan hosting ini tidak diberikan karena macet setiap hari. Saya bangun di pagi hari untuk memeriksa apakah server berbohong atau tidak. Saya menerapkan semua hasil yang saya terima. Grafik kosong tidak termasuk downtime, ini adalah laporan yang dilewati pada batas waktu setengah jam. Pengujian stres hosting tidak dilakukan.

Saya sangat berharap bahwa perusahaan ini akan berhenti memberikan layanan.
