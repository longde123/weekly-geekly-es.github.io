Casas inteligentes de entusiastas o hola de nuevo



Saludos a la audiencia Geektimes!

Somos el proyecto SmartLiving.ru , que está discutiendo activamente y desarrollando una plataforma abierta para construir una casa inteligente MajorDoMo.

Nuestro proyecto no es tan nuevo, pero gracias al programa de soporte inicial, tenemos la oportunidad de declararlo más ampliamente y no limitarnos a los detalles técnicos de la implementación de una oportunidad en particular. ¿Por qué es Smart Home de entusiastas? Porque somos tales :-)

Un poco de historia y filosofía

Acerca de las casas inteligentes Probablemente, personalmente, aprendí por primera vez a finales de los 90, aunque antes de eso había encontrado más de una vez elementos de automatización del hogar, como lámparas encendidas con algodón. Pero el primer proyecto que estudié con suficiente detalle fue el proyecto de código abierto MisterHouse escrito en Perl. En realidad, a lo largo de los años, me inspiró a crear MajorDoMo.

El proyecto MajorDoMo "oficialmente" comenzó en 2012 con el lanzamiento del sitio web y el lanzamiento del primer artículo sobre Habré con una descripción general de las capacidades del sistema. Después de un tiempo, salió otro artículo que se hizo bastante popular: "Un día típico bajo la supervisión de Alice". Desde entonces, ha fluido mucha agua: el proyecto se ha desarrollado técnicamente, la comunidad de usuarios ha crecido, gradualmente la información sobre el sistema divergió en la Web. Ahora hemos llegado al escenario cuando el entusiasmo y el tiempo libre de varias personas no son suficientes para el desarrollo posterior del proyecto. Las perspectivas de apertura requieren un enfoque profesional; de lo contrario, pueden cerrarse y, por lo tanto, se decidió crear una startup completa sobre la base del proyecto con la asignación de recursos apropiados para su lanzamiento y desarrollo. Entonces, a pesar de la edad del proyecto relativamente considerable (según los estándares de los productos de TI), todo lo más interesante recién comienza.

Sea como fuere, los principales "pilares" del proyecto siempre han permanecido y permanecerán sin cambios:
  • Franqueza
  • Disponibilidad
  • Libertad de Elección

Apertura

Código abierto: cualquiera puede conectarse al proyecto en GitHub , crear su propia sucursal y enviar recomendaciones sobre cómo cambiar la sucursal principal.

Licencia MIT: el proyecto se puede usar tanto para fines no comerciales como comerciales sin ninguna otra restricción que no sea indicar su origen.

Una comunidad abierta (y activa): el foro del proyecto sirve no solo como generador de nuevas ideas y formas de aplicación, sino también como una excelente plataforma de capacitación y asistencia mutua.

Accesibilidad

Descargar gratis sin registro y SMS : la versión estable actual siempre está disponible para descargar e instalar. De la misma manera que los cambios más recientes en el código fuente se instalan a través del sistema de actualización incorporado.

Fácil de instalar: el instalador para Windows, instrucciones de instalación para varias distribuciones de Linux, imágenes preparadas para Raspberry.

Libertad de elección

Quizás una de las características más importantes del sistema es la libertad de elección. El usuario mismo decide qué componentes construir su Smart Home, desde la elección de la plataforma del servidor hasta la selección de módulos específicos, porque no existen barreras para la integración de "todo y todo". Para cada tarea, puede elegir la solución más óptima, sin preocuparse de que el equipo de un fabricante no pueda "comunicarse" con su hermano de otro; si MajorDoMo admite ambas clases de equipos, entonces solo debe preocuparse por la lógica de interacción.

Entre los protocolos / equipos admitidos podemos distinguir: Z-Wave, KNX / EIB, Modbus, Noolight, Uniel, Orvibo, MQTT, 1-wire, WirenBoard, MegaD ... Aquí puede agregar un número infinito de dispositivos diferentes basados ​​en Arduino.

Además, puede integrarse con muchos servicios y aplicaciones utilizando API abiertas. Creada en tecnologías web, la plataforma le permite usar muchos ejemplos listos.
