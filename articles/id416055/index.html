Penugasan dan dukungan FQDN server 3QX

Pengguna kami secara berkala memiliki pertanyaan tentang tujuan dan dukungan FQDN dari server 3CX (yang sekarang dapat disediakan oleh 3CX): prinsip-prinsip memilih FQDN, menetapkan FQDN ini ke kunci lisensi, dll. Dalam artikel ini, kami akan menjawab banyak pertanyaan ini.

Saat memasang 3CX v15 dan 15.5, kunci lisensi dilampirkan ke FQDN yang dipilih oleh administrator selama instalasi server. Karena itu, sangat penting untuk segera memilih FQDN yang benar, sehingga nantinya Anda menghindari pekerjaan yang tidak perlu.

Mereka sering mengajukan pertanyaan, mengapa kita membutuhkan FQDN untuk server 3CX? Bukankah lebih mudah bergaul dengan alamat IP? Kami menjawab:

  • Untuk koneksi jarak jauh (SSL) aman dari klien 3CX ke layanan web yang menyampaikan status pengguna, riwayat panggilan, obrolan, manajemen klien terpusat dan fungsi lainnya, yang dapat disebut istilah umum: komunikasi terpadu.
  • Untuk memberikan akses jarak jauh yang aman ke antarmuka manajemen 3CX
  • Untuk menerapkan toleransi kesalahan, sehingga telepon IP dapat terhubung ke FQDN tunggal, yang dapat diselesaikan baik ke alamat IP dari server utama dan ke alamat server cadangan.
  • Bekerja pada alamat IP dinamis. Jika ISP Anda tidak memberikan alamat IP statis, 3QX FQDN memberi Anda layanan DNS dinamis.
  • Mengamankan lisensi Anda. Lisensi yang Anda beli diberikan ke domain Anda. Dalam kasus penggunaannya yang tidak sah, itu tidak akan mungkin untuk menggunakan PBX pada kunci ini, karena Anda akan segera melihat bahwa seseorang menggunakan domain Anda dengan alamat IP palsu.

Pertama, pertimbangkan untuk memilih FQDN untuk server 3CX, asalkan Anda menginstal kunci lisensi ini untuk pertama kalinya (belum ada FQDN yang terikat) dan menggunakan 3CX v15 SP4 atau lebih tinggi.

3CX Domains


Pengguna 3CX v15 dengan berlangganan aktif untuk pembaruan produk dapat menerima nama domain publik dari 3CX di salah satu zona domainnya. Layanan ini disediakan gratis. Selain nama domain, pengguna secara otomatis menerima sertifikat SSL tepercaya gratis untuk FQDN ini dari Let's Encrypt. Setelah Anda memilih nama host (userpart) 3CX dan zona domain (hostpart) yang ditawarkan kepada Anda di antarmuka installer selama instalasi, FQDN yang dihasilkan (userpart.hostpart.tld) â€‹â€‹dilampirkan ke kunci lisensi yang diaktifkan.

Selain itu, FQDN tambahan dibuat untuk layanan WebMeeting sesuai dengan prinsip ini: userpart.hostpart.tld -> userpart-tld.3cx.net

Domain umum: 

customer.3cx.de -> customer-de.3cx.net Customer.3cx.us -> customer-us.3cx.net

Domain tingkat kedua: 

 customer.3cx.co.uk -> customer-uk.3cx.net customer.3cx.co.nz -> customer-nz.3cx.net customer.3cx.com.tr -> customer-tr.3cx.net customer.3cx.co.za -> customer-za.3cx.net

Domain Pengecualian: 

 customer.elastix.com -> customer-ecom.elastix.net

Domain kota dikurangi menjadi 3 huruf: 

 customer.3cx.barcelona -> customer-bar.3cx.net

Subdomain dari tingkat kedua: 

 customer.north.3cx.us -> cusomer-north-us.3cx.net

3CX berhak untuk mencabut FQDN yang diberikan kepada Anda setiap saat, yang memiliki karakteristik sebagai berikut:

  • Melanggar merek dagang perusahaan lain
  • Digunakan untuk tujuan selain hosting 3CX
  • Berisi penghinaan dan teks lain semacam ini
  • Diperlukan oleh perusahaan 3CX itu sendiri (dalam hal ini, 3CX menghubungi mitra untuk mentransfer FQDN dengan benar)



Kami juga mencatat bahwa pengguna 3CX tidak memiliki hak untuk meminta nama domain 3CX yang sudah diambil oleh pengguna lain.

Semua nama domain dikeluarkan jika tersedia. Jika FQDN tidak digunakan untuk waktu yang lama, FQDN dibebaskan untuk pengguna lain. Dalam hal ini, tidak dijamin bahwa FQDN yang sebelumnya Anda tautkan ke kunci lisensi Anda masih tersedia.

Domain sendiri


Jika Anda menginstal server 3CX di bawah FQDN perusahaan Anda sendiri (dan tidak dikeluarkan oleh 3CX), perlu diingat bahwa ini memerlukan edisi 3CX Pro atau Perusahaan. Anda juga harus membeli sertifikat SSL tepercaya untuk nama domain ini sebelumnya (atau sertifikat wildcard untuk seluruh domain). Sayangnya, sertifikat Let's Encrypt gratis tidak dikeluarkan dalam kasus ini.



Kunci lisensi 3CX akan terikat pada FQDN Anda sendiri, dan FQDN dari portal WebMeeting akan dibentuk sebagai berikut: userpart.hostpart.tld -> userpart-hostpart-tld.3cx.net

Contoh: 

 customer.domain.com -> customer-domain-com.3cx.net customer.domain.co.uk -> customer-domain-co-uk.3cx.net pbx.customer.domain-part.co.nz -> pbx-customer-domain-part-co-nz.3cx.net

Lisensi FQDN 3CX Gratis


Lisensi 3CX gratis (dengan pengecualian kunci afiliasi NFR) yang belum digunakan selama 3 bulan dilepaskan dari FQDN yang terkait. FQDN ini kemudian dapat didaftarkan oleh pengguna lain. Jika FQDN sudah diambil oleh pengguna lain, Anda akan melihat pesan Digunakan pada tahap aktivasi kunci (lihat di bawah).

DNS TTL dalam edisi 3CX yang berbeda


Waktu caching untuk catatan DNS (TTL) tergantung pada jenis lisensi yang Anda miliki. Untuk lisensi Standar dan Pro, itu diinstal pada pukul 6 sore. Untuk lisensi Enterprise - 300 detik. Karena periode singkat ini, dimungkinkan untuk mengalihkan pengguna ke alamat IP dari server cadangan dengan sangat cepat (hingga 5 menit), karena semakin rendah TTL, semakin cepat catatan FQDN A diperbarui ke alamat IP saat ini.

Kemungkinan kesalahan


Seperti yang dikatakan, selama instalasi pertama, installer 3CX mengikat FQDN pilihan Anda ke kunci lisensi Anda. Tetapi jika FQDN yang dipilih sudah digunakan, kesalahan berikut akan muncul: 

 Error creating FQDN: FQDN already in use. Please choose another one

Dalam hal ini, cukup pilih FQDN yang berbeda.

Harap dicatat bahwa instalasi berulang 3CX dengan kunci yang sama akan "menarik" FQDN yang terkait dengannya. Jika Anda menentukan FQDN yang berbeda, kesalahan akan muncul: 

 Error creating FQDN: License key already bound to another FQDN

Dalam hal ini, Anda harus pergi ke Portal Pengguna 3CX dan memeriksa FQDN mana yang dikaitkan dengan kunci. Jika FQDN terlampir tidak cocok untuk Anda, lepaskan dari kunci, dan hanya kemudian melanjutkan dengan instalasi 3CX.

Kesalahan aneh berikut juga mungkin terjadi: 

 Domain zone XYZ is exhausted and can not be used. Please choose another domain.

Dalam hal ini, hanya ada satu solusi - memilih domain tingkat atas lainnya.
