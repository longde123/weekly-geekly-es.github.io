JavaScript Meetup SuperJob: Videobericht

Am 10. Oktober hatten wir ein JavaScript Meetup SuperJob . Wir erzählen, was die Teilnehmer besprochen haben, teilen Videos und Präsentationen.



Alexander Kukhta , leitender Entwickler von SuperJob, sprach zum Thema "Erfahrung mit JSON: API auf dem Client".

Alexander vertritt das Plattformteam, das für die Architektur und Leistung der JavaScript-Anwendung des Unternehmens verantwortlich ist.

Als erster Redner teilte Alexander mit, wie das Frontend in SuperJob funktioniert. Dann sprach er darüber, wie die Jungs ungefähr 1.500 Raw-Entitäten auf dem Client speichern, wie sie mit ihnen in React-Komponenten arbeiten und wie sie es getan haben, damit die Anwendung nicht langsamer wird.


→ Präsentation von Alexander
→ Demo von der Aufführung


Peter Solopov , Leiter des SuperJob-Entwicklungsteams, sprach über "Type Systems and Frontend Typing".

In Peters Bericht finden Sie Antworten auf die Fragen:

  • Welche Typsysteme gibt es und wie haben sie sich entwickelt?
  • Was sind die Hauptunterschiede zwischen Flow und TypeScript?
  • Wie wird Flow in einem großen Projekt wie SuperJob verwendet?

Und dort finden Sie eine Menge guter Mathematik, die viele wahrscheinlich von der Studentenbank vergessen haben.


→ Präsentation von Peter


Alexander Savin , SuperJob-Frontend-Entwicklungsmanager, sprach zum Thema „Erstellen eines JSON-Servers: API auf Node.js und PostgreSQL“ .

Alexander teilte seine Erfahrungen beim Erstellen eines Servers gemäß der JSON: JavaScript-API.
Er erklärte, mit welchen Tools Sie schnell die API für eine vorhandene Datenbank erhalten und wie Sie ein solches Tool mithilfe von Node.js und PostgreSQL selbst erstellen können.
Alexander ging auch auf das Thema möglicher Schwierigkeiten bei der Verwendung des „First DB“ -Ansatzes ein und erläuterte, was in der Anwendungsarchitektur vorgesehen werden muss, um diese zu minimieren.


→ Präsentation von Alexander
