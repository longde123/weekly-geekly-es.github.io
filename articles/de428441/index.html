GitHub-Kauf abgeschlossen. Was wird als nächstes passieren?

Der Kauf von GitHub durch Microsoft ist nun abgeschlossen. Kürzlich teilte der neue CEO von GitHub, Nat Friedman, die einleitende Ankündigung mit, die wir hier veröffentlichen. Details zum zukünftigen Service unter dem Schnitt.



Ich freue mich, Ihnen mitteilen zu können, dass die Übernahme von Microsoft GitHub abgeschlossen wurde. Montag 29.10 ist mein erster Tag als CEO. Millionen von Menschen verlassen sich jeden Tag auf GitHub, und ich bin stolz darauf, dieses Unternehmen führen zu können.

Als wir den Kauf im Juni ankündigten, teilte ich zwei Prinzipien für GitHub, die im Verständnis festgelegt werden sollten:

  • GitHub wird in drei unabhängigen Paradigmen operieren: als Community, Plattform und Unternehmen. Dies bedeutet, dass GitHub seinen Wert für Entwickler, einen unverwechselbaren Community-Geist und Offenheit beibehält. Wir unterstützen Entwickler immer bei der Auswahl einer Sprache, Lizenz, eines Tools, einer Plattform oder einer Cloud.
  • GitHub wird seine Philosophie beibehalten. Wir lieben GitHub, weil wir uns um die Entwickler kümmern. Ich verstehe und respektiere dies und ich bin mir sicher, dass wir weiterhin schöne, schnelle und zuverlässige Tools entwickeln werden, die Entwickler lieben.

Letztendlich ist es meine Aufgabe, GitHub für Sie besser zu machen.

Ich habe mich in den letzten Monaten mit Hunderten von Entwicklern getroffen. Die Leidenschaft für GitHub ist unglaublich - sowohl in Bereichen, in denen wir erfolgreich sind, als auch in Bereichen, in denen Sie möchten, dass wir mehr tun. Ich habe viel aus dieser Interaktion mit den Entwicklern gelernt. Die Kommunikation mit unseren Benutzern wird der Hauptteil der Arbeit von GitHub sein.

Drei Ziele werden für uns das Hauptziel sein:

  • Machen Sie GitHub zu einem besseren Ort, um produktive Communities und Teams zu erstellen.
  • Stellen Sie GitHub Entwicklern auf der ganzen Welt zur Verfügung.
  • Zuverlässigkeit, Sicherheit und Leistung.

Konzentrieren wir uns zunächst darauf, die täglichen Praktiken bei der Verwendung von GitHub zu verbessern und unser Projekt für Papierschnitte gemeinsam zu erweitern. Wir werden grundlegende Aufgaben wie Suche, Benachrichtigungen, Probleme / Projekte und die Plattform auf Mobilgeräten verbessern. Und natürlich GitHub-Aktionen. Wir wollen sie wirklich verbreiten.

Wir glauben an die Macht der Gemeinschaft. Als größte Entwickler-Community der Welt bringt GitHub mehr als 31 Millionen Entwickler zusammen, die erstellen, arbeiten, teilen und integrieren können.
