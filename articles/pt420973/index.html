Configurando o UFW no Ubuntu Server 16.04 com o exemplo BigBlueButton

UFW (Uncomplicated Firewall) é uma interface de tabelas de ip projetada para simplificar o processo de configuração de um firewall. A ferramenta iptables é confiável e flexível, mas não será fácil para um novato configurá-lo. Eu sou apenas novo neste negócio.

Por padrão, ACCEPT está ativado no iptables e fica assim:



No nosso exemplo, não há restrições. A política padrão é ACEITAR.

CONEXÕES DE ENTRADA - ENTRADA.
COMPOSTOS DE SAÍDA - SAÍDA.
PRÓXIMO - ROTAÇÃO (POR EXEMPLO, É NECESSÁRIO ESCREVER UMA ROTA NO Computador atrás do gateway).

Você pode ativar o ufw com o comando sudo ufw enable
E imediatamente após esse comando, a política do iptables muda.

A área selecionada é o local para regras do usuário, elas são registradas com os comandos:

Para porta:

sudo ufw allow ssh (pelo nome da porta)
sudo ufw allow 22 (especifique o número da porta especificamente)
sudo ufw allow 80/udp (especificando especificamente o número da porta e o protocolo tcp ou udp)

Para intervalo de portas: 

 sudo ufw allow 16384:32768/udp

Removendo regras: 

 sudo ufw delete allow 80

Estamos interessados ​​em linhas com xxx.xxx.xxx.x (xxx.xxx.xxx.x é o endereço IP do seu servidor), pois isso indica que o serviço está escutando na placa de rede solicitações recebidas de todas as redes (0.0.0.0:*) . As linhas com 127.0.0.1 não nos interessam, pois essa é a interface interna; portanto, estamos interessados ​​nas linhas com 0.0.0.0 na 4ª coluna (destacadas em negrito), por exemplo

tcp 0 0 0.0.0.0:80 0.0.0.0:* LISTEN 1359/nginx -g daemo

como diz que a porta 80 está aberta a todas as interfaces do servidor e aceita solicitações de todas as redes.

Então, de acordo com isso, fazemos as regras:

sudo ufw allow 22 para ssh
sudo ufw allow 80 para servidor web nginx ou apache

Várias regras para freeswitch: 

 sudo ufw allow 5060 sudo ufw allow 5090 sudo ufw allow 2855 sudo ufw allow 2856 sudo ufw allow 5066 sudo ufw allow 8081 sudo ufw allow 8082

para serviço de nó:

sudo ufw allow 3000
Em seguida, você precisa examinar a documentação, na qual veremos que é necessário abrir as seguintes portas:

sudo ufw allow 16384:32768/udp - para trabalhar com WebRTC
sudo ufw allow 443
sudo ufw allow 1935

Isso conclui a configuração básica do ufw para o servidor BigBlueButton 2.0.

Acrescentarei apenas que é desejável desativar o ssh ou alterar a porta ou permitir essa porta apenas para uma rede específica, por exemplo, para a rede da sua empresa.

Por exemplo, seu gateway possui duas interfaces de rede, uma virtual (para usuários) 192.168.3.2 e a outra real 105.xxx.xxx.xxx (para o provedor), aqui para a regra ufw usamos um ip real ou, melhor dizendo, indique a rede à qual isso pertence ip.

Primeiro, altere a porta no arquivo de configuração do servidor SSH, você pode usar qualquer editor de texto: 

 vim /etc/ssh/sshd_config

Em seguida, excluímos a regra que permite a porta 22:

Sudo ufw delete allow 22 - às vezes chamado de OpenSSH.

Criamos uma regra que permite a conexão com a nova porta somente da rede corporativa: 

 sudo ufw allow from 105.../26 to any port 4321

Em seguida, reinicializamos o serviço ssh: 

 /etc/init.d/ssh restart

E veja o resultado 

 netstat -ntlp | grep LISTEN 

 tcp 0 0.0.0.0:4321 0.0.0.0:* listen 5676/sshd

e nas regras do ufw, veremos quem tem acesso à porta: 

 ufw status 

 4321 ALLOW 105.xxx.xxx.xxx/26
