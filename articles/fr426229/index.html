Code de conduite: Pourquoi les développeurs de noyau Linux menacés de supprimer leur code - Comprendre le conflit

En septembre, le conseil d'administration de la Fondation Linux a adopté un nouveau code de conduite pour les développeurs du noyau Linux, le Code de conduite (CoC). Tel que conçu par les auteurs, il était censé résoudre les problèmes liés à la discrimination et améliorer l'environnement émotionnel des développeurs.

Certains programmeurs ont appelé cela une tentative de surréglementation et ont commencé à menacer de supprimer le code qu'ils ont écrit du noyau Linux. En savoir plus sur la situation ci-dessous.


/ photo Liam Quinn CC

Contexte


Au milieu du mois dernier, Linus Torvalds a annoncé qu'il ferait une pause dans la gestion du développement Linux. Selon Torvalds, il a besoin de temps pour réfléchir à sa manière de communiquer avec les gens (le créateur de Linux est connu pour ses remarques dures sur ses collègues). Dans le bulletin d'information de LKML, il s'est excusé auprès de tous ceux qui ont quitté la communauté à cause de ses critiques.

Pendant le «repos» de Linus, la direction a été assumée par Greg Kroah-Hartman, qui est engagé dans le support d'une version stable du noyau. Il a proposé l' adoption d'un nouveau code de conduite - Code de conduite. Ils ont décidé de mettre en œuvre le document afin de changer la culture de communication dans la communauté.

Le code de conduite est basé sur la norme du Covenant Contributeur , qui est également utilisé par les développeurs de Ruby on Rails, Swift, Golang et JRuby. Le document décrit les règles de conduite avec les autres programmeurs et souligne l'interdiction de toute discrimination.

Développeurs contre code de conduite


De nombreux développeurs ne sont pas d'accord avec l'introduction du CoC: ils craignent que les nouvelles règles conduisent à une réglementation excessive. Certains ont même exhorté tous ceux qui n'étaient pas d'accord non seulement à quitter la communauté, mais à interdire l'utilisation de code écrit par eux pour le noyau.

Tout a commencé avec une lettre du programmeur Joey Pabalinas, dans laquelle il critiquait le document. Selon lui, le code ne résout pas les problèmes de la culture de la communication, mais au contraire, rend l'atmosphère de la communauté hostile. Joey pense que CoC divisera les participants en «bonnes» et «mauvaises» et exclusions non méritées de la communauté.

La procédure CoC est particulièrement déplaisante, selon laquelle il est possible d’interdire à tout développeur de valider son code si les participants décident de ne pas aimer l’identité de l’auteur. Les membres de la communauté sont également préoccupés par le libellé vague du code, qui peut être interprété de différentes manières (par exemple, «commentaires désobligeants»). Cependant, le document ne décrit pas de cadre clair pour déterminer la sanction des auteurs.

Enfin, la troisième raison qui soulève des questions parmi les développeurs est les différences culturelles. Étant donné que CoC offre un style de communication unifié, les personnes de différents pays (y compris ceux pour lesquels l'anglais n'est pas considéré comme officiel) peuvent commencer à éprouver des difficultés à choisir la «langue correcte».

Y a-t-il une menace pour le noyau Linux


La "légende de la communauté open source", Eric Raymond , a suggéré que les développeurs mécontents étaient vraiment capables d'empêcher leur code d'être utilisé dans le noyau Linux.

Le système d'exploitation est publié sous la licence publique générale GNUv2. Les opposants au Code de conduite citent le fait qu'il ne contient pas de clause interdisant la résiliation de l'accord entre l'auteur du code et le projet open source (il existe une telle clause dans GNUv3). Par conséquent, ils sont convaincus qu'ils ont le droit d'obliger la Linux Foundation à supprimer le code du noyau.


/ photo Matt Madd CC

Et il y a déjà eu des précédents similaires dans la communauté open source. Il y a deux ans, le programmeur Azer Koçulu a reçu une lettre des avocats du messager Kik dans laquelle ils demandaient de renommer le module du même nom (kik), publié dans le gestionnaire de paquets npm. Azer a refusé, puis les représentants de Kik ont ​​écrit à l'administration du NPM. Ceux-ci ont soutenu la réclamation et ont changé le propriétaire de kik sans la permission de l'auteur.

En réponse, Kochul a rappelé tous les modules npm qu'il avait écrits et les a téléchargés sur GitHub . Mais parmi eux se trouvait l'un des modules populaires du pavé gauche, composé de onze lignes de code. Il est nécessaire pour le fonctionnement de Node.js, Babel et un grand nombre d'autres systèmes.

Voici à quoi ressemble le code du «module discord»: 

module.exports = leftpad;

function leftpad (str, len, ch) {
  str = String(str);

  var i = -1;

  if (!ch && ch !== 0) ch = ' ';

  len = len - str.length;

  while (++i < len) {
    str = ch + str;
  }

  return str;
}
