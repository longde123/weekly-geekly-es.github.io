我们完全攻击Joker 2019：俄罗斯最大的Java会议怎么样

在十月份，这不是我第一次坐飞机从遥远的伊尔库茨克出发向圣彼得堡方向行驶。 我之前参加过JUG Ru Group的Java会议，但仍然没想到这次会打破纪录，并且有超过2000名Java程序员和支持者聚集在Expoforum。 铁杆，刻录，工程报告，实用笑话和录制实时视频节目将我们包围了两天。 我马上就做出保留，因为他们已经在枢纽上写了有关Joker 2019的信息，例如： dbelob- “ Joker 2019：本年度的胜利 ”， gayka_m8- “ 我们如何记住最后的JokerConf ”。 但是现在可以使用最终照片了，可以计算收视率并且可以使情绪平静下来，因此我对会议的印象被削弱了。




第一天就像第一回合


在所有参与者的入口处都遇到了一只黑鸭。 小丑会议的秘密吉祥物在所有来访者中怒不可遏。 尽管这一事实至少没有阻止许多人与他拥抱和合影。 快速注册，大型展览（我们将在稍后讨论）和大量的自由空间迎接了会议，当然，很快就有了积极进取的开发人员。

我正忙于录制与发言人的访谈，因此错过了部分报告。 我必须说，起初我本人受到了质疑，这是不寻常的。 播客如何习惯自己成为提问者。 但是随后一切都按预期进行，面试是很平常的事情。 与录制播客的主要区别在于团队合作，JUG Ru Group团队以工业规模提供了演讲者-一切就像在他们所在领域选择专业人士一样。



该会议由Juergen Hoeller和Josh Long的联合报告“ Reactive Spring reisited ” 开幕 ，在此期间将近一个小时，您可以使用Spring Boot 2.2享受有关Spring Framework 5.2的实时编码会议。 乔什·朗（Josh Long）几乎不停地即时提供材料。



从春季的主要公众人物那里了解如何为云体系结构创建下一代反应式微服务的外观特别有用。 我立即注意到，我设法查看了收视率，该报告将前十名的报告分成了一半。 对于那些没有参加会议的人，进行了广播，可以通过屏幕参加第一大厅的这个和其他演示。

在第一次报告期间，我在第二个房间里，听了Vladimir Ozerov的报告“ 如何在Hazelcast中组织多线程 ”。 在我看来，他的报告包含了大量的说明性材料。 而且，如果您现在想了解Hazelcast中多线程的情况-正是您必须访问的报告。



令我惊讶的是，该程序包含了大多数英语报告，并且在大多数情况下，只有一份俄语报告。 那一点也没有吓到我。 而且，在百年的Joker会议上，我们都习惯了实时编码格式。 塞巴斯蒂安·达施纳 （ Sebastian Daschner）的报告“致力于向更高生产力的开发工​​作流程迈进”是这一趋势的杰出代表 。 塞巴斯蒂安（Sebastian）几乎不留痕迹地使用了演讲时间，并用俄语感谢了听众。 该报告未包含在会议的前十名报告中，但是我还是很喜欢。



用塞巴斯蒂安的话说：“本次会议展示了哪些方法和技术支持团队制定更具生产力的开发工​​作流程。” 重述演讲的本质是完全没有意义的，有必要将自己沉浸在一个简单而催眠的叙事中，在该叙事中，控制台，IDEA和迷人的故事相互交织。

报告之后，所有发言人都在讨论区。 这是一种独特的交流体验，不会被记录在案。 一些演讲者被小时折磨了，听众没有放过我们的英雄。 救赎只是随着下一次演讲的开始才出现，然后不是全部。 在第一天，大约有二十多名发言人在四个房间讲话，并突破了讨论区。



会议的发现之一是来自Baruch Sadogursky jbaruch和JFrog-DevOps SpeakEasy的新视频项目的现场直播。 在为期两天的会议中，进行了大约两次广播，会议参与者可以在这些会议期间进行现场直播。 在斯蒂芬·钱（ Stephen Chin）和巴鲁克·萨多古尔斯基（Baruch Sadogursky）的参与下发行了第一期。 随后，双方都担任首席面试官，后来都进入了Joker 2019的主舞台。



会议的第一天以Baruch Sadogursky的演讲结束，他在报告中谈到了系统管理员的阴谋-“开发人员的DevOps（或反对他们？！）”。 听众说-“在#jokerconf上来自jbaruch的报告“开发人员的DevOps”-观看很有趣，但是如果您认真听讲，它可以帮助开发人员“成长”或其他。 有趣而有意义。” 根据听众的不同，该报告在会议最佳报告中排名第二。

而且，如果Baruch的报告仍以免费广播的形式结束并可以记录，那么随后的聚会和四个BOF将仅保留在会议的全职参与者的记忆中。





宴会开始后的30分钟，这段段落中开始了四个平行的BOF会议，并且有可能从一个焦点转向另一个，讨论如下：

  • “性能：商业服务吗？”
  • 《恐怖故事》
  • “最佳的微服务框架”
  • “大四以后有生命吗？”

同时，他们正走在第三大厅-闪电谈话，但是，我再也没有力量参加那些自称发言人的小型报告。

第二轮


第二天以Evgeny Borisov， EvgenyBorisov和Kirill Tolkachev tolkkv的精彩对子报告“ Spring Reactive Ripper” 开始 。 这是他们早些时候在JPoint 2019上发布的报告的新版本，但包含了30％的新材料。

尽管一开始听起来有些不安，但该报告在各个方面都很火热。

尽管许多人可能不同意我的观点，但是由于该计划还包括严厉的Tagir Valeev的 报告 -“ Java 9-14：小型优化 ”，可靠的消息人士说，该报告在顶级会议报告的排名中排名第一。 这就像明显的非显而易见性，与基准紧密混合在一起。 Tagir进行了许多小的优化，显示了从版本9到版本14的Java某些部分的性能如何变化，并且非常完美。



顺便说一句，根据传统，根据填写反馈表的访问者的反馈意见，选择了Joker 2019的十大报告。

不幸的是，我错过了许多有趣的报道，但是我设法使自己沉浸在展览区的工作中。 当然，这里是无休止的天堂，吸引着大量的游客抢劫和摔落。 可以参加许多比赛和任务-这是回报。



如果您浏览互联网，则可以找到农作物的精美照片：



但是，看台的主要价值不是在精美的商品上，而是在专家身上。 在几乎任何展位和任何时间，都可以找到专家并讨论使用他们所提供产品的精妙之处。 当然，每个人都充分利用了什么。



在看台上，生活在沸腾。 比赛，讨论，社交，空缺。 JUG RU集团本身的展台也是如此。



闭幕主题演讲由斯蒂芬·钱（Stephen Chin）主持，尽管一段时间后，巴鲁克·萨多古尔斯基（Baruch Sadogursky）并没有坐下来加入他。 他们一起结束了关于“解密繁忙的编码器的技术炒作”主题的报告。



再见


在展览区，有一个用绳子固定的架子，除其他外，可以清楚地看到会议参与者是否只为鸭子而来。 并不是说结果可以用于统计研究，但至少很有趣。



在会议结束时，我觉得每个参与者都发现了一些不同之处。
