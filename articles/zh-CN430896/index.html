Linux Foundation已为GraphQL和Ceph建立了资金-为什么需要它们，以及对它们的期望

非营利性联盟Linux Foundation宣布立即成立两个新基金： GraphQL Foundation和Ceph Foundation 。 接下来是关于它们的更多信息。


/图片Roderick Eime CC BY-ND

为什么需要新的组织


Linux Foundation的主要目标是促进和标准化Linux。 但是，该联盟也在开发整个开源生态系统和相关产品。 最近，已添加GraphQL查询语言和Ceph对象存储网络。

GraphQL由Facebook在2012年开发。 这是描述如何请求数据并用于与服务器交互的语法。 它是作为REST API的替代品而创建的。 至于Ceph，它出现在2007年，七年后，红帽接管了该项目。 最初，该解决方案是为超级计算机开发的，但是后来他们开始使用Ceph来部署云基础架构。 根据2017年的数据（ PDF ），Ceph使用OpenStack上三分之二的云。

总体而言，在过去三年中，GraphQL和Ceph在各种规模的公司中都广受欢迎：从IT巨头到小型创业公司。 例如，GraphQL用户中有数百个组织，其中包括Coursera，GitHub，《纽约时报》，Twitter和Pinterest。 云提供商（Rackspace，Linode），大型金融组织（Bloomberg，Fidelity），汽车制造商（BMW）等使用 Ceph的对象存储网络。

Linux基金会看到了这些解决方案的潜力，并决定支持它们。 该联盟建立了两个新的GraphQL基金会和Ceph基金会。

GraphQL Foundation会做什么


创建了一个单独的基金来加速GraphQL周围的技术生态系统的发展。 Linux基金会和许多公司将提供财务和组织支持：GitHub，Netflix，Shopify，Twitter，Pinterest等。 该工具的创建者之一Lee Byron 希望该基金会能够帮助GraphQL成为行业标准。

到目前为止，GraphQL基金会正在组建中-该基金的最终利益相关者名单将在稍后获得批准。 然后将开始选择技术委员会和市场部门的代表。 他们计划在几个月内完成所有这些工作。 基金会开始从事GraphQL项目后。 他们的列表可以在GitHub上找到 。

Ceph Foundation的功能


Ceph开发基金包括三十家IT巨头，包括Canonical，Intel，Red Hat和SUSE。 该基金的宗旨和目标与GraphQL Foundation相同：使开发人员摆脱不必要的任务，围绕产品形成生态系统并加快其开发。

来自Canonical，Amihan，Red Hat，ProphetStor和其他公​​司的基金成员指出 ，他们的个人经验和专业知识已经帮助对Ceph进行了许多改进，包括机器学习系统和与应用程序容器一起使用的功能。 Linux Foundation的支持将成为存储网络进一步发展的“驱动器”。

也许新基金还将帮助消除在大型IT基础架构中使用技术时出现的问题 。 例如，与内存不足（OOM）错误相关的问题。 Ceph社区根本没有足够的资源来解决它们。 现在情况必须改变。

期待什么


过去，Linux基金会创建了许多基金会。 例如， Cloud Native Computing Foundation （用于促进容器）或Academy Software Foundation （其成立是为了促进电影和动画行业中的开源软件）。 总体而言，该基金已经支持了80多个开放项目，并且正在积极开发中。


/图片Christopher Michel CC BY

但是，尽管如此，Linux Foundation产品组合中还是有相对失败的项目。 例如，由于平台本身的不完善， IT社区将Tizen操作系统的前景视为 “模糊”。 错误的界面特别突出。

另一方面，对于Ceph和GraphQL，一切都会变得更好。 Linux基金会的成员每天都在增长，并且该基金会正在增加用于支持其项目的资金。 GraphQL基金会和Ceph基金会都有积极发展的机会。
