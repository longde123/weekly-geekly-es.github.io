Yealink Meeting Server 2.0 - Fitur Konferensi Video Baru

Dalam artikel terakhir: Yealink Meeting Server - solusi komprehensif untuk konferensi video, kami menjelaskan fungsionalitas versi pertama Yealink Meeting Server (selanjutnya YMS), kemampuan dan strukturnya. Akibatnya, kami menerima dari Anda banyak permintaan untuk menguji produk ini, beberapa di antaranya telah tumbuh menjadi proyek terpadu untuk pembuatan atau modernisasi infrastruktur konferensi video.
Skenario yang paling umum adalah mengganti MCU lama dengan server YMS, sambil mempertahankan armada perangkat terminal yang ada, dan memperluas menggunakan terminal Yealink.

Ada tiga alasan utama untuk ini:

  1. Skalabilitas MCU yang ada tidak mungkin atau mahal tidak masuk akal.
  2. Akumulasi hutang untuk dukungan teknis sebanding dengan biaya solusi turnkey VKS modern.
  3. Pabrikan meninggalkan pasar dan dukungan berhenti dilakukan sama sekali.

Banyak dari Anda yang pernah mengalami peningkatan Polycom, misalnya, atau dukungan LifeSize, akan memahami apa yang sedang dibahas.

Fungsi baru Yealink Meeting Server 2.0, serta memperbarui jajaran klien terminal Yealink, tidak memungkinkan Anda untuk memasukkan semua informasi dalam satu artikel. Oleh karena itu, saya berencana untuk membuat serangkaian publikasi kecil tentang topik-topik berikut:

  • Ikhtisar YMS 2.0
  • Server YMS Cascading
  • Integrasi YMS dan S4B
  • Terminal Yealink Baru
  • Solusi multi-ruang untuk ruang konferensi besar

Apa yang baru?


Selama tahun berjalan, sistem menerima beberapa pembaruan signifikan - baik dalam fungsi maupun dalam skema lisensi.

  • Integrasi dengan server Skype For Business disediakan - melalui gateway perangkat lunak bawaan YMS dapat mengumpulkan konferensi video dengan partisipasi pengguna S4B lokal dan cloud. Pada saat yang sama, lisensi YMS kompetitif reguler digunakan untuk koneksi. Tinjauan terpisah akan dikhususkan untuk fungsi ini.
  • Fungsi cascading server YMS telah diimplementasikan - sistem dapat diinstal dalam mode klaster untuk meningkatkan kinerja dan penyeimbangan beban. Fitur ini akan dijelaskan secara rinci di artikel selanjutnya.
  • Jenis baru "Broadcast" lisensi telah muncul - pada kenyataannya, ini bukan siaran, tetapi langkah pertama untuk mengoptimalkan biaya lisensi dalam konferensi asimetris. Bahkan, jenis lisensi ini memungkinkan peserta yang hadir yang tidak mengirimkan video / suara mereka sendiri ke konferensi, tetapi yang dapat melihat dan mendengar peserta dengan lisensi penuh. Dalam hal ini, kita mendapatkan sesuatu seperti webinar atau konferensi peran di mana para peserta dibagi menjadi pembicara dan penonton.
    Lisensi Broadcast dilengkapi dengan paket dengan jumlah koneksi kelipatan 50. Dalam hal 1 koneksi, pemirsa 6 kali lebih murah daripada speaker.

Langkah pertama




Halaman utama server menawarkan untuk memasukkan antarmuka pengguna atau panel kontrol admin.

Kami membuat pintu masuk pertama dari bawah admin.



Pada permulaan pertama, Wisaya Instalasi langkah-demi-langkah ditampilkan, memungkinkan Anda untuk mengkonfigurasi semua modul sistem yang diperlukan (kami akan mempertimbangkan lebih detail di bawah).

Langkah pertama adalah mengaktifkan lisensi. Di versi 2.0, proses ini telah mengalami beberapa perubahan. Jika sebelumnya cukup dengan menginstal file lisensi, yang dikaitkan dengan alamat MAC pengontrol jaringan server, sekarang prosedur dibagi menjadi beberapa tahap:

  1. Anda harus mengunduh sertifikat server (* .tar) yang Yealink berikan, melalui perwakilan - melalui kami, misalnya.
  2. Menanggapi impor sertifikat, sistem membuat file permintaan (* .req)
  3. Alih-alih file permintaan, Yealink mengirim kunci lisensi
  4. Tombol-tombol ini, pada gilirannya, dipasang melalui antarmuka YMS, dan mengaktifkan jumlah port koneksi simetris yang diperlukan, serta paket lisensi Siaran, jika berlaku.



Dalam rangka. Impor sertifikat di bagian Lisensi di halaman rumah.

Untuk mengekspor file permintaan, klik tautan “Lisensi Anda belum diaktifkan. Silakan aktifkan . " Dan buka jendela "Lisensi Aktivasi Offline"



Kirim file permintaan yang diekspor kepada kami, dan kami akan memberikan Anda satu atau dua kunci aktivasi (terpisah untuk setiap jenis lisensi).

File lisensi diinstal melalui kotak dialog yang sama.

Akibatnya, sistem menampilkan status dan jumlah koneksi kompetitif untuk setiap jenis lisensi.



Dalam contoh kami, lisensi diuji dan memiliki tanggal kedaluwarsa. Dalam kasus versi komersial - mereka tidak terbatas.

Antarmuka YMS memiliki beberapa opsi terjemahan, termasuk Rusia. Tetapi terminologi dasar lebih familiar dalam bahasa Inggris, jadi saya akan menggunakannya untuk screenshot.

Halaman beranda admin menampilkan informasi singkat tentang pengguna / sesi aktif, status dan jumlah lisensi, serta informasi sistem server perangkat keras dan versi semua modul perangkat lunak.

Setelah menginstal lisensi, Anda harus memulai pengaturan server - Anda dapat menggunakan asisten.



Di tab Network Association , atur nama domain server YMS - nama itu bisa nyata atau fiktif, tetapi perlu untuk konfigurasi lebih lanjut dari terminal. Jika tidak nyata, maka dalam pengaturan pada klien nama domain didorong ke alamat server, dan IP asli server didorong ke alamat proxy.

Tab Waktu berisi pengaturan SNTP dan zona waktu - ini penting untuk pengoperasian kalender dan distribusi yang benar.

Ruang Data - kontrol dan batasan ruang disk untuk berbagai kebutuhan sistem, seperti log, cadangan, dan firmware.

Kotak Surat SMTP - pengaturan surat untuk milis.

Versi baru YMS memiliki fitur yang bermanfaat - Alokasi Sumber Daya Angka .
Sebelumnya, penomoran YMS internal telah diperbaiki. Ini bisa membuat kesulitan ketika berintegrasi dengan IP-PBX. Untuk menghindari persimpangan dan membuat penomoran fleksibel Anda sendiri, Anda perlu mengonfigurasi untuk setiap grup yang dapat melakukan panggilan melalui panggilan numerik.



Dimungkinkan tidak hanya untuk mengubah kedalaman bit angka, tetapi juga untuk membatasi interval. Ini sangat nyaman, terutama ketika bekerja dengan IP-telephony yang ada.

Untuk pengoperasian penuh server YMS, Anda perlu menambahkan layanan yang diperlukan.



Di sub-bagian Layanan SIP , layanan dasar ditambahkan untuk berfungsi menggunakan koneksi SIP. Bahkan, penambahan turun ke beberapa langkah sederhana di setiap tab - Anda perlu memanggil layanan, pilih server (dalam mode cluster), adaptor jaringan dan, jika perlu, edit port koneksi.

Layanan Registrasi - bertanggung jawab untuk mendaftarkan terminal Yealink

Layanan Panggilan IP - melakukan panggilan

Layanan REG Pihak Ketiga - pendaftaran terminal perangkat keras pihak ketiga

Peer Trunk Service dan REG Trunk Service - Integrasi IP PBX (dengan dan tanpa registrasi)

Skype for Business - integrasi dengan server atau cloud S4B (lebih banyak di artikel terpisah)

Lebih lanjut, dengan cara yang sama, Anda perlu menambahkan layanan yang diperlukan di Layanan H.323 ayat, Layanan MCU dan Layanan Traversal .

Setelah pengaturan awal, Anda dapat melanjutkan untuk mendaftar akun. Karena fungsi ini tidak secara praktis berubah selama proses pembaruan dan dijelaskan dalam artikel sebelumnya, kami tidak akan membahasnya.

Pengaturan detail dan penyesuaian


Kami akan sedikit menyentuh pada konfigurasi panggilan (Kebijakan Kontrol Panggilan) - beberapa opsi berguna telah muncul di sini.



Misalnya, Tampilkan video asli adalah tampilan video Anda sendiri di konferensi.

Alamat push iOS - memungkinkan Anda untuk menerima notifikasi pop-up pada perangkat iOS dengan Yealink VC Mobile diinstal.

Siaran interaktif - memungkinkan peserta-pemirsa untuk terhubung ketika lisensi Siaran diaktifkan.

RTMP live and Recording - termasuk fungsi konferensi penyiaran dan perekaman. Tetapi penting untuk diingat bahwa setiap rekaman / siaran tidak hanya memuat server tambahan, tetapi juga menggunakan lisensi penuh untuk 1 koneksi kompetitif. Ini harus diperhitungkan saat menghitung kapasitas port server dan jumlah lisensi.

Kebijakan Tampilan Video - pengaturan tampilan.



Sebagai kesimpulan, pertimbangkan submenu "Kustomisasi".



Di bagian ini, Anda dapat menyesuaikan antarmuka YMS agar sesuai dengan identitas perusahaan Anda. Adaptasi template surat surat dan catatan IVR untuk Anda sendiri.

Banyak modul antarmuka grafis mendukung penggantian dengan opsi pengguna - mulai dari latar belakang dan logo, berakhir dengan pesan sistem dan screensaver.

Kesimpulan


Antarmuka administrator ringkas dan intuitif, meskipun setiap pembaruan memiliki fungsi tambahan.

Saya tidak melihat titik mendemonstrasikan antarmuka konferensi video aktif dalam artikel ini - kualitasnya masih pada sistem perangkat keras VKS tingkat tinggi. Lebih baik tidak memikirkan hal-hal subjektif seperti kualitas dan kenyamanan, lebih baik mengujinya sendiri!
