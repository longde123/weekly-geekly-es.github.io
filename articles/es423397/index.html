Integración de 3CX con 1C: Gestión comercial: método para instalar una extensión de configuración

Ante las numerosas solicitudes de socios y clientes, lanzamos la integración del servidor 3CX v15.5 con el popular sistema 1C: Enterprise.

La integración es una extensión de la configuración típica "Trade Management, Edición 11 (11.4.3.167) para Rusia" ( archivo .cfe ), que se instala en la configuración de forma estándar (ver más abajo). Desde el lado del servidor 3CX, se carga una plantilla CRM para interactuar con el servicio web de configuración publicado 1C.

Extensión de configuración 1C: la gestión comercial implementa una API REST que asigna los números de llamadas entrantes a los números de contraparte en 1C. Cuando una llamada llega a 3CX, solicita a través de REST de 1C los datos de la contraparte asociados con la identificación de llamada del suscriptor. Si se encuentra la contraparte, sus datos se agregan a los contactos de 3CX y se muestran en el cliente web de 3CX (nombre y apellido).

Si Enable Call Journaling está habilitado en 3CX, 1C mantendrá un registro de llamadas en 1C.

1C: requisitos de configuración de la empresa


Para usar la extensión, se requiere una versión de la plataforma 1C: Enterprise no inferior a 8.3.11. La configuración típica "Gestión comercial, edición 11" (11.4.3.167) se entrega en modo de compatibilidad con la plataforma 8.3.10. Para usar la extensión 3CX, debe establecer el modo de compatibilidad de configuración en No usar. Si la configuración es compatible, debe habilitar la capacidad de cambiar la configuración.

Lo demostraremos en las capturas de pantalla:

En Configurator 1C, vaya a la sección Configuración - Soporte - Configuración de soporte.



Haga clic en el botón Habilitar capacidad de cambio y confirme en la ventana emergente.



En la ventana Configuración de reglas de soporte, seleccione Objetos con la regla "Cambios permitidos": el objeto del proveedor se edita mientras se mantiene el soporte y Objetos con la regla "Cambios no recomendados": el objeto del proveedor no es editable.



Haga clic con el botón derecho en la sección raíz de Gestión de comercio, seleccione Propiedades y configure el Modo de compatibilidad como No usar.



Elija Configuración - Actualizar configuración de la base de datos.



Y haga clic en Aceptar para completar los cambios.



Conectando una extensión para interactuar con 3CX


Cierre el Configurador y ejecute la configuración 1C en modo Enterprise. En el menú superior izquierdo, seleccione Todas las funciones ...



Seleccione Administrar extensiones de configuración.



Asegúrese de que el Alcance esté configurado cuando instale la extensión de configuración - Infobase , haga clic en el botón Agregar y agregue la extensión especificando el archivo 3cx1cextension.cfe del archivo.



Después de instalar la extensión, la pestaña Administrar extensiones de configuración debería aparecer como se muestra a continuación.



Publicación del servicio HTTP 1C: Enterprise Extension en un servidor web


La publicación de la configuración 1C en el servidor web se realiza de manera estándar, teniendo en cuenta las características del entorno de software.

  • El procedimiento de publicación se describe en detalle en 1C: ITS en este enlace .
  • Instrucciones para configurar servicios web para 1C v8.3 e IIS 8.5 en el recurso Infostart .
  • Comentario del artículo Publicación de 1C: bases de datos Enterprise 8.3 en el servidor web IIS en Microsoft Windows Server 2008/2012 .

Al publicar, configure la opción predeterminada de Publicar servicios de extensión HTTP.



Si hay usuarios en la configuración (en casos excepcionales, se utilizan configuraciones sin usuarios), debe cambiar la cadena de conexión a la base de datos 1C: Enterprise.

Considere esto usando el servidor web IIS como ejemplo:

En el directorio de publicaciones, busque el archivo default.vrd.



Cambie la cadena de conexión en él, agregando las credenciales del usuario en cuyo nombre la base de datos responderá a las solicitudes HTTP. La cadena de conexión debería verse así.



Instalación de la plantilla CRM 1C: Enterprise XML en 3CX


Abra la interfaz de administración de 3CX y vaya a Configuración - Integración CRM - pestaña del lado del servidor.

En la sección Integraciones de CRM disponibles, seleccione 1C y haga clic en Descargar seleccionados.



Luego, en el campo Parte del dominio, especifique la URL del servicio HTTP de configuración 1C publicado.
Para contabilizar las llamadas en 1C, habilite la opción Habilitar el registro de llamadas.


Capacidades de integración 1C: Enterprise con 3CX


Como se mencionó anteriormente, la extensión compara los identificadores de llamadas entrantes con los números de contraparte en la base de datos 1C. Si se encuentra una coincidencia, el Cliente Web 3CX mostrará el nombre y el apellido de la persona que llama.

Contabilidad de llamadas en 1C


Si el registro de llamadas está habilitado en la plantilla CRM, 1C registrará las llamadas entrantes y salientes con los siguientes parámetros:

  • Fecha de llamada
  • Tipo de llamada: entrante / saliente / perdida por extensión / sin respuesta de externo
  • Duración de la llamada
  • Número externo (identificador de llamadas)
  • Número de extensión (extensión 3CX)
  • Contraparte 1C (si se encuentra cumplimiento)



En el registro de llamadas, puede especificar las condiciones de selección y clasificación de las columnas que le interesan para analizar rápidamente la situación de las llamadas perdidas o fallidas en la organización.



Revista técnica de interacción 3CX y 1C


El diario técnico de 3CX mantiene un registro de todas las llamadas de 3CX a 1C a través de la API REST.



Este registro puede ser utilizado para diagnósticos por el administrador o el programador 1C.

Preguntas frecuentes


  1. ¿Qué sucede si no se encuentra un cliente? Por el momento, la plataforma 1C no tiene la capacidad de invocar el formulario para agregar un nuevo cliente a través de un enlace HTTP.
  2. Si el contacto se muestra en el cliente web, ¿es posible abrir su tarjeta completa en 1C? Por el momento, la plataforma 1C no tiene la capacidad de llamar a una tarjeta de cliente a través de un enlace HTTP.
  3. ¿Es posible hacer una llamada saliente desde un cliente 1C delgado o grueso? Es posible, pero para esto, se debe implementar el soporte adecuado de la API REST de 3CX.
  4. ¿Es posible redirigir una llamada desde 1C o implementar un grupo de "administrador fijo de cliente"? Es posible, pero para esto, se debe implementar el soporte adecuado de la API REST de 3CX.
  5. ¿Es posible implementar un panel de monitoreo en tiempo real (panel de telefonía) de llamadas en 1C? Actualmente, con una llamada entrante, 3CX transmite solo la identificación de llamada del suscriptor sin información de servicio auxiliar (por ejemplo, ocupación de una línea específica, número de extensión del destinatario de la llamada, etc.). Se transmite información de servicio adicional después de completar la llamada. Además, el panel involucra la interacción del usuario con las llamadas, pero esto requiere soporte de la API REST de 3CX.
  6. ¿Puedo ver llamadas perdidas en la tarjeta de contraparte? Utilice el registro de llamadas en 1C, que implementa la extensión. En él, puede seleccionar rápidamente los contratistas que le interesan.
