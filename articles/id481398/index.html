Perintah Linux dasar untuk penguji dan banyak lagi

Kata Pengantar


Halo semuanya! Nama saya Sasha, dan saya telah menguji backend (layanan Linux dan API) selama lebih dari enam tahun. Pikiran tentang artikel itu datang kepada saya setelah permintaan berikutnya dari penguji yang akrab untuk memberitahunya apa yang dapat dibaca pada perintah Linux sebelum wawancara. Biasanya, seorang kandidat untuk posisi QA engineer diperlukan untuk mengetahui perintah-perintah dasar (jika, tentu saja, itu berarti bekerja dengan Linux), tetapi bagaimana Anda memahami perintah-perintah yang harus Anda baca ketika mempersiapkan sebuah wawancara jika Anda memiliki sedikit atau tanpa pengalaman Linux?

Oleh karena itu, walaupun sudah ditulis tentang ini berkali-kali, saya masih memutuskan untuk menulis artikel lain "Linux untuk pemula" dan daftar di sini perintah dasar yang perlu Anda ketahui sebelum wawancara di departemen (atau perusahaan) di mana mereka menggunakan Linux. Saya memikirkan perintah dan utilitas mana dan dengan parameter apa yang paling sering saya gunakan, mengumpulkan umpan balik dari rekan-rekan saya, dan memasukkan semuanya ke dalam satu artikel. Artikel ini secara kondisional dibagi menjadi 3 bagian: pertama, informasi singkat tentang dasar-dasar I / O di terminal Linux, kemudian ikhtisar dari perintah yang paling dasar, dan bagian ketiga menjelaskan solusi dari masalah khas di Linux.

Setiap tim memiliki banyak pilihan, semuanya tidak akan terdaftar di sini. Anda selalu dapat memasukkan ` man <command> ` atau ` <command> --help` untuk mempelajari lebih lanjut tentang perintah.

Contoh:

Jika perintah terlalu lama untuk diselesaikan, itu dapat diselesaikan dengan menekan Ctrl + C di konsol (sinyal SIGINT dikirim ke proses).

Sedikit tentang output perintah


Ketika suatu proses dimulai di Linux, 3 aliran data standar untuk proses ini dibuat: stdin , stdout dan stderr . Mereka diberi nomor 0, 1, dan 2, masing-masing. Tetapi kami sekarang tertarik pada stdout dan, pada tingkat lebih rendah, stderr . Dari namanya, mudah ditebak bahwa stdout digunakan untuk menampilkan data, dan stderr digunakan untuk menampilkan pesan kesalahan. Secara default, ketika perintah dijalankan di Linux, stdout dan stderr mencetak semua informasi ke konsol, namun, jika output perintah besar, mungkin lebih mudah untuk mengarahkan ulang ke file. Ini bisa dilakukan, misalnya, seperti ini: 

 [user@testhost ~]$ man signal > man_signal

Jika kita mencetak isi file man_signal , kita akan melihat bahwa itu identik dengan apa jadinya jika kita cukup menjalankan perintah ` man signal` .

Operasi pengalihan ` > ` menggunakan stdout secara default. Anda dapat menentukan pengalihan stdout secara eksplisit: ` 1> `. Demikian pula, Anda dapat menentukan pengalihan stderr : ` 2> `. Anda dapat menggabungkan operasi ini dan dengan demikian memisahkan output normal dari perintah dan output pesan kesalahan: 

 [user@testhost ~]$ man signal 1> man_signal 2> man_signal_error_log

Anda dapat mengarahkan ulang stdout dan stderr ke satu file sebagai berikut: 

 [user@testhost ~]$ man signal > man_signal 2>&1

Operasi pengalihan ` 2> & 1` berarti mengarahkan stderr ke tempat stdout diarahkan.

Alat lain yang nyaman untuk bekerja dengan I / O (atau lebih tepatnya, itu adalah cara yang nyaman untuk komunikasi antarproses) adalah pipa (atau pipa ). Pipa sering digunakan untuk menautkan beberapa perintah: perintah stdout diarahkan ke stdin berikut ini, dan seterusnya dalam rantai:

cd


Ubah direktori saat ini. 

 [user@testhost ~]$ pwd /home/user [user@testhost ~]$ cd /home/ [user@testhost home]$ pwd /home

Jika Anda tidak meneruskan nama direktori sebagai argumen, variabel lingkungan $ HOME , yaitu direktori home, akan digunakan. Mungkin juga nyaman menggunakan ` ~ `, karakter khusus yang berarti $ HOME : 

 [user@testhost etc]$ pwd /etc [user@testhost etc]$ cd ~/test/ [user@testhost test]$ pwd /home/user/test

mkdir


Buat direktori.

Terkadang Anda perlu membuat struktur direktori tertentu: misalnya, direktori di direktori yang tidak ada. Agar tidak memasukkan mkdir beberapa kali berturut-turut, Anda dapat menggunakan opsi -p - ini memungkinkan Anda untuk membuat semua direktori yang hilang dalam hierarki. Juga dengan opsi ini mkdir tidak akan mengembalikan kesalahan jika direktori ada.

rm


Hapus file. 

 [user@testhost ~]$ ls qqq test test2 [user@testhost ~]$ rm qqq [user@testhost ~]$ ls test test2

Opsi -r memungkinkan Anda untuk menghapus direktori secara rekursif dengan semua isinya, opsi -f memungkinkan Anda untuk mengabaikan kesalahan saat menghapus (misalnya, file yang tidak ada). Opsi-opsi ini memungkinkan, secara umum, untuk menjamin penghapusan seluruh hierarki file dan direktori (jika pengguna memiliki hak untuk ini), oleh karena itu, mereka harus digunakan dengan hati-hati (contoh lelucon klasik adalah " rm -rf / ", dalam keadaan tertentu itu akan menghapus jika bukan keseluruhan sistem, ada banyak file penting untuk kinerjanya). 

 [user@testhost ~]$ ls test test2 [user@testhost ~]$ ls -ltr test2/ total 4 -rw-rw-r-- 1 user user  0 Nov 26 10:40 temp drwxrwxr-x 2 user user 4096 Nov 26 10:40 temp_dir [user@testhost ~]$ rm -rf test2 [user@testhost ~]$ ls test

cp


Salin file atau direktori. 

 [user@testhost ~]$ ls temp test [user@testhost ~]$ cp temp temp_clone [user@testhost ~]$ ls temp temp_clone test

Perintah ini juga memiliki opsi -r dan -f , mereka dapat digunakan untuk menjamin bahwa hirarki direktori dan folder akan disalin ke lokasi lain.

Karena suatu proses memiliki penangan sinyal, kill tidak selalu mengarah pada hasil yang diharapkan - penghentian instan dari proses tersebut. Untuk "membunuh" suatu proses pasti, Anda perlu mengirim sinyal SIGKILL ke proses tersebut. Namun, ini dapat menyebabkan kehilangan data (misalnya, jika proses harus menyimpan beberapa informasi ke disk sebelum mengakhiri), jadi Anda perlu menggunakan perintah ini dengan hati-hati. gema


Tampilkan satu baris teks. 

 [user@testhost ~]$ echo "Hello" Hello

Opsi -n untuk tidak melengkapi garis dengan jeda baris di akhir, dan -e untuk mengaktifkan interpretasi melarikan diri dengan "\" patut diperhatikan. 

 [user@testhost ~]$ echo "\tHello\n" \tHello\n [user@testhost ~]$ echo -n "\tHello\n" \tHello\n[user@testhost ~]$ [user@testhost ~]$ echo -ne "\tHello\n" Hello

Juga, menggunakan perintah ini Anda dapat menampilkan nilai-nilai variabel. Misalnya, di Linux, kode keluar dari perintah yang terakhir selesai disimpan dalam variabel khusus $? , dan dengan demikian Anda dapat menemukan kesalahan apa yang terjadi pada aplikasi yang terakhir berjalan:

Memecahkan masalah Linux yang khas


Ubah pemilik file


Anda dapat mengubah kepemilikan file atau direktori menggunakan perintah chown : 

 [user@testhost ~]$ chown user:user temp [user@testhost ~]$ ls -l temp -rw-rw-r-- 1 user user 31 Nov 26 11:09 temp

Dalam parameter, perintah ini harus diberikan pemilik dan grup baru (opsional), dipisahkan oleh titik dua. Juga, ketika mengubah pemilik direktori, opsi -R mungkin berguna - maka pemilik seluruh isi direktori juga akan berubah.

Ubah izin file


Masalah ini diselesaikan dengan menggunakan perintah chmod . Sebagai contoh, saya akan memberikan instalasi hak "pemilik boleh membaca, menulis, dan mengeksekusi, grup diizinkan untuk membaca dan menulis, sisanya - tidak ada": 

 [user@testhost ~]$ ls -l temp -rw-rw-r-- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod 760 temp [user@testhost ~]$ ls -l temp -rwxrw---- 1 user user 31 Nov 26 11:09 temp

7 pertama (ini adalah 0b111 dalam representasi bit) pada parameter berarti "semua hak untuk pemilik", 6 kedua (ini 0b110 dalam representasi bit) - "baca dan tulis", well, 0 tidak ada artinya bagi yang lain. Bitmask terdiri dari tiga bit: bit yang paling signifikan ("kanan") bertanggung jawab untuk eksekusi, bit berikutnya ("tengah") untuk menulis, dan yang tertua ("kiri") adalah untuk membaca.
Anda juga dapat mengatur hak menggunakan karakter khusus ( sintaksis mnemonic ). Misalnya, dalam contoh berikut, hak eksekusi untuk pengguna saat ini dihapus terlebih dahulu dan kemudian dikembalikan:

 [user@testhost ~]$ ls -l temp -rwxrw---- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod -x temp [user@testhost ~]$ ls -l temp -rw-rw---- 1 user user 31 Nov 26 11:09 temp [user@testhost ~]$ chmod +x temp [user@testhost ~]$ ls -l temp -rwxrwx--x 1 user user 31 Nov 26 11:09 temp

Perintah ini memiliki banyak kegunaan, jadi saya menyarankan Anda untuk membaca lebih lanjut tentang hal itu (terutama tentang sintaksis mnemonik, misalnya, di sini ).

Cetak isi file biner


Ini dapat dilakukan dengan menggunakan utilitas hexdump . Berikut ini adalah contoh penggunaannya.

 [user@testhost ~]$ cat temp Content of a file. Lalalala... [user@testhost ~]$ hexdump -c temp 0000000  C on  tent    ofaf  il 0000010  e . \n  L ala  lala .  . . \n 000001f [user@testhost ~]$ hexdump -x temp 0000000  6f43 746e  6e65 2074 666f  6120 6620 6c69 0000010  2e65 4c0a  6c61 6c61 6c61  2e61 2e2e 000a 000001f [user@testhost ~]$ hexdump -C temp 00000000 43 6f 6e 74 65 6e 74 20 6f 66 20 61 20 66 69 6c |Content of a fil| 00000010 65 2e 0a 4c 61 6c 61 6c 61 6c 61 2e 2e 2e 0a |e..Lalalala....| 0000001f

Menggunakan utilitas ini, Anda dapat menampilkan data dalam format lain, namun, opsi ini untuk penggunaannya paling sering berguna.

Cari file


Anda dapat menemukan file dengan nama di pohon direktori menggunakan perintah find :

 [user@testhost ~]$ find test_dir/ -name "*le*" test_dir/file_1 test_dir/file_2 test_dir/subdir/file_3

Pilihan lain dan filter pencarian juga tersedia. Misalnya, Anda dapat menemukan file di folder tes yang dibuat lebih dari 5 hari yang lalu:

 [user@testhost ~]$ ls -ltr test total 0 -rw-rw-r-- 1 user user 0 Nov 26 10:46 temp_clone -rw-rw-r-- 1 user user 0 Dec 4 10:39 created_today [user@testhost ~]$ find test/ -type f -ctime +5 test/temp_clone

Cari teks dalam file


Perintah grep akan membantu mengatasi tugas ini . Dia memiliki banyak kegunaan, berikut adalah contoh yang paling sederhana.

 [user@testhost ~]$ grep -nr "content" test_dir/ test_dir/file_1:1:test content for file_1 test_dir/file_2:1:test content for file_2 test_dir/subdir/file_3:1:test content for file_3

Salah satu cara populer untuk menggunakan perintah grep adalah menggunakannya dalam sebuah pipa :

 [user@testhost ~]$ sudo tail -f /var/log/test.log | grep "ERROR"

Opsi -v memungkinkan Anda untuk membuat efek grep 'dan sebaliknya - hanya garis-garis yang tidak mengandung pola yang diteruskan ke grep yang akan ditampilkan .

Lihat paket yang diinstal


Tidak ada perintah universal, karena semuanya tergantung pada distribusi Linux dan manajer paket yang digunakan. Kemungkinan besar salah satu dari perintah berikut akan membantu Anda:

 yum list installed apt list --installed zypper se —installed-only pacman -Qqe dpkg -l rpm -qa

Lihat berapa banyak ruang yang dibutuhkan pohon direktori


Satu use case untuk perintah du :

 [user@testhost ~]$ du -h -d 1 test_dir/ 8,0K test_dir/subdir 20K test_dir/

Anda dapat mengubah nilai parameter -d untuk mendapatkan informasi lebih rinci tentang susunan direktori. Anda juga dapat menggunakan perintah dalam kombinasi dengan mengurutkan :

 [user@testhost ~]$ du -h -d 1 test_dir/ | sort -h 8,0K test_dir/subdir 16K test_dir/subdir_2 36K test_dir/ [user@testhost ~]$ du -h -d 1 test_dir/ | sort -h -r 36K test_dir/ 16K test_dir/subdir_2 8,0K test_dir/subdir

Opsi -h dari perintah sortir memungkinkan Anda untuk mengurutkan ukuran yang ditulis dalam format yang dapat dibaca manusia (misalnya, 1K, 2G), opsi -r memungkinkan Anda untuk mengurutkan data dalam urutan terbalik.

"Temukan dan ganti" dalam file, dalam file dalam direktori


Operasi ini dilakukan dengan menggunakan utilitas sed (tanpa tanda g pada akhirnya, hanya kemunculan pertama "teks lama" dalam string yang akan diganti):

 sed -i 's/old-text/new-text/g' input.txt

Anda dapat menggunakannya untuk beberapa file sekaligus: 
 [user@testhost ~]$ cat test_dir/file_* test content for file_1 test content for file_2 [user@testhost ~]$ sed -i 's/test/edited/g' test_dir/file_* [user@testhost ~]$ cat test_dir/file_* edited content for file_1 edited content for file_2

Tumpukan lalu lintas jaringan


Untuk mencegat lalu lintas jaringan di Linux, utilitas tcpdump digunakan . Untuk membuang lalu lintas di port 12345, Anda dapat menggunakan perintah berikut:

 [user@testhost ~]$ sudo tcpdump -i any -A port 12345

Opsi -A menunjukkan bahwa kita ingin melihat output dalam ASCII (oleh karena itu, ini bagus untuk protokol teks), -saya menunjukkan bahwa kita tidak tertarik pada antarmuka jaringan, port - port yang lalu lintas dibuang. Alih-alih port, Anda dapat menggunakan host , atau kombinasi dari host dan port ( host A dan port X ). Dan opsi -n juga dapat berguna - jangan mengonversi alamat menjadi nama host dalam output.
Bagaimana jika lalu lintas biner? Maka opsi -X akan membantu kami - menampilkan data dalam hex dan ASCII:

 [user@testhost ~]$ sudo tcpdump -i any -X port 12345

Harus diingat bahwa dalam kedua kasus penggunaan paket IP akan menjadi output, jadi pada awal masing-masing akan ada header biner IP dan TCP. Berikut adalah contoh dari output untuk permintaan " 123 " yang dikirim ke server dengan mendengarkan pada port 12345:
