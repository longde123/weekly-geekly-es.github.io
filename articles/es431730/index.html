Windows Server 2019

La historia de Windows Server se remonta a más de 25 años: Windows NT 3.1 Advanced Server se lanzó el 27 de julio de 1993. En octubre de 2018, tres años después de la versión principal anterior, se lanzó Windows Server 2019. Windows Server 2019 desarrolla y mejora las capacidades inherentes a las versiones anteriores. Detalles sobre esto, ¡debajo del corte!



A partir de Windows Server 2016, se adoptó un nuevo ciclo de lanzamiento. Ahora hay dos canales de distribución: LTSC (canal de servicio a largo plazo), un lanzamiento que se lanzará en 2-3 años, con soporte primario de 5 años y soporte extendido de 5 años, así como Canal semianual, lanzamientos que se lanzan cada seis meses, tienen un ciclo de soporte principal de 6 meses y un soporte extendido de 18 meses. ¿Para qué son estos dos canales? Microsoft está innovando activamente en su plataforma en la nube Azure. Esto es compatible con máquinas virtuales Linux, contenedores con Linux y Windows, y muchas otras tecnologías.

Los clientes que usan estas tecnologías en la nube también quieren usarlas en sus centros de datos. El Canal semianual cierra la brecha entre Azure y los centros de datos locales. Los lanzamientos semestrales están destinados a empresas que tienen un desarrollo dinámico y han cambiado a un modelo de servicio flexible para proporcionar servicios de TI a las empresas. Las versiones de LTSC están destinadas a empresas que utilizan aplicaciones bien establecidas con un ciclo de soporte largo, por ejemplo, Exchange Server, SharePoint Server, SQL Server, así como roles de infraestructura, centros de datos definidos por software e infraestructura hiperconvergente.

Windows Server 2019 es exactamente el lanzamiento en el canal LTSC. Incluye todas las actualizaciones funcionales de Windows Server 2016 y posteriores versiones semestrales.

Los principales esfuerzos de los desarrolladores de Windows Server 2019 se centraron en cuatro áreas clave:

  1. Hybrid Cloud : Windows Server 2019 y el nuevo centro de administración del Centro de administración de Windows facilitan el uso de los servicios en la nube de Azure: Azure Backup, Azure Site Recovery, Azure Update Management, Azure AD Authentication y otros.
  2. La seguridad es una de las prioridades más importantes para los clientes. Windows Server 2019 tiene capacidades integradas para dificultar que los ciberdelincuentes penetren y se establezcan en el sistema. Estas son las tecnologías Defender ATP y Defender Exploit Guard conocidas para Windows 10.
  3. Plataforma de aplicaciones : los contenedores se están convirtiendo en una tendencia moderna para el envasado y la entrega de aplicaciones a diversos sistemas. Al mismo tiempo, Windows Server puede ejecutar no solo aplicaciones nativas de Windows, sino también aplicaciones de Linux. Para hacer esto, Windows Server 2019 tiene contenedores de Linux, el Subsistema de Windows para Linux (WSL) y tamaños de imagen de contenedor significativamente reducidos.
  4. Infraestructura hiperconvergente : le permite combinar arquitectura estándar, informática y almacenamiento en un solo servidor. Este enfoque reduce significativamente el costo de la infraestructura al tiempo que proporciona un excelente rendimiento y escalabilidad.

Centro de administración de Windows


Windows Admin Center (WAC) es una nueva herramienta de administración del servidor. Se instala localmente en la infraestructura y le permite administrar instancias locales y basadas en la nube de Windows Server, computadoras con Windows 10, clústeres e infraestructura hiperconvergente.

WAC complementa en lugar de reemplazar las herramientas administrativas existentes, como mmc, consolas de Server Manager. La conexión a WAC se realiza desde el navegador.



Para completar las tareas, se utilizan WinRM, tecnologías de control remoto WMI y scripts de PowerShell.

Puede publicar WAC y administrar servidores desde fuera del perímetro de la organización. Los servicios de aplicación proxy y autenticación multifactor de Azure AD ayudan a proteger este acceso desde el exterior, y el uso de Microsoft Enterprise Mobility + Security (EMS) le permite otorgar o denegar el acceso según el cumplimiento del dispositivo con las políticas, los riesgos, la ubicación y otros factores. El uso de una aplicación web en lugar del acceso remoto del trabajador para la administración es, en mi opinión, la estrategia correcta para la seguridad.



Analítica del sistema


Windows Server 2019 se ha vuelto más inteligente. Usando la nueva característica de System Insights, se implementa el análisis predictivo para permitir la transición de la gestión reactiva de la flota de servidores proactiva. El modelo de aprendizaje automático tiene en cuenta los contadores de rendimiento y los eventos para predecir con precisión los problemas con espacio libre en el subsistema de disco, identificar tendencias para la informática del procesador, la conectividad de red y el rendimiento del almacenamiento.



Nuevos subsistemas de almacenamiento


Servicio de migración de almacenamiento


Windows Server 2019 introdujo una nueva tecnología para migrar datos de servidores antiguos a servidores nuevos: el Servicio de migración de almacenamiento.

La migración tiene lugar en varias etapas:

  • Inventario de datos en varios servidores.
  • Transfiera rápidamente archivos, carpetas de red y configuraciones de seguridad desde servidores de origen
  • Capturar el control y reemplazar el identificador del servidor y la configuración de red del servidor antiguo al nuevo

Los procesos de migración se administran utilizando el Centro de administración de Windows.



Sincronización de archivos de Azure


Azure File Sync transforma los servidores de archivos tradicionales y expande el almacenamiento a volúmenes prácticamente inalcanzables en la vida real. Los datos se dividen en varios niveles: un caché activo es información almacenada en discos del servidor de archivos y accesible a la máxima velocidad para los usuarios. A medida que los datos se enfrían, los datos se trasladan sin problemas a Azure. Azure File Sync se puede usar junto con cualquier protocolo para acceder a archivos: SMB, NFS y FTPS.

Réplica de almacenamiento


Esta tecnología de protección contra desastres apareció por primera vez en Windows Server 2016. En la versión 2019, apareció un soporte limitado para la edición estándar de Windows Server. Ahora, las pequeñas empresas también pueden hacer una copia automática (réplica) del almacenamiento en máquinas virtuales de Azure si la infraestructura de la empresa no tiene un segundo centro de datos remoto.

Espacios de almacenamiento directo


El espacio en disco local es un componente necesario para construir una infraestructura hiperconvergente y un servidor de archivos escalable. Windows Server 2019 tiene soporte integrado para memoria no volátil, algoritmos de deduplicación mejorados, incluso en volúmenes con el sistema de archivos ReFS, escalabilidad de hasta 4 PB por clúster y mejoras de rendimiento.

Cambios de clústeres de conmutación por error


Han aparecido conjuntos de clústeres, aumentando la escalabilidad a cientos de nodos.
Para garantizar el quórum en clústeres con un número par de nodos, se utiliza un recurso especial: un disco testigo. En los días de Windows Server 2012 R2, era necesario asignar un disco para el almacenamiento de un disco testigo, y en Windows Server 2016 se hizo posible usar una carpeta de red o un disco testigo en la nube. Las mejoras en Windows Server 2019 están asociadas con requisitos de infraestructura reducidos para pequeñas empresas. Un disco testigo puede ser un disco USB conectado, por ejemplo, a un enrutador.

La migración de clúster entre dominios y otras mejoras en el servicio de clúster de conmutación por error han aparecido.

¿Qué hay de nuevo en la plataforma de aplicaciones?


Ahora puede ejecutar contenedores basados ​​en Windows y Linux en el mismo nodo contenedor utilizando el mismo programa de administración Docker. Esto le permite trabajar en un entorno heterogéneo de nodos de contenedor y proporcionar a los desarrolladores la flexibilidad para crear aplicaciones.

Los contenedores recibieron una compatibilidad de aplicaciones mejorada, el tamaño de las imágenes de Server Core y Nano Server se redujo significativamente y se mejoró el rendimiento.

Windows Server 2019 ha recibido muchas otras mejoras. Puede familiarizarse con ellos en la sección de Windows Server del sitio web oficial de documentación docs.microsoft.com.

Requisitos del sistema


La documentación describe los requisitos mínimos para instalar Windows Server 2019. Debe comprender que, dependiendo de los roles y componentes, la ejecución de aplicaciones, los requisitos del servidor pueden aumentar.

  • Procesador de 1,4 GHz de 64 bits
  • Conjunto de instrucciones compatible con X64
  • Soporte NX y DEP
  • Soporte para CMPXCHG16b, LAHF / SAHF y PrefetchW
  • Soporte de traducción de direcciones de segundo nivel (EPT o NPT)

Requisitos mínimos de RAM:

  • 512 MB (2 GB para la opción de instalación "Servidor con escritorio")

Requisitos de almacenamiento:

Windows Server 2019 no es compatible con ATA / PATA / IDE y EIDE para arranque, archivo de intercambio o unidades de datos. La cantidad mínima es de 32 GB.

Adaptadores de red:

  • Adaptador Ethernet con un ancho de banda de al menos 1 GB.
  • Cumple con la especificación de arquitectura PCI Express.
  • Admite el protocolo de arranque remoto PXE.

Comparación de ediciones de Windows Server 2019 Standard y Datacenter


Windows Server 2019 viene en dos ediciones: Standard y Datacenter.

La edición para centros de datos tiene características avanzadas: soporte para infraestructura hiperconvergente, espacios de disco locales, derechos de licencia extendidos cuando se usa la virtualización.
