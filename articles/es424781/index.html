As√≠ que ahora es el momento de ejecutar ignite :

Veamos qu√© significa este c√≥digo.


Motor Engine


La clase ignite.engine.Engine es el marco de la biblioteca, y el objeto de esta clase es el trainer : 


 trainer = Engine(process_function)

Se define con la funci√≥n de entrada process_function para procesar un lote y sirve para implementar pases para la muestra de entrenamiento. Dentro de la clase ignite.engine.Engine , sucede lo siguiente: 


 while epoch < max_epochs: # run once on data for batch in data: output = process_function(batch)

Volver a la funci√≥n process_function : 


 def process_function(engine, batch): model.train() optimizer.zero_grad() x, y = _prepare_batch(batch, device=device) y_pred = model(x) loss = criterion(y_pred, y) loss.backward() optimizer.step() return loss.item()

Vemos que dentro de la funci√≥n nosotros, como es habitual en el caso del modelo de entrenamiento, calculamos las predicciones y_pred , calculamos la funci√≥n de loss , loss y gradientes. Este √∫ltimo le permite actualizar el peso del modelo: optimizer.step() .


En general, no hay restricciones en el c√≥digo de la funci√≥n process_function . Solo notamos que toma dos argumentos como entrada: el objeto Engine (en nuestro caso, el trainer ) y el lote del cargador de datos. Por lo tanto, por ejemplo, para probar una red neuronal, podemos definir otro objeto de la clase ignite.engine.Engine , en el que la funci√≥n de entrada simplemente calcula las predicciones e implementa una pasada a trav√©s de la muestra de prueba una vez. Lea sobre esto m√°s tarde.


Entonces, el c√≥digo anterior solo define los objetos necesarios sin comenzar el entrenamiento. B√°sicamente, en un ejemplo m√≠nimo, puede llamar al m√©todo:

Algunos detalles
Veamos un poco el c√≥digo anterior. El lector puede haber notado la siguiente l√≠nea de c√≥digo:

y probablemente hubo una pregunta sobre el tipo de objeto obtenido de train_evaluator.run(train_eval_loader) , que tiene el atributo de metrics .


De hecho, la clase Engine contiene una estructura llamada state (tipo State ) para poder transferir datos entre controladores de eventos. Este atributo de state contiene informaci√≥n b√°sica sobre la era actual, la iteraci√≥n, el n√∫mero de eras, etc. Tambi√©n se puede usar para transferir cualquier dato de usuario, incluidos los resultados del c√°lculo de m√©tricas.
