第一部分，理论


（指导音）：每个人都知道现代路由器应该在两个频段上工作。2.4和5 GHz。甚至没有讨论。在这种情况下，建议仅将路由器与802.11ac一起使用，因为在几个房间中5 GHz的802.11n并没有太大帮助，因为该范围内的信号会由于障碍物而衰减或减弱。多亏了802.11ac，由于最初使用一堆新算法可以达到更高的理论速度，因此即使衰减很大，也可以达到可接受的实际速度。

除了包装盒上的标准外，通常还会写上诸如N600，AC1200，AC1750等的名称。这是路由器无线网络可以工作的最大速度。但是，结果却很棘手-通过对不同范围内的最大连接速度求和。例如，命名为N600的上一代路由器具有2T2R方案，并支持以300 Mb / s（150 + 150）的速率将客户端连接到2.4 GHz频带中的两个空间流，以及类似地在5 GHz时高达300 Mb / s。总计600 Mbps。 AC1750路由器通常在2.4 GHz频段（150 + 150 + 150）中具有三个流，而在5 GHz频段（433 + 433 + 433）中具有三个流。总计1750。

首先要注意的是：当然，这与路由器无关，与路由器无关，甚至与内置的以太网交换机无关。 ACxxxx公式仅表示内置的Wi-Fi接入点。如果只有千兆位WAN端口的路由器可以发出/泵出1750 Mbit / s的速度，那么很少有人会出于某种原因进行思考。同样，市场上有许多路由器的无线公式为AC750 / 1200，但具有用于连接Internet的100 MB端口。

第二个体系结构警告：在任何实际任务中都不能总结ACxxxx公式的速度，因为至少在实际路由（读取Internet连接）中，即使在交换（读取家庭网络中的传输）中，ACxxxx公式的速度始终会成为瓶颈。假设我们有一个千兆路由器（带有千兆WAN端口和以太网交换机），在千兆Internet连接上具有公式AC1750。如果无线客户端非常适合此公式（AC 3x3-并且您仍然必须寻找这样的公式），则可以从Internet获得最大速度，在本地网络上，您获得的速率将不会超过1000 Mbps。从一个这样的无线客户端转移到另一个无线客户端时，理论上您甚至可以仅依赖于1300 Mbit / s的一半。如果您在网络之间进行转移，4和5 GHz-共同点是2.4 GHz的客户端速度。

第三点警告（许多Wi-Fi用户通常已经知道）：任何声明的Wi-Fi速度都是一对路由器（接入点）-客户端（无线适配器）中的通道连接速度。该速度已纳入我们正在讨论的公式中，适配器和路由器本身在其接口中显示了该速度，但是，在速度测试，下载torrent或下载文件中可以感觉到的实际数据传输速度通常不在实验室条件下，大致来说是一半（为简单起见，请考虑由于协议开销和不可避免的干扰。如果接入点必须与两个客户端进行通信（例如，从一个客户端到另一个客户端的传输），则实际速度会降低一半（以下有解释）。另一方面，AC 2x2点将能够将500兆位互联网分发给两个2x2 AC客户，每次给每个大约200 Mbit / s（请记住，其理论/公式最大值为867 Mbit / s，实际吞吐量为400-500 Mbit / s）。

从这里可以清楚地知道拥有100兆位WAN端口的AC1200型号的权利。想象一下现在和未来2-3年的情况：电视通过Wi-Fi传输大量内容（例如，来自NAS的高质量视频流），并行进行备份，几台笔记本电脑正在上网，有人正在下载种子（我什至没有提到智能手机） ）这里的互联网最大为100兆比特，其余的恰好高达300 Mbit / s，而AC867 5 GHz中可能只有400 Mbps。网上冲浪也可以通过2.4 GHz连接。当然，在具有100兆位交换机的路由器上安装NAS通常是不明智的。

理想情况下，对于不同的设备而言，最好以不同的流连接到路由器，以免影响速度。然而，他们如何真正联系是一个复杂的问题。因此，线程越多-设备空闲的可能性就越大。每个流的速度越高，即使设备在同一频道上工作，设备也将以足够的速度工作的可能性越大。重要的一点：100-150兆位足以从网络外部（例如，从提供商）输入信息，但是当几个生产性设备开始在网络内部工作时，它们变得很少。

值得一提的是所谓的三频路由器。现在，这些设备实际上具有三个接入点：一个在2.4 GHz上，两个在5 GHz上不重叠（同时，802.11ad标准中具有第三个60 GHz频段的型号已经出现，但这是一个完全不同的故事，因为有多少用户没有真正体验过5 GHz）。如果需要最大速度的设备（例如高性能笔记本电脑和计算机）连接到5 GHz的不同点，则这种路由器具有优势。但是即使在这种情况下，考虑到上面所说的，也不能指望所承诺的公式“ AC_to_infinity”，而只能指望其中一个的最大速度。

但是，即使您拥有3x3适配器（如Macbook Pro中的适配器），或者甚至是极为罕见的4x4，您也应记住，经典的Wi-Fi技术在为客户端设备提供服务时仍提供清晰的时间间隔：接入点进行传输（或接收）数据的必要部分，首先是来自一个客户端，然后是来自另一个客户端，依此类推。同时，它不能一次传输到两个客户端。所有这一切都需要时间，结果，真正的高速客户端会失去速度，变慢，事实证明他们浪费了点资源。

这种情况无法持续，并且几年来我们已经获得了MU-MIMO技术，从理论上讲，它应该允许同时传输到多个客户端。但是，其实施仍然停滞不前。首先，她有很多局限性。例如，真正同时服务的设备的最大数量限制为四个，并且不支持来自客户端的同时回发（对于监视摄像机和积极分发种子的用户来说，这是一个关键的减号）。其次，在宣布MU-MIMO的许多路由器中，它实际上是行不通的。第三，它应该由很少的客户端自己支持（这里是3x3的问题）

因此，基于上述所有内容，现在最好的选择是具有千兆端口的路由器，该端口可根据802.11ac标准工作，并且总速度至少为AC1200。
