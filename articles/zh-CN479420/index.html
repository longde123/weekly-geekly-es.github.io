打开麦克风：后端。 来宾注册



12月12日，我们邀请学生参加致力于后端开发的“开放式麦克风”。 我们从站立的人那里借用了这个想法，并希望帮助演讲者在轻松的氛围中“浏览”技术报告。 我们将在19:00聚集在Badoo的办公室（莫斯科，Tsvetnoy大道2号）。

该活动旨在帮助初学者和实践演讲者查看演示文稿的主题并获得同事的反馈。 因此，如果您对声明的主题感兴趣，则想讨论它们并分享建议，请注册 。

我们厨房的座位数量非常有限，我们希望收集最感兴趣的来宾，他们会为您提供建设性的高质量反馈，因此可以轻松进行应用程序的预审核。

该格式将由五位发言人进行测试，他们只有十分钟的时间表达他们的想法。 细节剪下。

程序


主题№1


关于演讲者：Alexander Lobtsov， Lamoda，技术交流客户交流，开发人员和好人。

关于主题：与用户的全渠道通信体系结构

  • 沟通中的客户旅程
  • 企业如何看待它
  • 正如发展所见
  • 共同点

个人动机：有制定和实施的计划，我们需要反馈和建议-通常，这是对笔的测试以及您可以在会议上开始演讲的主题。

主题№2


关于演讲者： VTB .NET开发人员Roman Grishin 。 自2013年以来从事IT。已婚，育有一女。

关于主题：

  • 整体架构最适合哪种情况
  • 微服务不是黄金药。 只会伤害他人的情况

个人动机：获得与听众交谈的经验和对该主题的反馈

主题№3


关于演讲者：Trifonov Max，已经发展了20多年。 哪里可以找到的创始人和项目经理（用于电子商务的API服务，b2b，公开版本已准备就绪）。

关于主题：

  • 用于API的json-rpc
  • 后端架构
  • 异步PHP（作为后端API的一部分）
  • 共享内存
  • 后端请求分配策略（云VPS-省钱）
  • 部署和监控

个人动机：了解如何完全公开地表达您的想法并获得他们的评估（包括堆栈，实施以及服务本身）。远离并调整演示文稿本身。

主题№4


关于发言人：Dmitry Shagarov，从事IT工作超过15年。 数据工程，研究和DevOps的主要活动。

关于主题：

  • 在实践中可以用作配置管理系统
  • 日常DevOps任务的自动化
  • 关于IaC（基础架构即代码）概念的知识很少。

个人动机： “不要在会议和聚会上喝免费的咖啡，现在是时候偿还社区的技术债务，开始播种理性，良好和永恒的种子了。 我认为您可以从打开麦克风开始。”

主题№5


演讲者简介：美国一家主要对冲基金的首席开发商Mikhail Smurov 。 在分布式团队中进行外包工作。

关于主题：

  • 如何在大型项目中构建后端架构
  • 为什么微服务在高负载系统中很好
  • 为什么机器学习并不总是对企业有利。

个人动机：尽可能在简洁的框架中进行检查，以谈论此类项目中的复杂系统和管理模型。

值机在这里 。 19:00聚会。
请等待参与确认（我们将发送至注册时指示的邮件）。

待会见！
