Mengapa aplikasi remote control perlu mengetahui lokasi saya?

Berikut ini adalah pola menarik dari perilaku aplikasi yang tidak ramah yang dapat dengan mudah dihindari jika programmer dan pengusaha beralasan seperti orang biasa.

Saya memiliki sound system Pioneer / Onkyo. Ini adalah teknik yang baik yang datang dengan aplikasi remote control Android (tidak terlalu jelek).

Setelah aplikasi diperbarui dan menunjukkan log perubahan standar dengan "perbaikan bug dan peningkatan", tetapi ketika dibuka muncul:



Tuhan, mengapa remote control tahu lokasi saya? Saya memutuskan bahwa ini adalah karena omong kosong pemasaran, jadi saya menolak. Kemudian aplikasi menolak untuk bekerja.


Mengesampingkan pelanggaran tata bahasa berbahaya di jendela sembul. Kami memiliki situasi: program digunakan untuk bekerja dengan baik, tetapi sekarang sudah rusak, karena pengembang tidak berpikir bahwa seseorang akan berani menolak untuk meminta data rahasia.

Seperti beberapa pengguna lain, saya mengirim keluhan. Beberapa minggu kemudian mereka menerbitkan pembaruan:
APA YANG BARU Indikasi perubahan izin untuk menunjukkan lokasi perangkat hanya jika perlu.

* Saat menggunakan Android 4 atau 5, indikasi terjadi ketika menginstal aplikasi atau memperbarui ke versi terbaru.

* Mengapa saya perlu lokasi perangkat? Jawab: Untuk mengatur perangkat nirkabel di sekitar Anda memerlukan data SSID dan titik akses. Informasi lokasi perangkat tidak digunakan untuk tujuan lain apa pun.

Aplikasi baru berfungsi bahkan jika Anda melarang pelacakan. Ternyata!

Alasan nomor 1


Kesalahan pertama dilakukan oleh pengembang Pioneer / Onkyo. Mereka seharusnya menunjukkan pesan seperti "Hei, kami ingin tahu lokasi Anda, jadi kami bisa ...", dan kemudian membiarkan pengguna membuat pilihan berdasarkan informasi apakah akan memberikan informasi atau tidak.

Jika pengguna menolak, berikan beberapa cara lain untuk melakukan operasi secara manual.

Alasan nomor 2


Wise Google telah menautkan "lokasi" ke fitur-fitur seperti pemindaian BLE dan pemindaian WiFi, karena secara teoritis mereka dapat digunakan untuk menghitung lokasi. Artinya, jika Anda berada di dekat jalur akses X, maka Anda dapat menganggap lokasi Anda.

Bagus, tentu saja. Tetapi apakah mungkin untuk mengajukan pertanyaan yang lebih dimengerti, misalnya: "Apakah Anda ingin mengaktifkan pemindaian WiFi? Ini mungkin mengungkapkan lokasi Anda. " Ini mungkin bukan formulasi terbaik, tetapi situasi saat ini benar-benar membingungkan dan membuat marah pengguna.

Ini adalah keluhan umum tentang Android. Dan dalam waktu dekat situasinya tidak mungkin untuk diperbaiki.

Izin bicara lancar


Pada zaman Nokia, aplikasi meminta izin secara tiba-tiba . Bercanda bahwa browser web akan meminta izin untuk terhubung ke jaringan, dan kemudian bertanya apakah Anda benar - benar ingin terhubung ke situs web yang aman.

Pikirkan aku bercanda? Ini adalah situasi umum pada Symbian:


Android dan Apple seharusnya membebaskan kita dari kerja keras ini. Dengan moderasi direktori yang menyeluruh dan aplikasi yang diperiksa ulang, tidak perlu mengganggu pengguna dengan permintaan izin.

Kemudian terjadilah:


Pengembang menyalahgunakan kepercayaan mereka dan mulai mendorong izin yang tidak perlu ke dalam program mereka untuk mendapatkan keuntungan dari pengguna.

Orang tidak mau memikirkannya. Direktori aplikasi tidak ingin mengontrol aplikasi. Pengembang tidak ingin membuat masalah bagi pengguna. Regulator tidak ingin data dikirimkan tanpa persetujuan. Pengguna mengatakan mereka ingin mengelola izin, tetapi mereka tidak benar-benar memahami konsekuensinya.

Benar-benar rumah gila! Mungkin mencari baterai AA untuk remote konvensional ...?
