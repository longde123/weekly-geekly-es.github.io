值类别



从C ++ 11开始，任何表达式只能属于以下三个值类别之一：lvalue，xvalue或prvalue：


  • xvalue（到期值）-表达式，它标识可以移动的非临时对象
  • 左值（左值）-表达式，它标识函数的非临时对象，该对象不能移动
  • prvalue（纯右值）-表达式，用于初始化对象


这三个类别可以分为三个重叠的组：


  • glvalue（广义左值）将左值和左值分组。 glvalue在内存中具有地址（具有标识），因此通常可以分配一个值（如果不是const）
  • 右值将prvalue和xvalue分组。 rvalue可以移动（xvalue）或根本不属于现有对象（prvalue）。 可以将右值传递给移动构造函数，移动赋值运算符或函数

因此，xvalue在内存中有一个地址，可以移动。



从上表中可以看到，左值，左值引用和右值引用类别的命名变量都具有相同的表达式类别：左值（以红色突出显示）。 例如，这意味着，当右值引用传递给函数时，将选择左值引用重载： T&& x=T(); f(x); T&& x=T(); f(x);

本文中的缩写


构造函数，运算符和析构函数的缩写：


  • DC-默认构造函数
  • PC-参数化构造函数
  • CC-复制构造函数
  • Ca-复制分配（可以重复使用分配的存储，例如用于std :: string）
  • Mc-Move构造函数
  • 马-搬家作业
  • Va-转化分配运算符
  • D-析构函数

值类别的缩写：


  • LV-Lvalue： T LV;
  • LR-左值参考： T& LR = LV;
  • XV-Xvalue： move(LV)
  • PRV-纯右值。 函数调用的文字或结果，返回类型不是引用： T f() { return T(); } T f() { return T(); }
  • FR-转发参考： auto&& FR = LV;
  • CLV-常量LV。 CLR-常量LR。 CXV-常量XV。
  • CPRV-常量PRV。 CPRV仅适用于类和数组。 像cont int这样的非类非数组CPRV将像int一样隐式转换为PRV，例如在函数const int f()返回值中）
  • RV是XV或PRV
  • CRV是CXV或CPRV

其他缩写：


  • PF-完美转发（使用模板化转发引用将参数传递给功能）

分配值类别


值类别的分配可以视为以下所有部分的基础。 在不进行显式类别转换的情况下将一个类别显式分配给另一类别时，将发生隐式转换： FROM_TYPE x; TO_TYPE y; y = x; FROM_TYPE x; TO_TYPE y; y = x; 在下表中，您可以看到将From类型的表达式分配给To类型的变量时会发生什么：


  • 减号表示不可能进行这种分配
  • 加号表示可以进行这种分配，并且不需要调用复制/移动运算符或构造函数。
  • 在其他情况下，会有文本，这意味着在这种分配过程中将调用哪些复制/移动操作符和构造函数。

从返回的表达式类别到函数返回值类型的隐式转换不能用红色字体标记-有关详细信息，请参见下面的“从函数返回”部分。

2-创建的用于保存引用初始化程序的临时对象将一直保留到其引用范围的结尾（请参阅下一节）。 无法将返回值转换为函数类型。
3-传递文字时，需要在右侧输入auto的类型说明符： auto&& = T{2};


注意事项：


  • 非常量左值和右值引用不能接受任何常量类型。 但是非const非引用类型可以接受const类型。
  • 右值引用（T &&）可以接受xvalue或prvalue（XV或PRV）。
  • 左值参考（T＆）可以接受左值或左值参考（LV或LR）
  • 非引用类型通过转换，复制，移动或RVO接受任何类型
  • const左值引用（const T＆，const auto＆）和自动转发引用（auto &&）接受任何类型，而无需复制或移动
  • 常量和非常量右值引用（T &&，常量T &&）不能接受常量或非常量左值（LV或CLV）。
  • LR不能接受常量或非常量右值引用（XV或PRV或CXV或CPRV）。 CLR可以接受它们。
  • 具有const修饰符的类型不能分配给没有const修饰符的任何类型（auto除外，如果const在右侧显式，则可以变为const）。
  • 传递prvalue（传递返回值，文字或构造函数结果）与传递xvalue相同，但在分配给非引用类型时避免调用move构造函数
  • 传递const prvalue（传递const类型函数的返回值）与传递const xvalue相同，但在分配给非引用类型时避免调用副本构造函数
  • 自动引用可以成为const，因此它们可以接受const和非const变体。 而且，它们可以成为const引用，以使临时持久化直到引用范围结束。 但是，如果未在右侧明确指定，则auto *引用将无法成为const。 这就是为什么您不能将rvalue分配给auto＆的原因
  • const引用可以接收一个临时值，并使它持久直到引​​用范围的结尾（非const引用不能）。 这仅在当前作用域（局部函数引用和函数参数）内起作用。 如果reference是类的成员，则临时对象将在构造函数或函数的结尾处被破坏，并且引用将继续指向被破坏的对象，这是未定义的行为。
  • Auto x = CLV; 将推断出auto为非const类型。

用临时对象初始化常量引用


C ++允许使用临时对象初始化常量引用。 在这种情况下，将延长临时对象的寿命。 范例： 


 struct T { int i = 1; }; const T& t = T(); cout << ti;

但是，此生存期扩展仅在创建临时对象的块末尾有效。 因此，如果在构造函数中用临时对象初始化了类的常量引用成员，则临时对象将在构造函数的结尾处被破坏，并且引用将继续指向被破坏的对象，这是未定义的行为： 


 class A { public: // Will not compile: value-initialization of reference type //A() : t() {} const T& t; }; class B { public: // Will compile in some compilers, but temporary object will be destructed at the end of constructor B() : t(T()) { cout << "In constructor: " << ti << endl; } const T& t; }; class C { public: // Will compile, but temporary object will be destructed at the end of constructor // Address sanitizer will show the problem C() : t(std::move(T())) { cout << "In constructor: " << ti << endl; } const T& t; }; C c; cout << "C: " << cti << endl;

没有地址清理器，该程序将输出一些垃圾，而使用地址清理器，将显示错误。 因此，不应使用C ++功能或应谨慎使用。
