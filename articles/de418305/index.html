Wie viele Objekte gibt Python beim Ausführen von Skripten aus?

Einige Python-Programmierer sind sehr überrascht, wenn sie herausfinden, wie viele temporäre Objekte der Python-Interpreter zuweist, während ein einfaches Skript ausgeführt wird.

Mit CPython können Sie Statistiken zu zugewiesenen Objekten abrufen. Dazu müssen Sie diese mit zusätzlichen Flags kompilieren. 

./configure CFLAGS='-DCOUNT_ALLOCS' --with-pydebug make -s -j2

Lassen Sie uns die Schlussfolgerung lesbarer machen: 

 def print_allocations(top_k=None): allocs = sys.getcounts() if top_k: allocs = sorted(allocs, key=lambda tup: tup[1], reverse=True)[0:top_k] for obj in allocs: alive = obj[1]-obj[2] print("Type {}, allocs: {}, deallocs: {}, max: {}, alive: {}".format(*obj,alive))

Wo:

  • Zuweisungen - Wie viele Objekte wurden seit dem Start des Interpreters zugewiesen?
  • Freigabe - wie viele Objekte wurden gelöscht (manuell oder automatisch)
  • lebendig - die Anzahl der lebenden (aktuellen) Objekte (Zuordnungen - Freigaben)
  • max - die maximale Anzahl lebender Objekte seit dem Start des Dolmetschers

Wie Sie sehen können, konnte die leere Python-REPL 17.328 Zeilen und 10.550 Tupel zuweisen. Das ist eine verrückte Menge an Objekten! Hier müssen Sie berücksichtigen, dass Python automatisch zusätzliche Module importiert, die bei leeren Skripten nicht importiert werden, damit REPL funktioniert.

Frames und Codeobjekte


Im obigen Beispiel finden Sie viele Frame- und Codeobjekte. Warum werden sie gebraucht?

Kurz gesagt, jedes Codeobjekt speichert in sich einen Block kompilierten Codes, der wiederum von Rahmenobjekten ausgeführt wird, die nach dem Prinzip eines Aufrufstapels arbeiten . In Python ist der beliebteste Block eine Funktion. Jede neue Funktion benötigt ein eigenes Codeobjekt, und jeder Aufruf dieser Funktion benötigt ein separates Frame-Objekt, in dem Python lokale Variablen speichert. Zusätzlich zu den lokalen Variablen speichert jedes Rahmenobjekt viele Hilfsdaten, die zur Ausführung der Funktion benötigt werden.

Woher kommen all diese Objekte?


Python ist eine sehr dynamische Sprache und Sie müssen dafür bezahlen. Um dynamische Funktionen zu unterstützen, erstellt er eine große Anzahl temporärer Objekte, die eine unterstützende Rolle spielen.

Wenn Sie beispielsweise eine einfache Funktion deklarieren, werden mindestens 5 Wörterbücher, 5 Tupel und 4 Listen erstellt. Diese Objekte bleiben bis zum Ende des Skripts erhalten. Alle diese Objekte speichern wiederum andere Objekte (ihre Elemente) in sich selbst. Dies sind Dutzende, manchmal Hunderte von zusätzlichen Objekten, die für die interne Beschreibung der kompilierten Funktion verwendet werden. Die Beschreibung der Durchschnittsklasse kann Hunderte von Containerobjekten (Wörterbücher, Tupel, Listen) hervorheben. Leider ist es hier nicht möglich, die genaue Anzahl der zugewiesenen Objekte automatisch zu berechnen, und diese Zahlen sind ungefähr.

Damit Python schnell eine große Anzahl von Objekten zuordnen kann, wird ein großes und mehrschichtiges System verwendet , das die Zuordnung von Objekten im Speicher optimiert.

Manchmal fragt man sich, wie viele Details die interpretierten Sprachen vor uns verbergen. Mit Python können Sie guten Code schreiben, ohne über viele Probleme und Details nachdenken zu müssen.
