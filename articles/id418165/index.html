Quasar, Sobaken dan Vermin: mengungkap detail kampanye mata-mata cyber yang sedang berlangsung

Menggunakan alat akses jarak jauh, Quasar, Sobaken, dan Vermin, penjahat cyber memantau agen pemerintah Ukraina dan mencuri data dari sistem mereka. Cybergroup ini pertama kali disebutkan dalam laporan pada Januari 2018, menarik perhatian ESET pada pertengahan 2017 dan hari ini terus mengembangkan perangkat lunaknya.

Dalam laporan ini, kami mengungkapkan rincian kampanye saat ini, memberikan informasi tentang malware dan menjelaskan metode yang digunakan penyerang untuk menyebar, menargetkan, dan menghindari deteksi.


Profil Penyerang


Grup ini tampaknya tidak memiliki pengetahuan teknis yang luar biasa atau akses ke kerentanan zero-day. Namun, ia telah berhasil menerapkan rekayasa sosial untuk menyebarkan malvari dan pekerjaan rahasia untuk waktu yang lama.

Kami berhasil melacak pekerjaannya hingga Oktober 2015, tetapi ada kemungkinan bahwa grup memulai aktivitasnya lebih awal.

Penyerang menggunakan tiga modifikasi .NET malvari: Quasar RAT (Remote Administration Tool), Sobaken (turunan dari RAT dari Quasar) dan RAT Vermin khusus. Alat-alat secara bersamaan digunakan pada target yang sama, mereka berbagi infrastruktur dan terhubung ke server C&C yang sama. Penjelasan yang mungkin untuk penggunaan tiga modifikasi paralel adalah fakta bahwa mereka dikembangkan secara independen satu sama lain.

Para korban


Perangkat lunak berbahaya digunakan dalam serangan terhadap lembaga pemerintah Ukraina. Menurut telemetri ESET, ratusan korban di berbagai organisasi dan beberapa ratus file yang dapat dieksekusi terkait dengan kampanye ini dicatat.

Distribusi


Menurut data telemetri kami, penyerang menggunakan email sebagai saluran distribusi utama untuk tiga RAT. Mereka menggunakan rekayasa sosial untuk meyakinkan korban untuk mengunduh dan menjalankan malware.

Dalam kebanyakan kasus, nama file ditulis dalam bahasa Ukraina dan terkait dengan pekerjaan para korban. Berikut adalah contoh file tersebut:

  • "PETUNJUK untuk Mengatur Perlindungan Dinas Militer Ukraina dan anggota Keluarga itu" ("Perintah untuk memastikan keamanan personil militer tentara Ukraina dan anggota keluarga mereka")
  • "Aku akan menghukum draf baru, penunjukan revisi Viluchennya" ("Draf baru dari perintah untuk memeriksa penarikan")
  • "Donning OVK Zb—ñlshennya limitu" ("Divisi Pengadaan Don HVAC. Meningkatkan batas kredit")

Selain teknik dasar rekayasa sosial (menarik perhatian investasi), penyerang menggunakan tiga metode teknis. Ini kemungkinan akan semakin meningkatkan kinerja kampanye.

Metode # 1 : Aplikasi email menggunakan karakter override Unicode kanan-ke-kiri , yang mengubah arah pembacaan karakter, untuk menyembunyikan ekstensi sebenarnya. Sebenarnya, ini adalah file yang dapat dieksekusi menggunakan ikon Word, Excel, PowerPoint atau Acrobat Reader, agar tidak menimbulkan kecurigaan.

Contoh nama file: seperti yang ditunjukkan pada Gambar 2, "Memindahkan kayu bakar padat (kayu bakar) untuk pembakaran tanpa perlindungan _ <> xcod scr" ("Mengangkut kayu bakar untuk pemanasan") akan terlihat seperti file .DOCX untuk pengguna yang kurang perhatian.


Gambar 2. File yang dapat dieksekusi yang menyamar sebagai dokumen Word

Metode 2 : Aplikasi email yang menyamar sebagai arsip ekstrak-sendiri RAR.

Contoh: e-mail dengan aplikasi "Punishment_MOU_Dodatka_do_Instrucii_440_ost.rar" ("Urutan Kementerian Pertahanan, Lampiran Instruksi No. 440"), seperti yang ditunjukkan pada Gambar 3. Di dalam arsip RAR ada file yang dapat dieksekusi yang disebut "Punishment_MOU_Dodatki_do__struktur_do__struktur _do__struktur _do__struktur _ _ _ _ _ _ _.

Agaknya, korban menjalankan file ini, menunggu pembongkaran lebih lanjut dari isi arsip self-extracting, tetapi tanpa disadari meluncurkan file berbahaya yang dapat dieksekusi.

Metode # 3 : Dokumen Word + mengeksploitasi CVE-2017-0199. Kerentanan ini diterapkan ketika korban membuka dokumen Word yang dibuat khusus. Proses Word mengirimkan permintaan HTTP ke file HTA yang berisi skrip berbahaya yang terletak di server jarak jauh. Kemudian skrip berbahaya diluncurkan oleh mshta.exe . Informasi publik pertama tentang kerentanan ini muncul pada April 2017 , dan Microsoft menutupnya dengan merilis pembaruan keamanan untuk semua versi Windows dan Office.

Menurut telemetri ESET, penyerang ini mulai menggunakan metode ini pada Mei 2017. Para penyerang menggunakan hxxp: // chip-tuning lg [] ua / untuk mengirimkan file HTA dan muatan akhir.

Instalasi dan Kegigihan


Prosedur instalasi sama untuk tiga modifikasi malvari yang digunakan oleh grup ini. Dropper membuang file payload (Vermin, Quasar atau Sobaken) ke folder %APPDATA% , ke subfolder dengan nama perusahaan yang sah (biasanya Adobe, Intel atau Microsoft). Kemudian, seperti yang ditunjukkan pada Gambar 4, ia membuat tugas dalam penjadwal untuk menjalankan komponen setiap 10 menit untuk memastikan ketekunan. Beberapa versi juga menggunakan metode menggunakan fungsi pintas Panel Kontrol Windows untuk membuat folder mereka tidak dapat diakses dari Windows Explorer. Folder-folder ini tidak akan terbuka ketika diklik di Windows Explorer, sebagai gantinya, halaman "Semua Tugas" terbuka.

Penargetan


Penyerang menggunakan banyak trik agar malware berfungsi hanya pada mesin target. Mereka sangat berhati-hati untuk menghindari sistem analisis dan kotak pasir otomatis.

Metode # 1 : memeriksa tata letak keyboard di Windows
Malvar memeriksa apakah tata letak keyboard Rusia atau Ukraina diinstal. Jika tidak, pekerjaannya segera berhenti.

Metode # 2 : Verifikasi Alamat IP
Malvar mendapatkan alamat IP komputer yang terinfeksi melalui permintaan ke layanan yang sah
Ipinfo.io/json. Pekerjaan akan selesai jika alamat IP tidak ditemukan di Ukraina atau Rusia, atau jika terdaftar di salah satu dari beberapa penyedia layanan cloud yang dipilih atau pengembang perangkat lunak anti-virus. Kode yang melakukan verifikasi ditunjukkan pada hasil pembongkaran pada Gambar 5 dan 6.

Metode 3 : Menguji Emulasi Jaringan
Sistem analisis otomatis sering menggunakan alat-alat seperti Fakenet-NG, di mana komunikasi DNS / HTTP berhasil dan menghasilkan hasil apa pun. Penulis Malvari mencoba mengidentifikasi sistem tersebut melalui pembuatan nama situs / URL acak dan memeriksa bahwa tidak ada koneksi ke URL ini (pada Gambar 7), karena itu akan berada pada sistem nyata.

Aplikasi Steganografi


Sejak pertengahan 2017, penyerang telah menggunakan steganografi, menyembunyikan komponen berbahaya dalam gambar di situs hosting gratis saveshot.net dan ibb.co.

Steganografi adalah ilmu yang memungkinkan Anda untuk menyembunyikan data "dalam tampilan penuh", di dalam informasi lain yang tidak diklasifikasi. Dalam kasus kami, file EXE berbahaya dienkripsi dan disembunyikan dalam file JPEG, seperti pada Gambar 9. Malvar mengunduh dan mendekodekan JPEG, mengekstrak data tersembunyi, mendekripsi file EXE dari data ini dan meluncurkannya.


Gambar 9. Contoh gambar JPEG yang digunakan untuk menyembunyikan komponen berbahaya yang terkirim (diubah ukurannya dan dihapus)

Proses dekripsi cukup rumit dan dapat digambarkan sebagai berikut:

1. Mengunduh file JPEG dari URL yang dikodekan dalam biner.

2. Kekuatan kasar dari kata sandi delapan digit dengan menghitung hash-nya dan merekonsiliasi dengan hash yang di-hardcode dalam file biner yang diunduh. Langkah ini menggunakan CPU secara intensif, dibutuhkan lebih dari 10 menit untuk menyelesaikannya di komputer biasa. Kemungkinan besar, ini adalah langkah lain untuk melawan sistem analisis malware otomatis.

3. Memproses file JPEG dan mengekstraksi data yang tersembunyi di dalamnya, seperti yang dapat dilihat dari kode yang dibongkar pada Gambar 10 dan 11. Algoritma yang digunakan oleh malware sangat mirip dengan yang digunakan di JSteg, salah satu algoritma steganografi tertua dan paling sederhana untuk file JPEG. Ini menyembunyikan data dalam LSB (bit paling tidak signifikan) dari koefisien transformasi cosine diskrit dari file JPEG. Data tersembunyi seperti itu biasanya tidak memengaruhi gambar yang terlihat oleh mata telanjang, tetapi algoritma khusus dapat dengan mudah mendeteksi keberadaannya. Namun, algoritma steganografi ini sangat mudah diimplementasikan, yang mungkin merupakan alasan untuk dipilihnya oleh penulis malvari.

4. Ekstraksi dan pembongkaran data menggunakan GZip.

5. Dekripsi data yang terkompresi pada AES dengan kata sandi yang diperoleh pada langkah No. 2.

6. Decoding base64 mendekripsi data.

7. Menulis file EXE ke disk dan eksekusi.

Akibatnya, penulis ancaman ini meninggalkan ide untuk menggunakan steganografi dan mulai menggunakan hxxp: // chip-tuning lg [] ua untuk mentransfer file yang dapat dieksekusi yang tidak terenkripsi.

Malvari Mods


Penyerang ini menggunakan tiga modifikasi malvari dalam serangan mereka. Di bawah ini kami memberikan gambaran singkat tentang masing-masing dan berkonsentrasi pada deskripsi fitur karakteristik.

Quasar


Quasar adalah RAT sumber terbuka yang tersedia di GitHub. Kami melihat beberapa kampanye di mana cybergroup ini menggunakan file biner Quasar RAT. Kampanye yang diketahui pertama berlangsung dari Oktober 2015 hingga April 2016. Yang berikutnya diadakan pada Februari 2017. Artefak kompilasi menunjukkan jalan ke PDB n:\projects\Viral\baybak_files_only\QRClient\QuasarRAT-master\Library\obj\ Release\Library.pdb

Kampanye lain dengan RAT Quasar, menggunakan server perintah para penyerang ini (mailukr.net), dilaksanakan pada bulan Juli - September 2017. Para penyerang menggunakan versi lama Quasar RAT yang disebut "xRAT 2.0 RELEASE3". Artefak penyusun di jalur penetes menunjukkan ke PDB N:\shtorm\WinRARArchive\ obj\Release\WinRAR.pdb

Tersedu-sedu


Sobaken adalah versi yang dimodifikasi secara signifikan dari RAT Quasar. Jika Anda membandingkan struktur program Quasar dan Sobaken, Anda dapat mengamati banyak kesamaan - lihat Gambar 12.

Para penulis Sobaken mengurangi fungsionalitas malware, sehingga file yang dapat dieksekusi menjadi lebih kecil dan lebih mudah disembunyikan. Mereka juga menambahkan trik untuk memotong kotak pasir dan lainnya yang dijelaskan di atas.


Gambar 12. Evolusi terselubung. RAT kiri Quasar v1.3, di tengah dan kanan - dua versi Sobaken

Kuman


Vermin adalah backdoor khusus yang hanya digunakan cybergroup ini. Untuk pertama kalinya, malware tersebut didokumentasikan dalam laporan Palo Alto Networks Januari 2018. Backdoor muncul pada pertengahan 2016 dan masih digunakan. Seperti Quasar dengan Sobaken, ia ditulis dalam .NET. Untuk mempersulit analisis, kode dilindungi menggunakan sistem perlindungan kode .NET Reactor komersial atau pelindung sumber terbuka ConfuserEx.

Selain itu, seperti Sobaken, ia menggunakan Vitevic Assembly Embedder, perangkat lunak gratis untuk menyematkan DLL yang diperlukan dalam file yang dapat dieksekusi utama, yang tersedia di Visual Studio Marketplace.

- Rekaman audio

Alat Perekaman Audio (AudioManager)


Komponen Vermin lengkap yang dapat merekam suara dari mikrofon komputer korban. Ia menerima tiga perintah Vermin: StartAudioCapture, StopAudioCapture dan SetMicVolume. Data yang diterima dikompres menggunakan codec Speex dan diunggah dalam format SOAP ke server Vermin C&C.

Keylogger (KeyboardHookLib)


Keylogger di Vermin adalah file executable sederhana yang terpisah yang memotong semua penekanan tombol dan menulisnya ke file dalam bentuk terenkripsi. Itu juga menulis isi buffer dan nama-nama windows aktif. Keylogger itu sendiri tidak dapat berkomunikasi dengan server Vermin C&C; Backdoor utama digunakan untuk mengirimkan informasi yang dikumpulkan.

Jalur menuju PDB di keylogger mengonfirmasi koneksi dengan malware Vermin:
Z:\Projects\Vermin\KeyboardHookLib\obj\Release\AdobePrintLib.pdb

Pencurian Kata Sandi (PwdFetcher)


Komponen pencurian kata sandi Vermin terpisah digunakan untuk mengekstrak kata sandi yang disimpan dari browser (Chrome, Opera). Bagian utama dari kode ini sepertinya disalin dari sebuah artikel dari Habr . Beberapa sampel juga mengandung kode untuk mengekstraksi informasi dari browser Firefox, tetapi sepertinya tidak digunakan. Seperti yang ditunjukkan pada Gambar 13, komponen ini juga mengandung jalur PDB yang mirip dengan komponen keylogger, yang mengkonfirmasi koneksi dengan Vermin.


Gambar 13. Kompilasi artefak yang memungkinkan Anda untuk mengaitkan komponen pencurian kata sandi dengan Vermin

Alat Pencurian File USB (UsbGuard)


UsbGuard.exe adalah komponen opsional yang digunakan oleh Sobaken dan Vermin. Ini adalah program kecil terpisah yang memonitor drive USB yang terhubung ke komputer dan menyalin semua file yang sesuai dengan filter yang dikonfigurasi oleh penyerang. File-file yang dicuri kemudian ditransfer menggunakan modul backdoor utama. Banyak jalur berbeda ke PDB telah ditemukan dalam sampel komponen ini, yang jelas mengikatnya dengan Vermin.

Sejak April 2018, komponen pencurian file telah digunakan sebagai alat terpisah. Ini menyalin file dan segera mengunggahnya ke server yang dikendalikan oleh penyerang.

Kesimpulan


Di antara banyak serangan dengan penggunaan malware yang ditujukan untuk target berpangkat tinggi di Ukraina, kampanye ini tidak mendapat banyak perhatian. Mungkin ini karena penggunaan kode dari sumber terbuka. Namun, kelompok ini telah pindah ke mengembangkan alat sendiri.

Penggunaan beberapa keluarga malware dan mekanisme infeksi, termasuk rekayasa sosial dan steganografi, selama tiga tahun terakhir mungkin disebabkan oleh fakta bahwa penyerang bereksperimen dengan metode atau bekerja dalam beberapa kelompok.

Penerapan teknik-teknik sepele yang berhasil (misalnya, mengirim RAR dan EXE melalui email) menekankan pentingnya perlindungan dari faktor manusia.
