NAS sederhana untuk rumah

Ini adalah artikel lain tentang membangun NAS lain untuk digunakan di rumah. Apa yang mendorong saya untuk menulisnya? Mulai mengumpulkan NAS untuk saya sendiri, saya membaca semua artikel google di mana orang-orang mengatakan bagaimana memilih dan mengumpulkan penyimpanan untuk diri mereka sendiri. Berkat artikel ini, saya menghindari beberapa kesalahan saat memilih perangkat keras dan perangkat lunak. Saya mengumpulkan konfigurasi konfigurasi yang pertama kali cocok untuk saya hingga hari ini. Karena itu, saya yakin seseorang mungkin menemukan artikel saya bermanfaat.

Benar, saya harus mengatakan bahwa menurut profesi saya seorang administrator sistem dan ini memberlakukan persyaratan tertentu pada konfigurasi, dan pada artikel - detail dalam deskripsi dengan ilustrasi di ~ 2 Mb. Jadi ya, NAS membangunnya sendiri. Tetapi tanpa fanatisme. Dalam artikel itu, saya tidak membatasi diri pada deskripsi besi, tetapi juga mengatakan bagaimana saya menggunakan mesin yang dihasilkan. NAS


Pada saat itu, saya memiliki pengalaman bekerja dengan Synology DiskStation DS411 dan dengan D-Link DNS-320. Saya jatuh cinta dengan Synology OS! Stabil, nyaman, dapat dimengerti, repositori Anda dengan aplikasi, pembaruan bermanfaat - ini semua tentangnya. Besi itu sendiri juga senang dengan waktu kerja dan kesunyiannya. Saya tidak senang hanya dengan harganya (akhir 2014). Saya belum siap memberikan 20.000 rubel. hanya untuk menyimpan film dengan musik di sana. Secara alami, ada model yang lebih murah untuk dua disk, tetapi empat disk untuk penyimpanan adalah suatu kehormatan, dan ada dua di komputer yang tidak bergerak.

Menggunakan DNS-320 sebagai contoh, saya melihat bahwa jika saya memilih yang termurah, saya akan mendapatkan lebih dari perangkat sederhana. Sebenarnya, mendistribusikan file dalam LAN adalah satu-satunya hal yang dapat dilakukan oleh kotak ini secara normal.
Pada tahap ini, menjadi jelas bahwa NAS harus dirakit dari komponen komputer.

Pertanyaan dengan HDD

Menjelajahi harga penyimpanan bermerek, saya secara bersamaan memikirkan konfigurasi disk di sistem. SERANGAN? JBOD? Volume terpisah? Di satu sisi, menyimpan film dengan musik di atas reid adalah pemborosan. Jika kehilangan, mereka dapat diunduh lagi. Di sisi lain, ada beberapa folder yang harus aman dan lebih disukai tanpa mengembangkan skrip cadangan khusus (cadangan harus dipantau dan diperiksa). Selain itu, disk yang saya miliki memiliki ukuran yang berbeda. Dengan demikian, setelah perakitan RAID apa pun, saya akan kehilangan ratusan gigabyte pada "pemangkasan" disk untuk disk terkecil dan keseluruhan untuk checksum (misalnya).

JbodIni terlihat menarik dalam arti bahwa seluruh volume semua disk tersedia, dan jika terjadi kegagalan, data akan tetap berada pada sisanya. Tapi Anda masih perlu mencadangkan sepasang folder penting itu. Selain itu, pertanyaan tetap terbuka untuk saya apa yang akan terjadi pada seluruh array jika salah satu drive gagal. Apakah array akan terus bekerja atau apakah semuanya harus dipasang kembali / dipulihkan?

Pemilihan sistem operasi


Karena penyimpanan sekarang dirakit dari komponen komputer biasa, pilihan OS sangat luas. Linux, Windows, XPenology, FreeNAS, NAS4Free. Linux Klasik (Ubuntu, Debian, dll.) Tidak segera tersedia. Saya bukan spesialis besar di dalamnya, dan pengikatan masing-masing fungsi akan disertai dengan jam kerja. FreeNAS dan NAS4Free adalah solusi yang sangat menarik dan fungsional, tetapi saya bingung dengan artikel tentang flash drive yang memuat OS ini. Selain itu, sejak Karena sistem ini didasarkan pada FreeBSD, dalam hal perluasan fungsionalitas atau masalahnya, saya akan menemui Linux yang kurang dikenal sama dengan saya. Dengan HDD, semuanya bekerja dalam format Linux yang sama. Seperti yang ditulis dengan benar dalam satu artikel :
Anda harus memahami apa yang terjadi di bawah tenda.

XPenology tampak paling menarik sejak itu memiliki pengalaman positif bekerja dengannya. Ya, pada dasarnya itu adalah Linux yang sama, tetapi saya berhasil menyelesaikan semua masalah dan tidak banyak masalah melalui antarmuka web.

Oh ya, Windows. Yah, itu tampak membosankan dan tidak praktis bagi saya. Saya menginginkan sesuatu yang ringan dan baru, dan bukan hanya server Win lain yang sudah bosan bekerja.

Dan di sini saya menemukan artikel tentang Habler StableBit DrivePool - array disk dengan file duplikat. Dalam artikel tersebut, penulis membuat ulasan kecil tentang solusi perangkat lunak untuk bekerja dengan array disk. Tujuannya adalah untuk meninggalkan RAID, tanpa kehilangan integritas file. Dan jika ada masalah, jangan "perdukakan" dengan pemulihan. Setelah membacanya, saya menyadari bahwa StableBit DrivePool adalah yang saya butuhkan! Prinsip dari program ini dijelaskan dengan baik di akhir artikel oleh penulis.

Program ini dibayar, tetapi pada tahun 2014, dalam hal rubel, harganya hanya 700+. Saya tidak mandi uap dan membeli lisensi. Sekarang harganya jauh lebih tinggi, jadi inilah infonya: untuk tahun ini programnya belum diperbarui. Ini bekerja dengan stabil, tidak ada stok yang ditemukan.

Setelah menemukan cara untuk menggabungkan disk ke dalam kumpulan, pilihan OS menjadi jelas - Windows. StableBit DrivePool hanya tersedia untuk platform ini. Saya memilih Server 2008R2.

Lembut


Untuk bekerja dengan torrent, saya pertama kali menginstal μTorrent klasik. Ia memiliki fitur yang luar biasa dalam pengaturannya: Anda dapat menentukan folder yang μTorrent akan terus memantau keberadaan file torrent. Segera setelah file baru terdeteksi, program mulai mengunduh. Dan ada tanda centang lain "Hapus file torrent saat pengunduhan dimulai." Apa yang terjadi? Saya di komputer saya, unduh file torrent dan letakkan di folder khusus di NAS. Setelah 5-10 detik, itu menghilang. Ini berarti μTorrent “mengambilnya” dan mulai mengunduh. Tanda centang lain yang disebut "Tambah ekstensi acak ke file hingga sepenuhnya diunduh" memungkinkan Anda melihat di folder unduhan file mana yang masih diunduh dan yang sudah diunduh. Menurut skema ini, untuk mengunduh torrent, saya tidak memerlukan antarmuka μTorrent sama sekali.

Saya melangkah lebih jauh dan menginstal YandexDisk di server saya, menginstruksikan μTorrent untuk mencari file torrent di folder Yandex.Disk. Sekarang, duduk di tempat kerja dan memilih film untuk saya sendiri malam itu, saya meletakkan file torrent pada Yadisk dan menghilang setelah 20-30 detik ...

Tentu saja, dia mendistribusikan film torrent, tetapi beberapa film favorit saya yang saya simpan untuk saya sendiri bahkan tidak setetes pun. Secara umum, jaringan torrent tidak menjadi lebih buruk sama sekali jika saya menghentikan distribusi.

Jadi, saya ingin NAS bekerja hanya ketika saya membutuhkannya. Dan dia tidur sepanjang waktu. Dan membangunkannya melalui jaringan, bukan tombol. Ada teknologi Wake-on-LAN untuk ini.. Fitur ini harus didukung oleh motherboard dan pengontrol jaringan (jika eksternal). Untungnya, pada banyak model fungsi ini, seperti pada saya. Komputer bangun dengan bantuan "paket ajaib" yang dikirim ke jaringan dengan alamat MAC kartu jaringan. Ada program gratis khusus untuk ini, atau, misalnya, router Asus saya memiliki fungsi seperti itu. Tetapi selama operasi, ternyata Windows itu sendiri mengirimkan sihir ini ketika mengakses sumber daya jaringan. Dan ketika mengakses melalui RDP dan ketika mengakses folder jaringan. Ini baik dan buruk. Hal yang baik adalah bahwa tidak ada program tambahan yang dibutuhkan. Tapi ini buruk dengan ini, jika Anda menghubungkan folder bersama pada klien sebagai drive jaringan, atau menambahkannya ke favorit Anda:



kemudian ketika Anda menghidupkan klien, "paket ajaib" akan secara otomatis dikirim ke server dan membangunkannya. Wifi


Ketika saya berpikir tentang tempat untuk meletakkan NAS di apartemen, saya memutuskan untuk mencoba Wi-Fi. Fashionable, nyaman, modern. TP-LINK TL-WN881ND telah dibeli:



Semuanya terhubung melalui router Asus RT-N12. Semuanya keren sampai saya duduk untuk menonton film dengan kualitas yang baik. Ukuran normal .mkv 1080p ~ 20 GB. Di tengah film, gambar dan suara mulai mengganggu. Media Player Classic menunjukkan bahwa semua buffer-nya kosong dan hampir tidak menarik video lainnya. Begitu pula dengan semua film "berat". Pada salinan sederhana dari sejumlah besar informasi, saya juga memperhatikan bahwa setelah beberapa saat kecepatannya menurun secara signifikan. Pencarian dangkal untuk informasi tentang masalah ini tidak memberikan hasil, dan lebih mudah bagi saya untuk mendapatkan kabel twisted pair ke server daripada berurusan lebih lanjut dengan Wi-Fi.

Dalam kasus apapun tidak ingin memfitnah pada wi-fi dan membuat pernyataan keras tentang kesiapannya untuk beban yang panjang dan tinggi. Tetapi jika Anda akan menghubungkan NAS Anda melalui itu, maka perlu diingat bahwa mungkin ada masalah.

Kesimpulan


Apa kesimpulannya? NAS itu sendiri adalah hal yang sangat berguna. Skenario untuk penggunaannya dapat ditemukan sebanyak yang Anda suka, terutama jika Anda menggunakan OS penuh. Saya sarankan di komentar untuk membagikan pengalaman atau ide Anda, yang untuknya Anda dapat menggunakan NAS rumah Anda di samping "penyimpanan file".

Misalnya, backup data dangkal. Jika pada LAN, maka Cadangan Cobian yang sama dijadwalkan ke folder khusus. Jika di Internet, maka BitTorrent Sync. Dalam kasus Kobian, jangan hubungkan folder cadangan sebagai drive jaringan. Dalam kasus BitTorrent Sync, gunakan salinan bayangan di server ( apa itu dan bagaimana mengkonfigurasinya ). Virus kriptografi tidak lagi menakutkan.

Saya sering berguna sebagai server rumah di tempat kerja ketika saya perlu memeriksa akses ke sumber daya kerja dari IP lain dari Internet. Segala macam VPN, OpenVPN, dll.

Server media. Bersamaan dengan Smart TV, semua fungsi multimedia dapat sepenuhnya ditransfer ke mereka. Jika TV memiliki dukungan browser dan keyboard normal dengan mouse, maka bahkan mengunduh konten dapat dilakukan melalui TV.
