Klien HTML5 untuk VMware vSphere Keluar Musim Gugur Ini - Yang Perlu Anda Ketahui

Di VMworld 2018 pada bulan Agustus, VMware mengumumkan bahwa platform virtualisasi vSphere North akhirnya akan menerima klien HTML5 yang berfungsi penuh.

Rilis, yang telah menunggu sejak Maret 2016, dijadwalkan untuk Oktober. Mari kita bicara tentang fitur-fitur baru.


/ foto Libreshot PD

Sedikit latar belakang


Pada Mei 2016, VMware memutuskan untuk mengikuti kursus pengembangan thin client berbasis browser. Kemudian perwakilan perusahaan mengumumkan penghentian dukungan untuk C # -client for vSphere (yang merupakan klien besar). Sebagai gantinya, mereka menawarkan pengguna sistem Host Client dan vSphere Web Client. Yang pertama memungkinkan Anda untuk mengelola masing-masing server ESXi melalui antarmuka web, dan yang kedua - untuk bekerja dengan infrastruktur virtualisasi melalui vCenter.

Klien Web didasarkan pada teknologi Flash. Namun, tak lama setelah rilis klien Flash, pengguna mulai mengeluh bahwa itu benar-benar "melambat". Itu sampai pada titik bahwa administrator dari beberapa perusahaan "mundur" dan menunda pembaruan untuk mempertahankan kemampuan untuk bekerja dengan klien C #.

VMware memperhatikan keluhan pengguna dan mulai mengembangkan klien baru untuk vSphere, yang sudah berbasis pada HTML5.

VMware pertama menyarankan agar administrator menggunakan klien vSphere Fling HTML5, yang dibuat khusus untuk vSphere 6.5. Solusi VSphere Client Fling v3.0 yang telah kami jelaskan di salah satu materi kami. Salah satu kemampuan kuncinya adalah fungsi menyebarkan VM dari template ke Perpustakaan Konten dan template OVF.

Tetapi Client Fling adalah semacam solusi perantara. Dan musim gugur ini, itu akan digantikan oleh klien HTML5 yang berfungsi penuh. Ini akan tersedia untuk vSphere 6.7.

Dalam keadilan, kami mencatat bahwa bekerja pada Fling tidak berhenti. Versi vSphere Client Fling v3.41 sekarang tersedia untuk pengguna, di mana pengembang telah menambahkan otomatisasi PowerCLI. Pada saat yang sama, mereka berjanji untuk merilis pembaruan untuk Fling setiap minggu.

Kenapa HTML5


Keamanan (dibandingkan dengan Flash)


  • WhiteHat Security memperkirakan bahwa produk Adobe menyumbang 14% dari semua kerentanan aplikasi web. Salah satunya ditemukan awal minggu ini - kita berbicara tentang CVE-2018-15967 . Ada beberapa detail tentang itu (misalnya, diketahui bahwa dengan bantuannya penyerang dapat memperoleh akses ke data pengguna), namun, Adobe mencatat tingkat signifikansinya sebagai "penting". Para pengembang telah meluncurkan tambalan untuk itu, secara bersamaan merilis lima tambalan lagi .

  • HTML5 adalah solusi open source (dan bukan milik Flash seperti milik) dan dikembangkan dengan dukungan perusahaan seperti Facebook, Google dan PayPal. Oleh karena itu, diyakini bahwa situasi keamanan dengan HTML5 lebih baik. Pada saat yang sama, semua pembaruan untuk HTML5 diinstal bersama dengan pembaruan untuk browser (mereka tidak perlu diinstal secara terpisah, seperti halnya dengan Flash).

HTML5 memuat lebih sedikit perangkat


  • Menurut hasil pengujian , perangkat yang bekerja dengan HTML5 mengkonsumsi energi 17% lebih sedikit daripada dalam kasus Flash. Ini secara signifikan dapat menghemat daya baterai gadget seluler.

Kompatibilitas Browser


  • Beberapa pengguna klien vSphere Flash mengalami masalah kompatibilitas browser. Sebagai contoh, administrator sistem dari perusahaan transportasi Greenbrier Companies mengatakan bahwa ia harus mengoperasikan keluarga dengan klien desktop yang berbeda, dan ia harus menggunakan browser yang berbeda untuk bekerja dengan mereka.

Semua fitur ini mempengaruhi tingkat popularitas kedua teknologi. Menurut Statista, pada 2011 Flash menggunakan setengah dari situs web, dan pada 2016 - hanya 10%. Pada saat yang sama, menurut data untuk tahun ini , 73% dari semua situs menggunakan HTML5. Pada saat yang sama, Adobe sepenuhnya menghentikan dukungan untuk Flash pada tahun 2020 - ini adalah alasan lain mengapa VMware telah mengarahkan upaya untuk mengembangkan klien berdasarkan HTML5.

Apa yang baru?


Alat untuk admin


  • Dalam versi HTML5 , alat umpan balik telah ditambahkan yang memungkinkan administrator sistem untuk mengambil tangkapan layar dalam sistem dan mengirimkannya langsung ke layanan dukungan teknis.

  • Para pengembang juga meningkatkan utilitas untuk memperbarui kluster Update Manager vSphere. Versi baru manajer "diajarkan" untuk secara otomatis memperbarui host ESXi. Sejumlah hotkey juga ditambahkan (termasuk Alt + F4, yang kekurangannya di klien Flash menyebabkan beberapa pengguna tidak puas ).

Peningkatan kegunaan


  • Informasi host sekarang dapat dipantau di tingkat cluster menggunakan halaman status penyimpanan terpisah. Anda juga dapat menggunakan semua "fitur" browser: bookmark, navigasi, zoom.


/ foto Richard Masoner CC

Plugin muncul


Untuk meningkatkan kinerja , sembilan plugin tersedia untuk klien vSphere 6.7 HTML5. Di antara mereka, ada plug-in untuk Virtual SAN , sistem penyimpanan yang ditentukan perangkat lunak, dan plug-in untuk vSphere Integrated Containers , sebuah teknologi yang membantu mengelola kontainer virtual Docker di vSphere.

Namun, beberapa pengguna percaya bahwa begitu banyak plugin tidak cukup untuk operasi normal dan dalam beberapa kasus Anda harus menggunakan fungsionalitas klien web lama. Misalnya, dalam versi HTML5 belum akan ada plugin untuk vRealize Orchestrator - pusat data dan sistem manajemen cloud.

Menanggapi hal ini, VMware berjanji untuk secara bertahap menambah versi masa depan vSphere semua plugin yang didukung oleh klien sebelumnya, serta untuk meningkatkan dan memperluas kemampuan klien HTML5 berdasarkan keinginan pengguna.
