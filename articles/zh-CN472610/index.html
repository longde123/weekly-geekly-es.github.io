HTML元素的研究是由AdvancedWebRanking的专家进行的。 我们分析了前20名Google中的800万个页面，其中包含3000万个搜索查询。 原始研究在这里 。 使用精炼数据进行详细分析-在有关Catzin Rosu的MOZ 文章中 。


我们介绍了研究结果，这对于理解技术页面优化的当前趋势将很有用。


Google可以理解的元标记


本部分讨论Search Console帮助列表中列出的meta标签。 扫描页面时，Googlebot会识别这些标签。




<meta name =“描述” content =“ ...”>


Description是页面内容的元描述。 通常由110-150个字符组成。 本质上，这是一个注释，使搜索引擎和用户可以了解页面上放置了哪些内容。


Google在搜索结果中生成摘要时通常会考虑描述的内容（但Yandex经常会自行决定生成摘要）。

研究表明，有54.9％的网站有完整的描述。 在4.7％的网站上，注册了meta标签，但content属性为空（即，标签为空）。 在0.2％的网站上，该标签是完全没有内容属性注册的。



<标题>


标题从技术上讲不是元标记，但是由于它是写在<head>部分中的，并且它传达有关页面内容的信息，因此也称为元标记-与描述类似。


从SEO角度来看，标题至关重要。 即，标题的内容作为摘要标题显示在Google和Yandex的搜索结果中。 搜索引擎在确定页面的相关性时会注意标题。

尽管标题很重要，但已满78.3％。 有趣的是，仅在标题被写入的页面的5.6％上，标题的内容完全重复了h1。 也就是说，大多数站点仍然使h1和标题不同，从SEO角度来看这是正确的。


至于标题的长度，我们认为标题的长度不应超过60个字符-在摘要的标题中显示的字符数应与之大致相同。 您可以做更长的标题，但是最重要的信息应该放在前50-60个字符中。



标题最多可以包含30个字符。 例如，产品或商品的名称可能很短。 另一方面，如果可以填充标题，则不应在标题中留空白（例如，除了产品名称之外，还应标明某些功能-颜色，尺寸，材料等）。


技术优化是一项长期而细致的工作。 如果您没有时间这样做，请自动执行该过程。 在PromoPult平台的SEO模块中 ，您可以消除网站上的错误，提高其优化，可用性和转换的水平。 系统将审核您的站点超过60个区域，为检查清单创建工作计划并计算最佳预算。 对您来说-仅批准预算，访问网站并接受工作。 实际上，对您而言，所有事情都在“后台”发生。

<meta name =“机器人| googlebot”>


漫游器元标记告诉搜索漫游器是否可以索引和跟踪页面上的链接。 googlebot元标记仅为Googlebot（对于Yandex-yandexbot）定义抓取规则。


与robots.txt中的disallow指令不同，robots meta标记中的规则被认为是防止对各个页面进行索引的更可靠的方法。


在19.7％的网站googlebot上发现了漫游器元标记-占1.7％。 也就是说，在大多数页面上，扫描规则是使用robots.txt，X-robots-tag设置的，或者根本没有设置。


最受欢迎的漫游器元标记的值为“ index，follow”。 它允许页面索引和点击。 前5个机器人的meta标签-直方图上。



<meta name =“视口” content =“ ...”>


此meta标签告诉浏览器如何在移动设备上显示页面。 视口的存在向Google表示该页面已针对移动设备进行了优化。 此meta标签在62.4％的页面上。 也就是说，有37.6％的页面没有针对移动设备进行优化。


在从2019年7月向Mobile-First索引过渡的过程中，针对移动设备的优化非常重要。 它不仅涉及视口元标记，还涉及移动可用性和页面加载速度的提高。 我们对这些主题进行了研究，到目前为止，情况远非理想。


这些研究是：


  • 电子商务中的移动可用性：俄罗斯TOP-20在线商店的分析
  • 电子商务中网站加载的速度：对俄罗斯48家顶级在线商店的分析

<meta charset =“ ...”>


charset属性指示HTML 5中文档的编码（通常为UTF-8）。使用此meta标记，可以在48.8％的站点上设置编码。


这并不意味着在其他站点上未指定编码。 毕竟，可以用另一种方式定义它-例如，元标记<meta http-equiv =“ Content-Type” content =“ text / html; charset = utf-8”>。 最主要的是要指示编码，否则在浏览器中显示页面的内容可能会出现问题（无法理解的字符集，而不是可读的文本-仍然会发生！）。


<meta http-equiv =“刷新” content =“ ...; url = ...”>


此meta标记将用户发送到其他URL，并用作简单的重定向方法。 请注意，并非所有浏览器都支持它，可能会误导用户。


W3C 不建议将重定向与元标记一起使用。 最佳选择是301重定向。 毫不奇怪，只有0.1％的网站使用了刷新元标记。 这些页面可能是有关过时的设计器的页面，在这些页面上无法访问服务器级别的管理，并且无法在管理面板中配置301重定向。


<meta name =“ rating” content =“ ...” />


此元标记将页面标记为包含成人内容。 使用SafeSearch时不会出现此页面。


Google 建议将以下元标记用于成人内容：


<meta name =“ rating” content =“ adult” />


<meta name =“ rating” content =“ RTA-5042-1996-1400-1577-RTA” />


这些元标记位于1.7％的页面上。


<meta name =“ google”内容=“ nositelinkssearchbox”>


这是一个奇异的元标记，仅在800万个站点中的1000个站点上使用。


Google搜索结果有时会在您的网站上显示一个搜索框。 nositelinkssearchbox元标记告诉Google您不需要显示此字段。


很难想象有必要向Google提供这样的建议的情况。 因此，对这种元标记的需求低并不奇怪。


<meta name =“ google-site-verification”内容=“ ...”>


在Google Search Console中验证网站所有权的一种方法是通过此meta标签。 它位于被分析站点的16.6％。


<meta name =“ google”内容=“ notranslate”>


另一个特定的meta标签。 他告诉Google如果用户的语言与页面的语言不同，则不建议翻译该页面。 此标记仅使用0.1％的网站。


结构化数据（JSON-LD）


结构化数据是用于显示页面信息和对其内容进行分类的标准化格式。


使用Microdata，RDFa或JSON-LD描述结构化数据。 Google可以理解所有这些格式，但建议使用JSON-LD（在Yandex搜索中， 尚不支持此格式）。

使用JSON-LD的数据标记（无论其类型如何）在34.1％的站点上可用。 JSON-LD通常用于标记站点搜索。 因此，摘要中某个网站的其他搜索字符串可能会出现在Google搜索结果中。 社交资料，徽标，本地业务数据的布局也很受欢迎。



在此处阅读有关JSON-LD的更多信息。 我们还写了关于以不同方式快速设置微布局的文章。


Rel =“规范”属性


此属性告诉搜索引擎，放置它的页面是建立索引的优先级。 它用于消除由于各种原因而出现的重复页面（打印版本，分页页面，具有动态参数的页面等）。 出现在40％的页面上。


meta name =“关键字”


自Google 宣布对网页进行排名时不考虑关键字meta标签已经过去了10年。 但是，已满32.2％。 在3.2％的页面上，元标记已注册，但具有空值。


有趣的是，Yandex在确定页面与搜索查询的对应关系时可以考虑关键字meta标签。 但是现在它通常是不满的。

标头h1-h6


h1-h6标头使您可以构建文档。 因此，它们对于SEO很重要。 尽管如此，仅在59.6％的页面上找到h1标题，h2-占58.9％，h3-占49.6％。


在收集了所有标题的信息之后，事实证明， 最流行的是使用频率为h3 （占标题总数的42％）。


有趣的是，研究发现23,116 h7头，甚至7,276 h8头。 问题是使用它们的合理性，因为很少有人开出h5-h6的处方。


Alt属性


这是<img>标记的属性。 它指示图像的备用文本。 如果在浏览器中禁用了该文本显示，则显示该文本而不是图像。


对于SEO，alt属性很重要，因为搜索引擎了解图像的含义。 这使您可以在搜索图像中占据有利位置，并吸引更多流量。 但实际上，alt仅填充了11.9％的图像。 在6.4％的情况下，已注册alt，但其值为空。



在我们的博客上了解有关优化图像以供搜索的技术。


语言定义


为了指示页面（文档）的公共语言或内容中的单个单词，HTML标记中使用lang =“ *”属性。 该属性出现在65％的分析页面上。


hreflang =“ *”属性用于指示备用语言版本。 Google 建议使用 hreflang指定页面的本地化版本。 Yandex也提供了类似的建议。 在21.6％的页面上找到此属性。


Google标记管理器


Google跟踪代码管理器是一个代码管理系统，可让您向网站或移动应用程序添加/更新跟踪代码和其他代码片段（代码）。


对网站进行分析的结果表明，片段* googletagmanager.com / gtm.js仅占页面的4.3％。


Rel =“ nofollow”属性


如果Google看到链接的属性为rel =“ nofollow”，则它不会跟随该链接并且不会传输链接权重。 通常，此属性用于指向不可信来源的链接和广告内容。


对800万个页面的分析显示，有1280万个链接的链接具有rel =“ nofollow”属性。 那是平均一个页面的1.6 nofollow链接。


Google在2019年9月宣布 ，除了rel =“ nofollow”属性外，它还将识别另外两个属性：


  • rel =“ sponsored”-用于广告（付费）链接；
  • rel =“ ugc”-用于网站用户留下的链接（例如，在评论中）。

在其他情况下，当您不想转移页面的重量时，仍使用rel =“ nofollow”。


网站管理员的新属性开始使用了一段时间：新闻发布两周后，发现了278个赞助商链接和123个ugc链接。


Yandex尚未针对rel =“ nofollow”引入任何创新。


Facebook开放图


开放图谱标记（协议）使您在社交网络上共享的内容看起来完全符合您的需求。 该协议是为Facebook开发的，但它也支持VKontakte，Pinterest，Twitter，LinkedIn，Telegram，WhatsApp，Viber等。



最常见的标签是：


  • og：标题-标题；
  • og：url-链接到页面；
  • og：类型-页面的类型（文章，新闻，视频，类别等）；
  • og：site_name-网站名称；
  • og：图片-链接到将在帖子中显示的图片。

标签og：description（页面描述）和og：locale（站点本地化-语言）的通用性大约少2倍。


社交媒体链接


对社交网络链接的分析表明，最受欢迎的社交网络是Facebook。 77.3％的页面链接到它。 第二位是Twitter（65.2％）。 但是您必须了解，这是西方研究。 自然，在俄罗斯情况就不同了。



有趣的是，尽管该社交网络已于2019年4月关闭，但仍有12.7％的页面仍链接到Google+。 这可能只是惯性。


rel =“上一个” /“下一个”


自2019年3月以来，Google 不支持这些页面导航属性。 有趣的是，属性支持实际上是在几年前被禁用的，但是Google才在今年正式宣布这一点。


现在，Google 建议不要将rel =“ prev” /“ next”属性，而是将内容放置在一页上，而不是将其分成多页。


研究表明，在0.3％的页面上使用了rel =“ prev”属性，在3％的页面上使用了rel =“ next”。


总结一下：仍有工作要做


了解平均网页的外观可以了解当前趋势。 然后出现了问题。


网站管理员为什么很少为图片开alt选项？ 为什么只有60％的页面有h1标题？ 为什么标题和说明不完整？ 他们为什么不急于实现JSON-LD微标记？ 但这是基本的SEO ...


我想说的是，这一切都没关系-毕竟，已经分析了Google TOP-20中的页面。 也就是说，即使没有Altos，Google也认为它们在总体上是权威的。


但是，不要忘记SEO是一个复杂的因素。 我们仅考虑了很小的技术方面。 即使我们只进行网站优化，也有60多种不同的作品。 以及链接，提及，本地化，行为因素...


因此，从基本技术SEO的“扭曲”中，您的网站绝对不会受到影响。 而且，如果您认为并非其他站点上的所有事情都那么好，那么增长远不止于实际。
