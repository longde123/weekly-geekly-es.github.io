使用案例的SJ-meetup功能建模

通常，IT产品需求的最大部分是系统与环境之间的行为和交互。 7月18日，在SuperJob的一次会议上，我们的系统分析师讨论了以用例技术为重点的功能建模。 与功能建模的其他实践相比，讨论了规范方法，非规范方法。

会议由Superjob的系统分析师 Ekaterina Evstifeeva主持开幕 ，主题为“在实践中使用案例” 。

程序：

  1. UC的定义，它的优点和在需求分类中的位置。
  2. UC形式及其组成
  3. 用例级别和将其分组的规则。
  4. 编写好用例的规则。
  5. 实践练习。


→ 演讲

Superjob的首席系统分析师 Sergey Nuzhnenko darkboatman谈到了使用 案例 进行功能建模的基础知识 。

程序：

  1. 该系统，其结构和功能。 IT系统的类型。
  2. IT系统功能建模的基本技术概述。
  3. 用例-好处和适用性。
  4. 用例和结构分析。
  5. 用例建模活动。
  6. 用于定义系统边界和范围的用例。
  7. 用例进行详细的行为建模。
  8. 其他功能建模实践与用例。


→ 演讲

Superjob的系统分析师 Ksenia Lyutova谈到了主题“当您尝试 拿出 自行车时会发生什么（有关基于真实事件的用例的故事） 。 ”

程序：

  1. 我们如何学会描述系统的行为：从Skype上的消息到600张纸的文档。
  2. 客户不理解要求：应该责怪谁，怎么办？
  3. 为什么开发人员更喜欢与我交谈而不是阅读我的文档？
  4. 结论和建议（如何做和如何做）。
