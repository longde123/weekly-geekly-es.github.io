Pemantauan konsumsi energi matahari oleh komputer / server

Pemilik pembangkit listrik tenaga surya mungkin dihadapkan dengan kebutuhan untuk mengelola konsumsi energi perangkat akhir, karena mengurangi konsumsi dapat memperpanjang masa pakai baterai di malam hari dan dalam cuaca mendung, dan juga menghindari kehilangan data jika terjadi shutdown keras.

Sebagian besar komputer modern memungkinkan Anda untuk menyesuaikan frekuensi prosesor, yang di satu sisi menyebabkan penurunan kinerja di sisi lain untuk meningkatkan masa pakai baterai. Di Windows, pengurangan frekuensi dilakukan secara manual melalui antarmuka program kontrol, di Linux melalui widget bilah tugas dan melalui konsol (cpupower - CentOS, cpufreq-set - Ubuntu).

Di Linux, menjalankan perintah melalui konsol memungkinkannya dieksekusi secara otomatis ketika peristiwa tertentu terjadi.

Utilitas agen konsumsi-usps dari set bebas UmVirt Solar Power Station memungkinkan Anda untuk mengeksekusi perintah yang mengontrol kinerja prosesor tergantung pada data operasional stasiun tenaga surya.

Konfigurasi khas untuk mode 12 volt:

  • Jika tegangan pada panel di atas 16 volt, atur mode kinerja
  • Jika tegangan pada panel di bawah 16 volt atau tidak diketahui, setel mode hemat daya
  • Jika tegangan baterai kurang dari 11,6, perintah shutdown

Perintah shutdown dapat berupa:

  1. shutdown yang halus (poweroff),
  2. mode tidur (systemctl menangguhkan),
  3. hibernasi (systemctl hibernate),
  4. urutan perintah.

Urutan perintah contoh: 

./suspend.py && systemctl suspend

Menjalankan perintah ini akan menyimpan mesin virtual saat ini ke disk dan menempatkan komputer ke mode sleep. Perintah ini dapat diklaim oleh programmer dan penambang jika kompilasi program "besar" seperti Firefox, Chrome, LibreOffice dan lainnya, ketika waktu kerja (uptime) dapat digunakan untuk siang hari.
