Postgres Pro 10.4.1
Postgres Professional telah merilis rilis Postgres Pro Standard / Enterprise 10.4.1 baru berdasarkan PostgreSQL 10.4 dan Postgres Pro 10.3.2. Mereka menyertakan semua peningkatan di PostgreSQL 10.4, serta sejumlah peningkatan penting:

  • modul jsquery yang diperbarui bertanggung jawab atas permintaan ke tipe data jsonb;
  • Utilitas pg_probackup untuk mengelola cadangan dan pemulihan kelompok basis data telah diperbarui ke versi 2.0.17 dan dilengkapi dengan fitur-fitur baru; misalnya, mode DELTA ditambahkan, di mana semua file data dalam direktori dibaca dan salinan tambahan dibuat untuk halaman yang telah diubah sejak salinan sebelumnya;
  • Modul pg_pathman , yang bertanggung jawab untuk mempartisi basis data besar dan terdistribusi, telah diperbarui ke versi 1.4.11, di mana sejumlah kesalahan telah diperbaiki;

Odyssey
Yandex memperkenalkan versi open source pertama dari Odyssey puller multi-threaded untuk PostgreSQL. Ini dirancang untuk beban berat dalam sistem multi-core: lebih dari 10k koneksi TLS dan lebih dari 100k TPS per server dengan koneksi terbuka yang jauh lebih sedikit ke PostgreSQL dibandingkan dengan beberapa pgbouncer.
Odyssey dapat:

  • skala secara linear melintasi inti;
  • bekerja dalam mode penyatuan yang berbeda, dengan otentikasi khusus, pembatasan;
  • secara otomatis mengembalikan transaksi dan mematikan proses server ketika koneksi terputus;
  • meneruskan kesalahan dengan benar.

Sharding bawaan untuk PostgreSQL
Dalam artikel ini , Robert Haas tidak menjelaskan cara kerja sharding di PostgreSQL, tetapi bagaimana ia membahas masalah yang masih perlu diselesaikan:

  • Eksekutor PostgreSQL tidak dapat secara bersamaan menjalankan query pada berbagai partisi asing.
  • Kita harus belajar memberi lebih banyak pekerjaan pada sumber daya pihak ketiga (pushdown lebih banyak).
  • ASAM Sekarang situasi menjadi mungkin ketika suatu transaksi dilakukan pada beberapa node dan tidak dilakukan pada yang lain. Haas menyebutkan Stas Kelvich (Postgres Professional), yang mengusulkan cara realistis menggunakan snapshots global.
  • Otomatisasi instalasi dan dukungan sharding. Sejauh ini terlalu banyak yang dilakukan secara manual.
