欢迎来到9月18日的DINS Java EVENING

我们邀请对Java开发感兴趣的每个人参加下一个开放的DINS Java EVENING，它将在我们位于Staro-Petergofsky 19的办公室举行。

今天晚上，来自Wrike的Denis Repp将讨论如何组织产品关键部分的技术债务工作。 Infobip的Andrey Markelov将在具体示例上展示如何组织对Java服务响应时间的分析。

删减部分–有关报告和演讲者的更多信息，一个注册参加会议的链接，以及上次会议的资料。



报告书


“内燃发动机上的飞船。 用技术债务来生存下去”（Denis Repp，Wrike）


-如果翘曲引擎在AI-95上运行怎么办？
-如果机舱中唯一的加热器是烤面包机怎么办？
-如果蜂蜜舱用一个螺母和三个钉子固定在船体上怎么办？
-机长，将密钥传给16，或者让我们最后处理技术债务！
在报告过程中，Denis提供了解决方案，以解决如何组织产品关键部分中使用技术债务的过程，如何使过程可预测和透明，必须处理哪些错误以及如何解决这些错误。

丹尼斯（Denis）在金融机构或项目中工作了8年多，一种或另一种与金钱有关的方式。 在这段时间里，他在开发可靠性和容错能力，技术债务，应用程序重新设计，作为开发人员和团队负责人进行集成方面积累了丰富的知识。 现在，他在Wrike帮助建立电子商务，负责开发人员的成长，工作组织和团队管理。

“分布式跟踪：分析Java服务的响应时间”（Andrey Markelov，Infobip）


在现代系统中，数十种独特的服务可以参与处理客户请求。 通常，客户对所有这些都不感兴趣，但是响应时间和吞吐量很有趣。 在报告中，Andrei将展示如何使用特定示例来组织对响应时间的分析。 该报告对于任何级别的实践工程师都是有用的。

十一年来，Andrei一直在金融和电信领域开发Java应用程序。 除业务项目外，他还积极参与开源产品的开发，包括他们自己的Atlassian产品插件，这些插件在社区中非常受欢迎。 Atlassian社区的活跃成员，使用Prometheus，Docker和Redis的辩护律师。
