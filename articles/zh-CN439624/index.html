为什么要在轨道上存储数据

初创企业Cloud Constellation获得了1亿美元的投资，用于建立一个基于空间的云数据仓库-SpaceBelt。 我们告诉它它将如何工作，并了解空间数据中心的利弊。


/ RawPixel / PD

太空中可靠的云


窃取数据的一种方法是直接访问设备。 在某些行业中，这种情况发生的频率比乍看之下的频率更高。 例如，驱动器失窃是医学数据泄漏的最常见原因之一 。 没有高物理防护标准的数据中心也会发生类似事件。

在Cloud Constellation，我们认为您可以通过将数据传输到难以到达的轨道上来解决安全问题。 他们的SpaceBelt项目建议使用卫星来创建空间存储。

该初创公司已经从中国公司休斯中国控股有限公司获得了 1亿美元的投资，以实现其想法。 但是，正如开发人员所说，这个数量是不够的。 启动系统和创建基础设施将需要 4.8亿美元。 因此，Cloud Constellation现在正在设计卫星的“填充”，并为下一轮融资做准备。

如何运作


最初的SpaceBelt项目包括 12颗卫星，但其数量减少到了8颗。 其中三个将用于数据存储-“通用驱动器”的容量将为5 PB。 所有数据都会被复制，因此一台设备的故障不会导致信息丢失。 其余五颗卫星将充当控制信号的接收器和发送器。

信号将从位于地球上的特殊终端发送。 首先，信息将到达位于非对地静止轨道的连接卫星（不属于SpaceBelt的一部分）。 然后从那里-降落到更低位置的Cloud Constellation航天器（海拔从160到2,000公里）。 SpaceBelt系统将使用激光彼此“通信”。


Cloud Constellation客户关系总监Dennis Gatens在IoT Time播客中表示，如有必要，他们将能够根据指定的参数创建自定义卫星。 因此，客户将收到个人卫星存储。 但是，生产和发射卫星的公司仍然未知。

最初，Cloud Constellation与Richard Branson的航空部门维珍轨道公司（Virgin Orbit）签订了使用LauncherOne运载火箭发射卫星的合同。 但是在开发过程中，事实证明，为SpaceBelt设计的设备对于LauncherOne来说太重了（设备的质量超过了允许的500千克）。 因此，启动将由另一个组织进行。 Cloud Constellation目前正在与欧洲航空航天局Arianspace进行谈判。

潜在的想法


首先，卫星数据仓库的运营具有成本效益：业主无需支付电费，因为太阳能电池板捕获的光将成为能源。 没有重力将减少机械装置可移动部分的磨损，例如在硬盘驱动器中，外层空间的租用仍然免费。


/ Flickr / 最佳Picko / CC BY

据IBM称，将数据存储在轨道上并通过卫星传输比建立水下电缆基础设施更有利可图。 电缆应具有高度的变形防护，耐高压，抗震能力，并且不能在世界任何地方延伸。 即使从最偏远的地区也可以访问空间中的数据。

明显的局限性


尽管具有所有这些优点，IT专家指出了实施此类解决方案的许多潜在困难。 主要问题是带宽低，将无法快速访问信息。

“地球的基础设施本身是另一个瓶颈：地球表面的接收器将必须得到进一步的保护。 1cloud.ru开发负责人Sergey Belkin说，访问它们比在轨道上飞行的服务器容易。 “也许用于组织地面数据中心的现代方法和标准将有助于提高可靠性。”

可能的问题之一也可能是卫星的维护 。 轨道上有越来越多的碎片，因此设备将需要受到严格的保护。 万一发生严重损坏，很可能将无法修复卫星。 缺少对服务器的访问权可能会对存储支持产生不利影响。

此外，空间数据中心将需要在卫星的开发和发射方面进行大量投资。 例如，SpaceX以9000万美元的价格发射了8吨货物。 要将一个总质量为30吨的十二个机架的数据中心送入轨道，将花费约3.3亿美元。 在地球上，您可以以相同的数量构建更大的数据中心。
