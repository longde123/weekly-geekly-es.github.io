OpenLiberty是OpenSource应用程序服务器中未知的美洲虎


该报告是英文的。

OpenSource越来越成为我们生活中不可或缺的组成部分。 这个过程在包括俄罗斯在内的世界各地都在发生。 怎么了 大型供应商正朝着OpenSource迁移，以简化和统一解决方案在云中的部署。
云时代的Java和应用服务器会发生什么？ Oracle引入了在服务器和工作站上使用Java的费用。 Java EE变成了Jakarta EE。 通常，开发人员使用基于纯jvm的解决方案在私有云和公共云中的容器中进行部署，从而通过从应用程序服务器中删除通常的JEE库来节省资源。

但是，如果应用程序服务器如此轻巧，如此灵活，以至于它将利用容器中的Enterprise Edition平台，而对所消耗的资源的影响却很小，该怎么办？ 但是，如果我们可以使应用服务器成为微服务体系结构的基础平台呢？

我将告诉您如何在OpenLiberty开放式应用服务器的基础上，使用容器来构建应用，以及有关Java社区（OpenJDK，AdoptOpenJDK等）的有前途的结构，有关Jakarta EE和用于创建微服务应用的新MicroProfile标准的信息。

塞巴斯蒂安·达施纳（Sebastian Daschner）


Java Champion ，Java开发专业人员（包括EE）的作者兼讲师。 他着有《 现代Java EE应用程序的体系结构 》一书。 Sebastian参与JCP，帮助制定未来的Java EE标准，在JAX-RS，JSON-P和Config专家组中工作，并在各种开源项目中进行协作。 由于对Java社区和生态系统的贡献，他被公认为Java冠军，Oracle和JavaOne Rockstar开发者冠军。

除了Java，Sebastian还是Linux和Docker等容器技术的活跃用户。 他是博客的作者，可以通过@DaschnerS在Twitter上找到。

现代的在线服务有一个很大的缺点。 您不拥有上载给他们的数据，也不控制这些数据的分发。 在任何时候，您已经投资多年的开发帐户都可以从服务中断开，而无需给出任何理由和希望恢复。

我们将讨论如果按照“数据属于创建它的用户，服务属于使用它的用户”的原则进行开发，Internet将会是什么样。

由于我不是律师或政治家，而是Java架构师，所以我将从技术角度来研究问题。 在现代云世界中，这可以替代经典方案“浏览器-网站-数据库”。 五年前，所有替代方案在技术上似乎都不太可行，但是现在随着Docker，Kubernetes，Helm等云服务和技术的发展，似乎至少在技术上存在替代方案。
