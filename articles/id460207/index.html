Bagaimana admin Linux mengelola paduan suara anak-anak menggunakan sistem pengenalan musik di bawah Ubuntu 16.04. Mikrofon dan Akord

Apa yang dibutuhkan musim panas untuk bersenang-senang? Butuh musik! Tetapi jika musik direkam dengan catatan di atas kertas, dan Anda adalah administrator sistem biasa, dan Anda ditugaskan membuat paduan suara anak-anak, maka sebagai salah satu solusi untuk masalah ini, saya menyajikan beberapa program yang bermanfaat. Mereka ditulis dalam Java dan tersedia di repositori Ubuntu 16.04 dan di Github untuk memecahkan masalah pengenalan mesin dari catatan, membuat bagian musik, mengunduh dan mengedit produk musik yang dihasilkan ke dalam format populer dan mendistribusikan file (serta head-slap) kepada masing-masing seniman. Topiknya, saya harap, akan menarik bagi pembaca blog kita.





Bayangkan situasi nyata di kamp anak-anak. Itu perlu untuk dengan cepat mengumpulkan paduan suara dan mengajari anak-anak sebuah lagu baru, tetapi tidak ada yang menginginkan posisi direktur musik yang ditentukan oleh meja staf. Tanya administrator sistem lokal. Dan itu pergi!

Memiliki satu laptop dengan Ubuntu 16.04 diinstal, admin menyelesaikan masalah sebagai berikut.

1. Instal editor musik gratis MuseScore yang kuat.

Tampilan umum dengan lagu anak-anak terbuka "My Russia".



Anda dapat membuka file MIDI:



Bekerja dengan file karaoke.




Jika ada yang memiliki keyboard MIDI, maka itu juga dapat dihubungkan ke program untuk mengedit dan memasukkan catatan.

2. Beberapa skor populer diambil. Contohnya ada di screenshot. Lagu-lagu itu, tentu saja, sangat populer, itulah sebabnya mereka telah membentak, bahkan dalam paduan suara, bahkan dalam solo!

3. Tapi kita hidup di abad ke-21. Diputuskan untuk mencoba menyanyikan sesuatu yang baru. Tapi ternyata bersenang-senang bernyanyi di program, tidak ada yang tahu catatannya. Lalu, apa yang dilakukan administrator Linux?

A. Masukkan sistem pengenalan OCR TESSERACT. Misalnya, dari sini: https://github.com/tesseract-ocr/tesseract/wiki . Hati-hati, Anda harus menginstal bukan 4.0, tetapi versi sebelumnya - 3.5.
Pastikan untuk memeriksa instalasi bahasa yang tepat digunakan dalam karya musik dan catatan (Inggris, Italia, Jerman, Prancis, Rusia).

B. Menginstal sistem pengenalan catatan yang luar biasa! Satu-satunya batasan adalah pemindaian 300DPI, tidak lebih dari 3,5 MB, satu halaman dalam PDF.
Anda harus memasukkannya dari sini: 

$> git clone https://github.com/Audiveris/audiveris.git cd audiveris gradlew.bat clean build ./gradlew clean build ./gradlew run ./gradlew run -PcmdLineArgs="your,arguments,here"

Ada cukup skor default untuk bekerja dengan satu atau dua lembar skor, tetapi untuk hal-hal orkestra saya sarankan membangun dan menjalankan setidaknya -Xmx = 11G.

Jika semuanya terinstal dengan benar, maka Anda harus mulai dari direktori tempat perintah git clone bekerja.

Anda perlu menggunakannya seperti ini:

2. Buku -> buku Trancribe

3. Tunggu pengakuan catatan.

4. Ekspor Musik XML 3.0 yang dihasilkan melalui Buku-> Ekspor.

5. Buka file dalam MuseScore dan ekspor dari itu ke format MP3.

Kami mendengarkan dan menikmati.



UPD1. Asli untuk perbandingan.



6. Jika Anda tidak sabar untuk merekam bagian mikrofon burung bulbul lokal dengan falsetto, maka ada juga program aplikasi mikrofon Nootka untuk kasus ini.
Ini adalah pelatih vokal yang bagus. Itu dapat membuat Anda memainkan skala dengan catatan, dapat menguji pendengaran Anda, dapat membantu Anda menyetel instrumen. Grafik menunjukkan seberapa akurat Anda masuk ke catatan, dan jika Anda mendapatkannya, maka ia menggambarnya pada alat musik. Misalnya, melodi yang sama dalam kinerja saya.



7. Situasi kehidupan lain. Tidak ada gitar di tangan, dan saya ingin mendengarkan solo gitar.
Atau buat pengaturan. Bandingkan kualitas pengaturan yang berbeda. Memperkenalkan program
TuxGuitar. Gratis, buka repositori Ubuntu.



Untuk mengatasinya, Anda harus mengekspor file Music XML 3.0 yang sebelumnya diterima ke
Guitar Pro, Tux Guitar atau format PowerTab. Sayangnya, dalam versi baru MuseScore langsung
ekspor telah dinonaktifkan. Saya memberi contoh dengan ekspor yang dibuat sebelumnya.


UPD2. Komentar itu mengangkat masalah multifaset.
Teman-teman yang terkasih, saya membawa beberapa tangkapan layar dari editor musik yang terkait dengan pembuatan skor polifonik (orkestra).

Menu utama -> File -> Buat skor.

Dalam skor -> Pemilihan instrumen orkestra (suara).

Pilihan nada suara.

Skor polifonik kosong yang dihasilkan selesai.


Kesimpulan singkat. Catatan itu ditulis untuk orang-orang yang benar-benar ingin mendengar nada asing, memiliki keterampilan administrasi, tetapi tidak memainkan alat musik.
Paduan suara juga harus dikelola.
