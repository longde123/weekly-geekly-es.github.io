您的Windows很旧吗？

在Windows系统上有关Habré最新文章的激烈讨论中，大量高级用户和不太熟练的用户都倾向于使用“过时的” Windows 7，因此不急于切换到Windows10。原因可能大不相同-从对隐私策略和遥测的不满到常见的习惯和不愿意跟随Microsoft营销人员的领导。

在本文中，我想考虑开发人员需要Windows 7甚至更早版本来为其工作职责提供高质量解决方案的情况。 Windows 7在企业界的份额仍然很大，这已经不是什么秘密了，而且由于微软将其扩展支持延长至2023年，因此在不久的将来不太可能发生任何重大变化。 更不用说一个事实，即服务器2003和2008中有相当古老的系统，它们运行某种旧的但极为必要的软件，而无需任何人根据“工作-请勿接触”的原则进行移植。 但是，更新“过时”的操作系统要比听起来快得多且容易得多。

在需要测试新开发或补充现有开发的功能时，出现了安装测试操作系统以测试可操作性的问题，这不是在实际工作系统上进行操作的最佳方法。 事实证明，即使您拥有可用的映像或安装盘，2011型号的Windows 7 SP 1也与过去十年中获得数百次安全性和稳定性更新的用户计算机上的Windows 7完全不同。

如果您尝试在某些Windows Server 2008或Windows 7 SP1上使用常规工具安装更新，则会发现更新系统无法正常工作，并不断搜索更新。 许多人都知道这种情况正在发生，包括由于过时的OS服务堆栈。

让我们考虑将Windows 7完全完全和快速更新到当前状态的方法，包括（可选）不将虚拟机连接到Internet的所有步骤，对于Windows Server 2008 R2 SP1，Datacenter和Enterprise来说都是相同的。 我们需要从MSDN映像或您喜欢的虚拟化系统或真实计算机上的安装DVD中进行适当安装的适当大小的Windows 7 SP1的全新安装。 此阶段的密钥输入和激活可以跳过，以后再执行。
对于在zashniks中没有正式映像并使用文件托管服务下载它们的用户，不要忘记在Microsoft网站上检查该映像可用文件的校验和，例如，可以使用Total Commander的试用版来进行此操作。

成功安装操作系统本身之后，我们将需要从microsoft.com下载具有相应位深度的操作系统的三个更新。 首先，我们需要Windows 7 SP1和Windows Server 2008 R2 SP1的服务堆栈更新KB3177467- 2018年10月服务堆栈更新。 安装SSU之后，需要安装更新汇总KB3125574并重新启动。 如果安装失败，则可能需要预安装KB3020369。

最后一步是安装月度汇总软件包， 此处提供KB4462923的最新版本。 将来，使用每月汇总更新窗口11月7日查询将最新版本的累积更新定位在搜索引擎中，其中11月是更新包的月份。 从每月的第二个星期三开始在欧洲提供新的软件包；原则上不需要安装以前的软件包。

因此，在安装了这三个更新之后，Windows 7会立即进行全面更新，并具有所有的稳定性和安全性修复程序。 除初步下载服务包外，所有操作都可以在没有Internet连接的情况下完成。 此外，更新服务堆栈将使您可以通过Windows Update快速接收新更新，而不会出现问题。 请注意，其他Microsoft软件（例如Office和Dot NET Framework）的更新是单独进行的。
作为偏执狂的一小部分。 如果您在搜索栏中找到Windows软件质量改进程序，则我希望禁用此项目，因为Microsoft并不打算进一步开发Windows7。这可能就是为什么您还应该删除diagtrack服务的原因。
