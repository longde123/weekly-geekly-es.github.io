Memperkenalkan 3CX V16 Update 2 dan PC Remote Management melalui WebMeeting

Pembaruan 3CX v16 Pembaruan 2


Minggu lalu kami merilis Pembaruan 2. 3CX V16 2. Jika Anda tidak menginstal versi beta Pembaruan 2 , ingat bahwa pembaruan termasuk manajer koneksi jarak jauh 3CX SBC (Pengendali Perbatasan Sesi) baru. Selain itu, sekarang saat membuat pengguna baru, buklet PDF dikirimkan kepadanya melalui email - panduan pengguna singkat 3CX dalam bahasa Rusia. Dan akhirnya, kami menambahkan beberapa opsi penyesuaian widget yang menarik untuk situs web 3CX Live Chat & Talk .

Versi Baru 3CX SBC




Dalam 3CX Pembaruan 2, layanan 3CX SBC ditulis ulang dari awal. Ini digunakan oleh banyak klien untuk menghubungkan telepon IP di kantor jauh. SBC baru menawarkan fitur-fitur berikut:

  • Koneksi yang disederhanakan. Membuat koneksi SBC baru sekarang sangat mudah. Tambahkan koneksi SBC di antarmuka 3CX, bagian SIP Trunks. Setelah itu, instal SBC di kantor jarak jauh dan tentukan FQDN dari PBX dan kunci otentikasi koneksi SBC. Ketika koneksi dibuat (ini dapat dilihat pada antarmuka 3CX), restart telepon IP - mereka akan terhubung secara otomatis .
  • Manajemen koneksi SBC lengkap. Memperbarui dan mengkonfigurasi ulang koneksi SBC sekarang dilakukan dari jarak jauh dari antarmuka 3CX.
  • Memantau koneksi SBC. Setiap koneksi SBC menunjukkan statistik terperinci untuk administrator sistem - pembaruan status koneksi, informasi sistem, dan pencatatan peristiwa. Juga ditambahkan pemberitahuan email dari berbagai acara.
  • Pembaruan otomatis SBC usang. SBC berbasis Linux dapat diperbarui secara manual atau otomatis. Saat memperbarui, koneksi SBC baru dibuat - pilih dan klik tombol "Kirim Konfigurasi" - koneksi akan dikonfigurasi secara otomatis.

Kustomisasi widget komunikasi untuk situs Obrolan dan Bicara Langsung 3CX




Pembaruan 2CX v16 2 memperkenalkan opsi penyesuaian baru untuk widget Obrolan dan Bicara Langsung 3CX:

  • Mode operasi tambahan. Opsi "Obrolan dan Telepon - abaikan kepemilikan antrian" mengabaikan tindakan "Bawa diri Anda" di Antrian Pusat Kontak. Pengunjung situs selalu memiliki panggilan suara dan video terlepas dari apakah salah satu operator “menerima” panggilan.
  • Pemberitahuan suara. Sekarang lebih nyaman bagi pengunjung situs untuk berkomunikasi dengan operator berkat pemberitahuan suara ketika menerima pesan obrolan.
  • Ubah ukuran jendela obrolan. Sekarang Anda dapat menyesuaikan ukuran vertikal dan horizontal dari jendela obrolan atau bentuk komunikasi offline.
  • Ikon widget yang diperkecil. Anda dapat mengatur jenis widget yang diperkecil - lingkaran kecil atau panel.
  • Tampilkan widget di ponsel. Anda dapat menampilkan atau menyembunyikan widget jika pengguna mengakses situs dari perangkat seluler.

Unduh atau perbarui plugin untuk WordPress CMS atau instal widget Javascript versi untuk CMS lainnya.

Menginstal Pembaruan 3CX v16 2


Harap dicatat bahwa 3CX SBC dan 3CX Live Chat & Talk hanya berfungsi dengan Pembaruan 2 3CX V16 dengan sertifikat SSL tepercaya. Sertifikat yang ditandatangani sendiri tidak didukung.

Pembaruan diinstal seperti biasa - dari bagian "Pembaruan" pada antarmuka 3CX. Anda juga dapat menginstal distribusi Pembaruan 2 3CX v16 untuk Windows atau Linux:

  • Windows: Pembaruan 3CX v16 2
  • Linux: 3CX v16 pada Debian 9 ISO - Dari layar Saluran Pembaruan 3CX, pilih Stabil.
  • Instalasi manual pada Debian dan Raspbian .

Kinerja SBC dalam Pembaruan 3CX V16 2


Dalam persiapan untuk peluncuran Pembaruan 2 3CX v16, kami melakukan pengujian beban terhadap 3CX SBC Remote Connection Manager. Dan sekarang Anda akan melihat bahwa layanan baru memiliki kinerja luar biasa!

Kinerja SBC pada Raspberry Pi


Dalam tes SBC kami selama 30 menit. mendukung percakapan simultan di 40 ponsel IP. Lebih dari 600 indikator BLF dikonfigurasikan pada ponsel. Kami menggunakan berbagai codec - PCMA, PCMU, G722 dan G729 melalui terowongan terenkripsi, dan mendapatkan hasil berikut:

  • Penggunaan CPU. SBC baru memuat prosesor Raspberry PI sekitar 20%.
  • Penggunaan RAM. Layanan hanya membutuhkan 18% dari 1 GB RAM yang terpasang di Pi.
  • Lalu lintas jaringan. Lalu lintas rata-rata berjumlah 11,23 Mbps - sekitar 5% dari bandwidth adaptor jaringan Pi pada 224 Mb / s. Selama pengujian, kami melakukan panggilan uji dengan codec PCMU. Rata-rata jitter adalah 2,54 ms., Dan maksimum - 7,24 ms. tanpa kehilangan paket.
  • Kondisi suhu. Sensor suhu menunjukkan pemanasan maksimum hingga 59,5 ° C pada beban puncak.

Kinerja SBC pada Intel x64


Pada prosesor yang cukup umum, Core i7-2600 SBC juga menunjukkan hasil yang sangat baik - berkat arsitektur multi-ulir, dapat menggunakan semua core CPU. Di tes selama 30 menit. 100 telepon IP secara simultan menerima panggilan dalam Antrian (distribusi - panggilan prioritas). Pada saat yang sama, lebih dari 1700 indikator BLF dikonfigurasikan. Nilai sendiri hasilnya:

  • Penggunaan CPU. Saat menggunakan codec yang berbeda - PCMA, PCMU, G722 dan G729 melalui terowongan terenkripsi, beban CPU rata-rata tidak melebihi 6,25%.
  • Penggunaan RAM. Pada mesin dengan RAM 8 GB, konsumsi memori rata-rata tidak melebihi 19%, mis. kurang dari 2 GB.
  • Lalu lintas jaringan. Lalu lintas rata-rata pada beban maksimum adalah sekitar 34,6 Mbps. - 3,5% dari bandwidth adaptor gigabit. Selama percobaan, kami melakukan panggilan uji dengan codec G729. Jitter rata-rata adalah 0,61 ms., Dan maksimum - 2,92 ms. tanpa kehilangan paket.

Perlu diingat bahwa beban jaringan dan kinerja server PBX dan server SBC adalah faktor penting yang mempengaruhi operasi IP-telephony. Oleh karena itu, hasil tes disediakan hanya untuk menilai kemampuan SBC baru. Ketika merencanakan jaringan nyata, mereka harus digunakan dengan mempertimbangkan faktor-faktor lain.

Apa pun peralatan yang Anda gunakan, jangan lupa bahwa kinerja SBC sangat dipengaruhi oleh mode logging (mode Verbose mengkonsumsi sumber daya secara signifikan - matikan segera setelah masalah teratasi).

Pembaruan WebMeeting 3CX - Utilitas untuk manajemen PC jarak jauh


Bersamaan dengan rilis 3CX v16 Update 2, kami merilis pembaruan 3CX WebMeeting, yang sekarang termasuk utilitas remote control untuk PC peserta konferensi. Selain itu, ada transfer file apa pun melalui panel obrolan - mereka akan tersedia untuk semua peserta dalam rapat.

Utilitas Manajemen Remote PC




Kontrol PC jarak jauh telah ditingkatkan untuk penggunaan yang cepat dan mudah - Anda tidak perlu menginstal perangkat lunak pihak ketiga dan membuat pengaturan khusus. Mulai saja WebMeeting dan kirim permintaan remote control. Fitur-fitur berikut tersedia:

  • Instalasi klien instan. Peserta konferensi menerima tautan untuk menginstal aplikasi Remote Control Client - dan Anda dapat langsung terhubung!
  • Papan klip dua arah. Jika Remote Control Client diinstal pada kedua peserta sesi, clipboard umum untuk teks tersedia.
  • Tampilan layar. Fungsi pemilihan layar menggunakan API pengaturan tampilan Windows untuk bekerja dengan sejumlah layar. Pilihan layar tidak ditawarkan jika hanya ada satu layar pada PC yang dikelola.
  • Koneksi aman. Dukungan penuh untuk kerangka kerja keamanan Windows, UAC dan peluncuran dengan hak istimewa Run as Administrator.
  • Akses minimum. Utilitas berfungsi dengan pengguna dengan hak terbatas dalam sistem.
  • Rekam sesi remote control. Jika setelah menghubungkan klik tombol "Rekam", perekaman sesi remote control akan dimulai. Di akhir pekerjaan, Anda akan menerima video yang sudah selesai.

Remote Control Client kompatibel dengan Windows 7 x64 dan lebih tinggi, dan harus diinstal pada PC yang dikelola.

WebMeeting Transfer File Obrolan




Pembaruan WebMeeting 3CX ini memperkenalkan transfer file yang telah lama ditunggu antara peserta konferensi.

  • Transfer file dengan satu klik. Seret file ke panel obrolan konferensi atau klik ikon "Unggah file" dan pilih file. Transfer file hingga ukuran 64MB didukung.
  • Pemantauan Transfer File. Indikasi visual pengiriman dan penerimaan setiap file.
  • Penyimpanan sementara. File bersama ditempatkan dalam penyimpanan sementara, dan setelah akhir konferensi mereka secara otomatis dihapus.
