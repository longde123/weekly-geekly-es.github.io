Memilih sistem penyimpanan file untuk kerja tim



Kami bekerja dengan volume besar media: video, render, foto, ilustrasi. Untuk memastikan kerja kolektif, kami membutuhkan akses umum yang konstan ke semua file ini.

Pada titik tertentu, server kami sendiri sudah tidak cukup lagi untuk kami, dan kami mulai mencari penyimpanan cloud yang akan memenuhi kebutuhan kami.

Kami membandingkan penyimpanan cloud populer untuk bisnis: Google Drive, DropBox, Citrix ShareFile dan Microsoft OneDrive.

Persyaratan penyimpanan cloud kami:

  • Volume data tidak terbatas - kami memiliki banyak data, rata-rata sekitar 10TB. Saya tidak ingin terus-menerus berpikir betapa saya harus membeli lebih banyak tempat bulan ini dan mengapa kuota tiba-tiba berakhir.
  • Versi file dan logging - git mengajarkan kita bahwa semua perubahan dapat dilihat dan digulung kembali. Oleh karena itu, file harus persis sama: perubahan apa pun, penghapusan harus dapat dibalik dan mudah dikontrol.
  • Izin - Tidak ada lagi folder publik yang tersedia untuk semua orang. Setiap karyawan harus memiliki ruang lingkup sendiri.
  • Unggah tanpa registrasi - pelanggan tidak lagi harus mencari layanan hosting file untuk mengirimi kami file yang berat. File-file harus segera diunggah ke repositori kami tanpa layanan perantara.

Seperti sebelumnya



Ketika ada lebih sedikit data, dan tidak ada layanan cloud (untuk uang yang masuk akal), kami harus menyimpan server yang kuat dengan hard drive renyah dalam array RAID. Akses ke sana adalah melalui SMB. Itu perlu untuk terus merawatnya, mengubah hard drive, cadangan. Sekali sebulan sesuatu terjadi padanya: kecepatan tulis / baca turun tajam, atau folder atau file menjadi tidak dapat diakses karena karakter yang tidak valid dalam nama, dll.

Ketika ada lebih banyak karyawan, mereka mulai bekerja dari jarak jauh, dan bahkan dari sistem operasi yang berbeda: Windows, macOS. Untuk mengakses server, saya harus menggunakan VPN, yang menyediakan akses ke file, meskipun lambat. Pada titik tertentu, menjadi jelas bahwa pendekatan ini sudah ketinggalan zaman dan kami mulai mencari pengganti.

Kapasitas Penyimpanan dan Harga


Kami menyimpan data dalam jumlah besar, rata-rata sekitar 10TB. Selama bekerja aktif, angka ini dapat meningkat beberapa kali. Proyek harus disimpan selama beberapa bulan setelah pengiriman ke klien, atau bahkan selamanya. Oleh karena itu, kami memerlukan penyimpanan tanpa batas sehingga setiap bulan kami tidak perlu memikirkan berapa gigabyte yang perlu kami beli.

Pembatasan tersembunyi


Biasanya, jika sesuatu diindikasikan sebagai tidak terbatas, Anda perlu mencari teks abu-abu dalam kondisi. Setiap layanan memiliki karakteristiknya sendiri, yang tidak dapat dilihat secara sekilas. Bagi beberapa orang, mereka begitu aneh sehingga mereka membuat mereka sama sekali tidak berguna untuk tugas kita.

  • Dropbox - ukuran file unggahan maksimum 50GB
  • Citrix ShareFile - ukuran file unggahan maksimum adalah 100GB. Tidak terlalu banyak, tetapi tertahankan.

Sinkronisasi selektif


Akuntan Svetlana tidak memerlukan video di komputer 2TB yang bekerja dengan videografer Andrey. Untuk melakukan ini, sistem harus memiliki fungsi sinkronisasi selektif, yaitu, pengguna harus secara manual mencatat file mana yang ia butuhkan secara lokal di komputer, dan yang hanya berdasarkan permintaan. Keempat layanan mendukung fitur ini. Namun, aplikasi Google Drive memungkinkan Anda untuk menandai hanya folder di root disk, yaitu, Anda tidak dapat mengaktifkan sinkronisasi hanya untuk Pembukuan -> Laporan -> 2018 , hanya mungkin untuk seluruh folder Pembukuan .

Hak akses


Karyawan yang berbeda harus memiliki akses hanya ke folder dan file tertentu. Sistem harus menyimpan catatan rinci tentang perizinan dan perubahan akses file. Keempat sistem memiliki fungsi seperti itu. Citrix ShareFile dan DropBox dapat mengaktifkan notifikasi ketika mengakses file dan folder tertentu. Ini mungkin berguna untuk file dengan informasi yang dilindungi secara khusus. Misalnya, Anda dapat mengatur peringatan tentang akses ke file bank klien dan beberapa dokumen, dan perhatikan bahwa seseorang di luar jam kerja tiba-tiba beralih ke file-file ini.

Versi

Versi File

Siapa yang secara tidak sengaja menghapus file dari folder bersama? Untuk nomor berapa kita punya cadangan terakhir di sana? Adakah yang bisa bertahan? Ini adalah situasi yang sangat tidak menyenangkan. Terutama menyenangkan ketika seseorang meluncurkan virus enkripsi dari rumah dan memecah semua file yang ia akses. Selain situasi darurat, sering kali perlu memutar kembali ke versi file yang lebih lama untuk membandingkan perubahan atau memperbaiki masalah. Sebelumnya, untuk ini, Anda harus menghubungi administrator yang mendapatkannya dari cadangan malam. Dan jika perubahan terjadi pada siang hari, maka mereka hilang.

Sekarang kita dapat langsung mengakses versi file apa pun. Sangat menyenangkan bahwa Anda dapat melihat perubahan yang dilakukan oleh pengguna. Ini mengecualikan situasi di mana Anda dapat mengatakan " itu bukan saya, itu sudah ada ."

Dalam beberapa kasus, menginstal perangkat lunak pihak ketiga pada komputer tidak diinginkan atau tidak mungkin, misalnya, pada sistem perusahaan. ShareFile memiliki akses melalui (S) FTP dan WebDAV. Google Drive memiliki program pihak ketiga untuk mengakses drive melalui WebDAV, tetapi semuanya memerlukan transfer akses ke akun ke pihak ketiga. Dropbox tidak memiliki dukungan untuk protokol pihak ketiga, akses hanya dimungkinkan melalui klien.

Klien untuk protokol ini dibangun di sebagian besar sistem operasi. Benar, klien macOS untuk beberapa alasan sering me-mount WebDAV dalam mode hanya baca.


Penyimpanan Citrix ShareFile dapat dihubungkan melalui WebDAV tanpa menginstal perangkat lunak pihak ketiga
