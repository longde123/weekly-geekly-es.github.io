使用nest，@ nestjsx / crud和TestMace快速创建CRUD


当前，REST API已成为Web应用程序开发的标准，使您可以将开发分为独立的部分。 对于UI，当前使用了各种流行的框架，例如Angular，React，Vue等。 后端开发人员可以从多种语言和框架中进行选择。 今天，我想谈谈NestJS之类的框架。 我们在TestMace上积极地将其用于内部项目。 使用nest和@ nestjsx / crud包 ，我们将创建一个简单的CRUD应用程序。


为什么选择NestJS


最近，JavaScript社区中出现了很多后端框架。 而且如果在功能方面它们提供类似于Nest的功能，那么绝对可以胜出-这就是架构。 下列NestJS功能可让您创建工业应用程序并将开发规模扩大到大型团队：


  • 使用TypeScript作为主要开发语言。 尽管NestJS支持JavaScript，但是某些功能可能无法使用，尤其是在涉及第三方软件包时；
  • DI容器的存在，使您可以创建松耦合的组件；
  • 框架本身的功能分为独立的可互换组件。 例如，在后台 ， express和fastify都可以用作框架；对于使用DB，从盒子中嵌套可以为typeorm ， 猫鼬 ， sequelize提供粘合剂 ；
  • NestJS独立于平台，并支持REST，GraphQL，Websockets，gRPC等。

该框架本身受前端Angular框架的启发，在概念上有很多共同点。


NestJS安装和项目部署


Nest包含nest / cli软件包，它使您可以快速部署应用程序的基础框架。 全局安装软件包： 


npm install --global @nest/cli

安装后，我们将使用名称nest-res t生成应用程序的基础框架。 这是使用nest new nest-rest命令完成的。

用基础配置工作


作为本文的DBMS，我选择了PostrgreSQL。 在我看来，他们没有争论口味，这是最成熟的DBMS，具有所有必需的功能。 如前所述，Nest提供了与用于处理数据库的各种软件包的集成。 因为 我的选择落在PostgreSQL上，选择TypeORM作为ORM是合乎逻辑的。 安装与数据库集成所需的软件包：


yarn add typeorm @nestjs/typeorm pg


按顺序，每个软件包的用途是：


  1. typeorm-直接来自ORM的软件包；
  2. @ nestjs / typeorm-NestJS的TypeORM软件包。 将要导入的模块添加到项目模块中，以及一组辅助装饰器；
  3. pg-使用PostgreSQL的驱动程序。

好的，已经安装了软件包，现在您需要启动数据库本身。 为了部署基础，我将使用docker-compose.yml并包含以下内容：


docker-compose.yml 
 version: '3.1' services: db: image: postgres:11.2 restart: always environment: POSTGRES_PASSWORD: example volumes: - ../db:/var/lib/postgresql/data - ./postgresql.conf:/etc/postgresql/postgresql.conf ports: - 5432:5432 adminer: image: adminer restart: always ports: - 8080:8080

如您所见，此文件配置了两个容器的启动：


  1. db是直接与数据库一起的容器。 在我们的例子中，使用的是postgresql版本11.2。
  2. 管理员-数据库管理员。 提供用于查看和管理数据库的基于Web的界面。
