Activiti-业务流程引擎

Activiti框架（Java）-XML任务流（bpm）及其管理的描述。 在这里，我将描述基本的基本概念以及如何构建简单的业务流程。

Activiti的主要概念是过程和任务。 流程是由定向流和分支互连的所有任务。

我将涉及以下方面：

  • -最原始形式的Activiti
  • -用户，角色
  • -连接SpringBoot
  • -REST API
  • -工作和代表

流从任务到任务的移动是逐步进行的，每个这样的步骤都会在等待输入和任务执行的同时暂停该过程，所有中间操作都保存在数据库中。

我将在下面指出在哪里采取什么措施。 让我们从一个简单的示例开始-程序开发过程，该过程包括编写代码和测试。 以下是流程图。



这都是一个过程，它具有ID，Name和其他特征。



它具有：

过程的开始，两个任务“开发”和“测试”，一个分支（网关），过程结束。 换句话说，一切都是这样的：

  • 加载bpm描述
  • 开始过程
  • 开始之后，我们立即进入Develop任务
  • 在执行Develop之后，它将进入测试阶段，并根据测试结果结束或再次返回到开发过程。

Activiti包含一组服务

这里是主要的：

  • RepositoryService：控制流程描述的加载
  • RuntimeService：启动进程
  • TaskService：执行任务
  • FormService：访问任务变量
  • HistoryService：访问流程历史记录
  • IdentityService：用户和角色
