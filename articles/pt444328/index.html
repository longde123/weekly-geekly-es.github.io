O redesenho da interface do usuário costuma ser uma perda de tempo

Como prefácio, observo que trabalho principalmente no back-end e prefiro o código apropriado. Estou envolvido no desenvolvimento de web e software há mais de quatro anos e trabalhei com muitas estruturas de front-end e servidor.


Nova interface do Twitter

Antes que todos os designers de interface do usuário apareçam, permita-me dizer que aprecio o design . O produto ou o site deve ser pelo menos conveniente de usar e, se você tiver um olho bem direcionado e uma mão firme , certamente criará algo com uma aparência agradável. Apenas pare de refazê-lo o tempo todo.

Alterar a interface do usuário, na minha opinião, é uma perda de tempo em 95% dos casos. Deixe-me explicar.

Ninguém liga


Veja o nosso novo design ! E o design de materiais , venha ver! Desculpe, mas, além de corrigir a interface do usuário, onde isso afeta a usabilidade do seu aplicativo, ninguém fica louco por como o redesenho melhora o aplicativo.

As pessoas usam o aplicativo por causa de sua funcionalidade, e não porque parece bonito.

Você acessa o Google Maps não porque é bonito, mas porque mostra onde encontrar tacos deliciosos. Você usa o Facebook não por causa de uma boa interface do usuário, mas para conversar com amigos e compartilhar fotos. Você vai à loja de jogos gamestop.com não porque gosta do fluxo de anúncios e da navegação confusa nos menus suspensos caóticos, mas para comprar o Anthem .


Um pouco demais, não?

A interface do usuário só deve ser atualizada se afetar a capacidade do usuário realmente usar seu aplicativo.

Se você achar que os usuários não podem ir para a página x porque ela está oculta no menu, mova o link. Ligue para a página. Mova o botão de logout para que faça sentido para todos. Publique formulários de maneira compreensível. Este não é um redesenho, mas simplesmente um bom design que deve refletir as necessidades da empresa e de seus usuários.

Sopre para uma audiência regular


Eu odeio reaprender a nova interface. Especialmente quando o redesenho se torna muito ambicioso. Você precisa entender que cada novo design da interface do usuário requer um esforço adicional para entender onde algo foi atualizado.

Usuários regulares não gostam de mudanças. Quando eles aprendem algo, gostam de trabalhar no piloto automático, é muito mais fácil. Quanto mais você muda, mais esforço é necessário para dominar os novos itens.

Pare de mudar tudo, pare de sobrecarregar-nos com trabalho extra, forçando-nos a reexaminar os aplicativos que amamos.
