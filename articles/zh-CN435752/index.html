Moscow Python Conf ++ 2019-第一次会议，我们自己准备一些演讲者



“如果你想做得好，那就自己做。” 在2019年4月5日，我们正在进行第四届Moscow Python Conf大会 ，我决定进行一个奇怪的实验：看看将报告的准备工作颠倒会怎样。

和往常一样，“征文启事”打开了（开门了，他什么都没发生），大约一个半月半，组织者说服了顶尖演讲者提供报告，他们击败了工作和家庭，然后数十人程序，每个人都呼气。 一个很好的方法，但有一个严重的缺陷：很多事故。

即使是经验丰富的演讲者也无法就所提议的主题做出强有力的报道。 知名公司中的“活跃”发言人可能很忙，或者想参加另一个会议。 仅仅因为程序委员会的十个人不知道IVI.ru肠道中巨大的Python球，并且没人给他们写信，就可以绕过许多有趣的话题。

我决定尝试遵循Goblin Slayer的方法，而不要掷骰子：自己拜访公司，亲自进行采访，选择主题并从头开始准备演讲者。 您应该已经从这样的报价中看到了他们的眼睛。。。将没有照片，但是我会告诉您切割后发生了什么。

我们将会议从秋季移到了春季，所以在秋季莫斯科Python Conf ++＃3结束后，我立即去了人们。 大型IT公司充当人们的角色，与我已经熟悉或在同一HighLoad ++上发言的DevRels一样：hh.ru，Domklik，Avito，S7，Tinkoff，Iponweb，Yandex，Rambler，Cyan –名单很长。 我向所有人提出了同样的建议：

  1. 我随身带了杯咖啡，并与了解该公司Python的开发人员交谈。 如果存在有希望的主题，我会与您可以一对一打电话的人联系，并讨论代码。
  2. 打电话给开发人员，收取发票。
  3. 如果有一个主题和主题，那么我建议根据自己的方法从头开始准备一个演讲者。 以这种方式准备的发言人将自动进入会议程序。

有趣的是：在我接触过并使用Python的公司中，没有人拒绝。 大概每个人都想看看一桶咖啡的样子。 像这样：


采访公司。 有没有Python？ 如果我找到了？


我在每个人中发现了Python和有趣的话题。 通常看起来像什么？ 在我周围，有1到10个开发人员，团队负责人和技术专家，我向他们简要介绍了我的提案，一个专业会议的概念，并建议我告诉我“在Python的参与下公司正在做什么”。 起初不确定，这个故事很快就变成了“我们还有……”的讨论，大约二十分钟后，我继续澄清问题，为会议选择了最有希望的主题。

这次采访的关键是要记住为什么我们要参加一次专业会议。 自从上学和上学以来，我们一直坚信会议是人们学习和学习新事物的地方。 这是我们的大脑对“为什么要参加会议”这一问题产生的“自动”响应。 自动且不正确-演讲者在30至40分钟的演讲中为尽可能广泛的听众做准备，您可以从中学到什么？ 学习和学习新事物与互联网有关。 官方文档，reddit，黑客新闻，媒介，Habr，stackoverflow和其他资源。

会议有哪些有趣的话题？


会议-关于交流。 Internet上有所有信息。 但是，作为开发人员，我们不需要所有信息。 我们需要最重要的问题和答案。 在使用技术和解决“很酷的公司”中的各种任务方面的个人经验。 您可以在大型专业会议上通过一杯咖啡（然后在aftapati喝啤酒）谈论所有这一切。

如果您从这个角度来看会议，为什么会有发言人？ 演讲者设置话题进行对话，并讨论公司的内部经验：所有与stackoverflow和Habr无关的内容。 我在面试中正在寻找这样的内容，问的问题是“您的情况如何，而官方文档和stackoverflow中没有写这些内容？”。 总是会发生一些有趣的事情：每个进行Python开发的大公司都有自己的独特蛇形建筑，带有自己的任务，困难，问题和解决方案。

关于Python开发人员所做的一切的莫斯科Python会议。 后端，数字破碎机，机器学习，大数据，Web开发，移动应用程序，测试和代码部署以及许多其他主题。 例如，Iponweb为Python中的每个客户端生成唯一的管理面板。 演讲者将告诉您出现了什么困难，以及如何为这些任务正确地将Django内部化。

通过Zoom进行逐项收款。 什么是发票？


在面对面访谈结束时，我收集了开发人员的联系人，并与他们安排了20-30分钟的小规模的开发人员联系，并通过Zoom进行了呼叫。 对于视频通话非常方便：两个用户无限制的免费视频通话，您可以将视频写入计算机，Skype对等没有典型的“ Skype通话”问题。 我认为唯一的缺点是带有隐藏的“加入计算机音频”按钮的奇怪的可用性解决方案：视频通话中连接的参与者可以观看视频，但听不到对方的声音，也听不到任何声音。 但是，如果您事先解释了此按钮，则不会有其他问题。

我已经与开发人员讨论了代码-是否有足够的材料来编写报告。 在全职头脑风暴中，人们对许多事情的记忆与实际情况有所不同，随后呼吁开发人员纠正了这一差距。

从我的角度来看，我召集所有人，向公司提出最合适的话题。 作为回应，公司给了我一名发言人：通常这是我在Zoom上与他交谈以获得代码的人。

演讲者培训


平均而言，在报告上花费了十几个视频通话，每个视频通话30-50分钟。 准备工作以访谈的形式进行，我问了一百万个问题，并挖掘出公司的独特经验，演讲者充当了知识的来源。 大致准备工作包括四个部分：

  1. 发票的收取：首先，将在前面阶段中所选主题收集的所有内容都复制到此处，然后我用问题轰炸演讲者，直到我看到足够的材料和价值以进行30分钟的报告为止。 通常这是2-3次打电话，在这段时间里演讲者返回小组并澄清细节和细微差别。
  2. 报告大纲 ：在检查了发票之后，我和发言人进行了整理，以便我们得到一个有趣的故事。 通常，我会使用一种气味，但在极端情况下，您可以使用现成的新闻方法。 这是最困难的阶段，在编写故事时，不需要某些事实，但是可能会出现新的有趣的细节，您需要进一步了解这些细节。
  3. 制作幻灯片是最简单的步骤。 有了演示计划，数百次演讲的经验并理解为什么需要幻灯片，它们很容易制作。 清单，图表，漂亮的图片和箭头在意想不到的地方，将有助于听众了解演讲者的故事，看到关键字并不会失去故事的轮廓。
  4. 切割双面幻灯片 。 我们在演讲中使用的最大秘诀：演讲者在他面前看到论文线索，这些线索被放在演讲者笔记部分，并将报告分成许多很多小的孤立故事。 运作方式，您可以在此视频中看到：
