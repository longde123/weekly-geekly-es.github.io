公开网络研讨会“工作：一个人，另一个人”

朋友们下午好！ 这个星期四，我们已经开始以“开发经理”的速度发布新视频。 今天，我们想与您分享上周举行的一次公开网络研讨会的内容。

公开网络研讨会“工作：一个人，另一个人”

一方面，“这不是我的工作”这句话听起来很丑陋：某种程度上说，这不是积极主动的，也不是命令性的。 另一方面，所有工作都是不可能完成的。 也就是说，做“不是您的”工作-等同于不做您的工作！ 这已经充满了...

那么，如何将您的工作与您的工作区分开？ 什么是工作？ 如果我们仍然对最终结果感兴趣，该怎么办？ 在“开发经理”课程的下一个公开课中，我们讨论了所有这些以及更多内容。



讲师-Biglion Technologies开发总监Stanislav Mikhalsky 。

工作和活动水平

工作的正式定义是：“工作是一个人的活动，旨在创造价值或满足他人的需求。” 这里的关键词是活动。 首先，任何活动都需要资源的可用性（在本例中，这是这次）。 其次，活动应导致最终结果 。 并且这两点（资源和最终结果）应始终严格控制。

另一个关键点是活动水平 。 如您所知，活动的活动是不同的。



更详细地考虑活动级别：

  1. 手术室。 在这个级别上的工作恰好是工作，即身体动作。 假设我们制造砖头。
  2. 战术上的。 这是关于优化运营流程。 例如，关于用更高生产率的输送机更换输送机。 这将使您能够以更快的速度和最低的成本获得质量最好的砖块。
  3. 战略性的。 制砖的目标是什么？ 盖房子？ 在市场上卖砖头？ 制定战略库存库存？
  4. 目标设定水平。 总体任务是什么？ 你为什么要这样做？ 为什么要砖头？ 人生苦短，还有许多其他有趣的活动。

采取哪种级别的选择是我们的。 而且他并不总是有意识地做。 毕竟，您可以通过两种方式帮助饥饿的人：给他钓鱼或教他如何钓鱼。 在我们的管理现实中也是如此：我们可以快速地为某人做某事（操作级别），并且我们可以花时间并教育一个人，以便他以后可以自己做（战术级别）。

艾森豪威尔矩阵

艾森豪威尔矩阵是一个简单但极为有效的工具，如果您及时记得的话。



这个想法很简单：我们根据两个标准来划分任务：紧急-非紧急； 重要-不重要。 这样我们得到一个2 x 2的矩阵，这是一个非常常见的可视化工具。 顺便说一句，在“开发经理”课程中进行培训时，经常会出于各种原因而使用此类矩阵，从而体现了某些管理难题。

因此，我们有4个正方形：

  1. 重要紧急。
  2. 重要的非紧急。
  3. 不用急
  4. 不重要，不紧急。

下两个正方形对我们来说并不重要。 但是，“火灾”和“目标”必须能够区分和看到，而且还需要管理目标。 从定义中已经很清楚，“火灾”是迫不及待的事情：紧急报告，服务器崩溃等。

什么是“火灾”的问题：

  • 他们无止境；
  • 它们分散了我们对最重要的2号平方的注意力。但是，正是从这个意义上讲，任何人在广义上的基本工作都在于此。 从个人的角度（为自己工作：接受第二次教育，学习外语，进阶培训课程，去健身房等）以及从工作的意义上（引入新流程，提高系统稳定性，使项目现代化等）。 ）

显然，目标非常重要。 但是它们与“火灾”的不同之处在于，如果您现在不执行这些操作，将不会有任何东西落在您的头上，老板也不会发誓。 但是请不要忘记，在您扑灭另一场“大火”的那一刻，未来变成了灰色，远处逐渐消失，有时甚至消失了 。

摆脱火灾的最好方法是系统地解决问题，即在战术层面上。 领导者的第一个错误和不作职务的第一个例子是在业务层面上扑灭大火 。 是的，您可以自行扑灭，但此后您必须了解问题，教导下属解决此问题并将任务委派给他。 这是战术。

在战略层面上该做什么？ 使火灾不可能发生或不太可能发生？ 总的来说，这是对的，但这并不是一个详尽的答案。 这里也有细微差别，但是关于它们的故事不在本网络研讨会的讨论范围之内。

因此：

  1. 操作级别。 快来自己修复。
  2. 战术水平。 来，修理（如果没有您的帮助），教导，委派。
  3. 战略层面。 我们消除了火灾的根本原因（这是理想的，众所周知，这并不总是可以实现的，但是您需要为此努力）。

但是，让我们回到矩阵，再次讨论平方2的重要性。

石头，鹅卵石和沙子。

让我们讲一个寓言。 教授聚集了学生，并在他们面前放了一个空罐。 之后，我将大石头放进去。

-罐子装满了吗？ -问教授。
-满！ -学生回答。

然后老师把鹅卵石放在同一个罐子里。 它掉落在大鹅卵石之间，掉落下来，填满了空座位。 罐子再次装满，但不拥挤，学生们确认了这一点。 然后，教授将倾倒在整个剩余空间中的沙子倒入罐子最难以接近的角落。

在此之后，有解释说，大石头是我们生活中最重要的事情：亲人，家人和父母的健康。 鹅卵石-一种职业，以“种树，盖房，买车”的风格取得各种成就，等等。沙尘是日常工作（工作中的丑闻，各种失败和其他事情）。

结论很简单 ：在我们的生活中，“装满罐子”的顺序非常重要。 如果先倒沙子，再倒卵石，那么石头（对我们来说最重要的东西）将不再适合。



这就是为什么专注于2号正方形如此重要的故事。

为什么别人不能工作？

当您从事他人的工作时，您就不会自己做。 这是主要且最明显的思想。 但是对于她的理解，您必须首先回答他的工作。 如果管理层将确保一切都没有中断的任务交给您，那么原则上您有理由跳槽。 不仅如此，如果您采取另一种方法，而不是扑灭自己，而是逐步改写系统（阅读，在战略层面上采取行动，查看根源），系统将继续定期发生故障，并且管理层可能会认为您没有履行职责（您与管理层不同意采取系统性措施，因此并没有改变上级的期望）。

因此，您应该以一种很好的方式与领导保持同步，并清楚地了解自己的工作。 这是一个标记，因为如果您不做自己的工作，但总是忙于某些事情，则意味着有人让您去做别人的工作。

我们在做别人的工作的事实还有其他副作用：

  • 我们阻止他人学习；
  • 我们弥补了系统的缺陷。

经理时间

您经理的时间可以分为以下几个部分：

  1. 老板的时间。 一段时间以来，您的老板总是因为明显的原因而离职。
  2. 系统时间。 我们正在谈论一些标准事物的不断实施（保持时间表，填写杂志）。
  3. 是时候采取主动了。 虽然有点，但是。 但是，即使是很小的一部分，也依次分为您可以执行自己真正想要的时间和下属花费的时间。 在这里，不禁让人想起唐纳德·瓦斯（Donald Wass）阅读最多的文章之一，“经理与他的时代，或谁会得到猴子”。

猴子在谁的肩膀上？

想象一下，您已经完成了老板的所有任务，沿着走廊走着，以为现在有一个小时的时间来启动现代化项目。 而此时，下属在走廊上“捉住”了您，而后者甚至没有打招呼，就报告了问题。 他做得如此之快，以至于您无法完全了解情况，而是想更详细地了解所有内容。 然后您会自动回答，您一定会这样做，但是稍后。

结果如何？ 下属遇到一个问题（带着猴子在他的肩膀上）来找您，与您交谈并离开。 现在，您正带着猴子在肩上行走，猴子不断对您窃窃私语：“去了解这个问题，收集信息，打电话给其他人，等等。” 同时，下属本人也绝对不受约束（他也报告了这个问题，所以他可以去冰沙吧）。

不仅如此，下属不仅将责任转移给您，还使您受到控制。 过了一段时间，谁会（如果不是他的话）会问：“那么，您是如何发现，澄清，弄清楚的呢？”

最终，在您进入工作场所之前，您可能会被各种大小的猴子所笼罩，拥有相同动物的访客可以在工作场所等着您。
真的，如果从猴子身上取走猴子，傻瓜会拒绝什么？

只有一种出路 ：确保下属带着他的问题离开（他带着一个问题-他带着一个问题离开）。 更好的是，让唐纳德·瓦斯（Donald Wass）阅读其下属的文章，然后，如果发生某些情况，请告诉他们：“嘿，带走猴子！”

反模式

有许多既定的表达式和管理策略可以称为反模式：

  1. 如果您希望它做得很好，请自己动手做。 这个故事将您锁定在操作级别。 自己捉鱼，喂饱饥饿的村庄。 逗你的自我，但仅此而已。
  2. 你做得不好-努力工作。 就是无法以高于操作水平的水平工作。
  3. 变得无可替代，成为一切中的佼佼者。 向各个方向展示了自己之后，您将再次将自己束缚在操作级别上，并不再考虑如何简化，委派等。记住，操作系统令人上瘾。
