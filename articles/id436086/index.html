Gambaran Umum Pembaruan 4 untuk Veeam Cloud Connect

Tahun Baru dimulai dengan produk-produk baru dari Veeam. Pekan lalu, Pembaruan 4 untuk Veeam Cloud Connect tersedia untuk penyedia layanan. Biarkan saya mengingatkan Anda bahwa Veeam Cloud Connect membantu mengkonfigurasi replikasi mesin virtual, meluncurkan replika di cloud penyedia layanan, dan mengatur repositori jarak jauh untuk cadangan di situs penyedia. Saya sudah menulis tentang alat ini di sini . Hari ini saya akan menunjukkan kepada Anda apa yang baru di dalamnya.

Berikut adalah perubahan utamanya:

  • pengikatan Cloud Gateway ke klien tertentu;
  • Replikasi ke Direktur VMware vCloud. Sebelumnya, Anda hanya bisa membuat replika di tingkat vCenter. Klien tidak dapat terhubung ke replika mesin virtual melalui konsol tanpa kegagalan.
  • backup pada kaset dengan klien mengikat ke rekaman tertentu.

Detail di bawah potongan.





Pool cloud gateway


Sekarang klien dapat diberi kumpulan cloud gateway yang terpisah (Cloud Gateway Pool). Jika klien memiliki persyaratan saluran khusus untuk tugas replikasi (misalnya, enkripsi saluran sesuai dengan GOST atau kecepatan terjamin), ini dapat diatur dalam Veeam Cloud Connect yang baru. Untuk melakukan ini, Cloud Gateway Pool yang didedikasikan untuk klien dibuat. Mungkin ada beberapa - untuk toleransi kesalahan atau penyeimbangan beban. Saya akan tunjukkan satu contoh.



Selanjutnya, buat Gateway Pool dan tambahkan Cloud Gateway yang diperlukan untuk itu.









Ketika Gateway Pool dibuat, tetapkan ke klien yang sesuai. Ada opsi failover ke Cloud Gateway lainnya jika terjadi sesuatu dengan kumpulan khusus.



Replikasi dalam Direktur vCloud


Sekarang Anda dapat mengkonfigurasi replikasi mesin virtual tidak hanya untuk infrastruktur VMware dari penyedia cloud, tetapi ke organisasi vCloud Director yang dipilih. Versi 8.10, 8.20, 9.0, 9.0.0.1, 9.0.0.2, dan juga 9.1 didukung. Ingat bahwa Versi Perangkat Keras dari mesin virtual yang direplikasi tidak boleh lebih tinggi dari penyedia. Begini caranya dalam praktiknya.

Di Cloud Connect, kami membuat profil klien berdasarkan akun di vCloud Director.



Kami memilih organisasi tempat replika akan ditambahkan.



Kami memilih Gateway Pool untuk klien yang diberikan.



Di tab Sumber Replika, kami melihat organisasi yang dipilih di cloud. Klik Terapkan, dan klien dapat terhubung ke Cloud Connect.



Untuk konektivitas jaringan antara sumber daya di cloud penyedia layanan dan infrastrukturnya, diusulkan untuk menggunakan Veeam Network Extension. Dengan itu, Anda dapat melakukan sebagian (mengembalikan mesin individual dari replika) dan menyelesaikan failover (memulihkan seluruh situs). Anda dapat menggunakan fungsionalitas NSX Edge untuk tujuan ini. Itu jauh lebih nyaman.

Sekarang mari kita lihat seperti apa konfigurasi replikasi di sisi klien.

Untuk terhubung ke Cloud Connect di konsol Veeam Backup and Replication, klien memilih penyedia layanan.





Klien masuk ke Cloud Connect menggunakan akun vCloud Director, ke mana replika akan ditambahkan di masa mendatang.



Setelah otorisasi, klien melihat sumber daya yang dialokasikan untuk replika: nama organisasi, CPU, Memori, Penyimpanan, dll.).



Selanjutnya, Anda dapat mulai membuat pekerjaan replikasi. Di sini semuanya, seperti dalam versi sebelumnya, ke tab Tujuan.





Pada tab Tujuan, klien memilih vApp yang akan direplikasi oleh mesin virtual, dan kebijakan Penyimpanan.



Kemudian tugas tersebut dikonfigurasi, seperti pada versi sebelumnya. Ini adalah bagaimana pekerjaan replikasi jadi terlihat.



Dan ini adalah replika yang sudah jadi di cloud penyedia layanan.



Semua replika dapat diakses melalui panel kontrol vCloud Director. Setiap mesin virtual dapat dinyalakan / dimatikan, diuji, dikloning dan semua yang dapat dilakukan dengan mesin virtual biasa. Jika topiknya menarik, di posting selanjutnya saya dapat memberi tahu Anda lebih banyak tentang pengaturan semua ini.





Rekam cadangan ke kaset


Dengan Pembaruan 4, kemampuan untuk menetapkan kaset tertentu untuk klien tertentu telah ditambahkan. Sebelumnya, saya harus melihat-lihat di repositori untuk memahami apa dan di mana itu. Ini lebih kepada celengan fasilitas untuk penyedia.





Untuk masyarakat umum, Pembaruan 4 akan dibuka pada 22 Januari. Di cloud CloudLine , Anda dapat mengambil kumpulan sumber daya uji untuk mencoba fitur-fitur baru Veeam Cloud Connect.
