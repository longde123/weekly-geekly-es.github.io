Windows Server 2019

Sejarah Windows Server sudah ada lebih dari 25 tahun yang lalu: Windows NT 3.1 Advanced Server dirilis pada 27 Juli 1993 Pada Oktober 2018, tiga tahun setelah rilis utama sebelumnya, Windows Server 2019 dirilis. Windows Server 2019 mengembangkan dan meningkatkan kemampuan yang melekat pada rilis sebelumnya. Detail tentang ini - di bawah potongan!



Dimulai dengan Windows Server 2016, siklus rilis baru telah diadopsi. Sekarang ada dua saluran distribusi: LTSC (saluran servis jangka panjang) - rilis yang keluar dalam 2-3 tahun, dengan dukungan primer 5 tahun dan 5 tahun, serta Saluran Semi-Tahunan - rilis yang dirilis setiap enam bulan, memiliki siklus dukungan utama 6 bulan dan dukungan diperpanjang 18 bulan. Untuk apa kedua saluran ini? Microsoft secara aktif berinovasi dalam platform cloud Azure. Ini adalah dukungan untuk mesin virtual Linux, wadah dengan Linux dan Windows, dan banyak teknologi lainnya.

Pelanggan yang menggunakan teknologi ini di cloud juga ingin menggunakannya di pusat data mereka. Saluran Semi-Tahunan menjembatani kesenjangan antara Azure dan pusat data lokal. Rilis semi-tahunan ditujukan untuk perusahaan yang dinamis dalam pengembangan dan telah beralih ke model layanan yang fleksibel untuk menyediakan layanan TI untuk bisnis. Rilis LTSC ditujukan untuk perusahaan yang menggunakan aplikasi mapan dengan siklus dukungan yang panjang, misalnya, Exchange Server, SharePoint Server, SQL Server, serta peran infrastruktur, pusat data yang ditentukan perangkat lunak, dan infrastruktur hyper-konvergen.

Windows Server 2019 persis rilis di saluran LTSC. Ini mencakup semua pembaruan fungsional dari Windows Server 2016 dan rilis semi-tahunan berikutnya.

Upaya utama pengembang Windows Server 2019 difokuskan pada empat bidang utama:

  1. Hybrid Cloud - Windows Server 2019 dan pusat admin Windows Admin Center baru membuatnya mudah untuk menggunakan layanan cloud Azure: Cadangan Azure, Pemulihan Situs Azure, Manajemen Pembaruan Azure, Otentikasi Azure AD, dan lain-lain.
  2. Keselamatan adalah salah satu prioritas terpenting bagi pelanggan. Windows Server 2019 memiliki kemampuan bawaan untuk mempersulit penjahat cyber untuk menembus dan mendapatkan pijakan dalam sistem. Ini adalah teknologi Defender ATP dan Defender Exploit Guard yang dikenal untuk Windows 10.
  3. Platform aplikasi - wadah menjadi tren modern untuk pengemasan dan pengiriman aplikasi ke berbagai sistem. Pada saat yang sama, Windows Server dapat menjalankan tidak hanya aplikasi Windows asli, tetapi juga aplikasi Linux. Untuk melakukan ini, Windows Server 2019 memiliki wadah Linux, Windows Subsystem untuk Linux (WSL), dan ukuran gambar kontainer yang secara signifikan berkurang.
  4. Infrastruktur Hyperconverged - memungkinkan Anda untuk menggabungkan arsitektur standar dan komputasi dan penyimpanan dalam satu server. Pendekatan ini secara signifikan mengurangi biaya infrastruktur sambil memberikan kinerja dan skalabilitas yang sangat baik.

Pusat Admin Windows


Windows Admin Center (WAC) adalah alat administrasi server baru. Itu diinstal secara lokal di infrastruktur dan memungkinkan Anda untuk mengelola contoh berbasis Windows Server, komputer Windows 10, cluster, dan infrastruktur hyper-konvergensi di tempat.

WAC melengkapi daripada menggantikan alat administratif yang ada seperti mmc, konsol Server Manager. Koneksi ke WAC dilakukan dari browser.



Untuk menyelesaikan tugas, WinRM, teknologi kontrol jarak jauh WMI dan skrip PowerShell digunakan.

Anda dapat mempublikasikan WAC dan mengelola server dari luar batas organisasi. Autentikasi multi-faktor Azure AD dan layanan aplikasi proxy membantu melindungi akses ini dari luar, dan menggunakan Microsoft Enterprise Mobility + Security (EMS) memungkinkan Anda untuk memberikan atau menolak akses tergantung pada kepatuhan perangkat terhadap kebijakan, risiko, lokasi, dan faktor lainnya. Menggunakan aplikasi web alih-alih akses pekerja jarak jauh untuk administrasi, menurut pendapat saya, adalah strategi yang tepat untuk keamanan.



Analisis sistem


Windows Server 2019 menjadi lebih pintar. Menggunakan fitur System Insights baru, analitik prediktif diterapkan untuk memungkinkan transisi dari manajemen armada server yang reaktif ke proaktif. Model pembelajaran mesin memperhitungkan penghitung kinerja akun dan peristiwa untuk secara akurat memprediksi masalah dengan ruang kosong pada subsistem disk, mengidentifikasi tren komputasi prosesor, konektivitas jaringan, dan kinerja penyimpanan.



Subsistem Penyimpanan Baru


Layanan migrasi penyimpanan


Windows Server 2019 memperkenalkan teknologi baru untuk memigrasi data dari server lama ke yang baru - Storage Migration Service.

Migrasi berlangsung dalam beberapa tahap:

  • Inventarisasi data di berbagai server
  • Cepat mentransfer file, folder jaringan, dan konfigurasi keamanan dari server sumber
  • Mengambil kontrol dan mengganti pengidentifikasi server dan pengaturan jaringan dari server lama ke yang baru

Proses migrasi dikelola menggunakan Windows Admin Center.



Sinkronisasi file azure


Azure File Sync mengubah server file tradisional dan memperluas penyimpanan ke volume yang hampir tidak dapat dicapai dalam kehidupan nyata. Data dibagi menjadi beberapa tingkatan: cache panas adalah data yang disimpan pada disk server file dan dapat diakses dengan kecepatan maksimum bagi pengguna. Saat data mendingin, data dipindahkan dengan mulus ke Azure. Sinkronisasi File Azure dapat digunakan bersama dengan protokol apa pun untuk mengakses file: SMB, NFS dan FTPS.

Replika penyimpanan


Teknologi perlindungan bencana ini pertama kali muncul di Windows Server 2016. Dalam versi 2019, dukungan terbatas untuk edisi Windows Server Standard muncul. Sekarang, perusahaan kecil juga dapat membuat salinan otomatis (replika) penyimpanan di mesin virtual Azure jika infrastruktur perusahaan tidak memiliki pusat data jarak jauh kedua.

Ruang Penyimpanan Langsung


Ruang disk lokal adalah komponen yang diperlukan untuk membangun infrastruktur hyper-konvergen dan server file yang dapat diskalakan. Windows Server 2019 memiliki dukungan bawaan untuk memori yang tidak mudah menguap, peningkatan algoritma deduplikasi, termasuk volume dengan sistem file ReFS, skalabilitas hingga 4PB per cluster, dan peningkatan kinerja.

Perubahan Clustering Failover


Cluster set telah muncul, meningkatkan skalabilitas hingga ratusan node.
Untuk memastikan kuorum dalam kelompok dengan jumlah genap, sumber daya khusus digunakan - disk saksi. Pada masa Windows Server 2012 R2, itu perlu untuk mengalokasikan disk untuk penyimpanan disk saksi, dan pada Windows Server 2016 menjadi mungkin untuk menggunakan folder jaringan atau disk saksi cloud. Perbaikan pada Windows Server 2019 dikaitkan dengan berkurangnya persyaratan infrastruktur untuk perusahaan kecil. Disk saksi dapat berupa disk USB yang terhubung, misalnya, ke router.

Migrasi cluster antara domain dan peningkatan lain dalam layanan cluster failover telah muncul.

Apa yang baru dalam platform aplikasi


Sekarang Anda dapat menjalankan kontainer berdasarkan Windows dan Linux pada node wadah yang sama menggunakan program manajemen Docker yang sama. Ini memungkinkan Anda untuk bekerja di lingkungan yang heterogen dari node wadah dan memberikan pengembang fleksibilitas untuk membuat aplikasi.

Kontainer menerima peningkatan kompatibilitas aplikasi, ukuran gambar Server Core dan Nano Server berkurang secara signifikan, dan kinerja ditingkatkan.

Windows Server 2019 telah menerima banyak perbaikan lainnya. Anda dapat membiasakan diri dengan mereka di bagian Server Windows dari situs web dokumentasi docs.microsoft.com resmi.

Persyaratan sistem


Dokumentasi menjelaskan persyaratan minimum untuk menginstal Windows Server 2019. Anda perlu memahami bahwa tergantung pada peran dan komponen, menjalankan aplikasi, persyaratan server dapat ditingkatkan.

  • 1,4 GHz prosesor 64-bit
  • Kumpulan instruksi yang sesuai dengan X64
  • Dukungan NX dan DEP
  • Dukungan untuk CMPXCHG16b, LAHF / SAHF dan PrefetchW
  • Dukungan terjemahan alamat tingkat kedua (EPT atau NPT)

Persyaratan RAM minimum:

  • 512MB (2GB untuk opsi instalasi "Server dengan Desktop")

Persyaratan Penyimpanan:

Windows Server 2019 tidak mendukung ATA / PATA / IDE dan EIDE untuk boot, swap file, atau drive data. Jumlah minimum adalah 32 GB.

Adaptor Jaringan:

  • Adaptor Ethernet dengan bandwidth minimal 1GB.
  • Sesuai dengan Spesifikasi Arsitektur PCI Express.
  • Mendukung PXE Remote Boot Protocol.

Perbandingan edisi Windows Server 2019 Standard dan Datacenter


Windows Server 2019 hadir dalam dua edisi: Standar dan Datacenter.

Edisi untuk pusat data memiliki fitur-fitur canggih: dukungan untuk infrastruktur hyper-konvergen, ruang disk lokal, hak lisensi yang diperluas saat menggunakan virtualisasi.
