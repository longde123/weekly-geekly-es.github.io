Curso C ++ en el centro CS, segunda parte

Computer Science Center es una iniciativa conjunta del Computer Science Club en POMI RAS, JetBrains y la Yandex Data Analysis School .

El centro existe para permitir que estudiantes talentosos y graduados se desarrollen en áreas de interés para ellos: informática, ciencia de datos o ingeniería de software.

El curso C ++ en dos partes en el centro es impartido por Valery Mikhailovich Lesin , valery-l , profesor en el centro CS y magistratura conjunta ITMO y JetBrains "Ingeniería de Software / Ingeniería de Software", director técnico de Simlabs .

La segunda parte ayudará a dominar la mayoría de las capacidades de C ++ moderno. Lo más probable es que no se convierta en un gurú al final del segundo semestre, pero la habilidad debería ser suficiente para usar C ++ de manera efectiva en la mayoría de los proyectos, si las ventajas son aplicables allí.

Exploraremos las nuevas características del lenguaje y la biblioteca estándar: mover semántica, bind / function, enable_if / if constexpr, coroutines y otros. Veamos algunas tareas típicas: programación multiproceso, uso de bibliotecas en Windows y Linux, y otras.

Que tengas una linda vista!

1. Biblioteca estándar. Composición, contenedores STL, tipos de contenedores, contenedor de vectores, iteradores, cómo funciona el vector. Sdt :: lista de contenedores, adaptadores, contenedores asociativos, iteración de mapas, búsqueda e inserción en el mapa. Indexación, eliminación del mapa (conjunto). Otros contenedores asociativos.


2. Mueva la semántica y el reenvío perfecto. Programas rápidos Optimización del valor de retorno. Lvalue vs Rvalue. Cómo distinguir un objeto temporal de uno no temporal. Conversión & <-> const & <-> &&. Mover constructor frente a otros constructores. Devuelve el valor vs && de la función. Devolver objetos grandes por valor, transferir por valor. Problema de argumento de reenvío. Nuevas reglas para enlaces en C ++ 11. Reenvío perfecto.


3. Algoritmos y functores. Algoritmos estándar, descripción de algoritmos. Operaciones de no modificación y modificación. Ordenar y buscar. Contenedores vs iteradores. Functores estándar. Impulsar gama. Concepto de rango.


4. enlace y función. Tarea: llamar a una devolución de llamada. Un poco de magia - std :: bind. Cómo evitar copiar. ref / cref. Usando bind con functors. Métodos de llamada. Enlace de anidamiento. std :: function. Efecto sinérgico de enlace y función. Rendimiento Cómo funcionan los marcadores de posición.


5. Excepciones. Formas de manejar errores. intenta ... atrapar. Tipos de excepciones Usando RAII. Pasando parámetros. Pila de clase Garantías de seguridad para excepciones. Garantizado sin excepciones. Copiar pila. Requerimientos de pila reducidos. Especificación de excepciones. Operador noexcept y especificación noexcept (C ++ 11). Contenedores STL. Garantías y requisitos de pila.


6. A SFINAE o no a SFINAE. Selección de código según el tipo. Selección de tipo por condición. Prueba de reducibilidad. Verifique la clase base. SFINAE Verifique el contenedor. Selección por tipo de propiedad.


7. Innovaciones en C ++. Funciones anónimas, auto, decltype, plantillas variadas, paquete de parámetros, expansión de paquete y expresión de pliegue. Inicializadores de miembros de datos no estáticos, si constexpr.


8. Filas. Entrada / salida. Codificación (juego de caracteres). String std :: basic_string. Operaciones básicas de cadena. impulsar algoritmos de cadena. Formato de archivos de texto vs binarios. Flujos de E / S, salida, entrada, estado de flujo, entrada / salida formateada. Manipuladores, secuencias de archivos.


9. El preprocesador. Uso y aplicabilidad de macros. Definición de constantes, compilación condicional. Incluye guardia. Función macro Convirtiendo una ficha en una cadena, concatenando fichas. Macro multilínea, define predefinida. afirmar Macro variable, macro sobrecarga. Pimpl idioma.


10. Bibliotecas. Bibliotecas estáticas y dinámicas. Estático vs Dinámico. Descarga de bibliotecas sin vinculación previa. Funciones exportadas. Tecnología COM.


11. Multithreading. Proceso e hilos. Tipos de multitarea. Situaciones problemáticas. Crear secuencia El objeto de flujo es std :: thread. Funciones auxiliares Exclusión mutua. Gestión de cerraduras. Captura de excepciones. Rechazo de excepción. Variables de condición.


12. Multithreading. Parte 2. Cálculos diferidos. Futuro y promesa. Asíncrono y Futuro. Cola de eventos / mensajes. Contratos, conceptos.
