Database yang Dikelola di Selectel: Selamat Datang di Beta


Hari ini kami memperkenalkan versi beta terbuka dari Managed Databases untuk PostgreSQL, yang penggunaannya akan gratis untuk periode beta.

Basis data adalah salah satu komponen paling penting dan kompleks dari setiap sistem informasi atau aplikasi. Pembuatan, konfigurasi, dan proses manajemen basis data manual dapat memakan waktu berminggu-minggu atau bahkan berbulan-bulan.

Dengan pertumbuhan bisnis, yang mengarah pada pertumbuhan infrastruktur, diperlukan untuk memastikan skalabilitas basis data. Pada saat yang sama, keandalan dan toleransi kesalahan mereka diletakkan di tempat pertama, karena ketersediaan layanan yang disediakan tergantung pada ini. Menerapkan persyaratan ini membutuhkan waktu yang berharga untuk menyelesaikan masalah bisnis dan mengembangkan aplikasi Anda. Selain itu, perusahaan tidak selalu memiliki waktu, uang, dan spesialis yang memenuhi syarat untuk menyelesaikan masalah ini.

Itu sebabnya kami memutuskan untuk membuat layanan manajemen database yang sepenuhnya Dikelola Basis Data, berkat itu Anda dapat fokus pada pengembangan bisnis Anda, dan bukan pada pemeliharaan infrastruktur.

Apa yang Dikelola Database


Managed Databases (DBaaS) adalah layanan yang memungkinkan Anda untuk dengan cepat menyebarkan cluster basis data di cloud. Konfigurasi, pemeliharaan, dan keandalan cluster disediakan di sisi Selectel. Database yang Dikelola berjalan dengan kekuatan komputasi platform Selectel Cloud . Layanan ini memungkinkan para insinyur dari semua tingkat keahlian untuk dengan cepat dan mudah menggunakan cluster basis data berkinerja tinggi dan andal tanpa membuang waktu untuk konfigurasi mereka.

Database yang Dikelola memecahkan sebagian besar masalah yang dihadapi perusahaan dan pengembang saat membuat cluster database dari awal:

  • penentuan infrastruktur optimal untuk penyebaran basis data;
  • Memperbesar infrastruktur saat bisnis tumbuh
  • keandalan database dan manajemen toleransi kesalahan;
  • memperkirakan biaya infrastruktur.

Alih-alih menjalankan perintah untuk menginstal dan mengkonfigurasi database dari terminal, sekarang dimungkinkan untuk menggunakan cluster database yang sudah jadi dalam beberapa klik di panel kontrol Selectel.

Manfaat penggunaan:

  • Hemat waktu dan uang ketika menggunakan cluster database yang andal dan berkinerja tinggi
  • kurangnya kebutuhan untuk memilih dan mengkonfigurasi server untuk hosting database;
  • Tidak perlu mengkonfigurasi pengaturan cluster database
  • Pembuatan cluster database toleran-kesalahan yang mudah yang melindungi terhadap crash aplikasi.

Prinsip kerja


Layanan Dikelola Database terdiri dari dua bagian: tingkat pengguna dan tingkat data.


  • Tingkat pengguna adalah antarmuka interaksi pengguna dengan layanan, pada kenyataannya itu adalah API yang melaluinya pengelolaan cluster basis data: pembuatan dan penghapusan cluster, basis data dan pengguna. Pada titik ini, zona tanggung jawab pengguna berakhir dan operasi aktual dari layanan Database Dikelola dimulai.
  • Lapisan data - tingkat penyebaran langsung dan penyimpanan cluster database yang dibuat di platform cloud Selectel. Semua cluster terisolasi satu sama lain. Level ini adalah titik masuk bagi aplikasi Anda untuk bekerja dengan database.

Pengguna tidak menyiapkan infrastruktur server, tidak menginstal sistem operasi, dan tidak melakukan pengaturan server awal - layanan melakukan ini untuknya. Sebagai gantinya, pengguna memilih versi database PostgreSQL, konfigurasi server, dan tipe kluster.

Setelah cluster digunakan di platform cloud, pengguna membuat basis data, ia muncul dalam beberapa menit dan melaporkan kesiapannya untuk bekerja. Kemudian pengguna basis data dibuat, ia diberikan akses ke basis data yang diinginkan.

Itu saja, cluster database siap bekerja: sambungkan, buat tabel, isi dengan data dan baca.

Seorang pengguna dapat terhubung ke database menggunakan klien apa pun untuk bekerja dengan PostgreSQL atau dari aplikasinya. Untuk mendapatkan data untuk terhubung ke database, buka tab Pengaturan. Informasi cluster akan menampilkan alamat untuk menghubungkan ke server cluster yang dibuat. Anda dapat membaca lebih lanjut tentang menghubungkan ke database di basis pengetahuan .

Kami meluncurkan layanan dengan dukungan untuk dua versi DBMS PostgreSQL - 11 dan 12, salah satu database relasional yang paling populer dan produktif.

Fitur beta dari layanan:

  • 2 versi DBMS tersedia: PostgreSQL 11.6 dan 12.1.
  • 3 konfigurasi server tersedia:
    • 2 vCPU, 4 GB RAM, 32 GB disk lokal;
    • 2 vCPU, 8 GB RAM, 64 GB disk lokal;
    • 4 vCPU, RAM 16 GB, disk lokal 128 GB.

Membuat cluster database:

  • cluster dari satu server - cluster untuk bereksperimen, mengembangkan atau menguji aplikasi Anda;
  • cluster toleran terhadap kesalahan dari tiga server, 1 master dan 2 replika - untuk aplikasi di lingkungan produk.

Fitur Beta:

  • Anda bisa membuat hingga 50 database dan 50 pengguna dalam sebuah cluster.
  • Seorang pengguna dapat membuat tidak lebih dari dua cluster di setiap wilayah platform cloud.

Cara mulai menggunakan


Proses mempersiapkan database cluster sangat sederhana. Setelah mendaftarkan akun dan memasuki panel kontrol, cukup ikuti beberapa langkah:

Langkah 1


Pergi ke proyek platform cloud atau buat yang baru.


Langkah 2


Buka tab Databases dan buat sebuah cluster .


Langkah 3


Isi formulir dengan memilih konfigurasi dan pengaturan cluster.


Langkah 4


Setelah membuat cluster, Anda perlu membuat basis data, pengguna, dan menetapkan mereka hak akses.


Langkah 5


Lihat alamat untuk menghubungkan ke database dalam pengaturan cluster.


Langkah 6


Sambungkan ke database, misalnya, menggunakan utilitas psql . 

# psql "host=<ip-address-1>,<ip-address-2>,<ip-address-3> port=6432 user=<db-user-name> dbname=<database-name>"

Rencana pengembangan


Kami berupaya memberi Anda lebih banyak opsi basis data di Selectel. Antrian implementasi sudah memiliki:

  • membuat dan mengelola basis data melalui API;
  • penskalaan sumber daya yang cepat dan mengubah jumlah replika dalam kelompok kerja;
  • membuat cadangan dan memulihkan data;
  • alamat domain tunggal untuk menghubungkan ke cluster database;
  • Mekanisme sederhana untuk memperbarui versi DBMS minor;
  • memantau kinerja dan kesehatan cluster basis data;
  • beralih server master ke replika atas permintaan pengguna;
  • Dukungan MySQL dan Redis.
