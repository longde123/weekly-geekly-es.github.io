Aktualisieren eines Laptops mit Windows 10 1903 - vom Baustein bis zum Verlust aller Daten. Warum kann ein Update mehr als ein Benutzer sein?

Microsoft Corporation auf der neuesten Version des Betriebssystems Win10 zeigt uns die Wunder der Möglichkeiten der Aktualisierung. Jeder, der keine Daten aus dem Update von 1903 verlieren möchte, ist zu cat eingeladen.

Einige Punkte, die zur Unterstützung von Microsoft selten beachtet werden, sind die Annahmen des Autors des Artikels, werden als Ergebnis von Experimenten veröffentlicht und erheben keinen Anspruch auf Zuverlässigkeit.

  1. Es gibt eine Liste von Anwendungen, die ein Update eindeutig überstehen. Einige ältere Anwendungen können das Update aufgrund nicht dokumentierter Funktionen beschädigen.
  2. Windows 10 fördert aktiv das Konzept, dass der Benutzer der beste Software-Tester ist.
  3. Wenn Sie versehentlich mit einer großen Sammlung von Multimedia- und Mobilgeräten von Microsoft arbeiten, kann das System aufgrund nicht dokumentierter Indizierungsalgorithmen zusammenbrechen

Selten erwähnte Wikipedia-Informationen zu UWP

Lesen Sie nur für Hardcore-Entwickler
Universal Windows Platform (Universal Windows Platform, kurz für UWP) - eine Plattform, die von Microsoft erstellt und erstmals in Windows 10 eingeführt wurde. Mit dieser Plattform können Sie universelle Anwendungen erstellen, die sowohl unter Windows 10 als auch unter Windows 10 Mobile ausgeführt werden, ohne Änderungen vorzunehmen im Code. Das Erstellen solcher Anwendungen in C ++, C #, VB.NET und XAML wird unterstützt. Die API ist in C ++ implementiert und wird in C ++, VB.NET, C #, F # und JavaScript unterstützt. Entwickelt als Erweiterung von Windows Runtime (der in Windows Server 2012 und Windows 8 eingeführten Plattform) können Sie Anwendungen auf verschiedenen Hardwareplattformen ausführen.

Ich kaufte einen frischen Laptop unter 10 ku.

Ich war überrascht, dass nach dem Anschließen der zweiten Festplatte die Mediendateien sehr lange indiziert wurden. Um mit Multimedia auf externen Geräten zu arbeiten, habe ich vor langer Zeit einen Zune-Player installiert. Das System wurde fehl am Platz aktualisiert. Infolgedessen durfte ich mit dem Update von 1903 freundlicherweise den Zeitpunkt für das Update auswählen.

Ich wählte ...

In der Regel wird Windows 10 aktualisiert, wenn Updates angezeigt werden. Aber! Das Update von 1903 war umfangreich und nach drei Betriebsstunden begann der Computer, völlige Dinge zu zeigen.

Ich habe mit der Installation des Updates begonnen - ich habe den Benutzer verloren. % Benutzername% .0001 ...
Es gab einen Benutzernamen, der sich nach einem Neustart geändert hat. Es stellte sich heraus, dass dies eine Reaktion auf den Media Player ist.

Es gab zwei Scheiben. Ein System, ein anderes für Daten.

Die zweite Scheibe stellte sich als Ziegel heraus.



Dies bedeutet, dass die Festplatte aus unbekannten Gründen vom Anfang und Ende des Windows-Snap-Ins unter einem unbekannten Dateisystem um Megabyte abgeschnitten wurde.

Ich schaue was passiert ist.

Es wurde notwendig, das Snap-In auszuführen, um diese Änderungen zu entfernen.
Das Schlimmste ist jedoch, dass das Update aufgrund des Mediaplayers nicht aufgezeichnet werden konnte
Benutzersystemeinstellungen. Wahrscheinlich hat niemand darüber nachgedacht.

Und die Startschaltfläche - die UWP-Anwendung - verschwand für immer, nachdem versucht wurde, den Benutzernamen an die Registrierung zurückzugeben.

Ein paar Worte zum Abschluss des Artikels.

  1. Ohne die Struktur des C-Laufwerks hätte es höchstwahrscheinlich einen Ziegelstein gegeben. Wenn es nur eine Festplatte gäbe, würde sich die Wahrscheinlichkeit eines Datenverlusts um ein Vielfaches erhöhen.
  2. Das Update hat den Domäneneintrag zerstört, das Programm muss neu konfiguriert werden, auch Visual Studio von Microsoft hat einen solchen Angriff nicht erlitten.
  3. Es wurde experimentell festgestellt, dass UWP-Anwendungen Benutzerinformationen immer noch irgendwo speichern, aber es gibt keine effektive Möglichkeit, mit Benutzerinformationen zu arbeiten. UWP besteht außerdem der Verdacht, dass Entwickler für Android und iOS es nicht eilig haben, ihre Informationen zu portieren Bei Anwendungen für Windows Mobile wird der Standard in Zukunft nicht mehr unterstützt und entwickelt.
