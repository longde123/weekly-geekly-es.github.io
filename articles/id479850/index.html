Instal Exchange 2019 pada Windows Server Core 2019

Microsoft Exchange adalah prosesor besar yang mencakup penerimaan dan pemrosesan surat, serta antarmuka web untuk server surat Anda, akses ke kalender dan tugas perusahaan. Exchange diintegrasikan ke dalam Active Directory, jadi bayangkan itu sudah digunakan.

Nah, Windows Server 2019 Core adalah versi Windows Server tanpa antarmuka grafis.

Tidak ada Windows tradisional di versi Windows ini, tidak ada yang diklik, tidak ada menu Mulai. Hanya jendela hitam dan garis perintah hitam. Tetapi pada saat yang sama, ada area yang lebih kecil untuk serangan dan peningkatan level entri, karena kami tidak ingin ada yang menggali sistem kritis, kan?

Panduan ini juga cocok untuk server dengan GUI.


1. Hubungkan ke server


Buka Powershell dan masukkan perintah: 

Enter-PSSession 172.18.105.6 -Credential Administrator

Opsional: Aktifkan RDP. Ini menyederhanakan instalasi, tetapi tidak sama sekali. 

 cscript C:\Windows\System32\Scregedit.wsf /ar 0

Dalam gambar dari Ultravds, RDP sudah disertakan.

2. Hubungkan server ke AD


Ini dapat dilakukan melalui Windows Admin Center atau melalui Sconfig di RDP.

2.1 Tentukan server DNS atau pengontrol domain



Di Windows Admin Center, kami terhubung ke server, pergi ke bagian jaringan dan tentukan alamat ip pengontrol domain atau server domain DNS.


Melalui RDP, masukkan "Sconfig" di baris perintah dan masuk ke jendela konfigurasi server biru. Di sana kami memilih titik 8) Pengaturan Jaringan, dan kami melakukan hal yang sama dengan menentukan server domain DNS.

2.2 Kami bergabung dengan server ke domain



Di WAC, klik "Ubah ID Komputer" dan kami diberi jendela yang familier untuk memilih workgroup atau domain. Semuanya seperti biasa, pilih domain dan gabung.


Menggunakan Sconfig, Anda harus terlebih dahulu memilih item 1, pilih apakah kami bergabung dengan workgroup atau domain, tentukan domain, jika kami bergabung dengan domain. Dan hanya setelah selesainya prosedur kami akan diizinkan untuk mengubah nama server, tetapi bahkan untuk ini Anda harus memasukkan kata sandi lagi.

Powershell membuat ini lebih mudah: 

 Add-Computer -DomainName test.domain -NewName exchange -DomainCredential Administrator

3. Pasang




Jika Anda menggunakan RDP, Anda harus menginstal komponen yang diperlukan sebelum menginstal Exchange itu sendiri. 

 Install-WindowsFeature Server-Media-Foundation, RSAT-ADDS

Selanjutnya, kita perlu mengunduh gambar disk dengan installer Exchange. 

 Invoke-WebRequest -UseBasicParsing -Uri 'https://website.com/ ExchangeServer2019-x64.iso -OutFile C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

Mount ISO: 

 Mount-DiskImage C:\Users\Administrator\Downloads\ExchangeServer2019-x64.iso

Jika Anda akan melakukan ini semua melalui baris perintah, Anda hanya perlu memasang disk yang diunduh dan masukkan perintah: 

 D:\Setup.exe /m:install /roles:m /IAcceptExchangeServerLicenseTerms /InstallWindowsComponents

Kesimpulan


Seperti yang Anda lihat, menginstal Exchange pada Windows Server Core, serta memasuki domain bukanlah proses yang menyakitkan, dan mempertimbangkan bagaimana kami menang dalam keamanan, itu sangat berharga.

Saya sangat senang bahwa memasukkan server ke dalam AD menggunakan Powershell lebih mudah daripada menggunakan GUI atau Windows Admin Center.

Sangat disayangkan bahwa opsi instalasi Exchange ditambahkan hanya untuk Exchange 2019, sudah saatnya.
