E-book dan formatnya: berbicara tentang EPUB - sejarahnya, pro dan kontra

Sebelumnya di blog, kami menulis tentang bagaimana format e-book DjVu dan FB2 muncul.

Topik artikel hari ini adalah EPUB.


Gambar: Nathan Oakley / CC BY

Format riwayat


Pada 90-an, solusi eksklusif mendominasi pasar e-book. Dan banyak produsen pembaca memiliki format sendiri. Misalnya, NuvoMedia menggunakan file dengan ekstensi .rb. Ini adalah wadah dengan file HTML dan file .info yang mengandung metadata. Keadaan ini mempersulit pekerjaan penerbit - mereka harus mengetikkan buku untuk setiap format secara terpisah. Sekelompok insinyur dari Microsoft, yang telah disebutkan oleh NuvoMedia dan SoftBook Press, menangani situasi ini.

Pada saat itu, Microsoft akan menaklukkan pasar e-book dan sedang mengembangkan aplikasi pembaca untuk Windows 95. Kita dapat mengatakan bahwa membuat format baru adalah bagian dari strategi bisnis raksasa TI.

Jika kita berbicara tentang NuvoMedia, maka perusahaan ini dianggap sebagai produsen pembaca elektronik massal pertama, Rocket eBook . Memori internal perangkat hanya delapan megabyte, dan masa pakai baterai tidak melebihi 40 jam. Adapun SoftBook Press, mereka juga mengembangkan pembaca elektronik. Tetapi perangkat mereka memiliki fitur khas - modem bawaan - memungkinkan Anda untuk mengunduh literatur digital langsung dari toko SoftBookstore.

Pada awal 2000-an, kedua perusahaan - NuvoMedia dan SoftBook - dibeli oleh perusahaan media Gemstar dan bergabung ke dalam Gemstar eBook Group. Organisasi ini telah menjual pembaca selama beberapa tahun (misalnya, RCA REB 1100 ) dan buku digital, tetapi pada tahun 2003 ia keluar dari bisnis .

Namun kembali ke pengembangan standar tunggal. Pada tahun 1999, Microsoft, NuvoMedia, dan SoftBook Press mendirikan Open eBook Forum, yang termasuk mengerjakan draft dokumen yang menandai awal EPUB. Awalnya, standar itu disebut OEBPS (singkatan dari Open EBook Publication Structure). Itu memungkinkan untuk mendistribusikan publikasi digital dalam satu file (arsip ZIP) dan menyederhanakan transfer buku antara platform perangkat keras yang berbeda.

Kemudian, Adobe, IBM, HP, Nokia, Xerox, dan penerbit McGraw Hill dan Time Warner bergabung dengan Open eBook Forum. Bersama-sama, mereka terus mengembangkan OEBPS dan mengembangkan ekosistem literatur digital secara keseluruhan. Pada tahun 2005, organisasi ini berganti nama menjadi Forum Internasional tentang Publikasi Digital, atau IDPF .

Pada 2007, IDPF mengubah nama format OEBPS menjadi EPUB dan mulai mengembangkan versi keduanya. Dia diperkenalkan ke masyarakat umum pada tahun 2010. Kebaruan itu hampir tidak berbeda dari pendahulunya, tetapi menerima dukungan untuk grafik vektor dan font yang tertanam.

Pada saat ini, EPUB telah menaklukkan pasar dan menjadi standar standar bagi banyak penerbit dan produsen gadget elektronik. Format ini sudah digunakan oleh O'Reilly dan Cisco Press, ditambah lagi didukung oleh Apple, Sony, Barnes & Noble, perangkat ONYX BOOX.

Pada 2009, proyek Google Books mengumumkan dukungan untuk EPUB - ini digunakan untuk mendistribusikan lebih dari satu juta buku gratis. Format mulai mendapatkan popularitas di kalangan penulis. Pada 2011, Joan Rowling berbicara tentang rencana untuk meluncurkan situs web Pottermore dan menjadikannya satu-satunya titik penjualan digital untuk buku-buku Potteriana.

EPUB dipilih sebagai standar untuk distribusi literatur, terutama karena kemampuannya untuk menerapkan perlindungan salinan ( DRM ). Semua buku di toko online penulis masih tersedia hanya dalam format ini .

Versi ketiga format EPUB dirilis pada 2011. Pengembang telah menambahkan kemampuan untuk bekerja dengan file audio dan video dan catatan kaki. Saat ini, standar terus berkembang - pada 2017 IDPF bahkan bergabung dengan konsorsium W3C, yang menerapkan standar teknologi untuk World Wide Web.

Bagaimana cara kerja EPUB?


Buku dalam format EPUB adalah arsip ZIP. Ini menyimpan teks publikasi dalam bentuk halaman XHTML atau HTML atau file PDF. Juga dalam arsip adalah konten media (audio, video atau gambar), font dan metadata. Mungkin juga berisi file tambahan dengan CSS atau dokumen gaya PLS dengan informasi untuk layanan pembuatan pidato.

Markup XML bertanggung jawab untuk menampilkan konten. Sebuah fragmen buku dengan audio dan gambar internal mungkin terlihat seperti ini : 

<?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE html> <html xmlns="http://www.w3.org/1999/xhtml" xmlns:epub="http://www.idpf.org/2007/ops" xmlns:ev="http://www.w3.org/2001/xml-events" epub:prefix="media: http://idpf.org/epub/vocab/media/#"> <head> <meta charset="utf-8" /> <link rel="stylesheet" type="text/css" href="../css/shared-culture.css" /> </head> <body> <section class="base"> <h1>the entire transcript</h1> <audio id="bgsound" epub:type="media:soundtrack media:background" src="../audio/asharedculture_soundtrack.mp3" autoplay="" loop=""> <div class="errmsg"> <p>Your Reading System does not support (this) audio</p> </div> </audio> <p>What does it mean to be human if we don't have a shared culture? What does a shared culture mean if we can't share it? It's only in the last 100, or 150 years or so, that we started tightly restricting how that culture gets used.</p> <img class="left" src="../images/326261902_3fa36f548d.jpg" alt="child against a wall" /> </section> </body> </html>

Selain file konten, arsip berisi dokumen navigasi khusus (Dokumen Navigasi). Ini menggambarkan tata letak teks dan gambar dalam sebuah buku. Aplikasi pembaca mengaksesnya jika pembaca ingin "melompati" beberapa halaman.

File lain yang diperlukan dalam arsip adalah paket. Ini termasuk metadata - informasi tentang penulis, penerbit, bahasa, judul, dan sebagainya. Ini juga termasuk tulang belakang subbagian buku. Contoh dokumen paket dapat ditemukan di repositori IDPF di GitHub .

Keuntungan


Keuntungan dari format ini adalah fleksibilitasnya. EPUB memungkinkan Anda untuk membuat tata letak dokumen yang dinamis yang menyesuaikan dengan ukuran layar perangkat. Ini adalah salah satu alasan utama mengapa format ini mendukung sejumlah besar pembaca (dan perangkat elektronik lainnya). Misalnya, semua pembaca ONYX BOOX bekerja "di luar kotak" dari EPUB: dari dasar dan 6-inci Caesar 3 ke premium dan Euclid 9,7-inci.


/ ONYX BOOX Caesar 3

Karena formatnya didasarkan pada standar populer (XML), mudah dikonversi untuk dibaca di Internet. EPUB juga mendukung elemen interaktif. Ya, mereka adalah elemen serupa dalam PDF, tetapi Anda dapat menambahkannya ke dokumen PDF hanya menggunakan perangkat lunak berpemilik. Dalam kasus EPUB, mereka ditambahkan ke buku dengan markup dan tag XML dalam editor teks apa pun.

Kekuatan EPUB lainnya adalah fitur-fiturnya untuk orang dengan masalah penglihatan atau disleksia. Standar ini memungkinkan Anda untuk memodifikasi tampilan teks di layar - misalnya, sorot kombinasi huruf tertentu.

Juga EPUB, seperti yang telah kita catat, memberi penerbit kemampuan untuk mengatur perlindungan salinan. Jika diinginkan, penjual buku elektronik dapat menggunakan mekanisme mereka untuk membatasi akses ke dokumen. Untuk melakukan ini, ubah file rights.xml dalam arsip.

Kekurangan


Untuk membuat publikasi EPUB, Anda perlu memahami sintaks XML, XHTML, dan CSS. Pada saat yang sama, Anda harus bekerja dengan sejumlah besar tag pengenal. Sebagai perbandingan, standar FB2 yang sama hanya mencakup set tag yang minimal diperlukan - cukup untuk fiksi penyusunan huruf. Dan untuk membuat dokumen PDF tidak memerlukan pengetahuan khusus sama sekali - perangkat lunak khusus bertanggung jawab untuk semuanya.

EPUB juga dikritik karena kompleksitas desain komik dan buku-buku lain dengan banyak ilustrasi. Dalam hal ini, penerbit harus membuat tata letak statis dengan koordinat tetap untuk setiap gambar - ini membutuhkan banyak waktu dan upaya.

Apa selanjutnya


IDPF saat ini sedang mengerjakan spesifikasi baru untuk format tersebut. Misalnya, salah satunya akan membantu membuat tutorial interaktif dengan bagian tersembunyi . Buku yang sama akan terlihat berbeda untuk guru dan siswa - dalam kasus kedua, misalnya, jawaban untuk tes atau pertanyaan kontrol akan disembunyikan.


Gambar: Guian Bolisay / CC BY-SA

Fitur baru ini diharapkan dapat membantu mengatur kembali proses pendidikan. Saat ini, EPUB cukup aktif digunakan oleh universitas besar, misalnya, Universitas Oxford. Beberapa tahun yang lalu, mereka menambahkan dukungan EPUB 3.0 ke aplikasi perpustakaan digital mereka.

IDPF juga membuat spesifikasi untuk menyematkan catatan kaki Open Annotation di EPUB. Standar ini dikembangkan di W3C pada 2013 - ini menyederhanakan pekerjaan dengan jenis anotasi kompleks. Misalnya, dengan bantuannya, Anda dapat meletakkan catatan ke bagian tertentu dari gambar JPEG. Selain itu, standar mengimplementasikan mekanisme untuk menyinkronkan perubahan dalam anotasi antara salinan satu dokumen EPUB. Catatan format Open Annotation dapat ditambahkan ke file EPUB sekarang, bagaimanapun, spesifikasi formal untuk mereka belum diadopsi.

Pekerjaan juga sedang dilakukan pada versi standar baru - EPUB 3.2. Format WOFF 2.0 dan SFNT akan muncul di dalamnya, yang digunakan untuk mengompresi font (dalam beberapa kasus, mereka dapat mengurangi ukuran file hingga 30%). Selain itu, pengembang akan mengganti beberapa atribut HTML yang usang. Misalnya, alih-alih elemen pemicu terpisah untuk mengaktifkan file audio dan video, standar baru akan memiliki elemen audio dan video HTML asli.

Draf spesifikasi dan daftar perubahan sudah tersedia di repositori W3C GitHub.
