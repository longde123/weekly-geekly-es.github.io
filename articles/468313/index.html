Introduciendo la fuente de Cascadia Code

¡Cascadia Code finalmente está aquí! Puede instalarlo directamente desde la página de versiones del repositorio de GitHub o recibirlo automáticamente en la próxima actualización de Windows Terminal.



Espera, ¿qué es el código Cascadia?


Cascadia Code se anunció el pasado mes de mayo en el evento Build de Microsoft. Es la fuente monoespaciada más reciente enviada desde Microsoft y proporciona una experiencia nueva para las experiencias de línea de comandos y editores de código. Cascadia Code fue desarrollado de la mano con la nueva aplicación de Windows Terminal. Se recomienda usar esta fuente con aplicaciones de terminal y editores de texto como Visual Studio y Visual Studio Code.

Este artículo en nuestro blog .

Programando ligaduras


¡Cascadia Code admite la programación de ligaduras! Las ligaduras de programación son más útiles al escribir código, ya que crean nuevos glifos al combinar caracteres. Esto ayuda a que el código sea más legible y fácil de usar para algunas personas.



Nota: Si está utilizando Visual Studio Code, deberá habilitar las ligaduras de fuente en su configuración para poder verlas.



¿Mencionamos que es de código abierto?


Sí, has oído bien! Cascadia Code está licenciado bajo la licencia SIL Open Font en GitHub , ¡así que no dude en contribuir como desee!

Nota: Actualmente no hay un marco de prueba adecuado integrado en el repositorio, por lo que todas las solicitudes de extracción deberán probarse y validarse manualmente. Esto ralentizará las aprobaciones de solicitudes de extracción, así que tenga paciencia con nosotros.

¿Por qué se llama Cascadia Code?


El nombre Código Cascadia se originó en el proyecto Terminal de Windows. Antes de su lanzamiento, el nombre en clave de Windows Terminal era Cascadia. De hecho, ¡algunos de los archivos fuente dentro de la Terminal todavía usan este nombre! Como homenaje a la Terminal, nos gustó la idea de nombrar la fuente después de su nombre en clave.
Agregamos Código al final del nombre de la fuente para ayudar a indicar que esta fuente estaba destinada a la programación. Específicamente, ayuda a identificar que incluye ligaduras de programación.

Para garantizar que Cascadia Code fuera la elección correcta para el nombre de la fuente, realizamos una encuesta en Twitter junto con otros nombres que estábamos considerando. Estábamos muy agradecidos por la participación de todos y estábamos muy emocionados de que Cascadia Code fuera el ganador.



¿Quién diseñó el código Cascadia?


Muchas gracias a Aaron Bell, el diseñador de Cascadia Code. Tuvimos la suerte de trabajar con él durante el año pasado y ha hecho un gran trabajo. Aaron Bell es diseñador de fuentes con su propia compañía, Saja Typeworks , y ha trabajado con Microsoft durante muchos años. ¡Fue el diseñador de Selawik en 2015! Puedes seguirlo en Twitter en @aaronbell y también estará activo en nuestro repositorio de GitHub. ¡Planeamos continuar trabajando con él para ayudar a mejorar Cascadia Code y hacer que sea una fuente excelente para todos!

¿Qué sigue para el código Cascadia?


A partir de hoy, la versión 1909.16 del Código Cascadia está disponible públicamente en GitHub. Seguirá la sintaxis de versiones de Windows a medida que la fuente reciba actualizaciones con glifos nuevos y refinados.

Mantente conectado
