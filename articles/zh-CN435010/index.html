数字歌词：大量有关编程和程序员的歌曲和剪辑

人们普遍认为音乐能力和编程人才是相互联系的 。 许多专业音乐家对精确科学都情有独钟。 例如，鲍里斯·格里本什基科夫（Boris Grebenshchikov）毕业于圣彼得堡国立大学应用数学系，广播电台的吉他手乔尼·格林伍德（Jonny Greenwood）童年时期，在编程时呼吸不畅。

要在两个活动领域中的任何一个领域取得成功，您都需要具有创造性和系统地思考并识别模式的能力。 但是，即使这些类之间没有直接联系，也不能否认大多数程序员都对音乐感兴趣。 有些甚至写歌。 包括关于他的职业。

我们将谈论他们。


图片凯蒂·衣柜 （ Katie Wardrobe） / CC BY-SA


音乐作者



代码猴子

乔纳森·库尔顿（Jonathan Coulton）无需介绍。 通过这位美国作家艺术家的所有作品，一个怪胎主题通过了。 Valve的游戏中他的歌曲为数百万听众所熟知。 好吧，他自己作为程序员的经验启发了乔纳森（Jonathan）创作了名为Code Monkey的热门歌曲 。

这首关于无意义的工作和不愉快的爱情的半自传式流行朋克歌曲很快赢得了Slashdot用户的喜爱，随后甚至在美国电视上播放了同名卡通的开场白。



OpenBSD发行歌曲

开源社区中不断交战的成员的武库是最多样化的。 OpenBSD团队有很多音乐。 从3.0版开始，操作系统的每个重要发行版都附带了一个组成部分和一本说明该插图的漫画书。

有时，产品只是在宣传新产品的功能，但有时会被用来攻击项目的意识形态敌人，例如Cisco和Apache员工。 我们建议您收听有关OpenBSD内部厨房的令人惊讶的高质量曲目 （意想不到的是嘻哈音乐与中世纪民谣的有机结合），该歌曲专门用于发行该项目歌曲的首张CD。

顺便说一句，已经有三个这样的收藏。


紧急集体一

Emergent Collective One是IT专业人员（主要是Flash开发人员）汇编的音乐。 流派光盘非常折衷：它既包含舞蹈音乐又包含实验电子产品。 特别是在这种背景下，关于2000年代中期示例PHP的黑帮说唱特别引人注目。 该专辑是根据知识共享许可发布的， 可以在Bandcamp上免费下载 。


IE对我很刻薄

我们以一首歌结束了本节，幸运的是，这首歌已经失去了意义。 程序员斯科特·沃德（Scott Ward）在加拿大的一家初创公司使用.us做前端工作， 在Internet Explorer下写了一首关于布局难题的歌曲 。 数年过去了，IE和Scott工作的公司一样也被遗忘了。 但是您总是可以听这首歌，并记住过去。 然后松一口气。



主题封面



免费软件歌曲

理查德·斯托曼（Richard Stallman）很难被爱，也无法恨。 GNU Emacs和GCC的创建者为自由软件的普及做出了巨大的贡献。 但这并不能阻止他的往绩。 事实证明，理查德很久以来就构成了世界各国人民歌曲的怪异模仿 。 最著名的是免费软件歌曲 ，它是根据保加利亚民间Sadi moma bela loza的动机演奏的。

理查德（Richard）充满讽刺意味地从事自己的职业，并且不认为自己是诗人还是音乐家，这是令人高兴的。 我们邀请您享受由大师本人演奏的免费软件颂歌。



公关墨西哥

我们名单上的第一支俄罗斯队。 PR-Mex是Nizhny Novgorod的音乐项目，由1C开发人员Leonid Pautov和Artyom Eremin组成。 一段时间以来，这些家伙演奏重金属，但幽默的材料使他们成名。

二人组成立后的第六年，于2010年发行了第一张录音室录音，并在RuNet中迅速流行。 这个著名的联合曲目Eminem和Dido的戏仿讲述了一个普通程序员的故事，该程序员未能尝试与1C执行董事联系，并同时发疯。

作曲成功后，小组开始应邀表演 。 尽管最近几年该团队已经平静下来，但许多Runet用户仍然记得它。



可扩展性

硅谷俘获了新的狂热。 现在几乎每个雄心勃勃的创业公司都有自己的无伴奏乐队 ，他们定期举办联合音乐会 。

苹果员工在主题演讲中唱歌，Facebook员工将乐队命名为The Vocal Network，InTune则代表LinkedIn。 对此类音乐的需求正在增长：每年的竞争都在加剧。 在同一时间，Google组成了至少三个和无伴奏合唱团-Googapella，Alphabeat和ScaleAbility； 后者将进行讨论。

ScaleAbility已经存在了十多年。 他们假装很受欢迎，不像在Google那样专注于编程。 他们的曲目包括有关内部技术支持 ， Go编程语言的歌曲，甚至是乔纳森·库尔顿（Jonathan Coulton）先前提到的歌曲的翻唱。

在新年假期中，他们院子里将有12天的圣诞节变更，其中列出了节日期间出现的技术问题，而不是礼物。



迪伦·贝蒂

系统架构师，伦敦.NET社区的杰出人物和兼职音乐家Dylan Beattie模仿流行歌曲并在世界各地的会议上进行表演。 此外，他还学习俄语，并定期访问俄罗斯，包括分享他的经验。

他将拥有各种口味的曲目： 在Pink Pink Floyd的音乐中， Dylan抱怨大公司中的过时做法，在模仿Peter Gabriel （Peter Gabriel）时谈到专业发展的重要性，并在《 You Give REST Bad Name》中责骂不良的API开发人员。


会议模仿



SUSECon

播放流行歌曲的有趣视频是吸引会议注意力的好方法。 此策略已被IT公司采用。 SUSECon的组织者年复一年地发布模仿著名歌曲的高质量剪辑。 SUSE品牌始终在竞争对手中脱颖而出，这种促销方式非常适合他。

有趣的是，公司员工自己参加了这些歌曲的专业录制。 他们既有披头士乐队的模仿（ CoDe Together ），针对机器的狂怒（ 以的名义编码 ），也有更多的商业材料。 例如，在模仿Maroon 5（ SUSE。是的，请 ）的情况下，他们在一个几乎空着的数据中心中组织了一场演唱会惊喜-类似于原始视频，该乐队参加了随机婚礼。 然后，他们将Mark Ronson和Bruno Mars的Uptown Funk变成了一首关于无需重新启动即可进行补丁的歌曲（ Uptime Funk ）。

这些视频成为会议形象不可或缺的一部分，因此明年组织者甚至发起了一场音乐短剧比赛。



Java会议

Java会议的组织者特别喜欢宣传音乐视频，以消除Java作为一种过时的，专有的企业语言的神话。

挪威JavaZone是该列表中的第一位。 多亏了她的宣传视频，它的流行程度远远超出了Java社区。 您一定已经看过他们的《闪灵》 （写一次，毕生难忘 ）和《权力的游戏》 （字节与电线之歌）的模仿。 八年前，在组织JavaZone 2010时，组织者发布了模仿扑克之星的歌手Lady Gaga（ Lady Java ）的模仿作品。

一年后，甲骨文制作了自己的模仿说唱视频并将其发布时间定于JavaOne2011。尽管引起了共鸣-该视频获得了超过一百五十万次观看，但该公司不再制作类似的剪辑。


最终，2014年JPoint俄罗斯会议的组织者不仅发布了“激动人心”的说唱视频 ，还说服了Nikita Dzhigurda在摄像机上朗诵一首赞扬这种编程语言的诗 。
