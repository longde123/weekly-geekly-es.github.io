Windows Server 2019

Windows Server的历史可以追溯到25年前：Windows NT 3.1 Advanced Server于1993年7月27日发布。 在上一个主要版本发布三年之后的2018年10月，Windows Server 2019发布了.Windows Server 2019开发并改进了先前版本中固有的功能。 有关此的详细信息-削减！



从Windows Server 2016开始，采用了新的发行周期。 现在有两个发行渠道：LTSC（长期服务渠道）-每2-3年发布一次，并得到5年主要和5年扩展支持，以及半年一次渠道-每六个月发布一次，主要支持周期为6个月，扩展支持为18个月。 这两个渠道分别是什么？ 微软正在积极创新其Azure云平台。 这是对Linux虚拟机，具有Linux和Windows的容器以及许多其他技术的支持。

在云中使用这些技术的客户也希望在其数据中心中使用它们。 半年通道弥合了Azure与本地数据中心之间的鸿沟。 半年发行版适用于动态发展的公司，并已转换为为企业提供IT服务的灵活服务模型。 LTSC版本适用于使用支持周期长的完善应用程序的公司，例如Exchange Server，SharePoint Server，SQL Server以及基础结构角色，软件定义的数据中心和超融合基础结构。

Windows Server 2019正是LTSC通道上的版本。 它包含Windows Server 2016和后续半年版本中的所有功能更新。

Windows Server 2019开发人员的主要工作集中在四个关键领域：

  1. 混合云 -Windows Server 2019和新的Windows管理中心管理中心使您可以轻松使用Azure云服务：Azure备份，Azure站点恢复，Azure更新管理，Azure AD身份验证等。
  2. 安全是客户最重要的优先事项之一。 Windows Server 2019具有内置功能，使网络犯罪分子难以渗透并在系统中立足。 这些是Windows 10已知的Defender ATP和Defender Exploit Guard技术。
  3. 应用程序平台 -容器正成为将应用程序打包和交付给各种系统的现代趋势。 同时，Windows Server不仅可以运行本机Windows应用程序，还可以运行Linux应用程序。 为此，Windows Server 2019具有Linux容器，用于Linux的Windows子系统（WSL）和显着减小的容器映像大小。
  4. 超融合基础架构 -使您可以在单个服务器中结合标准架构以及计算和存储。 这种方法可显着降低基础架构的成本，同时提供出色的性能和可伸缩性。

Windows管理中心


Windows管理中心（WAC）是新的服务器管理工​​具。 它在基础结构中本地安装，并允许您管理Windows Server，Windows 10计算机，群集和超融合基础结构的本地实例和基于云的实例。

WAC是对现有管理工具（例如mmc，服务器管理器控制台）的补充而不是替代。 通过浏览器连接到WAC。



为了完成任务，使用了WinRM，WMI远程控制技术和PowerShell脚本。

您可以从组织的外部发布WAC并管理服务器。 Azure AD多因素身份验证和代理应用程序服务有助于保护此访问不受外部影响，并且使用Microsoft Enterprise Mobility + Security（EMS），您可以根据设备对策略，风险，位置和其他因素的遵守情况来授予或拒绝访问。 我认为，使用Web应用程序代替远程工作人员访问进行管理是正确的安全策略。



系统分析


Windows Server 2019变得更加智能。 使用新的System Insights功能，实施了预测分析，以实现从被动式服务器到主动式服务器机队管理的过渡。 机器学习模型考虑了性能计数器和事件，以准确地预测磁盘子系统上的可用空间问题，确定处理器计算，网络连接性和存储性能的趋势。



新的存储子系统


存储迁移服务


Windows Server 2019引入了一项新技术-存储迁移服务，该技术可将数据从旧服务器迁移到新服务器。

迁移分为几个阶段：

  • 各种服务器上的数据清单
  • 从源服务器快速传输文件，网络文件夹和安全配置
  • 捕获控制并将服务器标识符和网络设置从旧服务器替换为新服务器

迁移过程使用Windows管理中心进行管理。



Azure文件同步


Azure File Sync转换了传统的文件服务器，并将存储扩展到现实生活中几乎无法实现的卷。 数据分为几个级别：热缓存是存储在文件服务器磁盘上的数据，用户可以以最大速度访问。 随着数据冷却，数据被无缝地移动到Azure。 Azure File Sync可以与访问文件的任何协议结合使用：SMB，NFS和FTPS。

存储副本


此灾难保护技术首次出现在Windows Server 2016中。在版本2019中，出现了对Windows Server Standard版本的有限支持。 现在，如果公司的基础结构没有第二个远程数据中心，则小型公司还可以在Azure虚拟机中自动创建存储副本（副本）。

直接存储空间


本地磁盘空间是构建超融合基础架构和可伸缩文件服务器的必要组件。 Windows Server 2019内置了对非易失性内存的支持，改进了重复数据删除算法（包括具有ReFS文件系统的卷），每个群集的可扩展性高达4PB，并提高了性能。

故障转移群集更改


群集集已经出现，将可扩展性扩展到数百个节点。
为了确保具有偶数个节点的群集中的仲裁，使用了一种特殊的资源-见证磁盘。 在Windows Server 2012 R2时代，有必要为见证磁盘分配用于存储的磁盘，而在Windows Server 2016中，可以使用网络文件夹或云见证磁盘。 Windows Server 2019中的改进与小型企业减少的基础架构要求相关联。 见证磁盘可以是连接到例如路由器的USB磁盘。

域之间的群集迁移和故障转移群集服务中的其他改进已经出现。

应用程序平台的新功能


现在，您可以使用相同的Docker管理程序在同一容器节点上运行基于Windows和Linux的容器。 这使您可以在容器节点的异构环境中工作，并为开发人员提供了创建应用程序的灵活性。

容器提高了应用程序兼容性，服务器核心和Nano Server映像的大小显着减小，并且性能得到了提高。

Windows Server 2019已获得许多其他改进。 您可以在docs.microsoft.com官方文档网站的Windows Server部分中熟悉它们。

系统要求


该文档描述了安装Windows Server 2019的最低要求。您需要了解，根据角色和组件，运行的应用程序，服务器要求可能会增加。

  • 1.4 GHz 64位处理器
  • 符合X64的指令集
  • NX和DEP支持
  • 支持CMPXCHG16b，LAHF / SAHF和PrefetchW
  • 二级地址转换支持（EPT或NPT）

最低RAM要求：

  • 512MB（“带有台式机的服务器”安装选项为2GB）

储存要求：

Windows Server 2019不支持ATA / PATA / IDE和EIDE进行引导，交换文件或数据驱动器。 最小数量为32 GB。

网络适​​配器：

  • 带宽至少为1GB的以太网适配器。
  • 符合PCI Express体系结构规范。
  • 支持PXE远程启动协议。

Windows Server 2019 Standard和Datacenter版本的比较


Windows Server 2019有两个版本：标准版和数据中心版。

数据中心版具有高级功能：支持超融合基础架构，本地磁盘空间，使用虚拟化时的扩展许可权。
