Ode ao dia de um programador

Olá Habr!

Mais uma vez, desenvolvendo outro projeto em uma festiva manhã de sexta-feira, visitei de repente a musa, que compartilho com você.

Sabe, isso não é bobagem de um comediante!
Ode ao dia de um programador.
(nota do autor)

Em primeiro lugar, e esta é a lei!
Memorize uma equipe 

git clone

Então, para não entrar na vala
Combo de 

 composer install

e 

 composer update

Se você não é um "atacante qualificado"
Veja o que mostra 

 git branch

A seguir, e estas não são apenas palavras:
Olha o que vai mostrar 

 git branch

com bandeira 

 -a

Se você criar um novo local 

 git checkout

teste de acordo com a tradição
Não se esqueça de realizar reconhecimento,
Para o assunto de nomear ramos
Então tente a sua sorte
Alimentando o nome git branch
Não aconteceu que Cuming Out :
Não se esqueça de fazer o checkout imediatamente
E então para parar o fracasso
Veja novamente o que o ramo git dará
Depois de sentar-se sobre a tarefa noturna
Adicionar tudo: 

 git add . #

Esclareça os principais pontos: 

 git commit -m "[ ]"

Para que seu commit não se dobre
Teste a equipe 

 git pull

Se o seu código não é completo absurdo
Estamos aguardando o comando do servidor 

 git push
...
Uma missão furiosa terminará em breve.
Repositório. Em seguida, mescle a solicitação.
Com a consciência limpa, despeje o café
Somente quando a revisão do código estiver concluída.
E se os testes passarem,
Isso é bem-vindo ao Prod.
Aqui está um dia tão bom,
Se cada etapa estiver correta .

Convido todos a participar do trabalho.
