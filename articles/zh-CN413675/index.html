小丑2018：匿名Java开发人员俱乐部

为什么说小丑是个好现象？ 实际上，这是唯一可以使用Java进行交流的地方。


同时，Joker是个假期。 这不是一群老朋友感到无聊的地方，什么样的Java在刹车，什么样的Hibernate是拐杖，为什么在那里没有编译某些东西，等等-如果您知道我的意思。


小丑是一个您可以在背心上哭泣，获得建设性反馈并受到新技术启发的地方。 这是一个由热情的开发人员组成的特殊俱乐部，具有自己的规则和功能，我们将在本文中进行讨论。


不久将有一个新的小丑-连续第六个。 如果您有兴趣立即找出日期和时间，请看猫下，否则请继续阅读。


隐藏文字
什么 ：Joker是我们最大的面向经验丰富的开发人员的国际Java会议；
哪里 ：圣彼得堡， Expoforum ；
时间 ：2018年10月19日至20日


我们在说话！


我们聚在一起聊。 讨论和交流相关主题。 当然，Joker也会讲话-与他的发言人和他们的报告。 一年又一年，该计划的质量越来越好。 我们为内容，内容的质量以及各种主题和难度级别而拼命拼搏。 我们有来自世界各地的发言人。


并非总是如此。 您所看到的只是数十个人，尤其是计划委员会和演讲者本人的艰苦努力的结果。


我们的演讲者


文卡特·亚拉马尼亚姆


Venkat是世界上最好的演讲者之一，当然也是会议上最受期待的演讲者之一。 做出有力的报告，总是收集最高的评分。 我们将对他进行Habr采访， 关注网站上的新闻并订阅我们在Habr上的中心 。


关于报告。 现在，每个人都在谈论有效，廉价地实施变更，但这在实践中可行吗？ Venkat是敏捷开发人员，在多年的工作中，他成功地培训了数千名开发人员并参与了许多企业，他当然知道“有效实施”的秘密。 这就是为什么我们邀请他发表“别走复杂，奔跑”的报告 。 Venkat将分享他的观察结果，实际项目的经验，并讨论问题的原因和典型陷阱，展示其中的模式以及它们在不同项目和产品中的出现方式。


乔希·朗


也许世界上最酷的Spring传播者会告诉我们有关Reactive Spring的世界：基于Netty的Web运行时，Spring WebFlux，Spring Data Kay，Spring Security 5.0，Spring Boot 2.0，Spring Cloud Finchley。 看来，对于一个报告来说，这是非常大量的信息，但是请不要害怕：乔希（Josh）有办法将所有这些作为一个完整的反应性Spring进行讨论。





尼古拉·帕洛格


Nikolay是一位知名的演讲者，是与最新Java功能相关的主题的博客作者。 他的Twitter不断发布最新消息。
这些报告基于他自己的实践，他用自己的双手尝试一切。 他还因其著作《 Manning Publisher的Java模块系统》以及参与许多FOSS项目而闻名。


您可能已经猜到了，他的演讲“ Java 11在这里！” 从版本9到版本11，将致力于Java功能。 您想长时间使用的所有内容，但又害怕考虑： var ，新的API和旧版本的改进，多版本JAR，性能的改进等等。


格里特·格伦瓦尔德


Gerrit是一位出色的IoT演讲者，是JavaFX的熟练掌握者和一个桌面编程大师，被称为JavaOne星，JUG组织者，许多FOSS项目的创建者和参与者等。


Gerrit有很多报告，现在他将与他一起去Joker。 最明显的选择是“ 多设备控件-一种不同的UX方法 ”，主要针对对JavaFX，Swift，HTML5和类似事物感兴趣的UX开发人员。 这个故事将涉及一个声称具有以下特征的研究项目：假设我们在Web应用程序中具有普通的不方便的HTML表单。 让我们以通常的方式而不是使用移动设备来填写它们：该字段显示在监视器屏幕上，并且从手机中输入文本。 实际上，移动电话已成为表单元素的控制面板。 该报告不仅包含所有这些梦想如何实现的梦想，还讨论了解决问题的各种现实方法，从JavaFX与Gluon一起使用到移动电话上的Swift与浏览器中的Polymer的结合。


奥列格·谢拉耶夫（Oleg Shelaev）


Oleg是VM运行时的鉴赏家，也是为数不多的可以清晰，简单地讲俄语的有关运行时复杂事物的人之一。 特别是，Oleg直接谈到了GraalVM设备-该报告将是关于她的。 关于如何在不损失抽象性能的情况下编写现代Java代码。





奥列格·杜卡（Oleg Dokuka）


Oleg是Reactor 3的提交者，他积极推广Spring堆栈，并向公众提供基于它们的各种反应式解决方案。 他是《 Spring 5的反应式编程》的作者，也是著名的演讲者和实践工程师。


Oleg为我们带来了这样的核心报告，甚至在标题中写着：“ 反应性核心：如何构建发布服务器？ ”。 开发您的图书馆？ 您决定支持Reactive-Streams规范吗？ 是否想了解RxJava 2或Reactor 3的功能？ 还是您是铁杆爱好者？ 那么这份报告适合您！ 在报告期间，您将遍历操作员开发的各个方面，因此您将学习如何处理它，并了解为什么反应式库是按这种方式安排的，而不是其他方式。


瓦迪姆·捷斯科（Vadim Tsesko）


Vadim是Odnoklassniki平台的领先开发人员。 在此之前，他从事各种工作，例如水声学，静态代码分析研究，Yandex的高负载分布式服务等。


瓦迪姆将谈论“ ok.ru/音乐喷射器分销商”。 这种东西可以提供音频流的字节，通过成千上万的连接将速度提高到100 Gb / s。 故事将讲述Odnoklassniki如何以某种方式改造分销商，从而在不损失可扩展性和容错能力的情况下，显着减少资源负载。 对于具有密集型I / O的高负载服务的开发人员，将有很多有关分发，复制，容错存储和其他必要信息。


经典小丑


当然，Joker将与之相关联的所有内容：


  • 益智游戏（Tagir Valeev，Barukh Sadogursky）。

  • 主题演讲： 夏洛克的第二部分（叶夫根尼·鲍里索夫（Evgeny Borisov），巴鲁克·萨多古尔斯基（Barukh Sadogursky））；




Baruch，Eugene和Tagir是我们会议的定期发言人。


Baruch Sadogursky是JFrog的一名开发倡导者，实际上做了3件事：与Bintray和Artifactory开发人员在一起，为他们编写代码，并谈论在博客和会议上的印象，例如JavaOne，Devoxx，OSCON，当然还有JPoint和小丑，还有许多其他人。 如此连续十多年。





Evgeny Borisov是谈论春天的最著名的俄语演讲者。 他参加了大量的企业项目。 从一个简单的程序员变成了一个架构师，并且厌倦了日常工作，他成为了一个自由艺术家。 如今，尤金（Eugene）开办了自己的创业公司，为各种受众提供咨询，写作和举办课程，研讨会和大师班：以色列陆军军官的JEE现场课程，罗马尼亚人的Spring，加拿大人的Hibernate，白俄罗斯人的故障排除和设计模式，澳大利亚人等


Tagir Valeev是在JetBrains工作的著名发言人。 他从事IntelliJ IDEA静态代码分析器，检查和快速修复。 他还向OpenJDK投掷补丁程序，开发了开源库StreamEx和Java HuntBugs字节码分析器。 在哈布雷（ Labré ）被称为兰尼（lany） 。





我们看，听，交流！


我们不仅变得很多，而且变得很多。 近年来，参与者的数量已超过1000，这使Joker和JPoint（我们在莫斯科的第二次会议）成为俄罗斯Java世界的主要活动。



访问Joker意味着要加入俄罗斯最大的专业Java开发人员社区。


反过来，Joker会尽一切可能尽可能舒适有效地观看，收听，参与报告和讨论报告。 这么说很奇怪，但是许多顶级世界会议仍然没有配备讨论区-我们不仅拥有讨论区，而且考虑周全。




由于存在讨论区，因此很多人参加会议不是听报告，而是与现场演讲者见面-技术的创造者，发明家，知名专家和从业人员-亲自问他们的问题并获得明智的答案。



有人想在一个很酷的外国公司工作，这是他与未来的同事和经理会面并了解如何到达那里以及是否有必要的难得机会。 顺便说一句，如果雇主不付您会议费，也许这一次您应该花钱去上下一任小丑，这已经由雇主承担了吗？



有趣的是，在成千上万的人中，有这样的访客，他们本身并不比演讲者还酷。 他们也很高兴谈论他们的工作和特定的事情。 我们现在正在思考如何区分此类参与者-也许给他们一个特殊的徽章？



为了使参与者更容易在人群中找到志趣相投的人，我们正在创建自己的机器人。 您可以将自己的兴趣带入该机器人，它将找到喜欢您的人。 实际上，他已经参加过以前的会议，但是该会议将有一个完全由我们的开发人员编写的全新版本-更稳定，更完美。


最后，在晚上，有可能在所谓的BoF上进行交谈 。 这就像是一个利益讨论小组，其中没有划分为“发言人”和“听众”，每个人都可以发言并讨论该小组中的问题。



对于傍晚时光的恋人，“什么？ 在哪 什么时候？ 分成小组，棘手的问题，所有这些都是成年的。



我们看，听， 写 ！


在我们的俱乐部中，人们很聪明而又踏实（还可以吗？）。 对于我们来说，重要的是报告不仅听起来漂亮，而且在实践中也适用。


假设您下周开始工作，想应用您所看到的一切……而忘记了特定的代码。 报告中的信息太多了，最有趣的就是从我脑海中飞了出来。



幸运的是，Joker能够记住所有（或几乎所有）内容。 绝大多数报告都被记录并上传到YouTube。 几天后就会通过特殊链接向参与者提供参赛作品。 （其中某些记录有一天将公开提供，但这不会很快出现，也不会完整）



（您可以在此处观看过去的小丑的视频）


另外，可能根本不参加现场直播，而是观看高质量的在线广播。 居住在圣彼得堡的现场旅行不合理有很多原因。 有人只想放松一下-呆在家里躺在沙发上观看直播。 而且，如果您想在办公室中广播，请购买在线公司票（仅适用于公司），在办公室中放一个大型等离子显示器，并在整个Java团队面前集会。


玩得开心！


最后但并非最不重要的是，气氛。 那些曾经去过Joker的人不会对任何东西感到困惑-这种气氛是由参与者和演讲者自己创造的。


为了进行交流更加有趣，我们决定带来各种不寻常的事物。 将会有复古计算机，VR或AR等展览。 我们仍在指定确切的位置，然后我们将详细介绍有关哈布雷的所有内容。



在公司展位之间的报告之间，可以解决棘手的问题，参加抽奖活动，赢得宝贵的奖品以及收听小型报告。





对于那些没有参加会议的人，可以在线观看我们网站上的广播：在两次报告之间，我们将对发言人，主要参与者和赞助商进行在线采访。 实际上，这是排他性内容，主要供在线广播的观看者使用。



当然，将会有一场盛大的啤酒晚会（和非啤酒-不喝酒的人）。 聚会不仅是一种娱乐的方式，而且还是与志趣相投的人继续对话的一种方式。 有时人们会把事情留在晚上，他们甚至在BoF都不会谈论。



我们算！


在我们的俱乐部，习惯数数。 要到达Joker，您需要购买一张票，并且这里涉及一些数学。


从各种来源的评论（尤其是对哈布雷的评论）来看，很明显，如果一个人在公告发布后没有时间立即以大幅折扣购买机票，那么最后一天他只能在雇主的帮助下购买机票。 我们仔细地检查了数学并计算了自己购买门票的人的公平价格：从最昂贵的机票的3.2万下降到18，即大约1.7倍。


如果您是学生 ，则应该出示学生证，您将获得特别折扣-机票的价格是原来的一半！ 大学教师享有相同的折扣。 （糟糕的是，它不能与其他折扣结合使用。这就像我们的“学术计划”一样，是在社会急需强大专家的社会面前履行社会契约。）


是的，它仍然是不错的钱。 有人会说：我宁愿读一本书，甚至是StackOverflow的答案，所有这些都会便宜得多。 但是事实是，互联网上的会议和文本是完全不同顺序的现象，并且一个不能替代另一个。 反过来，现实世界中事件的组织是一件昂贵而复杂的事情。 会议在不断发展，场地在变化，越来越多的国际演讲者到来，等等。 而且我们仍然不乘坐劳斯莱斯。 所有这些增长和发展都付出了巨大的代价。


原则上，如果您不是学生，并且打算在会议宣布之后立即购买最便宜的机票，则只能在两天中的某一天购买，并且可以以优惠价参观上一个小丑（今天或明天在邮件中等待个人促销代码）-只有七千卢布。 这不是值得做的，但数学允许。


提示：如果您购买一日票，最好在第一天来，因为晚上会举行聚会和BoF。 尽管从原则上讲，我们没有限制，但可以转到第二个。
