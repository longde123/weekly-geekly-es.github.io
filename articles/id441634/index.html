Apakah ada kehidupan di bawah Windows 98, bagian dua - tentang perangkat lunak

Saya sedikit memperketat dengan sekuelnya. Sebagian karena pekerjaan, sebagian karena kemalasan, sebagian karena kenyataan bahwa tugas itu agak lebih banyak. Tetapi saya tidak akan berbicara tentang semua pencarian, saya hanya akan memberikan hasilnya.

Meskipun pertama-tama Anda perlu menentukan apa yang dimaksud dengan kehidupan :-)



Biasanya, komputer retro dirakit untuk game retro. Dengan mereka, sebagai suatu peraturan, tidak ada masalah khusus (well, kecuali relevan untuk era itu). Karena itu, anggap "hidup" ini tidak sepadan. Saya melihat kehidupan dari sudut pandang yang lebih sehari-hari - duduk di Internet, mengetik teks, mendengarkan musik, menonton film. Meskipun, tentu saja, yang terakhir tidak terlalu tergantung pada OS seperti pada perangkat keras yang digunakan. Toilet saya harus cukup untuk DVD dan divx, tetapi dengan x264 dan codec yang lebih baru, mungkin tidak dapat mengatasinya.

Tentang besi


Agar tidak kembali, izinkan saya mengingatkan Anda tentang setrika. Merakit komputer, relevan sekitar tahun 2001.
Celeron Tualatin 1200 @ 1600 (133 di bus), 512 megabytes memori (motherboard pada chipset 815 tidak lagi melihat).

Kartu video telah berubah sejak artikel terakhir dipublikasikan - bukannya Matrox saya menginstal GeForce2 MX200. Pilihannya, bahkan untuk tahun 2001, bukanlah yang terbaik, tetapi GeForce3 tidak dapat ditemukan, MX440 yang ada sudah sedikit kemudian. Dan saya melihat MX200 dipasangkan dengan pentiums4 pada konfigurasi 2004-2005, orang bahkan mencoba memainkannya di San Andreas.

Sistem operasi


Sebagai berikut dari topik, OS - Windows 98SE, Rusia.

Di alam, ada perakitan win98IF dengan sekelompok driver dan perangkat lunak, tetapi saya tidak menyukainya karena tumpukan itu. Terlalu banyak yang tidak perlu, bahkan Photoshop berhasil mendorong. Anda tidak bisa mendapatkan Windows sederhana bahkan dalam versi minimal. Oleh karena itu, saya memilih untuk menginstal paket pembaruan dari maximus pada OS yang bersih.

Ada juga di alam 300-megabyte auto-patcher dengan ekor yang menempatkan semuanya sendiri, tetapi lebih dipenjara untuk versi bahasa Inggris, dan saya tidak ingin mengerti apa yang terjebak dalam 300 megabyte ini.

Selanjutnya saya akan memberikan tautan ke perangkat lunak yang digunakan terutama ke mega.nz, tempat saya mengunduhnya sendiri - karena saya kurang lebih yakin akan keaktifan mereka.

Nah, secara umum, agar tidak melangkah jauh, biarkan semuanya berada di satu tempat.

Meskipun hal pertama menempatkan IE6sp1 .
Kemudian, Pembaruan Kumulatif Maximus Decim 3.05 (pembaruan, dukungan untuk disk besar, dukungan untuk penyimpanan usb, dll.).
Kemudian DirectX 9c .
Kemudian MD InternetExplorer 6.0sp1 Pembaruan Komponen 3.4 - Pembaruan untuk IE6
Pembaruan Komponen Akses Data MD - semua jenis komponen untuk mengakses data (XML, ODBC, dll.).

Kemudian KernelEx-4.5.2.exe - "emulator windows untuk windows", memungkinkan Anda untuk berjalan pada program Windows 9x yang dirancang untuk versi OS yang lebih baru, menambahkan pengaturan ke properti program seperti mode kompatibilitas dari garis NT. Bukan obat mujarab - beberapa program tidak mulai menggunakan kernelex juga. Tetapi hidup masih agak lebih mudah.



Selesai semuanya dengan menginstal .Net 1.1 dan 2.0 - hanya untuk memilikinya.

Yah, murni karena alasan estetika, ia menambahkan dukungan untuk kecantikan dengan ikon 32-bit ( RP9.7 ).

Internet

Tetapi hampir semua tautan dari hasil pencarian sudah rusak. Meskipun Anda dapat memesan salinan yang disimpan dan mencoba membaca sesuatu di sana.

Akibatnya, setelah banyak hiburan, saya sampai pada kesimpulan bahwa browser asli yang masuk akal terakhir untuk 98 adalah Opera 9.64, tetapi itu sudah tidak cukup. Akibatnya, menggunakan KernelEx, saya berhasil meluncurkan Opera 12.02 dan Firefox 10.0.12 ESR .

Firefox membuka sedikit lebih banyak situs, opera lebih mudah untuk sistem dan bekerja lebih stabil. Tetapi segala macam kerangka kerja modern tidak menyukai kedua browser.

Video


Di sini saya beristirahat cukup lama - di mesin virtual saya hanya bermain avishki secara normal, dan bahkan tidak di semua pemain. Di pemutar media, misalnya, dimungkinkan untuk memaksa, tetapi dalam vlc tidak ada suara.



Saya mencari-cari untuk waktu yang lama, kemudian memutuskan untuk menunda untuk meluncurkan pada perangkat keras nyata. Di sini, di vlc 2.0.0 yang sama baik video dan suara muncul. Jadi jelas ada beberapa masalah dengan mesin virtual. Kualitas gambar, bagaimanapun, adalah normal hanya di iklan, dan MKV dan MP4 lemah. MKV (h264 / mp3) terbagi menjadi kotak, dan MP4 (h264 / aac) kehilangan bingkai. Tapi ini adalah pertanyaan kedua, saya mencapai reproduksi, dan Anda dapat memilih codec dan pemain untuk waktu yang lama. Setidaknya ada solusinya, tetapi fine tuning harus dilakukan pada perangkat keras tertentu.

Pertukaran data


Menimbang bahwa sekarang sudah ada banyak masalah bahkan dengan windows xp (selusin terus-menerus berusaha untuk menonaktifkan dukungan untuk smb1, dll.), Untuk bertukar file dengan 98 Windows, saya baru saja mengangkat ftp server di server rumah saya, karena protokol belum digunakan sejak telah berubah. Dan folder yang dibagikan sangat tidak stabil untuk saya - kadang-kadang ternyata menjangkau, dan setelah beberapa menit "server tidak ditemukan", ia mulai meminta kata sandi atau sesuatu.

Jadi lebih mudah untuk meningkatkan ftp dan menyalin file melalui FAR Manager yang sama.



Dengan mengorbankan pencetakan ke printer jaringan. Di tempat saya ada printer jaringan klasik, dengan koneksi melalui Ethernet. Oleh karena itu, semuanya agak mudah dengan itu daripada dengan printer yang dipakai bersama pada Windows. Yah, dia adalah Hewlett Packard (HP Color LaserJet CP2025dn), yang menyederhanakan hidup bahkan lebih dalam hal driver. Dulu ada Samsung, tapi saya menyingkirkannya. Byak. Driver untuk 98 saya menginstal Windows HP Color LaserJet (MS) standar. Saya tidak perlu cantik, saya hanya perlu mencetak.

Dan pencetakan itu sendiri dilakukan dengan menggunakan utilitas ACITS LPR Client . Sudah diatur dan dikonfigurasi tanpa masalah. Instal utilitas, lalu atur printer sebagai lokal dengan menginstal driver yang diperlukan. Dan kemudian ubah port ke ACITS LPR Remote Printing, tentukan IP printer - dan hanya itu. Instruksi dengan gambar disertakan di sana. Baik, atau Anda bisa menggunakan yang ini .
