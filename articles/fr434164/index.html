Migration d'IBM Notes / Domino vers Zimbra Collaboration Suite

Dans les commentaires du dernier article, la question de la possibilité de migration d'IBM Notes / Domino vers Zimbra Collaboration Suite a été soulevée. En effet, contrairement à la transition de MS Exchange à Zimbra Collaboration Suite, dont le processus est décrit en détail dans l'un des articles précédents, la migration depuis IBM Notes / Domino est beaucoup plus difficile. Cependant, comme nous le savons, il n'y a pas de situations désespérées, et maintenant nous allons parler de quoi et comment transférer du serveur IBM Notes / Domino vers le serveur Zimbra Collaboration Suite.



Tout d'abord, il convient de noter qu'en raison de la grande complexité du processus, la migration est mieux confiée à une personne possédant une vaste expérience dans IBM Notes / Domino afin qu'il puisse résoudre rapidement les problèmes qui surviennent pendant le processus de transfert de données. Les solutions possibles aux problèmes émergents peuvent être trouvées sur le forum officiel de Zimbra, ainsi que dans le wiki de Zimbra.

Pour la migration directe, nous utiliserons l' assistant de migration de Zimbra Collaboration Server pour Lotus Domino . Cette application pour le système d'exploitation MS Windows aidera à transférer les comptes d'utilisateurs nécessaires vers IBM Notes / Domino, le contenu de leurs boîtes aux lettres, ainsi que les données de calendrier et la liste de contacts du carnet d'adresses vers le serveur avec Zimbra. Pour que cela fonctionne, vous devrez saisir les informations du compte administrateur IBM Notes / Domino afin d'obtenir les données utilisateur à importer. Notez que l'assistant de migration ZCS pour Lotus Domino crée indépendamment des comptes pour les comptes transférés depuis le serveur IBM Notes / Domino s'ils n'ont pas été créés auparavant. Nous allons démarrer l'assistant de migration ZCS pour Lotus Domino sur l'ordinateur sur lequel IBM Notes / Domino est installé. Pour qu'il fonctionne correctement, il aura besoin des dernières versions des fichiers .dll fournis avec IBM Notes / Domino.

Avant la migration, il est très important d'effectuer un travail d'information avec les employés. Il est nécessaire d'aviser chaque employé dont le compte sera transféré sur le nouveau serveur qu'il utilisera bientôt la nouvelle application de messagerie avec une nouvelle interface. En outre, il est nécessaire de fournir à chacun des informations sur le nom de domaine sur lequel Zimbra Collaboration Suite fonctionnera, ainsi qu'une paire nom d'utilisateur / mot de passe, qui sera utilisée pour accéder aux outils de messagerie et de collaboration. La migration elle-même est mieux planifiée pour le week-end ou les vacances, lorsque la charge sur le courrier électronique sera minime.

Nous vous recommandons également de migrer les boîtes aux lettres en petites portions afin que le processus ne prenne pas beaucoup de temps et que vous ayez la possibilité de modifier rapidement les enregistrements des boîtes aux lettres migrées, afin que de nouvelles lettres arrivent déjà sur le serveur Zimbra et ne restent pas sur IBM Notes / Domino.

Après avoir démarré l'assistant de migration ZCS pour Lotus Domino, vous serez invité à saisir certaines informations dans plusieurs boîtes de dialogue. Dans la première fenêtre intitulée Destination d'importation, vous devez indiquer:

  • Nom d'hôte - nom de domaine du serveur avec Zimbra
  • Admin Port - port d'accès aux services d'administration de Zimbra (7071 par défaut)
  • Utiliser une connexion sécurisée - utilisée pour établir une connexion sécurisée avec le serveur
  • Nom d'utilisateur et mot de passe - données du compte administrateur sur le serveur Zimbra.

Dans la fenêtre intitulée Domaine de destination, vous devez saisir le domaine sur le serveur avec Zimbra, vers lequel les comptes d'IBM Notes / Domino seront transférés.

Dans la fenêtre Informations sur le serveur LDAP , nous devons saisir les informations nécessaires pour se connecter au serveur LDAP:

  • Serveur LDAP - adresse du serveur LDAP interne ou externe
  • Port LDAP - Port d' accès LDAP (389 par défaut)
  • DN utilisateur et mot de passe - Entrez le nom distinctif et le mot de passe utilisateur pour accéder à LDAP
  • BasDN - Entrez le nom distinctif de base pour rechercher

Les informations de la fenêtre Importer la source sont généralement renseignées indépendamment si l'assistant de migration ZCS pour Lotus Domino peut détecter le serveur IBM Notes / Domino. Si cela ne se produit pas, vous devrez saisir:

  • Nom du serveur Domino - nom de domaine du serveur avec IBM Notes / Domino
  • Admin ID File Path - chemin d'accès au fichier ID avec les données de l'administrateur
  • Chemin du fichier INI Notes - chemin vers le fichier INI Notes
  • Migrer les dossiers privés - Cochez cette case pour transférer les dossiers privés des utilisateurs vers le serveur Zimbra. Pour ce faire, vous devez indiquer au programme le chemin d'accès aux fichiers d'identification des utilisateurs dont vous prévoyez de transférer les comptes, ainsi que le chemin d'accès aux fichiers avec leurs mots de passe

Après cela, dans la fenêtre Boîtes aux lettres source et destination, nous devons sélectionner les boîtes aux lettres qui seront transférées au serveur depuis Zimbra. Cette fenêtre a plusieurs boutons qui ouvrent différentes fenêtres. Ainsi, par exemple, dans la fenêtre du sélecteur d'objets , vous pouvez rechercher et sélectionner les comptes à migrer vers un nouveau serveur. Pour ce faire, vous devez commencer à saisir le nom de la boîte aux lettres et la fonction de saisie semi-automatique vous proposera des options appropriées. Dans la fenêtre du générateur de requêtes , vous devez créer une requête pour effectuer une recherche par utilisateur dans la base de données LDAP.

Après cela, les comptes précédemment sélectionnés apparaîtront dans la fenêtre des boîtes aux lettres source et destination précédemment vide, qui sera transférée sur le serveur Zimbra. Si vous changez d'avis sur le déplacement d'une boîte aux lettres, vous pouvez la supprimer de la liste. Si la liste est compilée correctement, vous pouvez continuer.

À l'étape de vérification du compte cible , l'assistant de migration ZCS pour Lotus Domino vérifie s'il existe des comptes sur le serveur Zimbra avec des noms de compte identiques à ceux migrant depuis le serveur IBM Notes / Domino. S'il n'y a pas de compte avec un nom identique, il sera créé.

Dans la boîte de dialogue Provisioning du compte , l'administrateur devra configurer les comptes créés après la migration sur le serveur Zimbra. En particulier, vous devez entrer un mot de passe qui sera utilisé par ces utilisateurs par défaut, et également spécifier les groupes d'utilisateurs auxquels appartiennent les comptes migrés.

Une fois tous les comptes créés, vous pouvez commencer à importer le contenu des comptes. Pour ce faire, dans la boîte de dialogue Importer des éléments , sélectionnez les composants à transférer:

  • Importer des courriers - transférer la correspondance par courrier
  • Importer des contacts - Transférer des contacts
  • Importer des tâches - transfert de tâches
  • Importer un calendrier - transfert de données de calendrier

Dans la fenêtre suivante, intitulée Options d'importation, il vous sera demandé de sélectionner plusieurs options:

  • Importer le dossier Courrier indésirable - s'il faut transférer le dossier Spam
  • Importer le dossier Éléments supprimés - Faut-il déplacer la "Corbeille" avec les éléments supprimés
  • Importer les e-mails reçus après - Indiquez la date à partir de laquelle les messages doivent être transférés
  • Nombre de boîtes aux lettres à importer simultanément - Spécifiez le nombre de boîtes aux lettres importées simultanément

Une fois la migration terminée, vous serez invité à en effectuer une autre. Si vous avez profité de nos conseils et divisé la migration en plusieurs parties, vous pouvez prendre une petite pause pour modifier les entrées dans le relais de messagerie afin que les lettres adressées aux utilisateurs nouvellement migrés arrivent sur le serveur Zimbra.

Notez que si vous êtes confronté à la tâche de transférer uniquement le contenu des boîtes aux lettres d'IBM Notes / Domino vers Zimbra, vous pouvez utiliser la migration à l'aide de l'utilitaire imapsync . Le processus d'une telle migration est décrit dans un article précédent .
