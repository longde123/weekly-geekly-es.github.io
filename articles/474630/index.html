Semana de la seguridad 45: vulnerabilidades de Chrome y BlueKeep Wildlife

Google lanzó una actualización del navegador Chrome el 31 de octubre, en la que se cerraron dos vulnerabilidades graves. Uno de ellos (CVE-2019-13720) fue utilizado en ataques reales y fue descubierto ( noticias , investigación ) por expertos de Kaspersky Lab. Una vulnerabilidad de uso posterior libre (CVE-2019-13720) permite que se ejecute código arbitrario en sistemas con Windows de 64 bits y las versiones de navegador Chrome 76 y 77.


El problema se detectó mediante el sistema de prevención automática de exploits, que forma parte de las soluciones de seguridad de Kaspersky Lab y tiene como objetivo identificar ataques previamente desconocidos. El código Javascript malicioso que lanza el ataque ha sido inyectado en el sitio web de noticias coreano. Los investigadores han llamado a esta campaña Operation WizardOpium, y hasta ahora no hay signos de combinar esta actividad maliciosa con otras operaciones cibercriminales. El código de explotación sugiere una operación más grande que explota otras vulnerabilidades en software común.

Este no es el único ataque "en vivo" que se descubrió la semana pasada. El 3 de noviembre, se publicó una descripción interesante de un ataque a computadoras con la vulnerabilidad BlueKeep en el blog Kryptos Logic. Este problema en la función Servicios de escritorio remoto en Windows 7 y Windows 2008 Server se descubrió en mayo. A pesar de que el parche se lanzó no solo para estos sistemas operativos (relativamente) modernos, sino también para Windows XP y 2003 Server no compatibles, en el momento en que se lanzó el parche, había alrededor de un millón de sistemas vulnerables. En septiembre, se publicó un exploit para BlueKeep como parte del paquete Metasploit. Incluso en un sistema sin parches, puede cambiar la configuración para hacer imposible la explotación de la vulnerabilidad. Sin embargo, los investigadores procedieron de la alta probabilidad de que muchos sistemas no se actualicen y configuren correctamente. ¿Cómo determinar cuándo comenzarán a atacar de verdad?


Con la ayuda de hanipots, sistemas deliberadamente mal configurados en los que la vulnerabilidad permite que se ejecute código arbitrario de forma remota y sin autorización. El 2 de noviembre, el investigador Kevin Bumont anunció (su versión de los eventos está aquí ) que los chanipots que le pertenecen comenzaron a caer espontáneamente "en la pantalla azul". Un análisis de los bloqueos mostró que los ataques a los Servicios de escritorio remoto se están llevando a cabo, y su naturaleza coincide con las capacidades del código publicado como parte de Metasploit. Después de una penetración exitosa desde el servidor del atacante, los comandos de PowerShell se descargan y ejecutan secuencialmente, hasta que finalmente se descarga la carga útil, el cryptominer. Naturalmente, después de la publicación del exploit, tales "bromas" eran inevitables, pero en este caso también tenemos un ejemplo interesante de análisis de ataque, que comienza como el dolor de cabeza habitual del administrador: el sistema se bloquea y nadie sabe por qué. (Durante mucho tiempo) es hora de actualizar, pero el número de sistemas con la vulnerabilidad BlueKeep no ha disminuido mucho en cinco meses y ahora se estima en más de 700 mil.
