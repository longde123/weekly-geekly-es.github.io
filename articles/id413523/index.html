Ikhtisar bekerja dengan Tautan Universal: pro dan kontra

Di artikel ini, atas nama Getloyal , sistem penargetan ulang otomatis untuk aplikasi seluler, kami akan menunjukkan cara kerja Universal Link dalam kampanye pemasaran ulang, dan apa saja pendekatan untuk menyiapkan iOS dan Android. Dan juga pertimbangkan pro dan kontra utama dari teknologi ini.


Dalam kampanye pemasaran ulang, konversi untuk membeli menggunakan Deeplinks beberapa kali lebih tinggi daripada menggunakan tautan konvensional. Deeplink tidak hanya dapat membuka aplikasi, tetapi juga layar tertentu di dalam aplikasi. Agar tautan mengarah ke layar yang diinginkan di dalam aplikasi, Anda perlu mengonfigurasi skema URL. Implementasi skema URL (struktur diplinks) merupakan bagian integral dari proses pengembangan ponsel.

Hanya sedikit orang yang tahu bahwa pada konferensi WWDC15, Apple berhenti mendukung skema URL untuk tautan langsung dan memperkenalkan cara baru untuk menerapkannya pada platform seluler - Seamless Linking. Tautan semacam itu memungkinkan pengguna untuk langsung masuk ke aplikasi tanpa membuka Safari.

Dimulai dengan iOS 9, Apple memperkenalkan Universal Links. Perbedaan utama antara Universal Links dan skema URL kustom yang biasa adalah bahwa sebelum tautan menuju ke halaman Instagram Yegor Creed tampak seperti ini: instagram: // user? Username = egorkreed, dan dengan munculnya Universal Links, tampilannya seperti ini: www.instagram.com / egorkreed .

Anda masih dapat menemukan panduan untuk mengimplementasikan skema URL, tetapi tidak di situs web Apple. Bagaimana dengan Android? Google juga untuk privasi dan UX, jadi sejak Android 6.0, perangkat mendukung Android App Links. Ini hampir sama dengan Tautan Universal di iOS.

Banyak pengembang telah beralih untuk menggunakan Tautan Universal - ini menyederhanakan konfigurasi tautan dan meningkatkan pengalaman pengguna.

Bagaimana cara kerjanya?


Ketika Anda mengklik Tautan Universal, sistem operasi secara otomatis menentukan apakah aplikasi diinstal pada perangkat. Jika demikian, pengguna membuka aplikasi. Jika tidak, pengguna diarahkan ke situs, ini mungkin merupakan pengalihan standar ke AppStore.

Dengan mengunduh aplikasi yang mengimplementasikan teknologi ini, pengguna juga mengunduh file yang berisi aturan perutean tempat mengirim pengguna dalam satu kasus atau yang lain.


Contohnya


Ada perusahaan yang telah menerapkan Tautan Universal dalam aplikasi mereka, dan melakukan hal yang benar karena semua orang mendapat manfaat dari ini:

  1. Pengguna: berbagi konten menjadi lebih nyaman. Level UX telah meningkat secara signifikan.
  2. Mitra: mitra promosi juga berada di posisi pemenang: CRE (konversi menjadi keterlibatan kembali) telah meningkat, Anda dapat meningkatkan skala, pengiklan akan senang.
  3. Produk: konversi untuk membeli meningkat, penargetan ulang bekerja lebih efisien, biaya untuk menarik pengguna yang unik berkurang.

Implementasi


Menerapkan Tautan Universal tidak sulit. Tetapi pendekatannya sedikit berbeda tergantung pada platform.

iOS:

  1. Mengaitkan aplikasi dengan domain (domain dapat berupa pelacak Anda sendiri atau iklan) dalam pengaturan aplikasi melalui Xcode;
  2. Buat file asosiasi - halaman yang berisi data dalam format JSON, yaitu konten yang tersedia dalam aplikasi;
  3. Di file appdelegate, Anda perlu mengonfigurasi aturan untuk memproses klik pada Universal Link. Jika aplikasi diinstal, halaman yang diminta akan terbuka, jika tidak diinstal atau tidak dapat menyediakan konten yang diperlukan, maka halaman tersebut akan terbuka di Safari.

Android:

  1. Menerapkan skema diplink yang mengarah ke halaman tertentu dalam aplikasi;
  2. Tambahkan aturan ke aplikasi untuk meminta Tautan Aplikasi untuk membuka aplikasi;
  3. Publikasikan file asosiasi dalam format JSON di domain Anda (bisa berupa domain aplikasi Anda sendiri atau domain pelacak iklan).

Kontra Tautan Universal:


  • Kompleksitas implementasi. Terlepas dari kenyataan bahwa Tautan Universal dikonfigurasikan dalam beberapa klik, ada nuansa dalam proses pengaturan di mana Anda dapat membuat kesalahan.
  • Masalah pelacakan. Terkadang ada masalah dengan tampilan reattributions. Kesalahan seperti itu sulit ditangkap dan pelepasan aplikasi dengan Universal Link ditunda.
  • Kompleksitas berbagi. Jika Anda membagikan Universal Link di iOS melalui messenger atau menempelkan tautan ke bidang URL Safari, situs akan dibuka di Safari, tetapi jika Anda menambahkan tautan ke Facebook (Aplikasi), Twitter (Aplikasi), Mail (Aplikasi) atau ke spanduk di situs, aplikasi akan terbuka .

Manfaat Tautan Universal:


Kami di tim Getloyal memperhatikan bahwa sebagian besar perusahaan seluler telah menerapkan skema tautan URL dalam versi Android aplikasi, pada aplikasi iOS myapp: // tautan juga berfungsi. Banyak yang mungkin bertanya-tanya: "Mengapa kita membutuhkan Tautan Universal, jika semuanya sudah berfungsi?" Untuk menjawab pertanyaan ini, kami telah memberikan daftar manfaat di bawah ini:

  • Keunikan. Tidak seperti skema URL, Tautan Universal tidak dapat ditempati oleh aplikasi lain, karena mereka menggunakan protokol HTTP atau HTTPS standar;
  • Fleksibilitas. Tautan Universal berfungsi bahkan jika aplikasi tidak diinstal. Jika aplikasi tidak diinstal, pengguna membuka konten yang diinginkan di Safari;
  • UX yang luar biasa. Satu tautan: baik di situs dan di aplikasi;
  • Dukungan untuk platform iklan. Banyak situs, misalnya, MyTarget, tidak lagi memungkinkan Anda untuk memasukkan diplink tradisional ke dalam tautan;
  • Kemudahan berbagi , nyaman bagi pengguna untuk membagikan tautan yang dimulai dengan ..., karena tautan seperti itu terlihat lebih akrab.
