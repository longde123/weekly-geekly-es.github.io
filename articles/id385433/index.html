Mozilla akan merilis Firefox 64-bit untuk Windows pada 3 November

Dan pada akhir 2016, dukungan NPAPI akan dihapus dari Firefox


Firefox 42, versi browser Mozilla berikutnya, akan tersedia baik sebagai aplikasi 32-bit dan sebagai program 64-bit. Versi terbaru, hanya tersedia di server FTP, akan memanfaatkan arsitektur x86-64 (AMD64 / Intel 64). Sebagai contoh, itu akan dapat mengatasi lebih dari 4 GB memori. Saat ini, di bawah Windows hanya ada Firefox 32-bit. Untuk pengguna Linux dan Mac OS, Firefox 64-bit telah tersedia untuk waktu yang lama. Firefox 42 akan keluar 3 November tahun ini.

Selama lima tahun terakhir, Mozilla memulai proses pengembangan Firefox 64-bit, lalu menjatuhkannya , lalu mulai lagi . Sejak 2014, setelah memberikan prioritas pada proses pengembangan versi 64-bit Firefox untuk Windows, hasilnya telah ditunda beberapa kali. Seperti biasa, berita pertama adalah yang terbaik untuk menarik dari pelacak bug , karena proses pengembangan terbuka.

Pada tanggal 3 November, versi berikutnya dari Firefox 42 akan dirilis, tetapi tidak ada opsi yang ditawarkan untuk menginstal versi 64-bit. Selain itu, tidak akan ada tautan bahkan di halaman unduhan. Majelis akan menjadiDapatkan secara eksklusif di server FTP sebagai ujian. Seorang juru bicara Mozilla mengatakan semacam "perubahan dari mitra" diharapkan, sehingga tidak akan ada pengumuman lengkap. Versi 64-bit tidak akan memiliki dukungan untuk plugin menggunakan NPAPI.

64-bit di sini mengacu pada sistem dengan arsitektur prosesor pusat x86-64di mana OS 64-bit diinstal. x86-64 adalah ekstensi 64-bit dari 32-bit x86 dengan kompatibilitas ke belakang. Prosesor dari hampir semua komputer pribadi modern menggunakan arsitektur x86-64. OS 32-bit tidak memungkinkan menangani lebih dari 4 GB RAM. Selain itu, angka ini termasuk RAM itu sendiri dan memori akselerator video. Sistem 64-bit memiliki batasan lebih tinggi, misalnya, Windows 10 dapat menggunakan hingga 2 TiB RAM. Sistem operasi 64-bit memungkinkan Anda menjalankan aplikasi 32-bit. OS 32-bit tidak dapat menjalankan program 64-bit. Kelebihan kode 64-bit tidak berakhir dengan jumlah RAM yang bisa dialamatkan: program bisa lebih cepat.

Tidak seperti Google Chrome, yang menggunakan arsitektur multi-threaded, Firefox menggunakan proses tunggal, meskipun pengerjaan arsitektur multithreaded berlangsung . Sekarang versi rilis Firefox untuk Windows adalah 32-bit dan terlampir dalam satu proses, yang tidak dapat mengatasi lebih dari 3 GiB dalam sistem 32-bit dan 4 GiB dalam 64-bit. Ada 64-bit build malam untuk Windows. Pada 3 November, untuk pertama kalinya, versi rilis Firefox 64-bit akan dirilis untuk Windows.

Saingan Firefox telah lama membangun 64-bit di gudang senjata mereka. Kerakusan Chrome, yang memungkinkannya melampaui 32 bit ruang alamat dan dengan mudah menggunakan semua sumber daya komputer, dikenal oleh pengguna dan telah menjadi bahan lelucon, meskipun kemudian dioptimalkan oleh Googlebekerja dengan memori. Bahkan proyek yang didasarkan pada kode sumber Firefox gratis menyalip leluhur dalam hal ini: Pale Moon dan Waterfox telah lama tersedia dalam versi 64-bit untuk Windows.

Versi Win64 dari Firefox 42, yang akan dirilis pada 3 November, akan kekurangan dukungan untuk plugin NPAPI. Secara umum, Mozilla akan dihapusdukungan untuk NPAPI dari Firefox pada akhir tahun depan. Ini berarti bahwa plugin populer seperti Java atau Unity web player tidak akan berfungsi. Keputusan untuk menghapus dukungan NPAPI ditentukan oleh fakta bahwa fungsi plug-in sudah didukung secara native oleh browser itu sendiri: di Firefox Anda dapat menonton video berkat HTML5 dan keberadaan codec yang diperlukan dan bermain game 3D menggunakan WebGL. Plugin NPAPI semakin menunjukkan usia teknologi dalam bentuk masalah kinerja (yang mempengaruhi masa pakai baterai pada perangkat seluler), crash, dan insiden keamanan informasi.

Dukungan untuk plugin NPAPI telah dihapus dari Chrome. Untuk Flash, pengembang di Mozilla akan membuat pengecualian karena telah menjadi bagian penting dari kehidupan sehari-hari pengguna Web. Mungkin, pada periode perkembangan tertentu, ia juga akan dipindahkan. Oracle memahami penolakan Java dan menawarkan solusi sendiri. Unity ditinggalkan bekerja di Unity Web Player. Sebagai gantinya, diusulkan untuk menggunakan ekspor ke WebGL. Dalam versi Unity 5.4 yang keluar pada bulan Maret tahun depan, dukungan plug-in browser akan hilang.

Setelah rilis (3 November), Firefox 42 64-bit build untuk Windows akan tersedia di https://ftp.mozilla.org/pub/mozilla.org/firefox/releases/latest/. Siklus pengembangan Firefox terdiri dari mengubah secara berurutan status bangunan malam menjadi versi untuk pengembang, versi beta, dan kemudian rilis. Versi beta dari browser 64-bit sudah tersedia .
