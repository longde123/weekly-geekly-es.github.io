Cowon D2：音乐长寿

最近一次有关智能手机市场新产品的对话为我写这篇文章带来了启发。 一位熟人谈到了他是如何获得下一位的，她是多么的出色，不像其他人那样。 使用新的无线耳机聆听音乐有多方便，声音听起来多么美妙。 他看上去，听着，比较过，没有留下深刻的印象。

我本人并不经常更换小工具。 在过去的十年中，我一直在使用它们中的一个，而且我不急于让他休息。



实际上，我们将进一步讨论逐渐消失的多媒体播放器的代表。 关于他如何与我保持了这么长时间。

该设备于2007年购回。 一个2 GB的模型，即使到那时，其内部存储器的数量还是很可笑的。 但是播放器配备了SD卡连接器，这极大地改变了问题。

播放器体积小巧，带有2.5英寸彩色屏幕和电阻式传感器。除了耳机和平板电脑（仅适合测试性能）之外，还有一个笨拙的手写笔，大多数用户会很快将其扔到远处的抽屉中。



这个玩家与当时产生的成千上万的其他玩家有什么区别？

  • 很大的自治权。 制造商声称可以播放52小时。 实际上，整个后壁都被电池占据了。
  • 支持多种音乐格式：mp3，ogg，flac，ape，wma。
  • 高质量的声音和“摇滚”录音室耳机的能力。
  • 无需诸如iTunes之类的支持软件。

总的来说，对我来说，第一次就足够了。 但仅适用于第一个。

哦，是的，D2是最早配备触摸屏的玩家之一。

当更多容量的存储卡开始出现-32Gb或更多时，我开始注意到，当存储空间已满时，播放器开始表现异常：冻结并关闭。 在风扇论坛上有很多讨论，对制造商的功能要求，但是...

但是制造商并不急于处理发现的错误并满足社区的要求。 毕竟，这将不会带来有形的利润。 “您的设备已过时，请购买我们的新型号”模型看起来更加有利可图。

但是，尽管制造商不屑一顾，但该设备还是陷入了热恋，并获得了相当广泛的粉丝群。



社区的力量创造了许多设计主题，更新了固件，为某些耳机创建了均衡器预设，开放的Rockbox固件的狭窄领域广为人知的端口成为了所有这些活动的典范。 这给了玩家一些非常缺乏的东西：

长时间没有暂停就可以播放，这是官方固件开发人员长期以来所要求的。 可配置的淡入淡出（平滑淡入淡出和恢复），支持ReplayGain（对专辑或整个加载的库的音量级别进行标准化），甚至还支持last.fm加密（一旦相关）。 尽管推出Doom是出于兴趣，但许多插件甚至游戏对我都不感兴趣。 还有很多不同的接口。 我喜欢最简单的文字文字。

Rockbox安装


如上所述，这是多媒体播放器的免费固件。 官方网站： https : //www.rockbox.org/ 。

通过修改引导加载程序将其放置在标准驱动器旁边。

首先，我们从github中抽取源代码（我从我心爱的Debian稳定器中完成所有操作）： 

$git clone 'https://github.com/Rockbox/rockbox'

我们放置了构建工具链所需的依赖项： 

 #apt install build-essential texinfo libtool-bin flex bison

放置工具链： 

 #tools/rockboxdev.sh Download directory : /tmp/rbdev-dl (set RBDEV_DOWNLOAD or use --download= to change) Install prefix : /usr/local (set RBDEV_PREFIX or use --prefix= to change) Build dir : /tmp/rbdev-build (set RBDEV_BUILD or use --builddir= to change) Make options : (set MAKEFLAGS or use --makeflags= to change) Restart step : (set RBDEV_RESTART or use --restart= to change) Target arch : (set RBDEV_TARGET or use --target to change) Select target arch: s - sh (Archos models) m - m68k (iriver h1x0/h3x0, iaudio m3/m5/x5 and mpio hd200) a - arm (ipods, iriver H10, Sansa, D2, Gigabeat, etc) i - mips (Jz47xx and ATJ-based players) r - arm-app (Samsung ypr0) x - arm-linux (Generic Linux ARM: Samsung ypr0, Linux-based Sony NWZ) y - mips-linux (Generic Linux MIPS: AGPTek Rocker) separate multiple targets with spaces (Example: "sma" will build sh, m68k and arm)

然后选择选项“ a”。 接下来，将进行泵送和组装工具链的过程。 除非指定了不同的前缀，否则这些都将位于/ usr / local中。 但是，您可以将其移动到/ opt内的单独目录，使用checkinstall进行练习，甚至编译完整的软件包。

一段时间后，工具链准备就绪。 放置修补程序： 

 $cd rbutil/mktccboot; make && cd -

在输出中，我们进入rockbox / rbutil / mktccboot / mktccboot目录，通过该目录我们将组装引导程序。 

 $mkdir build; cd build; ../toold/configure && make

在列表中，选择项目“ 33”，然后选择项目“ B”（引导程序）。 正在进行构建过程。 输出是文件“ bootloader-cowond2.bin”

打开官方固件包装： 

 $unzip D2_2.59.zip

和补丁： 

 $../rbutil/mktccboot/mktccboot D2_2.59/1/d2N.bin bootloader-cowond2.bin d2n.bin OF entry point: 0x2000006c New entry point: 0x203204fc

瞧，我们得到了d2n.bin文件，我们将其复制到播放器的内存中。 我们断开播放器与USB的连接，将其打开，一个简短的固件过程即可完成。

然后，我们可以使用相同的控件收集摇滚箱本身（选择“ N”）： 

 $tools/configure && make && make zip

或从站点下载完成的一个： 

 wget 'https://build.rockbox.org/data/rockbox-cowond2.zip'

收集或下载的固件目录被扔到存储卡上（以前是未包装的）。 就我而言（我更喜欢使用读卡器）： 

 cp -r .rockbox /media/sdb1-usb-Mass_Storage_Dev

我们将存储卡放入播放器中，并获得免费且功能齐全的固件。
