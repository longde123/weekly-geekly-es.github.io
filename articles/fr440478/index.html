Sortie de 3CX v16 Beta 1 avec prise en charge de Raspberry Pi

Nous sommes heureux de vous présenter la prochaine version de notre 3CX v16 Beta 1. La version finale approche à grands pas! se rapprocher encore plus de la version finale. Permettez-moi maintenant de vous présenter les nouvelles fonctionnalités, améliorations et prise en charge de la nouvelle plate-forme qui sont apparues dans la version bêta 1.

Travailler sur la plateforme Raspberry Pi



Pas si souvent, un produit logiciel complexe est adapté pour prendre en charge la nouvelle architecture matérielle. Oui, ce n'était pas facile à mettre en œuvre pour un système en temps réel, qui est 3CX.

Mais nous l'avons fait! 3CX v16 fonctionne désormais sur des appareils de la taille d'un savon comme le Raspberry Pi 3B +. Nous avons préparé le guide d'installation de la v16 sur le Raspberry Pi, qui est donné ci-dessous. Installez et utilisez gratuitement !

Enregistrement de conversations sur un stockage externe


3CX v16 Beta 1 comprend un nouveau système de gestion d'enregistrement de conversation qui prend en charge le transfert de fichiers d'enregistrement vers un stockage externe, la recherche, la sélection et la suppression automatique d'anciens enregistrements. Maintenant, vous pouvez automatiquement, selon le calendrier, transférer les enregistrements de conversation vers Google Drive ou un serveur FTP en activant l'option d'archivage à distance. Ainsi, vous libérez un espace précieux sur le disque (ou la carte Flash) du serveur PBX. De plus, cette solution accélère la mise à niveau ou la réinstallation de 3CX.



La section Enregistrement de conversation a également une nouvelle fonction de purge . Il supprime les enregistrements locaux ou externes. Il vous permet également de transférer des enregistrements locaux vers un stockage externe (archive). Vous pouvez désormais réduire le coût du PBX matériel en déplaçant l'archive de conversation vers un stockage cloud externe peu coûteux. La fonction Filtre vous permet de sélectionner des enregistrements de conversations pour la période souhaitée ou vers des numéros de téléphone spécifiques. En savoir plus sur ces fonctionnalités dans notre guide .

Panneau opérateur ATS dans une fenêtre séparée




Le client Web de 3CX v16 Beta 1 a reçu une mise à jour importante - les onglets du panneau de commande PBX peuvent maintenant être ouverts dans des fenêtres distinctes. Les gestionnaires et les opérateurs de centre d'appels peuvent utiliser des fenêtres distinctes pour surveiller les appels de la même manière que dans le client 3CX pour Windows. En général, toutes les fonctions client pour Windows liées à la surveillance des appels ont déjà été transférées au client Web - tout fonctionne dans le navigateur!

Screencast dans 3CX WebMeeting sans plugins


Lors de l'introduction de tout nouveau système, une question importante se pose - la complexité du lancement initial. En essayant de simplifier le début de l'utilisation de la vidéoconférence, nous avons utilisé la dernière technologie du navigateur Chrome 72 et finalisé WebMeeting pour refuser d'installer un plug-in supplémentaire.



Les utilisateurs peuvent désormais afficher leur écran avec ou sans le plug-in 3CX Click2Call (anciennes versions de Chrome).

Sélectionnez simplement le mode d'affichage - plein écran ou fenêtre d'application - et votre bureau sera affiché dans la conférence! Pour utiliser cette fonctionnalité, installez la dernière version de Chrome .

Installation d'un PBX 3CX sur un Raspberry Pi


À partir de 3CX v16, vous pouvez installer des PBX sur les derniers appareils Raspberry Pi, modèle 3B +. Le système est installé sur le Raspbian Stretch OS, la distribution Debian Linux pour l'architecture Raspberry Pi ARM. Vous pouvez contrôler le système d'exploitation Debian soit via l'interface Web 3CX (commandes d'administration de base), soit via l'interface de ligne de commande si vous êtes un utilisateur Linux expérimenté.

Veuillez noter que l'installation de 3CX sur le Raspberry Pi est recommandée pour les systèmes avec 8 appels simultanés avec 16-24 utilisateurs. Chaque utilisateur peut configurer jusqu'à 8 indicateurs BLF en moyenne (la transmission de l'état BLF met une charge sur le serveur PBX).

Exigences supplémentaires:

  • Carte mémoire: 32 + Go Class 10 Micro SDHC
  • Source d'alimentation: compatible avec Raspberry Pi, 2,5 A, Micro USB

Préparation d'une carte SD pour Raspbian OS


Pour installer Raspbian sur une carte micro-SD, téléchargez et décompressez l'image Raspbian Stretch Lite à partir d'ici .



Gravez l'image sur une carte SD à l'aide des utilitaires Etcher ou Win32 Disk Imager . Insérez ensuite la carte et démarrez l'appareil.

Après avoir chargé le système d'exploitation, connectez-vous avec les informations d'identification d'administrateur par défaut (utilisateur pi , mot de passe raspberry ). Le mot de passe par défaut pi peut être modifié avec la commande passwd .

Notez que sur Raspbian Linux, utilisez la commande sudo pour effectuer des tâches administratives. Par exemple, pour rechercher les mises à jour de packages dans les référentiels, utilisez la commande sudo apt update .

Définir le nom d'hôte et l'adresse IP du périphérique


Pour définir le nom d'hôte sur le système Pi, exécutez l'utilitaire de configuration Raspbian avec la commande sudo raspi-config .



Sélectionnez 2. Options réseau> Nom d'hôte N1 et OK dans la boîte de dialogue d'avertissement. Saisissez le nom d'hôte et redémarrez l'appareil. Vous pouvez vérifier le nouveau nom d'hôte avec la commande hostname.

Pour définir l'adresse IP statique de l'interface eth0, modifiez le fichier /etc/dhcpcd.conf à l'aide de l'éditeur nano: sudo nano /etc/dhcpcd.conf



Accédez à la fin du fichier et modifiez les valeurs des paramètres: ip_address, routers (passerelle par défaut) et domain_name_servers. Utilisez les paramètres appropriés pour votre réseau, par exemple: 

 #eth0 static IP configuration interface eth0 static ip_address=192.168.1.111/24 static routers=192.168.1.1 static domain_name_servers=192.168.1.1 8.8.8.8

Enregistrez les nouveaux paramètres et redémarrez le périphérique de sudo reboot . Vous pouvez vérifier la nouvelle adresse IP avec la commande ifconfig eth0 .

Installer 3CX sur Raspbian




Pour installer 3CX sur Raspbian, connectez-vous via SSH en tant qu'utilisateur pi, puis:

Définissez la clé 3CX APT pour l'authentification des paquets: 

 wget -O- http://downloads-global.3cx.com/downloads/3cxpbx/public.key | sudo apt-key add -

Créez un fichier source 3CX APT (versions et test): 

 echo "deb http://downloads-global.3cx.com/downloads/debian stretch main" | sudo tee /etc/apt/sources.list.d/3cxpbx.list echo "deb http://downloads-global.3cx.com/downloads/debian stretch-testing main" | sudo tee /etc/apt/sources.list.d/3cxpbx-testing.list

Mettre à jour les sources et packages APT 

 sudo apt update

Vérifier la disponibilité du package PBX 3CX 

 sudo apt-cache policy 3cxpbx

Installer une version stable du PBX 3CX 

 sudo apt -t stable install 3cxpbx

Vous pouvez également installer une version de test. 

 sudo apt -t testing install 3cxpbx



L'installation des packages prend environ 5 à 10 minutes, selon les performances de l'appareil.



Une fois l'installation terminée, sélectionnez la méthode de configuration initiale de 3CX: (1) via un navigateur Web et (2) via la ligne de commande. Si la première option est sélectionnée, entrez l'URL de configuration initiale dans le navigateur: http: // <device_ip>: 5015 (voir le manuel détaillé ).

sudo /usr/sbin/3CXWizard --cleanup l'assistant de configuration initiale 3CX avec la commande sudo /usr/sbin/3CXWizard --cleanup initialise la configuration 3CX. Cela efface toutes les données, y compris les sauvegardes et les enregistrements de conversations! Si nécessaire, enregistrez ces données sur un autre appareil.

Suppression de 3CX dans Raspbian


Pour désinstaller 3CX dans Raspbian, utilisez l'une des commandes suivantes:

  • sudo apt remove 3cxpbx - Les fichiers système 3CX sont supprimés, enregistrant les données de travail (enregistrements de conversation, base de données PBX, fichiers utilisateur, etc.). À utiliser si vous prévoyez de réinstaller 3CX à l'avenir.
  • sudo apt remove --purge 3cxpbx - supprime complètement le système 3CX, y compris les données opérationnelles.
