Reconocimiento de color y luz con APDS-9960


Recientemente, apareció un artículo en Habr.com que, entre otras cosas, informó sobre un sensor de luz. Hace algún tiempo encontré y adquirí algo interesante: un módulo fabricado por RobotDyn basado en el sensor APDS-9960, que también sabe cómo medir el nivel de iluminación. Después de buscar y no poder encontrar referencias a este dispositivo en este recurso, decidí que esta era una buena razón para escribir un artículo.


En el artículo, me gustaría presentar brevemente a los lectores las posibilidades que ofrece este sensor y examinar con más detalle cómo se puede usar para determinar el color y medir el nivel de iluminación.


APDS-9960 es un sensor de Avago, es un sensor digital combinado con varias funciones diferentes interesantes y útiles.
Él sabe cómo reconocer los gestos, determinar la proximidad, y también sabe cómo registrar la intensidad de la luz ambiental y determinar el color.
Esto es exactamente lo que se discutirá en este artículo: con la ayuda de los antiguos STM32VLDISCOVERY y APDS-9960, mediremos la iluminación y determinaremos el color en toda su riqueza de tonos de rojo, verde y azul.


Sin embargo, antes de entrar en la parte práctica, permítanme escribir algunas palabras sobre las características generales del APDS-9960.


El diagrama funcional de APDS-9960 se muestra en la figura a continuación.


Reconocimiento de gestos


La idea de cómo se ve el reconocimiento de gestos en el APDS-9960 se muestra muy bien en este video .


La documentación describe el principio de registro de gestos:
Para reconocer el gesto, se utilizan cuatro fotodiodos direccionales que registran la luz reflejada (en el rango infrarrojo) emitida por el LED incorporado.



Función de detección de proximidad


A juzgar por la descripción de la misma documentación, el mecanismo de detección (aproximación) funciona según el mismo principio exacto que el reconocimiento de gestos.


Reconocimiento de color y nivel de luz ambiental (Color / ALS)


Según el diagrama funcional, el sensor determina el nivel de color / luz utilizando los fotodiodos apropiados. También se afirma que el APDS-9960 tiene filtros incorporados que bloquean los rangos ultravioleta e infrarrojo.
Simplificado, se ve así: las señales grabadas por los fotodiodos se miden con el ADC, se ingresan en el búfer y luego los datos se envían a través de i2c.



Los gráficos en la imagen de arriba están tomados de la documentación del sensor, la respuesta espectral de Color Sense (RGBC) se presenta en la parte superior izquierda.


La señal RGBC de los fotodiodos se acumula durante el período de tiempo establecido por el valor del registro ATIME. Para SparkFun (en su "apds9960.h") este valor está determinado por la constante DEFAULT_ATIME y es igual a 219 que corresponde a 103 ms.


La ganancia es ajustable de 1x a 64x y se determina configurando el parámetro CONTROL OTRA VEZ. La constante DEFAULT_AGAIN, que, a su vez, es igual a 1, que corresponde a una ganancia de 4 veces.


Parte práctica


Personalmente, solo estaba interesado en la función Color / ALS en APDS-9960, así que decidí considerarlo con más detalle y escribí un pequeño código que demostraba su funcionamiento.
Intenté intencionalmente hacer que el código fuera lo más compacto, conciso y extremadamente simple de entender posible; Todo el código se presentará al final del artículo.


Por lo tanto, toda la documentación (dibujo, pinout y diagrama de circuito) para el módulo está disponible en el sitio web del fabricante.



Conecte nuestro módulo APDS-9960 a STM32VLDISCOVERY


APDS9960 usa la interfaz i2c para comunicarse con el mundo exterior, por lo que para STM32VLDISCOVERY usamos el bus I2C1 conectando el pin del módulo SCL al pin PB6 y el pin SDA al pin PB7, respectivamente. No olvides conectar la alimentación y el cable común. Las interrupciones en este caso no se utilizarán, por lo que se puede omitir la salida de Int. En mi foto está conectado, pero no se usa.



Y ahora un pequeño código. Dado que toda la comunicación con el módulo ocurre usando i2c, crearemos la configuración necesaria y definiremos las funciones de lectura / escritura para i2c.


Inicialización de I2C.


Inicialización 
void I2C1_init(void) { I2C_InitTypeDef I2C_InitStructure; GPIO_InitTypeDef GPIO_InitStructure; RCC_APB1PeriphClockCmd(RCC_APB1Periph_I2C1, ENABLE); RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB| RCC_APB2Periph_AFIO , ENABLE); GPIO_InitStructure.GPIO_Speed = GPIO_Speed_2MHz; GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_OD; GPIO_InitStructure.GPIO_Pin = GPIO_Pin_6 | GPIO_Pin_7; GPIO_Init(GPIOB, &GPIO_InitStructure); I2C_StructInit(&I2C_InitStructure); I2C_InitStructure.I2C_ClockSpeed = 100000; I2C_InitStructure.I2C_OwnAddress1 = 0x01; I2C_InitStructure.I2C_Ack = I2C_Ack_Enable; I2C_Init(I2C1, &I2C_InitStructure); I2C_Cmd(I2C1, ENABLE); }

Leyendo el registro.


Función para leer un valor de un registro 
 uint8_t i2c1_read(uint8_t addr) { uint8_t data; while(I2C_GetFlagStatus(I2C1, I2C_FLAG_BUSY)); I2C_GenerateSTART(I2C1, ENABLE); while(!I2C_CheckEvent(I2C1, I2C_EVENT_MASTER_MODE_SELECT)); I2C_Send7bitAddress(I2C1, APDS9960_I2C_ADDR<<1, I2C_Direction_Transmitter); while(!I2C_CheckEvent(I2C1, I2C_EVENT_MASTER_TRANSMITTER_MODE_SELECTED)); I2C_SendData(I2C1, addr); while(!I2C_CheckEvent(I2C1, I2C_EVENT_MASTER_BYTE_TRANSMITTED)); I2C_GenerateSTART(I2C1, ENABLE); while(!I2C_CheckEvent(I2C1, I2C_EVENT_MASTER_MODE_SELECT)); I2C_Send7bitAddress(I2C1, APDS9960_I2C_ADDR<<1, I2C_Direction_Receiver); while(!I2C_CheckEvent(I2C1,I2C_EVENT_MASTER_BYTE_RECEIVED)); data = I2C_ReceiveData(I2C1); while(!I2C_CheckEvent(I2C1, I2C_EVENT_MASTER_BYTE_RECEIVED)); I2C_AcknowledgeConfig(I2C1, DISABLE); I2C_GenerateSTOP(I2C1, ENABLE); while(I2C_GetFlagStatus(I2C1, I2C_FLAG_BUSY)); return data; }

Para que un módulo funcione correctamente, primero debe configurarse correctamente. Específicamente para el reconocimiento del color y la iluminación, debe hacer lo siguiente:


1) Definir el registro ATIME. De forma predeterminada, cuando se inicia el módulo, el registro ATIME tiene el valor 0xFF y si no cambia nada, esto afectará la sensibilidad del sensor; la sensibilidad será baja. 


 i2c1_write(APDS9960_ATIME, DEFAULT_ATIME);

2) en el siguiente paso establecemos el campo de parámetro OTRA VEZ (ALS y Control de ganancia de color) del registro Control Register One (0x8F) en el valor correspondiente a la ganancia igual a x4 (DEFAULT_AGAIN es igual a AGAIN_4X). 


 i2c1_write(APDS9960_CONTROL, DEFAULT_AGAIN);

3) active la opción ALS configurando el bit AEN del registro de registro de habilitación (0x80)
4) encienda la alimentación del módulo configurando el bit PON del mismo registro


así: 


 i2c1_write(APDS9960_ENABLE, (APDS9960_PON | APDS9960_AEN));

Esa es toda la configuración. Nuestro sensor está listo para el trabajo y la defensa, puede comenzar a medir todos los colores.


Pero primero, mida el nivel de iluminación 


 Colour_tmpL = i2c1_read(APDS9960_CDATAL); Colour_tmpH = i2c1_read(APDS9960_CDATAH); Colour_Clear = (Colour_tmpH << 8) + Colour_tmpL;
