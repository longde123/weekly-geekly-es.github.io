Analizador de código fuente de Microsoft Application Inspector



Microsoft lanzó Microsoft Application Inspector, una herramienta de código abierto multiplataforma (!) Para analizar el código fuente.


Application Inspector se diferencia de las herramientas tradicionales de análisis estático en que no intenta identificar patrones "buenos" o "malos"; informará que detectará, según el conjunto inicial de más de 500 plantillas de reglas para detectar funciones, incluidas las funciones que afectan la seguridad, como el uso de criptografía, etc.


El objetivo principal de Application Inspector es una identificación sistemática y escalable de las funciones del código fuente que no se encuentran en otros analizadores estáticos típicos. Esto permite a los desarrolladores y profesionales de seguridad verificar los objetivos previstos de los componentes, por ejemplo, que la biblioteca utilizada solo hace lo que dice.


Las aplicaciones modernas dependen en gran medida del software escrito fuera de su empresa, lo que conlleva riesgos. Application Inspector puede ayudar a determinar características interesantes de la aplicación examinando el código fuente, ahorrando un tiempo considerable en comparación con el análisis "manual".


La aplicación es una herramienta cliente basada en .NET Core, por lo que funcionará en Windows, Linux o macOS y no requiere privilegios elevados, así como una base de datos local, comunicaciones de red o telemetría. Para iniciarlo, simplemente use la línea de comando estándar de dotnet para llamar, por ejemplo, dotnet ApplicationInspector.dll si está ejecutando Linux o macOS o Windows AppInspector.exe.

Puede agregar / editar / eliminar reglas o patrones estándar según sea necesario. También es posible agregar sus propias reglas a una ruta separada al conjunto predeterminado y guardar el conjunto predeterminado o excluirlas usando las opciones de la línea de comandos.


Áreas de prueba y aplicación:


  • Flujo de control: ejecución dinámica de código, gestión de procesos.
  • Criptografía: Cifrado, Hashing, Secretos, Aleatorización.
  • Operaciones del sistema operativo: sistema de archivos, variables de entorno, operaciones de red, cuentas de usuario, registro de Windows.
  • Datos: JSON / XML, Secretos / Claves de acceso, Datos personales confidenciales, SQL / ORM.
  • Manejo de datos: serialización de objetos (XML / JSON), Flash, PDF, Silverlight, uso o análisis de medios de audio / video, Bluetooth, celular, RPC.
  • Marcos: desarrollo, pruebas, dependencias.

De manera predeterminada, el Inspector de aplicaciones genera un informe basado en html para ilustrar las características, un resumen del proyecto y los metadatos encontrados. Las opciones de formato de salida JSON y TEXT también son compatibles.
