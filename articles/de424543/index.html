Java 11 / JDK 11: Allgemeine Verfügbarkeit

Mitstreiter, Java-Kämpfer von Habr! Diesen Tag haben wir so nah wie möglich gebracht.


Heute sind die Release-Version von JDK 11 , die Referenzimplementierung von Java 11 und die erste LTS-Version von Java verfügbar , die in einem schnellen 6-monatigen Release-Zyklus veröffentlicht wurde. Die größte und wichtigste Version der Neuzeit, die der gesamten Java-Community den Anstoß geben wird, auf die neuesten und fortschrittlichsten Technologien umzusteigen.




Seit der Veröffentlichung des 28. Builds sind vier Wochen vergangen, und in dieser Zeit ist kein einziger P1-Level-Bug aufgetreten. Dies bedeutet, dass Java 11 als veraltet und für den vollen industriellen Einsatz bereit angesehen werden kann.


→ OpenJDK-Assembly kann und sollte von hier heruntergeladen werden


Dies ist eine große und wichtige Version, die 17 große Funktionen und unzählige Fehler im Tracker kombiniert. Lassen Sie uns auf die größten Merkmale zurückblicken:


  • JEP 181 : Nestbasierte Zugriffskontrolle
  • JEP 309 : Dynamische Klassendateikonstanten
  • JEP 315 : Verbessern Sie Aarch64 Intrinsics
  • JEP 318 : Epsilon: Ein No-Op-Müllsammler (experimentell)
  • JEP 320 : Entfernen Sie die Java EE- und CORBA-Module
  • JEP 321 : HTTP-Client (Standard)
  • JEP 323 : Syntax lokaler Variablen für Lambda-Parameter
  • JEP 324 : Schlüsselvereinbarung mit Curve25519 und Curve448
  • JEP 327 : Unicode 10
  • JEP 328 : Flugschreiber
  • JEP 329 : Kryptografische Algorithmen ChaCha20 & Poly1305
  • JEP 330 : Starten von Single-File-Quellcode-Programmen
  • JEP 331 : Low-Overhead-Heap-Profiling
  • JEP 332 : Transport Layer Security (TLS) 1.3
  • JEP 333 : ZGC: Ein skalierbarer Garbage Collector mit geringer Latenz (experimentell)
  • JEP 335 : Veraltet die Nashorn JavaScript Engine
  • JEP 336 : Veralten Sie die Pack200-Tools und die API
