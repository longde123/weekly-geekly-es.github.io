Terkadang untuk mengetahui bahwa HDD sedang sekarat, Anda perlu merespons kesalahan yang terdeteksi oleh perangkat lunak aplikasi

Saya membuat aturan untuk selalu menginstal semua pembaruan OS, mungkin dengan sedikit keterlambatan (hingga satu minggu), saya juga menginstal aplikasi khusus (APP Center) dari produsen motherboard, yang memantau pembaruan driver dan beberapa program, dan juga termasuk program yang bermanfaat seperti , misalnya, menyetel operasi kipas atau memperbarui UEFI (pada 2018, saya harus memperbarui 3 kali karena masalah yang teridentifikasi dalam prosesor Intel). Dalam aplikasi ini, hingga saat ini, semuanya berjalan sesuai harapan. Setiap hari saya memeriksa untuk melihat apakah ada sesuatu yang baru dan biasanya segera mengaturnya. Itu sampai baru-baru ini. Ketika pembaruan perangkat lunak kartu suara tiba (itu benar-benar menghapus aplikasi klasik dan menginstal yang baru), saya menunggu sedikit dan memutuskan untuk menginstal. Masalah kecil Pusat APP adalah menawarkan untuk menginstal atau memperbarui sejumlah program yang tidak saya butuhkan, misalnya, antivirus, dan yang dibeli cukup memuaskan bagi saya, jadi sebelum memutakhirkan, saya selalu memeriksa daftar dan menghapus daw yang tidak perlu. Jadi kali ini. Tapi ada yang salah.

Secara umum, biasanya ada dua tab, di bagian bawah ada tombol "Refresh", di sisi lain "Instal", di setiap "Batalkan". Hanya konten dari satu tab yang terlihat pada satu waktu. Yang ketiga muncul hari itu. Sekarang dimungkinkan untuk kembali ke momen sebelum kejadian dan memposting tangkapan layar di sini, dan kemudian kembali ke momen saat ini (saya belum menghapus cadangan, meskipun dalam format lama), tetapi saya mencoba untuk tidak menggunakan Acronis True Image tanpa perlu mengembalikan - bahkan mencoba untuk mendapatkan masalah terlebih dahulu selesaikan dengan cara biasa. Dan terlebih lagi, saya tidak melompat ke berbagai kondisi sistem karena keinginan untuk mengambil tangkapan layar (meskipun pemulihan berlangsung cepat dan mulai langsung dari OS yang berfungsi, Anda hanya perlu mengizinkan reboot). Karena itu, saya mengandalkan memori. Dan dia bisa mengecewakan Anda: hari itu saya membaca beberapa kali. Tampaknya, dia memang logis, tetapi dia tidak memperhitungkan sesuatu.

Sebelum memperbarui, saya tidak melihat tab ketiga, di mana aplikasi juga menempatkan daw berlawanan pembaruan driver ke kartu jaringan dan HDD. Jika saya ingin memperbaruinya, maka tidak pada hari ini. Karena itu, ketika saya melihat sesuatu yang tidak terduga, saya segera berhenti memperbarui. Yah, karena saya melihat bahwa beberapa driver sudah diinstal, dan ada yang tidak, saya kembali ke salinan pagi sistem menggunakan Acronis True Image 2019, yang kadang-kadang saya lakukan dalam kasus seperti itu untuk memperbarui besok, tetapi sementara itu melakukan hal-hal lain. Saya bekerja sepanjang hari tanpa masalah, dan keesokan paginya, Acronis tidak membuat salinan cadangan karena kesalahan hard disk. Solusi yang jelas adalah menjalankan chkdsk. Verifikasi gagal. Kemudian chkdsk c: / scan / v / r. Ada kesalahan dalam cache Chrome, untuk memperbaikinya, perlu reboot. Saya setuju. Ini adalah hal terakhir yang dilakukan.

Saat boot, sistem jatuh ke BSOD dan mulai reboot ke BSOD berikutnya dan seterusnya dalam loop. Desktop tidak mencapai. Bukan masalah: cadangan saya disimpan di HDD USB eksternal, tempat Acronis Survival Kit diinstal, yang saat boot memungkinkan Anda untuk memulai pemulihan cadangan di WinRE. Kegagalan. Program merekomendasikan boot dari Linux Acronis flash drive dan melanjutkan pemulihan. Tapi saya tidak melakukan flash drive ini sebelumnya. Tapi ada flash drive RecoveryDrive, itu boot dari itu (untuk menggunakan komputer orang lain, saya harus menunggu - itu masih pagi) dan meletakkan sistem yang bersih di komputer. Menginstal True Image dan membuat flash drive USB yang dapat di-boot dengan versi Linux dari perangkat lunak ini.

Saya memulihkan salinan sistem malam sebelum kejadian dan kembali memeriksa hard drive - kesalahan yang sama. Saya mencari-cari di majalah System, saya menemukan bahwa masalahnya kemungkinan besar dalam satu filter sistem file (kesalahan terjadi segera setelah memuatnya, dan tanggalnya terlalu kuno). Kali ini, Pusat APP (dalam salinan ini) tidak lagi menawarkan untuk meningkatkan, jadi ia pulih ke salinan terakhir sebelum kejadian. Berhasil memperbarui semua driver, memeriksa semuanya lagi. Selain itu diinstal smartmontools. Ternyata selama ini SMART berhasil menugaskan kembali lebih dari 2.200 sektor (hampir 100%), secara umum, HDD rusak, meskipun masalahnya bukan pada cacat pada permukaan disk, tetapi hanya perangkat lunak. Tapi setidaknya itu berhasil. Sampai saya membeli berlangganan Acronis True Image 2020.

Diperbarui dan sekali lagi tidak dapat membuat salinan. Lebih tepatnya, kadang-kadang salinan dibuat, kadang-kadang bukan karena kesalahan HDD. Menulis dukungan teknis Acronis (alat reguler tidak mengidentifikasi masalah) dan terus mencoba menemukan apa yang tidak berfungsi saat ini. Saya juga memeriksa drive tersembunyi dengan chkdsk - tidak ada masalah. Saya sampai di SeaTools - perangkat lunak dari produsen HDD hanya untuk kasus-kasus seperti itu. Jadi itu mengungkapkan masalah - tes DST tidak lulus. Diusulkan untuk membuat flash drive Linux dan melanjutkan perbaikan dengan mem-boot darinya. Yang saya lakukan. DST juga tidak lolos sampai akhir, meluncurkan tes penuh, dan kemudian perbaikan singkat. Setelah itu, DST berlalu. Dan Acronis True Image 2020 mulai membuat salinan tanpa masalah. Tetapi masih ada pertanyaan.

Setelah perbaikan, smartctl menunjukkan bahwa jumlah sektor yang dipindahkan tidak meningkat, chkdsk (termasuk disk tersembunyi) juga tidak menunjukkan peningkatan jumlah sektor buruk, tampaknya masalah belum sepenuhnya diselesaikan dan mungkin muncul di masa depan. Dan ini berarti Anda mungkin harus mengubah HDD. Ngomong-ngomong, dukungan Acronis menjawab pada hari Senin (saya mengirim permintaan pada hari Sabtu), tetapi saya sudah memecahkan masalah (walaupun sudah tidak mungkin untuk membeli Acronis True Image untuk rubel di Rusia, tetapi masih ada dukungan untuk individu, saya sudah terbiasa sejak 2010) Saya tidak akan mengubah perangkat lunak, saya akan membayar dalam Euro sekarang).

Mungkin spesialis pemulihan data akan dapat mengklarifikasi apakah tindakan dan kesimpulan saya benar. Saya harus segera mengatakan bahwa sekarang di Windows 10 pemulihan salinan VSS yang sudah dikenal tidak memberikan kepercayaan pada operasi normal selanjutnya.
