Chrome googlé

Même si vous n'avez pas de compte Google, le navigateur gratuit Chromium communique toujours avec les serveurs Google en arrière-plan. C'est assez étrange, car les gens installent Chromium au lieu de Chrome juste pour obtenir un programme propre sans liaison commerciale. Cependant, lors de la création de Chromium en mode normal, les blobs binaires de Google sont toujours téléchargés et installés.

Le projet ungoogled-chrome est un ensemble d'indicateurs de configuration, de correctifs et de scripts spéciaux pour supprimer l'intégration avec Google, améliorer la sécurité et les paramètres de gestion.

Un utilisateur anonyme de Github a emprunté les paramètres de plusieurs projets existants, qui ont également essayé de maximiser Chromium à partir de fonctionnalités inutiles afin de créer une configuration minimaliste propre: il s'agit du Chromium de la distribution Debian , du kit de correctifs Inox , ainsi que du navigateur Iridium , en se concentrant sur la protection de la vie privée.

Les développeurs des trois projets mentionnés ont fait des efforts considérables pour se débarrasser de la présence de Google dans un navigateur gratuit. Mais le "Chrome effrayé" est encore plus paranoïaque dans cette entreprise.

Par exemple, l'auteur a remplacé de nombreux domaines Web mentionnés dans le code source de Chromium par des alternatives inexistantes - pour garantir que le navigateur ne puisse envoyer aucune information n'importe où à l'insu de l'utilisateur.

De plus, tous les blobs binaires ont été supprimés du code source: programmes, bibliothèques et toutes autres formes de code machine. Il n'y a que les codes machine qui sont nécessaires pour l'assemblage (par exemple, icudtl.datpour prendre en charge Unicode ou *_page_model.bin, qui définit les modèles de nettoyage des pages dans le DOM Distiller).

Dans un navigateur «googlé», la fonctionnalité spécifique aux domaines Google, par exemple, Google Host Detector, Google URL Tracker, Google Cloud Messaging, Google Hotwording, etc., est complètement désactivée.

L'auteur a ajouté quelques paramètres supplémentaires uniquement à votre goût. Tout le monde ne l'aimera probablement pas.

  • Le moteur de recherche "No Search" a été ajouté à l'omnibox pour désactiver complètement la recherche de l'omnibox.
  • Désactivez le formatage automatique de l'URL dans l'omnibox lorsqu'elle disparaît de la vue http://, etc.
  • Désactiver les boîtes de dialogue JavaScript lors de la fermeture d'une page (événements onbeforeunload)
  • Forcer toutes les fenêtres contextuelles dans de nouveaux onglets.

En outre, l'auteur a modifié certains paramètres standard des navigateurs Iridium et Inox. Par exemple, il interdit le ping obligatoire des adresses IPv6 lorsqu'il détecte la disponibilité IPv6.

En général, vous pouvez «google» Chromium de différentes manières. Outre Iridium et Inbox, une autre option est désormais apparue.

Apparemment, beaucoup de gens ont envie d'obtenir un simple navigateur minimaliste, sans "cloches et sifflets" et sans fonctions intellectuelles, rapide et fiable. La création d'un tel navigateur à partir de zéro est presque impossible, mais le nettoyage de Chrome à partir de scories est une bonne idée.

Le seul problème est que les navigateurs se développent très rapidement. De nouvelles versions sortent plusieurs fois par an. Dans le même Chromium, la version stable est désormais la 53.0.2785.116, mais Chrome 1 et Chromium 1 ont été lancés récemment - le 2 septembre 2008.

Ce n'est pas seulement l'amour des développeurs pour les chiffres. Chaque nouvelle version introduit en fait de nouvelles normes Web, de nouvelles fonctionnalités HTML5 et la prise en charge de nouvelles technologies. Le processus va vite. En raison d'une telle "fluidité", il est très difficile de garder une trace du nouveau code.

L'auteur du Chromium "effrayé" avertit honnêtement qu'en raison du développement rapide du projet Chromium, de nouvelles fonctionnalités qui violent la confidentialité peuvent constamment pénétrer le code, ainsi que de nouveaux bugs.

Pour une raison quelconque, l'auteur cherche à publier des versions en parallèle avec la branche principale de Chromium. Mais quelle idée intéressante ce serait de créer un navigateur ultra-minimaliste pour lequel les mises à jour ne sont pas du tout nécessaires. Désactivez Canvas, WebGL, WebRTC et d'autres choses modernes, désactivez les historiques, les cookies, l'accès du navigateur à l'équipement et plus encore. Créez une version de Tor pour une navigation sécurisée, avec des fonctionnalités minimales, ne laissant qu'un bloqueur de publicités et un nettoyeur de page comme le DOM Distiller susmentionné. Au moins pour les appareils mobiles, un tel navigateur minimaliste serait utile.
