Saluda a la nueva terminal de Visual Studio

Aprovechando el impulso del recientemente anunciado Developer PowerShell , nos complace compartir la primera vista previa del nuevo terminal de Visual Studio. Esta nueva experiencia de vista previa es parte de Visual Studio versión 16.3 Vista previa 3.



En lugar de construir todo desde cero, el terminal de Visual Studio comparte la mayor parte de su núcleo con el terminal de Windows . Para usted, eso se traduce en una experiencia de terminal más sólida y una adopción más rápida de nuevas funciones.

Este artículo en nuestro blog .

Habilitar el nuevo terminal de Visual Studio


Para probar la vista previa del terminal, primero deberá habilitarla visitando la página de Características de vista previa . Vaya a Herramientas> Opciones> Funciones de vista previa, habilite la opción Experimental VS Terminal y reinicie Visual Studio.



Una vez habilitado, puede invocarlo a través de la entrada del menú Ver> Ventana de terminal o mediante la búsqueda.



Crear perfiles de terminal


Al iniciar el terminal, se abre automáticamente una instancia integrada de PowerShell. Sin embargo, puede personalizar la experiencia de inicio mediante el uso de perfiles de shell.

Con los perfiles de shell, puede apuntar a diferentes tipos de shells, invocarlos utilizando argumentos únicos o incluso establecer un shell predeterminado que mejor se adapte a sus necesidades.

En futuras actualizaciones, planeamos optimizar la experiencia rellenando previamente el terminal con algunos perfiles básicos. Mientras tanto, puede agregar manualmente perfiles adicionales en la página de Opciones del terminal.



A modo de ejemplo, así es como puede establecer perfiles para algunas opciones populares:

Símbolo del sistema del desarrollador


Ubicación de la carcasa: 

C:\Windows\System32\cmd.exe

Argumentos: 

 /k "C:\Program Files (x86)\Microsoft Visual Studio\2019\IntPreview\Common7\Tools\VsDevCmd.bat"

Desarrollador PowerShell


Ubicación de la carcasa: 

 C:\WINDOWS\system32\WindowsPowerShell\v1.0\powershell.exe

Argumentos: 

 -NoExit -Command "& { Import-Module 'C:\Program Files (x86)\Microsoft Visual Studio\2019\Preview_master\Common7\Tools\vsdevshell\Microsoft.VisualStudio.DevShell.dll'; Enter-VsDevShell -InstanceId f86c8b33}"

Nota: Deberá actualizar el argumento anterior para que coincida con su configuración específica. Puede extraer la información del argumento buscando en la cadena Target el acceso directo de Developer PowerShell.



WSL


Ubicación de la carcasa: 

 C:\WINDOWS\sysnative\wsl.exe

¡Pruébalo y dinos lo que piensas!


Si bien estamos entusiasmados de compartir esta vista previa, queremos garantizar una experiencia sólida antes de habilitar esta experiencia en la versión de lanzamiento de Visual Studio. Como resultado, el terminal inicialmente solo estará disponible en las versiones de vista previa de Visual Studio 2019.

Como próximos pasos, buscaremos ofrecer mejoras en torno al renderizado (el terminal actualmente necesita ser redimensionado para renderizar correctamente), accesibilidad y temas. También agregaremos nuevos impulsores de productividad, como múltiples instancias de terminal e integración más profunda con Visual Studio.

Nos encantaría saber cómo se adapta a su flujo de trabajo y cómo podríamos mejorar aún más su experiencia de terminal. Envíenos sus comentarios a través del portal de la Comunidad de desarrolladores o mediante la función Ayuda> Enviar comentarios dentro de Visual Studio.
