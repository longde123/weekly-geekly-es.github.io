Cómo construir una arquitectura de TI en una empresa con SAP Enterprise Architecture Designer

En este artículo de la serie de materiales sobre SAP Data Management Suite, hablaremos sobre la aplicación para construir la arquitectura de TI en la empresa: SAP Enterprise Architecture Designer.

Artículos anteriores del ciclo:
SAP Data Management Suite como un complejo para trabajar con Big Data en empresas

El servicio SAP Enterprise Architecture Designer (EA Designer) le permite describir, modelar y analizar paisajes de información, requisitos funcionales, procesos comerciales, estructuras y flujos de datos y otros artefactos dentro de una empresa.





EA Designer es una herramienta para crear y mantener una arquitectura corporativa completa dentro de una empresa. Con EA Designer, puede describir rápidamente su arquitectura de TI en su empresa y luego visualizarla fácilmente.

Todos los modelos se colocan en un único repositorio, lo que simplifica la colaboración en el proceso de creación de una arquitectura corporativa común.

El servicio también admite comunicaciones entre objetos de diferentes modelos y diagramas. Un análisis de influencia y origen le permite visualizar y rastrear estas relaciones para comprender las relaciones entre los objetos y el efecto de los cambios en uno de los objetos de la cadena.



EA Designer admite la creación y edición de los siguientes tipos de gráficos:

Procesos de negocios Los diagramas de procesos empresariales le permiten identificar, describir y descomponer procesos empresariales. Hay soporte para dos variantes de BPMN 2.0:

  • BPMN 2.0 descriptivo. Comúnmente utilizado por usuarios comerciales. Contiene un subconjunto de objetos BPMN 2.0 adecuados para diseñar y analizar procesos empresariales.
  • BPMN 2.0 ejecutable. Usualmente utilizado por los desarrolladores de procesos, incluye todos los objetos BPMN 2.0 estándar y le permite simular un proceso de negocios teniendo en cuenta todos los detalles técnicos. Esto hace posible utilizar este modelo para una mayor implementación del proceso en sistemas BPM, por ejemplo, SAP Process Orchestration. También es posible la ingeniería inversa de modelos de SAP BPM o Eclipse BPMN2 Modeler.




Modelo conceptual de datos Ayuda a desarrollar y analizar la estructura conceptual de los sistemas de información, para identificar los principales objetos que deben representarse, sus atributos y la relación entre ellos.

Modelo de datos físicos Ayuda a diseñar y analizar la estructura de la base de datos. Puede rediseñar cualquier base de datos compatible. Se admite la generación directa de estructuras de bases de datos en SAP HANA basadas en el modelo físico.

Diagrama de flujo de datos Ayuda a diseñar y analizar la transferencia de datos entre almacenes de datos y su transformación. Puede realizar ingeniería inversa de los flujos de datos y crearlos en SAP Web IDE.



Arquitectura corporativa (Diagrama de arquitectura empresarial) . Ayuda a analizar y documentar la organización, sus capacidades y procesos, aplicaciones y sistemas que los soportan, así como la arquitectura física en la que se implementan.



Diagramas NoSQL (NoSQL Document Schema, JSON) . Ayudan a desarrollar, analizar y documentar estructuras JSON utilizadas en sistemas. Puede realizar ingeniería inversa de esquemas JSON (o heredarlos de documentos JSON) y crear archivos de esquema JSON.

Lista de requisitos Muestra una lista jerárquica de objetivos comerciales, requisitos funcionales, etc.

Según la información de los modelos, puede crear rápidamente informes en varios formatos comunes, distribuyéndolos entre las partes interesadas.

EA Designer está disponible tanto en las instalaciones como en las opciones de implementación en la nube. El usuario trabaja con la aplicación a través de un navegador.
