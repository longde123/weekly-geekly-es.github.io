Zimbra Collaboration Suite中的GAL设置

如今，所有协作系统的标准功能都是公司的地址簿或Glabal地址列表，其中包含有关职位，工作地点以及所有员工的工作电话号码和邮寄地址的信息。 由于这样的通讯录可供每个员工使用，因此公司内部的通讯变得尽可能快速和真正无缝。 但是，即使像地址簿这样简单的事情，也存在许多细微差别和陷阱。 让我们看一下如何在Zimbra Collaboration Suite中实现它。



在Zimbra中创建内部GAL是最简单的方法。 管理员只需创建一个资源帐户，就可以在该地址簿中存储服务器上所有帐户的联系信息。 此后，创建的资源帐户的地址簿将可供所有用户毫无例外地查看，并显示在公司员工的联系人列表的常规列表中。

您可以在创建域时在管理员控制台中以及在命令行上使用以下命令为内部GAL创建资源帐户： 

zmgsautil createAccount -a gal@company.ru -n InternalGAL --domain company.ru \ -s server.name -t zimbra -f _InternalGAL zmgsautil forceSync -a gal@company.ru -n InternalGAL

我们提请您注意以下事实：在Zimbra中，在每个域上只能创建一个资源GAL帐户。 这种限制远远超过了连接多个源以获取联系人的能力。 这就是Zimbra中的GAL如此灵活的原因。

创建GAL的资源帐户后，用户将可以按姓名，姓氏，邮寄地址和职位搜索员工。 此外，创建GAL后，用户在写字母时将在地址栏中自动完成。 如果您安装了Zextras聊天扩展程序，则另一个优点是在线聊天联系人列表中所有员工的外观。

但是，此处介绍的创建GAL的方法并不适合所有情况。 例如，如果企业IT经理通过MS Exchange或其他协作系统迁移到Zimbra，该怎么办？

特别是对于已经建立了IT基础架构的企业，Zimbra允许使用外部GAL，该GAL是从二手系统中提取的。 资源帐户的联系人来源可以是外部LDAP或AD。

为了使Zimbra从AD显示企业的地址簿，您需要在Zimbra管理控制台中选择您的域，然后在域设置中启用GAL配置向导。 在设置过程中需要指定的主要内容：

  • 服务器类型
  • 服务器地址
  • LDAP过滤器
  • 自动填充过滤器
  • 搜索授权数据
  • GAL同步选项

输入所有必需的设置后，无需离开GAL设置向导就可以测试输入配置的功能。 如果一切正确完成，则第一次同步后，来自AD的联系人将出现在您的GAL中。 它们还将立即出现在自动完成列表中。 您可以在命令行中配置从AD接收GAL。 为此，请输入以下命令： 

 zmgsautil createAccount -a gal@company.ru -n ExternalContactsGAL --domain company.ru -t ldap -p 1d zmprov mds gal@company.ru ExternalContactsGAL \ > zimbraGalSyncLdapBindDn zimbra@ad.company.ru \ > zimbraGalSyncLdapBindPassword ********* \ > zimbraGalSyncLdapFilter "(&(objectClass=contact)(mail=*))" \ > zimbraGalSyncLdapSearchBase dc=ad,dc=company,dc=ru \ > zimbraGalSyncLdapURL ldap://ds1.ad.company.ru:3268 \

Zimbra还允许您使用多个源来编译GAL。 可以使用命令zmgsautil addDataSource -a gal@company.ru -n AnotherGAL --domain company.ru -t ldap -p 1d来实现 。 使用此命令，将创建GAL的其他信息源，管理员只能将其配置为从必要的源接收信息。 这是通过一组熟悉的命令完成的： 

 zmprov mds gal@company.ru AnotherGAL \ zimbraGalSyncLdapBindDn uid=zimbra,cn=admins,cn=zimbra \ zimbraGalSyncLdapBindPassword ******* \ zimbraGalSyncLdapFilter '(&(mail=*)(zimbraAccountStatus=active)(!(zimbraHideInGAL=TRUE)))' \ zimbraGalSyncLdapSearchBase dc=anothercompany,dc=ru \ zimbraGalSyncLdapURL ldap://ldap.anothercompany.ru:389 \

因此，Zimbra中的GAL的配置非常灵活，不仅可以创建完整的通讯簿，而且还可以帮助将Zimbra集成到企业的现有IT基础结构中，帮助从MS Exchange迁移到Zimbra，还可以合并来自多家企业或在一个地址簿中分支。
