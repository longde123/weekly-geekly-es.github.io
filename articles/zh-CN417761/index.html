GitLab正在从Azure迁移到Google Cloud Platform。 搬迁新闻和维护日期


来自GitLab.com博客的帖子翻译。


GitLab.com计划在7月28日迁移到Google Cloud Platform。 我们告诉您现在和将来将为您带来什么。


对于我们来说，始终将提高GitLab.com的性能和可靠性作为首要任务 。 两者之间都在逐步增长，与此同时，我们计划了能够提供真正突破的重大变更：使GitLab成为Kubernetes的云（ 云原生 ）应用程序。


我们准备向原生云迈出一大步：从Azure迁移到Google Cloud Platform（GCP）。 Azure对我们来说是一个很好的提供商，但是GCP更好地支持Kubernetes，这对于我们的深远计划很重要。 这将立即产生影响，包括默认情况下启用的加密数据存储以及由于GCP与我们的CDN紧密集成而导致的更快的缓存。


迁移到GCP的维护时间


有一篇有关迁移的博客文章 ，但是在这里我们向社区简要介绍了我们计划在下周末（7月28日）进行迁移。 维护将分两个阶段进行，我们都在通知。


您应该知道的是：


在维护期间，以下服务将不可用：


•SaaS网站（ GitLab.com将不可用，但about.gitlab.com和docs.gitlab.com将继续工作）
•Git ssh
•Git https
•注册表
•CI / CD


7月21日，星期六，世界标准时间13:00（莫斯科时间16:00）


作为测试的一部分，我们将于7月21日（星期六） 莫斯科时间16:00进行短期维护。 这是准备情况的最终测试。 维护需要1个小时。


7月28日，星期六，世界标准时间10:00（莫斯科时间13:00）


在出发当天，我们将在莫斯科时间13:00开始维护。 根据计划，GitLab.com将被停止2个小时。 如果有任何变化，我们将通过以下渠道进行报告。 在此期间结束时，GitLab.com将在GCP上运行。


• GitLab状态页面
• GitLab状态高音扬声器


GitLab页面上的自定义域


如果您在GitLab页面上有一个单独的域：


•我们将离开代理，您将有时间更改DNS。
•从7月28日开始，GitLab页面将移至35.185.44.232。
•在完成迁移之前，请勿将DNS更改为新地址。
•当禁用Azure中的GitLab页面地址时，我们将分别在博客上报告。


如果您在迁移期间需要支持，请写信给GitLab支持 。


祝我们好运！
