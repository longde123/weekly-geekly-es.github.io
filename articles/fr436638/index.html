Redistribution des fenêtres entre les moniteurs après s'être réveillé du mode veille

Vous avez 2-3 moniteurs, sur un navigateur et un éditeur de code, sur le deuxième manuel, sur le troisième mail, vous avez trié les fenêtres selon vos besoins. Ensuite, nous sommes allés verser une tasse de café, le moniteur éteint, vous êtes venu, avez tiré la souris et tout à coup toutes les fenêtres étaient sur un seul moniteur principal, tout était en tas, vous avez encore levé les fenêtres et la situation se répète à chaque fois.

Ce sujet est abordé dans de nombreuses ressources, y compris le forum Microsoft, mais il est difficile de trouver une solution et une description claire du problème.

Un tel problème peut ne pas être observé par les propriétaires de moniteurs sans communication bidirectionnelle ou avec DDC / CI désactivé par défaut (sur le DELL précédent, ce n'était pas là, il a été désactivé dans le menu). Si vous n'avez qu'un seul moniteur, les fenêtres ne se reconstruisent pas non plus.

Il était une fois, de nombreux utilisateurs se plaignaient d'un tel problème dans Windows, dans lequel, après avoir éteint le moniteur, les fenêtres restaient hors de vue, et, apparemment, les développeurs ont décidé de corriger ce point, s'il n'y a pas de moniteur, alors les fenêtres passeront à celui qui est allumé.

Lorsqu'un moniteur est connecté via DisplayPort, lorsqu'il passe en mode veille, le système pense qu'il n'est pas là, toutes les fenêtres sont levées quand elles se réveillent sur un autre moniteur, et lorsque ce moniteur principal est détecté après une fraction de seconde, toutes les fenêtres s'envolent. Phénomène désagréable.

Les utilisateurs du forum se plaignent de cela, certains lorsqu'ils commencent à effectuer le rendu pour toute la nuit, diminuent simplement la luminosité, certains définissent des écrans de démarrage et éteignent le moniteur par timeout.

Certaines personnes, même sur le forum Microsoft, demandent de faire un tel réglage que, après avoir éteint le moniteur, les fenêtres resteront dessus ou seront jetées sur un autre écran.

Il existe de nombreuses solutions ici, si le moniteur prend en charge la désactivation du DDC, vous pouvez le faire.
Vous pouvez couper le contact, mais c'est aussi une sorte de décision cruelle.

Je n'ai pas eu l'occasion de voir comment cela se passe sur les cartes vidéo Nvidia, mais sur AMD, il suffit de désactiver simplement les services AMD External Events Utility, pour Nvidia, le service Deamon du service de mise à jour NVIDIA est désactivé.

Après ces manipulations, vous pouvez même éteindre le moniteur avec un bouton et les fenêtres ne s'enfuiront pas, très pratique pour ceux qui ont beaucoup de moniteurs et doivent éteindre le moniteur pendant des périodes.

Et pourtant, un tel moment, j'ai un DVI-HDMI connecté au moniteur chinois avec un cordon chinois, et il s'est toujours montré de cette façon auparavant, sur le deuxième moniteur, j'ai un code et quelque chose à travailler, mais sur le 34 pouces principal, je regarde des films quand Je m'assois sur le canapé, je ne veux pas vraiment que le deuxième moniteur me distrait, je l'éteins et les fenêtres ne se reconstruisent pas, apparemment le même DDC / CI n'est pas implémenté sur cette dentelle.

Veuillez ne pas juger strictement et il serait utile que vous ajoutiez vos propres options pour résoudre ces problèmes.

Merci de votre attention, j'espère que quelqu'un le trouvera utile.

UPD: Lorsque les fenêtres ne sont pas collectées sur un moniteur, mais restent allumées, vous pouvez appuyer sur WIN + P et sélectionner "uniquement l'écran de l'ordinateur" ou "uniquement le deuxième moniteur" au lieu de "développer".

Vous pouvez également sélectionner une fenêtre via Alt + TAB, puis via WIN + la flèche (gauche / droite) la déplacer vers le moniteur, selon l'emplacement des moniteurs dans les paramètres.
