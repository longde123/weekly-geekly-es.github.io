Bibliotecas gratuitas para crear y editar archivos PDF

Hace una semana, se lanzó una biblioteca gratuita para generar archivos pdf de la versión OpenPDF 1.1.0 , la actualización más importante de esta biblioteca en los últimos tiempos.

OpenPDF es una biblioteca de Java para crear y editar archivos PDF, que se distribuye bajo la licencia dual LGPL / MPL. La biblioteca se basa en la bifurcación iText. En realidad, ella tiene una historia así:

LibrePDF/OpenPDF

forked from rtfarte/OpenPDF

forked from kulatamicuda/iText-4.2.0

forked from daviddurand/iText-4.2.0

forked from ymasory/iText-4.2.0

Todos los tenedores lideran la historia de una versión de iText 4.2.0, que se lanzó hace ocho años y se convirtió en la última versión bajo una licencia gratuita. Los propios desarrolladores de IText se cambiaron a AGPL y se concentraron en crear la quinta versión para vender con mayor éxito el software comercial. Hasta la fecha, han crecido hasta la versión iText 7 . Pero el proyecto gratuito tampoco murió, sino que continuó desarrollándose por las fuerzas de toda la comunidad.

En el pasado, la biblioteca se ha mejorado en algunos aspectos, se han solucionado varios errores. Aunque los cambios no son demasiado significativos, el desarrollo continúa y todos pueden contribuir. Si necesita alguna funcionalidad específica al generar archivos PDF, sugiera un parche e implemente esta función para todos.

Desde la antigua versión "original" 4.2.0, se han realizado los siguientes cambios.

  • Compilación Java 8
  • Si el archivo PDF finaliza inesperadamente, se genera una IOException, no un PDFNull
  • parche adoptado para arreglar NPE en forma XFA (método escapeSom)
  • Se acepta la funcionalidad UnembedFontPdfSmartCopy de Vicente Alencar
  • ubermichael adoptó la funcionalidad de pie de página RTF
  • corrección de compatibilidad para admitir bibliotecas criptográficas más recientes Bouncy Castle (1.54)
  • algunas correcciones de NPE y soporte de marco retorcido de Maven

OpenPDF es una forma conveniente de implementar la generación de documentos PDF en su proyecto, incluida la firma digital criptográfica. Este es un módulo listo para usar y gratuito adecuado para su uso en la gestión de documentos electrónicos. Y dicho sistema se puede utilizar, por ejemplo, con certificados para firmar un PDF para certificar y aprobar documentos PDF. O integre con el servicio de firma digital de GlobalSign , un servicio de firma basado en la nube. Aunque este último ya funciona, por ejemplo, con la solución para firmar documentos de Adobe Sign en Adobe Document Cloud.

El soporte para bibliotecas criptográficas Bouncy Castle es muy útil. En el corazón de la arquitectura de Bouncy Castle hay un conjunto de API de bajo nivel que implementan todos los algoritmos criptográficos. Hay soporte para API criptográficas estándar de alto nivel de las plataformas Java y C #.

Las API Java de Bouncy Castle en la versión 1.53 incluían alrededor de 400 mil líneas de código y admitían una gran cantidad de algoritmos, incluida la generación básica de certificados X.509, el trabajo del proveedor JCE / JCA, PKCS # 10, PKCS # 12, S / MIME, OpenPGP, DTLS, TLS, OCSP, TSP, CMP, CRMF, DVCS, DANE, EST, Attribute Certificate (AC), que se utilizan para asociar información adicional con un certificado de clave pública.

Cabe señalar que, además de OpenPDF, existen otras bibliotecas gratuitas para generar PDF. Por ejemplo, hay veraPDF o la maravillosa biblioteca Apache PDFBox . Este último es activamente compatible y tiene un rico conjunto de funciones:

  • Extraer texto Unicode de archivos PDF.
  • Combine varios archivos o divida un PDF en partes.
  • Extraiga datos de formularios PDF o complételos.
  • Validación de PDF contra PDF / A-1b.
  • Imprima PDF a través de API Java estándar.
  • Guardar como imágenes como PNG o JPEG.
  • Cree PDF desde cero, con fuentes e ilustraciones incrustables.
  • Firmar digitalmente PDF.

Así que hoy en día no es necesario utilizar herramientas propietarias para crear o editar archivos PDF. Hay muchas alternativas gratuitas para esto.
