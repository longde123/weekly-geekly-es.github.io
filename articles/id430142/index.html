Apakah Windows Anda sudah sangat tua?

Sebagai diskusi panas dari artikel terbaru tentang HabrÃ© menunjukkan pada sistem Windows, sejumlah besar pengguna maju dan tidak terlalu memberikan preferensi mereka untuk "usang" Windows 7 dan tidak terburu-buru untuk beralih ke Windows 10. Alasannya bisa sangat berbeda - dari ketidakpuasan dengan kebijakan privasi dan banyak telemetri ke kebiasaan biasa dan keengganan untuk mengikuti jejak pemasar Microsoft.

Dalam artikel ini saya ingin mempertimbangkan kasus ketika Windows 7, dan bahkan versi sebelumnya, diperlukan oleh pengembang untuk solusi berkualitas untuk tanggung jawab pekerjaan mereka. Bukan rahasia lagi bahwa pangsa Windows 7 masih sangat besar di sektor korporasi, dan karena Microsoft memperpanjang dukungannya hingga 2023, ada sesuatu yang tidak mungkin berubah secara dramatis dalam waktu dekat. Dan ini belum lagi fakta bahwa ada sistem yang cukup kuno dengan Server 2003 dan 2008, yang menjalankan beberapa jenis perangkat lunak lama, tetapi sangat diperlukan, yang tidak perlu ditelusuri berdasarkan prinsip "bekerja - jangan sentuh". Namun, memperbarui sistem operasi "usang" Anda jauh lebih cepat dan lebih mudah daripada kedengarannya.

Saat ini diperlukan untuk menguji perkembangan baru atau menambah fungsionalitas yang sudah ada, muncul pertanyaan menginstal sistem operasi uji untuk menguji operabilitas, yang bukan ide terbaik untuk dilakukan pada sistem kerja nyata. Dan kemudian ternyata bahwa bahkan jika Anda memiliki image atau disk instalasi yang berfungsi, Windows 7 SP 1 dari model 2011 jauh dari sama dengan Windows 7 pada mesin pengguna yang telah menerima ratusan pembaruan keamanan dan stabilitas selama dekade terakhir.

Dalam kasus ketika Anda mencoba menginstal pembaruan menggunakan alat biasa pada beberapa Windows Server 2008 atau Windows 7 SP1, Anda menemukan bahwa sistem pembaruan tidak berfungsi dan mencari pembaruan tanpa henti. Banyak orang tahu bahwa ini terjadi, termasuk karena tumpukan layanan OS yang ketinggalan zaman.

Mari kita pertimbangkan cara untuk memperbarui Windows 7 sepenuhnya dan cepat ke keadaan saat ini, termasuk, secara opsional, tanpa menghubungkan mesin virtual ke Internet, semua langkahnya sama untuk Windows Server 2008 R2 SP1, Datacenter dan Enterprise. Kami akan membutuhkan instalasi bersih Windows 7 SP1 dari ukuran bit yang sesuai, dibuat dari gambar dari MSDN atau DVD instalasi pada sistem virtualisasi favorit Anda atau mesin nyata. Entri kunci dan aktivasi pada tahap ini dapat dilewati dan dilakukan nanti.
Bagi mereka yang tidak memiliki gambar resmi di zashnik dan menggunakan layanan hosting file untuk mengunduhnya, jangan lupa untuk memeriksa file checksum dengan yang tersedia untuk gambar ini di situs web Microsoft, Anda dapat melakukan ini menggunakan versi percobaan Total Commander, misalnya.

Setelah berhasil menginstal OS itu sendiri, kita perlu mengunduh dari microsoft.com hanya tiga pembaruan untuk OS dengan kedalaman bit yang sesuai. Pertama, kita membutuhkan KB3177467 Servis pembaruan tumpukan untuk Windows 7 SP1 dan Windows Server 2008 R2 SP1 - Oktober 2018 pembaruan tumpukan layanan. Setelah menginstal SSU, Anda perlu menginstal rollup pembaruan KB3125574 dan reboot. Jika instalasi gagal, Anda mungkin perlu menginstal sebelumnya KB3020369.

Langkah terakhir adalah menginstal paket rollup bulanan, versi saat ini yang tersedia KB4462923 di sini . Di masa depan, versi terbaru dari pembaruan kumulatif terletak di mesin pencari menggunakan pembaruan bulanan rollup windows 7 permintaan november, di mana november adalah bulan paket pembaruan. Paket baru tersedia di Eropa mulai dari Rabu kedua setiap bulan, pemasangan paket sebelumnya pada prinsipnya tidak diperlukan.

Jadi, setelah menginstal ketiga pembaruan ini, Windows 7 sepenuhnya diperbarui dengan semua perbaikan stabilitas dan keamanan saat ini. Semua tindakan, kecuali untuk paket layanan pengunduhan awal, dapat dilakukan tanpa koneksi internet. Selain itu, memperbarui tumpukan layanan akan memungkinkan Anda untuk menerima pembaruan baru melalui Pembaruan Windows dengan cepat dan tanpa masalah. Harap dicatat bahwa pembaruan untuk perangkat lunak Microsoft lainnya, seperti Office dan Dot NET Framework, dibuat secara terpisah.
Sebagai sebagian kecil dari paranoia. Jika Anda menemukan Program Peningkatan Kualitas Perangkat Lunak Windows di bilah pencarian, maka saya lebih memilih untuk menonaktifkan item ini, karena Microsoft tidak berencana untuk mengembangkan Windows 7. Ini mungkin mengapa Anda juga harus menghapus layanan diagtrack.
