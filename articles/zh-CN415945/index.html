带有Aliexpress的新中文PLC：Wecon LX3VP / LX3VE

我将继续介绍与之配合使用的可编程逻辑控制器的一系列说明。 这次将再次是关于Wecon活动中的“与Aliexpress合作的中国控制人”。

但是首先您需要回顾为什么我对它们产生了兴趣。 在为项目选择PLC时，不可避免地会出现价格质量比的问题。 当然，在任何地方使用西门子或施耐德（Schneider）经过时间考验的控制器都是不错的选择，但这并不总是允许预算的-它们非常昂贵。

以前，当我们不得不在项目中放置更便宜的控制器时，我们选择了家用Aries PLC，但是由于许多原因，我们开始寻找替代方案。 除其他外，我提请注意LX3V / LX3VP / LX3VE系列的中国公司Wecon的廉价PLC。 我已经写过关于最简单，最便宜的型号LX3V-0806MT-A2。

今天，我们将讨论更高级的型号：LX3VP-1616MR-A和LX3VE-1412MT-A。



LX3VP / LX3VE的主要优缺点


优点：

  • 与三菱FX2N兼容的软件架构。 对我来说，这很关键-编程时，我想要一些清晰且受欢迎的东西，而不要研究另一辆“独家自行车”
  • 使用Wecon PLC Editor免费软件进行编程
  • 该软件中有一个PLC模拟程序，可让您无需控制器本身即可调试程序。 没看到Eka？ 同时，我与之合作的另一个中文Xinje PLC，具有类似的软件架构，没有对该软件进行仿真。 这会使已开发程序的支持复杂化。
  • 通过标准的Micro USB电缆下载程序
  • 多达6个RS-485数据传输端口，其中多达5个具有Modbus
  • 安装DB Board时最多2个以太网端口
  • 能够连接扩展板DB板和扩展模块。 例如，ARIES PLC110没有此类选项
  • 官方网站上有一个论坛，技术支持和示例程序
  • 低成本，特别是如果您通过Aliexpress或直接在Wecon上购买


缺点：

  • PLC上只有1个带有Modbus的RS-485端口，其余的仅在连接DB板时使用，即需额外付费
  • 没有俄语文档。 但是此缺点是有条件的，因为该网络在俄语中具有许多有关对Mitsubishi FX2N PLC进行编程的信息，Wecon LX3在命令级别与之兼容。 因此，我们可以根据三菱FX2N的材料安全地研究Wecon编程，我做到了

扩展选项


如果LX3VP / LX3VE PLC的基本功能还不够，则可以通过安装扩展模块和可选的DB Board卡来对其进行扩展。 因此，您可以增加输入和输出（包括模拟）的数量，并增加数据端口的数量。

PLC总共最多可以增加256个输入和256个输出。

扩展模块安装在处理器模块右侧的DIN导轨上，并通过短环连接至扩展模块或先前安装的模块。 如果有人记得的话，这与Simatic S7-200的情况非常相似。

在某些PLC型号上，例如，在简单的LX3V-0806MT-A2中，不可能连接扩展模块。

用于4 AI的模拟量输入模块LX3V-4AD -10 ... 10V / -20 ... 20mA / 4 ... 20mA：



与扩展模块不同，DB Board直接安装在特殊位置的PLC机柜上。 根据PLC型号的不同，DB板可能有1个或2个座位，所有LX3V PLC型号都至少有1个座位，甚至LX3V-0806MT-A2也是如此。

DB Board可以为单一（短）或双重（长）尺寸。 因此，它们的座位为单人或双人。



选择DB板时，需要考虑PLC上有多少个座位以及它们的大小。 例如，PLC LX3V-0806MT-A2具有一个用于短DB板的座位。
LX3VE-1412MT也有1个座位，但在长DB板下。
LX3VP-1616MR有2个座位，其中1个用于矮个DB板，而1个用于个长DB板。
自然，在长板的座位上，您可以安装一块短的DB板，但是在短板上，您不能安装一块长的DB板。

数据接口和协议


在考虑中的LX3VP / LX3VE型号上，只有1个RS-422 Com0端口和2个RS-485端口：Com1和Com2。

同时，RS-422 Com0和RS-485 Com1是通用RS-422 / 485端口的两个组成部分，因此可以通过Com0或Com1同时进行数据传输。

圆形端口RS-422 Com0最初是为了与三菱FX2N兼容而引入Wecon PLC中的，该端口用于下载程序。 顺便说一下，用于编程的RS-422电缆也是兼容的。 但是，Wecon还为其控制器安装了USB端口以进行编程，这更加快捷，方便。

Com0 / Com1支持Wecon LX3VP / LX3VE协议，但Modbus不支持。 可以将HMI连接到Com0 / Com1，但是它们必须根据Wecon LX3VP / LX3VE协议工作。 首先，当然是Wecon本身的HMI。

HMI Samkoon和Weintek当前不支持Wecon协议。 但是它们支持三菱FX2N协议，并且由于Wecon支持Com0 / Com1，因此它也可以使用三菱FX2N协议连接到PLC。 但是在这里，我冒着风险和风险，在HMI Samkoon软件模拟器中设法联系LX3VE，但是必须检查这种连接在实际使用中的可靠性和正确性。

Wecon LX3VP / LX3VE协议很好，因为您不需要在PLC程序中进行任何配置，只需将PLC连接到HMI并向其输出数据即可。
该协议使用固定参数：9600、7、1，偶数。

通过Wecon协议进行PLC和HMI操作


接线图



RS-485 Com2端口支持Wecon LX3VP / LX3VE和Modbus协议。

可以使用具有2个RS-485 Modbus端口的DB Board LX3V-RS458-BD添加其他RS-485端口。

因此，在LX3VE-1412MT-A中，在DB板下方有1个位置，您最多可以安装4个RS-485，其中Modbus最多可以安装3个。
在LX3VP-1616MR-A上，DB板有2个位置，您最多可以安装6个RS-485，其中Modbus最多可以安装5个。

LX3VP / LX3VE没有以太网端口，但是您可以使用具有1个以太网端口的DB Board LX3VP-ETH-BD来添加它们。
因此，可以在LX3VE-1412MT-A中安装1个以太网端口，在LX3VP-1616MR-A中安装2个以太网端口。
