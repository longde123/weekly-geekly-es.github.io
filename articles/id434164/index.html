Bermigrasi dari IBM Notes / Domino ke Zimbra Collaboration Suite

Dalam komentar pada artikel terakhir, muncul pertanyaan tentang kemungkinan migrasi dari IBM Notes / Domino ke Zimbra Collaboration Suite. Memang, tidak seperti transisi dari MS Exchange ke Zimbra Collaboration Suite, proses yang dijelaskan secara rinci di salah satu artikel sebelumnya, bermigrasi dari IBM Notes / Domino jauh lebih sulit. Namun, seperti yang kita ketahui, tidak ada situasi tanpa harapan, dan sekarang kita akan berbicara tentang apa dan bagaimana mentransfer dari server IBM Notes / Domino ke server Zimbra Collaboration Suite.



Pertama-tama, perlu dicatat bahwa karena kerumitan proses yang besar, migrasi lebih baik dipercayakan kepada seseorang dengan pengalaman luas dalam IBM Notes / Domino sehingga ia dapat dengan cepat memperbaiki masalah yang muncul selama proses transfer data. Solusi yang mungkin untuk masalah yang muncul dapat ditemukan di forum Zimbra resmi, serta di Wiki Zimbra.

Untuk migrasi langsung, kami akan menggunakan Wisaya Migrasi Server Kolaborasi Zimbra untuk Lotus Domino . Aplikasi untuk sistem operasi MS Windows ini akan membantu mentransfer akun pengguna yang diperlukan ke IBM Notes / Domino, isi kotak surat mereka, serta data kalender dan daftar kontak dari buku alamat ke server dengan Zimbra. Agar bisa berfungsi, Anda harus memasukkan informasi akun administrator IBM Notes / Domino untuk mendapatkan data pengguna untuk impor. Perhatikan bahwa Wizard Migrasi ZCS untuk Lotus Domino secara mandiri membuat akun untuk akun yang ditransfer dari server IBM Notes / Domino jika belum dibuat sebelumnya. Kami akan memulai Wisaya Migrasi ZCS untuk Lotus Domino di komputer tempat IBM Notes / Domino diinstal. Agar dapat berfungsi dengan benar, diperlukan file .dll versi terbaru yang disertakan dengan IBM Notes / Domino.

Sebelum migrasi, sangat penting untuk melakukan pekerjaan informasi dengan karyawan. Penting untuk memberi tahu setiap karyawan yang akunnya akan ditransfer ke server baru bahwa ia akan segera menggunakan aplikasi email baru dengan antarmuka baru. Selain itu, semua orang perlu memberikan informasi tentang nama domain tempat Zimbra Collaboration Suite akan berfungsi, serta pasangan nama pengguna / kata sandi, yang akan digunakan untuk mengakses email dan alat kolaborasi. Migrasi itu sendiri paling baik direncanakan untuk akhir pekan atau hari libur, ketika beban pada email akan minimal.

Kami juga menyarankan Anda memigrasi kotak surat dalam porsi kecil sehingga prosesnya tidak memakan banyak waktu dan Anda memiliki kesempatan untuk dengan cepat mengubah catatan untuk kotak surat yang dimigrasi, sehingga surat baru akan tiba di server Zimbra dan tidak tetap di IBM Notes / Domino.

Setelah memulai Wisaya Migrasi ZCS untuk Lotus Domino, Anda akan diminta untuk memasukkan informasi tertentu di beberapa kotak dialog. Di jendela pertama berjudul Impor Tujuan Anda harus menunjukkan:

  • Nama Host - nama domain server dengan Zimbra
  • Admin Port - port akses ke layanan administrasi Zimbra (7071 secara default)
  • Pakai Koneksi Aman - digunakan untuk membuat koneksi aman ke server
  • Nama Pengguna dan Kata Sandi - data akun administrator di server Zimbra.

Di jendela berjudul Destination Domain, Anda harus memasukkan domain di server dengan Zimbra, ke akun mana IBM Notes / Domino akan ditransfer.

Di jendela Informasi Server LDAP , kami harus memasukkan informasi yang diperlukan untuk terhubung ke server LDAP:

  • LDAP Server - alamat server LDAP internal atau eksternal
  • Port LDAP - port akses LDAP (389 secara default)
  • DN dan kata sandi pengguna - Masukkan Nama Khusus dan kata sandi pengguna untuk mengakses LDAP
  • BasDN - Masukkan Nama Dibedakan Dasar untuk mencari

Informasi di jendela Impor Sumber biasanya diisi secara independen jika Wisaya Migrasi ZCS untuk Lotus Domino dapat mendeteksi server IBM Notes / Domino. Jika ini tidak terjadi, Anda harus memasukkan:

  • Nama Server Domino - nama domain server dengan IBM Notes / Domino
  • Jalur File ID Admin - jalur ke file ID dengan data administrator
  • Catatan INI File Path - path ke file Notes INI
  • Migrasikan Folder Pribadi - Centang kotak ini untuk mentransfer folder pribadi pengguna ke server Zimbra. Untuk melakukan ini, Anda harus memberi tahu program jalur ke file ID pengguna yang akunnya ingin Anda transfer, serta jalur ke file dengan kata sandi mereka

Setelah itu, di jendela Source and Destination Mailboxes, kita harus memilih kotak surat yang akan ditransfer ke server dari Zimbra. Jendela ini memiliki beberapa tombol yang membuka berbagai jendela. Jadi, misalnya, di jendela Object Picker Anda dapat menemukan dan memilih akun yang akan dimigrasi ke server baru. Untuk melakukan ini, Anda harus mulai memasukkan nama kotak surat, dan fungsi pelengkapan otomatis itu sendiri akan menawarkan Anda opsi yang sesuai. Di jendela Pembuat Kueri , Anda harus membuat kueri untuk mencari oleh pengguna di basis data LDAP.

Setelah itu, akun yang dipilih sebelumnya akan muncul di jendela Kotak Surat Sumber dan Tujuan yang sebelumnya kosong, yang akan ditransfer ke server Zimbra. Jika Anda berubah pikiran tentang memindahkan kotak surat, Anda dapat menghapusnya dari daftar. Jika daftar dikompilasi dengan benar, maka Anda dapat melanjutkan.

Pada tahap Verifikasi Akun Target , Wizard Migrasi ZCS untuk Lotus Domino akan memeriksa apakah ada akun di server Zimbra dengan nama akun yang identik dengan yang bermigrasi dari server Notes / Domino IBM. Jika tidak ada akun dengan nama identik, maka akan dibuat.

Dalam kotak dialog Penyediaan Akun , administrator harus mengonfigurasi akun yang dibuat setelah migrasi di server Zimbra. Secara khusus, Anda harus memasukkan kata sandi yang akan digunakan oleh pengguna tersebut secara default, dan juga menentukan grup pengguna yang dimiliki oleh akun yang dimigrasi.

Setelah semua akun dibuat, Anda dapat mulai mengimpor konten akun. Untuk melakukan ini, di kotak dialog Impor Item , pilih komponen yang harus ditransfer:

  • Impor Surat - mentransfer korespondensi surat
  • Impor Kontak - Transfer Kontak
  • Impor Tugas - transfer tugas
  • Impor Kalender - mentransfer data kalender

Di jendela berikutnya, disebut Opsi Impor, Anda akan diminta untuk memilih beberapa opsi:

  • Impor folder Email Sampah - Apakah akan mentransfer folder Spam
  • Impor folder Item Terhapus - Apakah akan memindahkan "Keranjang Sampah" dengan item yang dihapus
  • Impor pesan email yang diterima setelah - Tunjukkan tanggal dari mana pesan harus ditransfer
  • Jumlah kotak surat yang akan diimpor secara bersamaan - Menentukan jumlah kotak surat yang diimpor secara bersamaan

Setelah berhasil menyelesaikan satu migrasi, Anda akan diminta untuk melakukan yang lain. Jika Anda memanfaatkan saran kami dan membagi migrasi menjadi beberapa bagian, Anda dapat beristirahat sejenak untuk mengubah entri dalam relai email sehingga surat-surat yang ditujukan kepada pengguna yang baru dimigrasi tiba di server Zimbra.

Perhatikan bahwa jika Anda dihadapkan dengan tugas hanya mentransfer isi kotak surat dari IBM Notes / Domino ke Zimbra, Anda dapat menggunakan migrasi menggunakan utilitas imapsync . Proses migrasi tersebut dijelaskan dalam artikel sebelumnya .
