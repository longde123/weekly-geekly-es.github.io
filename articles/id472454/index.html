Tinjauan server VPS murah

Alih-alih kata pengantar


atau bagaimana hal itu terjadi, artikel ini muncul


yang menjelaskan mengapa dan mengapa pengujian ini dilakukan

Berguna untuk memiliki server VPS kecil di tempat yang nyaman untuk menguji beberapa hal. Biasanya diperlukan bahwa itu juga tersedia sepanjang waktu. Ini membutuhkan kelancaran pengoperasian peralatan dan alamat IP putih. Di rumah, kadang-kadang cukup sulit untuk menyediakan kedua kondisi ini. Dan mengingat bahwa biaya menyewa server virtual sederhana sebanding dengan biaya layanan penerbitan alamat IP khusus oleh penyedia Internet, menyewa server semacam itu mungkin dapat membenarkan biaya. Tetapi bagaimana cara memilih dari siapa untuk memesan VPS seperti itu? Ada sedikit kepercayaan pada ulasan tentang berbagai sumber. Oleh karena itu, muncul ide untuk memilih penyedia layanan terbaik dengan kriteria sederhana - dengan kinerja server sewaan.

Pilihan peserta


Saya tidak mulai membaca ulasan dan ulasan sama sekali untuk mencari tahu dalam praktik pribadi layanan yang ditawarkannya. Ternyata, ada layanan pemilihan server virtual, misalnya:

  • poiskvps.ru
  • vds.menu
  • vps.today
  • hosting101.ru
  • hostings.info
  • hosters.ru
  • hostadvice.com

Setiap layanan tersebut menawarkan untuk memasang filter yang diperlukan (misalnya, jumlah RAM, jumlah core dan frekuensi prosesor, dll.) Dan mengurutkan hasilnya berdasarkan beberapa parameter (misalnya, berdasarkan harga). Diputuskan untuk membagi peserta menjadi dua kelompok: kelompok pertama akan menyertakan penawaran dengan hard drive, dan yang kedua - pada memori flash. Jelas bahwa ada lebih banyak jenis drive dan kinerja kecepatan drive dengan antarmuka SAS akan berbeda dari drive dengan antarmuka SATA, dan kinerja NVD SSD akan berbeda dari SSD lainnya. Tapi kemudian, pertama, kita akan memiliki terlalu banyak kelompok, dan kedua, kinerja HDD dari SSD umumnya lebih berbeda daripada kinerja HDD yang berbeda di antara mereka dan SSD yang berbeda di antara mereka.

Metodologi pengujian


Setiap server diuji selama seminggu. CPU, RAM, subsistem disk, dan jaringan mengalami pemuatan. Tes berjalan sesuai jadwal, di-host di cron.

Hasilnya dikumpulkan dan diproses dengan mentabulasikan nilai-nilai dan merencanakan grafik dan (atau) grafik. Alat-alat berikut digunakan.

Tes sintetis:

  • sysbench
  • cpu, tes umum: sysbench --test=cpu run ( nilai : total waktu)
  • memori, tes umum: sysbench --test=memory run (nilai: total waktu)
  • file i / o, tes dan perintah (ukuran blok di semua tes adalah 4k; nilai: kecepatan transfer):

    • Urutan berurutan tunggal dibaca dengan kedalaman antrian simulasi 32: sysbench --num-threads=1 --test=fileio --file-test-mode=seqrd --file-total-size=2G --file-block-size=4K --file-num=32
    • Rekaman berurutan single-threaded dengan simulasi kedalaman antrian 32: sysbench --num-threads=1 --test=fileio --file-test-mode=seqwr --file-total-size=2G --file-block-size=4K --file-num=32
    • Bacaan acak delapan ulir dengan kedalaman antrian simulasi 8: sysbench --num-threads=8 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=8
    • Rekaman acak delapan thread dengan simulasi kedalaman antrian 8: sysbench --num-threads=8 --test=fileio --file-test-mode=rndwr --file-total-size=2G --file-block-size=4K --file-num=8
    • Single threaded random read dengan kedalaman antrian simulasi 32: sysbench --num-threads=1 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=32
    • Rekaman acak single-threaded dengan simulasi kedalaman antrian 32: sysbench --num-threads=1 --test=fileio --file-test-mode=rndwr --file-total-size=2G --file-block-size=4K --file-num=32
    • Single threaded random read dengan kedalaman antrian simulasi 1: sysbench --num-threads=1 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=1
    • Rekaman acak single-threaded dengan simulasi kedalaman antrian 1: sysbench --num-threads=1 --test=fileio --file-test-mode=rndwr --file-total-size=2G --file-block-size=4K --file-num=1
  • hardinfo:

    • CPU Blowfish
    • CPU CryptoHash
    • CPU fibonacci
    • CPU N-Queens
    • FPU FFT
    • FPU Raytracing

Untuk memeriksa kecepatan jaringan, tes speedtest (speedtest-cli) digunakan.

Kesimpulan


Di antara semua peserta tes, tempat pertama diambil oleh tarif dengan SSD dari RUVDS. Performa prosesor yang lebih baik dan kinerja disk yang baik memungkinkan mereka mengambil tempat pertama dalam tarif mereka. Selamat kepada pemenang. Saya juga ingin mencatat adminvps hosting, ionos dan regru, mereka berjuang dengan bermartabat. AdminVPS menunjukkan kinerja disk yang sangat baik, tetapi tertinggal dalam kinerja prosesor.
