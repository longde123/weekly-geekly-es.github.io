Modo de imagen en imagen en Chrome 69


En la versión 69, apareció la extensión PictureInPicture, que le permite mostrar videos en la parte superior de todas las ventanas. Decidí probar esta función y compartir los resultados.

Inmediatamente encontré un ejemplo, pero el botón de imagen en imagen para cambiar al modo de imagen en imagen (en adelante PiP) no estaba disponible, fui a verificar el indicador de cromo: // flags / # enable-picture-in-picture , el valor es "Predeterminado", para esta bandera es equivalente a "Habilitado", pero aun así trató de habilitarlo manualmente.



No sirvió de nada, resultó que necesitábamos otra bandera de Chrome: // flags / # enable-Surface-for-videos , el valor "Predeterminado" es equivalente a "Disabled".



Encendido y ganado. También escriben que es necesario habilitar la bandera # enable-experimental-web-platform-features, pero funcionó sin mí.



Ejemplo


Hagamos un ejemplo simple, la etiqueta <video />, un botón para activar / desactivar el modo de imágenes incrustadas, que procesa los eventos de imágenes incrustadas disponibles. 

<html> <head> <title>Chrome 69</title> </head> <body> <video width="320" height="240" controls> <source src="https://www.w3schools.com/html/mov_bbb.mp4" type="video/mp4"> </video> <br/> <button> PiP</button> <script> const video = document.querySelector('video'); const button = document.querySelector('button'); if (!document.pictureInPictureEnabled) { button.textContent = ' PiP  .'; button.disabled = true; } button.addEventListener('click', () => { if (document.pictureInPictureElement) { document.exitPictureInPicture() .then(() => { /**/ }) .catch(() => { /**/ }); } else { video.requestPictureInPicture() .then(() => { /**/ }) .catch(() => { /**/ }); } }); video.addEventListener('enterpictureinpicture', () => { button.textContent = ' PiP'; }); video.addEventListener('leavepictureinpicture', () => { button.textContent = ' PiP'; }); </script> </body> </html>

Verifique la disponibilidad del modo: 

 if (document.pictureInPictureEnabled){ // PiP  }

Es simple, si la propiedad existe, devuelve verdadero o falso.

Hacemos una solicitud para iniciar el modo: 
 button.addEventListener('click', () => { video.requestPictureInPicture() .then(() => { /**/ }) .catch(() => { /**/ }); });

Si se obtiene el permiso, ejecute lo contrario catch. La solicitud definitivamente será rechazada si no es activada por un evento de usuario.

Después de iniciar document.pictureInPictureElement devolverá nuestro elemento <video />.

Para salir del modo, puede hacer clic en la cruz en la esquina superior derecha de la ventana de imágenes incrustadas.

Puede hacer una salida mediante programación: 

 button.addEventListener('click', () => { if (document.pictureInPictureElement) { document.exitPictureInPicture() .then(() => { /**/ }) .catch(() => { /**/ }); } });

Después de salir de document.pictureInPictureElement, volverá a ser nulo nuevamente.

Eventos de imágenes incrustadas


Hay dos eventos disponibles, modo PiP activado y desactivado, respectivamente: 

 video.addEventListener('enterpictureinpicture', () => { //  button.textContent = ' PiP'; }); video.addEventListener('leavepictureinpicture', () => { //  button.textContent = ' PiP'; });

Resumen


Por el momento, este modo solo funciona con la etiqueta <video />, pero puede expandirse.
La ventana de imágenes incrustadas aparece en la esquina inferior derecha, hasta ahora no podemos afectarla.
El modo PiP es incompatible con el modo de pantalla completa, o uno u otro, lo cual es bastante lógico.
Solo puede haber una ventana de imágenes incrustadas a la vez.
La pestaña en la que el usuario ha habilitado el modo PiP se indica mediante el icono (visible en el KDPV).

UPD : La nueva API es la división W3C llamada WICG (Web Platform Incubator Community Group).
Con esta API, los sitios pueden controlar cuándo abrir y cerrar la ventana de imágenes incrustadas, crear su propio botón de administración y recopilar estadísticas cuando el usuario usa imágenes incrustadas.

Algunos navegadores ya admiten esta función, pero no se basan en W3C
(Por ejemplo, Opera y Yandex.Browser).
Firefox y Edge aún no han anunciado la implementación de la nueva API.
