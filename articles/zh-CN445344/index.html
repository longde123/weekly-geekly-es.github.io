OpenVox统一通信平台


您说的标题多么大声。 星号上新的PBX制造商？ 并非如此，但是设备非常新鲜有趣。


今天，我想向您介绍Openvox统一通信系统，制造商对于结合这些非常好的通信似乎有自己的见解：）


设备制造商OpenVox正在缓慢但肯定地朝着完全模块化的结构发展。 首先，他制造了GSM设备，可以在其中使用不同的模块组合及其编号，然后出现模拟网关，最后，引入了一个支持几乎所有必需的电话连接标准的新平台：FXO / FXS / E1 PRI / BRI / GSM / 3G / LTE


任何有兴趣的人请削减


因此，有一个机箱-高2个单元，尺寸为43厘米x 33厘米x 8.8厘米，它有11个插槽用于安装其他模块，每个插槽用于一个模块。 插槽编号直接显示在前面板上。


当前存在哪些类型的模块？


E1接口


Openvox ET200X模块允许您连接1到4个E1数字流。 另外，它可以配备一个Octasic板，用于消除硬件回声。

这些模块具有1个10/100 Mbps网络端口和一个用于灾难恢复软件的USB端口，以及用于指示连接状态的LED。 支持PRI / SS7 / R2协议，也提供具有更详细技术描述的数据表 。 正如Openvox的最佳传统一样，内部当然还有Asterisk。


模拟接口


制造商发布了用于连接模拟线路的3种模块变体。
8 FXO上的VS-AGU-E1M820-O用于连接外部线路。


8 FXS上的VS-AGU-E1M820-S，用于连接内部电话，传真机或廉价的DECT基站。



并在4条FXO和4条FXS线上混合VS-AGU-E1M820-OS


GSM接口


支持当前支持的GSM / 3G / LTE模块：分别为VS-GWM420G / VS-GWM420GW-E和VS-GWM420L-E。


我在上一篇文章中更详细地研究了它们。


带有Intel Celeron处理器VS-CCU-N2930AM的模块



是的是的 这是一款成熟的64位计算机，基于具有4核和最高2.16 GHz频率的Celeron N2930处理器。 默认情况下，SO-DIMM内存条设置为2 GB，但是您可以将DDR3L 1333扩展到8 GB。
板上焊接了容量为16 GB的SSD驱动器。 有两个网络接口可用，一个在10/100 / 1000Mb，另一个在10 / 100Mb。 一个用于外部监视器的VGA输出，和两个USB接口，例如，用于卸载备份或存储对话。

因此，我们逐渐采用了已安装的软件。
与该机箱中的其他任何模块（3G / FXO / FXS / E1）一样，该模块是完全自主的。 分别下载，更新并具有单独的IP地址。 对于VS-CCU-N2930AM，甚至是单独的网络接口。


Openvox促进了Issabel的开放式统一通信系统 ，该系统是Elastix项目的分支。 我认为我不需要对Issabel进行评论，因为实际上它与著名的Elastix几乎没有什么不同。


让我提醒您一些不熟悉开放电话软件的人：


  • 无限SIP订户
  • 无限数量的外部SIP中继
  • 通过API（AMI / AGI / ARI）与外部系统集成
  • 无需软件费用和进一步支持
  • 安装需要直臂

结论


该系统结构紧凑，高效节能，适合希望将设备全部集成在一起的中型和大型企业。 目前，所有这些模块的自动配置都不够，也就是说，他们自己的PBX软件严重短缺。
