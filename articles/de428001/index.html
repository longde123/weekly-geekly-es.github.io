Bekannte Probleme, Fehler und Korrekturen in Windows 10-Updates für Oktober 2018



Fast zwei Wochen sind vergangen, seit Microsoft im Rahmen des Patch Tuesday vom Oktober 2018 Updates für Windows 10 veröffentlicht hat - und es sind lange zwei Wochen vergangen. Dieser Satz von Patches kann als der schlechteste Dienstag von Patches von Microsoft seit langer Zeit bezeichnet werden, da viele Fehler gefunden wurden, die nicht nur Windows 10-PCs unbrauchbar machen, sondern (in einigen Fällen) auch zu Dateiverlusten durch Benutzer führen.

Alles war so traurig, dass Microsoft ab Oktober 2018 Updates für Windows 10 zurückrufen musste, um herauszufinden, was genau die Probleme verursacht hat, es zu beheben und dann den aktualisierten Satz mit Windows Insidern zu überprüfen.

In diesem Artikel werden die schlimmsten Fehler im Zusammenhang mit diesen Updates und Lösungen zur Behebung dieser Probleme beschrieben.

Oktober 2018 Update löscht einige Benutzerdateien


Der verheerendste Fehler in diesem Service Pack ist ein Fehler, der Benutzerdaten während des Aktualisierungsprozesses löscht. Solche Benutzer haben keine andere Wahl, als Daten aus Sicherungen, Schattenkopien, durch Systemwiederherstellung oder durch Dateiwiederherstellungssoftware wiederherzustellen.

Es kam zu dem Punkt, dass Microsoft empfahl, keine Computer zu verwenden, die unter diesem Fehler litten, damit die Dateiwiederherstellungssoftware eine bessere Chance hatte, sie wiederherzustellen.

Eine Woche später gab Microsoft bekannt, dass das Löschen von Dateien darauf zurückzuführen ist, dass die Einstellung "Umleitung bekannter Ordner" [Umleitung bekannter Ordner] auf den Computern der Benutzer aktiviert wurde.

"Vor der erneuten Veröffentlichung der Oktober-Updates ergab unsere Untersuchung, dass eine sehr kleine Anzahl von Benutzern ihre Dateien während des Oktober-Paket-Updates verloren hat", heißt es in der Ankündigung von Microsoft. - Dies geschah, wenn die KFR-Funktion (Known Folder Redirection) zuvor aktiviert war und der Benutzer die Dateien im ursprünglichen „alten“ Ordner belassen hat, anstatt sie in den neuen Ordner zu verschieben, der umgeleitet wird. Bei KFR werden bekannte Windows-Ordner umgeleitet, einschließlich Desktop, Dokumente, Bilder, Screenshots, Videos, Kamerarollen usw. von ihrem Standardspeicherort c: \ users \ username \ zu einem neuen Speicherort. Als Teil des Feedbacks des vorherigen Service Packs vom April meldeten Benutzer das Erscheinen einer zusätzlichen, leeren Kopie der „bekannten Ordner“ auf dem Gerät “, erklärt John Cable, Direktor für Programmverwaltung, Windows-Dienste und -Lieferung. „Basierend auf dem Feedback der Benutzer haben wir im Oktober Updates eingeführt, mit denen diese leeren Kopien bekannter Ordner entfernt werden. Diese Änderung führte zusammen mit einer weiteren Änderung, die die Reihenfolge der Anwendung von Aktualisierungen aktualisierte, dazu, dass die ursprünglichen „alten“ Ordner zusammen mit ihrem Inhalt gelöscht wurden und nur „neue“ unberührt blieben.

Leider können Benutzer, die keine Sicherungs- oder Schattenkopien zum Wiederherstellen von Dateien hatten, diese nur mit einer speziellen Software zum Wiederherstellen gelöschter Dateien wie Recuva wiederherstellen.

Inkompatibler Tastaturtreiber ließ HP Computer fallen


Nach der Installation des neuesten Service Packs entdeckten Benutzer von HP Computern mehrere Betriebssystemfehler in Form eines „Blue Screen of Death“, auf dem der Fehler WDF_VIOLATION angezeigt wurde. Diese Fehler wurden durch einen inkompatiblen HP Tastaturtreiber (Version 11.0.3.1) verursacht, der mit Updates installiert wurde



Laut einem Begleitartikel hat Microsoft den Treiber erkannt, der das Problem verursacht hat, und es von Windows Update deinstalliert. Für Benutzer, die den Treiber installiert und den Computer noch nicht neu gestartet haben, hat Microsoft ein weiteres Update veröffentlicht, mit dem der inkompatible Treiber entfernt wird.

»In Zusammenarbeit mit HP identifizierte Microsoft einen Treiber von HP mit bekannten Inkompatibilitäten mit bestimmten HP Geräten, wenn Windows 10-Versionen 1803 und 1809 ausgeführt wurden. Am 11. Oktober entfernte Microsoft den Treiber aus dem Update, um die Anzahl der betroffenen Geräte zu verringern. Wir haben auch das Update KB 4468304 veröffentlicht, mit dem ein inkompatibler Treiber von Geräten entfernt wird, für die ein Neustart erforderlich ist. HP arbeitet aktiv an diesem Problem. "

Benutzer, die den Treiber installiert, den Computer neu gestartet und mit dem Absturz des Betriebssystems konfrontiert sind, können die Wiederherstellungsanweisungen verwenden .

Audioprobleme nach der Installation von Updates


Für eine Weile schickte Microsoft Benutzern versehentlich einen Intel-Audiotreiber, was dazu führte, dass das Audio auf ihren Computern in den Versionen von Windows 10 1803 und 1809 nicht mehr funktionierte. Die Benutzer, die unter dem Treiber litten, hatten unter Windows 10 einen Tonmangel, und das Betriebssystem gab die Meldung „Kein Audio Ausgabegerät ist installiert “[fehlendes Audioausgabegerät].



Microsoft hat die Verteilung dieses Treibers bereits eingestellt. Wenn Sie jedoch darunter gelitten haben, können Sie die Wiederherstellungsanweisungen verwenden .

Probleme mit der Bildschirmhelligkeit


Nach der Installation der Updates beschweren sich einige Benutzer über die sehr geringe Helligkeit des Displays nach jedem Neustart des Computers. Sie müssen den Pegel manuell wiederherstellen. Möglicherweise kann dies nach der Installation des aktualisierten Grafikkartentreibers behoben werden, aber die meisten Leute mit diesem Problem sagen, dass es nicht hilft.

Leider sind diesem Problem derzeit keine Lösungen bekannt.

Microsoft Edge- und UWP-Anwendungen können keine Verbindung zum Internet herstellen

Nach der Veröffentlichung von Windows 10 1809 erfordern Microsoft Edge- und Microsoft Store-Anwendungen die Aufnahme von TCP / IPv6-Unterstützung, ohne die sie nicht online gehen können. Um den Support zu aktivieren, müssen Sie:
  • Einstellungen öffnen.
  • Wählen Sie "Internet und Netzwerke".
  • Wählen Sie "Adaptereinstellungen ändern".
  • Klicken Sie mit der rechten Maustaste auf den Adapter, mit dem auf das Internet zugegriffen wird, und wählen Sie "Eigenschaften".
  • Stellen Sie sicher, dass das Kontrollkästchen neben Internet Protocol Version 6 (TCP / IPv6) aktiviert ist.
  • Klicken Sie auf OK, um die Eigenschaften zu beenden.


Danach können Sie überprüfen, ob Edge und Anwendungen online gehen. Wenn dies nicht hilft, können Sie die Registrierung bearbeiten. Führen Sie dazu regedit.exe aus und wechseln Sie zum Zweig

"HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ Tcpip6 \ Parameters"
Dort müssen Sie die Variable DisabledComponents suchen und ihren Wert auf 0 setzen.



Ihre Warnung erforderlich Warnung in Bezug auf Intel Audio-Treiber


Wenn Sie versuchen, die neuesten Updates zu installieren, wird die Benachrichtigung "Ihre Aufmerksamkeit ist erforderlich" angezeigt, unter der der Treiber "Intel Audio Display (intcdaud.sys), KB4465877" angezeigt wird. Sie können das neueste Windows 10-Update nicht installieren.



Diese Meldung wird angezeigt, weil Microsoft Inkompatibilitäten mit Intel-Treibern festgestellt hat, die Probleme mit dem Oktober-Update verursachen können. Microsoft sagt, wenn Sie diese Meldung sehen, müssen Sie "Zurück" drücken und warten, bis die aktualisierten Treiber herauskommen.

Fehler beim Entpacken der Zip-Archive


Ein weiterer Fehler beim Upgrade auf Build 1809 führt dazu, dass dem Benutzer beim Extrahieren von Dateien aus Zip-Archiven mithilfe der integrierten Windows-Funktionalität keine Bestätigung zum Überschreiben von Dateien von oben angezeigt wird . Dieser Fehler verhält sich inkonsistent, da bei einigen Benutzern die Dateien ohne Bestätigung überschrieben wurden und bei einigen überhaupt nichts passiert ist. Dieser Fehler wurde im neuesten Windows 10 19H1-Update behoben.
