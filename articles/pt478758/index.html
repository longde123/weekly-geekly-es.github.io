Ótimo guia de tags UTM: como descobrir de onde os usuários vêm


Os métodos de marcação de link foram inventados há muito tempo. Mas, como sempre, diferentes padrões foram misturados e escolher o formato certo não é tão simples. Deixaremos de lado os casos em que o proprietário do site fizer objeção a links marcados (e esse não é um caso tão raro). Vamos entender em detalhes o que e quando colocar as tags UTM e o que fazer com elas a seguir.


Os sistemas de análise (Yandex.Metrica e Google Analytics) determinam facilmente de onde vem o tráfego: de pesquisas, redes sociais, listas de discussão ou por link direto. Mas existem nuances. Por exemplo, se o tráfego vier de um site externo, você aprenderá sobre ele, mas sem detalhes: de qual página ou por qual link - ele pode permanecer um mistério.


A boa notícia é que existe um sistema universal de rastreamento de tráfego - tags UTM. Se o link patrocinado for marcado usando UTM, você poderá acompanhar facilmente as fontes de referência até um link específico na página, anúncio ou palavra-chave.


Explicamos o que são as tags UTM, como compor para diferentes tipos de publicidade, como gerá-las, como e onde rastrear, quais são os problemas e como evitá-los. Dentro há muitos exemplos. Será útil para iniciantes e profissionais de marketing com experiência.

Tags UTM: o que é e por que são necessárias


As tags UTM são parâmetros adicionados aos URLs para obter informações detalhadas sobre o tráfego.


Um exemplo Você deseja colocar dois links de publicidade para sua loja no mesmo site sobre atualizações de moda. O primeiro link é transversal na barra lateral, o segundo é nativo no artigo sobre moda. Nos dois casos, o link leva ao diretório: https://clothingshop.ru/catalog .


Qual é a desvantagem de simplesmente vincular?


No sistema de análise, você verá o número total de visitas, mas não saberá em qual link os visitantes clicaram. Portanto, você não conseguirá entender qual abordagem de publicidade nesse recurso é mais lucrativa.


E se você adicionasse tags UTM ao URL, definitivamente rastrearia o tráfego de cada link e avaliaria a eficiência do posicionamento.


A seguir, descreveremos em detalhes como fazer isso.


O que é UTM


UTM significa Módulo de rastreamento do Urchin. "Urchin" é o nome da empresa que criou e começou a usar essas tags. A propósito, o Urchin adquiriu o Google em 2005 e criou, com base em seu sistema de análise - o Google Analytics.


As etiquetas UTM provaram ser uma ferramenta conveniente e versátil. Portanto, as tags começaram a ser usadas no Yandex.Metrica e em outros sistemas de análise.


Como são as tags UTM


Um URL comum se parece com isso: 


 https://bestbuyshop.ru/catalog

Esse link é compreensível e familiar para todos. Mas vamos adicionar tags UTM a ele e ver como ficará agora: 


 https://bestbuyshop.ru/catalog?utm_source=facebook&utm_medium=cpc&utm_campaign={campaign_id}&utm_term={keyword}

O URL tornou-se pesado e menos compreensível do que era originalmente. Para facilitar a percepção, dividiremos em duas partes:


  • a primeira parte é diretamente o endereço da página para a qual você precisa ir;
  • a segunda parte (do sinal?) são as mesmas tags UTM.

Certamente você já encontrou esse tipo de link mais de uma vez ao clicar em anúncios:



Se você não conhece as tags UTM, esse código pode parecer chocante. Mas nem tudo é tão complicado quanto parece.


Em que consiste as tags UTM


Tags UTM são parâmetros GET (transmitidos ao servidor usando um link).


O parâmetro GET é muito simples de determinar: se você vir um ponto de interrogação na URL, tudo o que estiver localizado após esse sinal será parâmetros GET.


Um único URL pode ter vários parâmetros GET. Aqui estão as regras básicas de sintaxe:


  • cada parâmetro GET consiste em dois elementos - uma chave e um valor. Entre esses elementos há um sinal de igual (chave = valor);
  • Os parâmetros GET são separados pelo sinal & (e comercial).

Vejamos novamente o exemplo de um link de tag UTM que fornecemos acima: 


 https://bestbuyshop.ru/catalog?utm_source=facebook&utm_medium=cpc&utm_campaign={campaign_id}&utm_term={keyword}

Tudo isso após o sinal "?" São os parâmetros GET:


Parâmetro Valor
utm_source facebook
utm_medium cpc
utm_campaign {campaign_id}
utm_term {keyword}

Quantas tags UTM existem


Existem 5 tags UTM universais que você pode usar para marcar todos os links. Essas tags são reconhecidas pelo Google Analytics e Yandex.Metrica.


Três das cinco tags são necessárias, elas sempre devem ser colocadas no URL:


  • utm_source - indica a origem do tráfego. Por exemplo, o site no qual você postou um artigo de convidado com um link para seu recurso. Ou uma plataforma de publicidade (por exemplo, o Facebook), na qual você coloca anúncios com um link para acessar seu site.
  • utm_medium - determina o tipo de tráfego. Por exemplo, ao colocar anúncios usando o modelo de pagamento por clique, esse parâmetro geralmente define o valor como "cpc" (custo por clique). Para um banner de publicidade - "banner", para um link em um boletim informativo - "email".
  • utm_campaign - esse parâmetro é necessário para indicar a qual campanha publicitária o link se refere. Em alguns casos, valores dinâmicos podem ser usados ​​neste parâmetro. Por exemplo, se você colocar anúncios direcionados no Facebook, poderá especificar o parâmetro dinâmico {campaign_id} e o identificador da sua campanha de publicidade será inserido automaticamente no link.

Duas tags podem ser usadas opcionalmente:


  • utm_term - esse rótulo é necessário para determinar a palavra-chave pela qual seu anúncio foi exibido. A tag é opcional, mas você deve usá-la para marcar links nas campanhas de pesquisa Yandex.Direct ou Google Ads. Para fazer isso, use o parâmetro dinâmico {keyword}. O parâmetro utm_term também é adequado para especificar outras informações (por exemplo, a data em que o link do anúncio foi publicado).
  • utm_content - ajuda a distinguir declarações se outros parâmetros forem idênticos. Por exemplo, se você usar dois links patrocinados no mesmo artigo, use a tag utm_content com valores diferentes para cada link para rastrear cliques em cada um deles.

As tags listadas são usadas com mais frequência. Eles são suficientes para obter estatísticas de tráfego detalhadas. Há também tags adicionais:


  • utm_nooverride - usando esse rótulo, você pode determinar onde ocorreu o primeiro contato com o cliente ao rastrear conversões associadas. Por exemplo, um usuário clicou em um anúncio de publicidade direcionada, foi ao site e se inscreveu no boletim. Então ele seguiu o link da carta de confirmação e se registrou no site. Sob condições normais, o sistema de análise contará o email como a fonte, embora na verdade o usuário tenha trazido anúncios segmentados. Para identificar corretamente a fonte, adicione o rótulo utm_nooverride com o valor "1" no link no site. Portanto, o sistema de análise ignorará o clique nesse link e o anterior será levado em consideração. Preste atenção! O rótulo utm_nooverride é entendido apenas pelo Google Analytics.
  • utm_referrer - ajuda a rastrear e levar em conta as transições corretamente na presença de um redirecionamento Javascript. Este rótulo é compreensível apenas para o Yandex.Metrica.

A recomendação . Se você exibir anúncios para si mesmo ou trabalhar em um projeto sozinho, poderá nomear os valores dos parâmetros para as tags UTM como desejar. O principal é que você entenda o que cada valor significa.


Se seus funcionários ou colegas precisarem acessar a análise, use nomes claros e padronizados para que todos compreendam claramente o que, por exemplo, significa: utm_campaign=derevyannie_stoly23_test_fb_wide .


Por conveniência, crie um documento no Planilhas Google (ou Google Docs) e anote as regras para marcar links com tags UTM. Compartilhe o acesso ao arquivo com colegas e todos entenderão.


Sintaxe da tag UTM


Vamos compor etiquetas UTM para a campanha publicitária da loja online de roupas masculinas e, ao longo do caminho, explicaremos o que e como fazer.


Aqui estão os detalhes da campanha:


  • página de destino - https://brandclothes.ru/winter_sale ;
  • exibir anúncios na pesquisa do Google (sistema de anúncios - Google Ads);
  • o nome da campanha é "Venda de casaco de inverno";
  • crie 2 anúncios para a campanha;
  • keywords: “casacos masculinos de inverno”, “venda de casacos masculinos”.

Passo a passo, faremos a marcação UTM.


Primeiro de tudo, você precisa atribuir um valor a cada rótulo. Para fazer isso, escreva o nome do próprio rótulo, um sinal de igual ("=") e o valor do parâmetro.



Existem dois pontos importantes aqui:


  1. Começamos com as tags necessárias.
  2. Os rótulos obrigatórios são criados com base no princípio “do geral ao particular” - primeiro, o parâmetro de nível mais alto (fonte), depois o tipo de publicidade, campanha, anúncio e palavra-chave.

Atribua valores aos rótulos:


  • utm_source = google - aqui podemos especificar qualquer outro valor, o principal é deixar claro depois que o tráfego veio da pesquisa no Google. Por exemplo, você pode especificar o valor google-search. Observe que, se você usar duas ou mais palavras no valor do parâmetro, conecte-as com um hífen ou sublinhado. Se você deixar um espaço entre as palavras, o link estará incorreto.
  • utm_medium = pesquisa - usando esse valor, indicamos que tipo de publicidade é usada na campanha (pesquisa).
  • utm_campaign = muzhskie-palto é o nome da campanha que especificamos no Google Ads.
  • utm_content = ad1 - com esse parâmetro, podemos rastrear qual dos dois anúncios é mais eficaz. Os rótulos anteriores serão os mesmos para os dois anúncios e, no rótulo utm_content, apontamos para um anúncio específico. Assim, para a segunda declaração, haverá um valor utm_content = ad2).
  • utm_term = zimnie-muzhskie-palto - especifique a palavra-chave. Observe que é melhor usar o alfabeto latino para a operação correta das tags UTM. Para fazer isso, escreva palavras em russo na transliteração (manualmente ou usando serviços especiais).

Estamos prontos para todos os elementos nos quais o link marcado será composto. Agora vamos compor em etapas.


1. Especifique o URL da página de destino (para a qual direcionaremos tráfego): 


 https://brandclothes.ru/winter_sale

2. No final do URL, adicione um ponto de interrogação: 


 https://brandclothes.ru/winter_sale?

3. Coloque a primeira etiqueta UTM: 


 https://brandclothes.ru/winter_sale?utm_source=google

4. Lembre-se de que os parâmetros GET são conectados por um e comercial. Colocamos um e comercial: 


 https://brandclothes.ru/winter_sale?utm_source=google&

5. Avançar - coloque o segundo rótulo: 


 https://brandclothes.ru/winter_sale?utm_source=google&utm_medium=search

6. Novamente, colocamos um e comercial, depois a terceira marca e assim por diante. Após o último rótulo, um e comercial não é necessário.


Para o primeiro anúncio, obtivemos o seguinte link: 


 https://brandclothes.ru/winter_sale?utm_source=google&utm_medium=search&utm_campaign=muzhskie-palto&utm_content=ad1&utm_term=zimnie-muzhskie-palto

O link para o segundo anúncio parecerá quase idêntico. Somente os valores dos dois últimos parâmetros são diferentes - utm_term e utm_content: 


 https://brandclothes.ru/winter_sale?utm_source=google&utm_medium=search&utm_campaign=muzhskie-palto&utm_content=ad2&utm_term=muzhskie-palto-rasprodazha

Preste atenção! Se você exibir anúncios apenas no Google e usar apenas o Google Analytics, poderá colocar tags em qualquer ordem. O Google os reconhece independentemente do rótulo que você possui no início e no final.


Para trabalhar com o Yandex.Direct e o Yandex.Metrica, é recomendável colocar as tags sequencialmente: especifique utm_source primeiro, depois utm_medium e assim por diante (como no nosso exemplo). Para não ficar confuso, siga esta regra ao marcar links com tags UTM para qualquer sistema de publicidade.


Quais são as formas de rastrear tráfego além do UTM


Não apenas as tags UTM são usadas para rastrear os cliques.


gclid


Marque links automaticamente nos anúncios do Google Ads. Portanto, a marcação aparece no link:



Nuances Gclid:


  • Essa marcação é usada apenas no Google Ads. O sistema gera automaticamente um identificador para rastrear links de clique. Se você colocar anúncios também no Yandex.Direct e em outros canais, ainda precisará usar outras tags (por exemplo, o mesmo UTM).
  • As tags gclid são compatíveis apenas com o Google Analytics. Portanto, você não pode coletar rápida e facilmente estatísticas de resumo em um único local sobre a eficácia das campanhas publicitárias em diferentes sistemas.

yclid


Essa marcação é semelhante ao gclid - é usada apenas no Yandex.Direct. É assim que o link com a marcação yclid se parece:



Assim como o gclid, a marcação automática do Yandex.Direct tem as mesmas desvantagens: as tags são compatíveis com o Yandex.Direct e são levadas em consideração pelo Yandex.Metrica. Outros sistemas de análise (como o Google Analytics) não os entendem.


Outros sistemas de anúncios têm rótulos semelhantes:


  • Anúncios no Facebook - fbclid;
  • Yandex.Market - ymclid;
  • etc.

Tudo é o mesmo com eles: apenas o sistema de publicidade "nativo" os entende.


de


Este é o rótulo com o qual você pode determinar a fonte do clique no link de publicidade. Reconhecido pela Yandex.Metrica.


Este é o URL marcado com de: 


 https://bestshop.ru/catalog/?from=ad-platform.

O rótulo from passa apenas um parâmetro (fonte de salto), enquanto a marcação UTM fornece cinco parâmetros.


Parâmetros estáticos e dinâmicos de tags UTM


Você pode usar parâmetros estáticos ou dinâmicos na marcação UTM. Com parâmetros estáticos, tudo é simples: o sistema de análise corrigirá os valores que você especificar manualmente.


As configurações dinâmicas permitem rastrear estatísticas avançadas sobre cliques em anúncios. Por exemplo, adicionando parâmetros dinâmicos ao link do anúncio Yandex.Direct, você pode descobrir:


  • em que posição o anúncio estava quando foi clicado;
  • Para qual consulta de pesquisa o anúncio foi exibido;
  • Em qual região o anúncio foi exibido etc.

Parâmetros dinâmicos no Yandex.Direct


O link no qual todos os parâmetros dinâmicos Yandex.Direct disponíveis são indicados é semelhante a este: 


 http://www.site.ru/?type={source_type}&source={source}&added={addphrases}&block={position_type}&pos={position}&key={keyword}&campaign={campaign_id}&name={campaign_name}&name_lat={campaign_name_lat}&retargeting={retargeting_id}&ad={ad_id}&phrase={phrase_id}&gbid={gbid}&device={device_type}&region={region_id}&region_name={region_name}

Você pode ver a lista completa dos parâmetros disponíveis aqui .



Vamos prestar atenção a vários parâmetros:


  • {ad_id} - ID do anúncio. Se esse parâmetro estiver no link, o sistema de análise receberá dados nos anúncios em que o usuário clicou. Se você executar várias variações de anúncios para a mesma página de destino (para testar e encontrar o anúncio mais eficaz), use um parâmetro dinâmico - isso é mais conveniente do que escrever manualmente um identificador de anúncio para cada link.
  • {campaign_name} é o nome da campanha. Substitui automaticamente o nome da campanha publicitária.
  • {position_type} - determina o tipo de bloco se o anúncio foi exibido na página de resultados de pesquisa Yandex. Podem estar os seguintes valores: premium - exibições premium, outras - o bloco à direita ou o bloco na parte inferior, nenhum - o anúncio é mostrado na YAN ou em redes externas. Use o parâmetro {position} junto com este parâmetro - ele determina a posição exata do anúncio no bloco.

Dimensões dinâmicas no Google Ads


O sistema de publicidade do Google tem seus próprios parâmetros dinâmicos - ValueTrack.


Este é o link com os parâmetros dinâmicos do Google: 


 http://mysite.ru/?utm_source=google&utm_medium=cpc&utm_campaign={network}&utm_content={creative}&utm_term={keyword}

Analisaremos os parâmetros indicados aqui:


  • {network} - define a rede de anunciantes da qual o clique foi recebido. Pelo valor do parâmetro, o sistema de análise identifica com precisão a origem do clique: g - na pesquisa do Google, s - no site do parceiro de pesquisa, d - na Rede de Display (CCM).
  • {creative} - o identificador exclusivo do anúncio.
  • {keyword} - transfere a palavra-chave da conta do Google Ads que corresponde à solicitação do usuário (se o anúncio aparecer na página de resultados de pesquisa). Se o anúncio aparecer na Rede de Display, o parâmetro passará uma palavra-chave que corresponda ao conteúdo do site.

Na Ajuda do Google, você encontrará uma lista completa das opções do ValueTrack.



Como usar tags UTM em diferentes campanhas: exemplos


Tags UTM para SEO


Eles não são necessários aqui. Para analisar o tráfego de pesquisa, basta instalar os códigos do sistema de análise no site e conectar os painéis dos webmasters Yandex e Google. Essas etapas são suficientes para rastrear cliques da pesquisa e entender em que consultas você recebe cliques.


Publicidade contextual


Para campanhas de publicidade contextual, é importante monitorar o tráfego e garantir que você esteja recebendo dados precisos e corretos. Não é necessário usar tags UTM:


  • Se você executar campanhas apenas no Google Ads, precisará usar a marcação gclid "nativa". Isso é ainda melhor - a marcação é definida automaticamente, você não precisa fazer nada manualmente.
  • Se você preferir executar campanhas apenas no Yandex.Direct, a marcação yclid ou Openstat será suficiente.

Recomendamos o uso do UTM se você estiver usando vários sistemas de publicidade ao mesmo tempo. As tags UTM são universais e ajudarão a rastrear o tráfego em um único sistema de coordenadas.


A estrutura das tags UTM para a campanha tem a seguinte aparência (exemplo):


Para Yandex.Direct


Parâmetro Transferir dados Valor do parâmetro
utm_source Sistema de publicidade yandex
utm_medium Canal de tráfego {source_type} *
utm_campaign ID da campanha {campaign_id} *
utm_content ID do anúncio {ad_id} *
utm_term Palavra-chave {keyword} *

* aqui usamos parâmetros dinâmicos para não prescrever parâmetros estáticos para cada anúncio manualmente.


Exemplo de URL com tags UTM para Yandex.Direct: 


 http://bestbuyshop.ru/catalog/?utm_source=yandex&utm_medium={source_type}&utm_campaign={campaign_id}&utm_content={ad_id}&utm_term={keyword}

Para o google


Parâmetro Transferir dados Valor do parâmetro

utm_content ID do anúncio {creative} *
utm_term Palavra-chave {keyword} *

* Utilizamos parâmetros dinâmicos que o Google Analytics “entende”.


Exemplo de URL com tags UTM para anúncios do Google: 


 http://bestbuyshop.ru/catalog/?utm_source=google&utm_medium={network}&utm_campaign={campaignid}&utm_content={creative}&utm_term={keyword}
