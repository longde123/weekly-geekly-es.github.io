Parte 1 do Docker de aprendizagem: o básico

As tecnologias de conteinerização de aplicativos são amplamente usadas no desenvolvimento de software e na análise de dados. Essas tecnologias ajudam a tornar os aplicativos mais seguros, facilitam a implantação e aprimoram sua escalabilidade. O crescimento e o desenvolvimento de tecnologias de contêineres podem ser considerados uma das tendências mais importantes do nosso tempo.

O Docker é uma plataforma projetada para desenvolver, implantar e executar aplicativos em contêineres. A palavra "Docker" recentemente se tornou um sinônimo da palavra "containerization". E se você ainda não está usando o Docker, mas ao mesmo tempo trabalha ou trabalha nas áreas de desenvolvimento de aplicativos ou análise de dados, o Docker é o que você certamente encontrará no futuro.

Se você ainda não sabe o que é o Docker, agora tem a chance de dar o primeiro passo para entender esta plataforma. Ou seja, tendo dominado este material, você entenderá o básico do Docker e simultaneamente preparará a pizza.

Metáforas e Docker


Somos constantemente confrontados com metáforas. Se você olhar para o dicionário de Ozhegov, a metáfora é "uma comparação figurativa oculta, comparando um objeto, um fenômeno a outro". As metáforas nos ajudam a compreender a essência de novos fenômenos para nós. Por exemplo, os contêineres virtuais podem ser comparados aos contêineres de plástico comuns. Essa comparação, comparando as propriedades de contêineres comuns já conhecidos por nós com as propriedades de contêineres virtuais, ajudará você a conhecê-los e entender sua essência.


Container

Como você entende, iniciaremos a conversa sobre o Docker com o conceito de "contêiner".

Container


Como um contêiner plástico comum, um contêiner Docker tem as seguintes características:

  1. Você pode armazenar algo nele. Algo pode estar no contêiner ou no exterior.
  2. Pode ser carregado. Você pode usar o contêiner do Docker no computador local, no computador de um colega, em um servidor de provedor de serviços em nuvem (como a AWS). Isso torna os contêineres do Docker semelhantes aos contêineres comuns, nos quais, por exemplo, eles transportam várias bugigangas em forma de coração quando se mudam para uma nova casa.
  3. É conveniente colocar algo no contêiner e é conveniente tirar algo dele. Um contêiner comum tem uma tampa nas travas, que devem ser removidas para colocar algo no contêiner ou remover algo dele. Os contêineres do Docker têm algo semelhante representando sua interface, ou seja, mecanismos que lhes permitem interagir com o mundo externo. Por exemplo, um contêiner possui portas que podem ser abertas para que um aplicativo em execução no contêiner possa ser acessado a partir de um navegador. Você também pode trabalhar com o contêiner usando ferramentas de linha de comando.
  4. Se você precisar de um contêiner, poderá solicitá-lo na loja online. Um contêiner vazio pode ser comprado, por exemplo, no site da Amazon. Os contêineres chegam a esta loja de fabricantes que os fabricam em grandes quantidades usando moldes. No caso de contêineres do Docker, o que pode ser comparado com o molde, a imagem do contêiner, é armazenado em um repositório especial. Se você precisar de um determinado contêiner, poderá fazer o download da imagem correspondente do repositório e, usando-o, criar esse contêiner.

Obviamente, recipientes de plástico, diferentemente dos recipientes do Docker, ninguém o enviará de graça e, quando você os receber, eles estarão vazios. Mas sempre há algo interessante nos contêineres do Docker.

Organismos vivos


Outra abordagem para pensar sobre os contêineres do Docker é compará-los com instâncias de organismos vivos. Uma "instância" é algo que existe de alguma forma. Este não é apenas código. Este é o código que causou a existência de algo maior que ele, algo, figurativamente falando, vivo. Como outros organismos vivos, instâncias de recipientes nascem, vivem e morrem.


Monstro convocado para a vida

Os contêineres do Docker são imagens animadas do Docker.

De software


Os contêineres do Docker podem ser comparados não apenas aos contêineres comuns ou aos organismos vivos. Eles podem ser comparados com os programas. Afinal, contêineres são programas. E, em um nível fundamental, um contêiner é um conjunto de instruções que são executadas em algum processador, processando algum tipo de dados.


Um contêiner é um programa

Em tempo de execução, o contêiner do Docker geralmente executa algum tipo de programa dentro dele. Ela realiza certas ações no contêiner, ou seja, faz algo útil.

Por exemplo, o código executado em um contêiner do Docker pode ter enviado o texto que você está lendo para o seu computador. É bem possível que o código executado no contêiner do Docker receba os comandos de voz que você fornece ao Amazon Alexa e os converta em instruções para alguns outros programas que funcionam em outros contêineres.

Usando o Docker, você pode executar vários contêineres no mesmo computador ao mesmo tempo. E, como qualquer outro programa, os contêineres do Docker podem ser iniciados, parados, excluídos. Você pode examinar o conteúdo e criá-lo.

Conceitos do Docker


MachinesMáquinas virtuais


Os antecessores dos contêineres do Docker eram máquinas virtuais. Uma máquina virtual, como um contêiner, isola o aplicativo e suas dependências do ambiente externo. No entanto, os contêineres do Docker oferecem vantagens sobre as máquinas virtuais. Assim, eles consomem menos recursos, são muito fáceis de transferir, iniciam mais rapidamente e atingem um estado saudável. Neste artigo, você pode encontrar uma comparação detalhada de contêineres e máquinas virtuais.

▍ Imagem do contêiner do Docker


Já falamos sobre "imagens" acima. O que é isso Boa pergunta O que o Docker chama de termo "imagem" ou, em inglês, "imagem" não é o mesmo que, por exemplo, fotografia (esse é um dos significados da palavra "imagem").


Imagens do Docker não são fotos

As imagens dos contêineres do Docker podem ser comparadas com desenhos, cortadores de biscoitos ou moldes para a fabricação de produtos plásticos. As imagens são modelos imutáveis ​​usados ​​para criar os mesmos contêineres.


Os contêineres do Docker parecem cortadores de biscoito

A imagem do contêiner do Docker contém a imagem do sistema operacional base, o código do aplicativo, a biblioteca da qual depende. Tudo isso é organizado como uma única entidade, com base na qual você pode criar um contêiner.

▍ Dockerfile


O Dockerfile contém um conjunto de instruções, após o qual o Docker monta a imagem do contêiner. Este arquivo contém uma descrição da imagem base, que será a camada da imagem original. Entre as imagens de base oficiais populares, destacam-se python , ubuntu , alpine .

Você pode adicionar camadas adicionais à imagem do contêiner, em cima da imagem base. Isso é feito de acordo com as instruções do Dockerfile . Por exemplo, se o Dockerfile descreve uma imagem que você planeja usar para resolver tarefas de aprendizado de máquina, ele pode conter instruções para incluir as bibliotecas NumPy, Pandas e Scikit-learn na camada intermediária dessa imagem.

E, finalmente, a imagem pode conter, além de todas as outras, outra camada fina, os dados armazenados nos quais podem ser alterados. Essa é uma pequena camada de volume que contém o programa que você planeja executar no contêiner.

Container Container Docker


Para iniciar o contêiner, precisamos primeiro da imagem do contêiner e, em segundo lugar, do ambiente em que o Docker está instalado, capaz de entender um comando do docker run image_name formulários, docker run image_name . Este comando cria um contêiner a partir da imagem e o inicia.

▍ Repositório de contêineres


Se você deseja permitir que outras pessoas criem contêineres com base em sua imagem, envie-a para a nuvem. O maior repositório desse tipo é o repositório do Docker Hub . É usado ao trabalhar com o Docker por padrão.

Já discutimos muitas coisas. É hora de juntar tudo e comparar o trabalho com os contêineres do Docker com a pizza.

Cozinhando com Docker



Pizza pronta é um recipiente

  • Uma receita de pizza é um Dockerfile . Ele nos diz o que precisa ser feito para atingir a meta, ou seja, obter um contêiner de trabalho.
  • Os ingredientes que compõem a pizza são camadas da imagem do recipiente. Para a nossa pizza, você precisará de bolo, molho e queijo.

Se você colocar a receita e os ingredientes em cima da mesa, acontece que em um só lugar tudo o que você precisa é coletado para cozinhar pizza. Tudo isso pode ser comparado com a imagem do contêiner do Docker.
Na receita ( Dockerfile ), você pode descobrir qual sequência de ações você precisa executar para cozinhar pizza:

  • O Korzh está pronto para uso, não o alteramos. Pode ser comparado com a imagem básica do sistema operacional Ubuntu. Esta é a camada inferior da imagem, é adicionada à imagem primeiro.
  • Em seguida, o queijo é adicionado ao bolo. É como adicionar uma segunda camada à imagem como uma biblioteca externa como o NumPy.
  • Em seguida, por cima do queijo, adicione o molho. Este é o código do aplicativo que deve ser executado no contêiner.

Agora é hora de cozinhar pizza no forno.


Oven é uma plataforma Docker

O forno no qual a pizza é cozida lembra a plataforma Docker. O forno está instalado na cozinha, com a sua ajuda você pode cozinhar alimentos. Da mesma forma, o Docker é instalado em um computador para "cozinhar" recipientes.

O forno, se for elétrico, é ligado girando o botão do controlador de temperatura. O docker run image_name é como um controlador de temperatura, cuja “rotação” faz com que o sistema crie e inicie um contêiner.

Pizza pronta - este é o recipiente do Docker.

E comer pizza significa usar o aplicativo em execução no recipiente.

Como fazer pizza, a preparação para o trabalho dos contêineres do Docker leva algum tempo, mas no final e nos dois casos, resulta algo saboroso.
