Gotify: un proyecto de código abierto para enviar notificaciones y enviar mensajes al servidor



Para aquellos que administran servidores y / o proyectos web, existe una aguda cuestión de conciencia sobre lo que está sucediendo con sus "barrios". Hay muchas decisiones en el mercado sobre alertas sobre fallas, estado de conexión y otros parámetros, incluso a través de SMS. Estamos hablando de MRTG, Twilio, F-Droid, servicios de Google y muchos, muchos otros. El único problema es que la mayoría de estas soluciones tienen una especialización muy limitada y no es posible influir en sus funciones. MRTG dará información sobre el tráfico y la conexión, Google enviará SMS. Twilio le da cierta libertad en términos de escribir scripts JavaScript para satisfacer sus necesidades, pero lo siento, este es un proyecto comercial. Es decir, "vamos, paga el botín o vete".

Entonces, en esta familia de herramientas, tenemos una reposición de código abierto: Gotify es un proyecto simple de cliente-servidor para recibir y enviar notificaciones push y comandos, incluso a través de una aplicación de Android. Y definitivamente vale la pena contar un poco más al respecto, creo que será útil para aquellos que estaban buscando algo similar y al mismo tiempo gratis, pero finalmente comenzaron su propio proyecto para mascotas. Es posible que esta publicación le ahorre un par de cientos de horas de su tiempo.

Lo que Gotify ofrece


El proyecto ha estado existiendo y desarrollándose pacíficamente en GitHub durante varios años, y durante este tiempo adquirió un servidor, un cliente web, una aplicación de Android, una API y, por supuesto, documentación. Decidí hablar sobre Gotrify por dos razones: es de código abierto y funciona no solo en "recepción", sino también en "retorno", todo tipo está en GitHub, además está configurado usando scripts bash. Y por supuesto, es gratis.

La mayoría de las soluciones existentes están destinadas exclusivamente a recibir notificaciones del servidor / dispositivo de destino, pero no a una interacción completa entre el cliente y el servidor. Es decir, puede recibir una notificación de que su servidor se ha bloqueado o que Internet ha desaparecido. No siempre es conveniente preguntarse qué está sucediendo en el "otro lado" y se puede hacer en la misma aplicación / ventana. La mayoría de las veces, para obtener este tipo de información, debe cortar una VPN o (Dios no lo quiera) algún TeamViever y verlo "con las manos". Mencioné anteriormente que Twilio da algo de alcance a través de scripts JS auto-escritos, pero es pagado, así que por.

De hecho, todo el proyecto Gotify se divide en tres partes: servidor, cliente y aplicación. El servidor funciona de manera comprensible sobre cómo enviar y recibir mensajes, el cliente solo para recibir y la aplicación solo para enviar.



Además, el proyecto tiene un cliente API (y documentación), una plantilla API, una plantilla de complemento, su propio sitio web, una biblioteca de configuración Go que admite JSON, YAML, TOML y variables de entorno, así como una construcción de servidor, cliente y, de hecho , Aplicación de Android. Lo único que puede molestar es la aplicación de Android. No hay nada más indefenso, irresponsable e inmoral en el mundo que monitorear la infraestructura a través de un teléfono móvil. Pero la tecnología está avanzando y sabemos que tarde o temprano iremos a esta basura. No es que fuera el conjunto mínimo necesario para comunicarse de forma remota con el servidor, pero cuando la comunidad de código abierto comienza a cortar sus propias herramientas, es difícil que se detenga.

Por separado, vale la pena mencionar que para comunicarse con el servidor solo necesita autenticación y el token de la aplicación, que el servidor devuelve a través de una solicitud REST. En el futuro, utilizando este token, puede recibir mensajes a través de cualquier cliente http, por ejemplo, a través de curl o HTTPie : 

$ curl -X POST "https://push.example.de/message?token=<apptoken>" -F "title=my title" -F "message=my message" -F "priority=5" $ http -f POST "https://push.example.de/message?token=<apptoken>" title="my title" message="my message" priority="5"

De acuerdo con el manual en el sitio web oficial del proyecto, se puede enviar al servidor con diferentes prioridades a Golang y Python:

Golang 

 package main import ( "net/http" "net/url" ) func main() { http.PostForm("http://localhost:8008/message?<apptoken>", url.Values{"message": {"My Message"}, "title": {"My Title"}}) }

Pitón 

 import requests #pip install requests resp = requests.post('http://localhost:8008/message?token=<apptoken>', json={ "message": "Well hello there.", "priority": 2, "title": "This is my title" })

También para Linux y Mac, también es posible escribir sus propios complementos. Estas son sus características disponibles, que se enumeran en el sitio web del proyecto:

  • complementos individuales para cada usuario;
  • registro de controladores http personalizados;
  • enviar mensajes como una aplicación;
  • Sistema de configuración basado en YAML en WebUI;
  • almacenamiento persistente para cada complemento de usuario;
  • Mostrar instrucciones generadas dinámicamente para los usuarios.

El sistema de complementos Gotify se basa en el sistema de complementos Go estándar, que se describe aquí . ¿Por qué se pueden usar en Gotify? los desarrolladores citan el ejemplo de recibir webhooks de GitHub, Travis CI y otros, sondear feeds a través de fuentes RSS, Atom y expandir la funcionalidad estándar de WebUI y, por supuesto, enviar alertas de alerta sobre bloqueos. En esta página puede encontrar una plantilla para el complemento y la explicación del código. Para que los complementos funcionen correctamente, los desarrolladores recomiendan usar Docker, pero sin él, es muy posible configurar Gotify para que funcione, aunque el envío y la recepción de mensajes correctos en este caso no están garantizados.

No proporcioné aquí el texto completo de la documentación básica del proyecto; puede leer todo usted mismo. A juzgar por las revisiones, el proyecto resultó ser bastante reflexivo, al menos por otro lado, que no se utiliza la notificación por SMS (lo cual es inconveniente si está en el extranjero o en el sótano), tratando de ignorar activamente el modo de ahorro de energía que se introdujo en Android 6-7 . Gotify es curioso incluso si es de código abierto, es decir, gratuito y nada impide bifurcar, recoger un archivo y llevar esta herramienta a un estado adecuado para las necesidades de ciertas personas / equipos.
