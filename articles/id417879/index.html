Cara mengambil ujian dari Microsoft secara gratis [misalnya, kompetisi terbaru]

Halo, Habr! Saya pikir banyak orang tahu tentang ujian sertifikasi kami. Tetapi mungkin sedikit yang pernah mendengar bahwa kami terkadang melakukan berbagai kegiatan, yang dengannya Anda dapat mengikuti tes gratis. Sebagai contoh, kami mengatur sesi sertifikasi di berbagai acara Microsoft, menyelenggarakan kontes online dan banyak lagi. Di bawah potongan Anda akan menemukan kisah salah satu peserta dalam kontes online CloudPro 2020 tentang cara mendapatkan voucher gratis, serta mempersiapkan dan lulus ujian Microsoft Certified Professional.



Saya memberikan lantai kepada penulis.

Selamat siang, rekan! Nama saya Natalia Efimtseva , belum lama ini saya lulus ujian 70-532 "Mengembangkan Solusi Microsoft Azure" . Sebenarnya, kita akan membahas ini nanti.



Waktu yang dihabiskan: ~ 40-60 jam selama 30-45 hari.

Biaya: 0 RUB (menerima voucher untuk program Cloud Pro 2020 ).

Durasi Sertifikat: Tidak Terbatas.

Basis teknologi


Microsoft mulai terlibat dalam teknologi dan produk untuk waktu yang lama, sekarang menakutkan untuk menulis angka ini - lebih dari 10 tahun yang lalu. Sebagian besar waktu dengan teknologi Microsoft terhubung selama pekerjaan saya langsung dengan Microsoft (8+). Lalu ada periode kerja di Google (3+). Tetapi tahun lalu telah berkonsentrasi pada konsultasi independen dan partisipasi dalam proyek sebagai ahli eksternal. 3-4 tahun terakhir di Microsoft telah dikaitkan dengan Microsoft Azure (kemudian semuanya dimulai sebagai Windows Azure), Google juga memiliki proyek cloud, tetapi dengan fokus pada Google Cloud Platform.

Saat ini, sebagian besar masalah yang saya temui terkait dengan cloud, IoT dan ML. Entah bagaimana ternyata sekitar 70% tugas terkait dengan Azure. Rasio ini terdiri dari faktor objektif dan faktor "bias". Distribusi dan ketersediaan Azure di Rusia adalah tujuan: Microsoft telah melakukan pekerjaan besar di sini (ini adalah masalah yang berkaitan dengan akuisisi dan pembelian, pelokalan, dukungan, pelatihan spesialis TI, proyek Azure lokal, program untuk mitra Gold \ Silver Azure, dll. p.). Untuk "bias" - kebetulan bahwa justru dengan saya mereka sering datang kepada saya dengan tema Azure, karena Azure tidak pernah meninggalkan fokus saya, bahkan saat bekerja dengan platform cloud lainnya.

Dari beberapa proyek di mana ia mengambil bagian dan informasi tentang yang tersedia secara online, ada tertulis dalam artikel " Microsoft Azure Cloud dalam proyek nyata ." Saya ingat proyek dengan Raxel Telematika (CASCO pintar), karena itu adalah proyek pertama yang terkait dengan IoT dan cloud. Ketika semuanya dimulai, Azure (dan vendor lainnya) tidak menawarkan layanan IoT khusus. Sekarang di Azure untuk solusi semacam itu, di mana klien memiliki perangkat pintar (untuk CASCO, khususnya, perangkat pintar di dalam mobil) , layanan khusus disediakan - Azure IoT Hub dan IoT Edge.

Sangat menarik bagaimana proyek itu ditransformasikan. IoT Edge, menurut saya, adalah langkah besar ke depan, karena memungkinkan kita untuk memahami bahwa cloud hanya dapat menjadi bagian dari solusi modern, banyak fungsi dapat dilakukan pada perangkat itu sendiri (perangkat pintar modern jauh lebih pintar daripada perangkat yang pernah kita miliki sebagai komputer utama), yang, pada gilirannya, menyimpan waktu untuk pemrosesan dan transfer data, dll. Tugas TI modern berubah dengan cepat dan menjadi lebih dan lebih ambisius, sehingga sangat nyaman untuk bekerja dengan teknologi yang juga berkembang dan tidak hanya menawarkan layanan cloud dasar. Ini memungkinkan Anda untuk memperbarui proyek dan melanjutkan, daripada menemukan kembali roda.

Motivasi


Pada titik tertentu, saya menyadari bahwa masuk akal untuk lulus ujian Azure, karena mengerjakan proyek yang menggunakan Azure.

Pertama, sertifikat tidak akan "berlebihan" untuk mengkonfirmasi kompetensi di bidang tertentu. Saya tidak bisa mengatakan bahwa sertifikat itu menjamin bahwa seseorang memahami sesuatu secara mendalam dan sempurna, tetapi dalam kombinasi dengan faktor-faktor lain, ini dapat berarti bahwa orang tersebut “mengetahui” dan bahwa ia dapat menetapkan tujuan untuk dirinya sendiri dan mencapai tujuan ini (mis. (memiliki keterampilan untuk merencanakan dan mengimplementasikan rencana).

Kedua, Azure telah berubah secara signifikan: ada dukungan kuat untuk teknologi dan struktur pihak ketiga dan solusi untuk bisnis besar, serta layanan barunya sendiri. Yaitu bagi saya, ujian itu seperti memeriksa pengetahuan "lama" (maksud saya dasar-dasar dan fondasinya, termasuk dari sudut pandang teknis, bagaimana Azure berfungsi, dan bukan hanya fitur yang terdokumentasi dari layanan dan layanan tertentu, yang banyak di antaranya telah mengalami positif yang cukup kuat perubahan), serta kesempatan untuk mendapatkan yang baru dan mengatur yang sudah diterima.

Ketiga, ketika saya membaca kursus Azure di musim semi, para siswa mendekati saya, termasuk dengan pertanyaan tentang sertifikasi Azure, dan tertarik pada bagaimana prosesnya diatur. Saya memutuskan untuk mempersiapkan masalah ini dan mencoba membantu mereka. Setiap informasi dan teori harus selalu diperiksa setidaknya dalam praktiknya secara mandiri - banyak nuansa selalu ditemukan.

Halaman ujian resmi


Inilah halaman ini . Ujian 70-532 memberikan status Microsoft Certified Professional (MCP). Dapat dikatakan bahwa ini adalah langkah awal untuk mendapatkan sertifikasi "lebih kompleks". Kadang-kadang ditanya apakah MCP memiliki tanggal kedaluwarsa atau tanggal kedaluwarsa. Mungkin kita dapat mengatakan bahwa status MCP tidak kedaluwarsa. Ujian itu sendiri (tetapi bukan status MCP Anda) mungkin tidak lagi tersedia untuk lulus (atau diganti dengan ujian lain yang diperbarui) atau teknologi atau produk mungkin tidak lagi relevan dari waktu ke waktu, tetapi status MCP yang diperoleh untuk teknologi atau produk tertentu akan tetap berada di Anda "Celengan".



Saya perhatikan bahwa tidak seperti MCP, ujian pada level berikut, misalnya, Microsoft Certified Solutions Expert (MCSE) atau Microsoft Certified Solutions Developer (MSCD), yang MCP merupakan komponen yang diperlukan, hanya aktif selama satu tahun. Untuk mempertahankan status aktif MCSE dan MCSD, Anda harus mengikuti ujian baru sekali setahun. Untuk detail lebih lanjut tentang topik ini, lihat Apakah Sertifikasi Microsoft Berakhir? dan Perubahan Sertifikasi Microsoft dan Selamat Tinggal pada MCSD Azure Solutions Architect . Contoh status ujian "lama" saya ada di bawah.



Biaya ujian


Harap dicatat bahwa biaya lulus ujian untuk Rusia adalah 80 USD. Jika Anda memiliki harga yang berbeda ditampilkan (misalnya, untuk AS adalah 165 USD), pastikan bahwa negara tempat Anda berada dan berencana untuk mengikuti ujian telah dipilih dengan benar.



Saya perhatikan bahwa pada akhirnya saya menerima voucher untuk ujian gratis, mengambil bagian dalam proyek CloudPro 2020 . Jika tidak ada voucher, saya masih akan lulus ujian. Voucher bukan prasyarat atau prasyarat untuk memutuskan lulus ujian, itu hanya bonus yang bagus.

CloudPro 2020


Kompetisi CloudPro 2020 memberi Anda voucher sertifikasi gratis. Diperlukan untuk lulus tes kecil (sekitar 10 pertanyaan). Tesnya sederhana. Salah satu pertanyaan memiliki kesalahan kecil dalam jawabannya. Agar tidak dianggap sebagai spoiler, saya tidak akan menunjukkan pertanyaan dan jawaban yang spesifik, tetapi saya akan mengatakan bahwa Azure memperhatikan keamanan dengan serius dan semua interaksi dengan portal manajemen melewati HTTPS. Tapi saya yakin semua orang tahu dan mengerti ini, jadi maknanya tetap benar.



Saya melakukan tes dan menerima surat berisi informasi tentang voucher. Yang menyenangkan adalah bahwa voucher tersebut juga menyertakan kemungkinan pengambilan kembali. Anda tidak keberatan =) Saya tidak perlu upaya kedua, tetapi kehadirannya menambah keyakinan bahwa itu membantu dalam upaya pertama - efek psikologis.

Kompetisi ini diadakan hingga 30 Juni 2018, tetapi ada kemungkinan besar bahwa itu akan digantikan oleh kompetisi baru. Oleh karena itu, saya pasti akan mengikuti proyek ini, saya sarankan Anda ingat tentang kesempatan ini.

Peraturan ujian


Aturan terperinci untuk melakukan ujian dan FAQ Microsoft tersedia dalam bahasa Rusia di tautan ini . Di sini Anda dapat menemukan banyak informasi yang berguna, misalnya, tentang aturan dan ketentuan pengulangan (saya mengetahui bahwa Anda dapat mencoba untuk lulus ujian yang sama tidak lebih dari 5 kali dalam 12 bulan); tentang format dan kemungkinan jumlah pertanyaan ujian (lihat Informasi dasar tentang ujian ); tentang ketersediaan Formulir Perubahan untuk pertanyaan ujian (walaupun saya tidak mengerti bagaimana melakukannya dalam praktik, karena dilarang mendokumentasikan / menyimpan pertanyaan dengan cara apa pun, dan bagaimanapun juga tidak mudah untuk mengingat semuanya secara terperinci, tetapi akan ada keinginan - prosesnya masih ada).

Dilarang (dan tidak hanya tidak disarankan) untuk menggunakan kesedihan otak dalam persiapan ujian. Saya mengutip dalam bahasa Inggris (karena tidak perlu menerjemahkan istilah-istilah dalam paragraf ini): "" Brain dump, "yang berkaitan dengan ujian sertifikasi, adalah sumber, seperti situs web, yang berisi konten ujian sertifikasi yang memiliki telah diperoleh secara curang dengan maksud untuk membaginya dengan atau menjualnya kepada kandidat ujian. Konten yang termasuk dalam kesedihan otak persis sama atau secara substansial mirip dengan item yang muncul pada ujian. Karena konten ujian diperoleh secara curang, menggunakan jenis materi ini untuk mempersiapkan atau selama ujian merupakan kecurangan. Penyedia "tempat pembuangan otak" melanggar hak kekayaan intelektual Microsoft dan perjanjian kerahasiaan. "

Untuk jawaban yang salah, poin tidak ditarik: “Untuk tebakan, tidak ada penalti yang dikenakan. Jika Anda memilih jawaban yang salah, Anda tidak akan mendapatkan poin untuk pertanyaan ini. Untuk jawaban yang salah, poin tidak diberikan ”(lihat tautan ).



Material




Untuk mempersiapkan - untuk mempersiapkan, tetapi tidak sepanjang hari dan malam, karena mengandalkan pengetahuan teoritis dan praktis yang tersedia. Pengetahuan yang diperbarui pada versi elektronik buku " Ref Ref 70-532 Developing Microsoft Azure Solutions ", tetapi edisi pertama. Azure dan ujian 70-532 sendiri telah mengalami perubahan tertentu, sehingga buku tersebut telah diperbarui. Khususnya, dalam edisi pertama buku, seluruh bab dikhususkan untuk Layanan Cloud, tetapi sekarang tidak lagi relevan (karena ada Layanan Aplikasi, dll.), Juga, misalnya, dalam edisi pertama tidak ada bagian yang dikhususkan untuk Azure Search, Cosmos DB dan informasi tentang model penyebaran sumber daya baru - ARM (Azure Resource Manager).

Saya perhatikan bahwa memahami bagaimana Layanan Cloud dan / atau Azure WebSites bekerja tidak akan berlebihan, ia berada pada tingkat pemahaman kerja (dan bukan pengaturan, dll.) Dan membantu untuk beradaptasi dengan Layanan Aplikasi. Dan dengan Cosmos DB, misalnya, saya berurusan dengan proyek, jadi saya memutuskan bahwa ini sudah cukup. Tentu saja, lebih baik disiapkan untuk edisi kedua, jika sudah dekat, jika tidak, maka Anda dapat memilih edisi pertama dan juga membaca materi yang hilang. Ngomong-ngomong, setelah setiap bab dalam buku ini ada tes dengan 5 pertanyaan tentang topik yang dipelajari - tes ini membantu Anda menyetel ke format ujian dan menguji diri Anda (baik pengetahuan dan kemampuan untuk menjawab pertanyaan pengujian).



Saya menyaksikan sebagian kursus " Sesi Teknis untuk Persiapan Ujian 70-532: Pengembangan pada Microsoft Azure " dari portal CloudPro 2020. Sebagian, karena saya lebih suka bahan cetakan. Tetapi dalam hal ini - kepada siapa itu lebih mudah.

Terkadang ini adalah bahan ulasan, terkadang contoh implementasi berdasarkan skenario indikatif. Buku gratis, formatnya elektronik, pemformatan dan ilustrasi bagus, cocok untuk pengenalan topik. Ujian itu sendiri


Momen organisasi


Sekitar 3 jam dialokasikan untuk ujian (ujian tidak terganggu, jadi pergilah dan masukkan dengan biaya Anda sendiri). Pusat sertifikasi diminta untuk menyerahkan dua dokumen identifikasi (tampaknya itu adalah paspor Rusia dan hak atau paspor asing). Saya berkenalan dengan aturan untuk melakukan pengujian (dalam bahasa Inggris) dan menandatangani untuk mengonfirmasi bahwa saya memahami segalanya dan setuju.

Saya menyerahkan semua hal, baik elektronik maupun non-elektronik, yaitu tidak hanya telepon, jam tangan, tetapi seluruh tas ransel dengan segala yang ada di dalamnya (termasuk pulpen dan notebook). Sebagai gantinya, sebuah spidol dan notepad dikeluarkan (jika ada sesuatu yang perlu dituliskan, misalnya). Saya pribadi belum menggunakan semua ini. Saya memasuki ruang ujian (selain saya, ada 2 atau 3 siswa lagi) dan melanjutkan ke ujian. Karena jam diserahkan, lalu penghitungan waktu sedikit hilang (selalu ada penghitungan mundur di halaman pemeriksaan, tapi itu relatif), rasanya hampir sama di kasino (meskipun tidak ada di sana) ketika Anda tidak tahu jam berapa di luar tembok (di ujian, tentu saja, Anda bisa menghitung, tapi saya semata-mata tentang sensasi). Saya menghabiskan sekitar satu setengah jam untuk ujian itu sendiri. Hasil dilaporkan segera setelah ujian.

Format ujian


Ujian ini tersedia dalam berbagai bahasa, termasuk Rusia, tetapi saya memilih bahasa Inggris. Saya perhatikan bahwa pengetahuan bahasa Inggris akan diperlukan untuk membaca kedua skenario (persyaratan), yang menurutnya serangkaian pertanyaan akan ditanyakan, dan untuk masalah organisasi (jenis pertanyaan, kemungkinan kembali ke pertanyaan, dll.), Jadi pilihlah yang lebih dekat.

Sebelum ujian (waktu ujian tidak dihabiskan untuk ini), lihat kuesioner elektronik di mana Anda menjawab pertanyaan tentang apa yang mendorong Anda untuk mengikuti ujian, pengalaman apa dengan Azure yang Anda miliki, dll. Menurut penafian dan informasi di sini, jawaban atas pertanyaan ini tidak memengaruhi pertanyaan yang mungkin muncul dalam ujian. Saya kira memang begitu, tetapi ketika menjawab pertanyaan, waspada, kewaspadaan tidak pernah diajukan.



Microsoft lulus ujian pertama dalam teknologi untuk waktu yang sangat lama, jadi saya belum siap untuk format yang diubah (atau menurut saya). Saya yakin bahwa mungkin untuk kembali ke pertanyaan apa pun nanti, tetapi tidak demikian. Seluruh ujian dibagi menjadi beberapa blok. Blok adalah pertanyaan yang disatukan oleh satu skenario atau beberapa topik teknis. Jadi, Anda hanya dapat kembali dan memeriksa saat Anda berada di dalam blok (Anda tidak dapat beralih di antara blok, mis. Satu blok selesai dan yang lainnya dimulai). Saya tidak berpikir bahwa itu mengganggu saya. Saya menandai jawaban untuk verifikasi dan kembali beberapa kali, tetapi pada kenyataannya tidak mengubah jawaban ini. Kemudian dia berhenti menandai, dan mulai menjawab segera (karena entah Anda tahu, atau tidak, atau Anda mengecualikan yang tidak cocok dan memilih yang menurut Anda paling benar, well, dalam hal ini, "siksa" intuisi saya lagi).

Saya punya 40 pertanyaan, menurut informasi dari FAQ pertanyaan mungkin ada sekitar 40-60. Sangat menarik (lagi dari FAQ, yang saya baca selama persiapan artikel) bahwa ada pertanyaan yang tidak diperhitungkan saat memformat skor akhir (lihat tautan), tetapi ternyata ini bukan kasus saya, karena dan jadi hanya ada 40 pertanyaan. Saya pikir tidak masuk akal untuk mempelajari perhitungan nilai akhir. Prosesnya tidak dijelaskan dengan jelas, jadi semuanya heuristik dan tebakan, hampir bagaimana menyelidiki algoritma peringkat untuk hasil pencarian dan akan lebih mudah untuk menentukan algoritma (algoritma tentu saja lebih sulit, tetapi secara praktis akan lebih mudah untuk menentukan, mungkin).Jadi bersiaplah dan jawablah dengan benar.
