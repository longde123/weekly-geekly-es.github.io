莫斯科，4月18日-QIWI SERVER PARTY 4.0

4月18日，星期四，我们将再次聚集QIWI SERVER PARTY mitap。

在会议上，我们将不仅详细讨论技术问题，还讨论优秀的程序员是否需要软技能，从培训学员那里学到的错误以及员工是否需要导师或足够的经理。



谈论什么：


  • 开发人员为什么要编写自动测试；
  • 指标：如何停止恐惧并开始了解您的系统；
  • 软技能和团队合作精神；
  • 具有共同代码所有权的产品的技术稳定性；
  • 为什么没有必要将自己局限于一种技术或编程语言的框架；
  • 实习生的指导和培训-如何不这样做。

有关报告的更多信息-已删减。

安东·尤里耶夫（Anton Yuryev）将讨论如何为项目带来最大利益的自我测试，以及开发人员如何自行编写测试（如果附近没有测试人员或只是想这样做）以及应该考虑什么。 开发人员经常面临选择-快速进行所有操作，而不进行测试，或将所有内容都覆盖在测试中，然后忘记“快速”一词。 自动测试和所谓的“闪烁”测试失败会阻止一个人专心，只会烦人。 该报告将讨论编写测试和方法方面的主要错误，这些错误和方法有助于将自动测试用作开发人员库中的有用工具。

自动测试：让爱而不是战争

  • 什么是可测试的设计？ 您在设计和编写代码时应该考虑什么，以便以后在编写和支持自动测试时不会遭受痛苦？
  • 金字塔还是冰淇淋？ 创建有效的测试套件。
  • 为什么会破裂？ 减少发现发红测试原因所需的精力量

Pavel Chernyak （服务器应用程序开发人员）将讨论使用哪些度量标准来处理代码，以及如果您想实现更多目标，如何使用这些度量标准正确覆盖代码，而不仅仅是显示带有数字的仪表板。

指标：如何停止恐惧并开始了解您的系统。

  • 让我们谈谈指标及其类型。
  • 如何收集指标，以便所有人都清楚。
  • 没有人需要度量标准。

Dmitry Roenko将谈到软技能的话题。 它们是否真的像所讨论的那样重要，以及它们如何对所有者本人和团队都有用。

Stas Dubrov将讨论在共有代码所有权的情况下如何确保项目的技术稳定性，包括有关敏捷的积极工作。

亚历山大·普罗科皮耶夫 （ Alexander Prokopyev） -有关为何编程语言本身不应该成为寻找和培训专家的决定性因素。 Sasha还将介绍我们在实习生方面的经验以及防止职业倦怠的方法。

培训主题将继续Yaroslav Storozhenko ，他将以导师的眼光扩展实习生的故事，并描述初学者进一步发展的阶段。
