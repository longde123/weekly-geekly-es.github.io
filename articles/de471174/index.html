Der Einstieg in Git und GitHub ist mit GitHub Desktop 2.2 einfacher als je zuvor



Jeder, der Git verwendet, weiß, dass es eine steile Lernkurve gibt. Wir haben von Entwicklern gelernt, dass die meisten Leute dazu neigen, von einem Kumpel zu lernen, sei es ein Mitarbeiter, ein Professor, ein Freund oder sogar ein YouTube-Video. In GitHub Desktop 2.2 veröffentlichen wir die erste Version eines interaktiven Git- und GitHub-Tutorials, das Ihr Freund sein und Ihnen den Einstieg erleichtern kann. Wenn Sie Desktop noch nicht kennen, können Sie das Tutorial unter desktop.github.com herunterladen und ausprobieren.

Dieser Artikel in unserem Blog.

Mach dich bereit


Um sich einzurichten, helfen wir Ihnen bei zwei wichtigen Aufgaben: Erstellen eines Repositorys und Verbinden eines Editors. Wenn Sie Desktop zum ersten Mal öffnen, wird eine Begrüßungsseite mit der neuen Option "Erstellen eines Lernprogramm-Repositorys" angezeigt. Wenn Sie mit dieser Option beginnen, wird ein Tutorial-Repository erstellt, das Sie durch die Kernkonzepte der Arbeit mit Git mit GitHub Desktop führt.



Es gibt viele Tools, die Sie benötigen, um mit Git und GitHub zu beginnen. Das wichtigste davon ist Ihr Code-Editor. Im ersten Schritt des Tutorials werden Sie aufgefordert, einen Editor zu installieren, falls Sie noch keinen haben.



Lerne den Github-Fluss


Als Nächstes führen wir Sie durch die Verwendung von GitHub Desktop, um Änderungen am Code lokal vorzunehmen und Ihre Arbeit an GitHub zu erledigen. Sie erstellen einen neuen Zweig, nehmen eine Änderung an einer Datei vor, schreiben sie fest, übertragen sie an GitHub und öffnen Ihre erste Pull-Anfrage.

Wir haben auch gehört, dass neue Benutzer anfänglich Verwirrung zwischen Git, GitHub und GitHub Desktop erfahren. Wir behandeln diese Unterschiede im Tutorial und stellen sicher, dass die Erklärungen verstärkt werden.



Mach weiter mit deinem eigenen Projekt


In GitHub Desktop 1.6 haben wir die nächsten Schritte vorgeschlagen, die auf dem Status Ihres Repositorys basieren. Wenn Sie das Lernprogramm abgeschlossen haben, schlagen wir in ähnlicher Weise die nächsten Schritte vor: Erkunden von Projekten auf GitHub, zu denen Sie möglicherweise beitragen möchten, Erstellen eines neuen Projekts oder Hinzufügen eines vorhandenen Projekts zu Desktop. Wir möchten immer, dass GitHub Desktop das Tool ist, das Ihre nächsten Schritte klar macht, unabhängig davon, ob Sie sich im Fluss Ihrer Arbeit befinden oder ein neuer Entwickler sind, der gerade erst anfängt.



Was kommt als nächstes?


Mit GitHub Desktop 2.2 machen wir das Produkt, das unsere Benutzer lieben, für Neulinge zugänglicher. Wir werden das Tutorial basierend auf Ihrem Feedback wiederholen und weiterhin auf der Verbindung zwischen GitHub und Ihrem lokalen Computer aufbauen. Wenn Sie etwas erstellen möchten, aber nicht wissen wie, stellen Sie sich GitHub Desktop als Ihren Freund vor, um Ihnen den Einstieg zu erleichtern.
