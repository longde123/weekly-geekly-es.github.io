Mengapa Anda tidak boleh menyewa VPS / VDS untuk 200 rubel atau bagaimana memilih server virtual yang tepat

Artikel ini muncul dengan topik di forum.searchengines.guru, di mana sering orang mencoba menemukan VPS lebih murah, membangun dan memantau jutaan direktori yang hampir tidak berguna bagi pengguna, kecuali untuk pembuatnya (dan saya akan menjelaskan mengapa di bawah) untuk menemukan penawaran termurah, yang merupakan kesalahan terbesar dalam kasus webmaster baru yang meremehkan kemampuan hosting atau melebih-lebihkan manfaat dari server virtual murah dalam kasusnya.

Pertama, mari kita tentukan mengapa server virtual mungkin diperlukan?

  • untuk meng-host situs web, termasuk situs yang memerlukan instalasi aplikasi tertentu;
  • untuk hosting aplikasi yang membutuhkan penyesuaian server web, atau OS tertentu;
  • untuk memberikan langkah-langkah keamanan pribadi, pengaturan firewall khusus, organisasi jaringan pribadi, pembangunan terowongan VPN.

Semua ini dapat dilakukan dengan cukup mudah, karena Anda memiliki akses root, dan pada kenyataannya, VPS Anda mungkin tidak berbeda secara pemrograman dari server khusus, dengan pengecualian pembatasan yang diterapkan oleh virtualisasi. Dan ini adalah poin yang sangat penting ketika memilih server virtual yang tepat.

Dalam penawaran penyedia hosting, Anda sering dapat menemukan dua jenis virtualisasi - OpenVZ dan KVM . Dan keduanya memiliki kelebihan dan kekurangan. Alasan utama ketidaksukaan terhadap OpenVZ adalah bahwa banyak penyedia menggunakan penjualan besar pada node mereka, karena virtualisasi memungkinkan Anda untuk menjual kembali CPU dan RAM berkali-kali. Akibatnya, penyedia yang ingin mendapatkan hasil maksimal, dengan biaya minimum, mengisi node dengan pelanggan sebanyak mungkin, sepenuhnya lupa bahwa pelanggan akan tumbuh. Mendekati konsumsi sumber daya 100% nyata dari node. Namun, virtualisasi perangkat keras KVM juga bukan tanpa cacat yang memungkinkan sumber daya dijual kembali berkali-kali, tetapi ada kemungkinan untuk hanya menjual kembali CPU berkali-kali, RAM dialokasikan secara ketat dan dijamin. Yang mengurangi kemungkinan menjual lebih banyak.

Dan pada prinsipnya, jika kita lupa tentang kerugian yang tidak mungkin untuk menginstal Windows pada OpenVZ , maka virtualisasi ini akan lebih baik, karena akan memungkinkan (dalam kasus kami) untuk mengkonsumsi RAM lebih optimal, karena menurut statistik bahkan 50% dari RAM yang kami alokasikan pelanggan tidak dikonsumsi oleh mereka. Tidak, kami tidak akan menambah jumlah pengguna. Tidak mungkin. Untuk kebaikan semua, RAM tambahan akan memungkinkan pemrosesan tugas yang lebih cepat, yang akan mengurangi beban pada CPU, karena proses akan berjalan lebih cepat, antrian akan dibebaskan lebih cepat untuk memproses gangguan berikutnya, yang secara umum, tidak peduli betapa mengejutkannya, akan mengarah pada penurunan konsumsi, termasuk RAM.
