iOS CSS da morte


Um código foi publicado no github que leva a forçar a reinicialização dos dispositivos iOS (11/12 GM) ao visitar uma página html. Também faz com que o Mac OS High Sierra / Mojave congele ao usar o Safari.


O código que leva à reinicialização forçada é um código html com um grande número de divs aninhados e uma função "louca" de desfoque de fundo: 


... div { backdrop-filter: blur(10px); -webkit-backdrop-filter: blur(10px); width:10000px; height:10000px; } </style> </head> <body> <div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div><div> ...

Também foi publicada a página html PoC que implementa esse bug (os usuários do Safari correm seu próprio risco e risco).



Supõe-se que o bug tenha se arrastado em um nível inferior ao webkit e possa levar a consequências mais graves. Além disso, o uso desse bug pode ser aplicável em empresas sociotécnicas e piadas estúpidas, por isso aconselho que você seja extremamente desconfiado de todos os links, especialmente porque todos os navegadores no iOS são, de fato, um complemento para o Safari.
