Linux，微软和马克思主义

Linux项目及其领导者Linus Torvalds的历史经常被引用为软件开发中精英管理原则的示例。 然而，天真地宣称成功仅由发烧友的努力来确保，却没有大公司的参与，完全没有利他主义，这是天真的做法。


Microsoft和Linux 。


不想低估老式黑客的作用，对孤独者和大量志愿者充满热情，值得注意的是，没有大公司的帮助，Linux不可能飞得如此之快。 让我们看看大型企业与Linux开发人员社区之间的关系是如何发展的。 让我们从第一个最具争议的代表开始-微软。


初始处置


像任何其他私人公司一样，微软的目标是获利，而且越多越好。 大型商业公司也有一种使命，但通常公司会遵循这一使命，因为这并不妨碍他们为股东牟利。


为了实现这些简单的目标，Microsoft使用了各种与Linux开发人员社区相关的工具-从笨拙，不明显，故意和微妙的角度出发。 我们必须赞扬该公司的“ gutta-percha”策略，该策略不惧怕遵循反复试验的道路。


一开始，MS认为Linux和开放源代码软件可用于另一个Netscape，这会阻止该公司开展业务并需要清理。 做生意这么长时间了，比尔·盖茨和史蒂夫·鲍尔默。


Linux的创建者几乎总是为自己设定平淡无奇的目标。 例如，编写一个可用来运行终端仿真程序并发送电子邮件的OS。 由此产生的结果对我们来说是众所周知的。


20多年前，有人问他是否可能与MS合作，当时这似乎很棒。 Linux诞生已经7年了，在取得巨大成功之后，Linus并没有寻求胜利，只是设定了现实的目标。


-如果Bill Gates打电话给您并愿意为Linux编写软件该怎么办。
“如果微软曾经为Linux编写程序，那我赢了。”

因此，最初的冲突基础只能是Linux与Microsoft竞争的地方，而这主要涉及两个OS的竞争。


第一阶段-拒绝


微软现任首席执行官萨蒂亚·纳德拉（Satya Nadella）做出了很多努力，以确保前任首席执行官史蒂夫·鲍尔默（Steve Ballmer）的遗忘被遗忘，但是互联网记得他如何称呼Linux为癌性肿瘤。


Linux是一种癌症，它在知识产权意义上将其自身与所接触的一切联系在一起 。

他称Linux只是20年陈旧的Unix OS的克隆。


Linux本身是一个已有20多年历史的操作系统的克隆。 就是这样 那就是您今天可以得到的，它是一个已有20年历史的系统的克隆版本。 我并不是说它对某些客户没有位置，但这并不是一个创新的主张 。

在另一种说法中，“有远见的人”表达了一个相当直觉的想法，也许纯粹是偶然，指出Linux与共产主义类似 。


没有一家名为Linux的公司，几乎没有Linux路线图。 然而，Linux从地球有机地涌现出来。 您知道，它具有人们非常非常喜欢的共产主义特征 。

在美国，“共产主义”一词具有强烈的否定含义，在意识形态领域内对美国所面临的一切构成威胁。 然而，为了使Linux现在成为普遍威胁，史蒂夫·鲍尔默（Steve Ballmer）偶然地说了一些确实值得的东西。



事实是，按照马克思的理论，资本家双重压迫工人。 首先 ，它低估了工作； 其次，它疏远了工人的劳动成果。 如果许多读者直接熟悉第一段，那么第二段将涉及更微妙的问题。


我们正在谈论一个人将自己的一部分投入劳动的事实。 当工人被剥夺了酌情处置其劳动产品的机会时，他不仅在物质上而且在精神上被盗。 马克思认为，这是简化的劳动异化。


通常，Linux和开放源代码软件不允许资本家利用雇员并以有利于他的方式疏远其工作结果。 每分钟 可卡因 鲍尔默的见解理解了这一点，他开始疯狂地跳上舞台，大喊“开发人员，开发人员……”。


我远没有招募STR的创始人，但即使Richard Stallman和Linus Torvalds对国际音乐郑重发誓要献出自己的生命，使工人阶级摆脱资本主义的压迫，他们仍无法为自己的使命做更多的事情。


第二阶段-EEE


微软很长一段时间并成功地与其他公司竞争使用了拥抱，扩展，熄灭策略。 该策略本身源自对Fortran编译器和库大惊小怪的时代，甚至更早-在Windows OS诞生之前。


利用微软自己的专有开发技术，抵制竞争对手的主动行动，围绕开放标准的游戏也不是微软发明的。 IBM，英特尔，苹果也知道如何扼杀竞争对手。 但是，拥有可疑的EEE忍者掌心的是Microsoft。


这只是MS如何使用EEE策略的几个示例。


  • DHTML，作为1990年代网络标准的扩展，包括HTML，CSS和JavaScript；
  • Active Directory，而不是Kerberos和LDAP；
  • Visual和J ++ J / Direct，作为Java，JNI的扩展和替代。

当然，即使是微软公司也无法发明世界上所有的自行车，在某些情况下，即使是被截断的形式，这些标准也成功地运用于雷德蒙德。


  • CIFS / SMB；
  • TCP / IP
  • HTTP / SSL
  • SMTP / POP（S）/ IMAP（S）。

彗星与诉讼之间的典型案例 微软于2007年成立。当时，微软员工Ronald Alepin提供了证据，该证据将在很长一段时间内被记住。 在其中，他揭示了拥抱，扩展和熄灭雷德蒙德公司的本质。


问：好的。 现在，对于陪审团来说，在这种情况下，微软员工所使用的拥抱意味着什么？

答：它用来表示Microsoft将采用其他公司软件的标准或规格以及接口的策略。

问：好的。 扩展指的是什么？

答：一旦包含了规范，Microsoft将对其进行扩展并添加Microsoft专有的其他接口。

问：好的。 当您说添加Microsoft的其他专有接口时，这在技术上对其他ISV和OEM有什么影响？

答：嗯，结果是或影响是，曾经由社区接管的曾经属于社区发展的财产，基本上是由行业和行业参与者的工作所挪用的，由Microsoft接管。

然后，Microsoft接受了它，并通过其专有扩展，使它基本上无法继续提供给首先负责制定规格和标准的行业参与者。

问：好的。 当Microsoft使某些ISV和OEM无法使用这些API时，它们在技术上创建产品的能力对这些ISV和OEM产生了什么影响？

答：这会降低他们创建产品的能力，特别是将与Microsoft产品互操作的产品。

EEE游戏的目的是降低另一家公司与Microsoft竞争市场的能力。


微软与Novell之间的交易


作为“ 拥抱，扩展，熄灭 ”战略的一部分， Microsoft在2006年与Novell达成了非常含糊的协议，引起了Linux OS市场的轰动。


双方同意放弃以财务协议为依据的相互专利起诉。 MS已承诺支付：


  • 2.4亿美元用于购买具有转售权的SUSE许可证；
  • 在5年内投资3400万美元用于培训针对Linux / Windows解决方案的销售经理；
  • 1200万美元用于营销；
  • 一次性支付专利费用1.08亿美元。

Novell承诺在5年内至少支付4000万美元，确切的金额与Novell软件的销售挂钩。



这笔交易真是令人震惊，并且对Linux的供应商和用户都怀有敌意。 她闻到一股难闻的气味，似乎MS正在把Badman带到她的保护伞下，以换取他愚蠢地犯了专利侵权的事实。


没有人能证明Linux内核代码违反了Microsoft的专利，但这并不能阻止后者长期困扰Linux商业用户。


第三阶段-收养


显然，EEE策略不适合GPL和兼容许可证。 原因是GPL代码具有病毒的属性-包含该代码的程序会自动变得相似。 您已经教过所有人进行创新，然后悄悄地将更高级的软件与用户群私有化，因此您无法在程序中添加自己的东西。


在1990年代后期，通过复制FUD和威胁诉讼以“一个人留下来”来粉碎Linux是不现实的，企业应该怎么做？


在任命新首席执行官后，萨蒂亚·纳德拉（Satya Nadella）成为公司掌权的史蒂夫·鲍尔默（Steve Ballmer）时，工作方式发生了巨大变化。 一夜之间，我们看到了一家新的，更成熟，更负责任的微软公司，该公司放弃了黄铜指关节，并准备好遵守规则。


MS开始发送笑话和笑话的Linux内核补丁，但随后进行了纠正 。 开源Power Shell ， Visual Studio Code 。 期待已久MSSQL Server for Linux没有人感到惊讶。 Linux开始渗透 OS本身-用于Linux的Windows子系统。 然后是GitHub，该公司正在努力确认开源软件的承诺。


可以而且应该说所有这一切都是出于商业性质的考虑，该公司对黑客没有特别的同情。 这是可能的，但是如果没有比尔·盖茨和史蒂夫·鲍尔默的旧时代，但是毫不客气的百万美元时代，IT形势看起来会好得多。


当前处置


双方都应该对目前的状况感到满意。 该公司已成功完成了从动荡模式的退出，而不再完全依赖于Windows操作系统的盒装销售。 Azure平台获得了发展动力，这在很大程度上归功于Linux。


至于Linux，征服了所有的高峰，该项目失败了，对创建者来说是最可取的，到目前为止，工作站是针对Windows的，而“ Linux桌面年”的口号仍然永远是绿色的笑话。


我认为，由于这场对抗，微软损失的不只是Linux。 首先，Linux的损失是什么？ 其次，由于捍卫了工作站，该公司失去了广阔的移动设备市场。 除了盲目敌视之外，还有哪些因素阻止了MS在Google之前在Linux内核上推出其移动操作系统？


如果新的MS策略长期存在，那么Linux，Windows的每个拥护者，甚至那些使用iOS，MacOS，FreeBSD或OpenBSD的拥护者都将从中受益。 而且只有Minix用户根本不会有任何感觉。
