Memulai dengan Git dan GitHub lebih mudah dari sebelumnya dengan GitHub Desktop 2.2



Siapa pun yang menggunakan Git tahu bahwa ia memiliki kurva belajar yang curam. Kami telah belajar dari pengembang bahwa kebanyakan orang cenderung belajar dari seorang teman, apakah itu rekan kerja, profesor, teman, atau bahkan video YouTube. Di GitHub Desktop 2.2, kami merilis versi pertama tutorial Git dan GitHub interaktif yang dapat menjadi teman Anda dan membantu Anda memulai. Jika Anda baru menggunakan Desktop, Anda dapat mengunduh dan mencoba tutorial di desktop.github.com .

Artikel ini di blog kami.

Bersiaplah


Untuk menyiapkan, kami membantu Anda melalui dua bagian utama: membuat repositori dan menghubungkan editor. Saat Anda pertama kali membuka Desktop, halaman pembuka akan muncul dengan opsi baru untuk "Buat Gudang Tutorial". Dimulai dengan opsi ini membuat repositori tutorial yang memandu Anda melalui konsep inti bekerja dengan Git menggunakan GitHub Desktop.



Ada banyak alat yang Anda butuhkan untuk memulai dengan Git dan GitHub. Yang paling penting dari ini adalah editor kode Anda. Pada langkah pertama tutorial, Anda diminta untuk menginstal editor jika Anda belum memilikinya.



Pelajari aliran github


Selanjutnya, kami memandu Anda melalui cara menggunakan GitHub Desktop untuk membuat perubahan kode secara lokal dan mendapatkan pekerjaan Anda di GitHub. Anda akan membuat cabang baru, membuat perubahan ke file, melakukan itu, mendorongnya ke GitHub, dan membuka permintaan tarik pertama Anda.

Kami juga mendengar bahwa pengguna baru pada awalnya mengalami kebingungan antara Git, GitHub, dan GitHub Desktop. Kami membahas perbedaan-perbedaan ini dalam tutorial dan pastikan untuk memperkuat penjelasannya.



Teruskan proyek Anda sendiri


Di GitHub Desktop 1.6, kami memperkenalkan langkah-langkah selanjutnya yang disarankan berdasarkan kondisi repositori Anda. Sekarang ketika Anda menyelesaikan tutorial, kami juga menyarankan langkah-langkah selanjutnya: menjelajahi proyek di GitHub yang mungkin ingin Anda berkontribusi, membuat proyek baru, atau menambahkan proyek yang ada ke Desktop. Kami selalu ingin GitHub Desktop menjadi alat yang membuat langkah Anda lebih jelas, apakah Anda sedang dalam pekerjaan Anda, atau Anda adalah pengembang baru yang baru memulai.



Apa selanjutnya


Dengan GitHub Desktop 2.2, kami membuat produk yang disukai pengguna kami lebih mudah didatangi oleh pendatang baru. Kami akan mengulangi tutorial berdasarkan umpan balik Anda, dan kami akan terus membangun koneksi antara GitHub dan mesin lokal Anda. Jika Anda ingin mulai membangun sesuatu tetapi tidak tahu caranya, anggap GitHub Desktop sebagai teman Anda untuk membantu Anda memulai.
