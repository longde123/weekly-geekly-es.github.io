Fraktal Gerasimov. Menemukan sebuah pola. Meja hitam

Saya menemukan pola ini ketika saya melihat posting pengguna xcont . Setelah menemukan publikasi ini , saya menarik perhatian pada fakta bahwa pola diulang tidak hanya dengan peningkatan skala menurut angka Fibonacci.



Saya bertanya-tanya apakah ada pola dalam pola ini. Tetapi hanya memiliki 2 parameter x dan y , saya memutuskan bahwa saya perlu menunjuk sesuatu yang lain, yang umum di antara semua pola yang diperoleh. Kemudian saya perhatikan bahwa jika kita mengambil 4 kotak pertama di lapangan, dalam hal apa pun kita mendapatkan 3 opsi untuk awal pola, jika garisnya:

naik (↑)



bawah (↓)



atau tidak pergi * (-)



Untuk penunjukan, saya memutuskan untuk menggunakan simbol-simbol ini ↑, ↓, dengan syarat memanggil mereka berputar (seperti putaran partikel). Dan di sini saya mulai membuat tabel dependensi dari spin ini ke x , y .

Pertama, mari kita lihat apakah ada pola jika hanya y , x yang diubah, ambil 4


Dan sekarang tentang propertinya

Kita melihat urutan putaran - ↑ - ↓ - ↑ - ↓ - ↑ - ↓

Pola berulang dengan urutan tertentu
y = 3,7,11 ... ( ↑ )

y = 5,9,13 ... ( ↓ )

y = 2,4,6,8,10,12 ... ( - )


Kita mendapatkan hal yang sama jika x = 3, kita melihat urutan putaran ↑ - ↓ ↑ - ↓ ↑ - ↓ ↑ - ↓ ↑

y = 2,5,8,11 ... ( ↑ )

y = 4,7,10,13 ... ( ↓ )

y = 3,6,9,12 ... ( - )

Saya pikir saya bisa mensistematisasikan urutan ini dan menyusun tabel dari putaran ini, dan inilah yang saya dapatkan.

Saya hadir untuk Anda, "Table of Black".



Yang paling menarik adalah tabel ini memiliki dependensi dan properti.

Pertama, kami memperoleh beberapa rumus: →
jika x genap dan y genap → -, juga jika x ⋮ y atau y ⋮ x → -
x = y → -
x = y + 1 → ↑
x = y-1 → ↓
Dan sekarang tentang sifat-sifat tabel itu sendiri, jika kita mengambil x = y sebagai titik referensi, maka ke arah mana pun kita memiliki urutan pantulan cermin (ditunjukkan oleh garis merah dan hijau).








Sebenarnya menemukan digunakan dan digunakan. Saya memiliki meja hingga 12, karena pada awalnya saya melakukan semua ini di atas kertas.



Tetapi menggunakan algoritma JavaScript, Anda sendiri dapat memeriksa nilai yang lebih besar.

PS Saya tidak tahu bagaimana ini dapat digunakan, tetapi mungkin algoritma ini dapat diterapkan pada komputer kuantum.

Tautan: satu dan dua .
