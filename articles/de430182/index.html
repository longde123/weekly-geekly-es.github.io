CodeOne 2018 als JavaOne, jedoch nur in Maske

Oracle Code One




Code One (JavaOne bis April 2018) ist eine jährliche Konferenz zu Oracle-Produkten und der Programmiersprache Java, die in verschiedenen Städten stattfindet. Viele Java-Entwickler möchten an dieser Veranstaltung teilnehmen und ihre Teilnahme im Voraus planen. Ich, der Testautomatisierungsingenieur bei Sbertech, hatte das Glück, an einer Konferenz teilzunehmen, die dieses Jahr im Oktober in San Francisco stattfand. Unter Katze meine Eindrücke und Gedanken über die Reise. Die Termine der Konferenz unter der Schirmherrschaft von Code One im Jahr 2019 finden Sie hier .

Auf der Code One 2018 wurden mehr als 600 Berichte vorgestellt, von denen sich etwa 200 auf Java beziehen. Der Rest bezieht sich auf die Themen Container, Serverlos, Cloud-Anwendungen, Datenbank, Big Data, Data Science, DevOps, Modern Web usw. Am selben Tag (21. bis 25. Oktober) organisierte Oracle erstmals eine weitere Open World- Konferenz. Beide Veranstaltungen fanden in den Geschäftszentren des Moscone Center statt. Code One fand im Moscone Center West statt.



Die Kosten für die Teilnahme variieren ab dem Zeitpunkt der Registrierung:



Um sich für die Konferenz anzumelden, benötigen Sie ein Oracle-Konto. Während des Registrierungsprozesses werden Sie vom System aufgefordert, eine bequeme Zahlungsmethode auszuwählen. In meinem Fall erfolgte die Zahlung per Überweisung. Eine Woche später erhielt ich eine elektronische Einladung.

Organisatorische Nuancen


Vor der Konferenz veröffentlichen die Organisatoren den allgemeinen Konferenzplan auf der Website .

Registrierung


Es ist am bequemsten, am Tag vor der Veranstaltung zu akkreditieren, da Sie sonst am Eröffnungstag in großen Schlangen stehen könnten.



Bei der Akkreditierung wurde ein Plastikabzeichen mit einem Namen (für den Verlust einer Geldstrafe von 70 USD) sowie Plastikgutscheine (Mittagessen) für jeden Konferenztag ausgestellt. An separaten Regalen im Erdgeschoss erhielten die Teilnehmer Oracle-Souvenirs (2018 war es ein Rucksack) und ein Ticket für das Oracle CloudFest.

Aktionsplan


Alle Berichte und Workshops fanden im zweiten Stock statt. Ein großer Bereich mit zwei Hallen mit 12 Hallen mit Zonen: Java Magazine, Oracle Quiz, Oracle Shop und einer Zone mit Tischen. Gleichzeitig fanden 20 bis 24 Sitzungen statt. Sie begannen um 8-45 oder 9-00 Uhr morgens und endeten um 17-00 Uhr und später.







Das Programm mit den Themen Berichte und Workshops wird auf der Code One-Website im Block Sitzungskatalog veröffentlicht .

Bei der Teilnahme an einer so großen Konferenz ist es wichtig, einen eigenen Besuchsplan zu erstellen. Viele Sitzungen finden in verschiedenen Teilen des Gebäudes statt, und es ist unmöglich, sie alle gleichzeitig abzudecken. Wählen Sie das für Sie interessanteste aus, erstellen Sie einen Zeitplan und lernen Sie Java kennen (ich habe ungefähr 30 Minuten gebraucht, um mich mit den Berichten des nächsten Tages vertraut zu machen und diejenigen auszuwählen, die mich interessieren). Ich empfehle Ihnen, auf den Verfasser des Berichts zu achten und sich mit seinen früheren Reden vertraut zu machen. Wählen Sie Sicherungsberichte: Wenn Sie in den ersten Minuten feststellen, dass der Sprecher „nicht Ihnen“ gehört, besteht die Möglichkeit, eine weitere Rede zu halten.

Es ist besser, sich im Voraus für Workshops anzumelden, da Sie sonst möglicherweise nicht daran teilnehmen. Stellen Sie sicher, dass Wi-Fi in der Werkstatt stabil funktioniert (wenn Sie das neueste Laptop-Modell haben, z. B. ein MacBook, ist es ratsam, einen Adapter für RJ-45 zu haben, da Wi-Fi intermittierend ist und der Signalempfangspegel nicht überall stabil ist).

Berichte


Insgesamt konnte ich in 4 Tagen 25 Sitzungen besuchen. Das Thema war das vielfältigste, sogar das gute alte Selen von Kito Mann traf sich.

Unten finden Sie eine Liste der Berichte, die mir gefallen haben.

1. Java 5, 6, 7, 8, 9, 10, 11: Was haben Sie vermisst? - Henri Tremblay überprüfte die Hauptfunktionen jeder Version von Java SE und sprach über die neue Java-Release-Richtlinie. Empfohlen, den Artikel Java Is Still Free von Java Champions zu lesen.



2. Ganzheitliche Tests bei LinkedIn - Sajid Topiwala sprach darüber, wie sie Microservices auf LinkedIn testen, über das interne Tool Epsilon und wie die gesamte Entwicklung in die Trunk-basierte Entwicklung geht.





3. Die Zukunft von Java SE - Ein Bericht von Sharat Chander und Aurelio Garcia-Ribeyro über das neue Java & OpenJDK-Release-Modell, welche Funktionen in Java weiter erwartet werden sollten und welche Projekte jetzt beachtet werden sollten („Keine Java-Roadmap mehr, die was nennt Funktionen werden in den nächsten Versionen veröffentlicht. ”©).



4. Migration von Java UI-Clientanwendungen auf OpenJDK 11 - eine Geschichte über die großen Veränderungen in der Java UI-Welt von Philip Race und Sergey Bylokhov.


5. Erstellen einer Architektur für die Automatisierung von Webtests - Elias Nogueira und Tatiane Aguirres Nogueira sprachen über die wichtigsten Punkte, die Sie bei der WEB-Automatisierung beachten sollten. Ich empfahl den Autoren, Allure für Berichte und Selenoid für die Autotest-Infrastruktur zu verwenden.



6. Kotlin als modernisiertes Java - ein Bericht von Roman Elizarov. Der Sprecher konzentrierte seine ganze Aufmerksamkeit auf die Hauptmerkmale der Sprache. Die Einreichung des Materials war vor allem Lob.



7. Mikrometer: Es ist SLF4J, aber für Metriken! - Jon Schneider hat das Mikrometer-Tool überprüft, mit dem die Metriken Ihrer Anwendung erfasst werden. Ab M4 ist dieses Tool in SpringBoot 2.0 sofort einsatzbereit. Mikrometer ist wie SLF4J, aber nur zum Sammeln von Metriken. Der Autor erklärte, warum wir Metriken in unserer Anwendung verwenden sollten, und gab Anwendungsbeispiele an.

8. Von einer großartigen IDEE profitieren: Mit IntelliJ Superuser werden - Maxim Novak zeigte, wie ein Profi auf seinem Gebiet bei IDEA arbeiten sollte. Quellprojekt mit Hotkeys.

9. Lambda Programming Laboratory - Lambda- Workshop von Jeff Dinkins, Stuart Marks, Maurice Naftalin und José Paumard. Die ursprünglichen Aufgaben und Antworten finden Sie auf github .

10. Weitere Insider-Geheimnisse der Java-Community - Vortrag von Stephen Chin, Yolande Poirier und Sebastian Daschner. Der Bericht enthüllte die Geheimnisse der Java-Community. Wie man Java Champions wird, welche Schritte Sie benötigen, um Ihren Artikel auf Java Twitter zu retweeten, wie Sie Artikel für das Java Magazine schreiben. Die Autoren diskutierten JavaOne4Kids und ermutigten zum aktiven Erfahrungsaustausch.

Zusammenfassung


Code One hat mich aufgrund seiner Größe und Multidirektionalität im Vergleich zu den Konferenzen, die ich zuvor besucht habe, sehr beeindruckt: SeleniumConf Chicago, Joker, Heisenbug, SQADays usw. Natürlich wollte ich alles auf einmal behandeln, aber leider wurden viele interessante Berichte gleichzeitig gehalten.

Die Berichte waren von unterschiedlicher Qualität, einschließlich sehr geringer Qualität. Daher habe ich mich manchmal sogar gefragt, wie ein solcher Redner für die Teilnahme an der Konferenz ausgewählt werden könnte. Es besteht kein Zweifel, dass die JUG.ru-Konferenzen gewinnen: Die Jungs bemühen sich, nur qualitativ hochwertige Berichte zu erstellen, und sie machen es gut. Aber natürlich gibt es bei Code One mehr Weltmarktführer.

Ich war mit der Organisation der Veranstaltung zufrieden, mit Ausnahme der langen Schlangen zur Mittagszeit und eines schlechten WLAN-Signals. Wenn wir es noch einmal mit JUG.ru vergleichen: Bei Joker und JPoint habe ich nie ein Problem mit dem Netzwerk bemerkt. Außerdem bietet JUG.ru die Möglichkeit, Online-Sendungen in jedem Raum zu kaufen. Code One kann nicht aus der Ferne angesehen werden.

Übrigens wäre es schön, in Begleitung von 1-2 Kollegen zu solchen Konferenzen zu gehen, um die aus den Berichten erhaltenen Informationen zu diskutieren und mehr Material aufzunehmen.

Und was denken Sie, sind solche Reisen zu Konferenzen nützlich?
