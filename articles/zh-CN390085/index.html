Raspberry Pi如何抱怨互联网缓慢


Raspberry Pi Zero（照片：Raspberry Pi）

关于通讯渠道速度慢或通讯不畅的问题与服务提供商联系是一种乏味且效率低下的例程。这是任何国家都具有的特征，尽管在某些地区，情况仍然没有那么简单。但是即使在美国首都华盛顿也存在沟通问题，在这里并不少见。

华盛顿的一位居民（以在Reddit上的昵称而闻名-AlekseyP）决定在通道带宽低于承诺的带宽时自动将消息发送给提供商。在这种情况下，提供者为Comcast，承诺的通道宽度为150 Mbps。 AlekseyP使用Raspberry Pi Zero作为通信速度检查系统的控制元素。一旦速度降至50 Mbps以下，计算机就会捕获情况，并将数据发送到Comcast Twitter帐户。

消息是：“嘿，@ Comcast，为什么我在华盛顿特区支付150降价/ 10的费用后，互联网速度会降低XX / XX？ @ ComcastCares @ xfinity #comcast #speedtest。”



对于那些想在工作中尝试该系统的用户，用户在此处发布了该程序的源代码。但是，他说：“我不是一个好的程序员，所以不要谈论我的代码丑陋或可以改进的事实。”

在今年1月1日之后，用户出现了创建类似系统的决定，传入通道的速度降至2 Mbps，传出通道的速度降至9 Mbps。在Reddit上的帖子中用户说：“我知道，有些人可能会说我不应该抱怨传入信道上的50 Mbps，但是当他们宣传150 Mbps，而我得到10-30时，我并不高兴。” Malinki网络端口的带宽为90 Mb / s，但是当速度降至50 Mb / s的阈值以下时，发明人也开始抱怨。据他介绍，平均信道容量约为82 Mbps。

同时，AlekseyP声称提供者每次都响应其自动消息，询问其地址和帐户信息，之后速度返回到声明的值。

根据用户的说法，所有Internet服务提供商都应得到公司承诺的速度。

我想知道在CIS中做类似的事情是否有意义，公司会做出反应吗？
