Applications pour les livres électroniques sur le système d'exploitation Android. Partie 2. Librairies et applications de lecture de livres

Dans la première partie de l'examen des demandes de livres électroniques sur le système d'exploitation Android, les raisons pour lesquelles toutes les applications pour le système Android ne fonctionneront pas correctement sur les livres électroniques avec le même système d'exploitation ont été décrites.



C'est ce triste fait qui nous a incités à tester de nombreuses applications et à choisir celles qui fonctionneront sur les salles de lecture (bien qu'avec des limitations).

Brièvement, les raisons des difficultés dans le travail des applications sur les lecteurs sont les suivantes:

  1. Les "lecteurs" ont un écran noir et blanc; l'affichage des couleurs dans les applications ne devrait pas être fondamentalement important;
  2. Les écrans des lecteurs se mettent également à jour assez lentement, de sorte que les applications ne doivent pas afficher de contenu évoluant rapidement;
  3. Les demandes ne doivent pas être payées, car les livres électroniques n'ont pas de magasin Google Play; nécessaire pour installer des applications payantes (mais le contenu payant dans l'application n'est pas exclu!);
  4. Les candidatures doivent être fondamentalement compatibles avec les lecteurs, même si les trois conditions précédentes sont remplies.

La première partie du matériel a présenté les résultats des tests des applications bureautiques avec des liens vers des versions utilisables des fichiers d'installation APK.

Dans cette (deuxième) partie, les résultats du test de deux catégories d'applications liées spécifiquement au processus de lecture des livres seront présentés: Librairies et Applications alternatives pour la lecture de livres (c'est-à-dire non préinstallées sur les livres électroniques vendus).

À la suite du test des applications, leur problème commun a été révélé: dans leur écrasante majorité, ils ne sont pas optimisés pour les livres électroniques en termes de contraste d'image.

En règle générale, ils fournissent des mesures pour limiter le contraste d'image excessif sur les smartphones et les tablettes. Habituellement, à cet effet, l'arrière-plan du texte n'est pas complètement blanc, mais légèrement assombri (gris, jaunâtre, avec imitation des pages d'un vieux livre, etc.); et les lettres du texte ne sont pas entièrement noires, mais gris foncé.

Mais les livres électroniques n'ont pas besoin de limiter le contraste, car leurs écrans ont déjà un contraste plus faible que les smartphones et les tablettes. Une limitation supplémentaire du contraste peut entraîner un manque de contraste.

À cet égard, les utilisateurs sont recommandés dans les applications où il est possible d'ajuster la couleur d'arrière-plan et les symboles (lettres), de définir l'arrière-plan aussi clair que possible et la couleur des lettres aussi noire que possible (ou vice versa - pour les amateurs de l'image "inverse").

Les applications ont été testées sur les livres électroniques ONYX BOOX avec les versions d'Android 4.4 et 6.0. Avant d'installer l'application, l'utilisateur doit s'assurer que l'application est compatible avec la version d'Android sous laquelle fonctionne son livre électronique.

Les données suivantes sont indiquées dans la description de l'application:

  • nom (exactement comme indiqué sur le Google Play Store; même s'il contient des fautes d'orthographe ou de style);
  • développeur (parfois des applications portant le même nom peuvent être publiées par différents développeurs);
  • but de l'application
  • Version Android requise
  • Lien vers le fichier APK fini vérifié sur le MacCenter;
  • Un lien vers cette application dans le Google Play Store (pour un examen plus détaillé de l'application et des avis; vous ne pouvez pas y télécharger le fichier d'installation APK);
  • un lien pour télécharger le fichier d'installation de l'APK de l'application à partir d'une autre source (le cas échéant);
  • Une note indiquant les fonctionnalités possibles de l'application;
  • quelques captures d'écran d'une application en cours d'exécution.
