Bagaimana kami terintegrasi untuk toko SAP di Eropa dengan meja kas di Rusia melalui 1C



Baru-baru ini, kami menerbitkan sebuah artikel tentang mengapa pengecer asing di Rusia membutuhkan layanan "pembukaan toko turnkey". Hari ini kami ingin berbicara lagi tentang cabang-cabang Rusia merek asing dan rantai ritel, dan berbicara tentang solusi percontohan, yang membantu mereka memenuhi persyaratan undang-undang negara kita.

Mengapa docking seperti itu diperlukan antar program


Sangat sulit bagi perwakilan perusahaan asing yang menemukan pasar Rusia untuk pertama kalinya dengan cepat beradaptasi dengan persyaratan undang-undang kami. Dan sistem informasi yang digunakan oleh mereka tidak memungkinkan pelaporan akuntansi dan pajak sesuai dengan undang-undang kami. Pada saat yang sama, kebutuhan untuk memiliki informasi yang terkonsolidasi pada semua kantor perwakilan dalam sistem informasi dasarnya juga tidak menghilang, karena itu diperlukan untuk analisis dan keputusan manajemen. Oleh karena itu, pengecer asing memerlukan perangkat lunak yang dapat mengintegrasikan beberapa sistem dan menyinkronkan data di dalamnya.

Mari kita ambil contoh. Salah satu pelanggan Pilot - rantai toko pakaian wanita asing menoleh kepada kami untuk koneksi semacam itu. Dia perlu mengintegrasikan sistem berikut:

  • sistem akuntansi kantor pusat yang berlokasi di luar negeri (SAP);

  • Sistem perdagangan front-office untuk meja kas dan peralatan perdagangan lainnya ( Profi-T );

  • sistem akuntansi untuk pengunjung di toko (Penghitung SM);

  • sistem akuntansi ("Akuntansi 1C").


Oleh karena itu, sistem integrasi harus memiliki fungsi dasar berikut:

  • impor bermacam-macam dan harga (dari sistem SAP utama);

  • impor pemberitahuan pengiriman barang yang dikirim ke Rusia (dari sistem SAP utama);

  • impor data barang yang benar-benar diterima (dari file deklarasi pabean yang disediakan oleh pabean pabean);

  • pembentukan dokumen tentang penerimaan barang di toko (termasuk berdasarkan terminal pengumpulan data - TSD);

  • pembuatan dokumen untuk berpindah antar toko (termasuk berdasarkan data TSD);

  • pembuatan dokumen inventaris di toko (termasuk berdasarkan data TSD);

  • impor dan konsolidasi data penjualan dari semua toko di jaringan (dari sistem front-office perdagangan Profi-T);

  • ekspor data konsolidasi tentang hasil penjualan (ke sistem SAP utama);

  • melaporkan distribusi dan kinerja barang di berbagai bagian;

  • ekspor data kinerja untuk menyediakan pelaporan pajak ke program akuntansi (dalam "Akuntansi 1C").

Untuk menyelesaikan tugas yang ditetapkan di hadapan kami , spesialis Pilot mengembangkan sistem RBS (Retail Backoffice System) pada platform 1C 8.3. Hari ini sudah berhasil digunakan oleh pengecer mode terkenal di pasar Rusia. Sistem RBS dirancang untuk mengumpulkan, memproses, dan mengkonsolidasikan data tentang aktivitas pelanggan untuk memantau aktivitas, membuat keputusan manajemen, serta menyerahkan laporan keuangan kepada badan pemerintah.

RBS ada dalam beberapa modifikasi - untuk setiap pelanggan sendiri, karena jelas bahwa dalam hal ini tidak ada solusi sirkulasi universal. Bagaimanapun, pengecer datang dari berbagai negara dan hampir semuanya memiliki sistem informasi dasar dan fitur akuntansi sendiri. Selain itu, berbagai program tunai dapat digunakan (misalnya, Profi-T, Retail Pro). Namun demikian, komposisi modul RBS dasar bervariasi sedikit dari versi ke versi, tetapi pada saat yang sama, program ini dimodifikasi secara serius untuk memenuhi persyaratan dan karakteristik masing-masing pelanggan.

Bagaimana RBS Bekerja


Sebagai bagian dari contoh kami, kami ingin menunjukkan deskripsi operasi salah satu modifikasi RBS terbaru. Urutan pergerakan dan pemrosesan data dalam bentuk elektronik adalah sebagai berikut:

1. File idoc dengan daftar barang dan harga eceran diterima dari sistem SAP kantor pusat di direktori pertukaran data pada server di Rusia.

2. Daftar barang dan harga eceran dari file idoc dimuat ke dalam database RBS (dimungkinkan mode otomatis dan manual). Nilai harga bersifat periodik dan, jika ada item dalam database, mereka hanya dicatat jika terjadi perubahan.

3. Setelah pembentukan batch dan pengiriman barang ke unit Rusia, layanan TI kantor pusat menghasilkan file pengiriman barang - Pengiriman barang. Ini dilakukan secara terpisah untuk setiap toko. File pengiriman dikirim ke server di Rusia di direktori pertukaran, serta ke mitra yang terlibat dalam logistik dan bea cukai barang, dan menggunakan file-file ini untuk mendistribusikan barang ke toko dan menghasilkan file pernyataan pabean.

4. RBS, berdasarkan data dari file pengiriman Barang, menghasilkan dokumen "Pemberitahuan Pengiriman" dalam database-nya.

5. Ketika barang tiba di pos pabean di Rusia, perusahaan mitra yang bertanggung jawab untuk logistik dan bea cukai membentuk file pernyataan pabean dalam format yang disetujui oleh Layanan Pabean Federal Rusia. File deklarasi pabean diunggah ke server RBS di direktori pertukaran. File ini harus berisi informasi yang diperlukan berikut:

  • Pengidentifikasi Produk Unik

  • Kode HS

  • Biaya Faktur

  • Biaya pengiriman

  • Biaya asuransi

  • Jumlah Tugas

  • Pengidentifikasi pengiriman produk untuk setiap pengiriman yang ditujukan untuk toko tertentu.

6. Manajer stok di kantor pusat mengunggah data deklarasi pabean menggunakan pemrosesan khusus, yang membuat dokumen â€œPabean deklarasiâ€ dalam database RBS. Ketika dokumen Deklarasi Pabean diposting, dokumen Penerimaan dibuat dalam database RBS.

7. Daftar barang dan harga eceran dari basis data RBS diunggah ke server mesin kasir Profi-T Center (mode otomatis dan manual dimungkinkan).

8. Daftar barang dan harga eceran dari database server Profi-T diunggah ke register kas toko, setelah itu mereka siap untuk menjual barang.

9. Data penjualan di toko, pembayaran dan pengembalian dari pelanggan secara real time diterima dari meja kas di server Profi-T. Dalam hal terjadi pemutusan, mereka menumpuk di box office dan diunggah ke server Profi-T setelah pemulihan.

10. Server Profi-T dalam mode otomatis dengan frekuensi yang ditentukan (secara default, 1 kali per hari pada pukul 2 pagi) mengunggah data dari register kas ke file di direktori pertukaran.

11. Server RBS dalam mode otomatis dengan frekuensi yang ditentukan (secara default, sekali sehari pada pukul 4 pagi) mengunduh data dari meja kas dari file yang diunggah oleh server Profi-T ke direktori pertukaran.

12. Data hasil penjualan diunggah ke program Akuntansi 1C melalui file CSV.

Diagram berikut menunjukkan diagram pergerakan data, panah biru adalah data keluar, panah hijau adalah data yang masuk:



Dengan demikian, dengan RBS, pelanggan menerima solusi yang mencakup berbagai tugas yang terkait dengan penyimpanan dan pemrosesan data. Secara khusus, ini adalah:

  • Menambah dan menyimpan informasi tentang produk;

  • Menambah dan menyimpan informasi berkala tentang harga eceran;

  • Menambah dan menyimpan informasi tentang perpindahan barang, termasuk tanda terima, penghapusan, kapitalisasi, perpindahan;

  • Menambah dan menyimpan informasi penjualan dalam konteks toko, barang, karakteristiknya;

  • Menambah dan menyimpan informasi tentang pembayaran oleh pelanggan;

  • Menambah dan menyimpan informasi tentang operasi impor, termasuk pemrosesan deklarasi pabean dan data pasokan;

  • Menambah dan menyimpan informasi tentang nilai tukar;

  • Memuat data dari file SAP idoc dengan informasi produk;

  • Mengunduh data dari file SAP idoc dengan informasi tentang tanda terima barang;

  • Unduh data dari file Profi-Tsvs dengan informasi penjualan;

  • Membongkar hasil kegiatan perusahaan untuk periode ke file csv untuk diunggah ke program Akuntansi 1C;

  • Mengunggah data barang dan harga ke file csv untuk diunggah ke sistem Profi-T;

  • Mengekspor data penjualan ke file idoc untuk diunggah ke sistem SAP;

  • Membongkar data tentang perpindahan barang (perpindahan, posting, penghapusan) ke file idoc untuk diunggah ke sistem SAP;

  • Perhitungan biaya barang;

  • Pencetakan dokumen primer;

  • Pelaporan pembentukan.

Sistem RBS memungkinkan perusahaan memulai operasi di Rusia untuk mengurangi waktu untuk memulai bisnis, cepat beradaptasi dengan undang-undang Rusia, dan juga mendapatkan alat yang mudah untuk memantau dan mengendalikan kegiatan.
