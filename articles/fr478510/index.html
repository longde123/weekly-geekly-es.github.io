Nous vous invitons à DINS QA EVENING 12/12/19: créez un Jenkins Pipeline et apprenez comment paralléliser le lancement des tests avec leur aide

Nous sommes heureux d'inviter toutes les personnes intéressées par l'automatisation des tests à la prochaine SOIRÉE DINS QA ouverte, qui se tiendra dans nos bureaux à Staro-Petergofsky, 19 ans.

Ce soir, Alexander Mikhailov de Yandex.Money partagera son expérience en écrivant Jenkins Pipeline et montrera comment cela aide dans les tests. DINS Ilya Gulyaev vous expliquera comment paralléliser le lancement de tests sur plusieurs machines et le faire le plus efficacement possible, en utilisant le même Jenkins Pipeline.

Sous la coupe - plus sur les rapports et les conférenciers, un lien pour vous inscrire pour participer à la réunion, diffuser des informations, des documents de la dernière réunion.



Rapports


«Jenkins Pipeline. Qu'est-ce que c'est et comment l'utilisons-nous dans les tests? »(Alexander Mikhailov, Yandex.Money)

Écrire Jenkins Pipeline est facile! Pour la première fois, cela semble compliqué, mais sa syntaxe est assez simple et directe. Je vais vous dire comment écrire un pipeline avec un minimum de travail. Je vais vous montrer l'exemple de l'utilisation du pipeline dans notre équipe. Et si quelqu'un supprimait tous vos 10+ emplois! Le rapport intéressera le développeur junior et QA Automation.

Au cours des 4 dernières années, Alexander a automatisé les tests dans des projets fintech à Java et Kotlin. Pendant ce temps, j'ai testé à la fois le backend et le frontend, et pas seulement. Il travaille actuellement dans une équipe fournissant l'automatisation des tests en tant que service.

«Lancement de test distribué» (Ilya Gulyaev, DINS)

Tout le monde a besoin d'un test distribué! Nous découvrirons comment (pas) écrire nos vélos en utilisant le pipeline Jenkins et intensifierons le lancement de tests pour des centaines de voitures. Je vais vous parler des écueils et comment les éliminer. Le rapport sera utile à QA Automation à tous les niveaux. Quiconque a déjà rencontré un test distribué ou pense qu'il sera bientôt confronté.

Ilya est engagée dans l'automatisation des tests - de l'écriture de tests à l'adaptation des frameworks de test aux tâches du projet. Il aime lire le code et comprendre comment tout fonctionne de l'intérieur. Il a de l'expérience dans l'écriture de plugins pour Jenkins. Il croit que les vélos valent la peine d'être conduits, pas de les construire.
