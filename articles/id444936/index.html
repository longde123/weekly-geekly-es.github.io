Firmware ZXHN H118N dari Dom.ru tanpa solder dan programmer


Halo

Saya mengeluarkannya dari lemari berdebu. Saya benar-benar membutuhkan ZXHN H118N dari Dom.ru.

Masalahnya ada pada firmware pendeknya, yang terkait dengan dom.ru penyedia (ErTelecom), di mana Anda hanya dapat memasukkan login dan kata sandi PPPOE untuk menghubungkan ke Internet.

Fungsi ini cukup untuk ibu rumah tangga, tetapi tidak untuk saya.

Karena itu, kami akan merombak router ini!

Kesulitan pertama adalah Anda tidak dapat mem-reflashnya juga. Dom.ru menangani ini dengan seksama. Itu hanya mungkin pada firmware Dom.ru. Tapi di sini di dom.ru firmware lama, yang dapat Anda putar kembali (downgrade), dimungkinkan untuk mengisi versi asli dari firmware ZTE.

1. Masukkan di bilah alamat 192.168.0.1/html/menu.html, lalu klik karakter apa pun - ini akan menampilkan webcam pabrik.

2. Buka halaman "Manajemen"

3. Tab "Pembaruan Firmware"

4. Pertama, mem- flash firmware dari DOMRU - DOMRU__ZXHN + H118NV2.0.1d_E04_OV.img (Firmware lama untuk home.ru dengan kemampuan untuk meningkatkan).

5. Setelah firmware, router menerima IP - 192.168.0.1. Namun melalui WEB tidak memungkinkan untuk masuk ke wajah. Hal ini dimungkinkan hanya melalui TELNET - Pengguna: root Pass: root (siapa yang peduli).

6. Menekan tombol reset, menerjemahkannya ke status default.

6.1. Mungkin router akan mengubah alamat menjadi 192.168.1.1, dan juga dalam pengaturan perangkat Anda yang terhubung ke router Anda perlu memasukkan rincian secara manual
IP 192.168.1.2
Mask 255.255.255.0
Gateway 192.168.1.1
Tanpa ini, pada 192.168.1.1, saya dialihkan ke halaman 404

7. Ada akses ke WEB dengan pengguna: admin pass: admin

8. Selanjutnya, dari titik 4. Hanya berkedip dengan firmware asli yang diambil dari arsip - ZXHN + H118NV2.0.1d_E04_OV.img

9. akses ke WEB dengan pengguna: admin pass: admin

PS Di Internet, banyak forum diisi dengan topik serupa, dan terlalu banyak metode dan informasi. Ini adalah metode khusus yang membantu saya, saya harap ini akan membantu Anda.

Perbarui Pada versi Hardware Versi baru V2.3 membahas celah ini. Hanya berlaku untuk versi yang lebih lama.
