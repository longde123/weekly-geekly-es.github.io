Diseño de paneles de control para el sitio de comercio electrónico de análisis web. Parte 2: Boletines informativos por correo electrónico. Panel operacional

El panel de la lista de correo operativa reflejará los indicadores financieros e indicadores del sistema ESP para cada uno de los correos promocionales y los datos mensuales agregados para las cartas de activación. Primero tiene que descubrir cómo configurar el seguimiento de datos.


Datos, ¡no puedes escaparte de nosotros!

¿Cómo rastrear listas de correo?


Tomaremos Unisender como un servicio ESP. Este servicio le permite realizar un conjunto limitado de cargas: no puede obtener todos los datos de envío por un período a la vez, solo puede cargar datos para cada envío. No todos se sienten cómodos trabajando con la API, por lo que utilizaremos la carga manual de los datos del boletín. Al aprender a analizar los correos enviados a Unisender, no habrá problemas con otros servicios.


Informe de correo electrónico enviado

Enviamos cartas de activación y promocionales. Para cartas promocionales, realizamos estadísticas de la siguiente forma:

  • Fecha de envio
  • Tiempo de envío
  • Línea de asunto
  • Contenido
  • Etiquetas UTM
  • Enviado
  • Entregado
  • Descubrimientos únicos
  • Transiciones únicas
  • Darse de baja
  • Agregando al Spam

Para las letras de activación, recopilar estadísticas para cada letra es laborioso: podemos enviar de 10 a 20 cartas por día. No es necesario realizar un seguimiento de las estadísticas diarias de los correos electrónicos desencadenantes: solo debe verificar si enviamos dichos correos electrónicos o si algo se rompe. Para ellos, los informes mensuales serán suficientes. Para los correos de activación, mantendremos estadísticas en la siguiente forma:

  • Mes de boletines
  • Boletín de noticias
  • Etiquetas UTM
  • Entregado
  • Leer
  • Transiciones

Estos datos se pueden obtener descargando del Unisender un informe resumido en la letra de activación el 1 de cada mes y el 1 del siguiente mes y restando algunos valores de otros.

Para los fines del informe operativo, necesitamos evaluar los ingresos y la tasa de conversión de cada carta de los correos promocionales y de cada carta de activación mensualmente. Para combinar datos sobre sesiones y transacciones de contadores de sistemas de estadísticas en línea y nuestros datos de sistemas ESP, utilizaremos un montón de etiquetas utm.

Recopilamos datos sobre correos promocionales


Marcamos cada envío promocional con un identificador de campaña único, que es asignado por el servicio ESP. Usando este identificador, conectamos los datos de la sesión con los datos sobre el envío de cartas. Para vincular los datos de ingresos, debemos hacer coincidir cada pedido desde la carga de CRM a la fuente del pedido desde el contador de estadísticas en línea vinculando los datos por número de pedido. A continuación, conecte los datos mediante el identificador único de las campañas de correo electrónico de ESP con los datos de CRM.


Modelo de datos promocionales

Pregunta para pensar . Podemos vincular a los datos de la lista de correo solo sobre aquellas sesiones que ocurrieron el día de la distribución, y podemos vincular datos sobre todas las sesiones que ocurrieron desde esta lista de correo. Ato los datos sobre todas las sesiones de la lista de correo para tener en cuenta todas las sesiones repetidas que generó esta lista de correo.

Recopilamos datos sobre correos desencadenantes


Marcamos cada boletín informativo con un identificador de campaña único y un nombre único. Por ejemplo, enviamos una carta en una cesta abandonada y una carta en vistas abandonadas. Luego, tales letras en utm_campaign tendrán diferentes números de campaña, y en utm_term escribiremos manualmente el nombre de las letras, abandon_cart y abandon_view, respectivamente. Para vincular sesiones y transacciones desde el contador de estadísticas e ingresos de CRM, utilizaremos números de campaña únicos, y para el enlace final de estos datos a la descarga desde ESP durante meses, usaremos utm_term.
