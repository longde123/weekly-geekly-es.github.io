Mengumumkan .NET Core 3.1 Pratinjau 1

Kami mengumumkan .NET Core 3.1 Pratinjau 1 . .NET Core 3.1 akan menjadi rilis kecil yang berfokus pada peningkatan utama pada Blazor dan desktop Windows, dua tambahan besar dalam .NET Core 3.0 . Ini akan menjadi rilis dukungan jangka panjang (LTS) dengan perkiraan tanggal pengiriman akhir Desember 2019.

Anda dapat mengunduh .NET Core 3.1 Preview 1 di Windows, macOS, dan Linux.

  • .NET Core 3.0 SDK dan Runtime
  • Gambar Docker

ASP.NET Core dan EF Core juga merilis pembaruan hari ini.

Visual Studio 16.4 Pratinjau 2 dan juga dirilis hari ini. Ini adalah pembaruan yang disarankan untuk menggunakan .NET Core 3.1 Pratinjau 1. Visual Studio 16.4 termasuk .NET Core 3.1, jadi hanya memperbarui Visual Studio akan memberikan Anda berdua rilis.

Detail:

  • Catatan rilis .NET Core 3.1

Masalah yang Diketahui: Penginstal Visual Studio 16.4 dapat menghapus instalan .NET Core 3.0 Runtime saat menginstal .NET Core 3.1. Kami menyarankan Anda menginstal ulang atau memperbaiki .NET Core 3.0 SDK dalam hal itu.



Perbaikan


Ada beberapa perbaikan yang ditargetkan yang direncanakan untuk .NET Core 3.1. Perbaikan berikut belum tersedia, tetapi diharapkan dalam pratinjau nanti.

Di .NET Core 3.0, .NET Core Desktop Runtime Installer (termasuk WPF dan Windows Forms) tidak menginstal .NET Core Runtime (termasuk CoreFX dan CoreCLR) untuk Anda. Kami akan mengubahnya, sehingga penginstal Desktop Runtime mandiri. Bundel Runtime dan Hosting, untuk skenario server, sudah berfungsi seperti ini.

C ++ / CLI adalah komponen yang dijanjikan dari rilis .NET Core 3.0. Itu disampaikan, mengingat bahwa itu adalah ketergantungan WPF, tetapi pengalaman pengembang dalam Visual Studio dan SDK hilang. Kami akan memberikan pengalaman itu dengan .NET Core 3.1 dan Visual Studio 16.4. C ++ / CLI hanya didukung di Windows.

macOS 10.15 Catalina mencakup persyaratan keamanan baru, bahwa aplikasi harus diaktakan . Kami akan memenuhi persyaratan ini untuk .NET Core SDK, untuk .NET Core 3.1 dan semua rilis .NET Core lainnya yang didukung. Jika Anda menggunakan .NET Core untuk mengirimkan aplikasi macOS, kami sangat menghargai bekerja dengan Anda pada persyaratan notaris.

Penutupan


Tujuan utama .NET Core 3.1 adalah untuk memoles fitur dan skenario yang kami kirimkan di .NET Core 3.0. .NET Core 3.1 akan menjadi rilis dukungan jangka panjang (LTS) , didukung setidaknya selama 3 tahun.

Silakan instal dan uji .NET Core 3.1 Pratinjau 1 dan beri kami umpan balik. Itu belum didukung atau direkomendasikan untuk digunakan dalam produksi.

Jika Anda melewatkannya, lihat pengumuman .NET Core 3.0 dari bulan lalu.
