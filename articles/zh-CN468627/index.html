ADAM-6200 I / O模块


远程I / O模块旨在与各种类型的外围设备进行通信。 这是构建工业系统的基本要素。 它们既可以从其他设备接收信号，也可以向设备发送控制信号，并通过MODBUS TCP，MQTT，HTTP等与SCADA中央控制系统集成在一起。

工业上使用的上一代I / O设备通常使用RS-232 / 485串行接口进行连接，这使得此类系统的可扩展性变得困难。 ADAM-6200系列模块的特点是能够通过以太网连接，以及用于2个以太网端口的内置交换机，从而允许串联连接设备。

除了信号传输之外， ADAM-6200模块还可以充当可编程逻辑控制器的角色来解决简单的自动化任务。 由于条件图形逻辑语言的支持，即使没有编程语言知识也可以对其进行编程。

本文讨论了设备的特性，防止故障的范围和其他功能。

技术指标


ADAM-6200系列具有各种型号，可满足任何工业需求。 可以连接设备以进行联合自治工作（点对点），这使您可以获得所需端口的任意组合。

  • ADAM-6217-8个模拟输入通道
  • ADAM-6224-4通道模拟输出
  • ADAM-6250-15个数字I / O通道
  • ADAM-6251-16个数字量输入通道
  • ADAM-6256-16路数字量输出
  • ADAM-6260-6个中继通道
  • ADAM-6266-4个中继通道


支持的协议


  • Modbus TCP是用于通过IP连接设备的行业标准。 该系列中的所有设备均支持该协议。
  • MQTT是一种现代的物联网协议，通常用于连接到Microsoft IoT中心等云系统。 MQTT内部的消息以JSON格式传输。
  • HTTP REST API -Web界面和REST API可轻松将设备集成到基于任何平台的现有解决方案中：NodeJS，PHP，Python等。
  • SNMP-用于配置网络设置和收集统计信息。

HTTP REST API


HTTP API的多功能性使它很容易将ADAM-6200设备集成到任何现有平台中，而无需特定的行业协议。

例如，让我们分析一下接收模拟输入状态的请求：

使用条件逻辑进行编程


不需要大量计算能力的简单自动化任务可以使用图形条件逻辑直接在ADAM控制器上进行编程。 例如，在发生事故时激活警报器，在发生传感器时关闭阀门等。 GCL环境几乎不需要编程技能，并允许您使用鼠标光标创建完整的自动化脚本。



在Adam / Apax .NET Utility中开发GCL脚本。 作为条件，可以使用数字或模拟输入，离散信号计数器，内部计时器等的状态。 规则可以相互参照，形成更复杂的条件。 在文档中阅读有关GCL开发环境的更多信息。

串行连接（菊花链）


ADAM-6200系列的设备板上有两个以太网端口，可通过菊花链将它们串联连接。 这大大简化了网络拓扑，并且在构建网络的远程部分时无需额外的交换机。



与使用RS-232 / 485接口的连接相反，通过以太网连接，可以轻松地将控制器集成到任何IP网络中，以及使用VPN隧道通过Internet连接远程对象，而无需使用其他接口转换器。

断电保护（自动旁路）


当链中的中间设备断电时，自动旁路功能会自动激活。 在此模式下，设备充当被动连接，就像直接连接电缆一样。 在这种情况下，UTP连接受到限制，因此重要的是考虑两个相邻点之间的电缆总长度，并且每侧不超过50米，因为无源连接的最大长度为100米。 该功能的电池寿命长达4天。


借助自动旁路，即使中间设备断电，通信也不会中断

结论


通用ADAM-6200 I / O模块具有广泛的协议支持：Modbus TCP，MQTT，HTTP REST，并且可以轻松集成到经典SCADA系统以及任何现代软件产品中。

对GCL语言程序的支持使您无需使用其他设备即可解决简单的自动化任务。 对等互操作性使设备之间可以直接进行数据交换，而无需使用中间数据处理服务器。

顺序连接设备的能力使得无需附加设备即可轻松构建大型网络，而自动旁路功能可防止由于设备故障而造成的线路中断。
