Andrei Belyaev在jug.msk.ru会议上关于Java的反思

在2019年9月26日上星期四，在CROC公司在莫斯科Java开发人员社区jug.msk.ru的会议上，Andrey Belyaev发表了一份报告``减少代码中的反射量''。



关于演讲者


Andrey目前在Haulmont从事开发人员倡导工作，参与CUBA平台的开发。 他代表公司参加了包括今年在内的Oracle Code One会议。

关于报告


Andrey Kogun打开会议，回顾即将举行的会议（最近的会议是Joker 2019 ），并介绍会议的发言人安德烈（Andrey）。



该报告专门讨论Java编程语言中的反思主题。 研究了JVM中的反射调用的功能，其替代用途， LambdaMetafactory机制的使用，代码生成的示例。 提出了各种方法的微基准，代码在GitHub上 。



该报告提到了两个Web框架（ Micronaut和Quarkus）的功能 。 Quarkus是一个非常年轻的Red Hat框架。 受到他的启发，第二天，他在其余 文章中添加了一个在集合中 使用它的示例 ，这是为最近的一篇文章写的。

在报告的末尾，通过对各种方法的评估对结果进行了评估，这些标准包括可支持性，速度，使用AOT的可能性以及实施的复杂性等标准。



传统上，在此类会议上，有机会就报告的主题提出问题，在轻松的氛围中与同事交流，在休息时用茶和咖啡包子来增强他们的力量。
