Utusan seluler mandiri


Saya mengusulkan untuk mempertimbangkan penyebaran prototipe mobile messenger (cloud di saku Anda), yang memungkinkan Anda untuk mengatur komunikasi di area yang tidak memiliki infrastruktur jaringan data tradisional, serta jaringan pribadi yang hanya diakses oleh pengguna dan data harus ditransmisikan melalui saluran komunikasi yang aman. Pada saat yang sama, mobilitas berarti laptop untuk berkomunikasi dengan klien messenger yang diinstal. Menggunakan ponsel dimungkinkan dengan implementasi konsep selanjutnya.

Dalam komunitas teknologi modern, komunikasi messenger biasanya datang untuk mentransfer data ke server data tunggal (topologi bintang), sehingga memaksakan pembatasan berikut pada kebebasan komunikasi:

  1. Harus punya server. Dalam pengertian tradisional, ini adalah server dari messenger online besar (WhatsApp, telegram, sms, ICQ). Atau itu adalah server dari subnet PC kecil yang menggunakan aplikasi kecil dan protokol pertukaran standar (ICQ, jabber, dll.). Pada saat yang sama, server tidak hanya menyediakan relay paket dari pengirim ke penerima, tetapi juga fitur-fitur yang berguna seperti: menyimpan pesan sampai klien pergi ke jaringan, mencatat, mengirim massal, satu daftar kontak jaringan, dan sebagainya.
  2. Kehadiran jaringan komputer, router baca, router, titik akses, menara operator seluler, pengorganisasian atau memberikan akses ke jaringan lokal / global tunggal. Infrastruktur ini, di tempat pertama, biaya relatif banyak uang, dan kedua, itu diam, yaitu, penyebarannya dilakukan sekali dan untuk waktu yang lama, dan sebagai aturan, untuk banyak uang. Dan ketiga, itu membutuhkan keterampilan pengalaman dan administrasi.

Saya akan memberikan beberapa contoh:

  1. Sebuah tim yang terdiri dari 10 orang berangkat ke tempat yang jauh dari peradaban untuk melakukan pekerjaan. Ini bisa seperti tim ahli geologi, penambang, sekelompok mahasiswa magang atau kantor keliling dari karyawan organisasi. Di darat, diperlukan untuk menyebarkan jaringan untuk komunikasi antara anggota tim dengan cepat, sambil mempertimbangkan bahwa mereka dapat melakukan tugas, misalnya, hingga 10 km.
  2. Anda pergi beristirahat di pangkalan wisata bersama teman-teman Anda di rumah-rumah terpisah atau bahkan tenda. Anda perlu menggunakan jaringan seluler untuk berkomunikasi dan mentransfer beberapa data.

Dengan demikian, dengan mempertimbangkan hal tersebut di atas, saya akan mencantumkan kriteria berikut yang harus dipenuhi oleh jaringan seluler lokal yang serupa:

  • Kecepatan transfer data rendah. Ini merujuk pada pengiriman pesan teks tanpa file media.
  • Kekompakan. Perangkat harus ringan dan kecil.
  • Otonomi. Perangkat harus mengkonsumsi sejumlah kecil daya, yang memungkinkannya ditenagai oleh USB atau baterai portabel.
  • Nirkabel Transmisi data harus dilakukan melalui saluran transmisi data nirkabel.
  • Keamanan Saluran data harus dilindungi dari akses yang tidak sah.
  • Harga Seluruh rangkaian peralatan seharusnya tidak lebih dari peralatan untuk mengatur jaringan IP.
  • Kemudahan penggunaan. Penyebaran jaringan harus dibatasi dengan menekan beberapa tombol dan memasukkan nama pengguna dan kata sandi.
  • Rentang sinyal yang ditransmisikan. Kisaran transmisi melalui saluran nirkabel harus disediakan tidak hanya karena kekuatan sinyal output dan lebar saluran, tetapi juga oleh kemungkinan membangun jaringan transmisi data mesh yang memungkinkan menyampaikan pesan antar pelanggan, sehingga menciptakan jangkauan jangkauan yang hampir tak terbatas. WI-FI yang sama difokuskan pada pekerjaan di dalam ruangan hingga 30 meter, dan total Bluetooth dirancang untuk aksesori.
  • Akses ke jaringan global. Kemampuan untuk mengakses jaringan Internet global tunggal, jika ada.

Sebagai messenger yang serupa, saya mengusulkan konsep sistem berikut:

  1. Server pesan. Meskipun merupakan kelemahan dari solusi klasik, itu memberikan keuntungan yang jelas. Oleh karena itu, disarankan untuk tidak membuangnya dengan beralih ke p2p, tetapi menggunakan PC mini seperti Rasbery atau Beaglebone sebagai server. Persyaratan dasar: kehadiran tombol eksternal, OS linux, keberadaan indikator, serta kemampuan untuk menghubungkan antarmuka radio untuk transfer data. Anda dapat menggunakan Jabber sebagai server pesan. PC mini diinginkan dengan baterai bawaan.
  2. Stick RF-USB dengan fungsi membangun jaringan mesh, diusulkan untuk menggunakan 6LoWPAN. Teknologi ini memungkinkan Anda untuk mendapatkan jaringan ipv6 dari jaringan mesh dalam bentuk antarmuka jaringan untuk klien PC. Nilai tambah yang besar adalah memungkinkan Anda untuk menyesuaikan beberapa bagiannya, sehingga beradaptasi dengan tugas yang ada. Sebagai klien, diusulkan untuk menggunakan program percikan yang didistribusikan secara bebas. Dianjurkan untuk menggunakan radio dengan frekuensi 433 MHz.
  3. Untuk masa depan Jalur akses / pengulang WIFI RF433 dengan catu daya otonom untuk menghubungkan peralatan portabel tidak melalui USB, tetapi melalui wifi.

Saya menyarankan use case berikut:

  1. Anda akan terhubung ke server mini melalui Ethernet, pergi ke panel adminnya dan gunakan moncong web untuk mengatur parameter yang diperlukan dari server pengiriman pesan dan jaringan mesh sekali.
  2. Untuk menghubungkan klien ke jaringan mesh, diusulkan untuk mencolokkan stik USB-nya ke server dan klik tombol "hubungkan", setelah itu pengaturan jaringan untuk pekerjaan lebih lanjut di dalamnya akan ditulis ke tongkat.
  3. Nyalakan gantungan kunci dengan radio di USB laptop / PC dan sambungkan ke server jabber, setelah menentukan parameter server di klien.
  4. Perbarui daftar kontak pada klien dan mulai perpesanan.

Itu saja. Jaringan ini sekarang dapat selalu dibawa bersama Anda dan dimasukkan terlepas dari lokasi saat ini di ruang pelanggan. Jaringan akan secara otomatis membangun topologi yang benar dan klien terhubung ke server.

Sejauh ini, ini hanya konsep yang ditentukan oleh kebutuhan pribadi akan sistem semacam itu. Saya mengundang komunitas untuk membahas pro dan kontra, dan menjawab pertanyaan: "Apakah kita membutuhkannya?"
