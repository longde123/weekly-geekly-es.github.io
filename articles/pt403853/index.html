Log e banco de dados de eventos de IoT. Início rápido com o servidor iRidium

Muitas vezes, você precisa ver quando e o que aconteceu no sistema de automação. Para projetos sérios, você precisa ter acesso a um banco de dados de tais eventos, geralmente por um longo tempo.



Um acidente aconteceu, a porta se abriu, o sensor funcionou. A temperatura caiu para -10, então o tubo estourou. Alguém veio ao escritório à noite - os registros das câmeras foram perdidos, mas aqui estão os registros dos sensores de movimento. Em geral, uma coisa útil.

Com base no iRidium, esses bancos de dados são criados de maneira simples e local, sem o uso de nuvens ou servidores SCADA. Basta adicionar as tags desses protocolos que você precisa e marcá-las para salvar no banco de dados.



Por padrão, os bancos de dados são salvos na pasta "_user directory_ \ Documents \ iRidium pro documents \ Server \ Database".

Ao mesmo tempo, há uma base principal na qual você precisa entender um pouco, mas há uma base personalizada que registra apenas os dados necessários no formato desejado. Aqui está a API do banco de dados, se alguém quiser entender em detalhes.

Momento agradável: se você usar até 50 tags, a licença do servidor para o integrador é totalmente gratuita.
