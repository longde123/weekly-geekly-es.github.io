与Microsoft Dynamics 365的3CX集成

在本文中，我们将向您展示如何将Microsoft Dynamics 365 CRM集成连接到3CX IP PBX。

目前，3CX正在客户端3CXPhone上完成集成支持，并使用REST API技术将所有CRM集成转移到PBX服务器端。 这样，您就可以在3CX管理界面中为所有用户一次配置集成。 管理员不需要绕过所有工作站，甚至不需要在用户计算机上安装任何应用程序-只需使用基于3CX浏览器的Web客户端即可。

对于来电，将在Web客户端界面中打开来自CRM的联系人迷你卡，其中包含姓名，姓氏和电话号码。 卡中有一个小按钮，单击可直接在Microsoft Dynamics 365 CRM界面中打开完整的卡。

回想一下，所有CRM集成都免费包含在3CX Pro许可证中 。

配置Microsoft Dynamics 365


登录到您的Microsoft Dynamics 365帐户，然后转到Azure门户以创建身份验证所需的应用程序。 转到“ Azure Active Directory”>“应用程序注册”，“新应用程序注册”按钮。



指定应用程序参数：

  • 名称：3CX服务器端
  • 应用类型：本机
  • 重定向URI： https：// 3CX_URL / api / oauth2crm 3CX_URL-3CX服务器网址


单击“创建”按钮后，您将看到应用程序数据，特别是应用程序ID，然后在3CX一侧显示该数据。



接下来，您需要指定访问Microsoft Dynamics 365中信息的权限。转到“设置”>“所需权限，添加”按钮。



单击选择API，然后选择Dynamics CRM Online API。



设置所有可用权限，然后按“选择并完成”。



您可以删除Windows Azure Active Directory权限，因为 集成工作不需要它。 身份验证应用程序已创建。

现在，您应该获得租户ID，该ID也将在服务器端3CX上使用。 转到“ Azure Active Directory”>“应用程序注册，端点”按钮。



提交端点URL中存在的租户字符串。 例如，OAUTH 2.0 AUTHORIZATION ENDPOINT具有URL login.microsoftonline.com/dedc4ebb-183f-42e2-9192-92eb425a027d/oauth2/authorize 。 这是租户行dedc4ebb-183f-42e2-9192-92eb425a027d。

配置3CX CRM服务器集成


转到“设置”>“ CRM集成”，在“服务器端”选项卡，然后下载Microsoft Dynamics 365集成。


然后从下拉列表中选择DynamicsCRM。



指定在先前步骤中复制的应用程序ID和租户参数。 “域”是您的Dynamics 365门户的网址，然后单击“授权”。 将打开一个新选项卡，询问该应用程序的权限。



单击接受-授权令牌将出现在“授权”按钮下。 保存更改。

最后，您可以通过单击“测试”按钮来测试集成。
