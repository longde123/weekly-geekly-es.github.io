IT中的绅士行为规则：ITIL的历史

在2019年初，ITIL信息技术基础架构库正在等待自2011年以来最重大的更新。近30年来，它已在全世界范围内使用-私营企业和政府机构。 回顾创建ITIL的原因及其变化。


乔纳森·穆勒 （ Jonathan Mueller） CC的图像

创建ITIL：培训动物园解决方案


在1980年代后期，英国收集了用于组织IT领域流程的第一套最佳实践。 为了更好地了解ITIL的业务来源，有必要简要介绍一下当时英国IT行业的状况。

在1980年代，英国对政府机构进行了重点计算机化。 一段时间后，每个州机构和私人承包商都开发了自己的方法来管理IT流程。

迅速发展的行业环境完全是零散的。 它的每个元素都按照自己的规则行事，并且在系统的不同部分中经常重复出现相同的错误，这不仅会导致高昂的成本。 因此，一套最佳实践至关重要。

当时在英国，有一个部门负责信息技术的实现及其统一，即中央计算机和电信局 （CCTA）。 他的职责包括帮助私人公司为政府开发产品和服务。 最初，CCTA主要从事设备的采购，但是在80年代后期，他们转向实施面向业务的IT管理系统方法。

英国政府设定了CCTA任务，将信息技术领域组织流程的最佳做法与唯一务实的目标相结合：增加政府在计算机化方面的支出的回报。

该机构又指示研究小组从各行各业的约2.5万家大小企业的代表那里收集必要的信息。 一位参与创建图书馆的专家认为，这一想法是要突出最成功的发展，并形成一种标准化的方法来构建日常流程，该流程可以在将来绝对适用于任何组织。

IBM公司在代码的编译中发挥了重要作用，当时IBM公司已经开发出自己的系统方法来管理IT服务。 她将其命名为信息系统管理体系结构（ISMA）。 根据IBM的建议以及在ISMA的许多方面，形成了ITIL的原型-政府IT基础结构管理方法（GITIMM）。 该文档包含几个部分，包括变更管理，问题管理和可访问性管理。

后来，CCTA董事会成员Roy Dibble坚持更改该文件的标题。 “政府”和“方法”一词被排除在外。 第一个不适合，因为假定这些开发将由私人公司使用，第二个不符合项目的本质（因为这是一组最佳实践）。 结果，只剩下了IT基础架构管理。 1989年，它变成了IT基础结构库，简称ITIL。

IT食谱：ITIL的三个版本


1989年同一年， 出版了ITIL库的第一本书-“ IT服务水平的管理”。 在接下来的七年中，出现了新的销量。 其中包括：“变更管理”，“问题管理”，“配置管理”，“成本管理”，“可用性管理”等。 总计，ITIL v1 包括 30多本书。

与将来的版本相比，ITIL v1具有明显的技术偏见。 但是，这并没有阻止欧洲的大公司和政府机构立即开始将其引入实践。 ITIL于1996年进入美国，并开设了其中所规定实践的第一门认证课程。

围绕英国的IT管理方法，一群坚定的个人和组织逐渐形成。 社区的形成是图书馆创建者从一开始就计划的。 最初，它是一群“活跃用户”，随后在IT服务管理论坛 （itSMF）中成长，并在不同国家设有办事处。

同时，IT管理概念也在发展。 使武器投入使用的私营公司现在试图将技术与业务流程联系起来，为此必须简化和更新库。 关于ITIL v2的工作始于1990年代后期。 该系列的第一卷-ITIL服务支持-于2000年发行。 到那时，CCTA被大不列颠政府商务办公室取代，它在协调图书馆工作时拦截了警棍。

在准备第二版ITIL时，开发人员决定将自己限制在七册中-每册都涉及信息技术领域项目开发的一个单独方面。 他们被称为 ：服务支持，服务交付，服务管理实施计划，应用程序管理，信息和通信技术基础架构管理，安全管理和业务透视图。 因此，ITIL的重点从技术组件转移到服务组件。

在2000年，Microsoft使用ITIL作为Microsoft Operations Framework （MOF）开发的基础， Microsoft Operations Framework （MOF）为IT基础架构的有效运行提供了一系列建议。

ITIL的第二版在2006年底之前仍然有用。 届时，英国国家商会已授予 HP编译ITIL词汇表的权利。 2006年，他被重新发行。 它的发布标志着向ITIL第三版的平稳过渡。

该库的更新版本于2007年5月发布。 卷的数量再次下降-这次降至五个：“服务策略”，“服务设计”，“服务转换”，“服务运营”，“持续服务改进”。 每个卷都是在相应领域中自动化流程的一组建议，并描述了为此所需的工具。


罗兰·唐老 CC图片

ITIL v3的第三种形式描述了26种流程和功能，是上一版本的三倍。 而且它与 IT在业务流程中的集成更加紧密相关 。

在此期间，除了前面提到的MOF：ISO 20000标准，HP的ITSM参考模型，IBM的IT流程参考模型之外，还形成了几个基于ITIL的IT管理系统。

2011年，更新了ITIL的第三版。 从概念上讲，它仍然是ITIL v3，但结构进行了稍微重新设计： 添加了许多流程，有些流程已进行了修订。 例如，修改了“服务战略”，引入了诸如“业务关系管理”，“战略管理”，“需求管理”和“项目协调”之类的实体。

ITIL的未来：实践与实践


2013年，商标和ITIL开发权移交给了英国政府与Capita的合资企业Axelos。 Axelos授予组织使用ITIL知识产权的许可，授权审查员，并负责实施新版本的图书馆及其补给。 正是这家公司宣布了即将进行的更新。

用Axelos负责人的话来说，ITIL 2018将与DevOps，Agile和Lean方法学兼容。 这对于行业来说确实是必需的：许多公司已经结合了这些实践，并且更新库将简化它们的相互集成。

ITIL的发展及其整个历史的当前趋势是一个很好的例子，它说明了起源于公共部门的概念如何打破了关于官僚主义和缓慢性的陈规定型观念，并构成了《财富》 500强公司中最大的公司所需要的工具的基础。
