Présentation de la police de code Cascadia

Cascadia Code est enfin là! Vous pouvez l'installer directement à partir de la page des versions de notre référentiel sur GitHub ou le recevoir automatiquement dans la prochaine mise à jour de Windows Terminal.



Attendez, quel est le code Cascadia?


Cascadia Code a été présenté en mai lors de l'événement Microsoft Build. Il s'agit de la dernière police monospace fournie par Microsoft, qui offre une nouvelle expérience en ligne de commande et des éditeurs de code. Cascadia Code a été développé conjointement avec la nouvelle application Windows Terminal. Cette police est recommandée pour une utilisation avec des applications de terminal et des éditeurs de texte tels que Visual Studio et Visual Studio Code.

Ligatures logicielles


Cascadia Code prend en charge les ligatures logicielles! Les ligatures logicielles sont plus utiles lors de l'écriture de code, car elles créent de nouveaux glyphes en combinant des caractères. Cela aide à rendre le code lisible et «convivial» pour certaines personnes.



Remarque: si vous utilisez Visual Studio Code, pour utiliser les ligatures, vous devrez activer les ligatures de police dans les paramètres.



Avons-nous mentionné qu'il est open source?


Oui, vous avez bien entendu! Cascadia Code est disponible sur GitHub sous la licence SIL Open Font , vous pouvez donc contribuer si vous le souhaitez!

Remarque: il n'y a actuellement aucune infrastructure de test appropriée dans le référentiel, donc toutes les demandes d'extraction doivent être testées et approuvées manuellement par Microsoft. Cela ralentit la coordination des demandes de tirage, veuillez donc être compréhensif.

Pourquoi s'appelle-t-il Cascadia Code?


Le nom Cascadia Code vient du projet Windows Terminal. Avant sa sortie, le nom de code pour Windows Terminal était Cascadia. En fait, certains des fichiers source dans Windows Terminal utilisent toujours ce nom! Nous avons aimé l'idée de nommer la police par le nom de code du projet. C'est un tel hommage à Windows Terminal.

Nous avons ajouté «Code» à la fin du nom de la police pour indiquer que cette police est principalement destinée à la programmation. En particulier, il permet de déterminer qu'il inclut des ligatures logicielles.

Pour nous assurer que Cascadia Code était le bon choix pour le nom de la police, nous avons mené une enquête sur Twitter ainsi que d'autres options pour les noms que nous avons envisagés. Nous sommes très reconnaissants à tous d'avoir participé et nous sommes très heureux que Cascadia Code ait été le gagnant.



Qui est le concepteur de Cascadia Code?


Un grand merci à Aaron Bell, concepteur de Cascadia Code. Nous avons eu la chance de travailler avec lui au cours de la dernière année et il a fait un excellent travail. Aaron Bell, concepteur de polices et propriétaire de Saja Typeworks , travaille avec Microsoft depuis de nombreuses années. Par exemple, en 2015, il était le concepteur de Selawik ! Vous pouvez suivre ses activités sur Twitter ( @aaronbell ), et dans notre référentiel sur GitHub. Nous prévoyons de continuer à travailler avec lui pour améliorer le code Cascadia et en faire une excellente police pour tout le monde!

Que se passera-t-il ensuite avec Cascadia Code?


La version de police Cascadia Code 1909.16 est actuellement disponible sur GitHub. Il suivra la syntaxe des versions de Windows et recevra des mises à jour liées aux glyphes nouveaux et améliorés.

Restez en contact


Pour plus d'informations sur les mises à jour liées au code Cascadia, n'hésitez pas à suivre notre référentiel sur GitHub ou les pages Twitter Kayla ( @cinnamon_msft ) et Rich ( @richturn_ms ). Nous sommes très heureux de publier notre police dans le domaine public et nous espérons que vous commencerez à l'utiliser.
