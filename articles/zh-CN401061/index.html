加湿和空气净化系统

嗨，极客时间！


在上一份出版物中，我分享了我对安装在公寓中的空调系统进行现代化改造的想法，并通过在寒冷季节增加加湿来提供新鲜空气。
考虑到所有这些细微差别，我决定在寒冷的季节进行自然测试，此时室外温度将至少为0-+5度。


KDPV的作者是Vasya Lozhkin。

告知您测试正在进行中，我现在已经在考虑分享其中的一些结果，因为自动化所需的必要设备仍在使用中，冬天临近尾声，这种解决方案的完整结果很可能只能在下一个供暖季节才能估算出来。

流动/不流动


秋天，在“天堂之下”订购了一个雾气系统，并在一个月之内送给我。该套件包括一根5米长的挠性管，10个喷嘴，连接管子和喷雾器的三通，连接适配器以快速连接到供水系统。首先，我检查了喷雾器和水的消耗情况。从最细的细流（连接而不是莲蓬头）开始，在水压作用下，一个尖头会形成细雨云，直到打开的水龙头最大，流量可能会发生变化，但形成的云雾保持在相同范围内-大约800毫米固体角度为30度。通过计时器测量流量计-在30分钟内将约2.5升收集到一个桶中，用3升的罐子测量。

下一步是检查雾云如何在通道块内，管道和格栅中扩散，以及热交换器的散热片如何工作。

实验性

喷涂胶带


由于马上kurochit /穿孔升级的室内机不希望连接到一个三通管被卷绕成通过在发泡塑料插头空调货盘沿着气流离开。
我一开始就是这样打开水的-两个喷雾器都根据需要起作用，我没有注意到多余的污迹。
更换了冷凝水排放盘。我将室内机以最低速度启动到通风模式（仅风扇工作），然后再次打开水。从视觉上看，什么都看不见，因为一切都关闭了-但是从声音中可以清楚地看到喷雾器正在供水。顺便说一下，它们发出的噪音甚至比空调以最小速度运行时产生的噪音还要大。
起重机打开了15分钟，即向空调供应约2.5升的冷水。室内机至少工作5分钟，中速工作5分钟，最大风扇速度工作5分钟。一直以来，他“挥手”在室内机出水管上的热交换器后面，拿着纸巾等。在第10分钟（切换到最大速度之前）的某个位置，热交换器的下半部分变湿，在此期间，右侧锅中的水位升至约7毫米。在第15分钟，由于缺少塞子，水开始从管道上滴下，即锅中的水位已经很高（右角约10毫米）。我去了，关闭了进料的水龙头，听说冷凝泵是如何开始工作的-浮子掉了。泵工作了10分钟，用水关闭了通风模式。
原则上，我认为该实验是成功的，因为尚未确认主要问题并且水不会超出空调。

第一次保湿


当然，在收到肯定的结果表明细雨没有超出空调的限制之后，进行更全面的检查是非常“痒”的。准备工作，实际测量和将结果记录在桌子上花费了我一天的时间。拆除了空气分配格栅和调节器。将新鲜空气混入系统的风扇与电源断开。钻了一条运河，将一条管道插入了内部，并将喷雾剂拧到平台/扎带上。老实说，我忘了拍张照片，但与带胶带的照片并没有太大区别。我从同事那里拿了一个设备-Testo 410-2风速计，并对其进行了测量。测量程序-每个尺寸100x500mm的孔进行5次测量，每点10秒。测量是在2个进气口和一个排气格栅（流入幼儿园和卧室，从卧室排气）上进行的，很难到达第二个排气格栅-它在柜子上方，在它前面是装有玩具的包装，而此时最小的儿子在房间里睡着了 他从系统中供应了冷水，试图从12:35开始以大约10分钟的周期供应/关闭水。供水前的首次测量显示卧室供水格栅上的湿度为41.4％。结果在下表中。

从上表可以清楚地看到，在一个半小时的起雾系统中，房间的湿度上升到了55-60％的可接受水平，而其中的40％则有点令人不舒服（在室外起到了+5度的作用），空气的温度通过托盘并通过带有冷自来水的加湿热交换器下降1-2度，这是可以接受的。

测试正在进行中


目前，已经进行了类似于第一个测试的4个测试。基本上，与供水/关闭时间有关的变化-10 / 10、5 / 15、5 / 10、10 / 20，外部温度以及公寓内部湿度对湿度的依赖性在所有测试室内半小时的湿度中也起作用增加了50％以上。 5/10个循环的时间格式显示了加湿系统稳定性的最佳指标，即5分钟的水用于雾化，10分钟的水仅用于通风，但在这种情况下，每小时约有3.3升水排入下水道。同样非常值得注意的是，通道块的右半部分更润湿了空气（将空气供应到卧室的空气）。我的看法是，这是由于这样的事实，空气不仅不仅通过湿式热交换器，而且还通过货盘上方，那里收集了足够的水。
在这方面，我认为要从室内机中取出一个雾化器，即离开两个雾化器之一，然后离开左侧（沿空气）。因此，用于托儿所的空气将通过热交换器被加湿，用于卧室的空气将被托盘中的水弄湿，并且空调器内部的供水量将减半。

Arduino的


就像我在上面写的那样，arduino设备正在开发中，但是不幸的是铁件将不包含在本出版物中。
我为自己选择了必要的设备：
  1. DHT22 / AM2302，温湿度传感器-3个 在空气加热器后安装一个，以控制从街道供应到系统的新鲜空气的温度，在送风和回风管道中安装两个传感器。
  2. MH-Z19，红外二氧化碳传感器-1个。安装在回流管中。
  3. ACS712，5A电流传感器-1个 监视室内机的运行情况，打开/关闭风管风扇，打开/关闭排水泵。
  4. AC-DC 220V / 5V 700mA，Arduino电源-1个
  5. ARM PIC AVR DSP，继电器模块4通道Arduino-1个 给室内机，风扇，石英辐射器和空气加热器的供水阀供电。
  6. Arduino Nano 3.0控制器-1个

单独的清单不是arduino设备：
  1. 管道加热器。到目前为止，我正在考虑购买工厂自动化产品还是自己进行功率调节-自动化产品的成本要高出2.5倍。
  2. 供水阀。意外地从洗衣机/洗碗机中以供水阀的形式找到了解决方案。
  3. 杀菌（石英）壁照射器（我借此机会感谢mr_klimenko的建议）。它将安装在室内机最脏的部分-一个返回适配器，用于将风道连接到过滤器。

对系统自动化功能的更详细研究提出了两个问题，这些问题仍在寻找正确的解决方案中。即，风扇和空气加热器性能的平滑调整，在Google正在研究的同时，针对此问题有几种解决方案。

该出版物旨在讨论该决定，并不假装是正确和全面的，我准备并希望听到任何建设性的批评，愿望，评论等。
