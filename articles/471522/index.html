Askozia Cómo funciona Autoprovisioning Plug & Play

Al desarrollar Askozia PBX, nos enfrentamos a la tarea de configurar automáticamente los teléfonos y resolverlos a nuestra manera.

Autoprovisioning Plug & Play (PnP), esta tecnología es compatible con muchos fabricantes: Yealink, Snom, Fanvil.

Las principales ventajas de la configuración automática del teléfono:

  • Facilita la configuración inicial : no es necesario ir a la interfaz web de cada dispositivo. Es suficiente para indicar la correspondencia de la dirección MAC del dispositivo y la cuenta en el servidor de autoajuste.
  • Simplifica el soporte : realmente es más fácil si necesita cambiar la configuración del dispositivo. Controlamos la configuración nuevamente en el servidor
  • Es posible reducir la configuración al conjunto de códigos de estrella "* 911 * <SIP_ACC>"; en algunos casos, esta función simplemente no está disponible. No todos los trabajadores de oficina podrán configurar un teléfono IP, pero marcar una combinación de números es una tarea simple.

Describamos cómo funciona Autoprovisioning Plug & Play. Al final del artículo, un enlace al código fuente de un pequeño script PHP que implementa la funcionalidad del servidor PnP.

Estudio de caso


Tuvimos un caso interesante cuando presentamos la telefonía a uno de nuestros clientes. El principal problema era que el cliente estaba en otra ciudad. Al mismo tiempo, uno de los requisitos era que después de desempacar el paquete con los teléfonos y enchufarlos a la red, la telefonía debería funcionar de inmediato.

El problema se resolvió de manera relativamente simple. El cliente ha reservado una serie de direcciones IP para nosotros. Instalamos el equipo en nuestra oficina y lo empacamos en una caja y lo enviamos por mensajería.

Usando la configuración automática del dispositivo, la tarea se volvería mucho más simple.

Esquema PnP simplificado


El teléfono al comienzo del trabajo envía una solicitud de suscripción SIP SUSCRIBIR a la dirección 224.0.1.75 IP de multidifusión.
Leer más sobre 224.0.1.75
224.0.1.75: se trata de IP de multidifusión (para multidifusión) "reservada" para servidores SIP.
ver networksorcery.com/Enp/protocol/sip.htm
224.0.1.75 SIP, Protocolo de inicio de sesión (todos los servidores).

En respuesta, espera recibir una respuesta de NOTIFICACIÓN con instrucciones de configuración.



SUSCRIBIRSE Ejemplo 
2019/09/02 09:26:41.543856 172.16.32.148:5059 -> 224.0.1.75:5060 SUBSCRIBE sip:MAC0015657322ff@224.0.1.75 SIP/2.0 Via: SIP/2.0/UDP 172.16.32.148:5059;branch=z9hG4bK42032775 From: <sip:MAC0015657322ff@224.0.1.75>;tag=42032772 To: <sip:MAC0015657322ff@224.0.1.75> Call-ID: 42032772@172.16.32.148 CSeq: 1 SUBSCRIBE Contact: <sip:MAC0015657322ff@172.16.32.148:5059> Max-Forwards: 70 User-Agent: Yealink SIP-T21P 34.72.14.6 Expires: 0 Event: ua-profile;profile-type="device";vendor="Yealink";model="T21D";version="34.72.14.6" Accept: application/url Content-Length: 0


Los titulares más importantes e interesantes.


  • De : la dirección de amapola del dispositivo es 0015657322ff
  • Evento : describe exhaustivamente el dispositivo, el fabricante, el modelo y la versión de firmware
  • Contacto - dirección del dispositivo
  • Call-ID : este encabezado es interesante al configurar dispositivos DECT desde Yealink, transfiere el identificador de línea (número de serie del tubo), delimitador " _ "

Una vez que el servidor PnP ha recibido dicha solicitud, debe responder

NOTIFY ejemplo 
 2019/09/02 09:26:41.550125 172.16.32.153:57593 -> 172.16.32.148:5059 NOTIFY sip:172.16.32.148:5059 SIP/2.0 Via: SIP/2.0/UDP 172.16.32.148:5059;branch=z9hG4bK42032775 Max-Forwards: 20 Contact: <sip:172.16.32.148:5059;transport=UDP;handler=dum> From: <sip:MAC0015657322ff@224.0.1.75>;tag=42032772 To: <sip:MAC0015657322ff@224.0.1.75> Call-ID: 42032772@172.16.32.148 CSeq: 3 NOTIFY Content-Type: application/url Subscription-State: terminated;reason=timeout Event: ua-profile;profile-type="device";vendor="MIKO";model="MikoServerPnP";version="1.8" Content-Length: 40 http://172.16.32.153:84/0015657322ff.cfg


En un mensaje de NOTIFICACIÓN, la información más valiosa se encuentra en el cuerpo del mensaje. Como regla general, en el cuerpo debe pasar un enlace para obtener el archivo de configuración: 

 http://172.16.32.153:84/0015657322ff.cfg

Si varios servidores PnP se ejecutan en la red, entonces quién es el primero que responderá al dispositivo lo configurará.

El teléfono, al recibir NOTIFY, intenta cumplir con la solicitud en la dirección especificada.

Ejemplo de solicitud y respuesta del servidor

Un ejemplo de implementación del servidor está disponible en github https://github.com/boffart/MikoServerPnP

Para que este servidor PnP funcione, debe:

  • PHP 7.1.9
  • Tomas de php
  • BusyBox v1.26.2
  • Las solicitudes de difusión deben estar permitidas en la red

Características del servidor PnP


  • Escucha las solicitudes enviadas a la dirección ' 224.0.1.75:5060 '
  • Cuando se inicia, se inicia el servidor web ( busybox httpd )
  • Le permite crear una configuración de teléfono simplificada
  • Le permite enviar a Yealink NOTIFICAR para reiniciar

El uso de un servidor PnP le permite usar "Enlaces únicos".


Supongamos que le damos el archivo por el enlace: 

 http://172.16.32.153:84/0015657322ff.cfg

El enlace directo habitual al archivo. Obviamente, esto NO es seguro. Conociendo la dirección MAC del teléfono y la dirección del servidor, puede intentar obtener una configuración con inicios de sesión y contraseñas.

Cuando se trabaja con un servidor PnP, es posible proporcionar un enlace único para cada solicitud de SUSCRIPCIÓN: 

 http://172.16.32.153:84/?mac=0015657322ff&hash=0a67f5290

Un ejemplo de una fórmula para calcular un hash: 

 hash = md5(MAC + DATE + PID)

PID es el ID de proceso del servidor PnP. Solo la raíz puede reconocerlo.
Elegir tal hash es prácticamente imposible.

Si se produce una apelación a través de un enlace no válido, entonces prohibimos la plaga.

Reiniciar Yealink NOTIFICAR significa sin autorización


Si, si, sin autorización .
No pude cerrar el dispositivo desde la versión actual del firmware desde esta posibilidad.

Solo ejecuta el comando 

 php -f MikoServerPnP.php socket_client_notify <IP_PBX> <PORT_SIP_PBX> <IP_PHONE> <PORT_PHONE>

Y el teléfono se reiniciará. Al colgar dicho comando en cron puede lograr un efecto aterrador. Por supuesto, esto es posible si conocemos la dirección IP y el puerto SIP del teléfono.

Configuración del servidor PnP


Ubicado en settings / settings.json 

 { "url": "http://<pbx_host>:<http_port>/", "http_port": 84, "pbx_host": "172.16.32.153", "pbx_sip_port": "5060", "vm_extension": "*001", "feature_transfer": "**" }

Lista blanca de MAC


Se puede describir en settings / mac_white.conf . Separador - avance de línea.

Lista negra de MAC


Se puede describir en settings / mac_black.conf . Separador - avance de línea.

Archivos de configuración del teléfono


Debe colocarse en el directorio configs .
Usando el servidor PnP, puede crear las configuraciones más simples para Yeakink y Snom: 

 php -f MikoServerPnP.php mk_config SIP_ACCAUNT SECRET MAC
