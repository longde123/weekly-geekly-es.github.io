Sayangnya, saya dihadapkan dengan fakta bahwa Skype mulai mengkonsumsi lalu lintas yang berlebihan baru-baru ini. Tentu saja, saya mengerti bahwa tidak ada pembaruan untuk 500+ MB yang dapat diunduh, dan ketika saya menutup messenger, unduhan berhenti dan tidak dilanjutkan setelah restart, tetapi setelah 10-12 jam kerja saya melihat situasi kedua. Selain itu, lalu lintas utama masuk, meskipun keluar hadir. Kemudian saya merasa sangat tidak nyaman.

Saya mencoba menjalankan perintah berikut (cukup masukkan di kotak dialog apa saja dan tekan Enter, teman bicara tidak akan melihat pesan ini):

Faktanya adalah bahwa setelah mencoba memeriksa aktivitas program pada perangkat lain, memang benar bahwa ini tidak dilakukan pada saat lalu lintas masuk sedang berlangsung (sayangnya, setelah memulai kembali, situasinya tidak terjadi lagi), setiap aktivitas mencurigakan berhenti.

Jadi saya berpikir sekarang apakah penyerang berhasil mendapatkan akses ke sistem saya atau tidak atau memperhatikan bahwa saya sedang memeriksa apa yang dilakukan Skype dalam sistem dan menghentikan aktivitas, karena ada alasan untuk mempercayai ini, karena baru-baru ini, ternyata, artikel keluar: Skype tidak dapat memperbaiki bug keamanan yang buruk tanpa menulis ulang kode besar-besaran :
Peneliti Stefan Kensek menemukan kerentanan serius pada messenger Microsoft yang memungkinkan penyerang mendapatkan akses penuh ke sistem operasi korban. Skema ini bekerja berdasarkan prinsip "pembajakan DLL" dan mengalihkan penginstal pembaruan Skype ke kode berbahaya, dan bukan ke file pembaruan yang diperlukan. Saat memasang versi baru, Skype menggunakan file yang dapat dieksekusi terpisah untuk komponen pembaruannya sendiri. Proses ini dapat dengan mudah dieksploitasi untuk tujuan jahat.

Seorang penyerang dapat "membajak" proses pembaruan dengan memuat DLL yang terinfeksi ke folder sementara. Pustaka palsu berperan sebagai sistem dan mengamankan file DLL yang dapat diedit oleh pengguna normal tanpa hak administrator (UXTheme.dll, misalnya). Pemasang pertama mendeteksi DLL jahat dan menginstal kode berbahaya pada sistem korban. Pekerjaan sudah selesai.

Peneliti menekankan bahwa di Windows Anda dapat menggunakan beberapa metode "pembajakan DLL" sekaligus. Dia juga mencatat bahwa sistem operasi lain, seperti macOS dan Linux, mungkin juga rentan terhadap serangan semacam itu. Kerentanan dapat dengan mudah dieksploitasi untuk berbagai tujuan, dan dalam kasus eksekusi yang sukses, hacker mendapatkan akses ke tingkat "Administrator on steroid". Dengan cara ini, ia dapat melakukan apa saja dengan komputer korban.

Microsoft belajar tentang kerentanan keamanan pada September tahun lalu, tetapi raksasa Perangkat Lunak memutuskan untuk tidak memperbaiki masalah ini, karena perawatannya akan memerlukan penulisan ulang sejumlah besar kode di dalam aplikasi. Sebagai gantinya, Microsoft memutuskan untuk menunda perbaikan sampai rilis versi baru Skype, yang mengarahkan semua sumber dayanya.

Apa status masalah ini sekarang, sayangnya, saya tidak tahu, tetapi perilaku Skype, terus terang, aneh dan saya tidak meninggalkan messenger tanpa pesan lagi.
