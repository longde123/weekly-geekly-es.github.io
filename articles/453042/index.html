Preguntas frecuentes sobre Windows Terminal Build 2019

La semana pasada, Microsoft celebró su conferencia Build 2019 en el Centro de Convenciones del Estado de Washington en Seattle . Build es un gran evento con la asistencia de varios miles de personas de todo el mundo para aprender todo sobre la tecnología actual, más nueva y futura orientada al desarrollador que viene de Microsoft.


¡Tuvimos el placer de conocer a muchos de ustedes en nuestro stand y responder a todas sus preguntas!



Preguntas frecuentes


A continuación se presentan algunas de las preguntas más frecuentes que hemos escuchado (y que aún escuchamos), junto con respuestas oficiales:


¿Cuándo y dónde puedo obtener la nueva Terminal?


  1. Puede clonar el código fuente de la Terminal desde GitHub en https://github.com/microsoft/terminal y compilarlo en su máquina.
    a. Nota: asegúrese de leer y seguir las instrucciones en la página REPO REPARE antes de intentar construir el proyecto. ¡Hay algunos requisitos previos y pasos de inicialización necesarios antes de que el proyecto se construya!
  2. Terminal como vista previa estará disponible para su descarga desde Microsoft Store para el verano de 2019.
  3. Nuestro objetivo es entregar Windows Terminal v1.0 para fines de 2019, pero trabajaremos con la comunidad para entregar v1.0 cuando Terminal tenga una calidad suficientemente alta.

¿La Terminal de Windows está reemplazando el Símbolo del sistema y / o PowerShell?


Para responder a esta pregunta, aclaremos algunos términos y conceptos:


  • Símbolo del sistema y PowerShell (como WSL / bash / etc. En * NIX) son shells, no terminales, y no tienen interfaz de usuario propia.
  • Cuando se ejecuta un shell / aplicación / herramienta de línea de comandos, Windows se inicia automáticamente y los conecta a las instancias de la Consola de Windows (si es necesario)
  • Windows Console es la aplicación de interfaz de usuario estándar "similar a un terminal" que se incluye en Windows y que los usuarios han utilizado durante los últimos 30 años al ejecutar herramientas de línea de comandos en Windows NT, 2000, XP, Vista, 7, 8 y 10


Entonces, la pregunta se vuelve a plantear como "¿Está Windows Terminal reemplazando a la consola?"


La respuesta es "No":


  • La consola de Windows continuará distribuyéndose dentro de Windows durante las próximas décadas para garantizar la compatibilidad con los muchos millones de scripts, aplicaciones y herramientas de línea de comandos existentes / heredados.
  • Windows Terminal se ejecutará junto con la Consola, pero probablemente se convertirá en la herramienta preferida de los usuarios que deseen ejecutar herramientas de línea de comandos en Windows
  • Windows Terminal se puede conectar a Símbolo del sistema y PowerShell, así como a cualquier otro shell / herramienta / aplicación de línea de comandos. Podrá abrir pestañas independientes conectadas a Símbolo del sistema, PowerShell, bash (a través de WSL o ssh) y cualquier otro shells / herramientas que elija

¿Cuándo puedo obtener la nueva fuente?


Pronto! No tenemos una línea de tiempo establecida, pero estamos trabajando ansiosamente para finalizar la fuente. Una vez que esté listo para ser lanzado, se abrirá en su propio repositorio.
