WSL 2 ahora está disponible en Windows Insiders

¡Nos complace anunciar que a partir de hoy puede probar el Subsistema de Windows para Linux 2 instalando Windows build 18917 en el anillo Insider Fast! En esta publicación de blog, cubriremos cómo comenzar, los nuevos comandos wsl.exe y algunos consejos importantes. La documentación completa sobre WSL 2 está disponible en nuestra página de documentos .




Comenzando con WSL 2


Estamos ansiosos por ver cómo comienza a usar WSL 2. Nuestro objetivo es hacer que WSL 2 se sienta igual que WSL 1, y esperamos escuchar sus comentarios sobre cómo podemos mejorar. La instalación de documentos de WSL 2 explica cómo comenzar a trabajar con WSL 2.


Hay algunos cambios en la experiencia del usuario que notará cuando empiece a usar WSL 2. Estos son los dos cambios más importantes en esta vista previa inicial.


Coloque sus archivos de Linux en su sistema de archivos raíz de Linux


Asegúrese de colocar los archivos a los que accederá con frecuencia con las aplicaciones de Linux dentro de su sistema de archivos raíz de Linux para disfrutar de los beneficios de rendimiento de los archivos. Entendemos que hemos pasado los últimos tres años diciéndole que coloque sus archivos en su unidad C cuando usa WSL 1, pero este no es el caso en WSL 2. Para disfrutar del acceso más rápido al sistema de archivos en WSL 2, estos archivos deben estar dentro del sistema de archivos raíz de Linux. También hemos hecho posible que las aplicaciones de Windows accedan al sistema de archivos raíz de Linux (como File Explorer! Intente ejecutar: explorer.exe . En el directorio de inicio de su distribución de Linux y vea qué sucede), lo que hará que esta transición sea mucho más fácil.


Acceda a sus aplicaciones de red Linux con una dirección IP dinámica en las compilaciones iniciales


WSL 2 incluye un gran cambio de arquitectura utilizando la tecnología de virtualización, y todavía estamos trabajando para mejorar el soporte de redes. Dado que WSL 2 ahora se ejecuta en una máquina virtual, deberá utilizar la dirección IP de esa VM para acceder a las aplicaciones de red de Linux desde Windows, y viceversa, necesitará la dirección IP del host de Windows para acceder a las aplicaciones de red de Windows desde Linux. ¡Nuestro objetivo es incluir la capacidad de WSL 2 para acceder a aplicaciones de red con localhost tan pronto como podamos! Puede encontrar todos los detalles y pasos sobre cómo hacerlo en nuestra documentación aquí .


Para leer más sobre los cambios en la experiencia del usuario, consulte nuestra documentación: Cambios en la experiencia del usuario entre WSL 1 y WSL 2 .


Nuevos comandos wsl


También hemos agregado algunos comandos nuevos para ayudarlo a controlar y ver sus versiones y distribuciones de WSL.


  • wsl --set-version <Distro> <Version>
    Use este comando para convertir una distribución para usar la arquitectura WSL 2 o la arquitectura WSL 1.


    : la distribución específica de Linux (por ejemplo, "Ubuntu")


    : 1 o 2 (para WSL 1 o 2)

  • wsl --set-default-version <Version>
    Cambia la versión de instalación predeterminada (WSL 1 o 2) para nuevas distribuciones.

  • wsl --shutdown
    Inmediatamente termina todas las distribuciones en ejecución y la máquina virtual de utilidad ligera WSL 2.


    La VM que alimenta las distribuciones de WSL 2 es algo que nuestro objetivo es administrar por completo para usted, por lo que lo activamos cuando lo necesita y lo cerramos cuando no lo necesita. Puede haber casos en los que desee cerrarlo manualmente, y este comando le permite hacerlo finalizando todas las distribuciones y cerrando la VM WSL 2.

  • wsl --list --quiet
    Solo enumere los nombres de distribución.


    Este comando es útil para las secuencias de comandos, ya que solo generará los nombres de las distribuciones que haya instalado sin mostrar otra información como la distribución predeterminada, las versiones, etc.

  • wsl --list --verbose
    Muestra información detallada sobre todas las distribuciones.


    Este comando enumera el nombre de cada distribución, en qué estado se encuentra la distribución y en qué versión se está ejecutando. También muestra las distribuciones predeterminadas con un asterisco.
