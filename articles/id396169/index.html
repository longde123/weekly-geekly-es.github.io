Skype sepenuhnya bergerak ke cloud

Dukungan pelanggan untuk OS yang lebih lama berakhir pada November 2016




Sejak awal keberadaannya, Skype bekerja pada prinsip P2P, yaitu, ia membangun koneksi langsung antara pengguna. Arsitektur Skype menyediakan kehadiran "super nodes" (supernode), yang merupakan pengguna dengan komputer dan saluran komunikasi yang cukup kuat. Melalui mereka, lalu lintas peserta lain dari jaringan P2P terlewati (kemampuan untuk menonaktifkan supernode pada PC Anda hanya muncul dengan Skype 3.0). Pada Desember 2010, Skype mulai menjadi tuan rumah beberapa supernode di cloud Amazon EC2 .

Setelah membeli Skype, Microsoft melanjutkan pekerjaannya. Pada 2012, ia hampir sepenuhnya memigrasi supernode ke servernya , yang dihosting di Azure cloud.

Jadi, pada 2012, infrastruktur lebih terpusat: alih-alih 48+ ribu supernode, jaringan P2P Skype mulai melewati lalu lintas melalui 10+ ribu supernode mega di hosting Microsoft. Perusahaan menjelaskan bahwa ini "secara signifikan meningkatkan keamanan sistem dan kualitas layanan, mengingat jumlah pengguna yang meningkat secara dramatis." Setiap supernode dalam sistem lama dapat melayani sekitar 800 pengguna, dan node Microsoft baru dapat menampung hingga 100 ribu koneksi.

Perubahan dalam infrastruktur Skype yang menjalankan Microsoft - transfer supernode lengkap ke cloud Azure dan transformasi mereka menjadi mega-supernode - telah menjadi perubahan paling dramatis dalam infrastruktur Skype dalam sejarah layanan ini.

Microsoft harus membantah tuduhan itukarena itu membuat perubahan pada arsitektur Skype untuk memberikan otoritas penegakan hukum akses yang lebih nyaman ke komunikasi pengguna. Perusahaan menekankan bahwa peralihan ke hosting internal "super node" tidak menyediakan untuk memonitor dan merekam panggilan. "Super nodes" membantu pelanggan Skype menemukan satu sama lain di jaringan sehingga mereka dapat terhubung untuk melakukan panggilan Skype.

Pada 2012, direktur pengembangan dan manajemen Skype Mark Gillet menjelaskanbahwa supernode hanya diperlukan agar klien Skype individual dapat menemukan satu sama lain di jaringan dan membuat koneksi, dan pertukaran data lebih lanjut di antara mereka sudah akan langsung dalam bentuk terenkripsi, tanpa keterlibatan server pusat. Supernode berpartisipasi dalam komunikasi hanya dalam situasi spesifik tertentu ketika diperlukan, misalnya, selama obrolan video grup, tempat host mengumpulkan video dari semua peserta dan menyiarkannya ke jaringan, yang mungkin tidak memiliki bandwidth saluran yang cukup.

Salah satu caranya, tetapi pada tahun 2016 dapat dinyatakan bahwa Microsoft sedang menyelesaikan transfer infrastruktur Skype ke cloud hosting. Kemarin, dalam blog perusahaan menerbitkan pesan, di mana pengembang berbicara tentang mengapa infrastruktur bermigrasi ke cloud. Menurut mereka, ini memastikan stabilitas komunikasi yang lebih tinggi, terutama fungsi-fungsi seperti berbagi file dan panggilan video, serta inovasi terbaru - obrolan video grup seluler , penerjemah Skype , dan bot Skype .

Jelas, fungsi-fungsi tersebut tidak dapat bekerja dalam arsitektur peer-to-peer biasa tanpa server pusat. Misalnya, transfer file sebelumnya hanya tersedia jika pengirim dan penerima online. Di versi baru klien, saat mentransfer melalui cloud, Anda dapat mengirim file, bahkan jika penerima saat ini terputus dari Jaringan.

"Transfer infrastruktur ke cloud masih berlangsung dan belum selesai," -menulis wakil presiden Skype dan Skype untuk Bisnis Gurdeep Pall (Gurdeep Pall). Dia menyebutkan beberapa kegagalan dalam Skype, termasuk kegagalan dalam sinkronisasi dan pemberitahuan yang tertunda. Perusahaan bekerja untuk memecahkan masalah teknis, membuat perubahan pada kode klien dan mendorong semua pengguna untuk menginstal versi terbaru dari klien , di mana bug ini diperbaiki dan bekerja dengan infrastruktur cloud baru didukung sepenuhnya.

Sehubungan dengan migrasi ke infrastruktur baru , rencana untuk dukungan versi untuk berbagai sistem operasi berubah .

Paket dukungan untuk platform berbeda ditunjukkan di bagian dukungan di situs web resmi.

Misalnya, dalam versi Skype untuk Linux, yang dirilis seminggu yang lalu, persyaratan berikut ditunjukkan: Ubuntu 10.04+, Debian 6.0+, Fedora 16+, OpenSUSE 12.1+, RAM 256 MB, kehadiran perangkat lunak tambahan: Qt 4.6, D-Bus 1.0.0, libasound 1.0.18, Pulse Audio 1.0 ( Pulse Audio 4.0 direkomendasikan untuk kinerja terbaik), BlueZ 4.0 (disarankan).

Klien untuk Android versi 6.2 dan lebih tinggi hanya bekerja dengan Android 4.0.3+ . Pengguna dengan versi Android yang lebih lama ditawarkan untuk menggunakan klien Skype 4, di mana tidak akan ada panggilan video grup dan fungsi terbaru lainnya. Hal yang sama berlaku untuk klien untuk iOS: klien terakhir hanya bekerja di iOS 8 dan di atasnya.

Dukungan Skype untuk Windows Phone 8, Windows Phone 8.1 dan Windows RT telah dihentikan pada 1 November 2016 .

Mulai 1 November 2016, dukungan untuk OS X versi lama juga berhenti.Klien baru hanya berfungsi pada Mac OS X 10.9 atau lebih tinggi. Pengguna lain didorong untuk menjalankan klien web dengan fungsionalitas terbatas.

Di masa depan, Microsoft bermaksud untuk fokus pada pengembangan aplikasi UWP (Universal Windows Platform) yang cepat dan produktif untuk Windows 10, aplikasi asli untuk iPhone dan Android, serta versi klien web untuk semua sistem operasi lain, seperti versi Windows, Mac dan Linux sebelumnya. . Klien web ini akan berjalan pada teknologi web ORTC atau WebRTC.





Microsoft mencatat bahwa memindahkan Skype ke cloud membutuhkan upaya teknis yang luar biasa, tetapi ini merupakan langkah yang mutlak diperlukan untuk memastikan pengembangan layanan di masa depan.

Sayangnya, Microsoft tidak mengatakan apa-apa tentang penerapan enkripsi ujung ke ujung. Protokol Skype adalah milik dan tidak berdokumen, kode program ditutup. Tidak ada yang tahu persis bagaimana komunikasi dienkripsi, bagaimana enkripsi yang andal digunakan, dan keterbatasan apa yang dimilikinya. Skype tetap menjadi semacam kotak hitam. Tidak ada antarmuka pemrograman (API) untuk menambahkan enkripsi Anda sendiri dan menggunakannya pada jaringan Skype.

Semua messenger menambahkan enkripsi end-to-end satu demi satu. Fungsi enkripsi menjadi wajib bagi pengirim pesan instan. Ini terutama benar sekarang ketikaPenyadapan massal lalu lintas Internet sedang diperkenalkan bahkan di Rusia .

Tanpa enkripsi yang andal, mungkin akan sulit bagi Skype untuk mempertahankan posisinya di pasar messenger dalam menghadapi persaingan semacam itu.
