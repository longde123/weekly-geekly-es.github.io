可弯曲的智能手机和其他Android Dev Summit信息



现在，Google正在运行Android Dev Summit，开幕式主题演讲已经举行。 通常说来的消息是针对Android开发人员的，但也有一些新闻引起了广大群众的关注：“弯曲智能手机”。 仔细观看了在线广播后，我们在主题演讲中介绍了此类设备的支持以及其他信息。

翻盖式归还


如您所见，智能手机的屏幕越来越大。 但是现在没有设备可以自己做了，它们使用了整个前端-下一步是什么？ 事实证明，借助折叠屏风可以将其对折。



同时发布了两个公告。 三星在活动中展示了一个带有“ Infinity Flex Display”屏幕的原型设备：它可以用作普通的智能手机，但显示屏没有那么大，但可以部署，并且内部隐藏了7.3英寸的屏幕。 Google在Android Dev Summit上宣布了对Android中这种方法的支持：例如，这种支持意味着您可以开始在折叠设备上观看视频，然后对其进行分解，然后它将在同一位置的大屏幕上继续播放。

三星智能手机计划于明年发布，谷歌增加了“等待其他制造商的型号”。 尽管情况尚处于原型阶段，但并非所有细节都一目了然（按字面意义：三星在黑暗中展示了其设备，无法完全看到它）。 但是无论如何，奇怪的是Google在如此早期就支持如此大胆的实验。 进行比较：Essential是世界上第一家发布“爆炸式”手机的公司（尚不清楚他们将如何获得它），它是自己制作的-只有在爆炸式爆炸声蔓延到各处时，Android才获得官方支持。

显然，谷歌确实相信“折叠”方法的未来。 似乎Android开发人员对在其应用程序中支持此功能的前景并不感到鼓舞。 在Android Dev Summit场景中听到了诸如“我们很兴奋”之类的字眼，听众却坐着这样的样子：




主题演讲还有什么？

过去和未来


首先，总结了10年Android的成果-从一切开始到未来。 有趣的是，这部分与我们的“ Android的10年” 枢纽非常相似：即使当我们切换到前面的内容时，我们已经描述了三个提到的开发方向中的两个。 但是还有一些奇怪的细节。

在谈到Android 1.0发行之前的时间时，现场直接意识到iPhone的输出极大地影响了正在发生的事情。 Google活动不仅响起了“ iPhone”一词，而且还响起了“ awesome”一词-地球没有打开。 但与此同时，该公司提供了自己的故事版本，与知名版本略有不同。 据她说，当iPhone出现在Google上时，他们已经在考虑两种不同的Android手机：它应该先释放一个快速按钮，然后再进行触摸。 iPhone的输出并未使公司对触摸屏很酷的事实大开眼界，而只是使它加快了已经在计划中的第二台设备上的工作。 相信吗？



未来的三个发展方向是什么？ 我们已经写过的其中两个是“智能手机智能手机”（AI，ML，仅此而已）和“智能手机以外的Android”（电视，汽车，手表）。 第三是“安全与福祉”：他们说，智能手机能够改善生活，而不是使生活恶化，这一点很重要。 也就是说，Android Pie的最新创新不限于此，该公司将其视为战略方向。


Android Studio和ChromeOS


就Android Studio而言，有一些常用词（“我们认为您需要加快构建速度”）和一个较小的发行版（版本3.3发布了beta 3版），但是对于那些关注IDE开发的人来说，这不足为奇。 但是有什么新信息：明年，Chrome操作系统将正式支持Android Studio。



几个月前，ChromeOS有了运行基于Linux的应用程序（因此也运行Android Studio）的机会，发烧友便开始在Android开发中使用Pixelbook：这是一款功能强大的笔记本电脑，直接来自Google，而当您在其上开发Android应用程序时，该应用程序本身可以直接安装在Pixelbook上。 但是，尽管这种方法具有吸引力，但尝试过的人表示：“到目前为止，一切都很艰难，ChromeOS稍后将真正开始适合开发。” 现在看来，这个时候到了。

考虑到有多少专业人士对当前的MacBooks不满意（问题键盘，有争议的触摸屏，价格仍然很高），高端Chromebook可以争夺Android开发人员的心。 虽然它也有自己的细微差别：去年的2018年Pixelbook没有更新，而新的Pixel Slate仍然基本上看起来像是平板电脑。


Google Play审核问题


在这两个主题之间的主题演讲中，他们尝试着不这样说：“如果您不合理地从Google Play删除了您的应用程序并且无法挑战它，那么请知道我们对您的看法，在峰会上还会有更多关于它的信息单独讨论。”

尽管这句话语气不动，但这些话语背后隐藏着许多紧张气氛。 最近，越来越多的故事浮出水面，说明独立开发人员如何在算法兴起的时候丢失了他们的应用程序（不违反规则），并且在尝试恢复公正性时，他们仅从机器人那里收到了自动消息。 结果，社区对此表示不满，呼吁组织有关Android Dev Summit的抗议活动。 主题演讲中的话被翻译成简单的语言是：“没有抗议，我们要人为地弄清楚，我们同意谈判。”

让我们看一下这些谈判的结果-也许在单独的枢纽文章中有一个要点，它将对情况进行更详细的分析。


应用内更新API


他们还讨论了一个新的API ，应用程序在使用过程中可以使用该API告诉用户“让我们更新我”。 有两种选择：通常是阻止内容的广告（这是修复关键错误的紧急更新），下面是不太烦人的广告，您可以通过该广告继续使用该应用程序。

一方面，这当然是有用的。 另一方面，社区中有人担心他们可能会使用它（“我看到独立开发人员如何向他的应用程序出售某种广告渣土，并强行迫使所有人进行更新”）。 对于第三种方法，开发人员无论如何都已经可以做到这一点，只需要大量的工作，而无需本地化。
