GitHub Package Registry admitirá paquetes Swift

El 10 de mayo, lanzamos una prueba beta limitada del Registro de paquetes GitHub, un servicio de administración de paquetes que facilita la publicación de paquetes públicos o privados junto al código fuente. Actualmente, el servicio admite las herramientas familiares de administración de paquetes: JavaScript (npm), Java (Maven), Ruby (RubyGems), .NET (NuGet), imágenes Docker y mucho más.

Nos complace anunciar que agregaremos la compatibilidad con paquetes Swift al Registro de paquetes GitHub. Los paquetes Swift facilitan compartir sus bibliotecas y código fuente en sus propios proyectos y con la comunidad Swift. Trabajaremos en esto en asociación con los chicos de Apple.



Este artículo del blog de GitHub

Disponible en GitHub, Swift Package Manager es una herramienta multiplataforma única para construir, ejecutar, probar y empaquetar el código Swift. Las configuraciones están escritas en Swift, lo que facilita el establecimiento de objetivos, la publicidad de productos y la administración de dependencias de paquetes. Juntos, Swift Package Manager y GitHub Package Registry facilitan la publicación y administración de sus paquetes Swift.

Para los desarrolladores móviles, es importante tener mejores herramientas para ser más productivos. Con el desarrollo del ecosistema Swift, nos complace trabajar con el equipo de Apple para ayudar a crear nuevos flujos de trabajo para los desarrolladores de Swift.

Desde el lanzamiento del Registro de paquetes de GitHub, hemos visto una fuerte participación de la comunidad en el trabajo con la herramienta. Durante el período beta, buscamos aprender de la comunidad cómo el Registry Package satisface diversas necesidades y qué podemos hacer para mejorarlo. Si no ha probado GitHub Package Registry, puede solicitar una versión beta aquí .
