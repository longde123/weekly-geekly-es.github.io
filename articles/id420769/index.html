Looking.House - lebih dari 150 poin Looking Glass di satu situs


Hampir sejak awal keberadaan VPS Search, ia memiliki bagian dengan Looking Glass dots. Saya menulis tentang hal itu pada Januari 2014 dan pada saat itu 23 titik berada di peta. Fungsinya agak sedikit, titik-titik secara berkala "mati", dan tidak ada waktu untuk mengikuti mereka. Plus, untuk beberapa waktu sekarang, ada skrip untuk mengatur titik Looking Glass Anda, yang digunakan oleh banyak penyedia hosting. Namun, skrip ini memiliki 2 kelemahan: pertama, host harus menginstalnya sendiri di server (ya, ini juga kelemahan), dan kedua, tidak ada database terpusat yang mengumpulkan informasi tentang semua poin dari host yang berbeda. Tahun ini saya memutuskan untuk berurusan dengan bagian ini, dan sebagai hasilnya saya mendapatkan situs yang belum pernah saya lihat analognya, dan saya ingin mempresentasikan proyek LOOKING.HOUSE dengan senang hati .



Saya ingin segera mencatat bahwa saya memutuskan untuk membuat situs ini dalam bahasa Inggris sehingga berguna tidak hanya untuk pengguna yang berbahasa Rusia, apalagi, praktis tidak ada teks di dalamnya, sehingga bahkan tingkat minimal bahasa Inggris tidak akan menjadi penghalang untuk penggunaannya.

Saat ini, 156 poin dari 85 hoster di 40 negara telah ditambahkan ke situs. Di halaman utama Anda dapat melihat peta dengan titik-titik yang tercetak di atasnya. Jika kami tahu alamat pasti pusat data, maka kami menandainya di peta, jika tidak, kami menunjukkan titik di kota tempat server berada. Jika Anda mengklik suatu titik, Anda dapat melihat lokasinya dan, jika informasi ini diberikan kepada kami, nama pusat data. Di bawah ini Anda dapat melihat daftar negara, memungkinkan Anda untuk melihat poin yang tersedia di negara tertentu.

Untuk menambahkan satu poin, hoster hanya perlu memberi tahu kami tentang keinginannya dan menyediakan server virtual, dan kami akan melakukan pekerjaannya sendiri. Pada alamat IP poin adalah versi yang disederhanakan (dalam hal desain, tetapi bukan fungsionalitas) dari skrip, yang hoster dapat digunakan di situsnya sebagai bingkai atau tautan. Dengan demikian, 2 masalah diselesaikan dengan segera: hoster tidak perlu melakukan apa-apa, tetapi kita dapat menyusun peta dengan semua poin.



Dari sudut pandang teknis, setiap titik adalah server virtual dengan karakteristik minimal (sebagai aturan, RAM 256-512 MB, HDD 5-20 GB, 1 inti prosesor). Karakteristik ini lebih dari cukup untuk bekerja, karena selain nginx, tidak ada perangkat lunak tambahan yang diinstal untuk memeriksa kecepatan unduhan di server.



Pada halaman poin Anda dapat melihat informasi terperinci tentang hal itu: hoster yang menyediakannya, peta dengan lokasi, alamat IP dan IPv6 (jika didukung), kemampuan untuk mengunduh file berukuran 10/100/1000 MB untuk memeriksa kecepatan, dan fungsi jaringan: host, ping, ping6, traceroute, traceroute6, mtr, mtr6.



Pengguna dapat menjalankan setiap fungsi secara individual atau segera menjalankan seluruh daftar untuk dijalankan. Saya harus segera mengatakan bahwa di sini Anda dapat menemukan beberapa kesalahan tidak kritis, misalnya, tidak ada yang akan bersumpah ketika pengguna mencoba menjalankan fungsi ping6 untuk alamat IPv4. Untuk kenyamanan, hasil ping diuraikan, dan statistik tersedia untuk pengguna dalam versi yang lebih nyaman.

Versi situs ini masih cukup mentah, sehingga tidak memiliki beberapa fitur yang saya rencanakan untuk ditambahkan di masa depan. Dari ide-ide saya dapat memberitahu Anda hal berikut:

  • kemampuan untuk menentukan berbagai tombol untuk menjalankan fungsi;
  • kemampuan untuk melakukan satu fungsi dari beberapa titik sekaligus;
  • kemampuan untuk memeriksa kecepatan mengunduh file ke server;
  • kemampuan untuk memeriksa kecepatan langsung dari browser;
  • akun pribadi untuk menyimpan statistik pada pelaksanaan perintah;
  • menampilkan statistik untuk masing-masing poin dalam periode yang panjang.

Terlepas dari kenyataan bahwa saat ini lebih dari 150 poin telah ditambahkan ke situs, kami akan selalu senang dengan hosters baru dan poin mereka. Kami mengkonfigurasi server virtual sepenuhnya independen, karakteristik server minimum untuk titik: 256 MB RAM, 5 GB disk, alamat IPv4 khusus, akses root ke server.

Juga tersedia bagi pengguna adalah fungsi memperoleh informasi tentang alamat IP dan daftar catatan DNS domain .





Saya akan senang mendengar umpan balik dan saran tentang fitur baru.
