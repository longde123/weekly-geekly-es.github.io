4 maneiras de economizar em backups na nuvem


O backup de máquinas virtuais é uma das áreas que devem receber atenção especial ao otimizar as despesas da empresa. Explicamos como você pode configurar backups na nuvem e salvar seu orçamento.

Os bancos de dados são um ativo valioso para qualquer empresa. De várias maneiras, as máquinas virtuais se tornaram demandadas. Os usuários podem trabalhar em um ambiente virtual que fornece proteção contra corrupção de dados físicos e vazamento de informações confidenciais.

A maioria das grandes e médias empresas depende de uma maneira ou de outra da VM. Eles armazenam uma enorme quantidade de informações críticas. Portanto, é tão importante cuidar da criação de backups para que um dia não ocorra "oops" e, durante anos, o banco de dados reabastecido não se torne repentinamente danificado ou inacessível.

Como regra, as empresas fazem backup de suas VMs e as armazenam em data centers separados. E se de repente o centro de processamento de dados principal falhar repentinamente, será possível recuperar-se rapidamente do backup. Idealmente, quando o backup é armazenado em diferentes datacenters, como o Cloud4Y . No entanto, a maioria dos provedores não pode oferecer esse serviço ou pedir dinheiro adicional. Como resultado, o armazenamento de backup resulta em um centavo tão barato.

No entanto, o uso inteligente dos recursos da nuvem pode reduzir a carga financeira.

Por que exatamente a nuvem?


Os backups de VM são convenientemente armazenados em plataformas na nuvem. Existem muitas soluções no mercado que simplificam o processo de backup e restauração de máquinas virtuais. Com a ajuda deles, você pode organizar a recuperação ininterrupta de dados a partir de máquinas virtuais e fornecer serviço estável para aplicativos que dependem desses dados.

O processo de backup pode ser automatizado dependendo de quais arquivos e com que frequência você precisa salvar os dados. A "nuvem" não possui nenhuma estrutura rígida. A empresa pode escolher a funcionalidade e o desempenho que atendem às suas necessidades de negócios e pagar apenas pelos recursos consumidos.

A infraestrutura local não tem essa oportunidade. Você precisa pagar imediatamente por todos os equipamentos (mesmo ociosos) e, se precisar aumentar a produtividade, precisará comprar mais servidores, o que gera custos mais altos. O Cloud4Y oferece quatro maneiras de reduzir os custos de backup do banco de dados.

Então, como economizar?


Cópia incremental

A empresa deve fazer backup regularmente de dados críticos. Mas esses dados aumentam com o tempo. Como resultado, cada backup subsequente ocupa mais espaço e requer mais tempo para carregar no repositório. Você pode simplificar o procedimento armazenando backups incrementais.

A abordagem incremental pressupõe que você faça backup apenas uma vez ou em determinados intervalos (dependendo da estratégia de backup). Cada backup subsequente contém apenas as alterações feitas no backup original. Como os backups são menos comuns e o backup apenas de novas alterações, as organizações não precisam pagar pela transferência de grandes dados na nuvem.

Limitar arquivos ou seções de troca

Às vezes, a RAM de uma máquina virtual pode não ser suficiente para armazenar aplicativos e dados do sistema operacional. Nesse caso, o sistema operacional ocupa parte do disco rígido para armazenar dados adicionais. Esses dados são chamados de arquivo de troca ou partição de troca no Windows e Linux, respectivamente.

Normalmente, os arquivos de troca são 1,5 vezes maiores que a RAM. Os dados nesses arquivos são alterados regularmente. E toda vez que um backup é feito, esses arquivos também são copiados. Portanto, seria melhor excluir esses arquivos do backup. Eles ocuparão muito espaço na nuvem, pois o sistema os salvará a cada backup (os arquivos estão constantemente mudando!).

Em geral, o resultado final é fazer backup apenas dos dados que a empresa realmente precisa. E arquivos desnecessários, como um arquivo de troca, não devem ser copiados.

Duplicação e arquivamento de backup

Os backups de máquinas virtuais pesam muito, então você precisa reservar mais espaço na "nuvem". Portanto, você pode economizar reduzindo o volume de backups. E aqui a desduplicação pode ajudar. Esse é o processo de copiar apenas blocos de dados alterados e substituir cópias de blocos inalterados por substituí-los por um link para os blocos originais. Você também pode usar todos os tipos de arquivadores para compactar o backup final e economizar ainda mais memória.

Este tópico é especialmente relevante se você seguir a regra “3-2-1” em relação ao armazenamento de backups. A regra diz que, para garantir o armazenamento de dados confiável, você deve ter pelo menos TRÊS backups armazenados em DOIS formatos de armazenamento diferentes, enquanto UMA das cópias deve ser armazenada fora do armazenamento principal.

Esse princípio de garantir a tolerância a falhas pressupõe armazenamento redundante de dados; portanto, uma diminuição no volume de backup ficará claramente fora do lugar.

Política de armazenamento GFS (avô-pai-filho)

Como o procedimento para criar e armazenar backups é organizado na maioria das empresas? Mas de jeito nenhum! As organizações fazem backup e ... esquecem delas. Por meses ou até anos. Isso leva a custos de dados desnecessários que nunca são usados. A melhor maneira de lidar com isso é usar políticas de retenção. Essas políticas determinam quantos backups podem ser armazenados na nuvem ao mesmo tempo.

A política mais simples de armazenamento de backup é explicada pelo princípio de “primeiro a chegar, primeiro a ir”. Com essa política, um certo número de backups é salvo e, após atingir o limite, o mais antigo é excluído para liberar espaço para os mais novos. Mas essa estratégia não é totalmente eficaz, especialmente se você precisar garantir o máximo de pontos de recuperação no menor volume de armazenamento possível. Além disso, existem regulamentos legislativos e corporativos que exigem armazenamento de dados a longo prazo.

Esse problema pode ser resolvido usando a política GFS (Grandfather-Father-Son). "Filho" é o backup mais frequente. Por exemplo, diariamente. E "avô" é o mais raro, por exemplo, mensalmente. E toda vez que um novo backup diário é criado, ela se torna filho de um backup semanal na semana anterior. Esse modelo oferece à empresa mais pontos de recuperação com o mesmo espaço de armazenamento limitado.

Se você precisar armazenar informações por muito tempo, existem muitas, mas nunca são realmente solicitadas, você pode usar o chamado armazenamento a frio. O custo de armazenamento de dados é baixo, mas se a empresa solicitar esses dados, você terá que pagar. É como um armário escuro distante. Há muitas coisas nele que não serão nada em 10-20-50 anos. Mas até chegar a um, você passará muito tempo. No Cloud4Y, esse armazenamento era chamado " Arquivamento ".

Conclusão


O backup é um elemento essencial de segurança para qualquer empresa. Manter backups na nuvem é muito conveniente, mas às vezes o serviço é caro. Usando os métodos listados por nós, você pode reduzir as despesas mensais da empresa.
