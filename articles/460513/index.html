Flutter 1.7: novedades en la versión del 10 de julio de 2019

No hace mucho tiempo, hubo un lanzamiento de Flutter en Google I / O, pero el equipo de Flutter lanzó nuevamente una nueva versión con elementos nuevos e interesantes que no pude evitar compartir.


Foto de fondo libre desde aquí

De alguna manera sucedió que me sumergí en el desarrollo en Dart + Flutter. Además del hecho de que la aplicación funciona inmediatamente en iOS y Android y, al mismo tiempo, se genera el código nativo, me gustó la apertura del código (puede profundizar en las clases y ver el código del núcleo) y la apertura de la comunidad. Como contribución a la comunidad, estoy publicando esta versión gratuita de un artículo sobre un nuevo lanzamiento de los desarrolladores.

Soporte de AndroidX para nuevas aplicaciones



En principio, antes, la aplicación migraba a AndroidX sin problemas , pero ahora está allí de inmediato. Tuve que migrar todos mis proyectos, por lo que al crear otros nuevos, planeo usar esta función. Para hacer esto, al crear la aplicación, simplemente usamos el flutter create project_name --androidx

Soporte para paquetes de aplicaciones de Android y aplicaciones de Android de 64 bits


A partir del 1 de agosto de 2019, las aplicaciones de Android que usan el código nativo y usan Android 9 Pie deberán proporcionar una versión de 64 bits además de la de 32 bits. Flutter ya admite la creación de aplicaciones de Android de 64 bits. Pero, comenzando con la versión 1.7, se ha agregado soporte para crear los paquetes de aplicaciones de Android inmediatamente para 2 versiones.

Personalmente, me encontré con esto hace una semana, cuando al generar el archivo .apk ejecutable para Android cuando en lugar de 12MB, de repente se convirtió en 25MB. Después de eso generé un paquete y vítores, el tamaño es normal nuevamente.



¿Qué es el paquete de aplicaciones aquí ?

Nuevos widgets


Dado que Google, si entiendo correctamente, está cambiando al concepto de superficies, en lugar del concepto de aplicaciones, prestan mucha atención a los widgets que representan la funcionalidad para el usuario.

En esta versión, se ha agregado un nuevo control deslizante para rangos (muy probablemente para usar en sus dispositivos de temperatura), en el que nosotros, como desarrolladores, podemos entrar y hacer nuestras propias cosas hermosas.



Fuentes


Aunque soy desarrollador, también entiendo la importancia de las fuentes. Además, las aplicaciones se escriben inmediatamente para iOS y Android. La nueva versión mejora el manejo de texto para iOS . Se ha realizado una gran actualización para el soporte de tipografía enriquecida, que incluye varios estilos de dígitos, estilos, etc. Un ejemplo en github .



Mayor número de ejemplos a partir de los cuales puede crear código


A menudo cuelgo durante mucho tiempo ejemplos para tomar técnicas de programación desde allí o simplemente copiar y pegar el código. Una característica maravillosa, puede crear un proyecto a partir de un ejemplo e inmediatamente ejecutar Ejecutar. El equipo de Flutter escribió que el número de tales ejemplos ha aumentado.

flutter create --sample=material.AppBar.1 mysample

Si el ejemplo se puede generar de esta manera, aparece una pestaña adicional.
