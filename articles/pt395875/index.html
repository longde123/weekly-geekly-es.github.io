Remoção de linhas telefônicas usando gateways VoIP da Grandstream

Os clientes costumam entrar em contato conosco com um problema bastante comum. Em geral, pode ser expresso da seguinte forma: a empresa utiliza uma central telefônica analógica, à qual é necessário conectar alguns usuários de um local remoto. Pode ser um escritório vizinho, uma cidade vizinha ou até outro país - o principal é que a rede telefônica não chega até lá, mas a LAN alcança ou há acesso à Internet. Hoje, resolveremos esse problema com a ajuda dos gateways VoIP da série Grandstream GXW4xxx.



De fato, esta tarefa tem muitas variações. Por exemplo, encaminhando linhas PSTN para outro local ou combinando as capacidades de numeração de duas trocas analógicas remotas - muito semelhantes em significado e solução para o problema. Portanto, para não introduzir confusão desnecessária, determinaremos imediatamente os dados iniciais e tomaremos a seguinte ilustração:


Existe uma sede da empresa na qual a central telefônica analógica está localizada, com a numeração de telefones internos 2XX. Há também um escritório remoto com quatro funcionários que precisam atribuir os números de ramal 301-304. É necessário oferecer aos funcionários de diferentes escritórios uma oportunidade de ligar diretamente um para o outro usando um número abreviado interno.

Na sede da empresa, colocaremos um gateway Grandstream VoIP com portas FXO. Pode ser um modelo GXW4104 (4 portas) ou GXW4108 (8 portas). E no escritório remoto - um gateway com portas FXS - GXW4004 (4 portas) ou GXW4008 (8 portas). Para simplificar, assumimos que trabalhamos com os gateways de quatro portas GXW4104 e GXW4004, respectivamente.

Selecionamos quatro portas internas em nosso PABX analógico e atribuímos números internos a elas - 301-304. Conectamos o Grandstream GXW4104 à rede local, atribuímos um endereço IP estático e conectamos as portas internas do PBX às portas do gateway (nº 301 - com a primeira porta do gateway, nº 302 - com a segunda, etc.). Na filial remota, também atribuímos um endereço estático ao gateway GXW4004 e conectamos quatro telefones analógicos a ele.

Agora precisamos resolver duas subtarefas:
  1. Permitir que os assinantes da sede (№201, №202) liguem para os assinantes da sede remota (№301 - 304)
  2. Permitir que os assinantes do escritório remoto liguem para os assinantes da sede da empresa.

Todas as outras configurações do gateway GXW4104 podem ser deixadas no padrão. E essas configurações são suficientes para organizar a comunicação dos assinantes da matriz com os assinantes da filial. Sim, o gateway remoto GXW4004 nem precisa ser configurado, pois escuta as portas 5060, 5062, 5064 e 5066 por padrão e converte as mensagens enviadas para essas portas e as envia para as portas 1-4, respectivamente. As portas SIP ímpares 5061, 5063, 5065 e 5067 no gateway FXS são padronizadas como SIP sobre TLS.

No entanto, também somos obrigados a resolver o segundo problema - organizar a comunicação dos assinantes da filial remota com os assinantes da matriz.   1. No campo Stage Method (1/2) , escreva ch1-4: 1; Isso significa que, para todos os quatro canais, uma chamada sem discagem de ramal será usada. Vários escritórios remotos


Vamos mudar um pouco a tarefa. Suponha que precisamos conectar não uma, mas várias filiais remotas ao PBX central da empresa. Sejam três pontos de venda com um assinante em cada um: em


vez de gateways de quatro portas, seria mais lógico colocar adaptadores SIP de porta única baratos, por exemplo, Grandstream HT701, nesses pontos. Como, então, as configurações no gateway GXW4104 serão alteradas? Para fazer uma chamada de assinantes da matriz para assinantes remotos, não é mais necessário usar portas SIP diferentes, pois agora as chamadas para vários assinantes remotos serão roteadas para diferentes endereços IP. E aqui estamos diante do primeiro problema - o número de servidores SIP que podem ser configurados no GXW4104 (e até no GXW4108) é limitado a três, portanto, mais de três locais não podem ser conectados através de um gateway FXO.

As chamadas efetuadas da central de câmbio para locais remotos foram ativadas, pois cada HT701 escuta sua porta padrão 5060 e, quando uma chamada chega, inicia uma chamada para o telefone conectado a uma única porta FXS. Para que as chamadas sigam o contrário, é necessário configurar cada Grandstream HT701. Nós vamos para a interface web do gateway HT-701 e para a seção FXS PORT .

  1. No campo Servidor SIP Primário , especifique o endereço IP do FXO do gateway Grandstream GXW4104.

  3. No campo Prefixo do plano de discagem , especifique 991 para o primeiro ponto (telefone 301), 992 para o segundo (302) e 993 para o terceiro (303). Com base nesse prefixo, o gateway FXO da matriz entenderá qual porta deve enviar a chamada ao PBX.

Usamos telefones IP


No problema em consideração, o gateway FXO instalado na matriz é um elemento-chave que não podemos recusar, pois a central de trocas, de acordo com as condições da tarefa, não suporta o protocolo SIP. Se a substituição de uma central telefônica central é um procedimento trabalhoso e caro, a substituição de telefones analógicos por dispositivos habilitados para SIP em filiais remotas é bastante fácil e muitos de nossos clientes fazem isso. Isso simplifica um pouco o esquema: As


configurações do gateway GXW4104 não mudam em comparação com o exemplo anterior. Vamos considerar as configurações do telefone IP usando o Grandstream GXP1620 como exemplo . Abra a interface da Web e vá para a seção Contas - Conta 1 - Configurações gerais .

  1. No campo Nome da conta , especifique o número de telefone - 301, 302 ou 303.
  2. SIP Server IP FXO Grandstream GXW4104.



  3. Account 1 — SIP Settings — Basic Settings SIP Registration No.



  4. Account 1 — Call Settings Dial Plan Prefix 991 301, 992 – 302 993 303.




Isso conclui a configuração do telefone IP - agora você pode fazer chamadas para os telefones dos assinantes de PBX central. Note-se que nem todos os telefones podem funcionar nesta configuração. O principal requisito para os dispositivos é a capacidade de fazer chamadas sem registrar. Além disso, os telefones devem poder adicionar um prefixo ao número discado. O segundo requisito não é obrigatório, pois em vez de usar o plano de discagem, você pode rotear chamadas de diferentes telefones IP para diferentes portas SIP do gateway FXO, o que permite que o último envie corretamente as chamadas para o PBX. Você também pode usar a discagem em dois estágios, o que, no entanto, é menos conveniente. As marcas mais populares de telefones IP, como Grandstream, Yealink, Fanvil, Linksys atendem aos requisitos especificados e suportam o trabalho nesta configuração.
