Halo yang disana.

Banyak orang ingin memulai pemrograman aplikasi untuk Android, tetapi mereka memilih untuk tidak menggunakan Android Studio dan / atau Java. Mengapa Karena ini adalah pembunuhan yang berlebihan. "Aku hanya ingin membuat Snake dan tidak lebih!"



Ayo ular tanpa java! (dengan bonus di akhir)

Mengapa membuat tutorial ular lagi?
Jika Anda adalah pembuat kode python dan ingin mempelajari gamedev untuk android, Anda harus sudah googled "snake on android" dan menemukan ini (Bahasa Indonesia) atau terjemahannya (Rus) . Saya juga. Sayangnya, saya menemukan artikel itu sangat tidak berguna karena:

Kode mereka buruk

Masalah kecil:

  1. Menggunakan "ubin" dan "kepala" alih-alih "ubin" atau "sel" tidak masuk akal bagi saya. Kepala tidak cukup berbeda dari ubin menjadi variabel yang berbeda.
  2. Clock.schedule untuk self.update dipanggil dari ... self.update.
  3. Playground kelas dua diimplementasikan di awal, sementara SnakeApp kelas satu diimplementasikan di akhir.
  4. Nama untuk arah ("atas", "turun", ...) digunakan sebagai ganti vektor ((0, 1), (1, 0) ...).

Masalah utama:

  1. Sebagian besar objek dinamis (misalnya, buah) dilampirkan ke file kv, jadi Anda tidak dapat membuat lebih dari 1 apel karena Anda harus menulis ulang bagian ini
  2. Logika yang aneh untuk gerakan ular bukannya gerakan "sel-setelah-sel".
  3. Kode ini sangat panjang menjadi lebih dari 350 baris.

Artikel ini tidak jelas untuk pemula

Ini adalah pendapat PRIBADI saya. Selain itu, saya tidak menjamin bahwa tutorial saya akan lebih menarik dan jelas. Tetapi saya akan melakukan yang terbaik dan, untuk artikel saya, menjamin bahwa:

  1. Kode ini pendek
  2. Ular itu baik
  3. Instruksi akan memiliki implementasi langkah-demi-langkah yang jelas dan perkembangan halus dari "halo, dunia" menjadi ular siap dengan langkah-langkah transisi.

Metode yang tidak perlu


Mari kita coba untuk memindahkannya. Untuk melakukan itu, kita harus menambahkan fungsi Form.update dan menjadwalkannya. 

 from kivy.app import App from kivy.uix.widget import Widget from kivy.clock import Clock class Cell(Widget): def __init__(self, x, y, size): super().__init__() self.size = (size, size) self.pos = (x, y) class Form(Widget): def __init__(self): super().__init__() self.cell = Cell(100, 100, 30) self.add_widget(self.cell) def start(self): Clock.schedule_interval(self.update, 0.01) def update(self, _): self.cell.pos = (self.cell.pos[0] + 2, self.cell.pos[1] + 3) class WormApp(App): def build(self): self.form = Form() self.form.start() return self.form if __name__ == '__main__': WormApp().run()

Bagian bonus - gerakan halus


Karena langkah worm sama dengan cell_size, itu tidak semulus itu. Tapi kami ingin membuatnya sesering mungkin, tanpa menulis ulang seluruh logika permainan. Jadi, kita perlu membuat mekanisme menggerakkan pose grafis kita tetapi bukan pose kita yang sebenarnya. Jadi, saya menulis file sederhana:

smooth.py 

 from kivy.clock import Clock import time class Timing: @staticmethod def linear(x): return x class Smooth: def __init__(self, interval=1.0/60.0): self.objs = [] self.running = False self.interval = interval def run(self): if self.running: return self.running = True Clock.schedule_interval(self.update, self.interval) def stop(self): if not self.running: return self.running = False Clock.unschedule(self.update) def setattr(self, obj, attr, value): exec("obj." + attr + " = " + str(value)) def getattr(self, obj, attr): return float(eval("obj." + attr)) def update(self, _): cur_time = time.time() for line in self.objs: obj, prop_name_x, prop_name_y, from_x, from_y, to_x, to_y, start_time, period, timing = line time_gone = cur_time - start_time if time_gone >= period: self.setattr(obj, prop_name_x, to_x) self.setattr(obj, prop_name_y, to_y) self.objs.remove(line) else: share = time_gone / period acs = timing(share) self.setattr(obj, prop_name_x, from_x * (1 - acs) + to_x * acs) self.setattr(obj, prop_name_y, from_y * (1 - acs) + to_y * acs) if len(self.objs) == 0: self.stop() def move_to(self, obj, prop_name_x, prop_name_y, to_x, to_y, t, timing=Timing.linear): self.objs.append((obj, prop_name_x, prop_name_y, self.getattr(obj, prop_name_x), self.getattr(obj, prop_name_y), to_x, to_y, time.time(), t, timing)) self.run() class XSmooth(Smooth): def __init__(self, props, timing=Timing.linear, *args, **kwargs): super().__init__(*args, **kwargs) self.props = props self.timing = timing def move_to(self, obj, to_x, to_y, t): super().move_to(obj, *self.props, to_x, to_y, t, timing=self.timing)
