GitHub Package Registry unterstützt Swift-Pakete

Am 10. Mai haben wir die eingeschränkte Beta von GitHub Package Registry angekündigt , einem Paketverwaltungsdienst, mit dem öffentliche oder private Pakete einfach neben Ihrem Quellcode veröffentlicht werden können. Derzeit werden bekannte Tools zur Paketverwaltung unterstützt: JavaScript (npm), Java (Maven), Ruby (RubyGems), .NET (NuGet) und Docker-Images. Weitere werden folgen.

Heute freuen wir uns, Ihnen mitteilen zu können, dass wir der GitHub Package Registry Unterstützung für Swift-Pakete hinzufügen werden. Mit Swift-Paketen können Sie Ihre Bibliotheken und Ihren Quellcode problemlos für Ihre Projekte und für die Swift-Community freigeben.



Swift Package Manager ist auf GitHub verfügbar und ein einziges plattformübergreifendes Tool zum Erstellen, Ausführen, Testen und Packen Ihres Swift-Codes. Paketkonfigurationen sind in Swift geschrieben, sodass Sie Ziele einfach konfigurieren, Produkte deklarieren und Paketabhängigkeiten verwalten können. Mit dem Swift Package Manager und der GitHub Package Registry können Sie Ihre Swift-Pakete noch einfacher veröffentlichen und verwalten.

Für mobile Entwickler ist es wichtig, über die besten Tools zu verfügen, um produktiver zu sein. Mit dem Wachstum des Swift-Ökosystems freuen wir uns, mit dem Apple-Team zusammenzuarbeiten, um neue Workflows für Swift-Entwickler zu erstellen.

Seit dem Start waren wir erstaunt, wie aufregend es war, mit der GitHub-Paketregistrierung zu beginnen. Während dieser Beta-Phase möchten wir von Gemeinden und Ökosystemen lernen, wie es Ihren Anforderungen entspricht und was wir tun können, um es noch besser zu machen. Wenn Sie dies noch nicht getan haben, können Sie sich jetzt für die eingeschränkte Beta anmelden .
