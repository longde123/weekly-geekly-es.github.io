ADAM-3600-多功能工业控制器



ADAM-3600控制器是用于工业设施的远程控制，数据收集，监视和过程自动化的通用设备。
同时，它可用作对象（ USO ），数据采集和传输设备（ USPD ）和可编程逻辑控制器（ PLC ）的通信设备。
它具有用于I / O端口的扩展模块，以及用于Wi-Fi，4G / 3G / GPRS，ZigBee，GPS的无线接口模块。

该设备在RealTime Linux操作系统和Cortex A8处理器上运行。 支持开放式行业协议ModBus，DNP3，MQTT，HTTP REST API。 它可以集成到任何SCADA系统中 。 它还支持集成到云IoT系统中，例如Microsoft Azure IoT中心 ， Amazon AWS IoT等。

在本文中，我们将分析该设备的主要功能，并了解在工业控制系统的构建中使用控制器的场景。

技术指标


处理器： Cortex A8 600MHz
内存： DDR3L 256MB
USB 2.0主机： 1个
VGA： 1
串口： 1x RS232 / 485，2x RS485
以太网端口： 2
数字输入： 8（可扩展至40）
数字输出： 4（可扩展至20）
模拟输入： 8（可扩展至24）
模拟输出： 0（扩展到16）
mPCIe插槽： 1 x半尺寸，1 x全尺寸
I / O扩展插槽： 4
无线接口： Wi-Fi，3G，4G GPRS，ZigBee（可选）
自供电实时时钟：是
SD卡：是
温度范围： -40至70°C
电源： 8至30V

端口扩展模块


I / O引脚的数量可以根据客户的需求扩展。 ADAM-3600控制器中最多可以安装4个接口扩展模块。 模块共有6种类型：数字量输入/输出，模拟量输入/输出，热电偶（TC）和电阻传感器（RTD）的输入。

安装在前面板上的接口扩展模块

无线接口


借助mPCIe端口， Zigbee，Wi-Fi，3G / 4G / GPRS，GPS无线接口，可以轻松地将其添加到控制器中。 可以同时安装两个mPCIe模块：一个完整​​尺寸的模块和一个一半尺寸的模块。 支持两个品牌的研华模块，以及由华为，Quectel等制造的第三方模块。


根据天线类型，将安装适当的连接器：SMA或RP-SMA。 天线端子通过标准的U.FL连接器连接到板上。

网络功能


ADAM-3600板上有两个独立的以太网端口，可以在交换模式下使用，也可以分为WAN和LAN。 这对于以菊花链拓扑结构串联连接多个控制器非常方便，并且不需要在长度超过100米的线路上安装其他开关。


两个LAN端口可让您串联连接设备

故障转移模式和多WAN


对于远程关键基础设施，通信丢失可能会造成重大损失。 因此，为了提高通信的可靠性， ADAM-3600在故障转移模式下具有通道冗余功能。 如果主通道上的连接丢失，则控制器将自动切换到备用，例如从有线Internet到3G。 在EdgeLink Studio中配置通道的优先级和检查连接的地址。


设置通道优先级和故障转移模式

VPN支持


对当前版本的OpenVPN 2.4的支持使您可以将设备集成到企业的虚拟网络中。 提供UDP和TCP隧道，证书身份验证以及高级加密算法，例如AES-256-CBC。


在EdgeLink Studio中配置OpenVPN隧道

设备硬件ID


可以使用前面板上的DIP开关设置快速初始配置。 NodeID设置设备标识符和起始IP地址。 这使您无需进行初步配置即可安装大量设备，并且可以通过网络远程执行进一步的配置。


可以使用跳线设置设备ID

操作模式


ADAM-3600控制器的主要功能是多功能性。 它既可以是数据采集设备（记录和发送遥测），也可以是带有对象的通信设备（从控制中心发送控制命令）和可编程控制器（自动化控制）。
每个用例都有自己的开发环境。

数据采集​​单元和远程终端


ADAM-3600可以用作经典的数据采集和传输设备（DTD）。 附加功能之一是发送前的数据处理。 例如，可以计算标准偏差或原始电流或电阻传感器的读数，并将完成的数据转换为适用于各种数据库（例如MS SQL Server）的适当格式。 如果没有Internet连接，则可以在SD卡上执行保存数据。 恢复连接后，数据将无损失地传输到服务器。



使用MQTT协议，您可以将数据收集直接组织到云服务中，例如Microsoft Azure IoT中心，Amazon AWS IoT等。

设备是在EdgeLink Studio图形环境中配置的，该环境允许您在数据收集模式下或在远程终端中配置设备，而无需编写任何代码。 这使您可以大大简化实施。

EdgeLink Studio的负担得起的云提供商

可编程逻辑控制器


ADAM-3600固件基于RT Linux- 硬实时操作系统的特殊版本。 而且，该平台是完全开放的，开发人员具有对系统的root访问权限。 这使您可以使用系统编程语言来实现任何非标准解决方案。 研华提供用于C / C ++语言的SDK ，以及用于构建Linux项目的开发环境和工具链。

使用场景：实时监控油井


在我们网站的培训材料部分，我们发布产品使用方案www.advantech.ru/resources/ 。 使用ADAM-3600控制器的常见方案之一是对钻井和泵站的监控。 研华提供用于构建自动控制系统的全套解决方案，特别是具有Web界面的SCADA系统-研华WebAccess。

系统的基本任务：

  • 监视油温和“套筒”，油压，流量以及抽油机的其他参数； 以及读取并保存到数据库。
  • 通过有线（RS-232 / 485，LAN）和无线总线（ZigBee）连接传感器
  • 将传感器和继电器直接连接到数字和模拟控制器接口
  • 收集外围设备传感器读数以及有关石油生产和设备运行的统计信息。 初始处理后，数据将发送到SCADA服务器。



油井监控系统架构

结论


凭借其模块化设计， ADAM-3600控制器允许您根据客户需求选择接口端口和无线协议，因此您可以使用一台设备来完成大量任务。 提供了快速设置功能，可以轻松安装和库存大量设备。 在-40至70°C的工作温度范围内，可以在恶劣的条件下使用控制器。 对行业标准和现代物联网协议的支持使ADAM-3600可以集成到任何工业系统中。
