拥有一台小型VPS服务器非常有用，在这里可以方便地进行某些测试。 通常要求它也全天候可用。 这需要设备的平稳运行和白色IP地址。 在家里，有时很难同时提供这两种条件。 并且考虑到租用一台简单的虚拟服务器的成本可与Internet提供商发布专用IP地址的服务的成本相媲美，因此租用这种服务器可以很好地证明这一成本。 但是，如何从谁那里订购这样的VPS？ 对各种资源的评论几乎没有信任。 因此，提出了一个想法，即通过一个简单的标准-通过租用服务器的性能来选择此类服务的最佳提供商。

参加者的选择


我并没有开始阅读任何评论和评论，而是为了亲自了解它提供的服务。 事实证明，有虚拟服务器选择服务，例如：

每个此类服务都提供安装必要的过滤器（例如，RAM的数量，内核数和处理器频率等），并按某个参数（例如，按价格）对结果进行排序。 已决定将参与者分为两组：第一组将包括硬盘驱动器，第二组是闪存。 显然，驱动器类型更多，具有SAS接口的驱动器的速度性能将与具有SATA接口的驱动器的速度性能不同，并且NVD SSD的性能将与其他SSD的性能不同。 但是，首先，我们将有太多的组，其次，SSD的HDD性能通常比它们之间的不同HDD和它们之间的SSD的性能差异更大。

测试方法


每个服务器都测试了一周。 CPU，RAM，磁盘子系统和网络都承受了负载。 测试按计划运行，托管在cron中。

通过将值制成表格并绘制图表和（或）图表来收集和处理结果。 使用了以下工具。

综合测试：

  • 系统平台
  • cpu，常规测试： sysbench --test=cpu run (值:总时间）
  • 内存，常规测试： sysbench --test=memory run （值：总时间）
  • 文件I / O，测试和命令（所有测试中的块大小为4k；值：传输速度）：

    • 具有模拟队列深度32的单线程顺序读取： sysbench --num-threads=1 --test=fileio --file-test-mode=seqrd --file-total-size=2G --file-block-size=4K --file-num=32
    • 具有模拟队列深度32的单线程顺序记录： sysbench --num-threads=1 --test=fileio --file-test-mode=seqwr --file-total-size=2G --file-block-size=4K --file-num=32
    • 具有模拟队列深度8的八线程随机读取： sysbench --num-threads=8 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=8
    • 具有模拟队列深度8的八线程随机记录： sysbench --num-threads=8 --test=fileio --file-test-mode=rndwr --file-total-size=2G --file-block-size=4K --file-num=8
    • 具有模拟队列深度32的单线程随机读取： sysbench --num-threads=1 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=32
    • 模拟队列深度为32的单线程随机记录： sysbench --num-threads=1 --test=fileio --file-test-mode=rndwr --file-total-size=2G --file-block-size=4K --file-num=32
    • 具有模拟队列深度的单线程随机读取1： sysbench --num-threads=1 --test=fileio --file-test-mode=rndrd --file-total-size=2G --file-block-size=4K --file-num=1

讲台


所有地方都去托管与SSD。 RuVDS在激烈的战斗中赢得了第一名。 AdminVPS排名第二，REG.RU和American Ionos（1＆1）排名第三，领奖台上的所有其他托管服务均代表俄罗斯。


结论


在所有测试参与者中，第一名是RUVDS的SSD关税。 更好的处理器性能和良好的磁盘性能使它们在收费方面排名第一。 恭喜获胜者。 我还想指出托管adminvps，ionos和regru，他们有尊严地战斗。 AdminVPS表现出出色的磁盘性能，但在处理器性能方面落后。 REG.RU表现出相当好的处理器性能，但是就磁盘性能而言，运行并不顺利。 Ionos显示出相当平衡的结果。 其余的参与者的结果要差得多。 以自己的方式，Ihor表现出色。 他们的两种资费都位于表格的“最底端”；使用服务时，“目睹”性能很低。
