Acordeão, perguntas frequentes, spoiler e outros widgets suspensos



Você pode criar um acordeão, perguntas frequentes, spoiler e afins usando Div e JavaScript.
Mas melhor: detalhes e resumo

É suportado por todos os navegadores modernos e este é um código semanticamente corretamente formatado, cuja utilização terá vantagens:

  1. Pessoas com deficiência serão mais fáceis de usar seu site! Seu software (leitores de tela e similares) entende perfeitamente as tags html5 e as processará e informará corretamente as pessoas sobre o conteúdo.
  2. A coerência do texto será aprimorada e os mecanismos de pesquisa poderão indexar melhor o site, pois entenderão melhor como o texto visível e oculto está relacionado.
  3. Ele estará disponível para controlar elementos do teclado e outros dispositivos.
  4. A quantidade de código javascript a ser carregado é reduzida, o que aumenta a velocidade de carregamento da página, a velocidade de processamento e a correção.
  5. Melhorando o desempenho no Lighthouse, no Google PageSpeed e em outras ferramentas similares.
  6. Funciona quando o javascript está desativado.

Menos:

  1. Navegadores mais antigos não conhecem essas tags e não ocultam informações.

HTML: 

<details> <summary>- </summary> <p>  —   </p> </details> <details open> <summary>-  2</summary> <p>        </p> </details> <details> <summary>-  3</summary> <p>  —   </p> </details>

Por um lado, não é muito bonito; por outro, é neutro e pode se encaixar facilmente em muitos designs. A propósito, a visualização padrão da tag Detalhes é muito semelhante ao spoiler do Habr, apenas é necessário repintá-lo um pouco, sublinhar e obter o semanticamente correto, sem javascript e divs, spoiler Khabrov.



Infelizmente, o marcador padrão tem duas desvantagens:

  1. Navegadores antigos não o veem.
  2. Os navegadores Webkit não permitem alterar o símbolo do marcador.

Por esse motivo, o marcador padrão deve estar oculto e criar o seu próprio.


Considere o primeiro exemplo Detalhes / Resumo com um marcador de texto modificado:



CSS: 

 summary::-webkit-details-marker{display:none;} summary::-moz-list-bullet{list-style-type:none;} summary::marker{display:none;} summary { display:inline-block; padding: .3em .5em .3em .4em; font-size:1.4em; cursor: pointer; } summary:before { content: "+"; margin-right: .3em; } details[open] > summary:before { content: "–"; } summary ~ * { padding:0 1em 0 1em; } summary:focus { outline:0; box-shadow: inset 0 0 1px rgba(0,0,0,0.3), inset 0 0 2px rgba(0,0,0,0.3); } details{ display:block; margin-bottom: .5rem; }

Para os marcadores padrão, exibimos: none e mostramos uma alternativa com o resumo: antes de {content: "+";}

resumo: foco - toque com sombra de caixa, isso é necessário para o teclado, para que o elemento ativo fique visível e você possa se mover com a tecla Tab e abrir e fechar com um espaço.

Para a tag de resumo, defino display: inline-block - para que não se estenda até a largura total e apenas as palavras possam ser clicadas, e não a linha inteira.

 var details = document.querySelectorAll("details"); for(i=0;i<details.length;i++) { details[i].addEventListener("toggle", accordion); } function accordion(event) { if (!event.target.open) return; var details = event.target.parentNode.children; for(i=0;i<details.length;i++) { if (details[i].tagName != "DETAILS" || !details[i].hasAttribute('open') || event.target == details[i]) { continue; } details[i].removeAttribute("open"); } }
