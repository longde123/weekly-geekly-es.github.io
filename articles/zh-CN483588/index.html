如何尝试使用ILV阻止任何站点

大家好！ 今天，我想讲一个有关ILV的故事，ILV是一个其他人拥有电影的网站，与我的网站无关。

我有一个站点，用于查找流量稳定的虚拟服务器。 有时，由于有人将链接链接到某个流行资源上的站点，访问者数量激增，但是这种情况很少发生，您几乎总是可以看到访问者来自何处。 在新的一年即将来临之际，情况就这样发生了：该网站的每日访问量增长了将近2倍，我开始查看访问者来自何处。 大多数情况下，此类激增的来源是技术资源或接近技术资源，但在这种情况下情况有所不同：访问者从盗版网站的不同页面（带有电影）进入该网站，最终到达了我网站的404页。



起初，我并不重视这个事实，但是在第二天情况又重演了一次，我决定解决这个问题。 事实证明，统计系统没有记录到我的网站上的任何点击-当您打开其中包含电影的网站的任何页面时，带有404页面的我的网站的框架会全屏打开（https：//domain/404.php）。 在找出原因之前，我确定有人发布了指向我网站的断开链接，因此，切换时，用户位于404页上，这是偶然的，但是现在很明显，事实并非如此，而是有人故意安装了与我的网站框架。 根据经验，我发现如果您访问带有Yandex或Google（及其引荐来源）的电影的网站，则访问者看到了网站的真实内容并且可以观看电影，但是如果没有引荐来源，则访问者可以看到我网站的404页。

经过一番思考，我得出的结论是，除了将故障率从10％提高到70％之外，这种情况不会造成任何伤害，因此决定不采取任何措施并观察更多事件，突然所有事情都会得到纠正。



但是，这种情况并没有像我预期的那样发展。 一天后，我收到一封来自ILV的电子邮件，要求删除包含特定电影的网站页面，并向我的电子邮件info@poiskvps.ru发送了电子邮件，并向站点托管者发送了带有abuse@ispiria.net电影的电子邮件。



为什么写这封信给房东，并没有对我提出疑问，但我不清楚为什么收到了这封信。 这里有逻辑：ILV专家打开了一个禁止放映电影的站点，忽略了大写字母“找不到页面”的题词（因此不存在禁止放映的电影），在页脚中看到一封电子邮件，该电子邮件不属于他们感兴趣的领域，并且很有成就感债务向他发出了通知。 从ILV方面来看，一切都很清楚：我们去了该站点，查看了电子邮件地址并向其发送了通知，尽管这种常识性的策略在任何方面都不适用-我想知道如果他们在这种情况下看到电子邮件，他们会怎么做，例如info @ google.com



在与几个熟悉的主持人交谈之后，我得到了建议，说明情况不规范，唯一可以做的就是写信给ILV，答案是他们弄错了，而我与电影网站无关，我这样做确实是与电影网站有关。我应该去做

除此之外，我将404页移到了另一个地址（https：//domain/404a.php），并使上一个404th（位于框架中）完全为空。

但是，这并没有产生任何效果，一段时间后，我收到另一条通知，指出在该网站的另一页上有电影，然后是另一页上发现了非法内容。

故事的结尾几乎是在我收到第4条通知时，该电影站点被视为其他已经被阻止的站点的镜像，并且也会被阻止。 正如我后来发现的那样，除了有关ILV的来信以外，还提供了有关查找站点镜像的通知，但通信部仍未收到我的来信，从这里我可以得出结论，该部正在研究发送通知的地址。



短时间后，该站点被阻止，因此到我的站点的流量恢复正常。 然后新年假期到了，我已经忘记了这个故事，但是在NG之后的第一个工作日，我收到了ILV的来信，他们注意到这些信息与我与电影的拍摄地点无关。 不幸的是，我需要做什么没有答案。



直到现在，我还不知道这是什么：纯粹的巧合（但我几乎不相信这样的巧合），或者有人决定惹恼我并阻止该网站。 我正在考虑的一种选择是，使用我的服务拍摄电影的网站的所有者找到了一个虚拟服务器，并决定感激地“赶上”我的访客，尽管这一假设听起来似乎也不可行。

作为一种预防措施，您可以通过X-Frame-Options标头或使用JavaScript来阻止网站显示在框架中： 

if (window.parent.frames.length > 0) { window.stop(); }
