Redis mengubah lisensi lagi

Untuk kedua kalinya dalam enam bulan, Redis Labs mengubah model lisensi untuk sejumlah produknya. Sekarang perusahaan ini pindah dari Apache 2.0 Commons Clause ke Redis Source Available License (RSAL) Mari kita bicara tentang alasan keputusan ini dan fitur RSAL.


/ Pixino / congerdesign / PD

Sedikit sejarah


Ini bukan pertama kalinya Redis Labs mengubah lisensi untuk produk-produknya. Pada bulan Agustus tahun lalu, perusahaan mentransfer beberapa modul - Redis Graph, ReJSON, dll. - dari lisensi GNU AGPL ke Apache 2.0 Commons Clause. Dengan demikian, perusahaan melarang penjualan modul asli oleh pihak ketiga. Namun, ini menyebabkan konsekuensi yang tidak menyenangkan.

Pertama, lisensi baru telah menyebabkan kebingungan dan kesalahpahaman. Beberapa pengguna secara keliru memutuskan bahwa bekerja dengan modul sekarang hanya diatur oleh Apache 2.0 (tanpa Commons Clause). Kedua, larangan penjualan solusi Redis " memukul " perangkat lunak open source. Sebagai contoh, beberapa layanan dipaksa untuk menghapus pengembang Debian dan Fedora. Mereka harus memotong repositori modul Redis dan menggabungkannya menjadi proyek GoodFORM .

Tim Redis Labs juga menghadapi kesulitan yang tidak terduga. Pembatasan lisensi memperlambat pertumbuhan komunitas di sekitar produk, meskipun efeknya diharapkan justru sebaliknya. Semua ini mengarah pada fakta bahwa Redis Labs menciptakan lisensi mereka sendiri, disesuaikan dengan kebutuhan dan karakteristik perusahaan - RSAL.

Apa itu lisensi baru?


Menurut ketentuan RSAL , pengembang dapat menggunakan modul RediSearch, RedisGraph, RedisJSON, RedisBloom, RedisML dan beberapa lainnya dalam layanan mereka, mengubah kode sumber dan mengimplementasikannya dalam aplikasi. Keputusan akhir dapat didistribusikan dan dijual.

RSAL hanya membatasi jenis produk akhir. Aplikasi yang didasarkan pada modul yang ditunjukkan tidak boleh berupa basis data, alat untuk menyimpan dan mengindeks, mesin pencari atau perangkat lunak untuk bekerja dengan pembelajaran mesin.

Dalam semua kasus lainnya, perangkat lunak yang dikembangkan dapat digunakan dan didistribusikan dengan tanda: Perangkat lunak ini tunduk pada ketentuan Perjanjian Lisensi Redis Source Available .

Tujuan dari pembatasan ini adalah untuk menghilangkan komersialisasi modul oleh perusahaan yang bersaing tanpa merugikan komunitas Redis. Sedangkan untuk kernel Redis sendiri, seperti yang terakhir kali, tetap terbuka dan didistribusikan di bawah lisensi BSD. Untuk mendukungnya, perusahaan membuat tim terpisah yang akan mengelola pengembangan kernel, terlepas dari apa yang terjadi pada komponen lain.

Apa yang dipikirkan masyarakat


Beberapa perwakilan komunitas percaya bahwa pergantian lisensi yang berulang mungkin merupakan kesalahan lain. Matt Asay dari Adobe tidak setuju dengan pernyataan bahwa perusahaan besar yang menjual perangkat lunak sumber terbuka berdampak negatif terhadap pengembangan ekosistem sumber terbuka. Dia mengatakan organisasi seperti itu, sebaliknya, membantu mendistribusikan produk-produk sumber terbuka ke pasar global.

Visi Redis juga tidak dimiliki oleh Gordon Haff, kepala teknologi cloud di Red Hat. Dia percaya bahwa melalui lisensi, Redis berusaha untuk "duduk di atas dua kursi" - untuk mendapatkan keuntungan dari penjualan modul dan menjadi perusahaan sumber terbuka.

Red Hat Apache Guru Rich Bowen menyebut keputusan perusahaan "tidak berarti." Menurutnya, orang-orang yang datang ke open source berharap untuk melihat solusi gratis dan mereka hampir tidak ingin memahami batasan dan ketentuan apa pun. Anggota Open Source Initiative ( OSI ) setuju dengannya. Mereka mengklaim bahwa Redis bertindak bertentangan dengan definisi dan prinsip-prinsip perangkat lunak open source.

Ada yang melihat gunanya beralih ke lisensi baru. Sebagai contoh, kepala BaenCapital mencatat bahwa perusahaan yang membangun produk mereka berdasarkan perangkat lunak open source tidak etis . Oleh karena itu, tindakan Redis cukup dapat dipahami - dengan bantuan lisensi baru, perusahaan melindungi kepentingannya dan hak pengembang.

Salah satu pencipta Ansible, Michael DeHaan (Michael DeHaan) juga percaya bahwa jika semua perangkat lunak didistribusikan secara gratis, sebagian besar proyek tidak akan bertahan. Tidak semua perusahaan berhasil menarik investor, sehingga menjual komponen individu ke organisasi besar adalah salah satu cara untuk tetap bertahan di pasar.

Yang baru saja mengubah lisensi


Redis Labs bukan satu-satunya yang mencoba pendekatan perizinan yang berbeda. Jadi, pada Oktober 2018, MongoDB beralih dari GNU AGPL ke versi GNU 3 - Lisensi Publik Sisi Server (SSPL). Tujuan dari perubahan lisensi adalah sama dengan Redis - untuk mencegah perusahaan pihak ketiga dari "pengemasan" dan menjual kembali DBMS terbuka.

Para penulis proyek Confluent juga meninggalkan Apache 2.0 demi versi mereka - Lisensi Komunitas Confluent . Persyaratan lisensi baru melarang penjualan KSQL sebagai solusi berpemilik. Meskipun masih mungkin untuk mengimplementasikan layanan SaaS pada mesin SQL ini.

Ada contoh lain dari perusahaan di mana bagian dari solusi diterapkan untuk uang. Diantaranya adalah Elasticsearch, Hadoop, Berkeley DB dan puluhan lainnya.

“Ada lebih sedikit proyek gratis seperti kernel Linux, WordPress atau GIMP. Pengembang open source sedang membangun model bisnis dalam upaya untuk menemukan keseimbangan antara pendapatan dan distribusi produk tanpa merusak perusahaan, ”komentar Sergey Belkin, kepala departemen pengembangan penyedia infrastruktur virtual 1cloud.ru . "Tapi masih ada cukup banyak komunitas TI yang menentang perubahan konsep open source." Oleh karena itu, dalam waktu dekat, perangkat lunak yang sepenuhnya gratis tidak akan hilang dari pasar, tidak peduli bagaimana lisensi untuk produk individual dan komponennya berubah. ”
