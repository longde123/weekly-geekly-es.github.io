9 trik lagi untuk bekerja dengan Visual Studio

Baru-baru ini, kami berbicara tentang 18 trik untuk bekerja dengan Visual Studio. Hari ini kami telah memilih 9 tips lagi yang akan membantu meningkatkan produktivitas pekerjaan Anda dengan alat ini.

1. Ingin kembali ke baris kode yang diedit terakhir? Coba Ctrl + Shift + Bkspce untuk pergi ke lokasi edit terakhir.

2. Mudah memindahkan pointer instruksi (panah kuning) selama debugging menggunakan Set Next Statement.



3. Gunakan Ctrl + Shift + F12 untuk pergi ke sumber kesalahan berikutnya yang tercantum dalam Daftar Kesalahan.

4. Untuk memilih semua kemunculan kata yang dipilih, tekan Ctrl + Shift + titik koma untuk menggunakan dukungan dari beberapa titik penyisipan agar semuanya dipilih secara tersinkronisasi dan dapat diedit.

5. Hati-hati saat beralih dari CancelledToken ke JTF.SwitchToMainThreadAsync (). Bertentangan dengan intuisi, ini bukan pembatalan seluruh tugas, tetapi pembatalan switch ... sehingga kode akan terus bekerja di latar belakang.

6. Apakah Anda tahu bahwa Visual Studio berfungsi dengan Surface Dial? Gunakan ekstensi Surface Dial Tools gratis untuk memudahkan navigasi bookmark, kesalahan, debugging, scaling, dan banyak lagi.



7. Buat Visual Studio melakukan pengumpulan sampah sendiri dengan mengklik dua kali Ctrl + Shift + Alt + F12

8. Menemukan Cuplikan Kode di Visual Studio Menggunakan Asisten Pengembang
