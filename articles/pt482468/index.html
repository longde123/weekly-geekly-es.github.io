5 extensões e temas para o VS Code que podem mudar a vida de um desenvolvedor front-end

O autor do artigo, cuja tradução estamos publicando hoje, deseja falar sobre várias extensões e tópicos do popular editor de código VS que podem melhorar a vida do desenvolvedor.



SCSS IntelliSense


→ Página IntelliSense do SCSS

Criar variáveis ​​no SCSS é fácil e divertido. Como se lembrar deles? Aqui geralmente surgem dificuldades com isso.

Se você é como eu, possui o arquivo _variables.scss , que contém centenas de ícones em dólar e nomes de variáveis ​​que lhe vieram à mente às três da manhã. A extensão SCSS IntelliSense fornece dicas para nomes de variáveis ​​ao trabalhar no código do projeto. E você não precisa mais se lembrar desses nomes com todos os detalhes. Não há necessidade de vasculhar um arquivo com nomes de variáveis ​​para recuperá-los.

Veja como é a extensão do SCSS IntelliSense.

Renomear tag automaticamente


→ Renomear automaticamente a página de tags

Independentemente da estrutura de front-end usada, escrever HTML (ou JSX) pode ser uma tarefa tediosa. Especialmente em uma situação em que há algo como 50 linhas de algum código entre as tags de abertura e fechamento. Pode ser necessário alterar a tag H2 para H3 durante o curso do seu trabalho. Pode ser necessário substituir a div por uma span . De qualquer forma, se você resolver esses problemas sem usar ferramentas auxiliares, eles, em fragmentos de código em grande escala, se resumem a uma pesquisa difícil por tags de fechamento.

Por que não ligar para um carro para ajudá-lo a editar tags automaticamente? Para fazer isso, basta usar a extensão Renomear tag automaticamente. Ao editar a tag de abertura, ele muda e fecha automaticamente. A mesma coisa, exatamente o oposto, acontece se a tag de fechamento for alterada.

Veja como a tag de renomeação automática funciona.

Ação duplicada


→ Página de ação duplicada

A capacidade de criar arquivos e pastas duplicados usando o menu do botão direito do mouse é tão útil que deve ser incorporada ao VS Code. Mas esse editor, e sem essa oportunidade, é maravilhoso, para que ele possa perdoar essa falha.

Na verdade, para equipar o VS Code com o recurso descrito acima, basta instalar a extensão Duplicate Action. Quando ativado, o desenvolvedor possui o comando Duplicate file or directory menu de contexto.

Codestream


→ Página CodeStream

Se você descrever brevemente o CodeStream, podemos dizer que essa extensão é comparável ao sistema para comentar sobre o código no futuro.

Muitos programadores não gostam de deixar seu IDE. Mas, para explicar algo aos colegas e fazer perguntas, você precisa sair do ambiente familiar e usar o Trello, Slack, Asana, Bitbucket, Microsoft Teams. O CodeStream suporta muitas ferramentas e um IDE, que permite trabalhar com problemas e perguntas sem sair do ambiente familiar do editor de código. Muitos, trabalhando, tendem a entrar no chamado "estado do fluxo". Sair do editor geralmente tira a pessoa desse estado. Graças ao CodeStream, as questões relacionadas ao código podem ser resolvidas sem sair do seu ambiente familiar.

→ Página Coruja Noturna

Night Owl é um dos melhores temas já criados para o IDE. Por três anos, eu uso diariamente. Abaixo está uma captura de tela da minha tela de trabalho real. O título deste tópico sugere que ele se destina a quem fica sentado no computador à noite. É verdade que é adequado para o trabalho noturno, evitando que seus olhos se esforcem, e os tons suaves de azul usados ​​nele ajudam a resolver problemas complexos com calma.
