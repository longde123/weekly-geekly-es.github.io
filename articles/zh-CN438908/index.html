人们使用计算机网络的另一个重要原因是使用打印机等设备。 假设这是公司购买的非常昂贵的打印机或扫描仪，并且如果我们没有公用网络，则只能将其直接连接到一台计算机。 假设该用户每天打印不超过1页，而该公司中的所有其他用户每天最多打印2页。 然后，该公司将被迫为每个用户购买个人打印机，并且使用量将降至最低。 因此，该公司在网络基础设施上进行了投资，该网络基础设施使您只有一台昂贵且高质量的打印机可供连接到网络的所有员工使用，从而节省了资金，并充分利用了IT行业的预算。

因此，创建了一个广域网，“广域”一词是指连接许多本地网络的大面积地理区域。 它可以分布在全国各地，也可以组织在同一国家的不同城市或不同的国家中。 实际上，所有这些都是LAN。

假设有人输入youtube.com并按Enter键，则计算机会将HTTP请求发送到YouTube公共服务器。 服务器收到此请求时，会认为：“很好，这是一个HTTP请求，因此我将HTML文件发送回去。” 它创建一个HTML文件并将其发送回给我。 我的浏览器接受此文件并说：“我知道这个HTML文件是什么，我知道如何处理它以显示来自YouTube的视频。” 同样，eBay，CNN和Skype也会发生这种情况。 Internet基本上是这样工作的：有一个公用服务器，计算机上请求您的文件，这些文件会根据请求发送给您。 这就是互联网如何帮助全球化信息世界的方式。

在讨论Internet之前，请考虑其最重要的属性-速度。 让我们讨论一下位和字节之间的区别。 位是计算机理解的最小信息，可以是0或1。字节是由8位组成的信息单位。



因此，例如，一个字节看起来可能像这样：1 1 0 1 1 1 1 0 1.人们经常以这些单位感到困惑，因此请记住：数据始终以字节为单位。 当我说我有1 GB的RAM时，这意味着1 GB的RAM，但是当我说到我有10 Mbps的速度时，则意味着我正在每秒传输10兆位。 因此，数据以字节为单位，数据速率以每秒位数为单位。 如果您将这两个概念弄混了，那对您来说并不容易。 如果我说“每秒字节”，则表示“每秒8位”。 因此，请记住，速度始终由小写字母b（这是一个位）和大小（由大写字母B，这是一个字节）指示。 1 KB = 1024位，就像1 KB = 1024字节一样。

考虑一个例子。 128 KB文件将以1 Mb / s的速度在网络上传输多长时间？ 我们将文件大小从字节转换为位：128 x 8千位= 1024千位，即1兆位。 因此，在给定的连接速度下，此大小的文件的传输时间将不超过1秒。

如果您了解位和字节之间的差异，那么您将很容易理解其余视频教程的内容。 如果要谈论速度，我们需要提到三个降低速度的重要因素。 所有这些因素总是相互联系的。 当您将电缆从路由器连接到计算机时，它报告它以100 Mb / s的速度连接到网络。 今天，这是LAN网卡可以提供的最大速度。 但是请不要忘记，我们所说的速度就是您的计算机和路由器之间的速度。 您的Internet连接的速度可能为1 Mbps，因此，如果您尝试以100兆字节的速度连接到Internet，则这对您将不起作用，因为Internet只能传输1兆字节。 假设您要将文件从班加罗尔的办公室转移到纽约的办公室。 因此，通过网关路由器的文件以1 Mbps的速度到达班加罗尔的Internet提供商。 Bangaolra的供应商通过可提供100 Mb / s速度的高速通信线路连接到孟买的供应商。



孟买的ISP和迪拜的ISP之间的速度较慢，因为它们之间只有一条线，只能提供10 Mbit / s，但是迪拜和开罗之间又有一条100 Mbit / s的高速连接。 在开罗和马德里之间，也可能有高速互联网，然后速度下降等等。 这就是Internet上发生的情况，Internet是一堆具有不同数据速率的路由器。 即使我在班加罗尔拥有100 Mbps的高速连接，班加罗尔与纽约办事处之间的传输速度也将取决于通信线路上的最小连接速度，在这种情况下，是纽约IPS与纽约计算机之间的速度。办公室，即1 Mbps。

这就是Internet速度的工作方式。 当然，班加罗尔和纽约之间的联系并不是真正通过孟买，迪拜和开罗建立的，我以这些城市为例，实际的数据传输方式可能有所不同。 但是通常，数据包以这种方式从一个位置移动到另一个位置。 因此，速度取决于许多因素，包括网络路由不同部分的速度。 因此，第二个关键因素是延迟。 假设我们在波士顿有另一个办公室，我将其放置在纽约办公室的右侧。 当从纽约向波士顿发送数据时，它们经过的节点更少，因此，它将更快地达到目标。 结论如下：如果两个设备之间的距离较大，则可能会遇到通信延迟。

因此，如果您从新加坡访问google.com或youtube.com，则网络不会将您定向到美国的Google服务器，而是定向到距离您最近的服务器，也许会定向至新加坡本身的IPS服务器，而延迟不会那么大。 但是，如果开罗和迪拜之间的联系中断，您将不得不从纽约到班加罗尔走更长的路，可能要经过俄罗斯，中国和印度。 这是一个非常漫长的过程，在这里出现了第三个因素-可访问性，即，Internet工作所需的必要资源的可用性。

我们将在本视频教程中讨论的另一件事是网络拓扑。 拓扑有三种基本类型：“星形”，“环形”和“总线”。



“ Star”是迄今为止存在的最古老的计算机连接方案之一。 在这里，所有计算机或设备都连接到中央交换机。 如果其中一台计算机失去与交换机的连接，则不会影响其他计算机的连接，但是此计算机将完全失去与网络的连接。

在“环”中，每台计算机都连接到另一台计算机，并且如果相邻两台计算机之间的连接中断，它们仍然可以通过下一台计算机相互通信。



在“总线”中，每台计算机还使用自己的小型网络设备分别连接到网络。 因此，如果网络的一部分出现故障，则位于附近的计算机将能够相互通信，但是将失去与位于受损部分后面的网段的计算机的联系。



让我们看看如何在现代世界中实现它，回到第一张带有网络图片的幻灯片。 如您所见，该交换机和与其最接近的三台计算机都连接到公共星形拓扑网络，并且位于右上角的服务器也已连接到该交换机。 移动设备几乎以相同的方式连接到WiFi AP。 两个中央交换机根据“总线”方案进行连接，因此，如果它们之间的连接中断，则右侧的设备将能够相互通信，而不能与左侧的网段进行通信。 有时，交换机通过附加的通信线路连接，因此，即使某一部分的连接中断，也可以在其他网段上进行通信。
