Die erste Managed OpenShift-Lösung in der Cloud

Hallo Habr! Auf dem Red Hat Summit 2018 in San Francisco haben wir zusammen mit Red Hat OpenShift auf der Azure-Plattform vorgestellt, der ersten vollständig verwalteten und benutzerfreundlichen OpenShift-Containerplattform als Cloud-Service. Heute werden wir ausführlicher darüber sprechen und Ihnen sagen, wie Sie mehr Insider gewinnen können. Schau unter die Katze!



Nachdem im vergangenen Jahr OpenShift Dedicated on Azure angekündigt wurde, haben Microsoft und Red Hat eine neue Lösung basierend auf Benutzerfeedback erstellt. Es bietet Kunden viel mehr als die ursprüngliche Version des Produkts. OpenShift Dedicated ist eine vollständig verwaltete und anpassbare Lösung, die die Verwendung in Azure einfach und bequem macht.

Was bedeutet "voll steuerbar"? Red Hat und Microsoft werden OpenShift on Azure gemeinsam entwickeln, verwalten und unterstützen. Als vollständig verwalteter Dienst wird er als einzelnes Projekt fortlaufend unterstützt und bietet umfassenden Benutzer-Support, dessen Qualität dem Azure-Support nicht unterlegen ist. Laut unseren Kunden hatten andere Cloud OpenShift-Lösungen keine Vorteile, die sich aus der Nutzung von Diensten innerhalb eines einzigen Anbieters ergaben. Wir freuen uns, diesen wichtigen Bedarf decken zu können.

Trotz der Tatsache, dass OpenShift heute in vielen Cloud-Umgebungen verwendet wird, ist die kürzlich angekündigte OpenShift on Azure einfacher zu konfigurieren und mit nativen Azure-Integrationsmechanismen zu verwenden. Sie können jedoch auch Kubernetes, den zugrunde liegenden Azure Kubernetes-Dienst, verwenden.

Wenn Sie einen neuen OpenShift-Cluster erstellen möchten, müssen Sie keine Serviceanforderung erstellen und mehrere Stunden darauf warten, dass Netzwerkkomponenten eine Verbindung herstellen (wie dies in anderen Clouds der Fall ist). Stattdessen können Sie die Azure-CLI verwenden, um verschiedene Befehle auszuführen:



Das Abrufen eines neuen Clusters in wenigen Minuten ist schneller und einfacher als mit anderen derzeit verfügbaren öffentlichen Cloud-Lösungen.

Open Service Broker für Azure (OSBA) funktioniert auch mit OpenShift für Azure, einschließlich Azure Stack, und erkennt automatisch eine Liste verfügbarer Azure-Dienste wie Cosmos DB, Azure KeyVault usw. und zeigt OSBA an. OSBA erhöht die Effektivität aller von Ihnen bereitgestellten Dienste Verwenden von OpenShift in Azure, sodass Sie andere in Azure verschobene Objekte problemlos integrieren können. Dies ist eine weitere praktische Funktion, mit der Sie ein Projekt schnell starten und mit anderen vorhandenen oder geplanten Projekten verbinden können, um es in Azure zu starten.



Was für Kunden noch angenehmer ist, ist die kürzlich angekündigte Unterstützung von Azure Stack und Windows Container für die lokale OpenShift Container-Plattform. Dies ist ideal für Unternehmenskunden, die eine Hybridplattform für Cloud-Container verwenden möchten, deren Arbeit gleichzeitig sowohl auf RHEL als auch auf RHEL unterstützt wird Windows Erfahren Sie, wie Sie die OpenShift Container Platform jetzt auf Azure Stack bereitstellen.

Verfügbarkeit


Die Red Hat OpenShift Azure-Lösung wird voraussichtlich in den kommenden Monaten als offene Demo verfügbar sein. Die Plattformen Red Hat OpenShift Container und Red Hat Enterprise Linux auf Azure und Azure Stack sind derzeit verfügbar.

Wenn Sie mehr Einblicke wünschen


Am 16. Juni wird Microsoft in Moskau auf der DevOps Days- Konferenz zusammen mit Red Hat eine Meisterklasse "DevOps, Red Hat, Microsoft und alles in allem" abhalten. In der Meisterklasse zeigen wir, wie schnell und schön Ihre täglichen Aufgaben in der Red Hat OpenShift Container Platform unter Microsoft Azure gelöst werden.

Was wird?


  • Der source2image-Mechanismus in OpenShift. Sie (und Ihre Kollegen im DevOps-Team) nehmen einfach den Link zum Git und geben ihm OpenShift. Dann erledigt er alles selbst: Er stellt die Anwendung selbst zusammen und stellt sie bereit, zeigt die Metrik, protokolliert und skaliert sie auf den angegebenen Knoten auf die angegebenen Werte. Entwickler müssen sich nicht darum kümmern, was und wie auf Infrastrukturebene zu tun ist. Sie erledigen einfach die übliche Arbeit mit den üblichen Mitteln - sie schreiben Code, klonen ihre Repositorys - und warten, bis sie zusammenkommen.
  • Buchhaltung und Kontrolle. Stellen Sie WordPress aus einem Container-Image bereit, um Flugblätter für Ihre Katze zu erstellen. Wir zeigen, dass unser Container nicht einfach so startet, weil er Berechtigungen benötigt. Wir werden anhand dieses Beispiels zeigen und zeigen, wie die Sicherheitslage ist, und dem Container besondere Berechtigungen erteilen. Dann werden wir sehen, wie er startet, die Datenbank daran binden und den ersten Super-Blog-Beitrag erstellen.
  • Farbige Schrödinger-Quadrate oder Blau / Grün-Bereitstellung über die Jenkins-Pipeline, die wir mit einem (aber nicht genau) Klick in OpenShift erweitern - ein sphärisches Szenario im luftleeren Raum, das zeigt, wie die neue Version der Anwendung dem Benutzer ohne Ausfallzeiten und ohne Nervenverlust zur Verfügung gestellt werden kann sowohl Dev als auch Ops.
