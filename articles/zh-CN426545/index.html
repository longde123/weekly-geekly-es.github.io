5次BoF会议


会议进行了整整两天，因此我们在等待越来越多的牛肉：

  • 正确的API设计
  • Java 11-可以吗？
  • 工具（IDE，CI等）
  • 生产故障排除
  • 从FP到反应式

如果您突然感兴趣，BoF代表“羽毛的鸟”，其起源是从第一次IETF会议开始计算的 。 Java One的参与者可能还记得，bofs也经常出现。

BoF现在是我们会议上的传统格式。 圆桌会议或讨论小组之类的东西，每个人都可以参加。

演讲者和参与者之间没有分歧：每个人都平等参与。 这很重要，至少出于以下三个原因：

  • 该报告的格式非常有用，因为它专注于特定主题并且缺乏分散注意力的机会。 每个人都在坐着听演讲者。 BoF会议适合所有参与者之间的全面交流，并允许您涵盖广泛的相关主题。
  • 参加者中有许多很酷的专家。 他们在这次会议上没有报告，但是他们的冷静并不影响报告。 他们经常在我们的其他会议上演讲，或者在知名公司中担任领导职务。
  • 并非每个人都有时间在讨论区提问，即使他们与演讲者在一起的时间超过一个小时。

在这里，您可以抓住会议的最后台部分，并找出没人写过的东西。



讨论区


每次报告后，演讲者都会进入专门的讨论区，在这里您可以与他聊天并提出问题。

正式地，这可以在报告之间完成。 发言人没有义务，但通常会停留更长的时间-例如，在下一次报告期间。 有时，从主程序中跳过报告是有意义的（如果您购买了票，您仍然会有条目），然后将其用于与重要专家进行有重点的沟通。

不能搁置一边，但要参与讨论并提出您的问题，这一点非常重要。 受欢迎的演讲者有时会被一群想要的人包围，因此立即“抓住”演讲者，与他一起进入讨论区，站在前排并且不要害羞地提出有趣的问题是很有意义的。

交流不仅限于在Joker上进行的报告。 如果您在其他会议或YouTube上看到特定作者的演讲，也可以尝试讨论这些主题。
