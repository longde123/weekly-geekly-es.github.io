PVS-Studio 7.00

Hoy es un día importante: después de 28 lanzamientos de la sexta versión, lanzamos PVS-Studio 7.00, donde la innovación clave es el soporte para el lenguaje Java. Sin embargo, en 2018 se han acumulado muchos otros cambios importantes con respecto a C ++, C #, infraestructura y soporte para estándares de codificación. Por lo tanto, le presentamos una nota que resume los principales cambios ocurridos recientemente en PVS-Studio.

PVS-Studio es una herramienta para detectar errores y vulnerabilidades potenciales en el código fuente de programas escritos en C, C ++, C # y Java. Se ejecuta en Windows, Linux y macOS.

Las capacidades del analizador están bien demostradas por la extensa colección de errores en el código que encontramos durante la verificación de varios proyectos de código abierto.

Comenzaremos con el cambio, debido a que, de hecho, se decidió cambiar el número de versión de 6.xa 7.x. Esto es compatible con el analizador de lenguaje Java, que hemos estado preparando durante mucho tiempo.

No podíamos ignorar la plataforma de control de calidad del código SonarQube, tan popular entre los desarrolladores de Java, por lo que agregamos compatibilidad con el lenguaje Java a nuestro complemento SonarQube .

C, C ++


En 2018, nuestro equipo trabajó activamente en el desarrollo del analizador C ++. Han aparecido muchos diagnósticos nuevos, así como algoritmos mejorados que le permiten recopilar y procesar información recuperada del árbol de sintaxis.

CWE, CERT

Los mensajes del analizador C ++ (como el analizador C #) se clasificaron de acuerdo con la enumeración de debilidad común ( CWE ). CWE es un sistema de clasificación para vulnerabilidades potenciales y confirmadas. Es compatible con la comunidad para identificar problemas de software y crear herramientas automatizadas que se pueden utilizar para identificar y solucionar estos problemas.

Además, los mensajes se han clasificado de acuerdo con el estándar de codificación CERT C y el estándar de codificación CERT C ++. SEI CERT Coding Standards es un conjunto de estándares para escribir software (software) en C, C ++, Java y Perl, desarrollado por el Centro de Coordinación CERT (Centro de Coordinación CERT, CERT / CC) para aumentar la confiabilidad y seguridad del software.

La clasificación de los diagnósticos C y C ++ de acuerdo con estas advertencias permite utilizar PVS-Studio como una solución SAST .

Misra

En 2018, el analizador de código estático PVS-Studio comenzó a clasificar sus advertencias de acuerdo con los estándares MISRA C y MISRA C ++. Gracias al soporte de estos estándares, el analizador ha sido posible usarlo efectivamente para mejorar la seguridad, portabilidad y confiabilidad de los programas para sistemas integrados.

Más detalles: " PVS-Studio: soporte para los estándares de codificación MISRA C y MISRA C ++ ".

El soporte de MISRA se ha vuelto relevante en relación con el desarrollo en el analizador de soporte para diversos sistemas integrados, que se discutirá en la siguiente sección del artículo.

Extensión de anotación personalizada

A veces, los proyectos usan sus propias implementaciones de diversas funciones del sistema, por ejemplo, memcpy , malloc , etc. El analizador es muy bueno en la búsqueda de errores utilizando funciones estándar, pero hasta hace poco era imposible usar dichos algoritmos en el código de usuario. Ahora, usando la anotación V_FUNC_ALIAS, puede hacer coincidir los nombres de sus funciones con las del sistema.

Formato de comentario: 

//V_FUNC_ALIAS, implementation:sysf, function:f, namespace:ns, class:c

  • Implementación clave: establece el nombre de una función estándar para la que se define un alias.
  • Tecla de función : establece el nombre del alias. La firma de la función cuyo nombre se especifica en esta clave debe coincidir con la firma de la función especificada en la clave de implementación.
  • La clase clave es el nombre de la clase. Puede estar ausente
  • La clave del espacio de nombres es el nombre del espacio de nombres. Puede estar ausente

Considere un ejemplo: 

 //V_FUNC_ALIAS, implementation:memcpy, function:MyMemCpy

Ahora el analizador manejará las llamadas a la función MyMemCpy de la misma manera que las llamadas a memcpy . Por ejemplo, se emitirá una advertencia V512 para dicho código: 

 int buf[] = { 1, 2, 3, 4 }; int out[2]; MyMemCpy (out, buf, 4 * sizeof(int)); // Warning!

SAST


Hasta hace poco, en nuestros artículos, posicionamos a PVS-Studio como una herramienta para detectar errores en el código. Al mismo tiempo, casi no consideramos PVS-Studio en el contexto de seguridad. Este año corregimos esta situación y analizamos la herramienta desde el punto de vista de probar la seguridad de las aplicaciones y las prácticas de DevSecOps.

PVS-Studio es una herramienta para pruebas de seguridad de aplicaciones estáticas (Static Application Security Testing, SAST). En otras palabras, el analizador PVS-Studio detecta no solo errores tipográficos, código muerto y otros errores, sino también vulnerabilidades potenciales.

Para la comodidad de los especialistas que utilizarán PVS-Studio como una herramienta SAST, el analizador muestra sus advertencias sobre la enumeración de debilidad común , estándares de codificación SEI CERT . Tablas de cumplimiento de diagnósticos de PVS-Studio a varios estándares:

  1. Cumplimiento de CWE
  2. Cumplimiento SEI CERT

Este tema se describe con más detalle en el artículo " PVS-Studio como una solución SAST ".
