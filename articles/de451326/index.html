3CX V16 Update 1 Beta - Neue Funktionen für Chat- und Anrufverlaufsdienste zum programmgesteuerten Verwalten von Anrufen

Nach der jüngsten Veröffentlichung von 3CX v16 haben wir bereits das erste Update von 3CX V16 Update 1 Beta vorbereitet. Es implementiert neue Funktionen des Unternehmens-Chats und den aktualisierten Call Flow Service, mit dem Sie zusammen mit der Entwicklungsumgebung Call Flow Designer (CFD) komplexe Sprachanwendungen in C # erstellen können.

Aktualisierter Unternehmens-Chat


Das 3CX Live Chat & Talk- Kommunikations-Widget wird weiterhin aktiv entwickelt. In Update 1 hängt das Widget unabhängig von den Übergängen zwischen Seiten und Registerkarten. Jetzt können Besucher auf Ihrer Website herumreisen und das Chat-Fenster für die sofortige Kommunikation verfügbar lassen.

Interessante Möglichkeiten ergaben sich im 3CX Corporate Chat Service.



Für Nachrichten (a) stehen nun folgende Aktionen zur Verfügung:

  • Chat-Sitzung beenden - Beenden Sie einen Chat mit einem 3CX-Benutzer (oder Site-Besucher).
  • Anonymen Benutzer blockieren - Blockiert den Benutzer (IP-Adresse) für eingehende Nachrichten und Anrufe.
  • Löschen - Löscht den Chat.
  • Archivieren - Archivieren Sie den Chat (verschieben Sie ihn in den Archivordner) und löschen Sie ihn von der Webclient-Oberfläche. In Zukunft wird es neue Funktionen für die Archivierung von Chats geben.
  • Übertragen - Wählen Sie die Nebenstelle 3CX (eines anderen Benutzers) und senden Sie ihm die weitere Kommunikation. Bei der Kommunikation mit Website-Besuchern ist es praktisch, wenn Sie das begonnene Gespräch an einen anderen Spezialisten übertragen müssen.

Außerdem zeigt der Benutzer beim eingehenden Chat eine Benachrichtigung an, in der Sie schnell auf die Nachricht antworten können (b).

Wenn die Nachricht von der Site über das 3CX Live Chat & Talk-Widget kam, sind jetzt eine Reihe neuer Funktionen verfügbar.



  1. Eine eingehende Nachricht kommt als WebVisitor-Benutzer zur schnellen Identifizierung an der 3CX-Webclient-Oberfläche an.
  2. Wenn die Nachricht in die Agentenwarteschlange eingeht, wird automatisch eine Chat-Gruppe erstellt, zu der alle Agenten dieser Warteschlange hinzugefügt werden. Die Bediener sehen die Korrespondenz mit dem Kunden und können ihm gemeinsam antworten, bis einer von ihnen weiterhin individuell mit dem Kunden kommuniziert. Von der Seite des Site-Besuchers wird dieser Chat als Konversation mit einem Operator mit dem in der Widget-Konfiguration angegebenen Absendernamen angezeigt.
  3. Im Menü oben rechts stehen die zuvor beschriebenen Schnellaktionssymbole zur Verfügung - Archivieren, Weiterleiten, Nehmen.
  4. Mit der Aktion "Nehmen" kann einer der Betreiber der Warteschlange den Gruppenchat mit einem Site-Besucher "aufnehmen" und die persönliche Kommunikation fortsetzen. Wenn das Widget Anrufe tätigen kann, verfügt der Besucher über eine Schaltfläche "Anrufen", auf die er klicken kann, um mit der Kommunikation per Sprache oder Video fortzufahren.

Auch im Chat wurden intuitive Diskussionssymbole hinzugefügt. Mit ihnen können Sie schnell zwischen Chats mit Website-Besuchern und Kollegen (PBX-Benutzern) unterscheiden. Eine weitere praktische Funktion ist das Beantworten von E-Mails. Der Betreiber kann auf die E-Mail des Besuchers klicken und ihm nach Abschluss des Chats antworten. Die Adresse des Besuchers kann online oder über ein Offline-Formular abgerufen werden.

In diesem Video wird eine Demonstration all dieser Funktionen vorgestellt.


Call Flow Service und Call Flow Designer-Entwicklungsumgebung


3CX v16 Update 1 Beta enthält den neuen 3CX Call Flow Apps Service. Es unterstützt die neuen 3CX-Sprachanwendungen, die in C # geschrieben wurden. Bestehende Anwendungen können im neuen Call Flow Designer konvertiert und finalisiert werden . Der Anwendungsserver funktioniert unter 3CX v16 für Debian / Raspbian Linux und Windows gleich gut. In naher Zukunft wird eine vollständige REST-API für die Anrufverwaltung und die zugehörige Dokumentation hinzugefügt.

Weitere Informationen zum Konvertieren vorhandener 3CX-Anwendungen finden Sie in diesem Video.


Vollständiges Änderungsprotokoll in 3CX v16 Update 1 Beta.

Update installieren


Die Installation der Updates erfolgt über die 3CX-Verwaltungsoberfläche im Abschnitt Updates. Bitte beachten Sie, dass nach der Installation des Updates die Datenbank der vorhandenen Chats konvertiert wird. Der Chat ist derzeit bei 3CX-Anwendungen nicht verfügbar.

Sie können auch die vollständige Beta-Distribution für 3CX v16 Update 1 für Windows oder Linux herunterladen:

  • Windows: 3CX v16 Update 1 Beta
  • Linux: 3CX v16 für Debian 9 ISO - Wählen Sie im Bildschirm 3CX Update Channel die Option Beta.
  • Manuelle Installation unter Debian & Raspbian - Befolgen Sie die Installationsanleitungen für Debian und Raspberry Pi .
