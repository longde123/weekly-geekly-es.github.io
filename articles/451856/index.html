Instale openmeetings 5.0.0-M1. Conferencias WEB sin Flash

Buenas tardes, Estimados Khabravchianos e invitados del portal!

No hace mucho tiempo tuve la necesidad de crear un pequeño servidor para la videoconferencia. No se consideraron muchas opciones: BBB y Openmeetings, como solo ellos respondieron funcionalmente:

  1. Gratis
  2. Demostración del escritorio, documentos, etc.
  3. Trabajo interactivo con usuarios (tablero general, chat, etc.)
  4. No se requiere instalación de software adicional para los clientes

Empecé con BBB ... bueno, en realidad no creció juntos ... El primero es el requerimiento de hierro real, porque en virtual no garantiza el rendimiento; El segundo es la intensidad de los recursos. Sí, una buena imagen y un excelente sonido, pero para mis tareas no es comparable con los recursos consumidos.

Empecé a probar las reuniones abiertas. Como aficionado de otros usuarios probados y versiones estables, instalé la última versión estable 4.0.8 (no consideraremos este proceso aquí). Todo está bien, excepto que él está en FLASH. Bueno, si es así, se negó a trabajar en cromo, fue a Fox ... pero esto contradice el párrafo 4, porque no todos usan FF y no a todos les gusta. ¡Ya me molesté cuando vi que la versión 5.0.0-M1 se declaró sin FLASH! Aquí es donde todo comenzó. Diré de inmediato que no comenzó todo de una vez, aproximadamente 2 semanas, 1-2 horas al día, me llevó un lanzamiento completo.

Y así, puse ubuntu 18.0.4-LTS. Requisitos:

  • JRE 8
  • Servidor de medios Kurento

Comencemos con JRE8. Por defecto, 11 se coloca desde los repositorios, por lo que agregaremos a los repositorios y luego comenzaremos la instalación de la versión que necesitamos: 

sudo add-apt-repository ppa:webupd8team/java sudo apt-get update sudo apt-get install oracle-java8-installer

Después de la instalación, debe especificar la versión predeterminada de Java para ejecutar: 

 sudo apt-get install oracle-java8-set-default

verificar versión 

 java -version

debe emitir 

 java version "1.8.0_201" Java(TM) SE Runtime Environment (build 1.8.0_201-b09) Java HotSpot(TM) 64-Bit Server VM (build 25.201-b09, mixed mode)

ahora queda por establecer los directorios de inicio. 

 cat >> /etc/environment <<EOL JAVA_HOME=/usr/lib/jvm/java-8-oracle JRE_HOME=/usr/lib/jvm/java-8-oracle/jre EOL

Para el funcionamiento normal de las transmisiones de video / audio, necesita un servidor Kurento Media (KMS). Hay diferentes opciones de instalación, utilicé la opción Docker. La instalación y configuración de Docker no está incluida en este artículo, ya que Internet está lleno de información. Y así, inicie KMS 

 docker run -d --name kms -p 8888:8888 kurento/kurento-media-server:latest

Ahora procedemos a instalar los componentes relacionados:
MySQL - OM tiene una base de datos integrada, pero no se recomienda usarla en la versión de combate. Ponemos cualquier versión conveniente para usted. Adecuado de repositorios estándar. 

 sudo apt-get install mysql

Para conectar Java a MySQL, debe descargar el conector y colocarlo en la carpeta / webapps / openmeetings / WEB-INF / lib /. La configuración de la conexión MySQL se encuentra en el archivo /webapps/openmeetings/WEB-INF/classes/META-INF/mysql_persistence.xml
ImageMagick - Necesario para un tablero general, demostración de documentos e imágenes. También tomar de nabos estándar. 

 sudo apt-get install imagemagick

GhostScript : si queremos trabajar con pdf, en ninguna parte sin él. Los repositorios también son estándar.
OpenOffice o Libre Office : para mostrar todos los formatos de documentos de Office ...
FFMpeg y SoX : para la capacidad de grabar videoconferencias en diferentes formatos. La versión debe ser 10.3 o posterior. 

 sudo apt install ffmpeg sudo apt-get install sox

Bueno, ahora estamos listos para descargar openmeetings en sí.
https://openmeetings.apache.org/downloads.html
Descargar, descomprimir en la carpeta que necesitamos.

Todo parece estar listo para ejecutarse (especialmente si sigue las instrucciones oficiales ), pero hay un enlace aquí https: // localhost: 5443 / openmeetings / install . Si prestamos atención a https y al puerto 5443, entendemos que nada saldrá de nosotros. Por supuesto, puede ejecutar el script ./bin/startup.sh y se iniciará el servidor. Incluso puede iniciar sesión y configurarlo usando el enlace http: // localhost: 5080 / openmeetings / install , pero no funcionará bien. Ahora todos los navegadores, y especialmente Chrome, están luchando por la seguridad del usuario y solo permiten que https funcione con la cámara y el micrófono. A través de FF será posible iniciar sesión y permitir que la cámara funcione, pero esto nuevamente nos vincula a un solo navegador. Por lo tanto, procedemos a instalar y configurar SSL. Puede hacer un certificado por dinero, o puede hacerlo usted mismo, trabajar con esto no empeorará a OM.

OM versión 5.0.0-M1 se basa en TomCat, no en Apache. La configuración del servidor web se encuentra en la carpeta ./conf/. Ya describí cómo crear un certificado autofirmado e instalarlo en TomCate.

Bueno, https está configurado, ahora vaya a la carpeta ./bin y ejecute statup.sh y luego de iniciar el servidor, vaya al instalador web https: // localhost: 5443 / openmeetings / install . Aquí todo es simple e intuitivo, EXCEPTO la sección "Convertidores". Aquí debe registrar las rutas a nuestros paquetes instalados adicionalmente.

  1. ImageMagick Path / usr / bin
  2. Ruta FFMPEG / usr / bin
  3. Ruta SoX / usr / bin
  4. Ruta OpenOffice / LibreOffice para jodconverter / usr / lib / libreoffice (configuré la libra)

Además, la configuración nuevamente no es complicada.

Después del primer inicio de sesión, SIEMPRE vaya a "Administración" -> "Configuración", busque el elemento path.ffmpeg y elimine el valor "/ usr / bin" escrito en él. Guarda la configuración.

Bueno, en realidad nuestro servidor de videoconferencia está configurado y listo para funcionar.

Después de reiniciar el servidor, debe ejecutar

  1. Base de datos DBMS (si no usa el Derby incorporado)
  2. KMS
  3. script statup.sh

Puede hacerlo manualmente pero puede crear scripts de ejecución automática.

Para generar "fuera" en el firewall, debe permitir los puertos 5443.5080.8888

¡Disfruta tu uso!

PD: si la cámara no transmite la imagen y no ve a nadie excepto a usted, debe agregar el dominio y el puerto a las excepciones en el firewall. Si Casper está instalado, funciona normalmente y se salta todo (¡sorprendentemente!), Pero Avast y el Windows incorporado trabajan duro. tendrá que sangrar con la configuración.
