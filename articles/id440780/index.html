Google tidak akan memblokir pemblokir iklan pihak ketiga di browser Chromium



Namun Google ingat prinsip Jangan jahat dan mengabaikan perubahan yang direncanakan pada API browser Chromium , yang menyebabkan sebagian besar pemblokir iklan dan beberapa ekstensi lainnya menjadi tidak berfungsi .

Keputusan itu dibuat setelah publikasi penelitian , bagaimana perbedaan pemblokir iklan memperlambat pekerjaan Chromium (lihat di atas). Ternyata keterlambatan ini sangat sedikit sehingga hampir tidak dapat dianggap sebagai alasan untuk memperkenalkan API baru. Beberapa jam setelah kemunculan ini, salah satu pengembang Chromium secara resmi mengumumkan keputusan untuk menunda API baru.

Ingatlah bahwa konflik muncul karena API deklaratifNetRequest baru (bagian dari dokumen Manifest V3 ), yang membuatnya mustahil untuk sepenuhnya menggunakan API webRequest . Ekstensi menggunakan data API untuk memblokir konten dengan cepat selama pemuatan halaman. Dalam sistem baru, pemblokir tidak dapat memblokir acara, tetapi hanya melihatnya. Alih-alih, ekstensi ditawarkan untuk beralih ke API DeclarativeNetRequest - dan memberi tahu browser tentang peristiwa yang ingin mereka blokir. Ini seharusnya mempercepat pemuatan halaman di browser (karena ekstensi tidak akan lagi memperlambat arus utama), dan juga melindungi privasi pengguna, menurut Google.

Yang pertama membunyikan alarm adalah Raymond Hill, penulis uBlock Origin dan uMatrix. Dia menyatakan bahwa ekstensi pemblokiran iklannya “tidak akan ada” jika perubahan diterima. Pesimisme kemudian diungkapkan oleh pengembang ekstensi lain, termasuk F-Secure, NoScript, dan Ermes Cyber ​​Security. Misalnya, NoScript untuk Firefox tidak dapat diangkut ke Chrome.

Raymond Hill mencatat bahwa declarativeNetRequest API tidak lebih dari sebuah implementasi dari satu mesin penyaringan tertentu, dan implementasi yang agak terbatas (batas 30.000 pembatasan tidak cukup untuk mengerjakan daftar EasyList yang terkenal). " Raymond Hill juga mencatat bahwa API baru tidak mendukung beberapa fitur lain, termasuk memblokir elemen multimedia yang lebih besar dari ukuran yang ditentukan, menonaktifkan eksekusi JavaScript dengan memperkenalkan arahan Kebijakan-Keamanan-Konten, dan menghapus header cookie yang keluar.

Google mengklaim bahwa perubahan diperlukan untuk 1) keamanan; 2) operasi cepat pemblokir iklan yang dibangun ke dalam Chromium, karena ekstensi saat ini dengan mekanisme operasinya saat ini memperlambat browser, menambah waktu rendering halaman. Namun, Studi Kinerja Adblockers yang diterbitkan pada 15 Februari 2019 sepenuhnya membantah tesis ini.

Penelitian ini dilakukan oleh pengembang pemblokir iklan Ghostery dari startup Jerman Cliqz. Mungkin, mereka terutama tersinggung mendengar tuduhan dari Google, karena pemblokir mereka memiliki efek paling kecil pada kecepatan memuat halaman, memimpin di semua kategori dengan margin yang cukup bagus. Nah, ini bisa dipahami, karena Ghostery sebenarnya bukan pemblokir penuh, seperti uBlock Origin. Selain itu, pilihan ekstensi untuk pengujian juga dapat dikritik, tetapi bukan itu intinya. Hal utama adalah bahwa pemblokir iklan praktis tidak memperlambat pemuatan halaman, seperti yang dipikirkan banyak orang (termasuk Google).

Perbandingan menunjukkan bahwa "pemblokir konten paling populer sudah sangat efektif (mereka memiliki waktu keputusan rata-rata per permintaan kurang dari 1 milidetik) dan tidak boleh mengarah pada biaya overhead yang terlihat oleh pengguna." Selain itu, penelitian sebelumnya oleh Pajak Pelacak menunjukkan bahwa pemblokiran iklan sebenarnya mempercepat pemuatan halaman , dalam beberapa kasus separuhnya .

Jika kita berbicara tentang masalah, mereka lebih cenderung terhubung dengan pekerjaan ekstensi lain, daripada pemblokir.


Diagram dari studi DebugBear Desember 2018 tentang bagaimana berbagai ekstensi memuat CPU selama rendering halaman.

Membatalkan rencana Google hanyalah perbaikan sementara, kata Insinyur Chrome Devlin Cronin. Setelah penyempurnaan, Manifest V3 akan dikembalikan untuk diskusi, dengan mempertimbangkan persyaratan semua pengembang.
