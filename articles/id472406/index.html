Perluas pusat data selama pengiriman pizza



- Ya, pusat data ini dapat digunakan dan dikonfigurasi saat Anda memesan pizza!

Saya melemparkan kata-kata ini pada pertemuan itu, dan perhitungannya segera datang. Seorang kolega menawarkan untuk memesan pizza. Yang kalah membayar: jika saya berhasil mengumpulkan pusat data - maka dia, jika saya tidak punya waktu - maka saya.

Neraka

Secara keseluruhan, penuhi stress test: solusi hiper-konvergensi Cisco dengan pengaturan anti-pizza sederhana.

Apa ini


Solusi hyperconverged dari Cisco datang ke pengujian kami. Berikut ini adalah posting terakhir dengan rincian pengujian itu sendiri dan perangkat keras.

Apa yang kita miliki:

  • Empat node penyimpanan dari Cisco UCS HXAF240 M4.
  • Dua Cisco UCS Fabric Interconnect 6248UP.
  • VCenter yang ada untuk menambahkan cluster di sana.
  • Beralih dengan port yang beroperasi pada kecepatan 10 Gb / s (dalam produksi, silakan gunakan dua sakelar untuk toleransi kesalahan).

Mari kita mulai.

12:28

Saya memasang dan menghubungkan kelenjar di muka. Kami menghubungkan server ke FI, kami menghubungkan pasangan FI ke switch 10G.

Sekitar waktu ini, kolega saya Marina menginstal aplikasi dan mengetahui urutannya.





12:35

Hitung mundur pesanan telah habis. Penjual SLA memiliki satu jam untuk pengiriman, dan itu sudah cukup bagi saya. Pergi untuk menginisialisasi gugus FI.

Cluster dapat diinisialisasi dari konsol web jika Anda memiliki DHCP di jaringan, atau menyambungkan ke port FI COM dan memasukkan semua data. Saya lebih suka cara lama yang dapat diandalkan melalui konsol (kecepatan standar - 9,600).

Untuk menginisialisasi FI pertama, kami memasukkan manajemen IP dan IP kluster, kata sandi untuk akun admin dan parameter DNS, jika diinginkan.

12:41. Konfigurasikan FI Cluster

Sebagian besar pengaturan akan dibuat oleh penginstal HyperFlex, tetapi kami perlu menentukan peran port untuk FI (AI belum diterapkan di penginstal, dan belum dapat menebak apa dan di mana kami menghubungkannya). Dalam kasus kami, kami perlu menentukan port tempat server terhubung (Server Port), dan menentukan uplink ke sakelar hulu (Port Uplink).



Setelah menaikkan port server, UCSM (Cisco UCS Manager - mengendalikan FI dan server, menyalakan FI) akan menemukan server kami di belakangnya dan mulai menginisialisasi mereka. Ini akan memakan waktu.

Saat ini:





12:45 Casting Pemasang HyperFlex

Pemasang HyperFlex hadir sebagai templat. Tidak ada yang menarik di sini: kami menuangkan template OVA, bersama dengan memasukkan data tentang IP dan kata sandi untuk akses ke installer. Saya malas, jadi saya menyerahkan semuanya ke DHCP dan memasukkan kata sandi beberapa kali. VM hanya diperlukan saat membuat cluster dan operasi seperti menambahkan node ke sebuah cluster, membuat cluster Metro, atau memperbarui. Oleh karena itu, dapat dimatikan dan tidak membuang sumber daya.











13:05. Konfigurasikan HyperFlex Cluster

UCSM menemukan server kami, menentukan konfigurasi dan siap untuk melanjutkan.
Kami melihat alamat penginstal dan terhubung ke antarmuka WEB-nya, pilih item untuk membuat cluster standar.



Masukkan IP dan kredensial dari UCSM dan vCenter.

Pilih server yang akan ditambahkan ke cluster.

Selanjutnya, kita perlu menentukan VLAN dan menangani untuk jaringan manajemen, vMotion, jaringan untuk menyinkronkan data antara node dan jaringan untuk hosting mesin virtual.

Kami memilih server yang ingin kami tambahkan ke cluster, tentukan pengalamatan untuk hypervisors dan kata sandi root baru.



Klik Berikutnya beberapa kali dan minum teh.

Ini adalah tahap terpanjang, karena installer membuat profil untuk server di UCSM, terhubung ke hypervisor dan membuat semua pengaturan, menumpahkan layanan VM pada setiap host, menambahkan server ke vCenter.

Dalam hal ini, penginstal melaporkan langkah saat ini dan kesalahan yang telah terjadi. Jika kata sandi atau VLAN yang dimasukkan salah, akan ditawarkan untuk memperbaiki data dan mulai dari tahap sukses terakhir.

Pizza telah tiba!

Pizza sialan disampaikan jauh lebih cepat daripada yang saya inginkan. Ini adalah pertama kalinya aku menyesal membawanya lebih awal.



1:55 malam Membuat Datastores

Pemasang telah menyelesaikan tugasnya, dan cluster sebenarnya sudah siap. Untuk pekerjaan penuh, kami tidak memiliki cukup datastore. Kami pergi ke webcam pengontrol manajemen HyperFlex dan memilih bagian Datastores.

Secara default, server dibuat dengan sakelar standar (vSwitch) dan satu VLAN untuk jaringan VM. Tetapi Anda dapat membuat saklar terdistribusi (semua jaringan layanan tetap pada standar) dan juga memotong VLAN baru, tidak akan ada masalah. Selain itu, UCS memiliki kemampuan untuk berintegrasi dengan Cisco ACI, yang juga tidak akan berlebihan bagi pemilik Cisco SDNs.



Itu saja. Total waktu penyiapan pusat data adalah satu setengah jam. Pengiriman pizza - sedikit kurang dari 40 menit dengan SLA satu jam. Yah, setidaknya itu sebanding. Intinya: 1 jam 30 menit, saya berhasil menyebarkan cluster dan makan.
