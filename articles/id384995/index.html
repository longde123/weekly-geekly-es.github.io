Google berbicara tentang inisiatif baru untuk mempercepat halaman web di perangkat seluler: Laman Seluler yang Dipercepat

Hari ini, blog Google resmi telah mengumumkan pengumuman inisiatif terbuka baru oleh raksasa pencarian Accelerated Mobile Pages (AMP), yang dirancang untuk mempercepat halaman web di perangkat seluler. Kita berbicara tentang penggunaan oleh programmer web dari perpustakaan HTML AMP khusus, kode yang dalam status pratinjau teknis tersedia di github . Simbol tidak resmi perpustakaan adalah baut kilat "âš¡", dengan demikian menyiratkan kecepatan kerjanya pada perangkat seluler.

AMP HTML percaya bahwa kinerja halaman web sangat bergantung pada jumlah kode JavaScript untuk berbagai pustaka yang menerapkan berbagai fungsi dinamis. Alih-alih menggunakan Javascript secara eksplisit atau implisit untuk elemen seperti berbagai suara atau video, pengembang didorong untuk menggunakan Komponen Web yang dirancang khusus untuk AMP HTML. Daftar komponen tersebut dapat ditemukan di sini . Kode JS komponen ini tidak boleh memengaruhi kinerja halaman web.

Sekarang perpustakaan AMP HTML hanya berisi 4 komponen dan validator khusus, yang dengannya Anda dapat memverifikasi apakah halaman dibuat sesuai dengan standar AMP. Jika Anda menambahkan "# development = 1" ke URL, perpustakaan akan menjalankan serangkaian metode tegas untuk mengetahui apakah tata letak halaman cocok untuk prinsip-prinsip AMP dan menampilkan hasilnya di konsol browser.
