提升您的Speedtest节点

如果需要测量Internet连接的速度，则很可能会打开speedtest.net。 Speedtest允许您创建自己的测量节点，其他用户可以使用该测量节点，例如，这对于Internet提供商非常重要（主Speedtest服务器应放置在设备之后，这会降低订户的速度）。

根据katom详细的安装手册。

开发人员提供的系统要求^



Speedtest支持许多操作系统：

  • Windows Server（带有IIS 6、7、7.5或8的2008或2012）
  • Linux（2.6.18或更高版本的内核）
  • Mac OS X（在10.8上构建，以前的版本可以运行，但不完全受支持）
  • FreeBSD（内核7.3或更高版本）

我使用Ubuntu 18.04。

因此，让我们按顺序进行。 首先，您需要下载HTTP旧版后备应用程序和主服务器 ，以使Speedtest正常运行。

HTTP Legacy Fallback需要Web服务器才能工作，我使用apache + php。 将http_legacy_fallback.zip的内容解压缩到/ var / www / html中 。

在OoklaServer.tgz存档中，我需要OoklaServer-linux64目录 。 将其解压缩到具有第三方程序的位置，例如/ opt / speedtest 。

接下来，您需要启动OoklaServer并确保它在系统启动时启动。
检查它是否有效。 您需要在浏览器中打开http：// <您的ip>：8080

如果一切顺利，将出现以下消息：


现在，查看旧版后备HTTP。 打开http：// <您的ip> /speedtest/upload.php

应该有类似size=500东西

之后，必须在Speedtest中注册该节点。 为此，您必须在https://account.ookla.com/上注册，然后转到“ Speedtest服务器”部分。 之后，您可以单击“添加服务器”，将打开一个带有调查表的窗口：



必须填写调查表，然后等待服务器出现在常规列表中。

大约一天后，在我们网络的订户中，已经开始在speedtest.net的入口处自动选择他。
