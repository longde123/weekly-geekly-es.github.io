[叶卡捷琳堡，公告] Java Mitap-JUG.EKB



NAUMEN每六个月为Java开发人员-JUG.EKB举行一次会议。 下次会议将于9月13日举行。

我们选择此日期是因为今年9月13日是256天！ 如果不是在专业假期，还有什么其他聚会的机会呢？ 什么时候（如果不是今天）听报告，向演讲者提问，与同事讨论困难，寻求建议或帮助某人解决问题，然后在聚会后聚在一起？

什么是JUG.EKB
JUG.EKB会议不仅仅是与参与者相关的新信息。 这是与那些对Java和新技术无动于衷的人们的交流，这是与演讲者和有用联系人的悠闲对话！

每个人都可以在这里找到好处。 经验丰富的开发人员将能够向演讲者提出复杂的问题，参与讨论，解决自己项目的问题，进行咨询或与之分享经验，或者与之分享经验。 甚至准备您的报告并获得新的体验！ JUG.EKB会议的新手开发人员将能够找到志趣相投的人，学习很多新事物，了解要成为一名开发人员要学习的其他知识。

计划中有哪些报告

• 深层联系，烈性酒
Andrey Chernoprudov（叶卡捷琳堡瑙门）

研究和开发报告似乎很简单：打开链接。 我们将讨论整个开发周期，从复杂后端的深度开始，到在每个客户端上的实现结束。 他们到处都有耙，陷阱和拐杖……总的来说，就像您所爱。

• 轮渡不更改API-如何构建稳定的API
阿列克谢·斯托卡洛夫（Alexey Stukalov）和尤里·阿塔莫诺夫（Yuri Artamonov）（萨马拉哈蒙）

您是否知道在一个长期稳定的项目上更新此框架或该框架时会使代码爆炸并将其变成乱七八糟的感觉吗？ 作为CUBA平台的开发人员，我们对此非常痛苦。 因此，在平台存在期间，它更改了3个ORM，Vaadin的3个主要版本，更不用说Spring框架了。 同时，平台要解决的任务之一是维护稳定的API，以使我们的用户不会遇到上述问题，并迁移到平台中使用的所有库的最新版本。 我们将分享我们在构建API方面的经验，以及使我们能够解决的问题，这些技术使我们能够解决将基于CUBA的解决方案移植到最新堆栈的问题。
