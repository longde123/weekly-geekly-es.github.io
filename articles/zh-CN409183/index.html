Nvidia停止支持所有32位驱动程序

Nvidia在Nvidia支持的官方网站上宣布 ，无论视频卡的型号如何，对32位操作系统的图形驱动程序的支持已完成。 也就是说，将不再发布新版本的32位驱动程序。 最新版本将是适用于所有受支持系统的390.x GeForce Game Ready驱动程序的下一版本。

到目前为止，Nvidia已经发布了适用于Windows，Linux和FreeBSD的32位和64位版本的图形驱动程序。 现在仅保留64位版本。

该公司在一份正式声明中指出，将不会安装新的驱动程序并且无法在32位操作系统上运行，并且不计划将新功能和改进移植回GeForce Game Ready驱动程序390或更早版本。

但是，英伟达承诺将在2019年1月之前发布针对32位驱动程序的关键安全更新。

拒绝支持32位系统的可能性极低。 现在，这种系统已经很少见：例如，根据Steam的统计，目前只有不到2％的计算机在32位操作系统下工作。 显然，该公司不想为开发商提供宝贵的服务，以服务于少数群体。 尽管相同的FreeBSD操作系统的市场份额甚至更少，但是Nvidia幸运地没有将其从支持中删除。

一种或另一种方式，但32位操作系统上的Nvidia视频卡的少数用户必须坐在旧驱动程序上，这些驱动程序将继续在正常模式下工作。 Nvidia可能会在将来的驱动程序中引入的新功能将不可用。

通常，这是从32位操作系统切换到64位版本的另一个原因，尤其是因为64位应用程序在各个方面都比32位应用程序要好：它们更安全，更快，并且对32位内存固有的数据量没有限制。位寻址。 保留在32位OS上的唯一原因可能是某些特定的旧软件（16位和32位）或仅存在32位驱动程序的设备的限制。 例如，对于较旧的Mustek扫描仪，只有32位驱动程序，因此这些扫描仪在64位OS下变成砖头。

此外，该公司宣布终止对以下图形卡的支持 ：

  • NVS 315
  • NVS 310

在GeForce Game Ready驱动程序390版之后，这些产品通常会停产。 但是，对于他们来说，安全更新的发布时间甚至比32位驱动程序的发布时间还要长。 计划在2019年12月之前发布漏洞状态从低到严重的更新，而关键更新将在2021年12月之前发布。

NVS图形卡被定位为具有多个显示器的系统的标准商业图形。 该NVS 810系列的最新型号具有4 GB DDR3内存，最多支持八个显示器（八个迷你DisplayPort插槽）。 除了不再支持NVS 315和NVS 310卡之外，该公司还发布并继续支持NVS 510（最多四个显示器）和NVS 810型号，因此，只有两个显示器的NVS 315和NVS 310视频卡几乎都被正式淘汰了-用户推动购买新型号。

最后，Nvidia的另一个消息是， 不再支持Quadro四缓冲立体声技术 。

特别是，从Windows驱动程序的第396版开始，以下功能将不再支持：

  • 具有用户界面和NVAPI访问权限的3D DLP和Tridelity立体成像模式。
  • 适用于Windows 10的基于NVAPI的DirectAP立体声

排除以下编程接口：

  • NvAPI_D3D1x_CreateSwapChain（）
  • NvAPI_D3D9_CreateSwapChain（）

Nvidia写道，使用不支持的功能重新编译应用程序后，它们将开始生成错误“ NVAPI_NOT_SUPPORTED”。 作为具有四重缓冲的立体声技术的替代方法，建议切换为使用Microsoft（DXGI）立体声的本机软件接口。

回到从32位到64位应用程序的迁移，值得回顾的是，英特尔将在2020年放弃对BIOS的支持 ，而在固件中仅保留纯UEFI。 因此，无法在现代硬件上启动16位操作系统以及几乎所有32位操作系统。
