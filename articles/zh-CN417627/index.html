超级CRM或Mini ERP？ 业务搞砸了

我们的IT人员与众不同，他们不知道思维定型观念的发生频率，尤其是不是IT专业人员，而是IT爱好者。 马上想到：如果有代码，那就是PHP； Java仅用于大型项目； Python仅用于科学。 1C适用于nedoprogrammers。 显然，这些都是错误的判断，但它们并非从头开始出现-如果没有大规模的先例，它们是无法做到的。 一切都像古典神话一样。

这个神话也影响了公司软件。 商业环境中已经形成了许多明确的定型观念：“ CRM是一种销售计划”，“ ERP是昂贵的空间”，“ CRM适合每个人”，“ ERP适用于星际企业。” 同时，情况发生了重大变化，因此它们本质上无关紧要，但仍然继续处于头脑中。 结果，公司拒绝了必要的自动化程度，混淆概念，使用效率低下的工具等。

现在该清理了。 让我们从ERP vs CRM开始。



问题来自哪里？


让我们回想一下我们共同的过去-我们穿着相当笨重的手机，而且这种趋势似乎正朝着使重量和尺寸最小化的方向发展。 我们有（并且还活着）电视机，播放器，时钟，警报器，PDA规划器和电子笔记本，录音机，照片拍摄机以及其他可穿戴但不是非常电子的设备。 实际上，现在，电子助理的大部分功能都由一部智能手机执行-一部大型，功能强大且智能的智能手机。 这是技术的发展。

另一方面，诺基亚，摩托罗拉，西门子，索尼+爱立信，阿尔卡特曾经吵架。 好吧，甚至飞利浦也被穿甲打了。 不久之后，胆小的婴儿床和颜色模型开始打入三星市场。 现在怎么办 弯曲手指：苹果，三星，小米，索尼，魅族，华为，诺基亚，飞利浦，华硕，联想，一加和Doogee和LeEco等20个中国品牌。 这就是响应市场需求和高需求的业务的发展。

在俄罗斯（和世界）商业领域中，情况完全相同。

  1. 业务发生了变化。 如果能够为大中型公司提供较早的生产，则今天要归功于中国发达的经济以及来自中东的活跃物资，机器，材料，熔炉，办公设备等。 便宜得多。 这意味着一个只有10人的小公司可以拥有一个仓库，生产，预算，采购管理，物流以及自己的车队。 同时，对于这种业务的数百万个ERP的投资仍然是不可接受的，并且需要良好的会计处理。
  2. 同时，CRM系统也在开发中（例如，我们公司已经在市场上销售了17年，我们的RegionSoft CRM已经使用 12年了，并且已经经历了6个主要版本）。 他们进行了介绍，并最终确定下来，其中一些改进都进入了积压的过程，从积压到生产。 有一个增加软件功能和通用性的过程。 实际上，未被某些开发人员放弃的CRM系统已经发展到了ERP级别，除了某些模块（取决于客户群及其要求）之外。
  3. 业务的信息丰富性不断增长-即使是一家小型公司，在日常活动过程中也会积累大量数据，存储大量客户基础，建立知识库，日志，记录等。

    一个原始的例子：在CRM中启动客户，现在经理甚至可以在联系人之间进行收集-电子邮件，办公电话，手机，Skype，电报，社交网络上的帐户等。

    一个严重的例子：与客户的每次联系都伴随着联系，财务，运营信息的收集，大量的文件，调查，信件，邮件，记录的电话，请求等很快就使联系变得不堪重负。 所有这些都需要组织起来，因为无组织的信息不仅无用，而且会干扰工作和决策（由于分析和数据解释的失真）。

    公司内的所有部门密切相关，因为它们面临着吸引，满足，留住客户并获得收入的任务。 因此，后勤人员应该知道库存情况，仓库管理员-生产者将要询问多少和什么，卖方-将要生产多少，市场营销者-如何对市场施加压力以出售未出售的东西。 在统一的信息环境之外，无法建立这样的关系-您的电话坏了，期限到了最后，客户不满意，（在最好的情况下）在社交网络上弄丢了声誉，（在最坏的情况下）在法庭上乱涂乱画。
  4. 公司已经积累了执行单独任务的动物园软件，并且大多数时候彼此没有集成。 分散的软件会导致业务流程内部出现问题：自行判断-当库存分开而客户分开时，这可能会导致故障并因此而导致声誉损失。 这很耗时，不安全，在监视和管理方面非常复杂，最后又很昂贵-您必须为所有产品付费。

    例如，在一家有40名员工的公司中，除了工作软件（例如IDE和Photoshop）外，每个（！）员工还拥有：公司门户（聊天室+社交网络），自写公司门户（新闻，订单，解雇，时间跟踪） ，日历，休息日），具有甘特图的云服务（已处理了一百年的项目），Skype用于业务，内部Messenger，针对具有发票的客户的CRM（西），针对俄罗斯客户的CRM（自写），1C，2封邮件在不同的域上，有很多客户可以品尝，还有一些保管箱办公室- 索赔哪里sharyatsya文件。 蛋糕上的樱桃-具有嵌套层次结构的公司球文件夹。 所有这些都有登录名，密码，所有内容都有其自己的界面，窗口，表格……工作舒适吗？ 以及要付出多少！ 顺便说一下，几个月前，最后一位系统管理员因失去工作而被解雇，而工作落在了程序员身上。 参考IT基础架构混乱。

结果，创建统一的多功能应用程序的技术和经济需求已经成熟，该应用程序不仅可以为用户提供界面的一个窗口，而且可以有效地存储和处理信息。

为此，提供端到端自动化的CRM系统和ERP系统最适合其余部分。

这些系统之间的差异可能发生在功能，复杂性，成本等方面。 但实际上，关键在于：CRM系统的重点是客户，ERP系统的重点是业务内部，业务流程，我们将以表格形式详细检查差异：

不，不一定。 根据您公司的需求，您可以选择一种可以作为工具使用的解决方案，并且不会因自动化方面的700万投资而炫耀也不会超出预算。 我们随意将其称为“超级CRM”的混合功能，它既可以提高销售额，又可以优化成本，但同时又保留了ERP的属性，ERP对于大型公司来说是必不可少的（例如，人力资源职能和财务状况和会计-仅仅是因为俄罗斯公司的会计已经实现了很长时间的自动化：1C普及到每个人，而没有得到的地方，则有针对私人企业家的解决方案等）。

当然，即使是一家营业额不错的小公司，也可以找到钱来购买“成熟的” ERP，但结果却像跑车在驱动蘑菇：您不会享受速度，您会杀死悬架，并且会在车身上刮擦油漆。 您可以从窗户大喊：“我看见法拉利的熊了吗？” （读为“老兄，我在这里买了SAP，消防系统，值得，我们佩服”）。

对于CRM而言，对于ERP而言，我们可以通过一种简单且有点奢侈的方法为您提供建议：询问部门的每个负责人（或关键的“参与”员工）一些指标（今天的销售额，今天的订单量，一种产品的剩余数量）库存，制成品的数量等），并注明准备答案需要多长时间。 看一下搜索路径：准备数据是否需要时间，或者将其埋入Excel中还是会爬升以从文档中读取？ 如果没有快速清楚地命名数字，则需要使用CRM或ERP进行自动化（根据业务需求和概况）。

好吧，这是主要指标。 有更深层次的理由开始寻找您的CRM / ERP梦想供应商。

  • 您的公司具有自觉（或什至是正规化和规范化）的业务流程，因此您需要确保其不间断的运行并减少人为因素的影响。
  • 业务流程变得乏味：难以跟踪订单和与客户的关系，销售和请求的结构难以理解，必须手动重新计算库存。
  • 信息存储在单独的系统和数据库中-在工作过程中，经常需要导出或导入数据，会发生错误。
  • 您与客户的关系需要快速交货，装运，完美无缺地履行订单，并保持关系历史到最小的细节，包括折扣，特价，退货等。
  • 您会感到员工的生产力已达到顶峰，并且由于任何原因很难扩充员工。
  • 文件（发票，法案，合同，商业要约，技术和商业建议等）经常丢失，需要复制和恢复。
  • 该公司以预算或业务计划为生，但是编制和调整它们需要花费大量时间，并且每次都需要重新收集数据。
  • 公司员工合并多个职位和/或同时执行多项任务（物流店主，店员-销售员等）。
  • 有很多信息，但是在您看来，这是一个负担，在做出每个特定决定之前，您必须再次重新挖掘数据，这里没有存储现成的报告，也没有动态报告（按时间段）。
  • 内部流程的成本在增长，但其有效性（销量，订单，收入）却没有。
  • 该公司有自己的生产。
  • 该公司有几个办事处，分支机构，合作伙伴，有必要汇总信息并从中心管理所有部门。
  • 员工的工作效率开始下降。
  • 你被竞争对手淘汰了。
  • 市场上不可能有价格竞争，由于忠诚度和服务质量，有必要进行指导。

所列标记适用于任何领域的业务：从零售和广告代理商到IT公司以及自己烹饪的商店。

什么以及为什么要自己选择企业？


在每种情况下，您都需要收集需求并将其与软件功能进行比较。 一般建议如下：

  • 没有实际生产，也没有仓库-CRM，甚至不用担心；
  • 有生产和仓库，但对财务和人力资源没有特殊要求-具有相应功能（仓库，生产，KPI，项目管理，主要...）的超级CRM；
  • 一切都是必要的，包括复杂的财务会计和特殊人员管理-ERP。

无论如何，购买和实施CRM / ERP都需要投资：

  • 分析-并且您了解有关客户和公司状况的更多信息，具有最完整，最准确的信息
  • 信息存储-所有数据将统一起来，并随时可用于分析和采用微观和战略决策
  • 减少公司内部的交易数量-自动系统将处理很多例行和强制性的任务（阶段的翻译，提醒，邮件，文件的形成以及打印表格的填写，报告的准备等）
  • 公司管理中心-在一个程序的界面中（如在控制台上），您将获得一个工作工具，可用于管理操作和分析工作。
  • 中长期计划是公司计划在市场上站稳脚跟，而不是过度劳累和过度支付资源的最重要方面。

为什么我们如此专注于俄罗斯供应商，因为在线实施不再受任何限制？ 我们将简要回答：

  • 它们是按照俄罗斯和大多数苏联国家采用的商业规则建造的
  • 它们更便宜，甚至更昂贵，许可证本身和实施工作
  • 更实惠的培训和支持
  • 无需购买复杂的连接器即可使用主要文档
  • 它们与1C：Enterprise等会计系统更充分地集成在一起。

选择CRM / ERP系统的过程中需要寻找什么？


原则上，我们在这里和这里写过关于选择CRM的信息 ，它也适用于ERP。 但是，最重要的一点应该首先考虑。

  1. 是否平台，平台的质量如何？ ERP通常在平台的作用下提供，但值得将技术功能与营销外壳区分开。 编写自定义报告，API和脚本的功能是开发人员工具，仅此而已。 该平台几乎是一个完整的IDE，您的程序员可以在其中完成用于业务任务的CRM / ERP。 如果您做出这样的决定，请记住，平台本身很昂贵，而且要加上具有必要技能的程序员的薪水（提示-它要花很多钱）。
  2. 是否有云。 如果没有非常有价值的数据的小公司可以沉迷于云中，那么您需要提出对云ERP和Hyper CRM的切实安全要求-实际上，您将把整个公司的电子业务转移到云，收入卢布和业务流程的步骤中。 关于品味，但就个人而言，我们不会冒险。 实践表明，企业变得越严重，就越清楚地形成领导层，即公司系统应完全在其系统管理员的控制下，并且不应超出办公室，甚至不限于分布式办公室（当多个办公室合并为一个虚拟局域网VPN时）。
  3. 最低安全性功能：如何以及何时进行备份，将备份存储在何处，如何安排访问权限的分发系统以及提供哪些身份验证机制。
  4. 更新。 请注意网站上有关更新的信息。 如果没有，请询​​问供应商哪些更新发布，如何滚动（您是否需要中断系统中的工作），哪些更新是付费的，哪些不是。
  5. 技术支持 -找出提供的数量，免费软件包的数量以及付费软件包的功能，在这两种情况下提供技术支持的优先级是什么。
  6. 拥有成本和必要的基础设施成本 ：可能会有付费的附件，服务，DBMS，开发人员工具等。 这是正常现象，但最好事先了解这些事实。

有这样的统计数据：根据Aberdeen Group分析，实施现代ERP系统的中型公司可以支持变更，将营业利润增加21％，将管理成本减少22％，并将营业支出减少23％。 这些数字令人印象深刻，表明对ERP（以及CRM）的投资是合理的。

CRM被认为是迈向ERP的第一步。 当然，这是有关企业软件的另一个神话。 这些不是连续的元素，也不是同一枚硬币的侧面，甚至不是进化链。 最佳定义-CRM和ERP-是独立的程序，每个程序都可以解决特定时间点业务所面临的任务。

即使对于最小的企业，CRM系统也比存储在笔记本，平板电脑或员工头上的随机，分散的客户数据集要好。 客户关系是任何企业的命脉，CRM是必不可少的生命支持工具。 时间显示，是否需要ERP？

仅在这里是最简单的建议：任何工具仅在使用时才起作用。 不要让CRM或ERP变得懒惰。
